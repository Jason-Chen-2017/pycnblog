The core concept of the software system architecture golden rule is to treat software architecture as a first-class citizen in the software development process. This means that architecture should be treated with the same level of importance as code. The golden rule consists of three main components:

* **Documentation**: Documenting the software architecture is essential for understanding how the system works and how it can be maintained and evolved over time. Good documentation should include diagrams, descriptions, and constraints that clearly describe the system's structure, behavior, and dependencies.
* **Knowledge Management**: Managing the knowledge associated with the software architecture is equally important. This includes capturing and organizing the expertise and experience of the development team, as well as tracking changes to the architecture over time.
* **Integration**: Integrating documentation and knowledge management into the software development process ensures that they are treated as first-class citizens. This involves using tools and processes that enable developers to easily create and update documentation and knowledge artifacts as part of their daily work.

These three components are interconnected and reinforce each other. For example, good documentation makes it easier to manage knowledge, while managing knowledge helps ensure that documentation is up-to-date and accurate. Similarly, integrating documentation and knowledge management into the software development process helps ensure that they are treated as first-class citizens and are not overlooked or neglected.
