# 1.背景介绍


## 1.1为什么需要tcpdump过滤端口？

tcpdump是一个基于libpcap的命令行网络数据包抓取工具，可以用来分析网络传输协议，如TCP、UDP、ICMP等。我们可以使用它来获取本地计算机或远程主机之间的数据通信过程及其中的细节信息。但是，在实际应用中，我们通常只需要关注特定端口上的流量信息。而tcpdump自带的过滤条件仅支持过滤IP地址，但不支持过滤端口号。

因此，如果我们需要过滤端口号，我们就需要自己编写脚本或者直接调用tcpdump进行过滤。

## 1.2什么是tcpdump过滤端口？

tcpdump过滤端口，即通过指定tcpdump命令中的`-w`参数，将过滤后的流量保存到文件中。具体流程如下：

1. 使用tcpdump命令监听目标端口，例如：`$ sudo tcpdump -i eth0 -s0 -nnvtt 'port 80' > outfile`
2. 在上面的命令中，`-i eth0`，`-s0`，`-nnvtt`分别表示抓取网卡eth0的数据包，不对数据包进行切分，显示IP地址；`-w outfile`，输出过滤后的数据包到一个新的文件outfile。
3. 如果要过滤掉非80端口的流量，则可以使用以下命令：`$ sudo tcpdump -i eth0 -s0 -nnvtt '(not port 80)' > outfile`
4. `(not port 80)`表示匹配所有不是80端口的流量。

1. markdown语法简介

   Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的 HTML 文档。它非常适合用来撰写技术文章、读者手册、快速记录笔记等。下面是一些基本的语法规则。

   ```
   # 一级标题
   ## 二级标题
   ### 三级标题
   
   **粗体** or __粗体__
   
   *斜体* or _斜体_
   
   ~~删除线~~
   
   `单行代码` or `` `单行代码` ``
   
   [链接名称](链接地址) or <链接地址>
   
  ![图片描述](图片链接地址)
   
   |表格列名|表格列名|
   |-|-|
   |单元格|单元格|
   
   > 引用内容
   
   ---  分隔线
   
   四个空格或一个制表符    表示换行
   
   1. 有序列表项
   2. 有序列表项
   
   * 无序列表项
   * 无序列表项
   
   ```
   
   
2. git与github简介

   Git 是一款开源的分布式版本控制系统，用于管理个人或小组项目的开发工作。GitHub 是面向开源及私有软件项目的代码托管平台，为用户提供了一个空间，让更多的人一起参与到Open Source（开源）社区的建设中来。以下是关于git与github的一些基本介绍。
   
   #### 1.Git

   Git 可以理解为带有版本管理功能的一个软件。它可以方便地跟踪文件的修改历史，可以对不同版本之间的差异进行比较，还可以向后复原文件到某一特定的版本。

   #### 2.Github

   GitHub是一个面向开源及私有软件项目的托管平台，由GitHub公司(母公司Microsoft旗下)的CEO William Sanches创建。GitHub提供了各种 Git 相关服务，其中包括但不限于：

   1. 代码仓库：存储Git项目代码
   2. Issue Tracker：bug跟踪系统
   3. Wiki：团队共享的技术文档
   4. Pages：静态站点托管
   5. Projects：项目管理

    Github拥有庞大的开源社区，并被世界各地的开源项目所依赖，是世界顶级的程序员交流社区。

   #### 3.Git与Github的关系

   Git是版本管理工具，而GitHub是一个面向开源及私有软件项目的托管平台，因此GitHub作为代码托管网站与Git配合工作，实现版本控制、代码协作、项目管理等功能。
