# 1.背景介绍


## 什么是支付结算系统？
支付结算系统是由商户、支付机构、账户中心、支付结算平台三个部门组成的，主要职责是实现商户与用户之间的交易，包括资金支付、交易清算等。支付结算系统分为上游支付系统（比如微信支付、支付宝）、中游支付网关（比如蚂蚁金服的支付通道支付），以及下游集团支付中心（比如淘宝的支付宝集团）。支付结算系统涉及的交易类型有商品购买、货币转账、银行卡转账、会员充值、积分兑换等等。目前支付结算系统的主要运用场景有电子商务、网络购物、移动互联网支付、数字货币、保险等领域。

## 为什么要研究区块链技术？
区块链技术（Blockchain technology）近年来受到越来越多人的关注，其特点就是去中心化、可追溯性、不可篡改性等。通过采用区块链技术可以解决传统支付结算系统存在的问题，比如支付信息真伪不确定性、信用卡流量控制不当、付款方式多样化带来的成本高昂等。由于其开源、透明、安全、高效等特性，区块链技术在支付结算系统的应用也逐渐增加。

## 以太坊是什么？
以太坊（Ethereum）是一个开源、基于区块链技术的平台，提供了智能合约编程能力，旨在构建一个公共、平等的分布式计算环境。以太坊拥有独创的“分片”架构，通过分片将数据存储分布到不同的节点上，提升处理能力和性能。另外，它还支持动态编译、跨平台运行、智能接口等功能，是一种“自然而然”的区块链技术应用方案。因此，以太坊与区块链技术在支付结算系统的应用也呈现出蓬勃发展之势。

## 以太坊与区块链技术有何不同？
从根本上来说，区块链是一种分布式数据库，具有极高的弹性和容错性，可以存储大量的数据，并保证数据完整、准确无误。但是，随着链上的交易越来越多，数据的价值也就越来越高了。为了使得链上的资源得到更有效地利用，区块链技术在一定程度上进行了优化。例如，侧链技术通过扩展区块链网络，可以在保持主链完整性的同时，新增一条链，专门用于执行某些特定功能。

另一方面，以太坊与其他区块链平台最大的不同在于，它是一个能直接运行智能合约的平台，能够实现非托管的自动化执行，即用户只需提供数据、算法、逻辑，就可以生成任意类型的合约，而不需要依赖任何第三方审核。同时，以太坊还具有很多其它独特的特征，诸如激励机制、分片架构、动态编译等。这些差异导致以太坊的应用场景比传统的支付结算系统更加复杂，需要更多的知识储备才能掌握。

## 以太坊的适用场景
目前，以太坊的应用场景主要分为以下几类：
* 智能合约应用：以太坊提供了一种全新的编程模型——智能合约（Contract Programming Model）。智能合约可以让用户在区块链上部署自己的服务，实现各种商业场景下的自动化操作。例如，智能合约可以用来管理房地产或股票资产，代替实体公司管理自己的账户和产品。
* 去中心化交易所：以太坊的一个重要特性就是其支持跨链交互，允许用户在不同的区块链上进行交互，从而形成新的业务模式。例如，Decentralized Exchange (DEX) 可以基于以太坊和其他主流公链建立，让用户之间可以在不依赖中介的情况下进行快速、免手续费的交易。
* 分布式应用：目前，以太坊已经支持基于以太坊的分布式应用。例如，去中心化的借贷平台 MakerDao 将使用以太坊作为担保，帮助他人向借款人借款。DeFi（去中心化金融）应用也是以太坊的典型应用场景。
* NFT（非同质性代币）：近年来，NFT（Non-fungible Token）已经成为一种热门话题。以太坊目前也支持创建、交易、记录 NFT。例如，OPENSEA 是一款基于以太坊的 NFT 市场。
以上只是一些以太坊的典型应用场景，还有很多其它场景也正在尝试使用以太坊。
