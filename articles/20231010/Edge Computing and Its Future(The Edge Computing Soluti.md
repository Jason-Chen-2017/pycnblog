# 1.背景介绍


边缘计算技术（edge computing）是指将运算任务分布到靠近数据的位置进行计算的一种新型计算模式。基于物联网、嵌入式设备及传感器等异构信息环境下的数据处理能力，边缘计算可以更好地满足资源匮乏、低带宽、高延迟以及数据孤岛等多种实际应用场景下的需求。边缘计算的一个典型特征就是将海量数据集中存储于靠近数据的地方（边缘），然后根据需要对其进行处理，而不是像中心服务器一样处理所有数据。

然而，随着边缘计算技术的飞速发展，它也带来了诸多的技术挑战和应用前景。这些挑战包括如下几方面：

1. 成本效益：由于边缘计算设备的价钱昂贵且功能受限，因此不易普及开放给所有用户。如何降低边缘计算设备的部署成本和维持运营成本是一个关键性课题。
2. 数据迁移与安全：当大量数据集中存储在边缘时，如何进行有效的数据迁移就成为一个重要问题。同时，如何保障边缘计算设备上数据的安全也是一项挑战。
3. 服务质量：边缘计算设备上运行的服务质量不一定能够达到最佳状态。如何保证边缘计算服务的质量，提升性能和稳定性，还需要进一步研究。
4. 智能网关：如何能够智能地调度边缘计算资源以及向下流动数据也是当前和未来的一个重要挑战。
5. 可扩展性：边缘计算的可扩展性一直被视作一个比较难解决的问题。如何通过自动化部署、负载均衡、弹性伸缩等方式提升边缘计算平台的容量、性能和灵活性，也成为一个需要解决的课题。

对于这些技术挑战和应用前景，目前还没有统一的解决方案。但是，边缘计算行业正在蓬勃发展，一些新的技术和产品正在涌现出来，比如微服务、容器、边缘云、函数式编程、serverless计算、超融合等等。因此，本文将从宏观的角度出发，分析边缘计算技术的演变历史，阐述边缘计算的关键要素，以及一些典型的应用案例和未来方向。

# 2.核心概念与联系
## 2.1 边缘计算技术简介
边缘计算（edge computing）:指将运算任务分布到靠近数据的位置进行计算的一种新型计算模式。主要特点是把海量数据集中存储于靠近数据的地方（边缘），然后根据需要对其进行处理，而不是像中心服务器一样处理所有数据。


边缘计算的主要优点：
1. 大幅降低中心服务器处理数据的压力，减轻网络带宽压力。
2. 提升计算速度，增加了大数据分析能力。
3. 在特定区域内提供本地计算能力，降低中心服务器带宽压力。
4. 可节省边缘设备投入成本。

## 2.2 边缘计算技术与云计算的区别
边缘计算和云计算是两种截然不同的计算模式，它们之间还有很多共同之处。下面简要回顾一下两者之间的不同之处。
### （1）模型
云计算关注的是整体式计算，也就是所有节点共享一套计算资源，比如分布式存储、计算节点、网络资源等。在云计算模型里，所有节点都有自己的IP地址和计算资源，所有节点之间通过网络进行通信。边缘计算则是将计算任务分散到距离数据源更近的地方进行计算，因此模型上有所不同。
### （2）计算目标
云计算侧重于计算资源共享和弹性伸缩，主要用于处理大规模、复杂的计算任务，如Hadoop、Spark等；边缘计算侧重于将计算任务分布到本地端进行快速响应，主要用于处理实时、精细化、数据敏感的计算任务，如机器人、视频监控等。
### （3）基础设施
云计算依赖于底层的基础设施硬件、软件等，通常采用商用、公有云服务或私有云部署；边缘计算则依赖于各类边缘计算设备，比如移动设备、嵌入式设备等，不需要投入大量的基础设施硬件。
### （4）计算框架
云计算的计算框架主要基于开源框架如MapReduce、Hadoop等构建，支持多种编程语言；边缘计算的计算框架往往依赖于轻量级框架，如TensorFlow Lite、Google Edge TPU等。
### （5）数据模型
云计算的数据模型是具有高度抽象的数据集合和处理规则，比如Big Data框架中的HDFS、MapReduce、Spark等；边缘计算的数据模型则以实体对象作为中心，并与外部数据源对接，结合业务逻辑和规则引擎实现数据处理。
### （6）管理模式
云计算平台通常由第三方服务商运维和维护，拥有完整的管理工具和组件；边缘计算平台通常由设备制造商、设备运维商或个人开发者自行部署和管理。

综上所述，云计算和边缘计算是两种截然不同的计算模式，它们之间有很大的区别。云计算旨在通过共享计算资源和弹性伸缩降低成本，适用于处理大规模、复杂的计算任务；边缘计算旨在将计算任务分布到本地端进行快速响应，适用于处理实时、精细化、数据敏感的计算任务。另外，云计算依赖于底层的基础设施，需要购买云服务器等大量昂贵的硬件；边缘计算依赖于各种边缘计算设备，其硬件成本相对较低。
