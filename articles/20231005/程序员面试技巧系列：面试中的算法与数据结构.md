# 2.核心概念与联系
## 数组 Array（数组）
数组是一种线性存储的数据结构，它用连续内存空间存储元素，并且可以随机访问元素，方便进行数据的查找、插入、删除等操作。数组的优点在于效率高，缺点是大小固定，不容易扩充容量，同时随机访问速度较慢，适合于读取少量数据。常见应用场景包括动态规划中的状态转移方程、排序算法中的比较次数和基数排序。
## 链表 Linked List（链表）
链表是一种非连续的内存空间，它的各个节点存在前驱和后继关系，通过指针地址连接，因此可以在 O(1) 时间内找到特定元素。但由于每个节点可能存在多个指针，导致内存利用率低，插入、删除操作比较麻烦。常见应用场景包括浏览器的缓存管理、LRU Cache 算法。
## 栈 Stack（堆栈）
堆栈是一种先进后出的线性存储结构，按照后进先出的原则存储元素，只能在尾端（top）进行插入或者删除操作。它支持基本的 push() 和 pop() 操作，其他操作都要依赖于栈顶元素。
## 队列 Queue（队列）
队列也是一种先进先出的线性存储结构，不过不同的是队列只有两端（rear 和 front），在队尾添加元素，在队头删除元素。它支持基本的 enqueue() （入队）和 dequeue() （出队）操作，其他操作都要依赖于队列长度。
## 哈希表 Hash Table（散列表）
哈希表是一种快速索引查找的无序数据结构，通过 key-value 对来存取元素，key 通过某种方式映射到固定的地址上，这样就可以直接计算得到目标元素的位置。它具有很快的查询速度，同时也可以解决冲突问题，常见的冲突解决方法有开放寻址法、链表法等。
## 树 Tree（树）
树是一个抽象数据类型，用来模拟具有层次关系的集合或结构。它分为根结点、中间结点和叶子结点三个部分。常见的应用场景包括文件目录组织、二叉搜索树（BST）及其各种变形、图论等。
## 二叉树 Binary Tree（二叉树）
二叉树是树的一个子集，每个节点最多拥有两个孩子节点。它有左右之分，便于表示层次结构。二叉树的性质包括：一棵二叉树第i层上至多有2^i - 1个节点；左子树上所有节点的值均小于当前节点值，右子树上所有节点的值均大于当前节点值；整棵二叉树的左侧即为深度优先遍历序列，从左到右层次依次排列；如果对二叉树进行按深度遍历，每次遇到的结点都是该子树的最左边的结点；对二叉树进行按层次遍历，从上至下逐层遍历。常见应用场景包括二叉查找树、排序二叉树、平衡二叉树、堆、线段树、Trie 树等。
## 堆 Heap（堆）
堆是一种特殊的完全二叉树，每个节点的值都大于等于（或小于等于）其左右孩子节点的值。堆通常被用于实现优先队列，其中每一个元素都有相关的优先级。常见应用场景包括求最小值的堆、最大值的堆、二项堆、斐波那契堆等。
## 双向链表 Bidirectional Linked List（双向链表）
双向链表是在单向链表的基础上增加了一系列指向前驱和后继的指针，使得查找、插入、删除操作更加方便。它有利于在链表中间快速定位特定元素。
## 图 Graph（图）
图是一种由边组成的集合，边通常用“弧”来表示。它可用来表示现实世界的一切物体之间的相互联系，常见的应用场景包括网络结构、 social network、 Web 技术架构、交易所订单流转流程、拓扑图等。
## 排序算法 Sorting Algorithm（排序算法）
排序算法是将一组数据按照指定顺序重新排列的方式。排序算法的目的就是为了能够轻易地对这些数据进行比较、处理和检索。目前，经典的排序算法有冒泡排序、选择排序、插入排序、希尔排序、归并排序、快速排序、堆排序等。
