# 1.背景介绍


## 概念及其定义
“边缘计算（Edge Computing）”是一个由英特尔、高通、联发科等厂商提出的概念，主要是指将一些计算任务或者数据处理放在网络边缘进行，通过高速的数据传输，降低计算负担，减少云端存储的压力。具体而言，边缘计算就是将智能应用部署到用户终端设备中，即所谓的边缘计算中心。

边缘计算的定义比较模糊，需要结合具体场景、技术方案、解决的问题来理解。笔者认为边缘计算可以用于以下几种场景：

1. 实时响应性：针对物联网或工控领域的应用场景，实时性是最重要的。这种场景下，边缘计算可以在一定程度上保证用户的实时响应，比如远程监控、自动驾驶等。

2. 节省成本：由于边缘计算能够在不占用大量云端资源的前提下，降低计算任务的费用，因此有助于节约云端资源的开销。例如，在移动互联网、物流、金融、人工智能等场景都可以考虑采用边缘计算。

3. 隐私保护：边缘计算还可以提供隐私保护功能，即避免收集和共享用户敏感信息。此外，边缘计算可以帮助用户实现数据的分析、处理和分析等工作，降低对云端服务商的依赖。

4. 灵活性：随着云计算的普及和发展，越来越多的企业开始采用云端服务。然而，有些企业仍然倚赖于传统硬件设施。如今，边缘计算可以提供灵活性，让更多的业务流程移至边缘。

## 发展现状
目前，国内外已经有不少行业开始尝试边缘计算，包括但不限于智慧城市、智慧交通、智慧教育、智慧医疗等。中国最大的公司，腾讯（Tencent），近年来已经逐渐转向边缘计算，有关部门也推出了边缘计算的相关产品，比如腾讯云边缘计算平台（TKE）。

不过，真正落地的边缘计算项目非常有限，而且效果也并非特别好。部分项目存在较大的技术壁垒、性能瓶颈等难题。而另一方面，也有大量的机构、企业、研究人员等参与到边缘计算的探索当中。对于国内的行业以及领域，目前还没有统一的标准体系，各家公司都在自己的研发路线上积累经验。此外，不同行业的标准、规范、工具也不尽相同。因此，如何更好的落地边缘计算，也是一个值得探索和思考的问题。

# 2.核心概念与联系
## IoT
### 定义
IoT（Internet of Things）是一种基于物联网技术的网络形态，是由智能设备以及这些设备所生成的数据组成，可连接物理世界和数字世界，使之互联互通。它通过信息采集、传输、处理、应用等方式实现相互之间的通信、交换和融合，从而赋予人类生活新的能力。IoT的定义比较宽泛，通常把IoT分为五个层次：物理层（Physical Layer）、网络层（Network Layer）、数据层（Data Layer）、应用层（Application Layer）和服务层（Service Layer）。每一层都构建在上一层之上，上层的功能是为了完成下一层的功能提供支撑。所以IoT的最终目标是使整个社会变得智能化。

### 架构
IoT的架构如下图所示：

#### 物理层 Physical Layer
物理层包括传感器、节点、网络设备、连接器、集线器等。物理层的作用是把传感器的数据采集到网络中，然后通过网络传输给服务层。物理层通常部署在建筑物、工厂、仓储等周边环境中。

#### 网络层 Network Layer
网络层包括无线通信网络、蜂窝网络、WiFi网络等。网络层的作用是将物理层的数据包路由到达目的地，同时保持数据的完整性、可靠性和安全性。网络层的关键技术是路由协议、链路管理、数据压缩、加密和安全机制等。

#### 数据层 Data Layer
数据层包含协议栈、数据库、消息队列等。数据层的作用是接收物理层传送的数据，解析出相应的指令或数据，并转换成服务层能识别的格式。数据层的关键技术是MQTT、CoAP、LwM2M、OPC UA、LoRaWAN等物联网协议。

#### 应用层 Application Layer
应用层包括智能手机、电视、摄像头、传感器、路由器、防火墙等。应用层的作用是处理网络上传输的数据，执行指令，实现控制。应用层的关键技术是计算机视觉、自学习、人工智能、规则引擎等。

#### 服务层 Service Layer
服务层包含平台、中间件、应用服务等。服务层的作用是实现网络的安全、可用性、可扩展性，为终端设备、传感器之间的数据流动提供基础设施支持。服务层的关键技术包括消息队列、分布式计算、容器编排、微服务、API Gateway等。

## Edge Computing
### 定义
边缘计算（Edge computing）是一种与云计算相对立的新型的计算模式，其基本特征是将计算任务或者数据处理放在网络边缘进行，通过高速的数据传输，降低计算负担，减少云端存储的压力。边缘计算的理念是：将复杂且耗时的计算任务在距离源点较远的地方实现，并通过智能边缘端接入网络提供服务。随着移动互联网、物流、金融、人工智能等各种场景的需求，边缘计算已成为迅速发展的计算新范式。

边缘计算的定义比较模糊，需要结合具体场景、技术方案、解决的问题来理解。笔者认为边缘计算可以用于以下几种场景：

1. 实时响应性：针对物联网或工控领域的应用场景，实时性是最重要的。这种场景下，边缘计算可以在一定程度上保证用户的实时响应，比如远程监控、自动驾驶等。

2. 节省成本：由于边缘计算能够在不占用大量云端资源的前提下，降低计算任务的费用，因此有助于节约云端资源的开销。例如，在移动互联网、物流、金融、人工智能等场景都可以考虑采用边缘计算。

3. 隐私保护：边缘计算还可以提供隐私保护功能，即避免收集和共享用户敏感信息。此外，边缘计算可以帮助用户实现数据的分析、处理和分析等工作，降低对云端服务商的依赖。

4. 灵活性：随着云计算的普及和发展，越来越多的企业开始采用云端服务。然而，有些企业仍然倚赖于传统硬件设施。如今，边缘计算可以提供灵活性，让更多的业务流程移至边缘。

### 架构
边缘计算的整体架构如下图所示：

#### Core
Core组件包括硬件（如网卡、服务器、处理器）、运算框架（如TensorFlow、Spark等）以及应用程序。Core组件主要作用是在本地端运行，并负责数据的处理、转换、传输等工作。

#### Edge Device
Edge Device组件包括各种嵌入式设备（如摄像头、雷达等）、小型PC（如树莓派）等。Edge Device组件的主要作用是运行应用程序，根据应用程序的要求完成必要的运算，并将结果返回Core。

#### Cloud Server
Cloud Server组件是云端的服务器集群。Cloud Server组件的作用是作为数据中心的云端服务端，主要用来承载任务的分发和调度。

#### Data Storage
Data Storage组件包括本地文件系统、NoSQL数据库等。Data Storage组件的主要作用是用来存储应用产生的数据，并提供统一的访问接口。

#### Communication
Communication组件包括网络通信模块（如WiFi、4G LTE等）、终端设备配套模块（如摄像头采集模块等）。Communication组件的作用是对外提供网络连接，并且负责各种传输协议的实现，比如MQTT、COAP等。

## 对比
IoT、边缘计算两者之间有很多类似之处，但是也有区别。IoT是将物理世界和数字世界相互联互通的技术，而边缘计算则更关注计算的局部优化，并通过分布式计算的方式将任务分布到网络中的特定位置。IoT侧重于处理海量的传感器数据，而边缘计算侧重于完成单个任务的快速处理。IoT更加关注的是数据的收集和处理，而边缘计算更注重数据的处理效率和实时性。IoT的核心目的是实现物联网，而边缘计算则以边缘端为中心，提升数据的处理效率、实时性和计算密度。
