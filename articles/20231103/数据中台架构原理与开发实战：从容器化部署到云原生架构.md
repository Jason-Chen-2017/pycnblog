# 1.背景介绍


## 1.1 数据中台简介
数据中台是一个综合性的产品，由数据采集、清洗、存储、计算、分析等多个环节组成，能够将不同业务线的数据进行整合，并对外提供统一的服务，主要包括如下几个方面：

1. 数据采集
包括数据源的接入、数据的质量检测、数据传输、加工等；

2. 数据清洗
对数据进行有效过滤、重组、规范化、关联、补齐等处理；

3. 数据存储
包括数据的分类、格式、持久化、检索、统计等；

4. 数据计算
包括数据流计算、机器学习、图分析、时序分析等；

5. 数据分析
包括业务报表的生成、数据挖掘、运营决策支持等；

6. 数据服务
包括统一的访问入口、数据共享和管理、数据服务治理等。

传统的单体应用模式存在以下缺点：
- 系统耦合度高，不同业务模块间难以复用，重复建设造成资源浪费、效率低下、风险提升；
- 对不同数据源依赖不同技术栈，技术栈异构带来的效率低下；
- 数据质量和一致性无法得到保障，数据随意波动会导致整个系统崩溃。

为了解决这些问题，中小型公司往往选择云计算平台作为解决方案，但很多时候需要考虑如何构建云端的数据中台。云数据中台要兼顾速度、易用性和安全性，因此必须满足以下几点要求：

1. 数据价值加速：通过数据中台，数据可以不依赖于研发，只需关注数据本身的价值，自动获得更多收益；

2. 数据对内和对外分离：中央调度中心不需要直接接触所有数据，而只负责对内数据的收集、处理和管理；

3. 标准化的数据交换：不同业务部门之间也可以方便地共享数据，减少重复建设；

4. 数据安全：必须保证数据在中央管控之下安全，防止数据泄露、篡改或恶意攻击；

5. 云数据中台支持多云平台：统一数据接入和整合，同时也适应多云平台的需求；

6. 数据融合性强：可以充分利用各个业务线的优势和数据，实现更大的价值。

## 1.2 数据中台特点
数据中台具有以下几点特征：

1. 数据驱动：数据是价值的来源，数据中台的核心理念就是数据驱动的。如何把数据赋能到业务，真正体现了数据中台的核心竞争力，也是数据中台的根本优势。

2. 混合计算：数据中台具备全面的智能计算能力，可以运行各种计算任务，包括图像识别、语音识别、推荐算法、预测算法、大数据计算等。

3. 跨越组织边界：数据中台中的数据不仅可以在内部流通，还可以跨越组织边界。通过数据中台，可以搭建起多种不同的联盟关系，实现不同业务之间的共赢。

4. 可扩展性：数据中台可以通过增加计算资源、扩容规模等方式快速响应业务变化。另外，数据中台可以使用消息队列和数据湖等新型架构，具备高度可靠性、高可用性和可伸缩性，帮助企业降低成本和提升效率。

5. 个性化服务：数据中台还可以根据用户习惯、偏好、偏好组合和实际情况，为用户提供个性化的服务。例如，根据用户的年龄段、居住区域、消费习惯等等信息，推荐相关商品、服务。
