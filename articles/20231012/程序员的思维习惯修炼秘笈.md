# 1.背景介绍


现在全世界都在用人工智能、区块链、机器学习等新型技术，自动化手段极其强劲。随着信息爆炸的速度越来越快、内容越来越丰富，个人获取知识、分析信息、解决问题的能力也越来越需要更高的水平。但是技术日新月异，编程语言、框架、工具层出不穷。如何快速掌握一种新的编程语言、框架、工具的技能、方法论呢？如果想更好地提升自己的综合素质、保持学习热情和投入产出比，除了跟上技术的发展之外，更重要的是提升自己的思维习惯。

作为一个程序员或者软件工程师，对于如何更有效地学习、掌握某个编程语言、框架、工具，大家肯定有很多心得体会。有的认为“速成”是学习的方法，通过阅读文档、看视频、做实践练习，就可以掌握所需的技能；有的则认为“知其然而不知其所以然”，靠记忆和临时抓取的东西不可取；有的则认为“学以致用”，“知道了才行”。这些都是针对不同层次的学习者提出的建议，但这些建议在实际应用中往往适用性不强。因此，作者建议从以下几个方面出发，推进程序员的思维习惯的修炼：

1. 建立基于项目的学习机制。多年来，程序员群体一直缺乏系统的学习机制，缺乏对特定领域技术、工具、理论的整体把握，这就导致他们学习能力低下、效率低下，缺乏长期价值。程序员学习必须围绕一个或多个具体的项目进行，并逐渐深入理解它背后的理论和技术。项目学习可以让程序员有机会亲身参与到项目的开发和设计当中，充分发挥自己的才能，锻炼自己独立完成任务的能力。

2. 注重总结和归纳。每种编程语言都有它的基础理论、算法、设计模式等理论知识，这些知识的积累成为程序员核心竞争力。通过学习并运用最新的技术，程序员能够对各种问题形成清晰的认识和归纳总结。比如，知道什么时候应该使用内存管理、堆栈和指针，什么时候应该使用引用计数技术，什么时候该使用泛型技术等。

3. 从自己的工作实践中学习。要真正掌握某项技术、语言、工具，最好的方式还是通过实际应用来提高自己的理解。自己的实际项目经历往往是最好的学习材料，它们可以反映出程序员遇到的问题、需求、困难、以及解决方案，甚至还能帮助程序员建立起更好的职业操守。

4. 坚持持续学习。学习是一个长期过程，时间太短很容易被老旧的知识所淹没。相反，持续学习意味着不断更新自己的知识和技能，构建自己的知识结构和能力网络。只有将自己的学习反馈给社区，才能促进别人的学习，增强自己的影响力。

以上四点，既可以是作为程序员，也可以是作为一个公司的CTO，以及整个技术团队的领导者，教导他们逐渐建立对编程语言、框架、工具的全局认识，形成系统的学习机制，形成最优的学习计划和方法，并且坚持不懈地持续学习，最终提升自己成为一名合格的高级程序员、架构师。
