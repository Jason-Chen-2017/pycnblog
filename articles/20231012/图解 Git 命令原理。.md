# 1.背景介绍


Git 是目前最流行的分布式版本控制工具，并得到了广泛的应用。本文将通过梳理 Git 的底层命令、命令之间的关系以及各个命令的作用及内部机制等内容，帮助读者更好地理解、使用和掌握 Git。
# 2.核心概念与联系
## Git 的三个对象
在 Git 中，可以把仓库看作是一个保存各个版本文件快照的文件夹，称之为工作区（Working Directory）。初始状态下这个文件夹是空的，只有执行 git init 命令才会创建一个.git 目录，里面存储着整个仓库的重要数据。


仓库中有三个对象类型：blob、tree 和 commit 对象。blob 对象是二进制文件的数据块，即文件的实际内容。每个 blob 对象都有一个唯一 ID（由 SHA-1 计算得出），用于标识它的身份。

Tree 对象是一种树状结构，它记录了一个目录或文件名列表及对应的 blob 或 tree 对象指针，用于表示一个目录或者文件的内容。每个 Tree 对象也有一个唯一 ID。

Commit 对象是一个提交记录，记录着某个特定时间点上某个版本库的一个快照。除了包含指向 Tree 和 Blob 对象的指针外，Commit 对象还包含作者信息、提交信息、父 Commit 的指针等其他信息。每一次提交都会产生一个新的 Commit 对象。


## 分支（Branch）与标签（Tag）
Git 可以创建分支，每次提交时都会自动创建一个新的分支。用户可以通过创建不同的分支，独立开发和测试不同的功能，最后再合并到主干分支。

Tags 是轻量级的打标记，可以在某一个提交上打一个标签，用于标记重要的版本发布点。常用的 Tags 有两种：轻量级标签（Lightweight Tag）和 annotated 标签（Annotated Tag）。

Lightweight Tag 只是指向一个提交，通常不会包含任何信息，只能用来快速切换回该提交。

Annotated Tag 除了指向一个提交外，还包含额外的注释信息，例如是哪里发布的、代码的版本号、发布描述等等。

## 技术术语
为了帮助读者理解文章内容，我将用一些名词来定义一些 Git 的基本术语。这些定义可以帮助大家更好地理解文章。

Repository：仓库。用于存放项目代码、文档、资源文件的地方。每个仓库有一个唯一的 URL。

Clone：克隆。从远程服务器上下载一个完整的仓库到本地电脑，称为克隆。

Branch：分支。开发人员可以在仓库中创建多个分支，每个分支独立开发和测试不同的功能，最后再合并到主干分支。

Merge：合并。把两个分支的变更合并到一起。

Fork：分叉。复制一个项目，修改后再提交给原作者。

Remote：远程。远程主机上的仓库，可以在本地进行克隆、推送等操作。

Master Branch：主分支。默认情况下，新创建的仓库中就存在一个 master 分支，所有的提交都会被合并到 master 分支。

HEAD：当前位置。指向当前所在的分支和提交。

Status：状态。显示当前所在的分支、是否有未暂存的修改，以及未提交的改动。

Commit Message：提交信息。每次提交时，都会带上一段信息作为备注，方便记忆和版本回溯。

Stash：暂存。保存工作区的修改，方便临时切换分支。

Checkout：检出。切换分支、恢复暂存区和工作区。

Log：日志。查看提交历史记录。

Diff：比较。比较不同分支或者版本之间的差异。
