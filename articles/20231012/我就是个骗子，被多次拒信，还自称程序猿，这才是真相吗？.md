# 1.背景介绍


## 什么叫做骗子
其实“骗子”这个词，并不是人们的专有名词。古时候的人也用过这个词，主要用于指当事人所谓的造谣者，即在公众面前自称是某个人、组织或是某个行为。但是随着现代社会的发展，这种用法渐渐变得愈加严重，且不乏危害性。于是，人们开始反对把“骗子”作为一种独立的观念，而是将其与各种违法犯罪行为相联系。因此，很多年前，人们就开始质疑“骗子”这个词是否应该加以严格限制。
## 为什么会遭遇骗子的厄运
我们都知道，网络的普及让信息的传播变得越来越快捷，而且由于大数据分析的手段的广泛应用，使得网民的信息获取能力也越来越强，从而产生了越来越多的虚假信息和欺诈行为。因此，如何能够有效地保护自己的信息安全一直成为社会热点之一。
但与此同时，由于互联网时代的发展，计算机技术的快速发展，人们越来越依赖于数字化的信息交流，越来越需要互联网上的各种服务。在这些信息服务中，涉及到各种交易活动，如信用卡支付、购物、养老保险等。越来越多的人们开始利用这些支付服务，从而更多地受益匪浅。
另一方面，网络平台上充斥着大量恶意的广告和虚假信息。这些广告和虚假信息可能涉及到消费者的个人信息、身份认证、金钱交易和交易信息等，导致用户隐私泄露或资产损失。这也是为什么很多年前，一些网络借贷公司把这类信息用作投放策略，引诱用户点击进入以获取额外的金钱和信息。
为了抵御这些攻击，许多网站、APP都会设立各种反骗机制，如恶意账户检测、风控审核、联合惩戒等。但是，反骗只是第一步，如何帮助人们更好地保障自己的权益，尤其是那些正在受到骗子的威胁的人呢？
## 为什么要去区分“骗子”和“程序猿”
首先，骗子是一个很容易被误导的词汇。尽管其字面意义十分模糊，但它确实存在着。举个例子，在一场婚礼上，一位母亲看起来像骗子。她声称自己是在找男朋友，结果双方都觉得女方是真心的。很多年前，一些科幻电影里也出现过类似的情节。所以，给人混淆视听固然可取，但实际上还是应当注意分辨。
其次，程序员这个职业本身也存在着同样的问题。程序员的头衔往往以他们的工作经历和相关能力来刻画，而不是他们本人的品德操守。因此，给程序员贴上“骗子”的标签，并非没有道理。毕竟，程序员要处理的数据量和复杂性远超普通用户，他们必须掌握一定的技巧和能力。如果骗子企图通过努力掩盖自己的技能水平，那么他们的后果可能会很严重。另外，编程语言、工具链、框架等都是程序员的一项基本素养，如果骗子企图通过糊弄性话语来掩盖这一点，可能会导致他们进一步陷入歧途。
最后，我们可以想象一下，如果你是一个骗子，不小心被别人误认为“程序猿”甚至“程序媛”，那么你的麻烦就会非常大。除了你的经济损失和信任危机，你还可能失去自己的工作机会和升职空间。这无异于是自杀式的骚扰。那么，怎样才能保护自己的权益，防止因不正当的语言或行为受骗呢？
总结来说，区分“骗子”和“程序猿”是一件很重要的事情。只有我们清楚区分两者之间的差异，才能更好地保护自己的信息和财产安全。
