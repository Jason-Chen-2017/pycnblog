# 1.背景介绍

## 为什么要颁发GitHub贡献者证书？
越来越多的开发者都选择了使用GitHub来进行开源项目的协作开发工作。而对于开源项目贡献者来说，能够获得GitHub贡献者证书可展示自己对开源项目的贡献和影响力，从而得到肯定和认可。通过该证书，您可以在您的个人资料中显示相关信息，比如您的经历，参与的开源项目等，为自己加分。

除此之外，GitHub还推出了一个奖励计划——最佳贡献者计划，鼓励优秀的开源项目贡献者在GitHub上公开展示他们的贡献，促进整个开源生态的蓬勃发展。该计划还设立了一个奖励榜单，有缘的开发者都可以申请免费的奖品。因此，通过该证书和奖励计划，你可以展示自己对开源世界的贡献。

# 2.核心概念与联系
## 1.什么是GitHub贡献者证书?
GitHub贡献者证书是GitHub推出的一种权利徽章，旨在证明开发人员已经为开源项目作出了贡献。证书颁发给符合条件的个人开发者，并可以记录他们在GitHub上的相关信息。证书的内容包括：

1. 在GitHub上所提交的代码量
2. 对已有的issue或pull request所做出的评论
3. 在Discord、Slack或者其他沟通渠道上所作出的贡献
4. 在开源项目上所做出的重要贡献，如文档修改、功能实现等。

除了证书上的这些信息外，证书还可以指定个性化的徽章样式。它可以代表开发人员的能力和成果，也可以激励开发人员不断维护和完善开源项目。

## 2.GitHub贡献者排行榜
GitHub每月都会根据新提交的GitHub贡献者数量更新自己的贡献者排行榜。排行榜上榜的都是活跃的开发者，其中包括经常参与GitHub开源项目的开发者、投身开源事业的技术人士，还有有突出贡献的人物。排名前列的开发者都可以在其个人资料页面上看到自己的贡献者证书。


## 3.如何获得GitHub贡献者证书？
为了获得GitHub贡献者证书，首先需要满足以下条件：

1. 必须是GitHub上注册过的注册用户；
2. 必须在三个月内至少提交过一次有效的PR(Pull Request)，包括对已有issue的评论和提交新的issue和PR；
3. 必须至少完成三种类型的有效贡献。有效贡献类型包括：
   - 提交至少3个有效的PR(Pull Request)。
   - 对已有的issue或PR做出了有效的评论。
   - 在各类社区论坛、聊天室或者微信群上作出了有效的贡献。

满足以上条件后，GitHub就会审核你的PR列表和贡献历史，确认你是真正的一名贡献者。审核通过之后，你就可以在你的GitHub主页上看到一条提示，上面会有你的GitHub贡献者证书。当然，如果你之前一直在某个开源项目提交PR或参与讨论，并且被授予了证书，那你今后也只需再次满足提交PR和评论issue即可。

# 3.核心算法原理与具体操作步骤
## 1.PR(Pull Request)评级标准
GitHub会根据你的PR(Pull Request)的质量对其进行评级，评级范围如下：

1. Gold Badge：获得了5颗星，最高奖项。
2. Silver Badge：获得了4颗星，奖项略低于Gold Badge。
3. Bronze Badge：获得了3颗星，奖项略低于Silver Badge。
4. Ghost Badge：获得了2颗星，奖项略低于Bronze Badge。
5. Lowest Badge：获得了1颗星，奖项最低。

当你完成PR时，GitHub会自动评判你是否达到了最低要求，如果达到了，你就会得到相应的奖项。具体的流程如下：

提交PR -> 作者接受PR -> PR作者提交经验值（称为Commitment Points） -> GitHub审核你的PR -> 根据审核结果决定你是否获得相应的Badge。
