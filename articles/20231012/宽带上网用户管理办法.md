# 1.背景介绍


随着互联网技术的发展和应用的广泛普及，“宽带上网”已成为人们生活中不可或缺的一部分。根据电信运营商的规定，宽带上网的主要目的是为用户提供网上信息服务、网络存储等基础服务。而在实际运行过程中，由于多种因素的影响，如恶劣天气、突发事件等，也会对宽带上网用户产生不利影响。因此，为降低因突发事件或意外情况导致的宽带上网用户服务质量下降风险，推动宽带上网的管理便成为企业应尽的义务。
事实上，《信息安全法》第四条对“网络服务提供者”的宽带上网管理提出了相关要求。即各级工信部、直属单位和网站可以根据本法第十七条第五款的规定，制定并实施《宽带上网用户管理办法》。
《宽带上网用户管理办法》包括两个方面内容，一是关于用户管理的规范；另一是关于突发事件发生时，如何及时有效地采取措施保障用户的正常访问和使用，提高用户体验。《宽带上网用户管理办法》还应当具备保密性、安全性、法律约束力和执行效率等诸多重要特征。
近年来，我国市场上已出现了一批宽带上网的管理良好的企业。例如，腾讯公司就自成立以来屡获殊荣，其管理办法已由中国电信联合全国广播电视总局下达，将“网络服务提供者”分为“边缘计算服务提供者”和“应用云计算服务提供者”，对边缘计算服务提供者适用所需的管理规定。微软公司目前也制定了自己的宽带上网管理办法，也被中国电信联合全国广播电视总局认可。
但是，存在以下一些弊端：

1.管理办法一般仅涉及到对接线路的管理，而对业务的定义、用户群体、交费模式、产品质量等方面的规范却没有得到很好落实。另外，对于某些非宽带业务，也未进行监管。

2.行业内尚且存在很多矛盾和争议，如何协调不同方面的力量，让互联网更加安全，这是值得探讨的课题。

3.无论是“如何推进业务合规”还是“如何构建健康的生态环境”，都需要依靠更多、更好的管理机制来促进互联网经济的发展。

因此，为了推动宽带上网管理制度化、健康生态建设，推动宽带上网经济的持续增长，我国正在研究制定新的宽带上网管理办法，完善《宽带上网用户管理办法》。
# 2.核心概念与联系
## 2.1 用户管理的概念与基本概念
宽带上网用户管理是指管理进入宽带上网者的身份验证、授权、计费、资源分配、网络质量管理、流量控制、安全防范、数据隐私保护等一系列活动，确保宽带上网的顺利进行。而宽带上网用户管理的核心概念如下：
### 用户管理的分类
宽带上网用户管理根据管理对象，可分为“企业内部用户管理”、“单位用户管理”和“个人用户管理”。
- “企业内部用户管理”是指企业内职工、学生、教师等人员使用宽带上网的管理。在这种情况下，管理对象为企业内部职员和教师。管理办法一般包括身份确认、业务授权、账单结算、容量分配、资源保障、流量控制、终端管理等。
- “单位用户管理”是指企业外部单位人员使用宽带上网的管理。在这种情况下，管理对象为所在单位职工、学生、教师等人员。管理办法一般包括网络质量保证、设备安全管理、网络安全保障、个人信息保护等。
- “个人用户管理”则是指居住在公共区域的个人使用宽带上网的管理。管理办法一般包括帐号管理、流量控制、网络质量保证、用户体验提升等。
### 用户管理的基本概念
宽带上网用户管理的基本概念有：
- 用户（User）：指具有宽带上网权限的人员。
- 服务提供者（Service Provider）：指通过网络向用户提供各种网络服务的组织。
- 交费模式（Billing Mode）：指用户网络消费按时间、包月、流量、固定费用等方式收费的方式。
- 服务水平协议（SLA）：指服务提供者对用户的正常访问时段和时间内的网络质量给予承诺，以及用户因违反承诺而受到的惩戒的标准。
- 熟练程度要求（Proficiency Requirement）：指用户接受的网络技术水平、服务水平的最低要求。
- 用户位置（Location）：指用户所在地区、工作地点等。
- 攻击源（Attack Source）：指用户或其网络存在的网络攻击行为或安全威胁。
## 2.2 突发事件发生时的管理措施
突发事件发生时，可能带来多个不同的影响，如网络故障、电力供应不足、突发事件发生等，这些事件对宽带上网的正常运行产生重大影响。因此，必须通过预案和措施，从宏观角度和微观角度综合考虑，充分利用信息通信科技、电力等新兴技术，最大限度地减少或者避免对宽带上网造成的严重影响。
《宽带上网用户管理办法》既要遵循相关法律法规的要求，也要注重实际需求和实际效果，根据不同的事件类型和影响范围，制定相应的应急措施。其中，在网络故障、电力供应不足、突发事件发生等紧急情形下的管理措施如下：
### 情况一：网络故障
网络故障对宽带上网的正常运行造成影响，比如，某个用户无法登陆、数据传输失败等。应对此类事件，首先应先确定故障原因，然后采取事前、事中、事后三个阶段的救援措施，及时排查并补救故障，确保业务正常运行。在事前阶段，通过对用户终端及终端服务器进行测试、维护等方式，检测并修复系统故障。在事中阶段，根据网络波动、流量异常等其他因素，进行临时调整，以保证业务运行。在事后阶段，根据故障原因及日志进行分析，制定相应的维修方案，避免再次发生同样的故障。
### 情况二：电力供应不足
电力供应不足会造成网络拥塞，导致网络故障，引起业务中断。因此，应该高度重视电力供应，做好电力保障工作，保证有充足的电能供应。在电力供应出现短缺时，应当采取以下措施：
- 在电力购买前，首先对需求进行评估，判断是否可以通过改造系统等方式缓解困难，以及采用什么样的结构能满足系统的性能和稳定性要求。
- 通过建立好的电力供应保障渠道和制度，对远程供电系统进行管理，确保系统稳定运行。
- 针对突发电力供应不足事件，制定应急预案，提升电力供应能力。
### 情况三：突发事件发生
突发事件发生时，应对突发事件应当保持高度警惕，全面、准确地开展系统保护和监测工作。在突发事件发生时，应对可能出现的问题进行及时反馈，及时建立紧急预案，以最大程度地减轻用户和系统的损害。在预案的开发过程中，可参照事故应急预案模板，以确保系统的可靠性和可用性。
