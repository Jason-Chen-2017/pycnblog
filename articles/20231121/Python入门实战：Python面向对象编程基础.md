# 2.核心概念与联系
## 什么是面向对象编程(Object-Oriented Programming, OOP)？
面向对象编程（英语：Object-oriented programming，缩写：OOP）是一种编程方法论，它以对象为核心，关注对象之间的交互和协作关系，利用封装、继承、多态等机制来建立程序的抽象层次结构。在Python中，面向对象编程主要通过类、对象、实例三种基本概念来实现。
### 类（Class）
在面向对象编程中，一个类（Class）是用来创建对象的蓝图或模板。它定义了该对象的属性和行为，并负责存储、管理和维护类的实例。在Python中，通过class关键字定义一个类，并用self关键字来表示类的实例。如下所示：

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def eat(self):
        print("The animal is eating.")

    def sleep(self):
        print("The animal is sleeping.")
```

在以上代码中，Animal类是一个空壳子，只有两个方法eat()和sleep()。其中__init__()方法是构造器（Constructor），用来初始化对象。通过实例来调用这些方法。如：

```python
dog = Animal('Wangcai')
dog.eat()   # The animal is eating.
dog.sleep() # The animal is sleeping.
```

实例（Instance）是在创建类后创建的对象，它由类的 blueprint 和实际的值组成。对象也被称为实例（instance）。每个实例都有一个唯一的标识符（ID）用于区分它。比如，对于上面定义的类Animal来说，每个dog对象都是独一无二的，它们之间不会相互影响。

### 对象（Object）
对象是类的实例，或者说是类的一个实例化结果。对象包含了类中的所有变量和方法，是程序执行的最小单位。就像其他编程语言一样，Python没有提供直接支持对象类型的语法结构，但可以通过组合、继承和多态等机制来模拟对象。

### 实例变量（Instance variable）
实例变量指的是在对象创建时赋予的值，这些值会在整个对象的生命周期内保持不变。实例变量通常保存在内存中，可以通过对象的方法来修改和访问。如：

```python
class Dog:
    def __init__(self, age):
        self.age = age
    
    def bark(self):
        if self.age > 2:
            return "Woof!"
        else:
            return "Yip!"
```

在Dog类中，实例变量age用来记录狗的年龄。当创建一个Dog类的实例时，需要传入age参数的值，如：

```python
my_dog = Dog(3)
print(my_dog.bark())    # Yip!
```

在这里，我们创建了一个叫做my_dog的Dog类的实例，并且设置它的年龄值为3岁。然后调用my_dog对象的bark()方法，打印它吠叫的结果。

### 方法（Method）
方法是对象的一类操作，它允许对象执行操作。方法就是类的一个函数，只是属于某个类而不是对象。方法只能通过对象调用，不能独立调用。方法可以访问对象的私有变量，但是不能访问对象的私有方法。

### 属性（Attribute）
属性是指实例变量，它是对象的特征。它们通常包含关于该对象的描述信息，可以被读取或写入。属性也可以被操作，例如求和、添加元素等。

## 如何理解类、对象和实例之间的关联关系？
类（Class）、对象（Object）和实例（Instance）是面向对象编程的三个基本概念。

类是抽象的概念，是用来描述一类对象的共同特征和行为的集合，是面向对象编程的第一步。每当我们想要创建一个新的对象时，都应该先定义一个类。

对象（Object）是类的一个实例化结果，是程序执行时的实体。对象中包括了状态和行为两方面的信息。

实例（Instance）是根据类创建出的一个具体对象，是类实例化后的产物。实例包含了该对象所有的数据成员（数据属性）、数据方法和函数成员。

按照面向对象的思想，一切皆对象。这意味着，每个对象都是一个类的实例。换句话说，每一个类都包含着自己的一套属性和方法，你可以在这个类的实例上调用这些方法。

类和实例之间的联系非常密切。一个类可以有多个实例，但每个实例只能属于一个类。换句话说，每个实例都拥有自己的数据副本，且只能通过修改该副本来修改自己的状态。

最后，实例变量并不是真正的变量，它仅仅是保存了一个指向数据存储位置的引用。即使将实例变量的值赋予另一个对象，原始对象也不会受到影响。真正的变量保存在数据存储区中。
