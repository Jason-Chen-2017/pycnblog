# 2.核心概念与联系

在Java面向对象编程中，核心概念包括类、对象、继承、多态、抽象、接口等。这些概念是Java面向对象编程的基础，同时也是Java程序设计的核心所在。在本节中，我们将详细讲解这些概念的定义、特点和应用场景。

## 2.1 类

类是Java面向对象编程的基本单元，它是一种模板，用于定义对象的属性和方法。类可以被实例化为对象，每个对象都是类的一个实例。类的定义包括属性、方法和构造函数等。属性用于存储对象的状态，方法用于对对象的状态进行操作。构造函数用于创建对象的实例。

## 2.2 对象

对象是Java面向对象编程中的实体，它是类的一个实例。对象可以拥有属性和方法，可以通过这些属性和方法来操作对象的状态。对象是Java程序的基本组成单元，通过对象之间的交互来实现程序的功能。

## 2.3 继承

继承是Java面向对象编程中的一个重要概念，它允许一个类继承另一个类的属性和方法。通过继承，子类可以重用父类的代码，减少代码的重复性，提高程序的可维护性。继承也允许子类对父类的属性和方法进行扩展和修改。

## 2.4 多态

多态是Java面向对象编程中的一个重要概念，它允许一个对象在不同的情况下表现为不同的类型。通过多态，我们可以在程序中使用更加抽象的类型，而不需要关心具体的类型。多态可以提高程序的灵活性和可扩展性，同时也可以简化程序的设计和实现。

## 2.5 抽象

抽象是Java面向对象编程中的一个重要概念，它允许我们将某些属性和方法隐藏起来，只暴露出需要的部分。抽象可以用于实现代码的封装和模块化，提高程序的可维护性和可重用性。抽象还可以用于实现接口和抽象类的定义。

## 2.6 接口

接口是Java面向对象编程中的一个重要概念，它是一种规范，用于定义对象的行为。接口可以被实现，实现接口的类必须实现接口中定义的所有方法。接口可以用于实现代码的解耦和模块化，提高程序的可维护性和可扩展性。

## 4.3 多态

```java
public class Shape {
    private String color;

    public Shape(String color) {
        this.color = color;
    }

    public String getColor() {
        return color;
    }

    public void setColor(String color) {
        this.color = color;
    }

    public double getArea() {
        return 0.0;
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    public double getRadius() {
        return radius;
    }

    public void setRadius(double radius) {
        this.radius = radius;
    }

    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }
}

public class Test {
    public static void main(String[] args) {
        Shape circle = new Circle("红色", 5.0);
        System.out.println(circle.getColor());
        System.out.println(circle.getArea());
    }
}
```

在上述代码中，我们定义了一个Shape类，该类有一个color属性和getter和setter方法。然后我们定义了一个Circle类，该类继承自Shape类，并添加了一个radius属性和getter和setter方法。Circle类重写了Shape类的getArea方法，实现了计算圆的面积。最后，我们创建了一个Circle对象，并调用其getter方法来获取对象的属性值和计算面积。

Q9：什么是单例模式？
A：单例模式是一种设计模式，它限制一个类只有一个实例，并提供一个全局访问点。单例模式可以用于实现程序的全局资源管理，提高程序的可维护性和可扩展性。

Q10：什么是工厂模式？
A：工厂模式是一种设计模式，它定义了一个创建对象的接口，让子类决定实例化哪个类。工厂模式可以用于实现程序的对象创建和管理，提高程序的可维护性和可扩展性。

Q11：什么是观察者模式？
A：观察者模式是一种设计模式，它定义了一种一对多的依赖关系，让当一个对象的状态发生变化时，所有依赖于它的对象都得到通知并被自动更新。观察者模式可以用于实现程序的事件驱动和数据同步，提高程序的可维护性和可扩展性。
