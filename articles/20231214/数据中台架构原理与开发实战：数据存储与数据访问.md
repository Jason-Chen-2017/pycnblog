# 2.核心概念与联系

在数据中台架构中，数据存储和数据访问是紧密相连的两个概念。数据存储是数据的持久化存储，数据访问是数据的读写操作。数据存储的目的是为了提供数据的持久化和安全性，而数据访问的目的是为了提供数据的可用性和可访问性。

数据存储和数据访问之间的关系可以用以下公式表示：

$$
Data\_Storage \rightarrow Data\_Access
$$

这个关系表示数据存储是数据访问的前提条件。

## 4.1 数据存储的代码实例

```python
import sqlite3

# 创建数据库
conn = sqlite3.connect('data.db')

# 创建表
cursor = conn.cursor()
cursor.execute('''CREATE TABLE data (id INTEGER PRIMARY KEY, value TEXT)''')

# 插入数据
cursor.execute('''INSERT INTO data (value) VALUES (?)''', ('Hello, World!',))

# 查询数据
cursor.execute('''SELECT * FROM data''')
rows = cursor.fetchall()
for row in rows:
    print(row)

# 更新数据
cursor.execute('''UPDATE data SET value = ? WHERE id = ?''', ('Hello, Data!', 1))

# 删除数据
cursor.execute('''DELETE FROM data WHERE id = ?''', (1,))

# 关闭数据库
conn.close()
```

这个代码实例是一个简单的SQLite数据库示例，它创建了一个名为“data”的表，并插入了一条数据。然后，它查询了数据库中的所有数据，更新了一条数据，并删除了一条数据。最后，它关闭了数据库连接。
