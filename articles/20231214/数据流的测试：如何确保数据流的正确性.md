# 2.核心概念与联系

在进行数据流的测试之前，我们需要了解一些核心概念。

## 2.1 数据流

数据流是一种由数据组成的流动系统，数据可以在不同的节点之间进行传输和处理。数据流可以是有向的（即数据只能从一个节点流向另一个节点）或无向的（数据可以在多个节点之间流动）。数据流可以是同步的（即数据处理是同时发生的）或异步的（数据处理是异步发生的）。

## 2.2 数据流的正确性

数据流的正确性是指数据流在处理过程中是否符合预期的结果。例如，在一个金融系统中，数据流应该能够正确地处理交易请求、计算交易费用等。如果数据流的处理结果与预期结果不符，则说明数据流的正确性被破坏。

## 2.3 数据流的测试

数据流的测试是一种验证数据流正确性的方法。通过对数据流进行测试，我们可以确保数据流在处理过程中的正确性和可靠性。数据流的测试可以是手动的（即人工对数据流进行测试）或自动的（即通过程序对数据流进行测试）。
