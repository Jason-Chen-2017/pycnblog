# 2.核心概念与联系

在讨论数据库高可用性之前，我们需要了解一些核心概念和联系。这些概念和联系包括：

1. 数据库：数据库是一种用于存储、管理和查询数据的系统。数据库由一组数据组成，这些数据被存储在数据库中的表、视图、存储过程、触发器等结构中。
2. 数据库管理系统（DBMS）：数据库管理系统是一种软件，用于管理数据库。数据库管理系统提供了一组API和接口，用于创建、修改、删除和查询数据库。
3. 数据库高可用性：数据库高可用性是指数据库系统在任何时刻都能提供服务，即使发生故障，也能尽快恢复服务。数据库高可用性的关键是确保数据的安全性、完整性、实时性和可用性。
4. 数据冗余：数据冗余是指在数据库中存储多个相同的数据副本。数据冗余可以提高数据库系统的可用性，但也可能导致数据的不一致和冗余。
5. 数据备份：数据备份是指在数据库中创建多个数据副本，以便在发生故障时恢复数据。数据备份可以提高数据库系统的可用性，但也可能导致数据的不一致和备份。
6. 数据复制：数据复制是指在多个数据库服务器上存储相同的数据副本。数据复制可以提高数据库系统的可用性，但也可能导致数据的不一致和复制。
7. 数据分区：数据分区是指将数据库中的数据划分为多个部分，每个部分存储在不同的数据库服务器上。数据分区可以提高数据库系统的可用性，但也可能导致数据的不一致和分区。
8. 数据分布：数据分布是指将数据库中的数据存储在不同的数据库服务器上，以便在发生故障时恢复数据。数据分布可以提高数据库系统的可用性，但也可能导致数据的不一致和分布。
