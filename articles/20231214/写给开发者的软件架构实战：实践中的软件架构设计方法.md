# 2.核心概念与联系

在进行软件架构设计之前，我们需要了解一些核心概念。这些概念包括组件、组件之间的关系、组件之间的交互方式等。

组件是软件系统的基本构建块，它们可以是代码、数据或者其他资源。组件之间可以通过各种方式进行交互，如通信、调用等。

组件之间的关系是指组件之间的依赖关系。这些关系可以是直接的，也可以是间接的。组件之间的关系可以通过各种方式进行表示，如依赖图、组件图等。

组件之间的交互方式是指组件之间的通信方式。这些交互方式可以是同步的，也可以是异步的。组件之间的交互方式可以通过各种方式进行表示，如消息队列、RPC等。
