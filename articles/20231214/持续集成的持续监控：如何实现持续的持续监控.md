                 

# 1.背景介绍

持续集成（Continuous Integration，CI）是一种软件开发的最佳实践，它要求开发人员在每次提交代码时，自动地将代码集成到共享的代码库中，并立即进行构建和测试。持续集成的目的是提高软件开发的速度、质量和可靠性。

然而，持续集成只是软件开发生命周期中的一部分，它不能解决整个软件开发生命周期中的所有问题。例如，持续集成不能监控软件系统的性能、安全性和可用性等方面。因此，我们需要一种新的方法来实现持续的持续监控。

持续的持续监控（Continuous Monitoring of Continuous Integration，CMC）是一种新的软件开发方法，它可以在持续集成的基础上，进一步实现软件系统的监控和管理。CMC的目的是提高软件开发的效率、质量和可靠性。
