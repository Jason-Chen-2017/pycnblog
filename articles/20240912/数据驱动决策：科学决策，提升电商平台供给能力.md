### 1. 数据库查询优化
**题目：** 如何优化电商平台中的商品查询性能？

**答案：**
电商平台中的商品查询性能优化可以从以下几个方面进行：

#### 1. 使用索引

为经常查询的字段创建索引，如商品名称、分类、价格等。索引可以大大提高查询速度，但过多的索引会影响表结构的修改性能。

```sql
CREATE INDEX idx_product_name ON product (name);
CREATE INDEX idx_product_category ON product (category_id);
CREATE INDEX idx_product_price ON product (price);
```

#### 2. 避免全表扫描

通过使用WHERE子句中的条件，限制查询范围，避免全表扫描。

```sql
SELECT * FROM product WHERE category_id = 1 AND price BETWEEN 100 AND 200;
```

#### 3. 使用连接查询

在需要查询关联表的数据时，使用连接查询代替子查询或临时表，可以提高查询性能。

```sql
SELECT p.id, p.name, c.name AS category FROM product p
JOIN category c ON p.category_id = c.id WHERE c.name = '电子产品';
```

#### 4. 分库分表

当商品数据量非常大时，可以考虑分库分表，将数据分散到多个数据库或表中，降低单个表的压力。

```sql
-- 分库分表策略
CREATE TABLE product_1 (LIKE product);
CREATE TABLE product_2 (LIKE product);
-- 数据分布策略
INSERT INTO product_1 SELECT * FROM product WHERE id % 2 = 0;
INSERT INTO product_2 SELECT * FROM product WHERE id % 2 = 1;
```

#### 5. 缓存策略

对于经常查询的数据，可以使用缓存策略，如Redis缓存，减少数据库访问压力。

```python
import redis

# 连接Redis
r = redis.Redis(host='localhost', port=6379, db=0)

# 缓存商品信息
r.set('product:1', '{"id": 1, "name": "iPhone 13", "price": 7999}')

# 从缓存中获取商品信息
product_info = r.get('product:1')
print(product_info)
```
