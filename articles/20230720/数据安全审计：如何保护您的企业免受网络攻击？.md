# 2.基本概念术语说明
## 2.1 数据安全审计定义
数据安全审计（Data Security Auditing）又称数据安全检查或数据安全监察，是指由第三方机构或个人对特定组织或公司生产、经营和消费的各种类型的数据进行检查、调查、研究、评估，找出违反数据安全规范的行为，并提出合理化建议，通过制定内部标准和安全技术手段，提高数据安全水平，从而实现数据安全保障。

数据安全审计的目的是通过检测、识别、跟踪、记录和评估组织、产品、服务或过程中的安全事件和风险行为，使其不致发生任何危害。数据安全审计包括三个层次：应用级、管理级和最终控制级。应用级审核是最基础的数据安全审计方法。它由组织或单位定期审核产品或服务的安全性、隐私和可用性，确保用户的数据不受未授权访问或泄漏所导致的风险。管理级审核通常通过比较法律、法规、政策和控制框架，确定组织是否具备适当的控制能力和数据安全流程，是否在正确地实施安全措施，以确保组织的资源、利益和数据得到充分保护。最终控制级审核主要是基于大数据的分析，通过识别和预测数据模式和变化，对组织产生影响的个别事件、组织和数据目标进行追踪和监控，并提供具有战略意义的建议。

## 2.2 数据分类
数据分类的目的在于归类不同类型的数据，方便审计人员分析和清理，有效减少误报，提升效率。数据分类一般按照范围分为：

1. 静态数据：指的是可用于指导决策和执行活动的信息，不经常发生变化。例如客户信息、制造订单、销售数据等。
2. 动态数据：指的是用来支撑交易活动的信息，经常发生变化。例如产品库存、库存流动、业务决策等。
3. 会话数据：指的是用户通过网络上传输的数据，不属于前两种数据范畴，其中包括敏感词、垃圾邮件、诈骗电话等。
4. 用户数据：指的是用户自身产生的各种信息，如浏览记录、搜索历史、下载记录、联系方式、位置信息、设备信息等。
5. 系统日志：记录了系统运行过程中出现的所有事件。

## 2.3 数据安全检测方法
数据安全检测方法可分为三种：静态检测、动态检测、机器学习检测。

1. 静态检测：静态检测是指基于规则、策略、法律法规或框架制定的标准和检查点，对数据的结构和完整性进行检查。静态检测包括结构化检测、非结构化检测、面向模式的检测和基于行为的检测。
2. 动态检测：动态检测是指通过分析数据的时间序列、分布和关联关系，判断其变化趋势，并对数据源头和违规行为进行追溯。动态检测包括时序检测、关联检测、异常检测和流量检测。
3. 机器学习检测：机器学习检测利用机器学习算法对数据进行训练和建模，根据模型判断数据是否具有可疑或异常特征。机器学习检测目前较成熟的方法有深度学习、神经网络、支持向量机和随机森林等。

## 2.4 加密技术
加密技术是指在信息传输过程中对数据进行加解密处理，以达到保护数据的作用。常用的加密技术有以下几种：

1. 对称加密：对称加密是指两方都知道使用的密钥，采用同样的算法对消息加密和解密。常用算法有DES、AES等。
2. 非对称加密：非对称加密是指使用两个不同的密钥对数据加密，公开密钥与私有密钥配对，公开密钥用于加密，私有密钥用于解密。常用算法有RSA、ECC等。
3. Hash 函数：Hash函数是一种单向散列函数，把任意长度的数据映射成固定长度的值。常用的Hash函数有MD5、SHA-1、SHA-256等。
4. 数字签名：数字签名是一种公钥加密方案，它允许一方对信息发表声明，另一方验证该声明的真实性。数字签名由消息摘要、签名值和签名者的身份组成。
5. SSL/TLS加密：SSL/TLS协议提供的加密服务包括对称加密和非对称加密，并且还实现了认证和完整性校验功能。
