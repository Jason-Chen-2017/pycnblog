1）网络安全防护的基本原理和技术要素；

3）攻击手段检测和威胁情报搜集的方法和工具；

# 2.核心概念与联系
## 2.1 概念介绍
### 网络安全模型
网络安全模型(NSPM)是一种建立在信息安全领域里的认证、授权、访问控制以及风险管理等概念基础上的一种综合性框架，用于描述和评估计算机和网络系统所遭受的攻击和安全威胁。该模型由五个部分组成：身份识别和鉴别，数据保护，访问控制，数据流动，事件响应，其中身份识别和鉴别部分指出网络系统的用户应该被授予其权限的能力。数据保护部分主要针对敏感数据的加密存储、传输和访问方式等安全相关因素。访问控制部分即管理员对网络资源访问权限的控制，它可以定义哪些用户可访问特定系统功能或数据。数据流动部分描述了网络中不同类型的数据流动的安全问题，包括内部网络之间、外部网络与内部网络之间的流动。事件响应部分则指出发生故障时应采取什么样的补救措施，并定期检查网络系统的安全情况。

### TCP/IP协议
TCP/IP协议（Transmission Control Protocol / Internet Protocol）是互联网协议簇中的一部分，定义了数据如何在网络上传输，以及路由器及通信链路上如何处理数据包。它是Internet最基本的协议之一，它的两个主要版本分别是IPv4和IPv6。

TCP/IP协议中有四层：应用层，表示层，传输层，网络层。每层的任务如下：

- 应用层：应用层负责应用程序间的通信，例如HTTP协议、FTP协议等，向用户提供接口。
- 表示层：表示层实现不同形式的信息编码与压缩，如加密和解密，使得发送端和接收端都能明白对方的意图。
- 传输层：传输层提供处于网络连接中两台计算机之间的数据传输，确保数据完整性，采用可靠的传输协议如TCP、UDP等。
- 网络层：网络层负责将数据包从源地址传送到目的地址，在传输过程中寻找路径，保证数据包到达目标，一般使用IP协议。

### 防火墙
网络防火墙(Firewall)是一种特殊的网络硬件，用于隔离互联网内外不同的网络，基于规则控制，能够监视网络流量，阻断非法数据流入和传出，帮助组织提升网络安全，降低网络风险。防火墙通过检验数据包头部的信息，根据规则决定是否放行进入网络的包，或者丢弃不符合要求的包，限制某些IP地址的访问，阻止病毒的传播。

常见的网络防火墙分为三种：

1）边界防火墙(Border Firewall):边界防火墙通常部署在企业内部的边界网络环境，过滤所有进入和退出企业的网络流量。比如：DMZ（Demilitarized Zone），公共网络，办公室区域等。

2）DMZ防火墙:在一个单独的区域放置专用的防火墙，专门用于隔离内外网络。它的作用就是让内外网络通信互不影响，避免它们之间的互相干扰。

3）云防火墙:云防火墙是基于云计算平台的一个防火墙产品，它利用云服务器的分布式特性实现网络的安全隔离，保障云资源的安全运行。

### VPN
VPN（Virtual Private Network）虚拟专用网，是一种打破网络隔离的技术，通过虚拟化技术将多台物理服务器连接在一起，形成一个逻辑上的专用网络，让不同部门之间的数据传输更加安全、高效。VPN分为服务器端和客户端两种模式，客户端可以安装客户端软件，通过服务器端的VPN服务器与远端VPN服务器建立连接，在本地网络与远程网络之间传输数据，加密数据。

### WAF（Web Application Firewall）
WAF（Web Application Firewall）是网络安全产品中的一种，旨在对web应用进行安全保护，是目前应用最普遍的一种安全解决方案。其主要功能如下：

1）过滤SQL注入、XSS跨站脚本攻击、CSRF跨站请求伪造等攻击。

2）对攻击行为进行持续监控，发现异常行为及误操作，并自动阻止攻击行为。

3）提供针对攻击流量的实时封堵，有效抵御攻击。

4）支持多种功能模块设置，如IP黑名单，cookie过滤，文件上传过滤等，为网站提供额外的安全防护。
