# 2.核心概念与联系
## 文件(File)
文件是一个存储在磁盘上的数据集合。一个文件可以是文本文件或二进制文件。
### 操作方式
文件操作的方式主要分为以下几类：

1.读(Read)文件内容
2.写(Write)新的内容到文件
3.追加(Append)新内容到文件末尾
4.复制文件
5.移动文件
6.重命名/修改文件名
7.删除文件
8.获取文件信息
9.遍历目录文件

以上操作都是基于文件的不同属性和特征的，并且操作不同的文件对象。例如，写操作仅适用于可写文件；删除操作仅适用于可删除文件；重命名操作仅适用于存在的文件。因此，选择合适的方法才能正确完成指定的任务。

## 异常(Exception)
异常是程序运行过程中发生的某种不可预知的错误状态。当程序遇到异常时，会停止执行当前程序流程，跳转至异常处理代码执行。Python提供了异常机制，用来管理程序运行期间发生的错误或异常。
### 异常种类
Python有两种类型的异常：

1.系统异常：系统异常是指由Python解释器或操作系统引起的一般性错误，例如系统调用失败、内存分配失败等。在这种情况下，Python解释器无法处理该错误，只能终止程序执行。

2.用户自定义异常：用户自定义异常是程序员自己定义的异常，以便更加准确地描述程序的运行过程。可以根据需要自定义不同的异常，比如文件打开失败、参数错误等。

### 异常处理
在Python中，异常处理是通过try...except...finally语句实现的。try语句块内的代码可能引发异常，则except语句块会捕获这个异常。如果没有异常抛出，则不执行except语句块。finally语句块无论是否出现异常，都会执行。下面的例子演示了异常处理的基本用法：

```python
try:
    # some code that may raise an exception
except ExceptionType as e:
    # handle the exception
finally:
    # execute this block regardless of whether there was an exception or not
```

在上述示例中，try语句块中包含可能会抛出的异常代码，包括系统异常和用户自定义异常。如果try语句块中的代码引发了一个系统异常，那么就会进入except语句块，将系统异常对象保存到变量e中。如果try语句块中的代码引发了一个用户自定义异常，那么同样也会进入except语句块，只不过这个用户自定义异常的类型是由ExceptionType指定。finally语句块无论如何都会执行，即使没有任何异常抛出，也要执行这个块。
