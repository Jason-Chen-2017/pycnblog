# 1.背景介绍


Rust 是一门非常新的编程语言，它可以提供安全、高效的内存管理、丰富的抽象机制和丰富的类型系统，是构建系统应用程序和网络服务的最佳选择。但是学习它的语法、特性和编程范式会花费相当多的时间。本系列教程将帮助读者快速入门并熟悉 Rust 编程语言的一些主要功能，包括集合（collection）和迭代器（iterator）。
## 为什么需要 Rust 中的集合？
在 C 和 C++ 中，都没有内置的集合类或数据结构。为了解决这一问题，比如实现一个动态数组或者链表的数据结构，通常需要手动去管理内存和指针。随着工程项目的增加，程序中往往会出现很多类似的数据结构和算法，并且这些数据结构和算法之间存在复杂的关系，比如依赖某个节点是否为空等。这些数据结构和算法的组合又常常导致程序难以维护和修改。所以 Rust 提供了一种叫做“集合”（Collection）的抽象概念，用于管理特定类型的元素，其提供了更多的保证，如类型安全、效率、并发性等。
## Rust 中的集合有哪些？
Rust 有四种基本的集合类型：
- Vector: 在 Rust 中，Vector 是堆分配的顺序连续存储空间。优点是随机访问速度快，适合于动态增长的场景。
- Array: 在 Rust 中，Array 也是堆分配的固定大小的存储空间。优点是索引访问速度快，不适合动态增长的场景。
- HashMap: 在 Rust 中，HashMap 是哈希表的实现，能够以 O(1) 的时间复杂度找到 key 的位置。
- BTreeMap: 在 Rust 中，BTreeMap 是红黑树的实现，能够保持 map 中的元素有序。
除了以上四种基本集合之外，还有一些其他的集合类型，如 HashSet、BinaryHeap、LinkedHashSet、BinarySearchTree 等。
## Rust 中的迭代器有什么用？
迭代器（Iterator）是一个对象，它代表了一个序列中的元素。每一次调用 next() 方法，迭代器都会返回下一个元素，直到所有的元素都被迭代完毕。迭代器使得编写基于数据的算法变得十分简单。通过对容器的每个元素进行迭代，我们就可以对容器中的数据进行各种操作，如过滤、排序、聚合等。

## 总结
在 Rust 中，集合是对数据的逻辑封装，它们使得数据的处理和维护更加简单、安全、快速。迭代器则是一个用于遍历集合的工具，它能够简化集合的操作流程。理解并掌握 Rust 中的集合和迭代器，对于开发人员来说，是一项必备技能。
