# 1.背景介绍


## 概述
最大流问题(Max Flow Problem)是网络流（Flow Networks）问题中的一个重要子集，也是电力分配、社会运输、图割等许多复杂问题的基础。

在这个问题中，给定一个二部图G=(V,E)，其中V表示网络节点集合，E表示网络边集合。设c[u,v]表示从结点u到结点v的一条有向边上的容量，流f[u,v](u->v)表示从结点u到结点v的一条有向边所含有的流，流的值是一个非负实数。流网络的每条边上都有一个方向，即一条边(u->v)，其反方向的边也存在于网络中，但容量却可能不同。

最大流问题是要找出一个流f*,使得从源点s到汇点t的任意路径上的流的最小值。流网络可以看成是一个容量限制的网络，从结点u到结点v的一条有向边只能够承载某一特定容量的流，而不超过该容量。如果一条边上的容量为0，则意味着不能通过这一条边，也就是说这条边处于阻塞状态。流网络中的流可以进行增广、减容、分流等操作。

在实际应用中，最大流问题常用于计算网络流的最大可行容量，或确定某个固定割准则下的最大流。最大流问题是一个NP-难度的问题。因此，很多研究者都试图利用近似算法求解这一问题。然而，由于近似算法往往具有较高的时间复杂度，导致在实际应用中仍旧采用贪心或动态规划算法求解这一问题。

## 最大流算法分类及比较
### 1.Ford-Fulkerson方法
Ford-Fulkerson方法是一种贪心算法，它是一种残留网络流算法，可以用来解决最大流问题。它的基本思想是从源点向汇点遍历网络，每当找到一条增广路时，就发送尽可能多的流，直到遇到阻塞边或达到最大流量。Ford-Fulkerson方法是所有残留网络流算法中最简单、最快速的方法，时间复杂度为O(VE^2)。


### 2.Dinic算法
Dinic算法是另外一种流算法，基于拓扑排序进行流的增广。它的基本思想是维护一个队列，其中存放从源点s到汇点t的所有可增广路；每次从队头取出一条增广路，然后检查新加入的边是否满足容量约束条件，如果满足的话就添加到可增广路的队列中。这个过程一直重复，直到所有的可增广路都被处理完或者达到限定的流量上界。Dinic算法的平均时间复杂度为O(VE^2)，但是在实际应用中通常采用更快的实现方式，例如采用队列优化的实现。


### 3.Push-Relabel方法
Push-Relabel方法是另一种流算法，它的基本思想是在每一步迭代过程中，先压满所有可压入的边，再重标记可标记的边。在压边阶段，将流从边u->v传递给边u->w，如果新的残留容量小于等于当前残留容量，则更新残留容量；如果新的残留容量大于当前残留容量，则更新父亲结点指针。在重标记阶段，遍历残留网络的顶点，对于每个顶点，检查所有出边，以找到可标记的边。如果新的边的容量大于当前容量，则更新容量和父亲结点指针。Push-Relabel算法的平均时间复杂度为O(EV^2),但是比Dinic算法稍慢一些。
