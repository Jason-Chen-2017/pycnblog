### 4.2 公式推导过程

假设我们有n个用户，m个知识付费课程，k个健身服务，用户i对知识付费课程j的兴趣为 \(I_{ij}\)，用户i对健身服务l的需求为 \(D_{il}\)。我们可以使用以下公式来描述用户画像模型、推荐算法模型和效果评估模型：

1. **用户画像模型**：

   \(I_{ij} = \sum_{m=1}^{m} w_{ij} \cdot C_{mj}\) （兴趣得分）

   \(D_{il} = \sum_{k=1}^{k} w_{il} \cdot S_{kl}\) （需求得分）

   其中，\(w_{ij}\) 为用户i对知识付费课程j的权重，\(C_{mj}\) 为知识付费课程j的属性特征，\(w_{il}\) 为用户i对健身服务l的权重，\(S_{kl}\) 为健身服务l的属性特征。

2. **推荐算法模型**：

   \(R_{ij} = \sum_{i=1}^{n} \sum_{j=1}^{m} I_{ij} \cdot R_{ji}\) （知识付费课程推荐得分）

   \(R_{il} = \sum_{i=1}^{n} \sum_{l=1}^{k} D_{il} \cdot R_{li}\) （健身服务推荐得分）

   其中，\(R_{ji}\) 为用户i对知识付费课程j的评分，\(R_{li}\) 为用户i对健身服务l的评分。

3. **效果评估模型**：

   \(S_{ui} = \frac{\sum_{j=1}^{m} R_{ij} \cdot P_{uj}}{\sum_{j=1}^{m} R_{ij}}\) （知识付费课程满意度得分）

   \(S_{ul} = \frac{\sum_{l=1}^{k} R_{il} \cdot P_{ul}}{\sum_{l=1}^{k} R_{il}}\) （健身服务满意度得分）

   其中，\(P_{uj}\) 为用户i购买知识付费课程j的概率，\(P_{ul}\) 为用户i购买健身服务l的概率。

# 用户画像构建
user_interest = {
    'health_diet': 0.6,
    'yoga': 0.3,
    'running': 0.1

    'fat_loss': 0.5,
    'body_shaping': 0.5
}

# 推荐算法
def recommend_courses(user_interest, user_health):
    course_recommendations = {}
    for course, interest in user_interest.items():
        course_recommendations[course] = interest
    for service, health in user_health.items():
        course_recommendations[service] = health
    return course_recommendations

# 效果评估
def evaluate_recommendation(recommendation, user_rating):
    satisfaction_score = {}
    for course, rating in user_rating.items():
        satisfaction_score[course] = rating
    return satisfaction_score

# 示例数据
user_interest = {'health_diet': 0.6, 'yoga': 0.3, 'running': 0.1}

satisfaction_score = evaluate_recommendation(course_recommendations, user_rating)

   - `evaluate_recommendation` 函数根据用户对推荐课程和服务的评分（user_rating），计算满意度得分（satisfaction_score）。
   - 通过遍历推荐列表和用户评分，计算每个课程的满意度得分，并存储在字典中。

4. **示例数据**：
