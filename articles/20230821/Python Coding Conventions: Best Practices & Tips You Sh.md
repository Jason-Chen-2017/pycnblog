# 2.Python编程语言特性
## 2.1 使用文档字符串
要让你的代码更加清晰易懂，就需要用描述性的注释来解释每个函数的作用和用法。Python使用`docstrings`，它是Python中被广泛使用的一种方式来提供文档注释。你可以通过在开头的三引号之后编写文档注释来实现这个目的。例如：

```python
def my_function(arg):
    """This function takes an argument and returns its value."""
    return arg
```

文档注释应该足够详细，包括函数的作用、输入参数、输出值、返回类型等信息。这样做有以下几个好处：

1. 可以让其他程序员快速理解你的代码。他们不需要看源代码来理解你的意图，只需查看文档注释即可。
2. 如果你重新构想或修改了函数，就可以更新文档注释而不会影响到其他用户的代码。
3. 有助于代码重构。当你想要对一个函数进行改进时，只需编辑文档注释中的相关内容即可。

另外，如果一个类比较复杂，也可以给它的各个方法添加文档字符串。

## 2.2 使用空行分隔逻辑块
为了方便管理代码，建议在函数定义、循环、条件语句之前都插入空行。这样做可以让代码更加整洁美观。举例如下：

```python
if age < 18:
    print("You are still a minor.")

    if is_married:
        print("But you have a wife...")
    
    else:
        print("And your parents must be busy.")


print("Goodbye!")
```

上面的代码没有按照最佳实践的方式组织代码结构，尤其是在三个打印语句中间。为了避免混乱，最好像前面所示那样，每条语句之间插入一个空行。

## 2.3 使用缩进区分嵌套逻辑结构
当你嵌套多个逻辑结构（如条件语句、循环）时，建议用4个空格来表示缩进层次。这样做可以让代码更加整齐划一，减少各种语法错误导致的编译错误。例如：

```python
if x > y:
    for i in range(x+1):
        if i % 2 == 0:
            continue
        print(i)

    while z >= y/z:
        print(z*y)
        
        if x <= -1 or y >= 1:
            break

        z /= 2
        
else:
    print("Error! X cannot be greater than Y")
    
finally:
    print("All done!")
```

上面的代码虽然有点啰嗦，但是利用了缩进和空行，并没有导致太多的混乱。

## 2.4 使用适当的变量名
变量名应该具有描述性，且尽量不要与关键字、内置函数命名相同。这里有一个命名规则建议：

1. 全小写单词。
2. 用下划线连接多个单词。
3. 名称尽量短，但不要过长。
4. 当变量名代表的是某个对象的属性时，后缀应该用`_`作为间隔符。例如，一个学生对象的名字可以使用`student_name`。

## 2.5 使用恰当的数据类型
选择正确的数据类型可以让你的代码运行得更快，降低出错的可能性。你应当优先选择内置数据类型的子集而不是像Java那样使用Object作为所有数据类型的父类。比如，如果你的代码仅涉及整数运算，则应该使用整数类型；如果你需要处理浮点型数字，则应该使用浮点型；如果你需要处理字符或字符串，则应该使用字符串类型；如果你的代码需要处理布尔值，则应该使用布尔类型。
