# 2.基本概念和术语说明

## 数据中心（Data Center）

数据中心（Datacenter）通常指的是通过计算机网络连接、聚合、存储、处理数据的专用区域，它可以为各种组织机构、企事业单位提供包括电力、通讯、金融、商业等各种应用服务。

## 数据中心架构设计

数据中心架构设计是指对数据中心机房进行规划、布局、布线和架空，利用多种技术手段、工具和方法完成对数据中心整体结构的设计、优化和功能建设，确保数据中心能够有效支持业务运营和运行，满足数据安全、可用性、可伸缩性、可靠性要求。

数据中心架构设计有以下几个主要步骤：

1. 选择适宜的硬件环境：确定服务器的配置、数量和性能，确定配套的网络设备及其数量、规格和带宽。

2. 选择和规划存储结构：选择对企业最有利的数据存储结构。如冷热数据分离、基于SAN、NAS、云盘等。

3. 规划电源、冷却系统和空调系统：根据数据中心的需要设计冷却系统、电源系统及空调系统，确保它们具有足够的能耗。

4. 设计数据中心网络体系：建立数据中心网络，包括骨干网、边缘网和第三方网络。

5. 设计安全措施：建立数据中心内部的访问控制策略，确保数据的真实性、完整性和安全性。

6. 制订服务水平协议：制订数据中心服务水平协议，包括SLA（Service Level Agreement）、成本核算规则、费用预测、容灾备份和恢复、网络故障演练规则等。

## ITIL

ITIL（Information Technology Infrastructure Library）是由一群具有信息技术行业经验的专业人员开发的技术指南，旨在定义和规范信息技术领域的工作方式，为组织制定更好的管理和运营方案提供指导。

ITIL拥有六大核心原则：

1. 定义明确：该框架明确定义了信息技术领域的工作流程、角色和职责，并提供了一致的标准和衡量标准。

2. 分层管理：该框架的分层管理模式允许信息技术团队按照不同的层次，对技术服务提供商、客户和用户进行管理。

3. 专业知识：该框架强调专业化、技能匹配，并且确保所有参与者都有足够的知识和经验来实现技术服务。

4. 服务透明性：该框架提倡信息技术服务的公开、透明和透明度，让客户能够知晓服务提供商所提供的技术服务的相关信息和指标。

5. 信息生命周期管理：该框架关注技术服务的全生命周期，包括计划、设计、交付、使用、支持、更新和退出等阶段，并制定相应的流程和工具。

6. 客观诊断：该框架注重客观性，所有的参与者都应具备独立判断和判断能力，而不受主观偏见的干扰。

## PMBOK®

PMBOK是美国认可的项目管理和过程控制原理的称谓，描述了项目管理中使用的一系列模型、理论、方法、工具和过程。

1. 项目管理（Project Management）：这一理论基于专家团队的智慧和知识，通过将项目中涉及到的众多的人员、资源、条件和限制综合考虑，识别出问题、评估风险、制定计划并最终完成项目。

2. 过程控制（Process Control）：这一理论主要用于管理和控制企业的生产、流通、物流、人力、设备和材料等过程，确保企业在不同环节上遵守共同的标准和规范，以最大限度地提高企业的效率。

3. 物理法（Physical Science）：这一理论对化学工程、建筑工程、交通工程、海洋工程、水产工程和农业工程等领域的建设过程进行了深刻的理解和应用。

4. 系统构造（Systems Engineering）：这一理斯理论着眼于企业的总体运行情况，通过提炼系统的功能、结构和行为特征，识别并改善系统的运行效率，为企业创造效益。

5. 管理信息系统（Management Information Systems）：这一理论系统地阐述了管理信息系统的组成要素、运作原理、工作流程、工具、方法等，有助于提升管理人员的信息处理、分析和决策能力。
