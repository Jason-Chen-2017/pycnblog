# 1.简介
  

推荐系统（Recommendation System）是互联网领域的一个热门话题。相对于传统的搜索引擎，推荐系统可以更精准地满足用户的个性化需求。推荐系统背后的核心理念是“用户兴趣相似”。根据用户过去行为，给出其类似的新推荐物品或服务。同时，推荐系统还有一个重要的功能就是过滤垃圾信息、节省流量等。因此，推荐系统也被称为信息过滤器。由于其复杂的计算和存储过程，一些公司和组织已经开始在自己的网站上推广推荐系统。例如，亚马逊、苹果、facebook等都将推荐系统嵌入到其产品中。另外，还有一些公司和组织基于推荐系统进行营销活动、促进销售等。这些都是当下推荐系统发展的一个热点方向。
# 2.Collaborative Filtering(协同过滤)
协同过滤是一种基于用户与用户之间的交互数据，利用该数据预测目标用户对目标物品的兴趣并推荐合适的物品给目标用户的方法。其特点是简单、快速、效果好。它不需要考虑物品的详细信息，只需要对用户行为习惯进行分析即可。
最早提出的协同过滤方法叫做User-based CF (基于用户的协同过滤)，它的基本思路是根据用户的行为历史，将相似用户的兴趣比较起来，再找出他们感兴趣的物品作为推荐结果。如今，协同过滤的相关算法有很多种。以下简要介绍一下基于用户的协同过滤算法。

1. Item-based collaborative filtering （基于物品的协同过滤）
基于物品的协同过滤通过分析用户对物品的偏好，来预测用户对其他物品的偏好。
该算法先对所有物品建立倒排索引表，然后基于用户对物品的评分矩阵，计算两者之间的余弦相似度，再找出用户对某项物品的评分高且与其欧几里得距离最近的其他物品，然后按照它们的相似度排序，选出前N个最相关的物品作为推荐结果。这种方法优点是能够快速完成推荐任务，缺点是无法考虑物品属性的影响，因此推荐效果不一定很好。

2. Matrix Factorization based approach (矩阵分解法)
矩阵分解法属于基于矩阵的协同过滤算法。该方法将用户与物品之间的评分矩阵进行分解，得到隐含的特征向量，再使用这些特征向量进行推荐。
具体来说，首先对用户-物品矩阵进行奇异值分解，求得其中的两个低维张量，其中一个张量表示物品特征，另一个张量表示用户特征。通过最小化用户特征矩阵与物品特征矩阵之积与原始矩阵之间的差距，得到的即为用户的潜在兴趣，从而产生推荐结果。该方法的优点是考虑了物品属性的影响，缺点是计算量较大。

3. SVD based approach （SVD算法）
SVD（Singular Value Decomposition）算法是一种矩阵分解的方法，可以帮助找到两个矩阵，使得原始矩阵与这两个矩阵之积尽可能的接近。SVD算法可以用于基于用户的协同过滤和基于物品的协同过滤。一般情况下，SVD算法比其他算法都快很多。SVD可以帮助发现潜在因子（latent factors），这些潜在因子可以通过物品和用户的交互数据来预测其他用户对其他物品的兴趣。
