# 1.简介
  


数据安全一直是企业IT部门最关心的重点之一。近年来，随着互联网、物联网等新型数字化技术的发展，数据安全也越来越成为一个新的关注焦点。数据安全意味着信息在不被授权访问和篡改的情况下能够得到足够保护。企业对数据的安全威胁日益增长，数据泄露、数据泄露带来的损失已经成为了一个社会突发性事件。

为了应对数据安全风险，企业需要建立数据安全监控体系，采取预防性的措施，提高运营效率并降低风险。《7. 企业数据泄露隐患分析及应对措施》将从数据泄露的基本概念、分类、潜在危害、检测方式、预防手段和应对措施等方面详细阐述数据泄露的基本原理和防御方法。

2. 数据泄露定义

数据泄露，是指数据经由网络传输或存储到目标计算机中后被不法人员窃取、破坏、修改等所造成的严重后果。数据泄露可以是结构性的数据泄露（如人员信息被泄露）或者非结构性的数据泄露（如电子邮件和文件被盗）。

3. 数据泄露分类

按数据泄露的性质，分为结构性数据泄露、非结构性数据泄露、业务数据泄露。结构性数据泄露发生在组织机构内部，主要涉及公司的一些基础数据、关键财产等。这些数据如果被非法获取、使用，会导致严重的经济损失、人身伤亡甚至恶劣的社会影响。而非结构性数据泄露则与公司业务无关，比如个人信息的泄露、工作日志的泄露、系统漏洞等。业务数据泄露一般发生在金融机构、政府部门以及其他具有商业价值的计算机上，例如存款卡信息、信用卡凭证等，这些数据也同样属于非结构性数据泄露。

结构性数据泄露的基本过程如下：

1. 发现数据泄露：数据泄露通常首先发生在第三方可靠源，如银行数据库、电子邮箱等，任何由于员工错误、偶然操作等原因未能妥善保管数据而出现的数据泄露，都可能成为发生数据泄露的元凶。因此，企业应当定期检查所有可靠源的安全性，以找出潜在的数据泄露风险。

2. 收集数据：发现数据泄露后，企业要尽快收集相关数据进行全面调查，包括原始数据、备份数据、加密密钥、访问日志、操作记录等。一定程度上，还可以根据源头可靠程度对数据的收集范围做限制，避免将不相关数据一起收集。

3. 检测数据泄露：检查完毕后，企业应当按照“应急响应”“综合整治”“事故管理”“后续跟踪”的流程对数据泄露进行全面清理，删除所有明文数据、密钥、日志、文件等，同时收集有关人员的身份证明、工牌号等必要证据。之后，应积极开展外部清洗，并与安全部门保持密切联系，以便实施进一步行动。

4. 行动计划：通过检查、清除等一系列行为，企业将相应数据信息抹掉，但仍然留下了足够的信息，如原始数据、日志、操作记录等，用于后续追踪，以保证数据的完整性和真实性。企业应当制定行动计划，详细列出清除过程中的各项措施，以及对受损数据的保护措施。

5. 撤销访问权限：清除完成后，企业应立即撤销其对数据的所有访问权限，使得被泄露的数据再无人能够访问，保障数据的安全。同时，企业应充分利用各种渠道向用户发送通知、警示信息，提醒受损数据的安全隐患。

6. 维护合规标准：数据泄露属于比较严重的安全事故，一定程度上可能会影响公司业务，因而企业需遵守相关政策法律法规，及时修订合规标准，落实好数据保护责任，以防止发生类似事故。

非结构性数据泄露主要发生在企业业务中的数据泄露。具体过程如下：

1. 概念定义：对于非结构性数据泄露来说，主要是指公司业务上重要的数据没有做好备份和安全保护，导致它们被非法获取、篡改甚至毫无痕迹的情况。此类数据包括：人员信息、重要文档、软件漏洞等。

2. 检测目标：企业应当首先对有可能存在泄露风险的数据、信息进行评估，确认其是否属于重要信息，并确立清除它的优先级。

3. 收集数据：企业应积极收集可能含有敏感信息的数据，如业务数据库、服务器上的关键日志文件、程序文件等。

4. 识别泄露对象：确定了存在泄露风险的数据后，企业应识别受影响的人员，并要求其配合提供身份证明、工牌号等个人信息。

5. 处理方案：企业应该制定清除、保护、保障的方法，确保数据安全，包括设定核查问卷、数据备份等。同时，企业应该向受影响的个人、部门和个人提交保密协议。

6. 执行预防措施：经过多次检测和收集后，企业才能判定其潜在的安全风险。因此，企业应当注意使用合适的安全措施，如加密、VPN、访问控制等，有效防范各种攻击和威胁。

7. 提升应对能力：除了前文提到的安全措施外，企业应当密切关注自身业务的安全风险，做到定期检测、报告、预防、处置，提升内部安全防范能力。此外，还应当关注云计算平台、移动应用安全、网络安全等方面的防护措施。
