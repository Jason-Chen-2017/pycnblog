# 2.数组和链表的区别
数组（Array）和链表（Linked List）都是非常常用的数据结构。两者之间的主要区别如下：

- 存储方式不同。数组是定长数据结构，一旦分配空间后，元素数量不能改变；而链表则是动态数据结构，元素的数量可以随时增减。

- 查找方式不同。数组支持按下标随机访问，查找速度快；链表只能顺序访问，需要遍历整个链表才能找到指定元素。

- 插入删除效率不同。数组可以在末尾添加新元素，时间复杂度为 O(1)，但插入和删除操作可能导致数据移动和内存复制，效率低；链表可以在任意位置插入或删除元素，效率高，但需要花费更多的时间。

- 使用场景不同。数组适用于随机查询，频繁修改；链表适用于数据动态增加和删除，读取少量元素。

总结一下，数组和链表都有自己擅长的地方，平时要多用数组来处理静态数据集；而链表适合用来处理动态变化的数据，比如插入、删除操作频繁的情况。
# 3.数组
## 3.1 定义
数组（Array），也称顺序表，是一种线性存储结构，它用一组连续的内存空间，来存储相同类型或者不同类型的元素的集合。数组中的每个元素可以通过索引来进行访问，索引一般是一个非负整数。

举个例子：假设有一组整数：[1, 2, 3, 4, 5]，我们就可以把这些整数存储到一个数组中，每一项对应数组的一个位置。索引为0的位置保存的是整数1，索引为1的位置保存的是整数2，依次类推。


数组具有以下几个特点：

1. 有序性：数组中的元素按照一定顺序排列，使得同一个索引处的元素可以快速地被找到；
2. 大小固定：数组中的元素个数是固定的，一旦创建完成之后，它的大小就不会再变了；
3. 查询效率高：通过索引直接获取元素，时间复杂度为 O(1)；
4. 占用空间小：数组只需要分配一整块连续的内存，不需要额外的开销，所以比较节省空间。

## 3.2 操作
### 3.2.1 创建数组
创建一个长度为 n 的整数数组，其中 n 表示数组的容量：

```python
arr = [None] * n
```

初始化数组时，可以使用 for 循环逐个赋值的方式：

```python
n = int(input("请输入数组的长度："))
arr = []
for i in range(n):
    arr.append(int(input())) # append() 方法向数组的末尾添加新的元素
```

或者可以使用列表解析的方式：

```python
arr = [int(x) for x in input().split()]
```

### 3.2.2 获取元素值
通过索引获取数组中的元素的值：

```python
print(arr[i])
```

例如：

```python
>>> print(arr[0])
1
>>> print(arr[1])
2
```

### 3.2.3 修改元素值
通过索引修改数组中的元素的值：

```python
arr[i] = value
```

例如：

```python
>>> arr[0] = 10
>>> print(arr[0])
10
```

### 3.2.4 删除元素
删除数组中的某个元素，可以通过将待删除的元素标记为空或其他特殊值，然后重新调整数组中元素的位置来实现。这种方法很有效，但是为了维持数组中的每个元素的连续性，引入了大量额外的操作。最简单的方法就是利用切片（slice）操作，只保留数组中除了待删除的元素之外的所有元素。

```python
del arr[i]
```

例如：

```python
>>> del arr[1]
>>> print(arr)
[1, 3, 4, 5]
```

### 3.2.5 插入元素
对于插入元素，可以通过使用切片表达式来实现，比如 `arr[:j]` 表示切片前 j 个元素，`arr[k+1:]` 表示从第 k + 1 个元素开始切片到最后一个元素，然后再拼接一个新元素即可：

```python
arr = arr[:j] + [value] + arr[j:]
```

例如：

```python
>>> arr = [1, 2, 3, 4]
>>> arr = arr[:2] + [5] + arr[2:]
>>> print(arr)
[1, 5, 2, 3, 4]
```

### 3.2.6 求数组的最大值、最小值和平均值
利用 Python 中的内置函数 max(), min(), sum() 可以很容易地求出数组的最大值、最小值和平均值。

```python
max_val = max(arr)
min_val = min(arr)
avg_val = sum(arr) / len(arr)
```

例如：

```python
>>> arr = [3, -2, 5, 1, -3, 4]
>>> max_val = max(arr)
>>> min_val = min(arr)
>>> avg_val = sum(arr) / len(arr)
>>> print("最大值为", max_val)
最大值为 5
>>> print("最小值为", min_val)
最小值为 -3
>>> print("平均值为", avg_val)
平均值为 0.5555555555555556
```

## 3.3 时间复杂度分析
### 3.3.1 查找元素
查找数组中某个元素的最坏时间复杂度是 O(n)，因为数组中的元素位置和索引一一对应，数组的第一个元素的索引是 0，最后一个元素的索引是 n - 1。如果需要查找第 k 小的元素，那么需要先将数组排序，时间复杂度是 O(n log n)。

### 3.3.2 插入元素
向数组中插入一个元素的最坏时间复杂度是 O(n)，因为所有后面的元素都要向右移动一个位置，导致数组中的元素位置和索引不连续。

### 3.3.3 删除元素
删除数组中某个元素的最坏时间复杂度是 O(n)，因为所有后面的元素都要向左移动一个位置，导致数组中的元素位置和索引不连续。

### 3.3.4 合并两个数组
合并两个长度分别为 m 和 n 的数组的时间复杂度是 O(m + n)，因为要将两个数组中的元素依次放入到结果数组中，时间复杂度取决于合并过程中的元素数量。
