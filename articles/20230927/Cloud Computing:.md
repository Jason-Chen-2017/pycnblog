# 2.云计算概念、术语与基本要素：
## 2.1 云计算概述
云计算是一种新的网络计算模型，它利用计算机网络将中心数据中心和机房设备扩展到互联网上，并通过互联网向最终用户提供服务。
## 2.2 云计算概念
### 2.2.1 定义
云计算是一种新的网络计算模型，它利用计算机网络将中心数据中心和机房设备扩展到互联网上，并通过互联网向最终用户提供服务。目前，云计算主要由公共云、私有云和混合云三种类型。
- 公共云：公共云提供商托管在第三方云服务器供应商的服务器上，运行客户自己的应用程序，这种计算环境由第三方运营，提供给不同客户使用。公共云主要服务包括网络、存储、计算、数据库、安全、机器学习等领域。
- 私有云：私有云则指在自身的数据中心内，采用自己的硬件进行数据的处理、存储和计算，云服务供应商只对外提供租用的服务，且对数据进行安全保护。私有云可以根据客户的需求进行定制化开发。
- 混合云：混合云结合了公共云和私有云的优点，同时也兼顾其特有的优势。将公共云和私有云两者结合起来，构建更大的计算环境，实现公共云和私有云之间的资源共享和整合。混合云能够支持公共云和私有云之间的互操作性，并且还可以在公共云上运行一些轻量级的任务。

### 2.2.2 服务与特性
#### 2.2.2.1 可用性
云计算的主要优势之一就是高可用性。无论是在公有云、私有云或混合云中，都可以确保用户的服务至少99.9%的时间可用。它意味着系统的组件可以不间断地工作、运行和提供服务。
#### 2.2.2.2 弹性
云计算可以根据需要快速调整集群规模，满足用户的个性化需求。随时增加或减少计算资源，以满足用户的业务增长和减少所需的成本。
#### 2.2.2.3 按需计费
云计算提供了按需计费的方式。客户只需要付费用量，而不需要预先支付任何费用，这样就可以节省大量的成本。
#### 2.2.2.4 按使用付费
由于云计算按使用付费，因此，无论何时何地都可以使用计算资源。降低了物理资源、服务器运维和管理的开销，提升了效率。
#### 2.2.2.5 按容量付费
对于公有云来说，它按容量付费，即用户按使用的计算容量收费。这种方式具有很好的服务质量保证，并且可以让公共云提供商提供超额的能力。
#### 2.2.2.6 服务级别协议（SLA）
云计算的主要特征之一就是SLA（Service Level Agreement）。它是云计算的重要服务保障机制，它确保了用户得到可靠、可信赖的服务。
## 2.3 云计算术语
### 2.3.1 服务器虚拟化
云计算的第一个关键技术是服务器虚拟化。服务器虚拟化允许多个物理服务器组成一个逻辑服务器，从而将计算资源抽象化，便于云计算平台分配资源，并保证性能，同时可以提供资源的按需扩充。
### 2.3.2 微服务
云计算的一个重要理念就是微服务。微服务是一个单一功能单元，它能独立运行和测试，拥有自己的生命周期和开发人员，这些特点使得微服务成为一个架构风格。微服务架构将复杂的应用程序拆分成松耦合的服务，每个服务负责单一的功能，并通过API进行通信。
### 2.3.3 边缘计算
云计算的一个重要方向就是边缘计算。边缘计算就是把数据中心内部的数据和计算转移到附近的一小块区域，让那里的终端设备能够像云端一样访问数据，并对数据进行实时的分析。这种方式可以加快信息的传输速度和响应时间，降低通信链路的成本，提升计算性能。
### 2.3.4 大数据
云计算的一个关键用途就是大数据。云计算平台可以将大数据存储、处理、分析和展示。它支持海量的数据采集、计算和分析，可以帮助企业发现价值所在，并找到最佳的决策点。
### 2.3.5 分布式系统
云计算的一个关键技术就是分布式系统。分布式系统是指各个节点之间存在高度连接性的计算机系统。云计算中的分布式系统通常由多个服务器构成，这使得它们相比于中心化的本地数据中心具有更多的优势。
### 2.3.6 RESTful API
RESTful API是基于HTTP协议的WEB服务接口标准，它是Web服务的主要模式。云计算中广泛使用RESTful API，它支持跨平台、跨语言的应用程序开发，提高了云计算的可移植性和可复用性。
### 2.3.7 容器技术
容器技术是一个轻量级、可移植、自包含的软件打包技术。云计算中使用容器技术来部署应用程序。容器封装了一个完整的软件环境，包括代码、运行时、依赖库、配置等文件，可以让代码能够在各种不同的环境下运行。
