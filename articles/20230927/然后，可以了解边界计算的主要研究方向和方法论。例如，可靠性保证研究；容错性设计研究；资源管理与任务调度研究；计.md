# 1.简介
  

边界计算是一种新的计算机计算模式。它利用分布式计算技术和专用硬件平台部署在物理边缘、虚拟机和容器之上的应用服务。目标是将计算密集型服务与其所处位置的物理环境隔离开来，从而提高计算效率，降低成本，实现资源共享和节约资源，有效保障用户体验。随着边界计算的不断推进，越来越多的公司开始逐渐尝试将边缘计算应用到自己的业务中。如今，许多知名互联网企业都在部署边界计算平台作为核心业务。但也有很多不少公司仍然坚持着传统的云计算模式。因此，边界计算的落地方式还有待进一步探索，并结合自身需求制定更加合适的方案。

目前国内外已经有相当多的研究和实践都表明边界计算有潜力成为未来的计算机计算模式。特别是在超算中心和存储、网络等领域，边界计算平台已被广泛应用。但是，对于如何进行边界计算平台建设、部署、运行、维护、运维等方面，依旧存在一些缺憾。因此，边界计算的研究和开发还需要做更多的工作。

在本次分享中，我将介绍边界计算的主要研究方向和方法论。并以《边界计算的研究方向》及《边界计算的方法论》两个部分对边界计算进行介绍。希望能够给读者提供一个更全面的认识，帮助他们理解边界计算的研究成果和应用。

# 2.边界计算的研究方向
首先，我们来看一下边界计算的研究方向。边界计算的主要研究方向包括如下四个方面：

1. 可靠性保证
针对计算资源受损或失效导致的业务失败，可靠性保证研究就是要研判其发展方向和策略。可靠性保证的目的在于确保服务质量和可用性，防止计算资源出现故障时，对业务造成影响。可靠性保证的研究有以下几个方向：

- 基础设施可靠性保证
即使基础设施发生故障，应用仍然可以正常运行。这包括物理机房故障，服务器故障，交换机故障，路由器故障，存储设备故障等。因此，研究人员需要提出一套完整的解决方案，通过自动化手段，保证基础设施的可靠性。

- 服务质量保证
为确保计算资源的服务质量，需要深入了解计算资源的软硬件性能指标。因此，需要研究各类计算任务的处理时间，内存消耗，网络带宽等指标，以便制订相应的服务水平协议（SLA）。同时，也要考虑计算资源的使用规模，如何进行负载均衡，以满足业务的预期。

- 用户体验保证
边界计算平台还需要考虑用户的体验，尤其是响应速度，稳定性和可靠性。因此，研究人员需要开发新的算法和工具，以提升用户体验。此外，还应关注用户的隐私安全，以及如何避免数据的泄露和恶意攻击。

2. 容错性设计
为了应对计算资源的不可靠性，容错性设计是非常重要的。容错性设计旨在在计算过程中采用措施，使得计算任务不至于因资源失效而失败。容错性设计的目的是保证计算任务可以在某些情况下完成，在某些情况下不能完成，但仍然可以继续运行，并且达到最高的可用性水平。在边界计算中，容错性设计还需特别注意资源的可用性，以避免资源不足的问题。

容错性设计的研究分为以下三个阶段：

- 前期容错设计：首先要定义边界计算平台的范围，确定关键任务的分类，评估计算资源的能力和需求。然后根据计算资源的可用性要求设计平台架构，配置相应的资源分配机制。再制定相应的容错措施，以保证关键任务的执行。

- 中期容错设计：在前期容错设计之后，应该围绕关键任务的生命周期展开研究。研究人员应该研究在计算任务生命周期中的各种异常情况，包括资源不足，任务失败，通信故障等。根据这些异常情况的发生频率，周期和影响程度，确定容错的优先级，制订相应的容错策略。

- 后期容错设计：在中期容错设计之后，容错能力的提升还有待进一步努力。这包括扩大容错范围，改善任务调度机制，减少任务的延迟和通信开销，增加冗余机制等。

3. 资源管理与任务调度
边界计算平台部署到物理边缘上之后，资源就成为了最大的瓶颈。因此，资源管理也是边界计算的一个重要课题。资源管理的目标是充分利用计算资源，避免资源的浪费。资源管理的研究领域包括两方面：

- 资源调度与分配：资源调度就是决定哪些任务需要什么资源运行，资源分配则是决定如何分配资源。资源调度涉及到多个层面，包括物理资源的分配，任务的调度，多任务间的协同，以及软硬件的整合。

- 资源感知与动态调整：资源管理的另一个重要目标就是动态调整计算任务的执行策略，满足用户的需要。资源感知研究人员可以采取主动测量的方式获取计算资源的实际状态，并根据资源的可用性和利用率进行调整。

4. 计算系统安全
边界计算平台由于部署在边缘，因此需要考虑边界计算平台的安全问题。计算系统安全的研究领域包括以下几方面：

- 数据安全：边界计算平台的数据安全一直是个难题。因为数据可能会存储在本地服务器，同时还要考虑边缘数据中心的安全。因此，需要研究多种数据安全机制，如加密传输，数据过滤，访问控制，审计等，以保证数据安全。

- 计算系统安全：计算系统安全涉及整个计算系统的安全机制，包括操作系统安全，中间件安全，应用服务安全等。因此，需要深入研究计算系统的安全问题。如黑客攻击，木马病毒，恶意程序的感染等。

5. 数据中心网络优化
数据中心网络是边界计算中必不可少的一环。数据中心网络的优化能够显著地提高服务的吞吐率，降低延迟，提升数据中心的整体性能。因此，数据中心网络的优化研究很值得深入研究。数据中心网络优化的研究领域包括如下方面：

- 分布式网络：分布式网络架构下的数据中心网络应如何设计？分布式网络架构下，不同机柜之间可能需要通信，怎么样才能降低通信成本？

- 光纤网络：光纤网络的特点和优势有哪些？光纤网络又该如何部署在数据中心网络中？光纤网络需要考虑哪些问题？

- 路由优化：数据中心的路由器需要优化吗？如何更好地利用路由空间，提升网络性能？路由器的各项参数该如何设置？
