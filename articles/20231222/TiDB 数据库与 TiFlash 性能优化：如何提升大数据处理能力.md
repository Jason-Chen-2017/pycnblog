## 2.核心概念与联系

在本节中，我们将介绍 TiDB 数据库和 TiFlash 的核心概念，以及它们之间的联系。

### 2.1 TiDB 数据库核心概念

TiDB 数据库的核心概念包括：

- **分布式键值存储引擎（TiKV）**：TiDB 数据库使用 TiKV 作为底层存储引擎。TiKV 是一个分布式键值存储引擎，它可以在多个节点上运行，以实现高性能和高可用性。
- **SQL 协议**：TiDB 数据库使用 SQL 协议与客户端进行通信。客户端可以通过 SQL 协议向 TiDB 数据库发送查询请求，并接收结果。
- **分区**：TiDB 数据库使用分区来实现数据的水平分片。分区可以让数据在多个节点上分布，从而实现高性能和高可用性。
- **事务**：TiDB 数据库支持 ACID 事务。这意味着 TiDB 数据库可以确保事务的原子性、一致性、隔离性和持久性。

### 2.2 TiFlash 核心概念

TiFlash 的核心概念包括：

- **列式存储**：TiFlash 是一种列式存储引擎。列式存储可以有效地存储和处理大量的结构化数据。
- **高性能 OLAP 查询**：TiFlash 可以与 TiDB 数据库集成，以实现高性能的 OLAP 查询。这意味着 TiFlash 可以快速地处理大量的数据，以生成有用的报告和分析。
- **压缩**：TiFlash 使用压缩技术来减少存储空间。这意味着 TiFlash 可以在同样的存储空间中存储更多的数据。
- **分区**：TiFlash 使用分区来实现数据的水平分片。分区可以让数据在多个节点上分布，从而实现高性能和高可用性。

### 2.3 TiDB 数据库和 TiFlash 的联系

TiDB 数据库和 TiFlash 之间的联系如下：

- **集成**：TiDB 数据库和 TiFlash 可以集成在同一个系统中，以实现高性能的 OLAP 查询。
- **数据存储**：TiDB 数据库使用 TiKV 作为底层存储引擎，而 TiFlash 使用列式存储引擎。这意味着 TiDB 数据库和 TiFlash 可以共享数据存储资源。
- **查询优化**：TiDB 数据库和 TiFlash 可以共享查询优化资源。这意味着 TiDB 数据库和 TiFlash 可以共享查询优化算法和数据结构。
