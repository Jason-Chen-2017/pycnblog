# 2. 核心概念与联系

在Flink中，数据源和数据接收器是流处理应用程序的基本组件。数据源用于从外部系统读取数据，数据接收器用于将数据写入外部系统。Flink提供了许多内置的数据源和数据接收器，例如Kafka、TCP socket、文件系统等。此外，用户还可以定义自己的数据源和数据接收器。

数据源和数据接收器的性能取决于以下几个因素：

1. 并行度：Flink流处理作业可以具有多个任务并行执行，这些任务之间相互独立。数据源和数据接收器的性能与其并行度有关。

2. 吞吐量：数据源和数据接收器的吞吐量是指每秒处理的数据量。高吞吐量的数据源和数据接收器可以确保流处理作业的整体吞吐量。

3. 延迟：数据源和数据接收器的延迟是指从数据到达到数据处理完成的时间。低延迟的数据源和数据接收器可以确保流处理作业的整体延迟。

4. 可靠性：数据源和数据接收器的可靠性是指它们是否能够在错误发生时保证数据的一致性。可靠的数据源和数据接收器是流处理作业的关键要素。
