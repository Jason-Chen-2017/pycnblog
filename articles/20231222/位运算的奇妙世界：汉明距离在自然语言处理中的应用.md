### 2.1 位运算

位运算是计算机科学的基础，它操作的对象是二进制数。位运算包括：

- **位补码**：将数字转换为二进制补码表示。
- **位与**：将两个二进制数按位进行与运算。
- **位或**：将两个二进制数按位进行或运算。
- **位异或**：将两个二进制数按位进行异或运算。
- **位非**：将一个二进制数按位进行非运算。
- **左移**：将一个二进制数的所有位向左移动指定的位数。
- **右移**：将一个二进制数的所有位向右移动指定的位数。

### 2.2 汉明距离

汉明距离是一种度量两个二进制序列之间的不同性的方法，它定义为两个序列不同位置的位置数。汉明距离的公式为：

$$
H(x, y) = \sum_{i=0}^{n-1} \delta(x_i, y_i)
$$

其中，$x$ 和 $y$ 是两个长度为 $n$ 的二进制序列，$\delta(x_i, y_i)$ 是两个位置上的位取反的数量。

def haming_distance(x, y):
    assert len(x) == len(y)
    distance = 0
    for i in range(len(x)):
        if x[i] != y[i]:

    return distance

def semantic_encoding(text, vocab):
    text_vector = [0] * len(vocab)

def semantic_search(query, documents, vocab):
    query_vector = semantic_encoding(query, vocab)
    scores = []
    for document in documents:
        document_vector = semantic_encoding(document, vocab)
        score = haming_distance(query_vector, document_vector)
        scores.append((document, score))

### 6.1 汉明距离与曼哈顿距离的区别

汉明距离和曼哈顿距离都是二进制序列之间的距离度量，但它们的定义和计算方式不同。汉明距离只计算两个序列在同一位置上的位取反的数量，而曼哈顿距离计算两个序列在同一位置上的绝对值。

### 6.2 汉明距离与编辑距离的区别

汉明距离是二进制序列之间的距离度量，它只计算两个序列在同一位置上的位取反的数量。编辑距离是将一个字符串转换为另一个字符串所需的最少编辑操作数，它可以是插入、删除或替换。因此，汉明距离和编辑距离的定义和计算方式不同。
