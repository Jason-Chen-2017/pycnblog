# 2.核心概念与联系
自动化测试是指通过使用自动化测试工具和框架来实现软件的自动测试，以便在软件开发过程中更快速地发现并修复软件中的缺陷。自动化测试的核心概念包括：

- 测试用例：测试用例是用于验证软件功能是否正确的一组操作和预期结果。
- 测试框架：测试框架是用于实现自动化测试的基础设施，包括测试用例的编写、执行、结果分析等功能。
- 测试数据：测试数据是用于驱动测试用例的实际输入和预期输出。

自动化测试与手动测试之间的联系在于，自动化测试是手动测试的补充和扩展，它可以帮助开发者在软件开发过程中更快速地发现并修复软件中的缺陷，从而提高软件质量和开发效率。

3. 接下来，我们定义了一个名为 `test_add` 的测试方法，它用于测试计算器程序的加法功能。
4. 在测试方法中，我们使用了 `self.assertEqual` 方法来验证计算器程序的输出结果是否与预期结果一致。
