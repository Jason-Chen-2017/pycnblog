# 1.简介
  

## 1.PostgreSQL 是什么？
PostgreSQL 是一种开放源代码的关系型数据库管理系统(RDBMS)。PostgreSQL 是在过去几十年中快速发展的关系数据库管理系统，它最初由加利福尼亚大学伯克利分校的研究生于1986年杰出贡献的POSTGRES项目开发而来，并以其开源许可证进行发布。它的出现让很多公司将其作为企业级数据库的首选，例如 Apple 的 macOS ，Microsoft 的 SQL Server ，Red Hat 的 RHEL、CentOS等等。
## 2.PostgreSQL 的优点
- 功能强大，支持丰富的数据类型，具有完整的SQL标准支持。
- 提供强大的扩展机制，可以根据需求对系统进行高度定制。
- 支持高可用性，采用冗余设计模式，能够保证数据的安全性。
- 拥有成熟的商业支持和工具支持。
- 有丰富的第三方工具支持，可以与各种编程语言，数据库管理系统进行集成。
## 3.PostgreSQL 的应用场景
- web应用数据存储
- 数据分析
- 分布式数据库应用
- 海量数据处理
- 模型数据存储
- 物联网数据收集
- 网络设备数据采集
- 车联网数据分析
# 2.基本概念术语说明
## 1.数据库
- 在计算机上用于存放各种类型结构化或非结构化的数据的容器。
- 可以简单理解为一个文件柜，里面可以存放各种文件的盘。
- 每个数据库都有一个独特的名称，该名称用于标识数据库，并由用户创建。
- 数据库支持不同的编程语言访问，如 SQL (Structured Query Language)，MySQL，Oracle，MongoDB 等。
## 2.表（Table）
- 一组相关的记录的集合。
- 表由若干列和若干行组成。
- 表中的每一行代表一个实体对象，即一条记录。
- 表中的每一列都对应一个字段，每个字段用来保存记录中一个特定属性的值。
## 3.记录（Record/Row）
- 一条数据库表中的一条数据。
- 每个记录都对应着唯一的一组值，这些值按照相同的顺序排列。
- 每个记录都可以用一个主键（Primary Key）标识。
- 主键是一个唯一且永久存在的标志符号，通常是一个数字或者字符串。
## 4.字段（Field/Column）
- 一个数据单元，表示一个属性的值。
- 每个字段都有其名称和数据类型。
## 5.索引（Index）
- 提升查询效率的一种数据结构。
- 通过创建索引，数据库引擎可以快速找到某些数据，而不是通过扫描整个表。
- 创建索引需要考虑的数据量、更新频率、索引的唯一性和空间占用等因素。
## 6.视图（View）
- 把多个表组合成一个逻辑上的表。
- 只显示所需的数据，屏蔽复杂的数据处理过程。
- 可以使用 SQL 语句创建、修改和删除视图。
## 7.触发器（Trigger）
- 自动执行指定事件时运行的存储过程。
- 可以监听到数据变化、INSERT、UPDATE 和 DELETE 操作。
## 8.主键（Primary Key）
- 保证表中每条记录都是惟一的标识符。
- 主键只能由单个字段构成，而且字段的数据类型不能为NULL。
- 主键一般是用来标识记录的唯一身份，可以保证记录的完整性、正确性和一致性。
## 9.外键（Foreign Key）
- 用于实现两个表之间建立联系的约束条件。
- 外键可以确保两张表的数据的一致性。
- 外键也称作参照完整性约束。
## 10.序列（Sequence）
- 一个独立的序列生成器，用来产生唯一的整数序列。
- 如果没有定义主键，则默认情况下会生成一个名为“id”的字段作为主键。
## 11.事务（Transaction）
- 执行过程中要么全部成功，要么全部失败。
- ACID：Atomicity（原子性），Consistency（一致性），Isolation（隔离性），Durability（持久性）。
- 使用事务可以确保数据库数据的完整性、一致性、正确性及其完整性。
## 12.锁（Lock）
- 当多个用户同时对同一个资源进行读、写、更新、删除操作时，为了避免发生冲突而采取的一种策略。
- 不同的锁级别分别限制了对同一个资源的并发访问权限。
