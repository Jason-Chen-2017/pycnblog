# 1.简介
  

## 1.1 概念介绍
​		ElasticDL(弹性分布式深度学习)是一个基于Kubernetes的开源系统，它提供云原生环境下的弹性机器学习，通过部署和管理调度系统集群，可以在任意规模的数据集上并行训练大规模深度学习模型。ElasticDL的特点如下：

1. 基于容器化技术：ElasticDL 使用Kubernetes作为容器运行时环境，在资源不足或服务故障时可以自动调整节点池的大小。同时，ElasticDL 提供弹性伸缩功能，可方便地处理复杂的计算任务，如：数据增广、分布式训练等。

2. 数据集分片机制：ElasticDL 提供了基于数据集的分片机制，可将大型数据集划分成小块，并异步进行计算，降低数据集读取瓶颈。

3. 弹性伸缩：ElasticDL 通过弹性训练任务的管理，支持用户自动扩缩容节点数量，提高集群利用率及资源利用率。

4. 模型持久化：ElasticDL 支持在本地磁盘保存模型的检查点，同时支持持久化到远程存储中，实现训练结果的长期保存。

## 1.2 系统架构
​		ElasticDL的整体架构分为两层，第一层为前端服务，负责接收客户端提交的训练任务请求；第二层为弹性训练服务，由多个Worker组成，负责进行分布式训练。Worker根据训练任务参数，从分布式文件系统（如HDFS）下载数据集，并将数据集划分成小块，并发执行各自节点上的训练任务。每一个Worker节点上都有一个TensorFlow或者PyTorch进程，用于加载模型参数，读取数据块，更新模型参数。当所有节点上的训练任务完成后，Worker将得到全局的模型参数，并把它们保存在持久化存储（如HDFS）中。

## 1.3 使用场景
​		随着人工智能技术的飞速发展，越来越多的应用需要训练海量的深度学习模型。但是传统的深度学习框架如 TensorFlow 和 PyTorch 在面对海量数据的训练时，往往需要采用分布式计算的手段，例如：分布式训练模式、多机多卡模式。这些方法虽然可以有效减少内存消耗和加快训练速度，但仍然无法解决更大的数据集和更复杂的模型训练任务。为了处理海量模型训练的问题，同时保证模型的准确性，越来越多的研究人员、开发者、企业也纷纷加入到了分布式训练和超参数优化的阵营。

​		ElasticDL 的出现，正是为了应对这样的需求。ElasticDL 可以帮助用户快速、方便、弹性地训练出高性能的深度学习模型。用户无需进行任何改动就能在 Kubernetes 上获得分布式、弹性的训练能力。并且 ElasticDL 还提供了强大的特征工程模块、数据预处理模块、模型压缩模块、模型评估模块等，可以很好地满足不同深度学习任务的需求。
