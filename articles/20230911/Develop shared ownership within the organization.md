# 1.简介
  

企业越来越多地面临着赋予员工共同所有权的问题。根据已有的工作经验来看，员工通常会有自主决策、组织资源分配以及承担责任的权利。但是随着组织规模的扩大，管理层越来越难以有效地分享管理权力，导致员工不再独立完成工作。因此，企业需要找出一种新的方式来协调员工的职责和责任，以达到共享管理权力和利益的目的。
# 2.定义
“Shared Ownership”即共享所有权，这是指企业内部的各个部门、岗位和个人都有责任或者权利对某个或某些事项负起完整的管理角色，相互之间具有平等的义务。
# 3.概念
当今社会的公司都在采用数字化、全球化的方式，将商业活动高度集成在线上，使得全员参与业务，而每一个人又拥有自己的个性特征和需求。所以，如何管理好公司中的人员关系，让员工之间的工作能够实现最大化的效率提升，同时还可以保持员工的积极性和创造性，并且有助于整体工作目标的实现，就成为企业管理者们在努力寻求解决这个难题中最关心的事情之一。
# 4.核心算法原理和具体操作步骤
## 4.1 团队的创建流程及意义
首先，我们要搞清楚什么样的团队适合进行“共享所有权”。一般来说，比较典型的团队形态有两种，分别是项目驱动型和产品驱动型。前者更侧重于项目产出的交付质量，比如研发团队在某个产品上投入很多精力，可能在最后也只能回报不足；后者更注重产品质量，如设计团队可以负责某个产品的功能设计、美观设计以及用户体验设计，甚至可以直接参与产品运营活动。当然，无论哪种类型的团队，其实都离不开一些必要的基础设施设施才能正常运行。
那么，团队之间是否需要共享管理权力呢？如果想要共享管理权力，那就需要制定相应的管理制度，并落实到实际上。比如，研发团队和设计团队之间应该做好信息沟通、协作配合，确保各自负责产品开发的任务都是单一、统一的；也可以规定研发团队独自负责某个模块的设计，并由设计师团队去实现这一点。另外，还可以通过设立商业模式等方式鼓励员工之间的合作。
除了这些基础性的工作，在产品驱动型的团队中，还需要确保其中的关键人才都具备一些职业素养和能力。因为，只有具有强烈责任感和主动性的人才能真正把握产品的核心价值，只有他们才能进行关键的技术突破。而产品驱动型的团队往往聚焦于产品本身的交付，忽略了工程开发过程中存在的挑战和风险。因此，对于这种类型的团队，共享管理权力可能会带来更多的麻烦，因为管理者可能无法将知识体系完全转移到每个人手里，而是需要利用信息化手段来优化工作的节奏和结构。
## 4.2 沟通和协作工具
为了实现共享管理权力，企业的管理者应该选择合适的沟通工具。一个好的沟通工具可以帮助团队成员之间建立联系、增进互相理解，减少沟通上的摩擦。比如，团队微信群、QQ群以及普通话电话等社交平台都可以用于沟通。除了工具，还需要善用语言和文化差异。比如，研发团队员工可能习惯用英文进行讨论和交流，但产品团队则喜欢用中文进行沟通和分享，从而影响产品质量。因此，如果企业想让团队成员之间能充分融合，就必须要通过不同的沟通方式来互相学习。
另一方面，由于不同团队成员所处的不同位置，不同的工作职责以及个人能力，协作工具的选择也非常重要。比如，研发团队可能倾向于选择白板或大屏幕作为协作工具，而产品团队可能更偏爱办公室中的虚拟沙龙形式。因此，共享管理权力实际上会涉及到跨团队的协作工作，因此协作工具的选择还需要考虑到协作效率、可靠性和隐私保护等因素。
## 4.3 管理层责任划分
经过上面两步操作之后，管理层就会考虑如何实现“共享管理权力”，但是这只是第一步。接下来，管理层还需要为共享管理权力付诸行动。怎么样划分出各个团队的管理层责任，该把权力留给谁，留给谁，还有没有什么其他的考虑？通常情况下，企业都会设置管理层委员会，由领导层决定各个团队的管理层责任以及权力范围。具体做法如下：
- 各产品团队设置产品总监（Product Manager）作为主要管理者；
- 研发团队设置研发副总裁（Director of Engineering）作为主要管理者；
- 技术委员会设置技术总监（Technical Director）作为主要管理者；
- 有时，还有产品销售经理（Product Marketing Manager）等职位。
## 4.4 团队管理角色和责任
上述操作已经为各个团队引入了相应的管理角色。当然，不同类型团队也有自己特有的管理角色，例如产品团队设置产品经理（Product Designer），设计团队设置视觉设计师（Graphic Designer）。这样，管理层的角色设置可以更具体化，为不同团队的人才培养提供有限的资源。除此之外，还有一些更具体的管理角色应当被纳入考量：
- 每个团队都应该有高级技术专家，因为产品研发过程中，会遇到很多复杂的问题，比如性能优化、安全防护、网络流量控制等；
- 每个团队都应当设置战略规划专家，因为管理层希望能够洞察整个公司的发展趋势，并及时调整策略；
- 每个团队都应该设置商业计划师，他们的作用是将公司的商业计划转换成执行计划，制定具体的执行方案。

# 5.未来发展趋势与挑战
虽然“共享管理权力”可以提升企业整体的效率，降低管理压力，但仍然有许多挑战。例如，公司层面的信息沟通、协作工具以及不同类型团队的管理角色没有考虑到可能带来的管理层额外开支。另外，如何建立个人价值观和工作理念也是一个重要课题。未来，还需要持续地提升协作和管理水平，更好地实现共享管理权力，帮助企业构建更加包容、团结、有序的工作环境。
# 6. 附录常见问题与解答
1. 为什么要共享管理权力？
“共享管理权力”的概念源于“职业多样性”，它是指多个部门和岗位共同对某些事项负起责任、产生共识和管理角色。目前，许多大型公司都在探索如何更好地激活员工的潜力，同时还要促进内部团队的融合，以便于共同达成目标。

2. 共享管理权力的优势有哪些？
- 提升整体效率：“共享管理权力”可以提升企业整体的效率，降低管理压力。一个好的管理框架可以让各个团队快速响应市场变化、抢占市场先机。
- 降低管理成本：“共享管理权力”可以降低管理成本，尤其是在跨国公司或政府机构，员工的海外经历或工作经验可能会限制他们的思维能力。
- 提升个人职业成长空间：“共享管理权力”可以增加各个员工之间的协同合作能力，培养其个人职业生涯规划能力，提升个人价值观和工作理念。

3. 企业如何实现“共享管理权力”？
企业实现“共享管理权力”的方法主要有以下四种：
- 团队设置：企业可以根据职能或产品形态，设置符合要求的团队结构。其中，产品团队可以设置产品总监、产品经理、设计总监等关键角色，并落实相关管理制度。
- 人才培养：企业可以在产品研发、战略规划等不同领域，设置相应的管理专家，确保各个团队都有领先的技术专家。
- 沟通协作工具：企业可以选择合适的沟通工具，包括团队微信群、QQ群、邮件列表等。沟通工具还可以提供翻译、视频会议等形式，改善沟通效率。
- 管理层的责任划分：企业可以设置管理委员会，将各个团队的管理责任划分到不同的组别，确保各个团队之间平等协作。

4. “共享管理权力”存在哪些挑战？
“共享管理权力”所面临的主要挑战有以下几种：
- 分工问题：“共享管理权力”的方式导致管理层越来越依赖个人能力和经验，容易出现分工不明确的问题。
- 流程问题：“共享管理权力”可能导致一些流程环节被重复或缺失，导致管理效率下降。
- 消费问题：“共享管理权力”可能会消耗大量的人力资源和时间，降低员工的忙碌状态。
- 利益问题：“共享管理权力”可能会让管理层获得不公平的收益，并最终影响到所有人的工作表现。
