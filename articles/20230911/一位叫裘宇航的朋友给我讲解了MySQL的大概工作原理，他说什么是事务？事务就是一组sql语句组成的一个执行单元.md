# 1.简介
  

## 数据库系统（Database System）
数据库系统(DBMS)是管理关系型数据库的一套完整的计算机软件系统，由数据库管理员、开发人员、应用程序员和最终用户共同协作完成，目的是为了方便地存储、检索和更新数据。目前主流的数据库有SQL Server、Oracle、MySQL等。

## MySQL
MySQL 是一种开放源代码的关系数据库管理系统（RDBMS），由瑞典MySQL AB公司开发，属于 Oracle 旗下产品，采用客户/服务器模型，支持 SQL 和函数编程语言。MySQL 是最流行的开源数据库服务器端。它被广泛应用在 web 网站、移动App开发、大数据分析、物联网以及各种高性能交易系统中。

## 什么是数据库？
数据库，通常指结构化查询语言（Structured Query Language，SQL）数据存储的仓库，用来存储、组织、管理和保护数字信息。

## 数据库的作用
- 数据组织：数据库将所有的数据按照一定规则存放在一起，使得数据的输入、输出、处理变得容易，更加有效。
- 数据共享：数据库将数据共享给其他程序，使各个程序可以共享相同的数据。
- 数据安全：数据库提供数据访问控制功能，对数据的访问权限进行细致的控制。
- 数据备份：数据库提供数据的冗余备份，可以避免意外损失。
- 数据分析：数据库提供了丰富的统计分析工具，对数据进行快速准确的查询、报表生成、图形展示。

## 关系型数据库 VS NoSQL数据库
关系型数据库和非关系型数据库是两种截然不同的数据库技术。

### 关系型数据库
关系型数据库（Relational Database Management System，RDBMS），又称为企业级数据库，是建立在关系模型上的数据库。它保存着复杂的数据集合，其中每一个数据项都按一定的关系存放在一张数据库表里，并且表中的每个字段都定义了一种数据类型。关系型数据库基于二维表格模型，并且具有完整的数据定义和强的数据安全性。它的优点是结构清晰、灵活、适合处理各种类型的数据；缺点则是执行效率比较低，尤其是在海量数据的情况下。例如，MySQL就是一个关系型数据库。

### NoSQL数据库
NoSQL，即Not Only Structured Query Language，非关系型数据库。它是指不仅仅支持 Structured Query Language (SQL)，还支持非关系型数据模型。NoSQL数据库的最大特征是不仅仅面向文档，也支持键值对、图形及列族等多种非关系型数据模型。NoSQL数据库通常被用于大数据量、高并发、高可靠性的场景。典型的NoSQL数据库包括Couchbase、MongoDB、Redis、 Cassandra等。这些数据库的优点是扩展性好、数据自动分片、易部署和迁移等。但也存在一些缺点，比如性能问题、数据一致性问题等。例如，Memcached是一个简单的Key-Value数据库，也可以作为NoSQL数据库来使用。
