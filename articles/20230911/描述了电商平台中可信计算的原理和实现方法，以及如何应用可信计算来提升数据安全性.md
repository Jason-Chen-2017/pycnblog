# 1.简介
  

随着互联网业务的飞速发展，越来越多的公司开始考虑将业务转移到线上，这使得安全性成为重点关注的重要因素。在这一背景下，可信计算作为一种新兴的计算模型，越来越受到公司的青睐。随着云计算、区块链等技术的发展，可信计算也逐渐得到更多的关注。那么，电商平台中到底怎么样才能实现可信计算呢？本文试图通过结合实际案例来说明可信计算对于电商平台数据的安全性的影响及其解决方案。
# 2.可信计算概念
可信计算（Trusted Computing）是基于真实世界中的假设建立的计算系统，它可以支持各种高级特性，如机密计算、数据安全、溯源、透明性、可追溯性等。可信计算属于类别计算机科学研究领域，是在安全计算模型基础上的进一步的计算模型，具有高度的理论复杂度和实际意义。不同之处在于可信计算定义了三个关键要求，即实体、机制、完整性。实体要求计算系统能够对实体提供真实且可靠的数据；机制要求计算系统具有多种处理手段来保护数据不被篡改或泄露；完整性要求计算系统必须保证数据正确有效。
基于以上三个要求，可信计算可以分为三层架构：
- 第一层：实体层，即所谓的实体化需求，计算系统需要提供实体的身份认证、数据收集、存储、传输、分析、防止攻击等功能。
- 第二层：机制层，即所谓的行动控制需求，计算系统要具备处理能力，包括加密、签名、授权、访问控制、审计等，用于保障计算结果的真实性和完整性。
- 第三层：完整性层，即所谓的完整性保障需求，计算系统还需要确保计算结果的有效性、精准性、持久性、不可否认性和可验证性。

在电商场景下，平台与消费者产生交流信息后，会经历多个环节。其中最重要的一环便是数据的存储、传输和接收。如果平台不采用可信计算方式存储用户信息，可能存在如下风险：
- 数据被篡改：例如，用户A提交订单后，其他用户利用系统漏洞进行恶意操作，获取到平台保存的信息。
- 数据被偷窃：例如，用户B盗取其他用户的个人信息后，冒充其身份购买商品，造成严重经济损失。
- 数据泄露：例如，平台中的个人隐私被泄露给不相关的第三方。
因此，为了降低用户信息的风险，电商平台必须选择可信计算的方式存储用户信息。
# 3.可信计算案例：电子商务网站
假设某电子商务网站采用可信计算架构，整个计算过程由实体、机制、完整性三层组成，流程如下：
- 实体层：用户上传订单信息后，通过对用户输入的数据进行校验，并加上时间戳生成摘要。
- 機械层：生成的摘要数据在存储之前，先进行加密，保证数据不被他人获取、篡改。
- 完整性层：采用数据完整性验证机制，对存储的数据进行校验，保证数据的完整性，同时记录数据操作日志，用于数据追溯。
下面具体分析一下流程：
## 实体层
当用户上传订单信息时，平台接受请求，接收数据并对其进行校验，包括用户名是否合法、订单数量是否正确、金额是否足够等。然后根据配置参数自动生成相应订单编号，并将信息保存至订单数据库中。订单数据库是一个可信计算组件，用于对订单数据进行存储和管理。
## 機械层
订单数据库中的数据虽然已经经过加密，但仍然不够安全，因为这些数据是散列后的形式，只有平台的系统管理员才有权对其解密。为了保证数据完全不被泄露，平台可以采用哈希函数对数据进行编码，生成摘要值。然后再将数据存储至计算节点服务器上。
## 完整性层
计算节点服务器存储的数据都是经过编码后的，但由于无法知道用户真实订单信息，因此只能确定该订单是否在交易系统中被创建。所以，为了实现真正的订单完整性，平台还需在计算节点服务器上部署一个事务跟踪模块，记录每一次订单交易的全过程，方便之后追溯数据和检查问题。
通过这样的设计，平台既满足了数据加密、存储的要求，又保证了数据操作的完整性，并记录了每个订单交易的全过程。这样做既能有效地保障用户信息的安全，又减少了信息泄露、信息被篡改等风险。
# 4.结语
本文主要介绍了电商平台中的可信计算概念及其实践案例。本文认为可信计算的核心就是构建透明、可信任、可信赖的计算环境，用以保障数据安全、数据完整性和用户隐私。通过可信计算的方式存储用户信息，能够有效地降低用户信息的风险，提高平台整体的运营效率。
