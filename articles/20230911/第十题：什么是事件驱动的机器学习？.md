# 1.简介
  

“事件驱动”（Event-driven）机器学习，又称事件驱动数据分析（EDA），是指通过收集、处理、分析来自不同来源的事件数据并对其进行实时处理，通过产生新的事件、触发特定事件的条件等，对事件流进行实时的响应和分析，从而实现对复杂系统状态的实时监控和分析，进而形成预警或决策支持。

相对于传统的静态机器学习算法，事件驱动机器学习在数据处理上具有以下优点：

1. 数据积累及时性：事件驱动的机器学习可以处理来自各个来源的数据，并根据数据产生的事件及时更新模型，使得数据集不断增长。

2. 反应灵敏度高：事件驱动的机器学习可以快速响应变化，不间断地对最新数据进行处理，从而达到实时监测数据的目的。

3. 更好的结果预测：事件驱动的机器学习可以准确捕捉到数据中的真实信号，从而实现更精准的预测。

4. 数据规模大：由于采集的事件数据越来越多，处理这些数据所需的时间也变得越来越长，因此，随着数据量的增加，效率将会越来越低。

事件驱动机器学习的主要特点包括：

- 实时性：事件驱动的机器学习能够在短时间内做出反应，可以用于实时监控。

- 数据交互性：事件驱动的机器学习可以通过互联网或者消息传递机制收集来自不同来源的事件数据，然后对事件流进行实时处理，从而实现对复杂系统状态的实时监控。

- 数据抽象化：事件驱动的机器学习采用事件的方式对数据进行处理，把数据看作一系列的事件，从而实现数据的抽象化。

- 模型快速迭代：事件驱动的机器学习可以快速地更新模型，对最新数据进行处理，从而实现模型的持续优化。

- 时空依赖性：事件驱动的机器学习对历史数据和当前时间都有依赖关系，可以应用于包括物理系统在内的复杂系统中。

# 2.基本概念术语说明
## 2.1 定义
事件驱动的机器学习，也称事件驱动数据分析，是指通过收集、处理、分析来自不同来源的事件数据并对其进行实时处理，通过产生新的事件、触发特定事件的条件等，对事件流进行实时的响应和分析，从而实现对复杂系统状态的实时监控和分析，进而形成预警或决策支持。

## 2.2 相关术语
- 智能事件：智能事件是对某些事件的持续观察，目的是为了寻找或发现可能发生的事件类型，并尝试分类、理解它们的含义，从而开发出相应的预警机制，帮助组织者提前做出相应的响应。
- 事件检测与预报（EDAP）：事件检测与预报（EDAP）是指按照设计好的预测方法和模型，从事件数据中自动地识别、分类、预测和分析事件；同时利用各种机器学习技术，对预测结果进行评估、改善，使之更加准确可靠。
- 分布式计算：分布式计算是一种通过网络连接多个计算机并行工作的技术，它允许多个独立的计算机节点同时执行相同的任务。
- 事件驱动分析：事件驱动分析（ECA）是指通过对事件进行分析、处理、检索和跟踪，对复杂系统的运行情况进行实时监测，并及时做出反应，以便在必要时触发预警机制，提升系统的鲁棒性、稳定性、可靠性和可用性。
- 事件驱动模式：事件驱动模式是指系统中的某些状态或条件发生改变，导致产生事件，而当事件被处理之后，生成新的事件，并驱动系统继续运行。
- 事件溯源：事件溯源是指追踪一个事件如何从诞生到消亡，全程记录下它的历史过程，并依据这一历史过程对事件的起因、影响及其后果进行分析、预测，并提供相关建议。
- 抽象事件：抽象事件是一种表述方式，其中的信息元素通常不需要具体的值。可以用通用语言描述抽象事件，如“检测到系统故障”，“开始了新一轮的销售”。
- 可视化工具：可视化工具是指用来呈现信息的图表、图像、电子表格、报告等工具。
- 事件驱动的数据分析平台：事件驱动的数据分析平台（EDAP）是一个基于云端的工具，可以让用户管理、存储、分析、预测和可视化来自不同来源的事件数据。它具备数据采集、数据清洗、数据存档、数据过滤、数据转换、数据建模、模型训练、模型推理和模型评估等功能模块。
- 事件驱动的机器学习模型：事件驱动的机器学习模型是指通过收集、处理、分析事件数据，建立模型对复杂系统状态进行实时监控和分析的模型，它采用事件作为输入，输出预测模型的概率值和类别标签，能够快速响应变化，即使遇到新数据也不会陷入过拟合的困境。
- 事件驱动的预警机制：事件驱动的预警机制是指由事件驱动的机器学习模型根据用户设定的阈值，判断出现异常状况之后，立即向用户发送预警通知，促使用户及时发现并解决潜在问题，从而提升系统的效率和效益。
