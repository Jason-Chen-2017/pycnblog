# 1.简介
  

## 一、引言
在业务领域中，经常会遇到一些很复杂的事情，比如说商机到来、合作伙伴关系变化等。在面对这些复杂的状况时，如何快速准确地掌握业务信息、洞察问题、把握机会、制定应对策略？在正确评估和认识问题的同时，我们还需要注意将注意力集中到关键路径上，帮助我们更好地将注意力放在最重要、最紧迫的环节上，并提升效率、缩短响应时间。那么什么是关键路径呢？又该怎么做到它呢？

## 二、概述
关键路径（Critical Path）又称之为关键工程，它是指影响整个项目生产或交付成功的关键活动或事件流顺序。当某个事件发生时，该事件前面的所有事件都已经完成，才能开始处理后面的事件。通常情况下，对于一个项目而言，关键路径可以帮助团队在完成各项活动之前作出明智的判断，最大程度地减少因活动延误带来的损失。因此，好的项目管理方法应该强调识别关键路径，确保项目顺利推进、取得成功。关键路径思维法（Critical Path Thinking Method）通过分析关键事件的影响，以帮助组织者及团队成员更好地预测、组织、协同工作，从而实现关键路径管理目标。

关键路径思维法可分为如下四个主要阶段：

①识别关键事件：关键路径起点和终点都是非常重要的，因为它们直接决定着整个企业生命周期中的转型和增长，也是公司盈利能力的保证。因此，在开始分析项目的关键路径之前，首先要识别其中的关键事件，以确定它们之间的顺序关系。

②定义关键路径：定义关键路径可以根据组织内部的结构、资源、工作流程、各方的合作关系等多种因素。为了定义清晰，关键路径应由多个事件组成，并且每个事件均不得超过六个月的时间跨度。

③制订预案：制订预案可视为关键路径思维法的最后一步。基于关键路径的信息、数据、分析结果，组织者将结合团队的实际情况制定具体的计划方案。

④跟踪监控：关键路径的实施还依赖于严格的监督检查和跟踪。在关键路径实施之前，组织者及团队成员需要不断收集相关信息，以便于跟踪关键事件的实施情况，确保项目顺利进行。如此一来，关键路径思维法可帮助组织者及团队成员更好地理解项目整体发展方向，优化资源配置，提升工作质量，最终达成项目的目标。

总之，关键路径思维法是一个有效的方法，通过对关键事件的研究，它能够帮助组织者及团队成员更好地预测、组织、协同工作，达成项目目标。同时，也应充分利用它的理论基础和工具来辅助管理工作。

## 三、关键路径模型
关键路径模型是指用来描述工程活动（项目、产品、服务等）及其顺序的图形化框架，其中包括四部分元素：触发事件、活动节点、顺序关系、相互依赖关系。下面我们用关键路径模型来解释一下什么是关键路径。

### 1.触发事件
触发事件即系统中发生的事件或条件，触发事件可能有以下几类：

- 外部事件：指外部因素引起的系统的变化，例如环境变化、人员变动等；
- 操作事件：指用户执行的系统操作，例如提交表单、点击按钮等；
- 数据事件：指产生或存储的数据发生变化，例如增加、删除数据等；
- 时序事件：指系统某一功能之间存在依赖关系，导致发生先后的顺序关系，例如登录成功后才能查看用户信息。

### 2.活动节点
活动节点就是系统中进行的活动，通常来说，活动节点属于以下两种类型：

- 实体活动：指非独立的、不可分割的活动，例如填写表单、发送邮件；
- 协同活动：指多个实体活动配合完成的活动，例如提交申请、审批、核对等。

### 3.顺序关系
顺序关系是指活动之间是否存在先后次序关系，通常采用箭头表示。

### 4.相互依赖关系
相互依赖关系则表示两个活动之间是否存在相互影响，如果存在，则表现为斜线上的虚线，反之则为实线。

### 5.关键路径计算方法
关键路径计算方法（Critical Path Calculation Method），又称关键路径预算法，它是在约束条件下求解工程关键路径的问题。其基本思想是先确定初始条件、边界条件和限制条件，再分析工程的网络结构，进而得到工程关键路径的长度。下面以车库空置的例子来说明关键路径计算方法的步骤。

假设有一个车库，里面有N个车位，每个车位只能停放一辆汽车，可以选择任意一辆车进入，但不能选择进入相同车道。现在，车库有N辆车要进入，需要满足以下几个要求：

①没有两辆汽车进入同一车道。

②车辆进入车库之后，不能再进入车道间隔范围内的车位。

③车辆每隔M分钟至少要停留一次。

关键路径预算法的步骤如下：

①确定初始条件：将N个车位都填满，设初值为C(N)。

②确定边界条件：将N辆车均匀分布于车库的不同车道中，设边界值为B(N)。

③确定限制条件：对于每辆车的第i个停靠事件，其前面的k-1个停靠事件（k<=i-M），必须全都满足。设限制值为R(N)。

④求解路径长度：用动态规划的方法，递归地计算每个活动的长度，直到计算到起始节点。路径长度的值等于活动节点中的耗费时间乘以路径长度值。设N个活动的序列为{A1, A2,..., AN}，则第i个活动的长度可表示为L[i]=∑_{j=1}^{i-1}(L[j]*P[j][i])，其中*号表示笛卡尔积。所以，可以得到L(N)=∑_{i=1}^NL[i]。另外，也可以设置前向差分的办法，来避免重复计算路径长度。

⑤筛选出最优路径：设最优路径为{A^*_1, A^*_2,..., A^*_K}, K<=N。只要满足L(N)<L(K)，就可以终止，否则继续重复前三步，直到无可行解。
