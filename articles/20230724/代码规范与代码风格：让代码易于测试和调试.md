# 1.简介
         
代码规范与代码风格在提高代码质量方面扮演着重要角色。它可以减少编码时的错误率，并增强代码可读性、维护性和可靠性。本文将结合自身的经验和研究成果，阐述代码规范与代码风格所涉及的内容及其意义。
# 2.为什么要进行代码规范与代码风格？
项目进展需要具有良好的代码风格与规范。下面介绍几种常见原因，以及如何通过代码规范与代码风格改善代码质量。

1. 可读性提升：代码的可读性是衡量一个程序是否易懂的最重要因素之一。良好的代码结构和命名方式可以帮助开发人员快速理解程序逻辑和功能特性，从而更容易地做出修改或扩展，降低软件缺陷。

2. 更易于理解和维护：一个优秀的代码规范可以促使开发人员团队中的每个成员都快速理解该程序的设计目的，从而方便其他开发人员进行协作。

3. 提高代码健壮性：代码规范可以防止因疏忽造成的bug出现，同时还可以在不同编程语言之间共享相同的代码标准，降低代码重用率的风险。

4. 更好地运用工具：借助代码检查工具和自动化格式化工具，我们可以更加有效地发现代码中的问题，从而提升代码质量。

5. 培养代码习惯：良好的代码规范会强制开发人员保持良好的编程习惯，从而帮助他们顺利完成任务。

因此，想要改善代码质量，就必须重视代码规范与代码风格。以下是对代码规范与代码风格的定义：

代码规范是一套规则，用于规定软件工程中所有的源代码文件，如变量名、函数名称等，以及这些文件的组织方式、文件头部注释信息等内容。它提供了一系列可供遵循的指南和约束，用于确保编写的代码符合某些标准，降低软件开发过程中可能出现的风险，提高代码的可靠性、可理解性和可维护性。它不仅对整个软件系统适用，而且也应该作为独立的文档存在，以便于他人查阅。

代码风格是一种偏好、个人风格或习惯的观点，它体现了开发者对代码书写格式和布局方式的喜好程度。不同编程语言的代码风格也不同，但它们都属于规范的一部分。代码风格不仅适用于同一项目中的所有源代码，而且也应当被纳入到项目管理、评审和规范流程中，促进代码的一致性。

# 3.代码规范的主要内容
## 3.1 文件命名规范
文件命名采用小写加下划线的方式，并尽量保持简单，不要出现特殊符号。
举例:
my_file_name.cpp
another_file_name.h
main.py

## 3.2 行宽限制
每行字符数不能超过80个。每行末尾也不能有多余空白字符，只能有一个。如果有多余空白字符，IDE会警告。

## 3.3 缩进与空格
每级缩进4个空格，换行时缩进与上一行保持一致。代码块的左花括号{应在第一列。
```c++
int main() {
    // code block here...
}
```

## 3.4 括号与空格
所有控制语句（if、else if、for、while）的左括号{应在同一行。每条语句的结束都需要加分号；而表达式语句不需要。

```c++
if (a > b) 
    return a;  // wrong example
    
// correct way
if (a > b) 
    return a;  
```

方法声明后加分号。方法参数列表在方法名前后都加空格。

```c++
void foo(int x, int y);  // right
void bar(int x, int y); // wrong
```

变量类型和指针类型后都加空格。运算符两边都加空格。

```c++
double pi = 3.14;     // right
float radius=2.5f;    // right
char *str="hello";    // right
int n= i+j*k / l - m%n ; //wrong
```

## 3.5 注释规范
1. 总体原则：文件开头注明作者和创建日期，文件结尾注明作者姓名和版本号。

2. 函数头注释：函数头注释应包括函数作用、输入输出、返回值、异常情况等详细信息。

3. 注释风格：类注释一般与类声明放在一起，放在类的声明之前；接口注释一般与接口声明放在一起，放在接口声明之前；全局变量和宏注释一般与全局变量或者宏定义放在一起。

4. 中英文混排：中文和英文、数字之间用半角空格隔开；英文单词和句子中间用半角空格隔开。

## 3.6 模板规范
模板的类名使用大驼峰式命名法，模板的参数列表使用小驼峰式命名法。

```c++
template<typename T> class MyClass{};
```

函数模板与类模板的命名风格一致。

```c++
template <typename T, typename U> void swap(T& a, U& b){
  ...
}

template<class T> class Queue {};
```
