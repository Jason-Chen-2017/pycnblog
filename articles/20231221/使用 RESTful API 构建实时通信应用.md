## 2.核心概念与联系

在本节中，我们将介绍 RESTful API 的核心概念和与其他相关技术的联系。

### 2.1 RESTful API 的核心概念

RESTful API 的核心概念包括：

- **资源（Resource）**：RESTful API 中的资源是指网络上的一个实体，它可以是一个文件、一个图像或一个数据库记录等。资源被唯一地标识，通过 URI（Uniform Resource Identifier）进行标识。
- **表示（Representation）**：资源的表示是资源的一个具体的表现形式，例如 JSON 或 XML 格式。
- **状态转移（State Transition）**：RESTful API 中的状态转移是指从一个资源状态到另一个资源状态的转移。状态转移是通过 HTTP 方法（如 GET、POST、PUT、DELETE 等）来实现的。
- **无状态（Stateless）**：RESTful API 是无状态的，这意味着服务器不会保存客户端的状态信息。每次请求都是独立的，不依赖于前一次请求的结果。

### 2.2 RESTful API 与其他技术的联系

RESTful API 与其他实时通信技术有很多联系，例如 WebSocket、SockJS 等。这些技术都是为了解决实时通信的需求而设计的。

WebSocket 是一种基于 TCP 的协议，它允许客户端和服务器之间建立持久的连接，以实现实时通信。WebSocket 的优点是简单、高效、低延迟。但是，WebSocket 只能在 HTTP 连接建立后使用，这限制了其应用范围。

SockJS 是一个基于 WebSocket、Server-Sent Events（SSE）和长轮询（Long Polling）的实时通信库，它可以在不同的浏览器和环境下提供一致的实时通信功能。SockJS 的优点是兼容性好、易于使用。但是，SockJS 需要额外的库来实现，这增加了复杂性。

相比之下，RESTful API 具有更广泛的应用范围，可以在不同的环境下实现实时通信。同时，RESTful API 也可以与其他技术结合使用，例如 WebSocket、SockJS 等，以实现更高效、更高质量的实时通信功能。
