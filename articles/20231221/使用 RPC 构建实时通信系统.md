# 2.核心概念与联系

首先，我们需要了解一下 RPC 的基本概念。RPC 是一种在分布式系统中实现远程对象之间通信的技术，它允许程序调用另一个程序的过程，而不需要显式地引用它们所处的地址空间。在 RPC 中，客户端向服务器发送请求，服务器将请求处理完成后，返回结果给客户端。

在实时通信系统中，RPC 可以用于实现用户之间的通信。例如，当用户 A 向用户 B 发送一条消息时，用户 A 可以通过 RPC 调用服务器端的消息发送接口，将消息发送给服务器。服务器将消息存储到数据库中，并通过 RPC 调用用户 B 的客户端，将消息推送给用户 B。
