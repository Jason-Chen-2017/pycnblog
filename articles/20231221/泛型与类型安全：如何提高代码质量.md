# 2.核心概念与联系

## 2.1 泛型

泛型是一种编程技术，它允许程序员在编译时检查代码中使用的数据类型，从而避免运行时错误。泛型的主要优点是它可以提高代码的可重用性和可读性，同时降低运行时错误的风险。

在Java中，泛型使用尖括号`< >`括起来的类型参数来表示泛型类型。例如，以下是一个泛型的列表类型：

```java
List<String> list = new ArrayList<String>();
```

在上面的例子中，`String`是类型参数，表示列表中的元素类型。这样，程序员可以确保列表中的元素始终是`String`类型，避免了运行时错误。

## 2.2 类型安全

类型安全是一种编程风格，其中编译器会检查代码中的类型错误，确保程序在运行时不会出现类型错误。类型安全的主要优点是它可以提高代码的可靠性和安全性，同时减少运行时错误的风险。

在Java中，类型安全可以通过使用强类型系统来实现。强类型系统要求程序员在编写代码时明确指定变量的类型，以便编译器可以在编译时检查类型错误。例如，以下是一个类型安全的代码示例：

```java
int a = 10;
String b = "Hello, World!";
```

在上面的例子中，`int`和`String`都是明确指定的类型，编译器可以在编译时检查这些类型是否匹配。如果类型不匹配，编译器将报错。
