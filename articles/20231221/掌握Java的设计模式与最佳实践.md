# 2.核心概念与联系

## 2.1 设计模式

设计模式是一种解决特定问题的解决方案，它们可以帮助开发人员更快地编写高质量的代码。设计模式可以分为三种类型：创建型模式、结构型模式和行为型模式。

### 2.1.1 创建型模式

创建型模式是一种用于创建对象的设计模式，它们可以帮助开发人员更好地组织代码，提高代码的可读性和可维护性。创建型模式包括以下几种：

- 单例模式：确保一个类只有一个实例，并提供一个全局访问点。
- 工厂方法模式：定义一个用于创建对象的接口，让子类决定哪个类实例化。
- 抽象工厂模式：提供一个创建一组相关或相互依赖的对象的接口，不需要指定它们的具体类。
- 建造者模式：将一个复杂的构建与其表示相分离。这样的话，同样的构建过程可以构建不同的表示。
- 原型模式：用于创建新的对象，以便将对象状态用作其他对象的模板。

### 2.1.2 结构型模式

结构型模式是一种用于组合类和对象的设计模式，它们可以帮助开发人员更好地组织代码，提高代码的可读性和可维护性。结构型模式包括以下几种：

- 适配器模式：将一个类的接口转换为另一个类的接口，从而允许该类被其他类使用。
- 桥接模式：将一个类的接口分离到多个独立的类中，以便可以独立地改变它们。
- 组合模式：将多个对象组合成一个树形结构，以便更容易地对它们进行遍历。
- 装饰模式：动态地给一个对象添加一些额外的功能，不需要改变其自身的结构。
- 代理模式：为一个对象提供一个替代者，以控制对它的访问。

### 2.1.3 行为型模式

行为型模式是一种用于定义类之间的交互的设计模式，它们可以帮助开发人员更好地组织代码，提高代码的可读性和可维护性。行为型模式包括以下几种：

- 策略模式：定义一系列的算法，并将它们一个一个放在一个接口中，以便可以根据需要选择算法。
- 命令模式：将一个请求封装成一个对象，从而可以用不同的请求对客户进行参数化。
- 观察者模式：定义一个一对多的依赖关系，以便当一个对象发生变化时，其依赖关系被自动更新。
- 状态模式：允许一个对象在其内部状态改变时改变其行为。
- 模板方法模式：定义一个算法的骨架，但让子类决定其具体实现。
- 命令模式：将一个请求封装成一个对象，从而可以用不同的请求对客户进行参数化。

## 2.2 最佳实践

最佳实践是一种编程方法，它们可以帮助开发人员更好地组织代码，提高代码的可读性和可维护性。最佳实践包括以下几种：

- 单一职责原则：一个类应该只负责一个职责。
- 开放封闭原则：类应该对扩展开放，对修改封闭。
- 里氏替换原则：子类应该能够替换其父类。
- 依赖反转原则：高层模块不应该依赖于低层模块，两者之间应该依赖于抽象。
- 接口隔离原则：不应该将许多不相关的功能集中到一个接口中。
- 迪米特法则：最少知识原则，一个对象应该对其他对象有最少的了解。
