# 2. 核心概念与联系

## 2.1 容器化技术

容器化技术是一种应用程序部署的方法，它可以将应用程序和其所需的依赖项打包成一个独立的容器。容器化技术的主要优势是它可以实现应用程序的快速部署、扩展和管理。

容器化技术的核心组件包括：

- 容器（Container）：容器是一个应用程序和其所需的依赖项打包成一个独立的文件。容器可以在任何支持容器化技术的平台上运行。
- 镜像（Image）：镜像是一个容器的模板，它包含了应用程序和其所需的依赖项。
- 容器引擎（Container Engine）：容器引擎是一个用于管理和运行容器的软件。

## 2.2 云原生架构

云原生架构是一种基于云计算的架构，它可以实现应用程序的自动化部署、扩展和管理。云原生架构的主要优势是它可以实现应用程序的高可用性、弹性和可扩展性。

云原生架构的核心组件包括：

- Kubernetes：Kubernetes是一个开源的容器管理平台，它可以实现应用程序的自动化部署、扩展和管理。
- Docker：Docker是一个开源的容器化技术，它可以将应用程序和其所需的依赖项打包成一个独立的容器。
- Helm：Helm是一个Kubernetes应用程序包管理器，它可以实现应用程序的自动化部署、扩展和管理。
