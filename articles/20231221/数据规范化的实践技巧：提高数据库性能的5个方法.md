# 2.核心概念与联系

## 2.1 数据规范化的定义

数据规范化是一种数据库设计方法，它的目的是消除数据的冗余和不一致，以提高数据库的性能和质量。数据规范化可以通过以下几个方面来实现：

1. 消除数据的冗余：通过将重复的数据存储在单一的表中，以减少数据的冗余。
2. 提高数据的一致性：通过确保数据在不同的表中的一致性，以减少数据的不一致。
3. 提高数据的可读性：通过将相关的数据存储在同一表中，以提高数据的可读性。

## 2.2 数据规范化的级别

数据规范化的级别分为三个阶段：第一范式（1NF）、第二范式（2NF）和第三范式（3NF）。每个阶段都有其特定的要求和规则，用于确保数据的规范化。

1. 第一范式（1NF）：表中的每个列都有唯一的名称，并且表中的每个列都有唯一的值。
2. 第二范式（2NF）：表中的每个列都有唯一的名称，并且表中的每个列都有唯一的值，并且表中的每个列都有唯一的值。
3. 第三范式（3NF）：表中的每个列都有唯一的名称，并且表中的每个列都有唯一的值，并且表中的每个列都有唯一的值，并且表中的每个列都有唯一的值。

## 4.1 代码实例

在这里，我们将通过一个具体的代码实例来说明数据规范化的实践技巧。假设我们有一个学生信息表，其中包含学生的姓名、年龄、性别和地址等信息。我们的任务是将这个表规范化，以提高数据库的性能和质量。

```
CREATE TABLE students (
  id INT PRIMARY KEY,
  name VARCHAR(255),
  age INT,
  gender VARCHAR(255),
  address VARCHAR(255)
);
```

通过分析这个表结构，我们可以发现它满足第一范式（1NF）的要求，因为每个列都有唯一的名称，并且表中的每个列都有唯一的值。

接下来，我们需要将这个表规范化到第二范式（2NF）。为了达到这个目的，我们需要将学生的性别和地址信息分离到单独的表中，并将它们与学生表关联。

```
CREATE TABLE student_gender (
  id INT PRIMARY KEY,
  student_id INT,
  gender VARCHAR(255),
  FOREIGN KEY (student_id) REFERENCES students(id)
);

CREATE TABLE student_address (
  id INT PRIMARY KEY,
  student_id INT,
  address VARCHAR(255),
  FOREIGN KEY (student_id) REFERENCES students(id)
);
```

通过将学生的性别和地址信息分离到单独的表中，我们可以将学生表规范化到第二范式（2NF）。
