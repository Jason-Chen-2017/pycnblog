2.1. 基本概念解释

在深度学习中，数据加密技术主要分为以下几种：对称加密、非对称加密、哈希加密、安全多方计算等。不同类型的加密技术适用于不同的场景和需求。

2.2. 技术原理介绍：算法原理、具体操作步骤、数学公式、代码实例和解释说明

### 2.2.1. 对称加密

对称加密又称为公钥加密，是一种常见的数据加密方式。其基本原理是，加密和解密使用的是同一个密钥。该密钥在发送端由应用方生成，在接收端由应用方生成并用于解密。

```
public key encryption:
    crypt(message, key) -> ciphertext
    decrypt(ciphertext, key) -> plaintext
```

### 2.2.2. 非对称加密

非对称加密又称为公钥密码，是一种基于密钥的加密方式。其基本原理是，应用方拥有两个密钥，一个用于加密数据，一个用于解密数据。当应用方需要发送数据时，使用密钥加密数据；当接收端需要解密数据时，使用密钥解密数据。

```
public key encryption:
    crypt(message, key) -> ciphertext
    decrypt(ciphertext, key) -> plaintext
```

### 2.2.3. 哈希加密

哈希加密是一种将数据映射到固定长度输出的加密方式。其基本原理是，将输入数据哈希并输出固定长度的输出。哈希算法在计算过程中，无法预测或解密哈希值，因此可以保证数据的安全性。

```
public key encryption:
    哈希(message, key) -> fixed_length_output
     fix_length_input(output) -> original_data
```

### 2.2.4. 安全多方计算

安全多方计算是一种保证多个参与方之间数据安全性的技术。其基本原理是，在参与方之间共享数据，并通过安全的中间人进行数据处理。中间人将数据处理结果返回给参与方，并保证数据的安全性。

```
    参与方B -> 加密数据 -> 中间人 -> 解密数据 -> 参与方A
```
