# 2. 技术原理及概念

## 2.1. 基本概念解释

- 数据库：数据管理的工具，用于存储、管理和访问数据。
- 数据库管理系统（DBMS）：用于管理数据库的软件，通常包括数据创建、操作、查询、更新和备份等功能。
- SQL：结构化查询语言，用于操作数据库，包括数据创建、查询、更新和备份等操作。
- 关系型数据库（RDBMS）：指所有数据都存储在关系型数据库中的数据库管理系统。
- 非关系型数据库（NoSQL）：指不使用关系型数据库的数据库管理系统，如键值数据库、文档数据库等。

## 2.2. 技术原理介绍：算法原理，具体操作步骤，数学公式，代码实例和解释说明

### 2.2.1. SQL基本查询操作

SQL中的基本查询操作包括SELECT、FROM、WHERE、ORDER BY和LIMIT等。

- SELECT：从表中选择满足查询条件的数据。
- FROM：指定要查询的表。
- WHERE：筛选满足查询条件的数据。
- ORDER BY：对查询结果进行排序。
- LIMIT：限制返回的数据条数。

以以下 SQL 查询为例，查询名为“students”的表中，年龄大于等于18岁且成绩大于等于80分的学生的信息：
```sql
SELECT * FROM students
WHERE age >= 18 AND score >= 80;
```
### 2.2.2. SQL中的JOIN操作

JOIN 操作用于将两个或多个表连接起来，形成一个新的表。JOIN 操作包括 INNER JOIN、LEFT JOIN、RIGHT JOIN 和 FULL OUTER JOIN 等。

- INNER JOIN：只返回符合两个表连接条件的数据。
- LEFT JOIN：返回左表中的所有数据，以及右表中与左表中数据匹配的数据。
- RIGHT JOIN：返回右表中的所有数据，以及左表中与右表中数据匹配的数据。
- FULL OUTER JOIN：返回左表和右表中的所有数据，即使右表中没有与左表中的数据匹配的数据，也要返回。

以以下 SQL 查询为例，查询名为“students”的表中，以及“orders”表中，年龄大于等于18岁且成绩大于等于80分的学生的信息：
```sql
SELECT * FROM students
JOIN orders ON students.id = orders.student_id;
```
### 2.2.3. SQL中的UPDATE操作

UPDATE 操作用于更新数据库表中的数据。

-语法：UPDATE table_name SET column_name = new_value WHERE condition;

其中，table_name：要更新的表名，column_name：要更新的列名，new_value：要更新的值，condition：更新条件。

以以下 SQL 更新语句为例，更新名为“students”的表中，年龄大于等于18岁且成绩大于等于80分的学生的成绩：
```sql
UPDATE students
SET score = 90
WHERE age >= 18 AND score >= 80;
```
### 2.2.4. SQL中的DELETE操作

DELETE 操作用于从数据库表中删除数据。

-语法：DELETE FROM table_name WHERE condition;

其中，table_name：要删除的表名，condition：删除条件。

以以下 SQL 删除语句为例，删除名为“students”的表中，年龄大于等于18岁且成绩大于等于80分的学生信息：
```sql
DELETE FROM students
WHERE age >= 18 AND score >= 80;
```
