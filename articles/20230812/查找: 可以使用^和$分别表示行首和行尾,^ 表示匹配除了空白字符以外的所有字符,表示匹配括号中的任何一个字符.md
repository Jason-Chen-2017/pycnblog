# 1.简介
  

查找算法（search algorithm）是计算机科学领域中最重要的问题之一。它的任务是在一个集合或数组中找到一个或多个特定元素（目标值）。许多算法都可以有效地解决这一问题。例如，线性搜索算法可以在一个列表中依次逐个查找一个元素；二分查找算法可以在已排序的数组中快速找到某一特定元素；而哈希表查找算法可以在一个无序数组或链表中快速找到指定的键值对。
在本文中，将主要讨论简单查找算法——顺序查找、二分查找和哈希表查找。其中，顺序查找即从头到尾依次查找指定的值，速度快但效率低下；二分查找是一种更复杂的查找算法，通过折半的方式找到指定值的位置。哈希表查找是基于数组实现的高速查找方法，通过计算索引值映射到对应的内存地址来快速找到指定的值。本文将围绕这几种查找算法进行详细阐述，并给出具体的实例及其相关分析。

# 2. 基本概念术语
## 2.1 数据结构
首先，我们需要理解数据结构的概念。数据结构是指数据的存储、组织和处理方式，包括数组、链表、栈、队列等。数组是一种存储相同类型元素的一组连续内存空间，通过索引访问元素；链表是由节点构成的集合，每个节点保存数据和指针，可以通过指针连接各个节点；栈和队列都是先进后出的容器。

## 2.2 时间复杂度
时间复杂度是指运行算法所需的时间。对于线性查找、二分查找、哈希表查找算法，其时间复杂度如下所示：

* 顺序查找：O(n)

* 二分查找：O(log n)

* 哈希表查找：O(1)

通常情况下，当数据量足够小时，使用顺序查找或哈希表查找会更快一些。当数据量较大时，二分查找的平均时间复杂度更优秀。因此，一般情况下，如果能够确定查找范围，优先选择二分查找算法，否则优先选择哈希表查找。

# 3. 顺序查找
顺序查找（linear search）是最简单的查找算法。其基本思想是从头到尾逐个比较每一个元素，直到找到等于或者刚好大于待查的元素，返回该元素的下标。顺序查找算法的代码如下：

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1 # 没找到
```

例如，在[5, 3, 9, 7, 1, 8]中寻找目标值为7的元素，则执行过程为：

1. 第一次循环：i = 0 ， arr[i] = 5 < 7
2. 第二次循环：i = 1 ， arr[i] = 3 > 7
3. 第三次循环：i = 2 ， arr[i] = 9 < 7
4. 第四次循环：i = 3 ， arr[i] = 7 == 7 ，返回i=3

这种方法比较简单直接，但是时间复杂度很低，为 O(n)。因此，在实际应用中很少采用。

# 4. 二分查找
二分查找（binary search）是一种更加复杂的查找算法，其基本思想是折半查找法。它通过比较中间元素与待查元素的大小关系来决定下一步查找的区域。如果待查元素存在于数组中，那么二分查找算法一定能在 O(log n) 的时间内找到该元素；如果不存在，则必定返回不正确的结果。二分查找算法的代码如下：

```python
def binary_search(arr, low, high, target):
    while low <= high:
        mid = (low + high)//2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1 # 没找到
```

例如，在[5, 3, 9, 7, 1, 8]中寻找目标值为7的元素，则执行过程为：

1. 初始化 low=0, high=5 （数组下标范围）， mid=(0+5)//2 = 2 
2. 比较 arr[mid] 和 target：arr[mid]=7<target
3. 更新 low=3, high=5
4. 下一次循环，mid=(3+5)//2 = 3
5. 比较 arr[mid] 和 target：arr[mid]=9>target
6. 更新 low=4, high=3
7. 下一次循环，mid=(4+3)//2 = 3
8. 比较 arr[mid] 和 target：arr[mid]=7==target ，返回mid=3

二分查找算法的平均时间复杂度为 O(log n)，非常高效。因此，它被广泛应用于各种算法和数据结构的实现中。

# 5. 哈希表查找
哈希表查找（hash table lookup）是基于数组实现的高速查找方法。它利用的是哈希函数的映射关系，把元素映射到数组的索引上。哈希表查找只需花费常数时间就可以完成查找。哈希表查找的步骤如下：

1. 定义哈希函数 h(key) 。该函数接受关键字作为输入，输出是一个非负整数，称作哈希值。
2. 使用哈希值求得索引 j。根据哈希值和数组长度，计算出索引值。
3. 如果索引j处没有元素，则说明元素不存在。
4. 如果索引j处有元素，则比较该元素是否和待查元素相等。若相等，则定位到该元素；若不等，则继续搜索下一个元素。

哈希表查找的平均时间复杂度为 O(1)，且具有抗冲突能力，在不太大的概率下，仍然能够找到目标元素。

# 6. 小结
本文主要介绍了两种查找算法——顺序查找和二分查找。顺序查找时间复杂度为 O(n) ，即遍历整个数组才可找到元素；二分查找时间复杂度为 O(log n) ，最坏情况下需遍历 log₂ n 个元素。同时，本文也介绍了哈希表查找，它是一个基于数组实现的高速查找方法。
