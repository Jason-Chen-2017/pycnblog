# 1.简介
  

## 一句话总结
Google Cloud Bigtable是谷歌公司推出的NoSQL键值存储服务。它提供了高度可扩展、高可用、低延迟的数据存储服务，并支持分布式数据处理、分析和实时查询。本文将从产品介绍、系统架构、核心特性和优势等方面进行介绍。
## 名词解释
- NoSQL（Not Only SQL）：不仅仅是关系数据库，还包括非关系型数据库。相比于关系型数据库，NoSQL更注重高吞吐量、高可用性和易扩展性，适用于大数据、实时数据处理场景等。
- Key-Value Store：键值对存储系统，其中的每个键都对应一个值。键值存储一般由哈希表实现，读写速度快，但不提供顺序遍历、范围查询等功能。
- BigTable：一种基于Google F1硬件构建的分布式结构化存储数据库。它是一种能够快速处理海量数据的NoSQL存储数据库。
- Table：BigTable中的最小存储单元，是一个行列存放的二维数组。表由多个Tablet组成，每个Tablet是一个存储文件的集合。
- Row Key：每张表中唯一的一列，用作定位行的主键。Row Key通常采用整数或者字符串形式。
- Column Family：同一族内的不同列共享相同前缀。Column Family可以有不同的属性，如最多版本数、压缩设置等。
- Cell：单元格，代表了某个时间戳下的特定行的特定列的值。Cell的大小不受限。
- Cluster：集群是运行在GCE虚拟机上的一个集群。一个集群包含多个表和磁盘。集群具有主备模式，当主节点出现故障时，备节点会自动接管工作负载。
- Region：Region是一个逻辑划分，每个Region都有一个中心节点和多个数据节点。Region用于提升性能，防止数据倾斜。一个集群可以跨越多个Region。
- Mutation API：用于向BigTable写入数据的方法。Mutation允许用户设置时间戳，可批量写入数据。
- Read/Write Path：读写路径决定了客户端如何访问BigTable数据库。客户端首先连接到最近的集群，然后向所选表发送请求。Read Path支持过滤、索引和缓存。Write Path提供事务保证和冲突检测机制。
- Secondary Indexes：搜索索引可以让用户根据某些列或条件快速检索数据。Secondary Index允许创建多级索引，加快检索速度。
- Data Locality：数据局部性指的是数据的访问模式，即一个数据项被最近访问过的地方附近的数据也很可能被访问。BigTable通过将热点数据放在固态硬盘上，可以较好地满足数据局部性要求。
- Regional Clustering：区域集群使得数据能被分布在多个区域，有效减少网络延迟。同时，它还可以利用GCP内部的高速互联网连接，实现低延迟访问。
- Scalability and Performance：BigTable是一种分布式结构化存储数据库，因此可以横向扩展，提高存储容量和处理能力。它的并发控制策略可以有效地管理内存资源消耗。
- Automatic Failover：如果主节点发生故障，备节点会自动接管工作负载，保证服务可用性。
- Global Scale：BigTable具备全局分布式的特性，可以跨越全球任何地方，为客户提供低延迟和高吞吐量的数据存储服务。
- Cost-Effective：BigTable提供免费的体验和廉价的价格，使其成为许多企业的首选选择。
