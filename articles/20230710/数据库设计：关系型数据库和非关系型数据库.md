### 2. 技术原理及概念

### 2.1. 基本概念解释

关系型数据库 (RDBMS) 和非关系型数据库 (NoSQL) 是两种主流的数据库技术。关系型数据库采用关系模型，数据以表格形式存储，具有结构化、规范化的特点。非关系型数据库采用非结构化数据模型，数据以文件、键值对形式存储，具有高度可扩展性和灵活性。

### 2.2. 技术原理介绍

2.2.1 关系型数据库 (RDBMS)

关系型数据库采用关系模型，将数据组织成一个个表格，其中每个表格包含行和列。每个行代表一个实体，每列代表实体的属性。关系型数据库具有规范化的数据结构，适合存储结构化、规范化的数据。

2.2.2 非关系型数据库 (NoSQL)

非关系型数据库采用非结构化数据模型，将数据存储为文件、键值对等数据结构。非关系型数据库具有高度可扩展性和灵活性，适合存储非结构化、半结构化、高度可扩展的数据。

### 2.3. 相关技术比较

| 关系型数据库 | 非关系型数据库 |
| --- | --- |
| 数据结构：规范化的表格 | 非结构化数据模型 |
| 数据模型：关系模型 | 非结构化数据模型 |
| 存储方式：文件、SQL | 内存、缓存 |
| 索引：传统索引 | 全文索引、列族索引、哈希索引 |
| 事务支持：支持 | 不支持 |
| 数据一致性：高 | 低 |
| 适用场景：对数据结构要求较高的场景 | 对数据结构要求较低的场景 |
