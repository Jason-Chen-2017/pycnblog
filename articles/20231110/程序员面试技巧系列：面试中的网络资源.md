                 

# 1.背景介绍


## 一、互联网产品创新生态
### 1.1 一流团队技术能力
互联网公司如雨后春笋般涌现出了一批优秀的技术人才，其中，一流团队往往占据上风。一流团队技术能力可以体现在很多方面，比如：
- 技术修炼：对于一流技术人员来说，除了技术本身，还需要持续不断地学习提升自己的编程水平。许多一流技术人员会从事一些平台级开发工作，比如大数据、安全、高并发等领域。他们经过长时间的努力和锤炼，逐步掌握了解决复杂问题的能力。
- 沟通表达能力：在创业初期，公司的高管及部门之间往往存在很大的沟通差异，比如重点业务及细节问题的沟通、项目进展与管理的沟通、部门之间的沟通等。但随着时间的推移，一流技术人员的沟通表达能力日益强化，能够更好地和各种角色的人员沟通交流。
- 团队协作精神：一流技术人员往往积极参加集体活动，包括内部技术分享、线下技术沙龙、开源社区及技术会议等。这样的参与方式既能够加强技术人员间的沟通交流，又能培养其对行业及时信息的关注和获取能力。
- 个人成长能力：即使没有充分的培训和科研投入，一流技术人员也能通过自我驱动、刻苦训练、踏实执行等方式，自我成长。
### 1.2 海量技术资料无处不在
互联网作为世界性的高新技术传播者，技术信息的爆炸已经成为不可抗拒的趋势。每天都有大量新的技术论文、工具、书籍、视频、教程等内容涌现出来。而这些资料又难免受到质量门槛的限制，让许多技术人员望而却步。实际上，我们很少遇到质量很好的技术文章，因为互联网上的文章质量总是不太高。但是，既然有那么多的技术资料，那么如何才能让我们的技术水平不断提升呢？
除了技术人员自身的积累之外，还有几个重要的因素影响着技术资料的质量：
#### （1）权威渠道封锁
相比起像维基百科这样的权威渠道，国内的技术博客网站尤为缺乏用户的普及率和点击率，不利于技术知识的传播。国内互联网公司大都选择部署自己的技术博客，由于它们具有较高的知名度，所以虽然内容可能会少于权威渠道，但是用户量仍然会远远超过权威渠道。
#### （2）资料的过时性
与前文所述的权威渠道的封锁不同，互联网上的技术资料往往都会存在一定的时效性。前些年发布的技术文档或工具可能已经不适应当时的情况，而即使现在重新调研，仍然无法取得同样的效果。
#### （3）海量的技术信息
互联网的蓬勃发展已经造成了越来越多的技术信息涌现出来。而信息的泛滥也给技术人员提供了很大的学习和实践机会。
### 1.3 商业模式赋予技术垄断地位
互联网公司的崛起与疯狂背后，是商业模式的急速转变。随着经济的飞速发展，传统制造业的衰落，互联网的崛起已经将工业界带到了一个全新的高度。根据统计显示，2017年中国互联网用户规模达到19.2亿，占GDP的比例达到33%左右，较2013年翻了近一番，成为最大的智能手机市场。因此，无论是企业还是个人，都充分认识到互联网公司正在改变着全球产业链，拥有巨额的金钱和影响力。在这个过程中，技术也是最重要的一个元素。
当今世界，绝大多数的创业企业都在抢夺用户的时间，而互联网公司却真正享有垄断地位，这一点无疑是非常危险的。当消费者无法自由选择的时候，互联网巨头们肩负着巨大的商业责任，他们既有价值也有权力，他们可以任意的定价、选择服务的对象、甚至定向推广某种商品或服务，从而将用户的注意力引导到自己看得顺眼的页面。另一方面，这种垄断权力的确会削弱其他创业者的竞争力，因为新的创业者就要站在互联网巨头的对立面，寻找更有价值的新方向。

基于上述因素，当下的技术创业者还需要更加开阔眼界，更多的关注技术的价值所在。抛弃传统的技术博客网站，尝试关注那些值得关注的技术博客网站。这就是“程序员面试技巧系列：面试中的网络资源”系列文章的目的，帮助大家了解互联网上的最新技术资源，为自己的技术演进打下坚实的基础。
