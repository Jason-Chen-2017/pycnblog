## PCI DSS（Payment Card Industry Data Security Standards）标准

PCI DSS是一个信用卡行业数据安全规范。它确立了一系列的标准和要求，用于保障在PCI信用卡交易系统中存储、处理及传输个人身份信息（PII）数据的隐私性、完整性和可用性。该标准从收集、存储、传输、使用、共享、销毁等几个方面对PCI系统中的数据进行了详细的管理。而与此同时，PCI又推出了PCI SAQA（Security Assurance Qualification Assessment Program），作为其认证考试方案，对所涉及的各个方面的安全控制措施进行自我评估和测试，并将结果反馈给PCI信用卡组织。

随着互联网金融发展，用户越来越多地选择线上支付服务，导致用户的支付信息在互联网上的流动。为了确保用户的支付信息的安全，PCI推出了PCI DSS标准。DSS规范包括四大部分：

1. 收集与存储
	- 数据分类
		- PII数据
		- 个人识别信息（PII）数据是指那些可以直接或者间接与身份相关的信息，例如名字、地址、电话号码、信用卡号码、社会保险号码、医疗记录号码等。
	- 数据存储
		- PII数据应被妥善保管，包括备份、加密、访问权限控制等。
		- 需要定期审计和改进访问控制措施，避免出现不可预知的风险。
		- 不同级别的授权人员应具有不同的访问权限，应制定明确的数据使用和数据泄露协议。
		- 对传输过程中可能出现的风险（例如中间人攻击、数据篡改或恶意破坏），应制定适当的防御手段。
	- 系统监控
		- 通过日志跟踪和监控系统的运行状况，发现异常行为并做出响应。
		- 在事件发生时及时通知相关人员。
		- 为系统管理员提供可靠的数据源，建立实时警报。
	- 数据报告和审计
		- 每年需要向PCI组织提交报告，说明PCI业务中的安全风险、法规遵从情况、数据安全工作状态等。
		- 报告的内容应包括PCI业务中产生的个人身份信息数据安全风险、法律义务和违规情况、工作流程、有效应急措施、风险管理机制等内容。
		- 提交的报告应该为审计员提供一份权威的参考资料，可以用于合作和评估工作。
2. 使用
	- 不要在网络上传输敏感信息。
		- 用户上传的PII数据必须使用加密传输，并且严格保护用户密码和密钥。
		- 通信内容必须使用SSL/TLS协议加密。
		- 使用VPN进行端到端加密。
		- 在登录页面上强制要求输入验证码或其他形式的身份验证。
		- 对于数据连接过程中的用户输入，必须通过服务器端的校验，确保输入的正确性。
		- 不要在URL、POST参数、HTTP header中传输敏感信息。
	- 对接收到的信息进行适当的保护。
		- 只允许授权人员访问数据。
		- 定期审查访问日志，确保只有授权人员才有权访问数据。
		- 不要将敏感信息和个人信息分享给无关人员。
	- 注意PCI安全最佳实践。
		- 将PCI DSS应用于所有PCI业务流程。
		- 制定风险评估、监控、应急、合规和审计机制。
		- 充分关注PCI云服务，确保其安全运营。
3. 分发
	- 使用加密方式传输数据。
		- 不要在不安全的环境下传输数据，包括移动设备和不受信任的网络。
		- 定期更新加密算法和密钥。
		- 确认使用的第三方服务提供商和技术支持符合PCI标准。
		- 如有必要，可购买保护数据完整性的产品和服务。
	- 消除数据泄露风险。
		- 开发适当的应急预案，在发生数据泄露事件时及时通知相关人员。
		- 为用户生成删除请求单，使数据尽快从系统中删除。
		- 定期检查系统和应用程序的漏洞，及时更新补丁和升级。
4. 销毁
	- 应主动销毁敏感数据。
		- 当用户账户关闭或退订时，请删除相关联的敏感数据。
		- 删除敏感数据后，请立即在系统中清理痕迹。
		- 对于存储在硬盘上的数据，请设定密码或密钥，确保仅授权人员可以访问数据。
