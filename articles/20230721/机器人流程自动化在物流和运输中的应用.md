# 2.基本概念术语说明
## RPA（Robotic Process Automation）
> 机器人流程自动化(RPA)是一个基于计算机控制的方法，用于改善企业内部运营和管理，通过高度自动化的办公过程，减少人因介入带来的成本和重复劳动，从而提高工作效率、降低运营风险。RPA的关键特征包括完全的自动化，没有人类参与，使用的工具和技术都由计算机编程语言或脚本编写。它的应用范围涉及各个行业，包括金融、制造、零售、服务等多个领域。

## 概念理解
### 流程自动化的定义
流程自动化是指通过计算机控制的办公过程，通过高度自动化的方式实现各种工作的执行，包括人工手动流程的替代、批量数据的处理、自动信息提取等。

流程自动化分为两种类型：

1. 基于规则的流程自动化:根据预先设定的条件执行指定的任务。如邮件处理，根据收件人的邮箱地址发送确认电子邮件。

2. 无人值守的流程自动化:基于现有的计算机模型，通过交互的方式实现特定功能。如HR助理系统，通过收集表单数据和图片识别，进行员工招聘筛选。

### 机器人流程自动化(RPA)的特点
1. 全自动化:不需要人类的参与，全过程被机器自动完成。例如，当接收到客户订单时，机器人会自动填写所有表单并发出发货指令，无需人工操作。

2. 可定制化:不同的业务场景需要不同的RPA解决方案，可以灵活调整。例如，针对采购部门的需求，可以使用不同的模板，生成不同的报告，提升整体的效率。

3. 可扩展性强:支持自定义脚本，轻松应对复杂业务流程。例如，可以编写复杂的算法来处理巨大的文件，提升处理速度。

4. 数据驱动:流程自动化系统能够利用外部数据源，进行流程优化和自动化决策。例如，可以将销售数据导入系统，识别出高转化率的产品，自动触发促销活动。

5. 系统集成:通过集成第三方系统，可以实现跨部门的协同工作。例如，将HR系统的数据导入采购系统，完成采购请求的快速响应。

## 技术分类
RPA技术主要按如下几个方面分类：

1. 文本处理：主要包括文本数据提取、数据清洗、信息提取、自动文书批准等。

2. 文件处理：主要包括文档转换、数据导入导出、批量数据处理等。

3. 操作系统：用于控制计算机运行环境、打开应用程序等。

4. 智能接口：通过图形界面或API调用智能硬件设备，如工业机器人、图像扫描仪等。

5. 模块化设计：用于模块化设计、重用代码，提高复用能力。

6. 服务部署：可以通过云服务平台，实现远程部署和维护。
