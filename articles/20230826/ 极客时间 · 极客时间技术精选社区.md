# 2.基本概念术语说明
在技术系列文章中，我们可能会用到各种各样的术语、概念。下面是一些重要的术语或概念的解释，供读者参考。

## 1.数据结构
数据结构(Data Structure) 是计算机编程中的重要概念。它是计算机存储、组织、管理数据的集合。数据结构决定了数据元素之间的关系、组成和访问方式。数据结构是计算机中最基本的构造块，几乎所有的高级语言都提供了相应的数据结构支持。数据结构可以分为抽象数据类型(Abstract Data Type, ADT)和具体数据类型(Concrete Data Type)。

抽象数据类型一般用于定义数据类型以及它们之间关系和操作。例如，栈是一种抽象数据类型，它是一种特殊的序列，用于存放数据项，而队列则是另一种抽象数据类型，是一种FIFO (First In First Out)的数据结构。抽象数据类型与具体实现类之间的关系是通过接口和协议机制来实现的。例如，栈可以使用LIFO (Last In First Out)的数据结构实现，队列也可以使用顺序表或链表实现。抽象数据类型通常具有一些标准的方法或操作，包括创建、插入、删除等，通过这些方法可以实现具体数据类型的操作。

具体数据类型是指某种特定的抽象数据类型，如栈、队列等。具体数据类型通常是一个数组或链表，其元素由数据类型中的域和值组成。具体数据类型中的元素被组织成特定的逻辑结构，例如栈中的元素按照先进后出的方式存储；队列中的元素按照先进先出的顺序排列。具体数据类型可以直接用于各种应用程序。

## 2.数据编码
数据编码(Data Encoding) 是计算机编程中用到的一种编码方式，它将数据转换为二进制形式，从而在计算机中进行处理。常用的编码方式有ASCII、UTF-8、GBK、ISO-8859-1、Base64等。

ASCII是最早出现的字符编码方案，只用7位二进制表示所有的英文、数字和一些符号，它主要用于显示现代文本文件。由于它的兼容性，ASCII编码也是现今使用最普遍的字符编码。但是，由于它只能用来显示文本文件，对于二进制文件的处理就比较麻烦。当我们处理图像、音频、视频等二进制数据时，通常采用专用的二进制编码方案。比如，JPG图像就是采用JPEG编码，PNG图像就是采用PNG编码，MP3音频就是采用MP3编码，AVI视频就是采用MPEG-4编码。

## 3.加密算法
加密算法(Encryption Algorithm) 是通过某种规则把明文数据变换成密文数据，只有使用相同算法的解密算法才能重新恢复原来的明文数据。常见的加密算法有DES、AES、RSA、RC4、MD5、SHA1等。

DES（Data Encryption Standard），一种对称加密算法，速度较快，安全级别较高，目前仍然是企业级应用的标准。它使用56位密钥，对每个数据块进行加密，密钥长度为64位。

AES（Advanced Encryption Standard），一种对称加密算法，速度快，安全级别高，比DES稍微强些，已经成为个人和商业领域里最流行的加密算法。它使用128位密钥，对每个数据块进行加密，密钥长度为128位或192位。

RSA（Rivest–Shamir–Adleman）加密算法，是公钥加密算法，它可以对消息加密，私钥用于解密。RSA算法基于数论理论，是目前最有影响力的公钥加密算法之一。

RC4（Rivest Cipher 4），一种流密码，速度很快，安全性也很低，但已被AES替代。它不需要密钥就能生成随机的密钥流，并且易受各种攻击。

MD5（Message Digest Algorithm 5），一种哈希算法，它通过对输入的信息产生唯一的输出，其结果是固定长度的值。MD5的设计目标就是容易计算，而且不同的输入得到不同的输出，这一特性适合用于验证文件完整性。

SHA-1（Secure Hash Algorithm 1），一种哈希算法，和MD5类似，不同的是SHA-1摘要长度是160位。它的设计目标是强壮的抗碰撞能力，已被美国国家安全局（NSA）推荐为FIPS 180-2规范中的默认算法。

## 4.Paxos算法
Paxos算法是一个分布式算法，用于解决一致性问题。Paxos算法采用了两种消息：prepare请求和promise响应。

Paxos算法的角色有Proposer、Acceptor、Learner三种。其中，Proposer是提议者，负责发送prepare请求，发起投票；Acceptor是响应者，负责接受来自proposer的请求，并给予promise响应。learner是最终决策者，在接收到多数派的promise响应之后，根据Promised Number的大小，对Value进行决策，最终确定Value的值。

Paxos算法非常适合于解决分布式系统中存在多个节点的协调的问题，特别是在保证强一致性的情况下。Paxos算法的正确运行依赖于提案唯一性、保证消息传递顺序、超时机制等机制。
