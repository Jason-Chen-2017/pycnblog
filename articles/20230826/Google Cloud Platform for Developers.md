# 1.简介
  

## Google Cloud Platform(GCP) 是什么?
Google Cloud Platform 由一系列服务组成，包括计算、存储、网络、机器学习、APIs 和应用等资源，提供可靠、安全、可扩展且经济高效的基础设施服务。你可以通过 GCP 提供的各种服务，快速、方便地创建、部署、扩展你的应用程序、数据库或其他 IT 工作负载。
Google Cloud Platform 是基于 Google 的服务器托管服务商（G Suite）的全栈云服务解决方案，包括系统工具、管理工具、开发工具、基础设施即服务 (IaaS)，平台即服务 (PaaS) ，软件即服务 (SaaS)。在 Google Cloud Platform 上你可以构建、测试、运行、扩展以及管理应用程序，无论是在 Google 的云端还是私有云中。此外，它还包括分析、机器学习、数据流处理、容器和函数计算等多个领域的工具及服务。

## 为什么要选择 GCP？
如果你想获得最佳的产品性能、响应速度和可靠性，选择 GCP 就很合适。GCP 是一个全栈平台，包含了用于构建、测试、运行、扩展以及管理应用程序的各种组件，其中包括计算、网络、存储、数据库、分析、机器学习、开发工具和 API 等服务。GCP 服务全面且统一，你不必担心混乱的服务集成问题，也不用花费时间去为每个服务调配配置参数。

另一个重要原因就是 GCP 的费用结构，它提供了一定的免费额度，让你在尝试新服务时可以快速上手。如果你需要更高的容量和并发能力，或者需要更多的数据处理、存储空间，GCP 可以按需付费，而且价格不贵。如果你的业务需要高度的可用性，那么 GCP 则是最好的选择。

## Google Cloud Platform 的优点有哪些？
- 高可用性: Google Cloud Platform 的基础设施都有冗余备份保障，你可以放心地利用这些基础设施。并且，GCP 提供了多区域的服务，这样就可以实现灾难恢复。另外，Google 还提供系统监控、日志记录、故障排查、备份恢复和定期维护等服务。
- 便捷性: Google Cloud Platform 提供丰富的管理工具，例如云监控、账单管理、IAM 角色权限控制、操作审计等。它还提供简单易用的接口和 SDK 来帮助你实现自动化和编程。
- 可伸缩性: 你可以根据自己的需求增加或减少 GCP 集群中的节点数量。因此，你的应用程序始终会得到最好的服务质量。
- 低成本: 使用 GCP 并不需要任何投入，你只需付费使用那些实际使用的服务即可。它的定价模式也非常经济实惠。
- 无限的弹性扩展: 在 Google Cloud Platform 中，你可以随时增加或者减少集群节点的数量，以满足你的应用的快速增长或减少。这种弹性扩展使得 GCP 在调整和优化应用程序服务方面具有超强的能力。

# 2.基本概念和术语
## 概念
### 项目(Project): 是 GCP 中的核心概念之一，用来组织和隔离相关资源。你可以在一个项目中创建资源并进行管理。比如，你可以创建一个项目用于开发某个应用程序，或者创建一个项目用于测试特定数据集上的模型训练结果。

### 区域(Region): 是 GCP 物理位置的名字。不同区域有不同的物理位置和带宽，这使得它们可以提供不同的网络延迟和可用性水平。

### Zonal 和 Regional 机房：这是两种不同类型节点的描述方式，但它们代表的都是同一个概念。Zonal 机房表示的是运行于单个区域中的节点，而 Regional 机房则表示的是跨越多个区域的节点。

### 资源(Resource): 是 GCP 能够提供的计算和存储等实体。你可以通过资源来创建虚拟机实例、云存储桶、数据集等等。

### 标签(Label): 是资源的一个属性，你可以使用标签来标记资源。标签不会影响资源的实际使用，只是给予其一些附加信息。你可以通过标签来组织和查询资源，从而更好地了解资源的使用情况。

### 卷(Volume): 是持久化存储的一种机制，你可以将它附加到虚拟机上，并将本地文件系统和外部磁盘分区映射到该卷上。你可以在卷之间复制和共享数据。

### 快照(Snapshot): 是卷数据的静态副本，你可以在需要的时候使用快照来回滚到之前状态。

### 操作(Operation): 是一项异步任务，对一项 GCP 操作执行后会返回一个 Operation 对象，你可以通过该对象来查看任务进度和状态。当任务完成时，它会返回一个状态码，告诉你是否成功。

## 术语表
| 术语 | 描述                                                         |
| ---- | ------------------------------------------------------------ |
| API  | Application Programming Interface，应用程序编程接口，是一个界面，应用程序可以使用它与 GCP 服务通信。 |
| BOM  | Bill of Materials，物料清单，是对所购买产品的摘要，包含所有购买项目的名称、规格、价格、相关税率、小计金额和权重百分比等详细信息。 |
| CDN  | Content Delivery Network，内容分发网络，是基于全局的网络互联网，利用最短路径路由策略，向用户提供内容。 |
| CLI  | Command Line Interface，命令行界面，是指用户通过键盘输入指令的方式访问计算机。 |
| DDOS | Distributed Denial of Service，分布式拒绝服务攻击，是一种网络攻击手段，通过大量的网络请求占据网站资源，使服务器瘫痪，甚至导致系统瘫痪。 |
| DNS  | Domain Name System，域名系统，是互联网的一套协议，它将域名解析为 IP 地址。 |
| IAM  | Identity and Access Management，身份和访问管理，是 GCP 控制访问、管理用户和访问权限的一种机制。 |
| IAP  | Identity-Aware Proxy，身份感知代理，是一种基于网络的访问控制机制，可以对经过身份验证的用户访问 Google Cloud Platform 内的资源。 |
| JSON | JavaScript Object Notation，JavaScript 对象表示法，是一种轻量级的数据交换格式，它采用了键值对形式。 |
| KMS | Key Management Service，密钥管理服务，是一种云服务，它为用户管理加密密钥提供安全存储、管理、分配和轮换功能。 |
| L7   | Load Balancing Layer 7，第七层负载均衡器，是一种根据网络流量的源目的IP地址、端口号、协议类型进行负载均衡的设备。 |
| NAT  | Network Address Translation，网络地址转换，是一种方法，可以在私有网络内部使用公网 IP 地址，以访问 Internet。 |
| PIA  | Privacy Information Access，隐私信息接入，是指个人信息的收集、处理、传输、储存、删除、使用和分享的一系列流程、技术和制度。 |
| PPA  | Personal Protective Equipment，个人防护装置，是指在国际共产主义运动期间，为了保卫个人生命财产不受第三方侵犯而采用的法律武器。 |
| RBAC | Role Based Access Control，基于角色的访问控制，是一种授权机制，通过角色定义人员在不同范围内可以进行的操作，可以精确地控制对资源的访问权限。 |
| SLA  | Service Level Agreement，服务级别协议，是客户与云提供商之间关于云服务的契约。 |
| VPN  | Virtual Private Network，虚拟专用网络，是建立在公共网络上的安全私有网络，只能允许特定的用户连接到网络。 |
