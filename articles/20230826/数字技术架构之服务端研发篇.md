# 2.基本概念术语说明

　　1、操作系统（Operating System）

　　操作系统(OS)是管理计算机硬件与软件资源的程序，是用户与计算机硬件设备之间进行交互的接口，它控制硬件完成各种系统功能，分配处理器时间片，并提供各种服务给用户进程。操作系统控制着整个计算机系统，它是计算机硬件、各种软件和数据资源的管理者和调度者。操作系统是计算机软硬件系统的核心部件，是计算机系统的基石，它是独立于应用程序运行的最小的软件集合，负责管理底层硬件资源并向上提供可靠的服务。如Windows、Unix、Linux、MacOS等。

　　Intel公司的x86架构CPU(或称作PC机架构)是现代计算机中最常用的处理器架构，其操作系统一般包括四个部分：Boot Loader、Kernel、Application Programming Interface (API)、System Call Interface。其中，Boot Loader负责引导操作系统启动，读取操作系统的第一块硬盘扇区；Kernel是操作系统的核心部分，包括核心程序和驱动程序；API是提供给上层应用编程接口，实现系统调用；System Call Interface则是在应用程序请求操作系统服务时，由操作系统内核提供的一种特殊的指令，它负责从用户态切换到内核态，执行系统调用，然后再切换回用户态，将结果返回给应用程序。

  　　2、Web Server

　　Web服务器（英语：Web server）是一个运行在计算机上的服务器软件应用程序，它按照HTTP协议接收客户端发送的请求，解析客户端的请求信息，生成动态网页内容并返回给客户端。也就是说，它可以把网页当做一种服务提供出来，客户端通过浏览器访问这个服务，就像直接访问一个网站一样。

　　Web服务器软件通常包括Web服务器软件、数据库服务器软件、FTP服务器软件和其他辅助工具软件。最常见的Web服务器软件有Apache HTTP Server、nginx、Microsoft IIS和lighttpd。

　　其中Apache HTTP Server是目前使用最广泛的Web服务器软件，可以运行在 Linux、Unix、Mac OS X、Solaris 操作系统平台上。除了HTTP协议支持外，它还可以运行CGI（Common Gateway Interface）、FastCGI（Fast Common Gateway Interface）和Perl语言脚本。FastCGI 是一种基于进程模型的接口，它的工作方式类似于传统 CGI ，但由于它在 Web 服务器和应用程序之间的连接建立在第一次调用后就保持着，因此在同一个 Web 请求中可以避免重复加载，提升 Web 服务的性能。Apache HTTP Server 支持PHP、ASP、JSP、Perl、Python、Ruby、Tcl等多种脚本语言。它同时支持IPv4、IPv6双栈协议。

  　　3、应用程序容器

　　应用程序容器（Application Container）是一个轻量级、隔离环境的虚拟环境，用来托管和运行独立的应用软件。它可以提供一个环境，让应用运行在一个独立的沙箱环境中，提供一个独立的进程空间，因此降低了应用间的耦合性，防止出现互相影响的问题。应用程序容器可以让应用与运行环境隔离开来，使得应用的部署和维护更加方便、可控。目前常见的应用程序容器有Docker、LXC（Linux Containers）和Rocket。

  　　4、中间件

　　中间件（Middleware）是一个软件框架，提供了一个应用服务的整体平台，它允许多个应用系统集成到一起工作，实现各个系统之间的信息共享和交流，适用于大规模分布式应用系统。中间件通常包含消息队列、配置中心、服务注册与发现、负载均衡、日志收集、认证授权、缓存、事务管理等功能模块。

  　　5、数据库

　　数据库（Database）是长期存储、管理和维护数据的仓库，它按照结构化查询语言（SQL）组织数据，提供数据的查询、更新和删除功能。目前使用最普遍的数据库有MySQL、Oracle、PostgreSQL、MongoDB等。

  　　6、缓存

　　缓存（Cache）是指将暂时不必要的数据暂存到磁盘中进行保存，以便下次访问时可以快速获取。缓存的目的是减少对原始数据源的频繁访问，提高数据的响应速度。缓存分为两种类型：内存缓存和磁盘缓存。内存缓存又称作常驻内存，是指将热点数据复制到内存中缓存起来，以提高系统的访问效率。磁盘缓存又称作分级缓存，是指将热点数据写入磁盘，以减少内存的占用空间。
