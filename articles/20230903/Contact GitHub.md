# 2.基本概念术语说明
## 2.1 GitHub
GitHub是一个基于Web的开源代码仓库托管服务，也是目前最热门的同性交友网站之一。GitHub秉承着开放透明、协作共赢的价值观，将软件源代码及其相关资料进行有效集中存储、整理，并通过统一界面、API接口、工具来提高软件开发者的工作效率。Github为全球软件开发者提供了一种可靠、安全、开放的软件共享平台。
### 2.1.1 用户账户
每个注册了GitHub账号的用户都会获得一个独一无二的账号，用于身份认证和数据保存。
### 2.1.2 仓库（Repository）
仓库（Repository）是存放文件的地方，类似于文件夹一样，每个仓库可以包含多个文件夹和文件，每个仓库都有一个唯一且公开的URL地址。仓库分为公开仓库（Public Repository）和私有仓库（Private Repository）。公开仓库对所有人都是可见的，任何人都可以克隆或下载到本地查看，而私有仓库则必须要邀请才能访问。
### 2.1.3 分支（Branch）
分支（Branch）是用来开发不同特性或功能的独立线路，它与主干开发（Master Branch）同步，确保主干上的最新改动不会影响开发分支的稳定性。
### 2.1.4 撤销操作（Revert Operation）
撤销操作（Revert Operation）可以恢复某一个提交版本的文件状态，并且可以在没有远程连接的情况下使用，十分方便快捷。
### 2.1.5 Issue
Issue是一个用来跟踪BUG、需求、任务或者其它事务的机制。每个仓库可以单独设置Issue模板，将固定的格式要求贴在新建的Issue中。Issue有助于管理产品的迭代流程、反馈意见以及跟进开发进度。
### 2.1.6 Pull Request
Pull Request（拉取请求）是一个用于通知项目维护者已完成代码修改并希望将修改合并到项目主干中的请求。当某个分支的代码被接受时，可以通过创建Pull Request来展示自己的代码，并征求其他人的意见或建议。
### 2.1.7 Fork
Fork是GitHub的一个特殊功能，允许用户将其他用户的仓库复制到自己的账户下，然后在自己的仓库中进行修改后提交，最后再向原始仓库提交Pull Request，让原作者进行审核和合并。这样就可以在自己的仓库中贡献代码到其他人的开源项目中。
## 2.2 Git
Git是一个开源的分布式版本控制系统，用于敏捷高效地处理复杂的分布式系统。Git 的设计目标就是取代 SVN，因为 SVN 在某些方面（特别是网络传输）性能较差。由于 Git 是分布式版本控制系统，不需要一个中心服务器来统一管理版本信息，每个人的电脑上都可以有完整的版本历史。而且，Git 可以直接记录每次更新，生成专业的更改日志，而 SVN 只能记录隐藏的文本变化。
### 2.2.1 工作区域（Working Directory）
工作目录（Working Directory）即用户在本地磁盘上看到的仓库，里面存放着项目的元数据以及对象数据库。
### 2.2.2 暂存区（Staging Area/Index）
暂存区（Staging Area/Index）是一个临时的存储区，它像个缓存区，临时保存当前准备提交到HEAD的改动。
### 2.2.3 HEAD
HEAD指向的是当前的提交版本，也就是说当前处于哪个版本号，即使提交历史改变了，HEAD永远指向当前版本号。
### 2.2.4 分支
分支是用来开发不同特性或功能的独立线路，它与主干开发（Master Branch）同步，确保主干上的最新改动不会影响开发分支的稳定性。
### 2.2.5 远程库（Remote Repository）
远程库（Remote Repository）即托管在互联网或者局域网上的代码仓库。所有的版本库都有对应的远程版本库，通常用来和本地版本库保持一致。

我们可以从以下几个角度理解GitHub：
1. 版本管理：Git是一个开源的分布式版本控制系统，GitHub是Git版本管理工具的提供商，而且GitHub提供付费版和免费版两种服务，商业版具有更加强大的功能，如多人协作、私密仓库、企业版、项目管理、代码审查等。
2. 协作平台：GitHub为协作工作提供了强大的平台支持，包括代码托管、协作工具、项目管理、反馈沟通等。通过GitHub，我们可以轻松的创建、管理、协作开源项目。
3. 宣传平台：GitHub建立于2008年，创始人是克里斯托弗·史蒂文斯（<NAME>），GitHub的使命是“使用GitHub，让世界各地的开发者协作解决复杂的问题”。GitHub是一个免费的软件服务，提供了强大的功能和社交服务，它鼓励人们在自己的代码库中分享知识和技术，而非专注于公司的业务。

## 3.1 GitHub工作流程
GitHub的工作流程可以简单概括为：
1. 创建仓库：首先创建一个空的仓库（repository），然后添加文件到这个仓库。
2. 提交变更：将本地仓库中的文件提交到GitHub上的仓库。
3. 分支：当需要同时开发不同的特性或功能的时候，可以使用分支功能，在主分支（master branch)基础上创建一个新的分支（branch)，然后在新分支上开发特性或功能。
4. 合并：当开发完成一个特性或功能，就可以将该分支合并到主分支（master branch)。
5. Pull Requests：当合并到主分支之后，就可以创建一个Pull Request，通知其他成员进行代码审查。如果代码审查通过，就可以合并代码到主分支。

## 3.2 Git工作流程
Git的工作流程可以简单概括为：
1. 初始化仓库：创建一个空的Git仓库，然后添加文件到这个仓库。
2. 检出仓库：检出仓库，即切换到仓库所在位置，并获取最新代码。
3. 添加文件：将文件添加到暂存区，并提交到HEAD。
4. 提交变更：将暂存区的文件提交到HEAD，同时更新仓库的历史记录。
5. 分支：当需要同时开发不同的特性或功能的时候，可以使用分支功能，在主分支（master branch)基础上创建一个新的分坝（branch)，然后在新分支上开发特性或功能。
6. 合并：当开发完成一个特性或功能，就可以将该分支合并到主分支（master branch)。
7. 冲突：当两个分支中存在相同的部分代码时，就会发生冲突。解决冲突的方法一般是手动编辑冲突文件，然后提交。
8. 拉取请求：当合并完成后，可以在仓库页面上创建一个Pull Request，请求将分支上的代码合并到主分支。
9. 同步仓库：当仓库的远程版本库有新版本时，可以同步仓库，获取最新代码。

## 3.3 使用GitHub进行版本管理
GitHub提供了一系列操作，可以帮助我们轻松的进行版本管理。
1. 创建仓库：点击右上角的“New repository”按钮，填入仓库名称，选择仓库类型（public or private），勾选Initialize this repository with a README，然后点击“Create repository”。
2. 克隆仓库：点击仓库页面的Clone or download按钮，然后选择HTTP URL，复制仓库的URL。然后打开命令行终端，输入git clone + URL，克隆仓库到本地。
3. 查看仓库：点击仓库页面的Code按钮，然后点击浏览器中的“Open with GitHub Desktop”按钮，GitHub Desktop会自动检测本地版本库，并显示仓库信息，包括仓库名称、创建时间、最近更新时间、星标数量、描述、大小等。
4. 推送变更：点击GitHub Desktop左下角的“Commit to master”按钮，输入提交消息，然后点击“Sync”按钮，GitHub Desktop会自动把本地仓库的代码提交到GitHub上的仓库。
5. 创建分支：点击GitHub Desktop的“Current branch”标签，然后点击“New branch”按钮，输入分支名称，点击“Create branch”，创建分支。
6. 提交变更：在分支上做修改，然后点击GitHub Desktop的“Commit to [分支名称]”按钮，输入提交消息，然后点击“Sync”按钮，GitHub Desktop会自动把本地仓库的代码提交到GitHub上的对应分支。
7. 将分支合并到主分支：点击GitHub Desktop的“Current branch”标签，选择“master”分支，点击“Merge into current branch”按钮，然后选择“[分支名称]”分支，点击“Merge [分支名称]”按钮，将分支合并到主分支。
8. 创建Pull Request：点击仓库页面的Pull requests按钮，点击“New pull request”按钮，然后选择分支，点击“Create pull request”按钮，填写Pull Request信息，点击“Create pull request”按钮，发送Pull Request。

## 3.4 使用Git进行版本管理
Git提供了一系列命令，可以帮助我们轻松的进行版本管理。
1. 设置用户名和邮箱：执行如下命令：
    ```
    git config --global user.name "Your Name"
    git config --global user.email "Your Email"
    ```
2. 初始化仓库：执行如下命令：
    ```
    mkdir myproject # 创建项目目录
    cd myproject     # 进入项目目录
    git init         # 初始化本地仓库
    ```
3. 克隆仓库：执行如下命令：
    ```
    git clone https://github.com/username/myproject.git   # 从GitHub克隆仓库到本地
    git clone git@github.com:username/myproject.git    # 从SSH克隆仓库到本地
    ```
4. 查看仓库：执行如下命令：
    ```
    ls -a      # 查看仓库文件列表
    tree       # 以树状图形式查看仓库文件
    ```
5. 添加文件：执行如下命令：
    ```
    touch file1.txt   # 添加文件
    echo 'hello world' >> file1.txt   # 写入内容
    git add file1.txt                # 添加文件到暂存区
    ```
6. 删除文件：执行如下命令：
    ```
    rm file1.txt                   # 删除文件
    git rm file1.txt               # 删除文件，并添加到暂存区
    ```
7. 提交变更：执行如下命令：
    ```
    git commit -m "commit message"  # 提交变更，并添加提交消息
    git push origin master          # 把本地代码推送到GitHub
    ```
8. 创建分支：执行如下命令：
    ```
    git checkout -b new-feature     # 创建新分支，并切换到新分支
    git branch                      # 查看分支列表
    ```
9. 切换分支：执行如下命令：
    ```
    git checkout master             # 切换回主分支
    ```
10. 合并分支：执行如下命令：
    ```
    git merge new-feature            # 把new-feature分支合并到当前分支
    git branch -d new-feature        # 删除new-feature分支
    ```
11. 创建tag：执行如下命令：
    ```
    git tag v1.0                    # 为当前版本打tag
    git push origin --tags           # 把tag推送到GitHub
    ```
12. 拉取远程代码：执行如下命令：
    ```
    git fetch                         # 获取远程代码
    git rebase FETCH_HEAD             # 合并远程代码到本地代码
    ```
13. 解决冲突：如果在合并分支时出现冲突，则需要解决冲突。首先，Git无法自动合并冲突文件，需要我们手动解决。然后，手动编辑冲突文件，把它的内容删除掉，保留我们想要的代码。最后，再提交修改后的文件，继续合并分支即可。
