# 1.简介
  


数据工程（Data Engineering）是指利用计算机系统及软件工具处理、存储和分析海量数据的过程。与软件开发不同，数据工程通常涉及更复杂的数据处理流程和更丰富的任务需求。由于数据规模的激增和多样化，数据工程师需要具备高度的系统思维能力、计算机科学知识、统计学知识等。

作为一名数据工程师，你的职责是设计和构建能够支持业务发展的数据基础设施。在工作中，你将协助公司或组织进行数据采集、清洗、转换、加载、分析、可视化、模型训练、模型评估和部署等数据工程相关的任务。下面介绍一些可能会被聘用到数据工程部门的公司或者岗位，以帮助大家更好的了解这个职业。

本文主要介绍以下13个数据工程职位供应方面的信息，希望可以给读者带来一些参考：

1. 数据工程师
2. 数据架构师
3. 大数据分析师/总监
4. 数据仓库工程师
5. 数据治理工程师
6. 数据质量保证工程师
7. 数据挖掘工程师/主管
8. 机器学习工程师/研究员
9. 流程建模工程师
10. 数据采集工程师
11. 数据开发工程师
12. 高级数据分析师
13. 数据平台工程师

# 2.数据工程师

数据工程师负责管理和维护数据基础设施，包括收集、存储、清理、整合、处理、分析和报告数据。他们必须对数据的结构、样式、和价值有深刻理解，并有效地运用数据科学和技术手段进行探索性分析。

这些角色往往会和科学家、工程师或IT经理一起组成一个团队。他们有能力解决复杂的问题，从而能真正提升业务的效率。例如，数据工程师可以建立数据仓库、分析平台、数据集市或数据接口，从而提升公司的数据管理能力。

数据工程师通常具有以下要求：

1. 数据处理、存储和管理的专业知识：数据工程师必须熟悉各种数据库和文件系统，掌握SQL语言，还应该了解大数据处理的相关工具如Hadoop、Spark等。

2. 抽象思维能力：数据工程师必须具备高度的抽象思维能力。他们应该能够理解数据模型的层次结构、关系、模式，并且能够识别出数据的价值和意义。

3. 分析和数据驱动决策能力：数据工程师需要了解业务需求，对业务流程和商业模式有深入的理解。他们应该善于使用统计、数学、机器学习和图表技术，以及建立有见地的模型，以便对数据进行分析、预测和决策。

4. 文档编写能力：数据工程师必须擅长文档的编写，能够清晰地阐述项目的信息和进展。同时，他们也要善于沟通，保持与团队其他成员的交流。

数据工程师可能适合以下职位：

1. 数据分析师/分析总监: 数据分析师一般是指具备一些数据分析能力和分析方法的人。他们通过研究数据和业务发展趋势，寻找数据的价值并制定数据分析策略，以帮助公司提升产品价值。他们还必须做好数据安全和隐私保护工作，确保分析结果不被泄露或滥用。

2. 数据科学家：数据科学家是一个全面而专注的角色，他和数据分析师一样，要对数据进行深入的分析，来提升公司的产品价值。数据科学家往往要擅长抽象思维、逻辑推理、分析方法、数据可视化、数据建模等技能。同时，他们还要了解统计学、机器学习、编程语言等多个领域知识。

3. 数据产品经理/项目经理: 数据产品经理和项目经理都可以承担数据工程师的角色。他们负责设计数据产品和服务，包括数据建模、数据报告、仪表盘、数据接口、数据集市等。他们还需要对业务流程、市场环境有深入的理解，以确保产品符合用户需求。

# 3.数据架构师

数据架构师的主要职责是设计和实现企业的数据平台。他们一般有能力定义整个数据生命周期的各个阶段所需的数据结构、功能和性能。此外，数据架构师还可以结合业务需求和目标客户群体，制定相应的数据平台方案。

数据架构师可能适合以下职位：

1. 数据总监/经理：数据总监或经理通常负责管理整个数据平台。他们必须了解各个部门的数据需求，并且在数据平台上实施数据整合、数据集成、数据仓库、数据仓库规划、数据模型设计等工作。他们还要制定数据资源的配置计划、工具选型和使用规范。

2. 数据分析总监/经理：数据分析总监/经理通常会协助数据总监或经理设计数据平台上的数据分析功能。他们必须具有深厚的统计学和数学背景，能够理解业务需求和发展趋势，制定数据分析的方案并通过数据模型建立数据价值洞察。

# 4.大数据分析师/总监

大数据分析师/总监负责设计和构建大数据平台，包括数据采集、处理、存储、分析、查询和应用。他们必须具有良好的分析、建模和处理能力，掌握大数据框架和工具，包括Hadoop、Spark、Pig、Hive、Presto等。

除了具备大数据分析师的一般技能之外，大数据分析师/总监还必须精通SQL语言、NoSQL数据库、数据挖掘技术、机器学习算法、特征工程等领域知识。

大数据分析师/总监可能适合以下职位：

1. 数据分析师/分析总监：大数据分析师和分析总监一般都是独立的角色，但两者都应该具备大数据分析和建模的能力。他们首先需要理解业务需求和市场趋势，然后通过大数据技术、分析方法、工具和技术，提炼出数据价值，以帮助公司降低成本、增强竞争力。

2. 数据平台工程师：数据平台工程师一般和大数据分析师、分析总监配合工作。他们设计和实现数据平台的各个组件，包括数据采集、存储、处理、分析和应用等环节。他们还负责数据资源的配置、工具选型和使用规范的制定。

# 5.数据仓库工程师

数据仓库工程师负责设计和构建数据仓库，包括ETL（Extract-Transform-Load）流程和数据仓库模型。他们必须掌握SQL语言和相关数据库，以及ETL工具，如Sqoop、Flume、Impala等。同时，他们还应该掌握数据分层、数据质量控制、维度建模、多维分析、统计建模、OLAP引擎等技能。

数据仓库工程师可能适合以下职位：

1. 数据仓库管理员：数据仓库管理员负责管理整个数据仓库，包括数据流向、清理、规范化、维度建模、数据建模、ETL、优化、监控等。他们必须对数据仓库的生命周期有充分的理解，并且能够快速响应业务的变化，并提升数据分析的效率。

2. 数据模型er：数据模型er负责构建数据仓库模型。他们必须理解业务需求，并且能够通过数据建模工具和技术，提升数据分析效率、减少数据误差、提升数据可视化效果、促进业务分析。

# 6.数据治理工程师

数据治理工程师主要负责对数据的采集、存储、处理、访问、共享、变更、贯彻和保护等方面，确保数据质量、安全、可用性、完整性和一致性。他们必须对数据产生、使用和保护的生命周期有深刻的理解，并持续关注数据整合、数据质量、数据安全、数据共享、数据使用和数据保护等问题。

数据治理工程师可能适合以下职位：

1. 数据保护主任：数据保护主任负责管理公司内部或外部的数据保护事务，包括制定数据分类、合规性管理标准、定期数据备份、数据披露等。他们也必须负责对数据进行分类、加密、压缩、归档和备份，确保数据的可用性、完整性、一致性和安全。

2. 数据主题专家：数据主题专家一般是指有丰富的专业背景、对数据敏感度高且擅长解决复杂的问题的人。他们通过对数据的分析、挖掘、模型训练和分析，能够发现数据中的模式和趋势，并通过提升数据分析和决策效率，提升公司的竞争力。

# 7.数据质量保证工程师

数据质量保证工程师负责对数据质量、准确性、完整性、一致性、时效性、可用性、审计和合规性等方面进行评估、分析、检测和改进。他们必须掌握数据质量的理论知识，具备数据质量建设和改进的执行能力。此外，数据质量保证工程师还要对企业的数据敏感度高，具有一定竞争力。

数据质量保证工程师可能适合以下职位：

1. 数据质量测试工程师：数据质量测试工程师通常对数据的品质有较高的要求，因此，他们需要掌握基于规则、度量、自动化和人工审核等方法，对数据的质量进行全面检查。

2. 数据分析师：数据分析师一般是指具备一些数据分析能力的角色。他们通过研究数据和业务发展趋势，寻找数据的价值并制定数据分析策略，以帮助公司提升产品价值。同时，他们还需要掌握SQL语言、数据挖掘、机器学习和统计学等技能。
