# 1.简介
         
 Code Complete是由Steve McConnell撰写的一本著作，该书着重于编写高质量的代码，而不是教授如何编程。它的目标就是帮助开发人员改进他们的代码技能，包括代码的结构、设计模式、命名规范、注释、可读性等方面。该书全面的总结了软件工程实践中最佳编码实践、模板方法模式及其他各类编码原则，并给出了详细的操作指南，帮助开发人员实现可维护的、可扩展的、可复用的代码。通过阅读本书可以了解到什么样的编程方法才能编写出更优秀的代码，也会掌握一些常见的编码错误及其避免的方法。此外，这本书还包括了在软件工程领域的不同行业相关的代码实践，如系统架构设计、安全编程、Web应用开发等方面。
         # 2.目录
         本书共分七章，分别是：第一章《Introduction》介绍了Code Complete这个书的由来、编写这个书的背景和意图，还有作者对自己的期望；第二章《Getting Started》主要介绍了编码从无到有、从新手到专家需要具备哪些基本知识、编码风格、工具、测试技术和编码习惯的重要性；第三章《Smalltalk Best Practices》介绍了Smalltalk语言中的一些实用设计模式，以及这些设计模式如何适应大型项目的编码要求；第四章《Object-Oriented Programming》提供了一系列基于类的设计原则和技巧，旨在帮助开发人员编写更好的面向对象代码；第五章《Coding Standards》讨论了各种类型的代码标准，如缩进、变量名、注释、函数签名、命名空间、异常处理等，并提供了相应的代码实践建议；第六章《Software Design》提供了软件设计的不同阶段（需求分析、设计、实现、测试）所需遵循的最佳实践；第七章《Conclusion》对Code Complete做了一个总结和展望。
         # 3.前言
         在现代社会里，软件已经成为生产力支柱和经济基础。软件工程师正变得越来越重要，因为软件在改变着世界，成为连接人类与自然的桥梁。计算机科学与软件工程是一门独立的学科，它既有许多看上去不相干的研究领域，比如数学、物理、生物、心理、哲学等。但它们有很多共同之处，例如都涉及计算机的结构、计算理论、算法和数据表示等方面。因此，当一个软件工程师走进某个公司时，就有必要了解整个软件工程的过程、方法和原理。只有理解并熟练运用这些原理，才能提升个人的能力和工作效率。同样，每个软件工程师也应该根据自己的经验、爱好、志愿等方面，制定合适的学习计划。本书中的例子、图片、图表、参考文献，都会提供足够的帮助。
         # 4.关于我
