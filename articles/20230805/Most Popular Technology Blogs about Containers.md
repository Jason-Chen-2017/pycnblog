        # 3.基本概念术语说明
         ## 什么是容器？
         容器（Container）是一种轻量级虚拟化技术，它使得应用程序可以打包成独立的运行环境，运行在宿主机（Host）上，共享主机内核，同时具有自己的进程空间和网络空间。容器和传统虚拟机（VM）不同之处在于它不需要完整的操作系统作为运行环境，只需要一个小型的用户模式Linux内核，这使得启动速度加快、资源占用减少，并为云平台上的应用程序提供了一个沙箱环境。容器的概念最早由美国国家安全局提出，最初被称为“沙盒”。

         ## Docker
         Docker是一个开源的容器引擎，可以让开发者打包和部署应用程序，让不同的容器之间可以互相沟通、交流。它最初是在2013年发布的，基于Go语言编写，并以Apache License v2许可证开源。目前，Docker已成为全球最流行的容器引擎。

         ## Kubernetes
         Kubernetes是一个开源的容器集群管理系统，可以让你方便地创建，调度和管理容器化的应用。它最初由Google团队开发，并在2015年发布，基于Go语言编写，并以Apache License v2许可证开源。Kubernetes基于容器技术的核心理念是“容器编排”，通过声明式的方式管理容器化的应用。

         ## OCI (Open Container Initiative)
         OCI (Open Container Initiative)，即开放容器计划，是一个开放的开放源码组织，致力于围绕容器标准化社区，推动容器运行时和管理规范的发展。OCI定义了容器格式、镜像格式、运行时接口以及集群管理接口。

         ## CNCF (Cloud Native Computing Foundation)
         Cloud Native Computing Foundation，即云原生计算基金会，是一个由Linux基金会推出的非营利组织。它的目标是构建和维护一个全栈的云原生计算平台，以支持容器、微服务、无服务器计算、API 和应用编程模型。CNCF通过制定规范、工具、最佳实践、框架和平台等，鼓励云原生计算的发展。
