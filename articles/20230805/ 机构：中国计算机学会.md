# 1.简介
         
1978年成立于北京，是全国性、跨部门的研究、开发和应用计算机科学技术的刊物，其主要任务是：推广、普及、加强并长期运用计算机技术，促进科学技术和经济文化的进步。
         1998年1月，“中国计算机学会”与清华大学、中国科学院共同组建的“清华计算机科学技术协会”、“北京计算技术促进中心”（后改称“北京计算智造中心”）联合成立了“中国计算机科学技术学会”。“中国计算机学会”的宗旨是“推动计算机科学的科技进步、经济发展与社会进步”，坚持“立场新知、理论先行、创新引领”，“贯彻‘九无一异’精神”，推动计算机科学在我国的快速发展。
         2007年，“中国计算机学会”正式更名为“中国计算机学会计算机科学技术委员会”，直指“计算机科学技术”这个领域，通过建立工作动态发布平台、举办培训课程、组织学术会议、参加国际学术会议、举办期刊和报纸发行等方式，积极探索新的计算机技术，开拓视野，培育青年才俊，服务国家的兴亡。
         2010年底，“中国计算机学会”和“中国计算机科学技术学会”合并，成为一个整体实体。2013年12月，“中国计算机学会”作为信息通信业和计算机系统业重要力量的代表，在世界一流水平设立“中国信息通信业计算机技术委员会”。
         1986年，第一次全国计算机系统设计大赛成功举办。“中国计算机学会”提出“国产软件第一奖”、“国产软件开发者奖”、“计算机安全奖”等多项荣誉奖项。
         # 2.认识计算机科学
         1946年，“计算机科学”被设定为自然科学的一门分支。它是利用数学方法解决的问题。美国麻省理工学院教授Alan Turing的著作《Computing Machinery and Intelligence》中，把计算机科学定义为“研究如何构建自动计算机器，使其拥有独立于特定领域的、灵活而复杂的能力”。1950年，由图灵奖获得者艾伦·图灵在剑桥大学举办的图灵奖颁奖典礼上，提出了“图灵机”概念。计算机始于此时。
          1956年，Donald Knuth博士发现了一种新的计算机编程语言——Fortran。当时的计算机语言有汇编语言、C语言和Lisp语言。1962年，贝尔实验室的克劳斯·莫瑟爵士在ACM的图灵奖题目下设计了第一次真正意义上的图灵机。随后的电子计算机问世，并迅速蔓延到整个计算机界。
          1971年，科学家们提出了著名的冯诺依曼结构。他将数据存储、运算和控制功能结合到了一起，使计算机具有超级计算能力。1973年，IBM发明了第一个微型计算机。到目前为止，计算机已经发展得非常成熟，有着各种各样的应用场景。
         # 3.计算机的发展历史
          1.20世纪50-60年代，IBM、惠普和微软分别领导了PC机和工作站市场，并取得了非凡的发展。这些产品改变了个人电脑的形象，改变了用户对计算机的认识。
          2.20世纪70-80年代，微软公司决定推出新的操作系统Windows，其独特的窗口管理系统掀起了PC市场的浪潮。这一革命在当时引起了芬兰乔治・马库斯·安德森的注意，他提出了基于微软Windows的Macintosh操作系统。该系统为苹果电脑带来革命性的变化，也成为今日苹果电脑的基础。
          3.20世纪80-90年代，雅虎、谷歌和Facebook等互联网公司崛起，掌握了Internet的主导地位。随之而来的则是一场“互联网泡沫”崩盘。虽然如此，但网络的兴起让PC市场保持竞争优势。
          4.20世tplv年代，随着互联网的发展，微软、HP、惠普、IBM等厂商纷纷涉足硬件领域。AMD、英特尔、戴姆勒、三星等生产计算机组件的厂商都试图推出自己的芯片，以实现对PC性能的追求。
          5.20世纪90年代末，通用电气宣布放弃PC市场，转而投资于服务器市场。服务器成为主流，并慢慢取代PC成为企业的标配。
          6.2010年，谷歌宣布开源手机操作系统Android，为全球所有人带来便利。尽管如此，移动端市场依旧是竞争激烈的领域。
          7.2015年，微软宣布收购GitHub，并开启了GitHub占据主流地位的开源社区。2016年，苹果iOS系统迎来重大更新。
         # 4.计算机的种类及分类
         ## 4.1 数量层次
           1.中央处理器(Central Processing Unit, CPU)
              中央处理器(CPU)又称为“中控核心”或“引擎”，它是整个计算机系统的核心，负责执行各种指令。通常，一个计算机系统中只有一个CPU，不过也可能有多个CPU（多核），以提高计算速度。CPU通常被集成在一块小型晶体元件中，能够执行基本算术运算、逻辑运算、串口输入输出、定时和计时功能，还可执行浮点运算、整数运算、特殊运算、加密运算等多种类型指令。
              CPU的主要部件有程序计数器、寄存器、ALU(Arithmetic/Logic Unit)、控制单元等。

           2.随机存取存储器(Random Access Memory, RAM)，简称内存(Memory)。
              内存是计算机存储数据的地方，是最快的数据交换和处理设备。RAM是随机存取存储器，用于临时存储程序、数据及指令，可以实现信息快速检索。它的容量比硬盘小很多，价格便宜。

           3.显卡(Graphics Processing Unit, GPU)
              显卡(GPU)是图形处理单元，它用来在屏幕上显示图像。一般来说，PC端的CPU处理速度较慢，因此，出现了显卡。GPU的主要功能是进行快速的三维图形渲染、光栅化和视频处理。显卡的分支还有矢量处理单元VPU、超级图像处理器SIPP。

           4.输入设备、输出设备及接口设备。
              计算机中的输入设备包括键盘、鼠标、扫瞄仪、摄像头、陀螺仪、触摸板等；输出设备包括打印机、显示器、扬声器等；接口设备包括网卡、USB接口、串口接口等。

         ## 4.2 功能层次
           1.应用层: 是人与计算机之间的接口，用户通过应用层来访问计算机的资源，比如文字处理软件、办公自动化软件、网络游戏等。应用层直接面向最终用户，提供一系列的功能和服务，如Word、Excel、PowerPoint、Chrome、Safari、QQ、Skype等。

           2.操作系统层: 操作系统(Operating System, OS)是控制硬件设备和向应用层提供服务的软件。它管理着计算机系统的所有资源，分配给应用运行需要的硬件资源，提供应用层所需的服务，如提供进程调度、文件管理、网络通信、设备管理等。不同的操作系统往往都有自己独特的特性，比如Linux和Windows。

           3.网络层: 网络层(Network Layer)是在TCP/IP协议簇的最上层，负责网络通信方面的功能，比如路由选择、连接建立、地址转换等。网络层的数据包传输到达接收端后，再通过应用层来解析并传递给应用层。

           4.传输层: 传输层(Transport Layer)是一个抽象概念，它为应用层提供端到端的通信服务。传输层的功能包括端口号的绑定、流量控制、差错控制、可靠性保障、包排序和重复消除等。

           5.应用层: 应用层(Application Layer)是计算机系统中最高的一层，它为最终用户提供各种应用服务。应用层向下接入传输层，然后通过网络层传送数据。应用层支持各种网络服务，比如Web服务、邮件服务、文件传输服务等。

         # 5.计算机体系结构
         ## 5.1 概念
           计算机体系结构(Computer Architecture)是指计算机系统的硬件、软件及固件的总称。它是一个完整的计算机体系，包括了处理器结构、存储器结构、I/O设备结构、总线结构、编译系统、输入/输出系统、系统软件及网络结构等多个模块和环节。

           在计算机体系结构中，有两种类型的指令：定制指令(Custom Instruction)和系统调用指令(System Call Instruction)。定制指令是指为了提高某些特定的应用需求而设计的指令集合，比如高性能计算、图形处理等。系统调用指令是指允许软件或操作系统向操作系统内核请求服务的指令。

           计算机体系结构通常采用结构化方法，即把计算机系统分成若干个部件，每一部件有自己固有的功能，相互之间可以通过一定接口进行通信和交流。体系结构可以分为五大层次：
           1.硬件层：它包括处理器、存储器、输入/输出设备、总线等。
           2.操作系统层：它包括操作系统、系统调用接口、应用程序接口等。
           3.编译系统层：它包括编译器、汇编器、链接器等。
           4.网络层：它包括网络设备、协议栈、网卡等。
           5.应用层：它包括应用软件、运行环境、API(Application Programming Interface)等。

         ## 5.2 分支结构
           1.单板计算机(Single Board Computer, SBC)
              单板计算机(SBC)是一种比较简单的计算机系统结构。它只由一个主板、一个CPU、一些存储设备组成，没有其他外设。这种结构适用于嵌入式系统、智能手机等。

              一款常见的嵌入式系统就是树莓派(Raspberry Pi)。它由两块SD卡、一个MIPS级处理器、一张LCD显示屏、一个扬声器、一个按键、一个扩展卡和一个WIFI模块组成。虽然它的尺寸很小，但却可以轻易运行各种各样的应用程序。

           2.服务器
              服务器(Server)是一种多功能的计算机系统。它包括了一块主机板、内存、CPU、磁盘、网络适配卡、输入/输出设备等。它提供了众多的网络服务，如文件共享、数据库服务器、电子邮件服务等。服务器以共享的方式运行，可以同时处理许多客户请求。

           3.个人电脑(Personal Computer, PC)
              普通的个人电脑(PC)由主机板、处理器、内存、硬盘、显示器、机箱、网络接口卡、USB接口、外部设备等组成。PC可以运行多种操作系统，其中包括Windows、Linux、macOS、Android和IOS等。

              2007年发布的全新笔记本电脑(Laptop)也由主机板、处理器、内存、硬盘、显示器、机箱、网络接口卡、USB接口、USB设备、声卡、Touchpad、键盘、鼠标、摄像头等组成。它的尺寸很大，重量也不轻，但是它的配置较高，可以在工作、学习和娱乐的时候使用。

           4.集群服务器
              集群服务器(Cluster Server)是一种服务器的配置，它由多个服务器节点组成，以提升整体的效率和容错能力。它提供的服务和普通服务器相同，但整体性能得到了提升。

           5.云计算
              云计算(Cloud Computing)是一种通过网络提供计算机资源的服务模型。云计算服务商通过网络将其硬件、软件及服务提供给消费者。消费者通过Internet、移动终端或其他方式就近登录云计算服务商，并根据需要使用计算机资源，可以像使用本地计算机一样，享受高性能、高可用性和可伸缩性。

         # 6.计算机网络
         ## 6.1 分类
         ### 6.1.1 OSI模型
            1.应用层
               应用层(Application Layer)为运行在不同操作系统上运行的应用提供服务，如HTTP、FTP、DNS、Telnet、SSH、SMTP等。它负责网络通信过程中消息的传递、接收、格式化、安全以及错误恢复。

            2.表示层
               表示层(Presentation Layer)用于信息的编码与解码，如ASCII、UTF-8、JPEG、MPEG、TLS等。它负责对应用层发送的消息进行加工，以便网络中的两个实体之间能够正常通信。

            3.会话层
               会话层(Session Layer)用于建立、维护、管理会话。它确保两台计算机之间的数据传输的完整性，避免数据丢失和篡改，并协调通信双方的行为。

            4.传输层
               传输层(Transport Layer)用于为两台计算机之间的数据传输提供通路，它负责对数据包进行分割、重组、传输、校验、编码、解码、安全性等。

            5.网络层
               网络层(Network Layer)用于互连计算机之间的数据传输，它负责寻址、路由和传输数据包，还要处理网络错误、流量控制和拥塞控制。

            6.数据链路层
               数据链路层(Data Link Layer)负责数据帧的封装、解封装、传输、错误检测、重发、透明传输等。

            7.物理层
               物理层(Physical Layer)负责传输介质的调制解调、信号传输、噪声、衰减、加工等。它负责计算机系统中所有传输信道的管理。

         2.协议栈
             协议栈(Protocol Stack)是一系列按照顺序进行数据交换的规则。每个协议都定义了网络通信的过程、格式和目标。

             协议栈中经常使用的协议包括TCP/IP、IPv4、IPv6、UDP、ICMP、HTTP、FTP、SNMP、TLS、DHCP、SMB、NFS、SAMBA、SSDP、NetBIOS、ARP、RARP、VLAN、QoS等。

         ### 6.1.2 TCP/IP模型
             TCP/IP模型(Transmission Control Protocol / Internet Protocol)是一套标准协议族，由一系列协议组成，它们定义了网络互联互通的细节。

             每个协议都定义了一种功能，例如：
             1.网络互联：TCP/IP协议族的最主要协议是TCP/IP协议，它规定了数据在网络上传输的规则。
             2.域名系统：域名系统(Domain Name System, DNS)是一个分布式数据库，它把网站域名和IP地址联系起来。
             3.网络管理：网络管理(Network Management)协议定义了网络管理的过程，包括网络管理、路由管理、策略管理等。
             4.互联网邮件：互联网邮件(Internet Mail)协议定义了邮件的格式、交换方式、寄信人的验证、发送、接收、储存等。
             5.网络时间同步：网络时间同步(Network Time Synchronization)协议定义了计算机时间与UTC时间的同步过程。
             6.万维网：万维网(World Wide Web, WWW)协议定义了浏览器与服务器之间如何通信，以及数据的传输。
             7.文件传输：文件传输协议(File Transfer Protocol, FTP)定义了两个计算机之间的文件传输过程。
             8.电子商务：电子商务(Electronic Commerce, EC)协议定义了电子商务网站的功能和流程。
             9.远程访问：远程访问(Remote Access)协议定义了远程终端如何连接到网络。

         ## 6.2 技术标准
           1.Internet RFCs (Request for Comments) RFC文档是网络相关技术的规范。RFC文档详细定义了各种协议标准、接口标准、数据格式标准、操作系统标准和网络管理标准等。
           2.IEEE Stds (Institute of Electrical and Electronics Engineers, IEEE) 美国电气与电子工程师协会(Institute of Electrical and Electronics Engineers, IEEE)制订了专业的标准。
           3.IEEE 802.3u (Intel) 以太网标准规范802.3u是微处理器局域网(MIH)协议的一种，它提供基于IEEE802.3 LLC的低时延、无比吞吐量和互操作性。
           4.IEEE P802.1X (IEEE) 以太网认证标准802.1x是一种安全机制，它用于对网络访问控制。
         # 7.计算机编程语言
         ## 7.1 编程语言分类
        ### 7.1.1 脚本语言
        脚本语言(Script Language)是一种解释性的编程语言，其源代码不是预先编译成机器语言，而是作为数据来源、被解释器直接运行。

        常用的脚本语言有JavaScript、Perl、Python、Ruby等。它们是解释型语言，不需要编译，运行速度快，并且可以轻松嵌入其他应用程序。
        ### 7.1.2 面向对象语言
        面向对象语言(Object-Oriented Programming Language)是一种编程语言，它将程序看作一组对象的集合，每个对象封装了一个数据和一组操作，对象间通过消息传递进行通信。

        常用的面向对象语言有Java、C++、C#、Python、PHP等。它们是面向对象的语言，支持封装、继承、多态、抽象、多线程、异常处理等特性。

        ### 7.1.3 编译型语言
        编译型语言(Compiled Language)是指预先把源码编译成机器代码，再运行的编程语言。

        编译型语言通常包括C语言、C++语言、Java语言等。它们的编译过程十分复杂，需要编译器的帮助才能完成。

        ### 7.1.4 解释型语言
        解释型语言(Interpreted Language)是指运行时动态分析源码，根据源码逐条执行的编程语言。

        解释型语言通常包括Python、JavaScript、Perl语言等。它们不需要编译，运行速度快，但不能脱离环境运行。

    ### 7.2 主要语言
       目前，计算机领域主要的编程语言有：

        1.高级语言(High-Level Language)：这类语言由符号式的代码描述，并利用编译器或解释器生成高效的机器代码。如Java、C#、Python。
        2.低级语言(Low-Level Language)：这类语言直接将指令翻译成二进制代码，不依赖于编译器或解释器。如汇编语言、机器语言。
        3.脚本语言(Script Language)：这类语言由解释器直接运行，通常与操作系统及工具集成。如JavaScript。
        4.专用语言(Specialized Language)：这类语言是针对某种特定的任务或领域，如游戏编程语言、数值计算语言等。

    # 8.软件工程
     ## 8.1 软件开发模式
        软件开发模式(Software Development Pattern)是指一套完整的软件生命周期，它定义了软件开发的各个阶段及其相互关系。
        ### 8.1.1 概念
        软件开发模式(Software Development Pattern)是指一套完整的软件生命周期，它定义了软件开发的各个阶段及其相互关系。软件开发模式是一种设计方法和方法论，旨在指导软件开发人员遵循一定的方法、准则和标准，以有效地进行软件开发。软件开发模式有助于将业务、系统开发、维护和部署等活动组织起来，更好地进行系统的设计、开发、测试、评估和改善。

        ### 8.1.2 模式分类
        根据模式的类型，软件开发模式可以分为以下几类：
        1.创建模式(Creational Patterns)：创建模式是一种用来描述如何创建对象的模式。创建模式关注的是实例化对象的过程，以满足特定的应用要求。创建模式有工厂模式、抽象工厂模式、单例模式、原型模式、Builder模式、迭代器模式等。
        2.结构模式(Structural Patterns)：结构模式关注类的组合结构，用于组织类和对象的形式和结构。结构模式有代理模式、桥接模式、组合模式、装饰器模式、外观模式、享元模式、职责链模式、模板方法模式、状态模式、策略模式等。
        3.行为模式(Behavioral Patterns)：行为模式是用来描述对象之间的通信和交互方式。行为模式处理对象之间的职责分配、通信、协作、状态转换等。行为模式有命令模式、解释器模式、迭代器模式、Mediator模式、备忘录模式、Observer模式、Reactor模式、状态模式、模板方法模式、Visitor模式等。
        4.并发模式(Concurrency Patterns)：并发模式描述了如何构造具有多线程或异步处理功能的应用。并发模式有并发容器模式、同步原语模式、并发框架模式、Future模式、Actor模式等。
        5.健壮性模式(Robustness Patterns)：健壮性模式用于处理软件的运行环境、数据损坏、安全漏洞等问题。健壭性模式有恐慌模式、防火墙模式、限流模式、隔离模式、演练模式、数据源模式等。
        6.实践模式(Practice Patterns)：实践模式提供了经过实践验证的模式和最佳做法。实践模式有工程模式、发布模式、架构模式、项目模式、接口模式等。

        ### 8.1.3 模式应用
        实际应用中，软件开发模式经常会应用到以下几个方面：
        1.设计模式：软件设计过程中，可以使用各种设计模式来提升代码的复用性、可读性、可维护性和可扩展性。
        2.模式语言：软件模式语言是一种计算机语言，专门用于编写、理解和实施软件设计模式。
        3.敏捷开发：敏捷开发是一种短小而高效的开发方式，它以用户需求为驱动，采用迭代的方式完成开发工作。
        4.微服务架构：微服务架构模式是一种分布式系统架构模式，它将单体应用分解为多个小型服务，每个服务运行在独立的进程中，可以独立部署、独立扩容和升级。

    # 9.云计算
    ## 9.1 云计算概述
       云计算(Cloud Computing)是一种通过网络提供可伸缩计算、存储、应用服务的一种计算资源模式。它可以降低IT基础设施成本，使企业享受到公有云、私有云、混合云、社区云等多种形式的服务。

       云计算平台提供计算、存储、应用等资源给用户，用户可以根据需求使用这些资源，从而节约资金投入、提高工作效率和资源利用率。目前，主流的云计算平台有AWS、Azure、Google Cloud Platform等。

       云计算平台提供的服务主要分为以下几类：
       1.基础设施服务：提供基础设施服务，如计算、存储、网络、数据库、消息队列、文件存储、CDN等。
       2.平台服务：提供平台服务，如开发框架、应用托管、安全管理、监控告警等。
       3.第三方服务：提供第三方服务，如AI、大数据、云安全、机器学习等。
       4.应用程序服务：提供应用软件服务，如云应用、SAAS应用、PaaS应用、FaaS应用等。

    ## 9.2 AWS(Amazon Web Services)
    Amazon Web Services(AWS)是亚马逊云计算服务的市场领先者，是全球领先的公有云服务提供商之一。

    AWS产品主要分为以下几类：

    1.计算服务：AWS提供了多种类型的计算服务，如EC2、Lambda、Lightsail、Batch等。
    2.存储服务：AWS提供了多种类型的存储服务，如S3、EBS、Glacier等。
    3.网络服务：AWS提供了多种类型的网络服务，如VPC、API Gateway等。
    4.数据库服务：AWS提供了多种类型的数据库服务，如DynamoDB、RDS等。
    5.分析服务：AWS提供了多种类型的分析服务，如Athena、Emr、Redshift等。
    6.开发工具包：AWS提供了多种类型的开发工具包，如CodeStar、CloudFormation等。
    7.应用服务：AWS提供了多种类型的应用服务，如Sagemaker、Elastic Beanstalk、Mobile Hub等。
    8.安全服务：AWS提供了多种类型的安全服务，如IAM、CloudHSM等。
    9.企业产品：AWS提供了企业级产品，如Managed Streaming for Kafka、Storage Gateway等。
