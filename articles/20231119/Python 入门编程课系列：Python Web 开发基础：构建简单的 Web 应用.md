# 2.核心概念与联系
## HTTP协议简介
HTTP(HyperText Transfer Protocol)即超文本传输协议，是用于从WWW或其他网络服务器上请求、传输和显示信息的协议。它是一个客户端-服务端模型的协议，由请求消息和响应消息组成。客户端发送一个请求报文到服务器端，请求报文的内容包括请求的方法、路径、协议版本、请求头部及数据。服务器收到请求后，向客户返回一个响应报文，响应报文中包含状态码（表示请求成功或失败）、原因短语、协议版本、响应头部及数据。
### 请求方法
HTTP定义了如下几种请求方法：

1. GET：获取资源。
2. POST：提交数据。
3. PUT：上传文件。
4. DELETE：删除文件。
5. HEAD：获得报文的首部。
6. OPTIONS：询问支持的方法。
7. TRACE：追踪路径。
8. CONNECT：建立 tunnel 。

### URL的结构
URL(Uniform Resource Locator)，统一资源定位符，用于描述网络上的资源。它由以下几个部分构成：

1. scheme: 指定访问资源使用的协议，如http、https等。
2. host：指定要访问的主机地址。
3. port：指定要连接的端口号。
4. path：指定请求的路径。
5. query string：提供参数给服务器处理。
6. fragment identifier：指定文档内的一个位置。

### HTTP报文结构
HTTP报文由请求行、请求头部、空行、请求数据四个部分组成。

1. 请求行：包括方法、URI、HTTP版本。
2. 请求头部：包括自定义的头部字段和通用头部字段。
3. 空行：用于分隔请求头部和请求数据。
4. 请求数据：可以是POST的数据、表单数据或JSON数据。

响应报文也由相应行、响应头部、空行、响应数据四个部分组成。

1. 响应行：包括HTTP版本、状态码、状态原因短语。
2. 响应头部：包括自定义的头部字段和通用头部字段。
3. 空行：用于分隔响应头部和响应数据。
4. 响应数据：服务器返回的响应主体，可能是文本、图片、视频或音频数据。

## HTML、CSS和JavaScript
HTML(Hypertext Markup Language)即超文本标记语言，是一种用来创建网页的标记语言。HTML使用标签将文字、图片、音乐、视频等各种媒体文件按照一定格式展示出来，并且允许跨平台浏览。HTML由一系列标签组成，如<html>、<head>、<body>、<title>、<p>、<img>、<a>等。

CSS(Cascading Style Sheets)即层叠样式表，用于美化HTML页面，为其添加交互、版式、视觉效果。CSS使用样式规则，对HTML页面进行选择和修改。

JavaScript是一种动态脚本语言，用于增强网页的功能和表现。JavaScript既可以使用嵌入到HTML中，也可以单独作为外部文件引入到HTML页面中执行。JavaScript可用来制作网页特效、增加用户交互，还可以与服务器通信实现更多功能。

## SQL语言
SQL(Structured Query Language)即结构化查询语言，用于操作关系型数据库，包括MySQL、Oracle、PostgreSQL等。SQL语言由DML(Data Manipulation Language)、DDL(Data Definition Language)、DCL(Data Control Language)三个子集组成，分别负责对数据进行插入、更新、删除、查询；DDL用于定义数据结构，比如表结构、视图结构等；DCL用于控制数据库安全，比如权限管理等。

## NoSQL数据库
NoSQL(Not only SQL)即不仅仅是SQL，它是非关系型数据库中的一个术语，泛指非关系型数据库管理系统。目前最流行的NoSQL数据库是MongoDB和Redis。MongoDB是一个面向分布式文档存储的数据库，基于分布式文件存储架构，自动处理数据复制及负载均衡。Redis是一个开源的、高性能的键值对数据库，采用单线程结构，内部采用内存存储，提供多种数据结构，是当前最热门的NoSQL数据库之一。
