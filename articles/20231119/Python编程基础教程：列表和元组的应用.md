                 

# 1.背景介绍


## 1.1 什么是列表和元组？
- **列表（list）**：在Python中，列表是一个有序集合，它可以保存多个数据项。列表中的每个数据项都可以通过其索引来访问或修改。列表可以包含不同的数据类型，包括数字、字符串、元组甚至其他列表等。列表可以使用方括号`[]`创建。例如：

```python
my_list = [1, 'a', True]
print(type(my_list))   # Output: <class 'list'>
```

- **元组（tuple）**：元组也是有序集合，但是它的元素不能被修改。元组一般用于不想让外界修改数据的场景，或者作为函数的输出结果返回给调用者。元组可以使用圆括号`()`创建。例如：

```python
my_tuple = (1, 'a', True)
print(type(my_tuple))    # Output: <class 'tuple'>
```

## 1.2 为何要用列表和元组？
当需要存储多个数据项的时候，列表和元组是最合适的选择。由于列表和元组具有相同的功能，因此它们可以互换使用。例如，如果某个函数要求输入一个列表，但是得到的是一个元组，则可以先将元组转换成列表，然后执行后续的操作。另外，列表和元组提供一些额外的方法和属性来方便地处理集合数据。例如，列表还可以分片操作（slice）。

## 1.3 数据结构的关系
- **列表（list）** 是一种可变的、可包含重复值的序列，其元素通过索引下标来访问。列表支持随机访问和截取操作。
- **元组（tuple）** 是一种不可变的、可包含重复值的序列，其元素通过索引下标来访问。元组支持按照顺序访问，但不支持插入和删除操作。
- 在性能上，列表的访问速度较快，内存占用也比较大，适合存储大量数据；而元组的访问速度较慢，内存占用小，适合存储少量、少改动的数据。
- 从实现方式上看，列表是采用线性表的数据结构，使用指针来链接元素，可以快速查询和修改元素。元组是将元素保存在一起，元素不能再单独添加或删除。
- 从抽象程度上看，列表提供了高阶的抽象能力，能够存储多种类型的数据；元组只能存储一种类型的数据，并提供简单和安全的数据交换机制。

## 1.4 概念及联系
- **序列（sequence）**：一个序列是指一个连续的整体，其中每一个元素都是一定的类型，并且元素之间可以按照一定顺序排列。序列在计算机科学中有着重要的地位，比如：字符串、列表、元组、集合等。
- **列表（list）**：列表是一个有序序列，其元素在内存中以连续的方式排布。列表中的元素可以通过索引进行访问，所以列表是一种灵活的序列容器。列表可以容纳任意类型的对象，也可以为空。
- **元组（tuple）**：元组与列表类似，不同之处在于元组中的元素不能被修改。元组是另一种有序序列，其元素在内存中也是以连续的方式排布。元组也可以容纳任意类型的对象，而且不可变。
- **字典（dictionary）**：字典是一个无序的键值对集合，其中每个键对应一个值。字典在Python中扮演着非常重要的角色，很多高级数据结构如数组、链表等都要依赖字典才能完成相应的操作。字典是动态的，可以随时添加、删除键值对。

# 2.核心概念与联系
## 2.1 序列
序列是指一个连续的整体，其中每一个元素都是一定的类型，并且元素之间可以按照一定顺序排列。序列在计算机科学中有着重要的地位，比如：字符串、列表、元组、集合等。

## 2.2 列表
列表是一个有序序列，其元素在内存中以连续的方式排布。列表中的元素可以通过索引进行访问，所以列表是一种灵活的序列容器。列表可以容纳任意类型的对象，也可以为空。

### 创建空列表
```python
empty_list = []
```

### 创建列表
创建列表最简单的形式是使用方括号 `[]`，将逗号隔开的元素放在里面即可。

```python
numbers = [1, 2, 3, 4, 5]
names = ['Alice', 'Bob', 'Charlie']
mixed = [True, 1, "hello", {'name': 'Tom'}]
```

### 添加元素到列表末尾
使用方法 `append()` 可以向列表的结尾添加元素。

```python
nums = [1, 2, 3]
nums.append(4)  # nums == [1, 2, 3, 4]
```

### 插入元素到指定位置
使用方法 `insert(index, element)` 可以将指定的元素插入到指定位置。

```python
nums = [1, 2, 3]
nums.insert(1, 4)  # nums == [1, 4, 2, 3]
```

### 删除列表中的元素
使用方法 `remove(element)` 或 `pop(index)` 可以从列表中删除指定的元素。

```python
nums = [1, 2, 3, 2, 4]
nums.remove(2)         # nums == [1, 3, 2, 4]
nums.pop()             # nums == [1, 3, 2]
```

### 修改列表中的元素
使用索引（下标）可以直接修改列表中的元素。

```python
nums[1] = 4            # nums == [1, 4, 3, 2]
```

### 分片操作（Slice）
分片操作（slice）是指从列表中提取一段子列表的操作。使用语法 `[start:end]` 可以指定子列表的起始和结束索引。

```python
nums = [1, 2, 3, 4, 5]
sub = nums[1:3]        # sub == [2, 3]
```

### 查询元素是否存在于列表中
使用 `in` 操作符可以判断元素是否存在于列表中。

```python
if 3 in nums:
    print('Found')
else:
    print('Not Found')
```

### 获取列表长度
使用内置函数 `len()` 可以获取列表长度。

```python
length = len(nums)      # length == 3
```

### 遍历列表
遍历列表可以使用循环语句。

```python
for num in nums:
    print(num)
```

## 2.3 元组
元组也是一个有序序列，但是它的元素不能被修改。元组一般用于不想让外界修改数据的场景，或者作为函数的输出结果返回给调用者。元组可以使用圆括号 `()` 创建。

### 创建元组
创建元组最简单的形式是使用圆括号 `()`, 将逗号隔开的元素放在里面即可。

```python
numbers = (1, 2, 3, 4, 5)
names = ('Alice', 'Bob', 'Charlie')
mixed = (True, 1, "hello", {"name": "Tom"})
```

### 不支持修改元素
元组不允许修改元素。试图修改元组中的元素会导致运行时错误。

```python
numbers[1] = 10       # Raises TypeError
```

### 支持分片操作
元组也可以进行分片操作。

```python
numbers = (1, 2, 3, 4, 5)
sub = numbers[:3]     # sub == (1, 2, 3)
```

### 性能比列表更好
因为元组是不可变的，所以对于读取频繁的场景，元组的效率优于列表。但是，在写入频繁的情况下，使用列表可能更好，因为列表是可变的。
