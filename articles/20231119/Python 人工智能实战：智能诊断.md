                 

# 1.背景介绍


## 智能诊断简介
智能诊断(Intelligent Diagnosis)或称为“智能医疗”，是通过计算机分析患者症状、检查手术过程等信息，识别并预测病情，制定相应的治疗方案。早期的智能诊断服务主要集中在公共卫生领域，随着互联网的普及，各种健康数据越来越多，如心电图、血压计、血糖监测仪等，可以通过这些数据对患者进行诊断和治疗。但是这些数据的呈现方式、结构和处理方法都存在一些缺陷，很难进行高效地有效的诊断。因此，需要借助计算机科学和统计学的方法对这些数据进行建模、归纳和分析，从而达到更好的诊断效果。基于对健康数据的理解和分析，智能诊断技术将使病人的治疗过程更加准确、及时、可靠、经济、环保。2017年，美国国家健康保险中心启动了“National AI-driven Healthcare”计划，旨在通过建立一个由私人、民营企业、非盈利机构合作组成的“健康AI公司”网络，把美国最先进的人工智能技术应用于医疗健康领域。

## 数据收集
目前，智能诊断涉及面临的挑战包括数据的获取、存储、预处理、特征提取、模型训练、结果评估和应用。目前，绝大多数研究者和开发者采用的数据集通常来自于特定领域或者历史遗留数据。由于这些数据往往比较简单，而且所含的信息也相对单一，因此会存在很多的偏差。因此，如何构建具有代表性的数据集、获得质量高的数据以及真正有效地利用数据是智能诊断的关键。

### 病例描述
本案例中，我们将对诊断患者脑瘤的主要症状、检查项目和CT检查结论进行分类，判断患者是否患上脑瘤并给出治疗建议。

### 数据获取
本案例采用的是脑瘤数据集，该数据集由人类通过CT扫描来获取。数据集的分布情况如下：

| Label | Dataset           | Samples   | Features     |
|-------|-------------------|-----------|--------------|
| 0     | Malignant         | 95        | 301          |
| 1     | Benign            | 100       | 301          |
| Total |                   | 195       |              |


其中Label列表示分类标签，0表示恶性，1表示良性；Dataset列表示数据的来源（这里是CT扫描），Samples表示样本个数，Features表示特征个数。我们可以对数据集进行详细了解：

1. 数据的基本情况
CT扫描图像是一个三维矩阵，通常每张图像大小约为1mm x 1mm x 1mm。所以总体图片尺寸为 $D_x \times D_y \times D_z = (128 \times 128 \times 128)$。每张CT图像被切割为多个小块，每个小块的大小为 $P_x \times P_y$ 。所以每张图片切割后的小块个数为 $D_x/P_x \times D_y/P_y = 8\times 8=64$。

2. 标签定义
我们将标签分为两种，恶性和良性。每张图对应一个标签，0表示恶性，1表示良性。

3. 属性
属性数据包含：肿瘤部位，病变类型，大小，形态，边缘性质，局部密度，血管线条等。

4. 缺失值处理
本数据集无缺失值。
