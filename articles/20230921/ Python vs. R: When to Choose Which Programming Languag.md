# 2.相关术语
* **动态类型系统**：动态类型系统（Dynamic Type System）是指程序在执行时才确定变量的数据类型，每个变量都具有一种静态类型（Static type），当赋予一个不同类型的值时，系统会自动将其转换成正确的类型。例如，在Python中，`x = 1 + "hello"`，由于字符串不能与数字相加，因此系统将会报错。而在C语言中，同样的语句无法被编译通过，因为"hello"不是整数类型。
* **注释**：注释是为了方便阅读而添加进程序中的文字，不会在程序中产生实际的运行结果。通常用于描述代码块的作用或说明一些特定行为。在Python和R中，均可使用 `#` 来表示单行注释。
* **函数**：函数（Function）是将相关的代码打包，并命名的一个小段代码，它可以在程序中重复调用，减少代码的冗余性。函数通常接受输入参数（input parameter）、执行某些计算过程（computation process）以及输出结果（output result）。在Python中，定义函数使用 `def` 关键字，调用函数使用函数名加括号。
* **模块**：模块（Module）是保存各种代码的集合，它用来管理代码的分割，便于代码的维护和复用。在Python中，使用 `import` 或 `from...import` 命令导入模块，然后就可以使用模块里面的函数或者类。
* **面向对象编程**：面向对象编程（Object-Oriented Programming）是一种编程方法，基于对现实世界问题的抽象建模，将现实生活中的对象与属性、行为等特征联系起来，建立起对象模型，并通过编程语言实现对象的各种功能。面向对象编程最大的特点就是代码组织化，可以帮助程序员创建复杂的应用。在Python中，可以使用面向对象编程的方法，编写面向类的程序。
* **切片**：切片（Slice）是指通过指定起始索引和结束索引，从列表、元组、字符串等数据结构中提取子集的过程。在Python中，可以通过方括号 `[ ]` 指定切片的范围，即 [start:end:step] 。
* **类型注解**：类型注解（Type Annotation）是一种新的编程语言机制，它允许在代码中标注变量的类型信息，增强程序的鲁棒性。在Python 3.5版本引入，并随后在其它版本中得到更新。其基本语法如下：

```python
variable_name : data_type = value
```

* **单元测试**：单元测试（Unit Test）是指独立测试软件组件（如函数、模块、类）的最小可测试单位。单元测试用于确保各个模块的行为符合设计预期，保证软件在变化环境下的正确性。在Python中，单元测试可以使用 `unittest` 模块来实现。
