# 1.简介
  

随着信息技术的飞速发展、社会信息化程度的提升、医疗健康领域的数据积累、云计算等新技术的出现以及大数据的广泛应用，基于大数据分析所面临的种种挑战越来越多。其中，在医疗健康领域，如何有效、准确地使用海量医疗数据进行决策支持系统建设，是一个亟待解决的问题。近几年来，基于大数据的决策支持系统已经成为医疗领域的一项重大突破，被各大医院或科研机构广泛应用。本文将对目前国内外基于大数据分析的决策支持系统相关研究成果进行综述，并提出以下几个方面的建议：

1) 定义清楚什么是决策支持系统。首先，决策支持系统应该是指利用计算机技术及相关知识处理人类活动中的数据和信息，提供决策支持和指导的手段或工具。它既包括一般的决策支持系统，如E-Healthcare；也包括特定类型且涉及医疗诊断、诊治管理和经济评估等复杂决策任务的决策支持系统，如Precision Medicine Platform。

2) 对于医疗健康领域的决策支持系统，需要建立在现有的理论、方法、技术基础上，充分发挥数据处理、分析、模式识别、机器学习等计算机技术的优势。

3) 在构建和运用决策支持系统时，不仅要关注算法的效率和准确性，更需要注重降低数据获取、存储、处理等方面的成本，充分考虑到可靠性、隐私保护、合规性、易用性、扩展性等因素。

4) 提供详细的功能需求文档，以便参与者能够快速理解系统的目标、输入、输出、性能指标和标准等要求，并根据这些要求设计出高效、精准的决策模型。

# 2. 基本概念、术语和符号说明
## 2.1 数据集及其特点
### 2.1.1 数据集的定义
数据集（Data Set）是指由各种记录组成的集合。每个数据集都可以看作是一个具有相同特征(attribute)的数据表，其中每条记录都是一个向量或矩阵，记录了某种实体的一个实例的相关属性值。

### 2.1.2 数据集的特点
数据集可以从多个角度观察其特点。按照维度（Dimensionality）的大小，数据集可以划分为静态数据集（Static Dataset）、动态数据集（Dynamic Dataset）、混合数据集（Hybrid Dataset）。按照数据持续性（Durability），数据集可以划分为瞬时数据集（Instantaneous Dataset）、持久数据集（Persistent Dataset）、非实时数据集（Non-Real-Time Dataset）。按照数据形式（Formality），数据集可以划分为结构化数据集（Structured Dataset）、非结构化数据集（Unstructured Dataset）、半结构化数据集（Semi-Structured Dataset）。按照数据源头（Provenance），数据集可以划分为原始数据集（Raw Dataset）、中间数据集（Intermediate Dataset）、引用数据集（Reference Dataset）、派生数据集（Derived Dataset）。按照样本数量（Sample Size），数据集可以划分为较小数据集（Small Dataset）、中型数据集（Medium Dataset）、大型数据集（Large Dataset）。

## 2.2 大数据分析的分类
### 2.2.1 体系分类
大数据分析主要体现在四个层次上：

1. 底层分析：对宏观数据进行深入分析和挖掘。比如，从全国、国家、地区、城市、乡镇、村庄、社区甚至各个细节区域的多个维度分析，如人口分布、贫富差距、教育水平、职业倾向、生活方式、经济状况等，揭示其内部动态变化的规律，为决策提供依据。

2. 中层分析：通过多维数据分析，对热点问题进行及时监测和反应，并形成有效的政策指导。比如，针对医患纠纷、食药安全、环保危害等敏感问题，通过整合各类因素、用户画像、多源数据，结合自然语言处理、语音识别、图像识别等技术，挖掘数据的关联性、稀疏性、不一致性等特点，实现实时的、自动化的、准确的、及时的分析和预警功能，提升服务质量、降低成本。

3. 上层分析：对某些比较宏观的事件或者矛盾性问题，通过大数据分析，制定战略性策略，以优化资源配置、降低风险、保障公共利益等。比如，针对抢劫、欺诈、恶意骚扰等犯罪行为，通过大数据分析，建立模型，匹配出其背后隐藏的真相，帮助执法部门快速制定行动方案，有效阻止犯罪扩散。

4. 系统分析：整合不同业务领域、不同部门、不同人员的信息，形成完整的知识图谱，以满足个性化需求。比如，智慧城市、车联网、地图导航、消费习惯分析、金融风险预测等，通过系统分析，将不同数据源、不同算法、不同框架、不同平台、不同场景的信息结合起来，为用户提供多种精准的服务。

### 2.2.2 模型分类
基于大数据分析的决策支持系统大致可分为三类：

1. 预测类：主要用于预测趋势、规律，例如预测市场的总体趋势、某个产品的销售情况、股票市场的走势等。通常采用统计模型、机器学习算法、时序分析等。

2. 决策类：主要用于决策支持，例如产品推荐、零售业销售预测、投资组合建议等。通常采用优化模型、贝叶斯网络、遗传算法等。

3. 检测类：主要用于异常检测、人流量预测、舆情监测等。通常采用聚类算法、支持向量机、神经网络等。

### 2.2.3 技术分类
基于大数据分析的决策支持系统的技术分类又可分为三层：

1. 离线分析：基于离线数据处理、数据挖掘、机器学习等方法的分析，应用于超大规模、海量数据集上的高性能计算集群。离线分析通常侧重海量数据的处理、存储、检索，而非实时运算能力。

2. 在线分析：基于实时数据采集、传输、处理、分析、显示等方法的分析，应用于流式数据和实时数据上的高吞吐量网络。在线分析通常侧重实时性、速度、可扩展性，而且一般不依赖离线数据，而是直接从生产环境中获取数据。

3. 混合分析：基于离线和在线分析的方法的混合，包括多种技术的融合，以实现更高的性能和更好的效果。

## 2.3 概念、术语和符号说明
1.数据仓库：数据仓库是对企业所有相关数据进行汇总、整理、加工、再发布的数据库，是数据驱动决策的关键所在。它是数据仓库技术的核心，也是支撑数据驱动决策的基石之一。

2.维度：维度是指构成数据集的变量或属性，即描述事物的某种客观特性。数据仓库中的维度通常包括时间维度、地理维度、主题维度、事务维度、子维度等。

3.星型模型：星型模型是一种特殊的维度设计模型，即把一个事物的多个维度都作为独立的字段来建模。举例来说，如果我们想对电影进行分析，除了有明显的“导演”、“主角”、“年份”、“国家”等维度，还可能存在“故事设定”、“宣传片”、“广告片”等类型的维度。

4.雪花模型：雪花模型是一种多维度数据分析模型，它强调数据的透明性、多样性和容错性。通过切分维度、聚类、关联分析等方法，将复杂的数据进行抽象、提炼，最终形成清晰易懂、直观易读的结果。

5.事件驱动分析：事件驱动分析是一种数据分析过程，它根据业务目标定义相关事件，然后使用事件规则引擎对数据进行实时分析，实时生成报告和指导决策。

6.流水线数据管道：流水线数据管道是一种数据处理模式，它将多个数据源的数据按照固定顺序进行收集、转换、过滤、加载，然后流向下游应用系统进行进一步分析、处理、呈现。
