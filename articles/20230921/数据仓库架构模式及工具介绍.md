# 1.简介
  

数据仓库（Data Warehouse）是一种基于集成的数据集合，用于支持复杂报告查询和分析工作。它是一个面向主题的、随时间变化的数据存储区，以便为业务决策提供信息。数据仓库的特点包括以下几个方面:

1. 集成：数据仓库中所存储的数据是从多个源头汇总过后得到的一个集中存放地，使得不同的数据源的数据可以被整合、处理并形成一张统一的视图；
2. 主题：数据仓库按其主题划分，可以实现数据的分类和过滤，方便业务用户查找相关数据；
3. 时序性：数据仓库中的数据具有时序性，记录着历史数据，并提供对近期数据的分析支持；
4. 实时性：数据仓库的数据是根据真实情况而生成的，不受时空限制；
5. 共享：数据仓库是一个共享资源，所有业务部门都可以使用它，并通过它获取有关自身业务的信息。

数据仓库的建设需要做到：

1. 可用性：保证数据仓库的数据可用、正确、及时的提供给各级业务部门，提高业务生产力；
2. 统一性：所有数据源保持一致性，避免出现数据孤岛，提高数据质量；
3. 易扩展性：根据业务的发展及对数据安全的要求，对数据仓库进行适当的扩容或收缩；
4. 数据质量：确保数据质量，确保数据准确、全面、时效性强。

数据仓库通常是采用分层的方式来构建，按照功能的不同将数据仓库分为数据集市层、数据挖掘层、维度建模层、数据访问层等几层。如下图所示：


1. 数据集市层：一般存储的是一些静态数据，如客户信息、商品销售数据等，这些数据在整个企业都是固定的。

2. 数据挖掘层：该层主要用来进行数据分析、挖掘，以找到业务中最重要的价值点，如销售额最大、订单次数最多、顾客消费习惯、风险指标等。通过这些信息，数据挖掘层可以帮助企业制定目标，优化经营策略，提升企业竞争能力。

3. 维度建模层：维度模型层主要基于现有的静态数据构建出一个完整的、能够支撑企业各种业务活动的数据模型。数据模型的建立通常会涉及到大量的数据清洗和转换工作，但最终可以达到数据中立的效果。

4. 数据访问层：数据访问层是数据仓库的主要用户，可以直接通过SQL语句查询数据。数据访问层通过协调数据源和数据模型之间关系的能力，以统一的方式将数据呈现给用户，提高用户的工作效率。

数据仓库作为集中存放和处理企业各种类型数据的一座大厦，可以有效解决数据管理的很多问题。数据仓库的建设，无疑会对企业的决策过程产生重大影响。如何才能建设一个可靠、高效、结构化、精准的企业数据中心，就成为一个需要长久的探索。数据仓库架构模式及工具介绍将阐述当前数据仓库的发展趋势，并探讨数据仓库架构设计及选型的技术路线。
