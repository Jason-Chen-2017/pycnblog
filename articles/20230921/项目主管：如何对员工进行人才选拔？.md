# 1.简介
  

作为一名项目主管，你的任务之一就是需要识别并培养项目组成员的优秀者。你首先会收集到各种信息，包括项目需求、个人能力、项目经验等。你需要根据这些信息，结合公司组织架构，制定出一套人才选拔标准。然后你将分派任务给各个组员去完成。通常情况下，组员都会认可这个标准，并且取得了明显的成效。但也有一些低能或者不擅长的人物通过这种方式被忽视掉了。因此，如何对员工进行人才选拔，成为项目主管的一个重要职责。下面我们从以下几个方面来详细阐述一下“如何对员工进行人才选拔”。
# 2.角色定位
首先，让我们回忆一下自己在这份工作中扮演的角色定位。在我们的工作中，我们既要保证项目顺利推进，又要确保项目干系人的安康成长。因此，我们不能只盯着项目的结果做决定。我们还需要从各个角度考虑人才的素质，比如学习能力、沟通技巧、团队合作精神等。因此，在我们选择培养哪些人时，一定要综合考虑这些因素。
# 3.资源汇总
现在我们知道自己的主要工作内容了。接下来，我们就需要清楚地理解各种资源。我们可以从以下几个方面来汇总：
- 企业内部的人力资源：这是最基础也是最重要的资源。企业内的人才库存量极其庞大，每年都可能产生几百万的新招聘岗位。人力资源管理部门应当将这一切投入到人才培养、激励、晋升等领域。
- 招聘网站：海量的招聘网站能够吸引海量的候选人，其中很多网站也提供筛选人才的功能。例如，LinkedIn、AngelList等都是知名的招聘网站。
- 员工自荐：员工自荐并不是真正意义上的人才推荐，只是一种形式。但在企业里，员工自荐无疑是为员工提供信息的有效手段。员工自荐也是为企业创造价值的良好开端。
- HR咨询：HR一般是整个企业的人力资源部门，当人事部负责人需要帮助时，HR会提供必要的帮助。尤其是在雇佣管理、薪酬福利、培训管理等方面，HR都可以提供专业化的咨询。HR的咨询往往需要更加专业的知识和经验。
# 4.选拔标准
好的选拔标准应该具备以下几个特点：
- 客观性：能够客观地衡量每个人。不同的标准可能会导致不同的结果。例如，身高是否高于平均值，学历是否具有更高的竞争力。
- 层次性：人们容易把自己看成一个整体，因此，我们需要考虑不同层次的能力。比如，有的同事比较擅长团队合作；有的则非常擅长沟通；还有的比较擅长学习能力。因此，我们需要细化标准，让他们之间互相配合。
- 差异化：每位员工都拥有不同的能力，为了提升项目的质量，我们需要引入一定的差异化。比如，对于负责业务分析的人来说，要求必须熟悉SQL语言；而对于负责项目开发的人来说，要求必须精通Java编程。这样才能更好地分配工作。
# 5.评价机制
所谓的评价机制，是指用什么样的方式对员工进行筛选。我们通常有两种方法：一是轮岗制，即每隔几年评一次，每次由上一级评委评一次；二是集中制，即每季度集中对所有员工进行评估，选出最好的来留任。集中制由于需要每天做功课，因此效率一般较低。但是集中制有一个很大的优点——可以全盘考虑到每个人的能力，防止出现狭隘的、片面的评判。所以，在实际工作中，我们往往采用两者之间的折衷。
# 6.面试环节
面试是整个过程中不可或缺的一环。不同级别的面试官对人员的能力掌握程度都有着自己的判断标准。因此，我们需要设计一套独特的面试题目，使得面试官能够对员工的实际情况有一个客观的认识。而且，我们还需要注意和面试官保持良好的交流习惯。面试需要尽可能展现自己的能力，而不是套话和错误的解读。面试结束后，我们也需要对他们的表现进行反馈。如果他们表现不佳，我们需要采取适当的手段，如将他们调离职位等。
# 7.流程建议
最后，我想提供一些流程上的建议。首先，我们要了解如何收集信息，并根据这些信息制定出一套完善的人才选拔标准。其次，我们要掌握各种资源，从不同角度考虑人才的素质。第三，我们要根据评价机制对员工进行筛选，并且制定相应的反馈机制。最后，在人才挑选之后，我们还需要与面试官保持密切联系，帮助他们改进自己的方案。以上就是我的建议，欢迎大家围观。
