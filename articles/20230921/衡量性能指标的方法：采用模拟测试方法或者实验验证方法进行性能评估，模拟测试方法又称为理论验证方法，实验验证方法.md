# 1.简介
  


性能评价是软件开发过程中非常重要的一环。对软件系统的性能进行评价可以直接影响到其生命周期中各个阶段的关键点，如项目启动、项目维护、产品发布等。好的软件系统的性能应当具有良好的响应时间、资源利用率和稳定性等特点。衡量软件性能的一个最直接有效的方式就是用计时器或者其他工具记录某些关键变量的变化，比如运行时间、CPU使用率、内存占用率等。但这种方式只能粗略地了解到软件系统的整体运行状态，无法洞察出不同系统配置下软件系统的性能差异。因此，需要更高级、更精细的性能评测手段才能得出科学的结论。

2.性能测试方法

衡量软件系统性能的方法可以分为两类：
- 模拟测试方法：此类方法通过构造虚拟环境模拟真实环境中的各种场景，并逐步增加负载直到系统达到最大容量，然后测量不同负载情况下系统的行为和表现。模拟测试方法得到的是相对真实的结果，是一种理想化的测试方式。
- 实际测试方法：此类方法在生产环境或实验室中安装实际的软件，采集系统的实际数据并分析处理。实际测试方法可以提供更加真实的数据，从而更好地了解软件系统在实际运行时的表现。但是，由于要在真实的环境中测试，因此需要付出更大的成本。

基于上述两类方法，目前普遍使用的性能测试方法有两种：
- 功能测试（Functional Testing）：测试各模块之间的功能协同是否符合预期，比如输入输出的正确性、数据传输效率、异常处理能力等。功能测试主要关注系统的外部接口和功能特性，可以测试系统的可靠性。
- 可用性测试（Usability Testing）：测试用户对系统的可用性和易用性。可用性测试涉及的方面包括易用性、可用性、兼容性、适应性等。可用性测试主要用于发现系统中的逻辑错误、界面错误、兼容性问题和用户习惯性错误。

除此之外，还有一些其它性能测试方法，如压力测试（Stress Testing）、易用性测试（Aesthetic Testing）、接口测试（Interface Testing），等等。

除了以上两种性能测试方法，还有一些相关的测试技术，例如白盒测试、黑盒测试、灰盒测试、结构测试、安全测试、配置测试、鲁棒性测试等。这些测试技术都可以用于性能评测。

3.性能测试模型

根据性能测试目的的不同，通常会把性能测试分为不同的阶段，并采用相应的测试模型。

### 1.功能测试模型
功能测试模型是一种最简单的性能测试方法，它将测试目标放在系统的外部接口上。该模型主要是观察系统的响应速度、功能和错误信息等。模型的测试步骤如下：

1. 概要设计：需求分析、系统设计、测试计划书撰写。
2. 详细设计：系统架构设计、数据库设计、接口设计、功能模块设计、性能测试方案设计。
3. 测试环境准备：配置测试环境、设置性能测试数据。
4. 测试执行：功能测试、性能测试、压力测试。
5. 测试报告编写：测试总结、缺陷统计、回归测试。

功能测试模型的优点是简单，适合快速验证系统的性能指标。缺点是无法考虑系统的内部实现机制，容易受限于人的感觉和经验。另外，功能测试模型还存在着依赖于人的因素，可能导致结果不可重复。

### 2.可用性测试模型
可用性测试模型注重于验证用户对系统的可用性、易用性、兼容性和适应性。可用性测试的目标是发现用户错误、故障、功能不匹配、使用困难等潜在问题，目的是提升系统的易用性。可用性测试模型的测试步骤如下：

1. 概要设计：需求分析、系统设计、测试计划书撰写。
2. 详细设计：可用性测试方案设计、可用性测试用例设计、测试环境准备。
3. 测试执行：可用性测试、兼容性测试、手动测试、自动测试。
4. 测试报告编写：测试总结、缺陷统计、回归测试。

可用性测试模型是一个广义的模型，它既关注系统的外部接口，也关注系统的内部结构和操作流程。优点是可见性强、全面性高，但测试成本较高。同时，可用性测试也可能会受限于个人能力、环境条件等。
