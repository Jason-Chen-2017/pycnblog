# 1.简介
  

虚拟私有云（Virtual Private Cloud，VPC）是一种网络隔离的技术，可以将用户的计算、存储和网络资源从其它租户或者企业租赁给自己的服务平台，让用户享受到完整的自主权、可控性和弹性，在实际运用中，用户可以按照需求定义子网并创建VPC，实现对内网环境的隔离。
VPC功能：
- 提供对云资源的虚拟隔离，提升资源利用率和安全性；
- 为用户提供了更加灵活的网络配置，可以实现多个子网之间的通信；
- 可以针对不同业务场景进行不同的配置策略，实现更高级的资源调度；
# 2.基本概念术语说明
## 2.1 VPC
虚拟私有云（Virtual Private Cloud，VPC）是一种网络隔离的技术，是一个虚拟的网络环境，它在物理网络上建立一个逻辑上的隔离区，让不同的系统或部门都可以在这个环境中互相隔离、互不影响地运行，提升了系统的安全性、可用性和性能。通过VPC，你可以构建私有网络，里面含有自己的网络资源和组件，如服务器、路由器、交换机等，并且与其他私有网络隔绝开来。私有网络中的所有成员之间只能通过VPN或SSL VPN通道来进行通信。
## 2.2 子网
子网（Subnet）是在VPC网络中划分出的一段IP地址范围，用于分配网络资源，用户可以根据需要创建不同子网，设置不同的防火墙规则、访问控制策略，实现对内部网络资源的隔离。子网的网络号段最好不要冲突。
## 2.3 安全组
安全组（Security Group）是一个逻辑上的集合，包括了一组权限规则。每条规则都包含以下字段：方向、协议、端口范围、授权对象、优先级和描述信息。安全组是防火墙的入口，作用是在两个虚拟网络之间实施流量过滤。
## 2.4 Internet Gateway
Internet Gateway 是 VPC 的外网网关，它负责 VPC 和互联网之间的连接，同时也具有路由转发的功能。一个 VPC 只能创建一个 Internet Gateway 。
## 2.5 NAT Gateway
NAT Gateway 是一种网络地址转换（Network Address Translation，NAT）设备，能够帮助您的私有网络内的主机与互联网的连接。NAT Gateway 可帮助您解决因公网 IP 资源短缺而造成的网络拥塞。NAT Gateway 可以启用/禁用与 ECS、ECI 及 SLB Instance 的连接。
## 2.6 VPN Gateway
VPN Gateway 是 AWS 提供的基于 IPsec 协议的 VPN 服务。它可以通过一键部署的方式，快速构建跨区域的站点到站点 VPN 或 VPNs。VPN Gateway 不但支持静态路由、动态路由、IPv6、BGP、秘钥认证等功能，还可以很方便的扩展到多可用区部署。
## 2.7 VPC Peering
VPC Peering 是 VPC 间的对等连接，可以跨越同样的 Region 或 AZ 实现两个 VPC 之间的通信。当两个 VPC 处于不同 Region 时，需要借助 AWS Global Accelerator 来实现 VPC 间的通信。
## 2.8 VPC Flow Logs
VPC Flow Logs 是 VPC 中一个子网级别的日志服务，用来记录 VPC 中的网络流量。VPC Flow Logs 可以跟踪 VPC 内的所有网络流量，包括入向和出向的数据包数量、源地址和目标地址、传输协议、传输方向等。
