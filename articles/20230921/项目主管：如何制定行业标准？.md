# 1.简介
  

主管们每天都在制定公司、部门或者项目的标准，这是一种必要且普遍的工作。但作为一个主管，如何制定的标准有一些自己的心得体会。另外，不同行业和组织也存在着不同的标准，所以制定标准的过程还是需要考虑到行业差异。
# 2.标准的定义
标准的定义是相对而言比较模糊的。一般来说，制订标准应当满足以下四个条件：

①统一性：标准应当能够应用到所有相关方面，不论是工程、管理、财务还是法律方面的，都可以遵守同一个标准；

②透明度：标准必须能让所有相关方面参与者清晰了解到标准的目的、范围和要求，并知悉标准的执行流程、评审方式及相关的责任主体等；

③可操作性：标准必须具有实施能力，能够真正落实到企业内外实际情况，并能够被实际执行；

④自动化程度：标准的制订应当考虑到自动化程度的提高，比如数据采集、自动化测试等方法能够更快、更准确地发现不规范行为，从而更好地保障企业的运营安全。

# 3.标准的制订步骤
制订标准一般分为两个阶段：立项阶段和执行阶段。其中，立项阶段由产品负责人或部门负责人等部门负责人召开会议，汇总需求信息、调研行业共识、形成需求目标、制定计划并提交给上级领导批准，再进行审核确认，最后确定标准的相关细则和条例。

执行阶段主要包括以下几个步骤：

1. 发布通知：向所有相关方面的参与者（如合作伙伴、客户、供应商）发布制订的标准；

2. 执行方案：制订标准并将其推广到相关部门和人员手册中，以便他们能够按照标准实施工作；

3. 培训培养：标准要落地后，应该给相关人员提供专业技能培训，让他们熟悉并掌握该标准，并达成共识，以便全面地实施；

4. 测试评估：标准要落地后，还需要进行一系列的测试评估，以确定其有效性、经济效益和社会影响力；

5. 维护更新：随着企业经营和业务的不断发展，标准也会发生变化，需要根据实际情况进行更新；

6. 撤销撤销：如果标准的有效性、经济效益或社会影响力出现下降，或者有其他原因不能继续实施，就需要取消原有的标准。

# 4.标准的落地
制订出来的标准要落地到公司、部门甚至组织里去，其实就是把相关部门的各个成员、职能以及部门的工作内容进行整合、关联和编排，使之成为一个协同工作的集合体，并通过制定的标准作为依据，确保公司各个部门、各个岗位上的工作都遵循相同的标准，并且得到各方的认可和支持。

落地的时候，首先需要考虑到是否可以满足标准的自动化程度。例如，如果制订的标准涉及的数据采集，就可以考虑使用机器学习算法等方式进行数据采集的自动化处理。对于那些无法完全自动化完成的部分，则可以通过人工的方式进行监督和指导。

另一方面，落地时还需要考虑到标准的执行难度。不同的部门、岗位都有不同的复杂度和要求，因此落地过程中可能会遇到很多困难。不过，制订和落地标准是一个长期的过程，最终的目的只有一个——保障公司的合规、高效运行。
