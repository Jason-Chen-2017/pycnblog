# 1.简介
  

移动边缘计算（Mobile Edge Computing，MEC）是一种利用终端设备、移动网络、云服务等新型计算平台，在边缘位置提供计算能力，以提升移动应用的响应速度和降低成本的一种技术。它能够实现资源的高效共享、节约电量、降低通信成本、减少拥塞、满足用户实时交互需求。MEC系统的开发始于2010年美国国家超级计算中心的工作，现已成为许多重要应用领域的关键技术。

但是MEC技术并不是一个简单的技术产品，而是一个复杂的体系结构及其所涉及的相关的众多研究领域。因此，MEC技术是一个由多个子技术组成的整体，各子技术之间又存在相互影响、配合与衔接的关系，往往难以单独探究其中的原理和算法。在这个背景下，为了让读者对MEC技术有一个全面的认识，作者首先基于前人的研究成果，阐述了MEC技术概述、术语、系统结构、核心算法原理，并给出了几个典型案例研究。文章最后还会对MEC技术的未来发展方向进行展望，并讨论一些关键的挑战。

# 2. 基本概念
## 2.1 MEC概述
### （1）移动边缘计算技术概览
移动边缘计算（Mobile Edge Computing，MEC）是指利用无线通信、互联网、移动网络等新型计算平台，在边缘位置提供计算能力，提升移动应用的响应速度和降低成本的一种技术。MEC是一种新型的计算架构，其主要特征包括：边缘计算节点、移动应用程序、基于云的数据存储和服务。这些元素构成了一个庞大的体系结构，从最初的无线通信设备到智能手机、微控制器、穿戴设备甚至传感器，再到无处不在的无线基站、无线路由器、WLAN交换机等分布式网络系统。

MEC的系统架构由数据中心向移动设备延伸，通过各种分布式、异构、智能的计算设备完成任务，能够支持巨大的计算负载，例如视觉分析、机器学习、金融交易、物流运输、社交网络等应用。它通过连接无线通信设备和服务器，实现数据的自动传输、本地处理，加速应用的响应时间、降低成本。通过与运营商、内容分发网络等网络服务商进行协同，可以有效地将业务流程引导到合适的网络边缘位置，提升移动应用的可用性和响应能力。

### （2）MEC定义
按照Wikipedia中关于移动边缘计算（Mobile Edge Computing）的定义，移动边缘计算（MEC）是利用无线通信、互联网、移动网络等新型计算平台，在边缘位置提供计算能力，以提升移动应用的响应速度和降低成本的一种技术。MEC通常被描述为：“利用分布式、动态环境中的计算能力，使终端设备能够快速、精确地执行各种计算任务。”移动应用则可以通过Mec平台上提供的计算资源，有效应对当前和未来的计算需求。

### （3）MEC概念定义
- 服务：移动边缘计算（MEC）旨在提供服务，即提供基于云计算能力的计算服务，提供终端用户访问网络上可靠的计算资源。
- 边缘计算节点：移动边缘计算系统由具有计算功能的终端设备组成，它们可以离本地的终端用户很近，而且能够为用户提供计算能力。这些设备通常部署在城市的周围或路边，能够消耗大量的电能，所以其所在位置非常重要。
- 分布式计算：分布式计算指的是多个计算节点之间相互独立、缺乏通信能力的计算方式。MEC系统的设计目标就是通过在终端设备、移动网络、云基础设施等分布式环境中进行计算，来满足用户的计算需求。
- 智能路由：MEC通过智能路由选择路径，将任务分派到距离用户最近的计算节点，从而达到资源优化和数据迁移的目的。同时，MEC还可以根据用户的移动行为及时调整路由策略，提升系统性能。
- 可编程计算：MEC的计算单元是可编程的，它能够执行各种计算任务，并能够根据用户的需要和需求进行定制化。
- 数据本地性：MEC借助无线通信的特性，将计算任务发送到靠近用户的终端设备，这样可以保证数据和计算任务的本地性。
- 远程运算：MEC的计算资源通常位于距离终端用户较远的地方，通过无线通信和云服务等计算资源实现远程运算。远程运算可以帮助终端设备完成计算密集型任务，但也引入了延迟和功耗方面的成本。
- 边缘分析：MEC中的边缘分析是指将图像、音频、视频等数据从终端设备收集上传到云端，并进行分析处理的过程。通过这种方式，可以大幅度降低终端设备上的处理负担，提升终端用户的体验。
- 时变计算：MEC中的时变计算（Time-Varying Computing）是指对任务的执行时间或依赖条件变化，比如移动的位置、天气状况、网络状况等，进行重新调度和部署的计算方案。该方案能够避免过时或失效的资源分配，提升系统的资源利用率，缩短响应时间。

## 2.2 术语
- 边缘计算节点（Edge Node）：分布式计算系统中负责承担计算任务的主机设备，通常部署在用户所在区域或周边。
- 计算任务（Computing Task）：需要进行计算的任务，可以是传统计算机应用（如数据库查询、多媒体处理）、深度学习任务或移动边缘任务。
- 远程任务（Remote Task）：需要由移动边缘节点执行的计算任务，并将结果返回用户的移动设备。
- 任务调度（Task Scheduling）：用于确定计算任务应该发送到的特定边缘节点。
- 任务分派（Task Dispatch）：指将远程任务分配给边缘计算节点进行执行的过程。
- 任务编排（Task Orchestration）：指将不同的远程任务组合成一个计算任务，以提高资源利用率和减少调度开销的过程。
- 任务管理（Task Management）：指边缘计算节点上运行的任务生命周期管理和优化。
- 任务编排器（Task Scheduler）：是指在多台边缘计算节点上调度任务的模块。
- 任务编排器集合（Task Scheduler Collection）：由多个任务编排器组成，用于协同工作，完成分布式任务调度。
- 软硬件协同（SoftWare Hardware Co-Design）：是指在硬件与软件层面协同设计，兼顾资源利用率和任务响应速度。
- 通信互连（Communication Interconnect）：在分布式环境中，用于信息交换的网络互连组件。
- 任务重启（Task Restart）：当某条边缘计算路径出现故障时，任务调度器将自动启动重新执行失败的任务。
- 任务容错（Task Fault Tolerance）：即使某个任务的某些部分因节点故障无法正常执行，任务调度器仍然能够继续执行其他任务。
- 请求响应时延（Request Response Delay）：指请求客户端发起远程计算任务后，任务调度器收到任务请求到实际执行完成之间的总时长。
- 使用率（Utilization）：表示资源的利用率，用百分比表示，值越高表示资源利用效率越高。
- 服务质量保证（Service Quality Guarantee）：通过降低延迟、提高带宽利用率和降低系统损耗等手段，MEC能够提供更好的服务质量。
- 移动边缘云（Mobile Edge Cloud）：部署在移动边缘的云计算平台，用于承载移动边缘计算的应用服务和数据存储。
- 静态资源（Static Resource）：在本地计算环境中运行且不参与任务调度的计算资源，如硬盘存储、内存、网络带宽等。
- 动态资源（Dynamic Resource）：根据计算需求和任务特点进行动态配置和调整的计算资源，如CPU、GPU、DSP等。
- 用户态资源（User Mode Resources）：直接受到终端用户控制的计算资源，如摄像头、麦克风、GPS等。
- 资源管理器（Resource Manager）：是指用来管理系统中计算资源的模块，负责分配计算资源、监控资源利用率、进行资源调度和弹性扩容。
- 资源分配模型（Resource Allocation Model）：用于描述系统中各类计算资源的分配规则，如静态优先级、弹性优先级等。
- 拓扑调度（Topology Scheduling）：指根据拓扑结构、网络拓扑、任务依赖关系等，决定任务的路由路径和顺序的过程。
- 流量调度（Traffic Scheduling）：指根据当前网络环境、应用情况、任务类型、数据大小、时延要求等，动态调整任务的传输路径、负载均衡、拥塞控制等过程。
- 集群调度（Cluster Scheduling）：指将多个任务或多个设备聚集在一起，并统一调度的方式，提升整体的资源利用率和资源调度效率的过程。
- 模糊计算（Fuzzy Computing）：使用模糊理论、模糊数学方法、模糊规划算法、模糊学习等技术，实现计算任务的非确定性，并满足用户的个性化需求。
- 数据溯源（Data Provenance）：记录任务从源头到达目的地的整个过程，包括移动设备、服务器、数据中心、边缘节点等。
- 数据存储（Data Storage）：云端或边缘端用于保存计算任务结果和中间结果的临时存储介质。
- 数据推送（Data Push）：将任务执行结果主动推送到本地用户终端的过程。
- 数据拉取（Data Pull）：本地用户终端主动从云端拉取任务执行结果的过程。
- 数据集成（Data Integration）：将不同类型、不同粒度的计算结果进行整合的过程。
- 缓存和预测（Cache and Prediction）：为了减少通信成本，边缘计算节点采用缓存机制和预测机制。缓存机制将经常访问的数据保存在本地，降低延迟；预测机制是指根据历史数据预测未来数据的值，提升任务执行效率。
- 碎片规划（Fragmentation Planning）：当数据集大小超过本地存储容量限制时，需要将数据切片并存储到多个本地节点。碎片规划是在选择合适的切片策略时考虑的指标，如切片大小、切片数量等。
- 网络状态（Network State）：指移动边缘网络的广播链路状态、时延、丢包率、抖动情况等指标。
- 终端位置估计（Terminal Location Estimation）：在进行任务调度时，使用定位技术，对用户的真实位置进行估算。
- 终端干扰（Terminal Imbalance）：指移动边缘计算节点间的网络负载不平衡导致的计算资源不足或任务调度不准确的问题。
- 多尺度计算（Multi-scale Computing）：采用不同规模的计算资源，从而提升整体的计算性能。
- 分布式计算（Distributed Computing）：由多个计算节点分布式协作完成任务的计算方式。
- 负载均衡（Load Balancing）：是指将用户请求平均分配到多个计算节点，从而提升整体的计算性能。
- 自适应资源分配（Adaptive Resource Assignment）：是指系统根据当前的网络状况、应用特点、计算任务类型和负载状况，动态调整资源分配。
- 虚拟化（Virtualization）：通过虚拟化技术，将本地计算资源虚拟化，提供统一的计算环境，解决网络状况不稳定、计算资源不足等问题。
- 多框架协同（Multi Framework Collaboration）：是指支持多种应用框架和语言，包括但不限于移动应用程序、Web应用程序等。
- 时空协同（Spatio-temporal Coordination）：是指依据任务在不同时间和空间内的依赖关系，结合多源异构数据，进行任务的调度和协同。
- 安全边缘计算（Secure Edge Computing）：为了防止恶意攻击和数据泄露，MEC需要在网络边缘部署安全防护措施。
- 虚拟网络（Virtual Network）：通过虚拟网络技术，将多个私有云网络连接起来，形成公共的、安全的、透明的计算网络。
- 容器（Container）：一种轻量级、标准化的虚拟化技术，可以封装应用运行环境、依赖项、配置信息，用于部署、交付、管理和扩展应用程序。
- 容器编排工具（Container Orchestration Tool）：用于编排、管理、调度容器的工具。
- 边缘管理（Edge Management）：指管理边缘计算系统的过程，包括任务调度、资源管理、节点部署、安全防护等。
- 云导流（Cloud Egress）：指移动边缘计算节点向外发送数据、指令等过程。
- 云访问（Cloud Access）：指移动边缘计算节点通过云接口访问云服务的过程。
- 大数据分析（Big Data Analysis）：通过大数据处理、分析和挖掘，提取海量数据中的价值信息，促进智慧经济发展。
- 数据中心网关（Data Center Gateway）：一种用来对接不同网络类型的网络设备的协议，提供统一的管理、控制和数据转发功能。
- 小样本学习（Few-shot Learning）：是在无标注训练数据情况下，基于小样本学习方法，对数据进行分类、检测等。
- 漂移检测（Mobility Detection）：用于检测移动终端的移动模式、速度、路径、地理位置变化等。
- 边缘自治（Edge Autonomy）：指物理边界的限制，使终端能够做出更加独立的决策。
- 边缘弹性（Edge Elasticity）：是指能够动态调配计算资源、部署节点的能力，以适应终端设备的负载、节能等特性。
- 边缘自治（Edge Autonomy）：指物理边界的限制，使终端能够做出更加独立的决策。
- 边缘弹性（Edge Elasticity）：是指能够动态调配计算资源、部署节点的能力，以适应终端设备的负载、节能等特性。
- 大规模并行（Massive Parallelism）：是指对大规模数据进行分布式并行计算。
- 数据迁移（Data Migration）：指将数据从源节点迁移到目的节点的过程。
- 混合云（Hybrid Cloud）：是指混合IT架构，包括私有云、公有云和边缘云等。
- Fog computing（Fog computing）：一种云计算形式，侧重于在用户的终端设备、应用中部署计算节点。
- Edge cloud computing（Edge cloud computing）：一种云计算形式，侧重于在网络边缘部署计算节点。
- 移动计算平台（Mobile Compute Platform）：一种分布式系统，用于对接、整合移动终端、云服务、通信网络、地理信息等。
- 移动边缘交互（Mobile Edge Interaction）：是指终端用户与移动边缘计算节点之间的交互，包括采集数据、上传数据、执行计算、下载数据等过程。
- 移动边缘实时计算（Mobile Edge Realtime Computing）：是指实时的移动边缘计算，即在短时间内完成任务，在移动设备本地完成计算。
- 车联网（Car-to-Grid Communications）：车联网是利用汽车内部的感应系统来实现数据的收集、传输、处理和存储。
- IoT平台（IoT platform）：包括终端设备、传感器、计算平台、网关、云服务、通信网络等部件，打造一种物联网平台。
- 车辆联网（Vehicle-to-Grid Communications）：车辆联网与车联网类似，不同之处在于，车辆联网是利用车辆内置的感应系统来收集数据，而车联网是利用汽车的外部传感器来收集数据。
- 移动学习（Mobile Learning）：是指在移动设备上进行学习，以提高用户的学习效率、知识掌握速度和专业技能水平。
- 小移动边缘计算（Small Mobile Edge Computing）：是指在移动终端上嵌入计算节点，以降低成本和增加处理能力。
- 游戏机器人（Game Robots）：游戏机器人通过识别玩家动作、环境、物品、玩家属性等，获取有用的信息，并根据这些信息制作出精准的操作指令。
