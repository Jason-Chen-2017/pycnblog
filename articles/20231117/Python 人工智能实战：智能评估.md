# 2.核心概念与联系

## 概念理解
### 信用评级系统
1997年，美国联邦信用评级委员会(Federal Financial Institutions Examination Council, FFIEC)颁布了《公共行业信用评级管理办法》，该办法要求建立信用评级体系，并明确定义了信用评级的相关术语和标准。其次，该体系主要依据公众的信用报告和历史数据，根据不同的信用等级划分出不同的信用级别。再次，除了公众评级外，还有政府主管部门对信用债券、商业贷款、保险产品等进行评级，用于监督征信活动。

信用评级系统中最重要的两个组成部分就是：信用报告与评级。信用报告由个人或机构向评级机构提交的一份文档，描述个人在某项商品、服务上的信用情况。评级机构会根据收到的信用报告进行评级，并对个人进行信用等级划分。

### 评级模型
信用评级系统包括不同的评级模型，常见的有三种：
1. 单因素评级模型：即每一个信用报告只考虑一个因素，例如用户历史订单记录、投诉数量、满意度调查结果等。这种模型存在缺陷，容易受到单一因素的影响，并且无法准确反映客户的实际信用状况。
2. 多因素综合评级模型：即多个信用报告中使用不同权重分配的方式综合分析，得出最终的信用评级。多因素综合评级模型可以减少因素之间的交互作用，提高模型的鲁棒性和准确性。
3. 深层次多因素评级模型：即通过分析用户的行为习惯、偏好、欲望等深层次信息，来对用户的信用进行评级。深层次多因素评级模型可以更好的评估用户真实信用状况。

## 数据理解

这里我们会用到的数据集是CMU提供的案例数据集。该数据集包含来自五个银行的客户数据的四十万条信用报告，每一条信用报告都包含了一位客户的信用报告和评级标签，评级标签有五个档次，分别为A、B、C、D、E。其中，训练集和测试集的比例是8:2。

除此之外，还需要对这些信用报告进行特征提取。假设我们选择以下三个特征进行特征提取：
1. 用户历史订单记录：顾客在某个时间段内所做的所有交易记录，如交易金额、交易次数、交易频率等。
2. 投诉数量：顾客向银行投诉的次数。
3. 满意度调查结果：顾客完成的满意度调查的得分。
