                 

# 1.背景介绍


## 文件（File）
在计算机科学中，文件是指在存储设备上的数据集合，用来保存数据、文本、图形、声音、视频或程序等信息。文件具有存储、检索、修改、共享和保护等多种功能。我们通常把电脑中的各种文档、图片、音频、视频、程序等归纳称为文件。文件可以帮助我们进行数据的备份、整理、传输、编辑等工作。
## 文件读取（Read）
文件的读取就是将文件的内容从内存中读取出来。当计算机处理的数据量越来越大时，对数据的存储容量也会越来越大，因此需要通过文件的方式对数据进行管理、备份、转移、搜索等操作。文件读取涉及到两个主要概念：文件路径（Path）和文件名。
## 文件写入（Write）
文件的写入则是将数据从内存中保存到硬盘上。文件写入的过程包含三个基本步骤：打开文件、写入数据、关闭文件。文件写入后，可用于数据备份、恢复、备份、复制、打印、发送等操作。
## 操作系统（OS）
操作系统是管理计算机硬件和软件资源并提供用户接口的程序。不同的操作系统之间存在细微差别，但总体都遵循同一个标准化的接口规范，能够有效地管理计算机硬件资源和控制应用软件的运行。操作系统常用功能包括进程管理、内存管理、文件系统管理、网络通信管理、用户管理等。
# 2.核心概念与联系
## 文件路径（Path）
文件路径又称目录路径，它表示文件在磁盘上的位置。路径是一个字符串，以斜杠`/`分割各个层次的文件夹和文件名，用于唯一标识一个文件。文件路径的组成如下所示：
```
[盘符]:/[文件夹1]/[文件夹2]/.../[文件夹n]/[文件名].[扩展名]
```
其中，`盘符`代表磁盘驱动器名称；`文件夹`是指磁盘中的某个文件夹；`文件名`和`扩展名`分别对应文件的文件名和扩展名，两者中间用点`.`连接。对于Windows系统来说，默认的盘符为`C:`。
## 文件名
文件名即文件的实际名称，由字母、数字、下划线和其他特殊字符构成。文件名不允许有空格、汉字、中文标点、逗号、分号、感叹号、问号、点、冒号、竖线、正斜杠、反斜杠等特殊符号，最长不能超过255字节。文件名区分大小写，不能含有中文。

## 文件对象（File Object）
文件对象是对文件的一种抽象描述，包括文件名、文件路径、文件大小、创建时间、最后访问时间、最后修改时间等信息。通过文件对象可以对文件执行各种操作，如读取、写入、删除、移动、拷贝、重命名、搜索等。
## 绝对路径与相对路径
绝对路径是指从根目录（通常是磁盘驱动器的根目录）开始计算的完整路径。相对路径则是指相对于当前所在目录的相对路径，相对路径不需要指定具体的盘符和盘符路径。在Windows系统中，当前目录的相对路径一般以`.`开头，表示当前目录。
## 概览
通过以上介绍，我们可以看到文件是存储在计算机内、具有一定功能的重要信息资源。文件是存储在磁盘上的，而操作系统负责管理文件，确保文件安全、方便快捷地被访问、使用的。文件的读取、写入、操作和维护等操作通过文件路径完成，文件路径由文件名、盘符、目录和扩展名组成。
