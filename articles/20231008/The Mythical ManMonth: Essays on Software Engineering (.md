
作者：禅与计算机程序设计艺术                    

# 1.背景介绍


　　软件工程是一个很古老但却仍然非常重要的学科。在过去的一百年里，软件工程经历了极其曲折的发展，如硬件、数据库、操作系统、网络、人工智能等领域都产生了大量的成果。这些成果能够帮助人们解决复杂的问题，也促进了社会的进步。

　　但是软件工程领域的著述已经远远超过了三五十万字，而这一大部头的论文几乎每年都会产生新的研究方向和热点。其中就包括《The Mythical Man-Month: Essays on Software Engineering》一书，这本书是软件工程界最经典的著作之一。

　　1995年，出版社出版了此书。《Essays on software engineering》一书的作者是德国计算机科学家海克尔（Henri Cockcroft）和艾利斯蒂安·范农（Eric Feingold）。海克尔在大学期间参加过多年研究活动，对软件工程有着浓厚的兴趣。他写道：“在许多公司、大学、研究所、和其他组织中，我发现软件工程领域中的很多问题都是老生常谈的话题，比如开发效率低下、设计缺乏灵活性、缺少测试和管理、源代码质量低下等等。” 

　　范农则是美国软件工程领域的创始人之一，也是《电子报》（IEEE）标准委员会主席。在她看来，软件工程是一门需要高度的专业知识和技巧才能完成的学术工作。“软件工程需要对各种软硬件技术有深刻的理解和认识，它还涉及到对问题定义、功能规划、需求分析、设计实现、单元测试、集成测试、维护升级、用户界面设计、文档编写、项目管理等众多环节的流程化处理。” 

　　在这本书中，海克尔和范农通过大量论述和实践案例，深入阐述了软件工程的发展历史、理论基础、方法论和实践要点，并系统地讨论了软件开发过程中的各个阶段的关键技术难点及应对方式。从而帮助读者了解到，如何构建软件开发团队，如何合理分配任务和资源，如何制定软件质量保障计划，如何改善软件开发过程中的风险控制，如何进行项目管理，等等。 

# 2.核心概念与联系
　　《Essays on software engineering》一书围绕着两大核心主题：管理、人员组织。《Essays on software engineering》提到了软件工程中的三个主要阶段：需求开发、设计、实现。并举出了各个阶段的相关技术、方法、工具和人员。在每个阶段中，海克尔和范农都侧重阐述了其中的关键理念、关键技术、关键人员以及软件工程中所需的一些关键法律法规。因此，阅读这本书可以了解到软件工程的基本框架、主要角色及职责、各个阶段的关键技术要素、过程以及管理规范。

　　1.项目管理：项目管理是软件工程的重要组成部分，能够有效地组织、管理和控制整个软件开发生命周期中的各种活动。项目管理就是为了实现项目目标，确保项目进展顺利、按时完成而采取的手段。《Essays on software engineering》提到项目管理的两个核心主题：计划与协调。而海克尔认为，计划与协调是相辅相成的。项目计划是指计划一个项目或某个产品的目标、范围、资源和进度。项目协调则是指将项目计划、资源投入、质量控制、进度监控、人力资源管理、成本费用核算、风险控制等多个方面的信息转化为有效的行动。

　　2.人员组织：在软件开发过程中，需要有高素质的人才参与其中。《Essays on software engineering》提到六个角色：工程师、项目经理、软件顾问、客户代表、项目领导人、软件测试员。工程师负责分析需求、设计、开发，并配合项目经理完成开发工作。项目经理负责组织项目成员、管理进度，并制定开发计划和质量保证计划。软件顾问根据工程师给出的建议，提供软件开发方面的咨询和建议。客户代表对项目进行现场的支持和反馈。项目领导人总结项目的情况，向上级汇报并提出改进意见。软件测试员负责对软件产品进行测试和故障分析。

　　3.需求开发：需求开发即需求获取与分析。需求开发实际上是指对所开发的软件产品提出的用户需求进行收集、整理、分类、描述、评估、验证、以及转换为软件可执行的形式。需求开发一般包括需求确认、需求分析、业务分析、需求规格说明、验收审计等多个环节。

　　4.设计：设计阶段要求软件工程师深入考虑软件的结构、组成、行为、性能、用户接口等特性，以确定软件所应具备的能力。设计一般分为功能设计、界面设计、数据结构设计、模块设计、进程设计、数据库设计、组件设计、体系结构设计、编码规范、兼容性设计、安全性设计、版本管理策略设计、自动化测试设计等。

　　5.实现：实现阶段是开发人员按照设计文档进行源代码的编写，并进行编译、链接、调试等流程，使软件成为满足用户需求的实际可用产品。实现一般分为需求实现、模块实现、集成实现、系统测试、维护更新、风险控制、用户界面实现、文档编写、项目管理、代码审查等多个环节。
