# 2.核心概念与联系
## 数据仓库与数据集市
“数据仓库”（Data Warehouse）与“数据集市”（Data Mart）是两个常用的术语，用于描述基于多种异构源头收集的原始数据，经过提炼、清洗、转换、加工后得到的最适合支持业务决策的信息集合。两者之间的区别主要如下：

1. 逻辑层次上的不同。数据集市只是一种数据抽象，仅从某些维度上对原始数据进行汇总，可以看作是业务主题相关的零星信息，而数据仓库则是面向主题的综合性数据集合。数据仓库通常由多个数据集市组成，每一个数据集市都具有明确定义的主题、范围和粒度，因此更接近于业务需求，更易于理解；
2. 数据质量的差异。由于数据集市所抓取的数据往往是第三方提供的、可能存在噪声或错误的，因此质量难以确定，但数据仓库所提供的数据是经过人工或者自动清洗、转换、加工等处理之后的，其质量直接影响到最终结果，质量也更可靠。另外，数据集市更侧重于异构数据源，因此架构设计、存储维护等都是比较复杂的，而数据仓库则更偏向于结构化的宽表格型数据源，一般情况下架构简单、存储维护方便，缺点是不利于实时查询；
3. 使用层次上的不同。数据集市是针对特定应用场景下的一种数据服务，比如销售数据集市、供应链数据集市等，目的就是提供相关业务的信息，数据仓库则是对整个企业的数据进行综合性分析和决策支持。

## 类SQL语言与过程化语言
SQL语言和过程化语言是数据分析过程中最常用的两种语言。

1. SQL语言: SQL语言（Structured Query Language）是用于关系数据库管理系统的标准语言，包括Oracle、MySQL、SQL Server等，它提供了丰富的查询功能，能够满足各种复杂的数据分析任务。SQL语言的特点是基于关系，所有记录都存在数据库中，查询速度快。

2. 过程化语言: 过程化语言（Procedural Programming Language）是在编程语言的层次上实现程序的功能，它的基本单位是语句，包括赋值、条件判断和循环等。过程化语言的特点是基于过程，所有数据都存放在内存中，执行速度慢。

## 云计算与本地分析工具
云计算平台（Cloud Platform）是指通过互联网远程访问计算机硬件资源的系统。云计算平台的优势在于按需付费、弹性伸缩、灵活定制、数据安全保护等，使得公司可以快速部署和扩展服务器，降低成本。同时，云计算平台还可以减少服务器的运营成本，并通过云平台提供的大数据分析工具和服务，帮助企业降低数据分析和决策的成本。

本地分析工具（Local Analysis Tools）是指通过个人电脑或PC运行的数据分析工具，例如SPSS、Tableau、SAS等。本地分析工具的优势在于可以提供更快速的计算能力，但缺点在于无法像云计算平台那样快速部署大数据环境，需要耗费更多的时间和金钱投入。

## 大数据生态圈
数据采集、存储、处理、分析等领域，目前业界已经形成了一套庞大的生态圈。大数据生态圈主要分为四个阶段：

第一阶段：批处理（Batch Processing）—— 最初采用基于磁盘的数据库技术，通过离线的方式处理海量数据。

第二阶段：集群计算（Cluster Computing）—— 通过集群技术将数据分布到不同的计算机节点，并利用分布式计算进行快速分析。

第三阶段：流计算（Stream Computing）—— 通过流数据处理框架进行实时的分析，可以实时响应变化，适合处理实时数据。

第四阶段：分布式计算（Distributed Computing）—— 在海量数据上采用分布式计算框架进行分布式分析，大规模数据处理成为可能。
