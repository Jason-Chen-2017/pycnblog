# 1.背景介绍


SureChEMBL是一个基于免费、开放的数据源汇总、整理并提供查询服务的平台。其主要包括药物结构数据库、生化数据集成、虚拟Screening、科学文献检索、结构可视化等功能。SureChEMBL共收录了超过7万个来自各个国家和地区的化合物信息、病理数据及试验设计。可以用于分析单个化合物或蛋白质的活性，从而发现潜在的抗体并预测药物有效性。通过从海量数据中挖掘模式，SureChEMBL助力医药领域发现高价值的化合物供应商，提升疾病治愈率，优化药物开发流程。目前已经成为国际领先的结构数据库。
SureChEMBL的目标是在生命科学领域提供更高效、准确、全面的药物发现与预测能力，进一步促进人类健康。随着人类基因工程的迅速发展，对宏观环境的影响越来越加关注。而化学实体的变异已成为许多临床问题的关键。借助SureChEMBL，我们可以将宏观环境对药物产生的影响与单一化合物或蛋白质的潜在生物活性联系起来，帮助医生更准确地判断药物的适应症。
# 2.核心概念与联系
SureChEMBL数据结构分为两级：第一级是供应商级别的数据，包括药物结构数据库和生化数据集成；第二级是科研项目级别的数据，包括实验设计、科学文献检索、结构可视化等功能。每一项数据都有相应的编号，供用户查询。比如，对于一个供应商A的药物结构数据库，他可能有A001, A002,..., An，分别对应该供应商的不同药物。而针对某一项目，比如一个2019年在伦敦大学做的关于肝癌基因组的实验设计，它的编号可能是PRJ001-2019。当用户输入编号后，可以查看该项目的相关信息，如实验设计方法、实验结果等。此外，还可以进行结构可视化，查看指定蛋白质的三维结构。
# 3.核心算法原理和具体操作步骤以及数学模型公式详细讲解
在SureChEMBL中，采用图谱技术，构建了一个化合物相互作用网络。它把化合物分子式、活性值、绑定信息等相关信息存储为节点，把它们之间的关系用边表示。每一条边记录了一种化学反应的发生机理，可以用来预测化合物的活性值。算法和流程如下：

1）提取所有实验数据，包括化合物的分子式、活性值和结构。将数据导入SureChEMBL数据库中。

2）利用自然语言处理工具，对实验数据的描述文本进行处理，提取关键词、关系等信息。

3）根据关键词搜索实验数据，生成检索结果。

4）对于检索到的项目，利用路径搜索技术，找到其中与查询化合物最接近的化合物。

5）依据用户指定的活性范围、转换条件等进行筛选，选择出符合条件的化合物。

6）利用机器学习算法，对得到的候选化合物进行预测。利用图谱计算网络，建立化学反应的发生机理。

7）根据预测结果和实际结果，评估算法的性能。

为了处理高维度数据，SureChEMBL采用了分布式计算技术。数据导入时会自动切分到不同的服务器上，每个服务器只保存自己需要处理的部分数据。同时，SureChEMBL使用SQL语言进行数据库操作，避免了复杂的JOIN操作，实现了高效的数据访问。除此之外，SureChEMBL还通过建立机器学习模型，对数据进行分类、聚类、回归等预处理操作。

最后，由于SureChEMBL的分布式特性，可以在不断增加实验数据的情况下，不间断运行，保持最新的数据状态。因此，SureChEMBL可以快速响应实验数据，并给出最新的化合物预测。

# 6.附录常见问题与解答Q1：SureChEMBL数据是否属于公开数据？
SureChEMBL的所有数据都是属于公开数据。你可以自由地使用SureChEMBL的数据。但是，如果你要进行商业用途，需要遵守SureChEMBL的法律条款和隐私协议。

Q2：SureChEMBL的数据来源有哪些？
SureChEMBL的数据来源分为两个方面：首先，SureChEMBL提供的药物结构数据库，来自于合作的各家药物公司。其次，SureChEMBL接受来自不同实验室的实验数据，包括PCR实验、核苷酸组装、靶向试验、结构方程建模等。这些实验数据则来源于众多的科研机构，并经过整理保护。

Q3：SureChEMBL平台上的数据是如何整理、保护的？
SureChEMBL采用的是ISO/TC 20914认证的安全检测协议。我们在国际上使用AES加密算法对实验数据和结构数据进行加密存储，只有授权人员才能访问。另外，SureChEMBL会定期检查数据中的错误、异常情况，并对失效或损坏的数据进行清除。
