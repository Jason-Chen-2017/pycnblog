# 1.背景介绍


企业资源计划（Enterprise Resource Planning，简称ERP）是一个信息系统，它将企业的日常管理活动进行了规范化、自动化，并通过统一的数据平台将数据交给相关人员处理、分析和决策。ERP系统的目标是实现企业管理效率的提升，并减少管理成本、降低人力投入。其理念主要包括基于人的中心的管理、高度的自动化程度、数据集成、流程优化和决策支持等。

基于以上ERP理念，业务建模就是构建企业内部运行的过程及各个环节之间的关系和流程，进而指导ERP系统如何更好的运作。业务建MODEL的作用在于：

1. 提升企业管理能力。业务建模能帮助公司制定出符合要求的组织结构、业务流程、财务控制体系、人事管理制度、生产制造控制方案等，能够有效的指导公司各部门的工作职责划分，优化产品和服务的生产流程，提高管理效率；

2. 优化企业运营效率。业务建模使公司能够建立起完善的财务报表和金融数据，可以预测企业的经营风险和前景，并对资源的配置进行合理规划，增加企业的竞争优势；

3. 为公司创新提供坚实的基础。业务建模构建的是企业资源流转的路径图，如果不考虑流程上的不足和配套工具的缺失，将无法真正意义上开拓创新，而且会影响公司长期的发展；

4. 促进企业科技成果转化。业务建模所涉及到的知识产权，会成为该领域的基础性知识，在未来的科技革命中扮演着重要角色；

5. 维护企业核心利益。业务建模具有很强的社会意义和政治性，但也需要公司保持稳定的金钱观，不能随波逐流。

因此，业务建模对于企业管理者来说尤为重要。同时，业务建模是ERP系统建设的一项重要环节，因此作为一名资深技术专家或IT经理，其博文还得具备较强的逻辑思维、表达能力、沟通协调能力、执行力和团队精神。

在业务建模的过程中，企业面临着多方面的挑战。一方面，从实际情况出发，不同企业业务的复杂性和需求差异很难统一；另一方面，管理层在业务建模时需要经历分析、设计、组织、实施、验证等多个阶段，这对知识、技能、经验的要求也是相当高的。在这种情况下，如何充分发挥个人能力，根据业务特点进行个性化的业务建模方法，是成功的关键。

# 2.核心概念与联系
## 2.1 ERP概念
**企业资源计划（Enterprise Resource Planning，简称ERP)** 是一种信息系统，是一套完整的企业管理和协同软件，用于管理企业的信息和资产。该系统通过将企业内部的各种管理活动进行标准化、自动化、统一化，通过对数据的收集、整理、分析、存储、交换、传输、共享等功能，实现对企业的管理、监控、管理协调、资源利用、信息传递、信息分析等全过程的控制。ERP系统主要采用计算机软件来实现，包括数据库、网络通信、应用程序等软硬件，采用网络管理方式（如IP、TCP/IP、远程访问、HTML等）来实现信息共享和数据传递。

**实体(entity):** ERP的对象是一个个实体。实体又可分为两种，一是物理实体，即各种实物，如商品、设备、人力、材料等；二是虚拟实体，即各种虚拟的存在物，如人、组织、过程、文档、资产、订单、合同等。实体是指企业内部重要的管理对象，如客户、供应商、生产厂商、仓库管理员、销售人员、采购人员、财务人员、秘书等。每个实体都有其属性、行为和关系，通过实体间的联系和关系，构建起一个实体集合体。

**关系(relationship):** 实体之间存在各种关系，这些关系通过网络连接在一起。例如，A实体与B实体存在关系R，则表示A与B之间存在某种联系，如联系、合作、依赖、沟通、分享、归属、资源、使用等。不同的关系对应着不同的边界，如存在关系通常是一种一对一的关系，反映物理实体之间的依赖关系；使用关系通常是一种一对多的关系，代表物理实体之间的共用关系。

**数据(data):** 数据是用来描述客观事物的数值、文字、图像、声音等符号形式。ERP中的数据可以记录实体的各种属性、行为、关系、状态等信息，并按照一定格式存放在计算机数据库中。数据可用于下一步的决策、生产、管理和控制活动。

**应用(application):** 应用是ERP系统中的一个模块，通过应用可以实现对实体之间的信息交互和数据的共享。ERP的应用由ERP系统中的各种模块组成，如基础数据管理、人事资源管理、财务管理、采购管理、制造管理、项目管理、质量管理等。

**规则(rule):** 规则是ERP系统中的一类数据，它定义了整个ERP系统中各个模块的操作规则。规则控制了用户在ERP系统中的权限、数据输入、数据输出、数据的共享、数据的安全、数据流转等方面的行为。规则还可以定义特定场景下的事件驱动、规则引擎和事务处理机制。

**接口(interface):** 接口是ERP系统与外部系统的连接通道。接口由协议、规范、格式、API等组成。它定义了数据交互的方式和规则，使得ERP系统可以和其他系统进行交互，如其他ERP系统、外部系统、终端设备等。

## 2.2 ERP系统
**ERP系统** 由以下构成：

1. **信息管理系统（IMS）:** IMS是ERP系统最外层的主体，负责对实体、数据、关系、规则等信息进行收集、整理、存储和传递。它根据组织结构、流程设计、业务模型、管理控制对象以及法律法规等方面的要求，对企业的业务信息进行结构化、标准化、语义化、数据化处理。

2. **企业组织架构管理（ORM）:** ORM是ERP系统中的核心子系统之一，它以树状结构的组织结构，实时呈现和向上汇总企业内部各单位、部门、人员、岗位等信息，以及各组织之间的联系。ORM的核心功能是管理企业的组织架构，协助业务管理者了解企业的整体结构，促进业务运行的顺畅。

3. **信息采集系统（ICS）:** ICS主要用于对企业内部各种数据源（如财务数据、人力资源数据、SCM数据等）进行采集、清洗、转换，并存储到信息管理系统中。它采用统一的模式来采集数据，包括SQL、XML、文本等多种格式。

4. **智能决策系统（IDS）:** IDS是ERP系统的核心组件之一，它采用机器学习、人工智能、统计学等手段，基于历史数据、当前数据、规则、上下文信息等多方面因素，对管理决策进行优化，提升决策效率。IDS可提供业务预测、优化方向、KPI指标等指标数据，实现对管理决策的自动化支持。

5. **管理控制系统（MCS）:** MCS是ERP系统的核心子系统，它采用灵活、统一、自动化的方法，对实体、关系、数据、规则进行管理，并实施监控和控制。MCS的任务是对企业的管理控制、业务运营、信息安全、合规性管理等各方面进行管理、控制和保障。

6. **审计系统（AS）:** AS是ERP系统的支撑系统，用于对企业管理活动进行审计，追踪数据变化、对账核算等，确保管理活动的准确、完整和有效。

7. **外部系统集成（OIS）:** OIS是ERP系统的辅助系统，用于和外部系统进行集成，如SAP、Oracle、CRM、SharePoint、OA等。OIS的任务是实现信息共享、协同、分析、决策和应用。

8. **日志管理系统（LMS）:** LMS是ERP系统的一个支撑系统，用于对系统运行过程中的事件和异常进行记录、分析、报告和回溯。LMS的任务是保障系统运行正常、维护系统运行效率、提升系统运营效率。

ERP系统的构成与功能完善，使得ERP系统能够承载不同类型企业的信息和管理要求，满足不同行业不同规模的企业的管理需求。
