# 2.什么是大数据
## 2.1 定义
“大数据”的英文名称“big data”由两部分组成：数据（data）和高速（fast）。也就是说，“大数据”就是指在过去的一段时间内生成、储存、管理、分析的数据，这些数据数量巨大、数据类型多样、数据价值丰富，而且产生速度极快。“大数据”是数据科学、机器学习等新兴学科所研究的问题。“大数据”产生的背景主要包括以下几点：

1. 大规模：由于数据集的大小越来越大，单个计算机的运算能力不能完全支持其处理。为了解决这一问题，云计算、分布式系统、NoSQL数据库、实时计算框架等技术被开发出来。
2. 数据增长：数据也随着时间的推移不断增长。过去几十年间，由于技术革命带来的发展，大数据产业蓬勃发展。2010年至今，全球每天产生超过100亿条数据。
3. 数据价值：人们越来越重视数据的价值。在金融、生物医疗、互联网、制造领域，传统的基于规则的数据分析已无法满足需求。相反，越来越多的公司和组织都依赖于大数据技术来获取更多价值。例如，通过大数据分析，可以实现产品定位、推荐、预测、定向广告、精准营销、客户服务等功能。
4. 数据挖掘：“大数据”对数据的分析和挖掘方面带来了新的机遇。在当前的信息技术驱动下，数据量越来越大，挖掘者需要找到突破口来提升效率和准确度。例如，互联网数据源庞大，但目前仍难以将其挖掘出来提供商业价值。
5. 应用场景：在众多应用场景中，“大数据”正在发挥着越来越重要的作用。如金融行业，大数据可用于建模、风险控制、营销决策、欺诈监控等；电信网络运营商，通过大数据预测故障并提前调整资源调度；电子商务，利用大数据提升用户体验，促进交易及流通的规模；互联网公司，使用大数据提升用户黏性和新闻传播效果；制造业，从海量数据中挖掘出有价值的商业模式。

## 2.2 大数据分类
### （一）按结构分类
大数据按照结构分为离线数据、实时数据、结构化数据、非结构化数据、半结构化数据、标记数据、图像、文本、音频、视频等。如下图所示：


- **离线数据**：指的是已经在存储设备上进行一定形式的整理、清洗、结构化的原始数据。这是一种静态的数据，比如文件、日志等。对于这种数据来说，获取速度比较快，但缺乏实时性。适合于长期分析和统计。
- **实时数据**：指的是那些快速产生、交换、流通并且经过一定的处理才能得到有效应用的各种数据。实时的特点决定了它的时效性，一般不会太久。因为实时的数据往往来自各类传感器、摄像头、GPS等。它是动态的、敏感的、实时的。所以，实时数据处理的方法主要采用一些特殊的方法。例如，实时计算框架（如Storm、Spark Streaming），流处理工具（如Kafka Streams），以及实时查询语言（如Elasticsearch Query DSL）。
- **结构化数据**：结构化数据即表格型数据。它是指具有固定字段和结构的事先定义好的信息集合，是固定的结构数据，如电话簿、联系人、学生信息、车辆信息等。结构化数据的获取方式和特性差异很大，但数据处理方法上却有很多共同点。典型的结构化数据处理方法有关系型数据库（如MySQL、Oracle），搜索引擎（如Solr、ElasticSearch），数据仓库（如Hive、Impala），以及数据分析工具（如Pig、Hadoop MapReduce）。
- **非结构化数据**：非结构化数据指的是一些没有固定格式的数据，如网站访问日志、邮件、文档、照片、视频、音频等，其中，文本、图像、声音、视频属于非结构化数据。非结构化数据的获取方式和特性也不同，但它们通常使用大数据处理工具进行处理。典型的非结构化数据处理方法有数据采集工具（如Flume、Scribe），消息队列（如ActiveMQ、RabbitMQ），文本处理工具（如Lucene、StromIK）等。
- **半结构化数据**：半结构化数据指的是具有不同结构的半结构化数据，如JSON、XML、HTML、日志数据、元数据等。这种数据类型通常比较复杂，因为其结构既不是键值对映射关系，也不是固定结构的。这样的数据类型需要对数据进行解析才能转换成结构化数据，所以处理方法也比较繁琐。典型的半结构化数据处理方法有XPath、正则表达式、半结构化数据库（如MongoDB），以及半结构化数据分析工具（如XQuery、MapReduce）。
- **标记数据**：标记数据又称松散数据，它是指以文本、数字或其他符号表示的数据。这些数据只能作为信息的一种形式存在，不能直接用于任何分析。但是，由于无结构化、不可索引，因而又会引起一些问题。典型的标记数据处理方法有标注工具（如OpenNLP、Stanford NER），搜索引擎（如Solr、ElasticSearch），以及信息提取工具（如Apache Tika）。
- **图像数据**：图像数据即位图、矢量图形、光栅图形、声像图形等。它们都是数字图像。图像数据通常是具有高度维度的，通常采用JPEG、PNG、GIF、TIFF等多种格式保存。处理图像数据的方法主要有计算机视觉技术（如特征提取、聚类、理解、检索、识别、描述），以及图像识别工具（如OpenCV、TensorFlow）。
- **文本数据**：文本数据是指包括各种形式的文字、电子邮件、日志、评论、聊天记录、短信等。这些数据多是无结构的，因而难以分析。但文本数据由于结构简单，便于处理。典型的文本数据处理方法有文本处理工具（如Tokenizer、Stemmer、StopWordRemover），搜索引擎（如Solr、ElasticSearch），以及文本分析工具（如Apache Spark、Lucene）。
- **音频数据**、**视频数据**：音频数据和视频数据也属于非结构化数据，主要处理方式与图像、文本类似。

### （二）按特征分类
大数据按照特征分为结构化、半结构化、非结构化、汇总数据、元数据、质量数据、关联数据、反馈数据、倾斜数据等。如下图所示：


- **结构化数据**：结构化数据指的是固定字段和结构的数据，它可以直接用来分析。结构化数据具备明显的结构，易于进行处理。它还具备两个独特的特征：第一，所有数据项都被指定一个唯一标识符，如ID。第二，所有数据项都包含的时间戳，以方便对数据进行时序分析。结构化数据通常保存在关系型数据库中。
- **半结构化数据**：半结构化数据指的是具有不同结构的数据。半结构化数据往往是半结构化的，难以用来做统一的分析，但可以进行定制分析。半结构化数据通常保存在NoSQL数据库中。
- **非结构化数据**：非结构化数据指的是含有文本、图像、声音、视频等非结构化数据。非结构化数据往往与结构化数据混合在一起。非结构化数据无法直接用来做分析，但是可以通过非结构化处理工具进行分析。非结构化数据通常保存在HDFS文件系统中。
- **汇总数据**：汇总数据是指对某些数据集进行汇总、整合的过程。汇总数据往往包含来自不同源的多个数据集。汇总数据往往无法用来做分析，但可以用来评估性能或跟踪数据的变化。汇总数据通常保存在报告数据库中。
- **元数据**：元数据是关于数据的数据。元数据通常是在数据创建或上传过程中自动添加到数据中的数据。元数据可以帮助研究人员和分析人员更好地理解数据。元数据通常保存在数据仓库中。
- **质量数据**：质量数据是指关于数据质量的信息。质量数据可以帮助企业知道数据的真实性和有效性。质量数据通常保存在数据仓库中。
- **关联数据**：关联数据是指不同数据之间的联系。关联数据可以在数据库之间建立联系，从而使数据之间能够进行分析。关联数据往往比较难以检测，但它可以提供有用的见解。关联数据通常保存在图数据库中。
- **反馈数据**：反馈数据是指算法对特定任务的输出结果。反馈数据通常来自不同的数据输入，包括原始数据、模型训练数据、测试数据等。反馈数据通常会被用于改善算法或设计新的算法。反馈数据通常保存在报告数据库中。
- **倾斜数据**：倾斜数据是指数据中某些方面的分布比其他方面更加倾斜。倾斜数据可能会影响分析结果，因而需要对其进行处理。倾斜数据往往不能够独立于其他数据进行分析。倾斜数据通常保存在数据仓库中。

## 2.3 大数据发展趋势
根据国外一家名为The Economist Intelligence Unit的数据显示，截止2020年，美国的大数据市场估计约占到了全球八分之一的份额，以致于有很多人担心未来五到十年的发展趋势。美国的国家安全局（NSA）最新发布的一份报告称，美国在2016年进行数据泄露调查后，发现国防部、情报部门、贸易代表处等机构曾在暗网上传播大数据和个人隐私信息。为了应对这些威胁，美国政府采取了一系列限制措施，包括数据共享、保护用户隐私、实施数据治理、清理犯罪数据。此外，美国政府还在积极筹划对数据科学家进行更严苛的培训，加强对大数据技术的研发。但是，美国的“大数据”未来仍将充满挑战。根据IDC的数据预测，2020年，全球只有中国、印度、日本、韩国、台湾、德国等少数几个国家拥有自己的超大规模数据中心，据估计，到2025年，这一数据中心的数量可能达到五六百个。另外，5G和人工智能将会改变整个经济和产业链，尤其是金融领域，未来可能出现更多的消费升级潮。未来大数据将会逐渐成为企业数据驱动型的创新模式，推动金融、制造、零售、物流、生物医药、安全、供应链等各行各业的创新发展。
