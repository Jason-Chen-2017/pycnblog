# 1.简介
  

对于云计算领域来说，数据成为其主要支撑，数据的产生、存储、使用、共享、传输、保护，无不直接或间接影响着企业的核心业务。因此，作为一名架构师，更应该对数据管理进行全面考虑。 

数据治理的目的是为了保证数据的准确性、完整性、时效性、正确性，避免数据被篡改或者泄露造成损失。数据治理有三大基础任务：

1.数据发现：通过对业务系统中的数据资产进行自动化探索、分析及报告，发现异常、不合规的数据资源，从而帮助相关人员制定数据安全和合规策略；

2.数据价值管理：基于数据分类、标签化、编制数据资产清单等方法，建立数据价值评估体系，实施数据价值评估并制定相应策略，完善数据资产结构；

3.数据访问控制：基于用户角色、组织架构和数据权限等因素，为数据提供不同级别的数据访问控制，实现数据保密和管理；

数据治理需要建立信息 governance framework (IGF)，围绕数据治理主题建立四项核心理念：数据价值优先、数据身份可控、数据共享化、数据连续性与同步化。如此，才能真正做到数据真金白银的保管和运营。本专栏将通过以下系列文章，为读者详细介绍这些理念，阐述数据治理的核心知识、方法论以及实践技巧。

# 2.数据价值优先
数据价值优先理念基于数据生命周期法则，认为数据产生的目的就是用于为业务带来价值的目的，也就是说，数据价值直接决定了数据是否要被保存、使用、共享甚至被销毁。那么如何定义数据价值呢？可以从两个方面去衡量：

1.客观数据价值（objective value）：客观数据价值指的是能够对数据质量产生直接影响的数据价值，比如，商业价值、经济利益、个人利益等。

2.主观数据价值（subjective value）：主观数据价值指的是不能量化的数据价值，包括用户满意度、产品满意度、服务质量、工作效率、社会影响力等。

数据价值优先理念要求数据主管部门以客观数据价值为标准，确立数据价值观，防止数据价值偏离中心，确保数据价值导向。数据主管部门应充分关注数据的生命周期，定期深入挖掘数据价值，为数据提供有效保障，进一步提升数据价值。

# 3.数据身份可控
数据身份可控理念强调，数据所有者（data owner）不是数据生成者，也不是数据消费者。只有数据主管部门才拥有完整的数据。数据所有者、使用者、管理者、监督者、参与者都应该履行数据主人的职责，尤其是在保护、收集、使用、处理、共享、存档、归档、流转、删除等环节中。因此，数据主管部门应定期进行数据安全管理，精心维护数据隐私和完整性。

# 4.数据分享化
数据分享化理念鼓励数据所有者通过共享方式让数据得到更多的受众，促进数据共享成果更多地流动起来。同时，数据主管部门应建立有效的数据沟通机制，倾听、接受不同群体的需求，包括技术专家、管理者、使用者等，最大程度地激活数据潜在价值，实现数据价值共享。

# 5.数据连续性与同步化
数据连续性与同步化理念强调，数据的所有方面都需要保持一个高度一致性，没有任何一个环节出现数据孤岛。如此才能确保数据准确、完整、时间戳连贯，实现数据价值最大化。数据主管部门应制定数据完整性管理计划，推进数据备份方案，建立数据集成、同步化、准确性验证机制，确保数据在使用过程中的完整性和一致性。
