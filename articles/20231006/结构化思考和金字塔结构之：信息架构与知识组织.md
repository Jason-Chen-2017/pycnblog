# 1.背景介绍


信息架构(IA)是组织构建、运营、管理及沟通其业务目标所需的策略、计划、制度、工具、资源、关系等一系列信息的过程和方法。IA作为一个重要的信息生产、交流、沟通和管理的手段，在实现企业业务目标的同时也促进了人力资源和组织架构的优化，提升了企业的绩效和竞争能力。但是，缺乏有效的系统性、科学的方法论对IA管理非常不利。因而，人们开始认识到IA的理念、体系建设、应用实践之间存在着长期的、互相关联的联系，而这套体系的形成往往需要以系统性、科学的方式进行考虑。这种思维方式源自西方哲学家海德格尔所倡导的结构思想，认为世界由各个不同的层次组成，并通过连接不同层次的功能实现协同工作，达到全面而合理的效果。
著名科技思想家马丁·加德纳教授也提出，人类社会是一种金字塔结构的生态，其中包括三种不同等级的组织：（1）基层组织——最底层的组织和个人；（2）中层组织——相当复杂的团队和机构；（3）上层组织——整体的政权和组织形式。每个层次都依赖于较低层的组织提供功能支持。因此，我们必须以金字塔的视角来观察世界，充分理解我们的处境和局限，并找到一条从基层向上发展的道路。只有在这个层次上我们才能真正理解组织和人事之间的关系、责任、权力和关系。
在企业组织管理中，可以将组织划分为五层——战略层、关键层、机会层、人才层、平台层。根据企业组织规模的不同，每一层都会有一些特有的组织结构和职能，但它们都共享三个基本原则：任务重心、绩效目标和行为准则。在战略层，组织关注的是公司战略规划和商业模式的设计；在关键层，组织着眼于核心产品或服务的开发和营销；在机会层，组织要吸引和留住人才，创造价值创造条件，激发创新潜力；在人才层，组织要培养并建立优秀的人才，包括高管、专家、学生和其他角色；在平台层，组织通过提供各种平台和服务，为公司创造持续增值的环境。为了有效地运用信息技术、数字经济和网络空间，企业采用信息资产管理(IIM)系统，这是一个基于云计算、网络、数据分析、人工智能等领域的集成管理系统。
传统信息架构的管理方法主要集中在以下几个方面：

1. 分类法——按照管理目标和风险程度进行层次划分。
2. 目录编制——将大量信息分散分布在各个地方，使得管理者难以掌握完整的架构。
3. 结构图绘制——树状图、矩阵图、流程图等多种图表可以帮助管理人员快速了解架构概况。
4. 文档管理——文档分类、归档、索引、搜索等技术手段管理大量信息，耗费大量的人力物力。
5. 团队组织——各种各样的办公室、支部、组、小组，无时无刻不在交流与合作，沟通效率极低。

因此，我们必须重新定义信息架构的管理，提升管理效率，创新管理模式，确保信息资产管理系统顺畅运行。如何解决这一难题呢？《结构化思考和金字塔结构之：信息架构与知识组织》正是通过提出“结构化思考”的方式，深入探讨信息架构的本质和内涵，以及如何建立健康的、结构化的、协同的组织，有效应对信息爆炸、反噬、监测、预警、评估、决策、沟通等一系列现代信息社会新形势下的挑战。
