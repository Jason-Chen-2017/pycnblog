# 1.背景介绍


树（Tree）是一种抽象数据类型（ADT），它由n(n>=0)个结点组成一个具有层次关系的集合。每个结点代表一个元素或记录，并且分为若干子树。树通常被用来表示结构化的数据，例如文件目录、语法树等。树的常用术语包括：根节点、分支节点、叶子节点、路径长度、高度、深度、森林、子树、父亲结点、儿子结点、兄弟结点、堂兄弟结点、祖先结点、后裔结点、边（边可以是实线也可以是虚线）、重心、联通分量、顶点、权值等。以下是树的几何图像示例：

## 2.1 二叉树（Binary Tree）
在计算机科学中，二叉树是每个节点最多有两个子树的树结构。通常子树被称作“左子树”（left subtree）和“右子树”（right subtree）。二叉树常用于实现二元搜索树、二叉排序树及其他数据结构。以下是二叉树的几何图像示例：

### 2.1.3 满二叉树的性质
* 在满二叉树中，除最后一层外，每层上的结点数都是最大结点数的整数倍，并且最小的叶子节点都出现在左边的树中。
* 如果将满二叉树的第i层上第j个节点的两个孩子看做左孩子和右孩子，则其父节点就是满二叉树第i−1层上第floor((j+1)/2)个节点。

### 2.1.4 二叉查找树（Binary Search Tree）
二叉查找树或者是一棵空树，或者是具有下列性质的树：

1. 左子树的所有节点的值均小于根节点的值。
2. 右子树的所有节点的值均大于根节点的值。
3. 左右子树也分别为二叉查找树。

其中，左子树不空时，左子树上所有节点的值均小于根节点的值。右子树不空时，右子树上所有节点的值均大于根节点的值。

### 2.1.5 哈夫曼编码
哈夫曼编码是一种变长编码方式，对原数据进行无损压缩。它的基本思路是利用信息的熵来确定编码过程。

假设要处理一组字符集{c1, c2,..., cn}，这些字符的概率分别为{p1, p2,..., pn}。构造一颗二叉树，以便使树中每个内部节点表示一种字符的概率分布。在构造过程中，选择两个具有最小概率的字符作为父节点，构造一棵新的二叉树，直至不能再继续划分。最终得到的树只有两个分支，其左分支存放字符c1，右分支存放字符c2，以此类推。

这种编码方式的特点是，树的高度越低，压缩效果越好。

## 2.2 BST树
二叉查找树（BST，Binary Search Tree）是一种特殊的二叉树。它满足一下几个条件：
1. 每个节点的左子树只包含小于当前节点值的元素，右子树只包含大于当前节点值的元素。
2. 中序遍历输出的结果是从小到大的顺序。

因此，BST可用来快速查找某些元素，比如字典中的字母表查找、二维区域查找等。

BST树的存储方法有两种：
1. 采用链表法（Linked List Method），即将同一层的节点链接在一起。
2. 采用数组法（Array Method），即用数组来模拟堆栈的结构。

数组法优点是查找时间复杂度为O(logn)，缺点是空间复杂度较高。链表法优点是空间复杂度低，查找时间复杂度为O(logn)。因此，实际应用中通常采用数组法。
