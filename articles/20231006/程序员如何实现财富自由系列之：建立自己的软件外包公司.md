# 1.背景介绍


软件外包是一个颇受欢迎且新潮的行业。根据国内外统计数据显示，截至2019年，全球累计已有超过7亿美元的收入被软件开发商支配，并形成巨大的利润空间。相比个人拥有一套完整的软件产品，许多企业在雇佣程序员或者系统架构师从事软件外包工作，可以节省大量的时间、金钱和精力。同时也能享受到更高的质量保证，提升竞争力。但是，如何正确开展软件外包业务，成为一个成功的企业，还存在很多难题需要解决。软件外包人员面临的主要问题是：如何保障自己的权益，如何快速找到合适的人才？以及如何持续地扩张业务，使得企业获得足够的收入。
为了解决以上这些问题，作者将通过“程序员如何实现财富自由”系列的文章，分享他自己的经验和方法。文章会先从软件外包服务市场、优质软件外包公司以及软件外包员工三方面阐述软件外包是什么，给出不同的角度来看待软件外包。接着，讲述软件外包人员要面对的最重要的问题——如何保障自己的权益。文章将结合自己的实际案例介绍如何管理自己的项目、如何招聘合适的软件外包公司员工、以及如何经营自己的软件外包公司。最后，作者会以编程技术和专业经验总结一下这一系列文章的特点和长处，并试图找寻下一步的发展方向。
# 2.核心概念与联系
软件外包服务市场
软件外包服务市场指的是由不同类型的软件工程师组成的资源池。他们通常提供各种形式的软件外包服务，比如网站制作、应用程序开发、网络安全审计、硬件维修等。这些人均具有高度的专业技能，并经过良好的培训和职业化训练，能够按时交付客户需求。但是，软件外包服务市场往往存在一些问题。首先，大多数软件外包服务都是按项目进行的，很少能够直接与顾客签订长期的合同。其次，由于软件外包服务通常只承担极小部分的开发工作量，因此，很难建立起真正的协作关系。此外，软件外包服务的定价往往较高，通常需要购买许可证或月费才能使用服务。另外，虽然软件外包服务市场已经存在了一段时间，但仍然缺乏规模效应。由于软件开发者们的需求数量巨大且增长迅速，所以软件外包服务市场正逐渐变成了一个超级市场。
优质软件外包公司
优质软件外包公司是指那些提供世界一流服务水平和完善服务体系的软件外包公司。它们的独特的服务模式和严格的流程标准吸引了众多的软件工程师和系统架构师。优质软件外PACK公司一般都有丰富的知识和经验积累，并且能够创造性地利用各种外包服务项目。优质软件外包公司通常以年收入超过百万美元为目标，并在市场上占有一席之地。除此之外，优质软件外包公司还具有开展软件外包业务的能力、团队协作能力和市场资源优势。
优质软件外包公司的优势表现在以下几个方面：第一，拥有世界顶尖的软件工程师，他们掌握着独特的技艺和专长。优质软件外包公司在给予软件工程师工作报酬时往往不用考虑福利费，而是选择性地减免费用，甚至免费赠送软件。第二，优质软件外包公司通过专业的软件外包服务体系，确保了服务质量的一致性。第三，优质软件外包公司虽然为软件工程师提供了诸如培训、协助支付相关税费、制定内部绩效考核等福利补贴，但并没有任何其他类型的福利。第四，优质软件外包公司具有强大的市场资源优势，可以满足大量的软件工程师的求职要求。
软件外包员工
软件外包员工是指由软件工程师、系统架构师、项目经理等组成的外部代理人。软件外包员工按照固定薪酬、按计划安排工作、接受系统和项目管理训练、参加软件外包活动等条件，协助优质软件外包公司进行项目开发、维护及相关业务支持。软件外包员工可以按照公司标准提供服务，也可以私自工作，但是，由于外包人员不可能像其他普通劳动者一样得到足够的职业道德约束，所以，保障软件外包员工的合法权益是一个值得研究的课题。

案例一：如何建立自己的软件外包公司
软件外包企业通常由一群对软件开发有热情的工程师、具有软件开发经验的系统架构师和对市场有一定了解的专门人士组成。为了创立自己的软件外包公司，需要具备一些基本的素养和能力，包括：
1.领域知识：软件外包公司要围绕客户所需的业务领域展开活动。熟悉该领域的业务规则、竞争环境、用户习惯等，才能为客户提供符合自己需求的软件外包服务。
2.软技能：软件工程师、系统架构师和项目经理们共同组建软件外包公司，他们往往具备软技能，包括沟通能力、学习能力、团队精神、创新能力、适应能力、职业操守等。为了保持专业的服务水准，需要严格遵守服务规范和行为准则。
3.专业知识：软件外包公司需要有较强的专业知识才能做好技术支持。技术人员需要理解客户需求，掌握该领域的最新技术，能够快速地掌握新技术应用和解决方案。
案例二：如何管理自己的项目
作为一个项目经理，管理软件外包项目是一个非常重要的任务。下面，作者将通过实操案例，向大家展示如何管理项目。
比如，公司有一个合作伙伴A需要提供一款应用程序，软件工程师S负责设计软件需求文档，项目经理P负责监督软件进度，包括项目启动、计划跟踪、质量控制、项目评审等各个环节，软件工程师S通常采用结构化分析、面向对象技术和设计模式的方法，编写可测试的代码，提升软件质量；项目经理P负责沟通客户、制定项目计划、实施项目管理方法，确保项目顺利进行；软件工程师S和项目经理P一起合作，完成整个软件开发流程，最终交付给客户，完成合作协议。下面，我们再来具体展开软件项目管理中需要注意的事项。

1.项目计划阶段：
项目计划阶段一般分为需求调研、设计阶段、风险分析、预算制定四个阶段。
- 需求调研阶段：对于软件外包项目来说，需求调研的主要内容是搜集客户需求信息，包括需求描述、功能列表、性能指标、UI图、界面布局、使用场景等。需要尽可能多的搜集信息，以便为客户提供更精准的服务。
- 设计阶段：软件工程师通常采用结构化分析、面向对象技术和设计模式的方法，编制软件需求文档、设计类图、状态图等文件，确定软件架构和组件关系、接口设计。为满足客户的需求，需要细化业务逻辑，把复杂的功能分解为多个简单模块，并且对每个模块进行详细设计。
- 风险分析阶段：项目前期通常会遇到各种各样的风险，比如客户需求变化、技术问题、竞争对手攻击、内部政策限制、第三方介入等。针对这些风险，需要制定相应的应对策略和方法，防止出现意想不到的问题。
- 预算制定阶段：在项目启动阶段，需要制定预算，包括项目开发费用、技术支持费用、项目管理费用、测试费用、办公费用等。预算制定的关键在于估算每一项费用的金额，如果预算不足，可能会导致项目延期或失败。因此，在制定预算之前，需要先确认费用清单和所有人的预算分配情况。
2.项目执行阶段：
项目执行阶段主要包括计划、进度、风险、评审和闭包四个阶段。
- 计划阶段：在项目的早期阶段，需要根据需求、预算、进度和其它条件制定项目计划，包括软件开发时间、模块设计、接口设计、风险处理、进度跟踪等。计划中的关键点在于明晰需求范围、制定模块分配、确定进度和计划、识别风险并制定应对措施。
- 进度阶段：在项目开发过程中，需要对软件工程师完成的模块进行进度跟踪，确保每一个模块都按计划完成。同时，需要跟踪并解决项目中的风险，避免出现意料之外的问题。
- 风险阶段：当项目进入后期阶段时，可能会遇到新的风险，比如测试人员的问题、后端服务器的问题、系统崩溃、用户反馈等。项目经理需要根据实际情况及时调整计划，确保项目按时交付客户。
- 评审阶段：项目结束后，需要对项目进行评审，以判断其是否达到预期要求、是否存在重大问题、是否达到销售目标。项目经理需要根据评审结果做出决策，如追加模块、降低质量标准、采取补偿措施等。
- 闭包阶段：项目的最后一站，需要对项目进行总结、回顾、讨论、交流和整改，以确保项目顺利运行。项目经理需要汇总项目经历、提出意见建议，并取得广泛的反馈。

3.项目管理工具：除了上述的项目管理过程，还有一些软件项目管理工具可以帮助组织和管理软件项目。其中比较常用的有甘特图、工作日程表、计划工时表、缺陷管理工具、看板和敏捷冲刺方法等。当然，软件项目管理还需要结合实际情况，建立一套完整的管理体系，包括技术、管理、组织和激励三个层面的管理。
案例三：如何招聘合适的软件外包公司员工
招聘软件外包员工是一个重要且繁琐的过程。首先，需要选定相应的岗位，然后筛选招聘人员，逐一面试，确认合适人员。同时，也需要制定招聘标准，如职业道德、语言表达、英文阅读、技术基础等。招聘者必须具有以下几个方面的特质：
1.懂软件开发：招聘者必须熟悉该领域的开发流程和方法，并且能够熟练地运用相关工具。
2.承担压力：招聘者必须承受项目中各种工作上的压力，包括时间压力、工作量压力、质量压力、合作压力等。
3.不断学习：招聘者必须具备坚韧的学习能力，否则他们就无法胜任这份工作。
4.注重细节：招聘者的每一个细节都必须要做到位，包括语言表达、作息时间、个人品牌、个人风格等。
招聘时，也需要注意以下几点：
1.求职简历写作：求职简历一定要充满详细的信息，包括个人信息、教育背景、工作经验、项目经历、职称等。同时，简历中还要突出软件外包公司的优势，如支付能力、服务质量、项目管理能力等。
2.面试技巧：面试过程中，除了看候选人的基本信息、项目经历、编程技能、培训经历等外，还要注意语言表达、面试技巧、项目经验、团队协作能力、解决问题能力、时间管理能力等。
3.接受邀请：面试完毕后，招聘者应接受邀请，接受转介绍或是直接进入电话面试，而不是垫话相待。
4.拒绝无关人员：为了提高招聘效率，软件外包公司应该精心筛选外包人员，不能招聘非相关人员，比如项目经理、HR等。
案例四：如何经营自己的软件外包公司
软件外包企业往往由一群对软件开发有热情的工程师、具有软件开发经验的系统架构师和对市场有一定了解的专门人士组成。为了创立自己的软件外包公司，需要具备一些基本的素养和能力。下面，我们将介绍软件外包公司的运营管理方式。
1.招聘：软件外包公司的招聘工作由专门的招聘人员进行管理，它主要是为了吸纳优秀的软件工程师、系统架构师、项目经理等人才，为客户提供专业、可靠的服务。在招聘过程中，需要评估招聘人员的个人和专业技能、能力、工作态度、职业道德、个人魅力和经验，以及与客户的交流能力，以及对项目的认识和理解程度。
2.培训：软件外包企业需要对招聘人员进行必要的培训，以确保招聘者能够胜任软件外包工作。培训的内容包括计算机操作系统、软件开发环境、开发工具、业务流程、项目管理、团队合作、软件测试等，这些内容也是软件外包人员的必备技能。
3.管理：软件外包公司需要设定管理目标，制定一系列工作流程和规范，以确保项目的顺利进行。管理过程中，需要关注项目进度、质量、风险、合同履行、人力资源等方面。
4.营销：软件外包公司需要进行营销活动，如宣传推介、广告营销、媒体合作、合作伙伴拓展等。营销的目的在于促进软件外包的知名度和影响力，让客户觉得自己是在与专业人士交流。
5.激励：软件外包企业需要为员工提供激励机制，鼓励他们完成艰苦的工作。激励的手段包括奖金、股票、培训、期权等。激励越多，员工完成工作的积极性就会增加。
案例五：如何保障自己的权益
软件外包业务主要面临的两个难题是保障权益、扩张业务。保障权益涉及到法律法规的规范、隐私权的保护、知识产权的保护，以及对员工权益的保护。扩张业务主要面临着投入和回报之间的矛盾，如何减少风险、提高效益，就是扩张业务的一个关键。下面，我们将分享作者的保障权益的经验。
1.知识产权保护：软件外包业务的最大风险在于侵犯知识产权。软件开发工程师为了服务客户，往往会开发很多高质量的软件，因此，对自己的劳动成果拥有知识产权是十分重要的。知识产权保护的方式有许多，包括软件著作权保护、专利保护、商标注册、版权保护、开源协议保护等。
2.隐私权保护：软件外包公司一般不会直接向客户提供源代码，因此，软件外包企业需要对客户的隐私权进行保护。隐私权保护的最基本原则是，不收集和保存不必要的个人信息。
3.法律法规：软件外包公司和相关产业都受到很多法律法规的限制，比如合同约定和违法行为的保护、知识产权保护、消费者权益保护、劳动合同法律义务、政府行政部门对外包工作的监管等。因此，软件外包企业需要遵守相关法律法规，保障自身权益。
