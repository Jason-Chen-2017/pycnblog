# 2.核心概念与联系
## 2.1什么是Docker？
Docker是一个开源的容器化平台，可以将应用程序及其依赖环境打包成可移植的镜像文件，通过 Docker 镜像运行容器。它提供简单易用的命令接口与 API 用来构建、发布、管理以及部署应用程序。 Docker 的架构包含三个主要组件：

1. Docker 引擎(Daemon)：负责运行Docker，构建、运行和分发Docker容器。

2. Docker 对象(Objects)：用于定义构建镜像、运行容器等操作行为的对象，比如镜像（Image）、容器（Container）、网络（Network）、卷（Volume）。

3. Dockerfile：用来自动化创建Docker镜像的文件，可以通过Dockerfile指令来创建自定义镜像，使得Docker具备一键式打包能力。

## 2.2什么是容器？
简单来说，容器就是一种轻量级虚拟机。但它不像传统的虚拟机一样模拟整个操作系统，而是利用宿主机的操作系统功能，并且只提供必要的隔离功能，从而运行指定应用，解决资源限制问题，提高性能。

容器与虚拟机最大的不同点在于，容器并不是完整的虚拟机，因此启动速度非常快。相比之下，创建一个完整的虚拟机则要花费更多的时间，因为需要为每个虚拟机加载操作系统内核等所有内容。而且，容器可以直接访问宿主机的目录和资源，因此可以在宿主机上很方便地实现跨机器或跨云平台的数据交换。

## 2.3什么是Kubernetes？
Kubernetes 是 Google 在 2015 年推出的基于 Docker 及其它开源项目打造的开源容器编排调度系统。它的核心功能包括：

1. 服务发现和负载均衡：Kubernetes 提供了集群内部的 DNS 和基于 Label 的 Service 概念，可以方便地实现服务的发现与负载均衡。

2. 配置和自动化管理：Kubernetes 提供了一系列机制来保证集群中应用配置的一致性，并提供简单的更新方式来完成应用的扩缩容。

3. 存储编排：Kubernetes 支持动态的 Volume、PersistentVolume 及 PersistentVolumeClaim 机制，用户无需关心底层存储的挂载过程，使得存储编排变得十分容易。

4. 自我修复机制：Kubernetes 提供了健康检查机制，能快速检测到故障，并在短时间内重启或替换失败的应用容器。

## 2.4为什么需要容器技术？
容器技术具有以下几个优点：

1. 跨平台兼容：容器技术可实现应用程序的一次编译，随处运行，支持 Linux、Windows、Mac OS X等主流操作系统。

2. 更高效的资源利用率：由于容器共享宿主机操作系统内核，因此可以节省资源开销，提升硬件利用率。

3. 简化的部署流程：不需要在不同的机器上安装不同版本的软件，只需要拉取指定的镜像即可快速部署应用程序。

4. 弹性伸缩性：容器技术能够快速、灵活地扩展应用程序规模，满足用户业务增长需求。

5. 可观测性：容器技术提供了统一的日志记录和监控视图，可方便地进行运维操作和故障排查。
