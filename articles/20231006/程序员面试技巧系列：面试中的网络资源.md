# 2.核心概念与联系
## TCP/IP协议族
- TCP(Transmission Control Protocol)传输控制协议：是Internet上流通的协议，负责数据在两个应用程序间的通信。它通过建立可靠连接，保证数据包按序到达，没有任何重复或损坏。主要用来保证通信安全性，防止丢包、乱序等情况发生。
- IP(Internet Protocol)网际协议：是Internet上互连的协议，用于路由选择，分组转送，并确保数据完整性。
- UDP(User Datagram Protocol)用户数据报协议：是Internet上不可靠协议，可以提供广播和多播服务。它不保证数据包到达顺序，也不会重复发送，但会进行丢弃，并且可靠性较差。主要用作实时应用（如视频和语音通信）、文件传输、DNS等。


- HTTP(Hypertext Transfer Protocol)超文本传输协议：基于TCP的应用层协议，用于从WWW服务器传输超文本文档。
- DNS(Domain Name System)域名系统：用于域名和IP地址解析，由IETF制定，使用UDP协议。
- SMTP(Simple Mail Transfer Protocol)简单邮件传输协议：用于发送电子邮件，负责将信息传送给收件人。

# 3.核心算法原理和具体操作步骤以及数学模型公式详细讲解
## 请求/响应模式

1. client端向server端发送一个HTTP请求。
2. server端接收到这个请求后，创建一条新的连接，与client建立起通信，然后把请求返回给client。
3. 如果请求成功完成，那么server就会关闭此连接；如果出现错误或者超时，server端可能会继续等待下一次请求。
4. client接收到server的响应后，检查响应状态码，并根据相应处理结果做出相应的动作。比如，如果响应状态码是“200 OK”，则表示请求成功完成，可以继续处理响应的数据。

## TCP三次握手与四次挥手
TCP协议采用三次握手建立连接，四次挥手释放连接。

### 握手阶段
客户端与服务器之间首先要建立一个TCP连接，需要经过“三次握手”过程。

第一次握手：建立连接。

在客户端浏览器中输入URL并点击回车键之后，客户端与服务器之间开始了TCP的三次握手。

第三次握手：确认连接。

在第二次握手过程中，服务器还需要确认客户端的发送能力和接收能力是否正常，所以服务器会再次发送确认消息给客户端。客户端接收到服务器的确认消息后，也进入了“已连接”状态。这样，客户端和服务器之间的TCP连接就建立好了，可以进行数据的交换。


第二次握手：同步序列号。

第三次握手就是客户端和服务器相互告知序列号，以便双方都能够正确接收数据。至此，整个TCP三次握手流程结束。

### 挥手阶段
当客户端或者服务器想要终止TCP连接时，需要发送FIN、ACK和SYN+FIN包。

第一步：发送FIN包通知对方我方即将关闭连接。

第二步：接收FIN包。

第三步：发送ACK确认包。

第四步：接收ACK确认包。

最后，连接彻底关闭。
