# 2.核心概念与联系
## 2.1 近似算法
### 2.1.1 概念
近似算法(Approximation algorithm) 是一种在计算中常用的方法，它是一种随机化的搜索算法，通过将一个复杂的目标函数映射到另一个容易处理的目标函数，从而降低目标函数的计算复杂度，使得算法具有较好的时间性能。近似算法的应用主要包括求解最优化问题、图论问题、数值模拟等。近似算法作为一种高效的计算方法已经得到了广泛应用。比如，Google的PageRank算法就是一种典型的近似算法。近似算法可以有效地减少算法的时间复杂度，并取得较好的结果。但是，如何设计准确的近似算法以及其效果是否能达到要求仍然是一个悬而未决的问题。

一般来说，当目标函数具有很多局部最小值时，不能保证找到全局最小值，此时用近似算法就可以找到相对较优解，即所求解问题的近似解，具有很大的实际意义。举个例子，假设有一个求解圆周率问题，已知圆周率的值为3.14，但不知道它的精确形式，如何利用计算机计算它的近似值呢？

常见的近似算法有：

1. 分治法（Divide and Conquer）
2. 迭代法（Iterative Methods）
3. 递归法（Recursions）
4. 模拟退火算法（Simulated Annealing）
5. 遗传算法（Genetic Algorithms）
6. 蒙特卡洛法（Monte Carlo Methods）
7. 插值法（Interpolation Methods）

## 2.2 分治法
分治法(Divide and conquer)是指将一个复杂的任务，如找出数组中的最大元素、排序等，按照一些简单的方式分割成若干个子问题，分别求解子问题，然后再合并这些子问题的解，就得到原问题的解。其基本思想是将待解决的问题划分成两个或更多的相同或相似的子问题，递归求解各子问题，最后合并子问题的解得到原问题的解。通常情况下，子问题的规模逐渐缩小，最终得到原问题的解。分治法是一种经典的分而治之的方法，十分重要和普遍。

### 2.2.1 动态规划问题
动态规划（Dynamic Programming, DP）是一种运筹学的模型，是求最优化问题（Optimization problem）的一类算法。动态规划有三个要素：

- 一组状态变量：它表示在不同的条件下，算法所处的不同阶段可能的情况，也称为动态规划问题的“状态空间”。
- 选择变量：它描述了如何从当前状态转移到后继状态，也就是在每一步都做出决策，选择动作，也称为动态规划问题的“决策变量”。
- 奖励变量：它表示在转移过程中获得的价值或收益，也称为动态规划问题的“价值函数”。

动态规划的步骤如下：

1. 将问题分解成子问题，至少包含一个阶段；
2. 定义状态变量；
3. 确定选择变量；
4. 定义奖励变量；
5. 根据已知信息计算已知的状态或价值函数；
6. 由已知的状态函数推导出未知的状态函数；
7. 重复第三至第六步，直到所有的状态或价值函数都已计算出来。

动态规划常用于求解很多优化问题。其中，最短路径问题和矩阵连乘问题就是动态规划的典型应用。

### 2.2.2 多项式时间算法
对于一个计算问题，如果存在某一类算法，使该算法运行时间与输入的大小呈多项式关系，则称该算法为多项式时间算法。根据多项式时间算法，可以判定算法的好坏。对于某些问题，只能构造出多项式时间的算法。

举例如下：

- 判断某个数是不是质数，存在整数因子只有2、3、5时，用判断是否为2的倍数来检查；
- 求解最长公共子序列，动态规划可以在$O(nm)$的时间内求出最长公共子序列；
- 求解最短路径问题，Dijkstra算法的时间复杂度为$O(m\log n)$；
- 求解背包问题，在容量限制下的0-1背包问题，可用动态规划在$O(nW)$的时间内求解；
- 判断无向图是否连通，可以使用DFS和BFS，并进行计数。两者的复杂度都是$O(m+n)$。

因此，多项式时间算法提供了一个比较理想的时间界限，不仅能够处理规模巨大的计算问题，而且还能够适应各种环境的变化。比如，对于字符串匹配问题，在时间复杂度上，Rabin-Karp算法由于自身的优越性和优秀的分析性，已经成为多项式时间算法的代表。
