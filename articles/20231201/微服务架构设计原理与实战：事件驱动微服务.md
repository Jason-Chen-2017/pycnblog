                 

# 1.背景介绍

微服务架构是一种新兴的软件架构风格，它将单个应用程序拆分成多个小的服务，每个服务都运行在其独立的进程中，这些服务可以通过网络进行通信。这种架构的出现是为了解决单一应用程序的规模和复杂性带来的问题。

事件驱动微服务是一种特殊的微服务架构，它将业务流程拆分成多个事件，每个事件都可以被多个服务处理。这种架构的出现是为了解决单一事件处理的规模和复杂性带来的问题。
