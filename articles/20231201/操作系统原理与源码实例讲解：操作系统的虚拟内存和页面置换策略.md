# 2.核心概念与联系

虚拟内存和页面置换策略的核心概念包括虚拟内存、物理内存、页面、内存分页、内存保护、内存分配、内存碎片等。这些概念之间存在着密切的联系，它们共同构成了操作系统内存管理的基础架构。

虚拟内存是操作系统为程序提供的一个抽象层，使程序能够访问更大的内存空间，而不受物理内存限制。虚拟内存通过将程序的内存空间划分为固定大小的页（page）来实现。每个页都有一个唯一的内存地址，用于在内存中进行访问。

物理内存是计算机系统中的实际内存资源，用于存储程序和数据。物理内存是有限的，因此操作系统需要进行内存管理，以确保程序能够有效地使用内存资源。内存管理包括内存分配、内存保护、内存碎片等方面。

内存分页是虚拟内存的一个关键技术，它将程序的内存空间划分为固定大小的页，以便于内存管理。内存分页使得操作系统可以更容易地进行内存分配和内存保护。同时，内存分页还为页面置换策略提供了基础。

内存保护是虚拟内存的另一个重要功能，它确保程序在访问内存时不会越界或访问不被允许的内存区域。内存保护通过将虚拟内存地址转换为物理内存地址来实现，从而确保程序只能访问自己分配的内存空间。

内存分配是操作系统内存管理的一个关键任务，它负责为程序分配和释放内存资源。内存分配可以是动态的，也可以是静态的。动态内存分配允许程序在运行时请求内存资源，而静态内存分配则需要在程序编译时确定内存需求。

内存碎片是操作系统内存管理中的一个问题，它发生在内存空间被不规则地分配和释放时。内存碎片可能导致程序无法分配足够的内存资源，从而影响程序的性能。内存碎片的解决方案包括内存分配策略的优化、内存回收机制的设计等。
