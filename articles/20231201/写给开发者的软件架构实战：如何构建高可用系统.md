# 2.核心概念与联系

在构建高可用系统之前，我们需要了解一些核心概念。这些概念包括：

- 容错性：系统的容错性是指系统在出现故障时能够如何自动恢复。容错性是高可用性的重要组成部分。
- 负载均衡：负载均衡是指将请求分发到多个服务器上，以确保每个服务器的负载均匀分布。负载均衡是实现高可用性的关键技术。
- 故障转移：故障转移是指当一个服务器出现故障时，将请求转发到另一个服务器上。故障转移是实现高可用性的重要手段。
- 数据备份：数据备份是指将数据复制到多个服务器上，以确保数据的安全性和可用性。数据备份是实现高可用性的关键技术。

这些概念之间存在着密切的联系。容错性、负载均衡、故障转移和数据备份都是实现高可用性的关键手段。我们需要综合考虑这些概念，以确保系统的高可用性。

- 随机选择：随机选择是用于选择备份服务器的算法。我们可以使用随机选择算法来选择备份服务器，以确保数据的安全性和可用性。
- 一致性哈希：一致性哈希是用于实现数据分布和故障转移的算法。我们可以使用一致性哈希来实现数据分布和故障转移，以确保系统的高可用性。
