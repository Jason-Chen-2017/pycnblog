# 2.核心概念与联系

## 2.1所有权

所有权是Rust中最基本的概念之一。在Rust中，每个值都有一个拥有者，该拥有者负责管理该值的内存。当拥有者离开作用域时，Rust会自动释放该值占用的内存。这种自动内存管理机制使得Rust能够避免内存泄漏和野指针问题。

## 2.2借用

借用是Rust中的一种资源共享机制。当一个值被借用时，其他代码不能直接访问该值的内存。这样可以确保内存安全，避免数据竞争。

## 2.3生命周期

生命周期是Rust中用于跟踪所有权关系的一种机制。生命周期标记法使得Rust能够在编译时检查所有权关系，从而确保内存安全。

## 2.4模式匹配

模式匹配是Rust中的一种用于解构数据结构的方法。通过模式匹配，我们可以根据数据结构的结构进行不同的操作。

## 4.2所有权和借用

```rust
fn main() {
    let s = String::from("Hello, world!");
    let len = calculate_length(&s);
    println!("The length of '{}' is {}.", s, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```

在上述代码中，我们定义了一个`String`类型的变量`s`，并调用`calculate_length`函数来计算`s`的长度。在`calculate_length`函数中，我们使用`&String`类型的参数`s`来表示对`s`的借用。这样可以确保`s`的内存安全。

## 4.3生命周期

```rust
fn main() {
    let s1 = String::from("Hello, world!");
    let len1 = calculate_length(&s1);
    let s2 = String::from("Hello, Rust!");
    let len2 = calculate_length(&s2);
    println!("The length of '{}' is {}, and the length of '{}' is {}.", s1, len1, s2, len2);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```

在上述代码中，我们定义了两个`String`类型的变量`s1`和`s2`，并调用`calculate_length`函数来计算它们的长度。由于`s1`和`s2`的生命周期不同，我们需要使用生命周期标记法来表示这一点。在`calculate_length`函数的参数类型中，我们使用`&String`类型的参数`s`来表示对`s`的借用，并使用生命周期标记`s`的生命周期`'a`。这样可以确保`s`的内存安全。

## 4.4模式匹配

```rust
fn main() {
    let num = Some(5);
    match num {
        Some(x) => println!("The number is: {}", x),
        None => println!("The number is not present."),
    }
}
```

在上述代码中，我们定义了一个`Option`类型的变量`num`，并使用`match`语句来匹配`num`的值。当`num`的值为`Some(5)`时，我们会打印出"The number is: 5"；当`num`的值为`None`时，我们会打印出"The number is not present."。这种模式匹配机制可以帮助我们更好地处理不同的情况。
