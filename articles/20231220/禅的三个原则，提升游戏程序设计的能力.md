# 2.核心概念与联系

## 2.1 无意识

无意识是指在设计游戏程序时，不去强迫自己去思考和分析每个细节。相反，我们应该尽可能地让自己的身心放松下来，让自然的思维和创造力发挥作用。这样的设计方法可以帮助我们更快地找到更好的解决方案，同时也能让我们更容易地发现游戏中的潜在问题和优化空间。

## 2.2 无想

无想是指在设计游戏程序时，不去强迫自己去遵循某种固定的思维方式或者设计模式。相反，我们应该尽可能地保持开放和灵活的心态，尝试不同的设计方法和思维方式，以便找到更好的解决方案。这样的设计方法可以帮助我们更好地适应游戏中的不断变化和挑战，同时也能让我们的设计更加独特和创新。

## 2.3 无取

无取是指在设计游戏程序时，不去强迫自己去追求某种固定的目标或者效果。相反，我们应该尽可能地保持对游戏的热爱和兴趣，让自然的创造力和想象力发挥作用。这样的设计方法可以帮助我们更好地体验到游戏的乐趣，同时也能让我们的设计更加独特和有趣。

## 4.3 无取

假设我们需要设计一个更有趣的游戏，游戏中有多个玩家需要被移动。我们可以采用以下方法来实现无取：

1. 保持对游戏的热爱和兴趣。
2. 让自然的创造力和想象力发挥作用。
3. 不去追求某种固定的目标或者效果。

代码实例：
```python
class GameObject:
    def __init__(self, x, y, player_id):
        self.x = x
        self.y = y
        self.player_id = player_id

    def move(self, dx, dy):
        self.x += dx
        self.y += dy

class Game:
    def __init__(self):
        self.objects = []

    def add_object(self, object):
        self.objects.append(object)

    def move_objects(self, dx, dy):
        for object in self.objects:
            object.move(dx, dy)

    def get_object_by_player_id(self, player_id):
        for object in self.objects:
            if object.player_id == player_id:
                return object

game = Game()
player1 = GameObject(0, 0, 1)
player2 = GameObject(100, 100, 2)
game.add_object(player1)
game.add_object(player2)
game.move_objects(10, 0)
game.move_objects(0, 10)
player1_position = game.get_object_by_player_id(1).x
player2_position = game.get_object_by_player_id(2).x
```
