## 3.1 动态规划

动态规划是一种常用的算法优化方法，它可以用来解决一些具有最优子结构的问题。动态规划的核心思想是将问题拆分成多个子问题，并将子问题的解存储在一个表格中，以便于后续使用。

动态规划的具体操作步骤如下：

1. 确定问题的状态和参数。
2. 创建一个表格，用于存储子问题的解。
3. 使用递归关系来计算表格中的每个值。
4. 返回表格中的最终结果。

动态规划的数学模型公式可以表示为：

$$
dp[i] = \min_{0 \leq j \leq i-1} dp[j] + f(i, j)
$$

其中，$dp[i]$ 表示第 $i$ 个子问题的解，$f(i, j)$ 表示将第 $i$ 个子问题分解为第 $j$ 个子问题时的代价。

## 3.2 贪心算法

贪心算法是另一种常用的算法优化方法，它通过在每个步骤中选择最优解来逐步优化算法。贪心算法的核心思想是在每个步骤中选择最优解，并假设这个最优解会导致最终结果更优。

贪心算法的具体操作步骤如下：

1. 确定问题的目标函数。
2. 在每个步骤中，选择能够最大化或最小化目标函数的解。
3. 重复步骤2，直到问题得到解决。

贪心算法的数学模型公式可以表示为：

$$
\arg\max_{x \in X} f(x)
$$

其中，$f(x)$ 表示目标函数，$X$ 表示解空间。

## 4.1 动态规划示例

考虑一个经典的最长公共子序列问题。给定两个字符串 $s$ 和 $t$，找出它们的最长公共子序列。

```python
def longest_common_subsequence(s, t):
    m = len(s)
    n = len(t)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s[i - 1] == t[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
```

在这个示例中，我们使用了动态规划的思想来解决最长公共子序列问题。我们创建了一个二维表格 `dp`，用于存储子问题的解。然后，我们使用递归关系来计算表格中的每个值。最后，我们返回表格中的最终结果，即最长公共子序列的长度。

## 4.2 贪心算法示例

考虑一个经典的零钱兑换问题。给定不同面额的零钱和一个总金额，找出组成总金额的最少零钱数量。

```python
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for coin in coins:
        for i in range(coin, amount + 1):
            dp[i] = min(dp[i], dp[i - coin] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1
```

在这个示例中，我们使用了贪心算法的思想来解决零钱兑换问题。我们创建了一个一维表格 `dp`，用于存储每个金额所需的最少零钱数量。然后，我们使用贪心策略来选择最优解，即每次选择能够使总金额减少最多的零钱。最后，我们返回最终结果，即组成总金额所需的最少零钱数量。
