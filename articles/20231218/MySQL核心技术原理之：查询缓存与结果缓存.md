# 2.核心概念与联系

## 2.1查询缓存

查询缓存是MySQL中的一个核心功能，它的主要目的是将查询结果缓存在内存中，以便在后续的查询请求中直接返回缓存结果，从而减少数据库的负载，提高查询性能。

查询缓存的工作原理是：当一个查询请求到达时，MySQL首先会检查查询缓存中是否存在与之对应的结果。如果存在，则直接返回缓存结果；如果不存在，则执行查询并将结果缓存在查询缓存中，然后返回结果。

## 2.2结果缓存

结果缓存是MySQL中的另一个核心功能，它的主要目的是将查询结果缓存在内存中，以便在后续的查询请求中直接返回缓存结果，从而减少数据库的负载，提高查询性能。

结果缓存的工作原理是：当一个查询请求到达时，MySQL首先会检查结果缓存中是否存在与之对应的结果。如果存在，则直接返回缓存结果；如果不存在，则执行查询并将结果缓存在结果缓存中，然后返回结果。

## 2.3联系

查询缓存和结果缓存都是为了提高MySQL的查询性能而设计的。它们的主要区别在于：查询缓存是针对特定的查询语句进行缓存的，而结果缓存是针对查询结果进行缓存的。
