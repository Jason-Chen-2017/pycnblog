# 2.核心概念与联系

音视频处理和媒体技术涉及到的核心概念包括：

1. 编码与解码：编码是将原始音视频信号转换为数字信号的过程，解码是将数字信号转换回原始音视频信号的过程。
2. 压缩与解压缩：压缩是将原始音视频信号压缩为较小的数据文件的过程，解压缩是将压缩后的数据文件还原为原始音视频信号的过程。
3. 存储与播放：存储是将音视频信号保存到存储设备中的过程，播放是将存储在存储设备中的音视频信号播放出来的过程。
4. 传输与接收：传输是将音视频信号从一个设备传送到另一个设备的过程，接收是将传输过来的音视频信号接收到并处理的过程。

这些概念之间的联系如下：

- 编码与解码是音视频处理的基础，它们决定了音视频信号在数字域中的表示方式。
- 压缩与解压缩是音视频信号传输和存储的关键，它们决定了音视频信号在网络和存储设备中的占用空间。
- 存储与播放是音视频信号的本地处理，它们决定了音视频信号在用户设备中的播放质量。
- 传输与接收是音视频信号的远程处理，它们决定了音视频信号在网络中的传输速度和质量。

## 3.1 编码与解码

### 3.1.1 编码

#### 3.1.1.1 压缩法

##### Huffman 编码

Huffman 编码是一种基于哈夫曼树的压缩法，它根据符号出现的概率来分配编码。Huffman 编码的核心思想是将概率低的符号编码为短的二进制编码，概率高的符号编码为长的二进制编码。

Huffman 编码的具体操作步骤如下：

1. 统计符号出现的概率，将概率低的符号放入优先级队列中。
2. 从优先级队列中取出两个概率最低的符号，将它们合并为一个新的符号，并将新符号放入优先级队列中。
3. 重复步骤2，直到优先级队列中只剩下一个符号。
4. 根据合并的规则，构建哈夫曼树。
5. 根据哈夫曼树，为每个符号分配二进制编码。

##### Lempel-Ziv-Welch (LZW) 编码

LZW 编码是一种基于字符串匹配的压缩法，它将输入序列划分为最长没有出现过的连续子序列的序列，并将这些子序列编号为一个索引表中的索引。

LZW 编码的具体操作步骤如下：

1. 创建一个空的索引表，将输入序列的第一个字符作为表的第一个元素。
2. 读取输入序列的下一个字符，如果该字符没有出现过，将其添加到索引表中并返回其索引。
3. 如果该字符已经出现过，将当前字符和上一个字符组合为一个新的字符串，如果该字符串没有出现过，将其添加到索引表中并返回其索引。
4. 重复步骤2和步骤3，直到输入序列结束。
