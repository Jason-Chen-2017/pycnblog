# 3.基本概念术语说明
首先，本文将介绍编程思维训练的一些基本概念、术语和方法论。
## 3.1.定义
- **编程思维训练**（Programming Thought Training）：是指通过一定的训练方法或教材，用以训练学生的编程思维。
- **编程思维**（Programming Thought）：是指作为程序员进行计算机编程所需的分析、设计、编码等过程中的思考方式，包括反省性思维、解决问题能力、创新能力、总结能力、计划执行能力、团队合作精神、工具使用能力等。
- **编程语言**（Programming Language）：是计算机编程的工具，用于描述计算机指令的语法和语义。目前，最常用的编程语言有C、Java、Python、JavaScript、PHP、Swift、Go等。
- **编译器**（Compiler）：是一种翻译程序，将编写的源代码编译成机器码或目标代码。编译器主要分为前端编译器、后端编译器和整个编译器。前端编译器负责解析、检查源代码，生成中间代码；后端编译器负责优化中间代码，并生成可执行文件；整个编译器完成整个编译过程。
- **解释器**（Interpreter）：是一种运行时环境，它读取字节码并逐行执行。解释器也称为脚本语言，是一种不需要编译就能运行的编程语言。解释器一般运行速度慢，而且占用的内存较多，适用于快速原型开发。目前，主流编程语言如Python、JavaScript、Ruby、Perl、PHP都属于解释型语言。
- **类型系统**（Type System）：是编程语言的一种特征，用来描述变量的数据类型。在不同的编程语言中，类型系统的定义也不同，如静态类型语言（例如Java、C++等）、动态类型语言（例如Python、JavaScript等）、弱类型语言（例如Ruby、Perl等）。
- **调试**（Debugging）：是指程序运行出错或者存在逻辑错误时的排除、修复流程。
- **单元测试**（Unit Testing）：是指对模块或函数的自动化测试，目的是为了验证每个模块或函数的正确性。
## 3.2.编程能力测评
- **Turing Test**（图灵测试）：是一种编程能力的测试，要求参与者只能通过编写一段程序来回答问题，且不能询问任何其他问题。
- **Project Euler**（项目欧拉）：是一个提供编程题目，要求参与者编写程序来解决这些题目，以验证自己的编程能力。
- **CodeEval**（代码评估）：是一个提供编程题目，要求参与者编写程序来解决这些题目，并提交到网站上，以评估参与者的编程水平。
- **Cognitive Reflection Test**（认知反射测试）：是一个测试，要求参与者在阅读文献或文档的时候，进行反思，询问作者是否清楚表达了自己的观点。
- **OpenAI GPT-3**（开放AI语言模型GPT-3）：是一个强大的自然语言生成模型，可以根据输入给出的句子生成独特的文本。
- **CodeWars**（编程马）：是一个基于网页的编程竞赛平台，提供了各种编程题目供选手来解决。
## 3.3.编程习惯训练法
- **问题反馈法**（Feedback Method）：是一种实验性训练方法，以编写程序的方式帮助学生发现自己的编程习惯偏好。
- **图形编程法**（Graphical Programming Method）：是一种交互式训练方法，利用图形化界面向学生展示代码并让学生直接在图形上编辑。
- **编码模拟法**（Coding Simulation Method）：是一种编码练习的方法，学生需要根据课程给出的样例编写代码，然后用现实生活中的例子来推敲代码的意思。
- **分阶段训练法**（Stage Training Method）：是一种多阶段训练方法，先以小规模的形式训练学生的编程思维，再逐步提升编程能力。
- **遗忘曲线法**（Forgetting Curve Method）：是一种学习方法，即在教授知识的时候，学生可能表面上理解了知识点，但实际上却忘记了。
- **刻板印象法**（Stereotyping Method）：是一种认知测试方法，以学生做出的刻板印象作为判断标准来评价他的编程能力。
## 3.4.编码规范
- **Google Java Style Guide**（谷歌Java编程规范）：是由谷歌推广的一套开源的Java编程风格规范，旨在统一Java社区的编程风格。
- **Microsoft C# Coding Conventions**（微软C#编码规范）：是微软推行的一套开源的C#编程风格规范，旨在统一C#社区的编程风格。
- **Airbnb JavaScript Style Guide**（Airbnb JavaScript编码规范）：是一套JavaScript编码风格规范，旨在保持一致性并且易于使用。
- **GitHub Actions for Python**（GitHub Action Python编程规范）：是一种编程语言规范，旨在提升Python社区的编程风格。
- **Java Code Convention**（Java编程规范）：是一种流行的Java编码规范，适用于各种大小的团队开发。
- **Clean Code: A Handbook of Agile Software Craftsmanship**（整洁代码：一门软件匠艺）：是一本书，由Uncle Bob和Robert Martin所著，介绍如何写出可维护的代码。
- **Effective Engineerings: Methods for Building Quality Technical Systems**（有效工程：构建质量技术系统的方法）：是一本书，由Kent Beck所著，介绍如何构建高质量的技术系统。
## 3.5.抽象思维训练法
- **结构化编程法**（Structured Programming Method）：是一种编程思维训练方法，将算法视作结构化的程序块，通过结构化的方式组合实现复杂功能。
- **面向对象编程法**（Object-Oriented Programming Method）：是一种编程思维训练方法，将程序视作对象的集合，通过类、继承、多态等机制来组织代码。
- **函数式编程法**（Functional Programming Method）：是一种编程思维训练方法，将程序视作数学上的函数，通过纯函数来处理数据。
- **并发编程法**（Concurrent Programming Method）：是一种编程思维训练方法，允许多个任务同时执行，并通过锁机制来避免数据竞争。
## 3.6.数据库设计和查询训练法
- **SQL语句编写训练法**（SQL Statement Writing Trainning Method）：是一种数据库设计训练方法，要求学生能熟练编写符合SQL语法规则的SELECT、INSERT、UPDATE、DELETE语句。
- **数据库索引设计训练法**（Database Index Design Trainning Method）：是一种数据库设计训练方法，要求学生能熟练设计索引，并选择合适的索引列、索引类型、索引顺序等。
- **关系型数据库查询优化训练法**（Relational Database Query Optimization Trainning Method）：是一种数据库查询优化训练方法，要求学生能熟练使用索引和优化器，改善查询性能。
- **NoSQL数据库设计和查询训练法**（NoSQL Database Design and Query Trainning Method）：是一种非关系型数据库设计和查询训练方法，要求学生能熟练使用NoSQL数据库，并制定相应的设计原则。

# 4.1.程序员的大脑是由树突神经网络组成的
## 4.1.1.理解程序员的大脑的构造模式
程序员的大脑是由许多不同类型的神经元组成的，其中包括**树突神经元**（Dendrite）、**轴突神经元**（Axon）和**信号传导元件**（Synapse），它们构成了一个高度连接的网络。当程序员通过键盘、鼠标、触摸屏或其它输入设备输入信息时，大脑会将信息发送给大腿，大腿又将信息传递给手臂，手臂又传递给头部、眼睛等。这些信息通过轴突神经元被传输到大脑的其他区域，如脑干、皮质扫描、运动控制、注意力管理等。
树突神经元是接收输入信息的主要部位，当接收到的输入信息超过一定阈值时，便会兴奋并向轴突神经元传递信号。轴突神经元是一个单向结构，只有一个单独的出口，负责将信号传递到指定位置。信号传递的途径叫做突触（Synapses）。通常，轴突神经元连接在树突神经元上方，当树突神经元兴奋时，就会向其传递信息，信息也会在轴突神经元的回路中流动。当轴突神经元受到刺激时，突触的强度会增大，当突触的强度减少时，信号就会停止流动。
## 4.1.2.使用多重打击训练大脑的协调性
为了使程序员的大脑更容易理解和记住信息，我们可以使用多重打击训练大脑的协调性。多重打击就是指当我们重复地给予不同的信息时，大脑内部的神经元会对不同信息做出反应。例如，如果我们说“请打开电灯”五次，我们的大脑可能会在短时间内对其反应五次。这是因为当我们重复信息时，我们的大脑中的神经元会接收到许多不同的信号。此外，当我们阅读或听到一些新的信息时，大脑的活动可能会分散到不同的区域。这种训练会让大脑中的神经元更加灵活、充满活力。
# 4.2.写出有效的注释
## 4.2.1.为什么写注释很重要
程序代码是有生命周期的，当我们修改代码时，注释往往也是必须的。但是，如果你忽略掉了注释，那么你的代码将无法被别人理解。假设你正在编写一个软件，你必须保证你的代码能够被其他程序员使用。如果没有足够的注释，那些使用你的代码的程序员可能根本不会看懂你的代码。注释应该包括以下三种内容：
1. 对代码的作用和目的的描述
2. 对复杂度和难度的评估
3. 作者的信息、日期和版本号
## 4.2.2.注释风格和格式
每种编程语言都有自己的注释风格。不过，我们一般有两种注释风格：

### // 注释
// 这种注释符号只能用在一行上，只能用于单行注释，不能嵌套。

```java
// This is a single line comment.
System.out.println("Hello World!");
```

### /* */ 注释
/* */ 这种注释符号可以用于多行注释，也可以嵌套。

```python
"""This is a multi-line comment.
It can be used to provide more details about the code or add explanations."""
print("Hello World!") # This is another single line comment.
```

# 4.3.利用搜索引擎优化你的代码
## 4.3.1.搜索引擎是如何识别你的代码
搜索引擎通过检索大量网页内容，从而帮助用户找到想要的信息。搜索引擎的识别算法可以分为三步：
1. 关键字识别：搜索引擎通过分析页面内容和网址，来确定页面的主题、关键词等信息。例如，搜索引擎会对页面的标题、正文、url、标签等内容进行分析，来确定页面的关键词。
2. 页面摘要生成：搜索引擎会将页面中重复的内容、图片、链接等删除，只保留重要的内容和关键字，生成摘要。
3. 权重计算：搜索引擎会对每个网页进行权重计算，来决定该网页在搜索结果中的排序和显示位置。搜索引擎会给相同关键词出现在同一网页上的网页分配相同的权重，从而确保关键词出现在相关的结果中。
## 4.3.2.搜索引擎优化策略
1. 使用标题：搜索引擎通过标题来确定网页的重要程度，好的标题能够吸引用户点击。
2. 使用相似的URL：URL可以唯一标识一个网页，相同的URL会导致搜索引擎认为两个网页是同一篇文档。所以，我们需要使用独特的URL来表示网页。
3. 为网页添加副标题：由于同一个网站的网页数量众多，搜索结果中的网页标题可能会重复，所以，我们需要为网页添加副标题，来提高网页的识别度。
4. 将关键字放置在合适的位置：一般情况下，搜索引擎会将关键字放在网页中央，所以，我们需要将关键字尽可能放置在网页的主题上方。
5. 添加描述信息：除了关键词之外，搜索引擎还会使用页面中描述信息、图片、链接等来对网页进行分类、归档、排序等。所以，我们需要为网页添加描述信息，来辅助搜索引擎对网页进行分类、排序。
