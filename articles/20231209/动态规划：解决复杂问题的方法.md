                 

# 1.背景介绍

动态规划（Dynamic Programming，简称DP）是一种解决复杂问题的方法，它通过将问题分解为一系列相互依赖的子问题，并通过存储子问题的解决方案来避免重复计算，从而提高计算效率。动态规划是一种强大的解决方案，它在许多领域得到了广泛应用，如计算机算法、数学、生物学、物理学等。

动态规划的核心思想是将一个复杂问题分解为一系列相互依赖的子问题，并通过存储子问题的解决方案来避免重复计算。这种方法通常用于解决最优化问题，其目标是找到一个最优解，使得某个目标函数的值达到最大或最小。

# 2.核心概念与联系

动态规划的核心概念包括：状态、子问题、递归与迭代、边界条件和备忘录。

## 2.1 状态

在动态规划中，状态（state）是问题的一个阶段，用于描述问题在某一时刻的状态。状态可以是问题的一些变量的组合，用于描述问题的当前状态。

## 2.2 子问题

子问题（subproblem）是动态规划中的一个关键概念。在动态规划中，我们将问题分解为一系列相互依赖的子问题，每个子问题都有自己的状态。通过解决子问题，我们可以得到问题的最终解。

## 2.3 递归与迭代

动态规划可以通过递归（recursion）和迭代（iteration）两种方法来解决问题。递归是一种基于函数调用的方法，它通过对子问题的递归调用来解决问题。迭代是一种基于循环的方法，它通过对子问题的迭代计算来解决问题。

## 2.4 边界条件

边界条件（base case）是动态规划中的一个重要概念。边界条件用于定义问题的基本情况，即当问题的状态满足某些特定条件时，问题的解决方案可以直接得到。边界条件是动态规划的基础，它们用于初始化问题的解决方案。

## 2.5 备忘录

备忘录（memoization）是动态规划中的一个重要技巧。备忘录用于存储子问题的解决方案，以避免重复计算。通过备忘录，我们可以在解决问题时，如果遇到一个已经解决过的子问题，直接从备忘录中获取其解决方案，而不需要重新计算。这样可以大大提高计算效率。
