# 2.核心概念与联系
GraphQL的核心概念包括类型、查询、变量、片段和扩展。这些概念相互联系，共同构成了GraphQL的数据查询和操作系统。

类型用于描述数据结构，它可以是基本类型（如字符串、数字、布尔值等）或复合类型（如对象、数组、枚举等）。类型系统使得GraphQL能够确定查询的数据结构，从而提高查询的效率和准确性。

查询是GraphQL的核心操作，它用于获取数据。查询语法包括类型、字段、参数、片段和扩展等组成部分。查询语法使得开发者可以轻松地定义查询请求，从而获取所需的数据。

变量用于传递查询请求中的参数。变量可以是基本类型或复合类型，它们可以在查询中使用，以便根据不同的需求获取不同的数据。变量使得GraphQL能够处理动态查询，从而提高查询的灵活性和可重用性。

片段用于组合查询。片段可以将多个查询请求组合在一起，以便更好地组织和重用查询。片段使得开发者可以轻松地构建复杂的查询请求，从而提高查询的可读性和可维护性。

扩展用于增强查询功能。扩展可以用于添加新的查询字段、修改查询字段的类型或修改查询字段的参数等。扩展使得开发者可以根据需要自定义查询功能，从而提高查询的灵活性和可扩展性。
