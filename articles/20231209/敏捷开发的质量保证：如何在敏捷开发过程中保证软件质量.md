### 4.1 代码审查

```python
def add(x, y):
    return x + y

def sub(x, y):
    return x - y
```

在这个代码实例中，我们有一个简单的计算器类，包括两个方法：`add`和`sub`。在代码审查过程中，我们可以发现以下问题：

- 方法`add`和`sub`的参数名称`x`和`y`没有进行清晰的解释，这会导致代码难以理解。
- 方法`add`和`sub`的返回值类型没有进行明确的指定，这会导致代码难以维护。

为了解决这些问题，我们可以对代码进行修改：

```python
def add(x, y):
    """
    将两个数字相加
    """
    return x + y

def sub(x, y):
    """
    将两个数字相减
    """
    return x - y
```

### 4.2 测试驱动开发

在这个代码实例中，我们可以编写以下测试用例：

```python
import unittest

class TestCalculator(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(1, 2), 3)
        self.assertEqual(add(-1, 1), 0)

    def test_sub(self):
        self.assertEqual(sub(1, 2), -1)
        self.assertEqual(sub(-1, 1), -2)

if __name__ == '__main__':
    unittest.main()
```

在这个测试用例中，我们使用了`unittest`模块来编写测试用例。我们编写了两个测试方法：`test_add`和`test_sub`，分别对方法`add`和`sub`进行测试。
