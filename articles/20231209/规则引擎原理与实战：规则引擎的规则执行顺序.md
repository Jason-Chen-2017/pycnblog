# 4.具体代码实例和详细解释说明

在这里，我们将通过一个简单的例子来详细解释规则引擎的工作原理。

例如，我们有一个简单的规则引擎，用于处理订单。订单可以是已付款、已发货、已签收等不同的状态。我们可以定义以下规则：

1. 如果订单状态为已付款，则发送付款通知。
2. 如果订单状态为已发货，则发送发货通知。
3. 如果订单状态为已签收，则发送签收通知。

我们可以使用以下代码来实现这个规则引擎：

```python
class Order:
    def __init__(self, status):
        self.status = status

    def notify(self, message):
        print(message)

def handle_payment(order):
    if order.status == 'paid':
        order.notify('付款通知')

def handle_shipping(order):
    if order.status == 'shipped':
        order.notify('发货通知')

def handle_delivery(order):
    if order.status == 'delivered':
        order.notify('签收通知')

order = Order('paid')
handle_payment(order)
order.status = 'shipped'
handle_shipping(order)
order.status = 'delivered'
handle_delivery(order)
```

在这个例子中，我们定义了一个`Order`类，用于表示订单的状态。我们还定义了三个处理函数：`handle_payment`、`handle_shipping`和`handle_delivery`，用于处理不同的订单状态。

当订单状态发生变化时，我们会调用相应的处理函数来处理订单状态。例如，当订单状态为已付款时，我们会调用`handle_payment`函数来发送付款通知。
