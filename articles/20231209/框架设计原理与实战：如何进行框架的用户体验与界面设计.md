# 2.核心概念与联系

在框架设计中，我们需要了解一些核心概念，包括：

- 模块化：模块化是指将软件系统划分为多个模块，每个模块负责一个特定的功能。模块化可以提高软件的可维护性和可扩展性。
- 抽象：抽象是指将复杂的系统抽象为简单的模型，以便更容易理解和操作。抽象可以帮助我们更好地理解系统的结构和功能。
- 接口：接口是软件系统的一种规范，它定义了系统的外部行为和数据结构。接口可以帮助我们更好地组织代码，提高代码的可读性和可维护性。

这些概念之间有密切的联系。模块化和抽象可以帮助我们更好地组织代码，而接口可以帮助我们更好地定义系统的行为和数据结构。

```python
class Framework:
    def __init__(self):
        self.modules = []

    def add_module(self, module):
        self.modules.append(module)

    def remove_module(self, module):
        self.modules.remove(module)

    def run(self):
        for module in self.modules:
            module.run()

class Module:

        pass

    def run(self):

```
