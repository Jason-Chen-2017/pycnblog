# 2.核心概念与联系
无障碍设计和可访问性是两个相关但不同的概念。无障碍设计是一种设计哲学，它强调设计应该考虑到所有用户的需求，包括残疾人士、老年人士、视力和听力障碍者等。而可访问性则是一种实际的性能指标，用于衡量软件系统是否符合无障碍设计的要求。

无障碍设计的核心思想是“设计即解决问题”。这意味着在设计软件系统时，我们需要考虑到所有用户的需求，并确保软件能够满足这些需求。这包括但不限于：

- 易于使用：软件系统应该具有直观的用户界面，易于理解和操作。
- 易于访问：软件系统应该能够通过各种设备和平台访问，并且不受用户的技术水平和能力限制。
- 易于扩展：软件系统应该具有可扩展性，以便在未来添加新功能和支持新用户群体。

可访问性是衡量无障碍设计实现效果的指标。可访问性可以通过以下方面来衡量：

- 信息可读性：软件系统中的信息应该易于理解和获取。
- 操作可用性：软件系统应该具有直观的操作流程，用户可以轻松地完成所需的任务。
- 技术兼容性：软件系统应该能够兼容各种设备和平台，并且不受用户的技术设备限制。
