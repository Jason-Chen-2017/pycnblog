# 1.背景介绍


## 1.1 数据建模概述
在大数据时代，数据的数量、种类、结构和价值都远超传统行业的数据量。因此，如何有效地对数据进行建模、存储、管理、分析、应用以及优化至关重要。数据的建模可以帮助企业快速理解业务数据、发现数据中的模式、规律、关联关系、质量分布等，从而提升数据价值的发现和应用效率；其次，数据的建模还可以作为数据治理的基础，提升数据可靠性、整合性、可用性和易用性，并减少或消除数据孤岛带来的隐患。
## 1.2 数据建模概念及联系
### 1.2.1 数据模型分类
数据模型是指描述数据如何存储、组织、处理、分析和报告的一套概念或方法。数据模型分为实体-关系数据模型、对象-链接数据模型、层次数据模型、面向主题的数据模型、空间-时间数据模型、多维数据模型、星型数据模型、因子图数据模型等几种类型。每一种数据模型都由一组抽象数据结构及定义数据结构之间的关系的方法所驱动。数据模型是构成大数据系统的基础，是支撑大数据各种功能模块的关键。
### 1.2.2 数据模型和数据库设计的区别
虽然数据模型和数据库设计都是用来解决存储、组织、查询等问题，但是两者还是存在一些区别：

1. **建模方式**：数据库设计是将现实世界的实体和关系转化成计算机程序中的表格形式，是面向数据的过程，数据库设计人员通过识别出数据中存在的实体及其间的关系，确定数据库的表结构及字段，然后根据业务规则生成相应的索引，形成完整的数据库设计文档。数据建模则不同，它不是直接在计算机上画表格，而是通过人工的逻辑、知识、经验、直觉来分析现实世界，识别和抽象出数据实体、属性、关联和约束等抽象概念和规则，构建一个符合要求的数学模型，即数据模型。
2. **建模范围**：数据库设计主要是针对整个数据库的设计，而数据建模则侧重于单个数据对象的建模，例如某个医院的病人信息表、某个新闻网站的用户行为日志等。数据库设计需要考虑到硬件环境和性能瓶颈，数据建模则需要更加关注数据的体积、可用性、完整性、一致性等。
3. **建模目的**：数据库设计通常是在需求阶段就开始制定，目标就是建立一个功能完备的数据库，为后续的开发、测试、部署、运营等提供技术保障；数据建模则是为了更好地理解业务、提升数据价值、识别潜在问题、改进数据质量，目的是促使数据架构师和数据工程师一起合作共同创造有益的产品和服务。
4. **关注点不同**：数据库设计着眼于高层次的设计目标，侧重于数据库的总体架构和数据的逻辑和物理组织；数据建模着眼于低层次的设计目标，主要涉及数据模型、数据字典、元数据、关联规则、主题域等方面。
5. **工具不同**：数据库设计通常采用标准化的工具如ERWIN，表结构设计则依赖于UML或ERD等工具。数据建模则需要掌握专业的建模工具、工具箱等，有助于快速实现模型的验证、交流和迭代。
