# 1.背景介绍


关系型数据库（Relational Database）是指用二维表结构存储、组织、管理数据的一类计算机软件。关系型数据库与其他类型的数据库相比，它的特点是简单、灵活、稳定，具备较高的数据一致性，并可通过SQL语言访问。
在现代社会，企业应用中常用的数据库包括Oracle、DB2、MySQL等。不同的数据库之间也存在一些差异，例如功能、性能方面的区别，因此，本文主要讨论的是关系型数据库，也就是常说的RDBMS(Relational Database Management System)中的一种。

# 2.核心概念与联系
## 2.1 RDBMS概述及特性
关系型数据库管理系统（RDBMS，Relational Database Management System）是一个将复杂的数据结构存储于数据库中的系统。它具有高度的组织化和标准化特征，并且支持多种类型的数据结构，如表、记录、字段等。RDBMS的特征如下：

1. 数据结构：RDBMS所管理的数据是以表格形式存储的。表由行和列组成，每一个表都有一个唯一的名字标识，其中包含若干字段（Field）。字段可以存放不同的数据类型，如数字、字符、日期等。

2. 完整性约束：RDBMS对数据的完整性（Integrity）非常重视。它提供了各种完整性约束机制，如主键约束、外键约束、唯一约束等，用于保证数据的正确性和一致性。

3. 可扩展性：RDBMS可以通过添加新的磁盘空间或服务器来扩展其容量。通过分片（Sharding）的方法还可以实现更大的数据库容量。

4. SQL查询语言：RDBMS支持 Structured Query Language (SQL)，用于执行各种数据库操作，如插入、删除、更新、查询等。

5. 事务处理：RDBMS支持事务处理，确保数据一致性。事务是一个不可分割的工作单位，它要么全部完成，要么完全不起作用。

## 2.2 概念解析
### 实体（Entity）
实体是一个对象，它是某个事物的不可分离的属性集合。例如，一条“订单”实体可能包含“顾客姓名”、“产品名称”、“数量”、“金额”、“下单时间”等属性。在关系数据库中，实体通常用“表”表示。每个表都有一个唯一的名字，它包含了实体的所有属性。

### 属性（Attribute）
属性是实体的一个具体方面，它描述了一个事物的某些方面，是可变化的、区分不同事物的特征。例如，一个“顾客”实体的“姓名”属性就代表着这个顾客的真实姓名。在关系数据库中，属性就是表的字段。

### 键（Key）
键是实体的一个属性组合，它唯一地标识一个实体，同时也是关系数据库中最重要的概念之一。例如，“顾客”实体的主键（Primary Key）就是“身份证号”。键有两种类型：主键（Primary Key）和外键（Foreign Key）。主键唯一地标识了一个实体，外键则指向另一个表的主键值。

### 关联（Relationship）
关联是两个实体之间的一种联系，它代表两个实体之间的一种依赖关系。例如，一张“订单”表与一个“顾客”表相关联，表明这两张表互相关联。在关系数据库中，关联由两个表之间的外键决定。

### 函数依赖（Functional Dependency）
函数依赖是一个二元组，它指定了一个属性集合X中的元素A与属性集合Y中的元素B之间的依赖关系。如果对于任意两个元素x和y，属性X中的元素a映射到Y中的元素b，那么就可以说X依赖于Y。例如，{客户名，身份证号} -> {地址}就是一个典型的函数依赖。

### 码（Candidate Key）
码是一个属性或属性集，它唯一地标识了一个候选码。候选码是至少包含两个属性的键，但可能包含更多的属性。候选码是通过消除属性间的任何函数依赖而得出的。例如，“订单”实体的候选码有可能是“订单号”，因为“顾客”实体的属性与“订单”实体的属性无关。

### 范式（Normalization）
范式是关系模型理论中著名的术语。它是一种基于函数依赖规范化的数据库设计方法。范式是为了解决数据冗余的问题，它是关系数据库设计的三大范式之一。
第一范式（1NF）：要求所有的字段都要单值，即非复合数据类型。
第二范式（2NF）：要求一个表不能有多个候选键。
第三范式（3NF）：要求一个关系中的数据要么依赖主键，要么依赖码。
