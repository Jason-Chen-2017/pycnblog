# 2.核心概念与联系
## 2.1 什么是React？
React是一个用于构建用户界面的JavaScript库，它是Facebook于2013年推出的开源项目。它的主要特性包括：
- 使用声明式语法：React采用声明式编程方式，简洁的语法使得代码更易理解，也减少了对数据的处理逻辑的影响。
- 组件化开发：React通过将复杂的页面分割成多个小组件，更好的实现代码重用，降低了代码复杂度。
- Virtual DOM：React使用Virtual DOM（虚拟DOM）进行视图的更新，避免了直接操作DOM导致的性能问题。
- 数据驱动：React通过 JSX、PropTypes等特性，能够轻松地管理状态变化并相应更新视图。
因此，React可以帮助我们构建出高性能、可维护、可复用的前端界面。
## 2.2 为什么要用React？
首先，React的确解决了传统HTML页面的不足之处，如过多的DOM操作、复杂的事件绑定及数据模型的同步，使得前端代码变得臃肿难维护、耗时长，并且性能优化困难。因此，如果你的网站、APP需要承受较高的访问量和数据刷新频率，React可以非常有效地提升用户体验。其次，React还能让我们的代码更加可复用，并且可以有效地解决组件间通信和状态共享的问题，简化了前端的开发过程。最后，React拥有丰富的生态系统，包括各种UI组件、路由、数据流等。
## 2.3 React与其他框架的区别
React除了上面提到的优点外，还有一些值得注意的地方。比如，与Angular或者Vue相比，React的一个显著特点就是“单向数据流”。这是因为React是一个严格的单向数据流框架，数据只允许单向流动——从父组件传递到子组件，而不能反向流动——子组件只能修改父组件的数据。这个数据流规则有利于代码的编写和维护，也避免了一些常见的陷阱。除此之外，React还有一个独特的 JSX 模板语言，它是一种声明式的、基于 JavaScript 的模板语言，用来定义 UI 组件的结构和行为。
## 2.4 React组件之间如何通信？
在React中，props 和 state 是父组件和子组件通信的两种主要方式。其中 props 是父组件向子组件传递参数的方式，state 可以被看做是可变的 props，用于存储组件的内部状态。
父组件可以通过 props 将数据传递给子组件，子组件也可以通过回调函数将结果返回给父组件。但是，这种方式有很大的局限性，因为它依赖于父组件控制数据的变化。
React 官方推荐的方法是使用 Redux 或 MobX 来统一管理应用中的数据流。这样的话，任何组件都可以读取全局的数据源，并且触发更新通知，使得所有组件保持同步。
## 2.5 State 与 Props 有何不同？
State 和 Props 是 React 中两个最常用的属性。它们之间最大的区别是：State 是组件自身的属性，可以根据组件内发生的变化自动更新，Props 是外部传入的属性，是不可以改变的。简单来说，就是 State 更适合用于组件内的本地状态，Props 更适合用于从父组件接收的数据。当然，在某些情况下，它们也会混用，比如组件 A 通过 Props 接收组件 B 中的数据，但组件 B 本身又有自己的 State 属性。
