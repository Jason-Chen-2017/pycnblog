# 1.背景介绍


计算机的诞生离不开信息处理领域的许多发明、发现、创新。计算机的发展历史可以分成两部分：数字计算机时代（第1世纪至第30年代）和符号计算机时代（第31-50年代）。在数字计算机时代，计算机采用了模拟计算方式，计算结果只能用二进制（0或1）表示；而符号计算则逐渐成为主流，它引入了命题逻辑、集合论等概念，运算对象都是命题变量。
虽然符号计算在计算能力上更强一些，但是由于处理能力限制，符号计算并不能做到对计算过程进行全面细致地控制，因此也不能完全解决实际问题。另外，符号计算通常需要较大的存储空间和时间开销，往往导致其被应用于只能使用少量内存和计算能力的嵌入式系统中。同时，符号计算和数字计算之间的界限在逐渐模糊化，越来越多的数学和工程问题都可以使用符号计算求解。因此，符号计算的发展历史也是极其重要的。
# 2.核心概念与联系
计算的基础是二进制的加减乘除，也就是二进制算术运算。但是由于存在很多非数值（非零、非负）的情况，例如“真”、“假”，或者“奇”、“偶”。为了能够处理这种情况，数学家提出了整数集、实数集、复数集等概念。从某种意义上说，所有的集合都是人类思维的抽象工具，它们本身没有内涵和外延，是无限扩张的。
在古代，计算能力只有很有限的几个功能，例如：求平方根、阶乘、斐波那契数列。而到了近代，符号计算等符号主义方法的出现使得人们认识到，真正的计算是在运算过程中通过给予输入、输出及计算规则来实现的。我们所说的算法就是完成这样一系列运算的规则。
符号计算与数字计算之间还有一种天然的联系，即它们之间的转换关系。数字计算就是利用机器硬件的加工能力将数字信息转换成电信号再传送到运算器。而符号计算则是利用符号语言进行抽象思维，把抽象事物的属性和结构映射成形式语言中的符号。这一过程称之为编码，而解码过程则是符号计算的一项重要特征。符号计算的理念就是把复杂的问题表述成简单易懂的符号表达式，然后再通过自动化的方法求解这些表达式。

## 3.1 密码学的历史简介
由于密码学与电脑有密切的关系，所以大家了解一下其历史就好。密码学（cryptography）研究的是如何在不安全信道中发送数据，以及如何在接收端对数据进行正确性验证的技术。在世界范围内，密码学主要有两个支柱，即一是数学方法，包括代数学、统计学和概率论，二是计算机科学，包括通信加密算法、身份鉴别认证机制、数字签名和可靠传输协议等。
### 3.1.1 早期的密码学技术
密码学最早起源于罗马共和国，当时罗马共和国的军队无法保护敏感信息，于是便考虑建立一种加密系统来对敏感信息进行加密，将敏感信息经过加密之后转移到更安全的位置。英国人纳赛尔德·卡尔森于19世纪末发明了凯撒密码，通过替换每一个字母后面的字母，创造出一个变换矩阵，使得加密和解密非常容易。他还发明了滑动块密码，即将信息分割成固定大小的块，对每个块进行一次异或运算，得到整个消息的密文。
### 3.1.2 近代密码学技术的演进
19世纪末到20世纪初，由于信息技术的飞速发展，美国的政府越来越重视信息安全，美国政府开始向其他国家提供了各种安全服务。但由于对保密数据的需求不同，各国政府自行开发相应的保密算法和标准。随着互联网的普及，各国政府又发展出多种类型的密码算法和标准，如AES、RSA等，均属于公钥加密算法（public key encryption algorithm）。
20世纪90年代，中国政府颁布了《中国国家密码管理条例》（National Security Administration of China，NSA），为了应对各种信息泄露事件，该法律要求任何组织和个人使用公钥加密技术来保障敏感数据和信息安全。

2007年的“银河”事件引发了对公钥加密技术安全性的担忧，当时由于中国政府没有充分认识到公钥加密的潜在危险性，NSA便在网上发布了一份《网络安全监督管理规定》（National Information Security Audit Office of China，NISPO），详细规定公钥加密算法应当经过严格评估，保证其安全性。

2011年，美国国家标准与技术研究院（NIST）颁布了FIPS 140，即美国联邦信息处理标准，其中规范了RSA加密算法。FIPS 140将RSA用于加密的过程称之为公钥加密，将密文解密的过程称之为私钥解密。随着对公钥加密算法的日益关注，目前的公钥加密算法已经发展到了一种新的阶段，即符合FIPS 140标准的加密算法。

## 3.2 RSA加密算法
RSA（Rivest–Shamir–Adleman）加密算法，是一种基于对数的公钥加密算法，可以抵御到目前为止已知的各种攻击，而且还是公认的最优秀的公钥加密算法之一。它的基本原理是基于大质数的因数分解难题，这个问题是困扰数学家们几百年的难题。RSA算法是公钥密码体系中第一个能同时用于密钥生成和加密的算法，具有良好的性能、效率和安全性，目前被广泛使用。
RSA加密算法的特点是私钥只能由拥有者知道，并且整个过程需要两把密钥，一把用来加密，另一把用来解密。加密时，发送方用自己的公钥将明文加密后发给接收方；接收方用自己的私钥将加密后的密文解密。加密和解密的过程只能由双方共享同一个密钥。
### 3.2.1 RSA算法的基本原理
RSA算法的基本原理是利用数论的一些基本原理，即欧几里得算法和费马小定理。对于一对大素数p和q，计算出的三个质数n=pq，φ(n)= (p-1)(q-1)，p、q互质，并且p、q之间满足互质关系，于是可以得到：
n = pq; p ≤ q ≤ n/φ(n) = sqrt(n)
根据欧几里得算法，可以计算出关于pq的所有整数d的整数根d，令s=d mod φ(n), t=1 mod φ(n)。
首先，对消息m进行指数运算E(m)：c = m^e % n
然后，对密文c进行指数运算D(c): m' = c^d % n
假设e是合适的值，那么可以推导出d和n：
ed≡1 mod φ(n); gcd(e,φ(n))=1
将d=1 mod s，得到：
1 ≡ ed - 1 mod s * (t*s + 1)
因为φ(n)和t互质，所以t=1 mod φ(n)。所以：
1 ≡ e(d-1) / (1-t) * s ≡ ks + 1 ≡ k * s + (-1)^k * t
式中，k取值为整数，-1取值也为整数，所以可以将式子左右两边同时取模φ(n)：
1 ≡ e(d-1) / (1-t) mod φ(n) * s ≡ ks + 1 ≡ k * s + (-1)^k * t
由于t=1 mod φ(n)，t也可以由式子左半部推导出来：
t ≡ d^(φ(n)-1)/(-1)mod φ(n)
再根据费马小定理：
φ(n) = prod{p<q}{[(p-1)*(q-1)]}
而：
prod{p<q}{[(p-1)*(q-1)]} ≡ ((p-1)(q-1))(q-p+1)
式中，第一项是平方级数的前n项和，第二项是组合数学的恰当公式。因此，将φ(n)写成这样形式：
φ(n) = (p-1)(q-1)(q-p+1) = lcm((p-1),(q-1))*(q-p+1)
而lcm()函数可以计算最小公倍数。
将原来的：
ed ≡ 1 mod φ(n); gcd(e,φ(n)) = 1
修改成：
gcd(e,φ(n)) = 1; ed = de ≡ 1 (mod n)
则：
de ≡ e inv(phi(n)) (mod n)
其中，inv()函数返回x的乘法逆元。
于是，公钥(n,e)=(p*q,e)；私钥(n,d)=(p*q,d)。
### 3.2.2 RSA算法的应用场景
现代互联网的绝大部分通信是通过HTTP协议进行加密的，其中RSA算法是应用最广泛的一种公钥加密算法。例如，用户注册网站时，需提供邮箱验证码，邮箱服务器要先对收到的邮件地址和验证码进行加密，才能确定验证码的合法性。HTTPS（Hypertext Transfer Protocol Secure）加密通信协议，是一种基于SSL/TLS协议的HTTP协议的升级版本，采用RSA加密算法实现对传输的数据进行加密。在支付宝、微信支付等移动支付系统中，也是采用了RSA加密算法，作为通信加密的基础。当然，RSA加密算法也不是完美的加密算法，它还存在着各种安全漏洞和侧信道攻击。但是，无论在哪个领域，都不可避免地要面临着公钥加密算法的问题。
