# 1.背景介绍


自动化运维（Automation）是指通过计算机实现对资源和服务的管理自动化，从而提升IT组织效率、降低成本、优化资源利用率和可靠性。DevOps（Development and Operations）是在IT行业中崛起的一个术语，它是一种文化、运动或者是流程，旨在促进开发者、QA工程师、系统管理员等角色之间的沟通协作，更好的完成需求交付。在整个软件生命周期内，自动化运维和DevOps将在质量保证、持续集成、自动部署、动态扩展、弹性伸缩、容量规划、负载均衡等方面提供重要的支撑作用。因此，作为一个后端架构师，不仅需要掌握基础的业务逻辑编程技能，还要充分理解自动化运维和DevOps相关理论知识和实践方法。

2019年是自动化运维、DevOps大爆发的年代，自动化运维工具、平台和服务层出不穷。而如今最火热的Kubernetes、容器技术、云计算、微服务架构、Serverless架构……这些概念背后的理念、工具和实践方法也逐渐成为架构师必须关注的方向。
