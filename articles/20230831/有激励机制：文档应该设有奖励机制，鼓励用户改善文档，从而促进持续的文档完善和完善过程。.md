# 1.简介
  

文档对于参与者来说，无疑是最重要也是最难以衡量的一种信息载体。这不仅仅是对使用者来说，对于开发者来说更是如此。良好的文档能够帮助参与者快速理解、掌握某个功能或模块的用法，提升效率和质量，降低沟通成本，缩短开发周期。而对于某些开源项目的维护者来说，除了写好文档之外，还应当给予其关注度和奖赏机制，鼓励用户报告文档中存在的问题、错误或缺陷并修复，从而让社区的力量得到充分体现。

但是，许多开源项目没有给文档设置奖励机制，这就导致了一些参与者只能依赖于自己能够阅读文档中的内容才能了解项目的工作流程，而不能像使用商业软件一样通过付费获得支持和服务。由此带来的后果就是更多的人选择放弃项目，甚至放弃开源。

因此，为了促进文档完善的进程，提高参与者的参与积极性，鼓励他们改进文档，而不是放弃参与者，可以采取如下几种策略：

1. 设置文档评级机制
对开源项目的文档进行评级是很有效的方法。评级的标准和评级方式都可以在社区共同商议制定。可以参考GitHub等代码托管平台的README文件里面的“Stars”、“Watchers”、“Forks”以及“Issues”等标签。同时，也可以制订自己的评级规则，比如热点词、质量、时效、深度等指标，用于给各个版本的文档打分。这样做既能够鼓励参与者在评级后进行贡献，也为读者提供一个客观的反馈渠道。

2. 设置文档改进期刊
开发者提交的文档一般都会经过编辑、校对和审阅，但很少有人会对文档的改进提出建议。所以，可以通过举办文档改进期刊的方式，吸引广大的用户和开发者参与到文档的完善过程中。期刊的内容可以包括主题演讲、专题讨论、文章翻译、资源分享等形式。这样，就可形成一个比较活跃的文档交流平台，引起开发者的关注和参与度。

3. 提供各种学习途径
除了文档评级及文档改进期刊之外，还有很多其他方式可以使得参与者有机会学习到更多关于项目的信息。比如，可以举办系列培训课程，向新手介绍项目的构架和功能；可以邀请业内专家分享他们的经验，从而进一步丰富文档；可以开设问答栏目，回答一些用户可能遇到的问题；还可以建立知识库和论坛，把大家的心得、知识和见解分享出来。通过这些方式，可以增强参与者的参与意愿、参与感和参与能力，也可提升项目的知名度和影响力。

4. 为文档编写制定目标
除了上面提到的各种方式，还可以制定文档编写的目标。例如，可以设置日历或者周报，规定每个月或每周完成的文档任务。这可以确保文档的更新步调一致、任务分明，提高文档的质量、品牌化和推广度。

总之，为了鼓励和促进文档完善的进程，开源项目可以考虑设置文档奖励机制、文档改进期刊、提供学习途径、为文档编写制定目标等机制，帮助参与者在获得足够的激励后，积极投入到文档的完善中来，真正实现“知识共享”。
