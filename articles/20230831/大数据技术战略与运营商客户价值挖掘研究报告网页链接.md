# 1.简介
  

## 1.1 大数据定义
大数据（big data）作为一种新兴的信息技术形态，其广泛应用于各行各业、各领域包括但不限于金融、生物医疗、互联网、制造业等多个领域，得到越来越多的关注。
## 1.2 数据采集与存储方式
数据采集一般采用两种方式：
1. 拉取数据：即定时定期地将数据从数据源（如数据库或API接口）上拉取下来。优点是实时性高、全面性强；缺点是受网络波动、数据的完整性影响可能会丢失一些数据、速度比较慢、资源消耗比较大。
2. 流式数据采集：利用实时数据流的方式，即通过网络直接获取数据，数据量较大，但实时性差。优点是速度快、成本低，适合处理流式数据。缺点是无法保证数据的完整性、数据更新机制不够智能、数据结构复杂。
## 1.3 数据类型
数据可以分为以下几类：
1. 结构化数据：包括关系型数据模型和非关系型数据模型，例如MySQL、MongoDB等。结构化数据具有严格的数据结构、固定的数据类型，使得分析、查询变得更加简单、效率更高。然而结构化数据的缺点也是很明显的，在数据量、处理难度都有限的情况下，仍然无法满足海量数据进行分析的需求。
2. 半结构化数据：指的是非结构化、无固定格式的数据，例如日志文件、文本文档等。这种数据可以按照一定的规则进行解析、提取，从中获得有意义的洞察力。但是半结构化数据的缺点也是很明显的，很多时候只能通过一定规则或者大数据分析工具才能从这些数据中发现规律和模式。另外半结构化数据往往会有较大的噪音，需要清洗、归一化处理才能得到可用的分析结果。
3. 不完全结构化数据：也称非标准数据，是指存在字段值的缺失、顺序变化、冗余重复等各种不规范现象的数据。它没有固定的数据结构、无法预先定义数据类型。这种数据往往难以处理和分析，因此在分析阶段需要借助机器学习、深度学习等算法，对数据进行分析挖掘。不完全结构化数据也可以用于数据挖掘，对其中的信息进行抽取、关联、聚类等处理，进一步挖掘其价值。
4. 时间序列数据：是指随着时间的推移，某些特征在不同维度上的变化情况。它可以帮助理解和预测经济、政治、金融、社会、生产、消费行为等各个领域的发展趋势。但是由于数据量大、变化频繁、数据周期长，时间序列数据往往需要特定算法才能处理、分析。
5. 图像、视频、文本等多媒体数据：它们往往呈现出复杂的结构、多样的表达形式。对它们的分析、挖掘将需要比较新的方法论和工具。
## 1.4 数据分析任务
数据分析的任务主要包括以下几个方面：

1. 数据清洗：数据清洗就是将原始数据中可能存在的问题修复并转换成可以被分析的形式。清洗过程中要考虑到数据质量、一致性、完整性等因素。

2. 数据整合：数据整合是指将不同数据源的数据整合到一个数据仓库、数据集市或数据湖中，然后再根据业务需求进行数据集成及建模。在数据整合的过程中，还需考虑到数据价值、一致性、可用性、时效性等问题。

3. 数据挖掘：数据挖掘的目标是基于所收集的海量数据集合提炼有价值的信息。在这个过程中，需要对数据进行探索、分析、挖掘，并找到最具价值的模式。数据挖掘的方法有基于规则的、基于统计学习的、基于图论的等。

4. 数据可视化：数据可视化是指将分析得到的结果通过图表、报表等形式呈现出来，方便分析人员快速了解数据之间的联系。可视化结果应具有直观、易懂、交互性强等特点，同时避免过于复杂、混乱，且结果具有普适性。

5. 模型训练及评估：模型训练是指使用大量数据训练计算机模型，以便识别、预测、分类等其他数据处理任务。在模型训练过程中，还需考虑模型选择、参数优化、结果评估、业务运用等环节。

6. 系统部署：最后，数据分析系统应作为整体运营的一部分，可以对外提供服务。部署过程应考虑安全性、性能、可扩展性等问题。
