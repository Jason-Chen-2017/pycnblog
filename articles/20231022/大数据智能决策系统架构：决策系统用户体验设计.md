# 1.背景介绍


大数据领域涌现了很多关于如何构建智能决策系统的研究。比如：信息检索、知识图谱、机器学习、深度学习等方法在不断的创新中得到广泛应用，并取得了很好的效果。然而，如何利用这些方法进行业务决策的用户体验则是一个复杂的难题。

本文旨在通过分析大数据智能决策系统架构设计过程中的关键问题——用户体验设计（User Experience Design，简称UXD），从用户视角出发，阐述大数据智能决策系统设计中需要考虑的方面及其具体解决方案。

UXD主要包括以下六个方面：

1. 认知能力：用户是否能够快速准确地理解决策系统给出的建议？
2. 操作性：用户是否可以轻松地掌握和操作决策系统提供的建议？
3. 直观性：用户是否可以快速理解各种信息及结果？
4. 舒适性：用户是否可以轻松完成决策任务？
5. 即时反馈：用户是否可以在决策过程中获得即时反馈？
6. 可用性：用户是否可以通过不同设备、网络环境访问决策系统？

本文将从以下两个角度对大数据智能决策系统用户体验设计进行剖析：

第一，从技术层面上深入探讨大数据智能决策系统架构的设计与实现过程中存在的各类技术问题与用户体验优化要素，如界面设计、交互设计、功能模块设计、安全设计等，以及用户满意度调查研究等。

第二，从用户视角出发，客观地呈现大数据智能决策系统的用户体验特点及优化方向，指导企业制定用户体验策略和目标，提升用户体验质量。

# 2.核心概念与联系
## 用户
用户是大数据智能决策系统的最终消费者。用户分为终端用户（例如手机APP）、网页用户、后台管理人员等。

用户习惯于快速浏览、使用、分享，并且喜欢简单、易懂的产品或服务。所以，大数据智能决策系统的用户体验应做到与用户的生活习惯相符，具有尽可能低的学习成本。

## 清晰易读
清晰易读是指用户容易理解决策系统输出的信息及建议。因此，信息结构应该便于理解，帮助用户快速获取所需的信息。同时，系统设计要保持简单、清晰，并突出重点。

## 操作简洁
操作简洁是指用户可以在短时间内轻松完成决策任务。因此，决策系统提供的功能模块设计应减少选项过多、操作流程繁琐等弊端，并与用户的日常工作习惯和生活方式相结合。

## 直观感受
直观感受是指用户在决策过程中可以快速感知到系统处理过程的进展情况。因此，系统设计要与用户的直觉相匹配，让用户能够全面掌握系统的功能和结果，并提供较快、更准确的反馈。

## 激活能力
激活能力是指用户可以通过系统提示、反馈等方式快速接受、获取并执行建议。因此，系统设计应尤其注意优化反馈环节，帮助用户快速获取反馈，并激活自身的决策行为。

## 实时反馈
实时反馈是指用户可以随时掌握决策系统的最新结果和更新情况。因此，系统设计应高度集成、可持续的反馈机制，并根据用户实际需求提供动态且精准的服务。

## 兼容性
兼容性是指用户可以使用不同的设备、网络环境访问系统。因此，系统设计应充分考虑各种设备、网络环境下的用户体验，同时避免因平台差异带来的差异化用户体验影响。


## 数字化转型阶段，对于智能决策系统设计者而言，除了传统的系统架构设计、数据库设计之外，还需要着力于满足新兴的“人机协同”需求。这要求决策系统的用户界面、交互方式、界面响应速度、数据展示形式、查询效率、模型训练精度、推理时间复杂度等方面都得不到有效关注。

人机协同可以归纳为四种特性：

1. 数据收集及整理能力：人工智能算法赋能机器获取大量数据的能力正在成为实际需求；
2. 知识表达能力：信息技术的飞速发展让决策支持得以覆盖范围越来越广，需要专业人士精心编写算法和规则的能力才能有效参与到决策中；
3. 决策表达能力：有些决策任务需要与人交流才能产生共鸣，需要具备话语技巧、表情识别、自我表达等能力；
4. 执行能力：人机协同能够让决策系统的执行速度加快，降低了人工耗时，提高了决策效率。

因此，针对人机协同特性的用户体验设计，大数据智能决策系统需要进行如下方面的改善：

1. 引导用户获取更多信息：作为一个独立的决策系统，决策者需要主动询问和获取更多信息才有可能产生最佳决策；
2. 提供决策建议的可视化表示：采用可视化方式呈现决策建议，能够直观地展示给予决策者的建议及相关原因；
3. 人机交互界面优化：当用户与系统进行交互时，需要提供便捷、直观、易用的操作手段；
4. 集成用户反馈机制：集成用户反馈机制，能够即时反馈决策进程，提升决策的透明度和鲜明性；
5. 充分利用人工智能技术：在决策系统中，基于人工智能算法的强大计算能力能够支撑大规模数据快速采集、处理、分析，降低决策延迟；

总之，数字化转型阶段的大数据智能决策系统架构设计者需要以用户为中心，从用户需求出发，优化决策系统的用户体验。
