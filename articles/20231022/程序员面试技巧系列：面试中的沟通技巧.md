# 1.背景介绍


程序员的面试之路充满了坎坷与磨砺。作为一名技术人员，面试一定不是一件轻松的事情，但是只要努力就能克服这些困难，学会自我introduction、面试时注意细节、提高表达能力、充分准备……通过一步步地练习，掌握面试的各种技巧，才能在职场中脱颖而出！
面试是一个很重要的过程，因为只有通过面试，才有机会接触到这些优秀的人才；同时，面试也是为了让被面试者知道自己擅长什么，提升自己的能力；还有，面试是筛选简历、了解公司业务和现状的有效手段。

但是，有些人总喜欢玩弄面试技巧，使得自己和面试官无法真正的交流和互动，甚至出现一些负面的评价。因此，不妨从以下几个方面来进行学习：

①.针对性的学习：首先，你需要针对你的实际工作情况，选择适合的面试技巧。例如，如果你要参加的是移动开发的面试，那么你可能需要熟悉下Git、Gradle等相关知识，并对HTTP协议有一定的理解；如果你参加的是计算机网络或者数据库的面试，那你可能需要掌握TCP/IP协议的基本原理和应用。所以，不要盲目套用别人的面试技巧，一定要根据自己的实际情况来学习；

②.方法论的思维：学会借鉴他人的方法论，以方法论的思维来看待面试。每一个面试都不一样，但如果有一个指导原则的话，那就是：面试官给你一张纸，上面写着“面试官会问的问题”，然后开始你的表演。如果你听不懂题目，那就重新再听，直到你能够完整地回答出来为止。其实，无论面试官问什么题目，你都可以用自己的话概括一下它的意思，然后回答给面试官。这样做，不仅能够使你的表演更好听，而且可以更有效地保障你的答题效果；

③.坦诚的沟通：说到沟通技巧，就不得不提及坦诚的沟通。无论面试官问你什么问题，都应该尽量用最简洁、直接的方式回答，并且主动聆听面试官的反馈信息，快速调整自己的回复策略；另外，面试中不要出现负面的评论，保持尊重心态，面试结束后也要感谢对面试官的肯定；

④.耐心的等待：在面试中，一定要耐心的等待，不要着急，慢慢的和面试官交谈，慢慢的发现自己的问题，努力解决它，这样才能保证自己的提问技巧和时间管理水平；

最后，祝大家都能在职场上顺利、成功！
# 2.核心概念与联系
## 概念
### 备胎
在美国，一般有两种备胎：技术备胎和非技术备胎。

技术备胎主要包括Java、Python、JavaScript、HTML、CSS、Swift等编程语言方面的问题，相当于暗含了一定的编程基础知识，但一般要求面试者有比较扎实的编程能力。

非技术备胎主要是面试者的个人经验、生活阅历、兴趣爱好等方面，不会涉及技术知识，具有很强的启发性和广度。

### 面试官
面试官分为两类，一类是职业的面试官，他们由专业的招聘人员介绍相关职位，帮助求职者预测能力范围和职业发展方向；另一类是非职业的面试官，他们有多种类型，如技术经理、HR、导游、记者等，帮助求职者确认其真实身份、搭建良好的人际关系、谋取最大收益。

面试官除了提出的问题外，还会根据候选人的履历、个人特点、个人观点等，判断其综合素质和工作能力，最终给出建议。

### 沟通技巧
沟通技巧可分为八个层次，分别是信息层次、语言层次、观点层次、表达层次、专业层次、激励层次、责任层次、策略层次。

- 信息层次：信息传递的艺术，包括陈述信息、收集信息、分析信息、归纳信息等。通过有条理的信息传递可以让面试官快速理解、吸收、整合相关信息，从而得到更准确的答案。
- 语言层次：如何使用口头、书面语、电视讲话等语言，这个技能在面试中非常重要，因为很多面试官都是外国人，学习外文、英文、日文、韩文等语言可以增进沟通能力。
- 观点层次：面试中，通常会出现众多不同的观点，不同观点会影响候选人的判断，并影响其决策。通过分析不同观点、不同立场的意义、联系和区别，可以帮助候选人获得更多信息，做出更明智的决定。
- 表达层次：表达技巧就是指用词清晰、符合逻辑、保持情感流畅、控制感情变化等，通过正确的表达，可以有效传达信息。
- 专业层次：面试过程中，面试官往往需要由多个专业领域的知识、技能、经验结合进行判断，而掌握相应专业知识和技能则显得尤为重要。
- 激励层次：面试时，面试官往往会设定目标和任务，比如让候选人做某项测试或向某些人询问情况。在目标达成的过程中，候选人的职业道德也将得到锻炼。
- 责任层次：面试时，应当与面试者建立信任关系，面试官需要为候选人提供足够的帮助和支持，而不会期望候选人一上来就把问题说死，否则容易陷入自我催眠。
- 策略层次：在面试中，面试官可能会采用不同的方式，采取不同的策略，从而取得更好的结果。面试过程中应当保持灵活性和想象力，遇到新情况应立即转变策略。

沟通技巧的关键在于沟通的目的、方式以及结果。

## 联系
### 1.技巧总结

- “聆听”：沟通开始时，应当先听取面试官的介绍和考察，了解面试对象的工作情况、兴趣爱好等。聆听有助于了解面试对象擅长什么、擅长什么领域，判断其是否具备合适的技能。
- “分析”：接着，应当分析面试官提出的问题，分析面试官所期望的候选人的知识、技能、经历和能力。分析有助于候选人了解自己擅长什么、需要什么样的培训、训练等，以便更全面地展示自己的能力。
- “讨论”：接着，应该邀请面试官和候选人进行充分的讨论，让双方都有机会阐述自己的观点、意见、建议。讨论可以为对方提供宝贵的反馈，帮助双方更好地理解彼此的需求和理念。
- “展示”：最后，应当展示候选人的工作、生活、兴趣爱好等方面的情况，展示自己的专业知识、项目经验、解决方案等。展示可以激励候选人向面试官展现自我的价值，让面试官觉得自己是一个真正的合格者。

### 2.别人怎么说的
很多候选人都会提供自己的建议或担忧，给面试官留下一些建议。下面是一些最常见的建议供参考：

- 提问前要先检查自己的问题
- 在进入面试前要写好计划
- 不要过于贪婪，不要自暴自弃
- 在面试中要坚持正确的态度，没有不礼貌的人生
- 如果面试官不愿意给出答案，你就自己去查资料，再自己思考和想出答案
