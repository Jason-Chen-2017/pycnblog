# 2.核心概念与联系
## （一）移动应用开发与应用生命周期管理
移动应用开发是一种基于移动端设备的应用开发方式，它利用移动端设备的特性，例如摄像头、相机、位置服务、支付、推送消息、社交网络、地图等等，来满足用户不同场景下的应用需求。其生命周期如下：

1. 项目启动阶段：

项目启动阶段主要包括产品需求分析、产品设计、信息系统建设、开发环境准备等工作，在这个阶段完成产品的整体设计、开发计划、以及人员招聘。

2. 应用开发阶段：

开发阶段主要包括移动应用的模块化开发、架构设计、接口设计、界面设计、代码实现和测试，以及版本迭代更新等工作。

3. 部署发布阶段：

部署发布阶段主要包括编译打包、服务器配置、性能优化、数据迁移、安全评估、灾难备份等工作，目的是让应用运行正常。

4. 用户运营维护阶段：

用户运营维护阶段主要包括用户培训、数据分析、bug修复、功能升级、维护升级等工作，目的是维持应用的稳定、健康、安全、和便捷。

## （二）架构设计方法论
架构设计是指根据某个业务目标或解决方案，制定相应的架构，即确定系统的结构、功能和关系。架构设计的目的就是为了实现系统的可靠性、可用性、扩展性、可维护性、可操作性。架构设计方法论可以分为过程方法论和结构方法论。

### 1）过程方法论
过程方法论是指按照一般流程进行系统架构设计，可以包括以下几种设计步奏：

1. 需求分析：明确业务目标、系统范围、功能模块、数据流向、性能指标、可靠性要求、安全性约束、使用环境及依赖情况、外部接口、以及其他相关需求。

2. 概念结构设计：建立系统中的基本概念和组织结构，包括功能、子系统、层次、角色、职责、接口、协议、组件、资源等。

3. 数据流设计：描述系统的数据流动和处理，包括实体、属性、事件、消息、序列图等。

4. 进程协调设计：定义消息传递、进程间通信、同步、异步调用、事务处理、错误处理等机制。

5. 服务架构设计：描述系统的业务逻辑、服务调用、服务容错、负载均衡、集群规模、安全、稳定性、监控、配额管理、风险管理、可用性等。

6. 部署策略设计：选择部署方式、分区规划、资源分配、网络配置、权限控制、软件管理、自动化测试、监控管理等。

### 2）结构方法论
结构方法论是指采用一种层次化的方法，按照一定的标准，将系统分解成多个部分，并且每个部分都有自己的目标和特征。结构方法论可以分为静态结构方法论和动态结构方法论。

静态结构方法论是指系统的静态结构不发生变化。例如，人力资源信息系统的静态结构包括HR、资产、招聘、绩效、考勤、培训等部门。

动态结构方法论是指系统的动态结构不断地改变，根据不同的业务发展阶段、客户需要、竞争对手的变化等因素进行调整。例如，电子商务网站的动态结构可能由静态的前端、中间件和后端组成，随着业务的发展，可能会加入新的模块比如搜索、推荐、购物车等。

## （三）常见的云计算服务与厂商
云计算是一种虚拟化技术，它通过云服务商提供的服务器资源、存储空间、网络互联等云资源，在云上部署应用系统，以解决IT的存储、计算、网络等资源密集型的问题。目前最流行的云计算服务商有AWS、Azure、阿里云、腾讯云等。
