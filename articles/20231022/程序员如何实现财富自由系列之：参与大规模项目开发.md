# 1.背景介绍


　　编程是一种复杂而又有趣的工作，可以让你在短时间内获得成就感。因为它可以使计算机的硬件、软件甚至整个互联网都得到极大的改进。如果能扎实地掌握编程技巧和逻辑能力，并将其应用于解决各种实际问题，那么你就可以在一定程度上实现财富自由。

　　2019年7月份，美国最大的IT公司Facebook宣布全面启用旗下全职招聘计划，吸引超过2万名全职工程师加入社区，参与构建创新网络、社交媒体等业务。然而这一计划也引起了一些IT从业者的质疑，包括Google、微软、Facebook等科技巨头，认为其意在鼓励暴富、利用个人关系和权力勾结。

　　不过，研究表明，科技公司并不是纯粹为了钱。在 Facebook 的案例中，那些加入该计划的工程师都付出了很多，其中不少人在该计划内期积累了丰厚收入。这些人之所以被选择，除了他们已经有所收获外，还有其他原因吗？

　　2019年3月初，LinkedIn发布了一项名为“第十二次女性首席执行官”(CEO)的招聘，邀请全球各国的100多名高级管理人员到LinkedIn担任CEO。据报道，这些高管将会帮助打造一个全新的社交网络平台，提供更加专业化的服务给用户。这无疑是一个很有吸引力的行当。

　　不过，对于那些没有意识到自己或身边的人正在受到所谓的“数字货币”（Digital Currency）的驱动而产生的愤慨情绪，这样的消息可能会令人生畏。同时，如今数字经济和区块链技术日益火热，这也带来了更多的机遇。

　　值得注意的是，即使有充足的资金、技术资源或个人才华，想成为一名成功的CEO仍然具有相当的难度。如何保持工作的投入并从长远角度看待事物，才是成功的关键所在。只有真正投入其中，才能保证自己的生命不断延续，并最终获得成功！

# 2.核心概念与联系
## 什么是项目管理？
　　项目管理是指对一项任务进行分解，并将它们按顺序、合理地安排完成的过程。项目经理负责组织团队成员，制定工作计划和优先次序，协调资源和进度，确保项目按时、精准、可靠地完成。项目管理是一个企业最基本的岗位，也是许多高级管理人员的必备技能。

## 什么是Scrum？
　　Scrum是项目管理的一种方法。它基于增量开发的原则，把产品开发周期切分成短小的迭代周期，每一个迭代周期称为一个sprint，每个sprint包含若干个反映功能和业务变化的小功能开发任务。每个sprint结束后，团队成员将产品演示、评审，并收集反馈，以调整迭代方向。Scrum通过适应性反馈机制，优化工作流程和角色分配，提升协作效率和开发质量。

## 什么是敏捷开发？
　　敏捷开发(agile development)是一种敏捷的软件开发过程，其理念是快速反馈和迭代，要求响应客户的需求，在项目初始阶段，先建立起用户故事(user story)和产品原型。然后，再按照敏捷方法，即产品BACKLOG(产品待办事项列表)，按照顺序进行开发，将功能模块划分成多个小块，每一块开发完成后，立即测试，形成产品包(product package)。最后，客户或者测试人员接受产品包并提出反馈，将此反馈作为迭代方向的依据，调整产品的开发节奏。敏捷开发减少了项目开发时间，提高了项目交付质量。

## 什么是大规模项目开发？
　　大规模项目开发(large-scale project development)是指企业计划、组织、人员、技术资源及支持环境条件所限，而导致的某种无法克服的问题。一般来说，企业中有一些内部或外部的压力或风险，比如，外部竞争对手的增加、需求变化、政策法规等。因此，企业在设计、开展大规模项目前，要认识到以下几个问题：

1. 项目背景及目的；
2. 项目范围、目标、要求、边界；
3. 资源成本估算；
4. 投入资源预算；
5. 管理层规划和授权；
6. 流程、管理制度、沟通工具、质量控制标准等方面的考虑。
