# 1.背景介绍


## 分治算法（Divide and Conquer）
分治算法是一种用于高效解决复杂问题的算法设计策略，它将一个复杂的问题分成两个或更多的相同或相似子问题，递归地解决这些子问题，然后再合并其结果，得到最终的解决方案。简单来说就是一把分而治之的工具。它的主要思想是将待求解的大问题拆分成各个规模较小、性质相近的子问题，逐层求解，最后合并得到整个问题的解。

举例来说，假设要在数组中找出最大元素，那么可以采用如下方式：

1. 将整个数组平均切分成两半，分别找出左右两边的最大值。
2. 比较左右两边的最大值，取其中更大的那个数作为该数组的最大值。
3. 以此类推，直到只剩下一个元素或者只剩下两个元素时，直接比较并返回其中的最大值即可。

通过这种方法，我们就能找到整个数组中的最大元素。实际上，分治算法可用于解决很多复杂的问题。比如：排序算法，矩阵乘法，图像处理等等。在现代计算机科学和编程技术发展的历史进程中，各种复杂问题都经历了由简单到复杂的发展过程，而分治算法则始终占据着统领地位。

总结起来，分治算法包含以下三个基本要素：
- 一组子问题：将原问题划分为几个规模较小的相同子问题，每个子问题又可进一步划分为多个规模较小的子问题。
- 递归求解子问题：对每一个子问题，采用递归的方法进行求解，递归结束后再合并子问题的解得到原问题的解。
- 合并结果：合并子问题的解，得到原问题的解。

为了更好的理解分治算法，我们需要首先了解一下什么是二叉树。

# 2.核心概念与联系
## 二叉树
### 定义
在信息论和计算理论中，二叉树（binary tree）是每个结点最多有两棵子树的树结构。通常子树被称作“左子树”（left subtree）和“右子树”（right subtree），而根节点被称作“二叉树”。除根节点外，所有非叶节点都只有左子树或右子树。二叉树常用存储、表示和操作数据的方式，广泛应用于数据结构、算法及计算机科学领域。

### 特性
#### 满足二叉搜索树性质
对于一颗二叉树，若任意节点的左子树不空，则左子树上所有结点的值均小于它的根结点的值；若任意节点的右子树不空，则右子树上所有结点的值均大于它的根结点的值。同时，二叉查找树还具有唯一路径和最小关键字的特点。因此，二叉树也称为二叉搜索树。
#### 有序性
在任意一棵二叉树中，若任意节点的左子树不空，则左子树上所有结点的值均小于它的根结点的值；若任意节点的右子树不空，则右子树上所有结点的值均大于它的根结点的值。
#### 左中右遍历序列
给定一棵二叉树，按照某种次序对节点进行编号（有前序、中序、后序三种），记为前序序列、中序序列、后序序列。前序序列指的是根节点在先，左子树、右子树序列在后。中序序列指的是左子树序列、根节点、右子树序列的顺序。后序序列指的是左子树、右子树序列、根节点的顺序。例如，给定如下二叉树：

        5
      /   \
     3     7
    / \     \
   2   4     9

前序序列：5->3->2->4->7->9

中序序列：2->3->4->5->7->9

后序序列：2->4->3->9->7->5

一般情况下，由于时间、空间复杂度方面的限制，我们只需关注两种遍历方式：前序和中序遍历。前序遍历可以方便地实现快速排序，而中序遍历可以实现堆排序。另外，还有一种深度优先搜索（DFS）和宽度优先搜索（BFS）的方式也可以用来判断是否为二叉树。
