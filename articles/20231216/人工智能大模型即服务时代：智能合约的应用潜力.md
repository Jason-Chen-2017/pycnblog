# 4.具体代码实例和详细解释说明

在这里，我们将提供一个简单的智能合约代码实例，并详细解释其工作原理。

```solidity
pragma solidity ^0.5.16;

contract SimpleContract {
    uint256 public totalSupply;

    constructor() public {
        totalSupply = 1000000 * (10 ** uint256(18));
    }

    function transfer(address payable recipient, uint256 amount) public payable {
        require(amount <= totalSupply, "Transfer amount exceeds total supply");
        require(msg.value == amount, "Incorrect transfer amount");

        totalSupply -= amount;
        recipient.transfer(amount);
    }
}
```

这个智能合约的主要功能是实现一个简单的代币转移功能。合约的构造函数初始化总代币数量为1000000个，每个代币的精度为18位。合约的`transfer`函数用于将代币从发起者的地址转移到接收者的地址，并更新总代币数量。

在这个代码实例中，我们可以看到智能合约使用了以下几个关键概念：

- `pragma solidity`：用于指定Solidity版本。
- `contract`：用于定义智能合约的关键字。
- `public`：用于定义公共变量的访问修饰符。
- `constructor`：用于定义智能合约的构造函数。
- `require`：用于进行条件判断的关键字。
- `transfer`：用于实现代币转移功能的函数。
