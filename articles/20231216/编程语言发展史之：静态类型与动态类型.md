# 2.核心概念与联系

## 2.1 静态类型

静态类型（Static Type）是一种编程语言的类型检查方式，它在程序编译期间就进行类型检查，以确保程序中的变量使用与定义的类型一致。静态类型的主要优点是它可以在编译期间发现类型错误，提高程序的可靠性和安全性。

静态类型语言的代表性语言有C、C++、Java、Go等。

## 2.2 动态类型

动态类型（Dynamic Type）是另一种编程语言的类型检查方式，它在程序运行期间进行类型检查，以确保程序中的变量使用与定义的类型一致。动态类型的主要优点是它提供了更大的灵活性，允许程序员在运行时根据实际需要改变变量的类型。

动态类型语言的代表性语言有Python、Ruby、JavaScript等。

## 2.3 静态类型与动态类型的联系

静态类型和动态类型是编程语言中两种不同的类型检查方式，它们的联系在于它们都是为了解决编程过程中的类型错误问题而设计的。静态类型在编译期间进行类型检查，而动态类型在运行期间进行类型检查。

## 4.1 静态类型检查的代码实例

以C语言为例，我们可以编写一个简单的程序来演示静态类型检查的过程：

```c
#include <stdio.h>

int main() {
    int x = 10;
    float y = 3.14;
    int z = x + y; // 静态类型检查：x的类型为int，y的类型为float，因此z的类型为int
    printf("%d\n", z);
    return 0;
}
```

在这个程序中，我们定义了一个`int`类型的变量`x`，一个`float`类型的变量`y`，以及一个`int`类型的变量`z`。在计算`z`的值时，编译器会进行静态类型检查，确保`x`和`y`的类型一致。

# 6.附录常见问题与解答

Q：什么是静态类型？
A：静态类型是一种编程语言的类型检查方式，它在程序编译期间就进行类型检查，以确保程序中的变量使用与定义的类型一致。

Q：什么是动态类型？
