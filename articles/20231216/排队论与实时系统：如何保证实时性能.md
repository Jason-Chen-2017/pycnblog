# 2.核心概念与联系

在实时系统中，排队论主要关注的是系统中的任务调度和资源分配。实时任务可以被划分为多种类型，例如硬实时任务、软实时任务等。硬实时任务是指必须在确定的时间范围内完成的任务，而软实时任务则是指在满足一定的性能要求的前提下，可以允许一定延迟。

排队论中的任务可以被视为一个个独立的任务，这些任务需要在系统中的资源（如CPU、内存等）上进行调度。在实时系统中，资源分配和任务调度是密切相关的，因此排队论可以帮助我们理解实时系统中的性能瓶颈，并提供一种有效的方法来优化系统性能。

# 3.核心算法原理和具体操作步骤以及数学模型公式详细讲解

排队论中的算法主要包括：

1. M/M/1 队列模型：这是一个简单的排队模型，假设到达任务的分布遵循指数分布，服务时间的分布也遵循指数分布，系统中只有一个服务器。
2. M/M/K 队列模型：这是一个扩展的排队模型，假设到达任务的分布遵循指数分布，服务时间的分布也遵循指数分布，系统中有K个服务器。
3. G/G/1 队列模型：这是一个更复杂的排队模型，假设到达任务的分布可以是任意的，服务时间的分布也可以是任意的，系统中只有一个服务器。
4. G/G/K 队列模型：这是一个更复杂的排队模型，假设到达任务的分布可以是任意的，服务时间的分布也可以是任意的，系统中有K个服务器。

3. 系统吞吐率：θ ，可以用以下公式表示：
$$
\theta = \frac{\mu}{\lambda}$$
其中，λ 表示到达任务的平均率，μ 表示服务任务的平均率。

def m_m_1_queue(lambda_, mu_, t):

    p = 1 - np.exp(-lambda_ * t)
