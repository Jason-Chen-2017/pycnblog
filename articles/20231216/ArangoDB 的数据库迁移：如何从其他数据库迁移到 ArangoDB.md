# 2.核心概念与联系

在讨论如何从其他数据库迁移到 ArangoDB 之前，我们需要了解一些核心概念。

## 2.1 ArangoDB 的数据模型

ArangoDB 支持三种数据模型：图、文档和关系。图数据模型适用于具有复杂关系的数据，如社交网络、知识图谱等。文档数据模型适用于不同类型的数据，如 JSON 文档、XML 文档等。关系数据模型适用于传统的关系型数据库，如 MySQL、PostgreSQL 等。

## 2.2 数据库迁移的类型

数据库迁移可以分为两类：全量迁移和增量迁移。全量迁移是指将源数据库中的所有数据迁移到目标数据库。增量迁移是指将源数据库中的新增数据迁移到目标数据库。

## 2.3 数据库迁移的方法

数据库迁移可以使用以下方法：

- 导出/导入：将源数据库的数据导出到某种格式，然后将其导入到目标数据库。
- 数据库同步：将源数据库和目标数据库同步，以确保它们保持一致。
- ETL（Extract、Transform、Load）：从源数据库提取数据，对其进行转换，然后将其加载到目标数据库。

4. 连接到目标数据库。

数据库同步的算法原理主要包括以下步骤：

# 4.具体代码实例和详细解释说明

在本节中，我们将通过一个具体的代码实例来详细解释如何从其他数据库迁移到 ArangoDB。

假设我们要从 MySQL 数据库迁移到 ArangoDB。我们可以使用以下步骤：

1. 安装 ArangoDB。
2. 创建目标数据库。
3. 使用 MySQL 的 dump 工具将 MySQL 数据库的数据导出到 SQL 文件。
4. 使用 ArangoDB 的 import 工具将 SQL 文件中的数据导入到目标数据库。
5. 验证迁移结果。

以下是相应的代码实例：

```
# 安装 ArangoDB
wget https://download.arangodb.com/arangodb-latest-3.x.tar.gz
tar -xzf arangodb-latest-3.x.tar.gz
cd arangodb-latest-3.x
./install.sh

# 创建目标数据库
arangodb --database.create mydb

# 导出 MySQL 数据库的数据
mysqldump -u root -p mydb > mydb.sql

# 导入 ArangoDB 的数据
arangodb --database.import mydb mydb.sql

# 验证迁移结果
arangosh -u root -p arango -d mydb _db
```
