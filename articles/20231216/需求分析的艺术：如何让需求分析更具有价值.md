# 2.核心概念与联系

在需求分析中，我们需要关注以下几个核心概念：

1.需求：需求是客户或用户对软件的期望和要求，可以是功能性需求（例如，软件应该具有某个功能）或非功能性需求（例如，软件应该具有某个性能指标）。

2.需求分析：需求分析是收集、分析、验证和确认客户需求的过程，旨在确保软件能够满足客户的需求。

3.需求文档：需求文档是一份详细的文档，用于记录和描述软件的需求。需求文档应该包含需求的详细描述、需求的优先级、需求的验证方法等信息。

4.需求验证：需求验证是一种方法，用于确保软件的需求满足客户的需求。需求验证可以通过各种方法来实现，例如，用户测试、代码审查等。

5.需求跟踪：需求跟踪是一种方法，用于跟踪软件的需求是否已经实现。需求跟踪可以通过各种方法来实现，例如，需求跟踪矩阵、需求跟踪表等。
