## 4.1 缓冲区溢出

缓冲区溢出是一种常见的安全问题，它发生在程序尝试将数据写入一个缓冲区时，当缓冲区没有足够的空间容纳这些数据时。这可能导致程序崩溃，或者甚至允许攻击者执行恶意代码。

以下是一个简单的代码实例，演示了缓冲区溢出的问题：

```c
#include <stdio.h>
#include <string.h>

int main() {
    char buffer[10];
    strcpy(buffer, "Hello, World!");
    return 0;
}
```

在这个代码实例中，我们尝试将一个字符串复制到一个只有10个字节的缓冲区中。如果字符串的长度大于10个字节，则会导致缓冲区溢出。

为了解决这个问题，我们可以使用`strncpy`函数，它可以限制复制的字符串长度：

```c
#include <stdio.h>
#include <string.h>

int main() {
    char buffer[10];
    strncpy(buffer, "Hello, World!", 10);
    return 0;
}
```

在这个修改后的代码实例中，我们使用`strncpy`函数将字符串复制到缓冲区中，并限制复制的字符串长度为10个字节。这样可以防止缓冲区溢出。

## 4.2 格式字符串攻击

格式字符串攻击是一种常见的安全问题，它发生在程序使用格式化字符串函数（如`printf`或`sprintf`）时，当格式化字符串中包含用户输入的数据时。这可能导致程序执行恶意代码，或者泄露敏感信息。

以下是一个简单的代码实例，演示了格式字符串攻击的问题：

```c
#include <stdio.h>
#include <string.h>

int main() {
    char buffer[10];
    sprintf(buffer, "Hello, %s!", "World");
    printf(buffer);
    return 0;
}
```

在这个代码实例中，我们使用`sprintf`函数将格式化字符串写入缓冲区，并将用户输入的数据作为格式化字符串的一部分。如果用户输入的数据包含格式化字符串的格式，则可能导致格式字符串攻击。

为了解决这个问题，我们可以使用`printf`函数的格式化字符串参数，而不是使用`sprintf`函数：

```c
#include <stdio.h>
#include <string.h>

int main() {
    char buffer[10];
    printf("Hello, %s!", "World");
    return 0;
}
```

在这个修改后的代码实例中，我们使用`printf`函数的格式化字符串参数，而不是使用`sprintf`函数。这样可以防止格式字符串攻击。
