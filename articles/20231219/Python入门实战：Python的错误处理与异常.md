# 2.核心概念与联系

## 2.1 错误与异常的定义与区别

在Python中，错误（Error）和异常（Exception）是两个不同的概念。错误是指在程序运行过程中发生的问题，可以是编译时的错误（如语法错误）或运行时的错误（如访问不存在的变量）。异常则是指在程序运行过程中发生的不正常情况，可以是预期的异常（如文件不存在）或者是未预期的异常（如内存泄漏）。

错误和异常的区别在于，错误是程序运行过程中必然会导致程序失败的情况，而异常则是程序运行过程中可能会导致程序失败的情况，但并不一定会导致程序失败。

## 2.2 错误处理与异常处理的方法

Python提供了两种方法来处理错误和异常：一种是通过try-except语句来捕获并处理异常，另一种是通过raise语句来抛出错误或异常。

### 2.2.1 try-except语句

try-except语句的基本结构如下：

```
try:
    # 尝试执行的代码
except ExceptionType:
    # 异常处理代码
```

在try块中，我们尝试执行一段代码，如果在这段代码中发生了异常，则会跳出try块并进入except块，执行异常处理代码。如果在try块中没有发生异常，则会继续执行try块中的其他代码。

### 2.2.2 raise语句

raise语句用于抛出错误或异常。它的基本语法如下：

```
raise ExceptionType, message
```

其中，ExceptionType是异常的类型，message是异常的消息。

# 4.具体代码实例和详细解释说明

## 4.1 错误处理与异常处理的代码实例

```python
# 示例1：尝试访问不存在的变量
try:
    a = b
except NameError:
    print("变量b未定义")

# 示例2：尝试访问文件不存在的路径
try:
    with open("不存在的文件.txt", "r") as f:
        print(f.read())
except FileNotFoundError:
    print("文件不存在")

# 示例3：自定义异常
class MyException(Exception):
    pass

try:
    raise MyException("自定义异常")
except MyException as e:
    print(e)
```

## 4.2 错误处理与异常处理的解释说明

示例1中，我们尝试访问不存在的变量`a = b`，由于变量`b`未定义，会触发`NameError`异常，然后进入except块，输出`变量b未定义`。

示例2中，我们尝试访问文件不存在的路径`open("不存在的文件.txt", "r")`，由于文件不存在，会触发`FileNotFoundError`异常，然后进入except块，输出`文件不存在`。

示例3中，我们自定义了一个异常类`MyException`，然后尝试抛出这个异常`raise MyException("自定义异常")`，由于抛出了异常，则进入except块，输出`自定义异常`。
