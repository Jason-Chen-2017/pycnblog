## 4.1 代码实例

假设我们有一个简单的计算机程序，它需要处理两个整数的加法和乘法。这个程序的代码如下所示：

```
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

def main():
    a = 5
    b = 10
    result1 = add(a, b)
    result2 = multiply(a, b)
    print("Addition result: ", result1)
    print("Multiplication result: ", result2)

if __name__ == "__main__":
    main()
```

在这个代码中，我们可以看到`add`和`multiply`函数之间存在依赖关系。`main`函数依赖于`add`和`multiply`函数，而`add`和`multiply`函数又依赖于`__name__`变量。这种依赖关系可能导致代码的维护和扩展变得困难。

为了解决这个问题，我们可以使用禅宗的思想来重构代码。首先，我们需要直接体验代码的本质，并且保持内心的平静和智慧。然后，我们可以通过以下步骤来解决代码依赖问题：

1. 将`add`和`multiply`函数的依赖关系抽象出来，并将其放入一个新的函数中。这个新的函数我们可以称之为`calculate`函数。`calculate`函数将接受一个字典作为参数，字典中包含了操作和它们的参数。

2. 修改`add`和`multiply`函数，使它们不再依赖于`__name__`变量。而是将这个变量作为参数传递给`calculate`函数。

3. 将`main`函数的依赖关系抽象出来，并将其放入一个新的函数中。这个新的函数我们可以称之为`print_results`函数。`print_results`函数将接受`calculate`函数的返回值作为参数，并将其打印出来。

通过以上步骤，我们已经成功地解决了代码依赖问题。新的代码如下所示：

```

    return a + b

def multiply(a, b):
    return a * b

    if operation == "add":
        return add(a, b)
    elif operation == "multiply":
        return multiply(a, b)

def print_results(results):
    for key, value in results.items():
        print(key + ": ", value)

    a = 5
    b = 10
    operation = "add"
    results = calculate(operation, a, b)
    print_results(results)

    main()
```

在这个新的代码中，我们已经成功地消除了代码依赖问题。`calculate`和`print_results`函数之间的依赖关系已经被抽象出来，这使得代码更加易于维护和扩展。
