# 1.背景介绍


## 数据结构简介
数据结构（Data Structure）是计算机中存储、组织数据的方式。在编程语言里，数据结构往往指的是一组数据的集合以及对这些数据进行操作的一套规则。它主要分为以下几种类型：
- 基本数据结构：包括整型、浮点型、字符型、布尔型、指针等。
- 线性结构：包括数组、链表、栈、队列。
- 树形结构：包括二叉树、堆、森林、trie树。
- 图形结构：包括图、树图、网格图。
## 算法概述
算法（Algorithm）是指解题的方法、顺序或步骤。在程序设计中，为了完成特定功能，计算机执行一系列步骤来解决问题。因此，算法在计算机科学领域十分重要，它的作用不仅仅是指导生活，更重要的是用于解决实际问题。算法可分为两大类：
### 确定性算法
确定性算法（Deterministic Algorithm），也称为非随机化算法，它对于相同输入始终产生相同的输出结果。其关键特征就是确定的执行过程，每次运行得到的结果相同；如：冒泡排序、插入排序、选择排序等。
### 不确定性算法
不确定性算法（Nondeterministic Algorithm），也称为随机化算法，它对于相同输入可以产生不同的输出结果。其关键特征是执行过程中会涉及到随机性，可能每次都不一样；如：随机搜索算法、贪心算法、模拟退火算法等。
数据结构与算法结合起来就是一个完整的程序设计方法论。作为工程师和专家，必须掌握好两种算法类型及它们之间的区别，并学会分析和应用他们，提升程序效率。

数组的缺点也是显而易见的，数组大小一旦确定就不能更改，如果元素个数多于数组大小，则可能导致溢出。同时，数组内存在创建后就固定了，无法实现灵活地增加数组容量。
## 链表
链表(Linked List)是一种非连续的内存块，通过指针指向下一个节点。首部结点和尾部结点通常用哨兵节点进行标识。如下图所示：
链表的优点在于可以动态添加元素和删除元素，并且在内存上不受数组大小限制，不会造成内存碎片。同时，在遍历链表的时候，只需要初始化一个指针即可，速度快而且占用内存少。但是链表缺点也很明显，由于每个节点只包含一个指针，指针占用的空间比较大，所以链表比较浪费空间。此外，如果要查找某个元素，则需要从头开始一个个比较，时间复杂度较低。如下图所示：
## 哈希表
哈希表(Hash Table)是基于散列函数将元素映射到存储位置的一种数据结构。它通过散列函数计算出元素对应的存储位置，把元素存储到这个位置。在取出元素的时候，根据散列函数的计算结果直接定位元素所在位置即可。
哈希表的优点在于可以在平均时间复杂度 O(1) 下查找元素，这比数组和链表快得多。另外，通过调整散列函数使得两个不同元素被映射到同一个位置的可能性非常小。但是缺点也是很明显的，一是哈希冲突过多会导致查询时间变慢，二是哈希表占用的内存空间和元素个数都可能随着输入的变化而增长。

## 堆
堆(Heap)是一种特殊的树，满足一种堆序性质，即子结点的值总是大于等于其父结点的值。如下图所示：
堆可以采用两种方式实现：最大堆（Max Heap）和最小堆（Min Heap）。最大堆的每个父节点的值都大于等于其子节点的值。最小堆相反，每一个父节点的值都小于等于其子节点的值。在堆中加入新元素的时间复杂度都是O(log n)，删除堆顶元素的时间复杂度也是O(log n)。
