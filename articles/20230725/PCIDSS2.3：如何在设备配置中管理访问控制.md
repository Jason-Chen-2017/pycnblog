# 1.简介
         
PCI DSS 2.3 (Payment Card Industry Data Security Standard) 旨在保障支付卡机构（PCI）的信息系统、应用程序和网络的安全性、完整性、可用性和合规性。PCI DSS 2.3 包括六个主要部分。
第一部分是“PCI DSS 概述”，它涵盖了 PCI 机构、PCI 程序、PCI 要求、PCI 体系结构、PCI 授权管理和 PCI-DSS 改进计划等方面的信息。其中详细阐述了 PCI 标准的目的、作用、目标和适用范围。第二部分是“PCI DSS 的控制措施”，主要关注了 PCI 信息系统的管理、运营和维护等方面所需遵循的措施。第三部分是“PCI DSS 程序要求”，根据 PCI 机构的需求制定了一系列的制度和流程。第四部分是“PCI-DSS 体系结构概述”，概述了 PCI 体系结构，描述了各组织、部门之间的关系、流程和协作，并概述了通过严格的审核和测试来保护信息的关键环节。第五部分是“PCI 授权管理”，涉及到 PCI 授权的采购、开发、管理和更新等方面。第六部分是“PCI-DSS 改进计划”，将会对 PCI DSS 进行修订和完善，以提供更好的服务。
本文只讨论第三部分“PCI DSS 程序要求”中的第一章 “PCI 数据安全要求”。该章详细阐述了 PCI 数据安全实践的一般准则，即数据收集、存储和处理过程中的各种安全风险和攻击途径，以及应对这些风险的最佳做法。
# 2.前言
计算机安全一直是一个热门的话题。随着互联网的发展，越来越多的人依赖于网络服务，而网络服务又往往需要敏感数据（例如信用卡交易信息），因此越来越多的人担心网络安全事件的发生。而为了防止网络安全事件的发生，PCI DSS 在其第三部分 PCI DSS 程序要求 中就规定了 PCI 数据安全要求，其中包括了数据收集、存储、处理过程中所涉及到的各种安全风险以及相应的应对措施。
PCI DSS 定义数据安全要求的三个重要特征，分别是安全、隐私和完整性。安全要求是指保护个人信息和交易数据不受恶意攻击和泄露，确保信息的安全性；隐私要求是指不向任何第三方透露用户的个人信息和交易数据，仅由授权的个人访问和使用；完整性要求是指数据记录、传输、处理过程中不存在错误或漏洞。PCI DSS 提供的解决方案往往侧重于哪些方面呢？按照PCI DSS的定义，数据安全包括保密性、完整性、可用性、认证、审计、检测、响应和回溯等。对于个人信息和交易数据的安全，PCI DSS 给出了以下几种安全机制：
数据加密：采用加密技术将个人信息和交易数据存储在数据库或文件中时，能够抵御信息被截获、篡改、伪造等攻击，从而提高数据安全性。
访问权限控制：针对敏感数据集中的特定信息，对不同的用户授予不同级别的访问权限，如只允许管理人员查看，其他用户只能查看部分信息，从而保障个人信息和交易数据安全。
入侵检测和防护：设立相关的侦查、预警和处置团队，对出现异常的数据流量、异常的访问行为、异常的交易行为等行为进行监控和预警，从而对个人信息和交易数据安全形成威胁。
数据备份：建立数据备份机制，在本地和远程两个位置进行备份，以防止数据丢失和数据泄露，从而保持数据完整性和可用性。
PCI DSS 将数据安全要求分为4类：
1.网络安全要求：包括身份验证（AuthN），授权（AuthZ），加密（Confidentiality），数字签名（Integrity），访问控制（Accountability），错误检测和恢复（Recovery）。
2.应用安全要求：包括输入验证（Input validation），错误处理（Error handling），输出过滤（Output filtering），命令过滤（Command Filtering），环境隔离（Enviroment Isolation）。
3.数据库安全要求：包括数据采集（Data Collection），数据模型（Data Modeling），数据传输（Data Transfer），访问控制（Access Control），数据恢复（Data Recovery），日志记录（Logging），数据可移植性（Data Portability），数据完整性（Data Integrity）。
4.个人信息和交易数据安全要求：包括数据分类（Classification），个人标识（Personal Identification），数据共享（Data Sharing），第三方数据共享（Third Party Data Sharing），数据披露（Data Disclosure），数据限制（Data Limitation）。
# 3.PCI数据安全要求
## 3.1 PCI DSS 数据安全要求概述
PCI DSS 数据安全要求围绕着 PCI 行业的规范化、统一、健全、可操作的过程，针对个人信息和交易数据的收集、存储、处理过程中的各种安全风险以及相应的最佳做法进行了细化和明确。数据安全要求包含四个部分。
### 收集阶段要求：
- 数据收集策略：应当制定完整的、可靠的收集策略，确保收集的信息符合 PCI DSS 法律和法规的要求，并且应当进行定期、全面的检查。
- 数据收集人员：应当根据 PCI 职责分工，建立并配备有效的、全面的数据收集人员队伍，从事数据收集工作。数据收集人员应当具有相应的知识和技能，能够识别、分类、收集个人信息和交易数据。
- 收集数据手段：应当采取强化认证、评估和审计的方式，对个人信息和交易数据进行收集和存储。对于收集的数据，应当进行尽可能长时间的保存和管理，并通过监控、跟踪、分析等方式进行日常监测，发现异常情况时作出及时的反应。
- 存储设备：应当选择存储设备的类型、性能、功能和易用性，满足个人信息和交易数据的存档和检索要求。根据 PCI 法规，应当选择高度安全、难以破解的存储设备来存储个人信息和交易数据。
- 数据生命周期管理：应当设置有效的、符合 PCI DSS 法律和法规的生命周期管理政策，确保存储的个人信息和交易数据安全、有效地进行归档、运输、处理和销毁。
### 存储阶段要求：
- 存储区域：应当在合法、必要的区域内（例如，保存在 PCI 机构内部的专用区域）建立存储区域。
- 存储设施：应当选用合法、安全的存储设施，提供足够的空间容纳个人信息和交易数据，并考虑到人员活动和物理安全。存储设施应当具备备份和数据恢复的能力，能够迅速恢复丢失、损坏、被盗用的信息。
- 密码技术：应当使用高度复杂的密码技术来保护个人信息和交易数据，防止黑客获取和篡改信息。
- 责任制度：应当制定详细的、负责任的存储和处理政策，明确数据所有者的责任和义务，并由有关人员承担相应的义务和责任。
- 数据访问控制：应当设置符合 PCI 法规和 PCI DSS 协议的访问控制规则，对存储的个人信息和交易数据实施严格的访问控制，并对所有用户的请求进行审批和记录。
- 数据使用的限制：应当针对个人信息和交易数据的收集、使用、共享和泄露等操作，制定相应的使用条件和限制，避免个人信息和交易数据被滥用、误导或用于违背目的。
### 处理阶段要求：
- 安全传输协议：应当采用 SSL/TLS、SSH 或 IPSec 来实现网络通信中的数据安全传输。
- 数据加密：应当采用符合 PCI DSS 标准的加密技术来保护个人信息和交易数据，确保数据的机密性、完整性和可用性。
- 使用保密技术：应当采用合法、有效的保密技术来保护个人信息和交易数据，确保数据不会泄露给无权访问的第三方。
- 浏览器防火墙：应当部署强大的浏览器防火墙，实施自动扫描和过滤恶意的访问行为，阻断病毒和木马攻击。
- 文件筛选器：应当设置文件筛选器，确保存储在个人信息和交易数据中的文档不会被非授权的访问。
- 操作审计：应当对个人信息和交易数据的处理、流转和存档过程进行审计，记录并报告处理活动，确保数据的安全、正确的流动，不受未经授权的访问、使用和修改。
### 清除阶段要求：
- 数据销毁：应当对已存储的个人信息和交易数据进行销毁，确保数据无法恢复或被重新利用。
- 期限通知：应当在期限过去前通知数据所有者。
- 删除权限：应当设定删除权限的机制，只有相应的个人信息和交易数据主体才可以执行删除操作。
- 数据最小化：应当在确定没有风险之前，将个人信息和交易数据最低限度地保留下来。
- 实体审核：应当对所有处理的个人信息和交易数据进行实体审核，确保数据质量、完整性和可用性。
