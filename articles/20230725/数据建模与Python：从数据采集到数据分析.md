# 2.基本概念和术语
## 2.1 什么是数据建模？
“数据建模”是指将现实世界中各种数据模型化、归纳和整合，并将其转化成计算机系统能够理解和处理的形式，提高效率和准确性。数据建模有助于改进系统结构、降低开发难度、增加可维护性、提升用户体验，有效减少项目风险。数据建模技术已成为各行各业均需掌握的基础技能。

## 2.2 数据建模的定义、分类及适用范围
数据建模的定义：根据业务要求，将收集、存储、处理、分析、表达和传播等环节中存在的重叠和矛盾的活动和信息，按照数据模型的原则、规范和规则进行组织和标准化的过程。

数据建模的分类：按功能划分，数据建模可以分为以下几类：
- 实体关系模型：用于描述实体间的相互联系和依赖关系，是一种静态的数据模型，具有较强的业务含义。
- 对象模型：是实体关系模型的扩展，是一种静态的数据模型，通过封装、继承和关联等关系提供对象的表示。
- 动态数据模型：用于描述实体、关系及其变化的动态特性，是一种动态的数据模型，具有较强的物理和历史一致性质。
- 层次模型：用于描述不同层级的组织关系，是一种静态的数据模型，具有较强的业务含义。
- 事件驱动数据模型：是面向事件驱动架构的数据建模技术，也是一种动态的数据模型，具有较强的实时性和反应速度。
- 事实表数据模型：基于事实表的模型，是一种静态的数据模型，具有较强的容错性。

数据建模的适用范围：一般情况下，企业内部的数据建模一般按照需求进行，采用面向主题的建模方法，首先确定数据模型的主题区域、需要的业务信息、数据特征和关系、数据安全等，然后进行数据模型的设计、构建和测试。

## 2.3 数据建模的方法及工具
数据建模的基本方法：
- 数据流图法：通过分析数据流、实体之间的关系，生成描述实体和关系之间流动的图形，构建数据模型。
- 实体关系模型法：通过识别实体、属性、主键、外键等，建立实体之间的联系，确定实体之间的数据关系，设计实体关系模型。
- 对象模型法：对实体关系模型进行扩展，通过封装、继承和关联等关系描述对象之间的关系，构造对象模型。
- 面向服务的体系结构（SOA）：采用面向服务的体系结构（SOA），将数据建模作为一个独立的服务，由服务提供者和服务消费者共同完成，增强数据建模的过程透明性和自治性。

数据建模的基本工具：
- Excel：可以用来创建实体关系模型、对象模型和层次模型。
- PowerPoint：可以用来展示实体关系模型、对象模型、层次模型。
- Visio：可以用来创建数据流图、实体关系图、序列图等图形。
- UML：统一建模语言（Unified Modeling Language）是一种工业标准，用来建模和交流系统设计模型。
- SQL：结构化查询语言（Structured Query Language）是一种用于访问和处理关系型数据库的语言。
- Python：用来编程实现数据建模的工具。
- TOAD：Oracle数据库设计工具，可以用来创建数据库和对象模型。
