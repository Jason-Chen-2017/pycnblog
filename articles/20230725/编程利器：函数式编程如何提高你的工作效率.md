# 2.背景介绍
函数式编程(functional programming)，也称作泛函编程(generic functional programming)，是一种编程范式，它将电脑运算视为计算表达式的映射(即函数变换)。也就是说，编程是将输入数据转换为输出数据的计算过程，这个过程就是函数的应用。函数式编程强调只使用映射函数，而且要求无状态且有返回值的函数。这样做的一个显著结果就是易于理解和编写代码，更易于并行化处理。其优点包括代码可读性强、易于调试、单元测试容易实现、并行处理能力强、函数式编程范型具有简单性、一致性、不可变性等特点。

函数式编程的两大支柱是面向对象编程(object-oriented programming, OOP)和逻辑编程(logic programming, LP)。面向对象编程基于类的概念，将数据及行为封装成对象；逻辑编程则侧重于证明形式化逻辑问题的程序开发方法。

函数式编程更接近计算机科学的本质。计算机科学系统的本质是计算，而函数式编程将计算视为运算，是一种声明式的编程风格。通过纯函数和引用透明的特性，函数式编程使程序变得更加简单、可靠和可预测，因此得到广泛应用。

函数式编程在不同编程语言中都有不同的实现方式，其中，Java和C#是目前最流行的两种函数式编程语言。Scala、Clojure、Haskell、Erlang、F#等都是很受欢迎的函数式编程语言。函数式编程通常被认为是一种在计算机科学领域的高级编程范式。许多编程语言都支持函数式编程，如JavaScript、Scheme、Common Lisp等。

函数式编程工具箱包含很多优秀的函数库和框架，如MapReduce、Akka和Clojure；还有函数式编程语言编译器和运行时系统，如GHC、Erlang虚拟机、JVM上的Clojure虚拟机。

# 3.基本概念术语说明
## 3.1 函数
函数是一个接受输入参数（或不接受）并产生输出的过程。函数的定义由两个部分组成: 参数列表和函数体。例如，函数`f(x)`可以表示为`y = f(x)`. 函数可以作为另一个函数的参数或者结果返回给调用者。函数的第一个自变量是函数的输入参数，其他参数（如果有）叫做位置参数，而剩余参数叫做关键字参数。函数的返回值可以是一个单一的值，也可以是一个序列或结构。函数可能没有任何返回值（称为空函数），返回None。

## 3.2 纯函数
纯函数是一个满足以下条件的函数：

1. 没有副作用，即不依赖于外部变量或环境变量，不能修改传入参数的内容。
2. 返回值只依赖于输入参数。
3. 不存在循环和递归。

## 3.3 闭包(Closure)
闭包就是能够读取其他函数内部变量的函数。创建闭包的方法是在一个函数内嵌套了一个函数。外围函数的局部变量被保存到内存中，使得内嵌函数可以访问它。这样，当内嵌函数执行完毕后，外围函数依然可以使用这些局部变量。这种特性让闭包非常有用，比如回调函数。

## 3.4 高阶函数
高阶函数就是一个函数可以接收另一个函数作为参数，并且还可以返回一个函数。比如，map()就是一个高阶函数，它接收一个函数和一个序列作为参数，并返回一个新的序列，该序列的元素是对原序列每个元素运用该函数后的结果。

## 3.5 柯里化(Currying)
柯里化(Currying)就是把多参数的函数转换成一系列单参数的函数的技术。所谓柯里化，就是指将一个 n 个参数的函数，拆分成 n 个单参数的函数，然后返回新的函数。举个例子，假设有一个函数 fn，它接受三个参数 a、b 和 c，此时可以通过 currying 把它转换成多个单参数函数：

```python
def add(a):
    def inner_add(b):
        return lambda x: b + x
    return inner_add

fn = add(3)(4) # 返回值为: 7
```

上述代码中，函数 `add()` 用到了 closure 机制，先返回一个 inner_add 函数，此函数可以接收第二个参数 b 。然后，调用 `add(3)` 时会返回 inner_add 函数的一个实例，此时就可以将第三个参数 c 传给它，这样就构造出了一个新的函数 `inner_add`，它只接受一个参数 x ，并返回 b+x。最后，再将该函数作为参数传给 `add(3)` ，就获得了返回值为 7 的函数 `fn`。
