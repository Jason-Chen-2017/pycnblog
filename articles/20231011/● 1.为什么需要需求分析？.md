# 2.核心概念与联系
需求分析其实就是一种系统工程过程，它主要用来确定系统所需功能、性能、限制等各方面的要求。这里的各方面要求并非简单的需求文档，而是在需求分析过程中需要对方面的需求进行严格定义、梳理、归纳、分类、验证。这样才能更好的理解客户的真正需求，将其转换为符合产品设计和开发的标准化的需求清单。同时，需求分析还能够通过识别市场竞争对手的需求，制定差异化的解决方案，促进企业产品的竞争力提升。

什么是需求：需求是指对某件事情的要求、期望或偏好。例如，客户的某个诉求、产品的销售目标、解决某个问题的方法、做出某种决策所需的支持等。当需求出现时，通常要分成几个阶段进行研究。首先，需求的表达、获取和分析。第二，需求的定义、细化、确认、审查和拆分。第三，需求的评估、分析、排序、权衡和决策。第四，需求的收集和整理，确定最终的需求陈述、产品设计或开发计划。因此，需求分析的流程是相互独立的，但最终形成的需求结果有助于明确需求并为后续工作提供依据。

需求分析作为系统工程过程的一部分，需要考虑系统的生命周期中多个环节，如需求产生、需求的获取、需求的分析、需求的定义、需求的确认、需求的评价、需求的排序、需求的权衡、需求的决策等。需求分析的工具、方法和技术逐步地完善，可以帮助企业更好地理解需求并有效地将其转化为可执行的开发计划或产品设计。

我们可以先简要回顾一下数据结构和算法的基本概念和特点，然后再进一步阐述如何应用它们解决需求分析中的问题。
1.数据结构
数据结构是计算机科学领域中存储、组织和处理数据的抽象概念。它包括数组、链表、栈、队列、散列表、树、图等数据结构类型，每个数据结构都有自己的特性和优缺点。

2.算法
算法是一系列操作序列，用于完成特定任务。它由算法的输入、输出、基本操作（指令）和一个termination condition决定。算法的运行时间依赖于输入规模n的大小，对于相同输入规模n，不同的算法往往具有不同的复杂度。

3.时间、空间复杂度分析
为了在需求分析中分析算法的时间、空间复杂度，我们需要掌握相关的数学模型。比如，最坏情况下（worst-case），平均情况下（average-case），最佳情况下（best-case）。这些模型能够帮助我们更准确地判断算法的性能。
