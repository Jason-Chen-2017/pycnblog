# 1.背景介绍


移动应用（App）可用性一直以来都是一个值得关注的问题。在过去的几年里，随着智能手机的普及率的提升、用户需求的变动以及手机硬件性能的提升，移动应用的使用场景越来越广泛，无论是在生活中还是工作中。同时，随着互联网行业的不断发展，移动端App的开发也越来越受到人们的重视。
作为一个专业的产品经理，了解并准确地把握移动App的可用性至关重要。本文将从以下几个方面，探讨如何高效有效地设计出好的移动App可用性。
首先，我们要了解什么是App可用性。简单来说，App可用性就是通过技术手段或方法让用户能够顺利、流畅地使用App，做到最低限度上的正常运行，并且不会出现任何明显的错误或崩溃现象。我们可以从几个层次来定义App可用性：功能可用性、交互可用性、信息安全可用性、可靠性、兼容性等。每一层都会对App的运行给予不同的要求。如果我们想要打造一款流畅、顺滑、安全、可靠、兼容的App，就需要保证各个层级的可用性都得到满足。
其次，了解市场的需求和痛点。移动互联网的兴起和不断增长，已经成为消费者和企业都高度关注的问题。为了满足这些需求，App应当根据不同人的能力水平和设备类型进行优化。在这个过程中，我们需要反复验证和测试我们的产品。只有我们真正理解需求、痛点和客户的真实情况，才能更加深入地思考如何提供好的移动App可用性服务。
再者，强调可用性不是免费的。维护和改进App的可用性往往是花费时间成本和金钱投入的。所以，我们一定要小心谨慎地为维护和改进做好准备，从而帮助我们节省更多的时间和资源，更好地满足用户的需求。
最后，回归本源——用户需求。当我们为用户解决了某个痛点后，用户就会主动寻求帮助。所以，为用户提供有用的信息，帮助他们达成目标会更有价值。反之，如果我们只是想赚取用户的青睐而忽略了用户的需求，那么用户可能根本不会用我们的产品。因此，制作一款精良的可用性App需要充分地考虑用户需求。
# 2.核心概念与联系
我们首先来看一下App可用性相关的一些核心概念和联系。如下图所示：
首先，从人机交互层面，我们可以看到五种基本的可用性指标：可用性（Usability），易用性（Learnability），可发现性（Discoverability），可访问性（Accessibility），可信任性（Trustworthiness）。其中，可用性是衡量App整体质量的标准，是最基础也是最重要的一项。易用性则是指新手上手率，帮助用户正确、有效地完成任务；可发现性是指App的使用是否容易被找到，是搜索引擎优化（SEO）的基础；可访问性是指App的使用难易程度，是否可以便捷地进行屏幕阅读等；可信任性是指App在社交媒体上的影响力，能否吸引到足够多的人去评价和使用它。
然后，从技术实现层面，我们可以看到四类主要的方法：可用性辅助工具、可用性测试、可用性工程、可用性编码。它们的作用都是为了让App在所有层面上达到可用状态。例如，可用性辅助工具如翻译工具、校对工具、文本替代工具等，可以帮助用户快速、准确地理解App的内容；可用性测试可以检测App是否符合用户习惯、功能是否完整、错误处理是否令人满意；可用性工程师负责创建和维护App的可用性文档，帮助用户学习如何使用该App；可用性编码人员则负责确保App的所有功能都可以正常运行。
最后，从市场营销层面，我们可以看到三种营销策略：宣传策略、可接受性策略、评估策略。宣传策略包括向用户展示新功能、优惠券、信息等方式，可促使用户下载和试用App；可接受性策略则是设置较高的门槛要求，比如用户必须先满足某些条件才能下载或使用App；评估策略则是衡量App的满意度和效果，将结果反馈给用户，提升整个产品的可信度和用户体验。
综合以上三个方面，我们可以总结出App可用性的八大核心概念：可用性、易用性、可发现性、可访问性、可信任性、可用性辅助工具、可用性测试、可用性工程。它们之间存在着密切的联系，共同塑造出一个完美的可用性。

## （3）可用性模型
接下来，我们将从两个角度来阐述Word for the Day的可用性模型。
### （3.1）可用性成本模型
首先，我们来认识一下可用性成本模型。可用性成本模型是基于可预测的风险来描述产品在特定时间段内或短期内的使用和维护成本。可用性成本模型的核心假设是：产品的生命周期越长，其平均故障率就越高。可用性成本模型用来评估产品的可靠性、稳定性和经济性。
举个例子，苹果的AirPods Pro一经发布便迅速获得了非常大的用户关注。由于产品生命周期短，其平均故障率却很低，平均每年仅有一次严重故障。于是，AirPods Pro得到了广泛的应用。Apple公司利用其产品的生命周期短、平均故障率低的特点，建立了一个可用性成本模型：
“可用性成本 = 投资（研发+测试+生产+运营+售后）成本 + 产品生存期*平均故障率”
由此可见，AirPods Pro的可用性成本远远低于其研发、测试等其他环节。另外，AirPods Pro的生存期只有十天左右，故障率也相对较低。因此，可用性成本模型为我们提供了一种直观的评估方法。
### （3.2）可用性成本估算
我们来估算一下Word for the Day的可用性成本模型。首先，我们计算Word for the Day的研发成本、测试成本、运营成本、售后成本、销售收益等各项成本，并乘以各自相应的比例，得到总成本。假定Word for the Day的研发成本为10万元，测试成本为5万元，运营成本为10万元，售后成本为5万元，销售收益为15万元，则Word for the Day的总成本约为30万元。
可用性成本模型又有另外一个核心假设：可用性的减少或增加会带来经济成本的变化。假定Word for the Day的可用性下降3%，则对应的可用性成本下降15%。然而，考虑到用户对可用性的追求和消费者的习惯，我们认为Word for the Day的实际可用性可能偏高或偏低，所以此处的可用性成本估算只是一种参考值。
# 4.具体代码实例和详细解释说明
关于Word for the Day的可用性设计，我们还有一些细微的地方需要注意。例如，除了色彩外，字体的大小、间距、位置等都有可能影响App的可用性。那么，怎样才能让字体的大小、间距、位置等都有利于提高App的可用性呢？这里，我分享一些具体的设计建议。
## （1）设计字号大小
字号大小的选择应该根据App的背景色来确定。如果背景色为白色，则字号大小应该更大，如果背景色为黑色或浅灰色，则字号大小应该更小。
## （2）设计排版规则
排版规则的设计可以有助于提高App的可用性。排版规则应该满足易读、舒服、简洁、统一、连续、一致的原则。具体操作步骤如下：
1. 对文字长度进行限制，避免单行文字过长。
2. 使用斜体、加粗等格式化对文字进行装饰。
3. 避免单独使用数字、特殊符号等作为文字。
4. 将文字内容进行分组和结构化，形成适合阅读的文章格式。
5. 在每页或者章节末尾加入适当的空白，防止页面过多。
## （3）对齐
对齐是对视觉呈现方式的设计。如果字体的对齐方式不恰当，可能会产生视觉冲突，影响App的可用性。我们可以通过调整字号大小、颜色、间距、排列方式、排版规则等来调整对齐方式。如下图所示：
