# 1.背景介绍


## 消息队列
消息队列（Message Queue）是一种应用程序之间的数据交换方式，用于缓冲一系列的输入，并按顺序传送到其他应用程序，也可以用于在分布式系统中跨越进程、网络和计算机进行异步通信，它具有以下特点：
- 技术门槛低：无论是企业应用还是移动互联网应用，都可以利用消息队列快速实现数据传递；
- 可靠性高：消息队列保证消息传递可靠，即使中间节点出现故障也能确保消息不丢失；
- 扩展性好：消息队列能够根据需求随时增加消费者或生产者数量；
- 异步通信：消息队列提供了异步通信机制，允许生产者将消息放入队列而不必等待消费者处理，从而提升性能和吞吐量；
- 多模式支持：消息队列还支持主题订阅、持久化存储、死信队列等多种功能模式；

作为分布式系统中的重要组件之一，消息队列作为一个中心化的消息分发服务，主要用于解决以下三个关键问题：
- 数据冗余：由于多次复制，可以减少单点故障带来的影响；
- 业务连续性：在发生系统崩溃、机器宕机等异常情况时，可以保证消息的可靠传递；
- 流程控制：通过延迟消息或限流的方式，可以在一定程度上防止过载或流量洪峰；

目前主流的消息队列包括 Apache Kafka 和 Apache RocketMQ。Apache Kafka 是由 LinkedIn 开源的分布式发布-订阅消息系统，它是一个多用途分布式 messaging system，它最初被设计为一个统一的消息平台，后来逐渐演变成一个独立的产品。RocketMQ 是阿里巴巴集团开源的分布式消息队列中间件，其提供高可用、高并发、海量消息堆积能力。

本文首先会给出两个消息队列的基本概念与联系，然后对 Apache Kafka 和 Apache RocketMQ 的技术特性及优缺点进行比较分析，最后将重点介绍 Kafka 和 RocketMQ 在实际应用场景中的差异和共同点。
