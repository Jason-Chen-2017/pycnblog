# 2.核心概念与联系

## 图论Graph
图论是一种抽象数据类型，用一组顶点（vertex）和边（edge）来描述某些事物之间的关系。图可以表示现实世界中许多复杂的现象，例如城市交通网络、生物群落的微观结构等。图由节点和连接节点的边所构成。
### 基本术语
- 有向图：一个有向图是一个带箭头的图。如图所示，箭头表示方向。
- 无向图：一个无向图是一个没有箭头的图。如图所示，箭头只是用来表示边的方向。
- 简单图：一个简单图就是没有重复边的图。如图所示，有三条边（A-B、B-C、C-D），但是没有一条边（A-D）。
- 强连通图：一个强连通图指的是图中任意两个顶点之间都存在路径相连。如图所示，如果一条从顶点A到顶点D的路径可以从另一条从顶点B到顶点D的路径完全访问到顶点C，那么该图就是强连通的。
- 弱连通图：一个弱连通图只要不存在环就可以称作弱连通。如图所示，该图只有两条路径（A-B和A-C），所以不是弱连通的。
- 完全图：一个完全图就是所有顶点都被连接了的图。
- 子图：在图论中，一个子图是指一个图中的一些节点和对应的边组成的图。
## 排序算法Sorting Algorithm
排序算法是计算机用于将一串元素按一定顺序排列起来的一系列方法。不同的排序算法之间的差异主要在于它们的时间复杂度。常用的排序算法包括选择排序、冒泡排序、插入排序、希尔排序、归并排序。
### 选择排序Selection Sorting
选择排序是一种简单直观的排序算法。它的工作原理如下：首先在未排序序列中找到最小（大）元素，存放到排序序列的起始位置；然后，再从剩余未排序元素中继续寻找最小（大）元素，然后放到已排序序列的末尾。经过第一次循环之后，待排序的数据变成了[3, 5, 7, 1, 2]。第二次循环开始，先在已排序区间中找到最小（大）元素，假设为5，然后将这个元素放在序列末尾。此时待排序的数据变成了[3, 5, 7, 1, 2, 5].继续下去，再将剩余的元素依次填充到已排序序列中即可。
时间复杂度：O(n^2)
### 冒泡排序Bubble Sorting
冒泡排序是一种简单的排序算法。它重复地走访过要排序的数列，一次比较两个元素，如果他们的顺序错误就把他们交换过来。走访数列的工作是重复地进行直到没有更多的元素可以交换，也就是说该数列已经排序完成。
时间复杂度：O(n^2)
### 插入排序Insertion Sorting
插入排序是一种简单直观的排序算法。它的工作原理是通过构建有序序列，对于每个未排序元素，在已排序序列中找到适当的插入位置，并将其插入。插入排序在实现上，通常采用in-place排序，因而在从后往前遍历期间，需要反复移动元素，会产生较大的开销。
时间复杂度：O(n^2)
### 希尔排序Shell Sorting
希尔排序是一个基于插入排序的算法，它使得数据可以在恒定的时间内被排序。希尔排序又称递减增量排序算法，是不稳定排序算法。该方法又叫缩小增量排序算法，是开放寻址法的一种。它通过设计一个 gap 来使各组元素的比较和移位更加紧密，减少了相邻元素的比较次数。
