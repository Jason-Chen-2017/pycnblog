# 1.简介
         
在机器人流程自动化（RPA）领域，任务错误是最常见且严重的问题之一。本文将从自动化决策流程（ADP）理论及实践出发，阐述错误的来源、类型、表现形式及其对自动化系统的危害。并结合工程上常用的RPA工具与框架，通过案例分析，讨论任务错误的分类方法，以及如何进行系统性的改进。
# 2.概述
## 2.1 自动化决策流程理论
### 2.1.1 ADP理论简介
“精益制造”（Lean Manufacturing）是一种“精益生产”的方法，强调在每一次产出的过程中都关注客户需求，通过优化生产流程及制品质量来降低成本和提升效率，这种方式被广泛应用于零部件制造、制药、贸易等各个行业中。自动化决策流程（ADP）理论认为，当一个人的工作由计算机代替时，工作时间减少了，但由于其自动化程度不够高，会导致各种非期望的错误发生。因此，开发出一种有效的ADP方法论，可以帮助企业避免自动化决策流程出现的各种问题，如错判、遗漏或延迟、重复、过度依赖、不全面，从而更好地服务客户，创造更优质的产品及服务。
### 2.1.2 RPA与ADP之间的关系
自动化决策流程（ADP）理论指出，自动化将使得业务人员的工作时间缩短，而且使得人们能够更多的专注于其它活动，但同时也带来了新的挑战——如何处理由此产生的新问题？RPA（Robotic Process Automation，机器人流程自动化）是基于AI和人工智能技术所设计的用于完成重复性、机械性、流程化、标准化任务的计算机程序。它是一种综合型的技术，利用机器人系统及规则引擎的方式自动化处理业务流程，并支持知识库和信息检索系统。因此，两者是相关联的。当实现RPA后，企业就将需要关注两个方面：第一，如何识别并且克服由ADP所带来的问题；第二，如何最大限度的提升RPA的效率，确保其运行流畅、准确且可靠。
## 2.2 RPA演变历史及其应用场景
### 2.2.1 概述
RPA（Robotic Process Automation）是一种基于人工智能和计算机技术的自动化解决方案，能够实现大规模的企业级工作流程管理。由于处理过程的高度标准化、自动化、自动执行，RPA已成为越来越多企业管理部门的共同选择。目前，市场上已经涌现出诸如Oracle、SAP、Micro Focus、Zoho等知名企业，它们均采用RPA技术来改善他们的日常工作。但是，这项技术的推广仍然存在着一些问题。例如，对“精益制造”的认识不足、缺乏必要的基础理论支持、缺少实践经验、系统工程师能力不足等。因此，本文试图回答以下几个问题：
1.什么是任务错误、为什么会发生？
2.如何避免、克服任务错误的影响？
3.如何将任务错误分为哪些种类？又有哪些典型原因？
4.如何改进ADP？
5.哪些工具、框架及流程设计方法能帮助企业有效地应对任务错误？
### 2.2.2 自动化决策流程的产生背景
#### （1）引入自动化的背景
随着现代商业环境的复杂性提升及信息技术的发展，企业工作的重复性较大，手动执行的工作流程在一定程度上削弱了公司的竞争力。为了克服这一难题，研究者发现“精益制造”（Lean Manufacturing）方法是一种有效的方法。它强调以客户为中心，每一次产出的过程中都关注客户需求，通过优化生产流程及制品质量来降低成本和提升效率，这样做既提升了生产力，又降低了成本。
因此，在面向客户的需求的驱动下，出现了使用机器人代替人类的需求，而机器人也开始发挥其价值——可以完成很多重复性的、机械性的、流程化的、标准化的任务。机器人可以替代单个人员完成许多繁琐而乏味的重复性工作。此外，机器人还具有高度的自主学习能力，能够理解复杂的业务信息，根据组织目标及条件自动执行任务。
#### （2）传统决策流程管理的局限
传统的决策流程管理通常采用“等待—批准—执行”的工作模式。在这个过程里，工作人员首先等待，直到相关信息收集齐全，然后才能审阅这些信息。这种工作模式的缺陷在于效率低、易疲劳、易错误、耗时长。此外，这种模式往往没有考虑到自动化决策流程带来的额外复杂性，比如，不同的职责或部门之间的协作、流程的改变、错误修正等情况。因此，业界急需一种新的工作流程管理模式来弥补传统决策流程管理的不足。

#### （3）自动化决策流程的理论提出
为了解决传统决策流程管理存在的问题，作者提出了“自动化决策流程”（Automation Decision-Making）理论。它认为，作为人类事务的一部分，自动化决策流程既是不可或缺的，也是重要的。然而，只有在充分关注用户需求、充分利用计算机科学和相关技术的前提下，才能有效的使用这一技术。因此，需要制定一套适合业务的自动化决策流程策略，包括流程定义、流程配置、流程监控、工作分配、审批机制、系统集成及整体质量保证等方面。

## 2.3 任务错误的概念及分类
任务错误的概念来自心理学的一个观点，即人类行为中有些动作或事件在习惯或偏好中表现出来，即便实际上没有发生过，也可能引起注意或误导，甚至导致危险。例如，许多人由于过敏、抑郁症等病症，会在饮食上出现不良倾向，甚至导致早婚。因此，任务错误是由于习惯、情感、遗忘等原因导致的，是由于时间不足、能力不足或其它原因无法正确理解或操作某个系统、设备或功能。
任务错误主要包括以下几种类型：

1. 失误——指因错误而造成的失常或失败，例如，未按要求操作而造成的设备损坏、系统崩溃、数据丢失或泄露，或者意外的、失控的物理或电子控制。

2. 误判——指对任务结果进行判断时，错误的决定或偏差。例如，从某个角度看待事情，可能导致判断错误。

3. 遗漏——指在进行工作时忽略某些环节或步骤，导致不能正常完成任务，或导致产生错误结果。例如，漏掉了准备工作或材料。

4. 延迟——指某个任务超出预期的时间或持续时间。例如，工作任务完成太慢、不及时，或根本就不开始。

5. 重复——指按照相同的方式重复执行相同的操作，例如，手工记录数据或发送同样的指令。

6. 不完整——指完成某项任务时，结果仅有部分，或未达到预期效果。例如，生成的文件尚未全部填写或文档未完全编写完毕。

### 2.3.1 任务错误的典型原因
根据IBM的人机交互认证（HCI）研究，任务错误的典型原因如下：

1. 环境因素——如工作条件不适宜、混乱的办公环境、令人生厌的噪音，甚至是恶劣天气。

2. 理解障碍——如对指令或任务不清楚、知识不熟悉、掌握技能不足、不知道该怎么做。

3. 操作不熟练——如不能很好的掌握关键操作、无法做到细致入微。

4. 承受能力——如缺乏专注力、精神压力过大、记忆力差、学习能力不强、语言表达困难、情绪反应慢。

5. 交互知识缺乏——如缺乏对界面和控件的理解、对输入输出的信息需求不了解。

6. 认知风险——如对业务信息不熟悉、记忆力差、容易出现错误印象、注意力分散、缺乏对优先事项的把握、认知盲区等。

7. 数据缺乏——如缺乏有效的数据、有关数据的收集不全面、数据质量不符合要求。

## 2.4 RPA的基本流程设计方法
### 2.4.1 概述
流程是指业务中的各个环节、步骤及其顺序，是完成一项特定任务的各个阶段。流程设计就是根据企业的业务需求和流程标准，设立一个符合规则、有序、标准化的流程，用于顺利、高效地完成目标任务。流程设计方法是构建RPA流程的基石。一般来说，流程设计方法可以分为四大类：

1. 基于结构的设计方法——结构化方法、信息系统方法、基于角色的设计法、基于阶段的设计法。

2. 基于约束的设计方法——最小功能模型、模糊综合方法、先进综合方法、小步快走法、快速响应方法、合作团队设计法。

3. 基于目的的设计方法——功能设计法、体系设计法、工具包设计法、培训、实验测试、用户研究。

4. 基于范围的设计方法——功能模型、模块划分、模块设计、界面设计、操作手册设计、设计评审及修改。

### 2.4.2 流程设计的原则
1. 明确目标——如业务目标、操作对象、效率要求、风险管理、风险控制、灾难应对等。
2. 分清层次——目标、任务、步骤、环节、事务等。
3. 保证关键路径——保证关键路径上的任务可以按流程进行，并能够正确完成。
4. 维护一致性——每个人对流程的理解应该保持一致性。
5. 清晰的边界定义——如环节之间的交互、输入输出的接口等。
6. 允许适当的变通——如增加或减少环节、允许跳过环节、允许暂停或终止流程等。

## 2.5 RPA工程实践——如何防范任务错误
### 2.5.1 用例分析
用例分析是最重要的工程实践，目的是通过分析业务需求、功能需求和系统流程，找出流程中可能存在的错误，并定义出故障恢复流程。

分析流程：分析工作流程、判断流程的可操作性、建立流程模型、画出流程图。

检测流程：评估系统是否满足标准要求，检查数据输入、输出、存储是否有效、检查数据转换、计算过程是否有效、检查外部系统是否可用、检查反馈路径是否工作正常。

规避错误：对于每次错误，首先评估错误发生的原因，然后采取适当的措施纠正错误。常用的纠正方法包括重新输入、更改流程、调整参数、更改流程设计、检查文档、重启计算机、联系IT人员等。

文档控制：定期对流程文档进行审核、修改，确保文档准确无误。

建立流程监测报告：建立周期性、定性、定量的流程评估报告，总结当前系统的运行状况，及时发现并处置潜在问题。

### 2.5.2 错误分离——结构化方法
结构化方法将流程划分为不同的阶段，不同阶段对应的操作步骤相互独立，并对每个步骤的输入、输出、条件进行详细的定义。流程设计人员通过对每个阶段的操作步骤和条件定义，制定出各个步骤的运行顺序，防止发生错误。如：采购订单流水线（供应链管理）、销售订单流水线（销售管理）、生产订单流水线（生产管理）。

### 2.5.3 知识库——信息系统方法
信息系统方法将流程流程，设置知识库来描述流转的实体及其属性、实体间关系及其约束，并建立约束规则来约束业务人员的操作。通过知识库来指导业务人员的操作，降低流程出现错误的概率。如：库存管理知识库、采购订单知识库、销售订单知识库。

### 2.5.4 模块设计——角色设计法
角色设计法是一种先进的设计方法，它基于角色来划分功能、任务和活动，并对每个角色分别进行定义。在角色的定义中，可以定义每个角色的职责范围、输入、输出、条件、限制、配合角色、频繁出现的操作等。模块设计法可以帮助业务人员快速定位自己的角色，并根据自己的角色去完成相应的操作。如：仓库管理员、采购员、销售员、生产计划员、财务人员。

### 2.5.5 统一配置——先进综合方法
先进综合方法是一种以精益制造的方式进行设计，它对流程进行了规范化，并采用不同类型的自动化技术，如计算机辅助设计、人工智能、规则引擎、业务规则引擎等，通过统一配置实现系统的可靠运营。如：工厂配件订购系统。
