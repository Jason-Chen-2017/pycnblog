# 1.简介
  

## PostgreSQL 是什么？
PostgreSQL是一个开源关系数据库管理系统（RDBMS），采用POSTGRES的商标授权。它的目的是实现 SQL 标准的绝大多数功能并提供额外的特性，如：事务支持、触发器、数据类型、视图、约束、函数和存储过程等。同时它也是开源免费软件，可在任何操作系统上安装运行。

## 为什么要学习PostgreSQL？
1. PostgreSQL 的性能比传统 RDBMS 更好。
2. 功能更加丰富。
3. 支持完整的 ACID 事务处理特性。
4. 提供高可用性，灵活的数据复制和主从服务器方案。
5. 支持更多的编程语言。
6. 使用简单方便。
7. 大量的第三方工具支持。

# 2.基本概念术语说明
## 关系数据库
关系型数据库（Relational Database Management System）是基于关系模型来组织数据的数据库。关系模型把数据看做是二维表格结构中的记录。每个记录都对应着唯一的主键值，不同的记录之间通过某种联系方式关联起来。关系数据库管理系统就是用来管理关系型数据库的软件。

## SQL（Structured Query Language，结构化查询语言）
SQL 是一种用于关系数据库管理系统（RDBMS）的标准语言，用于存取、更新和管理关系数据库对象（关系表、视图、索引、存储过程等）。其语法类似于英文，易于学习且阅读，并且标准化，因此被广泛使用。

## DBMS（Database Management System，数据库管理系统）
数据库管理系统（Database Management System，DBMS）是指负责管理关系数据库的软件系统。它包括了数据库的设计、存储、数据操纵、网络通信和安全保护等功能。常用的 DBMS 有 MySQL、Oracle、MS SQL Server 和 PostgreSQL。

## 数据库系统
数据库系统是由硬件和软件组成的一个系统。硬件主要包括数据库服务器（通常使用服务器集群），磁盘阵列和存储设备；软件包括数据库管理系统（DBMS）、各种应用程序以及用于访问数据库的客户端程序。

## 数据模型
数据模型是用来描述现实世界中客观事物及其相互关系的一系列概念、符号和规则的集合。数据模型定义了如何表示和存储数据，以及对数据进行操作的方式。数据模型可以分为三大类：实体-关系数据模型、层次数据模型和对象数据模型。其中，实体-关系数据模型是最常用的模型之一。

## 模式（Schema）
模式（Schema）是指数据库对象的定义。数据库模式包含一系列的对象，包括表、视图、存储过程、触发器、索引和约束。一个数据库可以包含多个模式。

## 表（Table）
表（Table）是关系型数据库中最基本的对象。表是一种二维结构，每行代表一个记录，每列代表一个字段。一个表可以包含多个字段，但是所有的字段必须属于同一个表。

## 字段（Field）
字段（Field）是表中的一个属性或变量。每个字段都有一个名称和数据类型。字段数据类型定义了该字段可以存储的值的类型，比如整数、字符串、日期时间等。

## 记录（Record）
记录（Record）是表中的一条数据。记录由一组相关的字段值组成，这些字段值共同描绘了一个事物的某个方面。

## 属性（Attribute）
属性（Attribute）是指一个对象具有的静态特征或状态。在关系数据库中，属性由字段定义。属性也可指一个关系表中两张表之间的关系。

## 键（Key）
键（Key）是唯一标识表中每条记录的属性或字段。主键（Primary Key）是唯一标识表中每条记录的属性或字段。一个关系表只能有一个主键。

## 外键（Foreign Key）
外键（Foreign Key）是用来建立一个关系表与另一个关系表之间映射的字段。一个外键字段可以引用其他表的主键。

## 索引（Index）
索引（Index）是对数据库表中一列或者多列的值进行排序的一种结构。索引可以大幅度提升数据库的检索速度，但会降低插入、删除、修改记录时的效率。

## 视图（View）
视图（View）是只包含已存在的表或其结果集的虚拟表。视图不是实际的表，而是根据需要动态生成的表的集合。视图可以隐藏复杂的 Join 操作，使得用户只看到所需的数据。

## 联合（Join）
联合（Join）是将两个或多个表中的信息组合在一起的过程。如果两个表中存在相同的字段名，则可以通过指定字段名来连接这两个表。

## 事务（Transaction）
事务（Transaction）是指逻辑上的一组操作，要么都执行成功，要么都不执行。事务主要用于确保数据一致性和完整性。

## 锁（Lock）
锁（Lock）是用来控制对共享资源的并发访问的方法。在关系数据库中，锁又分为两种类型：排他锁（Exclusive Locks）和共享锁（Share Locks）。排他锁是独占锁，一次只能给一个事务使用；而共享锁允许多个事务同时读同一份数据。

## 触发器（Trigger）
触发器（Trigger）是一个在特定事件发生时自动执行的数据库操作。当满足触发条件时，触发器向数据库内插入一条新纪录，也可以修改或删除已有的记录。

## 函数（Function）
函数（Function）是用来对数据进行转换、过滤、验证、统计等操作的自定义命令。函数通常还包含有参数、返回值和异常处理机制。

## 存储过程（Stored Procedure）
存储过程（Stored Procedure）是一个预编译的 SQL 语句，它存储在数据库中，可以重复调用而无需解析和编译。存储过程能够提高数据库的性能，因为它可以在执行前减少解析和编译的时间。
