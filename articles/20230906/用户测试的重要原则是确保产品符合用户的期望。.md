# 1.简介
  

## 为什么要做用户测试？
用户体验一直是互联网应用开发中不可或缺的一环。用户体验不好，可能造成用户流失、用户黏性减弱等负面影响；用户体验好，可能吸引更多用户。如何提升产品的用户体验，是一个值得思考的问题。我们总结了一下用户体验的五个关键点，以此作为提升产品用户体验的思路:

1. 清晰明确：产品界面设计清晰，易于理解，操作流程无误，让用户快速上手。
2. 美观大方：界面视觉效果好，符合用户审美要求，不破坏美感。
3. 功能实用：产品提供的所有功能都有实际应用价值。
4. 可靠及时：服务及时响应用户反馈，在一个时间段内解决所有问题。
5. 服务满意度：用户对产品提供的服务满意度高，有利于推广产品。

这些关键点是衡量一个应用的用户体验是否优秀的指标，而用户测试则是一种有效的方法来验证产品的用户体验是否达到这些标准。

## 用户测试方法论
用户测试的方法论主要分为如下几步：

1. 用户需求调研：收集目标受众（即用户群体）的需求，确定用户痛点和诉求，分析用户行为习惯，搜集用户评价。
2. 报告结构设计：制定测试报告的内容结构，根据测试目的合理安排各项任务。
3. 测试设计：制作测试用例，并填写问卷调查问卷。
4. 测试执行：根据测试计划进行测试，记录用户测试过程中的相关信息。
5. 结果分析：通过数据统计分析，提取数据特征，归纳总结用户的体验感受。

## 用户测试应当遵循的六个原则
- 清楚定义目标：测试目标应该清晰且可测量。比如，测试某个功能是否被激活、某个按钮是否可用、某个字段是否填充，等等。
- 选择恰当的受访者：选择恰当的受访者群体，既要覆盖大部分的真实用户，又要保证真实场景，避免抽样偏差。
- 关注具体细节：测试过程中需要关注细节，包括按钮大小、位置、颜色、语音提示、系统反馈等。
- 避免模棱两可：当用户给出多个意见时，可以询问用户的想法，并且加强试题设置，增加难度。
- 不断迭代完善：每次测试后都需要再次测试，反复优化产品，直到得到满意的用户体验。
- 及时调整策略：随着产品的更新迭代，测试策略也需要跟着改变，保持客观公正，不要盲目妄动。

## 用户测试的两种类型
### 黑盒测试与白盒测试
#### 黑盒测试（结构测试）
结构测试是最简单但也是最不准确的测试方式，它的测试对象只是测试人员所熟知的系统构架图，因此它无法检测到软件运行时的各种逻辑错误。对于系统复杂、体系庞大的软件系统，这种测试方式更适合检验系统设计的正确性，而不是检测用户使用的操作和使用体验。一般来说，结构测试侧重检查以下方面：

1. 组件和子系统之间的接口关系。
2. 数据结构的组织结构。
3. 组件间的通信协议。
4. 模块的初始化和启动顺序。

#### 白盒测试（功能测试）
白盒测试是一种完整的测试方案，它将整个软件或系统视为一个白盒子，通过检查其中的每一处代码来识别并定位出现的问题。相比之下，黑盒测试仅会检查那些系统内部的硬件、网络或者驱动程序上的错误。白盒测试侧重检查以下方面：

1. 检查业务逻辑和处理流程的代码编写。
2. 检查输入输出接口的正确性。
3. 对函数的参数和返回值进行检查。
4. 数据库查询语句的语法正确性。
5. 检查数据加密的安全性。

白盒测试可以帮助发现系统设计、编码、资源管理等方面的错误，以及防止系统运行过程中出现的各种漏洞。

### 混合测试
混合测试是指采用白盒测试的方式，在代码和外部环境之间添加适当的混杂因素，模拟用户的操作场景。比如，通过构造恶意攻击流量或劫持设备、植入病毒、监听设备信息等手段，对软件进行攻击和保护测试。混合测试的目的是全面地测试软件的功能、性能、稳定性和安全性，从而更好地满足软件的目标和要求。
