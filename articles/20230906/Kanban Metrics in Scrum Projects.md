# 1.简介
  

Scrum作为敏捷开发方法论的代表，一直以来都被认为是一个很好的软件开发流程。但是由于其反应速度快、迭代周期短、适用性广、管理复杂等特点，因此在实际应用中受到了很多人的质疑。不过随着时间的推移，Scrum越来越受到社区的欢迎，也被越来越多的人认可。在这种环境下，越来越多的人开始关注Scrum的一些指标，包括管理者需要关注的指标、开发者需要关注的指标以及其他人员关注的指标。其中最著名的就是看板(Kanban)了。本文将从理解Kanban到了解相关的Kanban指标。
# 2.什么是Kanban？
Kanban，中文译为看板，是日本于1943年提出的一种制造流程的方法。它通过一张大的看板，将工作项分成不同的列，并安排不同的工序进行流动，最终达到高效率的生产过程。20世纪70年代末期，日本的“Toyota Production System”（简称TPS）即采用了Kanban管理方法。它的核心理念是生产线上尽可能少地引入停机，以确保产品质量。因此，“看板”这个名字也是直接取自于此。
# 3.Kanban的定义
Kanban（看板）由伊藤忠信博士在1943年提出。他希望通过实践证明，将工作重点放在将生产中的工作效率提升上，而不是围绕生产准备产线上的空白时间，达到同样的效率，而后才谈得上提升生产能力。因此，他提出了以下几个方面的观点：

1. 管理者：要确立能够有效管理工作的目标；
2. 团队：要建立一个精益求精的团队，具有高度的协作精神；
3. 生产线：把工作放在各个工序之间分配和执行；
4. 流程：每一个步骤都必须经过严格的掌控；
5. 沟通：所有团队成员都要有效沟通，才能更好地完成任务。
总之，Kanban将生产过程的工作重心转向精益求精的管理，强调要提升工作效率，并且在各种环节上都要严格遵守流程要求。
# 4.Kanban的价值
Kanban主要有以下几个优点：

1. 提升生产力：减少上下游的沟通成本，提升工作的连续性及时性，让工作获得高效率；
2. 改善流程：改进流程、监控方式及工艺水平，优化工作流，使企业的效益得到提高；
3. 提升产品质量：将制品的可靠性和可测性纳入考量，确保产品的质量符合标准，提升客户满意度。
# 5.Kanban的原则
Kanban在实践中还提出了以下原则：

1. 可见性：流程中的各项信息都可以清晰可见；
2. 专注力：集中注意力，避免分散注意力；
3. 承诺制：团队成员对工作负责，接受约束，团队始终达成共识；
4. 执行力：始终坚持计划、执行、检查三个环节，保证结果准确无误；
5. 跟踪性：对每一个任务都要做详细的记录。
# 6.Kanban的衡量方式
Kanban中的重要指标：

1. WIP：Work In Process，正在进行的工作项数量，反映了生产效率；
2. Throughput：生产流程中每日或每周完成的任务数；
3. Cycle time：从需求进入研发周期到制品交付完毕所需的时间，反映了工作效率及工作的紧迫程度；
4. Cumulative Flow：从需求到产品交付总的工作流量；
5. Lead Time：从需求到制品交付完毕经历的时间，反映了工作流的顺畅程度及产品的及时性。
这些指标虽然不能全面地反映Kanban的生产效率，但却可以提供一个比较客观的判断。因此，Kanban项目管理人员应该把目光放得远一点，并且建立一套完整的管理体系。
