# 1.简介
  

在线数据隐私风险管理（Online Data Privacy Risk Management，ODPRM）是指通过对数据的收集、处理、存储、传输、共享和使用的过程及相关技术手段的监控、管理、评估、保护和报告等活动，以识别、防范、分析并减轻各种数据泄露、侵权、违反规则和非法活动对个人信息、组织资产的影响，提升数据安全、舆论引导、法治建设和社会公平力度，促进信息经济的合理流通和可持续发展。
基于互联网的日益发展，网络用户的数量和数据量呈现爆炸性增长态势，传统的机械、电子化和半自动化的工作模式正在向网络机器的“人机协作+自动化”转变。而信息安全意识和专业能力则成为实现网络信息系统的关键。“数据即操纵”，网络行为留下了极其丰富的价值信息。
随着网络的信息化程度不断提高、网络服务的日渐普及，网络数据对用户的生命健康、财产安全、健康和安全产生的巨大影响越来越受到重视。特别是在当今数字化的社会里，互联网上产生的海量数据带来了新的隐私和安全风险，如何有效地保障用户的个人隐私、企业的商业利益和国家的安全利益，是网络世界的重要课题之一。

# 2.基本概念术语说明
## 2.1 数据信息系统
数据信息系统，英文全称Data Information System，简称DIsys。它是指采用计算机技术，按照确定的技术标准和流程编制的、系统地整理、汇总、编制和管理的数据资料和信息的一系列完整的管理制度，包括管理、调控、处理、应用、交换、储存、分发和销售这些数据资料和信息，进行科学地研究、分析、综合地利用，从而改善、提高企业管理决策、资源配置、生产效率、生活质量、社会效益、经济效益以及环境保护的能力。
数据信息系统通常由四个部分组成：基础设施、业务规则、数据采集、数据处理与分析。

## 2.2 隐私风险
隐私风险，是指某些情况下由于个人信息暴露、滥用或被恶意使用而导致的后果。隐私风险既涉及个人隐私本身，也包括个人信息泄漏所造成的社会和经济损失。隐私风险可能影响个人权利、自由、人身安全甚至生命健康。隐私风险分为以下几类：

1. 个人信息泄露：指个人信息外泄，将个人信息从事物中抽取出来用于非法目的。个人信息泄露通常会伤害个人、损害他人、违背公共利益，严重时可能危及生命、财产安全或违反国际或地区法律法规。

2. 信息收集偏差：指在收集个人信息过程中存在个人选择偏差或主观认知偏差。个人选择偏差往往会导致缺乏足够的信息、错误地搜集个人信息，使得个人信息不准确、不全面，最终造成个人信息泄露。

3. 数据主体不受信任：指信息主体对自己提供的个人信息是否真实、准确、完整不予信任。例如，用户提供虚假或不完整的个人信息，或许某些企事业单位会因此损害自身利益或对其他公司造成损害。

4. 不正确处理：指个人信息遭到不必要或者错误的处理。例如，未经授权对个人信息进行收集、加工、存储、散布、传输、阅览或者删除，甚至未经授权就将个人信息出售给第三方。

5. 恶意使用：指攻击者试图通过对个人信息的收集、使用、保存、泄露、删除、篡改等行为，获取个人信息及其相关信息，如身份证号码、住址、联系方式、通信记录等，从而达到非法使用、窃取或破坏个人信息。

6. 跨部门、跨组织数据共享：指不同部门或组织之间存在数据共享的情形。例如，两个部门在同一事务中需要用到相同的员工信息，这种信息就会被多个部门共享，导致信息泄露、不一致、冗余，甚至造成经济损失。

## 2.3 ODPRM架构设计方法
### （一）ODPRM概述
在线数据隐私风险管理(ODPRM)是指通过对数据的收集、处理、存储、传输、共享和使用的过程及相关技术手段的监控、管理、评估、保护和报告等活动，以识别、防范、分析并减轻各种数据泄露、侵权、违反规则和非法活动对个人信息、组织资产的影响，提升数据安全、舆论引导、法治建设和社会公平力度，促进信息经济的合理流通和可持续发展。ODPRM是运用大数据技术、网络安全技术、机器学习技术、数据挖掘技术、模型训练技术、算法优化技术等多种新兴技术手段，将网络数据及其相关上下文信息（如时间、位置、上下文内容等）进行收集、分析、处理、存储、共享等数据管理、安全管理、风险管理、法规遵守和舆情引导等功能综合运用的一个方案。主要目标是构建基于人工智能和模式识别的隐私风险预测模型，并通过数据实时监控的方式发现隐私数据中的异常行为，根据预测结果做出相应的隐私控制措施，提供数据质量保障、安全保障、社会公平保障以及经济收益保障。

### （二）ODPRM主要任务
ODPRM主要任务如下：

1. 数据收集阶段：数据收集阶段就是采集各种数据，包括个人信息、行为数据、上下文数据等。该阶段的目的在于建立一个长期且全局的全景数据集，其中包含各种上下文信息、个体特征、行为习惯、社交网络结构、兴趣爱好、金融交易历史、购买习惯、常用软件及设备使用习惯、手机设置、敏感信息数据等。
2. 数据清洗阶段：数据清洗阶段将数据中的噪声、错误、缺失信息剔除掉，同时将收集的数据规范化、转换为统一格式。数据清洗阶段要遵循包括合规性检查、实体识别、数据去重、数据标记、数据匿名化等步骤。
3. 数据建模阶段：数据建模阶段将数据按照不同的维度进行分析，生成一些统计模型和数据挖掘模型，目的是找到隐私数据中的模式和特征。
4. 数据增强阶段：数据增强阶段通过对隐私数据进行各种手段的增强，让数据变得更加难以识别、预测和检测。
5. 数据应用阶段：数据应用阶段就是将模型部署到实际系统中，并通过相关工具或接口对外提供服务。
6. 数据管控阶段：数据管控阶段是通过制定数据保密协议、数据安全措施、数据流动限制、数据使用权限分配、数据质量审核等机制，来保证隐私数据在整个流程各环节中的保密性、安全性、可用性、合规性和完整性。
7. 政策法规规章制定阶段：政策法规制定阶段是为了落实数据隐私法律义务，贯彻落实《数据安全法》、《网络安全法》、《信息安全管理条例》、《互联网信息服务管理办法》等法律法规，对收集、使用、共享、保护、销售等行为进行指导、监督和管理，通过制定相关政策和法规，实现数据安全、个人信息保护、社会公平和公共利益之间的沟通合作。

### （三）ODPRM的优点

1. 提升数据安全：数据收集和使用过程中存在的隐私风险一般都可以通过ODPRM来提升，一方面通过收集、分析、监控、预警等方式来发现隐私数据中的异常行为，并根据预测结果采取对应的保护措施；另一方面，通过数据加密、数据孤岛隔离、数据脱敏等技术手段来保护个人信息，降低数据泄露的风险。

2. 促进信息经济的合理流通：在现代互联网环境下，个人隐私越来越成为经济利益所在，收集、使用和共享个人隐私数据的过程中，要充分考虑用户的利益，制定公平公正的规则，充分尊重用户的隐私权，保障个人信息的安全和隐私权。

3. 提升数据治理水平：在现代数据治理中，集成化的数据治理平台是数据治理的关键支撑，可以实现数据采集、数据加工、数据输出等多个环节的自动化、智能化、精细化。通过自动分析、预测隐私数据中的异常行为，提供精准、及时的隐私管理建议，助力数据资产的可靠性、完整性、时效性、稳定性和隐私保护水平的提高。

4. 保持信息公开透明：ODPRM的设计理念鼓励数据主体及关联方，开放数据信息，不仅无偿地分享数据，而且能够提供信息的全面、真实、透明、客观，也不会妨碍数据的去标识化、匿名化。

5. 降低运营成本和响应速度：通过集成化的数据治理平台，降低运营成本，让数据管理人员专注于数据安全、控制体系建设、法规遵守、审计检查等关键环节的管理，提升政策法规制定效率和效果。
