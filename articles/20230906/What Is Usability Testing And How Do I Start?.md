# 1.简介
  

Usability testing（易用性测试）是一项用来确定产品或系统的可用性、易理解性、易学习性、易用性、舒适性等多方面性能指标的过程。其目的在于发现产品或系统在不同用户群体和各种使用场景下的实际可用性，帮助企业更好地了解客户对产品或系统的满意程度。
Usability testing 一般包括以下几步：
- 需求分析和方案设计
- 用户研究
- 测试环境搭建
- 执行测试
- 数据分析和总结报告
- 产品及/或流程更新
通过执行一系列的usability testing流程，可以有效地提高产品或系统的可用性、易用性、易理解性、易学习性和舒适性。同时，它也能为公司的市场策略制定提供可靠的数据依据。因此，在企业中推行usability testing是一个十分重要的环节，能够为产品和服务的推广和销售做出更好的决策。
# 2.背景介绍
Usability testing 是一项至关重要的测试活动，也是一种产品质量保证的方法。当产品或者系统逐渐流行起来时，usability testing 将成为一个必不可少的重要环节。在互联网公司，采用usability testing 作为产品开发周期的一个阶段，是为了确保产品具有良好的用户体验，尤其是在那些有限的资源情况下，具有明显的商业价值。从而促进产品的持续开发，并避免出现低效率或滞后的产品。
通常来说，usability testing 的过程如下：
- 第一步是进行需求分析，明确测试对象和任务。
- 第二步是开展用户研究，收集测试用例和参与者信息。
- 第三步是选择合适的测试环境，制定测试计划，组织测试人员。
- 第四步就是执行测试了，运用不同的测试方法和工具，例如，焦点小组法、因果分析法、问题反馈法、访谈法等，收集用户的反馈信息。
- 第五步是数据分析和总结报告，整理测试结果，评估测试效果。
- 最后一步是根据测试结果进行产品及/或流程的更新，调整产品规格或流程，优化用户体验。
其中，第一步到第三步较为简单，关键是如何收集用户的反馈信息，对产品或系统进行真正有效的测试。通过正确的设计测试用例、选择恰当的测试方法和工具，结合目标受众的需求，有效地收集到用户的反馈信息。
# 3.基本概念术语说明
## 3.1 接受范围
定义：Usability test主要测试的对象是目标受众，即目标受众是那些将要或已经使用该产品或系统的人。Usability test的受众一般来自一定年龄段、专业领域、职业层级、文化水平等多方面因素。
## 3.2 启发法
定义：Usability test是启发法。“启发”是指通过观察或通过研究发现人们的错误行为、弱点、不满意之处，再从这些错误行为、弱点、不满意之处中找寻共同的模式或模式框架，从而获得针对特定问题的解决方案或创新产品的能力。
## 3.3 可用性
定义：可用性，即产品或系统的使用方法、效果、功能是否满足用户的需要、可预测性和稳定性。可用性不仅包括用户对产品或系统的感知能力、理解能力、操作技巧、认知水平、应用习惯等等，还包括产品或系统自身的健壮性、安全性、可用性、可扩展性、可维护性、兼容性等。
## 3.4 可理解性
定义：Usability testing 中所使用的词汇，可用性中的可理解性。可理解性主要包含两个方面：（1）产品或系统的功能、操作方式和界面是否容易被使用？（2）用户对于产品或系统的理解、定位和判断是否准确、清晰、一致？
## 3.5 可训练性
定义：Usability testing 中所使用的词汇，可用性中的可训练性。可训练性主要指的是产品或系统是否容易被用户学会使用、熟练掌握。
## 3.6 演示比赛
定义：Usability testing中所使用的词汇，演示比赛。演示比赛是一种产品交流的形式，是由一群用户来演示自己的想法，收集反馈意见，评审之后作出最终决策。演示比赛是比较有效的反馈渠道。
## 3.7 灵活性
定义：Usability testing 中所使用的词汇，可用性中的灵活性。灵活性主要表现在产品或系统的设置、布局、功能的自定义能力、可配置性等方面。
## 3.8 直观性
定义：Usability testing 中所使用的词汇，可用性中的直观性。直观性主要指的是产品或系统的操作方式是否让人觉得容易理解、上手。
## 3.9 完整性
定义：Usability testing 中所使用的词汇，可用性中的完整性。完整性表现为产品或系统的各个方面是否齐全、完备。
## 3.10 可用性
定义：可用性是指产品或系统的使用方法、效果、功能是否满足用户的需要、可预测性和稳定性。可用性不仅包括用户对产品或系统的感知能力、理解能力、操作技巧、认知水平、应用习惯等等，还包括产品或系统自身的健壮性、安全性、可用性、可扩展性、可维护性、兼容性等。
## 3.11 易用性
定义：Usability testing 中所使用的词汇，可用性中的易用性。易用性描述了一个人的产品或系统的可理解性、可训练性、灵活性、直观性、完整性等方面的能力。易用性强调的是用户能够轻松上手、快速完成工作的能力。
## 3.12 接口 usability testing
定义：Usability testing 方法中的接口 usability testing。接口 usability testing 是指通过模拟用户的操作，测试应用程序界面、功能和数据之间的相互作用是否符合预期，以检查用户在使用过程中是否遇到困难、反馈是否准确、报错提示是否明确等问题。
## 3.13 非功能性测试
定义：Usability testing 方法中的非功能性测试。非功能性测试是指不考虑用户特性，只关注产品的总体功能和性能的测试。
## 3.14 操作性测试
定义：Usability testing 方法中的操作性测试。操作性测试是指用来测试用户对产品或系统的基本操作，比如登录、注册、上传图片、保存信息等操作。
## 3.15 视觉性测试
定义：Usability testing 方法中的视觉性测试。视觉性测试是指测试用户对产品或系统的视觉效果，比如色彩、形状、排版、动画、翻转等。
## 3.16 用户故事
定义：User story是一种简单的小型描述，用来阐述用户对某个要求的看法、理解、期望和愿望。它通常由以下几个要素构成：“As a(n) X, I want Y so that Z”。
## 3.17 可承受能力
定义：Usability testing 中所使用的词汇，可用性中的可承受能力。可承受能力是指某产品或系统在某种情况下可能发生的损失。
## 3.18 易学性
定义：Usability testing 中所使用的词汇，可用性中的易学性。易学性指的是产品或系统是否容易学会使用、理解。
## 3.19 可用性测试
定义：Usability testing 是一项用来确定产品或系统的可用性、易理解性、易学习性、易用性、舒适性等多方面性能指标的过程。其目的是发现产品或系统在不同用户群体和各种使用场景下的实际可用性，帮助企业更好地了解客户对产品或系统的满意程度。
## 3.20 周密性
定义：Usability testing 中的周密性。周密性是指测试工作人员的时间管理。
## 3.21 冗余性
定义：Usability testing 中的冗余性。冗余性是指同一任务或流程测试中有很多重复的步骤或测试方法。
## 3.22 易混淆性
定义：Usability testing 中的易混淆性。易混淆性是指测试过程中存在多个操作路径和方式，且存在不明确的地方。
## 3.23 耐心性
定义：Usability testing 中的耐心性。耐心性是指测试人员对产品或系统的可用性、易用性、易学性、易理解性等方面的能力。
## 3.24 紧迫性
定义：Usability testing 中的紧迫性。紧迫性指由于时间、资源限制等原因，不能快速启动或必须暂缓进行的测试。
## 3.25 效率
定义：Usability testing 中的效率。效率是指测试工作的效率，衡量标准可以是测试人员每天花费的时间、处理文档的数量、每个测试用例的测试时间。
