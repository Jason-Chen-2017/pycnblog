# 1.简介
  

版本控制工具 Git 是当今最流行的分布式版本控制系统。Git 本身提供了一套完整的功能，包括本地分支、远程分支、版本回退等等，非常适合团队协作开发。本系列教程将从一个实际案例出发，带领大家了解并掌握 Git 的基本用法，逐步实现对代码的版本管理，提升工作效率。
学习 Git 需要具备一些基本的命令技巧，并且会使用至少一种编程语言，比如 Python 或 C++。如果你还不熟悉这些编程语言，可以先学习一下它们的语法。
如果你已经具备这些条件，那就正式开始吧！
# 2.基本概念术语说明
## 2.1 什么是版本控制？
版本控制（Version Control）就是在不同时间点记录代码文件的历史记录，而后再根据历史记录进行版本回滚或者更新，可以有效地避免重复造轮子、保证项目的完整性。在软件工程中，版本控制主要用于：

1. 对源代码文件进行备份，确保数据安全；

2. 提供多人同时开发同一代码库时的并行开发支持；

3. 可以记录每次修改的内容及其原因，方便查错和追溯；

4. 将代码的生命周期变成可测量的事件，方便代码审计。

## 2.2 Git 的优缺点
### 2.2.1 Git 的优点
- 免费、开源
Git 是目前最流行的分布式版本控制系统，它免费而且开源，意味着任何人都可以访问、修改、扩展它的代码，并且可以持续改进它的功能。由于 Git 的设计思想简单、易于理解，所以能够快速上手，甚至还有 GUI 工具来辅助使用。

- 分布式结构
Git 把整个代码仓库作为一个整体，而不同时刻的所有提交都被存储在本地磁盘上，没有中心服务器，因此速度快、占用的空间小。对于较大的项目，Git 的分布式结构也使得克隆操作很迅速，克隆一份远端代码仓库只需要几秒钟。

- 可靠性高
Git 使用 SHA-1 哈希值来对每个 commit 进行标识，它是一个独特且不可伪造的 ID ，并且通过树对象来记录目录结构和文件信息。所有的提交都是完全透明的，你永远可以把整个历史看清楚，因为每一个版本的代码都可以在任意时间点被获取到。

- 没有合并冲突
Git 的核心理念就是快乐的合作，任何时候代码库中的某个地方出现冲突都可以通过简单的暂存、重试或合并的方式解决。Git 知道如何处理任何种类的冲突，不会导致数据的丢失。

- 强大的分支模型
Git 有多个分支功能，允许你创建独立开发线路，例如 dev 和 master 分支，并随时将特性分支与主分支合并。 Git 支持多种工作流模式，如 GitHub Flow、Git Flow 和 GitLab Flow。

### 2.2.2 Git 的缺点
- 不适合直接修改已发布的代码
由于 Git 的设计初衷是为了追踪源码，而不是像 SVN 一样直接修改已发布的代码，所以 Git 比较擅长追踪代码的变化情况，但不适合直接修改已发布的代码。如果要修改代码，建议在修改之前创建一个新分支，然后再合并到 master 分支。

- 命令行界面
由于 Git 的分支功能，以及其他一些 Git 独有的机制，它的命令行接口可能不是那么直观。如果你不习惯命令行，那么使用图形化界面的图形客户端或 Git 友好的 IDE 会更加容易上手。

- 学习曲线陡峭
虽然 Git 在各方面都比传统的版本控制工具好很多，但是它还是有一定难度的。要想上手，首先需要掌握一定的命令技巧，如 add/commit/branch/merge/checkout 等，之后还要理解底层的原理，才能达到独立使用 Git 的水平。
