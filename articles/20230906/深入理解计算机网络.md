# 1.简介
  

## 什么是网络？
互联网是一系列通过通信线路连接起来的计算机网络系统。网络由各种计算机设备、硬件设施及服务构成，并以特定的协议和标准运行。这些计算机设备通过通信媒体共享信息、资源及应用软件。
## 为什么要学习计算机网络？
当今社会是一个高度复杂的系统，需要复杂的软硬件协同工作才能完成各项任务。而人们生活中产生的数据、音频、视频等，都需要通过网络传送到处去，所以学习计算机网络可以使我们更加了解这个世界，了解它的工作机制，从而提高我们的能力和知识水平。
## 网络分层结构
网络按照网络功能不同，将其分为7层——物理层（Physical Layer）、数据链路层（Data Link Layer）、网络层（Network Layer）、传输层（Transport Layer）、会话层（Session Layer）、表示层（Presentation Layer）、应用层（Application Layer）。
### 物理层（Physical Layer）
物理层负责实现网络节点之间的物理连接，主要包括两个作用：一是传输比特流，二是分配线路速率。物理层使用多种不同的物理信道如光缆、铜接电缆、双绞铜线等。
### 数据链路层（Data Link Layer）
数据链路层提供可靠的字节流服务，它建立在物理层之上，两台主机之间的数据传输必须经过该层。主要包含以下四个方面：

1. 封装成帧
数据链路层把网络层的数据报组装成一个个的帧。每一帧包括数据和必要的控制信息（如地址等），目的地为下一跳路由器的MAC地址。

2. 透明传输
数据链路层应能够自动检测出和纠正数据错误，即透明传输。

3. 差错控制
数据链路层采用循环冗余检验（CRC）或者其他错误检测技术进行差错控制，使得接收端能够检测到无效的数据包。

4. 流量控制
数据链路层通过滑动窗口协议对发送方的发送速率进行限制，保证了网络的拥塞控制。

### 网络层（Network Layer）
网络层提供尽力而为的IP寻址功能，并确定如何路由到达目标计算机。主要包括五个方面：

1. IP地址管理
网络层为每台计算机分配唯一的IP地址，同时还提供域名解析服务。

2. 路由选择
当数据报从源点传输到目的地时，网络层需要决定使用哪条路径。此时，网络层使用路由表或其他路由算法计算最佳路径。

3. 分段与重组
网络层对传输的数据进行分段和重新组装。

4. 服务质量（QoS）
网络层提供丰富的服务质量保证，如延迟保证、带宽保证、抖动减少、故障恢复等。

5. 拓扑结构
网络层需要处理复杂的拓扑结构。例如，当网络出现环路时，就需要动态地改变路由。

### 传输层（Transport Layer）
传输层为上层的应用程序提供了端到端的可靠数据传输服务。主要包括六个方面：

1. 进程间通信
传输层支持多种类型的进程间通信，如TCP/UDP。

2. 可靠传输
传输层提供可靠的数据传输服务，确保应用进程之间的通信不被破坏。

3. 多路复用与多路分解
当多个应用程序进程需要访问相同的网络时，传输层需要根据应用需求，进行流控和分用。

4. 差错控制
传输层采用滑动窗口协议进行差错控制，确保发送端发送的字节流能够按序到达。

5. 连接管理
传输层负责建立和维护网络连接，包括三次握手和四次挥手。

6. 伪装与加密
传输层提供对数据的加密和解密服务，防止数据泄露。

### 会话层（Session Layer）
会话层提供包括安全性、同步、通知和数据完整性的功能。主要包括三个方面：

1. 运输实体标识
会话层定义网络中的实体标识符，用于区别不同的会话。

2. 事务控制
会话层提供事务状态管理功能，确保应用进程之间的交互正确性。

3. 对话管理
会话层管理包括IPSec、SSL、VPN、RADIUS、TACACS+等多种安全功能。

### 表示层（Presentation Layer）
表示层提供了数据格式化、压缩、加密等功能。主要包括五个方面：

1. 数据格式化
表示层定义数据格式，使得不同应用进程之间的数据交换变得简单易行。

2. 数据压缩
表示层对数据进行压缩，降低传输数据量。

3. 数据加密
表示层对数据进行加密，防止数据被窃听、篡改。

4. 数据描述语言
表示层定义数据描述语言，用于定义数据语法和语义。

5. 数据转换
表示层允许不同数据格式之间的转换。

### 应用层（Application Layer）
应用层为用户提供应用程序接口。主要包括十几个方面：

1. 文件传输
应用层的FTP、TFTP等协议用于文件传输。

2. 虚拟终端
应用层的SSH、Telnet等协议用于远程登录。

3. 电子邮件
应用层的SMTP、POP3、IMAP等协议用于收发电子邮件。

4. 消息传递
应用层的NNTP、XMPP、IRC等协议用于消息传递。

5. 数据库访问
应用层的SQL、LDAP、Kerberos等协议用于数据库访问。

6. 即时通信
应用层的IRC、Jabber、Gtalk等协议用于即时通信。

7. Web浏览
应用层的HTTP、HTTPS、WebDAV等协议用于Web浏览器访问。

8. 名称解析
应用层的DNS、NIS、SMB等协议用于域名解析。

9. 目录服务
应用层的LDAP、AD等协议用于分布式目录服务。

10. 内容分发
应用层的RMI、CORBA等协议用于远程过程调用。
