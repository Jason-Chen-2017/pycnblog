# 2. 数据模型
## 2.1 数据类型
信息检索一般分为两类，一类是针对文档或文本的检索，另一类是针对数据库的检索。下面我们逐一探讨。
### 文档检索（Document Retrieval）
文档检索又称为信息检索，它主要面向的就是存储在计算机中的文档或者文本文件，根据特定的查询词、关键词、短语或者主题，找到相关的文件。由于文档数量庞大，搜索起来耗时耗力，所以文档检索主要采用布尔模型。布尔模型表示为：文档d与查询q的相关系数（coefficient of relevance），即log(P(q|d)/P(q))，其中P(q|d)表示文档d中包含查询词q的概率，P(q)表示文档库中所有文档的总体概率。在布尔模型下，文档检索系统的性能受限于系统硬件的限制，并且往往无法精确给出搜索结果的准确位置。

### 数据库检索（Database Retrieval）
数据库检索又称为关系型数据库搜索，它是在一组数据库表内查找数据的过程。数据库检索根据用户输入的查询条件，在数据库表内查找符合要求的记录。数据库检utar是基于SQL语句查询的一种信息检索方式。数据库检索根据用户需求指定筛选条件，然后把数据记录按照指定的顺序排列好，再通过比较不同字段的数据项，找出满足条件的记录。数据库检索是建立在关系型数据库表结构基础上的信息检索方法。由于数据库中记录是结构化的，可以直接按照索引进行查找，速度较快；而且数据库记录的结构比较固定，通常具有相关的属性，字段值大小也比较一致，因此数据库检索系统查询效率很高。但是，由于数据量大、复杂、多样，数据库检索系统的准确性较差，不一定能找出用户想要的信息。

## 2.2 数据模型
信息检索系统可以采用的三个数据模型，分别为集合模型、网状模型和层次模型。其中集合模型是最简单的一种数据模型，每个文档或者其他对象都是一个元素，集合中所有的元素构成了整个数据集合。在集合模型中，任意两个元素之间的相似度是由统计学上的相关系数衡量的，相关系数的范围是-1~1之间，值为1表示完全相关，-1表示负相关，0表示无关。在集合模型中，文档之间没有任何联系，只能依赖文档集内已知的信息进行相似度计算。在这种情况下，任何一个文档的相似度计算都只能依赖于整个文档集。

网状模型则是另一种数据模型，它将文档集抽象成互相关联的网络结构。在网状模型中，文档之间存在着层次结构，不同级别的文档之间也存在着相关性。层级越高，相关性就越强，直至两个文档处于同一层级为止。在这种数据模型中，文档的相似度计算更加依赖于邻居节点，而不是整个文档集。

最后，层次模型也是一种常见的信息检索数据模型。在层次模型中，文档被组织成树形结构，树的根节点代表的是整个文档集，树枝代表不同的主题，叶子结点代表具体的文档。每一层次之间都是彼此相关的，只有当前层次的文档才知道它的上下文。层次模型可以有效地避免冗余数据，降低内存占用，提升查询效率。同时，层次模型还可以反映出文档集的内部结构，能够发现隐藏的模式和特征。

综上所述，在目前信息检索领域，三种常见的数据模型为：

1. 集合模型：适用于小型文档集合。
2. 网状模型：适用于广义文档集合。
3. 层次模型：适用于文档集合中存在主题结构。
