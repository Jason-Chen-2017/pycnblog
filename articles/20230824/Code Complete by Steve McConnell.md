# 1.简介
  

Code Complete（中文名：代码完善）一书是微软公司程序员必读之书。它从C++、Java、Basic等多个编程语言入手，分别详细阐述了面向对象编程、软件设计、编码规范和单元测试等内容。在书中，作者给出了具体的代码编写建议、架构设计模式及工具使用方法，并将这些内容编制成了一套完整的软件开发流程和 checklist，帮助程序员有效地提升编码水平，提高代码质量，解决日益复杂的软件开发问题。Code Complete也是一本具有实用价值的参考书，能够对各种编程语言及框架进行比较深入的分析，并给出一些建议，对程序员的职业生涯产生重大影响。

# 2. 基础知识
## 2.1 对象-oriented programming
在Code Complete中，首先介绍了面向对象编程(Object-Oriented Programming)的一些重要概念和术语。

2.1.1 Class（类）: 类的定义通常包含数据成员（data members）、函数成员（function members），还可以包括构造函数和析构函数。一个类的所有对象都拥有相同的属性和行为。类定义可以包含基类或接口，也可以继承其他类中的属性和函数。通过类，可以创建多个具有相同功能或特性的对象，并实现代码重用、代码共享、灵活性和可维护性。

2.1.2 Object（对象）: 对象是一个运行时实体，由类的实例化得到，每个对象都有一个唯一的标识符。对象是一个独立的实体，其状态由对象的各个数据成员的值决定，可以通过调用其成员函数来修改它的行为。对象可以作为参数传递给函数、存储在容器中、被返回作为函数的结果、动态地建立连接到其他对象上。

2.1.3 Encapsulation（封装）: 封装是指把信息隐藏在对象内部，使得该对象只能通过特定的接口访问其内部状态或行为。对象内部的数据可以被保护起来，只允许对外通过特定的接口进行访问；对于数据的修改只能通过特定的接口来完成。封装可以提高模块化程度、代码复用率、并行性和健壮性。

2.1.4 Inheritance（继承）: 继承是一种机制，用于创建新类，新的类从已存在的类中派生而来。新的类可以使用基类的所有属性和方法，同时还可以新增自己的属性和方法。继承可以提高代码的可重用性、降低耦合性、节省资源、提高代码的组织结构和可维护性。

2.1.5 Polymorphism（多态）: 多态是指不同的对象对同一消息作出不同的响应。多态意味着可以在运行时改变对象的类型，使得程序可以根据情况选择合适的方法执行相应的任务。多态可以让程序更加灵活、可扩展、可维护。

2.1.6 Abstraction（抽象）: 抽象是指仅关注对象所需的方面，忽略其余的细节。抽象能帮助我们更好地了解系统的整体结构和运行原理，并提高我们的理解和效率。

## 2.2 Software design
2.2.1 Module decomposition （模块分解）: 模块分解是一种软件工程方法论，它要求将复杂的软件系统划分为互相独立的模块，每个模块都应该只有少量的代码做主要工作。模块分解有助于降低模块间的耦合度，提高模块的独立性、可测试性和可移植性。

2.2.2 Design patterns (设计模式): 设计模式是软件工程领域的一个重要概念，它提供了经过验证的可重用方案，帮助开发人员快速构建应用软件。设计模式包括创建型模式、结构型模式、行为型模式和组合型模式四种类型。

2.2.3 Quality Assurance（质量保证）: 质量保证是指软件工程过程中，对软件的质量进行审查、检测和改进的一系列活动。它包括需求分析、设计、编码、测试和部署等环节。通过良好的设计和测试，保证软件的正确性、可用性和性能。

## 2.3 Coding guidelines and standards
2.3.1 Coding style guidelines （编码风格指南）: 编码风格指南是用来统一整个项目的编码规范。它包含命名规范、注释规范、缩进方式、变量命名规则、语句布局规则、语法约定等内容。每一位程序员都应该遵守编码风格指南，从而可以让代码保持一致、易于阅读和理解。

2.3.2 Code reviews （代码审查）: 代码审查是为了发现代码中的错误、漏洞、性能问题、可靠性问题、安全问题、兼容性问题、界面美观度问题等潜在问题。通过代码审查，可以提高软件质量，找出代码中可能存在的问题，并提供宝贵的反馈。

2.3.3 Unit testing （单元测试）: 单元测试是软件工程领域的一个重要技术，它是指针对程序模块（比如函数或者过程）独立的、自动化的测试。单元测试的目标是保证每个模块的正确性，检查输入输出的组合是否能够正常工作。

## 2.4 Tools usage 
2.4.1 Integrated development environment (IDE): IDE是集成开发环境，可以提供编辑器、编译器、调试器、版本控制、单元测试等功能。不同的IDE配备了不同的插件、模板，可以满足不同编程语言的开发需要。

2.4.2 Version control system (VCS): VCS记录软件的历史变动，可以帮助开发人员追踪代码的最新版本和修改记录，并可以帮助团队协作开发。目前，主流的VCS有Git、SVN和Mercurial。

2.4.3 Build automation tools (自动构建工具): 自动构建工具是在编译之前将源文件、依赖项、设置等生成最终可执行文件的工具。它可以提高编译效率、提前发现错误、减少手动操作，降低软件开发的复杂度。目前，主流的自动构建工具有Make、Ant、Maven和Gradle。

2.4.4 Debugging tools (调试工具): 调试工具可以帮助开发者定位软件错误、崩溃、卡顿等问题。通过调试工具，可以方便地逐步分析程序的运行状态、跟踪程序的执行路径，以及查找内存泄露、死锁等问题。

2.4.5 Performance analysis tools (性能分析工具): 性能分析工具可以用于分析应用程序的性能瓶颈，帮助优化应用程序的运行速度。它可以统计应用程序的运行时间、CPU占用率、内存占用率、网络带宽占用率、磁盘IO占用率等指标。
