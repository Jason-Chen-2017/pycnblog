# 1.简介
  

## DBSCAN算法简介
DBSCAN(Density-Based Spatial Clustering of Applications with Noise)算法是一种基于密度的空间聚类算法，它可以自动地将相似对象归类到同一个簇中，并且对异常值也不敏感。
DBSCAN算法基于这样的假设：如果一个点的领域内存在足够多的点，那么该点就被划分到一个区域中；而那些孤立的点则被看作是噪声或者是需要进行特殊处理。
## DBSCAN算法步骤
1. 确定扫描半径eps（epsilon），即两个邻近点之间的最大距离，一般取样本数据集的平均距离加上一些偏差，如0.01倍标准差或0.5倍样本方差。
2. 从样本集中任选一个样本点，计算它的领域范围（即样本点到半径eps内的点）并标记其邻居。若领域内没有其他点，则称这个点为孤立点。
3. 对所有邻居标记点，递归执行步骤2。
4. 直到所有的点都已经得到了标记，共分为若干个核心点、边界点和噪声点三类。
5. 对每个核心点，根据样本空间的距离定义距离同一个核心点的样本点作为邻居的条件，将邻居分到同一个类别。
6. 在确定核心点的同时，迭代过程会发现新连接的核心点，将其加入队列等待处理。
7. 一旦队列为空，或者未出现新的连接的核心点，则停止迭代。

## DBSCAN算法特点
1. 高效：DBSCAN算法的运行时间复杂度为O(n^2)，主要原因是采用了渐进增长的方式，通过剔除不是核心点、边界点和噪声点分类后的数据量减少，所以可以较快地找到聚类中心，并对异常值不敏感。
2. 可扩展性：DBSCAN算法可以在任意维度上运行，适用于高维空间中的数据集。
3. 灵活性：DBSCAN算法允许用户指定要查找的簇个数K，并且能够对不同数据集的效果进行比较。
4. 任意形状的核：DBSCAN算法允许用户提供自己的核函数，用于计算两个点之间的距离。

## DBSCAN算法缺陷
1. 局部线性：由于算法采用密度的定义方式，所以对非球形结构的点密度估计较差，因此可能出现许多较小但密集的簇，这可能会影响模型的结果。
2. 方向性：DBSCAN算法是无监督学习方法，因此无法考虑到样本点之间的相关性。
3. 数据依赖：DBSCAN算法要求数据的局部结构相对稳定，如果遇到局部改变的数据（如波动的环境），就会产生不可预料的结果。
