# 1.背景介绍


## 概述
随着移动互联网、云计算等新型互联网应用模式的兴起，互联网终端设备数量呈爆炸式增长。一方面，随之而来的网络带宽、速率急剧提升，增加了服务质量的要求；另一方面，设备类型、功能越来越复杂，网络拓扑也日渐复杂，越来越难以手动部署和管理。因此，网络架构设计应力求简单易用、高效可靠、弹性扩展，符合可维护性和可扩展性。
SDN（Software Defined Network）是一种新的网络架构模式，旨在通过软件定义的方式管理网络，实现网络功能的集中化、分布式、模块化和虚拟化，使得网络架构具备高度灵活性、自动化程度高、成本低、可靠性强等特点。通过部署控制器、交换机、路由器等网络硬件设备的软件定义，用户可以直观地描述网络结构并自动配置其各个组件。由于软件定义的特性，SDN 提供了强大的网络自动化能力、网络可靠性和服务质量保障，能有效降低运维成本、提升网络的可管理性、可扩展性。
基于 SDN 的网络架构设计及其关键技术均具有极高的实用价值。SDN 能够满足快速响应业务需求，提升网络安全性和可用性，实现网络自动化、智能化和自动伸缩，大幅度减少人工管理网络所耗费的时间和成本。在许多应用领域都已经得到广泛应用。其中最具代表性的是边缘计算领域，如物联网、智慧城市、边缘网络和边缘云等，还有车联网、5G 通信等其他领域。

## 优势
### 简单易用
SDN 架构可以提供高效的网络体系结构，并通过统一的控制平面管理各种网络资源和流量，使得网络自动化、智能化和自动伸缩成为可能。这种架构对运维人员来说更加友好，不再需要担心繁琐的网络管理和配置任务，只需关注网络中的应用层流量即可。
在物联网、边缘计算等应用场景下，SDN 可以实现端到端的数据处理，通过协议栈可编程，可以在边缘端直接访问微服务或本地应用程序，提升边缘计算性能和实时性。同时，SDN 在智能调度、弹性分配、QoS 等方面还可以提供超高的灵活性，可以根据不同应用的需求进行精细化调度和策略调整，极大地优化网络利用效率。

### 可靠性
SDN 架构将网络功能模块化，实现网络功能的集中化和分布式部署，每个网络节点可以独立运行，确保了 SDN 架构的可靠性。通过 SDN 可编程的路由器、交换机等网络设备，用户可以设置合理的配置规则，避免硬件故障导致的连通性问题。
另外，SDN 还可以通过冗余机制、流控机制等方式，保证网络的高可用性。即使单台设备出现故障，整个网络依然可以正常工作。在此基础上，可以搭建多层级的防护措施，如分级制的 ACL（Access Control List），支持 ACL 匹配优先级设置。

### 高效可伸缩
基于 SDN 的网络架构可以非常容易地部署和扩展，并且有很高的弹性。这意味着可以快速、方便地添加或移除网络设备，满足网络的动态变化。在智能调度、流量控制、负载均衡等方面，SDN 架构可以提供较高的效率和性能。
例如，通过统一的控制平面，SDN 架构可以实现弹性伸缩，自动识别网络中的设备状态异常，并迅速做出反应，保证网络的高可用性。
