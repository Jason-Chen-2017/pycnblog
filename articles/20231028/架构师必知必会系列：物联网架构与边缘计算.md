# 1.背景介绍


## 什么是物联网？
物联网（IoT）是由各种嵌入式设备、传感器、智能设备等通过互联网连接到一起的网络。智能设备之间可以共享信息，进行数据交换，实现远程控制、自动化、人机交互等功能。其中，嵌入式系统尤其具有成本优势，可快速部署，可满足各类需求，并应用于不同的领域。物联网的兴起促使产业界、政府部门、企业及创客们围绕这一新兴技术不断投入研发、设计、生产、应用，创造了许多创新产品、服务及生态圈。

## 边缘计算简介
边缘计算就是一种在本地处理数据的能力，特别适合处理那些对性能要求苛刻、延迟敏感、安全性要求较高的数据处理场景。而这种处理数据的能力又可以通过云端平台进行大规模集中存储、分析、处理。边缘计算被认为是物联网发展的一个重要方向，它将智能设备与云端数据中心结合，形成全新的计算模式。边缘计算包括如下五个主要组成部分：
- IoT终端设备：通过标准协议与边缘服务器建立连接，收集传感器数据并上报到云端。
- 数据中心节点：提供云端计算、存储和分析资源。
- 消息中间件：用来支持IoT终端设备之间的通信，例如MQTT、AMQP等协议。
- 边缘计算框架：基于消息中间件，开发用于处理数据的应用程序。
- 数据库：用于存储边缘计算结果、元数据等数据。

边缘计算已经成为物联网领域的一大热点。如何让智能设备能够承载边缘计算的处理任务，是一个重要研究课题。目前，研究人员已经提出了一整套架构来帮助企业管理复杂的边缘计算环境，并且推动该领域技术的成熟、应用和发展。如图所示，边缘计算架构主要包括三个层次。第一层，硬件，包括了设备制造商、技术公司以及其他的硬件供应商，这些硬件能够在任何需要的时候提供计算资源。第二层，消息传输，包括了一整套的消息传输机制，包括基于TCP/IP协议的互联网传输协议，比如HTTP、HTTPS、WebSockets等；基于行业通讯协议的内部通信机制，比如MQTT、CoAP等；以及边缘网关、路由器等基础设施。第三层，编程接口，提供了不同语言的接口，包括用于开发应用程序的SDK、API、CLI等。


# 2.核心概念与联系
## 2.1 物联网与边缘计算
物联网（IoT）和边缘计算是两个非常相关的领域。
### （1）物联网
物联网包含智能设备和终端设备、互联网、云端、设备传感器等构成。智能设备可以是传感器、智能电器、工控设备、机器人、垃圾收集器、摄像头等。终端设备可以是PC、手机、Pad、电视等。互联网连接着各种设备，包括终端设备和智能设备。云端集中存储、分析、处理数据。云端可以是互联网上的一个或多个服务器，也可以是自己购买的私有云或者公有云。设备传感器可以是温度计、湿度计、照相机、雷达、震动传感器、光敏电阻传感器、GPS模块等。

### （2）边缘计算
边缘计算包含五个关键要素：终端设备、数据中心节点、消息中间件、边缘计算框架、数据库。
#### 终端设备
终端设备通常包含微控制器、单板计算机、工控机械臂、智能手机、车载传感器设备等。它们通过标准协议与边缘服务器建立连接，收集传感器数据并上报到云端。
#### 数据中心节点
数据中心节点一般部署在本地，负责处理业务逻辑和存储数据。
#### 消息中间件
消息中间件用于支持终端设备之间的通信。消息中间件可以分为两种类型：
- 一类是面向终端设备的消息中间件，比如MQTT、CoAP等协议，用于通信终端设备之间的消息。
- 一类是面向云端的消息中间件，比如Apache Kafka、RabbitMQ等协议，用于在数据中心节点之间传递数据。
#### 边缘计算框架
边缘计算框架是基于消息中间件的应用程序开发工具包，用于处理数据。边缘计算框架包括三个部分：
- 数据采集模块：用于从终端设备收集数据。
- 规则引擎模块：用于基于数据生成业务事件。
- 数据分析模块：用于分析数据并做出业务决策。
#### 数据库
数据库负责存储边缘计算结果、元数据等数据。

### （3）联系与区别
物联网由智能设备、终端设备、云端、设备传感器等构成，它们通过互联网连接，实现数据的共享、远程控制、自动化、人机交互等功能。物联网的目标是打通整个产业链，实现价值密度最大化。边缘计算是物联网发展的一个重要方向，它将智能设备与云端数据中心结合，形成全新的计算模式。边缘计算的主要目的是实现在本地解决一些特殊的计算问题，比如对实时性要求苛刻的机器学习任务。物联网与边缘计算虽然有重叠，但还是有区别的。物联网关注的是整个产业链，如设备采集、数据共享、数据分析、决策等，是建立在“智能化”、“统一管理”、“智能协同”等理念之上的产业网络。而边缘计算关注的是“一带一路”，只关注局部区域的资源，是为了提升效率、降低成本、节省能源等目的的针对特定领域的计算方案。因此，边缘计算是建立在物联网之上的一种计算方案。
