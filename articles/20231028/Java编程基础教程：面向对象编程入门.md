# 2.核心概念与联系
## 2.1 什么是面向对象？
面向对象（Object-Oriented Programming，简称OOP）是一种计算机编程的方法论，是一种基于数据抽象、继承、多态等概念，将构成问题空间的对象作为程式设计的基本单元，从而将整个问题解决方案映射到对象上进行处理的编程方法。它允许对复杂的问题域进行分割，更好地关注问题的各个方面，更有效地组织解决问题的方式。一般来说，OOP通过封装、继承、多态以及其他特性实现了代码重用、可扩展性、灵活性和可维护性。面向对象方法论还有另外两个重要特征，即抽象（Abstraction）和继承（Inheritance），后者又称为组合（Composition）。

## 2.2 面向对象模型及关键术语
### 2.2.1 对象（Object）
对象是一个有状态和行为的实例，其状态指的是对象存在于内存中时所拥有的各种数据值，行为指的是对象可以执行的一系列动作。对象之间的交互方式和信息共享也遵循四个基本原则：封装、继承、多态和迪米特法则（最少知识）。

### 2.2.2 属性（Attribute）
属性是对象内部特征的表示，通常包括数据成员变量（Data Member Variable）和操作成员函数（Operation Member Function）。属性既可以直接访问，也可以通过访问器函数间接访问。属性包括数据类型、名称、初始值、访问权限控制等。

### 2.2.3 方法（Method）
方法是对象对外提供服务的接口或功能，它由函数或过程来实现。方法通常包括接受的参数列表、返回值类型、访问权限控制等。方法可以分为普通方法、静态方法、私有方法、保护方法、构造方法等。

### 2.2.4 类的定义
类是创建对象的蓝图，是对具有相同属性和方法的对象的集合。类的声明语法如下：
```java
public class ClassName {
   // property declaration and methods implementation
}
```
其中，public是访问权限修饰符，ClassName是类名，它应当采用驼峰命名法（首字母大写，多个单词之间没有下划线）。property声明语句用于定义类的属性，比如字段（field）、变量（variable）等；方法声明语句用于定义类的行为，比如方法（method）、函数（function）等。

### 2.2.5 类的对象（Instance）
对象是类的一个实例化结果，也就是说，每创建一个类实例，就会生成一个对象。对象是类的实例化结果，当程序运行时，虚拟机会为每个对象分配一块独立的内存空间，该内存空间用来存放对象中的所有数据和方法。

### 2.2.6 抽象类（Abstract Class）
抽象类是不能够实例化的类，只能作为父类被继承。抽象类可以包含属性、方法和抽象方法。抽象方法是没有实现的虚方法，它在子类中需要被实现。抽象类的目的是为了给派生类提供接口或共同的功能，而不是用来创建对象。抽象类的关键字是abstract。

### 2.2.7 接口（Interface）
接口是另一种抽象机制，它定义了一组抽象方法，但是不提供方法的实现。接口不能包含属性，因为属性是由类的实例来提供的。接口的作用类似于抽象类，但它们有以下不同点：
* 接口仅定义方法签名，不包含方法体；
* 接口不能包含属性，只提供方法签名，因此无法实例化；
* 接口可以多继承，一个类可以实现多个接口；
* 接口的实例可以赋值给任何引用变量，并且可以调用接口中定义的所有方法。

### 2.2.8 多态（Polymorphism）
多态（Polymorphism）是指具有不同形态的同一个接口的对象可以接受不同的消息，并以不同的方式响应这些消息。在面向对象编程中，多态是指允许不同类型的对象对同一消息做出不同的反应。多态使得代码更加灵活、模块化、可靠，具有很强的适应性和可维护性。多态的实现方式主要有三种：
* 显式多态（Explicit Polymorphism）：通过参数、基类指针或者引用、虚函数等方式实现；
* 隐式多态（Implicit Polymorphism）：通过编译器或者解释器自动进行转换，如重载、类型转换、模板方法等；
* 演算多态（Ad-hoc Polymorphism）：利用运算符重载、表达式模板、工厂模式等方式实现。
