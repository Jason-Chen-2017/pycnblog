# 1.背景介绍


## 什么是字典？
字典(dictionary)是一个由键值对组成的数据类型。在Python中，字典是一种无序的集合。字典中的每个键值对用冒号分隔开，每个键都是唯一的，值可以重复。字典中的键值对可以通过索引、键或值进行访问。字典是一个可变的对象，它可以随时添加或者删除键值对。
## 为什么要使用字典？
字典是一种非常灵活且实用的数据结构。当需要存储和管理大量数据时，用字典就能非常高效地处理。比如，我们需要存储学生信息、电话号码等，就可以用字典将名字作为键，其余信息作为值存储起来。通过名字直接获取到完整的个人信息。如果没有用字典的话，可能要用多重列表或者嵌套列表，而且难以维护。
## 如何创建字典？
创建字典有两种方式：

1. 通过关键字参数形式：`d = {'name': 'John', 'age': 30}`。注意：关键字不能重复，否则最后一个会覆盖之前的。

2. 使用dict()函数：`d = dict([('name', 'John'), ('age', 30)])`。这种方法也支持使用元组作为键。

也可以通过如下方式修改字典：
```python
d['name'] = 'Mike' # 修改key对应的value
del d['age'] # 删除某个key-value对
```
## 如何判断字典是否为空？
判断字典是否为空，只需检查字典是否为空即可。即`if not d:`。

## 什么是集合？
集合（set）类似于数学上的集合，也是不允许重复元素的集合。集合是用花括号{}定义的，元素之间用逗号隔开。基本上所有元素都在内存里。但是，由于历史原因，集合并不是真正的集合，而是一些独特的运算规则，用来表示数学意义上的集合，集合运算就是利用这些运算规则实现的。

集合是一种数据结构，用于存储元素的集合。它具有以下几个主要的特征：

1. 不重复性：集合中的元素不允许重复出现。

2. 无序性：集合中的元素不按任何特定顺序排列。

3. 没有索引：集合没有下标，因此无法直接访问集合中的元素。

集合中的元素只能通过迭代器来访问，不能通过索引来访问。所以，集合不适合于数字元素的精确查找。除此之外，集合还提供了集合操作的一些基本的算术运算符，如求交集、并集、差集等。

一般来说，集合的操作速度比列表、字典快很多。但是，对于不需要高速随机访问的应用场景，集合通常更加合适。

集合的特点包括以下几点：

1. 集合是一种无序的元素集合，它的元素不能重复。

2. 集合中的元素可以是不同类型的对象，且可以是任意对象，而列表、字典则仅限于特定类型。

3. 集合中的元素是无序的，不能通过位置访问集合中的元素，只能通过迭代器访问。

4. 集合支持集合操作，如求交集、并集、差集等。

集合与列表、字典的相同点：

1. 集合的元素无序，不可索引。

2. 集合支持集合运算。

3. 创建集合的方法与列表、字典相同。

集合与字典的区别：

1. 字典是无序的，键值对的顺序不固定；而集合是有序的，元素不可改变。

2. 字典的值可以是任意的对象，而集合只能存放可哈希对象的集合。
