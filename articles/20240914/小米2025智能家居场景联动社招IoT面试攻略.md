### 1. 家居场景下的IoT设备通信协议有哪些？

**答案：**
常见的家居场景下的IoT设备通信协议包括：
- **Wi-Fi：** 无线通信，覆盖范围广，数据传输速度快。
- **ZigBee：** 低功耗，短距离通信，适合智能家居设备之间的互联。
- **Z-Wave：** 同样用于短距离通信，功耗较低，但传输速度较Wi-Fi慢。
- **蓝牙：** 低功耗，短距离通信，适用于手机与智能设备的连接。
- **LoRa：** 长距离通信，低功耗，适用于远程监控等场景。

**答案：**

### 4. 请实现一个简单的智能家居系统，包括传感器、控制器和执行器。

**答案：**
实现一个简单的智能家居系统，可以按照以下步骤进行：

```go
package main

import (
	"fmt"
	"time"
)

// Sensor 传感器
type Sensor struct {
	temperature float64
	humidity     float64
}

// Controller 控制器
type Controller struct {
	heater    *Heater
	airConditioner *AirConditioner
}

// Heater 暖气
type Heater struct {
	on bool
}

// AirConditioner 空调
type AirConditioner struct {
	on bool
}

// ReadSensor 读取传感器数据
func (s *Sensor) ReadSensor() {
	// 模拟传感器数据读取
	s.temperature = 25.5
	s.humidity = 60.5
}

// SetHeaterControl 设置暖气控制
func (c *Controller) SetHeaterControl(on bool) {
	c.heater.on = on
	if on {
		fmt.Println("暖气开启")
	} else {
		fmt.Println("暖气关闭")
	}
}

// SetAirConditionerControl 设置空调控制
func (c *Controller) SetAirConditionerControl(on bool) {
	c.airConditioner.on = on
	if on {
		fmt.Println("空调开启")
	} else {
		fmt.Println("空调关闭")
	}
}

func main() {
	// 初始化控制器
	controller := Controller{
		heater:    &Heater{on: false},
		airConditioner: &AirConditioner{on: false},
	}

	// 读取传感器数据
	sensor := Sensor{}
	sensor.ReadSensor()
	fmt.Printf("当前温度：%.2f°C，湿度：%.2f%%\n", sensor.temperature, sensor.humidity)

	// 设置控制器状态
	controller.SetHeaterControl(true)
	time.Sleep(5 * time.Second)
	controller.SetHeaterControl(false)

	controller.SetAirConditionerControl(true)
	time.Sleep(5 * time.Second)
	controller.SetAirConditionerControl(false)
}
```

**解析：**
这个简单的智能家居系统包括一个传感器、一个控制器和两个执行器（暖气和空调）。传感器负责读取环境数据，控制器控制执行器的开关状态，并输出相应的控制信息。

### 5. 请实现一个智能家居系统的数据库设计。

**答案：**
智能家居系统的数据库设计可以考虑以下表结构：

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(50) NOT NULL
);

CREATE TABLE sensors (
    id INT PRIMARY KEY AUTO_INCREMENT,
    type VARCHAR(50) NOT NULL,
    value VARCHAR(50) NOT NULL,
    user_id INT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE controllers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    type VARCHAR(50) NOT NULL,
    state VARCHAR(50) NOT NULL,
    user_id INT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE devices (
    id INT PRIMARY KEY AUTO_INCREMENT,
    type VARCHAR(50) NOT NULL,
    status VARCHAR(50) NOT NULL,
    controller_id INT,
    FOREIGN KEY (controller_id) REFERENCES controllers(id)
);
```

**解析：**
这个数据库设计包括用户表、传感器表、控制器表和设备表。用户表存储用户信息，传感器表存储传感器数据，控制器表存储控制器状态，设备表存储设备状态。
