### 典型问题与答案解析

#### 1. 设计模式的基本概念与分类

**面试题：** 请简要介绍设计模式的基本概念与分类，并列举至少三种设计模式及其应用场景。

**答案解析：** 设计模式是解决软件设计中的常见问题的通用解决方案，分为创建型、结构型和行为型三类。

- **创建型模式**：涉及对象的创建，如工厂模式（Factory Pattern）、单例模式（Singleton Pattern）等。
  - **应用场景**：当需要根据不同条件创建不同类型的对象时，可以使用工厂模式。
  - **代码实例：**

    ```java
    public class FactoryPatternExample {
        public static Vehicle createVehicle(String type) {
            if ("car".equals(type)) {
                return new Car();
            } else if ("truck".equals(type)) {
                return new Truck();
            }
            throw new IllegalArgumentException("Unknown vehicle type");
        }
    }
    ```

- **结构型模式**：涉及类和对象的组合，如代理模式（Proxy Pattern）、装饰者模式（Decorator Pattern）等。
  - **应用场景**：当需要动态地给一个对象添加额外的功能时，可以使用装饰者模式。
  - **代码实例：**

    ```java
    public class DecoratorPatternExample {
        public static void main(String[] args) {
            Shape circle = new Circle();
            Shape redCircle = new RedCircle(circle);
            System.out.println(redCircle.toString());
        }
    }
    ```

- **行为型模式**：涉及对象间的通信，如观察者模式（Observer Pattern）、策略模式（Strategy Pattern）等。
  - **应用场景**：当需要实现对象之间的松耦合通信时，可以使用观察者模式。
  - **代码实例：**

    ```java
    public class ObserverPatternExample {
        public static void main(String[] args) {
            Subject subject = new Subject();
            Observer observer = new Observer();
            subject.register(observer);
            subject.notifyObservers("Data updated");
        }
    }
    ```

#### 2. 策略模式的面试题

**面试题：** 请解释策略模式，并给出一个使用策略模式的实际例子。

**答案解析：** 策略模式是一种行为设计模式，它允许在运行时选择算法的行为。它通过封装不同的算法变体，使它们之间可以相互替换，从而实现算法的动态选择。

- **策略模式定义：** 策略模式定义了一系列算法，将每一个算法封装起来，并使它们可以互相替换。该模式让算法的变化独立于使用算法的客户。

- **实际例子：** 一个支付系统可能需要支持多种支付方式，如信用卡、支付宝、微信支付等。

  ```java
  public interface PaymentStrategy {
      void pay();
  }

  public class CreditCardPayment implements PaymentStrategy {
      public void pay() {
          System.out.println("支付金额 via Credit Card");
      }
  }

  public class AlipayPayment implements PaymentStrategy {
      public void pay() {
          System.out.println("支付金额 via Alipay");
      }
  }

  public class WeChatPayment implements PaymentStrategy {
      public void pay() {
          System.out.println("支付金额 via WeChat");
      }
  }

  public class PaymentContext {
      private PaymentStrategy strategy;

      public PaymentContext(PaymentStrategy strategy) {
          this.strategy = strategy;
      }

      public void pay() {
          strategy.pay();
      }
  }
  ```

#### 3. 代理模式的面试题

**面试题：** 请解释代理模式，并给出一个使用代理模式的实际例子。

**答案解析：** 代理模式是一种结构型设计模式，它提供了一个对象的代理，用于控制对另一个对象的访问。代理可以代替原始对象接受所有的请求，并可以进行一些额外的处理。

- **代理模式定义：** 代理模式为其他对象提供了一种代理，以控制对这个对象的访问。

- **实际例子：** 一个常见的例子是远程代理，它允许客户端通过代理对象访问远程服务器上的对象。

  ```java
  public interface Image {
      display();
  }

  public class RealImage implements Image {
      private String fileName;

      public RealImage(String fileName) {
          this.fileName = fileName;
          loadImageFromDisk();
      }

      public void display() {
          System.out.println("Displaying " + fileName);
      }

      private void loadImageFromDisk() {
          System.out.println("Loading " + fileName + " from disk");
      }
  }

  public class ProxyImage implements Image {
      private RealImage realImage;
      private String fileName;

      public ProxyImage(String fileName) {
          this.fileName = fileName;
      }

      public void display() {
          if (realImage == null) {
              realImage = new RealImage(fileName);
          }
          realImage.display();
      }
  }
  ```
