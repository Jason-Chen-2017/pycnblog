# 1.简介
  

系统架构师（System Architect）是一个新型职位，也是云计算时代的一项重要角色。本文介绍系统架构师的定义、定位、个人能力要求及其工作难点。

# 2.定义
系统架构师（System Architect），通常由企业IT部门负责设计、开发和实施基础设施和应用层面的架构设计、规划、维护。他们涉足于组织结构、流程、工具、平台、协议等多个方面，通过设计整体的系统架构，从而建立可靠、高效、易用的系统环境，提升公司的运营、发展和竞争力。系统架构师既具备专业知识也要善于协调各个环节的关系，能够将不同技术、人员和团队的资源有效地结合起来共同解决问题。

# 3.定位与职责
系统架构师的主要职责包括但不限于以下几方面：

1. 服务oriented design：在服务性产品和服务提供方面，设计可满足用户需求的系统架构；
2. Infrastructure design：设计基础设施，包括网络、服务器、存储、数据库、操作系统、中间件等；
3. Platform architecture：研究和规划各个服务组件之间的集成和交互，根据业务需要设计不同类型平台的架构；
4. System integration and migration：设计、实施和管理系统架构和平台间的集成，评估和跟踪架构演进过程；
5. Service management：提供对系统运行和性能的全面支持，包括服务策略、规模化、故障处理、容量规划、迁移方案等；
6. Project Management：促进团队间的沟通、协作和绩效管理，制定和执行项目计划；
7. Strategic planning：结合战略意图和公司目标，制定战略计划并带领团队完成跨越组织边界的系统架构设计。

# 4.个人能力要求
系统架构师具有较强的工程、分析和商务素质，掌握多种编程语言、框架、平台和方法。除了系统架构师的职责外，他还需具备以下的个人能力要求：

1. Problem Solving Skills: 要有解决复杂问题的能力，精通各种系统架构设计方法和工具，对系统及其关键技术有深刻理解。
2. Communication Skills: 有良好的语言表达能力，能够准确、清晰地阐述系统设计意图，并与他人的沟通能力。
3. Leadership Ability: 在跨部门的复杂环境中保持高度的战略意识，主动推动组织目标的实现，持续关注并改善系统架构。
4. Business Acumen: 对客户的业务需求有敏锐的洞察力，理解用户痛点，与产品经理、交互设计师及其他相关人员进行密切沟通。

# 5.工作难点
系统架构师的工作难点主要体现在以下几个方面：

1. Balance between technical expertise and business acumen: 系统架构师必须全面理解系统的功能和非功能属性，理解行业和市场的发展趋势，具备丰富的业务阅历，并在此基础上判断哪些功能或技术需要投入更多的资源。
2. Complexity of system architectures: 在快速迭代的大环境下，系统架构师必须熟悉并掌握复杂的系统架构设计方法和工具，包括面向服务的架构、微服务架构、编排工具、容器技术、负载均衡等。
3. Flexible Schedule: 在长时间工作中，系统架构师需要有弹性，能够适应各种工作调度和加班安排，确保不影响日常工作开展。
4. Distributed teams: 大型分布式系统和系统架构，往往存在多名系统架构师承担多个职责。
5. Leverage across the organization: 系统架构师需要学习其他人的技巧和经验，在多个部门之间分享自己的系统架构知识。
