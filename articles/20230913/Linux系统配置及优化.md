# 2.核心概念与术语
## 2.1 Linux
Linux是一个开源的类Unix操作系统内核。它由林纳斯·托瓦兹（<NAME>）在2000年9月17日创建，他希望基于自由软件基金会的这一条创新的理念打造一个全新的操作系统，即使不受任何商业限制，任何个人或组织都可以免费获取、修改和重新分发该系统。其后的10多年时间里，Linux世界中已经涌现出了各种发行版、开源软件包、驱动程序、脚本语言和应用软件等。目前，Linux在服务器领域占据着举足轻重的地位。
## 2.2 文件系统
文件系统（File System）是指在计算机存储设备上用来存放信息的文件结构。Linux系统中，主要的文件系统包括Ext2、Ext3、Ext4、XFS、NTFS、FAT等。其中，Ext2、Ext3和Ext4都是比较老旧的文件系统，XFS和NTFS是较新的文件系统。在Linux系统中，通常我们使用Ext4文件系统作为主文件系统，其次才是使用XFS文件系统，最后才是使用NTFS文件系统。
## 2.3 磁盘分区
磁盘分区（Partitioning）是将硬盘分成不同的区域，用于存放不同类型的数据。每一个分区对应于一个独立的文件系统。一般来说，服务器需要至少三个分区，分别是根目录分区（/）、交换分区（Swap）、日志分区（Log）。而其他的分区根据需要可以增加或删除。
## 2.4 命令行接口
命令行接口（Command Line Interface，CLI）是一种用户与计算机之间进行沟通的方式。通过键入指令并按下回车（Enter）键，我们可以在Linux系统中执行各种操作。这种方式相对于图形界面（Graphical User Interface，GUI），显得更加灵活和高效率。
## 2.5 Shell
Shell（外壳）是用户与Linux系统之间的接口。它是用户运行各种程序的入口点。Shell本身是一个命令解释器，它会解析用户输入的命令，然后依照相应的语法规则将命令发送给操作系统。由于Shell是直接与操作系统通信的，因此它也称作命令行接口。
## 2.6 用户
在Linux系统中，每个用户都有一个唯一的用户名和密码，并被分配到特定的组。Linux系统允许多个用户同一时间访问同一台计算机，但每个用户只能独自使用自己的用户名和密码登录。为了提高安全性，建议设置复杂的密码。
## 2.7 权限
权限（Permission）是授予某个用户对文件的访问权力，也就是控制某个用户是否可以读取、修改或删除某个文件。Linux系统中的权限包括读、写、执行（可执行）权限。其中，rwx分别代表读、写、执行三种权限，分别用数字表示为r=4、w=2、x=1。所以，7表示所有权限（read、write、execute），0表示无任何权限。
## 2.8 文件类型与扩展名
文件类型（Type）是指文件的总体分类，例如常见的文本文件、图像文件、二进制可执行文件等。扩展名（Extension Name）则是指在文件名前面添加的后缀名。比如，“Hello World”这个文件的文件名叫做“hello”，它的扩展名是“txt”。
