# 2.RESTful的定义及其重要性
REST（Representational State Transfer）是Roy Fielding博士在2000年提出的一种设计风格。它的主要思想是通过HTTP协议传输资源状态，使得网络上的各类计算机可以互相传递信息，而无须先确立对方的接口，由服务端提供统一的API接口，让客户端可以更方便地调用服务端的资源，而不需要考虑网络通讯上的各种复杂细节。因此，REST被认为是Web上最流行的设计模式之一。 

从字面意义上理解，“表现层状态转移”表示的是资源的表现形式应该由系统的当前状态来决定，并通过HTTP协议传达给客户端，而非像其他协议那样通过特定的命令序列来完成请求动作。这种转移方式能够减少网络延迳、提高通信效率、减轻服务器负载、增加可伸缩性等诸多好处。

RESTful这个词的起源，主要不是为了取代其他协议，而是强调设计符合REST原则的API的能力，帮助用户更容易地发现、理解、利用、扩展和组合分布式系统的功能。基于这一原则，设计者们围绕资源、URI和HTTP方法进行交互体系的设计，并试图构建满足实时性、互斥性和分层系统的约束条件下的API。

正因如此，RESTful已经成为当今互联网应用的基础设施，涉及多个公司、多个开发团队、多个编程语言、多个平台，RESTful规范成为不同组织之间合作和互通的共识。并且随着互联网应用的发展，越来越多的人开始认识到RESTful的优势，从而开始逐渐使用RESTful规范来构建他们的系统。

RESTful Web Services是REST设计风格的一种具体实现，主要倡导通过标准化的接口来实现分布式系统之间的通信。它最初起源于企业内部的Web应用服务间通信需要，但是也逐步演变成一个独立的规范，并广泛用于Web API的设计和实现。目前RESTful Web Services已经成为一种事实上的标准，各家公司都开始制定自己的RESTful规范，而很多知名的RESTful服务也开始向全世界开放。

总结一下，RESTful Web Services规范是基于HTTP协议构建的一种网络应用程序编程接口，采用这种风格能够极大地提升系统的可伸缩性、可用性、容错性和性能。在RESTful Web Services中，资源通常通过URI标识，通过HTTP方法(POST/GET/PUT/DELETE)执行CRUD（创建、读取、更新和删除）操作，通过统一的接口标准、数据格式、状态码、错误处理等机制实现通信的可控性。使用RESTful规范可以有效降低系统的耦合度，提高系统的易用性和可靠性。所以，理解RESTful Web Services对于掌握和使用Web API至关重要。
