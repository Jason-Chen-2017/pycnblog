# 1.简介
  


大数据业务分析师（Data Analyst），是指能够进行复杂的数据分析、挖掘、建模等工作的一名职位。它与一般的软件工程师不同之处在于，它擅长于对大量数据进行有效的分析和处理，并且能够将这些数据转换成可用于决策支持、营销或产品开发等应用场景中的信息。
大数据业务分析师需要对业务领域内的数据情况有比较全面的了解，掌握数据挖掘、分析方法、模型构建、信息检索、知识发现等相关技术。同时也需要具有极强的团队精神和沟通能力，能够快速学习新技能，协调资源，从而取得优秀效果。

# 2.职责范围及要求

- 数据需求分析：了解客户信息、产品信息、市场信息、交易数据等；
- 数据采集：掌握主流数据源获取方式，包括网络爬虫、API接口等；
- 数据清洗：掌握常见数据清洗方式，包括缺失值填充、异常值检测、合并重复数据、归类汇总等；
- 数据分析：掌握数据分析工具、统计方法，如关联规则、聚类分析、决策树算法、机器学习算法等；
- 智能推荐：基于用户画像、历史行为等，实现基于物品和用户的个性化推荐系统；
- 数据报表制作：根据数据分析结果生成可视化图表，并配合文字叙述进行解释。

# 3.核心能力

- 抽象能力：掌握抽象思维能力，能够准确把握客户需求，并理解需求背后的真实原因；
- 统计分析能力：理解统计学原理、概率论、数理统计、机器学习等理论，能运用高级统计方法分析数据；
- 数据挖掘能力：掌握分类、回归、聚类、关联规则、异常检测、时间序列分析、图像识别等数据挖掘方法；
- 数据库及SQL语言能力：掌握MySQL/PostgreSQL等关系型数据库的使用方法，具备编写复杂SQL语句能力；
- Python编程能力：熟练掌握Python编程语言，可以进行数据分析、特征工程、机器学习任务的开发；
- 可视化表达能力：了解各种数据可视化技术，如柱状图、散点图、气泡图等，并善于应用；
- 英文口语表达能力：良好的书面表达能力和语言组织能力，能将复杂的问题阐述清楚，对面试官的提问予以回应。

# 4.行业竞争力

大数据业务分析师的薪酬待遇比普通程序员稍微高一些，但是并不意味着就一定能胜任大数据业务分析师这个岗位，因为还有许多其他的专业人员可以承担类似的工作。举个例子，假设你是某知名公司的CEO，你肯定不会被培养成为数据分析师，因为那样的话就太晚了。在大数据时代，数据分析师和软件工程师并不是同一种角色，后者更侧重于产品开发和软件维护，而前者更多地参与到数据探索、数据挖掘、数据可视化、数据服务等方面。所以如果你对自己所处行业的发展方向有较深的认识，并愿意尝试新的事物，那么大数据业务分析师是一个非常好的选择。
