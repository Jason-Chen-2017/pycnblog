# 1.简介
  

搜索（又称图搜）是解决计算机科学中最基础的问题之一，也是计算机科学和数学中的重要分支。在自然语言处理、机器学习、数据挖掘、信息检索、生物信息学等领域都涉及到搜索问题的各个方面。搜索算法是指将给定的一组信息（可能是图或其他复杂结构的数据结构），找到特定目标或满足某种条件的一组数据集合。例如，在网页爬虫或数据挖掘中，搜索算法用来收集和分析所需数据，而在信息检索系统中，搜索算法用于提取和组织有关信息。搜索算法广泛应用于图、排序、组合优化等方面。

图论是数学的一个分支，它研究的是一些图形结构，如网络、计算过程和关系等。图论的主要关注点在于定义图的表示方法、遍历图的各种算法、图的生成和识别。搜索算法在图论的很多应用上也被运用，比如图搜索、图匹配、路径规划、最大流等等。

2.历史背景
搜索算法最早是由图灵机的开发者约翰·图灵提出来的。但由于图灵机的存储容量有限、运算速度慢，所以直到1960年代末期，才逐渐出现可用于搜索问题的图论算法。其主要基于图论中对节点之间的连接和相关属性之间的比较的性质，采用了基于搜索树的启发式方式来进行路径搜索，并进行了扩展和回溯等操作，从而实现路径的构建。后来，基于这种启发式搜索方法的图搜索算法已经成为图论中经典的算法。

3.基本概念和术语
为了能够理解和应用搜索与图论算法，需要掌握以下基本概念和术语：

图（Graph）：图是一种数据结构，通常表示为一个二元组G=(V,E)，其中V为顶点集，E为边集，图中任意两个顶点间存在一条或多条边。

带权图（Weighted Graph）：带权图同样是一个二元组G=(V,E),其中E中每条边都有一个权值w(u,v)。对于某些问题来说，带权图更加适合表达复杂的关系和依赖关系。

节点（Vertex）：图G中的每个元素都称为节点。

边（Edge）：图G中任意两点之间连线称为边。

相邻节点（Adjacent Vertex）：如果两个节点之间存在一条边，则称它们是相邻节点。

路径（Path）：路径是一个顶点序列，通过若干条边，从一个节点到另一个节点。

无环路径（Acyclic Path）：一个路径没有环路，即不存在任何重复的边。

简单路径（Simple Path）：一个路径中不能含有重复的边。

回路（Cycle）：回路是一个首尾相接的路径。

中心节点（Centricity）：节点的中心度定义为该节点所属的所有回路长度之和除以该节点的度。节点中心度高的节点称为中心点。

距离（Distance）：两个节点之间的最短路径的长度。

生成树（Spanning Tree）：生成树是所有顶点构成的无回路子图。

哈密顿回路（Hamiltonian Cycle）：哈密顿回路是一个奇数长度的回路，且可以通过增加最后一个结点使其成为长度为偶数的回路。哈密顿路径则不一定是回路，而只要可以经过每条边一次即可作为哈密顿路径。

中心点分裂（Center Point Splitting）：在生成树中，通过把所有中心点分裂，就可以得到子图。

最小生成树（Minimum Spanning Tree）：生成树中连接所有顶点的权重最小的子图。

Kruskal算法（Kruskal's Algorithm）：Kruskal算法是一种最小生成树（MST）算法。它首先按照边的权重大小从小到大的顺序选取边，然后依次加入所选边形成生成树，直到加入的边超过所有顶点个数减去两，或者构造出的生成树是一个完整图时结束。

Prim算法（Prim's Algorithm）：Prim算法也是一种最小生成树（MST）算法。它从某个顶点开始，选择距离它最近的未探测到的顶点，然后沿这条边前进，直至探测完所有的顶点。

4.核心算法
搜索算法的关键是如何快速地找到目标路径或目标节点。基于图论的方法，包括Kruskal算法、Prim算法、A*算法、Bellman-Ford算法等。这里只介绍几种最流行的搜索算法。

Dijkstra算法：Dijkstra算法是用于寻找单源最短路径的算法，其思想就是维护一个优先队列，每次取队头节点距离目前已知路径最短的节点，并且更新其他节点到这个节点的距离。该算法的时间复杂度是O(|V|^2)级别。

BFS算法：BFS算法是一种广度优先搜索（Breadth First Search，BFS）算法，其思想是维护一个队列，每次取队头节点的所有相邻节点加入队列，遍历完所有节点之后才下一层继续查找。该算法的时间复杂度是O(|V|+|E|)级别。

DFS算法：DFS算法是一种深度优先搜索（Depth First Search，DFS）算法，其思想是从根节点开始，依次递归地遍历子树。该算法可以在不同的子树中回溯，因此适用于图的遍历。时间复杂度是O(|V|+|E|)级别。
