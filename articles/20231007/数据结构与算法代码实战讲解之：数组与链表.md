# 1.背景介绍


在计算机科学中，数据结构（Data Structure）是指相互之间存在一种或多种特定关系的数据元素的集合。数据结构决定了数据之间的相互关系及操作。数据结构可用于各种应用场合，如电子商务、信息检索、文件管理、数据库等。本文将以最常用的两种数据结构——数组和链表进行讲解。

## 1.1 数组
数组是存储相同类型数据的集合，它可以是静态的或者动态的。静态数组就是在程序运行前就已经分配好空间，大小也不能改变；而动态数组则是根据需要自动扩充它的容量。数组的下标从0开始计数，并且可以通过下标访问数组中的元素。数组的存储单元可以是连续的也可以是离散的。如果存储单元是连续的，那么数组可以被看做一个一维线性表。但是，如果存储单元是离散的，比如用指针指向数组中的元素，那么这种情况下数组不是线性表了。

## 1.2 链表
链表是一个数据项的链接，它由一系列节点组成。每个节点包含两个部分：数据域和指针域。数据域存储真正的数据值，指针域则用来记录当前节点指向的下一个节点的位置。第一个节点的指针域指向第二个节点，第二个节点的指针域指向第三个节点，依此类推，最后一个节点的指针域指向空（null），表示链表的结束。因此，链表也是非线性表，但比数组更灵活。

# 2.核心概念与联系
# （1）数组
数组（Array）是一种线性结构，由若干相同类型的元素按一定顺序排列而成的一块内存空间。

数组的基本属性：

- 序号(index): 从0开始的整数，用以标识数组的每个元素。
- 长度(length): 有多少个元素就有多长。
- 数据类型(datatype): 每个元素的类型相同。
- 固定长度: 如果创建了固定长度的数组，其长度就不能再变了。
- 分配方式: 静态分配的数组和动态分配的数组。

数组的优点：

1. 通过下标访问元素，具有随机访问能力，时间复杂度为O(1)。
2. 只需一次内存分配就可以初始化，降低开销。
3. 支持动态增长，灵活方便。

数组的缺点：

1. 插入和删除操作时效率低，因为要移动数组中各元素，时间复杂度为O(n)。
2. 需要预先知道最大容量，浪费空间。

# （2）链表
链表（Linked List）是由一系列结点（Node）组成的线性结构。每个结点都包含两部分信息：一个存储的数据元素和一个指向下一个结点的引用地址。链表首结点的引用称为空头指针（NULL Pointer）。

链表的基本属性：

- 无序: 不需要给出元素的索引，只需要直接访问即可。
- 动态大小: 在运行过程中，链表的大小可以动态变化。
- 插入和删除: 通过指针操作可以插入或删除元素，插入和删除的时间复杂度为O(1)，不需要对其他元素进行搬移。

链表的优点：

1. 插入和删除操作快，不需要整体移动。
2. 可以实现一些高级操作，比如按照某种顺序遍历链表。

链表的缺点：

1. 使用指针反复寻址比较麻烦。
2. 在遍历链表时，需要从头到尾遍历完才能确定是否遍历完成。
