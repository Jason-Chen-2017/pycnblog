# 2.核心概念与联系
## Garbage Collection

在Go语言中，垃圾回收（GC）机制是自动进行内存管理的一种重要功能。简单的说，它是一个后台进程，用于监视那些不再被程序使用的内存块，并释放它们以供后续程序使用。

一般来说，Go程序中的内存分配都是自动完成的，不需要手动申请和释放内存。但当一个变量引用的对象生命周期结束时，垃圾收集器就会回收该变量占用的内存空间。此外，由于垃圾收集器是在运行时对内存进行整理的，因此它的处理速度要比一般应用程序慢很多。但是，Go语言通过GC机制可以很好的解决内存管理的问题。所以，熟悉GC的工作原理非常重要。

## pprof工具

pprof是Go语言官方提供的一个性能分析工具，它提供了丰富的性能剖析工具，包括：CPU分析、内存分析、goroutine分析、阻塞检测等。通过分析这些数据，能够帮助我们找出程序的热点函数、延迟高的组件以及存在泄露的内存等。另外，pprof还支持导出图表，使得我们可以直观地查看程序的性能。
