## 2. 核心概念与联系
### 2.1 数据类型
MySQL数据类型是数据库中数据的类型，用于定义数据库表中字段的数据类型。MySQL支持多种数据类型，包括整数、浮点数、字符串、日期和时间等。

以下是MySQL中的一些常见数据类型：

- 整数类型：INT、TINYINT、SMALLINT、BIGINT
- 浮点数类型：FLOAT、DOUBLE
- 字符串类型：VARCHAR、TEXT、BLOB
- 日期和时间类型：DATE、DATETIME、TIME、TIMESTAMP

### 2.2 字段属性
字段属性是数据库表中字段的附加信息，用于定义字段的特性。字段属性对于数据库设计和开发至关重要，因为它们确定了数据库中数据的结构和约束。

以下是MySQL中的一些常见字段属性：

- NOT NULL：字段不能为空。
- UNIQUE：字段值必须是唯一的。
- DEFAULT：字段的默认值。
- AUTO_INCREMENT：自动递增字段。

### 2.3 数据类型与字段属性的联系
数据类型和字段属性在数据库设计和开发中是紧密相连的。数据类型定义了数据库中存储的数据类型，而字段属性定义了数据库中数据的结构和约束。

例如，在创建一个表时，您可以为表的字段指定数据类型，如INT、VARCHAR等。同时，您还可以为字段指定字段属性，如NOT NULL、UNIQUE等。这些数据类型和字段属性共同确定了数据库中数据的结构和约束。

## 4. 具体代码实例和详细解释说明
在本节中，我们将通过具体代码实例来说明数据类型和字段属性的使用方法。

### 4.1 数据类型的使用方法
以下是一个使用MySQL数据类型的示例：

```sql
CREATE TABLE students (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    age INT,
    gender ENUM('M', 'F')
);
```

在这个示例中，我们创建了一个名为"students"的表，其中包含四个字段：id、name、age和gender。id字段是自动递增的主键，name字段是一个不能为空的VARCHAR类型，age字段是一个整数类型，gender字段是一个枚举类型，只能取值为'M'或'F'。

### 4.2 字段属性的使用方法
以下是一个使用MySQL字段属性的示例：

```sql
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    customer_id INT,
    product_id INT,
    price DECIMAL(10, 2) NOT NULL,
    status ENUM('pending', 'shipped', 'delivered') DEFAULT 'pending'
);
```

在这个示例中，我们创建了一个名为"orders"的表，其中包含四个字段：id、customer_id、product_id、price和status。id字段是自动递增的主键，customer_id和product_id字段是整数类型，price字段是一个不能为空的小数类型，status字段是一个枚举类型，默认值为'pending'。
