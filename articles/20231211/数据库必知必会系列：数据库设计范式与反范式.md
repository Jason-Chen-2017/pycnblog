## 2.2 反范式

反范式是一种数据库设计方法，它的目的是为了提高数据库的性能和易用性。反范式的核心概念包括：

- 反范式的目的：提高数据库的性能和易用性。
- 反范式的方法：通过将多个表合并为一个表，从而减少数据库的查询和操作次数。
- 反范式的优点：易于使用，性能较高。
- 反范式的缺点：数据冗余，维护成本较高。

```sql
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);
```

### 4.1.2 第二范式（2NF）

```sql
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

CREATE TABLE address (
    id INT PRIMARY KEY,
    city VARCHAR(50),
    state VARCHAR(50),
    zip INT
);

CREATE TABLE student_address (
    student_id INT,
    address_id INT,
    PRIMARY KEY (student_id, address_id),
    FOREIGN KEY (student_id) REFERENCES student(id),
    FOREIGN KEY (address_id) REFERENCES address(id)
);
```
