# 2.核心概念与联系

设计模式是一种解决问题的方法，它们提供了一种解决常见问题的方法，并且可以帮助我们更好地组织和管理代码。设计模式可以分为三类：创建型模式、结构型模式和行为型模式。

- 创建型模式：这些模式主要关注对象的创建过程，包括单例模式、工厂模式、抽象工厂模式、建造者模式和原型模式。
- 结构型模式：这些模式主要关注类和对象的组合，包括适配器模式、桥接模式、组合模式、装饰器模式和外观模式。
- 行为型模式：这些模式主要关注对象之间的交互，包括策略模式、模板方法模式、命令模式、观察者模式和状态模式。

## 3.1 单例模式

单例模式是一种创建型模式，它确保一个类只有一个实例，并提供一个全局访问点。这种模式通常用于管理全局资源，例如数据库连接、文件操作等。

算法原理：单例模式通过在类内部维护一个静态变量来保存唯一的实例，并在构造函数中检查是否已经存在实例。如果存在，则返回已经存在的实例；否则，创建一个新的实例并返回。

具体操作步骤：
1. 在类内部定义一个静态变量来保存唯一的实例。
2. 在构造函数中，检查是否已经存在实例。如果存在，则返回已经存在的实例；否则，创建一个新的实例并返回。
3. 在外部，通过全局访问点获取单例实例。

数学模型公式：
$$
Singleton(x) = \begin{cases}
    instance & \text{if } instance \neq null \\
    new Singleton() & \text{else}
\end{cases}
$$

## 3.2 工厂模式

工厂模式是一种创建型模式，它定义了一个创建对象的接口，但不具体指定创建哪个类的实例。这种模式允许我们在不知道具体对象类型的情况下，创建对象。

算法原理：工厂模式通过定义一个创建对象的接口，并提供一个工厂类来实现这个接口。工厂类根据传入的参数创建不同类型的对象。

具体操作步骤：
1. 定义一个创建对象的接口，包括一个创建对象的方法。
2. 实现具体的工厂类，实现创建对象的接口。
3. 在外部，通过调用工厂类的创建对象方法获取对象实例。

数学模型公式：
$$
Factory(x) = \begin{cases}
    Product(x) & \text{if } x \in A \\
    OtherProduct(x) & \text{else}
\end{cases}
$$

## 4.1 单例模式实例

```python
class Singleton:
    _instance = None

    @classmethod
    def getInstance(cls):
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
```

在上述代码中，我们定义了一个类`Singleton`，它内部维护了一个静态变量`_instance`来保存唯一的实例。在`getInstance`方法中，我们检查是否已经存在实例，如果存在，则返回已经存在的实例；否则，创建一个新的实例并返回。
