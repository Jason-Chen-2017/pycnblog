# 2.核心概念与联系
在了解MySQL的创建和修改数据库表的过程之前，我们需要了解一些核心概念和联系。

## 2.1 数据库、表、列和行
在MySQL中，数据库是一个逻辑的容器，用于存储和管理数据。数据库可以包含多个表，每个表都包含多个列，列用于存储数据的具体信息。每个表的行表示一条记录，记录包含了多个列的值。

## 2.2 数据类型
数据类型是用于定义列中存储的数据类型的规则。MySQL支持多种数据类型，如整数、浮点数、字符串、日期等。选择合适的数据类型对于确保数据的准确性和效率至关重要。

## 2.3 约束
约束是用于限制表中数据的规则的规定。例如，可以使用NOT NULL约束来指定某个列不能为空，使用UNIQUE约束来指定某个列的值必须唯一。约束可以确保数据的完整性和一致性。

## 2.4 关系
关系是数据库表之间的联系。通过关系，我们可以实现数据的连接和查询。关系可以通过主键和外键来定义，主键是表中一个或多个列的组合，用于唯一标识每一行记录。外键是表之间的关联关系，用于连接不同表的数据。

# 4.具体代码实例和详细解释说明
在本节中，我们将通过具体代码实例来说明如何创建和修改数据库表。

## 4.1 创建数据库表的代码实例
以下是一个创建数据库表的代码实例：
```sql
CREATE TABLE employees (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL,
    age INT,
    salary DECIMAL(10,2)
);
```
在这个例子中，我们创建了一个名为employees的表，表包含四个列：id、name、age和salary。id列是表的主键，使用AUTO_INCREMENT属性自动生成唯一的值。name列是一个VARCHAR类型的列，不允许为空。age列是一个整数类型的列，salary列是一个小数类型的列，精度为10，小数部分为2。

## 4.2 修改数据库表的代码实例
以下是一个修改数据库表的代码实例：
```sql
ALTER TABLE employees
ADD COLUMN department VARCHAR(255);
```
在这个例子中，我们修改了employees表，添加了一个新的列department，用于存储员工的部门信息。
