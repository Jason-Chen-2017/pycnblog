## 4. 单例模式

### 4.1 核心概念

单例模式是一种创建型设计模式，它确保一个类只有一个实例，并提供全局访问点。这种模式主要用于解决一个类的多次实例化会导致资源浪费或数据不一致的问题。

### 4.2 应用场景

单例模式适用于需要全局访问的资源限制的情况，例如配置文件、日志系统、线程池等。

### 4.3 代码实现

```python
class Singleton(object):
    _instance = None

    @classmethod
    def getInstance(cls):
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance

    def __init__(self):
        self.value = None

singleton1 = Singleton.getInstance()
singleton2 = Singleton.getInstance()

assert singleton1 == singleton2  # True
```

### 4.4 优缺点分析

优点：

* 节省内存，减少不必要的实例化
* 提供了全局访问点，便于在整个程序中共享资源

缺点：

* 模式本身就已经存在一个全局状态，可能导致代码难以测试和维护
* 由于单例模式的全局状态，可能导致数据不一致或资源竞争的问题

## 5. 适配器模式

### 5.1 核心概念

适配器模式是一种结构型设计模式，它将两个不兼容的接口连接起来，使得它们可以平滑地工作在一起。适配器模式主要用于解决接口不兼容的问题。

### 5.2 应用场景

适配器模式适用于需要将两个不兼容的接口连接起来以实现新的功能的情况，例如将旧版接口与新版接口适配，以实现向后兼容。

### 5.3 代码实现

```python
class OldInterface(object):
    def method1(self):
        print("OldInterface method1")

class NewInterface(object):
    def method2(self):
        print("NewInterface method2")

class Adapter(NewInterface):
    def __init__(self, old_instance):
        self.old_instance = old_instance

    def method2(self):
        self.old_instance.method1()

old_instance = OldInterface()
adapter = Adapter(old_instance)
adapter.method2()  # OldInterface method1
```

### 5.4 优缺点分析

优点：

* 解决了接口不兼容的问题，实现了向后兼容
* 提高了代码的灵活性和可扩展性

缺点：

* 适配器类可能变得复杂，难以维护
* 适配器类可能需要为不兼容的接口编写大量的代码
