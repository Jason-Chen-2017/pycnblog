旅游业务管理系统是一个集旅游信息查询、预订、支付等功能为一体的综合性系统。系统主要功能包括：旅游信息查询、预订管理、订单结算、用户管理、系统管理等。下面详细介绍系统的设计和具体代码实现。

1. 系统架构

系统采用B/S架构，使用Java语言作为后端开发语言，使用Spring框架进行后端开发，使用MyBatis作为持久层框架，使用MySQL作为数据库。前端使用HTML、CSS、JavaScript进行开发，使用Bootstrap作为前端框架。

2. 系统功能模块

系统主要包括以下几个功能模块：

a) 旅游信息查询：用户可以通过关键字查询旅游景点、酒店、机票等信息。

b) 预订管理：用户可以在线预订酒店、机票、景点门票等。

c) 订单结算：用户可以在线支付订单，并获取支付凭证。

d) 用户管理：系统支持用户注册、登录、修改个人信息等功能。

e) 系统管理：系统管理员可以对系统进行配置、监控、维护等。

3. 系统具体代码实现

a) 旅游信息查询

在Java后端，创建一个TourismInfoController类，使用Spring MVC框架处理用户请求。

```
@RestController
@RequestMapping("/tourismInfo")
public class TourismInfoController {
    
    @Autowired
    private TourismInfoService tourismInfoService;

    @GetMapping("/query")
    public ResponseEntity<List<TourismInfo>> queryTourismInfo(@RequestParam String keyword) {
        List<TourismInfo> tourismInfoList = tourismInfoService.queryTourismInfo(keyword);
        return ResponseEntity.ok(tourismInfoList);
    }
}
```

在Java后端，创建一个TourismInfoService类，实现查询旅游信息的逻辑。

```
@Service
public class TourismInfoService {
    
    @Autowired
    private TourismInfoMapper tourismInfoMapper;

    public List<TourismInfo> queryTourismInfo(String keyword) {
        return tourismInfoMapper.queryTourismInfo(keyword);
    }
}
```
