# 1.背景介绍


## 1.1 概念介绍
### 1.1.1 数据仓库
数据仓库（Data Warehouse）是一个集成化、相对集中的存储库，用于存储所有在组织范围内产生、变更或删除的数据。它通常被用来支持企业决策过程、管理决策、分析数据、支持战略规划等，并从各种源头获取数据，如企业内部事务系统、财务信息系统、销售订单数据库、操作系统日志、客户关系管理数据库等。数据仓库的内容可以通过多种方式进行汇总、报告和分析，帮助业务领域人员做出快速、可靠的决策。其架构主要包括如下几个方面：
- 维度建模：数据的结构定义及数据质量检查，主要适合高级商务用户、金融机构、保险公司等大型机构。维度建模包括建立各个维度的定义、分类、层次等，并设计相关的视图或报表显示数据的汇总信息；
- 事实层建模：实际业务实体之间发生的交互与转换，主要适合中小型企业。事实层建模包括定义事实表的结构，通过明确业务规则和约束对事实进行规范化，确保数据一致性；
- 主数据建模：外部数据源（如厂商系统、采购系统等）引入到数据仓库时，需要按照统一标准对数据进行预处理，并将数据转存到规范的模式中。主数据建模一般用ETL工具完成，将原始数据导入到临时表或事实表，然后再应用数据转换脚本对数据进行清洗、转换、合并、拆分等操作；
- 数据加工层：数据仓库以外的其他系统和分析工具需要访问数据仓库里面的信息，如采用OLAP的方式提供数据分析查询服务。数据加工层的任务就是把复杂的数据转换成简单易懂的信息，使得后续的业务决策和分析工作更高效；
- 数据集市层：数据集市层用于支持多种数据源之间的集成。由于不同数据源的更新速度和准确性存在差异，因此数据集市层能够有效地整合多个源头的数据，为业务决策提供单一、集中的信息源。

### 1.1.2 OLAP
在线分析处理（Online Analytical Processing，简称OLAP），是一种基于多维分析方法的一套数据仓库技术。OLAP是指用多维数据结构（多维维度与度量）来分析和检索大量的数据，特别是在数据量很大的情况下。例如，在零售行业，经营决策通常需要分析顾客消费行为、价格决策、销售趋势、供应商信息等综合性数据。OLAP将原有数据经过预处理、清洗等措施，汇总到一张或多张多维表中，可以直接进行分析、检索。OLAP目前已成为数据仓库技术的一种重要工具，广泛应用于金融、证券、保险、制造、电子商务、零售等各行各业。

### 1.1.3 决策支持系统
决策支持系统（Decision Support System，DSS）是一个信息系统，用来帮助人们做出业务决策，并通过计算机辅助决策支持工具来支持决策过程。根据需求场景的不同，DSS可分为四类：规则引擎、统计分析、决策导则、直观分析。
- 规则引擎：规则引擎基于离散事件流、条件判断和正向推理等原理，通过一系列规则自动地对数据进行分析、归纳和推理，生成符合要求的建议或操作指南。其优点是灵活、精准、即时、自动化，但缺点也很突出，对实践经验、理解能力、逻辑思维能力都不足。
- 统计分析：统计分析方法借鉴了概率论和统计学的理论，用概率分布来描述变量间的依赖关系，并通过计算频率统计值、回归曲线和因子分析等方法进行数据分析。统计分析方法广泛运用于金融、制造、电信、医疗、物流、能源、公共安全等领域。
- 决策导则：决策导则是一套以明确目标为导向的演示建议，旨在帮助企业快速、有效地做出业务决策，提升工作效率。决策导则一般由业务专家编写，对工作流程、活动指标、关键结果等情况作出详尽的说明。由于决策导则的可读性强、结构化程度高、对业务规则的强调，因此在执行中往往容易遭遇滞后、误判等问题。
- 直观分析：直观分析方法是一种专门针对决策制定的方法，以图形化的方式呈现数据、信息和建议，以直观的形式揭示业务决策的意义。直观分析方法常用绘制决策树、条形图、柱状图、饼图等图表，将数据的变化关联起来，帮助决策者快速、准确地做出决策。直观分析法得到了很多企业的青睐，但它的弊端也十分明显，比如无法解释数据背后的原因、无法量化目标指标、不够灵活。

### 1.1.4 数据建模
数据建模是将已有的数据或者新收集到的信息转化为有价值的有序集合，用于分析、决策和给出业务解决方案。数据建模的目的通常是为了得到一个客观、全面、易于理解、易于维护的系统。数据建模的方法有实体关系模型、对象关系模型、半结构化模型、面向主题的模型、层次模型、网络模型等。数据建模的基本要素有实体、属性、关系、实体间的联系等。
