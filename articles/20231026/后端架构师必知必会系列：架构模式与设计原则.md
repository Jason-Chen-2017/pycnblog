# 1.背景介绍


在IT界，架构师是一个非常重要的角色，它负责整个IT系统或者业务的设计、开发、部署、运维、管理等工作。而架构师又分成两类：
- 传统意义上的架构师(Traditional Architect)：主要承担的是整体系统的设计和研发；
- 云计算时代的架构师（Cloud Native Architect）：主要负责云平台的应用架构设计和开发；

那么，传统架构师和云架构师之间的区别在哪里呢？为什么会产生这种区别呢？为什么很多人觉得传统架构师和云架构师之间没有可比性呢？

对于这个问题，首先需要搞清楚一些基本概念。

# 概念
## 模式(Pattern)
模式(pattern)是对一段重复出现的问题、结构或行为进行总结的一种描述或范例。最早由Eric Evans提出的面向对象设计模式[Evans97]，通过一套简单的规则并提供典型的示例来帮助工程师解决软件设计中经常出现的问题。这些设计模式有助于统一计算机科学的各种术语，并促进共同关注点的分离，从而更好地理解软件构造和设计的不同方面。

目前已经存在众多的设计模式，如发布-订阅(Pub/Sub)模式、职责链(Chain of Responsibility)模式、命令(Command)模式、迭代器(Iterator)模式等，它们能提供统一的解决方案和实现方法。使用合适的设计模式能让程序更容易维护、扩展和复用，从而有效降低软件开发和维护成本。

## 设计原则(Design principle)
设计原则(Design principle)是指一组能够引导软件工程师在不违反通用性的前提下对软件系统进行设计的准则和法则。这些原则一般都与特定的设计问题和设计目标密切相关。例如，面向对象设计原则通常是用于处理类的设计问题，其核心思想是将系统划分成具有明确功能职责的各个模块，并使各模块之间高度内聚，避免彼此干扰。

设计原则有助于架构师识别、消除和解决系统设计中的一般性问题。在软件设计过程中，应当优先采用经过充分论证的、能够指导项目成功的原则。

# 传统架构师和云架构师之间的区别
## 分工不同
传统架构师一般只负责系统整体设计和研发，而云架构师则更加侧重于架构设计、平台选择和应用设计。传统架构师的主要工作就是把复杂的需求分析转换成高效且可靠的解决方案。比如，传统架构师会根据业务流程图、系统框架图、数据库设计图等制定系统架构，然后采用图形化工具搭建出详细的系统设计文档。随着时间的推移，架构师还会发现一些新的架构和设计问题，并不断对系统架构进行改造和优化。他们也需要了解技术发展和行业趋势，持续追踪技术发展方向，不断提升自我能力。

而云架构师则更多的是从整体上考虑如何利用云计算资源构建健壮、弹性、安全的应用程序，因此他们更擅长应用领域的深入理解、分析、设计和实践。云架构师往往会参与到多个云服务提供商之间技术选型、架构设计、性能调优、监控和故障排查等一系列环节，充分体现了架构师的全面能力。

## 技术倾向不同
云计算虽然带来巨大的变化，但传统软件架构的设计方式仍然占据主导地位。传统的系统架构设计从简单到复杂依次是顺序流水线(Sequential Pipeline)，层次结构(Layered Architecture)，依赖关系(Dependency Hierarchy)和服务化(Service Oriented)。

传统的架构设计注重硬件资源的利用率，如数据库、文件服务器、内存缓存等，避免过度配置或者超卖资源，通过优化调度和负载均衡的方式提升系统响应速度。传统架构设计的另一个特征是基于组件的模块化，使系统可以灵活的部署和替换不同的子系统。同时，由于硬件资源的限制，传统架构设计还会涉及到数据一致性、容错性、可用性和可伸缩性等问题，在保证系统稳定性的同时，还要兼顾业务需求。

云计算环境下，云架构师更多的是关注如何利用云计算资源构建快速、可靠、安全、可伸缩的应用程序。因此，他们更偏爱基于微服务架构的服务化设计。服务化架构强调通过定义良好的接口和契约，使服务间通信变得简单、快速和自动化。云架构师还倾向于使用容器化技术部署应用程序，简化应用程序的部署和管理。同时，云架构师更关注应用程序的性能、可靠性和扩展性等指标，通过自动化工具和流程来提升系统性能。

## 职责范围不同
传统架构师一般都参与到整个软件架构的设计和研发中，因此，他们往往在不同的项目和产品中都担任架构师的角色。例如，一个电商网站的架构师可能仅负责电商网站的前端和后端的设计和研发，而另一个零售网站的架构师可能负责交易系统的设计和研发。而云架构师则更关注在云平台上运行的应用程序，因此，他可能会担任企业级云平台的主架构师。

由于传统架构师主要负责整体系统的设计和研发，所以他往往能抽象出一个全面的系统架构，而且很擅长根据业务、技术和市场的实际情况制定系统架构。传统架构师通常会以“务虚”的态度，去寻找问题的根源、做出假设，并试图找到完美的解决方案。他们关注系统的可伸缩性、可靠性、鲁棒性和易维护性等多方面的问题。另外，传统架构师还要考虑兼容性、安全性、可扩展性等复杂的因素，这些要求往往需要大量的研发经验和知识。

相比之下，云架构师所关心的只是如何利用云计算资源构建健壮、安全、可伸缩的应用程序，所以他更重视系统的性能、可靠性和扩展性，并且更偏爱微服务架构。云架构师往往拥有丰富的工程经验，并熟悉很多云计算相关的技术，包括容器编排、网络、存储、数据库等，并且善于利用工具和流程来优化和自动化软件的开发过程。
