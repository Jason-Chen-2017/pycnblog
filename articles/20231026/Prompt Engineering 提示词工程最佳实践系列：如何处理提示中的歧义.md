# 1.背景介绍


作为开发者、测试人员或项目管理者，我们经常会遇到一些让人头痛的问题，比如解决某个难题需要花费很多时间或者资源，导致无法按时完成任务，而我们又不知所以然地发现这些问题发生在哪里，究竟是因为需求定义不清晰、设计不合理、开发不当还是其他原因？

提示词（prompt）是用来描述任务、产品或者其它事物的词汇，一般都有很多层次的含义，比如“给我一份报告”，“提交一个申请”，“请帮忙修改一下表单”，等等。那么，什么时候该使用提示词，什么时候不该使用提示词呢？

提示词引起我们的注意，是因为它能够帮助我们更好地理解任务、产品或者其它事物背后的含义和意图，能够帮助我们更快地了解情况、制定计划并快速行动，还能促进团队间的沟通交流，提升工作效率。但是，使用提示词也可能造成一些误解和困惑，比如在日常交流中，不同的人可能会对同一个提示词下不同的含义理解，甚至产生争执。因此，如何正确使用提示词，尤其是在面对复杂、模棱两可、未知的场景时，就显得尤为重要。

提示词工程是一种工具集，可以帮助我们实现跨部门、跨公司、跨职能、跨行业的服务化方案，以及降低组织、团队、个人之间的沟通成本，缩短各项流程的时间。通过清晰准确地定义业务领域的关键词，以及相应的解决方案、操作指南、实例文档，使得业务相关知识的传播、使用、沉淀变得更加容易、高效。因此，对于提示词工程来说，如何处理提示中的歧义、细微差别、模糊概念、提升效率、改善协作、减少错误，以及其他相关问题都是至关重要的。

# 2.核心概念与联系
## 2.1 术语词典
提示词工程的主要分支是信息提示词词典（Info-Tips Dictionary）。该词典用于定义各种类型提示词，包括产品名词、功能名词、角色名词、场景名词、术语名词、名词组合名词等。其中，名词组合名词是指由多个名词组成的单个名称，如“制造商名称+设备型号”、“客户名称+订单编号”。

此外，除了使用正式的提示词外，还有一些非正式的表达方式，例如，口头禅（即“告诉我”，“再说一遍”，“你说了算”等），疑问句，肯定的否定句等。

## 2.2 信息提示词评审 checklist
为了保证提示词准确、有效、高效，评审人员应首先进行信息提示词初审。评审者应保证以下基本要求：

1. 审核的提示词使用范围应该尽量覆盖全公司、全行业、全国等。如果提示词不够全面，则很有可能造成误导。
2. 在确定采用何种形式时，需考虑提示词的表达风格、语言结构、内容结构、结构连贯性、表达逻辑与完整性、易懂性、精准性、一致性、上下文关系、场景适用性、文法规范性、输入方式限制性、使用频率偏好等方面因素。
3. 通过同行评议，收集、分析业内各类信息提示词的优缺点，根据实际情况选取采用哪些提示词。如：企业网站宣传、App内推送、微信群聊、前台拨打电话、短信通知等。
4. 将信息提示词分类整理。将不同类型信息提示词放在一起比较，根据其特点、作用、应用、场景及易混淆程度，对信息提示词进行分级分类，便于后续的维护。

## 2.3 案例研究
以零售场景下的商品购买提示词“商品名称+价格”为例，下面是一个具体案例：

**提示词**：“商品名称+价格”

**作用**：用于表示某一商品的名称和价格，具有实时反馈作用，可以直观地向顾客展示商品的信息。

**注意事项**：购买按钮须置于商品详情页，一般情况下，用户只需点一下购买按钮即可立刻完成购买，因此不需要额外地提供付款方式、配送信息等，否则影响效率；商品名称要简洁明了，没有冗余或重复信息，避免出现笼统的名称。

**提示词的表达风格**：如“请拿着您的商品名称和价格，准备充值后继续购买”、“感谢您光顾我们的超市，此商品现价为¥XX元”、“小心第三方串改您的购物车内物品信息”、“由于商品金额较大，需要您先支付货款才可以收货”。

上述例子仅代表一种提示词形式，实际场景中，提示词还存在多样化的表达方式，信息提示词词典可帮助我们快速找到、理解、记住各种类型信息提示词的不同表达方式。
