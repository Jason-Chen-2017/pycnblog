# 2.核心概念与联系
## 2.1 IP地址与MAC地址
IP(Internet Protocol)地址和MAC(Media Access Control)地址是两个重要但又容易被混淆的名词。它们都可以唯一标识网络设备。IP地址由一串数字组成，通常为四个十进制数，比如192.168.0.1；MAC地址由一串数字组成，通常为六个十六进制数，比如F8:7B:E6:B6:BD:C8。为了方便区分两者，一般会把前三个字节的IP地址称之为网络前缀（Network Prefix），后三个字节的IP地址称之为主机ID（Host ID）。
## 2.2 DNS域名解析
DNS(Domain Name System)域名解析就是根据域名查询对应的IP地址。DNS服务器记录了全球不同域名所对应的IP地址信息。当用户输入一个网址时，首先会通过本地的DNS解析器进行域名解析，解析出相应的IP地址，然后再访问该IP地址上的Web资源。如果本地的DNS解析器无法解析域名，那么就会向远端的DNS服务器请求解析。
## 2.3 HTTP协议
HTTP协议是互联网上应用层协议的一种，它负责数据通信，使得网页服务器和浏览器之间可以进行双向数据传输。其特点是简单、灵活、易于扩展，并且支持包括HTML、XML等在内的多种数据格式。在TCP/IP协议族中，它建立在TCP协议之上，是一个可靠、无状态的协议。
## 2.4 HTTPS协议
HTTPS(Hypertext Transfer Protocol Secure)，即超文本传输安全协议，是以安全为目标的HTTP通道，利用SSL/TLS protocols进行加密传输，保障了Web网站的机密性、完整性、身份认证和保护隐私。通过配置正确的证书，可以使得HTTPS协议具备很好的安全性。目前，HTTPS已经广泛应用于各类Web服务中，例如维基百科、Youtube、天气预报网站等。
## 2.5 TCP/UDP协议
TCP/IP协议是互联网协议簇，TCP/IP协议族包括TCP协议、UDP协议和IP协议。TCP协议实现了面向连接、可靠的数据流传输，保证数据准确性。而UDP协议则提供面向非连接、不保证数据的顺序、不保证可靠性的数据包传输。IP协议是TCP/IP协议族中的协议，提供底层数据包的传输。
## 2.6 URL地址与URI地址
URL(Uniform Resource Locator)地址表示的是在Internet上某一资源的位置，通常由协议类型、域名、端口号和文件路径组成。比如http://www.baidu.com/index.html。URI(Uniform Resource Identifier)地址是用于标识某一互联网资源的字符串。它由三部分组成：权限部分、SCHEME部分、路径部分。比如，https://docs.python.org/3/library/urllib.parse.html。
## 2.7 请求方法
请求方法指的是对资源的一种操作方式，包括GET、POST、PUT、DELETE等。GET方法获取资源，POST方法用来创建资源，PUT方法用来更新资源，DELETE方法用来删除资源。
## 2.8 Cookie与Session
Cookie是网站服务器存放在用户计算机上的小文件，用来保存用户信息。当用户访问某个网站时，浏览器会自动发送Cookie至服务器。这样服务器就可以根据Cookie信息进行session跟踪，识别用户。Session是在服务端保持状态的方法。客户端生成一个随机数，称之为Session_id，并通过此值与服务器进行交互。服务器可以通过Session_id识别不同的用户，进而处理请求。
## 2.9 CDN内容分发网络
CDN(Content Delivery Network)内容分发网络是构建在网络之上的内容分发系统，依靠部署在边缘服务器上的缓存服务器，通过中心平台的负载均衡，将用户的请求分配到最接近的内容源服务器上，提高网站的响应速度。目前国内外很多站点都在使用CDN服务，如Twitter、YouTube等。
## 2.10 RESTful架构
REST(Representational State Transfer)是一种软件架构风格，其主要目的是使得互联网应用程序的设计风格更加简洁、清晰，更适合互联网的需求。它的结构清晰、符合标准、易于理解、扩展方便，已经成为WEB领域的主流架构。RESTful架构是一种基于HTTP协议、全面健壮的面向对象的软件设计风格。RESTful架构有以下几个特征：
1. 客户-服务器分离：客户机和服务器之间互相独立，通过封装HTTP消息传递来完成通信。
2.  Stateless：所有的资源都是无状态的，每一次请求都必须包含所有相关信息。
3.  分层系统：RESTful架构通过分层的方式，系统中存在多个层次，每个层次都完成单独的功能。
4.  使用标准HTTP协议：RESTful API 接口应该仅仅使用HTTP协议，并遵循HTTP的方法、状态码、URL及Header规范。
