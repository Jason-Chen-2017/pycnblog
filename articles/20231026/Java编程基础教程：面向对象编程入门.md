# 2.核心概念与联系
## 2.1 对象、类、继承和多态
　　面向对象编程（Object-Oriented Programming，简称OOP）是一种基于类的编程方式，通过类（Class）和对象（Object）来进行数据和功能的封装。每一个对象都是类的实例，拥有自己的状态和行为。

　　首先，我们需要认识一下Java中的基本概念：

　　1) 类（Class）：类是一个抽象概念，它定义了一组具有相同属性和方法的对象，每个对象都可以属于某个特定的类。类定义了对象的静态特征，包括属性（如长宽、颜色等）和方法（如move()方法、jump()方法等）。

　　2) 对象（Object）：对象是类的实例，是实际存在的实体，它由类创建而成。对象是运行时的实体，程序在执行过程中创建对象，并通过该对象调用它的属性和方法。

　　Java中类的定义分为三部分：

　　1) 访问控制符：public、protected、private；

　　2) 类名：英文单词首字母大写，如Car、Person、Employee等；

　　3) 主体：包含属性、构造器（用于初始化对象）、方法（用于实现功能）等。

例如下面的代码定义了一个Car类，它有一个name属性和move()方法：
```java
public class Car {
    private String name;

    public void move(String direction) {
        System.out.println("正在" + direction + "行驶...");
    }
}
```

　　对于一个Car对象，我们可以通过new关键字创建它的实例，并设置它的name属性值：
```java
// 创建Car对象
Car myCar = new Car();
myCar.name = "奔驰";

// 使用Car的方法
myCar.move("前");
```

　　当编译这个程序的时候，编译器会检查Car是否有move()方法，因为Car类中已定义了move()方法。然后，编译器生成字节码文件Car.class。

　　接下来，我们再来看下类与类之间的关系：

　　1) 继承（Inheritance）：子类继承父类的所有属性和方法，并且可以增加新的属性或方法。

　　2) 实现（Implementation）：实现指的是接口与类的关系，接口是抽象的类型，而类是具体的实现。

　　3) 组合（Composition）：通过成员变量的方式将两个类相互关联起来。

　　4) 聚合（Aggregation）：通过成员变量的方式将两个类相互引用，使得这两个类可以独立地改变和扩展。

　　继承的形式如下：

```java
class Parent {... }; // 父类

class Child extends Parent {... }; // 子类
```

　　子类可以重写父类的属性和方法，也可以新增自己的属性和方法。

　　实现的形式如下：

```java
interface InterfaceA {... }; // 接口A

class ClassB implements InterfaceA {... }; // 类B实现接口A
```

　　类B可以调用InterfaceA中声明的所有方法。

　　组合的形式如下：

```java
class A {
    B bObj; // 成员变量
}

class B {
    A aObj; // 成员变量
}
```

　　这种方式使得两个类彼此独立，如果要修改其中一个，只能影响另一个，互不干扰。

　　最后，我们来看下Java中的多态：

　　多态是指允许不同类的对象对同一消息作出不同的响应。在Java中，多态体现为方法重载、方法覆盖、抽象类和接口。

　　方法重载（Overloading）是指在同一个类中，允许存在多个名称相同但参数列表不同的函数或者方法。如果一个函数或者方法的参数个数、类型或者顺序与已经存在的函数或者方法完全相同，那么它就是可以被视为重载版本的。

　　方法覆盖（Override）是指子类重新定义父类的方法，它使得子类实例能够调用定义在父类中的方法。

　　抽象类和接口可以用来实现多态。

　　总结：通过类、对象、继承和多态这四个基本概念，我们可以很好地理解Java面向对象编程。
