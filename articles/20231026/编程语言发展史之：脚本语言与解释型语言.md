# 1.背景介绍


脚本语言（script language）和解释型语言（interpreted language），是计算机科学领域里两个主要的语言类型。前者是一种静态编译型的编程语言，后者则是一种动态解释执行的语言。从结构上看，两者最显著的区别在于运行时的机制。脚本语言是在文本编辑器或者命令行界面下直接输入的命令序列，运行时立即对命令进行解释、执行；解释型语言需要首先将源代码编译成机器码并保存为可执行文件，然后再由该文件运行。由于解释型语言不需要预先编译，因此其运行速度更快，但它对于代码的编写要求比较高，也会导致代码的不易维护。
# 2.核心概念与联系
## 2.1 基本概念
### 2.1.1 编程语言定义
编程语言是人类为了解决计算任务而设计的符号语言，用于电脑、手机、网络设备等各种计算机平台之间的数据交换、控制和执行。简单的说，编程语言就是人们用一些符号语言（如英语、日语等）来描述计算机要执行的任务或指令。比如C语言就是一种编程语言，它可以让开发者快速地编写出功能丰富且可靠的应用程序。编程语言可以分为四种类型：低级语言、汇编语言、高级语言和脚本语言。
- 低级语言：又称机器语言，机器码（Machine Code）是指CPU能够直接识别和执行的机器指令集。汇编语言是一种高级的低级语言，它的目标是允许程序员以容易理解的方式来描述他们所处理的计算机的硬件。例如，汇编语言可以被转换成机器码并加载到内存中。
- 中间语言：中间语言（Intermediate Language，IL）是一种低级的编程语言，它经过编译器和解释器处理后得到的目标代码，可以直接被计算机执行。中介语言通常是一种机器无关的语言，通过中间层把编译器的输出翻译成机器能理解的指令。如Java字节码、.NET Microsoft Intermediate Language都是中间语言。
- 高级语言：高级语言（High Level Language，HLL）是基于人的思维习惯来实现计算机编程的一种编程语言。它提供了比低级语言更高的抽象级别、面向对象编程、函数式编程等特性。HLL一般都具有较强的编译性，生成的代码往往可以运行在各种不同的操作系统上。目前广泛使用的HLL有C、Java、Python、JavaScript等。
- 脚本语言：脚本语言是一种以声明性语言形式出现的脚本语言，它是一种高级的解释型语言，具有可移植性和跨平台能力。它通常具有脚本化的特征，只需简单的一小段代码就能完成某个功能。脚本语言可以使用解释器或编译器来执行，运行效率相对较高。如Perl、Ruby、Python、PHP等。
### 2.1.2 脚本语言与解释型语言
脚本语言（script language）和解释型语言（interpreted language），是计算机科学领域里两个主要的语言类型。前者是一种静态编译型的编程语言，后者则是一种动态解释执行的语言。从结构上看，两者最显著的区别在于运行时的机制。脚本语言是在文本编辑器或者命令行界面下直接输入的命令序列，运行时立即对命令进行解释、执行；解释型语言需要首先将源代码编译成机器码并保存为可执行文件，然后再由该文件运行。由于解释型语言不需要预先编译，因此其运行速度更快，但它对于代码的编写要求比较高，也会导致代码的不易维护。如下图所示：


脚本语言的特点包括：
- 跨平台：脚本语言可以在各种操作系统环境下运行，因而具备良好的可移植性。
- 可移植性：脚本语言是跨平台的，可以在多个操作系统平台上运行。
- 易学习：脚本语言语法简单，易于学习和掌握。
- 脚本化：脚本语言只需要一小段代码就可以完成特定任务，与解释型语言相比，灵活性较高。

而解释型语言的特点包括：
- 执行效率：解释型语言在运行时将代码编译成机器码，因此运行速度相对较慢，但是启动速度更快。
- 性能：解释型语言通常执行速度快，对内存要求较低，适合做后台运算任务。
- 不易调试：解释型语言运行时如果出错，难以定位错误位置，造成调试困难。
- 调试困难：调试困难，开发人员无法直接看到代码运行中的变量值。

综上所述，脚本语言的特点是简单、跨平台、易学、脚本化，而解释型语言的特点是执行效率高、内存占用低、调试困难，适用于后台运算任务。实际应用中，采用脚本语言作为编程工具，结合解释型语言作为底层支撑，能够实现快速部署及迭代。
