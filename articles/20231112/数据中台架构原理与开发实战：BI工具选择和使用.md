# 2.核心概念与联系
## 2.1 数据基础设施概览
数据基础设施包括以下几个方面：

1. 数据采集：即将各种各样的数据源（如网站日志、服务器日志、移动设备传感器数据）采集到统一的数据中心进行收集、汇总、整理和处理；
2. 数据存储：将经过清洗和处理的数据按照特定的规则存储起来，支持海量数据的快速检索、分析、计算；
3. 数据计算：利用大数据计算框架对数据进行高速计算和分析，提取有价值的信息，生成新的数据指标；
4. 数据共享：基于数据计算的结果和分析结果，建立数据共享平台，允许不同部门之间的数据交流和共享。

这些构成数据基础设施的四大组件，各司其职，协同工作才能实现数据分析的效率和精准性。而企业级数据平台，则是一套综合性的解决方案，它由多个不同子系统的组合组成，为公司业务领域提供数据服务和支持。
## 2.2 数据中台的构成
数据中台架构是一个集数据采集、存储、计算、共享、应用于一体的平台，它所提供的服务可以实现数据的采集、存储、计算、共享以及智慧应用。数据中台可以分为四大部分：

1. 数据接入层：主要负责数据采集，包括采集方式（离线采集、实时采集）、类型（结构化数据、半结构化数据、非结构化数据）、速度（秒级到分钟级）。
2. 数据存储层：主要负责数据存储，包括数据持久化、高可用、安全性、可扩展性。
3. 数据计算层：主要负责数据计算，包括数据清洗、转换、规范化、维度建模、聚合统计等，并提供高效、易用的查询接口和接口SDK。
4. 数据应用层：主要负责数据应用，包括数据可视化、智能推荐、机器学习、智能搜索、数据分析等，并提供统一的平台。

数据中台除了提供以上功能外，还需要完善数据质量保障机制、数据治理流程、数据驱动业务运营、数据服务标准、数据安全保障等模块。
## 2.3 数据中台的角色与职责
数据中台的角色有四种：

1. 数据仓库管理员（DWA）：主要负责DW的规划、计划、管理、监控、优化和支持。DWA需要具备丰富的IT知识、业务知识和工程经验。他需要掌握DW的概念、原理、架构和技术。同时，他也需要了解业务领域的背景信息，掌握数据情况的详细信息。比如，他需要知道DW的数据来源、目的、种类、数量、质量、时间范围等，并需要跟踪业务的发展变化，根据这种变化调整数据架构。DWA需要善于沟通，了解用户需求，并及时反馈信息。
2. 数据架构师（DA）：主要负责DW的设计、构建、测试、部署、运维。DA需要具备丰富的数据库、中间件、数据分析、分布式计算等技术背景。他们需要擅长业务理解和抽象，掌握DW中的关键技术，包括OLAP、OLTP、数据仓库等。他们也需要熟悉数据采集方式、数据清洗方式、数据转换方式等，以及在不同的计算框架下开发代码。同时，他们需要抓住DW的机会点，探讨最佳的架构设计和技术方案。DA需要具有独立思考能力，能够提出新颖的设计思路，能够应对复杂的部署、运维任务。
3. 数据开发工程师（DSE）：主要负责实现DW的数据开发、ETL、数据驱动业务运营、用户画像、知识图谱等。DSE需要掌握DW的相关技术，包括SQL、NoSQL、数据仓库工具、编程语言、数据平台工具等。他们需要理解业务和用户的需求，提出一系列的解决方案，并帮助公司落地执行。DSE需要协调资源，做好任务分配，确保项目的顺利执行。
4. 数据科学家（Data Scientist）：主要负责数据分析、挖掘和挖掘结果的应用。他们需要掌握数据分析的基本方法论和技术，例如机器学习、数据挖掘、信息检索、图谱等。同时，他们需要对业务现状及其机会点有充分的洞察力，并提出建议和意见，指导公司业务方向的优化和转型。
## 2.4 数据中台的目标与要求
数据中台的目标是在有限的时间内，通过提供一套通用的技术架构和工具，为公司提供数据服务。数据中台必须满足以下几点要求：

1. 服务灵活性强：能够根据公司的业务形态，快速、准确、有效地完成数据服务。
2. 用户友好性高：数据中台的界面设计应当简洁明了，并且符合用户习惯。用户只需简单配置即可使用数据服务。
3. 技术可靠性高：数据中台应当具备高度的可靠性和可用性。
4. 数据精准性高：数据中台的服务应当精准、可信、实时。
