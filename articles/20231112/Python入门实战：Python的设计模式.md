# 2.核心概念与联系
设计模式是一个用来帮助我们解决特定问题的一套方法论。它定义了一种解决问题的方法，可以反复使用。通过采用已有的设计模式，可以简化开发过程、提高代码可维护性、降低软件复杂度，并保证系统的稳定运行。设计模式有很多种，包括创建型、结构型、行为型等。以下我列举几个常用的设计模式。
- 创建型模式：单例模式、工厂模式、抽象工厂模式、建造者模式、原型模式。
- 结构型模式：适配器模式、装饰器模式、外观模式、代理模式、桥接模式、组合模式、享元模式。
- 行为型模式：策略模式、模板模式、观察者模式、迭代子模式、命令模式、状态模式、备忘录模式、访问者模式。

# 6.附录常见问题与解答
1.何为“面向对象”？
   “面向对象”（Object Oriented，简称OO），是一种编程思想，面向对象强调程序中的数据和函数都应该以对象形式出现。它通过封装、继承、多态等机制来组织代码，增强代码的灵活性、可扩展性、可维护性和可复用性。面向对象的程序设计倡导把功能划分成各个相互独立、完整且封装良好的类，每个类封装了数据和操作数据的函数。这样做可以有效地实现模块化、信息隐藏和代码重用，是目前程序设计中最流行的编程方法。
2.面向对象的三大特性是什么？
   OO编程具有三个重要特征：抽象、封装、继承。
   - 抽象：即将客观事物中普遍存在的特征和规律，如人的五官、行为等，抽象出来形成一个独一无二的概念——类，这个概念是人们认识客观世界的有机整体。类通过属性和方法来表现它的特征，属性存储实际的数据，方法负责处理数据。程序员只需要调用这些方法，就可以控制对象的行为。
   - 封装：即把对象内部的实现细节隐藏起来，只暴露给外部客户程序员必须使用的接口，使得客户程序员无法直接访问对象内部的实现。封装可以有效地防止数据被随意修改，提高安全性。
   - 继承：即允许某个类派生自另一个类，新类得到了基类的所有属性和方法，同时还可以新增自己特有的属性和方法。继承是OO的一个重要特征，它让程序变得模块化、灵活、可复用，同时也减少了代码量和避免了重复造轮子的风险。
