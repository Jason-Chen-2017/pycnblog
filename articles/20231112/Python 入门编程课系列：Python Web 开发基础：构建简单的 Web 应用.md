# 2.核心概念与联系
Web开发的一个重要组成部分就是Web服务器。Web服务器负责接收客户端发送的请求并响应数据。但是，这里有一个“客户端”这个词，到底指的是浏览器还是其他客户端呢？
答案是，Web开发里的“客户端”指的是浏览器，也就是我们通常说的前端开发。网站的后台则主要由后端工程师来完成，后端工程师利用相关的技术栈来构建网站的后台服务，提供数据查询、数据分析、订单处理等功能。
所以，这里的“客户端”并不仅仅局限于浏览器，它还包括手机APP、微信小程序、PC桌面应用程序等各种客户端。
为了实现Web开发，你需要掌握以下几个核心概念：

1. HTTP协议
HTTP（HyperText Transfer Protocol），超文本传输协议，是Web开发过程中必不可少的一种协议。Web开发的本质其实就是服务端和客户端之间的数据交换。所有通信的双方都遵守HTTP协议进行数据传输。
2. URL
URL（Uniform Resource Locator），统一资源定位符，用来标识互联网上某个资源地址。它由三部分构成：协议名://主机:端口/路径?参数#片段。
其中协议名：http或者https；主机：域名或者IP地址；端口：默认是80，有些情况下会变成不同的端口；路径：指定访问资源的具体路径；参数：GET方式传递的参数；片段：通常用于定位页面中的某一部分。
3. RESTful API
RESTful API（Representational State Transfer)，表述性状态转移，一种用来定义网络资源的接口标准。它的特点是简单易用、兼容性好、扩展方便。RESTful API广泛应用于移动互联网、web服务、大数据服务等领域。
4. 后端开发
后端开发者除了写代码之外，也要懂得如何设计和架构一个完整的后端系统，这涉及到很多技能，比如数据库设计、安全措施、缓存机制、高可用性、监控告警等。
5. 前端开发
前端开发者通常更关注用户界面和用户体验，他们把精力集中在网站的美观、易用、流畅、高效上面。这涉及到HTML、CSS、JavaScript、jQuery、Bootstrap、AngularJS等技术。
