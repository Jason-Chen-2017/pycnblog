                 

# 1.背景介绍



　计算机网络（英语：Computer Networking）指将分布在不同地点、组织的计算机系统连接成一个巨大的互联网，通过因特网进行信息交流。网络可以覆盖从无线电信道到光纤等各种传输介质，并提供多种功能，如数据传输、文件分享、电子邮件、即时通信、计算资源共享及其它应用服务。

　　当今世界上最大的互联网是基于TCP/IP协议族，它提供了一种简单而有效的通信手段。TCP/IP协议是Internet协议簇中的协议，包括TCP、UDP、IP等。网络层的主要功能是实现不同主机之间的通信，因此需要对网络通信进行管理。通信管理中最重要的是建立可靠的通信链路，确保信息能够按照预定的顺序到达目标。

　　现代网络系统通常都采用分层结构，层次依次从底层物理层到高层应用层。下图展示了互联网体系的基本组成要素：


　　数据链路层（Data Link Layer）：这一层向上封装原始比特流，传送至局域网中。在局域网中，物理层的作用是把比特流转换成信号，再经过传输媒体线路传输到另一端。数据链路层的作用则是把信号转换回比特流。该层主要包括两个子层：物理层和数据链路层接口层。


　　物理层（Physical Layer）：物理层用于定义物理通讯信道的参数，规定了各类传输媒体的特性，例如速率、波长、误码能力等。这一层主要负责处理器与传输媒体之间的物理层连接。它还控制着传输媒体本身的状态，包括打开或关闭传输媒体、调节信号强度等。

　　２．数据链路层（Data Link Layer）：数据链路层负责将分组从源节点通过本地网络传输到目的节点。分组在链路上传输时经过采样、编码、发送、接收、错误检测、解码等过程。它还适应于各种物理网络环境，包括短距离无线局域网、城域网、广域网等。数据链路层的任务之一就是透明地传输分组，使源节点的数据可被所有节点所接收。在数据链路层中，两个子层是以太网媒体访问控制（Ethernet MAC）和逻辑链路控制（LLC）。

　　　　1) 以太网媒体访问控制(Ethernet MAC): 是在局域网（LAN）上使用的以太网规定的数据链路层协议。MAC协议是IEEE标准化组织1980年制订的，它允许在多个主机之间直接发送以太网帧，而不需要任何路由配置。它还支持广播、多播、自动地址配置、差错控制等。

　　　　2) LLC: 负责透明地传输分组。它提供的数据链路层服务，包括对帧的封装和释放，报文的传递、流量控制和差错校验等。

　　３．网络层（Network Layer）：网络层实现从源主机到目的主机的通信路径选择和数据传输。网络层向上只提供简单灵活的、尽力而为的网络服务，不保证可靠性、QoS、流控等高级功能。网络层的主要任务是选择一条或多条通信路径，确保数据按顺序到达目的地。网络层的功能包括寻址、路由选择、拥塞控制、转发表管理、网际互连等。网络层的实现方式通常是协议栈，但也有一些应用层协议直接使用网络层的服务。

　　４．传输层（Transport Layer）：传输层用来提供进程间通信，包括端到端的可靠数据传输、面向连接、可靠连接和运输信道管理。在传输层中，有两种基本协议：传输控制协议（Transmission Control Protocol，TCP）和用户数据报协议（User Datagram Protocol，UDP）。传输控制协议是提供可靠数据传输的网络层协议，它提供面向连接的、可靠的、基于字节流的双向数据流服务。用户数据报协议是提供不可靠数据传输的网络层协议，它提供无连接的、尽最大努力的、基于数据报的单播传输服务。

　　　　1) TCP: 由四个序号字段、一个确认字段、窗口字段、选项字段、校验和字段、紧急指针字段共八个字段构成，提供可靠数据传输。TCP协议采用滑动窗口机制来进行流控，避免网络拥塞。

　　　　2) UDP: 不提供可靠的、基于字节流的服务，只是提供单播、多播、一对一、一对多、多对一、多对多的透明数据包传输。

　　５．应用层（Application Layer）：应用层为用户提供应用程序接口，包括文件传输、电子邮件、数据库查询等。应用层决定了向用户提供什么样的网络服务。应用层协议常用的有HTTP、FTP、SMTP、Telnet、POP、NNTP、TFTP等。
