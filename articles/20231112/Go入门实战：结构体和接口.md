# 2.核心概念与联系
## 概念解析
### 结构体
在Go语言中，结构体(struct)是一个用于定义各种数据类型的数据类型。每个结构体都有一个或多个成员变量，这些成员变量可以是不同的数据类型，也可以是其他的结构体类型。结构体可以实现面向对象编程中的对象属性和方法的概念。以下是简单的结构体示例:

```go
type Person struct {
    name string
    age int
}
```

Person结构体有两个成员变量name和age，分别对应于字符串类型和整数类型。可以通过new()函数创建结构体类型的实例。例如：

```go
p := new(Person)
p.name = "Alice"
p.age = 27
```

实例p代表一个Person类型的结构体对象，可以通过访问结构体的成员变量对其进行操作。

```go
fmt.Printf("Name is %s and Age is %d", p.name, p.age) // Output: Name is Alice and Age is 27
```

### 接口
在Go语言中，接口(interface)是一组抽象的方法签名。它定义了一种类型必须实现的方法，但不关心如何实现。接口由一组接口声明语句定义，其中每一条语句指定了一个方法签名。接口中的所有方法都是抽象的，不能有实际的实现。接口类型的值可以保存任何实现了该接口的类型的值。可以使用接口类型作为参数传递给接收者类型。以下是简单的接口示例:

```go
type Human interface {
    Speak() string
}

type Employee struct {}

func (e *Employee) Speak() string {
    return "Hello"
}

h := &Employee{}
var i Human = h
fmt.Println(i.Speak()) // Output: Hello
```

在这个示例中，Human接口仅有一个方法Speak(). 通过匿名结构体类型实现了Human接口。而Employee结构体类型实现了Speak()方法。在main函数中，Employee类型值h被赋值给接口类型Human i。因此，可以通过调用i的Speak()方法打印出"Hello"。

# 6.附录常见问题与解答
1.为什么要使用结构体？
结构体提供了一种灵活的方式来组织和管理数据，让代码更加容易阅读和理解。它允许将相关联的数据打包在一起，避免过多的参数传递，同时也简化了代码的修改和扩展。

2.什么是结构体的比较？
结构体的比较基于内存地址比较。如果结构体的指针相等，那么它们所指向的结构体也是相同的；否则，结构体是不同的。

3.什么是嵌套结构体？
嵌套结构体就是一个结构体里面包含另一个结构体，通常情况下，嵌套的层次越深，使用的内存空间就越小。

4.为什么要使用接口？
接口提供了一种协议，它定义了类型的行为，但不强制定义它的内部工作机制。它允许不同类型的对象之间进行交互，使得代码具有更好的可复用性和扩展性。

5.接口值的动态类型检查过程是什么？
当我们调用某个接口值时，动态类型检查会根据当前对象的类型和实现了接口的类型进行判断。只有当接口值和对象实现了同一个接口，并且这个接口的所有方法都可以被调用，才算是成功调用。

6.什么是反射？
反射是在运行时（runtime）检查程序的结构和内容，并借此构造和操纵程序的能力。通过reflect包中的Type和Value结构，可以获取运行时的类型和值信息。

7.如何创建自定义接口？
创建一个自定义接口需要提供至少一个方法签名。自定义接口可以方便的进行封装、继承和多态。

8.接口的赋值和复制过程是怎样的？
当我们把一个接口值赋予另一个接口值时，实际上只是保存了一个对底层对象的引用。当我们把一个接口值复制给另一个接口值时，实际上也是保存了一个对底层对象的引用。但是当我们调用接口方法时，就会触发一次动态类型检查。
