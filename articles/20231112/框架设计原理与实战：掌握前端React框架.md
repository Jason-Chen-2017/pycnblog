2.核心概念与联系
## 1.什么是React？
React（Rea ctive，反应性）是一个用于构建用户界面的JavaScript库。它是由Facebook于2013年创建的，并开源。React最早由知名的Instagram背后的工程师<NAME>以及工程师Michael Jackson领导开发。它的特点是构建可重用的UI组件，利用虚拟DOM提升性能，并且可以配合其他工具实现完全动态的应用。 Facebook甚至声称React是当前最热门的前端JavaScript框架。 

## 2.React的特点
- 使用JSX语法定义组件。 JSX是一种类似XML的语法扩展，用来描述HTML，它也是JS的一个子集，可以在JS文件中直接编写。
- 数据驱动视图。React通过数据渲染视图。通过setState()方法触发重新渲染，从而更新视图。
- 单向数据流。React遵循单向数据流，即父组件的数据流动到子组件，但不能反过来。也就是说，数据的改变只能通过props或者state方式向下传递，不允许直接修改。
- Virtual DOM。Virtual DOM 是 React 在内部使用的一种数据结构。它是真实 DOM 的一个克隆版本，不会影响实际的 DOM 。当状态发生变化时， React 会自动计算最小的差异，然后只更新发生变化的部分，使得浏览器保持响应速度。
- 组件化。React支持强大的组件化编程方式。你可以把你的应用划分成多个组件，每个组件封装自己的逻辑和 UI ，使得代码更加模块化、易维护。
- 轻量级。React体积小，只有几个核心的模块，加载时间短，适合移动端和PC端开发。
- 更好的可伸缩性。React的组件化设计使其更易于改造和扩展。你可以将已有的组件放入不同的应用中，也可以通过组合这些组件来实现复杂的界面。
- 可预测的渲染结果。React采用异步并行的架构，保证每次渲染的结果都是一致的。这可以避免因不可抗力（比如动画、AJAX请求）导致的闪烁感。

## 3.React的应用场景
### 1.移动端APP开发
React Native 是 Facebook 提供的一个针对 Android 和 iOS 平台的开源移动端跨平台开发框架，支持开发 native 应用一样的效果，但是开发效率更高。它是用 JavaScript 来编写组件，运行于原生环境，并且有着类似 React 的声明式编码风格。因此，你可以很容易的从 JSX 模板生成 native UI 组件。通过 JSX，你可以将组件拖拽到屏幕上，调整布局，设置样式属性。其最大优势就是你只需要学习 JSX 语法，就可以快速的开发出高质量的应用。

### 2.大型 Web 项目
React 可以帮助你开发大型 Web 应用。Facebook 的 Instagram 使用 React 来构建流畅的用户交互界面。在这种框架中，组件化的开发模式可以有效的管理应用中的代码，使得应用的代码量减少，同时也方便开发者进行功能模块的重构和迭代。React 的另一个优点就是它强大的服务器端渲染 (SSR) 技术，可以让你在服务端生成 HTML 页面，然后直接发送给浏览器，无需等待 JS 脚本执行完毕，可以大幅度提高网站的首屏渲染速度。

### 3.服务器渲染
由于 React 的独特特性——声明式编码风格，以及 Virtual DOM 机制，它被设计用来解决一般 web 应用中遇到的那些难题。如果你有基于 Node.js 的后端开发经验，那么 React 服务端渲染 (SSR) 可能是你最喜欢的技术。因为它可以将你的组件渲染成 HTML 字符串，然后再传送到浏览器显示。这样的话，用户在查看页面的时候就不会看到花费几秒钟的时间去加载 JavaScript 文件，而且搜索引擎也能收录到页面内容。而对于用户访问没有渲染的静态页面，则可以直接返回静态的 HTML 文档，不用等待 JS 执行完成。

## 4.React的架构设计
React的架构设计理念源自Unix哲学中的“everything is a file”，所有东西都可以看作文件，包括代码文件、配置文件、日志文件等等。为了更好的组织代码，React将组件化思想应用到了整个架构设计中。整个React架构由以下主要的模块组成：
- Core Module：核心模块包括调度（Reconciliation），渲染（Rendering），状态管理（State management）。
- Rendering Module：渲染模块处理组件树的生成及更新。
- Component Module：组件模块负责定义组件及其生命周期函数。
- Events Module：事件模块用于处理事件相关事宜。
- API Module：API模块用于处理与其他模块之间的接口调用。
