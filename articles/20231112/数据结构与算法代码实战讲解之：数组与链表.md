                 

# 1.背景介绍


## 什么是数组？
数组（Array）是一种用于存储一系列相同类型元素的数据结构。其特点是在内存中连续分布，可以随机访问。数组支持高效的随机访问，可以通过索引下标直接访问到数组中的元素。数组一般用一对方括号`[]`表示，元素之间通过逗号分隔。例如：`int arr[5] = {1, 2, 3, 4, 5};`。
## 为什么要用数组？
数组主要用来进行快速且方便的数据存取，尤其在某些场景下，比如动态集合、矩阵求和等等，它能提供相当大的加速优势。因此，应用到具体的问题上时，应当首先考虑是否可以转化为数组形式，而不必着急开发时间的投入。
## 数组有哪些优点？
### 便于编程
由于数组的连续性，对数据的访问速度非常快。这是因为通过索引访问的元素实际上都是数组中的物理位置，不需要进行任何计算或偏移量的计算。同时，对数组的操作也可以通过指针完成，使得程序逻辑更清晰易懂。此外，采用数组还有一个好处就是可以在运行时由编译器进行边界检查，避免数组越界的问题。
### 使用简单
在数组中，插入和删除元素只涉及两个地方，即数组起始地址和下一个空闲位置即可。因此，对于小规模的数据来说，数组操作比较容易实现。而且，由于数组大小固定，增加或删除元素时并不会导致数据的搬移，因此也不存在内存空间浪费的问题。因此，数组在很多情况下都是一个很好的选择。
### 支持动态扩容
对于一些大型的数据集来说，由于内存空间有限，无法一次性分配足够的内存空间，这种情况下，可使用动态扩容的方式来解决这个问题。数组支持在运行时自动扩容，这就保证了数组的适应能力。
### 提升性能
数组在某些时候还可以提升程序的运行性能。比如，假设有一个需要排序的整数列表，如果使用二叉查找树（Binary Search Tree），那么每次查找的时间复杂度都为logN。但是，如果采用数组的方式，只需将列表划分成一个个子列表，每个子列表的大小为500~1000，就可以使用快速排序算法，这样的话，单次查找的时间复杂度降低为O(n)，比二叉查找树要快得多。
# 2.核心概念与联系
## 单向链表
单向链表（Singly Linked List）是指链表中只有一端的链表，其中每一个节点除了保存数据之外，还保存下一个节点的地址。链表的头节点称为第一个节点，指向第一个节点的指针称为头指针。如下图所示：
单向链表的定义为：

```C++
struct ListNode{
    int val;
    struct ListNode* next;
}；
```

这里的`val`变量保存该节点的数据，`next`指针则指向下一个节点。每个节点都至少占用两个内存空间，即保存数据和下一个节点地址。

## 双向链表
双向链表（Doubly Linked List）是指链表中有两端的链表，每一个节点除了保存数据之外，还保存前一个节点和后一个节点的地址。双向链表可以从任意一个节点出发遍历整个链表。如下图所示：
双向链表的定义为：

```C++
struct Node{
    int data;
    struct Node* prev; // pointer to previous node in doubly linked list
    struct Node* next; // pointer to next node in doubly linked list
}；
```

这里的`data`变量保存该节点的数据，`prev`和`next`指针分别指向该节点的前一个节点和后一个节点。每个节点都至少占用三个内存空间，即保存数据、前一个节点地址和后一个节点地址。

## 循环链表
循环链表（Circular Linked List）是指最后一个节点的指针域指向头结点，形成一个环状结构。它简化了指针的管理，增强了表现力，所以在某些场合下会用到。如下图所示：
循环链表的定义为：

```C++
struct Node{
    int data;
    struct Node* next;
}；
```

这里的`data`变量保存该节点的数据，`next`指针指向下一个节点。每个节点都至少占用两个内存空间，即保存数据和下一个节点地址。

## 链表的操作
### 插入操作
插入操作可以分为两种情况：
1. 在头部插入新的节点
这种方式插入的新节点就变成了头节点，因此需要更新头指针。
2. 在中间或者尾部插入新的节点
这种情况插入的新节点的上一个节点的`next`指针需要更新，然后新节点的`next`指针指向当前节点。

### 删除操作
删除操作又分为两种情况：
1. 删除头节点
这种情况删除的节点就是头节点，因此需要更新头指针。
2. 删除非头节点
这种情况删除的节点的前一个节点的`next`指针需要更新。

### 查找操作
查找操作基本上与数组一样。

### 链表 vs 数组
相较于数组，链表的优势有以下几点：

1. 插入操作快，因为不需要移动元素。
2. 删除操作快，因为只需要修改指针域。
3. 可不按顺序存储，链表可以随意添加或删除节点，数组则只能追加。

不过，链表本身的额外消耗也不是零。比如，给定一个链表的长度为L，为了支持随机访问，链表通常至少需要额外L个指针，而数组不需要。同样地，在内存中的分配也不同，链表可能需要更多的内存空间。

综上所述，数组和链表各有千秋。在实际的应用中，需要根据实际情况做出取舍。
