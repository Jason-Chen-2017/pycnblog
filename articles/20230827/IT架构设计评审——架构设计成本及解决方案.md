# 1.简介
  

“架构设计成本”作为IT行业中一个非常重要的指标，能够准确地衡量企业在技术上是否已经形成了一套成熟、可持续的架构体系。但是，如何客观地评估这种“架构设计成本”是一个具有挑战性的问题。特别是在快速变化的时代背景下，如何合理有效地分配资源和管理风险，才能够帮助企业更好地为业务创造价值并提升竞争力？

为了解决这个问题，笔者根据现实需求，结合中国银行集团“支撑平台”自主研发的IT架构设计能力评估工具，对“架构设计成本”进行了科普及有效管理的方法论。通过此工具，企业可以快速了解自己的IT架构设计情况，对其进行评估分析，找出改进点，从而做到“架构设计成本”可控。

# 2. 概念和术语
## 2.1 “架构设计”的概念和定义
“架构设计”（Architecture Design）是指根据系统规划、功能需求、性能参数等要求，经过多次迭代演化，最终确定系统的结构、组件、服务等构架和布局，以满足系统性能目标、可靠性、可用性等要求的工程过程或方法。

## 2.2 “架构设计成本”的概念和定义
“架构设计成本”是指企业在设计、构建或维护IT架构过程中所花费的时间、金钱和资源。其主要包括以下三个方面：

1. 开发时间——指企业开发新功能或修复已知漏洞所耗用的时间。
2. 测试时间——指企业在开发新功能或修复已知漏洞后，需要测试整个系统的总时间。
3. 运维时间——指企业需要重新安装服务器、配置网络设备等过程所耗用的时间，这直接影响系统的运行效率。

基于上述三个方面，“架构设计成本”被定义为：

$$ArchCost = DevTime + TestTime + OpsTime $$ 

其中DevTime，TestTime和OpsTime分别代表开发时间、测试时间和运维时间。

## 2.3 架构设计评审流程
架构设计评审流程一般分为以下五步：

1. 确认目标
2. 收集信息
3. 提出假设
4. 使用模型/工具评估
5. 结果呈现及讨论

其中第四步通常由架构师或项目经理进行。本文将以前三步为主线展开。

## 2.4 模型和工具
本文将使用“银行集团“支撑平台”自主研发的IT架构设计能力评估工具，即“支撑平台”的架构设计评审模块。

该工具主要有以下几个特点：

1. 直观易用：交互式界面，图表展示，对于没有编程基础的用户来说，也容易上手。
2. 准确度高：采用基于业务模式和技术特征的指标体系，既考虑经济效益，也考虑社会效益。
3. 集成度强：平台内置数据模型、规则引擎、自动生成报告等一系列工具，无需独立部署。
4. 全面的评估指标：包括用户满意度，成本和效率，安全性，可扩展性，稳定性，可管理性，可移植性，鲁棒性等各项指标。

除此之外，还可以在平台上查看到所有参与人员的IT架构设计心得，不仅有助于了解整体架构设计状况，也能帮助决策者及管理者快速发现问题，避免技术债务积累，节约资源开支。
