                 

# 1.背景介绍


## 什么是编程规范？
在计算机编程中，代码规范是一个很重要的习惯，它能确保项目代码的一致性、可读性和简洁性。对于一个团队而言，代码规范可以减少代码混乱、提高效率和质量，增加协作合作的能力，为后续维护提供便利。有些公司甚至规定所有的代码都要符合某种代码规范，从而让其他开发者更容易阅读、理解和改进代码。因此，代码规范在软件开发中扮演着十分重要的角色。
## 为什么要做代码规范？
优秀的代码规范能够帮助开发者编写出更易于阅读、理解和维护的代码。以下几点可以作为衡量标准：
- 提高代码可读性。代码规范可以让代码更加易于被人理解，降低代码维护成本，提高代码的稳定性。
- 促进团队工作协作。代码规范有利于团队间的沟通交流，提高代码的交叉认识，也有助于团队成员之间建立信任关系。
- 增强代码质量。代码规范可以检查代码中的错误、漏洞和逻辑错误，并能够在代码提交前发现这些错误，保证代码质量的提升。
- 提高软件质量。代码规范能够对软件的运行速度、资源消耗等方面进行优化，改善用户体验。同时，代码规范也会给工程师带来额外的锻炼机会，培养他们的编程风格和技术素养。
# 2.核心概念与联系
## 概念
### PEP 8——Python编码规范指南
PEP（Python Enhancement Proposals）全称Python增强建议，由Python官方社区制订。其内容涉及Python语言各方面的相关议题，如文档规范、语法约定、代码风格等。PEP 8 是关于 Python 代码样式指南的一个发布版本。
### 代码行长度限制
一般情况下，每行代码字符数不超过79个，因为在较短的显示器上，有时无法完整显示79个字符。另外，更长的代码行可能影响代码的可读性。

如果你的编辑器或者IDE支持自动换行，那么在每行结束处按下回车键即可实现自动换行。

对于超长的函数或语句，建议考虑将它们拆分为多个小函数/方法。

### 分号
每个语句末尾都应该加上分号；

不要使用分号增强语句的可读性；

### 缩进
使用4个空格作为Python代码的缩进，不要使用Tab缩进。

```python
if True:
    print("True")
else:
    print("False")
```

### 函数名
函数名尽量用小驼峰命名法，即将单词首字母大写，然后每个单词的第一个字母小写，例如`def myFunction():`。

### 变量名
变量名通常用小驼峰命名法，不过也可以使用下划线命名法，例如`my_variable = "Hello"`。

### 类名
类名通常用驼峰命名法，例如`class MyClass:`。

### 模块名
模块名通常用小写下划线命名法，例如`import my_module`。

### 文件名
文件名一般采用小写字母和下划线组合，例如`hello_world.py`。
