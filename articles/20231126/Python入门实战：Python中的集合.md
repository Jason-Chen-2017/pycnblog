                 

# 1.背景介绍


## 什么是集合？
在计算机科学中，集合（Set）是指一个无序且元素不重复的集。它可以用于表示对象、元素或数据的集合。同样，在数据结构中，集合通常被用来存储具有相同特性的数据元素。Python 中的集合是内置类型之一，它提供一种高效的方法来处理非重复元素的集合。
## 为何要用集合？
集合有很多重要的应用场景。比如，假设我们需要实现一个游戏角色的攻击能力。这个角色可能有多个武器，而武器的组合也可能十分复杂。因此，我们可以使用集合来存储角色的攻击武器，并根据攻击力强弱进行优先排序。如果某个角色没有任何可用的攻击武器，那么我们还可以提醒他们装备武器。再如，对于文本的处理，我们可以将文本中的词语、字符等归类到不同的集合中，以便于后续的分析。
此外，集合提供了许多独特的运算方法，比如交集、并集、差集等，使得对集合的操作更加方便。另外，集合支持迭代器协议，可以让我们对其中的元素进行快速遍历。
# 2.核心概念与联系
## 集合相关概念
### 元素（element）
集合中唯一的基本单位。集合中的元素都具有相同的属性或者特征，并且不能有重复。换句话说，集合中的每个元素都是独一无二的。
例如，集合{a, b, c}中元素有{a, b, c}三种。
### 空集（empty set）
空集是一个没有元素的集合。它的真正含义取决于使用的上下文。
例如，某些语言的语法中，空集用 {} 表示。
### 子集（subset）
如果 A 是 B 的子集，则称 A 是 B 的真子集，记作 A ⊆ B 或 B ⊇ A。
例如，集合 {1, 2, 3} 是集合 {1, 2, 3, 4, 5} 的子集。
### 超集（superset）
如果 A 是 B 的超集，则称 A 是 B 的真超集，记作 A ⊃ B 或 B ⊉ A。
例如，集合 {1, 2, 3, 4, 5} 是集合 {1, 2, 3} 的超集。
### 相等关系（equality relation）
当两个集合含有的元素完全一样时，它们就是相等的。
### 不相等关系（inequality relation）
当两个集合含有的元素不完全一样时，它们就是不相等的。
例如，集合 {1, 2, 3} 和集合 {3, 2, 1} 就属于这种关系。
### 合集（union）
两个集合的合集指的是同时包含了这些元素的所有集合，即 A ∪ B 。
### 补集（complement of a subset）
如果集合 X 是一个给定的集合 A 的子集，那么 A 的补集 C 定义如下：C = A - X ，其中 X 指的是 A 的子集，X 是一个集合，而 C 也是集合。A 的补集指的是所有不属于 A 的元素构成的集合，记作 A' 。
### 交集（intersection）
两个集合的交集指的是同时存在于这两个集合中的元素组成的集合，即 A ∩ B 。
### 并集（union）
两个集合的并集指的是所有元素都属于这两个集合中的所有元素组成的集合，即 A ∪ B 。
## 集合操作的性质
### 对称性
对于两个集合 A 和 B，若 A ⊆ B 且 B ⊆ A，则称 A = B。
### 传递性
对于三个集合 A、B、C，若 A ⊆ B 且 B ⊆ C，则称 A ⊆ C。
### 结合律
对于四个集合 A、B、C、D，如果 A ∩ (B ∪ C) = (A ∩ B) ∪ (A ∩ C)，则称该等式成立，记作 A ∩ (B ∪ C) ≅ (A ∩ B) ∪ (A ∩ C)。
### 反身性
对于任意集合 A，若 A ⊆ A，则称 A 是自身的子集。
