# 1.简介
  

本文主要介绍了工程项目评估过程中所需面对的问题和相应的方法论。基于项目经验和专业知识编写。
# 2.工程项目评估的目的及意义
工程项目评估是软件开发过程中的重要环节之一。其目的在于确定新软件或服务是否能够满足客户的需求、解决实际问题以及预期目标。而工程项目评估的重要性不亚于需求分析、设计、编码、测试等环节。由于项目的大小、难度和风险性都不一样，因此需要采取不同的评估手段，以期达到事前对项目的管理、控制和预测。
# 3.工程项目评估所需面临的问题
工程项目评估通常分为2种类型：
1. “外”评估——组织与审计部门进行系统性的整体评估，即“项目管理计划”。它包括项目概述、资源需求、预算、进度安排、人员配比、过程改进建议等方面，从而为项目的正当性进行评价。
2. “内”评估——软件开发人员进行功能测试、兼容性测试、性能测试、安全测试等环节的测试工作。它包括开发成果的可行性、完整性、可靠性、用户满意度、反馈速度等指标，通过测试后提出质量改进建议，并根据这些建议对软件进行优化。
以上两种评估方式各有优缺点。“外”评估更侧重总体管理，适用于复杂、长期的项目；而“内”评估更注重软件产品本身的质量，适用于短时间内完成的项目。对于一些较小、简单、独立的项目，一般只做“外”评估；而对于成熟的大型项目，可以结合“外”评估与“内”评估，提高项目质量、降低开发成本、增强项目的生命力。
工程项目评估中还会遇到以下问题：
1. 项目难度难以估算
软件开发过程中，不同阶段的复杂程度存在差异。因此，就项目的难度难以准确衡量。同时，也没有统一的标准和方法来衡量不同阶段的复杂程度。
2. 测试不充分、测试环境不匹配
测试是项目评估的关键环节。但现实世界的软件开发往往是一个动态的环境，测试环境必然与开发环境存在差异。测试时的设备、网络环境、依赖项版本等都会影响测试结果。
3. 文档缺乏有效性
软件开发过程中，除了文档外，还有很多其他形式的记录信息。比如，可以把设计文档、测试用例、Bug列表、变更日志等集中存储起来，便于后续的评估。但是，这些文档往往过时、不全，并且很难跟踪实际情况。
4. 没有及时反馈
许多项目的重要环节（如需求、设计、编码）都是由多个团队协同完成的，每个人的工作进度及产出的文档不同。而工程项目评估则是各个团队相互配合，共同评估项目的进度、成果和质量。如果不能及时提供反馈，可能导致项目延期或者质量问题无法解决。
# 4.工程项目评估的基本原理
工程项目评估的基本原理有一下几个方面。
## 4.1 开放问题
工程项目评估的目的是评估项目是否适合做，要回答这个问题，首先要问清楚客户真正想要什么？项目将产生什么样的收益？客户会如何支付费用？项目应该完成什么任务？需要考虑这些问题才能做出评估报告。
## 4.2 客观分析
工程项目评估一般都是依据客观事物的特征进行的，客观性要求非常高。工程项目评估应有客观性，不能出现主观性和主观情绪。例如，对于项目进度不达标，评估人员应坦率地说出原因，不能指望工程师们自己能够快速发现和纠正错误。
## 4.3 集中反映
工程项目评估往往集中反映项目的所有方面，包括项目管理、人员培训、业务拓展等。工程师要有把握自己在整个项目中的作用，对自己的工作负责。
## 4.4 对称性
工程项目评估在评估过程中应保持一致性和对称性。严格按照评估协议的内容进行评估，避免虚假宣传和操纵结果。任何个人或团队都不能贬低他人的工作能力。
# 5.工程项目评估的常用工具和方法
工程项目评估中，有些工具和方法是必须要有的，比如，评估表格、评估流程图、评估记录等。另外，也有很多工具和方法能够帮助工程师更好地理解、掌握项目管理，提高效率。工程师需要了解项目管理的一些相关概念、原则、规范和方法，以及评估工具和方法。
## 5.1 评估表格
评估表格是工程项目评估的基础工具。它包括项目背景、范围、预期目标、约束条件、计划、资源分配、进度安排、考核标准、报告格式等方面。评估表格能够反映项目的主要方面，为之后的评估提供参考。
## 5.2 评估流程图
评估流程图用来描述工程项目评估的过程。它从工程项目评估的起始、逐步完善到最终完成三个阶段。工程师可以在这个图上看到各个环节之间的联系，以及评估人员的工作职责和角色。
## 5.3 评估记录
工程项目评估需要严谨、公正地记录。评估记录应采用结构化的方式，包括日期、时间、参与评估人员、评估内容、评估结论等。工程师可以从记录中了解到评估情况，为之后的评估提供依据。
## 5.4 会议纪要
会议纪要能够集中反映工程项目评估的各种信息，并提供给所有相关的人员。工程师可以利用会议纪要总结评估结果、提出改进建议，以及向领导层汇报。会议纪要可以起到工程项目评估成果传递、评估效果评估的作用。
## 5.5 其他评估工具
除了以上几种评估工具和方法外，还有其他各种评估工具和方法。工程师在选择评估工具和方法的时候，应当综合考虑项目的特点和实际情况，选取最适合的工具和方法。
