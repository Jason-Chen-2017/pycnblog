# 1.简介
  

Occupation codes是一门新的IT职业分类系统。它通过建立职业相关词汇的联系，从而将职业分为不同的类别，如研发、销售、客服等。这一系统的目的是帮助职业经理人更好地了解各个职业的要求、偏好和喜好。

2017年，在推动“职业编码”工作的过程中，英国国防部与英国政府机构共同发布了一份“Occupation Codes for the Future”（OFC）白皮书。该文件对未来职业分类的发展有了更加详细的阐述。其中提到，“Occupation Codes will allow professionals to better understand the requirements, preferences and preferences of each occupation.”它将基于以下几个方面建立职业代码：
1. 主要需求或职责要求
2. 技能要求
3. 适应性
4. 生活习惯
5. 情感需求

根据OFC的定义，职业代码应该具有如下特征：
1. 可追溯性：代码应该可以追溯到某个起源（通常是一个相关的政府部门），并且应该被记录下来并分享出去。
2. 专业性：应该涵盖专业领域的多个学科（例如，IT、医疗、金融、教育）。
3. 尊重生命：代码不应该仅仅根据表面的情况进行划分，应该由个人、组织或国家本身来决定职业的长期发展方向和社会价值。
4. 更多元化：代码应包括多种类型的人才，例如，有些职位需要熟练的工程技能、有些则需要具有艺术才能、有些则需要解决复杂的问题。
5. 人际交往：代码应反映不同个体之间的关系。比如，会计和财务分析师可能属于同一个类别。

OFC还建议，对于新出现的职位，应允许职业经理人们先定义自己的职业代码，然后其他人可以参考该代码。这样可以减少歧义，避免因一些职业缺乏共识导致的错误。
