## 1.2 相关背景

### 1.2.1 隐私计算

隐私计算技术旨在通过研究和开发技术来保障个人信息安全的保密性、可用性和完整性。通过采用加密技术、隐私保护协议和数据安全措施，隐私计算技术可以提供比传统机密共享方法更高的安全性。

隐私计算的两个基本要素是：

1. 数据保密性：要求对所有参与者的输入数据进行加密，确保只有授权方才可查看数据的内容；
2. 计算隐私性：保证参与者只能看到由自己输入或选择的数据的输出结果，无法得知其他参与者的任何数据输入。

其中，加密技术应用广泛，主要用于保障数据隐私的有效性。目前主流的加密方式有对称加密和非对称加密，对称加密的实现方案如AES、DES、3DES等，速度较快，使用简单；而非对称加密则需要配套的数字证书机制，解决了数据传输过程中安全风险的问题。

隐私保护协议则用于定义计算流程的规则，确保各参与者间的通信安全。在该层次，主要关注两种协议：差分隐私协议和同态加密协议。差分隐私协议允许多个参与方共享数据，但数据中的信息都是随机的，不能被恢复。同态加密协议则允许多个参与方分享相同的数据，但对每个参与方而言，其看到的数据都是相同的，且无论何时，其所看到的数据都符合预期。

数据安全措施则包括多种防御机制，如访问控制列表（ACL），数据加密，数据隔离，反垃圾邮件等。访问控制列表可以限制访问者对于数据的读取权限，数据加密可以将敏感数据加密，数据隔离可以实现各个业务线的数据不被彼此影响。反垃圾邮件则可以过滤垃圾邮件发送者，保障通信质量。

### 1.2.2 大数据

“大数据”指海量的数据集合。数据量越来越大、特征越来越复杂、价值越来越丰厚，传统的软件和硬件无法有效处理和分析大数据。为了解决这一问题，出现了很多的解决方案。其中，分布式文件系统 Hadoop 经过几十年的发展，成为处理大数据的重要组件之一。Hadoop 平台的基础是 HDFS (Hadoop Distributed File System)，它是一个由廉价商用服务器组成的网络分布式文件系统。HDFS 提供了一系列支持海量数据存储和处理的功能，如备份、容错、负载均衡等。

Hadoop 的另一项核心功能是 MapReduce，它是一个编程模型和运行环境，用于轻松处理大规模数据集。MapReduce 定义了两个阶段，第一阶段是“映射”阶段，即把输入数据切片并分配到不同的节点上执行；第二阶段是“归约”阶段，即把不同节点上的结果汇总为一个结果。MapReduce 模型可提供高效且容错的计算能力。

另一种重要的大数据处理技术是 Apache Spark。Spark 是开源的内存计算框架，在 Hadoop 和 Flink 之后又一个崛起的大数据处理技术。Spark 可以利用内存进行计算，有效降低计算资源消耗。与 Hadoop 相比，Spark 更加便于快速迭代和部署应用。

### 1.2.3 数据隐私保护

当前，数据隐私成为政府和企业共同关注的重点议题。国家也在落实数据保护法律法规，推动各种监管制度的建立。数据隐私保护应当作为政府监管的一部分，为个人的合法权利提供保障。

数据隐私保护包含四个维度：

1. 数据收集、存储和管理
2. 数据使用及应用
3. 数据共享与转让
4. 数据安全

### 1.2.4 微众银行

“微众银行”是中国乙⻓银行数字货币团队研发的区块链金融服务平台，为实体经济公司和个人提供全面的数字货币服务。微众银行将基于区块链技术，打造企业级去中心化的金融生态系统，实现真正意义上的“数字货币化”。

微众银行主要包括如下功能：

1. 存币：支持实体经济公司发行自己的数字货币，并通过跨境支付互联网电子钱包实现支付和结算；
2. 兑换：通过智能合约智能识别用户身份、验证交易资金、兑换加密货币并自动计费；
3. 交易：提供基于信托记账的资产组合管理和资产调配交易；
4. 支付：提供开放接口，外部应用可连接微众银行，实现电子支付；
5. 券商：支持第三方券商连接，对接内部业务系统和互联网银行。

微众银行的区块链底层技术是基于 POW(Proof of Work) 算法的瑞波币，它能产生足够多的工作量，难度可以逼近太阳每年赤道上的两侧，目前的共识时间是 10 分钟左右。未来，微众银行还会基于其他密码学算法、DAG(有向无环图)结构、隐私保护技术、状态通道等探索区块链的新应用场景。
