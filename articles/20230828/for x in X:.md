# 2.核心概念
## 2.1 迭代器 Iterator
在 Python 中，迭代器是一个可以遍历元素的对象，它提供了一种访问集合元素的方式。迭代器在需要访问数据集中的所有元素时会很有用，无需一次把整个列表都加载到内存中，能够节省内存开销并提升效率。

迭代器的作用主要有以下几个方面：

1. 支持集合数据的遍历，例如列表、元组等；
2. 可以逐步访问集合数据，避免一次性加载所有数据导致的内存占用过大的问题；
3. 提供了统一的接口，使得不同的数据类型具有相同的方法；
4. 有利于对数据的处理和处理流程的控制。

一个迭代器必须实现两个方法：iter() 和 next()。iter() 方法返回迭代器对象本身，next() 方法返回迭代器的下一个元素。当迭代完成后，抛出 StopIteration 异常。

下面给出示例代码：

```python
# 创建一个列表
my_list = [1, 2, 3, 4, 5]

# 将列表转换成迭代器
it = iter(my_list)

# 使用 next() 方法获取迭代器的第一个元素
print(next(it)) # Output: 1

# 使用 next() 方法获取迭代器的第二个元素
print(next(it)) # Output: 2

# 以此类推，直至迭代完成
print(next(it)) # Output: 3
print(next(it)) # Output: 4
print(next(it)) # Output: 5

# 此时再次调用 next() 会抛出 StopIteration 异常
# print(next(it))
```

上述代码创建了一个列表 `my_list` ，并将其转换成了迭代器。通过调用 `iter()` 方法将列表变成迭代器对象 `it`。然后通过调用 `next()` 方法从 `it` 中依次取出各个元素，并打印出来。

## 2.2 生成器 Generator
生成器是一种特殊类型的迭代器，它是一种迭代器对象但不像一般的迭代器那样只能用于一次性遍历集合元素。它的特点是在每次执行 next() 方法时，都会生成一个新的元素，这样就不需要存储之前所有的元素，从而节省了内存空间。而且，它还可以通过 yield 来产出值，这使得生成器更加灵活，能按照特定的逻辑产出需要的值。

下面给出示例代码：

```python
def my_gen():
    n = 1
    while True:
        if n <= 10:
            yield n
            n += 1
        else:
            return "Done"

g = my_gen()
print(next(g)) # Output: 1
print(next(g)) # Output: 2
print(next(g)) # Output: 3
print(next(g)) # Output: Done
```

上述代码定义了一个生成器函数 `my_gen`，该函数会生成数字 1-10，每产生一个数字就会调用 yield 函数产出这个数字。`yield` 返回的结果值会暂停函数的执行，记住当前位置并等待下一次执行。直到调用 `next()` 方法恢复函数执行，返回产出的数字或字符串。

## 2.3 匿名函数 Anonymous Function (lambda)
匿名函数就是没有名字的函数，也就是说我们不能给函数起名。匿名函数通常用来创建简单的函数表达式，如同其他编程语言中的 lambda 关键字一样。匿�名函数只有一行语句，通常只作为一个函数传递给另一个函数。以下为例子：

```python
sum = lambda a, b : a + b
result = sum(2, 3)
print(result) # Output: 5
``` 

上面代码创建一个匿名函数，接受两个参数 `a` 和 `b`，求和后返回结果。该匿名函数赋值给变量 `sum`，然后传入 `2` 和 `3` 作为参数调用，得到返回值为 `5`。
