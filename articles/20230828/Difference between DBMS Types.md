# 1.简介
  

随着互联网的蓬勃发展，各种新型应用诞生，数据的存储与查询需求也越来越高，传统的关系数据库(RDBMS)由于其结构化数据建模、灵活的数据模型及完善的SQL支持等优点已经不能完全满足当代互联网应用的需求。因此，基于NoSQL技术的非关系型数据库(NoSQL)也开始被广泛采用。然而，对于非关系型数据库来说，不同的数据库系统之间的区别仍然非常重要，特别是在大规模集群环境下，需要根据实际情况选择合适的数据库系统才能实现最优的性能。在本文中，我们将对关系数据库管理系统（RDBMS）与非关系数据库管理系统（NoSQL）进行详细比较，并探讨这些差异背后的内在逻辑，希望能够帮助读者更好地理解它们之间的不同之处，并在实际应用中做出更明智的决策。
# 2.基本概念术语说明
关系数据库管理系统（RDBMS），全称Relational Database Management System（关系数据库管理系统），它是一种用来存储和组织数据的数据库系统，由数据库管理员经过长期工程实践开发出来。关系数据库管理系统按照数据集合的范式设计，数据以表格的形式存放，表中的每行对应于某个元组或记录，每列则对应该元组的一个属性或者字段。关系数据库管理系统具有以下几个特点：
- 数据结构化：关系数据库管理系统强制要求所有的数据都要以结构化的方式组织，这样可以确保数据的一致性、完整性和正确性。
- 可靠性：关系数据库管理系统通过事务机制、查询优化、索引、数据库备份等手段来保证数据存储的可靠性。
- SQL支持：关系数据库管理系统具备完整的SQL语言支持，能够灵活地定义、修改和查询数据。

而非关系数据库管理系统（NoSQL）是指不仅仅是“没有”关系。NoSQL是指非关系型的数据库管理系统，也就是说，它不是基于关系模型来组织数据，而是提供了非关系模型，比如键值对、文档、图形和列族。NoSQL的出现主要是为了克服关系数据库固有的缺陷，其特点包括：
- 无模式：非关系数据库管理系统不要求每个对象都有预先定义的结构，而是可以灵活地存储各种类型的数据，这种特性使得它可以存储多种不同结构的数据。
- 动态 schema：非关系数据库管理系统不需要事先定义schema，而是直接存储数据并允许在运行时添加、删除字段。
- 没有固定 schema：非关系数据库管理SYSTEM可以存储任意结构的数据，不存在所谓的固定的schema。
- 不使用SQL：非关系数据库管理系统不依赖SQL语言作为查询语言，因为它提供的是不同的查询方式，比如键值查找、文档搜索等。

另外，在大规模集群环境下，相比关系数据库管理系统来说，NoSQL有如下优势：
- 高可用性：NoSQL可以使用分布式集群来提高数据库系统的可用性，避免单点故障。
- 扩展性：NoSQL可以在不停机的情况下动态增加存储容量，可以应对快速增长的数据量。
- 自动备份：NoSQL可以自动执行备份，从而保证数据的安全性。

综上，我们可以总结一下关系数据库管理系统与非关系数据库管理系统的区别和共同点：
- 面向对象数据模型：RDBMS是基于关系数据模型来组织数据的，而NoSQL提供了不同的数据模型，比如键值对、文档、图形和列族。
- 查询语言：RDBMS是使用SQL作为查询语言的，而NoSQL可以使用不同的查询语言，比如键值查找、文档搜索等。
- 架构设计：RDBMS是高度结构化的数据库系统，数据以表格的形式存储；NoSQL是松散耦合的数据库系统，数据可以有不同的结构。
- 数据复制：RDBMS通常使用主从复制的方式来实现高可用性，NoSQL的复制方式更加复杂。
- 数据访问方式：RDBMS通常通过SQL语句来访问数据，而NoSQL可以使用不同的访问方式，比如键值查找、文档搜索等。
