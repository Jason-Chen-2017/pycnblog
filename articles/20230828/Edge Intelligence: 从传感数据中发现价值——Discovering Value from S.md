# 2.传感数据定义、分类和特点
## 2.1 传感数据定义
传感数据(Sensor Data)指的是由传感器所生成或接收的原始数据，包括但不限于以下几种形式：

1. 静态传感数据：指传感器在空间中的固定位置上收集到的各种原始数据，例如环境温度、湿度、光照强度等。
2. 动态传感数据：指由于传感器本身的运动或变化，导致所记录的数据发生变化的原始数据，例如机器人的运动轨迹、车辆的速度、GPS坐标等。
3. 半静态/半动态传感数据：指静态传感数据的一部分，因为其具有某些限制条件不能再进行静态的采集，只能得到部分信息。例如传感器位于固定结构上，无法完全消除地面的摩擦，因此只能获得静态的粒子压力数据。
4. 事件传感数据：指指示某些特定事件发生的原始数据，例如火灾、爆炸、检测到人类或者其他动物的行为等。

## 2.2 传感数据的分类
根据传感数据所采集的信号类型、分布规律、时间、大小、周期性、速度、准确性、采集频率、数据密度等特性，传感数据可分为以下几个类别：

1. 传感器传感数据：指由单个或多个传感器收集到的原始数据，包括静态传感数据、动态传感数据、半静态/半动态传感数据和事件传感数据等。
2. 概念传感数据：指以人类认知或理解的方式抽象的、较难处理的传感数据，如图像、文本、声音等。
3. 虚拟现实传感数据：指以虚拟现实(VR)技术生成的、反映真实世界的数据，如人脸识别数据、手部追踪数据等。
4. 复杂传感数据：指由多个传感器组合生成或转换而成的原始数据，如交通状态数据、车流量数据等。

## 2.3 传感数据的特点
传感数据的特点主要包括：

1. 时效性：传感数据的时间尺度一般都很小，单位时间内采集的数据量也比较少，因此即使是较长时间的历史数据也是相对易失的。
2. 重复性：由于传感器与周边环境的影响，同样的事件或状况可能会被多次记录，因此数据的重复性很高，且极具价值。
3. 不连续性：由于传感器、传感系统、环境的复杂性、部署位置、移动速度等原因，传感数据往往不是完整的、连续的，而且存在着噪声和干扰。
4. 多维特征：传感数据既有数量性特征，如计数、温度、湿度等；又有空间性特征，如坐标、方向、距离等；还可能包含多种复杂的非数量性特征，如图像、声波、触觉等。
5. 可靠性：由于传感系统、传感器自身的各种故障或问题，传感数据可能存在不可靠、错误、缺乏的情况。另外，由于不同传感数据之间的联系、依赖关系、冲突、关系，传感数据可能被人为或非人为的干扰。
6. 隐私性：由于个人隐私或商业利益的保护要求，传感数据不宜向公众开放，可能需要严格控制获取权限和使用范围。
