# 1.简介
  

Google Analytics(以下简称GA)是谷歌推出的网站流量分析工具，其功能包括：统计网站流量、了解用户行为习惯、改善网站设计。由于其多维数据结构，方便了数据的查询，因此广泛用于广告投放、优化营销活动等领域。目前，GA在全球范围内已经成为事实上的标准，各大互联网公司均已将GA作为重要的增长性指标。
在GA的基础上，谷歌推出了跟踪广告的第三方服务——Google Ads Conversion Tracking(以下简称GATC)，该产品通过监控用户在电子商务网站或移动应用上的行为，精准地把每一次点击转换为客户。同时，GATC还集成了Google Search Ads、DoubleClick Campaign Manager和YouTube Advertising API，能够帮助广告主提高广告效果、增加转化率。相对于传统的SEO优化方法，GATC具有更精准的针对性，更加有效的广告投放效益。

但是，GATC仍然存在一些局限性。首先，基于第三方技术，可能会造成额外的性能开销；其次，只能跟踪特定类型的点击，无法捕获所有可能的广告购买点；再者，如果用户访问的是非广告链接，或者浏览器禁用了第三方Cookie，则会影响广告效果。因此，为了更好地实现广告效果优化，以及满足客户需求，谷歌近期推出了一个新的解决方案——标签管理系统(Tag Management System)。

2.基本概念术语说明
1. 事件类型 Event Types（又称“事件”）
GA支持对各种类型的事件进行跟踪，如浏览页面、下载文件、添加到购物车、注册等。这些事件被划分为不同的类别，即事件类型。比如，浏览页面事件对应于网页浏览情况，下载文件事件对应于文件下载情况，注册事件对应于新用户注册情况等。

2. 概念对象 Concept Objects（又称“目标”）
GA根据不同的业务场景设定相应的目标，可以从用户群体、页面内容、行为习惯、设备属性等多个角度定义目标。例如，一个电子商务网站的目标可能是“购买某种商品”，而一个零售网站的目标可能是“完成购买”。每个目标都由一个唯一标识符表示，便于跟踪、分析和报告。

3. 属性 Attribute（又称“指标”）
GA收集的数据主要包含两部分：事件数据和用户属性数据。其中，事件数据反映用户对目标的行为习惯、触发事件的时间、所在页面、设备信息等信息，是衡量用户参与度、流失率等的主要依据；而用户属性数据反映用户自身的信息，如年龄、职业、消费水平、兴趣爱好等，也是衡量不同用户的转化率、留存率等的重要参考因素。

GATC通过分析用户对广告的点击次数，结合广告设置的规则和逻辑，得出用户是否发生了购买行为，并反馈给广告主。因此，GATC要跟踪用户在电子商务网站或移动应用中的具体购买点，这一点要求GA提供更多的指标数据。

4. 标签 Tagging（又称“标签”）
标签由HTML代码片段组成，嵌入网页中或设置在第三方平台中。当网页被加载时，标签会自动向GA发送相关数据，例如页面浏览路径、搜索词、来源、IP地址、设备型号等。除了基础的统计数据外，还可以进一步收集用户自定义的事件数据，如购买商品时的商品名称、价格、数量、渠道来源等。

5. 数据层级 Data Hierarchy（又称“层级”）
GA将事件数据和用户属性数据分别存储在不同的数据库表中，通过不同的层级关系联系起来。最底层的层级是网站访客、设备、网络及位置信息，这些数据不仅包含事件数据中的用户信息，而且还可以反映网站的整体使用情况、市场份额分布、开发团队成员分布等。

在中间层级，则包含目标相关数据，如各种目标的转化率、激活率、跳出率等指标数据。在顶层，则包含标签数据，记录各种标签的运行数据。GATC通过标签数据可以精准地测量用户是否发生了购买行为。

6. 数据导出 Data Export（又称“导出”）
GA提供了两种数据导出方式，即实时导出和离线导出。实时导出实时生成CSV文件，方便后续处理；离线导出可以生成历史数据快照，可以根据需要导出最近30天或90天的数据。

7. 会话 Session
GA将用户访问网站的所有行为进行归纳，形成一条完整的用户行为序列，即会话。会话由用户产生的初始事件开始，逐步记录用户的点击、浏览、购买行为，直至结束。会话可以用来分析用户在网站的停留时间、在网站浏览的热门内容、用户在某个页面停留时间过长、用户在不同设备上交互频繁等问题。

8. 用户画像 User Profiling
GA利用用户的行为习惯、偏好、偏好等特征进行用户画像，并提供分析功能，如产品推荐、个性化广告等。通过用户画像，可以洞察用户的喜好、意愿、需求，并为他们提供更符合其预期的内容。
