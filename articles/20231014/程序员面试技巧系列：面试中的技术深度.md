# 1.背景介绍



前言：当今世界，IT界人才纷纷涌入市场，有些企业甚至以亿元、千万级计价。IT技术不断地向前发展，掌握开发技能的人成为未来发展最需要的人才。但是，这个IT行业的竞争环境日益复杂化，技术水平参差不齐，如何在公司任职，提升个人能力，成为真正的技术精英成了吸引人的招聘目标。但是面试中，技术水平的广度和深度的考察往往被忽略掉。而面试官对技术深度的评判更多的是依赖面试者的自我评估及运气，这无疑会造成很大的误导和负面影响。因此，为了能够准确且客观地评估候选人的技术深度，更好地为他们提供一份优秀的工作岗位， IT界应该从技术技能的广度和深度两个角度出发，努力打造一支高素质的技术团队。
 
本文将着重探讨技术深度问题，根据国内外实际情况，梳理技术人员的所得知识结构与职称层次之间的对应关系，总结出其中的规律。并针对性的分析了面试中常见的问题与面试官们的回答，希望能够帮助IT从业者更好地掌握技术深度和广度。
 
# 2.核心概念与联系
## 2.1 什么是技术深度？

技术深度是指一名技术人员对一个领域的理解程度，他可能已经是一个高级工程师或者架构师，但仅仅只是掌握某项技能或方法的应用经验的个人，其技术深度往往远低于具有一定职业经验的人士。那么究竟什么样的个人才能算是技术深度呢？
 
《程序员面试技巧系列：面试中的技术深度》一书中，作者列举了以下几类个人：
 
- 有一定编程经验的“菜鸟”程序员（Code Monkey）；
- 有一定开发经验的“术士”程序员（Guru）。Guru通常具备超过普通程序员几倍到十几倍的深度和广度。换句话说，Guru是跨越多个专业领域并拥有众多职位经验的专业人才。
- 深厚的专业技术底蕴的资深工程师（Expert）。Expert通常具备非常丰富的知识面，涉猎范围包括很多不同方向的技术，具备超强的学习能力和分析问题的能力，并且同时具备其他专业角色所要求的高度的执行能力。
- 对大量新技术产生兴趣，在快速发展的技术前沿中游走的“追随者”程序员（Entrepreneur）。这种程序员的突破能力和创新思维力是不可替代的。
 
## 2.2 技术深度和知识结构之间的关系

技术深度可以理解为一个技术人员对于某个领域的理解深度。但如何衡量技术深度，又该如何定义才能更准确？一方面，需要把技术人员的所得知识结构与职称层次之间的对应关系理清楚。另一方面，还要了解不同职称层次的人们对于自己的技术深度有何认识？
 
一个典型的技术人员的知识结构分为三层，分别是基础知识、技术知识和业务知识。其中，技术知识是其最终目的所在，也是其与职位匹配度最大的部分。
 
 
根据IT行业对各个专业技术人员的要求，IT技术人员的职称层次也被细分为以下几种：
 
  - **初级工程师**（Entry Level Engineer）：初级工程师一般从事一些简单的编程、数据库等相关工作，也就是说，他只需要扎实的编程基础即可胜任。初级工程师们通常仅熟悉基本的计算机编程语言如Java、C++等，并会使用各种工具、框架进行编程。然而，初级工程师们只能在较小的项目中深入研究某些技术细节，对于应用和解决实际问题却没有太多的经验。
 
  - **中级工程师**（Mid-Level Engineer）：中级工程师能够更好地理解软件开发的基本原理，通过学习新的技术，掌握更加深入的技术特性，可以用自己的双眼看世界，以更快的速度理解商业需求，提升产品质量，提高工作效率。
 
  - **高级工程师**（Senior Engineer）：高级工程师主要负责设计、编码、测试和部署软件系统，具备一定的应用开发能力，能够带领团队合作开发成功产品。高级工程师通常具备很强的领导能力、跨部门协调能力、资源分配能力以及对业务敏感度高。
 
  - **技术专家**（Technical Expert）：技术专家通常具有卓越的理论功底和复杂的技术知识，掌握一整套完整的软件开发技术。这些技术人员不仅对某项技术深入了解，而且能够利用自身的专业知识快速建立起对特定技术的理解。
 
  - **架构师**（Architect）：架构师的角色是负责设计软件系统的架构和功能，同时兼顾性能优化、安全性设计和交付质量保障等方面的工作。他们的重要工作就是解决复杂的问题，把方案落地到生产环境中。
 
以上是IT技术人员的职称层次，每个层次都包含一定的知识结构和技术知识要求。比如，初级工程师首先需要掌握基本的编程语言、数据结构和算法，然后可以选择自己擅长的某项技术深入研究，比如操作系统、网络协议等。而高级工程师则需要更为深入的技术理解、工作能力和领导能力，才能更好地领导团队完成工作。
 
## 2.3 面试官的技术深度评测方法
 
虽然技术人员的所得知识结构与职称层次之间存在着相互关联，但是每一个技术人员又有自己的特点，不同的面试官对于技术深度的评测方式也不同。因此，如何准确地评测面试者的技术深度，也是IT行业需要解决的问题。
 
《程序员面试技巧系列：面试中的技术深度》一书给出的技术深度评测方法主要包括四个方面：
 
  - **知识结构：**考察面试者对于所谓的“知识结构”有多了解。比如，面试者应当能清晰地表达自己的观点和想法，并可以用自己的语言准确描述各种技术概念，知道不同领域之间的区别和联系。
 
  - **经验水平：**面试者的经验水平决定了他的技术深度。面试官们一般不评测技术人员的专业知识技能，因为这是一个主观的标准，无法量化地衡量。而是通过考察面试者是否有过相关的项目经历，以及是否对某项技术有过独到的见解，来衡量技术深度。比如，面试者应当展示自己用不同编程语言解决过的问题，以证明自己的编程能力和解决问题的能力。
  
  - **专业度：**考察面试者是否擅长某个领域的技术。这是比较客观的方法，是衡量技术深度的一个客观标准。面试官们不认为一个技术专家一定比其他技术专家具有更高的专业度。比如，面试者应当分享自己曾经担任某些开发任务的经历，以增进彼此之间的了解。
 
  - **思路开阔程度：**面试官们不会直接考察面试者的技术深度，而是通过“回答问题”的方式来测试面试者的思维过程和处理问题的能力。比如，面试者可能会被问到一些高级技术问题，这些问题可能涉及到了很多的技术细节。面试官应当让面试者自行思考这些问题，如果他无法正确回答，就可以说明他的思路有待提升。另外，面试官们还可以通过问一些比较抽象的问题，来验证面试者是否具有深度思考能力。

最后，《程序员面试技巧系列：面试中的技术深度》一书给出了面试中关于技术深度的常见问题和面试官的回答，可以帮助读者更好地掌握技术深度和广度，为自己的工作选择到合适的人选做好准备。
