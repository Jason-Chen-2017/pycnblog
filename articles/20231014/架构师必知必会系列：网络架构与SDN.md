# 1.背景介绍


  SDN（Software Defined Networking）是一个热门话题，它的出现使得网络基础设施和业务应用之间更加松耦合、标准化，使得各种硬件设备可以智能地组合组网而互相协同工作。随着云计算、物联网、大数据等新兴技术的蓬勃发展，SDN也越来越受到各行各业的青睐。因此，对SDN知识的掌握至关重要，也是IT技能的重要保障之一。因此，笔者根据个人经验总结出如下一些需要注意的事项，希望对大家有所帮助。
  
  1）理解SDN
  - SDN的核心目标是将网络功能从硬件上完全卸载到应用程序中，实现网络自动化。
  
      通过将网络控制功能从底层网络硬件转移到SDN控制器上，并在控制器与交换机之间架起数据平面，将来自不同区域的数据流量通过这两者之间的协商，直接流转到正确的目的地址，并且这些动作可以通过可编程逻辑进行精细化控制，进一步提升网络的灵活性与安全性。
   
  - SDN的核心功能模块主要包括三个：控制器（Controller）、交换机（Switch）、主机（Host）。
  
  - 控制器: SDN控制器负责配置网络资源及规则，管理交换机的连接，分配IP地址、QoS策略等，同时还可以执行高级路由协议等。
  
  - 交换机: 交换机通常具有访问控制列表（ACL），能够分析网络流量的源地址和目的地址，过滤无效流量，执行分组的处理，如QoS、统计、缓存、转发等操作。
  
  - 主机: 传统的网络设备主要用于网络接入、分组传输、QoS调控等，而SDN将其扩展到主机上，运行用户态的程序，主要用于网络数据采集、分析、处理等，极大地提升了主机的计算性能。
  
  2）适用范围
  - 只适用于具有复杂网络拓扑、多租户、高性能要求等特点的企业。
  - 只适用于高速率、大容量的数据交换场景。
  - 只适用于少量边缘业务。
  
  3）适用的技术
  - 支持SDN的主要网络硬件、软件和中间件，如OpenFlow、OVS、Linux Kernel Bridge等。
  - 大量实践经验和开源项目，如ONOS、Floodlight、Ryu等。
  
  4）注意事项
  - SDN缺乏网络工程的基本素养。
  - 学习曲线陡峭。
  - 不建议完全依赖于SDN，而应结合传统网络和虚拟网络一起使用。
  
  5）推荐阅读
  《SDN权威指南》(第二版) 作者：何塞·史密斯著，曹阳译 电子工业出版社 2017年9月发布 本书的内容非常详尽，以具体的技术案例和实例来阐述SDN的核心概念、优势和局限，并以落实到实际应用的方法论来指导读者实际操作。
