# 1.背景介绍



量子纠错编码（Quantum Error Correction Codes, QECC）是为了解决错误传输的问题而提出的一种编码方法。在传送信息过程中，由于各种因素导致的信息损失、电路失灵、光线等一系列无法避免的错误都会导致信息的丢失或错误传输。QECC的基本思想是在编码过程中引入相干信息（entangled information），使得错误被更容易检测出来并纠正。因此，通过采用QECC可以降低信息传输中的错误概率，从而保障通信的可靠性。目前已经存在多种高效的量子纠错编码方案，比如量子位反转编码(QUBIC)、纠缠态-码元门控编码器(AQC)、物理层信道模型(Physical Channal Modeling, PCM)等。其中，QUBIC和PCM两种编码都可以在通信系统中用于纠错，而AQC则需要特定的物理介质支持才能实现纠错功能。在本文中，我们主要讨论基于QECC的通信协议——隐蔽无噪声双向通信（HISEC）。

在早期的通信系统中，通信双方之间通常使用时域上的信号来进行通信。但随着通信的增长，人们越来越依赖于光传输的方式来进行信息传递。因此，为了能将早期时域的通信技术发扬光大，人们开发了利用光作为载体进行通信的技术。然而，利用光传输方式带来的问题是，光的特性与时间上隔离，导致出现信号遮挡、干扰以及信道时延等一系列不可避免的错误。为了保证通信的可靠性，利用QECC可以对双方发送的信号进行纠错，提升通信的成功率。

隐蔽无噪声双向通信（HISEC）是第一个使用QECC的通信协议。它由两个节点A和B组成，它们共享一个已知的密钥K，并希望通过加密信息来进行通信。HISEC协议在传输过程中使用纠错码。具体来说，发送方首先将原始信息x转换为生成码z，然后使用纠错码E编码生成码z，并将其发送给接收方。接收方收到生成码之后，可以用纠错码E进行纠错，并恢复出原始信息x，同时也可以校验和验证生成码是否出现错误。如果发现错误，则可以重发消息进行纠错。这样就可以有效地消除传输过程中的错误。HISEC协议可以在无需调制解调器的情况下实现信息的安全传输。

# 2.核心概念与联系

## 2.1 概念回顾

1. 量子纠错编码(QECC): 量子纠错编码（Quantum Error Correction Codes, QECC）是为了解决错误传输的问题而提出的一种编码方法。在传送信息过程中，由于各种因素导致的信息损失、电路失灵、光线等一系列无法避免的错误都会导致信息的丢失或错误传输。QECC的基本思想是在编码过程中引入相干信息（entangled information），使得错误被更容易检测出来并纠正。因此，通过采用QECC可以降低信息传输中的错误概率，从而保障通信的可靠性。
2. 纠错码：纠错码（Error Correcting Codes, ECC）是一种能够纠正传输错误的编码方法。它的基本思想是在编码中加入冗余元素，使得接收方可以通过判断冗余元素是否发生错误来纠正误差。例如海明码就是一种最简单的纠错码。
3. 生成码与纠错码的关系：生成码(Generator code)是一种没有任何冗余元素的编码方法，编码之后的信息为原始信息的连续函数。而纠错码(Error correcting code)是一种可以纠错的编码方法，它在编码过程中会加入冗余元素。所以，生成码不适合用于传输信息，而纠错码是用于传输信息的编码方法。
4. 海明码：海明码(Hamming code)是一种最初的纠错码，它利用奇偶校验的方式进行纠错。当某一比特发生错误时，仅能确定这一比特以及它所对应的那些比特发生错误。通过这种方式，接收方只需要检查奇数个的位置就能确定是否发生错误。
5. 隐蔽无噪声双向通信(HISEC): 隐蔽无噪声双向通信（HISEC）是第一个使用QECC的通信协议。它由两个节点A和B组成，它们共享一个已知的密钥K，并希望通过加密信息来进行通信。HISEC协议在传输过程中使用纠错码。具体来说，发送方首先将原始信息x转换为生成码z，然后使用纠错码E编码生成码z，并将其发送给接收方。接收方收到生成码之后，可以用纠错码E进行纠错，并恢复出原始信息x，同时也可以校验和验证生成码是否出现错误。如果发现错误，则可以重发消息进行纠错。这样就可以有效地消除传输过程中的错误。HISEC协议可以在无需调制解调器的情况下实现信息的安全传输。

## 2.2 QECC相关术语

### 2.2.1 Qubit

量子比特（qubit）是量子计算的基本单元，具有两个量子态，分别对应于量子系统的两个可能状态，即|0〉和|1〉。由于量子纠错编码需要引入相干的信息，因此qubit还有一个可观测的量子态，即|+>和|->。

### 2.2.2 量子态

量子态是一个描述客观世界或物质存在状态的一组量子比特的集合，它们处于不同的量子态。任何量子态都可以表示为归一化的矩阵形式，其中矩阵的每一项都是复数。通过这个矩阵，我们可以对任意一个量子态进行操作。一般来说，我们用希腊字母表示矩阵，例如用 |psi> 表示某一量子态，那么 |psi><psi| 表示该量子态的能量或者模长。

### 2.2.3 流程图

1. 用户在两台计算机之间发送一段信息；
2. A发送一串二进制数据0110101……，接收方接收后使用检错码对其进行纠错，得到正确的数据；
3. 如果出现错误，重新发送正确的二进制数据；
4. 当接收端接收到正确的数据时，完成数据传输。


流程图展示了使用纠错码进行通信的流程。假设A和B之间可以发送二进制数据，使用QECC进行通信。首先，A发送一些0、1序列给B，B接收并检验。如果接收到错误数据，B立刻把正确的数据发送给A。当A接收到正确的数据时，结束数据传输。

### 2.2.4 BBP协议

BBP协议是一个全双工通信协议，可以直接在两台计算机之间发送信息，不需要中间节点。BBP协议的发送端采用一种编码方式把要发送的信息变成一串0、1序列。接收端按照相同的方式解析收到的0、1序列，即可获得要传输的信息。BBP协议的编码方式称为双点型编码，它的特点是需要进行周期性调制才能正常工作。

### 2.2.5 Ferroelectric switching (FSS)

费米子交换(ferroelectric switching)是指在两个极板上的两个极互相作用而产生极弱相互作用。在量子通信领域，这种现象称为量子互作。对极板施加足够的磁场，可以引起两个量子门的交叉激励，从而改变两个量子门的方向。可以利用这种特性实现量子通信。在QEC上，可以利用这种特性建立非接触通信。具体方法如下：

1. 编码器使用单个比特作为控制位，此比特的初始状态是0。
2. 将编码器的输出连至磁铁的两端，这样便形成了一根磁通。
3. 对于发送方，发送0信号时，磁通从左边进入极板；发送1信号时，磁通从右边进入极板。
4. 对于接收方，接收0信号时，它可以通过检测极板中的磁场来确定发送者实际发送的是0信号。
