# 2.核心概念与联系
## 2.1 软件外包与IT外包
软件外包（Software Outsourcing）也称作IT外包（Information Technology Outsourcing），是一种由外部服务机构提供计算机软硬件设备及相关的服务的服务方式。通常情况下，软件外包公司由专业人士通过远程协助的方式安装、配置和维护用户的电脑、网络设备和服务器。由于它属于信息技术服务外包，因此也是属于IT外包的一种。

软件外包公司包括：
1. 第三方软件开发公司；
2. 专业的软件开发、维护、咨询和培训公司；
3. 云计算、移动应用开发、硬件设备销售、系统集成、网络安全服务等等。

## 2.2 为什么要建立自己的软件外包公司
1. 赚钱能力：软件外包公司最主要的优势就是赚钱能力强，有时甚至超过了自己雇佣的人员。
2. 技术优势：软件外包公司可以把精力更集中地放在软件开发上，避免重复造轮子，提高开发效率。同时，通过提供专业化服务，获得与客户交流的机会，减少竞争对手的压力。
3. 自由时间：因为不需要你一直待在办公室，所以你可以尽情享受工作的快感，也不用担心忙碌。另外，你的创意也不再局限于自己公司内部，而是可以应用到世界范围内。
4. 个人价值和管理价值：虽然你可能只是搬运工，但是软件外包公司也可以产生巨大的个人价值，例如，通过提供软件服务给别人，在潜在客户身上建立声誉，并且建立个人名气。而且，管理软件外包公司就等于管理自己的一生。

## 2.3 软件外包公司的定位和功能
软件外包公司一般分为以下几个层级：
1. 服务型公司：向客户提供相关软件解决方案或服务，如提供软件设计、测试、编程等服务。例如国外的Adobe公司，它的产品和服务都涵盖了广泛的领域。
2. 平台型公司：提供开发者平台，供客户开发软件应用程序，并承担相应软件的销售及支持工作。例如微软公司的Visual Studio，AWS公司的Amazon Web Services，都是以软件平台的形式出现的公司。
3. 销售型公司：以软件服务为目的，按照合同收费的方式销售软件产品给客户。例如Google公司的Chrome浏览器，Mozilla公司的Firefox浏览器，都是以软件销售为主的公司。
4. 综合型公司：既提供软件开发和服务，又按合同收费提供软件产品的销售。例如Facebook公司，Twitter公司，都是综合型公司。

除此之外，软件外包公司还可以提供各类管理咨询、培训、法律、财务、HR等服务。

## 2.4 软件外包公司的服务对象
软件外包公司的服务对象主要包括：
1. 中小企业：软件外包公司可以直接开展业务拓展，为其提供专业化软件解决方案或服务。对于这些小型企业来说，通过软件外包公司，可以节省大量的人力资源投入，帮助其快速扩张规模，从而带动经济增长。
2. 大型企业：软件外包公司可以帮助这些公司完成复杂的任务，例如开发软件、运行维护、迁移系统等。例如亚马逊公司的AWS（Amazon Web Services），采用SAAS（Software as a Service）模式，为其提供IaaS（Infrastructure as a Service）服务，帮助其降低运营成本，提高生产力。
3. 政府部门：软件外包公司可以提供政务数据处理、政策跟踪、执法监控、事件响应等服务，使政府部门的工作更加有效率、规范化。
4. 金融机构：软件外包公司可以提供资产管理、债权转让、交易结算、支付结算等服务，帮助金融机构降低成本，提升营收。
5. 初创企业：软件外包公司可以帮助企业完成市场验证、渠道开发、品牌营销、运营优化等环节，使他们更容易进入市场。

## 3.1 如何运营一个软件外包项目
一个软件外包项目的生命周期可以概括为以下几个阶段：
1. 招聘需求：寻找软件工程师、web开发工程师等职位要求外包公司的职位描述，制订合同约定。
2. 项目实施：由专业的软件工程师组成团队，负责项目的计划、设计、开发、测试、部署等过程。
3. 项目交付：软件外包公司承诺在固定的时间内交付完整的软件，并提供相关的文档和技术支持。客户只需要按时付款，即可享受服务。
4. 后期维护：如果客户提出任何问题或需要额外的支持，软件外包公司都会立即赶来提供帮助。
5. 项目终止：如果项目后续出现新问题或无法满足客户的需要，软件外包公司也可以提前终止服务。

## 3.2 软件外包服务的收费方式
软件外包服务的收费方式可以分为以下几种：
1. 按人天计费：即按每个工程师的工作日工作量来收取费用。这样做的好处是明确指出工程师的工时费用，避免工程师因加班而产生不必要的损失。
2. 按月固定价格：即每月固定收取一定金额的服务费用。这样做的好处是降低费用风险，可避免公司欠费或欠薪。
3. 根据工程师的能力支付费用：即根据工程师的专业技能、项目经验等情况，收取不同的费用。这样做的好处是保证工程师的技术水平匹配。
4. 按成果量付费：即根据工程师完成的工作量来收取费用。这样做的好处是保障工程师的努力和付出的价值。

## 3.3 从需求分析到项目启动的流程
需求分析是整个外包项目的一个重要组成部分。
1. 收集需求：确定客户的需求并收集需求文档。客户需要提前准备好客户需求文档，通常包含以下内容：
  - 目标和功能
  - 期望效果
  - 财务需求
  - 时间要求
  - 其它要求
2. 提交需求：把客户需求文档提交给软件外包公司。软件外包公司审核完毕后，将需求文档发送给合适的工程师。
3. 项目评审：软件外包公司会对项目的合理性进行评审。只有合格的项目才可以正式启动。
4. 项目管理：项目管理包括项目策划、项目执行、项目收尾等，是在项目实际运作过程中要遵守的流程和规范。项目管理可以帮助软件外包公司管理各种资源、人员、物料、进度等，提高工作效率和质量。
5. 项目启动：当项目达到启动条件后，软件外包公司将正式开始项目开发。

## 3.4 如何选择软件外包公司
选择软件外包公司是一个很重要的决定，因为你不仅需要了解所提供的服务内容，而且还要考虑到该公司的服务质量、可靠度、质量保证体系、态度、工作环境和风险控制措施。下面介绍几点建议：

1. 选择合适的领域：软件外包公司通常服务于多个行业，因此选择面向客户的领域非常重要。例如，对于创业公司，可以选择为他们提供软件外包服务，而非为他们提供硬件销售或IT服务。
2. 确认服务质量：通常软件外包公司都有自己的服务标准，包括可靠性、速度、服务内容、预算、质量保证体系等。了解这些标准和规定，才能更准确地选择软件外包公司。
3. 确定合适的价格：软件外pack公司价格往往依赖于服务内容，不同类型的服务价格也不同。选择合适的价格是做决定的关键。
4. 选择可靠的服务商：选择可靠的服务商可以避免公司倒闭、人员变动或人才流失等情况。另外，选择可靠的服务商还可以为客户提供贴心的服务。
5. 不要盲目依赖外包服务：虽然软件外包公司可以在短期内收获大量利润，但最终它们还是需要为自己支付足够的本金和利息。为了防止出现亏损的情况，最好为自己设置一个收益保障机制，并偿还贷款或退还差额。
