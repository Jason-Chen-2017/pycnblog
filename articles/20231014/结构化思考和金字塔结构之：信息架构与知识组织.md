# 1.背景介绍


在当前信息社会的发展中，信息的整理、分类、检索、存储等作为人类社会发展的一项基本功能已经成为一个复杂而重要的过程。信息的整合使得人们可以快速地获取、处理并有效地传播信息。如今，随着互联网的普及、移动互联网的发展以及智能手机的普及，人们越来越依赖智能设备进行各种各样的信息交流和获取。同时，人们对自己掌握的知识也越来越看重、关心。如何高效整理和组织人们的知识成为信息时代的一个关键问题。

人类的知识组织往往有“条条大路通罗马”的美誉，然而随着互联网的发展，越来越多的人通过互联网、手机APP获得了海量的信息资源，这些信息如何管理、整理、分类、检索和传播，形成自己的知识结构，成为信息时代的关键问题。

本文将围绕信息架构与知识组织两个核心问题展开讨论，试图将传统的制度性方法和工具、人工分析等方法相结合的方式去解决信息架构与知识组织的难题。

## 1.1 信息架构的定义
信息架构（Information Architecture）又称“信息设计”，它是一个系统工程学术词汇，由 <NAME> 在1997年提出。其定义如下：

```
信息架构是指管理信息系统（MIS）和组织人员之间沟通交流的过程中所涉及到的所有方面的工作。主要目标是建立可供所有用户使用的、系统化的、完整的、适当层次化的、易于理解的信息结构。这种信息结构包括如何分级、编排、收集、处理、储存、检索、传输和使用信息。信息架构通常包括确定系统的角色、职责范围、边界、分层次结构、导航、搜索、学习、输出和评价标准等方面。
```

简单来说，信息架构就是用来控制和组织信息的一种系统设计。它包括物理布局、信息分类、信息组织结构和信息管理技术三个方面。其中物理布局是指信息存储位置的选择；信息分类是指信息按主题、类型和时间划分；信息组织结构是指各个信息对象之间的逻辑关系；信息管理技术则是指对信息的整理、分类、检索、存储、传输、交换、检索等手段和过程进行整体规划和调整。

## 1.2 信息架构与业务需求
信息架构与业务需求的关系，主要可以分为两类。第一类是以信息架构驱动业务决策。这一类信息架构除了做好信息组织，还需要考虑上下游关系、资源约束等因素，确保信息架构能够为业务提供有效的支持。比如网络营销的场景下，要保证信息流动顺畅、降低重复投放等。第二类是业务需要信息架构满足用户需求，比如客户满意度调查问卷、新闻咨询服务等。这一类信息架构包括信息采集、整合、搜索、运用、呈现等环节，都需要考虑到用户需求，满足用户的各种信息查询、分析和理解能力。

总而言之，信息架构的发展离不开人的因素，既要考虑需求、体验、流程、规则、操作习惯等非技术因素，也要注重技术的实现、架构师的技术修养和管理经验。因此，在给出信息架构方案之前，务必首先清楚需求背后的实际情况、业务特点、技术限制和实际困难。只有弄明白这些后才能确定信息架构的目标、路径、方式，才可能最终落实到项目上。
