# 1.简介
  

GitHub是一个面向开源及私有软件项目的托管平台，它提供一个功能强大的社交通讯工具，可以用来进行bug管理、任务管理、项目管理等。这是一种Git版本控制系统（版本库）的托管服务，具有很强的社会化特性，被越来越多的公司和个人所采用。
在实际应用中，GitHub作为一个社交平台，用户可以在这里发布各类开源或私有项目的源代码、文件以及文档等信息。开发者通过将自己在项目中的工作成果或者日常记录分享到GitHub上，也可以方便他人获取自己的项目资源并参与进来。同时，还可以基于GitHub上不同项目的贡献与反馈，为自己与他人的成长提供帮助。因此，GitHub已成为开源项目和个人项目的重要组成部分，也被认为是“世界上最大的同性交友网站”。
GitHub共有超过五百万开发者从事其各种开源项目开发、维护和社区建设。截止目前，GitHub已经有超过三千万的用户。由于GitHub是一个免费的远程仓库服务，用户对服务的可用性及稳定性负有一定的责任。GitHub自2008年成立至今，已经历经多个版本迭代。其在软件领域的知名度越来越高，已经成为一个被广泛认可和使用的平台。

 # 2.基本概念与术语
## 2.1 Git 是什么？
Git是一个开源的分布式版本控制系统，最初由林纳斯·沃尔克（Linus Torvalds）在2005年创造。Git于2008年由Linux内核开源社区中的杰出工程师布道者蔡奇（Cecil Caine）编写而成。Git支持以下的分支管理策略：
- 分支开发模型
  - master分支：主干分支，一般包含最新稳定版本的代码。
  - develop分支：开发分支，从master分支拉取最新代码，集成新功能，测试后合并到master分支。
  - feature分支：特性分支，开发新功能时创建，开发完成后合并回develop分支。
- git命令详解
  1. git clone [url]：克隆远程代码库到本地目录。
  2. git init：初始化一个空的Git仓库，创建一个名为.git 的隐藏目录。
  3. git add [file1 file2...]:暂存文件修改。
  4. git commit -m "commit message":提交暂存区改动到当前分支，需要添加-a选项则可以提交所有修改过的文件。
  5. git push origin branch-name:推送本地分支到远程分支。
  6. git pull origin branch-name:从远程分支拉取代码到本地。
  7. git status:查看当前状态。
  8. git diff HEAD -- filename:显示指定文件的不同之处。
  9. git log --oneline –graph –all:查看提交历史，包括分支信息。
  10. git tag tag-name:给当前版本打标签。
  11. git checkout branch-name:切换当前分支。

## 2.2 GitHub 是什么？
GitHub是一个面向开源及私有软件项目的托管平台，主要提供了一个功能强大的社交通讯工具，并且拥有众多优秀的功能。除此之外，GitHub还提供了代码托管、基本的wiki支持、issue跟踪、任务管理、项目管理等功能，极大地方便了开发者在线协作。其中，与Git一样，GitHub也是开源且免费的。GitHub主要提供以下服务：
- 在线代码托管服务
  提供Git版本控制系统的远程仓库服务，让开发者能够快速、方便地管理软件源代码。
- 支持Wiki
  可以为项目提供丰富的Wiki文档，帮助开发者更好地了解项目的功能和使用方法。
- Issue跟踪系统
  有助于项目开发者管理项目中的问题、需求以及 bug。
- 项目管理工具
  集成了项目管理、任务分配、工作流程等一系列强大的工具，方便开发者组织和管理项目。
- 其它功能
  GitHub还提供许多其他有益于项目管理的服务，如部署自动化工具、第三方插件、报告、代码搜索等。
