# 1.简介
  

## 1.1定义
### 1.1.1 计算机视觉
计算机视觉(Computer Vision, CV)是指让计算机理解或者处理图像、视频或相机拍摄到的信息。图像处理就是指从照片、视频、医疗影像中提取有用信息并创建可用的图像，而识别系统就是根据模式识别技术从图像或视频流中提取感兴趣的特征，通过对特征进行匹配或聚类，实现目标跟踪、分类、检索等功能的技术。
### 1.1.2 模型
机器学习(Machine Learning)是一门人工智能科学领域，目的是使计算机基于训练数据构建模型，从而对新的数据或任务做出预测和决策。其任务可以分为两大类：监督学习（Supervised learning）和无监督学习（Unsupervised learning）。监督学习指的是用已知数据的正确输出结果来训练模型，然后利用这个模型对新的输入数据进行预测和分类；无监督学习则不给定任何标签的训练数据，只知道数据间存在一些潜在关系，因此模型需要自我组织，以发现隐藏的结构、模式和模式之间的联系。
### 1.1.3 边缘计算
边缘计算(Edge Computing)指的是运行在设备端的计算，具有良好实时性要求、高效率、低功耗的特点，主要用于响应快速变化的需求，是一种应用于物联网、云计算、移动互联网及其他网络的技术。它的出现与物联网、智慧城市、车联网等新兴领域的浪潮同时发生。
边缘计算的主要应用领域包括车辆、无人机、视频分析、智能摄像头、机器人等。其中，车辆的边缘计算主要用于识别车牌、检测汽车安全问题，无人机的边缘计算主要用于摄像机记录、AI助攻等，机器人边缘计算则用于协同工作、提升性能。

## 1.2 发展历史
边缘计算在20世纪90年代由美国麻省理工学院教授威廉·布雷顿（<NAME>）在INFOCOM会议上首次提出。随后多家公司陆续参与了该技术的研发，包括Facebook、Google、微软、Intel等。由于其所处的边缘位置的特殊性，导致边缘计算能够提供极快的响应速度，在处理图像、音频、视频等数据时可以获得更高的效率。在最近几年，越来越多的人开始关注到边缘计算的应用。例如，在物联网（IoT）的发展过程中，越来越多的应用场景都转向了边缘计算。近几年，随着边缘计算的火爆，各个公司都推出了自己的边缘计算产品或服务。如阿里巴巴旗下的芯片阵列，百度的边缘端云平台，腾讯的边缘计算集群，华为的移动云平台，等等。

## 1.3 基本概念

### 1.3.1 概念
边缘计算(Edge computing)，也称“边缘计算”，是指在物联网边缘、车载等场景下，运行机器学习模型的技术。与传统机器学习不同，边缘计算需要考虑低延迟、低功耗等软硬件限制。在这个过程中，模型的迁移学习、模型压缩、安全加密等技术将会成为关键。

边缘计算通常采用轻量级、小型化的边缘终端设备作为计算节点，充当“边缘”“中央”“云”三者之间的媒介作用，因此也被称作边缘云计算、边缘云端计算等。其特点是资源受限，网络环境不稳定，运算能力低，但对数据传输速度要求极高，且对模型准确率要求高。因此，边缘计算往往应用在图像分析、物体追踪、智能决策等应用领域。

边缘计算的应用场景十分广泛，包括图像、语音、视频分析、物联网（IoT）、车载、智能手机、无人机等。以下是边缘计算的典型应用案例：

1. 智能摄像头：自动驾驶汽车中的摄像头是边缘计算的典型应用场景。由于摄像头的安装位置一般远离车轮中心线，摄像头捕获的图像信息很少会直接送到云端，而是先经过边缘计算处理之后再上传至云端。这样可以节省带宽资源，减少网络流量消耗，提高汽车的操控性能。另外，智能摄像头还可以用于拍摄路况、监测环境健康状况、活动异常情况等，帮助运营部门提前掌握安全隐患，并在必要时主动采取行动。

2. 无人机追踪：无人机的无线通信覆盖范围一般较窄，传感器信号传播距离短，无法覆盖整条无人机飞行路径，因而对无人机的实时定位、跟踪具有重要意义。无人机追踪的关键技术是空间定位和地图构建，常用的方法是地面站部署基站天线，利用无线电导航方式获取飞机位置信息。但是，这种方法对云端的依赖较大，在不可靠的网络环境、高海拔地区等条件下，定位精度和可靠性较差。为了解决这一难题，无人机可以使用边缘计算的方式进行位置估计，结合摄像头拍摄的遥感影像等信息进行实时建模，最终提供准确的无人机位置信息。

3. 运动检测：物体移动检测也是边缘计算的典型应用场景。在工业自动化、智能安防、智慧农业等领域，运动检测的主要目的在于监控、预警和控制。由于相机安装的位置偏离了采集对象的中心，相机捕获到的数据只能用来预警、监控，不能保证检测效果的可靠性。因此，边缘计算可以在相机之前安装传感器，对其捕获的视频帧进行边缘计算处理，从而得到检测效果的可靠性和实时性。

4. 语音识别：边缘计算在智能语音助手、智能客服、智能设备、多用户聊天机器人的应用中发挥着重要作用。语音识别是最基础的应用，使用边缘计算可以帮助降低延迟，提升识别率，解决连接质量不稳定的问题。

### 1.3.2 基本术语
#### 1.3.2.1 算力单位（Power Unit）
一般情况下，边缘计算的算力单位是比传统服务器多出几千倍的芯片，即每台服务器拥有的处理能力可以达到几万亿级。此外，边缘计算平台还包含很多分布式的边缘计算设备组成计算集群，每台设备的算力资源并不是独占的，而是共享集群的。因此，边缘计算的资源单元通常是一个计算模块或一组处理单元。
#### 1.3.2.2 远程数据管理（Remote Data Management）
远程数据管理(Remote Data Management, RDM)是指边缘计算平台上的数据存储、传输、处理等功能。它通过将存储和处理功能分布到各个边缘计算设备上，来实现数据的本地化和减少数据中心的负担。RDM通过智能调度和计算资源管理，将计算任务分配到可用资源上，提高平台整体的计算能力和稳定性。同时，它还支持实时同步，确保平台数据的最新性。
#### 1.3.2.3 数据中心内核（Data Center Kernel）
数据中心内核(Data Center Kernel, DCK)是在边缘计算平台中运行的操作系统内核，它负责处理平台内部各项任务，包括存储、网络、计算资源管理、应用编程接口等。DCK将平台的各项功能模块化，并且提供了统一的编程接口，使得开发人员可以方便地进行扩展和自定义。
#### 1.3.2.4 混合云（Hybrid Cloud）
混合云(Hybrid Cloud)是指将数据中心的部分计算资源迁移至云端，以提升平台整体的计算能力和灵活性。混合云的主要优点是解决了数据中心内核资源密集的问题，可以有效节省数据中心的算力开销，同时又可以利用云端的计算能力、存储能力、网络资源、弹性伸缩等优势。
#### 1.3.2.5 安全加固（Security Hardening）
安全加固(Security Hardening)是指通过软件层面的加固措施来增强边缘计算平台的安全性。加固措施如采用虚拟机隔离技术、密码学安全措施、加密存储等。它可以防止边缘计算平台遭受外部威胁，保障平台数据的完整性、可用性和安全性。
