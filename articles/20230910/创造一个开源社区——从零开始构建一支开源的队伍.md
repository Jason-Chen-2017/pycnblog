# 3.介绍
云计算是一个快速发展的新领域，越来越多的人开始关注云计算的最新技术。作为一个IT从业者，我们也经历了云计算的刚起步阶段。虽然云计算给予我们很多便利，但是在应用时却还是遇到了一些问题。云计算的最大问题就是缺乏可信任的平台。因为云计算的服务具有高度的弹性性，可以满足用户的各种需求，所以它可能带来很多的不确定性。如果不能信任云提供商的服务质量，那就很难保障用户的数据安全。另外，云计算平台还存在一个分布广泛的问题。不同地区的云提供商所提供的云资源规格、性能可能会千差万别，甚至同一供应商不同的产品之间功能也会有出入。为了解决这些问题，越来越多的公司和组织选择开源的方式来共享云计算平台的优点。这种模式让开发者们能够更好地利用自己的技能，快速、有效地提升技术水平，促进技术的进步。这也是开源的缘由所在。
虽然云计算和开源都是云端技术，但它们有着不同的使命。云计算服务于商业或金融领域，并且提供各种服务。而开源则专注于代码，它的使命就是向公众开放源代码，帮助开发者解决现实世界中遇到的问题。可以说，云计算的出现使得开源变得更加重要，因为只有开源才能真正实现价值创造。通过参与开源，你可以在学习过程中提升你的能力，获得反馈，扩展你的影响力，并最终为社会做出贡献。所以，你需要构建一个开源的团队，并以开源的态度参与到开源项目中。
# 4.什么是开源？为什么要参与开源项目？
开源(open source)最早被定义为一种计算机科学研究领域，即开放源代码及其相关法律条款。其理念意指任何人都可以在开放的基础上任意使用、修改或者再分发软件。可以想象，开源就是一种协作共享的工作方式，它鼓励开发者把自己的代码免费发布出来，接受大家的批评和改进。其次，开源对于行业内的竞争力和创新力是有巨大的促进作用。例如，Linux 操作系统、Git 版本管理软件、Python、Ruby、MySQL 数据库等都是开源项目。通过学习、研究开源项目，您可以掌握该领域的最新技术、业务模式和前沿发展方向，提升个人能力，增强个人品牌，建立个人关系网，推动社会的进步。
## 为什么选择参与开源项目？有哪些注意事项？
目前，互联网行业处于蓬勃发展的阶段，技术日新月异，每天都有新的技术出现。在这样的大环境下，开源项目无疑成为一个热门话题。下面是一些注意事项：
- 拥抱开源的风潮，是很好的时代选择。
- 学习开源项目的最新技术，了解这个领域的最新发展动态。
- 将自己的工作、研究成果以及知识产权纳入开源社区，保护自己的知识产权。
- 积极参与开源社区活动，贡献自己的力量。
- 善待他人的贡献，不要过度吹捧自己的才干，更要尊重别人的付出。
- 不要盲目跟风，选择适合自己的项目。
- 不要恶意破坏开源项目，保持善良风气，维持项目的良好氛围。
# 5.GitHub 的使用方法
Github 是最大的开源代码托管网站，面向开源及私有软件项目，支持所有类型、所有编程语言。它 offers 包括代码托管、任务协作、讨论协作、代码 Review 与统计分析在内的多种服务，而且 Github 是一个非常热门的开源项目社区。下面介绍一下怎么在 Github 上创建项目、提交代码、创建 PR（Pull Request）等常用操作。
## 5.1 创建一个仓库
首先，在 Github 上注册账号，然后点击头像左边的“+”按钮，选择“New repository”，输入项目名称、描述和许可证即可创建仓库。如图所示：


## 5.2 创建一个 README 文件
README 是每个项目的第一道关卡，它是对项目的一份概览。建议每个项目都有一个 README ，里面应该包含项目的背景、安装说明、使用说明、开发文档、代码目录结构等。在已有的仓库中点击“Add file”按钮，然后选择 “Create new file”，输入文件名 README.md ，点击右侧的铅笔图标编辑。填写完毕后，点击 “Commit changes” 完成。


## 5.3 添加 LICENSE 文件
LICENSE 表示项目的授权协议，在没有其他协议的情况下，默认使用 MIT License 。一般在项目根目录创建 LICENSE 文件即可。

## 5.4 使用 Git 命令行工具进行版本控制
版本控制是进行代码管理的一种有效的方法。借助 Git 可以对代码进行分支、合并、回滚等操作。在项目根目录执行如下命令，初始化本地仓库：

```
git init
```

接着，添加文件到暂存区：

```
git add.
```

最后，提交更改到远程仓库：

```
git commit -m "first commit"
```

这样，本地仓库已经准备就绪，可以向远程仓库推送代码。
## 5.5 提交一个 Pull Request（PR）
当完成了一个比较有意义的功能，或者完成了一个模块，就可以把代码提交到远程仓库。一般来说，远程仓库会要求先 fork 项目，然后 clone 到本地，在本地完成开发，测试之后，提交 PR （Pull Request）到远程仓库，等待审核和合并。如果需要修改代码，也可以继续 push 修改到远程仓库，然后再次提交 PR 进行审核。
