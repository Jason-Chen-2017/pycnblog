# 3.基本概念术语说明
## 3.1 OSI模型

OSI（Open Systems Interconnection，开放式系统互连）七层协议模型由国际标准化组织（ISO）发布，它将通信双方的硬件、软件及网络之间分成七个层次，并定义了每个层级中的功能及通信机制。从最底层的物理层到最上层的应用层，各层又分为不同的子层，其中物理层负责比特流的传输，数据链路层负责帧的传输，网络层负责包的传输，传输层负责报文的传输，会话层负责建立和管理通信连接，表示层负责数据的编码与加密，应用层负责向用户提供应用服务。如下图所示。



## 3.2 TCP/IP协议簇
TCP/IP协议簇是Internet最基本的协议，它是Internet上应用层最常用的协议簇，由一组标准协议组成，包括IP协议、ICMP协议、UDP协议、TCP协议等。下表是TCP/IP协议簇的各层功能以及对应的端口号。

| 协议 | 名称                  | 描述                                                         | 服务      | 端口          |
| ---- | --------------------- | ------------------------------------------------------------ | --------- | ------------- |
| IP   | Internet Protocol     | 因特网协议，主要用于互相通信的主机之间的通信。                | 寻址、路由 | 0~65535       |
| ICMP | Internet Control Message Protocol | 因特网控制消息协议，用于主机、路由器之间传递控制消息。        | 错误报告   | 0~65535       |
| UDP  | User Datagram Protocol | 用户数据报协议，是一种无连接的协议，可支持高速数据传输。           | 数据包交换 | 0~65535       |
| TCP  | Transmission Control Protocol | 传输控制协议，是一种面向连接的、可靠的数据流协议，用于两个应用程序间的通信。 | 连接oriented | 0~65535       |

## 3.3 DNS域名系统
DNS(Domain Name System)域名系统，它是一个分布式数据库，可以把域名解析成相应的IP地址。目前使用的DNS服务器遍布全球，域名解析过程：

1.首先本地客户端浏览器缓存，检查浏览器缓存是否存在该域名对应的IP地址；如果有，则直接访问，否则继续执行2步。
2.本地域名服务器缓存，检查本地域名服务器缓存是否存在该域名对应的IP地址；如果有，则返回给客户端，否则继续执行3步。
3.根域名服务器，首先向根域名服务器发送请求，获得该域名的服务器地址列表，然后再向这些地址服务器进行查询。
4.顶级域名服务器，获取域名服务器地址后，根据注册商的要求向顶级域名服务器发送请求。
5.权威域名服务器，获取顶级域名服务器的授权后，向其查询得到域名对应的IP地址。

## 3.4 HTTP协议
HTTP(Hypertext Transfer Protocol)是WWW(World Wide Web)的基础。HTTP协议通过请求和响应的方式，实现从浏览器或其他客户端到服务器端的通信。HTTP协议包括请求头和响应头两部分，其中请求头由请求方法、URL、版本、Accept、User-Agent等构成，响应头由状态码、服务器类型、字符集、日期、Content-Type等构成。HTTP协议可以进行GET、POST、HEAD、PUT、DELETE、OPTIONS、TRACE等请求方法。

## 3.5 HTTPS协议
HTTPS(Hypertext Transfer Protocol Secure)，即HTTP over TLS/SSL，是以安全通道建立在TCP/IP之上的HTTP协议。HTTPS协议存在两种模式：明文模式和加密模式。明文模式下，通信内容容易被窃听、篡改和伪造；加密模式下，通信内容在传输过程中通过加密算法进行加密，防止攻击者截获、修改或伪造通信内容。

## 3.6 Socket套接字
Socket套接字是应用层与传输层之间的一个抽象层，不同编程语言提供的接口不一样，但是一般都包含四个基本方法：connect()、bind()、listen()、accept()。创建Socket时，系统自动分配一个唯一的本地端口，绑定后监听等待客户端连接。当接收到客户端连接请求后，服务器调用accept()接受连接请求，并创建一个新的Socket用于与客户端通信。
