# 2.基本概念术语说明
## 2.1 Python概述
Python是一个强劲的动态编程语言，具有简单易懂的语法，并被设计成可以广泛应用于各种领域。Python支持多种编程范式，包括面向对象、命令式、函数式、面向元组的编程模型。另外，Python提供了可移植性保证，可以运行在许多类Unix和Windows操作系统上，甚至可以在嵌入式设备上运行。
Python还有一些独有的特征：
- 自动内存管理：Python采用垃圾收集机制自动回收不可达对象的内存空间，这一点非常方便程序员。
- 可扩展性：Python使用动态编译器，通过引入C语言接口，允许程序员嵌入Python解释器。
- 轻量化：Python的安装包大小仅占磁盘空间的一小部分，而且可以在不同平台之间共享。
- 丰富的标准库：Python的标准库非常丰富，其中包含常用的内置数据结构和函数，比如数组、哈希表、字符串处理、文件输入/输出等。
- 可互操作：Python程序可以与其他语言编写的库相连接，比如C/C++、Java、Ruby、Perl、Tcl、JavaScript等。
- 开源：Python的源代码完全免费开放，并且拥有庞大的社区支持。

## 2.3 Python编码风格
Python有两种官方的代码风格，即PEP8和Google Python Style Guide。它们的共同点是遵循缩进规则和空白字符的规范，因此可以使代码更容易阅读。
### PEP8
PEP8指的是Python Enhancement Proposal，它是描述Python代码风格的一种约定。其目标是促进Python代码的一致性，并保持代码的可读性。
PEP8中规定的代码风格包括如下方面：
- 使用四个空格缩进而不是制表符或其他字符;
- 每行最大长度不超过79个字符;
- 驼峰命名法;
- 模块名应该全部小写;
- 函数名、变量名采用小写字母下划线分隔;
- 私有成员以单下划线开头;
- 公开成员以双下划线开头;
- 如果一个模块或函数可能引起混淆，则给予其一个描述性名称。

很多Python IDE都会对代码符合PEP8的要求做出提示，如果遇到违反PEP8的情况，可以尝试修改代码。
### Google Python Style Guide
Google Python Style Guide就是由Google维护的一个Python代码风格规范。与PEP8相比，它更注重文档字符串（docstring）的规范和命名风格。除此之外，Google Style Guide还将一些细枝末节的地方进行了明确约束。
Google Python Style Guide的具体规范如下：
- 句子应该尽量完整，避免使用冗余的断句符号；
- 每行的字符数应该限制在79个左右，超出的部分应该分行并增加适当的缩进；
- 不要滥用注释；
- 类的名字采用驼峰命名法；
- 方法名、函数名采用驼峰命名法；
- 常量名全部大写，多个单词用下划线连接；
- 模块名全部小写，多个单词用下划线连接；
- 函数参数的名字应当具有描述性；
- 使用类型注解标注函数参数、返回值及局部变量的类型，这样可以增强代码的可读性；
- 不要滥用None，用空字符串或者空列表代替；
- try...except块需要指定异常类型；
- 文档字符串应当使用三引号而不是单引号；
- 不要在Python代码中使用shell脚本，而应该直接调用Python API；
- 需要注意的文件编码统一使用UTF-8。
