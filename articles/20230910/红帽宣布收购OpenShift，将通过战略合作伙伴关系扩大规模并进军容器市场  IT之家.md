# 1.简介
  

在IT领域中，云计算已经成为非常热门的技术方向。随着容器技术的兴起、编排系统如Kubernetes等崛起、基于微服务架构的应用流行等，容器技术的应用越来越广泛，容器集群管理工具也逐渐火爆起来。其中最著名的就是开源的Docker Swarm，Kubernetes以及CoreOS Rkt。

红帽宣布收购OpenShift，是一个不得了的消息！Red Hat宣布将加快推进其商用Kubernetes平台Red Hat OpenShift Container Platform的销售及支持业务。Red Hat OpenShift Container Platform是面向企业级应用环境设计的、适用于多租户环境的PaaS/SaaS平台。它包含了容器编排工具Kubernetes及围绕它的增值服务，例如监控、日志、存储、备份、网络、安全、以及混合云功能。红帽还将通过战略合作伙伴关系扩大规模并进军容器市场。战略合作伙伴包括专业的IT咨询公司、企业解决方案提供商、以及云服务供应商。这些合作伙伴将为客户提供基于红帽OpenShift产品的定制化支持及部署服务。

2.什么是OpenShift？
红帽OpenShift是基于Kubernetes的容器编排平台，由红帽公司推出。OpenShift主要包括两个组件：
- Kubernetes：它是容器集群管理系统，负责资源调度、容器组管理、存储卷管理等。
- Service Catalog：它是一个新的项目，旨在实现PaaS层面的应用程序交付和服务管理。OpenShift的Service Catalog让开发者可以轻松地发现、请求和消费云上可用的服务，包括数据库、缓存、消息队列等等。

总而言之，OpenShift是一个开放源码的云应用平台，它提供基于Kubernetes的容器编排、服务发现、自动伸缩、健康检查、应用路由、日志记录等功能。

3.为什么要收购OpenShift？
随着开源容器编排工具Kubernetes的蓬勃发展，越来越多的公司选择把Kubernetes作为自己的基础设施即服务（IaaS）产品之一。但是Kubernetes本身虽然能够很好地运行容器化应用，但还是需要进一步优化才能更好地满足用户的需求。而Red Hat OpenShift则为用户提供了完整的基于Kubernetes的平台，它包含了许多强大的特性，比如统一的身份验证和授权机制、完整的监控和日志体系、内置的服务发现机制等。而且Red Hat还提供了丰富的定制化服务，使得用户可以根据自身需求进行快速构建自己的PaaS/SaaS平台。因此，红帽宣布收购OpenShift的消息无疑会刺激OpenShift的关注度，也会加速OpenShift的推广。

4.收购的影响会带来哪些改变？
当然，OpenShift平台上的容器服务会得到广泛的应用。目前很多知名大型互联网公司、金融机构以及政府部门都采用了OpenShift平台来部署其业务应用。除此之外，IT界的其他IT咨询公司、云服务供应商以及互联网企业都也开始跟进OpenShift平台的发展。另外，由于OpenShift是基于Kubernetes的，这就意味着OpenShift的容器编排能力会被迅速普及，也会为容器技术的迅速发展做出贡献。

5.什么样的战略合作伙伴能够帮助红帽加快收购OpenShift？
红帽宣布收购OpenShift后，很多公司会马上加入到收购队伍中。主要参与收购的公司包括IBM、阿里巴巴、腾讯、微软、Red Hat以及其它来自于IT巨头的公司。这些公司都会在OpenShift和Kubernetes方面有所建树，并且他们都有能力和精力为红帽提供优质的支持和服务。

6.后续会发生什么变化？
红帽将继续推动OpenShift产品的发展，将不断完善它的功能并提供更好的服务。另外，Red Hat也将与更多的企业、组织以及创业团队合作，共同打造一个面向企业的全栈PaaS/SaaS平台。

最后，希望红帽OpenShift及相关合作伙伴能够携手推动容器技术的发展。
