# 2.计算机科学基础知识
## 2.1.编程语言与开发环境
### 2.1.1.编程语言
计算机程序一般需要某种编程语言编写，编程语言就是人类用来告诉计算机执行任务的指令集。目前流行的编程语言主要分为两类——解释型语言和编译型语言。
- 解释型语言：解释型语言不需要预先编译，直接运行源代码时，解释器解释执行源代码中的每一条语句。例如，Python，JavaScript等。
- 编译型语言：编译型语言需要进行预编译（编译）生成机器代码才能运行。例如，C，C++等。

选择编程语言的原则主要是根据以下几个方面：
1. 适合场景：不同类型的项目需求对编程语言的选择也会有所不同。例如，图形界面相关的应用通常采用解释型语言，后台服务、网络爬虫相关的应用通常采用编译型语言。
2. 技术的进步：目前计算机技术的发展速度非常快，新的编程语言、开发工具层出不穷。选择具有较高生态圈的编程语言可以更好地利用现有的优秀技术。
3. 社区支持：有很多开源社区提供各种编程语言的资料，帮助初学者快速了解语言特性和语法规则。同时，社区也提供了大量的库函数、框架等资源，可以降低学习成本。
4. 兼容性：由于不同的编程语言之间存在相互兼容的功能，所以同一个项目可以在多个编程语言中共存，提升开发效率。

### 2.1.2.开发环境
开发环境是指使用哪些软件和工具配合编程语言进行编程工作。主要包括文本编辑器、IDE、编译器、调试器和版本控制软件等。

1. IDE（Integrated Development Environment）集成开发环境：集成了常用的工具，如代码编辑器、编译器、调试器、版本管理软件，让用户只需简单配置即可轻松完成编程。典型的IDE有Visual Studio Code、Eclipse、IntelliJ IDEA等。
2. 文本编辑器：文本编辑器只是一种工具，并不能替代IDE的作用。最常用的文本编辑器是Notepad++、Sublime Text、VSCode等。
3. 编译器：编译器用于把源代码编译成可执行文件，由此运行该程序。编译器分为静态编译器和动态编译器。
    - 静态编译器：编译过程中所有的源代码都被一次性加载，输出目标文件后就结束了。静态编译器的缺点是编译时间长，而且运行时内存占用较大。
    - 动态编译器：编译过程中仅加载被修改过的代码，提升运行时的效率。但是动态编译器的更新频率较慢，而且要依赖于运行时环境，故其安装过程较复杂。
4. 调试器：调试器用于跟踪代码运行时的变量值、函数调用栈、异常信息等。有的IDE自带调试器，有的语言也内置了调试器，如Python的pdb模块、Java的JVM调试器。
5. 版本控制软件：版本控制软件用于记录代码的变动情况，方便团队协作开发。GitHub、Gitlab都是流行的版本控制软件。

除了以上这些工具外，还有其他工具如单元测试、性能测试、代码质量检查等，也需要开发人员掌握。

## 2.2.数据结构与算法
数据结构和算法是编程的基石，也是构建各种高效且健壮的软件的关键。
### 2.2.1.数据结构
数据结构是指按照怎样的规则存储数据，以及数据之间的关系和联系。常见的数据结构有数组、链表、树、图、哈希表等。
#### 2.2.1.1.数组
数组是一个连续的内存区域，可以存放相同类型的数据元素。数组的大小是固定的，一旦初始化后不可更改。数组的长度可以通过下标来访问元素。数组的插入和删除操作十分高效，因为仅需要移动数组中相应位置上的元素。
#### 2.2.1.2.链表
链表是由节点组成的无限序列，每个节点都保存一个数据元素和指向下一个节点的引用。链表中不存在环，可以往其中添加或者删除元素。链表的最大好处是可以在 O(1) 的时间内查找某个元素，但插入删除元素的时间复杂度是 O(n)。
#### 2.2.1.3.树
树是一颗有根结点和若干子树的集合。树的结构类似于倒立的棍子，每条边代表孩子的方向，从而构成了一系列的节点。树的高度定义了树的结构的复杂程度，通常用节点的个数来衡量高度。二叉树、二叉搜索树等都是树的一种。
#### 2.2.1.4.图
图是由顶点和边组成的集合，表示对象之间的连接关系。图常用于表示地图、组织结构、网络结构等复杂系统。图的存储结构可以采用邻接矩阵或邻接列表的方式。
### 2.2.2.算法
算法是对特定计算任务进行描述的清晰定义的计算过程。算法的准确性和有效性决定了一个算法的优劣。
#### 2.2.2.1.排序算法
排序算法是将一串数字按一定顺序排列的算法。排序算法一般包括以下几种：
- 插入排序：依次比较两个元素并交换位置，直至整个序列有序。平均时间复杂度O(n^2)，最坏时间复杂度O(n^2)，空间复杂度O(1)
- 选择排序：选择最小或者最大的元素，放到序列的首位，然后继续选择剩下的元素，直至整个序列有序。平均时间复杂度O(n^2)，最坏时间复杂度O(n^2)，空间复杂度O(1)
- 冒泡排序：两两比较两个元素，若逆序则交换位置，重复 n 次，最终整个序列有序。平均时间复杂度O(n^2)，最坏时间复杂度O(n^2)，空间复杂度O(1)
- 快速排序：选择一个元素作为枢轴，比它小的放左边，大的放右边，递归处理左右子序列。平均时间复杂度O(nlogn)，最坏时间复杂度O(n^2)，空间复杂度O(logn)
- 堆排序：堆是一种树结构，父节点的值总是大于等于它的子节点的值。先将序列构造成最大堆或最小堆，每次取堆顶元素放到序列末尾，反复迭代，直至整个序列有序。平均时间复杂度O(nlogn)，最坏时间复杂度O(nlogn)，空间复杂度O(1)
#### 2.2.2.2.查找算法
查找算法是确定某一特定元素是否在列表或集合中出现的过程。查找算法一般包括以下几种：
- 线性查找：从第一个元素开始向后遍历整个列表，若找到目标元素则返回；否则返回-1。平均时间复杂度O(n)，最坏时间复杂度O(n)，空间复杂度O(1)
- 分块查找：将序列划分成多个相同大小的子序列，并对每个子序列分别进行线性查找。平均时间复杂度O(m+n/m)，最坏时间复杂度O(nm)，空间复杂度O(1)
- 索引查找：维护一个索引表，在列表中查找时首先查看索引表，若找到目标元素的索引则直接定位；否则进行线性查找。平均时间复杂度O(logn)，最坏时间复杂度O(n)，空间复杂度O(n)
- 哈希查找：将列表中所有元素映射到一个固定大小的数组中，通过计算索引值得到元素在数组中的位置，然后比较数组元素的值。平均时间复杂度O(1)，最坏时间复杂度O(n)，空间复杂度O(n)
