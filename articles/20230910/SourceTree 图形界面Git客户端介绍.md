# 1.简介
  

## 什么是 Git？
Git 是目前世界上最先进的分布式版本控制系统（DVCS），是一个开源项目，它的设计目标是有效、高速且非常可靠地处理从很小到非常大的项目的所有方面。

## 为什么要使用 Git？
- 1.速度快
  - 由于 Git 的设计使它在速度上远远超过了其它 VCS （如 SVN）之上，尤其是在大型项目或多人协作的环境中。通过 Git 的分支模型及其基于引用的工作方式，开发者可以在短时间内完成大量并行的工作。
- 2.灵活性强
  - Git 可以用于管理各种不同的文件类型，包括文本文件、二进制文件和目录等。并且可以自定义不同级别的提交权限，允许开发者控制对代码库各个部分的访问和共享。
- 3.简单易用
  - 在学习曲线不陡峭的情况下，无论是在桌面环境还是服务器环境中，Git都可以轻松安装配置并使用，而且还提供了丰富的命令，极大地提升了日常工作的效率。
- 4.强大的分支管理能力
  - 使用 Git 分支，可以方便地进行复杂的集成测试或发布流程。同时，Git 提供了多种方法支持团队之间的合作和协作，让多个开发者可以各自拥有自己的分支，互不干扰地进行工作。
- 5.集成环境友好
  - 大多数流行的 IDE 和编辑器都内置了 Git 支持，可以更加便捷地进行版本控制。

## Git 发展历史及今日的主要功能特性
### 1. Git 的创始人
Linus Torvalds，是 Linux 操作系统的创建者之一。

### 2. Git 发展史
- 1991 年，linus 用 C 语言写了 git 的第一个版本
- 2005 年，git 迎来了第一次商业化运营，贝尔实验室和其他公司参与到了这个开源项目的开发和维护当中
- 2018 年，github 宣布收购 gitlab，gitlab 正式成为 git 的全球领航者。
- 2020 年，git 迎来了 10 岁生日 🎂😀😊🎉🥳👨‍💻✨

### 3. Git 的主要功能特性
- 1.快速和简单的分布式
  - Git 采用的是分布式架构，它的克隆、推送、拉取等操作都是本地执行，不需要联网，速度飞快。
- 2.数据完整性保障
  - Git 使用 SHA-1 哈希值来保证存储数据的完整性，所以说 Git 不容易出现丢失或遗漏的数据的问题。
- 3.完备的暂存区和工作区
  - 在 Git 中，有三个区域的概念：暂存区、工作区、仓库。暂存区用于保存准备提交的文件修改，工作区用于代码编写，仓库用于存放版本库。
- 4.丰富的合并策略
  - 通过多种合并策略，Git 能够自动合并文件冲突，减少开发者的手动合并工作量。
- 5.高效的日志系统
  - Git 有着优秀的日志系统，能够清晰地记录每次提交更新的内容。
- 6.强大的分支功能
  - Git 分支是 Git 的一个核心功能。每个开发者可以独立创建和管理自己的分支，代码合并也十分便捷。
- 7.强大的交互性
  - Git 对终端用户的命令做了优化，所有命令都有详细的帮助信息。
