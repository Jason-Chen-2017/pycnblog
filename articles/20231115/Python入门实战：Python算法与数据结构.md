# 2.核心概念与联系
## 2.1 数据结构
数据结构是指组织、存储和管理数据的方式。它是为了使数据更容易被访问、修改、添加、删除、查找等操作而建立的抽象数据类型。常见的有列表（list）、元组（tuple）、字典（dictionary）、集合（set）。不同的数据结构之间存在着相似之处，不同之处则在于它们的特征和特点。

## 2.2 算法
算法（algorithm）是指用来解决特定问题的一系列指令、操作或流程，是用来操作数据以达到预期目的的指令集、问题的求解步骤、指令的顺序。算法是一种灵活机械化、规范化的解决方案，它定义了一个清晰的目标，并提供了描述该算法所需步骤的规则。

## 2.3 抽象数据类型
抽象数据类型（Abstract Data Type，ADT）是数据类型与数据结构之间的桥梁。它是一种数据类型，代表一类数据结构以及一族操作它所支持的运算。在抽象数据类型的概念下，数据结构和算法可以看作是不同的模块，只不过算法提供的是对数据的处理方式，而数据结构则维护了数据的内部表示形式。

## 2.4 时间复杂度
时间复杂度（Time Complexity），又称时间/空间复杂度（Space Complexity），是衡量算法执行效率的重要参数。它反映算法运行过程中占用的内存数量和时间消耗。时间复杂度描述了算法的执行时间随输入规模的变化情况，一般用大 O 符号表示法表示，例如 O(n) 表示随着输入规模 n 的增大，算法的执行时间以恒定的速度增加。

## 2.5 分治策略
分治策略（Divide and Conquer Strategy）是指将一个大的问题分成两个或多个小的相同问题，递归地解决这些子问题，然后再合并结果，得到整个问题的解。分治策略常用于解决复杂问题，即使该问题的规模不能直接由计算机完成处理。

## 2.6 动态规划
动态规划（Dynamic Programming）是一种通过把大问题拆分成相互独立的子问题来求解复杂问题的方法。动态规划通常适用于存在重叠子问题和最优化原理的组合问题。

## 2.7 回溯法
回溯法（Backtrack）是指在搜索解空间树的同时尝试，失败时退回一步重新选择路径。回溯法常用于解决组合问题，例如组合数学中的排列和组合问题。
