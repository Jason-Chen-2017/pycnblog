                 

# 1.背景介绍


## 数据中台简介
数据中台是以数据仓库作为核心，通过构建一套完整的数据服务体系，提供给业务团队及其他部门使用的数据集成、整合和分析能力，并且该数据服务体系可支持不同部门的多个业务线共同使用，形成一个完整的数据价值网络。数据中台由三个层级组成：数据仓库层、数据采集层和数据服务层，如下图所示：

数据仓库层：
数据仓库包括各种类型的数据，如财务数据、人力资源数据、产品销售数据等，汇总到一起形成一系列的维度、指标和聚合结果表，并进行清洗、规范化、计算、验证、反范畴化、关联、链接和加工，以便更好地进行数据分析和决策支持。数据仓库除了能存储原始数据外，还需要经过ETL（抽取-传输-加载）过程对数据进行清洗、规范化、计算、转换、校验、赋权、审核、归档等处理，确保数据质量和完整性。

数据采集层：
数据采集层负责从企业各个系统、数据库或第三方平台获取实时数据并转换为标准化结构的数据格式，供数据仓库层进行后续分析和决策支持。数据采集层包括数据接入层、数据提取层和数据存储层。其中，数据接入层用于连接外部数据源，如数据库、文件系统、API接口等；数据提取层则用于从接入层获取数据并进行清洗、规范化、过滤和转换等处理，输出符合数据仓库要求的数据格式；数据存储层则将提取完毕的数据持久化存储至数据仓库层。

数据服务层：
数据服务层提供了一系列的基于数据仓库的业务应用服务，例如报告查询、分析预测、决策支持等，并将这些服务提供给业务团队及其他部门使用。数据服务层包括数据发现层、数据应用层和数据访问层。其中，数据发现层通过数据挖掘方法发现数据间的关联关系，形成数据驱动的业务规则；数据应用层则基于业务需求，通过应用服务，如数据分析、数据报表、数据推荐等，提供基于数据的业务价值；数据访问层则为其他部门提供数据分析、报告查询等应用服务。

数据中台具有以下优点：

1.降低了业务部门之间的沟通成本，实现业务数据共享和复用，统一数据源；

2.提升了数据分析和决策的准确性和效率，实现数据共享的同时对数据质量和正确性做出保障；

3.降低了数据采集、清洗和存储的复杂度和成本，可节省宝贵的人力物力资源；

4.实现数据治理，自动化数据采集、存储和服务交付流程，支持自动发现、生成数据价值网络，确保数据价值最大化。

# 2.核心概念与联系
## 多租户的定义
多租户就是一种允许多个租户共用相同的服务器、软件和硬件资源，每个租户只能看到自己的数据或服务，且不能随意访问或者篡改别人的数据或服务的计算机技术。换句话说，多租户就是一种虚拟化的解决方案，目的是为了让不同的用户或组织之间能够安全地共享系统资源而创建的一种方案。在多租户环境下，每一个租户都有其自己的账户信息、密码和权限，且只能在自己的账户范围内进行操作和访问。租户可以是个人或公司，但通常情况下，我们把组织称作租户。举个例子，假设有两个组织，分别是A和B，那么A和B的租户就可以分成两部分，比如A的租户可以是两个员工，而B的租户可以是五个部门。

## 多租户与单租户之间的区别
对于多租户来说，它带来的好处之一就是节约成本。因为无论是硬件还是软件，如果不做划分，都会浪费掉。另一方面，对于企业来说，无论是在生产上还是销售上，采用多租户的方式可以降低总拥有成本。但是，多租户也会存在一些问题，比如安全问题、配置管理问题、合规性问题等。所以，单租户和多租户两种架构都有其各自适用的场景。

## 普通用户与管理员角色的概念
多租户中，普通用户只能看到自己的数据和服务，管理员可以看到所有租户的数据和服务。管理员一般可以管理租户设置、租户成员和租户的业务设置等。当某个租户被禁用、需要注销等情况发生的时候，管理员可以很快的切断其与其它租户的联系，实现租户数据安全。因此，多租户架构下，管理员角色相对较重要。
