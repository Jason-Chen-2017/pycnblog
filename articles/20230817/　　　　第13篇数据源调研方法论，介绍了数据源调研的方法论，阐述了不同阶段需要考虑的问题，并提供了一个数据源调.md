# 1.简介
  

数据源调研是指对所需的数据进行收集、整理、分析、分类等过程，以制作出具有实际用途的数据集，对数据的价值和应用情况进行评估，从而确定数据的获取渠道和质量保证措施。数据源调研是信息化建设的基础工作之一，也是信息化企业经营管理的一项重要环节。 

由于缺乏对数据源调研的充分认识和知识储备，致使很多公司在决定采用什么样的数据源时出现较多不确定性，增加了信息化项目的风险。在这个过程中，通过对数据源调研的研究，能够对各类数据源的优劣特征进行评估、选择，以及了解数据需求方面提出的相关问题，从而对信息化建设、客户关系管理等领域起到积极作用。 

本文将结合个人的工作经验，对数据源调研方法论进行系统的介绍，并梳理其中的一些具体操作步骤，希望能给大家带来更多宝贵的信息。 

# 2.基本概念术语说明

2.1 数据源分类
数据源包括以下几种类型：

- 外部数据源：即从第三方机构、专家等获得的数据，如社会经济统计数据、天气预报数据、地理信息数据等。 
- 技术数据源：由IT部门或系统提供的数据，如业务数据库、电子邮件服务器日志数据、ERP系统数据、运维监控系统数据等。 
- 用户数据源：用户自行产生的数据，如问卷调查结果、手机通话记录、交易记录、新闻阅读记录等。 
- 内部数据源：企业内使用的各种数据，如人事信息、销售数据、库存状况、产品信息、财务数据等。 

2.2 数据源调研方式
数据源调研的方式一般包括四种：

- 市场调研：通过对企业的产品及服务、竞争对手的产品和服务等进行调研，得出针对特定市场的产品或服务的最优解。
- 市场分析：通过对已收集到的企业数据进行分析，将客户群体、市场消费行为、需求、资源、成本、获利、市场占有率等方面的数据进行综合分析，找出能够满足数据需求的人才、物品、服务、环境等因素。
- 竞品比较：通过对竞争对手产品或服务的相关数据进行比较，找出各方面的差距，分析竞争对手产品或服务的优点、缺点、机会、困难以及如何克服它们，为自己的产品或服务提供参考。
- 模型设计：根据已有的相关数据建立模型，并根据模型预测未来的市场趋势。

2.3 数据源调研阶段
数据源调研有五个阶段：

- 概念验证阶段：即定义目标、明确范围、准备阶段，主要用于确定市场、需求、解决方案以及数据的优先级，并制定相应的调研方案。 
- 收集数据阶段：即找到数据源，收集和整理数据，进行初步的数据分析。 
- 数据筛选阶段：对收集的数据进行初步的清洗、过滤、归纳，确保数据准确、完整。 
- 数据分析阶段：对数据进行整理、分析，找出主要的变量、潜在影响因素以及数据的关联关系。 
- 数据报告阶段：对调研结果进行总结、归纳、总结，并制作成文档。

3.1 数据源分类方法
数据源分类方法主要分为以下两种：

- 层次划分法：按数据源的大小、复杂度、量、质、有效性等因素进行层次划分，如全国性数据源、省级数据源、县级数据源、镇级数据源。 
- 主题分类法：基于数据的重要特性和目的，将数据集划分为不同的主题，如常规数据的采集、处理、存储、查询；业务数据的采集、分析、应用等。

4.2 数据源评价方法
数据源评价方法主要分为两大类：

- 内部指标：如投入产出比、盈利能力、运营效率、核心竞争力等。 
- 外部指标：如价格、易用性、数据可用性、定期维护、信息咨询等。

4.3 数据源分析流程
数据源分析流程一般包含以下五个步骤：

- 挖掘阶段：对数据进行初步的挖掘，收集各种数据，如市场信息、产品信息、客户信息、销售数据等。
- 数据预处理阶段：对收集的数据进行初步的清洗、过滤、归纳，确保数据准确、完整。 
- 数据分析阶段：对数据进行整理、分析，找出主要的变量、潜在影响因素以及数据的关联关系。
- 数据报告阶段：对调研结果进行总结、归纳、总结，并制作成文档。
- 数据应用阶段：实现对数据的集中应用、数据共享、决策支持。 

4.4 数据源建议和检查表
数据源建议和检查表可以帮助企业快速确定数据源，并避免因缺少必要的数据而造成的意想不到损失。该表格主要包括以下几个部分：

- 数据来源建议：指出企业应当购买哪些数据来源。 
- 数据收集和整理建议：指出企业应当采集哪些数据，收集这些数据的方法和时间。 
- 数据分类建议：指出企业应当将不同类型的数据分别放在何处，以及采取哪些分类方式。 
- 数据收集周期建议：指出企业应当每隔多少天、多少小时、多少分钟收集一次数据。 
- 数据质量管理建议：指出企业应当设置哪些数据质量管理标准，及时对数据质量进行检查和处理。
