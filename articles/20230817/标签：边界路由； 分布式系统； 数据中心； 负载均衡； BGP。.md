# 1.简介
  

## 概述
边界路由器（border router）又称边界网关，其主要功能是在数据中心内将不同地域的网络连接起来，实现不同地域之间的通信。边界路由器可以理解为一种私有网络，只有特定的运营商才有权力接入、控制和管理它。边界路由器与其他数据中心内部的网络之间通过一组专用的接口相连，这些接口可以是物理接口或者逻辑接口。从而，边界路由器可以使得数据中心能够更好的适应快速变化的业务需求。边界路由器主要由以下几种：

- VPN Edge Router：VPN边界路由器是一种基于IPsec的边界路由器，可以为企业提供VPN服务，且支持多协议VPN。VPN边界路由器通过VPN隧道封装VPN流量并在数据中心内发送出去。
- Core Router：核心路由器是一种可以对Internet、公司本地网段、合作伙伴网段进行路由和访问控制的路由器。核心路由器可以充当整个数据中心的中央控制器，为多个不同的运营商提供服务。核心路由器可以通过BGP协议来与互联网路由器交换路由信息，并根据本地策略进行策略路由。
- Access Router：接入路由器是一种专门用来访问公司局域网或企业Wi-Fi的路由器。因为通常每个办公室都需要接入Internet，所以需要一个专门的接入路由器来处理这种特殊的业务需求。
- SDWAN Router：SD-WAN路由器是一种基于软件定义网络（Software Defined Networking，SDN）的路由器，可以灵活地管理复杂的网络环境，提升网络可靠性和性能。SD-WAN路由器可以充当边界路由器，实现在多个数据中心之间建立连接。

边界路由器的功能与应用非常广泛。如今，边界路由器已经成为企业网络设备的标配设施，不仅用于连接不同区域的网络，还用于各种业务场景，如托管云计算、分布式系统等。无论是IPSec VPN、核心路由、接入路由还是SD-WAN，它们都为数据中心的网络提供了可靠的连接和快速的数据交换能力。因此，本文将详细介绍边界路由器的一些相关知识，包括其基本概念、工作原理、配置方法、使用限制等。

## 发展历史
随着互联网的迅速发展，互联网带宽的呈线性增长逐渐形成瓶颈，甚至出现过拥堵现象。为了解决网络通信中拥塞和延迟的问题，1996年美国斯坦福大学开发了TCP/IP协议栈，该协议栈打破了传统网络的设计框限，将网络结构拆分为多个子网，并引入虚拟链路技术，允许多个用户同时共享同一条物理链路，可以有效降低网络拥堵风险。但由于缺乏基础设施的支持，该研究无法实践，也没有引起足够大的关注。直到2007年，华盛顿大学的Cisco开发出第一个支持IPv6的互联网协议栈。经过一番发展后，新的互联网协议和通信方式不断涌现出来，但边界路由器却始终停留在视觉上，没有进入实际应用阶段。

2009年，美国AT&T贝尔实验室开发了第一款边界路由器——Tribono系统。该路由器是一个基于IP协议的边界路由器，可以为企业提供VPN、缓存、加密等功能。在其之上，IT部门被赋予了管理VPN系统、维护网络运行状况和分析业务情况的任务，使得其成为边界路由器的重要角色。

2011年，美国国家安全局宣布封锁该公司网站，该事件让边界路由器市场上出现了一场更严重的危机。2012年，AT＆T贝尔实验室推出Tribono系统新版本，加入了更多的功能特性，如支持动态BGP路由更新、兼容IPv6等。

2015年，AT＆T贝尔实验室正式宣布退出边界路由器市场。美国政府又宣布解除对该公司的限制，现在任何人都可以在线下注册、购买并部署边界路由器。
