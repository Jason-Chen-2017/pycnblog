# 1.简介
  

物联网（Internet of Things，简称IoT）是一种从网络连接起来的现代化技术，通过应用物理系统、信息技术、计算技术及其相关软硬件设备所形成的新型计算环境。IoT已经成为物流、运输、生产、医疗卫生、电力、环保、制造等各行各业不可或缺的一部分。随着IoT技术的飞速发展，涌现出了诸多颠覆性的创新产品，如智能路由器、智能视频球机、智能车载充电系统、智慧农业自动监控、智能运动健身器材等，具有极高的实用价值。 

作为物联网领域中的一个重要组成部分——即智能终端，智能终端是指能够进行复杂的计算任务和识别各种传感器信息并作出反应的硬件设备，包括传感器、微控制器、显示屏、摄像头等。目前，IoT终端有很多不同的类型，如智能手机、嵌入式系统、无人机、传感卡、电子词典、穿戴设备等。根据国际标准定义，目前最常用的智能终端类型为嵌入式系统，但也会出现一些其他类型的终端如智能手表、家庭安防、车载充电器等。 

为了使得用户获得更好的体验，提升产品的可用性、可靠性和服务质量，IoT终端应当具备良好的用户界面设计、用户友好操作、安全性能、鲁棒性、可维护性、可扩展性和高效能等特点。因此，物联网终端的开发者们需要不断提升自己对终端的认识、理解，通过提升技术水平、优化设计、完善功能实现、引入竞争机制，以满足用户的需求。 

而在互联网和云计算的帮助下，物联网技术已经逐渐演进成熟，越来越多的人群加入到物联网的怀抱中来，如商家、智能城市、边缘计算、住宅智能等。此外，随着5G、AIoT等物联网技术的发展，智能终端将进入新的时代，提升智能生活品质，促进经济社会的转型升级。 

# 2.基本概念术语说明
## 2.1 物理层、数据链路层、网络层、传输层和应用层
物联网技术借助现有的通信网络，通过物理层、数据链路层、网络层、传输层和应用层协议来实现通信。如下图所示：

① 物理层(Physical Layer):负责0和1序列信号的传输、物理特性的测定和描述，包括物理编码、调制、解调、同步、噪声的处理等功能。它主要用于规划、分配、控制数据的传递方式，同时也提供高速且稳定的地面信道通信。

② 数据链路层(Data Link Layer):主要功能是将上层的数据块打包成帧，并通过线路传送到相邻计算机。数据链路层还要负责重传、错误纠正、流量控制等。

③ 网络层(Network Layer):主要职责是将不同节点之间的通信请求转发到相应的目的地，选择合适的路径和路由。

④ 传输层(Transport Layer):传输层提供了通信的端到端的可靠性和流量控制。其中，面向连接的传输控制协议 TCP (Transmission Control Protocol) 提供可靠的、面向连接的字节流服务，而无连接的传输控制协议 UDP (User Datagram Protocol) 提供不可靠的、无连接的报文服务。

⑤ 应用层(Application Layer):主要负责网络应用程序的开发。例如，域名系统 DNS (Domain Name System)，网络浏览器 HTTP 和网络邮件 SMTP 都是属于应用层的协议。



## 2.2 MAC地址和IP地址
物联网中，每台终端都有一个唯一的物理地址(MAC Address)。该地址由数字组成，通常分为两部分：网络设备地址（NICA）和媒体访问控制（MAC）地址。网络设备地址（NICA）由厂商组织分配，用来标识网络适配器；MAC地址则由网卡制造商（如三星、苹果、微软、英特尔等）随机分配给每个设备。MAC地址的第二个部分，也就是最后的三位，通常由制造商自行分配。如中国移动的前两位为0xAC:0xD8；中国联通的前两位为0x7C:0xD9；中国电信的前两位为0xAC:0xD8。

与物理地址对应的，网络层的IP地址也很重要。IPv4地址是一个32位的二进制数，它通常被表示成四个十进制数（范围0～255），中间以点号隔开。IPv6地址是128位的二进制数，通常以8个“：”隔开，每个部分用两个十六进制数字表示，共有8组4个十六进制数字。

IP地址是由互联网工程任务组（IETF）指定的用于标识主机和路由器等网络结点的数字标签，由一串由点号分割的数字组成，经过路由查找后确定目标计算机的位置。IP地址可以分为以下几类：

1. IPV4地址：使用32位的整数表示，通常以点分割四段8位数字，格式为XXX.XXX.XXX.XXX。
2. IPV6地址：使用128位的二进制数组表示，通常采用8组4位十六进制数，格式为XXXX:XXXX:XXXX:XXXX::XXXX。
3. 私有地址：由用户私自分配，并指定用途的地址，通常以10.0.0.0/8、172.16.0.0/12和192.168.0.0/16为前缀，格式为XXX.XXX.XXX.XXX。
4. 公用地址：由ICANN（互联网名称与数字分配机构）管理的地址，全球范围内有效，格式为XXX.XXX.XXX.XXX。

## 2.3 概念术语
**设备注册**：是指把终端设备或者终端设备的一部分绑定到一个固定的IP地址上去，这样就可以使得该设备能够通过IP寻址的方式访问互联网。设备注册一般包括设备身份确认、安全配置、设置默认网关等工作。由于IoT设备数量庞大，设备注册工作又非常繁琐，往往需要耗费大量的人力资源，因此在IoT技术刚刚起步的时候，没有建立统一的设备注册中心，导致设备之间通信无法准确、快速地完成。目前，很多IoT平台都会提供自己的设备注册服务，也可以利用现有的注册中心服务。


**设备认证**：是指根据具体场景，通过指定的方法对终端设备进行身份鉴别验证，比如智能手机扫码登录、指纹识别验证等。由于IoT终端设备普遍具有较低的处理能力，无法执行复杂的加密运算，只能依赖于相对简单的密码或密钥进行设备认证。所以，目前很多公司采用基于密钥协商算法的设备认证方法，让终端设备主动生成一次密钥，然后发送至服务器端进行校验，如果校验通过则认为设备已成功注册。这种认证过程比较简单，安全性一般，但也能满足某些特定需求。


**数据收集**：是指终端设备上的传感器采集的原始数据经过处理后，通过网络传输到服务器端。数据收集一般包括数据清洗、数据转换、数据压缩、数据加密等工作。由于IoT终端普遍采集大量的原始数据，因此数据收集处理工作占据了很大的比例，但是目前的IoT平台对数据的处理和存储仍然存在一定的局限性。


**数据分析和处理**：是指接收到的原始数据经过分析、处理后，生成业务相关的数据。数据分析包括特征提取、异常检测、模型训练等工作，主要目的是为业务决策提供依据。由于IoT终端普遍采集大量原始数据，并且不同设备产生的数据格式、单位不统一，因此数据分析工作一般需要结合多个不同设备的不同数据进行特征提取和关联分析，才能得出有意义的结果。


**数据展示**：是指终端设备生成的业务数据经过平台的分析、处理、展示等工作。平台会根据用户的权限和偏好，对数据进行排序、筛选、聚合、统计等操作，形成一定的业务视图。目前，通过现有平台进行数据展示还是比较繁琐的过程，因为平台还没有开发出专门针对IoT的可视化分析工具。


**消息推送**：是指终端设备的指令、事件、告警等数据，通过平台的消息推送功能，透明地向用户通知。消息推送系统需要考虑终端的接收容量、传输速度、网络状态等因素，保证消息的及时、准确地发送到用户的终端上。目前，很多IoT平台采用短信、微信、邮件等即时通讯消息推送服务，虽然目前效果还不错，但也存在一些问题。


**规则引擎**：是指IoT平台中的规则引擎负责对业务数据进行规则配置，以便对数据的流动和处理形成预设条件下的触发器。规则引擎有助于智能平台提升灵活性和弹性，增强平台的响应速度、容错能力和可靠性。当前，在智能城市、制造业、智慧农业等领域，都有着广泛的应用场景。
