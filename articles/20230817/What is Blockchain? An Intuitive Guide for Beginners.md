# 2. Why do we need a blockchain?
# 2.1 概念层面上
区块链作为一种新的分布式数据库技术，其概念可以说颠覆了过去单机数据库技术体系中的所有模型。在目前信息爆炸的时代，个人或企业的数据越来越多、数据价值也在逐渐增长，传统的单机数据库不足以存储海量的数据。然而，单个节点的数据仍然存在丢失风险、伪造风险等隐患。区块链技术通过加密的方式保证数据的安全，从根本上解决了数据共享的问题。

2009年，斯坦福大学教授林武耀等人提出了“区块链：一种新的分布式分类账技术”，使得区块链技术真正成为一个“重新定义互联网的基础设施”。通过区块链技术，用户可以直接将数字货币交易、商业合同签署等各种操作指令通过互联网发送到世界各地，实现集体记账、去中心化，确保信息真实可靠、公开透明。

2014年底，日本团队开发的以太坊（Ethereum）项目推出之后，才开始引起越来越多人的注意。以太坊由开源社区在Github上发布，是一种基于区块链技术的计算机虚拟环境，可以让用户编写智能合约来自动执行各项任务，构建去中心化应用。以太坊的技术创新使其获得了巨大的成功。截止到2018年4月，全球范围内超过三千万人正在使用以太坊平台构建去中心化应用。

2.2 应用层面上
区块链可以解决许多实际问题。对于企业来说，它可以提供一种高效、低成本地提供稳定、安全的数据记录方式；对于个人用户来说，它可以方便快捷地进行消费付款、票据交易、数字资产管理等活动；对于政府部门来说，它可以在公共决策中提供透明、可验证的机制，并降低监管成本；对于金融机构来说，它可以建立起加密货币交易所、数字清算系统，并且能够防范网络攻击和欺诈行为。总之，只要涉及到数据交换、记账、计算、溯源、认证等多个环节，都可能受益于区块链技术。

# 3. How does the blockchain work?
# 3.1 结构
区块链由两大主要部分组成——分布式账本和共识机制。分布式账本记录所有的交易记录，每个记录被打包在一起形成一个区块，整个链条称之为区块链。共识机制负责验证交易顺序的正确性，并最终确定交易结果。


如上图所示，一条区块链可以由很多相互连接的区块组成，一条链上的区块是按照时间先后顺序串联起来的。每一个区块里面都保存着前一个区块的Hash值、当前区块里面的交易数据和Merkle树（用作快速定位某个交易的位置）。

区块链的交易过程通常由一个用户发起，他首先把交易请求发送给其他节点，然后其他节点接收到交易请求后会验证并广播到整个网络。其他节点收到交易请求后，会验证请求的内容是否符合要求，比如余额是否充足、支付方式是否有效等，如果验证通过则生成一笔交易，并将交易写入区块，等待下一个区块的广播。当最后一个区块产生的时候，交易即完成。

# 4. Terminology
# 4.1 Blockchain
区块链，是分布式数据库系统，它利用加密算法保证了数据信息的安全性、不可篡改性、全球透明、去中心化等特点。一条链上的区块之间通过哈希函数链接起来。一条链的运行依赖于共识协议。每个区块包括一个指向前一个区块的指针，这样一个链就像一个链表一样，最新的数据总是在头部，历史数据处于底部。

# 4.2 Mining
在比特币等加密货币系统中，矿工（Miners）通过竞争方式解决复杂的计算问题来获得比特币奖励。在区块链系统中，矿工被用来打包交易数据。每个节点都可以作为矿工，但是只有一个节点拥有独占的权利，即拥有生成新区块的资格。一个区块的产生需要消耗大量的计算资源，因此具有很强的无偏性，使得网络上产生了不同节点的区块，形成了一个高度分散的区块链。这种特性又带来了一个新的问题，那就是难以确定网络的正确运行状态。为了解决这一问题，各个节点之间需要达成一致意见，即“共识”。共识协议需要保证网络中只有合法的区块才能生效。

# 4.3 Transactions and blocks
区块链系统中，“交易”表示用户从一个账户向另一个账户转移数字资产的动作。“区块”是交易集合，构成一条链，其中第一个区块（Genesis Block）之前没有指向前一个区块的指针。

# 4.4 Public key cryptography
公钥密码学是一种非对称加密算法，允许用户对消息进行加密和签名，同时只能由持有私钥的实体解密。区块链系统中使用的公钥密码学算法通常采用椭圆曲线密码学（Elliptic Curve Cryptography，ECC），可提供高性能的加密运算。

# 4.5 Consensus protocol
共识协议用于确保区块链网络的整体运行平衡。不同的共识协议可以用来保证网络的可靠性、安全性、透明性和容错性。目前主流的共识协议有POW、POS和DPoS等。其中，POW（Proof of Work）、也就是工作量证明，是最古老的共识协议。它是指矿工需要花费大量的电力、空间和算力才能验证一条区块。POW可以避免一些错误交易被加入到区块中，但因为需要大量的硬件投入，导致挖矿难度一直在逐步提升。

POS（Proof of Stake）和DPoS（Delegated Proof of Stake）是基于委托制的共识协议。在POS中，持有币龄较大的用户（委托人）可以获得更多的权利，比如说获得新的区块奖励或者产生更多的交易。在DPoS中，可以把控制权交给一些小型的非盈利组织，比如说银行、交易所等，来共同维护网络的运行。DPoS能够减少网络拥堵、促进去中心化，但同时也会增加信任成本。

# 4.6 Smart contracts
智能合约（Smart Contracts）是指由区块链技术驱动的去中心化应用程序。它是一段部署在区块链上的程序，当其中的一条交易被广播到网络上时，合约就会被执行，并根据合约的逻辑对区块链上的数据做出一系列的修改。目前，最常用的智能合约语言有Solidity和Vyper。

# 5. Applications
# 5.1 Cryptocurrency transactions
区块链技术已经改变了现有的金融服务模式。在区块链网络中，用户可以利用加密货币进行无缝支付、结算以及交易，而且不需要第三方担保，完全依赖于网络的安全性。这极大的降低了交易成本、促进了交易的透明度和效率。例如，在比特币发行之前，人们通过中心化的交易所进行交易。在比特币流通市场里，各个交易所进行的交易和结算都相互影响。而在区块链系统里，所有的交易都是透明的，用户可以自行选择最佳的交易平台，也可以自己搭建属于自己的交易所。

# 5.2 IoT devices authentication
区块链可以用来验证物联网设备的身份，有效地保护物联网设备免受恶意攻击、隐私泄露等威胁。物联网设备通过与区块链系统的通信建立信任关系，可以获取到设备的所有操作记录。同时，设备的生产厂商也可以通过区块链系统发布产品认证信息，方便消费者核实。

# 5.3 Record keeping in supply chain management
供应链金融（Supply Chain Finance）旨在为零售企业和制造企业提供一种新型的金融服务。在供应链金融中，企业通过共享交易记录来协调各个环节之间的关系，来提高整个供应链的效率、成本和利润。区块链可以提供一套完整的交易记录系统，记录每一笔商品在各个环节的流动情况，确保产品流通的可追溯、可验证，促进经济的流通和繁荣。

# 5.4 Healthcare data sharing
区块链技术已经为医疗行业提供了一种全新的方式来处理医疗数据。在传统医疗服务中，数据主要存储在个人或医院的数据中心中，数据共享往往只能局限于内部人员，而无法达到患者的需求。而在区块链系统中，每个人的医疗数据都会存放在区块链上，任何人都可以访问这些数据，并且可以自由地共享、复制、分析。通过这种方式，区块链技术可以大幅降低数据收集、存储和管理的成本，提升数据的透明度、效率和价值。

# 6. Future trends
区块链技术已经成为各个领域的热点话题。区块链的基本思想是利用分布式网络技术来建立一个去中心化的、不可篡改的数据结构，使得数据可以被安全、可靠地传递、存储、共识、验证、确认。随着区块链的应用范围越来越广泛，它的发展方向将会不断变得更加深远。

未来区块链将会加速医疗、智慧城市、电子游戏、贸易、金融、供应链金融、保险、房地产、人工智能、农业、工业和制造等领域的发展。相信未来区块链将成为“革命性技术”，其应用将会遍及各个行业和岗位。
