# 1.简介
  

Cloudera是一个集数据处理、分析、存储于一体的大数据平台。它在Hadoop、Spark、Impala、Kudu等众多开源技术上进行了深度整合，提供了完整的大数据解决方案，包括Hadoop生态系统（HDFS、YARN）、Spark实时计算框架、Hive数据仓库、Sqoop、Flume、Kafka、Zookeeper、Kudu分布式列存数据库、Hue交互式SQL工具等，帮助企业实现大数据处理及分析的敏捷部署和快速迭代。Cloudera自2013年成立以来，已经成功地应用在多个行业，包括金融、电信、制造、零售、电子商务等领域。


近几年来，Cloudera公司的产品经理们逐渐开始关注大数据时代的新机会。基于对Apache Hadoop的不断深入理解和对Cloudera所提供的技术能力的深入考察，他们相继推出了CDH、Spark On Yarn等系列产品，这些产品除了技术上的革新之外，也兼顾功能和价值观上的突破。今天，我将着重介绍Cloudera CDH与CDP的优点，并阐述它们的不同之处。
# 2.基本概念术语说明
## 2.1 Cloudera Manager
Cloudera Manager是Cloudera大数据平台的管理中心，用于管理整个大数据平台，包括底层的Hadoop、Spark、Kudu集群、Kafka消息队列集群、数据仓库集群、Cloudera Navigator数据可视化工具、Sqoop、Flume、Hue等。用户可以通过浏览器访问CM界面或者通过命令行接口操作CM，CM具有以下功能：

1. 配置管理：通过图形化界面或API配置各种服务的安装路径、端口号、运行模式、性能参数、故障转移策略等；
2. 监控管理：提供丰富的监控指标、报警规则、告警信息，能够及时发现和定位集群中的异常情况；
3. 备份恢复：支持对各个组件的数据进行完整的备份，并可以从备份中还原数据；
4. 服务控制：允许用户方便快捷地启动/停止/重启服务，并且可以设置依赖关系，确保集群中所有服务都正常运行；
5. 用户权限管理：提供基于角色的授权机制，可以精细地控制用户对集群的访问权限；
6. 升级补丁管理：通过统一的升级发布机制，可以及时获取最新版本的补丁并自动应用到集群中。

## 2.2 Cloudera Director
Cloudera Director是一种新的基于云的分布式大数据部署、运维平台。它能够更轻松地完成大规模集群的部署、扩容、迁移等操作，降低运维成本，提升效率。Cloudera Director提供统一的管理控制台，使管理员可以在同一个视图下查看集群、节点、工作负载等资源的状态，并能够一键部署、扩展、回滚应用程序。管理员还可以使用Director API、命令行接口或Web UI管理集群、计划任务和事件通知等。


## 2.3 CDH(Cloudera Distributed Hadoop)
Cloudera Distribution for Hadoop，简称CDH，是基于Apache Hadoop，针对大数据场景做了高度优化。它能够同时处理海量数据、高吞吐量和复杂查询需求，且提供全面的安全性、可用性、可伸缩性和可靠性保证。它的主要特征如下：

1. 大数据计算：通过并行计算、HDFS和MapReduce等基础技术实现分布式的数据处理，并利用Cloudera Manager进行管理；
2. 数据分析：通过Spark SQL和Impala等引擎实现大数据的实时分析，并结合Cloudera Navigator进行数据可视化；
3. 消息传递：通过Kafka、Flume、Scribe等消息队列实现分布式日志收集和传输；
4. 分布式文件系统：Cloudera Manager 提供统一的界面，管理HDFS、S3和ADLSv2等多种分布式文件系统；
5. 可用性：通过HDFS、MapReduce和YARN等基础设施实现高可用性和可靠性；
6. 安全性：支持Kerberos认证、SSL加密、各种访问控制机制；
7. 监控管理：使用Cloudera Manager提供的监控模块对集群实时状态进行监测，并通过邮件、短信、Webhook、页面和其他方式发送告警；
8. 弹性伸缩：通过简单而直观的配置选项，实现实时的集群水平动态扩展。



## 2.4 CDP(Cloudera Data Platform)
CDP是一个基于Cloudera Manager的大数据集群，它提供了高级的安全性、易用性和可伸缩性，支持最新的大数据技术和云计算环境。它可以像CDH一样运行在自己的私有云上，也可以在公共云上部署运行。它的主要特点包括：

1. 弹性集群：CDP可以根据用户的需要快速创建、调整和删除集群；
2. 数据加密：CDP能够使用高级的加密技术，帮助用户在分布式环境中安全地存储、处理和传输数据；
3. 自动备份：CDP提供完整的备份和恢复功能，让用户能够轻松地从集群中恢复数据；
4. 高可用性：CDP能够通过多区域部署，确保集群的高可用性和可靠性；
5. 数据湖：CDP支持大数据生态系统中的数据湖，并为客户提供一站式的端到端数据湖解决方案；
6. 操作界面：CDP采用图形化界面，让用户更容易理解集群的运行状况；
7. 技术支持：CDP提供专业的技术支持，包括定制化咨询、培训和文档，以及免费的优先支持选项。
