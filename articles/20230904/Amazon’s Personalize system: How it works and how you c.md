# 2.基本概念、术语及其含义

## 2.1 Personalize 是什么？

Personalize是一个基于机器学习的个性化推荐引擎，由亚马逊Web Services提供。它通过分析用户行为数据并进行学习，可以根据用户兴趣、偏好、品牌偏好等信息，向用户提供符合个性化需求的商品推荐。

Personalize采用了协同过滤、无监督学习、因子分解机等机器学习算法，结合了聚类分析、关联规则挖掘、决策树等技术，能够快速生成个性化的推荐结果。

## 2.2 个性化推荐系统

推荐系统的目标是在不给用户任何明确反馈的情况下，为用户推荐他们可能喜欢的商品或者服务。目前市面上，个人化推荐系统的研究已经相当成熟，主要包括协同过滤（Collaborative Filtering）、基于内容的推荐系统（Content-Based Recommendation System）、基于模型的协同过滤（Model-based Collaborative Filtering）等。

### 2.2.1 协同过滤

协同过滤是指对物品之间的交互关系进行分析，将热门物品推荐给用户，这种推荐方式不考虑物品特征，只根据用户的历史交互记录，因此被称为“用户基础模型”。

基本思路是找到那些与目标用户之前观察到的物品相似度最大的那些用户，然后推荐那些最热门的、与这些相似用户喜欢的物品。简单来说，就是根据历史交互数据，预测目标用户可能感兴趣的物品，再把这些物品推荐给该用户。

### 2.2.2 基于内容的推荐系统

基于内容的推荐系统认为不同物品之间存在某种相关性，可以基于物品的内容来进行推荐。与协同过滤不同的是，基于内容的推荐系统还会考虑物品的特性，例如品牌、价格、描述、风格、图像等。

### 2.2.3 基于模型的协同过滤

基于模型的协同过滤融合了协同过滤和概率估计的思想，即基于用户和物品的行为序列进行建模，然后利用这些序列来预测用户对每个物品的评分。该模型建立在概率图模型之上，使用矩阵分解的方法求解。

## 2.3 Personalize 的术语及其含义

### 2.3.1 Dataset

Dataset，顾名思义，数据集。顾名思义，就是用来训练模型的数据集。在Personalize里，这个数据集包含两部分，第一部分是用户的行为数据，第二部分是物品的特征数据。

### 2.3.2 Item

Item，顾名思义，物品。在Personalize里，一个item指的是个性化推荐的对象，比如电影、音乐、App等。

### 2.3.3 User

User，顾名思义，用户。在Personalize里，一个user指的是个性化推荐的受众，比如网民、手机app用户等。

### 2.3.4 Event Type

Event Type，事件类型。顾名思义，事件类型是指用户的各种行为，比如点击、收藏、加入购物车等。在Personalize里，事件类型用数字表示，比如0表示点击、1表示加购物车等。

### 2.3.5 Contextual metadata

Contextual metadata，上下文元数据。顾名思义，上下文元数据是指关于用户的非正式特征，如用户的年龄、职业、居住地等。上下文元数据一般通过日志、网络上下文数据或其他渠道获取。

### 2.3.6 Interaction dataset

Interaction dataset，交互数据集。顾名思义，交互数据集就是指用户与物品之间发生的一系列事件。交互数据集包含事件类型、物品ID、用户ID和时间戳等信息。

### 2.3.7 Implicit feedback

Implicit feedback，暗示反馈。顾名思义，暗示反馈是指某个物品的点击、购买等行为只能通过其他行为产生，不能直接反映出用户的真实意愿。

### 2.3.8 Explicit feedback

Explicit feedback，显式反馈。顾名思义，显式反馈是指用户表达了明确的意愿，比如喜欢或不喜欢某个物品。

### 2.3.9 Personalization strategy

Personalization strategy，个性化策略。顾名思义，个性化策略是指用于推荐的算法，可以分为基于内容的推荐系统、基于协同过滤的推荐系统、基于深度学习的推荐系统等。

### 2.3.10 Personalized ranking

Personalized ranking，个性化排序。顾名思义，个性化排序是指根据用户的历史交互数据进行推荐后得到的推荐列表。
