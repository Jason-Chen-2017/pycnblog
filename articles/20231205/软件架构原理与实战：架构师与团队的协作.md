# 2.核心概念与联系

## 2.1 软件架构的核心概念

软件架构的核心概念包括：

- 组件：软件系统的基本构建块，可以是代码、数据、服务等。
- 关系：组件之间的联系，可以是依赖关系、组合关系等。
- 约束：组件和关系之间的约束，可以是性能约束、安全约束等。

## 2.2 软件架构与软件设计的联系

软件架构与软件设计是两个不同的概念，但它们之间存在密切联系。软件设计是软件开发过程中的一个环节，它涉及到软件的具体实现，包括算法、数据结构、接口等。而软件架构是软件设计的一个子集，它关注于软件系统的全局结构和组件之间的关系。因此，软件设计是软件架构的具体实现，而软件架构是软件设计的一个全局视角。
