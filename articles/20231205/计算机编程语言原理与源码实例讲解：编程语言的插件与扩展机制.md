# 2.核心概念与联系

在计算机编程语言中，插件与扩展机制是指在不修改原有语言源代码的情况下，为编程语言添加新功能的方法。这种方法通常包括两种：一种是通过插件（plugin）的方式，将新功能作为外部模块加载到运行时环境中；另一种是通过扩展（extension）的方式，将新功能作为内部模块添加到编程语言的核心功能中。

插件与扩展机制的核心概念包括：

- 插件（plugin）：插件是一种可以独立运行的软件模块，它可以在运行时动态加载到程序中，从而扩展程序的功能。插件通常包含一定的代码和数据，可以与其他插件或程序进行交互。

- 扩展（extension）：扩展是一种内部模块，它可以在编程语言的核心功能中添加新的功能。扩展通常是通过修改编程语言的源代码实现的，从而实现对编程语言的拓展。

插件与扩展机制的联系在于，它们都是为了实现编程语言的拓展和定制而设计的。插件通常更加灵活，因为它可以在运行时动态加载，而扩展则需要在编译时进行修改。
