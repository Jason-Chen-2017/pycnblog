# 2.核心概念与联系

模块化设计是一种软件设计方法，它将软件系统划分为多个模块，每个模块都负责完成特定的功能。模块化设计的核心概念包括模块、接口、抽象、封装、继承和多态等。这些概念在软件设计中起着关键作用，使得软件系统更加易于理解、维护和扩展。

模块化设计与其他软件设计方法，如面向对象设计、函数式设计等，存在密切联系。例如，面向对象设计中的类和对象可以被视为模块化设计中的模块和接口的特例。函数式设计则强调函数的模块化，将软件系统划分为多个函数，每个函数负责完成特定的功能。

模块划分是模块化设计的第一步，它涉及将软件系统划分为多个模块，每个模块负责完成特定的功能。模块划分的关键是确定模块的粒度，即模块之间的耦合度和内聚度。模块的耦合度是指模块之间的相互依赖程度，而内聚度是指模块内部功能的完整性。
