# 2.核心概念与联系

## 2.1 Docker容器化

Docker容器化是一种将软件应用程序与其所需的依赖项、库、环境等一起打包成一个独立的容器的方法。这使得软件可以在任何支持Docker的平台上运行，无需关心底层的操作系统和硬件环境。

## 2.2 Docker镜像

Docker镜像是一个只读的文件系统，包含了软件应用程序及其依赖项的所有文件。镜像可以被复制和分发，也可以被用来创建Docker容器。

## 2.3 Docker容器

Docker容器是一个运行中的Docker镜像实例。容器可以运行、停止、删除等操作，并且可以与其他容器进行通信。

## 2.4 Docker仓库

Docker仓库是一个存储Docker镜像的服务。Docker Hub是一个公共的Docker仓库服务，也是Docker社区的核心基础设施之一。

# 4.具体代码实例和详细解释说明

在本节中，我们将通过一个具体的代码实例来详细解释Docker容器化的实践。

## 4.1 创建Docker镜像

首先，我们需要创建一个Docker镜像。我们可以使用Dockerfile文件来定义软件应用程序及其依赖项。以下是一个简单的Dockerfile示例：

```
FROM ubuntu:18.04

RUN apt-get update && \
    apt-get install -y python3

COPY app.py /app.py

CMD ["python3", "/app.py"]
```

在这个Dockerfile中，我们使用了Ubuntu 18.04作为基础镜像，并安装了Python 3。然后，我们将一个名为`app.py`的Python脚本复制到容器内的`/app.py`路径，并指定了运行容器时使用的命令。

接下来，我们可以使用`docker build`命令来构建Docker镜像：

```
docker build -t my-app .
```

这个命令将在当前目录下构建一个名为`my-app`的Docker镜像。

## 4.2 推送Docker镜像到仓库

接下来，我们需要将创建的镜像文件推送到Docker仓库。首先，我们需要登录到Docker Hub：

```
docker login
```

然后，我们可以使用`docker push`命令将镜像文件推送到仓库：

```
docker push my-app
```

## 4.3 从仓库拉取Docker镜像

在需要运行软件的机器上，我们可以使用`docker pull`命令从Docker仓库拉取相应的镜像文件：

```
docker pull my-app
```

## 4.4 创建并运行Docker容器

最后，我们可以使用`docker run`命令创建并运行Docker容器：

```
docker run -d --name my-container my-app
```

这个命令将创建一个名为`my-container`的容器，并运行`my-app`镜像。
