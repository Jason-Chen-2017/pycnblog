# 2.核心概念与联系

在框架设计中，有几个核心概念需要理解：

1. **框架**：框架是一种软件设计模式，它提供了一种结构化的方法来组织和实现软件系统。框架通常包含一个基本的架构和一组预先定义的接口，开发人员可以根据需要扩展和修改这些接口来实现特定的功能。

2. **组件**：框架由一组可组合的组件组成。这些组件可以独立地实现特定的功能，并可以通过预定义的接口来组合和扩展。组件是框架的基本构建块，它们可以根据需要被替换、扩展或修改。

3. **插件**：插件是框架中的可扩展模块，它们可以在运行时动态加载和卸载。插件通常实现特定的功能，并可以通过框架提供的接口来集成到框架中。插件提供了一种可扩展的方法来实现特定的功能，而无需修改框架本身。

4. **依赖注入**：依赖注入是一种设计模式，它允许组件在运行时接收其他组件的实例。这意味着组件不需要知道它们所依赖的组件的具体实现，而是通过接口来访问它们。这使得组件更加可扩展和可维护，因为它们可以在运行时动态地更改其依赖关系。

```python
    def __init__(self):
        self.history = []

    def add(self, a, b):
        return a + b

    def subtract(self, a, b):
        return a - b

    def multiply(self, a, b):
        return a * b

    def divide(self, a, b):
        return a / b

    def add_history(self, a, b, result):
        self.history.append((a, b, result))

    def get_history(self):
        return self.history
```
