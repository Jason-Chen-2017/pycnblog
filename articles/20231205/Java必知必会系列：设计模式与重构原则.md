## 2.1 设计模式

设计模式是一种解决特定问题的解决方案，它们是解决问题的经验和最佳实践。设计模式可以帮助我们更好地组织代码，提高代码的可读性、可维护性和可扩展性。设计模式可以分为三类：创建型模式、结构型模式和行为型模式。

### 2.1.1 创建型模式

创建型模式主要解决对象创建的问题。它们提供了一种创建对象的方式，使得对象的创建过程可以独立于其他代码中。常见的创建型模式有：单例模式、工厂方法模式、抽象工厂模式、建造者模式和原型模式。

### 2.1.2 结构型模式

结构型模式主要解决类和对象的组合方式的问题。它们描述了如何将类和对象组合成更大的结构，以便更好地组织代码。常见的结构型模式有：适配器模式、桥接模式、组合模式、装饰模式和外观模式。

### 2.1.3 行为型模式

行为型模式主要解决对象之间的交互方式的问题。它们描述了如何在对象之间定义更好的沟通方式，以便更好地组织代码。常见的行为型模式有：策略模式、命令模式、观察者模式、责任链模式和状态模式。

## 4.1 设计模式的具体代码实例

我们以单例模式为例，来详细解释设计模式的使用方法。

单例模式的核心思想是确保一个类只有一个实例，并提供一个全局访问点。单例模式主要解决了对象创建和对象之间的关联问题。

单例模式的实现方式有两种：懒汉模式和饿汉模式。

### 4.1.1 懒汉模式

懒汉模式是在需要使用单例对象时才创建单例对象的实现方式。懒汉模式的实现方式有两种：线程安全的懒汉模式和线程不安全的懒汉模式。

#### 4.1.1.1 线程安全的懒汉模式

```java
public class Singleton {
    private static Singleton instance;

    private Singleton() {
    }

    public static synchronized Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

#### 4.1.1.2 线程不安全的懒汉模式

```java
public class Singleton {
    private static Singleton instance;

    private Singleton() {
    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

### 4.1.2 饿汉模式

饿汉模式是在类加载时就创建单例对象的实现方式。饿汉模式的实现方式有两种：线程安全的饿汉模式和线程不安全的饿汉模式。

#### 4.1.2.1 线程安全的饿汉模式

```java
public class Singleton {
    private static final Singleton instance = new Singleton();

    private Singleton() {
    }

    public static Singleton getInstance() {
        return instance;
    }
}
```

#### 4.1.2.2 线程不安全的饿汉模式

```java
public class Singleton {
    private static Singleton instance;

    private Singleton() {
    }

    public static Singleton getInstance() {
        return instance;
    }
}
```
