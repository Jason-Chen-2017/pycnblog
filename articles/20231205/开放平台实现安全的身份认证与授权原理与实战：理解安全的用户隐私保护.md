# 2.核心概念与联系

在讨论身份认证与授权之前，我们需要了解一些核心概念。

## 2.1 身份认证

身份认证是一种验证过程，用于确认一个实体（例如用户或设备）是否是所声称的实体。通常，身份认证涉及到两个方面：

1. 用户身份验证：用户需要提供有效的凭据（如密码、身份证或驱动证）以证明他们是谁。
2. 系统身份验证：系统需要确保用户的身份信息是准确的，以防止欺骗和身份盗用。

## 2.2 授权

授权是一种访问控制机制，用于确定用户是否具有访问某个资源的权限。授权涉及以下几个方面：

1. 用户权限：用户具有的权限可以决定他们是否可以访问某个资源。
2. 资源类型：资源类型可以是文件、目录、数据库等。
3. 访问权限：用户可以具有读、写、执行等不同类型的访问权限。

## 2.3 身份认证与授权的联系

身份认证与授权之间存在密切的联系。身份认证确保了用户是谁，而授权确保了用户具有访问资源的权限。在实际应用中，身份认证和授权通常是相互依赖的，它们共同确保了用户的身份和权限。
