# 2.核心概念术语说明
## 2.1 数据采集
数据采集是指从各个渠道（如互联网、文件、系统等）收集数据，经过初步的清洗和整理，然后存入数据库或文件系统等形式。其中包括以下三个主要阶段：

1. 爬虫（Crawler）：爬虫通常是指自动化程序，它会按照一定规则获取网站上的信息，并把它们抓取下来保存到本地。常见的网络爬虫有BeautifulSoup、Scrapy等，也可以采用开源的商业爬虫框架，如ScrapingHub、DataLad。

2. API接口（API Interface）：API接口即Application Programming Interface，它提供了各种数据服务的接口，使得应用可以调用相关服务获取数据。目前比较流行的金融数据API接口有Quandl、IEXCloud等。

3. 数据合成（Data Synthesis）：数据合成是指利用多个数据源构建一个统一的数据集。如利用各类数据源进行的预测模型训练、跨市场分析的股票期权组合，都会涉及到数据合成。

## 2.2 数据清洗
数据清洗又称数据预处理、数据转换，是指对原始数据进行初步的处理，将其转换成适合后续分析的数据结构。主要包括以下五个主要步骤：

1. 数据规范化（Data Normalization）：数据规范化是指将数据字段标准化，以便于数据之间的比较、计算等。

2. 数据类型识别（Data Type Detection）：数据类型识别是指根据数据的模式识别其所属数据类型，如数值型、字符串型、日期型等。

3. 数据缺失值填充（Missing Data Imputation）：数据缺失值填充是指通过某种方式填补缺失的数据。常用的方式有平均值、中位数、众数等。

4. 异常值检测（Anomaly Detection）：异常值检测是指发现数据中的异常值，并将其剔除掉。常用的异常检测方法有Z-score法、Isolation Forest算法等。

5. 特征抽取（Feature Extraction）：特征抽取是指从原始数据中提取出有意义的、可用于分析的数据，如指标、交易策略等。

## 2.3 数据仓库（Data Warehouse）
数据仓库是一个中心化的存储库，用于汇总企业各个系统的关系型数据。数据仓库存储数据的目的在于支持企业的信息分析、决策支持和风险控制等业务需求。它是企业内部数据资产的一个集合体，由各种来源的数据经过数据集成、清洗、转化和加工，得到易于查询的数据集。数据仓库的建设包括建模、规范化、ETL（Extract Transform Load）、数据质量管理、报表生成和数据仪表盘等环节。

## 2.4 数据分析
数据分析是指从数据中发现有价值的模式、规律、关联性和趋势，并通过统计、数学模型、机器学习等方法进行预测和推断。数据分析方法包括Descriptive Statistics、Inferential Statistics、Predictive Analytics、Time Series Analysis、Text Mining、Image Recognition等。

## 2.5 数据挖掘
数据挖掘是指利用大数据技术（如Spark、Hadoop）、统计学、机器学习等技术从海量数据中发现有价值的信息，并据此做出决策、调整策略、改善产品等。数据挖掘方法包括Clustering、Classification、Association Rule Learning、Regression、Density Estimation、Dimensionality Reduction、Graph Analysis、Pattern Recognition等。
