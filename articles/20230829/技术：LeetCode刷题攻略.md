# 3.基本概念术语说明
## 3.1 LeetCode题目分类
LeetCode共分为两大类：

 - 一类是经典难度问题：囊括了面试中出现频率最高的算法题；
 - 一类是入门难度问题：由简单到困难，适合刚接触算法但想进阶的小伙伴们练习。

除此之外，每道题还会提供两个额外标签，分别表示其难易程度和已有知识点依赖。

## 3.2 LeetCode图解题型
LeetCode采用二叉树图的形式展示题目的结构。其中，每个结点代表一种操作类型，如数组的遍历、链表的操作，并且每种操作类型的结点又可以有子结点构成不同的子树。

图中的箭头线条代表操作类型之间的关系。如题目要求在一组数字中找到两个最大的数字，则可以看作是从小到大的排序算法，题目中的遍历操作对应于对数字进行扫描，而题目中的比较操作则对应于判断数字大小。

## 3.3 LeetCode题目状态
LeetCode的题目分为三种状态：

 1.待解决：表示该题仍处于待解决状态，可能有多个算法模型可供选择；
 2.回答错误：表示提交的代码存在语法或逻辑错误；
 3.已AC：表示提交的代码完全正确。

## 3.4 LeetCode刷题建议
### 3.4.1 先刷标签集中的“简单”问题
首先，要保证自己对这些“简单”问题的掌握程度达到一定水平。一般来说，不会太复杂的算法题会比较容易理解和刷，因此应该优先着重刷“简单”的问题。

### 3.4.2 每天选择一道以上难度相似的题目
第二，每天选择几道以上难度相近的题目，这样可以让思路更加统一，并且比单刷某一难度下的所有题要有效得多。

### 3.4.3 多使用边界条件和特殊值测试
第三，多使用边界条件和特殊值测试，这是一件很重要的事情。比如，给定一个数组，如果数组为空，需要返回什么样的值？数组只有一个元素的时候呢？还是数组中没有重复的元素？这些都是需要考虑的边界情况。

### 3.4.4 使用模板快速写出代码
第四，使用模板快速写出代码，比如可以使用栈，队列或者哈希表等数据结构，减少时间复杂度，或者使用递归函数等方法，减少空间复杂度。

### 3.4.5 使用Lint工具检查代码规范
第五，使用Lint工具检查代码规范，确保代码风格一致性和错误率低。目前，很多编辑器都支持Lint功能，如VSCode、Sublime Text、Atom等。
