# 1.简介
  

数据平台（Data Platform）是一个很热门的话题，随着互联网、云计算等技术的不断发展，数据平台也逐渐成为一个被重视的主题。在互联网+、移动互联网时代，企业级数据平台的需求越来越强烈，越来越多的公司开始考虑如何构建适合自己的数据平台。本文将详细介绍企业级数据平台建设的几个重要因素。
# 2.企业级数据平台的特点
首先，企业级数据平台一定要面向企业用户，而非终端用户。企业级数据平台应当与内部管理系统、业务运营系统相结合，通过可视化的方式呈现数据，并实现数据共享、协同分析、决策支持等功能。其次，企业级数据平台必须能够处理海量的数据，因此，需要高效的数据处理能力和处理速度。最后，企业级数据平台还要具备完善的安全性保障机制，防止数据的泄露、篡改、恶意攻击等风险。
# 3.企业级数据平台的构成
企业级数据平台由如下几个部分组成：
- 数据采集：负责收集原始数据，对数据进行清洗、转换等操作，形成可用于分析的结构化数据。比如，可以利用ETL工具将多个不同数据源的数据汇总到一个数据仓库中。
- 数据存储：负责将经过处理后的结构化数据保存到数据库或文件系统中。比如，可以使用HDFS分布式文件系统保存数据。
- 数据分析：负责根据业务规则、数据挖掘方法等分析已有的结构化数据，得出有价值的信息。比如，可以使用Hive查询语言做复杂的SQL查询、使用Spark机器学习框架进行复杂的数据分析。
- 数据展示：通过可视化的方式呈现数据，并与内部管理系统、业务运营系统相结合。比如，可以使用Tableau、Davinci标注引擎等工具进行数据可视化。
- 数据应用：负责基于分析结果，提供信息服务。比如，可以基于分析结果设计报表、生成BI产品。
以上五大部分，作为企业级数据平台建设的基础，共同构建了企业级数据平台的整体架构。
# 4.数据采集方式
数据采集包括两种模式：“拉”模式和“推”模式。“拉”模式的主要特点是，在数据源发生变化时，通过定时任务定期地从数据源上抓取最新的数据；“推”模式则是由数据源主动发送更新的数据，然后由接收端处理。目前比较流行的是“推”模式，它更加高效，可靠。
# 5.数据存储方式
数据存储主要有以下几种方案：
- 文件系统：采用关系型数据库或NoSQL数据库无法处理海量数据时，可以使用文件系统存储，如HDFS、OBS等。
- 分布式数据库：分布式数据库通常比传统关系型数据库具有更好的性能，且可以水平扩展，适合存储海量数据。例如，Teradata、Greenplum、Redshift等。
- NoSQL数据库：NoSQL数据库不需要固定的模式，灵活地存储数据，具有较好的弹性扩展性，适合处理大量数据。例如，HBase、MongoDB等。
# 6.数据计算框架
数据计算框架主要分为离线计算和实时计算。
- 离线计算：离线计算一般采用MapReduce模型，它会把所有的数据加载到内存中并进行处理，然后输出结果。例如，Hive、Pig。
- 实时计算：实时计算则会实时地处理数据流，可以在秒级响应用户请求，例如Storm、Spark Streaming等。
# 7.数据安全机制
数据安全机制应该具备以下四个属性：
- 认证：确保只有受信任的实体才能访问数据。
- 授权：控制实体对数据的访问权限。
- 加密：保护数据免受恶意攻击。
- 漏洞管理：发现、修复漏洞。
# 8.数据治理机制
数据治理机制应当遵循GB/T 3108—2017标准，共计十二条，其中包括：
- 制定数据生命周期策略：定义数据从收集到存档、过期删除的整个过程。
- 管理个人信息：维护相关人员信息、机密信息和隐私政策。
- 建立数据分类制度：构建数据分类和数据质量体系。
- 建立数据共享制度：落实数据共享规则及流程。
- 制定数据使用条件：明确数据使用者的权利和义务。
- 定期评估数据准确性：持续跟踪数据质量，提升数据质量水平。
- 提供数据反馈渠道：构建数据使用反馈机制，促进数据使用质量改善。
- 建立数据质量监控体系：建立健全数据质量监测体系，评估数据质量水平。
- 建立信息服务管理机制：建立完善的信息服务管理体系，包括资讯服务、培训服务等。
- 落实数据审核制度：制订严格的数据审核制度，有效防范信息泄露。
