# 1.简介
  

迭代算法是一个非常重要的算法分类。算法一般分为两大类：贪心算法和动态规划算法，而在数据量较大、计算复杂度高的情况下，一般采用动态规划算法。那么什么时候应该使用贪心算法呢？或者说何时用动态规划算法更好一些呢？这个问题引出了我们要探讨的内容。
# 2.贪心算法
贪心算法又称为“贪婪算法”，是一种在每一步选择中都采取最佳方案，从而希望在全局最优解上收敛的算法。基本思想就是当下之最大（或最优）者，也就是在某种意义上是对全局最优的近似解。例如在旅行商问题中，每次选择路径中距离最短的路线作为下一个目的地。这样的方法虽然不是最优的，但是比起动态规划算法得到的结果通常会更快、更省时间。
对于这种情况，贪心算法简单有效，而且效率也不错。但是缺点也是明显的，比如在一些特殊问题中，往往找不到全局最优解，所以贪心算法并不能保证求得全局最优解。因此，需要结合其他方法进行分析和改进。
# 3.动态规划算法
动态规划（Dynamic Programming）是指将原问题分解成子问题，递归求解子问题，然后利用子问题的解来求原问题的解。动态规划以自底向上的方式解决复杂的问题。其主要的思想是通过把小问题的解保存起来，避免重复计算，从而减少运算的时间。动态规划解决很多背包问题、机器学习等关键性问题。它具有穷举所有可能状态的方法，但实际上是高效的。
对于比较规模化的问题，比如背包问题，动态规划是很好的解法。但是也存在一些局限性。首先，计算子问题的时间复杂度可能会比较高；其次，动态规划算法在处理多重子问题的时候容易陷入重复解的问题。因此，动态规loptimization问题应与贪心算法结合使用，来获得更优秀的解法。
# 4.总结
贪心算法适用于处理具有确定性解的优化问题，如图论问题，而动态规划算法适用于处理具有最优子结构的问题。两种算法可以互相结合，以更快速、更精确的找到全局最优解。当然，还有其他一些方法，如蒙特卡洛方法等等，值得我们去尝试。因此，当我们遇到一个比较复杂的问题，而没有太多的经验时，如何选择算法是一个值得考虑的问题。通过阅读相关文献，了解各种算法的特性、应用场景及局限性，可以帮助我们作出正确的决策。我们也可以从我们的视角理解为什么选择算法才是正确的，以及不同算法之间的利弊权衡，从而做出更加科学的判断。另外，基于此，我们还可以提出自己的迭代算法，或者集成多个算法来生成最终的结果。这无疑是一种创新。最后，祝大家工作顺利！
