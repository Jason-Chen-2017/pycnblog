# 2.核心概念与联系

## 2.1 模块

Go模块是Go程序的基本组成单元，它包含了Go程序的源代码、依赖关系信息和其他元数据。Go模块可以理解为一个Go程序的容器，它可以包含多个Go包。Go模块可以通过Go工具（如`go mod init`、`go mod edit`、`go mod tidy`等）来创建、编辑和管理。

## 2.2 包

Go包是Go模块中的一个组成单元，它包含了Go程序的源代码、变量、函数、类型、常量等。Go包可以理解为Go程序的模块化单元，它可以被其他Go程序引用和使用。Go包可以通过Go工具（如`go get`、`go build`、`go test`等）来下载、编译和测试。

## 2.3 依赖关系

Go模块和包之间存在依赖关系，这意味着一个Go模块或包可能需要其他Go模块或包来实现其功能。Go模块和包的依赖关系可以通过Go工具（如`go mod edit`、`go mod tidy`等）来管理。Go模块和包的依赖关系可以通过Go工具（如`go list`、`go mod why`等）来查询和验证。

## 2.4 版本控制

Go模块和包可以指定版本号，这意味着一个Go模块或包可以指定其他Go模块或包的版本号来实现其功能。Go模块和包的版本号可以通过Go工具（如`go mod edit`、`go mod tidy`等）来修改。Go模块和包的版本号可以通过Go工具（如`go list`、`go mod why`等）来查询和验证。
