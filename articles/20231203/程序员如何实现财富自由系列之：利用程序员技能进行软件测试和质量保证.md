# 2.核心概念与联系

在进行软件测试和质量保证之前，我们需要了解一些核心概念。这些概念包括：

- 软件测试：软件测试是一种验证软件是否满足需求和预期行为的过程。它旨在发现软件中的缺陷，并确保软件的质量。
- 软件质量：软件质量是指软件满足用户需求、可靠性、可维护性、性能等方面的程度。
- 测试策略：测试策略是一种用于指导测试活动的方法，它包括测试的目标、范围、方法、资源和时间等因素。
- 测试计划：测试计划是一种详细的测试策略，它包括测试的目标、范围、方法、资源和时间等因素。
- 测试用例：测试用例是一种用于测试软件的方法，它包括输入、预期输出、实际输出等信息。
- 测试用例库：测试用例库是一种存储和管理测试用例的方法，它包括各种不同类型的测试用例。
