TCP（Transmission Control Protocol，传输控制协议）是一种面向连接的、可靠的、因子的流量控制和拥塞控制的传输层协议。TCP 是应用层协议 TCP 上的应用程序（例如 HTTP、FTP、SMTP 等）使用的数据报文（数据包）。TCP 协议为数据报文提供了端到端的连接服务，确保数据报文的顺序、完整性和可靠性。

在互联网中，TCP 是最常用的传输层协议之一，它为应用程序提供了一种可靠的、面向连接的数据传输方式。TCP 协议为数据报文提供了端到端的连接服务，确保数据报文的顺序、完整性和可靠性。

# 2.核心概念与联系

在探讨 TCP 的流量控制和拥塞控制机制之前，我们需要了解一些核心概念。

## 2.1 数据报文（Packet）

数据报文是 TCP 协议中传输的基本单位，它包含了源地址、目的地址、协议类型和数据等信息。数据报文可以理解为一种封装了数据和元数据的数据包。

## 2.2 连接（Connection）

TCP 协议是一种面向连接的协议，这意味着在数据传输之前，需要建立一条连接。连接可以分为三个阶段：连接请求、连接确认和数据传输。

## 2.3 流量控制（Flow Control）

流量控制是一种保证接收方不被发送方淹没的机制，它通过接收方向发送方发送一个控制信息（窗口）来告知发送方接收能力。流量控制的目的是保证网络稳定性，避免因发送方发送速度过快导致接收方处理不过来而出现数据丢失。

## 2.4 拥塞控制（Congestion Control）

拥塞控制是一种保证网络不出现拥塞的机制，它通过观察网络状况和调整发送方发送速率来避免网络拥塞。拥塞控制的目的是保证网络稳定性，避免因网络拥塞导致数据包丢失和延迟增加。
