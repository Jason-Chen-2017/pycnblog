## 3.1 纠错输出码的基本概念
纠错输出码（FEC）是一种错误抑制编码技术，它可以在信道传输过程中抑制错误，提高信息传输的可靠性。纠错输出码的主要思想是在编码过程中，为原始信息添加一定的冗余信息，从而在解码过程中，即使信道出现错误，也可以通过冗余信息来纠正错误，从而实现信息的可靠传输。

## 3.2 纠错输出码的主要算法
在本节中，我们将介绍一种常见的纠错输出码算法——Hamming码的编码和解码过程。

### 3.2.1 Hamming码的编码过程
Hamming码是一种常见的纠错输出码，它的编码过程如下：

1. 首先，将原始信息按位分解，得到一个二进制序列。
2. 接着，为原始信息添加一定的冗余信息，以实现错误抑制。具体来说，可以在原始信息的某些位置添加冗余位，使得冗余位与原始信息的某些位置相同。
3. 最后，将原始信息和冗余信息组合在一起，得到最终的编码序列。

### 3.2.2 Hamming码的解码过程
Hamming码的解码过程如下：

1. 首先，将编码序列按位分解，得到原始信息和冗余信息。
2. 接着，通过检查冗余信息中的错误位，找出编码序列中的错误位。
3. 最后，将错误位修改为原始信息中的正确位，得到解码后的信息。

    # 添加冗余位
    n = len(bits)
    k = int(n / 2)
    p = int(np.ceil(np.log2(n)))
    r = p - k
    # 将信息和冗余位组合在一起
    encoded = np.zeros(n, dtype=int)
    encoded[:k] = bits
    encoded[k:] = bits[:r]
    return encoded

def hamming_decode(encoded):

    bits = np.array(list(encoded), dtype=int)
    # 找出错误位
    errors = np.zeros(len(bits), dtype=int)
    for i in range(len(bits)):
        if bits[i] != bits[i ^ (i & -i)]:
            errors[i] = 1
    # 修改错误位
    info = np.zeros(len(bits) - len(errors), dtype=int)
    for i, bit in enumerate(bits):
        if errors[i] == 0:
            info[i] = bit
    return info

print('解码后的信息:', decoded)
