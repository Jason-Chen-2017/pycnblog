                 

# 1.背景介绍

Docker 是一种开源的应用容器引擎，它可以用来打包应用及其依赖项，以特定的环境来运行。Docker 使用特定的镜像（Image）格式存储应用及其状态，以便在任何支持 Docker 的平台上运行。这种容器化的方法有助于提高应用的可移植性、可扩展性和可维护性。

Docker 的核心组成部分包括：

1. Docker 镜像（Image）：镜像是只读的并包含应用及其依赖项的模板。镜像不包含运行时环境。
2. Docker 容器（Container）：容器是从镜像创建的实例，包含运行时环境和应用。容器可以运行在任何支持 Docker 的平台上。
3. Docker 仓库（Repository）：仓库是一个存储镜像的集合，可以是公有的或私有的。
4. Docker 引擎（Engine）：引擎是 Docker 的核心组件，负责构建、运行和管理容器。
