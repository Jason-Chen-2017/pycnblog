# 2.核心概念与联系

## 2.1 实时流处理的核心概念

### 2.1.1 数据流

数据流是一种连续的数据序列，数据以高速流量流入系统，需要在流中实时地进行处理和分析。数据流可以来自于各种来源，例如 sensors、web logs、social media 等。数据流可以是结构化的，例如关系数据库表；也可以是非结构化的，例如文本、图像、音频、视频等。

### 2.1.2 窗口

窗口是对数据流的一个连续子集，可以根据不同的需求设置不同的窗口大小。窗口是实时流处理中一个重要的概念，它可以用于限制数据流中的处理范围，从而实现更高效的数据处理。

### 2.1.3 流处理模型

流处理模型是一种用于描述如何在数据流中实现数据处理和分析的模型。流处理模型可以分为两种：一种是基于事件的模型，另一种是基于流的模型。

1. 基于事件的模型：基于事件的模型将数据流中的每个数据点看作是一个事件，然后根据事件之间的关系来进行处理和分析。这种模型的优点是简单易用，缺点是无法处理数据流中的时间顺序关系。
2. 基于流的模型：基于流的模型将数据流看作是一个连续的数据序列，然后根据数据流中的时间顺序关系来进行处理和分析。这种模型的优点是可以处理数据流中的时间顺序关系，缺点是复杂度较高。
