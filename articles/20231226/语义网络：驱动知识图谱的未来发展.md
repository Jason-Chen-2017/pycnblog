# 2. 核心概念与联系

在本节中，我们将介绍语义网络和知识图谱的核心概念，以及它们之间的联系。

## 2.1 语义网络

语义网络是一种基于Web的信息资源，其目标是使这些资源具有可理解和可解析的结构和语义。语义网络的核心概念包括：

- **RDF（资源描述框架）**：RDF是语义网络的基本数据模型，用于表示实体、关系和实例之间的关系。RDF使用三元组（主题、属性、值）表示信息，例如（人，姓名，蒂姆·伯纳斯·李）。

- **OWL（Web Ontology Language）**：OWL是一种用于描述语义网络实体和关系的规范化语言。OWL允许用户定义自定义类和属性，以及这些类和属性之间的关系。

- **SKOS（简化知识对象结构）**：SKOS是一种用于表示知识对象结构的语义网络语言。SKOS允许用户定义类别系统、分类和关系。

- **SPARQL（RDF查询语言）**：SPARQL是一种用于查询语义网络数据的语言。SPARQL允许用户使用自然语言查询RDF数据库，并获得结果。

## 2.2 知识图谱

知识图谱是一种表示实体、关系和实例的结构化数据库，它可以帮助计算机系统理解和推理人类语言中的信息。知识图谱的核心概念包括：

- **实体**：实体是知识图谱中的基本元素，它们可以是人、地点、组织等。

- **关系**：关系是实体之间的连接，它们可以是属性、类型或其他类型的连接。

- **实例**：实例是实体和关系的具体实例，例如（蒂姆·伯纳斯·李，姓名，蒂姆·伯纳斯·李）。

- **图**：图是知识图谱的基本结构，它由实体、关系和实例组成。

- **图谱查询**：图谱查询是用于在知识图谱中查找特定实体、关系或实例的操作。

- **图谱推理**：图谱推理是用于在知识图谱中推导新关系和实体的过程。
