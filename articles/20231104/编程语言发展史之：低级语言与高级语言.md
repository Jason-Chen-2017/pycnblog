# 1.背景介绍


计算机编程语言，英文名称为programming language，起源于1940年代末期，作为人们在数字计算机中解决问题的工具而出现，如今天的各种编程语言。但是，早期计算机只能执行特定的程序，比如说能被计算机识别并翻译为机器码指令的程序。当时还没有可以像我们使用的英语、中文这样的自然语言，所以，为了方便程序员进行编程，工程师设计了一些变异语言，即低级语言（machine language）和高级语言（high-level language），用来减少编写程序时的低级细节。二者的关系就像是母语与非母语之间的关系。

在1950年左右，高级语言发明出来，它的作用主要是为程序员提供一个更加高效、简洁、易于理解的解决问题的方法。最初，高级语言只是一种抽象概念，是由汇编语言、模拟语言、子程序语言等一系列编程技术组合而成。随着时间的推移，高级语言逐渐成为主流，因为它有着比其前身更强大的能力。

1970年代中期至1980年代初，新一代计算机开始问世，计算机程序不再局限于特定的硬件，可以运行在各种各样的平台上。此时，对编程语言的需求也越来越强烈，为计算机带来了巨大的革命性变革。

早期的编程语言都倾向于具有某种特定功能，如操纵内存、调用操作系统等。这些功能往往比较简单，并且应用很广泛。例如，BASIC、Fortran、COBOL、Pascal、Simula 67等都是早期的低级语言。但是，1970年代后期至1980年代初期，人们发现，能够用简单的语句完成复杂任务的编程语言是不存在的。因此，需要一种新的编程语言来实现这种能力，这就是高级语言诞生的契机。

# 2.核心概念与联系
## 2.1 编程语言
在了解编程语言之前，我们首先要了解编程这个过程。从人类视角看待编程，可以分为三个层次：

1. 符号化层次：计算机只能识别和执行机器码指令，但程序员的思维方式却是面向问题的，他将一个个具体的问题抽象成抽象语法树(Abstract Syntax Tree，AST)。

2. 编译器层次：程序员通过编写源代码，经过编译器把源码转换为目标代码（目标代码就是机器码指令）。编译器可以把源码翻译成很多种目标代码，包括机器码指令、汇编代码、可执行文件等，根据目标代码的不同，编译器生成的程序可以在不同的操作系统或硬件平台上运行。

3. 解释器层次：如果目标代码不能立即运行，解释器则接手程序的执行，解释器读取源代码，逐行执行代码，遇到源代码中的语法错误时，抛出错误提示信息。解释器相对于编译器来说，速度快，占用资源少，不过对于那些只运行一次的代码或者短小的程序，使用解释器还是比较合适的。

计算机系统执行程序的方式一般分为两种：

1. 基于栈的虚拟机指令执行方式：在解释型语言中，程序就是一连串的指令集，系统逐条执行每一条指令，直到程序结束。而基于栈的虚拟机指令执行方式，类似CPU执行指令的方式，使用堆栈保存数据。在程序执行过程中，会创建变量、分配内存、调用函数、跳转控制流等，这些动作都在栈上完成。

2. 基于寄存器的虚拟机指令执行方式：针对虚拟机指令执行方式，目前比较成熟的是基于栈的指令集执行方式，而基于寄存器的执行方式，则是采用寄存器而不是栈存储数据。寄存器通常比栈小得多，因此，使用寄存器可以降低系统开销。另外，寄存器不需要分配内存空间，因此，可以使用它们作为通用目的的计算寄存器或临时寄存器。
