# 1.背景介绍


## 一、数据层面：
互联网企业在持续快速发展过程中，数据量不断增长，数据分析也成为重点。数据仓库、数据湖等技术解决方案已经成为IT架构演进中的一项重要方向。但是随着业务形态的不断变化，大多数公司仍然把主要的精力放在数据分析、数据建模上，对数据的价值却不足。因此，需要构建一个“数据中台”，作为数据的中转站，连接数据源和各种服务端应用，为客户提供更丰富、更智能的分析服务。
## 二、数据门槛低，但功能繁多：
数据中台的主要目的是为各种端到端服务提供统一的数据源。数据经过中转后可以为各个应用提供精准、及时、准确的用户画像信息。数据中台应当具备如下几个特征：
- 数据门槛低：数据中台不需要复杂的技术基础或前置条件，只要有数据即可接入。
- 功能繁多：数据中台除了数据存储、数据处理之外，还包括数据分发、数据安全、数据接口等一系列功能模块。数据中台应当能够满足各类应用对数据的需求。
## 三、数据孤岛问题：
互联网产品的规模越来越大，数据量的增加使得数据孤岛问题成为严峻的问题。如何利用数据资源实现业务价值的最大化？数据中台应当将业务数据、渠道数据、反馈数据、上下游数据等数据进行集成整合，将数据呈现给所有服务端应用，并通过数据分发实现不同业务之间的无缝衔接。
# 2.核心概念与联系
## 1、数据中台架构：
数据中台架构由数据采集中心、数据加工中心、数据仓库中心、数据服务中心、数据展示中心五个模块组成。其中，数据采集中心负责收集原始数据，包括各种来源的数据，包括日志文件、监控指标、第三方API接口等；数据加工中心则从原始数据中提取有用的数据，经过加工变换后输出可供应用访问的格式；数据仓库中心负责对采集和加工后的数据进行存储，以便数据调研、统计分析、数据报表等；数据服务中心则提供应用和其他服务端系统对数据集成查询、分析、计算、决策等功能；数据展示中心则提供图表、报表、仪表盘等多种形式的数据展现方式，为业务决策提供支持。
## 2、ETL（Extract Transform Load）组件：
数据中台中，ETL是最常用的模块。ETL即“抽取-转换-加载”三个词的首字母缩写，用于实现数据采集、清洗、标准化、规范化、编码、质量控制等多个环节的自动化。它包括四个步骤：
1. 数据抽取：指将各种数据源的日志、监控指标、第三方API等数据通过数据采集器从外部获取到本地机器。
2. 数据转换：指对获取到的数据进行清洗、规范化、标准化等处理，消除不必要的噪音和偏差。
3. 数据加载：指将处理好的数据导入数据仓库，供应用进行查询、分析、计算、决策等。
4. 数据分发：指将数据分发给其他服务端应用，通过数据接口方便地获取到所需数据。
## 3、数据中台数据迁移工具：
数据迁移工具主要包括两个部分，一是工具本身，二是平台。平台是运行工具的服务器，它通过数据同步的方式将数据同步到目标服务器，并提供监控、管理、审计等功能，帮助用户完成数据的迁移工作。目前国内外主流的数据迁移工具有MySQL的Mydumper、BinlogSyncer、DataX、Canal等。
