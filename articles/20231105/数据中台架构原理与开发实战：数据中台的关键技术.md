# 1.背景介绍

  
## 一、数据意义的提出和定义  
　　数据作为信息的重要载体之一，其价值在于通过对数据的分析，人们能够从中获得丰富的信息，帮助决策制定与运营优化，并实现对实体经济的改革。而数据的价值最大化则依赖于数据采集的质量、时效性、准确性及成本低廉，它的数据质量与准确性决定着其分析与挖掘的效率与精度，因此，数据采集与管理需要完善。传统的数据中心和数据仓库技术存在很大的技术壁垒，特别是对于快速变化的互联网企业来说，如何解决数据集成、加工、存储、计算和分析等多个环节之间的依赖性、灵活性和可靠性成为新的挑战。所以出现了数据中台这个概念。  
  
　　数据中台（Data Hub）由数据平台（Data Platform），数据湖（Data Lake），数据分析平台（Data Analysis Platform），数据应用平台（Data Application Platform），数据共享平台（Data Sharing Platform）五大组件组成，可以说是一个具有整体性的架构平台。它的作用是对业务、技术、资源进行集成和打通，实现数据驱动的目标。该平台基于开源中间件、云服务、大数据平台、机器学习引擎等支撑，具备统一数据价值的同时兼顾数据价值增长、实现数据价值保障的能力。  
  
　　数据中台的理念就是把企业最主要的数据需求纳入其中，围绕数据价值的内涵实现全方位、覆盖各个角度、全程协同的数据共享、分析、应用、价值。这样，企业就可以快速响应业务发展、重视数据价值投入与保障，实现数据价值的持续增长。  
  
## 二、数据中台架构设计模式  
　　数据中台架构的设计模式主要分为三种，包括推拉合一模式，提升分层模式，数据支撑层模式。下面我们就逐一进行讲述。  
  
### 1.推拉合一模式  
推拉合一模式是指将数据中台建设在公司内部主干网路上，形成一种自然流程。这种模式下，数据中台和公司其他模块紧密结合，职能清晰，数据交换方便，能较好地完成各模块之间的同步。如下图所示：  
  

　上面红色虚线框起来的区域是整个数据中台平台，其他蓝色虚线框起来的区域分别是公司的不同模块。当公司的数据产生时，首先被推送到数据中台。数据中台平台接收到数据后，会先进行统一处理和转储，然后按照不同维度进行传输，直至流向不同部门。当然，公司的各个部门也可以直接接入数据中台，也可按需获取数据。推拉合一模式虽然简单，但往往忽略了底层基础设施的升级更新维护，可能导致运行效率不高，且容易受到外部因素影响。  

### 2.提升分层模式  
提升分层模式是指将数据中台架构从整体层面分解到多个子系统之上。在数据中台架构中，数据源头主要集中在公司的数据中心，通过对数据的采集、存储、加工、分析等多个环节的集成和管理，形成一种数据栈，是整个数据中台架构的关键组件。而根据数据的特性、使用场景和需求，将数据按照其价值和重要性拆分为多个层级，如金融、电信、医疗、交通等，分别构建相应的金融数据中台、电信数据中台、医疗数据中台、交通数据中台等子系统，使得数据流转更加自洽和便捷。如此，数据中台架构可以有效降低整体复杂度，实现单个子系统的快速迭代和部署。  
  

　　提升分层模式下，数据中心负责采集原始数据，数据仓库负责存储经过清洗的大数据，数据应用平台则提供查询接口。每个子系统都独立部署，互相配合工作，最后汇总到数据中台上，实现不同层级之间的衔接。这种模式的优点是可以有效地提升各个子系统的耦合度和自治度，以及不同子系统之间数据共享和数据分析的弹性，但是依旧存在依赖性问题，难以形成一个完整的数据生态。  
  
### 3.数据支撑层模式  
数据支撑层模式是在现有数据中台基础上，增设数据支撑层。数据支撑层是对现有数据中台架构的补充，目的是实现数据完整性和可用性的最大化。数据支撑层负责数据共享、数据应用、数据分析、数据服务等多个环节的支持，比如构建数据基础设施、开发数据工具、搭建数据平台、建立数据基础服务、培育数据生态圈等，进一步提升数据中台架构的整体能力。如下图所示：  


　　数据支撑层模式主要用于解决数据中台架构的局限性，但由于存在数据支撑层的存在，势必增加数据平台的复杂度，不利于降低数据中台架构的规模。不过，数据支撑层模式在一定程度上弥补了推拉合一模式和提升分层模式的不足。因此，在实际中，如何选择合适的数据中台架构模式仍有待商榷。
