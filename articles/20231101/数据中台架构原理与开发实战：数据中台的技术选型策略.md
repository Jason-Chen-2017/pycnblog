# 1.背景介绍


## 数据中台简介
&emsp;&emsp;数据中台（Data Central）的全称是“数据集成中心”，它是一个独立于业务系统之外、提供数据采集、存储、加工和分析服务的系统。简单来说，就是把业务系统需要的各种原始数据按照既定流程整合到一起，提供业务系统多方面的支持。从这个角度看，数据中台属于信息集成的一种形式，具有很高的价值和意义。

在国内，数据中台通常被理解为一个外部化的、共通的计算环境或平台，里面可以容纳各类工具和服务。数据中台架构的创新主要体现在以下几个方面：

1.数据整合能力增强
   数据中台的核心功能是对不同的数据源进行整合，将来自各个业务部门的复杂的数据成为可供业务系统使用的单一数据源，并赋予其更丰富的维度和质量属性。通过数据中台的智能数据加工能力，业务系统能够基于更全面的理解业务需求，基于相互关联的多源数据做出更加准确和全面的决策。

2.数据价值明确
   数据中台的目的是清晰定义数据价值，围绕数据价值的产生和价值分配，实现数据的价值赋予和共享。因此，数据中台在创新过程中，一定要关注数据价值的明确性和实际应用场景，而不是僵化地遵循传统数据仓库的规范、模式和流程。

3.数据应用赋能
   在数据中台的布局上，可以设立多个层级，包括数据应用层、数据开发层、数据运营层等，这些层级都可以参与到数据价值赋予的过程当中。因此，数据中台需要构建和完善数据应用、数据服务和数据产品三个支撑层，进而打通不同层级之间的沟通渠道，赋能数据应用的完整循环。

4.数据治理优化
   数据中台还能促进数据治理改革，提升数据治理效率，推动企业走向科学数据管理，为业务决策提供更科学、可靠的依据。它不仅要解决数据分类、数据质量、数据存储、数据处理等底层问题，还要面临的重要挑战是如何保障数据安全、保护隐私、避免数据泄露、数据共享等方面的问题，从而让数据产生价值并有效满足用户的需求。

## 为什么需要数据中台？
### 数据中台带来的好处
数据中台的核心优点有以下几点：

1.数据资产集中管理
   数据资产指的是企业内部的各种数据，如客户信息、订单数据、商品销售数据、人力资源信息等。数据中台的建立就是为了能够对这些数据进行集中管理，方便各个业务部门进行数据共享、分析和加工，以达到业务需求的统一、优化、快速响应的效果。

2.数据价值明确
   数据中台建设和运行的关键在于数据价值的精确把握。数据中台的目标就是实现不同的数据价值明确，帮助各个业务部门更好地共同发挥自己的价值。

3.数据应用赋能
   数据中台可以使得业务部门在获得数据后能够更快、更高效地进行应用，从而实现数据价值赋予。

4.数据治理优化
   数据中台能够有效地提升企业的数据治理能力，通过数据分类、质量管控、数据备份、访问控制、数据保密等方式，降低企业数据被不法侵害的风险。同时，数据中台还可以通过数据标准化、数据信任机制等方式，让数据产生价值并更好地满足业务需求。

### 数据中台架构演进
&emsp;&emsp;数据中台架构的演变始于2008年，最初的版本是一个中心化的单体架构。随着时间的推移，越来越多的公司将数据中台部署在不同的地方，形成分散的架构，每个架构之间可能还有交流的环节。2019年最新版本的数据中台架构可以分为三层：

1. 数据调取层：负责对外提供数据接口，让业务系统能够获得原始数据；
2. 数据服务层：由若干数据集成服务组成，根据业务需求和系统规模进行优化配置；
3. 数据治理层：用于对数据进行分类、质量、共享和管理，并协助业务团队进行数据治理，提升数据价值。

数据中台架构如此多样化，背后的原因也不少，如垂直领域划分、多维度扩展、治理改革、更易用的数据接口等。总的来说，数据中台的优势在于将多个源头数据整合为一个数据中心，实现各个业务部门的数据更全面、更准确、更智能地结合在一起。
