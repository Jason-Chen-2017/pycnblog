# 1.背景介绍


数据中台作为一项新兴的架构模式，在最近几年涌现了很多优秀的产品或项目，比如说京东集团、美团点评、滴滴出行等。这些产品或项目都是基于数据中台架构来实现数据共享和数据分析的。那么，数据中台到底是什么？它解决了什么问题？如何实现数据共享？如何解决数据安全性问题？本文就来看一下数据中台的相关知识。

数据中台是一个“数据孤岛”：其中的数据只能通过数据中台进行交换和传输，而不能直接流通于各个业务系统之外。数据中台的目标就是将内部业务系统的原始数据进行统一收集整合、加工处理、加上上下游的应用层逻辑、对外提供统一的标准化接口供其他服务调用，最终形成一套完整的价值信息平台。数据中台一定程度上弥补了企业内外部不同部门之间的信息不对称、上下游系统的数据衔接问题，同时还可以提供具有数据生命周期管理、数据治理、数据质量保证、数据采集效率优化、数据共享协作等功能，有效满足企业对于数据价值的追求。

数据中台所面临的问题一般分为以下四个方面：

1.数据价值不确定性：由于数据的收集、处理过程存在一定随机性，因此产生的数据价值也可能发生变化。当数据价值发生变化时，如何确保上游的应用系统能够及时获取最新且准确的数据，是一个重要的难题。

2.数据价值多样性：数据中台除了具备数据共享能力以外，还应具备数据处理能力，才能提高数据的价值。数据处理能力是指对数据进行加工、统计、关联等操作，从而得到有用的商业价值。如何对多种类型的数据进行有效分类、加工处理，成为数据中台最主要的工作。

3.数据易用性要求：数据中台的运行环境要求很苛刻，系统的部署、运维、维护都需要考虑大量的时间和人力，其可用性（availability）、可靠性（reliability）、可伸缩性（scalability）也会受到极大的影响。如何根据数据中台的应用场景制定高可用、低延迟的部署架构，降低对用户的影响，是数据中台建设的一大挑战。

4.数据安全性问题：数据中台所承载的业务数据越来越复杂，越来越敏感，如果被不法侵入者获取到，那么对公司、对消费者都将造成严重损失。如何建立健全的运营机制，保障数据安全，也是数据中台建设的一大难点。

# 2.核心概念与联系
## 2.1 数据中台的定义
数据中台（Data Sandbox），又称为“数据集市”，是由内部不同业务部门或组织共同搭建的一个平台。其核心作用是“赋能、协同、发现”，以集中数据源、统一数据标准、打通数据壁垒、加速创新、加强协作为目的，构建出具备完整生命周期管理能力的价值信息体系。数据中台以数据湖为基础，通过不同渠道、不同手段收集和加工数据，并最终汇聚到数据湖之中。此外，数据中台还包括数据共享中心、数据治理中心、数据应用中心等多个子系统。

## 2.2 数据中台的组成
数据中台的组成一般包含三类子系统：数据仓库、数据湖、数据应用。其中，数据仓库用来存储、整理、计算和分析数据；数据湖则用来存储、保护和管理海量数据；数据应用中心则提供对外数据服务和应用，并支持业务智能的构建。除此之外，数据中台还包括运营平台、计算平台、工具平台、基础设施等多个支撑系统。


## 2.3 数据中台与数据的关系
数据中台构建成功后，数据就会进入数据湖，数据湖便成为企业数据资产。但这个数据资产只是存在于数据中台内，不会离开数据中台，真正意义上的“数据”还需要从原始数据中抽取、整理、加工、处理之后才能呈现在人们面前，变成企业提供的价值。所以，数据是数据中台的第一生产力。

但是，数据资产中绝大多数还是原始数据。数据资产其实是倚赖于数据中台生根发芽的，只有当数据中台成功运行起来之后，才能真正得到商业价值。而且，每天都会产生海量的数据，如何有效地管理、保护和处理这么多数据，才是数据中台的核心难点。

总结来说，数据中台是以数据湖为基石，实现集中、统一、共享、加工、价值创造的全流程数据管理平台。数据湖的数据必须通过数据中台才能产生价值，数据中台是整个数据生命周期管理体系的关键节点。只有将数据中台的各个组件成功融合、贯通，才能实现企业数据价值的实现。
