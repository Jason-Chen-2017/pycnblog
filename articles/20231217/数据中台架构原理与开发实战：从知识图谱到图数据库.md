# 2.核心概念与联系

## 2.1数据中台

数据中台是一种架构，主要包括以下几个核心组件：

- 数据集成层：负责将来自不同系统、源、格式的数据进行集成。
- 数据清洗层：负责将不规范、不完整、不一致的数据进行清洗和规范化。
- 数据标准化层：负责将不同业务、不同系统的数据进行统一和标准化。
- 数据共享层：负责提供数据服务，实现数据的发布、订阅和消费。
- 数据应用层：负责实现数据驱动的决策和智能化运营。

数据中台的核心目标是提供一个统一的数据资源、数据服务和数据应用平台，以实现数据驱动的决策和智能化运营。

## 2.2知识图谱

知识图谱是一种知识表示和处理方法，主要包括以下几个核心组件：

- 实体：表示事物、概念、属性等。
- 关系：表示实体之间的联系、依赖、属性等。
- 事件：表示发生的动作、行为、过程等。

知识图谱的核心目标是构建一个高质量、高效率、高可扩展性的知识库，以便进行查询、推理和推荐等应用。

## 2.3图数据库

图数据库是一种数据库类型，主要包括以下几个核心组件：

- 节点：表示事物、概念、属性等。
- 边：表示事物之间的联系、依赖、属性等。
- 图：表示一组节点和边的集合。

图数据库的核心目标是提供一个高性能、高可扩展性、高可靠性的数据存储和管理平台，以满足大规模、高并发、实时的应用需求。

# 创建命名空间

# 创建实体

# 创建关系
g.add((person, ns.has_name, "Alice"))

# 创建事件
