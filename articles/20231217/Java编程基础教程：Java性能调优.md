## 4.1 算法优化

我们来看一个简单的排序算法的例子，通过分析这个算法，我们可以了解到如何进行算法优化。

```java
public class Sort {
    public static void bubbleSort(int[] arr) {
        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            for (int j = 0; j < n - i - 1; j++) {
                if (arr[j] > arr[j + 1]) {
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                }
            }
        }
    }
}
```

从上面的代码可以看出，我们使用了一个简单的冒泡排序算法。这个算法的时间复杂度是O(n^2)，这意味着当输入数据量较大时，这个算法的性能会非常低下。因此，我们需要优化这个算法。

我们可以使用一个更高效的排序算法，例如快速排序或者归并排序来替换冒泡排序。这些算法的时间复杂度是O(nlogn)，这意味着当输入数据量较大时，这些算法的性能会更高。

```java
public class Search {
    public static int linearSearch(int[] arr, int target) {
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] == target) {
                return i;
            }
        }
        return -1;
    }
}
```

从上面的代码可以看出，我们使用了一个简单的线性查找算法。这个算法的时间复杂度是O(n)，这意味着当输入数据量较大时，这个算法的性能会非常低下。因此，我们需要优化这个算法。
