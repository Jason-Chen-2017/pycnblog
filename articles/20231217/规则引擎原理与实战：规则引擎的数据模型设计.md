# 2.核心概念与联系

在深入探讨规则引擎的数据模型设计之前，我们需要了解一些核心概念和联系：

1. **事实（Fact）**：事实是规则引擎中的基本数据结构，用于表示一些简单的数据或信息。事实通常包括属性和值，例如（name: John，age: 25）。

2. **规则（Rule）**：规则是一种用于描述事实之间关系的语句，它可以根据事实的属性值来生成结果。例如，如果一个人的年龄大于等于18岁，则这个人是成年人。

3. **决策（Decision）**：决策是规则引擎根据规则和事实来生成的结果，它可以是一种动作、一种状态或一种评估。例如，如果一个人是成年人，则他可以获得驾驶证。

# 4.具体代码实例和详细解释说明

在这个部分，我们将通过一个具体的代码实例来解释规则引擎的数据模型设计。我们将使用Python编程语言来实现一个简单的规则引擎。

```python
from rule_engine import RuleEngine

# 创建一个规则引擎实例
engine = RuleEngine()

# 定义一些事实
fact1 = {'name': 'John', 'age': 25}
fact2 = {'name': 'Jane', 'age': 18}

# 加载规则
engine.load_rule('if age >= 18 then adult')

# 执行规则并生成决策
engine.execute(fact1)
engine.execute(fact2)
```

在这个代码实例中，我们首先导入了`rule_engine`模块，然后创建了一个规则引擎实例`engine`。接着我们定义了两个事实`fact1`和`fact2`，分别表示John和Jane的年龄。

接下来，我们使用`engine.load_rule()`方法加载了一个规则`if age >= 18 then adult`，这个规则表示如果一个人的年龄大于等于18岁，则这个人是成年人。

最后，我们使用`engine.execute()`方法执行规则并生成决策。对于John，因为他的年龄是25岁，满足规则的条件，所以决策为`adult`。对于Jane，因为她的年龄是18岁，也满足规则的条件，所以决策为`adult`。
