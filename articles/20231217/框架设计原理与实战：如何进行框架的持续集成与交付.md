# 2.核心概念与联系

在了解框架设计的持续集成与交付之前，我们需要了解一些核心概念：

1. **框架设计**：框架设计是一种软件设计方法，它提供了一种可重用的、可扩展的基础架构，以便开发人员更快地构建新的应用程序。框架设计通常包括以下几个方面：

- 抽象：框架设计通过提供抽象来隐藏底层实现细节，让开发人员专注于实现业务逻辑。
- 模块化：框架设计通过将软件系统划分为多个模块来提高可维护性和可扩展性。
- 可扩展性：框架设计通过提供扩展点来让开发人员根据需要添加新功能。

2. **持续集成**：持续集成是一种软件开发方法，它要求开发人员在每次提交代码后立即进行构建和测试。这可以帮助发现和修复问题，并确保代码的质量和稳定性。

3. **交付**：交付是一种软件开发方法，它要求在软件开发过程中定期将软件产品交付给客户或用户，以获取反馈和验证软件的有效性。
