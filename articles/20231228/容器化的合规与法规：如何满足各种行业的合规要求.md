# 4.具体代码实例和详细解释

## 4.1 项目结构

项目结构如下：

```
order-management-system/
├── Dockerfile
├── app.py
├── requirements.txt
└── entrypoint.sh
```

- Dockerfile：用于构建容器镜像的配置文件。
- app.py：订单管理系统的主要应用程序。
- requirements.txt：应用程序的依赖项列表。
- entrypoint.sh：容器启动时执行的脚本。

## 4.2 Dockerfile

Dockerfile是一个用于构建容器镜像的配置文件。以下是Dockerfile的具体内容：

```
FROM python:3.7-slim

WORKDIR /app

COPY requirements.txt .

RUN pip install -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "app.py"]
```

这个Dockerfile使用了Python 3.7的镜像，设置了工作目录，复制了requirements.txt文件，并安装了应用程序的依赖项。然后复制了整个项目，暴露了5000端口，并设置了容器启动时执行的命令。

## 4.3 app.py

app.py是订单管理系统的主要应用程序。以下是app.py的具体内容：

```
from flask import Flask, request, jsonify
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///order.db'
db = SQLAlchemy(app)

class Order(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    customer_name = db.Column(db.String(100), nullable=False)
    product_name = db.Column(db.String(100), nullable=False)
    quantity = db.Column(db.Integer, nullable=False)

@app.route('/order', methods=['POST'])
def create_order():
    data = request.get_json()
    order = Order(customer_name=data['customer_name'], product_name=data['product_name'], quantity=data['quantity'])
    db.session.add(order)
    db.session.commit()
    return jsonify({'message': 'Order created successfully'}), 201

@app.route('/order/<int:id>', methods=['GET'])
def get_order(id):
    order = Order.query.get_or_404(id)
    return jsonify({'customer_name': order.customer_name, 'product_name': order.product_name, 'quantity': order.quantity})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
```

这个应用程序使用了Flask框架，定义了一个Order模型，用于存储订单信息。它提供了两个API端点，一个用于创建订单，另一个用于获取订单信息。

## 4.4 entrypoint.sh

entrypoint.sh是容器启动时执行的脚本。以下是entrypoint.sh的具体内容：

```
#!/bin/bash

# 等待数据库容器启动
until docker-compose ps -q app_db; do
    echo "Waiting for database..."
    sleep 1
done

# 启动应用程序
exec python app.py
```

这个脚本首先等待数据库容器启动，然后启动应用程序。这样可以确保应用程序只在数据库容器启动后才启动，从而避免了应用程序和数据库容器之间的依赖关系。
