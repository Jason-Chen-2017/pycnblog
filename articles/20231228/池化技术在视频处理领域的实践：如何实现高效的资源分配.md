# 2.核心概念与联系

池化技术是一种常用的资源分配策略，它可以帮助我们更有效地利用计算资源，提高视频处理的效率和性能。池化技术的核心概念包括：

1. 资源池：资源池是一种集中管理的资源，包括计算资源（如CPU、GPU、内存等）和网络资源（如带宽、延迟等）。资源池可以根据需求动态分配和释放资源，实现高效的资源利用。

2. 任务调度：任务调度是将任务分配到资源池中执行的过程。任务调度可以基于各种策略，如优先级、资源利用率、任务依赖关系等进行实现。

3. 任务监控：任务监控是监控任务在资源池中的执行情况的过程。任务监控可以帮助我们发现资源池中可能存在的问题，如资源不足、任务执行延迟等，从而进行及时的调整和优化。

池化技术与其他资源分配策略的联系包括：

1. 与分布式计算框架的联系：池化技术可以与分布式计算框架（如Hadoop、Spark等）相结合，实现高效的资源分配和任务调度。

2. 与云计算服务的联系：池化技术可以与云计算服务（如AWS、Azure、Aliyun等）相结合，实现高效的资源分配和任务调度。

3. 与容器技术的联系：池化技术可以与容器技术（如Docker、Kubernetes等）相结合，实现高效的资源分配和任务调度。
