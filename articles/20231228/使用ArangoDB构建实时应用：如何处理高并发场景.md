# 2.核心概念与联系

## 2.1 ArangoDB 的核心概念

ArangoDB 的核心概念包括以下几点：

1.多模型数据库：ArangoDB 支持文档、键值存储和图形数据模型。这使得 ArangoDB 可以处理各种类型的数据和应用需求。

2.分布式架构：ArangoDB 可以在多个服务器上运行，以实现高可用性和水平扩展。

3.高性能：ArangoDB 使用了一种名为 [FOX] 的高性能存储引擎，以实现快速的读写操作。

4.强大的查询能力：ArangoDB 支持多种查询语言，包括 SQL、JSON 查询和图形查询。

## 2.2 ArangoDB 与传统数据库的区别

ArangoDB 与传统的关系型数据库有以下几个主要区别：

1.数据模型：传统的关系型数据库只支持表格数据模型，而 ArangoDB 支持文档、键值存储和图形数据模型。

2.查询能力：传统的关系型数据库只支持 SQL 查询，而 ArangoDB 支持多种查询语言。

3.扩展性：传统的关系型数据库通常需要进行表结构设计和优化，而 ArangoDB 的数据模型更加灵活，可以在运行时进行调整。

4.实时性能：ArangoDB 的高性能存储引擎和分布式架构使其在处理高并发场景时具有优势。

# 4.具体代码实例和详细解释说明

在这里，我们将通过一个具体的代码实例来详细解释 ArangoDB 的使用方法。

## 4.1 创建数据库和集合

```
// 创建数据库
CREATE DATABASE mydb

// 创建集合
USE mydb
CREATE COLLECTION mycollection
```

## 4.2 插入和查询数据

```
// 插入数据
INSERT INTO mycollection { "name": "John", "age": 30 }

// 查询数据
FOR doc IN mycollection FILTER doc.age >= 30 RETURN doc
```

## 4.3 实时处理

```
// 实时监控
LET stream = FILTER { FOR doc IN mycollection RETURN doc.age >= 30 }
FOR doc IN stream
  LET age = doc.age
  IF age > 30 THEN
    UPDATE mycollection WITH { "age": age + 1 } FILTER doc._id == @doc._id
  END
RETURN doc
```
