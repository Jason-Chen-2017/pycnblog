# 2.核心概念与联系

DDD 是一種設計方法，它旨在將業務邏輯和數據模型與技術實現分離，以便在靈活地應對業務需求的同時保持高度的可維護性和可擴展性。DDD 的核心概念包括：

1. 领域（Domain）：這是一個具有特定業務邏輯和數據模型的範疇。在金融行業中，例如銀行、投資、保险等。
2. 实体（Entity）：這是一個具有特定業務邏輯和數據模型的實體。在金融行業中，例如客戶、帳戶、交易等。
3. 值对象（Value Object）：這是一個具有特定業務邏輯和數據模型的值對象。在金融行業中，例如金額、日期、百分比等。
4. 聚合（Aggregate）：這是一個具有特定業務邏輯和數據模型的聚合。在金融行業中，例如交易帳戶、投資組合等。
5. 域事件（Domain Event）：這是一個具有特定業務邏輯和數據模型的域事件。在金融行業中，例如交易完成、投資組合變更等。
6. 命令（Command）：這是一個具有特定業務邏輯和數據模型的命令。在金融行業中，例如交易、投資、取款等。
7. 查詢（Query）：這是一個具有特定業務邏輯和數據模型的查詢。在金融行業中，例如客戶信息查詢、帳戶餘額查詢等。

2. 定义业务规则和约束。
3. 设计数字模型和数据库架构。
