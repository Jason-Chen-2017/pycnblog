# 4.具体代码实例和详细解释说明

## 4.1 容器化的具体代码实例

在这个具体代码实例中，我们将使用Docker来实现一个简单的Web应用程序的容器化。首先，我们需要创建一个Dockerfile文件，如下所示：

```
FROM nginx:latest
COPY html /usr/share/nginx/html
```

这个Dockerfile文件定义了一个基于最新版本的Nginx的Docker镜像，并将一个名为html的目录中的文件复制到Nginx的默认文件夹中。接下来，我们需要构建这个Docker镜像，如下所示：

```
$ docker build -t my-web-app .
```

这个命令将构建一个名为my-web-app的Docker镜像，并将其推送到本地Docker镜像存储库中。接下来，我们需要运行这个Docker镜像，如下所示：

```
$ docker run -d -p 80:80 my-web-app
```

这个命令将运行my-web-app这个Docker镜像，并将其绑定到本地的80端口上。

## 4.2 Kubernetes的具体代码实例

在这个具体代码实例中，我们将使用Kubernetes来实现一个简单的Web应用程序的部署。首先，我们需要创建一个Deployment资源对象，如下所示：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-web-app-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-web-app
  template:
    metadata:
      labels:
        app: my-web-app
    spec:
      containers:
      - name: my-web-app-container
        image: my-web-app:latest
        ports:
        - containerPort: 80
```

这个Deployment资源对象定义了一个名为my-web-app的Deployment，它包含3个重复的Pod，每个Pod都运行一个基于最新版本的my-web-app镜像的容器，并将其绑定到80端口上。接下来，我们需要创建一个Service资源对象，如下所示：

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-web-app-service
spec:
  selector:
    app: my-web-app
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer
```

这个Service资源对象定义了一个名为my-web-app-service的Service，它将将请求从80端口转发到Pod的80端口，并将其公开为一个可以通过LoadBalancer访问的服务。
