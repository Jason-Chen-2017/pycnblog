## 1.1 背景介绍

线程池在 Java 中的出现，是为了解决传统的线程创建和管理模型的不足而产生的。传统的线程创建和管理模型主要有以下几种：

1. 单线程模型：只有一个线程在同时执行多个任务，这种模型简单易实现，但性能不佳。
2. 多线程模型：多个线程同时执行任务，这种模型性能更高，但线程的创建和销毁开销较大。

线程池是一种更高效的线程管理模型，它可以预先创建一定数量的线程，并将这些线程放入线程池中进行管理。当需要执行任务时，可以从线程池中获取线程进行执行，这样可以减少线程的创建和销毁开销，提高程序性能。

在 Java 中，线程池通常由 `Executor` 框架提供支持。`Executor` 框架提供了多种不同的线程池实现，如 `ThreadPoolExecutor`、`ScheduledThreadPoolExecutor` 等，可以根据不同的需求选择合适的线程池实现。
