# 2.Redis简介
Redis是一个开源的高性能、键值对数据库，它支持多种类型的数据结构，如字符串(String)、散列(Hash)、列表(List)、集合(Set)和排序集(Sorted Set)。Redis提供了一个基于内存的高速缓存服务，可用于提升Web应用的访问速度和降低数据库负载。Redis可以用于存储需要重复读/写的数据，例如热点新闻、用户信息等，通过Redis缓存可以减少后端数据库的压力，提升网站的响应速度和吞吐量。 Redis的主要优点包括：
- 高性能:Redis基于C语言开发，因此具有很高的执行效率。数据结构简单、对性能要求苛刻的情况下可以使用Redis。
- 数据类型丰富:Redis支持丰富的数据结构，包括字符串、散列、列表、集合和排序集。同时支持面向对象的语法，可以方便的进行数据操作。
- 持久化:Redis支持两种持久化方式，一种是快照持久化(RDB)，另一种是只追加文件持久化(AOF)。快照持久化可以在指定的时间间隔内将内存中的数据集快照写入磁盘，也就是从内存中创建的一个静态的拷贝，它恢复时直接加载所有数据到内存，非常适合数据备份或灾难恢复。而AOF持久化记录服务器执行的所有写命令，并在后台进程按顺序执行，采用的是日志的方式记录每个写操作，当发生故障时，可以通过重新执行这些命令来实现数据恢复。
- 事务:Redis的所有操作都是原子性的，也就是说要么成功执行，要么失败完全不执行。这确保了Redis的运行安全，使得多个客户端能够共同操作一个共享资源时不会出现并行更新导致数据的不一致的问题。事务可以一次执行多个命令，节省了客户端与服务器之间的网络延迟，提高了Redis处理请求的速度。
- 分布式:Redis支持主从复制，这意味着Redis主节点将数据的改动实时同步到从节点上，从而实现分布式集群。当主节点宕机时，可以由从节点提供服务，确保了Redis的高可用性。此外，Redis还支持Sentinel，它是一个独立的节点，它用来管理其他slave节点，并提供故障检测和自动故障转移的能力。
- 缓存:Redis除了支持数据的存储之外，还提供了数据缓存的功能。通过缓存可以减少数据库查询的次数，加快请求的响应时间。另外，Redis还支持多种数据淘汱策略，包括最少使用(LRU)、最近最少使用(LFU)和最近最久未使用(LRU)算法，可以根据业务需要设置不同的淘汱策略，进一步提升缓存命中率。
- 脚本:Redis支持Lua脚本，可以让客户端和Redis服务器之间交互执行脚本。Redis的脚本功能可以帮助解决一些重复性的工作，比如批量插入数据。
以上就是Redis的一些主要特性。除此之外，Redis还有很多其它优秀的特性，例如可扩展性，Redis支持无限的数量级的内存和数据类型，对主从复制、哨兵模式、集群模式支持非常友好。所以，如果你打算用Redis开发应用，那么一定要亲自尝试一下Redis，掌握它的功能和使用方法。
