# 1.简介
  

PEP 8（Python Enhancement Proposal）是由 Python 官方发布的一份编码风格指南。它是一种合理、可读性强、与语言特性紧密结合的编码风格指南，旨在统一 Python 社区中不同开发者的编程风格，使得编写出更易于理解、维护的代码。它提供了一系列的代码格式化、命名约定以及注释规范，这些规范可以有效地提高代码质量、降低错误率并提升代码的一致性。PEP 8 对 Python 的新手也是非常友好易懂的。本文将阐述 PEP 8 的背景、目标、适用范围、基本要求和最佳实践。

# 2.背景介绍
PEP 8 是 Python Enhancement Proposals（Python 增强建议）的缩写，它是一份官方文件，旨在推进 Python 社区关于代码格式化、命名约定的一套共同的标准化规则。它的首次发布是在 2001 年 7 月，并持续了近十年。PEP 8 中的“增强”一词，表明该文档主要针对那些想对 Python 代码进行改善的人。

PEP 8 提倡统一代码风格，而非制订一套自己的风格指南。因此，它不是 Python 语法规范，而仅仅是一份编码规范，涉及的知识面也比较广泛。例如，除了对代码格式化、命名约定等方面的要求，它还提出了一系列的最佳实践和建议，例如文档字符串的使用方式、如何命名变量、如何组织模块、什么样的文件需要放到版本控制系统中等。如果要成为一名合格的 Python 工程师，了解并遵守 PEP 8 有助于提升自己的代码能力，同时也会帮助团队保持代码的整洁、易读与一致。

# 3.基本概念术语说明
## 3.1 行宽限制
每行字符的最大数量叫做行宽限制。一般来说，推荐每行不超过79个字符，但对于一些特别长的句子或表达式，也可以适当调整行宽。

## 3.2 空白字符
空白字符包括空格（ASCII码值为32），制表符（ASCII码值为9），换行符（ASCII码值为10）。通常情况下，四个空白字符是一个好的代码阅读习惯。不要使用过多的空白字符，这样会影响代码的可读性。

## 3.3 行与缩进
每行的开头都应该缩进4个空白字符。绝对不能使用 tab 键缩进，因为不同的编辑器或 IDE 会有不同的显示效果，而且很容易造成混乱。

```python
if True:
    print("This line is indented with four spaces")
```

## 3.4 注释与文档字符串
PEP 8 不要求一定要写注释，但是建议良好的注释能够帮助自己和他人理解代码。

文档字符串（docstring）是用来描述一个函数、类或者模块作用的方法。PEP 8 中要求所有的文档字符串都要以三个双引号（""")开头和结束，并且建议所有模块、函数、类定义后都要有文档字符串。

```python
def my_function(arg):
    """This function does something."""
    # some code here
```

## 3.5 分号
每条语句末尾都要加上分号（;）。虽然这不是强制性的，但一般来说，能少打一个分号就少打一个吧！

```python
x = 1 + 2; y = x * 3
```

## 3.6 名称
PEP 8 中有很多的命名规范，其中最重要的有以下几点：

1. 采用单词小写拼写法，多个单词之间用下划线连接；
2. 模块名和包名采用小写且单词连在一起的格式；
3. 函数名采用小写，多个单词之间用下划线连接；
4. 类名采用驼峰命名法，即每个单词的首字母都大写；
5. 异常名采用驼峰命名法，以 “Error” 或 “Exception” 为后缀；
6. 全局变量名采用小写，多个单词之间用下划线连接。

还有其他一些命名规范，如常见缩写的全大写形式（如 HTTP，XML）等。总之，遵循 PEP 8 的命名规范可以让你的代码更易读、更易理解，并减少潜在的错误。
