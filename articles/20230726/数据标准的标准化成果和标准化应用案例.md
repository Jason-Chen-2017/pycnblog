# 1.简介
         
数据标准化的目的是为了保证数据的准确性、完整性、一致性、及时性、可移植性，这些要求其实都体现了数据标准化的价值。那么什么是数据标准呢？数据标准是由数据组织者或者标准化组织发布的一组规则或规范，它明确定义了数据记录和交换方式，为不同的数据提供统一的认识和理解。数据标准化作为一个综合性的工程，其核心工作就是确立、维护和完善数据标准。传统的数据标准化模式分为内部标准化、外部标准化和标准集成三种类型，而在互联网+数据中心的新时代背景下，数据标准化又面临新的挑战，例如海量数据、多样化的应用场景、异构系统和数据格式等，因此，如何构建统一、有效、高效的数据标准成为当前研究热点。近几年来，随着云计算、移动互联网、物联网、区块链等新兴技术的广泛应用，数据标准化带来的意义也越来越突出。 

本文首先简要回顾了数据标准化相关的基本知识和技术；然后，通过分析国际、中国、欧洲、美国等地区已有的标准化成果进行梳理；最后，结合实际案例，阐述数据标准化的目标和作用，并探讨数据标准化的标准化模型、标准化工具、标准化工作方式和标准化管理方法。 

2.基本概念术语说明 
## 2.1 数据标准化概念及相关技术 
数据标准化的概念是指对数据的整体把握、结构化、分类、管理、检索和共享的过程。它的核心是建立起标准，用以确保数据真实性、正确性、有效性、完整性、一致性及时性。数据标准化可以采用多种形式，如：  
1) 内部标准化：指企业内部建立数据标准，使所有业务部门之间、业务和数据资源之间的数据通信符合相同的规则。 
2) 外部标准化：指组织与国家之间或组织与第三方之间对数据标准制定和实施。 
3) 标准集成：指将多个不同标准按照某种规则融合到一起形成一个标准集。  
4) 标准机制：指标准制定者设定的行政措施和流程，它包括审批制度、核查、监督、解释、评估和追溯等。 
5) 数据模型：数据模型是数据结构的抽象表示形式，它表示事物的静态特征和规律性，通常借助于图表、词汇描述、表单等符号来呈现。数据模型用于建模、表示和理解数据的语义和结构。 
6) 数据仓库：数据仓库是一个基于网络的集成数据库，用来存储、整理、分析和报告有关业务、客户、产品和市场的信息。它是个重要的标准化资源，它存储的不仅仅是业务数据，还包括企业信息、人员信息、财务信息、法律信息、品牌信息等。 
7) 数据挖掘与分析语言：它是一种结构化查询语言，能够对复杂、结构化、半结构化的数据进行分析、挖掘、关联、分类和计量等一系列复杂的处理操作。

相关技术如下所示： 
1) 数据建模：数据建模的主要目的是建立关于数据特征和关系的逻辑模型。其主要工具有ERwin、JDEdwards、Lucidchart、Quickmodeler等。 
2) 数据转换：数据转换是将非标准数据转化为符合标准格式的过程。主要工具有XSLT（XML Stylesheet Language Transformations）。 
3) 数据治理：数据治理包括数据质量保证、数据可用性和完整性、数据授权、数据共享、数据资产管理、数据编制、数据披露、数据保护等。 
4) 数据集成：数据集成主要解决多个异构数据源间的数据同步、合并、转换、复制等问题。相关工具有Informatica Integration Server、Oracle Data Integrator、SAP Data Intelligence Platform等。 
5) 元数据管理：元数据管理是指对数据的描述、结构、特征和含义进行集中、标准化和表达的过程。相关工具有IBM InfoSphere Information Governance Catalog、MySQL Database Design Tools、Apache Hive Metastore、PostgreSQL Metadata Management System等。 
6) 联合查询语言：联合查询语言是一种SQL扩展语言，支持多种数据源之间的复杂查询。相关工具有EXASolution、Squirrel SQL Client、Teradata Warehouse Optimizer等。 
7) 数据访问控制：数据访问控制是指对数据安全访问进行限制和约束的过程。相关工具有IBM Security Access Manager for LDAP、OpenLDAP Directory Server、Oracle Advanced Security Administration Edition等。 
8) 数据保密性：数据保密性是指保护数据安全性和隐私的技术手段。相关工具有IBM Data Protector、Trusted Computing Group SecureHash Standard Algorithm Specification、Kalium Cryptographic Library等。 
9) 数据保护：数据保护是指防止数据泄露、毁坏、篡改、越权和恶意使用的技术手段。相关工具有IBM Data Loss Prevention、Microsoft Azure Advanced Threat Protection、EMC ECS Enterprise Content Protection等。 
10) 数据备份：数据备份是指对原始数据进行完整备份的过程。相关工具有Backup Exec、Acronis True Image Backup、Riverbed Steelhead NSX-Protect等。


## 2.2 数据标准化模型 
数据标准化模型分为内部标准化、外部标准化、标准集成、标准机制四种类型。其中，外部标准化模型最为流行。内部标准化模型需要建立数据标准化的自然流程，强调规范标准的制定、宣传、推广与实施。标准集成模型把不同标准按照某种规则融合到一起形成一个标准集。标准机制模型认为，标准制定者设定的行政措施和流程是指由标准制定者制定或设置的一套制度、规则或程序，它包括审批制度、核查、监督、解释、评估和追溯等。 

1) 内部标准化模型 
内部标准化模型的核心是建立数据标准化的自然流程。该模型适用于具有内部数据标准化需求的机构。它包括以下四步： 
1） 需求确定：要标准化的业务领域、用途、模式、载体等方面进行研究、分析、理解和设计。 
2） 标准设计：根据需求提炼出的标准主题、规则、过程和技术方案进行设计。 
3） 标准的实施：规范标准的制定、宣传、推广与实施。 
4） 技术支持：确保平台实现、部署和运营符合标准要求。 

2) 外部标准化模型 
外部标准化模型是指组织与国家之间或组织与第三方之间对数据标准制定和实施。该模型的核心是确立共同的价值观和制度框架。它包括以下五步： 
1） 情况说明：对标准制定前景和机遇进行充分反映，对待标准化进程中的各种因素和挑战。 
2） 方案探讨：明确制定标准的范围、对象、时机、条件、内容、手段和费用，以及标准化实践的各项要求。 
3） 评估风险：对标准实施的影响、后果以及合规性进行严格评估，明确标准实施的责任划分、流程和制度。 
4） 承诺目标：以承诺书的方式对组织的义务、权利和义务做出详细说明，以确定其承担的责任和义务。 
5） 执行实施：履行承诺并顺利执行标准实施，包括标准制订、培训、监控、审计等。 

3) 标准集成模型 
标准集成模型是指将多个不同标准按照某种规则融合到一起形成一个标准集。该模型包括以下两个步骤： 
1） 标准选择：在众多的标准中选取满足要求的标准。 
2） 标准集成：将选定的标准按照一定的规则集成到一起。 

4) 标准机制模型 
标准机制模型认为，标准制定者设定的行政措施和流程是指由标准制定者制定或设置的一套制度、规则或程序，它包括审批制度、核查、监督、解释、评估和追溯等。该模型包括以下几个方面： 
1） 行政决策：标准制定者对于标准实施的决策作出制定，遵循合理、透明、公正的原则，及时地反馈结果。 
2） 审核和监督：数据标准的实施应当对标准制定者、拥有者和使用者进行持续、全面的审查，定期进行自我评审，确保标准的正确实施和实效。 
3） 解释权：数据使用者有权要求获得标准制定者的解释权。 
4） 复核权：数据使用者有权要求复核。 
5） 纠错权：数据使用者有权提交数据错误发现，要求标准制定者予以更正。 
6） 结算权：当标准实施完成后，数据使用者有权要求得到结算报告。

## 2.3 数据标准化工具 
1) 数据采集工具：采用逐条或批量导入数据的方法。如：主数据、附加数据、日志文件、网络数据等。 
2) 数据校验工具：采用检查、验证、修正数据的办法。 
3) 数据清洗工具：对数据进行初步的整理、处理、清除、验证等。如：Excel清洗、R语言清洗、文本编辑器清洗等。 
4) 数据迁移工具：跨系统、设备的异构数据迁移，支持不同的格式数据。如：异构系统数据迁移工具、SQL数据迁移工具、Python数据迁移工具等。 
5) 元数据管理工具：对数据属性、结构、特性进行描述，并且能够支持多样化的使用模式。如：Informatica Metadate Hub、Dataiku、Metabase、MySQL Database Designer等。 
6) 联合查询语言：支持多种数据源之间的复杂查询。如：EXASolution、Squirrel SQL Client、Redshift Spectrum、Hive LLAP、Presto等。 
7) 数据访问控制工具：用于限制、约束数据安全访问。如：IBM Security Access Manager for LDAP、Apache Ranger、Oracle Fusion Middleware、Microsoft Active Directory等。 
8) 数据保护工具：用于保护数据安全性和隐私。如：IBM Data Protector、Trusted Computing Group Secure Hash Standard Algorithm Specification、Kalium Cryptographic Library等。 
9) 数据备份工具：用于对原始数据进行完整备份。如：Backup Exec、Acronis True Image Backup、Veritas NetBackup等。
