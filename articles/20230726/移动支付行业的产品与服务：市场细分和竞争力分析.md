# 1.简介
         
随着移动互联网、智能手机等新兴产业的发展，移动支付也成为越来越多的消费者选择。本文将通过分析目前国内外主要移动支付公司产品、服务和业务模式、各自的优势及客户价值，以及各自对行业发展趋势的预测和预期，阐述移动支付行业的整体情况和发展方向。同时，我们还将重点关注行业内的一些热门应用场景以及中国移动支付行业的发展前景。本文将从移动支付市场的细分入手，包括腾讯、微信、支付宝、银联等APP支付、公众号支付、电商支付等多个方面进行详细介绍，并根据行业现状和变化的实时数据对各个产品和服务的发展方向和竞争力作出评估和预测。本文的目的在于系统梳理移动支付行业的产品与服务，引起读者对该领域的相关研究和讨论，以便更好地参与到移动支付行业的竞争中来。
# 2.市场概念和关键词解析
移动支付（Mobile Payments）是指利用移动设备（如iphone、android等）进行支付的服务。移动支付市场共计7亿美元，占据移动互联网市场近9成份额；相对于PC端支付方式，移动支付在收费、用户习惯、支付手段、支付场景等方面都有明显优势。移动支付分为APP支付、公众号支付、电商支付三种类型，其中APP支付最受欢迎，占比64%，是最主要的支付形式；公众号支付是指网站或APP开通微信支付功能后，给公众号用户提供“扫码付款”等付款方式；电商支付是指第三方电商平台提供的线上支付服务，包括电子支付、实体支付和数字虚拟货币。
主要代表性应用场景：APP支付
超级app:腾讯充值、微信优惠券、微信红包、QQ钱包
日活百万+、支付宝等第三方支付平台开通支付功能
电商平台：1、天猫：淘宝升级版，提供淘宝扫码、快捷支付、免密支付；2、京东：提供京东金融支付、银联闪付；3、拼多多：提供拼多多POS支付、免密支付；4、苏宁易购：提供苏宁易购付款、银联闪付；5、斗鱼：提供斗鱼直播、虎牙直播收益抵扣；6、Uber：提供Uber GO支付。
其他应用场景：公众号支付
微信公众号、小程序：微信的公众号支付、微信小程序提供了扫码、免密支付、投放广告等功能；微信红包、微信优惠券都是基于公众号支付平台的应用场景。
个人中心及商户中心：微信支付个人中心、微信支付商户后台提供了查看、收付款、设置等功能；用户可以通过个人中心随时随地查询、收付款，提高了个人支付能力。
电商支付：第三方电商平台采用线上支付的方式，包括支付宝、微信、余额支付等。
移动支付行业的关键词：
1、支付宝：中国领先的移动支付平台之一，拥有超过20亿的日活跃用户，覆盖12亿人口；国内知名企业，包括支付宝集团、蚂蚁金服、芝麻信用、当当网、润禾互动、全民健身房等。
2、微信：第二大主流的移动支付平台之一，目前有2亿月活跃用户，用户规模广泛且持续增长。拥有海量公众号和小程序用户。
3、网银支付：相对于现金或其它支付方式，在移动支付领域更为普遍的支付方式，主要由中国银联、招行、建设银行等所提供。
4、POS支付：短时间、简易快捷的一种移动支付方式，只需要安装终端设备，打开APP即可完成支付。
5、分期付款：指用户下单时可指定还款期限，一次性还清所有本金和利息。
6、微聚合：微信的小程序一个，提供商家发布信息、商品推荐、线下店铺、拼团等功能。

## 3.1 APP支付概况
### 3.1.1 三大区域网络环境差异带来的支付问题
移动支付APP平台依靠网络环境，不断优化其运营策略来解决网络因素导致的支付问题。主要包括以下三个方面：

1. 网络延迟：即APP支付过程中的客户端-服务器端通信延迟。由于不同地区及网络带宽条件不同，APP支付过程中客户端-服务器端的通信延迟会比较突出。另外，随着公司业务发展，服务器负载可能会较高，当服务器资源紧张时，客户端-服务器端通信延迟也可能出现相应的变化。

2. 数据传输：即APP支付过程中上传下载的数据量大小。由于不同的移动网络制式，设备连接的上网方式、设备性能不同，因此APP支付过程中数据传输的效率也会存在差别。例如，对于较窄带宽、较慢的网络环境，APP支付过程中数据的上传下载效率可能就会较低，这就需要APP开发人员针对此种网络环境做优化。

3. 服务质量：即APP支付过程中各种服务的可用性和响应速度。由于移动互联网的高速发展，移动支付APP平台应对快速变化的市场形势和用户需求，可以实现快速迭代和快速响应，这就要求APP平台能够提供足够好的服务质量。例如，移动支付平台可能会遇到网络波动、设备故障等情况，如果APP平台不及时调整运营策略或者维修设备，可能会造成用户无法正常进行支付。

### 3.1.2 移动支付场景分类
根据移动支付的支付场景，可以将移动支付分为以下四类：

1. 闲聊付费：这类场景一般是指小额快捷的支付场景，如分享、打赏、红包等。在这种场景下，用户可能会用自己的闲暇时间，充分利用闲置的设备进行支付。

2. 用户行为聚合：这类场景一般是指用户在不同APP之间频繁切换的场景。例如，用户希望在淘宝、天猫和支付宝之间进行账户余额的同步，又或者是在微信和支付宝之间进行账单的汇总和结算。

3. 用户产品或服务交易：这类场景一般是指用户购买特定产品或服务的场景，如在线旅游、网络直播、社交媒体等。在这些场景下，用户可能需要在APP平台间进行选择、支付、确认、查看等一系列复杂的流程。

4. 其他场景：除以上四种场景外，还有很多支付场景都属于此类，如主题电影票务、主题演出门票购票、游戏充值、生活缴费等。

### 3.1.3 APP支付流程图
![](https://upload-images.jianshu.io/upload_images/17215953-6f2c9b6294f2d5aa.png?imageMogr2/auto-orient/strip|imageView2/2/w/1200)

APP支付的流程可以分为五步：

1. 注册：用户注册后，APP发送用户身份信息、支付密码等必要信息，验证用户身份。

2. 登录：用户登录后，APP检测是否具备支付权限，并请求用户进行手机支付密码或邮箱支付验证码等方式的二次认证。

3. 消费确认：用户选择待支付的商品或服务，点击支付按钮，APP根据商品或服务情况向用户展示待支付信息。

4. 支付授权：APP检测是否已获得用户授权，如用户之前未同意授权支付APP的某些功能，则显示授权提示框让用户选择是否同意。

5. 支付结果：APP接收到支付指令后，向支付平台发起支付请求，支付平台处理完支付请求后返回支付结果给APP。APP根据支付结果判断是否支付成功或失败，并反馈给用户支付状态。

## 3.2 APP支付产品结构及优势
### 3.2.1 APP支付产品结构
当前市面上主要的APP支付产品主要包括以下几类：

1. 普通支付：这是最普通的支付方式，用户直接进入商家支付宝页面，输入交易金额、选择支付方式等，然后等待支付成功通知。

2. 分期付款：顾名思义，就是用户可以在购买的金额基础上，分多次付款，每期还款额和期数由商家决定。

3. 拼团买单：这个支付方式的特点是将用户分为若干个组队的队员，每个队员都按照规定价格进行购买，最后只有团长才能享受到全部价格。这个产品的核心理念是尽可能降低平台风险，提升支付效率。

4. 会员付费：顾名思义，就是用户成为某个APP的会员后，会自动享受该APP的各种福利，比如减免、折扣等。会员付费往往可以帮助推动APP的发展，增加用户粘性，拉动渠道盈利。

5. 在线兑换：即类似于积分商城这样的APP，可以提供用户商品的在线购买，支持实物和虚拟商品，提供优惠券、积分、礼品等促销方式，提升用户参与度和留存率。

### 3.2.2 APP支付优势
移动支付的发展已经走到了顶峰，已经成为行业标杆，但是移动支付依然面临着巨大的挑战。首先是支付的各种安全隐患和用户数据泄露的问题，支付过程中用户的敏感信息可能会被恶意篡改甚至被泄露。另一方面，移动支付的开发仍处于快速迭代阶段，新技术革命迭起，商家支付平台的技术要求不断更新，但无论如何，如何保证整个支付流程的顺畅、准确和有效，这一点依然很重要。综合来看，移动支付APP平台的一些优势如下：

1. 安全性强：移动支付APP平台的支付方式和逻辑都比较简单，所以安全性比较高。另外，移动支付APP平台还要防止用户数据的泄露，所以移动支付平台的安全也不能完全依赖第三方支付平台。

2. 支付体验佳：移动支付APP平台的支付体验非常好，因为APP界面设计精美，易于理解。在APP内部的支付界面，可以提供丰富的付款方式，如支付宝，微信，银联等，这使得用户的选择范围变得多样化。

3. 投放生态：移动支付APP平台一直在努力探索新的支付方式，提升用户体验。除了聚合支付、会员付费等以往的营销方式之外，APP还可以提供更多类型的营销方式，比如返佣，免费试用，分享返利等，促进用户活跃度和忠诚度。

4. 方便快捷：移动支付APP平台的功能完善程度很高，用户可以快速找到想要的支付方式，点击一下就可以完成支付。另外，APP平台的操作逻辑简单，支付流转快，用户可以及时获得支付结果，解决了传统支付方式中因用户体验差，支付过程繁琐等问题。
