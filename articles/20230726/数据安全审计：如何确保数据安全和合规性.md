# 1.简介
         
随着互联网的发展，越来越多的数据、信息通过互联网传输到世界各地。网络存储的海量数据使得个人隐私、机密信息等存在很高的安全风险。为此，许多公司和政府部门都在紧盯数据的安全问题。数据安全审计就是一项技术，旨在监测和检测计算机系统中的各种数据和信息是否被未经授权访问、修改或泄露。由于数据的重要性，任何对数据的使用者和维护者，均应高度重视对数据的安全管理工作。因此，企业数据安全的事关重大，为此，建立专门的数据安全计划和管理体制是非常必要的。本文以实施数据安全审计为目标，阐述了相关的背景知识和概念。
# 2.相关概念和术语
## 2.1 数据安全管理的概念
数据安全管理（Data Security Management）：是指对计算机网络和系统中所有数据和信息进行控制和保护，防止其遭受恶意攻击、非法泄露、篡改、毁坏、伪造等侵害，实现信息安全。其关键任务包括：制定数据安全策略、评估风险、识别威胁、分析威胁、响应威胁、监控预警、恢复服务、整体保障、管理风险、应急处置等。
## 2.2 数据安全审计相关术语
### 2.2.1 概念定义
数据安全审计（Data Security Auditing）：指根据法律法规、公司、组织等不同标准对电脑系统和网络中的数据进行收集、分析和报告，发现并解决系统漏洞、安全缺陷、数据泄露等安全事件，提升信息安全水平。它通过收集、分析、比较、分析、报告等手段来检查和分析电脑系统和网络上数据资料的安全性、完整性、可用性、真实性等方面情况。
### 2.2.2 相关名词解释
#### 2.2.2.1 数据泄露
数据泄露：指在不经授权的情况下，泄露或者提供给无权访问的人员。
#### 2.2.2.2 安全事件
安全事件：包括计算机网络安全事件、实体安全事件和系统安全事件等。
#### 2.2.2.3 安全威胁
安全威胁：是指对企业、个人生活、财产安全造成严重损害的行为。安全威胁一般来自于外部、内部、技术环境等多方面。
#### 2.2.2.4 安全漏洞
安全漏洞：是指由于软件、硬件或程序设计错误、缺陷、配置失误或其他原因导致系统中存在的一种弱点。
#### 2.2.2.5 信息保护标准
信息保护标准（Information Security Standards）：指根据国家标准、国际标准、行业标准等统一认可的技术、管理、法律规范。
#### 2.2.2.6 数据隐私
数据隐私：指能够被某些人识别、用于个人交易和交流，且可能影响个体利益的信息。
#### 2.2.2.7 信息安全管理部门
信息安全管理部门（ISMD）：是一个独立的政府机构，负责全社会信息安全，包括打击犯罪、监督执法、加强国际合作、提升行业和人员素质。
#### 2.2.2.8 持续性安全运营
持续性安全运营（Continuous Security Operation）：是指企业在系统中采用一系列有效措施、确保安全，以保证系统不间断运行，并始终维持最高级别的安全水平。
#### 2.2.2.9 数据保护标准
数据保护标准（Data Protection Standards）：是在特定区域内，国际组织通过国际标准化组织（ISO/IEC）或行业协会设置的一套法律要求和技术标准，以保障个人或组织的隐私、自由、财产权、数据安全及其所处理的数据的合法权益，避免数据泄露、非法使用和滥用，从而促进经济、政治和社会的发展。
#### 2.2.2.10 信息安全管理委员会
信息安全管理委员会（ISMA Committee）：是一个由代表各领域专家组成的、面向政府部门和企事业单位的正式机构。ISMA主要负责监督、评价和改善政府部门和企事业单位的信息安全管理措施，以及将信息安全管理理论应用于日常工作。
#### 2.2.2.11 敏感信息
敏感信息：是指对个人隐私、商业秘密、生命健康和财产安全造成严重危害或负面的影响，需要适当加以保护的不当发布或传播的信息。
#### 2.2.2.12 加密技术
加密技术（Cryptography）：是一种将明文转变为密文的方法，目的是为了保护信息的机密性、完整性和可用性。
#### 2.2.2.13 网络安全
网络安全（Network Security）：是指保障网络通信安全，包括保障通信链路的安全、保障数据传输过程的安全、保障网络资源的安全和保障信息安全的措施。
#### 2.2.2.14 信息安全管理体系
信息安全管理体系（Security Information Management System）：是一种企业、组织和网络管理工具集合，用来管理对信息的安全性、完整性、可用性、真实性等方面情况的审查、检测、记录和响应。
