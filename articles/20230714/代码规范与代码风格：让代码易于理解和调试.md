# 2.基本概念术语说明
## 2.1 注释
注释是用来解释代码的。对于复杂的代码来说，有必要加入很多注释来提高代码的可读性，增强代码的易懂性，便于其他开发人员快速理解代码的意图。
注释一般分为两类：

1. 文件级注释：主要用于对整个文件的描述、作者、版权信息等；
2. 函数级注释：主要用于对函数的功能、输入参数、返回值等进行解释；

例如，文件头通常包含作者信息、版本号、创建日期等。函数头一般包括描述性注释、输入输出参数的注释以及用法说明。

```
/**
 * @author user1
 */
public class Main {
    /**
     * This is a method to add two numbers.
     * 
     * @param num1 First number to be added.
     * @param num2 Second number to be added.
     * @return The sum of the two numbers.
     */
    public int add(int num1, int num2) {
        return num1 + num2;
    }
    
    // main function for testing purposes
    public static void main(String[] args) {
        Main m = new Main();
        System.out.println("Sum: " + m.add(10, 20));
    }
}
```

## 2.2 命名规则
命名是一个相当重要的工作，尤其是对自己正在编写的代码来说。命名是变量、函数、类的名称。如果命名得不合适，则会造成难以阅读、理解的混乱。命名有什么样的规则呢？下面列举了一些常见的命名规则。

### CamelCase
CamelCase 是小驼峰命名法，它的特点就是每个单词的首字母大写，除第一个单词外，其他单词的首字母都采用小写字母。这种命名方式比较简洁，容易看懂，不会出现连续多个单词的缩写。

比如：`myVariableName`，`firstName`。

### PascalCase
PascalCase 是帕斯卡命名法，它的特点是所有的单词都是以大写字母开头。因此，它比CamelCase更加符合英文语法习惯。

比如：`MyClassName`，`FirstName`。

### snake_case
snake_case 是蛇形命名法，它的特点是所有单词都采用下划线连接。

比如：`my_variable_name`，`first_name`。

还有一种命名方式叫做 kebab-case ，用中横线代替下划线，比较类似于网站URL的写法。

比如：`my-variable-name`，`first-name`。

### 大驼峰法、下划线连接法、下划线连字符法
其实还有三种命名方式，它们之间有着各自不同的特点，我们可以通过记忆、理解和应用选择适合自己的命名方式。

比如：`EmailModel`，`customer_id`，`TABLE_NAME`。

## 2.3 格式化
格式化是指按照一定的规范，排列代码的位置和排版样式，使得代码易于阅读。主要有以下四种格式：

1. 紧凑型格式：这种格式中，代码都挤在同一行里面，占据很少的空间，并且不允许使用换行符。
2. 中间距型格式：这种格式中，代码被放在不同的行，通过缩进来区隔不同代码块。
3. 拖沓型格式：这种格式中，代码被分散在不同的行，每行长度不统一，适合代码量大的项目。
4. 统一型格式：这种格式中，代码的所有部分都被放在同一行或者多行，且每行都保持相同宽度，适合适合团队内部开发。

下面给一个典型的格式化例子：

```
import java.util.*;

class Example{
  private String name;
  private Date dateOfBirth;

  public Example(String n, Date d){
    this.name = n;
    this.dateOfBirth = d;
  }
  
  public static void main(String[] args){
    Example e = new Example("John", new Date());
    System.out.println("Hello world");
  }
  
}
```
