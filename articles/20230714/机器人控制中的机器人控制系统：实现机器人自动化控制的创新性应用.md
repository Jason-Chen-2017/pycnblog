# 2.基本概念术语说明
## 2.1 概念及术语
### 2.1.1 机器人控制系统（Robot Control System）
机器人控制系统(Robot Control System)，又称作机器人运动控制系统(Motion Control Systems)或机器人操纵系统(Robot Manipulation System)。它由机器人装置及其控制器组成，负责将电信号转换为能够让机器人移动的运动信号并执行相应的动作。它的主要任务包括基于感知、推理和规划等方面的信息处理、处理、分析、控制、决策等功能。

机器人控制系统分为机械控制、电气控制、模式识别控制、神经网络控制、混合控制等几类。

### 2.1.2 机械控制
机械控制，又称离散控制，是一种最原始的控制方式。机械控制通常依赖于物理特性如质量、刚度等不可微分的特性。如用机械臂直接驱动关节转动、关节角度监测、正反馈控制等。

### 2.1.3 电气控制
电气控制，又称连续控制，是通过电路或信号的方式来完成运动控制的一种控制方法。电气控制通过转变电压或电流的大小对电机或驱动器发出指令，使得电机或驱动器转动到要求的位置或状态。电气控制中有关信号处理、编码、调制、采样等技术也十分重要。

### 2.1.4 模式识别控制
模式识别控制(Pattern Recognition Control，PRC), 是指利用人工智能技术，从收集到的大量数据的分析中找寻规律性和模式，以此来指导机器的运动行为。典型的PRC模型如Kalman滤波法、支持向量机、多层感知机。

### 2.1.5 神经网络控制
神经网络控制(Neural Network Control，NNC)，是指采用人工神经网络来模拟人的大脑神经元工作原理，通过学习训练过程中的经验数据，模仿人类的行为准则，达到机器智能化的目的。与传统的模式识别控制相比，NNC具有更高的灵活性、鲁棒性、适应性、自主性，在复杂环境下更具备优势。

### 2.1.6 混合控制
混合控制(Hybrid Control)，是指结合多个控制方式，例如机械控制、电气控制、模式识别控制、神经网络控制，共同协调优化机器人的动作策略。同时，还可以考虑采用信息融合、空间感知、机器学习等技术进行控制。

## 2.2 机器人控制系统的结构
机器人控制系统由如下结构所构成:
- （1）控制器（Controller），用来根据输入的感知信息、预期目标及内部模型计算输出命令给电机驱动器或其他动力系统。
- （2）运动模型（Motion Model），是一个对机器人行为进行建模、描述、模拟的一组动态规划方程或参数关系。
- （3）感知模块（Sensor Module），用于检测、接收外部环境信息、获取机器人周围环境状态。
- （4）决策与规划模块（Decision and Planning Module），负责从感知及运动信息中提取有效信息，形成运动指令，然后转交给控制器进行执行。
- （5）运动处理模块（Motion Processing Module），用于执行运动指令，对指令的实时处理和校准进行调整。
- （6）仿真模拟模块（Simulation Module），主要用于开发、测试及验证机器人控制算法。
- （7）通信模块（Communication Module），用于控制系统之间的数据交互。
- （8）错误处理模块（Error Handling Module），用于控制系统出现故障后的错误处理。

## 2.3 机器人控制系统的功能特点
### 2.3.1 灵活性、鲁棒性、可控性、自主性
机器人控制系统具有灵活性、鲁棒性、可控性、自主性等属性。

1. 灵活性：机器人控制系统可以灵活地适应不同的条件，即可以根据不同的物理状况或任务环境，选择相应的控制策略和控制方式。

2. 鲁棒性：机器人控制系统具有较高的鲁棒性。对于一些必然会发生故障的特殊情况，仍可以保持正常运转。

3. 可控性：机器人控制系统可以被精确地控制，可以快速响应反馈信息及执行相关动作。

4. 自主性：机器人控制系统具有高度的自主性，可以在不同的环境条件下独立运行。当需要的时候，机器人可以自己寻找路径，做出运动。

### 2.3.2 生物特征鲁棒性
机器人控制系统能够克服各种生物特征，比如身体部位的缺陷、触手、皮肤病等。例如，无论在哪里遇到危险，只要没有被破坏，机器人都可以安全地继续工作。
