# 2.基本概念术语说明
## 数据集（Dataset）
数据集是指用来训练或测试模型的数据集合。通常情况下，数据集包括训练集、验证集和测试集三个部分，分别对应着模型的训练阶段、调参阶段和最终评估阶段。

- 训练集（Training Set）：用于训练模型的样本数据。模型学习时，用训练集中的样本进行学习，并调整参数使模型达到最优效果。
- 验证集（Validation Set）：用于评估模型性能的样本数据。模型训练好后，对验证集中的样本进行评估，以确定模型是否过拟合或者欠拟合，并调整参数再次优化模型。
- 测试集（Test Set）：用于给最终结果评估的样本数据。模型训练完成之后，用测试集中的样本进行评估，得到最终结果。

数据处理流程即数据预处理过程中所执行的各个步骤，如下图所示：

其中：

1. 数据收集与处理：第一步是获取原始数据，然后进行数据预处理。数据预处理过程一般包括清洗、合并、分割、重组、重命名、过滤、重采样、降维、数据转换等。
2. 数据准备：第二步是对数据进行清洗、标准化、归一化、缺失值处理等。这一步是为了保证数据质量，避免出现异常值或无法处理的值。
3. 数据分析：第三步是进行数据探索和统计分析，以便更好的了解数据集。这通常包括画图、数据可视化、分析指标计算等。
4. 特征工程：第四步是利用数据进行特征选择和构造，以提升模型的效果。特征工程包括特征提取、特征选择、特征降维等过程。
5. 模型构建：第五步是构建机器学习模型，以实现数据的分类和预测。此处通常使用深度学习方法，例如CNN、RNN等。
6. 模型训练：第六步是使用训练集进行模型训练，将模型参数更新，以优化模型的精度。
7. 模型评估：第七步是使用验证集对模型进行评估，确认模型是否过拟合或欠拟合。如果需要的话，对模型进行调整，重新训练模型。
8. 模型推断与部署：第八步是将模型应用于测试集，对最终结果进行评估。部署模型可以让其他人或系统利用模型来进行预测或分类工作。

## 特征工程（Feature Engineering）
特征工程是一个迭代的过程，旨在对数据进行抽象、转换、选择、过滤等操作，以更好的满足模型构建的需求。特征工程通常包含以下几个方面：

1. 特征抽取：特征抽取是指根据已有的信息创建新的特征。例如，可以根据客户购买行为生成用户性别、年龄、城市等特征。
2. 特征转换：特征转换是指根据原始特征生成新特征，如将某些连续变量离散化、用指标表示某种分布等。
3. 特征选择：特征选择是指根据模型的目标和性能要求，选择特定模型相关的特征，消除无关的特征。
4. 特征过滤：特征过滤是指丢弃一些不必要的特征，减少噪声和冗余信息。
5. 特征降维：特征降维是指对高维数据进行降维，简化数据的空间表示，同时保持原始数据的内在含义。

## 缺失值处理
缺失值处理是指对数据集中的缺失值进行填充或删除，以避免模型训练时出现错误。常用的方式有：

1. 删除缺失值：直接删除缺失值占比比较大的特征；
2. 插补缺失值：用样本外数据进行插补，如用均值、中位数等进行替换；
3. 回归/预测缺失值：对缺失值的影响可能是由相邻样本决定的，因此可以通过回归/预测的方式进行填补；
4. 概率预测缺失值：使用贝叶斯估计法对缺失值进行预测。
