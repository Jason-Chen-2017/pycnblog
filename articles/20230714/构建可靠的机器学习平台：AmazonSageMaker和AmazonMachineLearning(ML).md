随着互联网、移动应用和物联网的发展，越来越多的数据、模型和服务正在从单体应用向分布式系统演进，需要一个统一的机器学习（ML）平台作为数据处理、建模和部署的中心枢纽。Amazon Web Services(AWS) 提供了 Amazon SageMaker 和 Amazon Machine Learning (ML) 服务作为机器学习平台，他们提供的能力包括：
- 数据集成与管理：Amazon S3 存储桶可以用来导入和保存数据，并与其它 AWS 服务的输出数据连接起来，使用标记语言如 Apache Parquet 来进行高效的数据格式转换。
- 自动训练和调优：Amazon SageMaker 可以利用各种机器学习框架，例如 TensorFlow、MXNet、PyTorch 和 Scikit-learn，来自动地对模型进行训练、优化和调优。它还可以利用深度神经网络来训练图像分类或文本分类任务，提供可扩展性和弹性。同时，它提供了超参数优化、训练和部署的监控指标，以及实时日志跟踪等功能。
- 模型评估和验证：Amazon SageMaker 可以通过流行的评估指标来衡量训练的模型质量，并针对不同类型的任务提供不同的模型选择建议。另外，它提供一系列模型打分机制来快速有效地识别模型中的错误，帮助开发者找到最适合业务场景的模型。
- 模型部署与预测：Amazon SageMaker 支持在生产环境中运行训练的模型，并可轻松地将其部署到 API Gateway 或 AWS Lambda 中进行推理，提供低延迟和高度可伸缩性。它还支持容器化模型，可以将模型和依赖项打包成易于使用的 Docker 镜像，这样就可以在任何环境下部署模型。
- 智能监控与跟踪：Amazon SageMaker 提供了一系列用于监控 ML 工作loads 的工具和组件，包括警报、日志记录和分析，并且可以与其它 AWS 服务和工具进行集成，如 CloudWatch Logs 和 AWS X-Ray，以提供更全面的运营洞察力。

总之，Amazon SageMaker 和 Amazon Machine Learning (ML) 为机器学习流程中的各个环节提供一站式的解决方案，助力企业降低运营成本，提升研发效率，提升产品质量，让客户获得更好的服务。

# 2.基本概念术语说明

## 2.1 什么是机器学习？
机器学习 (Machine Learning) 是指由人工智能技术引领的信息处理及自我学习的过程，以统计模型的方式来发现数据内在的模式并应用到新的任务和场景中，以取得成功。简单来说，机器学习就是让计算机能够自己学习、改善自身的性能。

## 2.2 什么是监督学习？
监督学习 (Supervised learning) 是机器学习的一个分支，它通过对已知的输入-输出样本对进行学习，建立一个模型，该模型可以对新输入的数据进行预测或分类。监督学习的目的是要找到一条从输入空间到输出空间的映射函数，即找到能够精确预测每个输入的输出值。监督学习的训练数据通常有标签，即样本所属的类别或者目标变量。如果训练数据没有标签，则称为无监督学习。

## 2.3 什么是无监督学习？
无监督学习 (Unsupervised learning) 是指机器学习中的一种方法，其主要目的是对未标记的数据集进行学习，使机器能够从数据中提取结构。无监督学习包括聚类、密度估计和关联分析等方法。无监督学习的训练数据通常没有标签，即它们没有明确的目的或类型信息。

## 2.4 什么是深度学习？
深度学习 (Deep Learning) 是机器学习的一种分支，它是一类通过多层抽象的神经网络来进行学习的算法。深度学习的关键是通过多个隐藏层来学习特征表示，而不是简单地用线性模型拟合数据。深度学习的应用主要涉及计算机视觉、自然语言处理、语音识别、强化学习、推荐系统等领域。

## 2.5 什么是 SageMaker?
SageMaker 是一种基于 AWS 的开源机器学习服务，它提供高级的机器学习功能，包括数据准备、模型训练、部署和端到端监控。SageMaker 可以简化机器学习工作流程，让数据科学家和开发人员更快、更容易地完成机器学习任务。SageMaker 使用户可以创建训练作业，提交给 SageMaker，然后等待训练完成。当训练完成后，可以直接部署为 RESTful API 或容器化应用程序。SageMaker 还包括用于自动模型训练和超参数优化的功能。SageMaker 可以监控部署的模型的质量，并及时检测异常情况。
