# 1.简介
  

最短路径算法在图论、电路设计等领域有着广泛的应用。给定一个图G=(V,E)及两个顶点s和t，求从s到t的最短路径。已知有许多不同类型的最短路径算法，如Dijkstra算法、Bellman-Ford算法、Floyd-Warshall算法等等。本文将介绍几种最短路径算法，并总结其特点、适用范围和局限性。

# 2.基本概念术语说明
1. 图（Graph）: 图是由一些结点(vertex)和连接这些结点的边所组成的结构。

2. 顶点（Vertex）: 图中的一个节点称为顶点。

3. 边（Edge）: 连接两个顶点的线段叫做边。

4. 权值（Weight）或代价（Cost）: 在图中，每条边都有一个权值，表示连接两个顶点之间的距离或者代价。

5. 源点（Source Vertex）: 从图中任意一个点出发的路径，称为源点到其他各个点的路径。

6. 目标点（Target Vertex）: 从图中某个顶点到其他所有顶点的路径，称为该顶点到目标点的路径。

7. 最短路径（Shortest Path）: 沿着一条最少弧的路径称为最短路径。

8. 无向图（Undirected Graph）: 如果两个顶点间没有方向性，则称为无向图。例如，在社交网络中，每个用户都可以以自己喜欢的方式连接到其他用户，即两者之间没有明确的方向性。

9. 有向图（Directed Graph）: 如果图中存在方向性，则称为有向图。例如，在航空管制中，某些航班只能从一地到另一地，不能相反。

10. 无环图（Acyclic Graph）: 如果不存在从源点可达且不可逆的边，则称为无环图。无环图一定是树结构。

11. 负权值边（Negative Weight Edge）: 有时图中会出现负权值的边，表示从某一顶点到另一顶点的距离减去这个权值，得到的值可能是一个正值或者零。

12. DAG（Directed Acyclic Graph）: 有向无环图。如果不存在反向边，则称之为DAG。例如，一个工程项目有若干子工程，子工程完成后才能开始下一个工程。

# 3.核心算法原理和具体操作步骤以及数学公式讲解
1. Dijkstra算法: 是一种贪心算法。它用于计算某一节点到其余各节点的最短路径，时间复杂度为O(|E|+|V|log|V|)。该算法可以解决无向图和带权重的无向图。算法流程如下：

　　　　(1). 将源点s加入已处理集合S。

　　　　(2). 初始化源点s的距离为0，其余各点距离为无穷大。

　　　　(3). 从源点s出发，对未处理的点进行排序，按距离s到该点的距离进行排序。

　　　　(4). 选取距离s最近的未处理点q。

　　　　(5). 对未经过的直接连通的顶点r及其距离为di的边进行松弛。

　　　　　　　　(a). 如果q的距离+di比r当前的距离更小，则更新r的距离。

　　　　　　　　(b). 更新r的前驱。

　　　　(6). q加入已处理集合S。

　　　　(7). 当未处理集合为空时结束算法。

2. Bellman-Ford算法：该算法用于计算某一节点到其余各节点的最短路径。时间复杂度为O(|E|*|V|),当图中存在负权回路时，算法可能陷入无穷循环。算法流程如下：

　　　　(1). 初始化源点s的距离为0，其余各点距离为无穷大。

　　　　(2). 对每个边v->u进行 |V|-1 次松弛操作，使得所有的边最多经过一次，若经过多次，则说明存在负权回路。若没有负权回路，则算法终止。

　　　　(3). 求出从源点s到各点的最短路径。

3. Floyd-Warshall算法：Floyd-Warshall算法是一种动态规划算法，用来计算任意两点之间的最短路径。时间复杂度为O(|V|^3)。该算法适用于所有带权图，包括带负权值的图，但当图中存在负权回路时，算法可能陷入无穷循环。算法流程如下：

　　　　(1). 初始化源点s到其他各点的距离矩阵D。

　　　　(2). 对于每一对点p,q，若存在一条从p到q的路径，则更新D[i][j]的值，其中i<=j。

　　　　(3). 返回D。
