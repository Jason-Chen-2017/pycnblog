# 2.核心概念与联系

在讨论MySQL性能模式之前，我们需要了解一些核心概念。这些概念包括：

- 索引：索引是一种数据结构，用于加速数据库查询。MySQL支持多种类型的索引，如B-树索引、哈希索引等。
- 查询优化：查询优化是指MySQL在执行查询时，根据一定的规则和算法，选择最佳的执行方案。这些规则和算法包括查询的解析、优化和执行等。
- 缓存：缓存是一种存储数据的方法，用于提高应用的性能。MySQL支持多种类型的缓存，如查询缓存、表缓存等。

这些概念之间有很强的联系。例如，索引和查询优化是密切相关的，因为索引可以帮助MySQL更快地找到数据，从而提高查询性能。同样，缓存和查询优化也有密切的联系，因为缓存可以帮助MySQL避免不必要的查询，从而提高性能。
