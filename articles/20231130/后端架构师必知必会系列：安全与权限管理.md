# 2.核心概念与联系
在讨论安全与权限管理之前，我们需要了解一些核心概念。这些概念包括：

- 身份验证（Authentication）：身份验证是确认用户是谁的过程。通常，身份验证涉及到用户名和密码的比较。
- 授权（Authorization）：授权是确定用户是否有权访问特定资源的过程。授权涉及到用户的角色和权限。
- 密码学：密码学是一种数学学科，主要研究加密和解密的方法。密码学在身份验证和授权中发挥着重要作用。
- 加密：加密是将信息转换为不可读形式的过程，以保护信息的安全。加密技术在身份验证和授权中广泛应用。
- 会话管理：会话管理是一种机制，用于跟踪用户在系统中的活动。会话管理涉及到会话的创建、维护和终止。

这些概念之间的联系如下：

- 身份验证和授权是安全与权限管理的基本组成部分。身份验证确认用户是谁，而授权确定用户是否有权访问特定资源。
- 密码学在身份验证和授权中发挥着重要作用。例如，密码学的一种技术是密码哈希，可以用于存储用户的密码。
- 加密在身份验证和授权中广泛应用。例如，加密可以用于保护用户的密码和其他敏感信息。
- 会话管理与身份验证和授权密切相关。会话管理用于跟踪用户在系统中的活动，从而确保安全和权限管理的有效性。

    # 将加密后的密码与用户名进行比较

        return True
    else:
        return False

def authorize(username, resource):
    # 确定用户是否有权访问特定资源
    if username == "admin" and resource == "sensitive_data":
        return True
