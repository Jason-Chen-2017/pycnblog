# 2.核心概念与联系

## 2.1 Web框架

Web框架是一种软件框架，它提供了一种结构化的方式来开发Web应用程序。Web框架通常包括以下几个组件：

- 模型（Model）：用于存储和管理应用程序数据的组件。
- 视图（View）：用于显示应用程序数据的组件。
- 控制器（Controller）：用于处理用户请求和管理模型和视图的组件。

Web框架通常提供了一种更加简洁和可扩展的方式来开发Web应用程序，它们通常包括一些预先定义的组件和功能，开发人员可以根据需要进行扩展和修改。

## 2.2 Spring MVC

Spring MVC是Java中一个非常重要的Web框架，它是基于Spring框架的MVC架构实现的。Spring MVC提供了一种更加灵活和可扩展的方式来开发Web应用程序。Spring MVC的主要组件包括：

- 控制器（Controller）：用于处理用户请求和管理模型和视图的组件。
- 模型（Model）：用于存储和管理应用程序数据的组件。
- 视图（View）：用于显示应用程序数据的组件。
- 处理器拦截器（HandlerInterceptor）：用于在控制器方法之前或之后执行一些额外的操作的组件。
- 处理器适配器（HandlerAdapter）：用于将请求转发到控制器方法的组件。
- 视图解析器（ViewResolver）：用于解析视图名称并将其转换为实际的视图对象的组件。

Spring MVC的核心原理是基于MVC设计模式实现的，它将应用程序的控制器、模型和视图组件分离，从而实现了更加灵活和可扩展的开发方式。
