# 2.核心概念与联系

设备驱动程序是操作系统与硬件设备之间的桥梁，它负责将硬件设备的操作指令转换为操作系统可以理解的数据格式，并将操作系统发送给硬件设备的数据转换为硬件设备可以理解的格式。

设备驱动程序可以分为两类：内核模式驱动程序和用户模式驱动程序。内核模式驱动程序是操作系统内核的一部分，它具有最高的系统权限，可以直接访问硬件设备。用户模式驱动程序则运行在用户空间，它需要通过系统调用来访问硬件设备。

设备驱动程序的核心概念包括：硬件抽象层（HAL）、驱动程序框架、设备树、中断处理、DMA传输、内存管理等。这些概念将在后续的内容中详细解释。
