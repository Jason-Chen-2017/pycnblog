# 2.核心概念与联系
在本书中，作者提到了一些核心概念，如模块化、组件化、可扩展性、可维护性等。这些概念是软件架构设计的基础，我们需要理解它们的含义和作用。

模块化是指将软件系统划分为多个模块，每个模块负责完成一定的功能。模块化可以帮助我们更好地组织代码，提高代码的可读性和可维护性。

组件化是指将软件系统划分为多个组件，每个组件可以独立部署和维护。组件化可以帮助我们更好地实现软件系统的可扩展性和可维护性。

可扩展性是指软件系统的能力，可以在不影响其他部分的情况下，增加或减少功能和性能。可扩展性是软件系统设计的一个重要目标，因为它可以帮助我们更好地应对未来的需求变化。

可维护性是指软件系统的能力，可以在不影响其他部分的情况下，进行修改和更新。可维护性是软件系统设计的另一个重要目标，因为它可以帮助我们更好地应对 bug 和需求变化。

在本书中，作者还提到了一些关于软件架构设计的核心原则，如“单一职责原则”、“开放封闭原则”等。这些原则是软件架构设计的基础，我们需要理解它们的含义和作用。

单一职责原则是指一个模块或组件只负责完成一个功能。这个原则可以帮助我们更好地组织代码，提高代码的可读性和可维护性。

开放封闭原则是指一个模块或组件可以被扩展，但不能被修改。这个原则可以帮助我们更好地实现软件系统的可扩展性和可维护性。
