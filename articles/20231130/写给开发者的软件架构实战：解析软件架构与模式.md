## 4.1模块化设计模式

### 4.1.1代码实例

```python
# 用户模块
class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def get_info(self):
        return "Name: %s, Age: %d" % (self.name, self.age)

# 订单模块
class Order:
    def __init__(self, user, items):
        self.user = user
        self.items = items

    def get_total(self):
        total = 0
        for item in self.items:
            total += item.price * item.quantity
        return total

# 商品模块
class Item:
    def __init__(self, name, price, quantity):
        self.name = name
        self.price = price
        self.quantity = quantity

    def get_price(self):
        return self.price * self.quantity
```

### 4.1.2详细解释说明

在这个代码实例中，我们将一个电商系统划分为多个模块，如用户模块、订单模块和商品模块。每个模块都负责完成特定的功能。例如，用户模块负责处理用户的信息，订单模块负责处理订单的信息，商品模块负责处理商品的信息。通过将系统划分为多个模块，我们可以更好地组织代码，提高代码的可读性和可维护性。

## 4.3设计模式

### 4.3.1代码实例

```python
# 单例模式
class Singleton:
    _instance = None

    @staticmethod
    def get_instance():
        if Singleton._instance is None:
            Singleton._instance = Singleton()
        return Singleton._instance

# 工厂模式
class Factory:
    @staticmethod
    def create_user(name, age):
        return User(name, age)

    @staticmethod
    def create_order(user, items):
        return Order(user, items)

    @staticmethod
    def create_item(name, price, quantity):
        return Item(name, price, quantity)

# 观察者模式
class Observer:
    def update(self, subject):
        pass

class Subject:
    def __init__(self):
        self.observers = []

    def attach(self, observer):
        self.observers.append(observer)

    def detach(self, observer):
        self.observers.remove(observer)

    def notify(self):
        for observer in self.observers:
            observer.update(self)

# 使用设计模式
user = Factory.create_user("John", 20)
order = Factory.create_order(user, [Factory.create_item("iPhone", 1000, 1)])
subject = Subject()
subject.attach(user)
subject.attach(order)
subject.notify()
```

### 4.3.2详细解释说明

在这个代码实例中，我们使用了单例模式、工厂模式和观察者模式来解决软件系统的设计问题。

单例模式是一种确保一个类只有一个实例的设计模式。在这个例子中，我们使用了单例模式来确保Singleton类只有一个实例。

工厂模式是一种创建对象的设计模式。在这个例子中，我们使用了工厂模式来创建用户、订单和商品对象。

观察者模式是一种将多个对象绑定到一个主题对象上，以便这些对象可以在主题对象发生变化时自动更新的设计模式。在这个例子中，我们使用了观察者模式来实现用户和订单之间的通信。

通过使用这些设计模式，我们可以更好地组织代码，提高代码的可读性和可维护性。
