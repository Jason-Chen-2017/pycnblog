# 2.核心概念与联系

安全认证与访问控制是一种保护计算机系统资源的方法，它涉及到用户身份验证、授权和访问控制。在这个过程中，我们需要考虑以下几个核心概念：

- 身份验证：用户向系统提供凭据以证明他们是谁。这通常包括密码、个人身份信息（如社会安全号码）或其他类型的证书。
- 授权：系统根据用户的身份验证结果，决定用户是否可以访问某个资源。这可能包括读取、写入、删除等操作。
- 访问控制：系统根据用户的授权结果，控制用户对资源的访问。这可能包括基于角色的访问控制（RBAC）、基于属性的访问控制（ABAC）等。

```python
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.asymmetric import rsa

# 生成密钥对
private_key = rsa.generate_private_key(
    public_exponent=65537,
    key_size=2048,
    backend=default_backend()
)

public_key = private_key.public_key()

# 生成数字签名
message = b'Hello, World!'
signature = private_key.sign(
    message,
    padding.PSS(
        mgf=padding.MGF1(hashes.SHA256()),
        salt_length=padding.PSS.MAX_LENGTH
    ),
    hashes.SHA256()
)

# 验证数字签名
try:
    public_key.verify(
        signature,
        message,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        ),
        hashes.SHA256()
    )
    print('数字签名验证成功')
except ValueError:
    print('数字签名验证失败')
```
