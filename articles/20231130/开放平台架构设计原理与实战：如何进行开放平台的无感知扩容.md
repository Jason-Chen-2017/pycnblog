# 2.核心概念与联系

在开放平台架构设计中，核心概念包括：

1. 负载均衡：负载均衡是指将请求分发到多个服务器上，以提高系统性能和可用性。常见的负载均衡算法有：轮询、随机、权重等。

2. 扩容：扩容是指在系统负载增加时，动态地增加服务器资源，以保持系统性能稳定。扩容可以通过增加服务器数量、增加服务器资源等方式实现。

3. 无感知扩容：无感知扩容是指在系统扩容过程中，用户和应用程序对系统性能的变化不能感知到。无感知扩容是开放平台架构设计的一个重要目标。

4. 自适应扩容：自适应扩容是指在系统负载变化时，动态地调整服务器资源，以保持系统性能稳定。自适应扩容需要实时监控系统负载，并根据负载变化调整服务器资源。

5. 高可用性：高可用性是指系统在任何时候都能提供服务。高可用性是开放平台架构设计的一个重要目标。

6. 扩容策略：扩容策略是指在系统扩容过程中，采用的扩容方法和策略。扩容策略包括：预先扩容、渐进式扩容等。
