# 2.核心概念与联系
在讨论如何实现高可用性之前，我们需要了解一些核心概念。首先，我们需要了解什么是高可用性，以及如何衡量高可用性。高可用性是指数据库系统在满足所有业务需求的同时，能够保持稳定运行的能力。这意味着数据库系统应该能够在出现故障时，尽可能快地恢复并继续提供服务。

为了实现高可用性，我们需要了解一些关键概念，如主从复制、集群、冗余等。主从复制是一种数据库复制方法，它允许我们将数据库中的数据复制到另一个数据库实例上，以便在主数据库出现故障时，从数据库实例可以继续提供服务。集群是一种数据库部署方式，它允许我们将多个数据库实例组合在一起，以便在一个实例出现故障时，其他实例可以继续提供服务。冗余是一种数据保护方法，它允许我们在多个数据库实例上保存相同的数据，以便在一个实例出现故障时，其他实例可以继续提供服务。
