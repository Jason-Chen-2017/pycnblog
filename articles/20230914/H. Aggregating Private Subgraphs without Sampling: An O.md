# 1.简介
  

在大数据分析、广告推荐、网络安全等领域，大量私密数据的处理成为当前研究热点。如何有效地对私密数据进行建模、分析并推导出有价值的信息，是一项关键性任务。目前，多种私密子图聚合方法被提出用于解决这一问题。然而，这些方法均存在着较大的计算复杂度和高效率低下的聚合性能。为了减少计算量和提升聚合性能，本文针对此类方法进行了优化。

基于私密子图聚合技术，作者首先将原始的数据集分割成许多小的、可私密共享的子集，然后再聚合各个子集的结果以获取有意义的信息。目前，大多数私密子图聚合方法采用的方式都是随机抽样法（即每次选择一个子集进行聚合），这种方式存在着较高的聚合代价。因此，作者提出了一种新的方法——无需随机抽样，而是根据原始数据集中各个子集之间的依赖关系，通过在聚合过程中动态确定要聚合哪些子集，从而提升聚合性能。

除此之外，作者还设计了一个名为PALE（Private and Local Graph Embedding）的方法，该方法旨在利用私密子图聚合和本地嵌入技术来降低匿名性并改善聚合质量。通过采用强化学习（RL）的方式，作者可以训练模型以自动选择最优的聚合策略，使得聚合结果更加准确。最后，作者还比较了三种经典的私密子图聚合算法，证明了作者的方法的有效性和优越性。

本文总结了私密子图聚合相关的发展历史、基本概念及其定义，进而详细阐述了作者设计的无需随机抽样、优化聚合策略、PALE方法、RL方法以及实验评估等方法。此外，作者还给出了对这些方法的深入分析、未来的发展方向、以及与相关工作的比较和评估等。文章内容丰富且具有深度，文章结构清晰，语言流畅易懂，既有理论也有实践性，是一个十分有益的技术博客文章。


# 2.背景介绍

私密子图聚合（private subgraph aggregation）是指利用私密数据集中的信息提取有价值的模式和知识。其核心目的是将不同群体（比如，用户）间共享相同的私密数据拆分成多个相互独立的子集，并基于每个子集的数据进行分析。私密子图聚合的主要应用场景包括社交网络分析、文本分类、舆情监测、金融交易数据分析、图像搜索、生物信息学，以及其他类似的领域。

私密子图聚合相关的研究已经在多年前就开始了。2017年，一个叫做“Scaffolding”的研究团队发表了一篇名为《Scaffolds as a Service: Leveraging Personal Information for Private Data Analysis》的论文，提出了基于 scaffold 的私密数据聚合方法。其中，scaffold 是一种小型的、可私密共享的数据结构，由一些公开数据组成，其目的就是能够帮助数据持有者快速了解自己所拥有的私密数据。2019 年的一篇论文《A Survey on Privacy-Preserving Subgraph Aggregation》则对现有的私密子图聚合方法进行了综述。随后，随着 Big Data 和 AI 技术的兴起，越来越多的人们开始关注这个领域。

基于 scaffold 的私密数据聚合方法可以有效地实现对私密数据进行建模、分析并推导出有价值的信息。但是，这些方法存在着不足之处。首先，它们需要用户隐私保护措施，即不要透露任何有关个人信息的数据。其次，它们生成的模型通常需要大量的计算资源才能运行，而对于用户来说可能又不便宜。第三，这些方法只能提供粗糙的结果，因为很多私密数据其实是高度关联的。

于是，私密子图聚合方法的研究者们开始寻找另一种解决方案。他们考虑到私密数据之间往往存在依赖关系，所以可以基于某种模型或规则，来确定应该聚合哪些私密子集。这些方法被称为无需随机抽样的聚合方法。

目前，私密子图聚合相关的研究主要分为两大类。第一种类别是基于机器学习和优化的方法，如“Graph Convolutional Networks with Privacy Preservation”、“ReGraph: A Library for Reproducing Graph Algorithms in an Anyonymous Environment”。第二种类别则是直接聚合私密子集，如“Private & Local Graph Embeddings”和“Subgraph Fusion: Combining Multiple Subsets to Improve Performance”。但是，仍有许多工作没有被充分调查和完善。例如，无需随机抽样的聚合方法是否可以真正地提升聚合性能，以及如何设计相应的模型、训练策略。

最近几年，一些研究人员开始提出了基于强化学习（RL）的私密子图聚合方法，如“PrivGAN: Protecting User Privacy by Generative Adversarial Network via Attribute Disentanglement”，它利用 RL 对不同参与方进行博弈，以获取私密数据集中的有价值信息。作者认为，通过对私密子图聚合过程进行细致的设计，无需随机抽样的聚合方法可以在保证数据隐私的同时提升聚合性能。

# 3.基本概念术语说明

## 3.1 数据集划分

假设原始数据集共有 n 个记录，每个记录都有一个唯一标识符 id ，它可以用来索引到整个数据集中每条记录。

为了保护用户的隐私，一般会把原始数据集切分成多个相互独立的私密子集。这里的独立指的是子集内不包含任何记录，两个不同的子集甚至都不能共用同一条记录。因此，如果原始数据集有 m 个私密子集，那么其组合数目为 C(n+m−1,m)。

私密子集可能会与原始数据集的大小和分布有很大关系。由于无法直接访问原始数据集，因此需要建立一种机制，让所有参与者可以自主地选择自己感兴趣的私密子集，从而在不泄露个人隐私的情况下构建模型。

## 3.2 私密子图

如果有 k 个私密子集，那么对于任意两个私密子集 $S_i$ 和 $S_j$ ，都存在一个公开数据集 $R$ 和一个私密数据集 $D_i$ 。对于 $|S_i\cap S_j|$ 中的某个 $n \in N^*_{max}$ 条记录 $r \in R$ ，它的私密子图 $G_i(r)$ 可以定义如下：

$$ G_i(r) = (v,e)|V|=|\{u : v \in V,\{(u,v): e\in E\}\cap S_i\neq \emptyset\}|,$$ 

其中，$V$ 为 $r$ 的邻居节点集合；$\cup_{\forall u} (\{u,v:e\in E\}\cap S_i)=\emptyset$ 表示 $r$ 不属于 $S_i$ 或 $S_i$ 中没有边。

定义子图 $G(r)$ 为所有私密子图 $G_i(r)$ 的并。

## 3.3 有向图

对于有向图，我们定义 $G=(V,E)$ 为 $|V|=N$ 个节点和 $|E|=M$ 个边的有向图，表示为 $(v_1,e_1),(v_2,e_2),...,(v_N,e_M)$ 。

对于任何 $v \in V$ ，其邻居节点集合 $N(v)$ 可以定义如下：

$$ N(v) := \{u : e=(v,u)\in E\}$$ 

其中，$e$ 是指向 $u$ 的边。

## 3.4 无向图

对于无向图，我们定义 $G=(V,E)$ 为 $|V|=N$ 个节点和 $|E|=M/2$ 个边的无向图，表示为 $(v_1,e_1),(v_2,e_2),...,(v_N,e_M)$ 。

对于任何 $v \in V$ ，其邻居节点集合 $N(v)$ 可以定义如下：

$$ N(v) := \{u : e=min\{w\}(v,u)\in E\cup\{w\}(u,v)$$ 

其中，$e=\{(u,v),(v,u)\}$ 是指向 $u$ 的边。如果不存在这样的边，则令 $N(v)=\{v\}$ 。
