# 1.简介
  

## SQL注入(英语:SQL injection)
　　SQL注入，也叫SQL拼接攻击，是一种计算机安全漏洞。它允许恶意用户输入恶意的代码或函数，通过利用漏洞，达到非法获取数据库服务器权限，篡改数据、删除数据甚至获取更高级的系统访问权。所有支持SQL语言的应用程序都可能被攻击者利用该漏洞。

　　一个典型的SQL注入攻击包括以下几步：

　　1.攻击者将恶意的SQL代码插入到表单字段中；

　　2.服务器端接收请求后解析SQL代码并执行，导致正常用户无法预测或控制的内容被执行；

　　3.由于没有对输入进行有效过滤或转义处理，导致攻击者能够轻易地添加、修改或删除数据库中的敏感信息，甚至执行任意的系统命令。

　　一般来说，攻击者并不能准确预测出网站上每个输入框所对应的SQL查询条件。但是，可以通过一些技巧和工具（例如sqlmap）来自动化地检测和exploit这些漏洞。

## 概念
　　从本质上说，SQL注入就是利用SQL语法错误或者缺陷，以某种方式构造一个特殊的查询语句，在不经过授权的情况下，将其提交给应用层，最终实现非法控制数据库内容。如果攻击者精心设计了恶意的SQL注入攻击，就可以获取到系统管理员完全无需授权的权限。因此，SQL注入攻击可以说是任何系统安全漏洞中最危险的一种，也是黑客使用最有效的攻击手段之一。

　　正因如此，服务器端的软件需要严格检查所有输入的数据是否符合预期。对于SQL注入攻击来说，通常攻击者会尝试构造特殊的SQL查询语句来绕过数据库的输入验证机制。在正常情况下，输入的数据只能由指定的数据类型和格式构成。但是，通过SQL注入攻击，攻击者往往可以将原始输入的数据转换为合法SQL语句，然后通过特定的字符替换、注释掉等方式，将它们作为查询条件提交给数据库，最终获得受害者数据的所有权限。

　　　　由于大多数Web应用框架都是基于ORM技术构建的，因此安全性很容易遭到破坏。除了ORM框架外，大多数开源PHP框架中也存在SQL注入漏洞。攻击者可以通过构造恶意的SQL注入攻击，窃取或者篡改数据的重要信息。

　　　　通过SQL注入攻击，攻击者可以执行如下操作：

　　　　1.盗取或修改数据；

　　　　2.执行系统管理任务；

　　　　3.破坏数据库完整性；

　　　　4.执行任意代码。

　　因此，安全防护应首先保证数据安全，防止数据泄露、篡改、损坏等情况的发生。同时，还应采取措施检测和识别SQL注入攻击行为，采用参数绑定或者输入输出转义等措施减少SQL注入的风险。另外，应用更新版本、维护软件补丁和系统设置，始终保持系统最新版本并定期备份数据，也是为了最大限度地降低SQL注入的影响。
