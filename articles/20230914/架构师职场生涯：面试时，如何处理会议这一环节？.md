# 1.简介
  

本文将以面试官视角进行剖析架构师面试中经常碰到的“会议”问题。文章不会对架构师职场生态、技术栈及流程做详细介绍，只针对当前热门技术架构师面试中的“会议”问题进行了分析讨论。

# 2.什么是会议?
在我国，我们一般把参加工作或者实习或者毕业旅行等各种活动叫做“培训”，但如果要指导或组织一个“会议”，通常指的就是由管理者主持、带领人员参加的某种活动，例如：举行公司内部的业务会议、产品研发团队的分享会、技术团队的培训会议等。

# 会议为什么重要?
1.沟通
   会议是通信工具，可以有效促进各方之间的沟通，比如需求、项目计划、质量保证、人才选拔、组织架构调整等。
2.决策
   会议可以使得大家对某个议题达成共识并形成决策，在不同观点之间提供信息，增强决策过程中的多样性。
3.共同关心的事情
   会议往往就一些比较重要的事情如项目管理、质量保证等进行，避免无谓的争吵，也能集体认识到自己的价值观和方向，更好地共同前进。


# 3.如何应对“会议”的问题?
会议是一个复杂的事务，面试官很难准确回答“你的方案是否考虑到会议开销？”，但为了应对这种情况，面试官需要从以下几个方面提升自我应变能力:

1.团队建设
  - 首先，需要确保参会人员对相关知识和技能有一定的了解；
  - 其次，需要指导参会人员进行知识储备和相关的专业技能培养；
  - 第三，建议面试官对参会人员进行综合素质考核，评估他们对所参与的项目、业务和市场的理解、掌握程度，为他们安排相应的培训或指导工作。

2.制定工作计划和日程表
  - 提前预计可能出现的会议，及时通知参会人员，让参会人员做好准备；
  - 将会议安排在较忙的时间段，避免影响工作进展；
  - 在每天晚上，组织每个团队成员回顾和总结自己的工作，鼓励他们参加小组内的其他会议，增强团队凝聚力；
  - 不要迷信会议，而是要根据实际情况进行灵活分配和安排，确保工作按计划顺利进行。


3.对会议给出的建议及要求作出回应
  - 当面对会议上的议题和问题，应该表现出真诚的关心和耐心，以帮助提升自己；
  - 如果参会人员不赞同你的意见，请充分肯定他们的理解和能力，并且请教他们有关该议题的更多信息，让他们能够提供更好的建议；
  - 最后，对参会人员给出的建议和指导，保持开放心态，将这些建议和建议背后的原因分享给所有参会人员，希望大家可以听取并吸收学习，提升自己。
