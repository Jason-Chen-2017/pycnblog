# 1.简介
  

性能测试(Performance Testing)一直都是互联网应用开发过程中的重要环节。对于任何一个项目而言，尤其是互联网企业项目，都需要充分考虑到用户的体验、系统的响应时间和资源利用率等指标。常用的性能测试工具主要包括JMeter、LoadRunner、PerfMon等。由于市场上还有很多性能测试工具，比如Web Load Generator、ApacheBench、Apache Grinder等，所以本文将重点介绍两个性能测试工具：Apache JMeter 和 Load Runner。

# Apache JMeter
Apache JMeter是开源的基于Java的最流行的开源性能测试工具之一。它可以用来对各种类型负载生成高负载、多线程甚至分布式的测试计划，并分析出产生的结果。它的图形界面提供了直观的操作方式，适合用于复杂的性能测试场景。


如上图所示，JMeter的主要工作窗口包括：

1. Test Plan（测试计划）- 此处可配置HTTP请求的各项参数，同时也可配置不同的线程数、循环次数、定时器、数据处理、控制器等；
2. Thread Group（线程组）- 此处配置单个线程或线程组运行的相关信息；
3. Controller（控制器）- 可配置控制线程组中各个线程之间的数据交换、条件判断、持续时间等；
4. Samplers（取样器）- 负责生成测试数据，如CSV文件读取器、读写器、FTP/HTTP请求、数据库查询等；
5. Listeners（侦听器）- 可以在测试过程中实时获取系统信息、监控性能指标等；
6. Results Tree（结果树）- 在测试结束后，显示测试结果的摘要信息和详细统计信息。

除了图形化界面外，JMeter还提供了命令行的接口，通过脚本的方式执行性能测试，能够更好的满足不同场景下的需求。

# Load Runner
Load Runner是一个专门针对Oracle数据库的性能测试工具。它提供了丰富的功能和设置选项，能模拟真实用户的访问行为，精确测量系统资源的消耗。Load Runner提供各种类型的测试计划，如吞吐量测试、响应时间测试、事务响应时间测试等。它的图形化界面简洁易用，适合用于简单场景的性能测试。


如上图所示，Load Runner的主要工作窗口包括：

1. Summary View（概览视图）- 此处可查看测试计划、服务器资源、报表和设置信息；
2. Analysis View（分析视图）- 此处可查看性能指标数据的变化曲线、统计信息及警告信息；
3. Config View（配置视图）- 此处可进行性能测试的设置、调整和保存；
4. Graph View（图形视图）- 此处可实时显示系统资源的消耗情况；
5. Monitors（监视器）- 此处可以实时监控系统资源占用、CPU、内存、网络等状况。

除此之外，Load Runner还提供了命令行的接口，可以通过脚本自动化执行性能测试，可以更加灵活地管理测试环境和数据源。

# 为什么选择Apache JMeter ？
Apache JMeter的特点如下：

1. 流畅的操作界面：JMeter 的图形界面操作起来很容易上手，但却不失灵活性，能支持复杂的负载测试配置。
2. 支持丰富的协议类型：包括 HTTP、HTTPS、FTP、JDBC、LDAP、TCP、UDP 等。
3. 大量插件扩展：JMeter 提供了强大的插件扩展机制，能轻松实现定制化需求。
4. 跨平台兼容性好：JMeter 完全支持多种平台，如 Windows、Linux、MacOS、Solaris、AIX等。
5. 详细的测试报告：JMeter 有内置的 HTML 报告，并且提供了丰富的图表展示，帮助我们更快地理解测试结果。
6. 灵活的可编程接口：JMeter 提供了一系列的 API ，方便进行脚本化、批量测试。
7. 自动化测试：JMeter 提供了多种方式可以实现自动化测试，包括 HTTP 参数化、事务测试、自定义断言、脚本驱动等。

总结来说，Apache JMeter 是性能测试领域中功能最全面的测试工具之一。如果你想快速掌握性能测试的知识，Apache JMeter 是你的首选。

# 为什么选择 Load Runner ？
相比 Apache JMeter ，Load Runner 更注重于 Oracle 数据库的性能测试。它的特点如下：

1. 直观的测试结果：Load Runner 的测试结果图表清晰直观，能直观地看到各项指标的变化曲线。
2. 操作友好的测试计划编辑器：Load Runner 的测试计划编辑器具有高效的图形编辑功能，让用户无需编写代码即可创建复杂的测试计划。
3. 支持多种测试用例类型：Load Runner 提供了丰富的测试用例类型，包括吞吐量测试、响应时间测试、事务响应时间测试、错误检测测试等。
4. 易用的 SQL 模板编辑器：Load Runner 提供了 SQL 模板编辑器，能根据 SQL 请求模板自动生成多条 SQL 请求，有效减少手动输入 SQL 语句的时间。
5. 更好的内存和 CPU 使用情况：Load Runner 会根据硬件资源的限制自动调整测试计划，降低测试环境对机器的压力。
6. 灵活的历史记录查询能力：Load Runner 提供了查询历史测试结果的能力，帮助管理员回溯性能测试的历史数据。
7. 更广泛的兼容性：Load Runner 支持 Linux、Windows、macOS、Solaris、AIX 等平台，能更好地满足部署环境的要求。

总结来说，Load Runner 是面向 Oracle 数据库的专业性能测试工具。如果你需要针对 Oracle 数据库进行性能测试，建议使用 Load Runner 。
