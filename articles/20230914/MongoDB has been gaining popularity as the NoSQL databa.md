# 1.简介
  

NoSQL（Not Only SQL）数据库指的是非关系型数据库，其特点在于数据不仅存储在关系型数据库中，而且也可以存储在非关系型数据库中。NoSQL数据库有利于提高数据管理、应用程序开发效率、可伸缩性、易扩展性等诸多优点。

而MongoDB是一种NoSQL数据库，它是基于分布式文件系统的开源数据库。它的优点有：

1.灵活的数据模型：支持丰富的数据类型，如字符串、整数、浮点数、日期时间、数组、对象等；

2.自动分片：可以横向扩展到多个服务器上，进行数据的分布式存储和处理；

3.动态查询：提供灵活的查询语法，使得开发人员能够快速检索、过滤、排序数据；

4.高性能查询：通过索引机制实现快速查询，并支持对海量数据进行分页和排序操作；

5.ACID事务支持：保证事务执行的原子性、一致性和隔离性，确保数据完整性。

然而，MongoDB也存在一些缺陷：

1.缺乏完整的ACID兼容性：当前版本的MongoDB仍处于开发阶段，不完全支持ACID事务，例如更新和删除操作不能回滚。

2.慢查询性能：由于设计时没有充分考虑查询性能，导致了查询速度较慢。

因此，在实际应用场景中，需要根据需求选择合适的数据库，比如，对于具有复杂事务处理要求的业务系统，建议使用Oracle或PostgreSQL等传统的关系型数据库。而对于需要处理海量数据的灵活查询、实时搜索和分析的业务系统，则推荐使用MongoDB作为NoSQL数据库。

# 2.相关概念及术语
## 2.1 文档(Document)
文档是一个记录或者实体，它由字段及其值组成。一个文档类似于关系型数据库中的一条记录。

举例来说，一个文档可能包含一个名字、年龄、地址、电话号码等信息。

## 2.2 集合(Collection)
集合是一个类似于表格的容器，用来存放文档。每个集合由一个或多个文档构成，集合中的所有文档都有相同的结构，并且可以通过特定键来标识唯一的文档。一个集合类似于关系型数据库中的表。

## 2.3 数据库(Database)
数据库是一个逻辑上的概念，它包含了一系列的集合。数据库提供了对集合的操作接口，通过这个接口可以创建、修改、删除集合及其对应的文档。

## 2.4 客户端(Client)
客户端就是连接MongoDB数据库的应用。客户端可以运行在本地机器或者远程计算机上，通过网络访问数据库。

## 2.5 主键(Primary Key)
主键是一个文档字段，它唯一标识了一个文档。主键用于确定文档的位置。在MongoDB中，主键会自动生成，默认使用"_id"作为主键。

## 2.6 索引(Index)
索引是一个帮助数据库更快地找到集合中满足查询条件的文档的数据结构。索引是特殊的数据结构，它存储着指向文档的指针，加速了数据查找的过程。在MongoDB中，索引可以帮助数据库加速查询。

## 2.7 分片(Sharding)
分片是一个将大型集合分割成多个小集合的过程。在分片过程中，MongoDb将各个小集合分布在不同的服务器上，从而达到数据水平拆分的效果。每台服务器存储部分分片的数据，这样可以有效降低单机硬件的负担。另外，当数据量很大时，MongoDb还可以自动均衡各个分片之间的负载。
