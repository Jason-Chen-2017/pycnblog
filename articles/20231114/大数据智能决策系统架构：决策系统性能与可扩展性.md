                 

# 1.背景介绍


在现代企业中，由于信息化的发展、海量的数据量的产生、海量的业务数据的积累，以及对数据分析及决策的需求，导致了数据科学、人工智能等高新技术的迅速发展。同时，多种维度的信息不断涌现并得到有效整合后，给人的感觉就像“一张张网页”，对于快速决策、准确预测而言，确实具有不可替代的作用。然而，当数据量和业务数据越来越多时，传统的基于规则或统计算法的决策系统也面临着越来越多的挑战。如何提升决策系统的决策性能和可扩展性？如何利用大数据进行更加精准的决策？本文将从“什么是大数据”、“大数据应用场景”、“大数据分类及特征”三个方面阐述大数据是什么，它可以解决什么样的问题，以及如何通过大数据构建智能的决策系统。

# 2.核心概念与联系
## 什么是大数据
大数据是指通过各种渠道获取、存储、处理、分析和挖掘海量数据而来的信息，其包括如下四个层次:
- 数据采集层: 从不同数据源收集、汇总和存储数据；
- 数据清洗层: 对数据进行清理、标准化、结构化等操作，转换成适用于分析的格式；
- 数据分析层: 通过抽象、概括和计算的方式从数据中发现价值，并进行数据挖掘、分析、归纳、预测等工作；
- 数据展示层: 将分析的结果呈现给用户，包括报表、图表、图形等形式。

## 大数据应用场景
随着互联网的发展和物联网的出现，人们越来越关注数据的价值。在当下，大数据应用场景主要有以下几类：
- 互联网金融: 在线银行支付、保险、电商交易数据、社交媒体、广告数据；
- 移动互联网: 用户行为数据、地理位置数据、产品消费数据、社交数据、移动设备数据；
- 制造业: 智能生产系统、工业物流系统、机器故障诊断系统；
- 政务、电信、制造业、电子商务等领域。

## 大数据分类及特征
根据大数据的特点和应用场景，大数据可以分为四大类： structured data(结构化数据)、 unstructured data(非结构化数据)、 semi-structured data(半结构化数据)、 and real time data(实时数据)。

结构化数据：指的是以关系型数据库建模方式存储的数据。一般情况下，关系型数据库中的表结构比较固定，并且各列之间有明确的定义。比如电影数据库中的数据，包含多个属性，如导演、编剧、片名、年份等。结构化数据可以用SQL语言进行查询、检索和分析，但其缺点是数据量较小，且容易发生数据错误。因此，结构化数据适用于单一主题的数据，适合作为一个整体进行分析。

非结构化数据：指的是不受数据库模式限制的数据。例如，无论是在网页上还是在Word文档中都可以存在大量文本数据。这些数据既不能用关系型数据库管理，又没有固定的结构。分析非结构化数据需要借助于搜索引擎、推荐系统、聚类分析等技术。非结构化数据适合于包含复杂主题、海量数据的场景，能够帮助我们发现新的机会、 insights。

半结构化数据：指的是介于结构化和非结构化数据之间的一种数据类型。半结构化数据通常被称为“XML”或者“JSON”。XML 和 JSON 是两个文本格式。XML 使用标签来定义结构，允许丰富的元数据描述。而 JSON 只是一种用于传输数据的轻量级数据格式，简单易读，但是却缺少灵活的结构。分析半结构化数据需要使用 XML/JSON 解析器或工具。半结构化数据适用于多种数据源混杂、互相关联、动态变化的场景。

实时数据：指的是来自传感器、网络设备、应用程序、微博、微信等不同来源、不同形式的数据。实时的输入数据使得数据的准确性和完整性要求更高。实时数据通常使用消息队列和流计算框架进行处理。实时数据可以帮助我们更快地响应突发事件，并且可以提供实时的反馈，可以及时做出响应。
