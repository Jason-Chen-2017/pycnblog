                 

# 1.背景介绍


随着互联网的蓬勃发展,IT技术也越来越重要。IT技术已经成为经济、社会、国家等领域广泛运用的基础设施。同时,IT技术的快速发展带动了新的攻击手段的出现,这给信息安全带来了新的挑战。因此,如何保障企业的信息系统免受攻击和数据泄露是每一个公司都必须关注的问题。为了保护企业的信息系统,国际上已制定了一系列的安全标准,其中就包括安全策略。

安全策略（Security Policy）是指当发生重大的安全事件或者遭遇某种安全威胁时，公司应当采取的一套或多套行动的总称。从字面意义上理解,安全策略就是对公司及其管理者所实施的一系列举措、规章或规则。它既要强调保护信息系统的安全性,又要鼓励员工积极履行各项安全义务,以确保公司的合法权益不受损害。安全策略的作用主要体现在以下方面:

1. 加强信息系统的安全防范能力。通过制定并落实安全策略,可以提高公司信息系统的整体安全水平,保障公司的信息资产不被盗用、泄露、篡改、毒化、污染,有效防止各种信息泄露的风险。

2. 提升信息系统的运行效率。通过加强信息系统的网络隔离和访问控制,以及严格要求用户使用加密工具,可以减少公司网络上的非法入侵行为,提升信息系统的运行效率,降低信息泄露的风险。

3. 促进信息系统的可靠性维护。通过建立起信息安全档案、监控测评体系和定期安全审计等机制,可以更好地协助公司监控信息系统的运行状况,提升公司的可靠性和信任度,增强公司的信息安全运营能力。

4. 优化信息系统的管理体验。通过充分实施安全政策,能帮助公司更好地把握信息系统的价值,有效支持信息技术人员完成日常工作,降低管理成本,减少公司的信息化投入和风险。

根据以上四点考虑,企业在设计信息系统时应该将安全策略纳入考虑范围,着力确保信息系统具有良好的安全性和运行效率。此外,还需要充分利用云计算、大数据、物联网、人工智能等新兴技术,开发适用于信息安全的解决方案,最大限度地提升信息系统的整体安全性。

# 2.核心概念与联系
## 2.1 常见安全策略
安全策略分为内部安全策略和外部安全策略。内部安全策略指公司内部人员的安全防范措施。如,员工的个人保密制度、内部信息流动限制、内部员工培训、定期安全检查、内部安全日程安排等。外部安全策略则指公司在与外部环境的互联网通信过程中应当具备的安全防范措施,如,安全防火墙设置、运维安全管理、入侵检测、可疑网络流量清理、系统漏洞修复等。

一般来说,公司内部安全策略以信息分类、权限分配和密码管理为主,而公司外部安全策略则以安全防火墙、网络拦截和流量过滤为主。由于公司外部安全策略涉及到多个部门之间的配合,因此存在很多制约因素。

## 2.2 安全策略的作用
安全策略的作用主要体现在以下五个方面:

1. 提升信息系统的运行效率。安全策略能够帮助企业优化信息系统结构,降低信息泄露的风险,提升信息系统的运行效率。例如,对于信息系统中的敏感数据,可以通过对数据的访问权限进行细粒度控制来降低泄露风险。

2. 提升信息系统的可靠性维护。安全策略不仅会引导员工履行各项安全义务,还会在公司各层面开展信息系统的可靠性维护。例如,安全策略可以规范员工的做事方式,强化员工的健康安全意识,提供信息安全相关的培训课程。

3. 促进信息系统的业务连续性。安全策略能够带动公司业务流程的连续性,提升信息系统的正常运行。例如,安全策略可以定期检查公司的业务活动情况,发现异常情况立即进行处理,减轻公司的业务压力。

4. 促进信息系统的运营效率。安全策略不仅会帮助公司实现业务目标,而且还会帮助企业节省资金、降低运营成本。例如,公司可以通过降低网站的登录失败次数、采用验证码、增加薪酬福利来提升网站的访问速度。

5. 维护企业的声誉。安全策略可以提升公司的市场竞争力,推动公司的持续经营。例如,安全策略可以为公司提供充足的安全宣传和安全保障,使公司形象受到社会广泛认可。

综上所述,安全策略是企业应对重大安全事件、满足合规要求、保障信息资产安全、提升信息系统运行效率和可靠性维护等方面的重要工具。

## 2.3 安全策略的分类
安全策略的分类有如下几种:

1. 静态策略：静态策略是指公司在部署产品之前，通过制定管理决策并签署文件的方式，明确对产品安全的要求，并详细阐述，明确对产品、服务、操作的管理要求和流程。这些管理决策和文件一般不会发生变化，长久保存下来供产品生命周期内的使用和参考。

2. 动态策略：动态策略是指产品或服务的安全需求和功能在生产、部署和使用中不断变化的情况下，为了保证产品、服务的安全性，公司通过技术手段、产品功能、运营过程的改善和安全管理政策的制定，不断完善产品或服务的安全保障措施，来满足不断变化的安全需求。

3. 综合策略：综合策略是一种结合静态和动态安全策略的一种安全策略。它综合使用静态策略和动态策略，共同解决信息安全方面的问题。这种策略能够保持较高的安全水平，并且在整个产品生命周期中保持稳定性。

4. 事件响应策略：事件响应策略是指公司应对重大安全事件所采取的策略。这些策略包括恢复、修补、监控和漏洞披露。当发生重大安全事件时，企业首先要采取恢复、修补、监控和漏洞披露的相关策略，使信息系统得到恢复、可靠的运行。

5. 应急策略：应急策略是指处于紧急状态下的公司应对突发事件所采取的安全策略。应急策略可以采取快速部署、回滚和紧急应变等策略，快速恢复系统的运行。

## 2.4 信息安全风险管理法律法规
信息安全风险管理是指对信息系统可能出现的安全风险进行预测、分析、评估和管理，包括识别、评估和分析信息系统可能存在的风险、制定和实施信息安全战略、建立信息安全策略、评估和跟踪信息系统安全状况、使用威胁和威胁情报，并采取相应的组织、流程和技术手段来保护信息资产、建立信息安全监督机制。

信息安全风险管理的法律法规共分为两类:第一类是《计算机信息系统安全管理条例》(GB/T 35273-2011)，第二类是《网络安全法》(GB/T 35279-2016)。前者的立法背景是为了规范企业、政府、学术界和公众对信息安全管理的要求；后者的立法背景则是为了保护全球公民的人身、财产和信息安全。

《计算机信息系统安全管理条例》第十三条定义了信息安全风险管理的任务,要求企业开展风险评估、分析、管理和整改,保障信息系统安全,防止信息泄露、数据丢失、破坏和恶意攻击。它还要求企业落实信息安全工作主体责任制度,建立信息安全工作协同机制,健全信息安全预案制度,及时向社会和行业发布安全通知,采取有效措施加强信息安全风险防控。

《网络安全法》第九十六条将信息安全风险管理分为内部风险管理、外部风险管理、持续管理、流程管理三个阶段。在内部风险管理阶段,企业根据自身业务特点、产品特性和其他因素,制定相关的管理和应急策略,明确信息安全风险评估、分析、管理和整改的职责和工作程序,定期进行内部安全风险识别、分析和处置,促进信息系统完整性和可用性的最大化。在外部风险管理阶段,企业不断向外部合作伙伴、监管机构和专业人员学习和分享信息安全管理知识,积极参与国际和区域安全领域的合作与交流,加强信息安全对抗的能力,有效应对外部威胁。在持续管理阶段,企业不断寻找和消除隐患,确保信息系统的持续性运行和安全性,不断提升产品和服务质量。在流程管理阶段,企业通过制定管理制度、实施规范操作和管理流程,为管理人员创造条件,建立长效、有序、可追溯的安全管理机制。
