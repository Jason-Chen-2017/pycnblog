# 2.核心概念与联系
## 2.1 MySQL简介
MySQL是一个开源的关系型数据库管理系统，最初由瑞典 MySQL AB公司开发，后被 Oracle 收购。它是一个快速、可靠、简单的开放源代码的关系数据库管理系统，并获得了广泛的应用，被广泛地用于Web网站的数据库存储，微软 SQL Server、Oracle、DB2等企业级数据库产品也都采用了MySQL作为关系数据库管理系统。MySQL提供了丰富的功能特性，使其成为最具备竞争力的关系型数据库产品之一。
## 2.2 数据库的定义及分类
### 2.2.1 数据库的概念
数据库（Database）是建立在计算机系统内的一组存放在磁盘或其他永久存储设备上的集合。数据库系统管理各种应用程序的数据，处理用户提交的查询请求。数据库系统组织成若干个逻辑单位——数据库、表、字段、记录、视图、触发器、存储过程等，这些单位相互连接组成完整的数据库。

数据库可以分为三种类型：

1．关系型数据库：关系型数据库是建立在关系模型基础上的数据模型，它以行和列的形式存储数据，具有高度组织化的特点，是目前广泛使用的一种数据库。

2．非关系型数据库：非关系型数据库是一类新型的数据库，主要用于高速、灵活的处理海量数据。它不仅仅存储数据的格式，同时也能检索出需要的数据。非关系型数据库最显著特征就是不仅仅依赖于SQL语言，而且可以存储各种各样的数据格式。非关系型数据库通常以键值对存储数据，其中键是唯一标识符，值可以是任何形式的数据。非关系型数据库的代表产品有Couchbase、MongoDB、Redis。

3．半结构化数据库：半结构化数据库可以存储各种各样的数据，这些数据没有固定的格式，可以随时修改，而不需要重新组织整个数据库。半结构化数据库通常由文档数据库和图形数据库构成，它们具有特别适合动态数据更新的特点。

### 2.2.2 数据库的分类
根据数据库的功能范围、结构复杂度、数据冗余度、数据共享性等多维属性划分为以下五类：

1．客户机/服务器型数据库：这种数据库系统由两台独立的计算机组成，客户端应用程序通过网络访问服务器端数据库，数据库系统接收到请求并处理请求，返回结果给客户端。例如：Microsoft Access、Filemaker、MySQL/MariaDB、PostgreSQL、Oracle Database。

2．分布式数据库：这种数据库系统由多个计算机组成，数据按照分布式的方式存放，客户端应用程序通过网络访问任意一台服务器，数据库系统负责数据同步，保证数据一致性。例如：HBase、 Cassandra、 MongoDB。

3．联机事务处理型数据库：这种数据库系统支持在线事务处理（OLTP），为复杂的应用系统提供快速响应的能力。支持大规模并发写入操作，对实时性要求较高的应用系统。例如：SQL Server、MySQL/InnoDB、PostgreSQL。

4．联机分析处理型数据库：这种数据库系统支持在线分析处理（OLAP），为复杂的分析查询提供快速响应的能力。支持大规模数据集处理，并且通常都有专门的分析引擎来支持分析查询。例如：Vertica、Mondrian、SAP HANA。

5．嵌入式数据库：这种数据库系统只能在特定的应用环境中运行，不能单独部署，一般用于个人电脑、智能手机、嵌入式系统等领域。例如：SQLite、Firebird。
## 2.3 数据模型及范式
### 2.3.1 数据模型
数据模型是指用来组织和存储数据的方法论。数据模型定义了数据对象和关系之间的结构，是数据库系统的核心组件。数据模型的目的是为了简化数据库的结构，降低数据冗余，增加数据可靠性、一致性和安全性。数据模型有三个重要的基本要素：实体、属性、关系。

实体：实体是指现实世界中某些事物的抽象，实体间存在联系或者相关性。例如：人、地址、部门、商品等。

属性：属性是指实体所具有的某些特征，即客观事物性质，它可以是某个值的集合。例如：人名、年龄、手机号码、身份证号、邮箱等。

关系：关系是指两个或更多实体之间具有固定关系的联系。关系可以是一对一、一对多、多对多等。例如：联系人表关系为一对多；员工表关系为多对多；订单表关系为一对多。

### 2.3.2 数据模型的分类
#### 第二范式（Second Normal Form，2NF）
第二范式是关系模型的第三范式，也是关系模型中最严格的范式，它要求一个关系中的每个字段都直接与主键相关。也就是说，如果一个关系的每一行都只有一个值，那么这个关系就不是一个符合第二范式的关系模型。

第二范式的约束条件：

1．数据库表中的所有字段都必须直接与主键相关，而不是传递相关。

2．数据库表中的字段没有重复，不能出现相同的字段。

3．任意非主属性不能完全依赖于候选关键字。

4．不存在派生字段。

#### 第三范式（Third Normal Form，3NF）
第三范式是关系模型的第二范式，它是对第二范式的扩充，消除了含有组合属性的主码，确保每一个非主属性都与主键直接相关。

第三范式的约束条件：

1．除主键外的属性都不应该与主键的任何一部分相关。

2．非主属性不能传递函数依赖。

3．不存在传递依赖。

#### BC范式（BCNF，Boyce-Codd Normal Form）
BC范式（又称3.5NF）是关系模型的第四范式，是对第三范式的进一步推广，它确保每个子集依赖于主键，而不是依赖于其它非主属性，并且主码不能超过两个，不能有超键。

BC范式的约束条件：

1．一个关系模型中不存在主码和复合码。

2．关系模型中的任意一个属性都不可以依赖于其它非主属性。

3．任意非主属性都依赖于主码。

4．超键不能超过两个。
