                 

# 1.背景介绍


近年来，随着云计算、大数据、微服务架构等新兴技术的蓬勃发展，软件架构设计也越来越成为一个复杂而重要的课题。在这个过程中，架构师需要从多方面对系统进行需求分析、功能模块划分、技术选型以及后期维护等环节进行把关。但由于软件架构本身知识面广，技能要求高，所以有志于进阶的架构师多半都在不断学习、沉淀中。但是，到底应该如何学习和掌握软件架构的最新研究成果，确保自己的架构经验不断升级？
近几年来，软件架构领域的顶级大会——全球软件架构技术大会（GSART）屡屡举办，吸引了众多技术专家到分享学术经验、推广理论创新、培养实践能力、开拓创造新的机遇等各个方面，但似乎人们更多关注技术本身，缺少一个系统的思维框架，或者还停留在系统架构图、模型层面的观点。这就诞生了一系列“软件架构课”尝试解决这个难题。
尽管如此，作为一个软件架构师，我的亲身经历告诉我，首先必须要有一套系统的架构理论视角。基于这种理论视角，才能更好地理解架构设计背后的复杂逻辑、关键技术要素以及设计原则。只有真正系统地体认并思考过架构设计背后的这些原理、抽象、模式、方法、组件、对象等核心概念，才能正确评估和选用适合业务场景的架构设计方式。否则，只能看到一些烂摊子上的技术，毫无营养可言。
因此，我认为GSART精彩内容的总结应当包括：
# 1.架构理论：这是文章主要的内容。通过系统的学习和研究，读者可以构建起自己对软件架构设计的全局认识。系统的架构理论包括：基本概念、设计原则、模型、框架、模式、组件、对象及它们之间的关系等。读者需要了解这些概念、原则、模式、框架，以及他们之间的相互作用，并能够利用它们来指导自己的架构设计决策。
# 2.架构实践：阅读《写给开发者的软件架构实战：全球软件架构技术大会精彩内容总结》可以帮助读者锻炼自己对软件架构的理解和思考能力。在具体的代码实现和解读中，读者也可提升自己架构设计水平，获得架构设计过程中的敏捷性和风险控制能力。同时，文章还有助于加强读者的架构思维能力，使之能够更好地搭建起自己的软件架构知识体系。
# 3.架构案例分析：许多作者通过发布他们在实际工作中遇到的软件架构问题和解决方案，来增强读者对架构设计的整体把控力。这些例子既能激发读者内在的解决问题意识，又可为他人提供参考，更有利于将自己的经验转化为行动方案。对于架构师来说，这是一个非常有效的工具。
# 4.云计算平台架构：很多云计算公司正在投资软件架构相关的研究项目，这也是大会精彩内容的一个延伸。云计算平台架构涵盖了云计算基础设施、存储、网络、计算资源、安全、运营管理、开发框架、部署策略、测试策略等多个方面，具有很大的应用前景。云计算架构师需要针对云平台的特点、架构优劣以及产品开发流程进行充分研究，并有条理地把握架构设计的方向和方案，用最好的技术去解决客户痛点问题。
# 5.大数据平台架构：大数据平台所采用的架构模式是目前火热的研究热点。通过对大数据的各种处理和分析，云计算公司需要设计出相应的分布式、弹性、容错的大数据平台。而作为架构师，你可以站在浪潮之巅，站在各大厂商、研究机构的肩膀上，一起参与到大数据平台的设计中来。
