# 1.简介
  

## HTTP协议简介
HTTP（HyperText Transfer Protocol，超文本传输协议）是一个用于分布式、协作式和超媒体信息系统的应用层协议。HTTP协议是建立在TCP/IP协议之上的一个应用层协议，可以传递由HTML等媒体类型的数据对象。
HTTP协议是基于请求-响应模式的客户机/服务器模型，通信过程由客户端向服务器发送请求消息，并接收服务器返回的响应消息。HTTP协议可以支持多种类型的资源，如 HTML 文件、图像文件、视频文件、声音文件及其他各种数据对象。HTTP协议的默认端口号为80，通过请求方法（GET、POST、PUT、DELETE等），客户端可以指定从服务器上请求什么样的信息或者提供什么样的服务，还可以携带请求数据，服务器端根据请求处理后将结果返回给客户端。所以说HTTP协议是一种简洁、灵活、可扩展的通用协议。

## TCP协议简介
TCP（Transmission Control Protocol，传输控制协议）是一个互联网领域中的传输层协议，它负责实现面向连接的、可靠的、基于字节流的通信。TCP协议提供可靠性保障，即确保数据包按顺序到达接收端，并且没有任何错误，对收到的数据包进行确认，保证数据不丢失。此外，TCP协议还实现了拥塞控制，采用了流量控制、滑动窗口协议、重传机制等手段，有效地避免网络拥塞，提高网络吞吐量。TCP协议的缺点是性能低下，因为其握手阶段较长，而且需要等待确认信息。

## 为什么要选择HTTP协议
HTTP协议已经成为互联网上最广泛使用的协议之一。HTTP协议具有以下几个优点：

1. 可靠性：HTTP协议可靠性很好，因为它采用请求-响应的方式，客户端发送请求后，等待服务器返回响应，如果服务器出现故障或宕机，则会自动重试；而TCP协议不支持这种可靠性。

2. 简单性：HTTP协议简单易懂，使用起来也比较方便，无需复杂的配置。

3. 快速性：HTTP协议的速度快，相对于TCP协议来说，因为减少了握手的时间。而且，HTTP协议可以在同一个TCP连接上连续发送多个请求，而TCP只能通过新建新的连接才能实现多次请求。

4. 拓展性：HTTP协议支持多种类型的数据，比如HTML页面、图片、视频、音频等。而TCP协议只支持文本。

5. 隐私保护：HTTP协议可以保护用户的个人信息，因为它使用明文传输，用户无法查看数据包的内容。而TCP协议虽然可以加密，但是无法保护用户的个人信息。

因此，在很多情况下，HTTP协议都能取代TCP协议作为应用层协议，因为它提供了更简单的接口、更高的性能、更强的可靠性、更广泛的适应性。然而，正如TCP协议的缺点所显示的那样，HTTP协议依然有些局限性，比如慢速启动时间、依赖于连接、不能承载大文件等。

# 2.相关术语

## HTTPS协议
HTTPS（Hypertext Transfer Protocol Secure，超文本传输协议安全）是建立在SSL/TLS协议之上的HTTP协议，其主要目的是提供对网站服务器和浏览器之间交换数据的安全信道，可以防止中间人攻击。HTTPS协议需要到CA申请证书，费用较高，但可以通过证书认证、数字签名等方式验证服务器真伪。

## TLS协议
TLS（Transport Layer Security，传输层安全）是一种加密协议标准，它建立在SSL或SSH之上，是一种在两台计算机之间建立安全通信 channel 的方法。TLS协议能够通过身份认证、加密数据、完整性校验等方式确保通信安全。

## SSL协议
SSL（Secure Sockets Layer，安全套接层）是一套加密传输协议，使得互联网电脑用户可以经由Internet进行安全的通信。SSL协议由网景公司设计，网景公司在其之前的商业产品中广泛运用，如Netscape Navigator。SSL协议运行在TCP/IP协议族上，而其前身即标准SSL（Secure Socket Layer，安全套接层）早已废弃。
