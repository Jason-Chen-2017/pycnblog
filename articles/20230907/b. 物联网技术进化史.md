# 1.简介
  

物联网(IoT)作为近几年来人们关注的热点，已经成为继互联网、移动互联网之后的又一重要的网络架构。由于其独特的技术特征及对经济社会发展的深远影响，使得它引起了国际和国内外的高度关注和讨论。此前，许多关于物联网的研究文章或者报道都集中在物联网技术发展的不同阶段，但忽略了其历史脉络及技术演进的整体规律。本文基于笔者所做的一些相关研究工作，尝试从物联网技术发展的角度出发，梳理和总结物联网技术的发展历程。
## 1.背景介绍

物联网(Internet of Things, IoT)最早由英国剑桥大学提出并定义，主要是指通过计算机、传感器、各种终端设备和应用系统实现物理世界和信息世界之间的交流、控制和数据共享，从而进行智能化、自动化的活动。随着计算机硬件的不断升级，以及物联网技术的发展，目前已经成为了包括日常生活、工业领域、军事领域等多个领域的重要基础设施。

随着物联网技术的发展，越来越多的人开始关注其发展过程中存在的问题和弊端，例如网络带宽不足、安全性问题、计算资源的消耗、部署维护成本高、协议标准不统一等等。这些问题对于物联网的发展产生了很大的影响。

从20世纪90年代末到21世纪初期，物联网技术逐渐形成了一个由不同公司、组织和个人共同开发的开放的生态系统。这种生态系统为整个社区提供了共享知识、工具、设备、服务和支持。然而，这个生态系统也存在诸多问题，比如定制化的软件和硬件使得功能过于单一、供应商锁定、需求未被满足、更新迭代缓慢等。另外，物联网的普及率一直不高，仅占全球发达国家的1%左右，导致相关产业的发展受到限制。

## 2.基本概念术语说明

**边缘计算（Edge computing）**：将计算任务下发到靠近数据的地方，如基于终端的传感器、手机、车载终端等；

**云计算（Cloud computing）**：提供应用程序接口和平台，让用户能够购买、存储、管理和使用计算能力，通过网络连接可以访问到；

**移动互联网（Mobile internet）**：利用移动通信技术、无线通讯技术和数据处理技术，构建起覆盖全球的移动通信网络；

**RFID（Radio Frequency Identification）**：由射频识别技术所组成的信息传输系统，用于标识和跟踪物品，属于无线电子技术的一种应用技术，用于实现产品的物理定位、身份认证、流转记录等功能；

**LoRa（Long Range Communication）**：低功耗长距离通信技术，可有效解决地面无线通信信道的不利条件；

**Wi-Fi Alliance**：由IEEE 802.11基金会，将Wi-Fi协议标准化和推广，为世界各地的无线个人设备和企业建立广泛的标准；

**LoRaWAN（Low Power Wide Area Network）**：由LoRa技术组成的无线局域网技术，为物联网终端设备之间提供低功耗、高速率、长距离、安全、可靠的数据传输服务；

**MQTT（Message Queuing Telemetry Transport）**：一种发布/订阅消息传输协议，用于物联网设备之间的数据交换、通信和监控；

**RESTful API（Representational State Transfer）**：一种设计风格，主要用于客户端-服务器通信，用来定义请求方式、URL、参数、返回值等约束；

**OSGi（Open Service Gateway Initiative）**：Apache基金会推出的开源项目，是一个规范、一个框架和一个运行时环境，旨在简化动态模块化系统的开发、部署和使用，为应用程序开发人员提供一套完整的服务网格技术栈；

**容器（Container）**：一种封装技术，允许开发者打包应用程序和其依赖项到一个独立的单元，从而简化了应用的分发和部署；

**Kubernetes（K8s）**：Google开源的容器集群管理系统，具备简单易用、弹性伸缩、高可用和跨平台等优点，正在成为容器编排调度领域的事实标准。

**边缘计算虚拟机（EVM）**：将应用程序部署在边缘节点上，利用分布式计算环境、超级网路、存储等资源，从而实现数据分析、机器学习等计算密集型应用的快速部署和扩展；

**边缘云计算（ECloud）**：利用边缘计算虚拟机部署计算密集型应用，通过云平台提供网络、存储等服务，从而降低数据中心的使用成本；

### （1）物联网架构演变

从最原始的无线电网到有线网再到光纤网后，物联网技术的架构经历了漫长的演变。从网络层到应用层，有以下几个阶段：

1．从无线网到有线网

   - 第一代无线局域网（WLAN）

   - 时至今日仍在广泛应用，用作机器间、机器与人工之间的通信；

2．从有线网到光纤网

   - 第二代无线局域网（MAN）

   - 传统的有线传输技术和结构效率比较差，无法满足需求；
   - 随着新一代光纤技术的发展，基于光纤的互联网已成为物联网的主流架构模式；

3．从光纤网到互联网

   - 以太网（Ethernet）

   - 在电力、信号和电子工程方面取得重大突破；
   - 更好的兼容性、更快的速度、高性能、安全性和易用性；
   - 当时的互联网速度还不够快，导致通信延迟较大；

4．从互联网到物联网

   - 智慧城市（Ubiquitous City）

   - 将一系列互联网技术融入到一起，构成物联网的核心架构；
   - 提升通信、数据、分析能力，使得物联网技术得到迅猛发展；


1999年，安徽省政府举办了首届“物联网”科技创新大赛，来自全国各地的众多创客们集结到了一起，展开了一场关于“物联网智慧城市”的探索之旅。在经过两个多月的论坛期间，通过激烈的竞争、充满激情的探索和极富创造力的创意，最终设计出了中国第一代物联网智慧城市——“千鹏大厦”。

### （2）边缘计算的概念和技术发展

边缘计算(Edge Computing)是物联网的一种关键技术，也是支撑物联网架构的基石。在边缘计算中，把计算任务下发到靠近数据的地方，如基于终端的传感器、手机、车载终端等；这样就可以减少网络负担，并根据用户的实际需要提供最佳的响应速度。在提升效率和降低延迟方面，边缘计算还具有十分重要的作用。

早期的边缘计算技术主要包括：

1．专用计算资源

   - 采用商用服务器，如PC服务器、服务器阵列、网络服务器等；
   
2．客户机服务器的并行计算

   - 把客户机上的应用代码或指令集下发到服务器上执行，降低服务器负担；
   
3．网状计算结构

   - 通过路由器、交换机等网段，把服务器与客户机网络连接起来，实现服务器之间的通信；

4．无线传输技术

   - 使用无线传输技术，如Wi-Fi、蓝牙、Zigbee等，降低网络传输成本；

随着云计算、微服务架构、容器技术的发展，边缘计算已形成了一个完整的生态系统。其中包括：

1．边缘计算平台

    边缘计算平台主要包括物联网边缘平台、边缘云平台、终端平台等；
    
2．边缘应用

    边缘应用是物联网边缘计算领域的一个重要研究方向，涵盖了边缘计算、移动计算、物流计算、网关计算、网络分析、智能视频监控等众多领域；

3．边缘架构

    边缘架构是物联网边缘应用的底层支撑技术，包括RFID、LoRa、Zigbee、MQTT、RESTful等；
    

2014年，英特尔推出了第四代英伟达GTX1080显卡，为云计算领域注入新的活力。同时，华为麒麟990芯片的发布更加推动了边缘计算的发展，集成到华为手机中，并将主要用于5G通信领域。

2015年，阿里巴巴在无线终端上推出基于LoRa和NB-IOT技术的网络架构。阿里推出了边缘计算架构的新方向，即从终端到云端，把计算、网络、存储、数据处理、终端设备、传感器、智能APP等硬件资源聚合到一起。以便完成复杂的业务逻辑，获取更多价值。

2016年，微软Azure推出基于边缘计算的Microsoft Azure Stack，该方案专门用于开发、测试、部署、管理微服务应用程序。因此，微软正在向边缘计算转型，推出适用于边缘计算的云服务，并将其用于无人机、自动驾驶汽车、AR/VR等智能终端设备。

### （3）边缘云计算的概念和技术发展

边缘云计算(Edge Cloud Computing)是云计算的一个分支，是一种将云计算的核心能力——基础设施即服务（IaaS），通过边缘计算能力、数据中心的计算能力、应用服务的计算能力等组合形成的新型云计算模型。它是以边缘节点为核心，将整个物联网架构下沉到本地端，通过本地计算和资源综合协调、数据迁移、业务集成、应用交付等机制，实现资源按需分配，解决边缘计算和云计算的矛盾，实现智能化、低时延的分布式服务。

边缘云计算主要包括以下几种技术：

1．虚拟化技术

   - 虚拟化技术，如Docker、Kubernetes等，能够实现应用在云端和本地环境的无缝衔接；
   
2．容器技术

   - 容器技术，如Docker、Kubernetes等，能够实现物联网终端设备的本地化计算资源管理；
   
3．微服务架构

   - 微服务架构，如Spring Boot等，能够实现边缘节点内部服务的编排和部署；
   
4．边缘网络

   - 边缘网络，如WiFi、蓝牙、ZigBee等，能够实现物联网终端设备的高速、低时延的通信；
   
5．边缘数据中心

   - 边缘数据中心，如VDC、SDN等，能够实现物联网终端设备的本地化、安全、可靠的数据存储和处理；
   

2015年，百度推出了百度雄雀云，通过以边缘节点为核心，将整个物联网架构下沉到本地端，提升了服务质量和响应速度。目前，百度雄雀云正在布局第三代云计算平台，即边缘云计算平台，并将其部署到主要物联网运营商的骨干站点。百度雄雀云是在分布式系统、分布式数据库、消息队列等新一代云计算技术的驱动下，打造的一款特别针对物联网场景的云计算平台，为物联网终端设备和应用提供高可用、低延迟、安全、可靠的服务。

### （4）物联网协议栈演进

物联网协议栈是指物联网通信的底层链路技术。它包括物理层、数据链路层、网络层、传输层、应用层和业务层。

1．物理层

   - 物理层，主要包括物理连接媒介、信号标准、物理信道资源分配和控制、物理层安全保障、传输介质等功能；
   
2．数据链路层

   - 数据链路层，主要包括网络地址、帧格式、错误检测、流量控制、传输协议、流量管理、拥塞控制、差错控制等功能；
   
3．网络层

   - 网络层，主要包括网际协议（IP）、路由选择、网络互连、因特网控制报文协议（ICMP）、网际组管理协议（IGMP）、域名系统（DNS）、动态主机配置协议（DHCP）等功能；
   
4．传输层

   - 传输层，主要包括传输服务、多路复用、多路分解、端口号范围等功能；
   
5．应用层

   - 应用层，主要包括应用程序接口、文件传输、电子邮件、远程登录、电话呼叫等功能；
   
目前，物联网协议栈的版本已经历经了不同的演进。从最初的无线电通信协议、有线传输协议、互联网协议、TCP/IP协议等，到现有的物联网应用层协议如MQTT、XMPP、CoAP等，再到如LoRaWAN、LoRaHAT等物联网应用层协议。
