# 3.	相关概念
## 3.1 什么是DataException情？
DataException情，是指利用非法获取的数据进行犯罪活动。目前该词还没有统一的定义，各个法院、部门、部门规章制定者自行区分。在行业中，“DataException情”多被用于描述与金融、保险、银行等有关的犯罪活动。

## 3.2 为什么会发生DataException情？
DataException情一般发生在未经授权的情况下收集数据、储存数据、交换数据、共享数据，从而对个人或组织造成严重损害。例如，某房地产公司员工利用非法获得的财产信息贿赂，购买二手房，导致上市公司被起诉。

## 3.3 DataException情有哪些特征？
1. 数据造假、泄露
2. 违反相关法律法规
3. 泄露/非法使用他人的私密信息
4. 捏造事实、掩盖事实、胁迫他人
5. 涉及危害国家安全、经济利益、公共利益
6. 有恶意攻击性、矛盾性言论等

# 4.	案例分析——外卖骗局
## 4.1 事件背景
假设某餐饮集团旗下的外卖平台，发现当顾客点外卖时，系统会记录顾客的订单编号，但并不是真实订单号。而且在大量数据的查询下，很快查找到了很多重复的订单编号，显示多个顾客实际都是同一个订单号。

## 4.2 事件分析
为了解决外卖平台中的这个漏洞，公司决定建立订单数据中心。订单数据中心负责对订单信息进行收集整理，解决系统漏洞中出现的“身份混淆”问题，同时可以提供多种数据服务，包括数据分析、监控预警、风险控制等功能。

订单数据中心初步设想如下：

1. 根据顾客手机号码（或其他唯一标识）对订单信息进行聚合，确保数据准确无误；
2. 对数据做去重处理，避免重复数据进入系统；
3. 使用数据分析工具，对订单进行统计分析，比如各类型订单数量、各时间段订单量、客户行为习惯等；
4. 提供接口，方便业务部门对订单数据进行导入导出；
5. 在系统中加入身份验证机制，防止恶意入侵、破坏系统。

## 4.3 结论
这次案例展示了面对“DataException情”，一个成功的组织往往需要考虑更多方面的因素，而不是单一的漏洞修复，以保证其业务的持续发展与增长。
