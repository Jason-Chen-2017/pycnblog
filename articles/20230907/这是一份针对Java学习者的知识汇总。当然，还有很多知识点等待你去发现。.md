# 2.基本概念术语说明
## Java 虚拟机（JVM）
Java虚拟机（JVM），一种执行Java字节码的虚拟机。它负责字节码的解释、加载，并运行Java程序。目前，JVM有两个实现，分别是Sun HotSpot VM 和 Oracle JRockit VM。这两种VM的性能和适用场景都不同。Sun HotSpot VM是Oracle Java SE标准版的基础JVM，它是一个由Oracle开发并拥有完整权限的商业JVM。支持JIT编译器，支持动态类型检查，具有垃圾收集机制，支持高效的线程处理；其指令集类似于其他主流CPU架构的指令集，可以将字节码编译为本地机器码运行。

而Oracle JRockit VM则是一款由Oracle开发的开源JVM，其特色在于自带JIT（Just-In-Time）编译器，采用的是压缩寻址方式，并兼容OpenJDK标准库。由于它自身的压缩寻址方式，相对于HotSpot VM来说，它的启动速度和内存占用率会更低，但其稳定性和兼容性更佳。

除此之外，OpenJDK拥有自己独有的模块系统，允许用户开发第三方模块。OpenJDK还拥有多个OpenJDK开发组（OpenJDK Community Process）项目，如OpenJFX（JavaFX）、OpenCSV（CSV数据解析）等，都可以与OpenJDK一起使用。

## JDK vs JRE
JDK（Java Development Kit），是整个Java开发环境的集合，包括Java运行环境（JRE）和开发环境（JDK）。其中包括javac、java、javadoc等工具。JDK用于编译、运行、调试Java应用程序。JRE是运行Java应用程序的必备组件，包含Java的类库、虚拟机及其它必要的文件。

需要注意的是，JDK不是只有开发环境才需要安装。如果你只是想运行Java程序，那么只需要安装JRE即可。

## Java 语言
Java 是一门静态面向对象编程语言，是一种健壮、简单、可靠、可移植的programming language。创建Java的目的是为了给程序员提供一个可以构建跨平台(Windows, Linux, Mac OS X)、可扩展的应用软件的工具箱。Java支持面向对象编程、事件驱动模型、分布式计算、数据库访问、GUI编程以及许多其他功能。Java程序可以在任何具有JAVA虚拟机(JVM)的操作系统平台上运行。

Java有如下的一些特点:

1. 安全性: Java 具备可靠的安全机制来保证数据的保密性、完整性和可用性。

2. 可移植性: Java 可以在各种不同的平台上运行，包括 Windows、Linux、Unix、OS X、Solaris、HP-UX、AIX、IBM z/OS 等。

3. 易学性: Java 语法简单易懂，学习曲线低，开发效率高，源代码通常比同类语言更紧凑，尤其适合多线程开发。

4. 平台独立性: 通过 Java 的 JVM 技术，Java 程序可以在各种不同的平台上运行，不用担心平台兼容性问题。

5. 多线程: Java 支持多线程开发，通过线程间共享变量的方式进行通信，有效避免复杂的同步问题。

6. 自动内存管理: Java 有自动内存管理机制，能够自动释放内存，不需要手动管理内存。

7. 网络编程: Java 提供了丰富的网络编程接口，包括 java.net、javax.net、java.nio 等包。可以通过 Socket、ServerSocket、URLConnection 等类来编写网络程序。

8. JDBC: Java 为访问关系型数据库提供了JDBC接口。

9. RMI: Java 提供了 Remote Method Invocation (RMI) 技术，允许程序调用远程对象的服务。

10. 微内核: Java 使用微内核架构，这样可以降低系统开销和资源占用，使系统更加健壮、更容易维护。

11. 反射: Java 提供了反射机制，允许程序在运行期间获取类的信息，构造对象，调用方法和字段，动态修改代码。

12. 动态代理: Java 提供了动态代理技术，可以实现在运行期间根据需求生成代理对象。

除了以上几个重要的特点之外，Java 还有很多特色，例如：

1. 动态类型: Java 灵活的静态类型系统支持动态类型，可以使用对象作为参数传递，返回值也可以是对象。

2. 函数式编程: Java 支持函数式编程，支持 Lambda 表达式和 Streams API。

3. 反射式注解: Java 支持反射式注解，可以在运行期间获取注解的信息。

4. 脚本语言: Java 可以被用来开发脚本语言，比如 JavaScript。

5. 微服务: Java 的 Spring 框架支持创建微服务，可以在不增加额外负载的情况下扩展应用程序。

6. 异常处理: Java 提供了良好的异常处理机制，能够帮助定位和解决程序中的错误。
