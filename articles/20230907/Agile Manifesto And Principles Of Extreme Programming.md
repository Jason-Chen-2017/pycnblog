# 1.简介
  

Agile (aɪˈgl)是一个经典的敏捷方法论，它由英国人安东·马利克(<NAME>)于2001年提出。这是一种以个体和 interactions 为核心，而非 process 和 tools 的敏捷开发方法论。简单来说，它的特点就是快速响应需求变化，能够在短时间内完成工作。所以，Agile方法论认为，企业应该更多关注于客户需求的反馈及其快速迭代的方式，从而取得更好的竞争优势。
Agile方法论也是一个非常好的工具，它可以帮助企业将大型复杂系统分割成一个个小模块，并把它们组装起来，来满足用户的需求。此外，由于每个模块都可独立开发、测试、集成，所以开发团队可以轻松地对项目进行迭代。

Extreme Programming (XP)又称为极限编程，它是一种敏捷开发方法论，并不是某个公司或者组织独有的，而是源自于1996年IBM公司发起的一套开发方法。这套方法强调以客户需求为导向，不断重复以最小化设计和开发周期为目的的迭代过程。XP倡导用简单但完整的方法去构建高质量软件。所以，XP主要关注于快速交付满足用户需求的软件产品，并解决最严重的问题。其工作原理如下图所示： 


XP的核心理念就是『Test Driven Development（TDD）』。TDD是一个软件工程中的核心实践，它强调自动化测试驱动开发。通过编写单元测试，测试人员就能在编码之前就发现一些潜在的问题，减少了bug出现的可能性。通过这样的测试驱动开发方式，开发者不需要费心地花费大量的时间去调试软件，只需要关心实现功能的正确性即可。

2.Agile Manifesto 
以下是Agile宣言的要点。
* 个体和互动胜过流程和工具
* 面对面协作胜过单独决策
* 满足客户的需求比合同谈判更重要
* 通过快速反馈进行调整
* 客户始终居于首位

以下是该宣言的总结。它定义了一个企业应如何运用Agile方法论来达到快速响应、满足用户需求的目标。它不仅仅适用于IT行业，还可以用于其他行业，如医疗保健、金融服务等。


3.Agile四大价值观
* 透明性：客户和团队之间的沟通频率越低，价值的透明性越高。
* 开放性：软件开发过程中不同角色的人员可以参与进来，有助于产品持续改善。
* 顾客满意度：定期获得顾客的反馈是成功的关键。
* 精益求精：开发团队的每个成员都要尽可能做到全面，才能有效地提升效率和产出的质量。

4.Agile原则和重要原则
以下是Agile的六项原则。
* 推倒重来：迭代开发是一个不断修正错误、完善产品的过程。任何时候，只要有一个想法或决定被证实是错误的，就要推倒重来，重新评估它，直到找到正确的方向。
* 可运行的产品：所有开发的工作都应该经过测试，并且所有的软件都是可以运行的。只有软件开发出来了，才能真正成为生产力工具。
* 客户协作：与客户密切配合，以便了解他们的需求。客户总是第一位的，并且要定期提供反馈。
* 完美的计划：开发团队要制定详细的计划，以保证开发工作按时完成，并按照计划的要求提交软件。
* 快速反馈：软件开发的一个重要方面就是快速反馈。要快速的得到反馈，否则就无法调整方向，无法保持敏捷开发的节奏。
* 紧凑的队伍：没有人喜欢长时间坐在办公室里，因此开发团队的规模要适中。没有多余的人浪费时间，使得每天都有空闲的时间来学习新知识。

5.Scrum和XP的区别
Scrum和XP都是敏捷开发方法论，但它们有些不同。
Scrum是一个框架，它提供了一些指导原则和方法，但Scrum本身并不是敏捷开发方法论。Scrum涉及的不是具体的方法和流程，而只是Scrum会议上使用的各种术语，如角色、事件、工件、议程、墙板等。

另一方面，XP是一个具体的敏捷开发方法论，它强调编写自动化测试用例，并使用TDD这种开发方式。XP鼓励开发团队在设计和开发阶段进行设计评审，而不是在后期才进行设计评审。

6.Scrum和XP实践
Scrum的实践步骤：
1.产品负责人和业务分析师一起探讨用户需求。确定需求优先级，并制定sprint计划。
2.研发团队根据sprint计划，每天分配任务到自己的名字栏上。
3.产品负责人负责安排会议，确保团队的进度按计划执行。每隔几天，产品负责人和业务分析师都会召开会议，讨论产品的进展情况、问题和风险。
4.每日站会：在每日早晨，每个人都要登录到自己的名字栏，看一下自己昨天完成了哪些工作，明天打算做什么，遇到了什么问题。
5.迭代会议：每隔两周，产品负责人和业务分析师一起举行一次迭代会议，讨论本次迭代的结果、计划、问题和风险。每日站会是迭代会议的组成部分。

XP实践步骤：
1.收集需求
2.设计模型：确立基本的功能和用例模型，并编写测试用例。
3.小步快走：首先完成一个用户故事(story)，再逐步迭代，直到满足所有需求。
4.反复验证：测试通过之后，继续小步快走。
5.反馈循环：每两周反复检验，根据用户反馈及时调整和优化产品。
