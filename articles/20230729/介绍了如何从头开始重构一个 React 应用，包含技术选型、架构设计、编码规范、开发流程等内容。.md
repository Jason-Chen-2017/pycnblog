# 2.核心知识点
## 什么是重构？
重构（英语：refactoring）是一种对软件内部结构的改善，目的是为了提高代码的质量、更容易理解和修改。重构的最终结果并不是完全没有Bug，而是在不改变外部行为的情况下，提升代码的可读性、可扩展性、可靠性。因此，重构应当有利于软件的健康、功能的增强、性能的提高。本文主要讨论的是React应用的重构，但是重构的相关技术还包括HTML、CSS、JavaScript等。
## 为什么要重构React应用？
React应用的痛点和挑romise很多。其中最突出的是复杂性问题。React 组件化架构的引入让它可以构建丰富的界面功能，但同时也增加了代码的复杂度。基于这个原因，越来越多的公司选择用 React 技术进行大型应用的开发。这样的应用逐渐壮大，代码量也越来越庞大。随之而来的问题就是代码维护、扩展能力差、可测试性差等问题。因此，需要对React应用进行重构，让其更加易维护，易扩展。
## 有哪些重构策略？
### 增量重构
在重构的过程中，我们不能一次性把所有问题都解决掉。通常，会先做一些小的、局部的优化，验证方案是否有效后再扩展到整个应用。这样就可以保证重构的效率，避免因为一蹶不振导致失去信心。另外，也可以采用循序渐进的方法，先重构那些不需要太多修改的代码，从而减少风险。
### 小步快跑
其实，我们每天都在写React应用。所以，我们应该充分利用每天的编码成果。即便某次重构工作无法马上完成功能，也可以先完成一些较小的改动，确保每次迭代都能实现目标。这样既保证了工作进度的稳定性，又能及时发现和修复问题。
### 分而治之
相比单一重构，分而治之的方式能有效减少出错的概率。比如，可以先重构页面布局，然后逐步扩展其他模块。这么做的好处是可以在迭代过程中逐步细化和攻克问题，缩短反馈周期。
### 没有银弹
并不是所有的重构手法都是万金油。有的重构手法虽然简单，却很有效。比如，重命名变量、函数或者文件名，只需花费很少的时间就能完成。而另一些重构手法则需要付出更多的代价。比如，想要优化React应用的渲染性能，就必须牺牲代码的封装性，同时兼顾代码的可读性和可维护性。因此，只有找到适合项目的重构手法，才能真正提升应用的质量。
