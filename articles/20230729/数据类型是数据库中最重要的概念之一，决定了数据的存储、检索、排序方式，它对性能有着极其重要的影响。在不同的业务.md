# 1.简介
         

         在关系型数据库（RDBMS）中，数据类型分为三种:标量数据类型、集合数据类型及关系数据类型。其中，标量数据类型包括整型、浮点型、字符串型等，它们是最简单的一种数据类型，只有单个值。而集合数据类型则包括数组、列表、元组等容器类型的用法，例如，数组可以用来存储多个相同或不同的数据类型；列表可以用来存储一系列的值，并且可以按需增加或者删除元素；元组也可以用来存储一系列的值，但不能增加或者删除元素。
         
         而关系数据类型又包括表、记录、行、字段等各种概念，这些概念是由关系模型理论得出的，其目的就是为了更好的表示多对多、一对多、一对一这种复杂的关系。表是关系数据库中的基本单位，一般表都具有以下几个特性：
         1.主键：一个表只能有一个主键。主键通常是一个唯一标识符，其能够帮助我们快速索引到表中指定的一条记录。
         2.外键：如果两个表存在关联关系，那么它们之间的某个字段就形成了外键。外键用于描述两个表之间的联系。
         3.唯一性约束：唯一性约束确保某列或某组列不含重复值。如果某个字段被定义为唯一性约束，意味着该字段值的组合必须唯一。
         4.非空约束：非空约束保证该字段不允许为空值。
         5.默认约束：当一条记录被创建时，如果没有明确填写该字段的值，那么系统会自动将该字段设置为默认值。
         
         除此之外，还有一些其他约束，比如检查约束、枚举约束、表达式约束等。关系数据类型非常灵活，可以适应各种业务需求。但是，同时也带来了一定的复杂性。
          
         2.标量数据类型 
         根据使用场景不同，常见的标量数据类型如下图所示： 
         
         | 数据类型    | 描述     | 优缺点                     | 使用场景                                      |
         | ---------- | -------- | -------------------------- | --------------------------------------------- |
         | INT        | 整数类型 | 没有小数点，适合数值计算      | 需要数字精准计算，如身份证号码                |
         | DECIMAL    | 小数类型 | 有小数点，适合货币计算       | 比较适合金额的计算                             |
         | NUMERIC    | 定点数类型| 有小数点，适合货币计算       | 跟DECIMAL类似，但是它的总长度受限于系统变量      |
         | CHAR       | 定长字符串| 固定长度，占用空间大，节省空间 | 需要保存短字符串                              |
         | VARCHAR    | 可变长字符串| 动态长度，可根据输入调整大小 | 如果数据的长度并不是特别固定的情况            |
         | TEXT       | 文本类型 | 大容量存储，适合大文本        | 需要保存长文本                                |
         | DATE       | 日期类型 | 不保存时间，只保存日期      | 只需要日期信息                                  |
         | TIME       | 时间类型 | 不保存日期，只保存时间      | 需要保存时间戳                                 |
         | DATETIME   | 日期时间类型| 保存日期和时间              | 需要保存完整的时间                             |
         | BOOLEAN    | 布尔类型 | true/false                 | 需要存储逻辑值                                |
         | BINARY     | 二进制类型| 存储字节序列               | 可以存储图片、视频、音频文件等二进制数据       |
         | VARBINARY  | 可变长字节序列类型| 动态长度                   | 当要保存的字节数量并不固定的时候可以使用这个类型 |
         | JSON       | json类型 | 可存取结构化数据           | 需要存储json格式的数据                         |
         
         3.集合数据类型 
         集合数据类型包括数组、列表、元组等容器类型的用法。根据使用场景不同，常见的集合数据类型如下图所示： 
         
         | 数据类型   | 描述     | 优缺点                           | 使用场景                    |
         | --------- | -------- | -------------------------------- | ---------------------------- |
         | ARRAY     | 数组类型 | 通过下标访问元素，随机访问速度快 | 保存同一种类型的数据        |
         | LIST      | 链表类型 | 按序存储数据，按索引访问速度慢   | 有序集合的存储、访问         |
         | SET       | 集合类型 | 无序存储集合，无索引             | 集合的存储、去重             |
         | MAP       | 映射类型 | 存储键-值对                      | 用作字典的功能              |
         | TUPLE     | 元组类型 | 固定顺序存储数据，支持数据类型    | 有规律的保存多种数据        |
         | ENUMERATE | 枚举类型 | 提供友好的名称与编号             | 需要定义一组整数值的集合     |
         
         4.关系数据类型 
         以表的角度来说，关系数据类型主要是指表、视图、索引、触发器等概念。表可以理解为关系数据库中最基本的单位，它可以保存多种数据类型。为了更好地管理关系数据类型，数据库还提供了很多高级功能，例如：视图、索引、触发器、事务等。常见的关系数据类型如下图所示： 
         
         | 数据类型          | 描述                                                         | 优缺点                                   | 使用场景                                  |
         | ----------------- | ------------------------------------------------------------ | ---------------------------------------- | ----------------------------------------- |
         | TABLE             | 表                                                           | 结构化的集合，灵活方便                   | 保存复杂的结构化数据                      |
         | VIEW              | 视图                                                         | 逻辑上的表现形式，基于已有表生成           | 对特定查询做抽象                          |
         | INDEX             | 索引                                                         | 提升查询效率                             | 快速查找某些列的数据                       |
         | TRIGGER           | 触发器                                                       | 自动执行的存储过程，插入、更新、删除等事件 | 执行一些自动化的任务                        |
         | CONSTRAINTS       | 约束                                                         | 数据完整性和一致性                       | 设置字段值的限制条件                        |
         | PROCEDURE         | 函数                                                         | 操作数据库中的数据                       | 将多个SQL语句封装成一个操作，并赋予名称 |
         | SEQUENCE          | 序列                                                         | 生成唯一数字                               | 用于生成唯一标识符                         |
         | DATABASE          | 数据库                                                       | 数据库                                    | 可以划分多个用户、角色、权限的数据库       |
         | SCHEMA            | 模型                                                         | 定义数据库中表、视图、约束的模式          | 统一数据库设计风格                          |
         | SYSTEM VERSIONING | 版本控制                                                     | 对表数据进行历史记录                     | 查看指定时间内修改的数据                    |
         
         每个数据类型都有自己的特点和适用的业务场景，需要结合实际情况进行选择和使用。
