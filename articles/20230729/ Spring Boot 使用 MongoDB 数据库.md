# 1.简介
         
　　Spring Boot是一个轻量级、全面、构建RESTful应用的框架，它可以让你快速、方便地开发单个微服务或者完整的分布式系统。其主要目的是使项目启动更加简单、运行更加稳定。其中，Spring Data JPA模块可以帮我们非常容易地实现对MongoDB数据库的访问，而不需要额外编写ORM映射代码。本文将详细介绍如何在Spring Boot中使用MongoDB数据库。
         　　什么是MongoDB？
         　　MongoDB是一个开源NoSQL文档数据库，支持动态查询、高效率、分布式、索引和查询优化等特性。它是一个基于分布式文件存储的数据库，基于性能的横向扩展能力可以轻松应对大规模数据集合。相比于关系型数据库（RDBMS）来说，它具有更好的灵活性、可伸缩性、高可用性、可靠性、易用性等特点。因此，在某些场景下，例如大数据分析、电子商务平台、社交网络等，MongoDB可以发挥出巨大的优势。
         　　Spring Data MongoDB模块可以帮助我们连接到MongoDB数据库并对其进行CRUD操作。本文将介绍如何在Spring Boot中配置并使用MongoDB作为持久层数据库。
         
         # 2.基本概念术语说明
         　　1.数据库：数据库是存放数据的仓库，是管理数据的容器。不同数据库系统之间的差异性很大，但都遵循一般的数据模型。常见的数据库系统包括MySQL、Oracle、SQL Server、PostgreSQL等。


          　　2.关系型数据库：关系型数据库（Relational Database Management System，RDBMS）是指采用关系表格结构来组织、存储和管理数据的数据库。关系型数据库通常由一个或多个表格组成，每个表格由不同的列和行组成，每条记录都对应着唯一的关键字。典型的关系型数据库系统包括MySQL、Oracle、SQL Server、PostgreSQL等。


         　　3.非关系型数据库：非关系型数据库（NoSQL，Not only SQL），也称作不仅仅是SQL的数据库，是一种独立于数据库管理系统的，基于键值对（key-value）存储的分布式数据库。不同于传统的关系型数据库，NoSQL数据库的理论基础是数据无关性，这一点与关系型数据库不同。典型的NoSQL数据库包括HBase、Cassandra、MongoDB等。


         　　4.文档数据库：文档数据库（Document Database）是一种NoSQL数据库，是一种面向对象的数据库，其中数据被存储为JSON格式的文档。典型的文档数据库包括MongoDB、Couchbase等。


         　　5.键值对数据库：键值对数据库（Key-Value Database）是最简单的非关系型数据库，它将数据以键值对的方式存放在内存中，通过提供一个键即可获取相应的值。Redis就是一种键值对数据库。


         　　6.关系型数据库与非关系型数据库的区别：

            1) 数据模型：
              关系型数据库中的数据以表格的形式存储，表格之间存在固定模式和关系；而非关系型数据库中的数据则不受固定模式限制，可以自由的定义数据模型。关系型数据库适合存储结构化数据，如股票交易历史、商品价格变动等；而非关系型数据库适合存储半结构化、不可预测的数据，如照片、文本、视频等。

            2) 数据容错性：
              关系型数据库由于结构化数据，要求数据结构的一致性，在保证数据一致性的同时，也增加了对磁盘、内存、CPU等资源的消耗，因此，当发生硬件故障时，可能导致数据丢失。而非关系型数据库则不存在数据结构的要求，数据可以按需扩容，在保证高可用性的前提下，减少了对硬件资源的依赖。

            3) 查询速度：
              在数据量较小时，关系型数据库由于有固定的模式，查询速度快；而非关系型数据库则没有固定的模式，可以灵活的设计数据模型，因此查询速度可能会慢一些。但是随着数据量的增长，关系型数据库的查询速度会逐渐下降，这取决于优化技巧。

         　　总之，关系型数据库和非关系型数据库各有优缺点，根据不同的应用场景选择合适的数据库。目前主流的非关系型数据库有键值对数据库、文档数据库和列式数据库等，比较知名的有Redis、MongoDB、Couchbase等。
