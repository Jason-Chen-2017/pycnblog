# 1.简介
         
 MyBatis 是 Java 中的一个持久层框架，可以用于操作数据库，支持定制化 SQL、存储过程以及高级映射。 MyBatis 框架的诞生离不开它的继承体系和优秀的设计理念，它使得 XML 配置文件可以完全映射对象的关系型数据库表，并通过简单的接口和对象就可以灵活地访问数据库。
          在过去的一年里 MyBatis 在 GitHub 上已经超过了 7.6k star，随着它的热度越来越高，越来越多的人开始关注 MyBatis 作为 Java 中最流行的 ORM 框架。由于 MyBatis 框架在性能、功能和兼容性方面的全面优势，越来越多的公司开始逐渐转向 MyBatis 来管理他们的数据库操作，也促进了 MyBatis 在 Java 社区中的知名度和发展。
          本文将从以下六个方面详细分析 MyBatis 的优势和局限性：
          1.优势
          ● SQL 自动生成：MyBatis 可以根据用户指定的模型类自动生成对应的 SQL 语句，减少开发人员手动编写 SQL 语句的时间。此外，还可以通过注解的方式指定一些特殊规则（如缓存、关联查询等）来优化 SQL 查询性能。
          ● 基于对象的映射：MyBatis 通过反射机制，用自己的映射配置文件将数据库结果集转换成Java类型，极大方便了数据的处理。而且 MyBatis 提供映射标签，可以将数据库表直接映射到 Java 对象上。
          ● 支持多种数据库平台：MyBatis 几乎支持所有主流数据库产品，包括 Oracle、MySQL、SQL Server、DB2、SQLite、PostgreSQL 等。同时， MyBatis 提供了强大的插件扩展功能，可以对不同的数据库厂商的特性进行适配。
          
          ● 数据源连接池：MyBatis 默认采用数据库连接池连接数据源，不需要像 JDBC 那样频繁打开和关闭连接，提升了数据源效率。
          
          ● 支持 XML 和注解两种配置方式：MyBatis 支持使用 XML 文件或者注解的方式来配置映射关系。使用 XML 文件更加直观易懂，适合于小型项目，而注解则更加灵活，可读性更好，对于复杂系统来说更加推荐。
          
          ● 查询结果缓存：Mybatis 支持基于本地内存的查询缓存或EhCache缓存。
          
          ● 支持通过代码控制事务：可以通过 MyBatis 提供的 API 或注解来实现事务管理，只需要在配置文件中设置开启事务即可，无需手动编程。
          
          
          ● 支持mybatis-generator生成器：MyBatis官方提供了一个mybatis-generator的生成器，能够快速生成mapper.xml文件及DAO接口和Mapper映射实现。
          
          
          ● 无侵入式ORM框架：MyBatis 有着良好的分离与解耦的设计理念，不会影响其他模块的正常运行，方便应用解除第三方依赖。
          ……
          2.局限性
          ● 学习曲线陡峭：MyBatis 虽然非常简单，但它所涉及到的知识点却很多，特别是刚接触 MyBatis 的人可能要花费较长时间才能掌握。尤其是在 XML 配置文件的编写上，必须充分理解 MyBatis 的语法，才会编写出正确的映射关系。
          ● 缺乏面向对象思想：MyBatis 没有采用面向对象思想，导致一些高级特性不便使用，例如动态 SQL 和分页。
          
          ● 非规范ORM框架：MyBatis 只能对关系型数据库进行操作，对 NoSQL、NewSQL、图形数据库等其它类型的数据库无法直接进行操作。
          
          ● 线程不安全：由于 MyBatis 使用线程不安全的 Map 作为参数传递，导致 MyBatis 在高并发下容易出现线程安全问题。
          
          ● 缺乏原生数据库的特性支持：MyBatis 对数据库的原生特性支持不够丰富，比如游标读取、BLOB 数据类型等。
          
          ● XML 配置繁琐：XML 配置文件比较冗长，并且 MyBatis 不支持动态 SQL 。
          
          ……
          从以上六点优势和局限性分析，我们发现 MyBatis 有其独特的优势和局限性。 MyBatis 以简单易学著称，但它的局限性也是显而易见的。与 Spring Data JPA 或 Hibernate 比较， MyBatis 更偏重于简单，但是其性能相对要差些。因此， MyBatis 在实际项目中还需要结合 MyBatis Generator 这样的工具来帮助完成自动化的生成，而不是纯手工编写。
