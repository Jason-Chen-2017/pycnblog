# 1.简介
         
　　360企业安全中心(360 ECS)是一个集成了终端安全管理、设备安全管理、云安全管理、风险管理、运营监控等一体化的企业级终端安全解决方案。其目的是为了提升企业网络安全状况，优化企业业务流程，降低企业损失。借助 360 ECS 产品，客户可以快速发现并应对网络攻击、垃圾邮件、恶意软件、电子邮件病毒等各类威胁，有效保护公司资产。
         　　360 ECS采用“云+端”的架构模式，打通设备到云端、云端到终端的全链路覆盖，通过对终端的统一管理，实现了终端安全数据全自动收集、分析和处理，为企业提供高效、便捷的终端安全解决方案。360 ECS 适用于企业内部和外部的终端，覆盖企业员工使用的各类终端，既包括实体机、虚拟机、手机、平板电脑等物理机型，也包括路由器、交换机等网络设备。通过将终端和云端数据进行集中管理，360 ECS 提供了全方位的终端安全防护能力，包括设备运行状态监测、入侵行为检测、漏洞扫描、威胁情报共享、终端异常风险预警等功能。同时，360 ECS 提供了完善的权限管理机制，可根据组织架构、岗位职务、设备类型等多维度灵活配置访问控制策略，确保数据安全。
         　　360 ECS 的云端数据存储、分析和处理能力引起了广泛关注。作为国内第一家自主研发的云安全管理产品，360 ECS 把数据源头探索到了极致。它把终端的数据采集、加工、归纳、分析等环节全部托管在云端，不仅使得数据的获取更快、便捷，还能让数据源头和数据使用者分离，减少人员依赖，降低了整体成本。据统计，截至目前，360 ECS 在全球范围内已收集到超过三百万条终端数据，并已针对性地开展了数十亿条数据清洗工作，做到了数据真实可靠、完整可信。360 ECS 不断创新，在增值服务、数据服务、市场竞争力等方面都取得了显著成绩。
         　　2019年，360 ECS 宣布完成 A轮融资，估值约3亿美元。据悉，360 ECS 将于2020年5月份向美国证券交易委员会提交 IPO 招股书，寻求上市发行。
         　　# 2.概念术语说明
         ## 2.1 威胁情报
         ### （1）什么是威胁情报？
         威胁情报（Threat Intelligence）：指通过收集、储存、利用数字信息来识别、跟踪、预测、防范和抵御各种威胁的系统、方法、工具和过程，以及基于这些知识建立网络空间中威胁活动的数据库的技术、手段和能力。

         ### （2）威胁情报分类
         * 智能别名阻断（AI-based alias blocking）：通过构建生物特征检测模型，能够识别出在黑名单中的模糊、易混淆或错误别名，并自动屏蔽特定用户的设备登录。
         * 互联网安全威胁感知（Internet Security Threat Intelligence）：通过收集不同渠道的信息，如互联网域名数据库、社交媒体账号、智能机器人、网络流量等，识别网络安全威胁、溯源威胁。
         * 沙盒逃逸检测（Sandbox Escape Detection）：通过运行虚拟机、容器或者沙箱环境来检测恶意行为，从而减少恶意代码对主机的影响。
         * 漏洞检测（Vulnerability Scanning）：通过运行定制的扫描软件或网络攻击场景，识别计算机系统中存在的安全漏洞、弱点。
         * 漏洞动态跟踪（Vulnerability Tracking）：通过收集并分析常用系统组件和库，如操作系统、数据库、中间件等的安全漏洞情况，不断跟踪漏洞更新。

         ### （3）威胁情报作用
         * 预警系统：由于网络安全形势日益复杂化，许多企业面临着越来越严峻的网络安全风险。利用威胁情报可以预先识别出潜在威胁，及时响应并采取补救措施。
         * 攻击监测：威胁情报可以帮助企业发现其网络空间存在的攻击行为，并迅速采取相应的防护措施。
         * 安全协同：威胁情报也可以用于安全团队之间的合作，共同发现威胁、协调应对。
         * 数据分析：利用威胁情报，能够充分挖掘大量的网络日志和数据，从而对网络安全形势进行及时分析和预测，为企业提供有价值的安全数据。

        ## 2.2 终端安全
        ### （1）什么是终端安全？
        终端安全（Endpoint Security），又称端点安全、终端设备安全、个人安全，是指以信息技术手段和控制措施对个人终端设备进行安全管理的过程和活动。
        ### （2）终端安全管理要素
        * 管理体系：依据国家标准、法律法规、部门规章、部门职权划分等，对管理目标设立具有专门针对性的管理体系。
        * 技术保障：建立健全的技术保障体系，包括安全开发与测试、安全漏洞管理、安全评估、安全运营与管理等。
        * 人员培训：为管理人员提供必要的培训课程，明确知识结构和技能要求。
        * 事件响应：具备应急响应能力，并在发生重大、不可抗拒、诱人的安全事件时积极承担责任，及时处置。
        
        ## 2.3 数据安全
        ### （1）什么是数据安全？
        数据安全（Data Security）：是指信息技术管理范围的一项重要内容，也是运用信息技术为社会提供服务的过程中不可缺少的一环。数据安全包括对所有数据进行收集、存储、传输、使用、处理、共享、销毁、移动、授权、保密等全生命周期过程的保障，即保障信息在使用过程中的安全、可用、隐私、合法、完整和真实性。
        ### （2）数据安全定义
        数据安全应该是通过对数据收集、存储、传输、处理、共享、销毁等整个生命周期阶段的管理和控制，从而最大限度地提高数据安全性、可用性、完整性、正确性、保密性和效率性，保障数据在使用和流动中的安全、可用和隐私性。
        ### （3）数据安全保障要素
        * 安全分类：对不同级别的数据进行保护，分为内部数据、外部数据和关键数据三个层次。
        * 数据加密：在网络上传输的所有数据均需要进行加密，且对于重要数据进行保护，如身份验证信息、金融交易信息等。
        * 数据完整性保障：所有数据存储、处理过程均需保证数据的完整性。
        * 风险管理：数据安全应当配备成熟的风险管理体系，包括信息安全事件管理、信息泄露防治、安全漏洞管理、数据恢复、变更管理、审计等，有效避免或减小安全风险发生。
        * 流程控制：数据安全管理体系应当设置相关制度，如流程文档、内部控制体系、外部控制体系、审计机制、合规检查等，确保数据安全规范、流程和工作方式符合相关标准。
        
        ## 2.4 操作系统安全
        ### （1）什么是操作系统安全？
        操作系统安全（OS Security）是一种保障计算机系统运行正常、稳定、安全的计算机系统安全机制，它负责对计算机系统中的资源、数据和应用程序进行保护、限制、管理、防护和监视，以达到使系统免受非法攻击、病毒、木马、外界入侵等安全威胁的目的。
        ### （2）操作系统安全要素
        * 访问控制：基于用户的身份认证、访问控制、访问控制列表和密码加密技术实现对系统资源的合理、精准和有效访问。
        * 系统配置：系统安全配置的过程旨在评估系统的基础设施、系统配置、软件安装、网络连接等，并通过自动化配置工具、安全控制措施和流程改进，进一步增强系统的安全性。
        * 更新管理：系统安全更新管理可以确保系统的最新软件版本，以及时修复漏洞、漏洞利用、恶意攻击等安全风险。
        * 反恶意软件：通过安全补丁、入侵防御、网络安全策略、病毒库升级等方式提升系统的防病毒能力，促进系统的安全运行。
        
        ## 2.5 智能硬件安全
        ### （1）什么是智能硬件安全？
        智能硬件安全（IoT Security）指智能硬件嵌入到各种设备（如家庭装置、商业设备、建筑物等）中，对其中的网络协议栈、数据通信接口、操作系统、驱动程序等进行安全控制，保障智能硬件运行安全、不被篡改、隐私泄露、物理安全、可用性、完整性和合法性等方面的需求。
        ### （2）智能硬件安全要素
        * 设计过程：智能硬件系统的安全设计应该遵循安全需求分析、威胁建模、安全设计、攻击路径分析、攻击检测与应对、安全评估、漏洞管理和恢复等流程。
        * 安全配置：安全配置包括设备的身份认证、设备初始化、数据加密、关键数据完整性保障等，确保系统的可用性、完整性、合法性、机密性和安全性。
        * 安全审计：在智能硬件上运行的软件需要经过审计和监督才能确保系统的安全，因此，智能硬件上的安全审计体系应当完善，确保数据和操作在设备上执行的安全性。
        * 风险管理：智能硬件系统的安全风险应当由不同专业人士协同管理，包括网络安全专业人员、硬件安全专业人员、软件安全专业人员、运营安全专业人员和技术支持专业人员。
        
        # 3.核心算法原理与具体操作步骤以及数学公式讲解
        这里将对360企业安全中心中的一些算法进行简要讲解。
        ## 3.1 AI-based alias blocking
        360企业安全中心基于深度学习的人工智能模型，能识别出用户输入的错误、模糊或易混淆的昵称，并把相应用户的设备进行阻断，提高设备安全性。
        1. **基于生物特征检测模型的人脸检测**：
        　　通过模型训练、识别出用户输入的错误、模糊或易混淆的昵称，并用该信息对该用户的设备进行阻断。首先，构建一个生物特征检测模型，能够对用户的照片进行面部特征识别，如鼻子、眉毛、眼睛、嘴巴等，这样就可以通过对比相似度，判断输入的昵称是否与图像匹配。
        2. **基于关键字的搜索**：
        　　若没有找到匹配的图像，则可考虑使用搜索引擎搜索，利用关键词去搜索匹配的用户图片。此时，360企业安全中心将会使用Google搜索引擎进行搜索，搜索到的图片与用户输入的昵称进行匹配。
        3. **标记上锁设备**：
        　　无论使用哪种方式去阻止用户的设备登录，最终都会把已登录的设备或账户标记上锁，防止后续访问。
        
        ## 3.2 Internet Security Threat Intelligence
        360企业安全中心利用互联网安全威胁感知对网络安全态势进行监控。
        1. **网络攻击检测**：
        　　通过收集互联网域名数据库、社交媒体账号、智能机器人、网络流量等信息，对网络安全威胁进行持续的跟踪。通过对入侵行为的实时监测，及时发现网络攻击的发生，并及时发现攻击者的位置、攻击方法，以便进行及时的防护和响应。
        2. **威胁溯源**：
        　　利用威胁情报对网络安全威胁进行溯源，通过分析入侵事件的源头，发现攻击者的组织架构、技术能力、业务计划等信息。对疑似可疑的入侵行为进行调查，追溯其祖先、历史演变，并找出可行的攻击路径，以应对网络安全威胁。
        3. **风险预警**：
        　　借助智能机器学习，结合网络安全日志、设备数据等数据，对网络安全风险进行预警。对异常设备的行为、特征、位置等进行实时监控，预警发现异常行为，及时进行安全封堵。
        
        ## 3.3 Sandbox Escape Detection
        360企业安全中心使用沙盒逃逸检测技术，通过运行虚拟机、容器或者沙箱环境来检测恶意行为，从而减少恶意代码对主机的影响。
        1. **沙箱检测技术**：
        　　通过配置和运行不同的虚拟化技术，如Docker、VMware、KVM、Parallels等，可以在一个隔离环境中运行恶意代码，检测代码是否突破沙盒边界，判断其是否能够逃逸沙箱。
        2. **静态代码分析技术**：
        　　静态代码分析技术能识别出恶意代码，提前发现其恶意性。一般来说，静态代码分析会比较老旧的代码和脚本，无法捕获到最新的恶意攻击行为。因此，建议在部署环境中引入自动化检测技术，如编译时检查和单元测试，提升代码质量。
        3. **云环境检测技术**：
        　　云环境检测技术侧重于研究云计算平台的安全漏洞，发现云上运行的应用和服务是否存在安全问题。通过云平台的安全中心、API监控、日志采集等功能，对云上的服务器和服务进行安全检测。
        4. **实时监控**：
        　　沙箱检测技术的检测结果需要反馈给管理员，才能及时发现异常行为。因此，建议在管理界面中引入实时监控功能，及时跟踪运行的恶意代码，及时发现、阻断。
        
        ## 3.4 Vulnerability Scanning
        360企业安全中心利用漏洞检测技术对系统漏洞进行检测，识别出存在的安全漏洞，有效保障系统的安全。
        1. **漏洞扫描技术**：
        　　漏洞扫描技术可以识别出系统中存在的安全漏洞。漏洞扫描一般分为手动和自动两种。手动扫描指管理员手工检查系统漏洞，检查系统安全的运行环境、操作过程等；自动扫描指通过脚本、插件、工具对系统进行自动化扫描，系统在运行中自动检测漏洞。
        2. **漏洞分类和管理**：
        　　通过漏洞分类、管理和改进，可以有效地提升系统漏洞发现和管理的效率和能力。漏洞分类、描述和修复通常是漏洞管理的基础。漏洞管理应该包括漏洞扫描、通知、分析、预防、响应、接受和重构等。
        3. **漏洞发现频率**：
        　　漏洞管理的频率取决于系统中漏洞的危害和风险，一般来说，漏洞扫描的频率越高，漏洞的发现速度就越快。
        4. **漏洞修补周期**：
        　　漏洞管理的目的之一就是修补漏洞，因此，漏洞修补周期也是影响安全性的一个重要参数。一般来说，漏洞修正周期一般为几个月甚至几年。
        
        ## 3.5 Vulnerability Tracking
        360企业安全中心通过收集并分析常用系统组件和库的安全漏洞情况，不断跟踪漏洞更新，提升系统安全。
        1. **漏洞跟踪机制**：
        　　漏洞跟踪机制可以提醒管理员关注系统漏洞的变化情况，及时发现和更新漏洞，增强系统的安全性。通常情况下，漏洞跟踪机制应当包括漏洞报告、漏洞库、漏洞修复时间表等。
        2. **漏洞分类和描述**：
        　　漏洞分类和描述是漏洞管理的基础。一般来说，系统漏洞可以通过漏洞的影响程度、影响范围、触发条件、解决方案等来分类。漏洞描述应当详细说明漏洞的背景、触发条件、攻击流程、防护措施、威胁因素、漏洞利用、风险等。
        3. **漏洞识别和预防技术**：
        　　借助漏洞识别和预防技术，可以有效地防范攻击和渗透，提升系统的安全性。常用的漏洞识别和预防技术包括反病毒、白帽子、安全扫描、网卡管理、凭证管理、VPN、代码安全等。
        4. **漏洞信息共享机制**：
        　　漏洞信息共享机制可以帮助网络安全人员和其他第三方合作伙伴共享系统漏洞信息，共享漏洞信息可以促进互联网安全的发展。
