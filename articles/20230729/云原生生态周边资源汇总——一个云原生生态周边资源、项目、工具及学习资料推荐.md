# 1.简介
         
云原生(Cloud Native)这个词语在近些年随着技术的飞速发展而在业界流行开来。这是一种定义：云原生应用构建在可扩展性、弹性伸缩、灵活迁移、自动恢复、监控告警等核心能力之上，能够更好地实现业务目标并提升IT效率。

随着容器编排领域的兴起，越来越多的公司选择采用云原生的应用架构模式。为了更好的掌握云原生相关技术，了解最新动态，市面上出现了很多优质的云原生技术、开源社区资源、学习资料等。通过对这些资源进行整合归纳，不仅可以帮助大家快速入门云原生技术，也能有效的将自己的知识沉淀到一个文档中供后续查询参考。因此，云原生生态周边资源汇总就是基于此目的而产生的。

# 2.云原生基本概念与术语
## 2.1 什么是云原生?
云原生(Cloud Native)是一个新的软件开发方法论，它源自于 Google 提出的 Borg 框架。它的核心是“一切以云原生的方式编写”以及“容忍一切故障”。云原生意味着应用架构应该建立在可靠、安全、可扩展、弹性的基础设施之上，并且要遵循微服务架构的设计原则。

## 2.2 什么是云原生架构？
云原生架构指的是一套由基础设施、容器编排、微服务和编排调度等组成的架构模式。基础设施即网络、存储、计算等基础设施层，容器化是云原生架构的一个重要组成部分，它使得应用程序可以运行于一致的环境中，获得高度可移植性，从而在分布式系统中扮演重要的角色。微服务架构则是云原生架构中的核心模式之一，它将单个应用程序拆分为一组小型服务，这些服务之间通过轻量级的消息传递通信。

## 2.3 什么是容器化？
容器化是云原生架构的基础，它使得应用可以打包部署，使得部署环境与运行环境一致，具有高度的可移植性。容器化技术主要依托于 Linux 操作系统上的容器技术，包括 Docker 和 Rocket 等，它们允许用户创建独立的容器，其中包括应用程序的代码、依赖库、配置、环境变量和进程，并可以在其上构建、运行和管理。

## 2.4 为什么要用微服务架构？
微服务架构有以下四个优点:

1. 按需伸缩: 通过增加或减少服务的数量，就可以满足应用的不断发展需要；

2. 服务化开发: 将复杂的单体应用划分为一组小型、独立的服务，每个服务都负责一个独立的功能或业务领域，这样使得应用架构更清晰、模块化且易于维护；

3. 可复用性: 每个服务都可以被其他服务所重用，使得应用架构更加灵活；

4. 快速响应: 微服务架构可以让团队更快地响应变化，快速开发新功能或解决问题。

# 3.云原生生态周边资源
## 3.1 Kubernetes 生态资源
### 3.1.1 Kubernetes 官网及文档
Kubernetes 是开源系统，它的官网 https://kubernetes.io/ 上提供了详细的用户手册、教程、API 文档，并且还有大量的 Kubernetes 技术分享及案例展示。

### 3.1.2 Kubernetes 开发者指南
《Kubernetes 开发者指南》是一本由开放原子基金会联合创始人兼云原生基金会主席马若琳撰写的一本技术书籍。从 Kubernetes 最基础的构件到最佳实践，《Kubernetes 开发者指南》详细介绍了 Kubernetes 的各项技术特性，并提供云原生应用开发人员从零开始构建、运行和管理 Kubernetes 集群的指导。

### 3.1.3 Kubernetes 中文社区
Kubernetes 中文社区 (K8sChina) 是由国内多个云计算领域技术大牛共同组建的一个 Kubernetes 技术交流、学习和经验共享的技术群组。K8sChina 有着丰富的 Kubernetes 文章、资讯、视频教程，涵盖了 Kubernetes 各个版本和领域的最新信息，也是国内外许多 Kubernetes 爱好者们的摘要信息和讨论平台。你可以通过官方网站 http://www.k8schina.com/ 来关注 K8sChina 的最新动态，或者通过微信公众号 k8s_china 来加入我们的交流社区。

### 3.1.4 GitHub - kubernetes
GitHub 上有一个名为 kubernetes 的开源项目，里面有关于 Kubernetes 的所有资源，包括官网、文档、教程、视频、代码示例等。项目地址为：https://github.com/kubernetes。

### 3.1.5 CNCF Landscape
CNCF Landscape 是一个可视化工具，它把云原生领域的各种开源项目和工具通过不同的颜色和形状展示出来。你可以通过它来查看当前云原生生态的发展状况，以及了解不同云原生项目之间的关系。CNCF Landscape 项目地址为：https://landscape.cncf.io/. 

## 3.2 Prometheus 生态资源
### 3.2.1 Prometheus 官网及文档
Prometheus 是一款开源的服务发现和监控系统和时间序列数据库，其 GitHub 地址为：https://github.com/prometheus/prometheus 。其官网为：http://prometheus.io/ ，文档丰富。

### 3.2.2 Prometheus 开源大数据生态工具链
PromQL 是 Prometheus 查询语言，它提供了一些内置函数用于处理时间序列数据，如聚合、数据窗口、 joins、时间偏移等。这些函数可以用来查询和处理 Prometheus 收集到的指标数据，形成可视化的仪表盘和报警规则。开源大数据生态工具链 (OpenTSDB Exporter for Prometheus) 可以把 OpenTSDB 数据导入到 Prometheus 中。OpenTSDB 是 Yahoo! 提供的一款高性能的时间序列数据库。通过它可以收集和处理分布式系统的数据，并且支持采样、聚合和标签查询等高级特性。开源大数据生态工具链项目地址为：https://github.com/prometheus-community/opensdb_exporter.

### 3.2.3 Thanos
Thanos 是 Prometheus 项目的云原生版本，它能够在 Prometheus 之外存储和查询时序数据，并支持水平扩展和容错。除了支持 Prometheus 以外，还可以连接对象存储和云服务商的接口来实现长期保留数据。Thanos 项目地址为：https://github.com/thanos-io/thanos.

## 3.3 TiDB 生态资源
### 3.3.1 PingCAP 官网及文档
PingCAP 是一家国内知名的数据库公司，它的产品 TiDB 是一款开源分布式 HTAP (Hybrid Transactional and Analytical Processing) 数据库。TiDB 官网为：https://pingcap.com/docs/stable/, TiDB 文档丰富，值得一看。

### 3.3.2 TiDB Operator
TiDB Operator 是 PingCAP 推出的一款开源项目，它的作用是方便地在 Kubernetes 集群上部署和管理 TiDB 集群。TiDB Operator 提供了 CRD (Custom Resource Definition) API，使得用户可以通过 Kubernetes 中的 CustomResourceDefinition 对象来声明 TiDB 集群，然后由 TiDB Operator 来执行部署和管理工作。TiDB Operator 项目地址为：https://github.com/pingcap/tidb-operator.

### 3.3.3 TiUP
TiUP 是一款命令行工具，用于一键下载和安装 TiDB/PD/TiKV/Grafana/Pump/Drainer 等组件。它可以帮助用户快速部署本地测试环境、线上生产环境以及 Kubernetes 上的 TiDB 集群。TiUP 项目地址为：https://github.com/pingcap/tiup.
