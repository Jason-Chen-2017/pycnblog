# 1.简介
         
## 1.1 React简介
React（快速，简洁且可扩展的JavaScript库）是一个用于构建用户界面的JS库。其主要特性如下：

1. 使用JSX创建组件化开发模式；

2. 组件间数据流管理简单，易于理解和维护；

3. 利用虚拟DOM提升渲染性能；

4. 提供路由功能，可轻松实现单页应用、多页面应用和服务器端渲染；

5. 支持JSX语法，既支持静态类型检查又可灵活使用；

6. 社区活跃、文档完善、工具丰富、生态圈充满先锋队成员。

## 1.2 为什么要用React?
### 1.2.1 React解决了什么问题？
React最初是由Facebook开发的开源UI库，目标是为了解决网页前端页面更新频繁、不停留在页面导致用户闲置的时间过长的问题。随着Web的发展和互联网应用的普及，这个问题变得越来越突出。React的诞生正好解决了这个问题。

React的使用方法很简单，通过 JSX 来定义组件结构，组件之间通过 props 和 state 数据传递。它还内置了一套完整的生命周期，提供了丰富的功能，比如设置状态时触发某些函数。

因此，React可以帮助我们创建复杂的用户界面，并使得开发更高效、更容易控制。

### 1.2.2 优势
- 简洁：React采用声明式语法，用一套规则去描述视图，极大地方便了前端人员的编码工作。无需关注底层的DOM操作，只需要关心业务逻辑即可。
- 可复用性强：React将各个组件拆分成独立的模块，不同组件之间也可以进行组合，大大提高了组件的复用率和可维护性。
- 方便测试：React 提供了良好的单元测试方式，我们可以通过编写测试用例来验证代码是否按预期运行。
- 跨平台：React 基于JS，可以在 Web，Android，iOS，Windows等多种环境中运行，并且能够利用已有的生态系统，如 Webpack，Babel等。
- 大量的第三方库：React 生态圈里有大量成熟的第三方库，如 Redux，GraphQL，Dva，Mobx，styled-components等，可以让开发者更加快捷地完成项目。

### 1.2.3 框架比较
React、Angular、Vue三者都是使用虚拟 DOM 技术来提高渲染性能的 JavaScript 库，但是它们的对比，还需要看它们之间的编程模型。

1. Angular是以指令为中心的模板引擎，Angular中的每个指令都会生成对应的DOM节点。它的模板语法类似JSP，但比JSP更加灵活，所以写起来会相对麻烦一些。而React则是声明式编程模型，它基于组件来构建应用，使得视图和数据更容易被抽象，开发者不需要考虑底层DOM的操作。

2. Vue是以MVVM模式为基础的框架，它的双向绑定机制可以自动更新视图，使得开发者不必关心DOM的变化。它的模板语法类似HTML，所以学习起来比较方便。React同样也是声明式编程模型，不过React官方也在思考如何实现更加接近传统的编程模型。

3. 在选择React还是Angular的时候，通常会综合考虑一下个人能力，开发速度，项目规模，团队习惯等因素。

# 2.核心概念、术语
## 2.1 JSX
JSX 是一种JS语言的语法扩展。它是一种嵌入到js中的XML语法，最终会被编译为普通的JavaScript代码。 JSX 有以下特点：
- 可以直接在 JSX 中书写 JavaScript 表达式。
- 通过 JSX 描述的 UI 组件可以像 React Class Component一样声明属性，在渲染时接收数据。
- JSX 经过编译后仍然是纯粹的 JavaScript 对象，可以被当做参数传入任意函数中。

总结来说，JSX 是一种描述 UI 组件的语法，可以直接嵌入到 JS 文件中。它有利于提高 JSX 的可读性，方便 React 的开发者处理数据，以及与其他框架集成。
