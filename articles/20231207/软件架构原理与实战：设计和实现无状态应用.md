# 2.核心概念与联系

在无状态应用的软件架构中，我们需要关注以下几个核心概念：

1.无状态：无状态应用不依赖于外部数据库或存储系统来存储状态信息。这意味着所有的状态信息都存储在内存中，当应用程序重新启动时，状态信息将丢失。

2.分布式：无状态应用通常是分布式的，这意味着它们由多个节点组成，这些节点可以在不同的计算机上运行。这种分布式架构可以提高应用程序的可扩展性和可用性。

3.可扩展性：无状态应用的设计应该考虑到可扩展性，这意味着应用程序可以根据需要增加或减少节点数量，以满足不同的负载。

4.可靠性：无状态应用的设计应该考虑到可靠性，这意味着应用程序应该能够在不同的环境下运行，并且在出现故障时能够自动恢复。
