# 2.核心概念与联系

在分布式系统中，我们需要关注以下几个核心概念：

1.分布式一致性：分布式系统中的多个节点需要保持一致性，即在任何情况下，所有节点都应该达成共识。

2.分布式存储：分布式系统需要使用分布式存储来存储和管理数据，以实现高性能和可扩展性。

3.分布式计算：分布式系统需要使用分布式计算来处理大量数据和任务，以实现高性能和可扩展性。

4.分布式通信：分布式系统需要使用分布式通信来实现节点之间的通信，以实现高性能和可扩展性。

这些概念之间存在着密切的联系，它们共同构成了分布式系统的核心架构。

4.分布式通信算法：例如TCP/IP、UDP等。这些算法通过将数据包发送到多个节点上，以实现高性能和可扩展性。

在实际应用中，我们需要根据具体情况选择合适的算法来实现分布式系统的核心功能。

Q：分布式系统与集中式系统有什么区别？

A：分布式系统和集中式系统的主要区别在于数据存储和处理的方式。在集中式系统中，所有数据和计算任务都存储在一个中心服务器上，而在分布式系统中，数据和计算任务分布在多个节点上，以实现高性能和可扩展性。

Q：分布式一致性是什么？

A：分布式一致性是指多个节点在完成某个任务后，所有节点都达成共识，并且达成的结果是一致的。分布式一致性是分布式系统中的一个重要概念，它确保了系统的可靠性和一致性。
