# 2.核心概念与联系

## 2.1 规则

规则是规则引擎的基本组成部分，用于描述特定条件下的动作。规则通常由条件部分和动作部分组成，当条件部分满足时，动作部分将被执行。

## 2.2 决策

决策是规则引擎的核心功能，用于根据规则集合来选择最佳的动作。决策过程可以是基于规则的顺序执行，也可以是基于规则的优先级。

## 2.3 知识表

知识表是规则引擎的数据结构，用于存储规则引擎所需的数据。知识表可以是关系型数据库、NoSQL数据库或者内存中的数据结构。
