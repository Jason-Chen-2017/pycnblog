# 2.核心概念与联系

## 2.1所有权

所有权是Rust的核心概念之一，它规定了在Rust程序中的变量有一个拥有者，该拥有者负责管理变量的生命周期和内存。当拥有者离开作用域时，Rust会自动释放变量占用的内存。

所有权的设计目的是为了避免内存泄漏和野指针等问题，从而提高程序的安全性。

## 2.2内存安全

Rust的内存安全是通过所有权机制实现的。当一个变量离开作用域时，Rust会自动释放该变量占用的内存，从而避免内存泄漏。同时，Rust的所有权机制也避免了野指针的问题，因为Rust不允许在无效的指针上进行操作。

## 2.3并发和异步编程

Rust提供了并发和异步编程的支持，这使得Rust能够在多核处理器上实现高性能的并发任务。Rust的并发和异步编程模型是基于任务和通道的，这使得Rust能够在多核处理器上实现高性能的并发任务。

## 4.3并发和异步编程的代码实例

```rust
use std::thread;
use std::time::Duration;

fn main() {
    let v = vec![1, 2, 3];

    // 创建一个新的线程，并在其中执行一个闭包
    let handle = thread::spawn(move || {
        for i in &v {
            println!("{}", i);
        }
    });

    // 主线程继续执行其他任务
    for i in &v {
        println!("{}", i);
    }

    // 等待子线程完成
    handle.join().unwrap();
}
```

在这个代码实例中，我们创建了一个新的线程，并在其中执行一个闭包。主线程继续执行其他任务，而不需要等待子线程完成。
