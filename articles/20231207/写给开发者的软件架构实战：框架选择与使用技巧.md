# 2.核心概念与联系

在深入探讨框架选择和使用技巧之前，我们需要了解一些核心概念。这些概念包括：

1. 软件架构：软件架构是软件系统的高层次设计，它决定了系统的组件、它们之间的关系以及它们如何协同工作。

2. 框架：框架是一种软件设计模式，它提供了一个基本的结构和功能，以便开发者可以快速构建软件系统。框架通常包含一些预先实现的组件，以及一些接口，以便开发者可以扩展和修改这些组件。

3. 组件：组件是软件系统的基本构建块，它们可以独立地实现特定的功能。组件可以单独使用，也可以组合在一起，以实现更复杂的功能。

4. 接口：接口是软件系统的一种抽象，它定义了组件之间的交互方式。接口允许开发者在不知道具体组件实现的情况下，使用这些组件。

5. 依赖关系：依赖关系是软件系统中组件之间的关系，它们决定了哪些组件依赖于哪些其他组件。依赖关系可以是直接的，也可以是间接的。
