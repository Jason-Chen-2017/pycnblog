# 1.简介
  

Kanban（看板）是一个用来管理工作进展、保证产品质量、改善工作效率的敏捷软件开发流程管理方法。它与其他敏捷方法相比，最显著的特征在于把流程分解成可视化的工作卡片，并随着工作进行而逐步清晰化，从而降低了沟通和协作的成本。它的主要特点如下：

1. 可视化组织: Kanban通过将工作流程转化成可视化的工作卡片，让每项任务都具备一个对应的卡片，可以清楚地看到每项任务所需时间、人员配比、阻碍等信息。

2. 更快响应需求变化: Kanban的方法强调尽早交付产品或服务，同时允许团队适应需求的变化，因此能够更快反映市场需求的变化。

3. 减少工作风险: 由于Kanban制订了详细的计划，可确保每项工作按计划完成，防止出现计划外任务导致项目延期、进度滞后甚至损失。

4. 强调职责分离: Kanban鼓励每个人的职责范围小，这样才能有效控制工作流中的风险，提高团队士气。

# 2.基本概念
## 2.1 卡片墙
Kanban中的卡片墙是把流程任务以卡片的形式呈现出来的图形界面。一般来说，卡片墙以项目名称开头，按照工作阶段、子工作包、子任务的方式列出所有任务，每个任务用一个卡片表示。每个卡片上除了显示任务描述、优先级、实际花费时间、当前状态等信息外，还可以根据实际情况添加更多内容，比如截屏、文件上传、链接、评论等。


## 2.2 WIP（Work In Progress）
WIP（工作进行中）指的是当前正在处理的任务的数量。它是衡量Kanban是否取得成功的重要标准之一。Kanban要求每个任务都必须是独立且有限的时间内完成的，否则就会出现WIP超标的问题。

## 2.3 列（Lane）
列（Lane）是Kanban中的一个重要概念。它表示不同类型的工作，比如“待办”、“进行中”、“已完成”，或者可以自定义的任何名称。这些列之间通过卡片流动，表明工作的优先级和进度。在Kanban中，列是虚拟的，也就是说不存在真实存在的物理位置，因此每项工作都可以在多个列上存在。

## 2.4 循环
Kanban也采用了迭代的方法，称为“循环”。每个循环都有一个开始日期和结束日期。每个周期通常有固定的几个阶段，比如准备、启动、开发、测试、发布等。每个阶段都有一个对应的列，这些列通常分为多个子阶段，如准备阶段可以分为计划、定义、分配、资源分配等子阶段。在Kanban中，这个模型被称为“基准模式”，因为它既可以适用于整个项目，也可以细分到各个工作项。

# 3.Kanban的具体操作步骤
## 3.1 制定计划
制定计划时，需要决定每周、每月、每季度分别在哪些任务上投入精力。制定计划一般包括以下步骤：

1. 设置目标：制定计划前，要设定目标，目标应该是长远的，而不是短期的。例如，制定计划可以设置“每月部署10个新功能”这样的目标，而不是“每天新增一个功能”。

2. 分解任务：将每项任务分解为可管理的小块。如果任务过大，则建议将其拆分成多个任务。

3. 确定优先级：每项任务都要有优先级，优先级的级别是紧急、重要、次要、普通四种。

4. 拆分卡片：将任务拆分为不同的卡片，每张卡片只代表一个小的任务。

## 3.2 完成任务
完成任务时，需要对每个任务的执行情况进行跟踪。完成任务一般包括以下步骤：

1. 将任务移动至正确的列：将任务从待办移动到进行中，或者将任务从进行中移动到已完成。

2. 更新卡片内容：更新卡片的内容，如截图、链接、文件等。

3. 添加注释：根据任务的进度和状况添加注释。

4. 对卡片评估工作量：评估每张卡片实际花费的时间，以判断工作是否达成预期目标。

5. 检查任务完成情况：检查每个任务的实际花费时间，确保没有任务超出预算。

## 3.3 处理风险
当遇到突发事件、意外情况或疑难问题时，可能需要采取一些行动，处理风险。处理风险一般包括以下步骤：

1. 想清楚原因：不管是什么原因引起的风险，都必须先想清楚原因。

2. 使用KPI监控：在处理风险之前，最好设置关键绩效指标(KPI)，并将它们加入Kanban流程中。

3. 制定备用方案：如果无法立即解决问题，可以考虑制定备用方案，以免影响整体计划的进展。

4. 分析结果：处理完之后，要分析结果，发现问题、发现缺陷、调整措施、优化流程、总结经验教训等。
