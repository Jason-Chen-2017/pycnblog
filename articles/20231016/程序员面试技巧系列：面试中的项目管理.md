# 1.背景介绍


　　对于技术人来说，“面试”一直都是最频繁、最紧张的工作，面对千变万化的编程语言、框架、工具、解决方案、公司架构等各个维度的拼图，技术人必须具有项目管理的能力，能够准确把握技术路线图、分清优先级，不断完善方案并及时反馈结果，调整方向，确保最终结果符合预期。

　　从事项目管理工作的人通常会从以下几个方面进行考虑：

　　1）解决问题：面试官会问你关于某项产品或者服务的具体需求，这个时候你要想办法回答出自己的一些关键点和优势，以及你能做到哪些功能。

　　2）执行方案：面试官会评估你的技术水平、时间精力、业务知识、团队协作等条件，让你演示自己设计的项目管理流程或工具，让你可以说清楚整个项目的开发周期、任务分配、管理资源以及风险管理策略。

　　3）客户关系：项目管理除了涉及到相关产品的设计开发之外，还包括和客户的交流、沟通、合作等工作。面试官在了解了你的项目经验后，可能会追问到你的客户价值观和诉求，以及项目中遇到的困难和问题，希望你能给出可行的解决方案和方法。

　　4）长远规划：项目管理包含了一段漫长的时间，面试官可能问你，在这段时间里，你都做了什么？从哪里开始，一直到哪里结束。面试官则要给予你一个较为具体的时间表，让你知道每天的工作计划、进度安排，让你掌控全局。

　　以上几种情况，也是项目管理人员需要关注的重点。这才是面试官真正想了解你的主要目的所在。不过，实际上，这只是项目管理人员应当具备的基本素养和技能之一。如果你还欠缺其他方面的能力，比如分析逻辑思维、领导能力、商务谈判能力等，那么项目管理是无法胜任的。

　　当然，作为个人职业生涯规划的一环，你还可以对自己的职业发展规划有所帮助。比如，你认为自己属于哪一种类型的人才？职业发展方向如何？当前的就业市场状况又怎样？这些都可以作为个人职业规划的参考。

　　最后，作为一名技术专家、程序员和软件系统架构师，你是否了解到项目管理在实际工作中的应用？同时，你是怎样把项目管理融入自己的日常工作、生活中？欢迎大家分享您的经验与感悟。
# 2.核心概念与联系
## 2.1 定义与作用
　　项目管理（Project Management）是一门研究、实践和教育，用于组织、指导和监督工程、过程、产品、服务等活动的科学，是企业为了提高效率、降低成本而设计的一种制度、方法和系统。一般将项目管理定义为：
> 1) 对项目进行分解，明确目标和要求；
> 2) 使用适当的方法、工具和手段开展活动；
> 3) 将管理者与参与者的互动，统筹协调、达成共识；
> 4) 确保项目的成功实现，改善项目环境，提升企业竞争力。

　　项目管理通过完成对工程建设活动的规划、跟踪、控制、预测、优化、整合、交流和协同，能够更好地管理、组织、协调、协助工程建设的各个方面，改善工程建设的质量、效率和成本，促使工程建设取得可持续发展。

　　项目管理的实施目标可以概括为：

　　1）确保项目的成功实施：从立项、计划、实施、审计到收尾，严格遵守项目计划，保证项目成功实施；

　　2）提升项目质量：将项目的管理、组织、协同，转换为提升项目质量，比如通过增强工程结构、优化工艺流程、提升设备效率，提升产品性能和成本；

　　3）改善项目环境：充分利用项目管理能力，培养项目干部，提升项目绩效，改善项目环境，做到“两个一百八十度”，打造卓越的企业文化；

　　4）管理和支持企业发展：管理是企业持续发展不可或缺的内在因素，通过合理规划管理，能够充分发挥企业资源优势，推动企业长久健康稳定发展。

　　简单来说，项目管理就是通过对工程建设活动的计划、执行和结果进行有效的管理和控制，确保项目目标、计划、范围、进度、成本、质量、进展、风险等管理和组织目标得到满足，以达到预期的营收、利润或效益。
## 2.2 关键术语
### 2.2.1 项目管理角色
　　项目管理是一个综合性的领域，项目管理部门人员往往既熟悉项目管理的理论和实践，也兼顾不同角色的人员，他们应该具备以下四个核心角色：

　　1）项目策划者（Project Planner）：负责制定项目计划和目标，分析需求、整理方案，确定项目范围、任务分解、分配资源，管理进度，协助制定项目预算、费用等。

　　2）项目经理（Project Manager）：在项目启动前后，负责指导、组织、协调项目的实施和管理，保障项目的按时、按质、按时交付，并对项目结果进行评估和总结，定期向项目经理汇报。

　　3）项目主管（Project Director）：兼任项目策划者和项目经理的职责，负责处理全体项目管理人员的事务，掌控项目的整体进展和规划，为项目总监提供决策建议，并向CEO或总裁汇报工作成果。

　　4）项目总监（Project Supervisor）：既是项目策划者又是项目经理，负责对外代表公司，根据公司战略需求制定、实施项目，统筹协调各方面的参与者，收集意见建议，管理团队，制定和实施项目计划。

### 2.2.2 项目管理过程
　　项目管理的过程可以分为五个阶段：

　　1）项目启动阶段（Start-up Phase）：项目启动前的准备工作，包括研究工作、方案开发、项目会议等。

　　2）项目计划阶段（Plan Phase）：制订项目计划，包括策划项目、确定项目范围、任务分解、分配资源、计划进度、规划预算、费用等。

　　3）项目实施阶段（Implement Phase）：实施项目，包括跟踪项目计划和进度、组织实施、控制实施、监督实施、整合实施等。

　　4）项目收尾阶段（Close-out Phase）：收尾项目，包括审查、总结、评估和跟踪项目成果，确保项目的顺利完成，并对项目进行评估和总结。

　　5）项目管理总结阶段（Summary Phase）：项目管理总结阶段，包括对项目进行总结和对各方面的反馈，得出对企业的满意度和改进建议，并通过CEO或总裁的签字确认管理实施方案。

　　项目管理的各个阶段一般分为项目管理计划书、管理控制台和信息系统三个部分。项目管理计划书是项目策划者制定的完整计划，它记录了项目的范围、目标、任务分解、资源分配、预算、费用等所有内容。管理控制台是用来记录、管理项目的关键数据，例如，项目状态、进展、计划、费用、人员分配等。信息系统是提供项目的信息共享平台，通过在管理控制台和外部渠道获取的数据进行分析，反映出项目的实施情况，提高项目的可视性、透明性。
