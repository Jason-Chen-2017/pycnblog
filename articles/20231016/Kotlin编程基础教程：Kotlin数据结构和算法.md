# 1.背景介绍


## 数据结构
在计算机中，数据结构指的是组织、存储、管理数据的方式。数据结构主要分为两大类：
- 顺序存储的数据结构：也称线性结构，它包括数组（一组连续的内存空间），链表（连接多个节点的数据结构）等。
- 非顺序存储的数据结构：也称非线性结构，它包括树形结构（例如，二叉树，红黑树），图状结构（例如，森林，有向图），哈希表等。

常用的四种数据结构：数组、链表、栈、队列。

## 算法
算法是指用来解决特定问题的一系列指令或操作，并由计算机执行。算法可用于解决各种问题，如排序、查找、矩阵计算、字符串匹配等。

最基本的算法：冒泡排序、选择排序、插入排序、归并排序、快速排序、堆排序。

高级算法：Dijkstra算法、A*算法、贪婪算法、回溯算法、动态规划、BFS、DFS、并查集、堆、红黑树等。

# 2.核心概念与联系
## 顺序存储数据结构——数组
数组是一个具有固定大小的顺序集合，其中的元素可以通过索引(下标)进行随机访问。
### 描述
数组是最基本的数据结构之一，它可以保存相同类型元素的集合。数组是一种线性存储结构，也就是说，数组中的元素是按顺序存储在一块连续的内存区域。当需要访问数组中的元素时，通过索引就可以获取到相应位置的元素。通过索引，我们可以直接访问数组中的任一元素。数组的长度是固定的，创建后不能改变。
### 操作
数组支持以下两种操作：
1. 查找元素：根据索引获取数组中的元素值。时间复杂度为O(1)。
2. 修改元素：通过索引设置数组元素的值。时间复杂度为O(1)。
### 优点
1. 查询快：通过索引，可以很快地定位到数组中的任意一个元素。时间复杂度为O(1)，相对于其他线性存储结构来说，查询效率较高。
2. 支持动态扩容：数组的容量可以根据需要自动扩容。这使得数组的大小可以根据实际情况调整，灵活满足业务需求。
3. 使用方便：数组的实现非常简单，应用广泛，几乎所有的编程语言都提供了对数组的支持。
### 缺点
1. 占用内存过多：对于较大的数组来说，如果每个元素都是简单类型，那么内存占用将是占用空间的三倍以上，因为每个元素都会消耗掉指针和数据所占用的内存。
2. 只能存储一种类型的数据：数组只能存储同一类型的元素，如果要存储不同类型的元素，则需要定义不同的数组。比如，int型数组，float型数组等。

## 顺序存储数据结构——链表
链表是一种无序的动态集合，它的元素通常是按照顺序存储的，但也可以通过指针链接到任何元素。链表可以类比成一条链条，每条链条上的结点表示数据元素，结点之间存在指针关联关系，这样就能构成一个动态的、无序的序列。链表的特点是只保留当前结点和下一个结点之间的指针，不像数组那样会留下存储空间。
### 描述
链表也是一种线性存储结构，但是它在存储上不是连续的，而是在元素间保留着指针，通过指针可以在运行时动态地添加或删除元素。
### 操作
链表支持以下三个基本操作：
1. 尾部追加元素：在链表末尾添加新的元素。时间复杂度为O(1)。
2. 指定位置插入元素：在指定位置插入新的元素。时间复杂度为O(n)。
3. 删除指定元素：删除指定位置的元素。时间复杂度为O(n)。
### 优点
1. 可变长：链表的大小并不固定，随时可以动态变化。
2. 不受内存限制：链表只保留当前结点和下一个结点的指针，因此不受内存大小的限制，能够动态地分配内存。
3. 支持动态修改：链表支持在运行时添加或删除元素，不会影响到其他元素的位置。
### 缺点
1. 随机访问慢：链表没有提供随机访问的能力，只能从头开始遍历，因此在链表的中间或者头部进行查找的时候速度比较慢。
2. 需要额外的指针域：每个结点都需要额外的指针域来指向下一个结点，这增加了结点的开销。

## 栈
栈是一种线性数据结构，其中存放的数据项按先进后出(First In Last Out)的顺序排列。
### 描述
栈又称为堆栈，是一个先进后出的结构。栈的特点是先进的元素在上面，后进的元素在下面，从栈顶压入元素之后再弹出，最后一个压入的元素就是第一个弹出的元素。栈本质上是一个线性表结构，因此栈支持线性表操作的 push、pop 和 peek 操作。
### 操作
栈支持以下三个操作：
1. 入栈：把新元素放入栈顶。时间复杂度 O(1)。
2. 出栈：把栈顶元素移除。时间复杂度 O(1)。
3. 查看栈顶元素：查看栈顶元素。时间复杂度 O(1)。
### 优点
1. FILO 特性：栈遵循“先进先出”(FILO)的原则，也就是说，最后进入栈的元素，最先被移除；而第一个进入栈的元素，最后被移除。因此，栈特别适合于一些操作回退、撤销、替换等场景。
2. LIFO 特性：栈遵循“后进先出”(LIFO)的原则，也就是说，最后进入栈的元素，最先被移除；而第一个进入栈的元素，最后被移除。因此，栈适合于临时存储数据的场合，比如函数调用，子程序返回，表达式运算。
3. 有限容量：栈的大小是有限的，所以当栈满的时候就会出现栈溢出错误 (StackOverflowError)，即栈已满，无法继续容纳新的元素。栈的大小应该设置的足够小，以免栈溢出。
4. 支持动态扩容：栈是线性结构，可根据需要自动扩容。因此，栈在工作过程中不需要事先知道栈的最大容量，它可以自动调整自己的容量。
5. 支持共享数据：栈和它的数据项共用同一块内存，因此栈的实现非常简单，易于线程安全。
### 缺点
1. 内存占用空间大：栈在底层依赖数组实现，数组需要预先分配好空间。当栈满时，栈顶指针后面的元素需要进行移动，这样的话，内存占用就会增加。另外，栈要求元素类型一致，若元素类型较为复杂，还会增加内存的消耗。

## 队列
队列是先进先出的数据结构，典型代表就是排队买票，队首元素在队尾，队尾元素在队首。
### 描述
队列一般用来存放等待处理的任务，比如打印机任务队列。队列的操作有两个端点，分别是入队和出队。队尾加入元素，队头取出元素。队列具有以下性质：
- FIFO(First in First out): 先进先出，队首元素先出队，队尾元素后入队。
- LILO(Last in Last out): 后进先出，队尾元素先出队，队首元素后入队。
### 操作
队列支持以下三个操作：
1. 添加元素：往队列尾部添加元素。时间复杂度 O(1)。
2. 取出元素：从队列头部取出元素。时间复杂度 O(1)。
3. 判断是否为空：判断队列是否为空。时间复杂度 O(1)。
### 优点
1. 队列具有先进先出(FIFO)的特性，适合多人并发环境下对资源的保护和请求调度。比如，当多个用户同时访问数据库时，将资源分配给每个用户前先进入排队队列，确保资源按序被使用。
2. 队列具有多种应用形式，比如生产者消费者模式。
3. 可以保证系统的响应速度，通过积攒一定数量的请求，稳定地处理请求。
### 缺点
1. 消息延迟: 当消息积累过多时，可能会导致某些消息需要等待很久才会得到处理。
2. 消息丢失: 在一些情况下，消息可能会丢失。比如当消费者消费消息的速度大于生产者生产消息的速度时，可能导致消息队列中的消息堆积。
3. 队列容量限制: 如果队列满了，再向队列中添加消息就会发生阻塞，直到队列中的消息被消费掉为止。所以，需要合理设计队列的容量大小，避免因队列满而导致服务不可用。
