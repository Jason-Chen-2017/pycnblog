# 2.基本概念术语说明
## 数据结构的概念
数据结构是指相互之间存在一种逻辑关系的数据元素的集合，它定义了该集合中元素之间的关系和规则，并对集合中数据的访问和操作提供有效的方法。数据结构分为以下几种：

- 线性结构
  - 顺序表
  - 栈
  - 队列
- 非线性结构
  - 树
  - 图
  - 散列表
  - 堆
- 分组结构
  - 数组
  - 链表
  - 队列
  - 栈

## 抽象数据类型ADT(Abstract Data Type)
抽象数据类型(Abstract Data Type)，又称抽象数据类型或泛型数据类型，是指通过一个接口或一个模板定义出来的一种数据类型，而无需指定具体的实现方式。通俗地说就是描述一类数据所具有的性质的集合。抽象数据类型是指对数据中共性特征的定义，而具体的实现则由具体的编程语言来实现。抽象数据类型的概念很重要，因为它有助于简化编程工作，并使算法的设计和实现变得更简单。

## 时间复杂度、空间复杂度、平均时间复杂度、最坏情况时间复杂度
时间复杂度：反映运行一个算法所需要的时间。

空间复杂度：反映运行一个算法所需要的内存空间。

平均时间复杂度：当输入规模的增大时，平均时间复杂度也随之增大。

最坏情况时间复杂度：当输入规模达到某个值时，算法的运行时间最长。
