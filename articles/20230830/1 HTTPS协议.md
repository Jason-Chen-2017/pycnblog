# 1.简介
  

HTTPS（HyperText Transfer Protocol Secure）即超文本传输安全协议，是以安全为目标的HTTP通道，简单来说它是HTTP的安全版，HTTPS先在信息传输的端点（如网页服务器）对传输的数据进行加密，然后再通过Internet发送。它利用公钥私钥密码体制，认证客户端和服务器双方身份，并确保通信过程中的数据安全完整性。


主要特点如下：

- HTTPS协议需要到CA申请证书，一般免费证书较少，因而需要付费购买或自己颁发证书；
- HTTPS是属于TCP/IP四层协议，由于加了SSL/TLS协议，因此速度比http要快很多；
- HTTPS可以有效防止中间人攻击、数据篡改、截获、重放攻击等；
- HTTPS协议中可选的加密套件有很多，比如RSA、ECDHE、DHE等。

HTTPS的作用主要有以下几点：

1. 数据加密：HTTPS协议中采用混合加密机制，将数据经过网络从一端到另一端都加密传输，这样可以在攻击者获取到数据包时，只有被加密的数据才有效果；
2. 身份认证：HTTPS支持基于X.509数字证书认证机构验证用户和服务器的身份，可以确认服务器是否真的是请求方授权提供服务的网站；
3. 数据完整性：HTTPS协议提供数据完整性校验机制，一旦数据被篡改或者破坏，那么传输过程中将会出现错误提示；
4. 数据保密：HTTPS保证了数据传输过程中的隐私安全，用户的信息不会向第三方泄露。

# 2.基本概念术语说明
## HTTP
HTTP（Hypertext Transfer Protocol），超文本传输协议，是互联网上应用最为广泛的协议之一。

定义：HTTP 是一种用于分布式、协作式和超媒体信息系统的请求/响应协议，由 <NAME> 在1989年提出的，目的是允许客户机与服务器之间进行标准化的WWW通信。它规范了浏览器和万维网服务器之间交换数据的规则，包括HTML文档、图像文件、视频片段、音频流、文本等各种各样的信息对象。

HTTP协议工作在TCP/IP协议族上，默认端口号是80，HTTPS协议默认端口号是443。

## SSL/TLS协议
SSL(Secure Socket Layer)/TLS(Transport Layer Security)协议，是Netscape社区开发的一种安全协议，用来提供对网络间通信传输端点的安全通讯服务。目前已经成为事实上的标准协议。

SSL/TLS协议运行在TCP/IP四层之上，处于应用层和传输层之间，负责在两端进行安全通信。它能够实现数据加密、数字签名以及身份验证功能。

## CA证书
CA(Certificate Authority)证书，即数字证书认证机构，是一个非政府组织，它颁发证书给用户。CA是公信力的来源，公众可以很容易地验证CA颁发的证书是不是由可靠的实体签发的。当某个人想要用https访问一个网站的时候，他首先需要访问该网站所在服务器的域名，然后检查它的https证书，确定证书是否由可靠的CA颁发的，如果是，则继续正常访问；否则就显示警告信息，并要求用户进行核实确认。

## 公钥私钥密码体制
公钥密码体制是一种非对称加密技术，加密和解密使用的密钥是两个不同且不相同的密钥。公钥对外发布，私钥对内守护。公钥加密的内容只能通过私钥解密，私钥加密的内容只能通过公钥解密。公钥密码体制的优点在于：

1. 通信双方可以使用不同的密钥对数据进行加密，使得通信更加安全；
2. 可以同时实现客户端和服务器之间的身份认证，解决了单向认证问题；
3. 传送过程中使用公钥加密，接收方收到后再用私钥解密，效率高。

## 混合加密机制
混合加密机制又称分块加密，即将明文分割成固定长度的块（块大小取决于密钥长度），每一块使用不同的密钥进行加密，然后再连接起来。这种方法能够抵御针对某个明文块的单字节解密攻击，也能保证整个消息的完整性。
