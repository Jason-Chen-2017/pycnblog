# 1.简介
  

现代企业对于数据资产的保障和运维管理都越来越复杂，复杂程度是由于当今业务模式和数据处理的需求迅速增长带来的复杂的数据需要。传统的管理理论和方法已经不能适应如此多变的业务和数据处理需求，因此，我们提出了“基于可观察性的管理框架”（Observability-based Management Framework）。该框架从多个角度回顾和总结经验，认为数据资产的管理从根本上是为了服务于业务的目标、实现可预测性，所以如何科学有效地运维数据资产就显得尤为重要。数据资产的维护涉及数据采集、存储、处理、分析等各个环节，在不同的应用场景中存在差异很大的复杂性。因此，我们提出了一个新框架——“Everyday Data Infrastructure Maintenance for Big Organizations”，旨在通过“可观察性+数据质量”来建立对数据资产维护的科学机制。
# 2.基本概念术语说明
## 可观察性
可观察性（Observability）是指一个系统或组件所提供的一组性能指标，可以帮助我们了解系统运行状态、问题排查、可靠性分析、优化方向等。数据可观察性（Data Observability）也称为数据监控能力，其目的是监控数据流经系统的各个阶段，包括数据产生、传输、保存、检索、分析等。可观察性的设计原则是：能够从外部观察到系统内部工作原理和运行情况，能够清楚地理解系统中的所有变量，并能够准确快速地识别和定位问题。传统的管理理论、工具和方法无法满足这个要求，因为它们主要关注那些不能被直接观察到的隐私或不确定性，而忽略了有价值的数据、上下文和行为。因此，我们需要一种全新的管理框架来管理数据资产，它应该由四个层次构成：第一层是“观察（Observe）”，即运用可观察性技术来收集、整理、分析和展示数据。第二层是“知晓（Understand）”，即通过对数据的分析、归纳、比较等，推导出更加详细、全面和客观的数据信息。第三层是“共享（Share）”，即利用数据开放接口，将数据资产输出给合作伙伴，让他们参与到数据价值的生产和消费过程当中，促进数据价值和品牌价值的共赢。第四层是“实施（Implement）”，即采用一系列运营策略，让数据资产真正贡献于组织的业务目标。

## 数据资产
数据资产（Data Asset）是指作为信息基础设施的一部分，用于支持组织业务活动的信息资源，一般包括数据库、文件、报告、模型、应用程序、数据源、API等。数据资产的生命周期通常分为开发期、过渡期和终止期三个阶段，其中开发期包括数据开发、数据集市建设、数据产品构建等环节，过渡期包括数据集成、数据应用移植等环节，终止期包括数据清理和备份等环节。数据资产的运维管理非常重要，它负责数据的收集、处理、存储、查询、分析等工作，实现业务的顺利完成。但同时，它也面临着日益增加的数据规模、复杂性和不可避免的性能、稳定性、安全性等挑战。
