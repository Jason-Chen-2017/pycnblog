# 2.基本概念术语说明
## 什么是高可用性（High Availability，HA)？
高可用性（HA），是指在一个正常运行的状态下仍然保持运行的能力。它通常是指一个系统或系统组件能够在某个时间段内提供有效工作，但可能会受到各种故障或非预期的影响而出现中断或者失效的情况。对于企业级应用系统来说，高可用性意味着系统的稳定性和持续的业务服务能力。如果不能保证应用系统始终可用，那么就不能满足企业对客户的真实需要。因此，企业级应用系统都应该具备高可用性，确保其正常运行，不会因任何原因导致服务中断或异常终止。
## 什么是持续性（Disaster Recovery，DR）？
持续性（DR），是指应对突发事件或灾难时，系统及数据可以按照设定的业务规则及目标持续运行，以便用户及其他系统应用能够继续使用。持续性的主要目的就是实现业务连续性。通过将系统和数据复制到不同位置，并保证系统的可用性，实现持续性，可以最大限度地减少业务中断。对于企业级应用系统来说，持续性不仅仅是在发生突发事件时才需要考虑，而且在业务发展的过程中也需要持续关注、保障系统的正常运行。
## 为什么要做数据库高可用性与性能优化？
由于互联网应用系统发展的快速性和规模化，使得数据库的计算、存储和网络资源在不断增加，为了保证数据库的高可用性与持续性，数据库管理员必须采取措施，根据业务需求对数据库进行定期维护和优化，才能让数据库始终处于正常运行状态。
所以，数据库高可用性与性能优化是企业级应用系统设计的一项重要环节，也是解决性能问题、提升业务并发能力的有效手段。基于这两方面，本书将重点阐述SQL Server 2019在高可用性与性能方面的优化方法和技巧。
