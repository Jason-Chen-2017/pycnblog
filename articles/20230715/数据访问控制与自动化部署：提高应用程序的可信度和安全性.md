## 数据科学/人工智能时代背景下，数据的收集和处理成为了中心任务之一。数据处理过程一般需要保障数据安全、可用性和一致性。但是，越来越多的数据集中带有敏感信息，如何保证数据的安全、可用性、一致性以及隐私权受到尊重？这些都需要数据访问控制（Data Access Control）来解决。数据访问控制是在企业内部为用户提供有限权限的数据访问方式，通过控制数据对不同业务角色和人员的使用方式，来提升企业数据安全、可用性及其真实性。

随着数据科学/人工智能的兴起，数据获取途径日益广泛，比如从社交媒体、IoT设备等进行采集、爬取、传输等。当这些数据中所包含的个人身份信息（PII）被泄露出去，例如个人手机号码、住址、电子邮件地址等，具有严重隐私和商业风险。因此，在数据安全和个人信息保护领域，数据访问控制（DAC）成为一个重要的技术需求。DAC可以帮助企业管理个人信息，提升数据安全和隐私权，使公司的数据更加可用。

## DAC 的目的及意义

1. 数据拥有者：拥有数据的主体
2. 数据使用者：使用数据所涉及到的所有实体
3. 数据使用场景：各种各样的应用场景，如金融交易、业务决策、人员招聘、广告展示、营销推送等。
4. 授权模式：授予特定级别的使用权限，如只读、只写、只审查、读写等。

DAC 有以下作用:

1. 数据安全性：确保数据只能被授权的角色或用户访问，防止数据泄露或恶意攻击。
2. 可用性：确保数据能够被有效使用，不被篡改、丢失、修改或删除。
3. 数据真实性：保证数据准确无误，不得出于任何目的擅自修改、销毁。
4. 隐私权：保证数据只有数据拥有者才可获得，其他任何人都不可得知。

因此，DAC 是确保数据被正确、合法、安全地处理的重要手段。它能有效保障个人信息的隐私、数据安全和合规性。

## 相关概念

1. 个人信息：指以直接或间接方式识别特定個体的信息，包括姓名、地址、电话号码、身份证号、银行账号等；属于公开信息，任何人均可获悉。
2. 数据安全：是指保护计算机系统、网络系统及相关数据不遭受恶意攻击和破坏，从而为合法用户提供必要的服务，并保持正常运行状态。
3. 数据可用性：是指企业所存储、处理、传输的数据能否满足合理的查询和分析要求，同时也要确保数据能够被有效、完整、及时的提供给用户。
4. 数据真实性：是指数据是否符合预期且没有发生任何错误，即所呈现的数据是否可靠、真实、准确。
5. 授权模型：是一种定义数据访问权限的方式，用于确定哪些用户、角色或进程对数据的不同类型访问（如读取、写入、修改、审查、打印、传输等）具有什么样的权限。

# 2.基本概念术语说明

## 概念

- **敏感数据**：指与个人生活息息相关的数据，例如个人生物信息、照片、医疗记录、地理位置数据等。
- **数据分类**：将敏感数据按其敏感度分为两类：
  - 一类是**基础数据**：可以理解为数据主体（如人员、组织机构）无法对外共享的数据，如身份证号、手机号码、出生日期等。
  - 另一类是**敏感数据**：与个人生活息息相关的数据，如车辆数据、工作信息、财产、健康信息等。敏感数据需要对外共享，但不能完全公开，需要对数据使用者进行适当约束。
- **个人信息**：指以直接或间接方式识别特定个体的信息，主要包含个人身份信息（PII），如姓名、地址、电话号码、邮箱等，也包括个人财产信息、宗教信仰、出入境历史、居留国籍、婚姻家庭、教育程度、职业信息等。
- **数据信任边界**：确定个人信息数据可以流动至何处，是隐私权保护的一个关键环节。数据信任边界是指个人信息数据与其他信息数据之间的界线，该线定义了数据主体对于其他信息数据所持有的权利和义务，界线越小则信息泄露可能性越高，反之亦然。数据信任边界越宽则个人信息数据存在风险越低，但对个人信息保护的影响力也越大。通常情况下，个人信息数据仅存在于法定监管部门内，而跨部门共享则会受到相关法律法规的约束。
- **数据元（Data Element）**：数据元素是指那些能够唯一标识个人信息的个人信息项，如全名、性别、出生日期、年龄、联系方式等。数据元的数量决定了数据的复杂度和价值。
- **数据主体**：指个人或事物，在特定的环境和条件下所生成、制造或接收的有关个人信息。
- **数据源**：数据源是指个人信息的产生和经由的途径，如人员调查、笔记录入、专业记录、商业行为、网上浏览记录、设备采集、移动应用上传等。
- **数据安全和个人信息保护法律法规**：是指国际、国内法律、法规和规范，保护个人信息的安全和保密，为个人信息主体提供充分、公平、有效的数据使用权利，促进信息资源的有效利用和保护。
- **数据共享**：指两个或多个数据主体之间数据传递和转移。
- **数据使用者**：使用数据的人群，包括个人、单位、政府、社会团体等。
- **数据使用场景**：指不同阶段、不同业务领域和不同用户类型所需要的数据。
- **授权机制**：指根据个人信息的实际使用情况和场景，向数据主体提供足够权限以实现数据的安全、可用、真实和合法使用，并且需要严格遵守法律法规和秩序。
- **数据泄露**：指由于计算机系统、通讯网络或者其他电子设备缺陷、泄露、盗窃、篡改等因素导致的数据泄露，包括个人信息、金融信息、商业机密、制度信息等。
- **加密算法**：加密算法又称密码编码算法(cryptographic algorithm)，是一种把明文变换成其对应信息摘要的算法，用于将信息变换成未知形式，如字母表中的文字，用于区别数字、音乐、视频、照片等。加密算法包括对称加密算法、非对称加密算法、Hash函数等。
- **敏感数据类型**：指能够识别个人身份、财产信息或其它信息的敏感数据类型，如身份证号、银行卡号、车辆识别信息、民航相关信息、照片等。
- **数据虚拟化技术**：是指采用虚拟化技术将敏感数据划分为虚拟数据，并赋予使用者一些限定使用范围的使用权限，以减少数据泄露的风险。
- **数据提取技术**：是指采用规则、机器学习、数据库查询等技术从数据源中提取有价值、有效信息。
- **数据转换技术**：是指将多个格式或结构相同的数据转换为统一格式，以便进行集成、分析、处理等。
- **数据清洗技术**：是指对收集的原始数据进行预处理，对不完整、无效、错误的数据进行修正、清除。
- **脱敏规则**：是指对敏感数据进行掩盖，将真实信息隐藏起来，如对手机号码进行加长，或对姓名进行缩写。
- **数据库加密技术**：是指采用密钥加密的方法加密数据库中的敏感信息。
- **认证技术**：是指通过某种手段验证数据主体的真实性、合法性和完整性，包括手动检查、基于密钥的认证、动态令牌、短信验证码、双因子认证等。
- **访问控制列表**：是指数据主体和他们在系统中的访问权限，包括允许哪些人员和系统可访问哪些敏感数据。
- **强制访问控制**：是指数据访问控制系统将数据的所有者与数据使用者绑定，当数据所有者对数据使用者授予权限后，其对数据本身的使用就只能由数据使用者完成。
- **基于角色的访问控制**：是指按照使用者的职责分组，每个组定义了一系列访问权限，例如，管理员组只能查看和修改关键数据的权限，普通用户组只能查看数据的权限，这样可以有效保障数据主体的权益。
- **基于属性的访问控制**：是指根据个人信息的特定属性，对不同的信息使用者授权。例如，限制公共访问或对指定地区的访问只读。
- **基于加密的访问控制**：是指对敏感数据使用访问控制前先进行加密处理，加密后的数据只能被授权的人才能解密，不能阅读原始数据。
- **自动化部署技术**：是指利用DevOps和自动化工具对敏感数据流水线进行部署和管理，以提升敏感数据安全性、可用性和一致性。
