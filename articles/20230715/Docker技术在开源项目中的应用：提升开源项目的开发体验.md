## 1.1 什么是Docker？
[Docker](https://www.docker.com/)是一个开源的应用容器引擎，它是轻量级的虚拟化平台。通过创建独立运行环境，可以简化部署流程、加快交付效率，并节省系统资源。相比传统虚拟机方式，Docker容器具有以下优点：
- 更高效的利用系统资源：由于容器不需要进行硬件虚拟化，因此占用系统资源更少，启动速度也更快。
- 更轻松的迁移和扩展：由于容器内已经包含了所有运行环境，因此无须再安装操作系统，可直接移植到其他机器上。同样地，多个容器之间可以共享资源，有效提高资源利用率。
- 更便捷的制作和发布：Dockerfile文件提供了一种定义可执行环境的简单方法，使得应用的构建、测试和分发都变成一件轻松而自动化的事情。
- 更加灵活的调度策略：Docker提供动态管理能力，允许管理员将容器动态分配给系统资源。甚至可以在服务器发生故障时自动重启容器，保证应用的持续运行。

除了以上应用场景，Docker还有很多其它优秀特性，比如安全性高、镜像仓库分发速度快、自动化编排、微服务等。

## 1.2 为什么要使用Docker？
使用Docker的主要原因有以下几点：
- 可复现环境：通过Dockerfile和镜像，可以实现应用的可复现环境，降低开发、测试和部署环节的复杂度。
- 节省时间和精力：通过Docker，可以快速搭建起应用所需环境，大幅缩短开发周期，节省宝贵的时间。
- 提升效率：Docker让应用的打包、测试和发布都变得十分简单和自动化，大大提升了工作效率。
- 减少资源开销：容器之间共享资源，有效提高资源利用率。
- 避免兼容性问题：通过定制镜像，可以消除不同环境间的差异，解决依赖版本的问题。

# 2.基本概念术语说明
本章节介绍Docker相关的一些基本概念和术语。
## 2.1 镜像（Image）
Docker镜像是一个只读的模板，类似于一个ISO镜像一样，里面包含了某个软件的整个软件栈。你可以创建自己的镜像或者下载别人的镜像。一般来说，镜像包括运行某个软件所需要的二进制文件、配置文件、脚本、库文件等。
## 2.2 容器（Container）
容器是一个运行中的程序，是镜像的实体。你可以创建、启动、停止、删除容器。每个容器都是相互隔离的、自包含的，但它们可以访问宿主机上的相同的文件系统，也可以通过网络通信。
## 2.3 Dockerfile
Dockerfile是一个文本文件，其中包含了一条条的指令用来构建一个镜像。这些指令包括从基础镜像开始、指定标签、设置环境变量、安装软件包、复制文件、设置启动命令等。Dockerfile通常位于一个空白文件夹中，并以“Dockerfile”作为文件名。
## 2.4 注册中心（Registry）
镜像仓库又称为注册中心，用于存储Docker镜像。目前主流的镜像仓库有Docker Hub、Quay、Google Container Registry和阿里云容器镜像服务。镜像仓库是用来保存和分发Docker镜像的场所，通过镜像仓库，我们可以很方便地分享和使用别人的镜像。
## 2.5 Docker Compose
Compose是一个基于YAML文件的工具，用于定义和运行多容器Docker应用程序。Compose通过定义环境变量、卷、端口映射、外部依赖关系等参数，然后将这些不同的配置组合起来，生成最终的运行配置。通过Compose，可以快速启动和关闭应用程序的所有容器，还可以管理它们的生命周期。
## 2.6 Kubernetes
Kubernetes是一个开源的容器编排系统，可以管理容器集群中的容器化应用。其核心功能包括Pod的管理、Service的发现和负载均衡、存储卷管理、在线伸缩、密钥和证书管理、日志查看、监控告警等。
## 2.7 OCI（Open Containers Initiative）
OCI（Open Containers Initiative）是一个开放联盟，致力于创建一个开放、标准化的容器规范和架构，旨在通过开放流程促进容器标准的发展和落地。OCI也是一个兼容的、可移植的容器运行时标准，是Docker、rkt、Rocket、LXD等各种容器运行时技术的参考实现。
