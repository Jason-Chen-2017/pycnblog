# 2.核心概念与联系

在分布式系统中，服务降级策略的核心概念包括：

- 服务降级：服务降级是一种动态的性能调整手段，用于在系统出现故障时，动态地降低服务的性能，以避免系统崩溃或者出现严重的性能问题。
- 服务限速：服务限速是一种限制服务性能的手段，用于避免系统资源被耗尽，从而保证系统的稳定运行。
- 服务拒绝请求：服务拒绝请求是一种拒绝处理某些请求的手段，用于避免系统资源被耗尽，从而保证系统的稳定运行。

这些概念之间的联系如下：

- 服务降级是服务限速和服务拒绝请求的一种具体实现手段。
- 服务限速和服务拒绝请求是服务降级策略的一部分，用于实现服务降级的目标。
