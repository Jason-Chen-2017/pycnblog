# 2.核心概念与联系

在这一部分，我们将详细讲解智能手机和移动支付的核心概念。

## 2.1 智能手机

智能手机是一种具有高级功能的手机，它可以运行多种应用程序，并且可以通过互联网进行数据传输。智能手机的核心组成部分包括：

- 操作系统：智能手机需要一个操作系统来管理硬件资源和应用程序。常见的操作系统有Android、iOS等。
- 处理器：智能手机需要一个处理器来执行各种任务。处理器的性能会影响智能手机的运行速度和功耗。
- 内存：智能手机需要内存来存储应用程序和数据。内存的大小会影响智能手机的运行速度和存储能力。
- 存储：智能手机需要存储来存储应用程序和数据。存储的大小会影响智能手机的存储能力。
- 显示屏：智能手机需要显示屏来显示用户界面和内容。显示屏的大小和分辨率会影响智能手机的显示效果。
- 摄像头：智能手机需要摄像头来拍照和录像。摄像头的分辨率和功能会影响智能手机的拍照和录像效果。
- 通信模块：智能手机需要通信模块来进行通信。通信模块的类型和功能会影响智能手机的通信能力。

智能手机的发展历程可以分为以下几个阶段：

- 第一代智能手机：这些手机使用了基于Java的操作系统，例如Nokia的9210 Communicator。
- 第二代智能手机：这些手机使用了基于Linux的操作系统，例如Palm的Treo 650。
- 第三代智能手机：这些手机使用了基于Android的操作系统，例如Google的Nexus One。
- 第四代智能手机：这些手机使用了基于iOS的操作系统，例如Apple的iPhone。

## 2.2 移动支付

移动支付是指通过手机进行的支付。移动支付的核心组成部分包括：

- 支付平台：移动支付需要一个支付平台来处理支付请求和结算。支付平台可以是第三方平台，例如微信支付、支付宝等，也可以是商家自建平台。
- 支付方式：移动支付可以使用多种支付方式，例如银行卡支付、支付宝支付、微信支付等。
- 支付渠道：移动支付可以通过多种渠道进行，例如短信支付、扫一扫支付、二维码支付等。
- 支付安全：移动支付需要保证支付安全，例如使用加密技术、身份验证等。

移动支付的发展历程可以分为以下几个阶段：

- 第一代移动支付：这些支付使用了短信支付和扫一扫支付等方式，例如腾讯的腾讯钱包。
- 第二代移动支付：这些支付使用了微信支付和支付宝等第三方平台，例如微信支付、支付宝等。
- 第三代移动支付：这些支付使用了基于NFC的支付方式，例如苹果支付等。

public class PayActivity extends AppCompatActivity {

    private static final int SDK_PAY_FLAG = 1;

    private String orderInfo;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_pay);

        orderInfo = "支付金额：10元，订单号：1234567890";

        pay();

    private void pay() {
        PayTask payTask = new PayTask(this);
        String result = payTask.pay(orderInfo, true);

        Toast.makeText(this, result, Toast.LENGTH_LONG).show();

    @Override
    protected void onNewIntent(Intent intent) {
        super.onNewIntent(intent);
        setIntent(intent);
        pay();
    }
}
