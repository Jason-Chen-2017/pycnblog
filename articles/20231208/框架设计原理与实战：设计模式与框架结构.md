### 3.1 算法时间复杂度
算法时间复杂度是指算法执行所需时间与输入大小之间的关系。我们通常用大O符号表示算法的时间复杂度。例如，线性搜索算法的时间复杂度为O(n)，其中n是输入大小。

### 3.2 算法空间复杂度
算法空间复杂度是指算法所需的额外空间与输入大小之间的关系。我们也通常用大O符号表示算法的空间复杂度。例如，递归算法的空间复杂度通常为O(n)，其中n是输入大小。

### 3.3 算法稳定性
算法稳定性是指算法在对相同数据进行排序时，对于相等的元素，其排序结果是否保持不变。例如，快速排序算法是不稳定的，因为它可能会改变相等元素的顺序。

### 4.1 代码实例1：简单的框架设计
在这个代码实例中，我们将设计一个简单的框架，用于计算两个数的和。我们将使用面向对象编程的方式来实现这个框架。

```python

    def __init__(self):
        pass

    def add(self, a, b):
```

在这个代码实例中，我们定义了一个Calculator类，它有一个add方法用于计算两个数的和。我们创建了一个Calculator对象，并调用其add方法来计算两个数的和。

### 4.2 代码实例2：复杂的框架设计
在这个代码实例中，我们将设计一个复杂的框架，用于实现一个简单的计算器应用。我们将使用设计模式来实现这个框架。

```python
from abc import ABC, abstractmethod

class Calculator(ABC):
    @abstractmethod
    def add(self, a, b):
        pass

    @abstractmethod
    def subtract(self, a, b):
        pass

    @abstractmethod
    def multiply(self, a, b):
        pass

    @abstractmethod
    def divide(self, a, b):
        pass

class BasicCalculator(Calculator):
    def add(self, a, b):
        return a + b

    def subtract(self, a, b):
        return a - b

    def multiply(self, a, b):
        return a * b

    def divide(self, a, b):
        return a / b

calculator = BasicCalculator()
result = calculator.add(2, 3)
print(result)  # 输出：5
```

在这个代码实例中，我们定义了一个Calculator类，它是一个抽象类，包含了四个抽象方法。我们还定义了一个BasicCalculator类，它实现了Calculator类的所有抽象方法。我们创建了一个BasicCalculator对象，并调用其add方法来计算两个数的和。
