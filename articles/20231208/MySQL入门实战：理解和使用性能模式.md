# 2.核心概念与联系
在讨论性能模式之前，我们需要了解一些核心概念。

## 2.1查询性能瓶颈
查询性能瓶颈是指在执行查询时，由于某些原因，查询性能不能达到预期的水平。这可能是由于查询本身的复杂性、数据库结构的不合适或者硬件资源的限制等原因。

## 2.2查询优化
查询优化是指通过对查询计划进行分析和修改，以提高查询性能的过程。查询优化可以包括修改查询语句、创建索引、调整数据库配置等方法。

## 2.3性能模式
性能模式是一种针对特定查询的优化方法，可以帮助我们更好地理解查询的性能瓶颈，并提供一种优化的方法来提高查询性能。性能模式通常包括以下几个步骤：

1. 分析查询计划，找出性能瓶颈。
2. 根据性能瓶颈，提出优化建议。
3. 实施优化建议，并评估性能改善的效果。
