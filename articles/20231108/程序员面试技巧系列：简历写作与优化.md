# 1.背景介绍


为什么要写这个系列？我认为程序员面试是一个非常具有挑战性的工作。虽然每天都会有无数的编程技术问题涌现出来，但是作为一个技术人员，在面试中也需要不断提升自己的能力，提高自己的竞争力。很多时候，面试官很难确定到底应该问些什么问题，更糟糕的是，如果一份候选人的简历做得不好，甚至根本没有通过面试，就会影响到整个团队的薪酬水平。因此，技术博客对面试者的职业发展有着十分重要的作用。

在准备面试时，很多技术面试者都喜欢刷LeetCode题目或者看一些技术类的视频来增加面试题的难度，但这些方法并不能完全解决问题。即使真的被面试官问到了“用什么样的方法、数据结构解决了什么问题”，面试者依然可能因为没有答案而望而却步。为了帮助面试者更好的写出一份优秀的简历，以及顺利的通过面试，我想整合我所知道的面试经验，结合个人的实际情况，为你提供一些建议。希望通过我的分享可以帮助到大家。

本系列共包含以下7个主题：

1. 如何准备简历：从收集信息到编写CV到出版简历，这是一个循序渐进的过程。在准备简历的过程中，我会告诉你一些最基本的事情，比如如何写个性化的推荐信，怎么让你的CV更吸引人等等。同时，还会给出一些非技术相关的建议，比如如何将简历制作成一本书。

2. 算法：这是关于基础算法和数据结构的问题。如果你是计算机科班出身，或之前已经做过系统分析或开发项目，那么你肯定已经知道很多算法。但如果你是新手，又担心自己什么都不会，那么本主题可以帮你复习一下基础知识。我会逐步地讲解各种常见的数据结构，以及它们的应用场景。当然，还有一些算法比较复杂，但它们在一定程度上也是通用的。

3. 操作系统：这是一个了解计算机系统运行机制的非常重要的主题。掌握操作系统的底层机制，才能更好地理解计算机是如何运作的。对于技术人员来说，操作系统几乎占据了所有的资源。在本主题中，我会着重介绍操作系统的组成、调度、内存管理、进程管理等方面的知识。

4. TCP/IP协议栈：TCP/IP协议栈是一个非常重要的网络协议。如果你正在准备面试，也许就有机会问到它。本主题会教你常见的传输控制协议（TCP）和网际协议（IP）是如何工作的，以及它们之间又是如何通信的。同时，我会介绍一些特定协议，如IPv6、VPN等。

5. 数据库：数据库是现代互联网世界的基础设施。在本主题中，我会介绍一些数据库常用技术及其应用场景，以及SQL语言的使用技巧。

6. 设计模式：如果你是一个优秀的架构师，那么本主题必不可少。设计模式是一个经典的话题，它的出现就是为了解决软件工程中的重复问题。在本主题中，我会列举一些经典的设计模式，并展示他们的结构图，告诉你何时该使用何种模式。

7. 消息队列与流处理：消息队列与流处理都是实时计算领域的热门方向。消息队列可以在实时的环境下传递数据，而流处理则可以用来分析数据。本主题会介绍Kafka、Storm等分布式消息队列和Storm、Flink等流处理框架。
