# 1.背景介绍


## 分治法（Divide and Conquer）
分治法是指将一个复杂的问题分成两个或多个相同或者相似的子问题，递归地解决这些子问题，然后再合并其结果，得到原问题的一个解。它的特点就是将一个大问题分成各个小问题进行处理，直到不能分的时候才停止，最后得到原问题的解。分治法一般应用于集合、树形结构和图论等一些具有复杂性的数据类型，是一种高效的解决问题的方法。

## 分治算法分类
分治算法可以根据问题是否可以被划分成多个互相独立的子问题以及子问题之间是否存在重叠，将分治法又分为两类：
- 多项式时间算法：当一个问题可以在多项式的时间内求解时，它就属于多项式时间算法；
- Θ(n)复杂度的算法：该类算法的运行时间的上界是 O(n^c)，其中 c 是某个正整数，称为渐进时间复杂度。例如：快速排序、二分查找、希尔排序、归并排序等都属于此类算法。


## 分治算法的优缺点
### 优点
- 有利于大规模问题的解决。在大型问题中，很多时候可以将其分割成几个规模较小的子问题，利用计算机多处理器并行计算的方法，可以大大缩短算法的执行时间。
- 分治法往往具有较好的空间复杂度，因为只需要存储局部变量和一些中间结果，而不必存储整个输入或输出数组。
- 分治法的递归实现往往比较简洁，便于理解和实现。

### 缺点
- 在设计上要考虑每个子问题的最优子结构，并且减少子问题之间的依赖关系，避免出现所谓的"深而不当"的分治策略，从而导致低效率。
- 不同问题的最优子结构往往不是非常容易找到，因此设计出高效的分治算法并不总是件容易的事情。



# 2.核心概念与联系
## 二分搜索
二分搜索算法用于在有序序列中找出给定值或值的近似值，算法过程如下：
1. 首先设置两个指针，low 和 high ，指向序列的第一个元素和最后一个元素；
2. 根据输入的值和指针位置确定新的区间 [mid]， mid = low + (high - low) / 2；
3. 如果 mid 的值等于输入的值，则返回 mid；
4. 如果 mid 的值比输入值小，则 low = mid+1；
5. 如果 mid 的值比输入值大，则 high = mid-1；
6. 重复第 3～5 步，直至 low > high 为止，此时没有找到目标值；
7. 返回不存在的值 NULL 或负无穷大。

通过比较区间 [mid] 的大小，判断输入值应该在前半段还是后半段继续查找。如果输入值小于 mid 的值，则应该在后半段继续查找，因为输入值可能在右边；反之，应该在前半段继续查找，因为输入值可能在左边。以此类推，直到目标值被找到或越界。

## 分治算法的基本思想
分治算法的基本思想是把一个大问题拆分成若干个相互独立的子问题，分别求解这些子问题，然后再合并其结果，最终得到原问题的一个解。通俗来说，就是将一个问题分割成几个规模较小、与原问题形式相同的子问题，递归地解决这些子问题，最后将这些子问题的解组合起来，即得出原问题的一个解。

分治算法的三种分类：
- 一对一法：把原问题分解为 n 个子问题，每个子问题都和原问题一样大，并且有一个确定的解法。这样的子问题通常是直接处理原问题的子集，比如排序中的子问题是输入的一段连续元素。
- 一对多法：把原问题分解为 n 个子问题，每个子问题都和原问题一样大，但有着不同的解法。通常，子问题之间相互独立，并且可以自己求解。比如矩阵乘法，可以用分治法求解 A * B = C，也可以单独求解 A * b 和 B * a，然后再组合起来。
- 多对一法：把原问题拆解为 n 个更小的问题，但是这些更小的问题没有自己的解法，只有一个确定的答案，就是原问题的解。比如求最小值和最大值，可以把原问题分解为两个子问题——最小值问题和最大值问题，然后分别求解它们的解，最后取其中的一个作为原问题的解。
