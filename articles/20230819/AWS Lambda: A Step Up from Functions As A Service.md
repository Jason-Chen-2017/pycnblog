# 1.简介
  

​		AWS Lambda (λ) 是亚马逊云计算服务（Amazon Web Services）中的一项功能。它是一个运行在云端的无服务器函数计算服务。通过Lambda你可以运行代码而不必担心服务器管理或扩容的问题。你可以创建轻量级的函数来处理事件触发、定时执行或者从其他服务中响应数据变化。你可以选择在任何 AWS 服务上运行 Lambda 函数，包括 Amazon S3 和 Amazon API Gateway。除了无需管理服务器的弹性伸缩，还可以按需付费，降低了运行成本。Lambda最初被设计为支持高并发事件处理工作负载。目前，Lambda正在扩展到包括后台任务，图像识别/处理和机器学习等领域。

# 2. 基本概念术语说明
## 2.1 概念及应用场景
- lambda表达式：一个lambda表达式是一个匿名函数，他只定义了一个表达式，当调用这个函数时，就自动执行这个表达式。表达式一般比较简单，仅限于一些简单运算或逻辑判断。例如： `(x+y)` 或 `if x>y then return true else false` 。

- function：函数是计算机编程的基本单元，它接受输入参数并返回输出结果。通常情况下，函数要有入参和出参。lambda表达式就是一种特殊类型的函数，它没有名称，只有表达式。Lambda表达式的作用类似于普通函数一样，可以作为独立模块使用，也可以赋值给变量进行调用。

- trigger：触发器是指执行Lambda函数的事件。Lambda函数可以绑定触发器，当指定的触发器发生的时候，Lambda函数就会被激活。

- event source：事件源是指产生触发器的对象。例如，API网关可以作为触发器之一，当API网关接收到请求时，Lambda函数会被触发。消息队列可以作为触发器，当有新的消息进入队列时，Lambda函数才会被触发。

- runtime：运行环境是指Lambda函数在执行过程中需要使用的运行环境，例如Java、Python等。每种语言都有对应的runtime。

- handler：处理器是指Lambda函数实际执行的主体。处理器是一个可执行文件，它主要负责读取event data，执行用户的代码，然后生成输出结果。它可以是本地可执行文件，也可以是云端执行环境。

- IAM role：IAM角色（Identity and Access Management Role）是用于授予Lambda函数权限的机制。角色是一个非常重要的组件，它控制着Lambda函数对其他资源的访问权限。角色是IAM中非常抽象的概念，你可以将角色看作是Lambda函数的身份，角色决定了Lambda函数可以访问哪些资源，以及有什么权限。

- VPC：虚拟私有云（Virtual Private Cloud）是一个能够提供网络隔离的容器，你可以利用VPC让你的Lambda函数与其他AWS服务之间的通信保持安全。

- Cold start time：冷启动时间是指第一次执行Lambda函数的时间。由于Lambda函数是按需计费，所以第一次执行的时间往往较长。冷启动时间与初始化时间、执行代码的时间都密切相关。

- code package：代码包是指Lambda函数运行所需要的代码和配置。它包含了函数的入口点、依赖库、配置文件等。

## 2.2 配置
### 创建Lambda函数
首先登录到AWS Console ，点击左侧导航栏中的“服务”，搜索并选择“Lambda”。
然后点击“创建函数”按钮，按照提示设置函数名称、描述、运行时环境、角色、VPC、触发器等信息。
### 编辑Lambda函数
你可以编辑Lambda函数的名称、描述、运行时环境、角色、VPC、内存大小、超时时间等配置。
### 发布版本
当你完成函数的配置后，你可以点击“发布新版本”来部署最新代码。默认情况下，每个版本都会关联到一个别名，可以通过该别名指定特定的版本。你也可以删除旧版本。
### 配置触发器
你可以添加多个触发器，使得函数在不同的条件下被执行。例如，你可以创建一个S3上传成功触发器，当S3存储桶中的文件上传完毕后，该函数将会被激活。你也可以添加API Gateway触发器，使得函数在HTTP请求时被执行。

## 2.3 使用Lambda函数
### 执行Lambda函数
你可以手动执行Lambda函数，也可以通过触发器自动执行。当你执行某个函数时，系统会根据函数配置的触发器类型进行相应的操作。比如，当触发器类型为S3上传成功，那么系统会触发S3存储桶上的文件上传事件；当触发器类型为API Gateway，那么系统会发送HTTP请求到API Gateway API的路径上。
### 查看日志
你可以查看Lambda函数的日志，包括函数的执行时间、内存使用情况、执行状态、执行输入、输出等。
