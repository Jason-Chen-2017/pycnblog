# 1.简介
         

         ## 概述
         JavaScript中有很多内置的处理函数方法，比如Array对象的`forEach()`、Object对象的`keys()`、`values()`等都是由JavaScript运行时环境直接提供的功能。但是，对于一些比较特殊或者更高级的需求，JavaScript的标准库还提供了相应的方法，这些方法都需要通过调用内置函数实现。比如说，数组的遍历方法除了 forEach() 方法外，还有 Array.prototype[Symbol.iterator]() 的 Symbol.iterator 属性，可以通过调用这个属性返回迭代器对象，然后对其进行遍历。ES6中的 Map 和 Set 数据结构也提供了额外的方法，比如forEach() 方法，用于遍历 Map 或 Set 中的所有元素。这样看起来，似乎所有这些高级的数据结构的方法都可以用类似于 forEach() 的方式来进行处理。但是，实际上，这些方法还是会调用相应的内置函数或其他 API 来实现。
         
         在本文中，我们将要讨论一下另外一个内置方法：Reflect.get() 。Reflect.get() 方法是一个用来读取对象的属性（property）的内置函数，它支持多种参数形式，包括 name 参数指定要访问的属性名，receiver 参数指定要从哪个对象中获取属性，并在没有找到该属性时返回 fallback 参数的值。Reflect.get() 是 Proxy 对象使用的底层方法，很多 Proxy 对象的方法都会调用 Reflect.get() 函数。因此，了解 Reflect.get() 的工作机制十分重要。
         
         通过阅读本文，你可以学习到以下知识点：
         
         1. 什么是 Reflect？为什么要有 Reflect？
         2. 如何利用 Reflect 获取属性？
         3. 如何自定义属性访问行为？
         4. Proxy 使用场景及原理介绍
         5. Promise 使用场景及原理介绍
         6. Generator/Iterator 使用场景及原理介绍
        ...
         7. 更多待补充...
         
         ## 总结
         本文主要介绍了Reflect.get() 方法的作用和使用，分析了 Reflect.get() 是如何实现的，并且根据不同的参数组合，详细阐述了 Reflect.get() 的各种用法。希望能够帮助读者理解 Reflect.get() 方法背后的原理，更好的运用其来优化自己的代码。
