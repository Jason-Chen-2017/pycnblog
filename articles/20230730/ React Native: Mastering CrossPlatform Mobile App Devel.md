# 1.简介
         
 在React Native出现之前，开发者们只能通过原生代码开发移动应用，而不能跨平台运行在Android、iOS、Windows Phone、Firefox OS等多个平台上。React Native项目提供了一种可用于开发跨平台移动应用的方法，让你只编写一次代码，然后将其编译成各个平台支持的原生应用，你可以用JavaScript或TypeScript语言来编码你的APP，还可以利用npm包管理工具进行模块化开发。
          
          ## 1.1 什么是React Native？
          
          React Native是一个开源的移动开发框架，它允许使用JavaScript来开发iOS和安卓应用，甚至可以在Windows、MacOS和Linux等其他平台上运行。该框架使用自己的JS运行时环境，而不是浏览器中的JavaScript。

          ## 1.2 为什么要用React Native？
          
          ### 1) React Native使前端开发人员能够编写原生应用

          大多数前端开发人员从事的是网站开发，但React Native不仅仅局限于此，它的目的是提供一个统一的开发体验，让前端开发人员能够构建具有良好性能的原生移动应用。你可以利用React Native快速地创建自定义用户界面，并且集成第三方库来实现功能强大的功能。

          ### 2）React Native能跨平台运行

          使用React Native你可以创建适用于任何设备的移动应用。当你用JavaScript和React Native创建一个应用后，你可以编译成针对不同平台的原生应用程序。因此，你可以同时发布iOS和安卓版本的应用，还可以打包到其他平台，比如Windows、MacOS和Linux等。

          ### 3）React Native有很好的性能表现

          Facebook工程师使用React Native创造了名为“F8会议”的大型移动应用，该应用为全球超过50万人提供了高质量的视频会议体验。Facebook声称，它使用的React Native框架比其它所有框架都要快得多。虽然Facebook没有透露他们如何优化应用的性能，但是它已经证明React Native能够为移动应用带来出色的性能。

          ### 4）React Native社区活跃

          React Native社区非常活跃，有很多优秀的资源、教程和示例。开发者们经常能够找到他们需要解决的问题的答案，或者找到合作伙伴们分享的最佳实践。如果遇到了问题，也能得到及时的帮助。

          ### 5）React Native易于学习

          如果你是一位熟悉JavaScript的前端开发人员，那么学习React Native应该不是一件难事。该框架本身就是用JavaScript编写的，并且有丰富的文档和教程。另外，React Native提供了一些社区驱动的插件，让你能够轻松添加各种功能。

          ## 1.3 总结
          
          通过阅读本文，我们了解到React Native是什么，为什么要用它，以及它有哪些优点。我们还介绍了React Native的基本用法。对今后的学习和工作十分有益。希望本文对大家有所启发！感谢您的阅读！
