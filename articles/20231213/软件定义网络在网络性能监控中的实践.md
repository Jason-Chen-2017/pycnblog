# 2.核心概念与联系
在本节中，我们将介绍软件定义网络（SDN）的基本概念和与网络性能监控的联系。

## 2.1 软件定义网络（SDN）
软件定义网络（SDN）是一种新型的网络架构，将网络控制平面和数据平面分离，使网络控制逻辑可以通过软件实现。这种设计使得网络更加灵活、可扩展和易于管理。SDN的核心组成部分包括控制器、数据平面和应用层。控制器负责管理网络的状态和决策，数据平面负责实际的数据传输，应用层提供了各种网络功能和应用。

## 2.2 网络性能监控
网络性能监控是一种用于评估网络性能的方法，通过收集网络的各种指标，如延迟、吞吐量、丢包率等，以便进行性能分析和优化。网络性能监控的目的是为了确保网络的稳定性、可用性和性能。

## 2.3 SDN在网络性能监控中的联系
SDN技术在网络性能监控方面具有以下优势：

1. 通过将控制逻辑抽象为软件，SDN使得网络性能监控更加灵活和可扩展。
2. SDN提供了更好的可视化和分析功能，使得网络管理员可以更快地发现和解决性能问题。
3. SDN技术可以与各种性能监控工具集成，提供更丰富的性能监控功能。
