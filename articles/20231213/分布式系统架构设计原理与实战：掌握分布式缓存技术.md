## 1.1 背景介绍

分布式缓存技术的诞生和发展与互联网企业业务的快速发展紧密相关。随着互联网企业业务的快速发展，企业的业务系统需求也在不断增加，这导致了企业业务系统的性能、可扩展性、可靠性等方面的要求也在不断提高。为了满足这些需求，企业开始考虑使用分布式缓存技术来提高企业业务系统的性能、可扩展性、可靠性。

分布式缓存技术的核心思想是将企业业务系统中的一些数据存储在分布式缓存系统中，以便在企业业务系统需要访问这些数据时，可以直接从分布式缓存系统中获取数据，而不需要从数据库中获取数据。这样可以减少企业业务系统与数据库之间的网络传输开销，从而提高企业业务系统的性能。

分布式缓存技术的核心组成部分包括：分布式缓存系统、分布式缓存服务、分布式缓存协议、分布式缓存算法等。

        this.cacheServices = cacheServices;
    }

    public void put(String key, Object value) {

        // 存储数据
        cacheService.put(key, value);

    @Override
    public Object get(String key) {
        // 根据一致性哈希算法计算key的哈希值
        int hashValue = MurmurHash3.hash(key.getBytes());

        CacheService cacheService = findCacheService(hashValue);

        // 获取数据

- 缓存穿透：可以通过设置缓存空值和缓存空值的有效时间来减少缓存穿透。
- 缓存击穿：可以通过设置缓存过期时间和缓存重建策略来减少缓存击穿。
- 缓存雪崩：可以通过设置缓存过期时间和缓存重建策略来减少缓存雪崩。
