# 2. 核心概念与联系

在数据查询中，我们需要考虑以下几个核心概念：

1. 查询语言：查询语言是用于描述查询需求的语言。例如，SQL是一种流行的查询语言，用于查询关系型数据库。NoSQL是另一种查询语言，用于查询非关系型数据库。

2. 查询计划：查询计划是用于描述如何执行查询的方法。查询计划可以是预先定义的，也可以是动态生成的。例如，SQL查询可以使用预先定义的查询计划，而NoSQL查询可以使用动态生成的查询计划。

3. 查询参数：查询参数是用于描述查询需求的变量。例如，SQL查询可以使用查询参数来描述查询条件，而NoSQL查询可以使用查询参数来描述查询范围。

4. 查询结果：查询结果是查询需求的输出。查询结果可以是数据集、图表、图像等。例如，SQL查询可以输出数据集，而NoSQL查询可以输出图表。

5. 查询效率：查询效率是用于描述查询需求的执行速度。查询效率可以是查询时间、查询空间、查询资源等。例如，SQL查询可以使用查询时间来描述查询效率，而NoSQL查询可以使用查询空间来描述查询效率。
