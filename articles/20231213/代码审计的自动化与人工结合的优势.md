                 

# 1.背景介绍

随着软件开发的不断发展，代码审计在软件开发过程中扮演着越来越重要的角色。代码审计可以帮助开发者发现代码中的潜在问题，提高代码质量，减少bug，提高软件的安全性和可靠性。

在传统的代码审计方法中，人工审计是主要的方法之一。人工审计通常由专业的软件工程师或开发者进行，他们会根据一定的规则和标准来审查代码，以确保代码符合预期的质量和安全性。然而，随着代码规模的增加，人工审计的效率和准确性都受到了严重的限制。

为了解决这个问题，近年来，自动化代码审计技术逐渐成为了研究者和开发者的关注焦点。自动化代码审计通过使用计算机程序来检查代码，可以在较短的时间内检测出更多的问题，从而提高审计效率。然而，自动化代码审计也有其局限性，它可能无法理解代码中的复杂逻辑，或者误判某些问题为错误。

因此，为了充分利用人工审计和自动化审计的优点，研究者们开始探索代码审计的自动化与人工结合的方法。这种方法通过将人工审计和自动化审计结合起来，可以在保持审计准确性的同时，提高审计效率。
