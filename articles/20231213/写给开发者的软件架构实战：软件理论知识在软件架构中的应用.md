# 2.核心概念与联系
在软件架构中，我们需要关注以下几个核心概念：

- 模块化：将软件系统划分为多个模块，每个模块负责特定的功能。模块化有助于提高代码的可读性、可维护性和可重用性。
- 抽象：抽象是将复杂的系统简化为更简单的模型，以便更容易理解和操作。在软件架构中，抽象可以帮助我们将复杂的系统划分为更小的部分，从而更容易进行设计和实现。
- 分布式系统：分布式系统是由多个独立的计算节点组成的系统，这些节点可以在网络中进行通信和协作。在软件架构中，分布式系统的设计需要考虑如何实现高可用性、高性能和高可扩展性。
- 数据存储：数据存储是软件系统中的一个关键组件，用于存储和管理数据。在软件架构中，我们需要选择合适的数据存储方案，以满足系统的性能和可扩展性需求。
