# 2.核心概念与联系

在本节中，我们将从核心概念和联系的角度，深入了解SQL查询和事务处理的基本概念。

## 2.1 SQL查询的基本概念

SQL查询是数据库系统的核心功能之一，用于查询和操作数据库中的数据。SQL查询的基本概念包括：

- **查询语句**：查询语句是SQL查询的核心部分，用于定义查询的逻辑和语法。查询语句包括SELECT、FROM、WHERE、GROUP BY、HAVING、ORDER BY等子句。
- **数据库表**：数据库表是数据库中的基本数据结构，用于存储和管理数据。数据库表由一组列组成，每个列表示一种数据类型。
- **数据库索引**：数据库索引是数据库中的一种数据结构，用于加速查询操作。数据库索引通过创建一个数据结构，将数据库表中的数据排序，从而加速查询操作。
- **查询结果**：查询结果是SQL查询的输出结果，包括查询的数据和查询的结构。查询结果可以是一行一列的数据，也可以是多行多列的数据。

## 2.2 事务处理的基本概念

事务处理是数据库系统的另一个核心功能，用于确保数据的一致性、完整性和可靠性。事务处理的基本概念包括：

- **事务**：事务是数据库系统中的一种操作，用于对数据库中的数据进行修改。事务包括一组数据库操作，这些操作要么全部成功，要么全部失败。
- **事务的ACID属性**：事务处理的核心属性是ACID（原子性、一致性、隔离性、持久性）。这些属性确保事务处理的结果是可靠的、一致的和可预测的。
- **事务的隔离级别**：事务的隔离级别是事务处理的一个重要属性，用于确保事务之间的独立性。事务的隔离级别包括四个级别：读未提交、读已提交、可重复读、串行化。
- **事务的锁**：事务的锁是事务处理的一种机制，用于确保事务之间的互斥性。事务的锁包括共享锁和排它锁。共享锁允许多个事务同时读取数据库中的数据，排它锁允许一个事务修改数据库中的数据。
