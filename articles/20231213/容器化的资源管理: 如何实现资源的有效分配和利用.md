# 2.核心概念与联系
在容器化的资源管理中，我们需要了解一些核心概念，包括资源、容器、调度器、资源分配策略等。这些概念之间存在着密切的联系，我们需要理解这些概念的关系，以便更好地实现资源的有效分配和利用。

## 2.1 资源
资源是容器化环境中的基本单位，可以包括 CPU、内存、磁盘等。资源的分配和利用是容器化环境中的关键问题，因为有限的资源需要分配给不同的容器，以满足不同的应用需求。

## 2.2 容器
容器是容器化环境中的基本单位，是应用程序的运行环境。容器可以包含应用程序、库、依赖关系等，可以独立运行，不受主机操作系统的影响。容器之间可以相互依赖，需要合理的资源分配，以确保应用程序的正常运行。

## 2.3 调度器
调度器是容器化环境中的一个重要组件，负责将资源分配给不同的容器。调度器需要根据容器的需求、资源状况等因素，进行资源的有效分配和利用。调度器可以采用不同的策略，如最小化资源分配时间、最大化资源利用率等。

## 2.4 资源分配策略
资源分配策略是调度器中的一个重要组件，负责根据不同的需求和状况，进行资源的有效分配和利用。资源分配策略可以包括最小化资源分配时间、最大化资源利用率等。

## 4.2 代码实例
在Kubernetes中，我们可以使用YAML文件，来定义容器的资源分配策略。以下是一个简单的YAML文件示例：

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
  - name: my-container
    image: my-image
    resources:
      requests:
        cpu: 100m
        memory: 128Mi
      limits:
        cpu: 500m
        memory: 512Mi
```

在这个示例中，我们定义了一个名为my-pod的Pod，包含一个名为my-container的容器。我们为容器设置了资源请求和限制，以实现资源的有效分配和利用。
