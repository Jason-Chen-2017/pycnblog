### 3.1.1 所有权的传递

所有权可以通过以下方式传递：

1. 函数调用：当一个函数返回一个值时，所有权将被传递给该函数的调用者。
2. 赋值：当一个变量被赋值时，所有权将被传递给新的变量。
3. 移动：当一个值被移动到另一个变量时，所有权将被完全传递给新的变量。

### 3.1.2 所有权的规则

所有权系统遵循以下规则：

1. 每个值都有一个拥有者。
2. 拥有者可以在其作用域内访问和修改这个值。
3. 当拥有者离开作用域时，所有权将被转移给其他拥有者或释放。

## 3.2 Rust的类型检查

### 3.2.1 类型推导

Rust支持类型推导，这意味着开发者不需要手动指定变量的类型。例如，当你声明一个变量时，Rust会根据变量的初始值推导出其类型。

### 3.2.2 类型约束

Rust支持类型约束，这意味着开发者可以指定某个函数或结构体的类型。例如，当你定义一个泛型函数时，你可以指定该函数接受哪种类型的参数。

在这个例子中，我们定义了一个名为`main`的函数，它是程序的入口点。我们使用`println!`宏来打印出“Hello, world!”字符串。

## 4.2 一个简单的计数器程序

以下是一个简单的计数器程序的代码实例：

```rust
fn main() {
    let mut count = 0;

    loop {
        count += 1;

        println!("Count is: {}", count);

        if count > 10 {
            break;
        }
    }
}
```

在这个例子中，我们定义了一个名为`count`的变量，它的初始值为0。我们使用`loop`关键字来创建一个无限循环，每次循环中我们将计数器增加1，并打印出当前的计数值。当计数器超过10时，我们使用`break`关键字退出循环。
