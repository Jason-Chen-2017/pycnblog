                 

# 1.背景介绍

大表（Big Table）是一种数据库技术的革命性设计，它主要面向的是Web 2.0的互联网应用程序，这些应用程序需要处理海量数据和高并发访问。大表设计的核心思想是将数据存储在分布式、无模式的数据库中，这种数据库可以在需要时轻松扩展。大表的设计思想源于Google，Google的搜索引擎就是一个典型的大表应用。

大表的设计思想主要有以下几个方面：

1. 分布式存储：大表的数据存储在多个服务器上，这样可以实现数据的水平扩展。当数据量增加时，只需要增加更多的服务器即可。

2. 无模式数据库：大表的数据库没有预先定义的数据结构，这意味着数据可以随意存储，不需要事先知道数据的结构。这使得大表非常灵活，可以存储任意类型的数据。

3. 自动分区：大表的数据自动分区，这意味着数据可以在不同的服务器上存储，这样可以实现数据的水平分区。当数据量增加时，只需要增加更多的服务器即可。

4. 高并发访问：大表的数据库可以支持高并发访问，这意味着多个用户可以同时访问数据库，并且数据库可以处理这些访问请求。

5. 高性能查询：大表的数据库可以实现高性能查询，这意味着用户可以快速地查询数据库中的数据。

6. 高可用性：大表的数据库具有高可用性，这意味着数据库可以在发生故障时自动切换到备份服务器，保证数据的可用性。

7. 高扩展性：大表的数据库具有高扩展性，这意味着数据库可以在需要时轻松地扩展。

8. 低成本：大表的数据库具有低成本，这意味着用户可以在低成本下使用大表的数据库。

大表的设计思想主要面向的是Web 2.0的互联网应用程序，这些应用程序需要处理海量数据和高并发访问。大表的设计思想源于Google，Google的搜索引擎就是一个典型的大表应用。大表的设计思想主要有以下几个方面：分布式存储、无模式数据库、自动分区、高并发访问、高性能查询、高可用性、高扩展性和低成本。
