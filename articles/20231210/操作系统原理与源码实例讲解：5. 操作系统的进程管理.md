# 2.核心概念与联系

## 2.1 进程和线程的区别

进程和线程是操作系统中的两种执行单位，它们之间有以下区别：

1.资源隔离：进程间资源互相独立，每个进程都有自己独立的内存空间、文件描述符、打开文件等资源。而线程间资源共享，同一个进程内的多个线程共享进程的内存空间、文件描述符等资源。

2.创建和销毁开销：进程的创建和销毁开销较大，因为操作系统需要为进程分配和回收资源。而线程的创建和销毁开销相对较小，因为同一个进程内的多个线程共享资源，只需要为线程分配和回收资源即可。

3.调度和优先级：进程间的调度和优先级相互独立，每个进程都有自己的调度策略和优先级。而线程间的调度和优先级相互依赖，同一个进程内的多个线程共享进程的调度策略和优先级。

## 2.2 进程的状态和状态转换

进程有多种状态，如新建、就绪、运行、阻塞、暂停等。进程的状态转换可以通过系统调用、硬件中断、软件中断等方式发生。以下是进程的主要状态和状态转换：

1.新建（New）：进程刚刚创建，尚未分配资源，处于就绪队列中，等待调度。

2.就绪（Ready）：进程已经分配了资源，等待调度，可以立即执行。

3.运行（Running）：进程正在执行，占用处理器资源，其他进程需要等待。

4.阻塞（Blocked）：进程正在等待某个事件发生，如输入输出、文件锁定等，不能继续执行，需要操作系统的调度。

5.暂停（Suspended）：进程由于某种原因被暂停，如用户请求、资源不足等，需要操作系统的调度。

6.结束（Terminated）：进程已经执行完成，或者发生错误，结束了。

## 2.3 进程的同步和互斥

进程同步是指多个进程之间的协同执行，以确保它们之间的正确性和效率。进程互斥是指多个进程之间的互相排斥，以确保它们之间的独占资源。

进程同步和互斥可以通过以下方式实现：

1.信号量：信号量是一种计数器，用于控制多个进程对共享资源的访问。信号量可以用于实现进程同步和互斥。

2.互斥锁：互斥锁是一种特殊的信号量，用于实现进程的互斥。互斥锁可以用于保护共享资源，确保只有一个进程可以访问。

3.条件变量：条件变量是一种同步原语，用于实现进程之间的同步。条件变量可以用于实现进程同步，以确保它们之间的正确性和效率。

4.信号：信号是一种异步通知，用于通知进程发生某个事件。信号可以用于实现进程同步和互斥。

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // 子进程
        printf("子进程ID: %d, 父进程ID: %d\n", getpid(), getppid());
        sleep(1);
    } else {
        // 父进程
        printf("父进程ID: %d, 子进程ID: %d\n", getpid(), pid);
        wait(NULL);
    }

    return 0;
}
```

在上述代码中，我们使用fork()系统调用创建了一个子进程。子进程和父进程分别执行不同的代码块，并打印出自己的进程ID和父进程ID。父进程使用wait()系统调用等待子进程结束，并返回子进程的状态信息。
