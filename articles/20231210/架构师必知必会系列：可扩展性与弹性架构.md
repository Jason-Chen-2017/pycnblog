# 2.核心概念与联系

在讨论可扩展性与弹性架构之前，我们需要了解一些核心概念：

1. 可扩展性：系统在不影响性能和稳定性的情况下，能够根据需求增加资源。
2. 弹性：系统在面对突发的故障或变化时，能够快速恢复并保持稳定运行。
3. 高可用性：系统在任何时候都能提供服务，即使出现故障也能快速恢复。
4. 负载均衡：将请求分发到多个服务器上，以提高系统性能和稳定性。
5. 容错：系统在出现故障时，能够快速检测并进行故障转移，保持服务的正常运行。

这些概念之间存在很强的联系，可扩展性和弹性是为了实现高可用性和负载均衡，而容错是为了应对突发的故障。
