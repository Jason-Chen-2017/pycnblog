# 2.核心概念与联系

在框架设计中，有几个核心概念需要我们关注：

1. 模块化：模块化是指将大型软件系统划分为多个较小的模块，每个模块负责完成特定的功能。这样可以提高代码的可维护性和可读性，同时也可以实现代码的重用。
2. 抽象：抽象是指将具体的实现细节隐藏起来，只暴露出接口。这样可以让不同的实现细节之间保持相互独立，提高代码的可扩展性。
3. 继承：继承是指一个类可以继承另一个类的属性和方法。这样可以实现代码的复用，同时也可以实现代码的扩展。
4. 多态：多态是指一个接口可以有多种实现。这样可以让不同的实现细节之间保持相互独立，提高代码的可扩展性。

这些概念之间存在着密切的联系。模块化和抽象可以实现代码的模块化和可扩展性，而继承和多态可以实现代码的复用和扩展。在框架设计中，我们需要充分利用这些概念，以实现一个灵活的框架架构。
