## 3.1 单例模式

单例模式是一种常见的设计模式，它确保一个类只有一个实例，并提供全局访问点。这种模式通常用于管理全局资源，例如数据库连接、文件操作等。

单例模式的核心原理是通过一个全局变量来存储单例对象的引用，并在类的内部提供一个访问这个全局变量的方法。这样，无论从哪个地方访问这个单例对象，都会返回同一个实例。

以下是单例模式的具体实现：

```python
class Singleton:
    _instance = None

    @classmethod
    def get_instance(cls):
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
```

在这个实现中，我们使用一个类变量`_instance`来存储单例对象的引用。在`get_instance`方法中，我们首先检查`_instance`是否为`None`，如果是，则创建一个新的单例对象；如果不是，则直接返回已经存在的单例对象。

## 3.2 工厂模式

工厂模式是一种创建对象的设计模式，它提供了一个接口来创建对象，而无需知道具体的类。这种模式通常用于创建不同类型的对象，以便在运行时根据需要选择不同的对象类型。

工厂模式的核心原理是通过一个工厂类来创建对象，这个工厂类提供了一个接口来创建对象，而具体的创建逻辑由子类实现。这样，我们可以在运行时根据需要选择不同的工厂类来创建不同类型的对象。

以下是工厂模式的具体实现：

```python
class Animal:
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

class AnimalFactory:
    @staticmethod
    def create_animal(animal_type):
        if animal_type == "dog":
            return Dog()
        elif animal_type == "cat":
            return Cat()
        else:
            return None
```

在这个实现中，我们定义了一个`Animal`类和两个子类`Dog`和`Cat`。我们还定义了一个`AnimalFactory`类，它提供了一个静态方法`create_animal`来创建不同类型的`Animal`对象。通过调用`AnimalFactory.create_animal`方法，我们可以根据需要创建不同类型的`Animal`对象。
