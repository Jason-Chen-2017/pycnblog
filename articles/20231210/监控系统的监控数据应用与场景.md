# 2.核心概念与联系

监控数据是指监控系统中收集的数据，包括系统性能指标、资源利用率、错误日志等。监控数据应用与场景是指将监控数据应用于各种场景，以提高系统的运行质量和可用性。

监控系统的监控数据应用与场景包括以下几个方面：

1. 系统性能监控：通过收集系统性能指标，如CPU使用率、内存使用率、网络带宽等，可以了解系统的运行状况，及时发现问题，从而提高系统的稳定性和可用性。

2. 资源利用率监控：通过收集资源利用率指标，如CPU利用率、内存利用率、磁盘利用率等，可以了解系统资源的利用情况，及时发现资源瓶颈，从而提高系统的性能。

3. 错误日志监控：通过收集错误日志，可以了解系统中发生的错误，及时发现问题，从而提高系统的可用性。

4. 应用监控：通过收集应用性能指标，如请求响应时间、错误率等，可以了解应用的运行状况，及时发现问题，从而提高应用的可用性。

5. 网络监控：通过收集网络性能指标，如网络带宽、延迟、丢包率等，可以了解网络的运行状况，及时发现问题，从而提高网络的可用性。
