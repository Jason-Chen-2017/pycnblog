## 3.1 B-树

B-树是一种自平衡的多路搜索树，它用于实现数据库的索引和查询功能。B-树的主要特点是它可以在O(log n)的时间复杂度内完成查询操作。

B-树的基本结构如下：

1. B-树的每个节点都包含一个关键字和一个子节点数组。

2. 关键字是节点中的所有关键字的中间值。

3. 子节点数组中的每个子节点都包含一个关键字和一个子节点数组。

4. 子节点数组中的每个子节点的关键字都在节点的关键字中。

5. 子节点数组中的每个子节点的关键字都在节点的关键字中。

B-树的查询操作如下：

1. 从根节点开始查询。

2. 比较当前节点的关键字与查询关键字。

3. 如果查询关键字小于当前节点的关键字，则查询当前节点的左子节点。

4. 如果查询关键字大于当前节点的关键字，则查询当前节点的右子节点。

5. 重复步骤2-4，直到查询关键字与某个节点的关键字相等，或者查询关键字小于所有节点的关键字。

## 3.2 B+树

B+树是B-树的一种变体，它用于实现数据库的索引和查询功能。B+树的主要特点是它可以在O(log n)的时间复杂度内完成查询操作，并且它的叶子节点存储了所有的关键字和数据。

B+树的基本结构如下：

2. 关键字是节点中的所有关键字的中间值。

B+树的查询操作如下：

4. 如果查询关键字大于当前节点的关键字，则查询当前节点的右子节点。

5. 重复步骤2-4，直到查询关键字与某个节点的关键字相等，或者查询关键字小于所有节点的关键字。

6. 如果查询关键字与某个节点的关键字相等，则查询该节点的叶子节点。

7. 在叶子节点中查找关键字与查询关键字相等的数据。

## 3.3 索引

索引是数据库中的一个重要概念，它用于加速查询操作。索引是数据库中的一种数据结构，它用于存储数据的关键字和数据的地址。

索引的基本结构如下：

1. 索引包含一个关键字数组和一个数据地址数组。

2. 关键字数组中的每个关键字对应一个数据地址。

3. 数据地址数组中的每个数据地址对应一个数据。

索引的查询操作如下：

1. 从索引中查询关键字。

2. 找到关键字对应的数据地址。

3. 查询数据地址对应的数据。

        if root.left.left.height > root.left.right.height:

def rotate_left(node):

    node.right = new_root.left
    new_root.left = node

    node.left = new_root.right
    new_root.right = node
    new_root.height = max(node.height, node.left.height) + 1
    node.height = max(new_root.left.height, new_root.right.height) + 1

    def create_index(self, column):
        for row in self.table:
            key = row[column]
            if key not in self.index:
                self.index[key] = []
