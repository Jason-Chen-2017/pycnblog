# 2.核心概念与联系

异常处理的核心概念包括异常、异常处理机制、异常捕获和异常处理。下面我们来详细介绍这些概念：

## 2.1 异常

异常是程序在运行过程中遇到的错误或异常情况，可能会导致程序的崩溃或者产生不可预期的结果。异常可以是运行时异常（Runtime Exception），如NullPointerException、ArrayIndexOutOfBoundsException等，或者是编译时异常（Checked Exception），如FileNotFoundException、SQLException等。

## 2.2 异常处理机制

异常处理机制是用于处理异常情况的一种机制。在Java中，异常处理机制包括try、catch和finally关键字。当程序在try块中遇到异常时，它会立即跳出try块，并尝试执行catch块中的代码。如果catch块中的代码能够处理异常，则异常被处理；否则，异常会被传递给上层的catch块或者最终被抛出。finally块中的代码在异常处理完成后或者程序正常结束时都会被执行。

## 2.3 异常捕获

异常捕获是指程序捕获并处理异常的过程。在Java中，我们可以使用try-catch语句来捕获异常。当程序在try块中遇到异常时，它会跳出try块并执行catch块中的代码，以处理异常。异常捕获可以帮助我们更好地控制程序的执行流程，并在遇到异常时采取适当的措施。

## 2.4 异常处理

异常处理是指程序在遇到异常时采取的措施。异常处理可以包括以下几种：

1. 捕获异常：通过try-catch语句捕获异常，以便进行处理。
2. 处理异常：在catch块中处理异常，以便恢复程序的正常执行流程。
3. 忽略异常：在某些情况下，我们可能不需要处理异常，而是直接忽略它们。这种情况通常发生在我们已经预期了异常，并且已经采取了适当的措施来处理它们。

```java
try {
    // 在try块中执行可能出现异常的代码
    FileReader fr = new FileReader("nonexistent_file.txt");
} catch (FileNotFoundException e) {
    // 在catch块中处理异常
    System.out.println("文件不存在，正在尝试打开一个默认文件");
    FileReader fr = new FileReader("default_file.txt");
} finally {
    // 在finally块中执行清理操作
    System.out.println("程序结束");
}
```

在上述代码实例中，我们使用try-catch语句来捕获FileNotFoundException异常。当程序在try块中尝试打开"nonexistent_file.txt"文件时，它会跳出try块并执行catch块中的代码，以处理异常。在catch块中，我们打印了一条错误信息，并尝试打开一个默认文件。最后，我们在finally块中执行清理操作，并打印了一条程序结束的信息。
