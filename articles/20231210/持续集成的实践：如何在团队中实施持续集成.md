                 

# 1.背景介绍

持续集成（Continuous Integration，简称CI）是一种软件开发的实践方法，它旨在在开发过程中自动化地集成代码，以便更快地发现和修复错误。这种方法在软件开发团队中广泛使用，特别是在敏捷开发和极限编程等方法中。

持续集成的核心思想是：团队成员在每次提交代码时，都需要在集成服务器上自动构建代码。这样可以确保代码的质量，以及在发布前尽可能早地发现错误。
