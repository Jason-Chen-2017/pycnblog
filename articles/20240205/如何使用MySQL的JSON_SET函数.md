## Background Introduction

JSON (JavaScript Object Notation) is a lightweight data interchange format that is easy for humans to read and write and easy for machines to parse and generate. It has become increasingly popular as a way to store and manipulate structured data in databases.

MySQL 5.7 introduced native support for JSON data types and provided several functions to work with JSON data. One such function is JSON_SET, which allows you to modify the value of a specific property in a JSON document.

Before we dive into the details of JSON_SET, let's first take a look at some core concepts related to JSON and MySQL.

### Core Concepts

* **JSON**: A lightweight data interchange format that is easy for humans to read and write and easy for machines to parse and generate.
* **MySQL**: An open-source relational database management system (RDBMS).
* **JSON Data Type**: A data type in MySQL that stores JSON documents.
* **JSON Path**: A syntax for selecting values from a JSON document.

Now that we have a basic understanding of these concepts, let's move on to the core algorithm principle behind JSON_SET.

## Core Algorithm Principle and Operating Steps

The JSON_SET function in MySQL takes three arguments:

1. The JSON document to modify.
2. The path to the property to be modified.
3. The new value to set.

Here's an example of using JSON_SET:
```sql
SET @json = '{"name": "John", "age": 30}';
SET @json = JSON_SET(@json, '$.age', 35);
SELECT @json;
```
Output:
```json
{"name": "John", "age": 35}
```
In this example, we first define a JSON document containing the name and age properties. We then use JSON\_SET to modify the value of the `age` property to `35`.

Under the hood, JSON\_SET performs the following steps:

1. Parse the input JSON document.
2. Find the specified property using the JSON path.
3. Replace the value of the property with the new value.
4. Return the modified JSON document.

Note that if the specified property does not exist in the JSON document, JSON\_SET will add it with the specified value.

Let's now examine the mathematical model behind JSON\_SET.

### Mathematical Model

The mathematical model behind JSON\_SET can be represented as follows:

$$
\text{new\_json} = \text{JSON\_SET}(\text{old\_json}, \text{path}, \text{value})
$$

Where:

* $\text{old\_json}$ represents the original JSON document.
* $\text{path}$ represents the JSON path to the property to modify.
* $\text{value}$ represents the new value to set.

This formula shows that JSON\_SET modifies the original JSON document by replacing the value of the specified property with the new value.
