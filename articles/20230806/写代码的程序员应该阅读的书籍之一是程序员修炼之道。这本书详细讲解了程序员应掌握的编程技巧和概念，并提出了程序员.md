         # 2.编程相关术语的定义
         ## 2.1.程序和程序的组成部分
         - 程序：计算机软件、脚本或硬件设备的静态集合，是指令序列和数据定义的集合体，是用来实现某些特定功能的源代码集。
         - 程序的组成部分：程序可以分为四个部分：输入（input）、处理（process）、输出（output）。


         
         - **输入**：程序的输入是计算机接收外部信息或从其他程序中获得信息的途径。输入可能包括文件、命令行参数、鼠标点击等。
         - **处理**：程序的处理过程就是对输入进行计算、分析、判断、排序、存储等操作的过程。
         - **输出**：程序的输出一般是处理结果或反馈给用户的形式。如打印到屏幕、显示在页面上、保存到文件中等。
         ## 2.2.变量、数据类型和数据结构
         - 变量（variable）：程序运行过程中发生变化的值。变量由变量名、数据类型、存储位置、值组成。变量类型分为整数型、浮点型、字符型、逻辑型、数组型、指针型、结构型、枚举型等。
         - 数据类型（data type）：每个变量都对应着一种特定的数据类型。不同的类型的数据可用于完成不同的运算或任务。数据类型决定了变量可以存储什么样的信息，以及可以进行什么样的操作。
         - 数据结构（data structure）：数据结构是指计算机内储存和组织数据的方式，它将数据元素按逻辑关系和相互依赖组织起来，便于管理、访问和使用。数据结构可以简单地分为两类：线性结构和非线性结构。
         ### 2.2.1.线性数据结构
         - 一维数组（一维矩阵）：数组是最简单的线性数据结构。它是一种定长序列，其元素可以是相同数据类型的元素或不同数据类型的元素。
         - 二维数组：二维数组是指有两个下标的数组，其中一个下标表示行号，另一个下标表示列号。二维数组的每个元素称为一个矩阵元（matrix element），也可以称为元素。
         - 链表：链表是一种动态的线性数据结构，其元素除了存储值外，还包括指向下一个节点的链接地址。链表可分为单向链表和双向链表两种。
         ### 2.2.2.非线性数据结构
         - 栈（stack）：栈是一种先进后出（FILO）的数据结构，其操作方式类似于堆栈。栈中的元素只能从栈顶进行删除和插入操作，栈底永远固定不动。
         - 队列（queue）：队列是一种先进先出（FIFO）的数据结构，其操作方式类似于排队。队列的头部为进入队列的第一个元素，尾部为最后进入的元素。
         - 散列表（hash table）：散列表是一个无序的键值对集合，其基于数组和链表实现。散列表的查找、插入、删除操作的时间复杂度都为O(1)。
         ### 2.2.3.抽象数据类型
         - 抽象数据类型（abstract data type，ADT）：抽象数据类型是指由一组值的集合和一组操作所组成的数据类型。它允许用户定义自己需要的数据类型，同时隐藏实现细节。抽象数据类型可以通过接口来使用，使得不同的应用可以共享同一组操作。
         ### 2.2.4.堆和栈
         - 堆（heap）：堆是内存中用于动态分配的数据区。堆可以被应用程序随时使用，并随时释放掉，不会造成内存碎片。
         - 栈（stack）：栈是计算机运行时用于维护函数调用状态、临时数据的存储空间。栈提供了一个先进后出的存储方式，即先进入的先出。栈的大小是固定的，一旦超出容量就无法再添加新的元素。

         # 3.程序的执行模型
         ## 3.1.顺序执行模型
         - 顺序执行模型：这是一种最简单、直观的程序执行模型。程序中的语句从上到下依次执行，遇到条件或循环语句时，会选择符合条件或者继续执行循环。
         ### 3.1.1.控制流程
        ```java
        for (int i = 1; i <= n; i++) {
            if ((i % 2 == 0 && k > 0) || (i % 2!= 0 && k < 0)) {
                continue;
            } else {
                System.out.println("i=" + i);
            }
        }
        ```
         - 此段代码实现了一个求前n项中第k大或第k小的数字的程序。程序中首先循环n次，对每一个奇数i进行检查，如果i%2等于0且k大于0，则跳过当前循环；如果i%2不等于0且k小于0，则跳过当前循环；否则，输出i。
         ### 3.1.2.顺序执行的限制
        ```c++
        int main() {
            int a[10];   //声明数组
            for (int i = 0; i < 10; ++i) {
                a[i] = i * 2;     //初始化数组元素
            }
            return 0;    //程序结束
        }

        /*
         * Output: 
         * a[0]: 0      a[1]: 2    ...       a[9]: 18
         */
        ```
         - 在此段代码中，我们声明一个长度为10的数组a，然后使用for循环对其元素进行赋值。注意，for循环前面的“i=0”表示初始化的索引值，“<”表示终止条件，“++”表示自增步长。整个程序仅占用一行，但为了方便展示，我们把循环体放在了一起。
         - 然而，当执行这个程序时，可能会发现数组a中没有按照预期显示0-9的偶数。原因是在C++中，默认情况下，数组的长度是0，而不是10，因此，程序的执行不会把a[i]的值填满，而只是赋予前面声明的那些元素的值。要想正确地初始化数组，可以使用如下的方式：
         ```c++
         int main() {
             int a[10] = {0};    //声明数组并初始化
             for (int i = 0; i < 10; ++i) {
                 a[i] = i * 2;     //初始化数组元素
             }
             return 0;    //程序结束
         }
         ```
         - 在这里，我们用花括号{}对初值进行了初始化，并指定了长度为10的数组。这样，程序就会把数组的元素都初始化为0。
         ## 3.2.并发执行模型
         - 并发执行模型：在并发执行模型中，多个进程或线程在同一时间段内执行，共享CPU资源，互不影响。这种模型的特征是同时存在多个任务，彼此之间独立运行，但共享内存，也就是各个任务可以访问同一块内存空间。
         ### 3.2.1.多任务环境下的并发执行
         - **多任务系统**：多任务系统是指允许多个任务同时运行的操作系统。典型的多任务系统如MS-DOS、Windows、Unix/Linux等。
         - **多任务切换**：在多任务系统中，每个任务都由操作系统来负责切换执行。例如，当有新的任务需要调度运行时，操作系统就会从正在运行的任务中选择一个暂停执行的任务，并交换执行权。操作系统会在任务间频繁进行切换，实现多任务的平滑运行。
         - **多核处理器架构**：多核处理器是指拥有两个或更多独立的CPU芯片的处理器。每个芯片都有自己独立的缓存、寄存器、指令流水线、高速局域网、PCI总线等资源。由于多个芯片共同处理任务，因此称为多核处理器。
         ### 3.2.2.异步通信机制
         - **消息传递机制**：消息传递机制是指多个任务通过通讯的方式进行通信。最著名的消息传递机制是管道（pipe）和消息队列（message queue）。管道提供了字节流形式的数据传输，因此可以在不同进程间通信；消息队列提供了数据块形式的数据传输，可以根据优先级进行调度。
         - **信号量机制**：信号量机制是指多个任务通过信号量同步进行互斥和同步。信号量是一个计数器，每个任务获取一个信号量，当信号量计数器减至零时，表示某个资源被占用，不再允许访问，直到所有的任务释放信号量。信号量可以用作互斥锁或同步变量，也可以用作事件等待。
         - **互斥锁机制**：互斥锁是一个开关锁，任何时候只能有一个任务持有锁，其他任务只能等待锁释放才能获取锁。互斥锁可以用于资源共享的保护，防止竞争条件。
         ### 3.2.3.并发执行环境的限制
         - **死锁**：死锁是指两个或多个进程因竞争资源而陷入僵局，一直无法解除的情况。死锁的四个必要条件是：互斥条件、请求保持条件、不剥夺条件和环路条件。
         - **饥饿**：饥饿是指处于待命状态的进程，由于资源不足而导致一直无法运行的现象。
         - **性能瓶颈**：在并发环境中，当资源不足或竞争激烈时，往往会出现性能瓶颈。性能瓶颈是指某种计算能力或资源的利用率过低，导致系统整体性能下降的现象。
         ## 3.3.分布式系统
         - 分布式系统：分布式系统是指由不同节点构成的计算机系统，它们之间通过网络连接，形成了一个互联的分布式系统。分布式系统的特点是分布性、Failures Isolation、Scalability、Concurrency 和 Availability。
         ### 3.3.1.分布式系统的特点
         - 分布性：分布式系统由多个计算机节点组成，节点之间通过网络连接，可以随意分布。这一特点使得分布式系统具有非常强大的弹性，具有很好的扩展性。
         - Failures Isolation：分布式系统能够将故障隔离到各个节点，并且节点之间采用异步通信，因此故障不影响其他节点。这一特点可以缓解节点故障带来的系统瘫痪。
         - Scalability：分布式系统能够按需增加资源，提升系统的可靠性和可用性。例如，可以增加更多节点，提高系统吞吐量或响应速度。
         - Concurrency：分布式系统能够支持高并发，每个节点可以并行执行多个任务，有效提高系统的处理效率。
         - Availability：分布式系统具有高可用性，在任意时刻，系统的某个节点失败不会导致整个系统失去服务。
         ### 3.3.2.分布式系统的实现
         #### 3.3.2.1.远程过程调用（RPC）
         - RPC：远程过程调用（Remote Procedure Call，RPC）是分布式系统中不同节点之间的通信机制。通常是通过网络来实现的，客户端进程像调用本地过程一样调用远程过程，不需要知道底层的网络通信细节。
         #### 3.3.2.2.客户端-服务器架构
         - 客户端-服务器架构：客户端-服务器架构是分布式系统中经典的设计模式。客户端与服务器端分别位于分布式系统的不同节点上。客户端进程直接调用服务器端的过程，无需考虑底层网络通信。服务器端负责处理客户端请求，并返回结果。
         #### 3.3.2.3.远程服务调用（RMI）
         - RMI：远程方法调用（Remote Method Invocation，RMI）是Java提供的分布式系统的通信机制。Java中的RMI支持动态绑定，客户端可以调用远程对象上的方法，就像调用本地对象一样。但是，Java的RMI是建立在TCP/IP协议之上的，所以需要遵守TCP/IP协议。
     
           # 4.编码习惯
         ## 4.1.命名规范
         ### 4.1.1.驼峰命名法
         - 驼峰命名法又叫驼峰连接法，即将标识符的首字母和每个新单词的首字母大写，但不改变标识符中间的字母的大小写。例如，UserName -> UserName，PrintStudentInfo -> PrintStudentInfo，IMessageSender -> IMessageSender。
         ### 4.1.2.小写驼峰命名法
         - 小写驼峰命名法是指将所有字母转换为小写字母，然后按照驼峰命名法进行命名。例如，username -> username，printstudentinfo -> printStudentInfo，imessagesender -> IMessageSender。
         ### 4.1.3.下划线命名法
         - 下划线命名法是指多个单词之间用下划线分割，例如my_name。
         ## 4.2.注释的使用
        ```c++
        // single line comment

        /** 
         * multi lines comments
         */
        ```
         - 单行注释：//
         - 多行注释：/* */
         - 文档注释：/** */
