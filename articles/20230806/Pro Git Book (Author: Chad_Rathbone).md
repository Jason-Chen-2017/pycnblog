# 2.核心概念及术语
## 2.1 Git概念
Git是一个开源的分布式版本控制系统，它的优点很多，例如速度快、占用的空间小、可以快速解决冲突等等。Git的基本工作流程如图1所示。
图1 git基本工作流程
### 2.1.1 仓库（Repository）
仓库是一个保存项目所有文件的地方，通过Git管理的文件都放在仓库中。
### 2.1.2 分支（Branch）
分支是用来实现多人协作开发的功能。每个分支都是一条独立的线性历史记录，互不干扰，可帮助你开发不同的特性而不会影响主分支。在多人同时开发时，新建一个分支用于不影响其他开发者的代码。当开发者完成某些特性时，可以将该分支合并到主分支。
### 2.1.3 克隆（Clone）
克隆操作即将远程库克隆到本地电脑上，通常是在本地创建了一份属于自己的完整副本。
### 2.1.4 检出（Checkout）
检出操作会从指定分支或提交点创建一个新的工作区，这个工作区在本地磁盘上是一个完整的目录。
### 2.1.5 提交（Commit）
提交操作会将文件修改的内容暂存到暂存区，等待下一次提交。提交前可以查看修改了哪些文件，修改了多少内容。
### 2.1.6 拉取（Pull）
拉取操作即从远程库获取最新内容并更新本地仓库。
### 2.1.7 推送（Push）
推送操作则把本地提交的内容推送到远程库。
### 2.1.8 报错（Error）
报错即由于输入指令错误或没有按照要求进行操作，导致的运行失败。
## 2.2 Git术语
为了更好的理解和使用Git，下面列举一些重要的术语。
### 2.2.1 SHA-1值
Git采用SHA-1哈希算法生成唯一的对象ID，长度为40个字符的字符串。它表示一个特定的对象，比如提交对象、树对象、blob对象等。可以使用Git show 命令查看对象的信息。
### 2.2.2 HEAD指针
HEAD指针指向当前所在的分支或提交对象，它是一个指向最新版本的指针，比如指向当前正在工作的分支。HEAD指针可以通过Git checkout命令修改。
### 2.2.3.gitignore文件
.gitignore 文件用来指定忽略哪些文件，防止这些文件被纳入版本管理中。
### 2.2.4 暂存区（Index）
暂存区是一个缓存区域，临时保存你的改动，等待提交。
### 2.2.5 分支合并
分支合并就是把两个不同分支的修改合并成同一个分支，目的是保持分支之间同步，提高代码质量。
### 2.2.6 远程库（Remote Repository）
远程库是指托管在网络上的Git仓库，他可以作为本地仓库和其他仓库之间的传输通道。
