         # 1.对称加密算法（Symmetric Encryption Algorithm）
        ## 中文版

        对称加密算法又称为共享密钥加密算法，它利用同一个密钥进行加密和解密，加密时使用的是一种算法或模式，解密时也是用这种算法或模式。加密速度快，安全性高，但只适用于数据量较少的对话通信或其他实时通信应用。

        最常用的对称加密算法有DES（Data Encryption Standard）、AES（Advanced Encryption Standard）、Blowfish、RC4等。通常情况下，对称加密算法有两种方式进行实现，即ECB（Electronic Code Book）模式和CBC（Cipher-Block Chaining）模式。

        ## English Version 

        Symmetric encryption algorithms are also known as shared key encryption algorithms and they use the same key for both encryption and decryption. The algorithm used during encryption is a pattern or an algorithm while it's used during decryption in the reverse direction. Symmetric encryption algorithms are widely used for real time communication where data volume is relatively small but with high security level. 

        Some commonly used symmetric encryption algorithms include DES (Data Encryption Standard), AES (Advanced Encryption Standard), Blowfish, RC4 etc. Usually, symmetric encryption algorithms can be implemented in two ways namely ECB mode and CBC mode.

        # 2.非对称加密算法（Asymmetric Encryption Algorithm）
        ## 中文版

        非对称加密算法也叫公私钥加密算法，它是建立在对称加密算法之上的。公钥和私钥是两个不同的密钥，公钥用来加密，而私钥用来解密。由于加密和解密需要不同密钥，因此公钥可以对外公开，任何人可以通过公钥进行加密，而只有拥有对应的私钥的人才能解密。

        RSA算法是最著名的非对称加密算法。RSA是目前世界上使用最广泛的公钥加密算法，它支持两种密钥长度——512比特、1024比特和2048比特，同时还支持两种密钥分发机制——密钥交换和中心化。

        ## English Version

        Asymmetric encryption algorithms are also known as public-key encryption algorithms which rely on symmetric encryption algorithms to function. Public keys and private keys are different keys that work together. During encryption, one of them (public key) encrypts the message using another key (private key). Thus, only those who possess the corresponding private key can decrypt the encrypted message.

        The most famous public-key encryption algorithm is RSA (Rivest–Shamir–Adleman). RSA supports three types of key lengths – 512 bits, 1024 bits, and 2048 bits; and two key distribution mechanisms – key exchange and centralized key management.

    # 3.消息认证码（Message Authentication Code）
    ## 中文版

    消息认证码（Message Authentication Code）简称MAC，它是一种完整性校验的方法，它通过一个秘密函数把一个报文的全部或部分加以验证，以保证报文完整无误。它包含的信息可以保护数据的完整性、认证数据来源及完整性、并且防止重放攻击。

    MAC算法可分为三类：

    - 数据驱动型MAC算法：对整个报文计算HMAC，即利用HMAC算法将报文中的消息和额外的数据（如序列号）组合成消息摘要。然后对消息摘要进行加密运算得到完整性校验值。该方法具有抗修改性、抗重放攻击的能力，但它也容易受到中间人攻击。
    - 标注驱动型MAC算法：与数据驱动型相反，在数据尾部增加消息认证码，由接收端直接校验。该方法不需要发送方明确消息认证码的存在，具有简单有效的实现。但是它不能提供抗重放攻击等安全性保证。
    - 混合型MAC算法：结合了数据驱动型和标注驱动型的优点，其中前面提到的消息认证码保护整个报文完整性，后面的标签可以提供可靠但不保密的服务质量信息。

    ## English Version 

    Message authentication code (MAC) is an integrity check method. It verifies the entirety or part of a message by applying a secret function. It contains information that protects data from tampering, authenticates the source of data, and prevents replay attacks. 

    There are three classes of MAC:

    1. Data-driven MAC algorithm: Computes HMAC over whole message including additional data like sequence number. Then encrypts message digest to obtain the mac value. This technique has the ability to resist modification and replay attack. However, this makes it vulnerable to middle-man attacks.
    2. Label-driven MAC algorithm: Inverts the process of data driven MAC. Adds the authentication tag at the end of the message itself without any extra data. Easy implementation, but does not provide secure guarantees against replay attacks. 
    3. Hybrid MAC algorithm: Combines features of previous two techniques to achieve optimal security performance. Here, MAC protects the overall message integrity while the label provides service quality information with low security guarantees. 
    
    # 4.摘要算法（Digest Algorithm）
    ## 中文版

    摘要算法，又称哈希算法或者散列算法，它是指将任意长度的数据转换为固定长度的输出，且这个输出只能用于检验输入数据是否被修改过，不能用于解密。通常采用单向Hash函数，目的是为了计算出一个唯一的固定长度的值。目前常见的摘要算法有MD5、SHA-1、SHA-256、SHA-3等。

    SHA-256实际上是由两部分组成的，首先由输入消息经过压缩函数处理，然后再通过一个Hash函数求得其摘要。第一步的压缩过程包括选取一个候选消息块（512比特），选择其右边界，填充空格，将消息分为固定大小的块，最后将消息与填充方案混合。第二步的Hash函数负责将多条消息块连接成一条消息流，然后将消息流进行分块、填充，并将每条消息分割成512比特。然后将消息流经过哈希函数，将得到的一个固定长度的二进制串作为结果输出。

    ## English Version 

    A hash function is a cryptographic hash function that takes arbitrary length input data and produces a fixed-length output that cannot be reversed back. Hash functions are often used to verify the integrity of digital messages or files, typically by converting inputs into unique fixed-size values called hashes. Common hash functions include MD5, SHA-1, SHA-256, and SHA-3.

    SHA-256 consists of two parts. First, the input message is compressed by a compression function, then passed through a hashing algorithm to produce its digest. The first step involves selecting a candidate block (a set of 512 bits), finding its rightmost boundary, padding spaces, splitting the message into blocks of equal size, and combining it with the padding scheme. The second step involves concatenating multiple message blocks and performing further operations such as padding and chunking the stream into pieces each containing 512 bits. Finally, the resulting binary string obtained after processing the message through the hashing function is returned as the final result.
