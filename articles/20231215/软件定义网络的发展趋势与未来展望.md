                 

# 1.背景介绍

软件定义网络（Software-Defined Networking，简称SDN）是一种新兴的网络技术，它将网络控制平面和数据平面分离，使网络更加灵活、可扩展和可管理。SDN的核心思想是将网络控制逻辑从硬件中抽取出来，放到独立的软件平台上，从而实现网络的程序化管理。

SDN的诞生是为了解决传统网络的一些问题，如网络管理复杂、不灵活、不可扩展等。传统网络中，网络控制逻辑和数据传输逻辑紧密结合在一起，使得网络管理和优化变得非常困难。而SDN则将这两者分离开来，使得网络管理变得更加简单和灵活。

SDN的核心技术包括：

1. 控制平面分离：将网络控制逻辑从数据平面中抽取出来，放到独立的软件平台上。
2. 程序化管理：使用开发者熟悉的编程语言来编写网络控制逻辑，从而实现网络的自动化管理。
3. 流量分发：使用流量分发器来实现基于流的网络控制，从而实现更加精细化的网络管理。

SDN的核心概念与联系：

1. 控制平面与数据平面：控制平面负责网络的控制逻辑，数据平面负责网络的数据传输逻辑。它们之间通过Southbound接口进行通信。
2. 流量分发器：流量分发器是SDN的核心组件，它负责根据流表条目将数据包发送到正确的输出端口。
3. 流表：流表是SDN中的一种数据结构，用于描述如何处理特定类型的数据包。流表包括匹配条件、操作动作等信息。
