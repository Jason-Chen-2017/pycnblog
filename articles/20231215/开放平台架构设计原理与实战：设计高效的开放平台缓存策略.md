# 2.核心概念与联系

在开放平台架构中，缓存策略的核心概念包括缓存穿透、缓存击穿、缓存雪崩等。

缓存穿透是指在缓存中没有找到请求的数据，需要从原始数据源中查找。缓存穿透会导致系统性能下降，因为需要额外的查询操作。为了解决缓存穿透问题，可以使用预先加载或者使用特殊标记来标记缓存中的数据。

缓存击穿是指在缓存中有大量的请求同时访问一个缓存项，导致缓存项被击穿，需要从原始数据源中查找。缓存击穿会导致系统性能下降，因为需要额外的查询操作。为了解决缓存击穿问题，可以使用悲观锁或者使用分布式锁来保护缓存项。

缓存雪崩是指在缓存中大量的缓存项在同一时间内失效，导致系统性能下降，因为需要从原始数据源中查找。缓存雪崩会导致系统性能下降，因为需要额外的查询操作。为了解决缓存雪崩问题，可以使用随机失效时间或者使用分布式锁来保护缓存项。
