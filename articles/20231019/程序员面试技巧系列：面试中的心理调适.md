# 1.背景介绍


在面试中，候选人通常都要花费大量的时间来准备，而这些时间应该如何分配才能更有效地提高自己的能力？怎样让面试官更能够吸引到优秀的人才？这背后存在着许多复杂的问题。
作为一名程序员或计算机科学相关的专业人员，面试时面临着诸如自我介绍、项目经验、编程语言使用经验、算法能力、综合素质等多个方面的考察。而一个好的职场人士需要具备的软 skills，也是很值得探讨的。因此，如何掌控自己对面试的控制、进行“内外兼修”的学习及心态平衡等都是非常重要的。
面试心理调适是一个庞大的课题，涉及面试官、候选人、面试者三方面的因素。但本文将着重于面试官的行为及其影响，希望通过分析面试过程中面试官所采取的行为及应对方法，以及在实际工作中如何运用技巧来提升自己的面试效果与能力，对面试官有所帮助。
# 2.核心概念与联系
## 2.1 “放松式”的面试
“放松式”的面试意味着面试官放松了对候选人的评判标准，只看其个人表现、技能和相关经历，而不关心候选人的学历、行业背景、薪水、年龄、婚姻状况等额外的因素。这种方式对于初级开发工程师来说相当友好，因为他们可以把更多的时间用于自身的职业发展。但是，如果候选人有特殊的能力或兴趣，则可能被忽略甚至针对性地拒绝。因此，面试官应当在面试过程中做出一些调整，提高自信，保持一定心理距离，尊重候选人的个性。
另一种形式的“放松式”面试也会发生在整个流程的不同阶段。面试官在确定感兴趣方向后，为了应对高压情况，会提前进入面试环节，即使候选人并没有完全准备好。这样，候选人就会感到被逼到墙角，而面试官也会收获很多反馈。因此，在进入面试之前，面试官首先应该设法了解候选人情况，尽量做到充分沟通，保证自己清楚地知道候选人想要什么，以及他们的兴趣爱好、缺点、工作经验、能力等。
## 2.2 面试前的准备
面试前，首先应当准备好相关资料、工作报告和简历，确保面试官能够快速了解候选人。面试官应当给予充足的耐心，尤其是当面试候选人遇到了困难的时候。此外，面试官还应当亲自示范候选人作品，并且负责教导候选人使用开发工具，培养候选人解决问题的能力。
## 2.3 面试技巧
### 2.3.1 没有捷径可走时的策略
面试官往往缺乏常识、经验和知识，因此在面试过程当中往往会陷入纠结、迷茫之中。遇到这种情况，面试官应当保持冷静，不要着急下决定，而是先调整自己的状态，稍微休息一下，继续保持沉默。这种调整的方法既不会降低面试官的竞争力，也能够激励面试官向正确的方向前进。同时，也可以避免在谈话中出现自我中心主义、焦虑情绪或自恋情绪，从而给面试官留下积极的印象。
### 2.3.2 对不同的候选人都适用的技巧
不同的候选人都有各自独特的优势和长处。因此，面试官可以根据候选人特点制定相应的考察方式，来针对性地提升自己的能力。例如，对于具有丰富经验的工程师，面试官可以让候选人展示自己的项目经验，然后进行代码审查，以检验候选人是否能够快速理解设计思路和实现方案。另一方面，对于对新技术敏感、追求新事物的候选人，面试官可以询问候选人是否了解这些新技术，以便判断他们是否有能力理解新事物并实践。
### 2.3.3 面试官的注意力管理技巧
面试官的注意力在面试过程中起着至关重要的作用。过多关注自己的工作、面试中的问题，可能会导致疲劳、紧张、失去方向。因此，面试官应当精心安排面试时间，避免出现“太忙”的场面。另外，面试官应当做好答辩准备，专注于陈述自己的观点和论证，而不是烦躁地听别人发言。另外，面试官可以通过采用“聆听-记录-反馈”的方式来提升自己的注意力。面试官可以先听候选人的问题，再回顾答案，找出自己认为重要的问题，最后评估其真实性、正确性和切合情况，并修正自己的陈述。最后，通过将面试效果反映到“工作”上，面试官也能够更好地激励自己，改善自己的工作效率和表现。
