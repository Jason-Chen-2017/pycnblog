# 1.背景介绍


## 一、云计算概述
### （1）定义
> 云计算（Cloud computing），是一种利用网络提供按需服务的方式，将计算资源、存储资源和应用软件分散到互联网上，并通过高度可靠性的网络连接提供可扩展性的计算服务。目前，国际上主要的云计算服务提供商包括Amazon Web Services、Microsoft Azure、Google Cloud Platform等，它们的共同特征是利用计算、网络和存储等公有或私有云资源，为用户提供弹性可伸缩的计算平台、存储平台以及软件平台服务，让客户在云端享受到超大规模计算机算力、数据存储空间以及相应的IT服务，这些服务随用量的增加而自动扩张或收缩。

### （2）功能特点
#### 1. 按需弹性
随着互联网快速发展，电子商务、移动互联网、金融、生物医疗、新能源汽车等行业对数据的处理量越来越大，导致传统IT架构难以应付这种日益增长的数据量和复杂度。随着云计算的普及和发展，云计算可以根据需求快速弹性扩展计算能力，使得数据的处理变得更加灵活高效。例如，在AWS EC2、Azure VMWare和阿里云等公有云服务中，用户无需预购服务器、配置硬件，只需要按照实际使用的容量付费即可。此外，由于云计算平台提供的计算资源是按需分配的，所以当某段时间不使用时，无需为其支付额外费用，也不会影响其它正在运行的任务。

#### 2. 提供更多服务
云计算除了提供计算能力外，还可以通过提供软件、数据库、网络等基础设施服务来为客户提供更多价值。例如，AWS提供了大量的应用程序服务（Amazon Elastic Compute Cloud、Amazon Elastic Block Store、Amazon Simple Storage Service、Amazon Elastic File System、Amazon Relational Database Service、Amazon Machine Learning Service等），帮助客户构建各种Web应用、移动应用、游戏服务器、后台数据处理服务、网站托管服务等；微软Azure提供的平台即服务（PaaS）服务包括Azure App Service、Azure SQL Database、Azure Data Factory、Azure Blob Storage等，可以帮助客户快速开发、部署、管理和监控应用程序。

#### 3. 大数据分析
由于云计算平台提供的计算、存储和网络资源可以任意分配，因此可以非常方便地进行大数据分析。例如，AWS提供的EMR（Elastic MapReduce）集群可以帮助用户快速搭建 Hadoop 或 Spark 集群，使用 MapReduce 或 Spark 对海量数据进行分布式运算；Azure提供的 HDInsight 服务可以帮助用户快速搭建 Apache Hadoop 或者 Spark 集群，用于处理结构化或非结构化数据。这样就可以将数据从硬盘转移到内存，提升数据分析速度。另外，云计算平台提供的开源工具可以帮助用户进行数据采集、清洗、转换、查询、分析等，进行数据的科学挖掘和展示，为公司创造价值。

#### 4. 技术门槛低
云计算不需要高端技术的专业知识就可以获得足够的服务。云计算服务提供商通过提供廉价的基础设施资源，降低了客户技术上的门槛。例如，AWS提供了亚马逊自己的弹性云计算服务（Amazon EC2）、虚拟机镜像、VPC（Virtual Private Cloud）等服务，允许用户在不到五分钟内创建自己的云环境，甚至可以购买第三方的服务。因此，即使没有相关的专业技能，也可以轻松使用云计算服务。

## 二、云计算的发展历程
### （1）初期阶段：单一技术解决方案
早年，云计算刚刚出现时，主要技术就是OpenStack、Eucalyptus、CloudStack等。这些产品都是基于一套开源软件实现的，为公司提供了一个统一的虚拟化平台，让各个业务部门的运维人员可以利用共享的云资源部署软件。但由于缺乏整体性、集成性，使得公司内部多个部门之间存在信息孤岛，无法实现数据共享，难以满足各自的工作需求。

### （2）第二阶段：应用级服务
后来，出现了第一次公开的云计算服务，其中就包括了SaaS（Software as a Service）、PaaS（Platform as a Service）、IaaS（Infrastructure as a Service）三种不同层次的服务。SaaS服务提供软件解决方案，如雅虎旗下的搜狐邮箱、GitHub，用户可以直接购买软件服务，且不受限制地使用其中的功能，同时可以把自己的数据和服务导入到云端，由云端运维人员提供支持；PaaS服务则是面向应用开发者提供开发框架、中间件、开发工具，让用户只关注自己的业务逻辑；而IaaS层提供的是基础设施服务，如计算、网络和存储等资源，让用户可以快速部署应用程序、数据库等，同时对平台的配置、维护、扩展等负责。此时的云计算市场仍然比较小，主要是少数技术公司和少数企业独家垄断，而且仍然处于起步阶段。

### （3）第三阶段：混合云及多云
随着云计算的发展，商业模式也在不断升级。第三次的公开云计算市场虽然推出了新的服务类型——混合云服务，但仍然存在很多问题。首先，云计算只是解决某些特定问题的一种技术手段，并不是一种通用的商业模式。其次，服务的多样性和覆盖范围都无法满足客户的业务需要，比如某个领域的服务一直都是最好的，其他领域却很差，这样的服务架构无法满足用户的需求。

为了克服这些问题，出现了第四次公开云计算市场——多云服务，让用户可以选择不同的云服务供应商，组合成一个完整的云计算平台。多云平台的优势在于可以减少总拥有的基础设施数量，让用户得到更大的灵活性。另外，用户可以自己选择数据中心的位置，避免拥有一个数据中心拥有过多的设备，以便节省成本。但是，多云服务仍然面临着技术壁垒，因为各云服务供应商之间的兼容性、稳定性无法得到保证，因此用户也很难完全依赖一个供应商。除此之外，多云平台的成熟需要时间，目前仍处于起步阶段。
