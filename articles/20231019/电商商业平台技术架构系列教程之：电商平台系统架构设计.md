# 2.核心概念与联系

## 2.1	什么是系统架构设计？ 

系统架构设计是用来描述组织系统(或部件)之间相互连接关系、以及这些元素在运行时的行为方式的设计过程。其目的在于确定系统结构、组件和交互的基本形式，以及各个组件之间的相互作用关系。系统架构设计的目的是为了建立稳固、灵活、可伸缩、可维护的软件系统。

## 2.2	什么是微服务架构？ 

微服务架构是一种分布式应用程序设计范式，它将单体应用划分成一组小型的独立服务，每个服务运行在自己的进程中，彼此间通过轻量级的通信机制互相协作。它使得应用可以针对不同的任务或用途进行横向扩展或收缩。微服务架构的最大优点是可以按需伸缩，而且每项服务都可以独立部署、升级或替换。因此，微服务架构提供了一种服务治理的方式，其中服务独立部署，降低了耦合性，增强了健壮性、弹性、可靠性。

## 2.3	什么是SOA？ 

面向服务的架构（Service-Oriented Architecture，简称SOA），是一种面向服务的体系结构风格，用于实现面向服务的需求。SOA体系结构采用分布式、松散耦合的服务提供者模式，使得应用模块之间通过服务接口进行通信，以实现需求的自动化和重复利用。SOA把应用程序的复杂性分解为一系列的服务，服务之间存在松耦合关系，通过标准化的消息传递进行通信，并遵循标准的服务质量保证，以此提高可靠性、可扩展性和可复用性。
