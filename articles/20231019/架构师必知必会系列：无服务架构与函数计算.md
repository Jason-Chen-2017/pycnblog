# 1.背景介绍


## 函数计算简介
函数计算（Function Compute）是一个基于阿里云技术栈的新一代服务型一体化计算平台，提供海量函数计算服务，可实现低成本、弹性伸缩、按量计费等能力，可以帮助客户在全球范围内灵活部署、弹性伸缩及运行各种函数，提升业务应用效率、降低成本、节约资源。其主要优点如下：

1. 按需付费：采用按量计费模式，客户只需要付费实际使用的资源，按分钟或秒计费。

2. 高可用：具有极高的可用性和容错性，具备抗灾备能力，通过多区域部署及异地备份，确保用户业务的连续性。

3. 弹性伸缩：可根据用户需求随时增加、减少资源，保证资源利用率及响应时间。

4. 服务无限接近：按需调用，客户通过API接口调用到函数计算平台上所部署的各种函数，同时也支持从本地调用函数，实现更快的响应速度。

5. 安全可靠：通过阿里云的多项安全防护措施，包括网络攻击防护、身份验证、访问控制、数据加密传输等，提供安全可靠的计算环境。

6. 易用友好：兼顾开发者友好性及客户操作简单易懂，用户可以通过界面、SDK及API调用方式，轻松完成函数的部署、调试、发布等工作。

目前，阿里云 Function Compute 提供了五种产品形态，如 SCF、Serverless Kubernetes 、FaaS 、Wind FC 和 EventBridge 。其中，SCF 是最基础的产品形态，用于支持各种编程语言及运行环境，并提供了丰富的触发器及事件源配置，可满足不同场景下的需求；Wind FC 提供了无服务器的事件驱动型计算能力，可以针对事件进行数据处理、数据分析等，适合于对实时数据的处理；FaaS 可以快速创建、运行和管理函数，支持各种编程语言，以面向事件为驱动的工作流编排；Serverless Kubernetes 支持基于 Kubernetes 的容器集群，支持各种编程语言、框架及运行环境，通过声明式的 YAML 文件配置可直接运行函数，并提供自动扩缩容、弹性伸缩等能力；EventBridge 可将事件数据路由至目标端点，用于对各种消息队列、对象存储、数据库等进行集中监控。

无论是哪一种函数计算产品形态，都可以实现客户提交的代码执行，然后返回结果。无服务架构是指不需要预先定义函数模板和触发器，而是在函数调用时动态地按需加载代码、运行环境及资源，并在不损失状态和数据一致性的前提下进行计算。无服务架构与传统架构的差别在于它是按需按量计算，不需要预留资源，适用于处理海量数据的任务，并能够应对突发高负载的情况。因此，无服务架构正成为主流计算架构，它可以在不同的场景中发挥作用，最大限度地释放云计算资源，满足客户的各种计算需求。

## 为什么要学习函数计算？
现在很多公司都在考虑在生产环境中使用函数计算架构，这意味着许多创业公司正在涉足这一领域。那么为什么需要学习函数计算呢？以下是一些原因：

1. 快速响应业务变化：函数计算架构使得企业具备了“敏捷”的能力，能够在短时间内快速响应业务变化，而且无须停机。这种能力是传统架构无法企及的。

2. 节省成本：按量计费模式可降低计算资源的使用成本，节省大量金钱。

3. 节约资源：函数计算架构能自动适配资源，有效节省了企业的IT支出，减轻了运维负担。

4. 投资回报率高：函数计算架构由阿里云提供，可以让创业公司省去购买服务器、域名、公网IP等硬件投入的时间成本。而且阿里云有专门的内部团队跟进支持，能帮助企业避免风险、提升竞争力。

5. 降低运营复杂度：函数计算架构提供了按需弹性伸缩的能力，可以快速响应业务增长或减少带来的压力。而且阿里云有统一的监控中心，可以集中收集、分析和管理函数计算平台上的所有数据，并进行定期的故障诊断和报警。

总之，函数计算架构的应用正在席卷各行各业，只要具备云计算的基础知识、了解业务架构和功能模块，就可以顺利地上手函数计算架构。所以，学习函数计算是非常有必要的！
