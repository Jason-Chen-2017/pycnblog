# 1.背景介绍


物联网(IoT)是一个非常热门的话题。随着智能设备的不断增加，无论是传统设备还是新兴的无人机等，其数量在逐渐扩大，分布范围也越来越广，如何提升设备的连接、数据处理、分析处理能力和可靠性成为了新时代物联网的主要难点之一。而物联网的本质是把物理世界的数据转变成数字信息，并通过网络进行互联互通，因此，物联网所涉及的各个环节都需要有高性能、高可用、低延迟的特点。因此，物联网架构的设计需要考虑各种因素，包括但不限于设备的计算性能、通信性能、存储性能、安全性、成本效率等。另外，由于物联网应用面临的需求变化迅速，边缘计算作为新的IT基础设施层，也正在蓬勃发展。边缘计算是指将计算任务下沉到接近数据的终端设备，可以减少云端服务器的负载、降低网络带宽占用、提高计算资源利用率。边缘计算已成为物联网领域里的一个重要方向，它的架构也需要进行精心设计。本文中，作者结合自己的知识体系和经验，分享物联网和边缘计算之间的密切关系，阐述物联网架构与边缘计算的区别、联系、优势和局限性。
# 2.核心概念与联系
首先，了解物联网的基本概念和特征，包括物理世界、数字信息和网络通信三个方面。物理世界是指现实世界中的实体（如房屋、楼宇、车辆、家电等），这些实体可以通过传感器、加速度计、GPS等设备捕获周围环境的数据，产生数字信号；数字信息则是从传感器接收到的原始数据经过处理得到的有意义的信息，它具有自描述性，比如“这是一辆红色汽车”，“今天天气很好”等；网络通信则是指不同设备之间相互发送和接收数据的过程，比如手机APP通过WiFi、蓝牙、ZigBee等传输数据到服务器上。

其次，要理解物联网架构的基本组成，主要包括两部分，即物理层和数据层。物理层的作用是连接各种设备，实现数据采集、传输和处理，同时还需要保证数据安全、隐私和完整性。数据层的功能是按照业务规则处理收集到的信息，输出相应的结果。物联网架构通常包括一套硬件平台、网络平台、应用平台和服务平台四个层级。其中，硬件平台是指用于物理连接、数据采集、传输的设备，包括传感器、处理器、存储器、路由器等。网络平台是指连接设备的网络环境，包括无线传输、卫星传输等。应用平台是指用于应用部署、管理、处理数据的软件，包括基于云的服务器、移动应用、Web服务、AI平台等。服务平台则提供各种服务，包括认证授权、安全防护、设备管理、流量调控、计费管理等。

最后，要理解边缘计算的定义、特征以及其与物联网的关系。边缘计算就是把计算任务放在接近数据源头的终端设备上执行，可以降低云端服务器的负载、降低网络带宽占用、提高计算资源利用率。根据计算机集群的分布位置不同，分为如下三种类型：端边计算、分布式计算和超级边缘计算。端边计算又称为本地计算，指的是将某些计算任务交给端侧设备完成，如移动设备、嵌入式系统等，这种架构可以降低云端的负载、提高用户体验。分布式计算也称为云计算，将计算任务下沉到分布在不同位置的多台服务器上，采用分布式任务调度方式分配任务，这种架构可以降低单个服务器的负载、提高资源利用率。超级边缘计算也叫做异构计算，它融合了上述两种架构，将计算任务下沉到各种异构设备上，具有更大的弹性和扩展性。

物联网与边缘计算的区别与联系
除了物联网的架构和边缘计算的定义和特征外，还有以下几个方面的区别：
1、架构
物联网和边缘计算都是针对海量数据的处理。物联网的数据层一般采用云架构，其主要目的是为企业的业务提供支持；而边缘计算架构更偏向于集群化的方式，因此边缘节点对数据的处理效率更高，更适合那些对延迟敏感的应用场景。另一个差异是设备的分布范围。物联网涉及的设备一般都分布在企业内，受到内部网络环境限制；边缘计算通常聚焦于边缘地区，因为边缘节点对网络、计算性能有更高要求。

2、数据处理方法
物联网的关键是如何在物理世界和数字世界之间建立起联系，这涉及到数据的采集、传输和处理。物联网中的传感器、处理器和存储器属于物理层，它们分别用来收集、转换和保存数据。而边缘计算的数据处理一般采用微处理器或核处理器，它们运行在边缘节点上，能够实时响应和处理各种输入数据。另一方面，物联网的应用往往与特定业务相关，需要针对性地进行数据处理；而边缘计算则高度模块化，可以灵活应对各种应用场景。

3、计算资源
物联网的计算资源主要由传感器、处理器、存储器等硬件构成，它们能够捕捉、处理、存储环境数据。而边缘计算则依赖于云端服务器，云端服务器可以承载大量的计算任务，并且计算资源有限，无法满足所有边缘计算场景的需求。另一方面，边缘计算更关注计算效率，对计算资源的要求比物联网更高。

4、计算模型
物联网的计算模型主要以机器学习、模式识别和数据挖掘等为代表，目的是对采集到的数据进行分析、分类和预测。而边缘计算的计算模型则是基于云端框架开发的应用软件，它们能够快速响应和处理各种输入数据，同时还能满足诸如安全性、隐私保护、延迟控制等需求。

5、易用性
物联网应用通常需要第三方平台、应用程序和第三方服务的配合才能完成。而边缘计算通常不需要任何第三方组件，只需简单配置即可实现运行。另一方面，边缘计算还需要兼顾可靠性和实时性，以满足用户的高频率、低延迟的需求。

6、成本效益
物联网应用对硬件投资较为昂贵，但是其服务商往往能够提供大规模、高价值的服务。而边缘计算的云服务则具有高效率和低成本的特点，这也是边缘计算与物联网共存的原因之一。
