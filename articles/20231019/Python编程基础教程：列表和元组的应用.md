# 2.核心概念与联系
## 2.1 列表（list）
列表是一种有序的集合，元素可以重复出现，其中的元素都可以通过索引访问，并且支持切片操作。列表的创建可以使用方括号[]来表示，内部的元素通过逗号进行分隔。比如：[1, 2, 'hello', True]，列表的第一个元素索引是0，最后一个元素索引是-1（负数索引表示倒数第几个数）。
```
>>> lst = [1, 2, 'hello', True]
```

## 2.2 元组（tuple）
元组也是一种有序的集合，元素不可以重复出现，其中的元素都可以通过索引访问。元组的创建方式和列表一样，只不过加个圆括号就表示了元组。比如：('apple', 'banana', 'orange')，元组的第一个元素索引是0，最后一个元素索引是-1。
```
>>> tpl = ('apple', 'banana', 'orange')
```

## 2.3 列表和元组的区别
列表和元组在很多方面有些相似之处。共同点如下：

* 可以使用索引访问元素；
* 支持切片操作；
* 长度可变，可以添加或删除元素；

区别主要体现在以下方面：

### 存储位置
列表和元组都可以存储在内存中，但不同的是：列表是在栈区中，因此它的内存占用比较小，性能较好；而元组是在堆区中，因此它的内存占用比较大，性能较差。一般情况下，建议尽量使用元组，因为它具有不可变性，能够提高代码效率。另外，有的函数可能会修改元组的内容，造成错误，如果一定要修改的话，应该使用一个新的元组替换掉旧的元组。

### 创建方式
列表的创建方式是方括号，元组的创建方式则是圆括号。

### 使用场景
一般来说，列表适合用于存储多个相同类型的元素，例如[1, 2, 3]、[True, False]等；而元组通常用于存储多个不同类型的元素，例如(1, 'a')、('hello',)等。

### 赋值方式
由于列表是一个可变数据类型，因此可以对其中元素进行修改，而且可以在任何时候修改；而元组是一个不可变数据类型，因此一旦初始化完成后，其中的元素就不能再修改了。但是，可以通过下面的方法创建一个新的元组，然后对旧元组的元素进行复制，达到修改的效果。

```python
old_tpl = (1, 'a')
new_tpl = tuple([x for x in old_tpl])
print(new_tpl)   # (1, 'a')
```
