# 3.核心概念

# 3.1编程语言
计算机程序的开发一般都要依赖某种编程语言，比如C、C++、Java、Python、JavaScript、HTML、CSS等。

其中，目前最流行的编程语言莫过于Java，占据了市场绝对份额。

Java具有跨平台特性，可以在多个平台上运行，并且有成熟的框架支持，包括Spring和Hibernate等。

# 3.2程序设计理念
- 优雅简洁：编程语言中的关键词，变量名，函数名应该有描述性，清晰易懂。
- 模块化：所有的功能模块都应该被封装成独立的小类或者方法。
- 继承机制：提高代码复用性，减少重复的代码。
- 数据抽象：采用面向对象的方式，把数据和行为绑定起来。

# 3.3高级语言
- 编译型高级语言：Java、C++等，需要先编译，然后才能执行。
- 解释型高级语言：JavaScript、Python、Ruby等，不需要先编译，直接就可以运行。

# 3.4数据库
数据库是一个组织存放、存储和管理数据的容器。它将各种信息保存到文件、磁盘或其他介质上，以方便检索和访问。目前最流行的数据库系统是MySQL。

# 3.5网站开发
网站开发是指根据客户需求，制作和发布网络服务的过程。Web应用开发又称为WEB编程。网站开发的工具包括HTML/CSS/JS、PHP、JSP、ASP.NET等。

# 3.6面向对象的编程
面向对象编程(Object-Oriented Programming，OOP)是一种编程范式，将对象作为程序的基本单元，一个对象包含状态和行为。

- 封装性：通过把数据和行为绑定到一起，实现对象的内部封装。
- 继承性：一个对象可以从另一个对象继承属性和方法。
- 多态性：允许不同类型的对象使用相同的方法。

# 3.7反射机制
反射机制是在运行时动态获取类的属性和方法的一种机制。

可以通过Class.forName()方法加载类的定义，通过getDeclaredMethods()方法获取类的所有方法，通过getMethod()方法获取某个指定的方法。

# 3.8软件工程
软件工程是一门研究开发、维护和管理信息系统的学科。软件工程关注降低软件开发成本、提高软件质量和保证软件安全、可用性的有效方法和实践。

软件工程的基本目标是开发高质量、可靠、稳定的软件。软件工程的关键要素有：需求分析、设计、编码、测试、文档和品质管理。

# 3.9版本控制系统VCS
版本控制系统，也叫源代码管理系统，用于管理和维护源代码，通常分为集中式和分布式管理。集中式VCS，如SVN，是单一服务器上的中心化版本控制系统。分布式VCS，如Git，采用主从模式，每个开发人员都可以拥有一个完整的副本。

# 4.实践技术

## 4.1GitHub搭建个人博客

很多同学都想建立自己的个人博客，这里我们以GitHub Pages + Jekyll 来建立自己的博客，大家也可以按照自己的喜好选择其他方案。

1. 在GitHub上创建仓库

   - 方法1：进入https://github.com/new
   - 方法2：点击右上角头像 -> New repository


   创建仓库名称为`username.github.io`，其中`username`为你的GitHub用户名，注意不能使用特殊字符。

2. 安装Jekyll

   ```
   gem install jekyll bundler
   ```

   `gem`是一个Ruby包管理器，`bundler`是一个依赖管理器。

3. 下载模板


   ```
   unzip 主题压缩包.zip
   cd 主题目录
   bundle install
   ```

   `unzip`命令是解压命令，`bundle install`命令用来安装主题所需的依赖库。

4. 配置文件

   修改`_config.yml`文件，设置正确的基本信息。

   ```yaml
   # Site settings
   title: Your Blog Name   # 博客名称
   email: your@email.com    # 你的邮箱地址
   description: "Write an awesome description for your new site here." # 博客描述
   baseurl: ""     # 如果使用自己的域名，则填入子域名，否则留空
   url: "https://yourusername.github.io"      # 个人博客的网址
   twitter_username: yourtwittername        # 添加你的Twitter账号

   # Build settings
   markdown: kramdown    # 设置kramdown为Markdown引擎
   theme: minima         # 使用minima主题
   plugins:               # 插件列表
     - jekyll-feed       # 添加jekyll-feed插件，以生成Atom或RSS feeds
     - jemoji            # 支持表情符号
   ```

5. 编写博文

   把下载的模板解压后，进入`_posts`目录，新建一个`.md`文件，命名格式为`yyyy-mm-dd-title.md`。

   ```markdown
    ---
    layout: post
    title: Hello World!   # 博文标题
    date: 2020-12-31 12:00:00 +0800   # 发布日期
    categories: test   # 分类标签
    tags: [hello, world]   # 标签列表
    ---

    # 欢迎来到我的博客！

    这是我的第一篇博文。

   ```

   每篇博文必须包含YAML Front Matter。

   YAML Front Matter包含三部分：

   1. Layout：布局类型，固定为`post`
   2. Title：博文标题
   3. Date：发布日期，格式为`YYYY-MM-DD HH:MM:SS +/-TTTT`，例如`2020-12-31 12:00:00 +0800`表示中国标准时间2020年12月31日12点00分。
   4. Categories：分类标签列表
   5. Tags：标签列表

   正文部分写法与Markdown一致。

   用浏览器打开本地的博客网址，查看首页和博文页面，如果有错误提示，请检查配置是否正确。

6. 提交修改

   提交修改前，记得确认以下几点：

   1. _config.yml 文件中的信息填写正确
   2. _post 中的 YAML Front Matter 填写正确
   3. 博文内容正确
   4. 浏览器访问网址查看是否正常显示

   如果出现问题，可以先删除重新上传一遍，或者在评论区留言。

   ```bash
   git init        # 初始化本地仓库
   git add.       # 将所有文件添加到暂存区
   git commit -m "commit message"   # 提交修改，"commit message"为提交说明
   git remote add origin https://github.com/username/username.github.io.git   # 设置远程仓库地址
   git push -u origin master   # 将本地提交推送至远程仓库
   ```

   执行以上命令，提交修改至GitHub。

7. 部署完成

   当提交修改并推送成功后，等待GitHub Pages更新，几分钟左右，就可以在个人博客网址上看到自己的博客了。

   部署完成后，可以在网站根目录下的`.nojekyll`文件里加入`/* */`，以解决不识别中文的问题。
