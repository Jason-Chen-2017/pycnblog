# 3.基本概念术语说明
## 3.1 GitHub
GitHub是一个基于云端的代码仓库管理平台，它允许用户在线存储代码、分享想法、协作开发项目。这里有一个GitHub账号，你可以用来尝试本文所述的方法。

## 3.2 概念术语
### 3.2.1 Git
Git是一种开源的分布式版本控制系统，用于跟踪文件内容的变化，可以记录每次更新，从而提供版本历史。在GitHub中，Git被用来跟踪代码的改动，所以有必要掌握Git的相关概念和术语。

#### 3.2.1.1 本地仓库（Local Repository）
当你从GitHub下载代码到你的本地机器后，代码就保存在本地仓库（local repository）里。这意味着你对文件的修改、添加、删除都只能在本地仓库里进行。

#### 3.2.1.2 分支（Branch）
当你从本地仓库克隆一个仓库到本地时，默认情况下，会创建一个名为“master”的分支，你也可以创建其他分支。分支是Git中的重要概念，因为它允许多个开发人员同时并行开发代码，并最终将各自的改动合并到主干上。

#### 3.2.1.3 提交（Commit）
提交是指将文件暂存区（Stage Area）里的文件，保存到本地仓库的过程。提交之前，你需要先把文件暂存起来。

#### 3.2.1.4 推送（Push）
推送是将本地仓库里的文件上传到远程仓库的过程。推送之前，你需要先将本地仓库与远程仓库关联起来。

#### 3.2.1.5 拉取（Pull）
拉取是从远程仓库下载代码到本地仓库的过程。一般来说，你只需要执行一次拉取就可以得到最新代码。

#### 3.2.1.6 拉取请求（Pull Request）
拉取请求是一个功能，允许你向其他开发者请求把你的代码合并到主干（Master Branch）。其他开发者可以通过审查你的代码，决定是否接受你的请求。如果你对代码有疑问或建议，可以在评论栏中留言。

### 3.2.2 Docker
Docker是一个开源的容器虚拟化平台，它使你能够打包应用程序及其依赖项到一个轻量级的、可移植的镜像中，并可以在任何地方运行该镜像。你也可以通过Docker Hub或私服分享镜像。

## 4.3 使用AutoHotKey
你可能已经知道，AutoHotKey是一个免费、开源的键盘宏工具，它允许你使用快捷键执行常用任务。不过，正如之前所说，你没有金钱，不能白嫖他人的软件。那，我们为什么不自己动手制作一个呢？下面，我将带你走进程序员的世界，并用AutoHotKey制作一套软件来帮助你节省时间。

## 4.4 使用Windows计划任务
Windows计划任务是一个任务计划工具，它允许你设置自定义任务，让它们按照预定的时间自动执行。你只需要设置好计划任务，就可以自动关闭网络连接、锁定屏幕、关闭后台进程等。

### 4.4.1 添加任务
在开始菜单里搜索“任务计划程序”。


在出现的对话框中，单击“创建基本任务...”


在任务编辑对话框中，填写相关信息：

1. 任务名称：自定义任务名称。
2. 按钮：根据实际情况选择“立即执行”或“每天”。
3. 操作类型：选择“开始一个新的程序”选项卡。
4. 程序/脚本：浏览到AutoHotkey的安装目录并选择你的脚本文件。
5. 参数（可选）：如果你需要传入参数给脚本，请输入参数值。
6. 条件（可选）：你可以设置触发计划任务的特定时间。
7. 身份验证：如果你的用户名或密码更改了，需要重新认证。


### 4.4.2 配置计划任务
点击“配置”按钮，进入“任务详细信息”对话框。在“常规”选项卡下，设置如下选项：

1. 启用：默认启用计划任务，不需要额外操作。
2. 用户帐户：默认使用“NT AUTHORITY\SYSTEM”，无需修改。
3. 频率：根据你希望计划任务运行的时间选择。
4. 最初日期：计划任务首次运行的时间。
5. 结束日期（可选）：计划任务停止运行的时间。

在“Triggers”选项卡下，可以设置计划任务的触发条件。比如，你可以设置定时任务每天早上六点运行。


点击“确定”完成任务的设置，然后单击“关闭”退出。

至此，你已经设置好Windows计划任务，它会在指定时间自动执行你的脚本。

## 4.5 使用GitHub Actions
GitHub Actions是一个由GitHub提供的持续集成（Continuous Integration）和持续部署（Continuous Deployment）服务。它允许你自动构建和测试代码，并在每次代码被push到仓库时部署到生产环境。

### 4.5.1 在GitHub上创建仓库
在GitHub上注册账号，然后创建一个新仓库。名字可以任意起一个，比如“autohotkey-time-saver”。


### 4.5.2 安装GHA Runner
第一步，你需要安装GHA Runner。GHA Runner是一个在服务器上运行的服务程序，它监听GitHub上的仓库事件，并自动运行你定义的任务。我们需要在GitHub上安装GHA Runner才能运行AutoHotkey脚本。

1. 登录到你的GitHub账户。
2. 访问https://github.com/marketplace/actions/setup-node-jerkins，单击“Set up for development workflows”。


3. 在安装页面上，单击“Install this app”，然后授权GitHub安装你的APP。
4. 等待安装完成。
5. 返回https://github.com/{your_username}/{repo_name}/settings/actions，单击“Add runner”，然后选择安装好的Runner。


6. 将安装好的Runner添加到仓库。

### 4.5.3 配置Workflow
第二步，是配置Workflow。在仓库根目录下创建一个名为“.github”的隐藏文件夹。在里面新建一个名为“workflows”的文件夹。


在“workflows”文件夹内，创建一个YAML格式的配置文件，比如“autotimesave.yml”。

```yaml
name: AutoTimeSaver

on:
  schedule:
    - cron: '0 6 * * *' # run at 6 AM every day

jobs:
  build:

    runs-on: windows-latest
    
    steps:
      - name: Checkout
        uses: actions/checkout@v2

      - name: Run autohotkey script
        shell: pwsh
        run: |
          $env:Path += ";C:\Program Files\AutoHotkey"
          ahk /errorstdout "C:\path\to\your\script.ahk"
```

其中，cron: '0 6 * * *' 表示每天的6点运行，你也可以根据自己的需要调整运行时间。run: 中的脚本路径需要替换成你自己存放AutoHotkey脚本的绝对路径。

```powershell
$env:Path += ";C:\Program Files\AutoHotkey"
ahk /errorstdout "C:\path\to\your\script.ahk"
```

此处，我假设你已经在PATH变量里添加了AutoHotkey的安装目录。如果还没添加，你需要手动将安装目录添加到PATH变量里。

```powershell
echo "$env:Path"
```

单击“Start commit”，输入提交信息，点击“Commit changes”，保存你的工作进度。

### 4.5.4 查看日志
最后一步，是查看日志。在Actions页面，单击“AutoTimeSaver”下的最新日志。


在日志页面，你可以看到运行结果，包括运行耗时、退出码等。你可以点击日志中的任意一条信息，查看详细错误信息。
