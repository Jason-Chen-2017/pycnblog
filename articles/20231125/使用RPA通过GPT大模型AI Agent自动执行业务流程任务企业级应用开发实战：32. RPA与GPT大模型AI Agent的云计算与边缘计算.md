                 

# 1.背景介绍


人工智能（Artificial Intelligence，简称AI）在今天已经成为产业界主要驱动力。传统IT行业为了追求效率，多采用“手工”的方式来处理重复性工作，并将更多精力投入到机械化和自动化上。然而随着越来越多的人工智能产品问世，自动化的基础设施越来越丰富、应用场景越来越广泛。特别是在工业领域，传感器、工艺流程、工单、报告等信息渠道不断被自动化设备所替代，实现数字化转型变革的进程正在加速推进。
相对于传统的手动工作流程来说，采用AI的自动化工具可以节省时间、降低风险，提高效率。机器学习方法如深度学习（Deep Learning）等技术被用来处理大量数据，从而使计算机能够更好地理解、分析和预测。因此，机器学习模型已成为人工智能领域中最具影响力的工具之一。
工业界普遍认识到使用机器学习模型的重要性，比如涉及制造、销售等关键环节，都需要用到大量的数据进行训练和测试。但是如果用手工方式完成这些繁琐的重复性任务，很容易出现错误。于是，人们开始探索如何利用机器学习模型解决实际问题。机器学习模型的一些常见应用场景如图像识别、文本分类、异常检测、推荐系统等。当下主流的机器学习模型包括决策树、神经网络、支持向量机、深度学习等。这些模型的训练数据和目标都是手工提供的，效率不高，并且往往无法满足现实世界的复杂环境需求。
另一个方向是将模型部署到云端、边缘端或者嵌入式设备，用于实时处理数据。这样就可以通过连接设备的不同位置，实现跨越不同平台、地域甚至国家的数据共享，实现异地备份、异地灾难恢复等功能。这就为模型的部署提供了巨大的便利，减少了模型的训练时间，同时也降低了本地资源的使用成本。
基于上述原因，人工智能（AI）模型越来越多的被部署到边缘端，以及运行在云端。如今，实现自动化工作流任务的方案种类繁多，涵盖了各种业务场景。无论是面对传统的生产工序，还是快速响应市场变化的反应式流程，自动化的优势不可忽视。RPA（Robotic Process Automation，即机器人流程自动化）技术是一个成熟的解决方案，它为企业提供了一个可编程的自动化引擎，可以根据业务需求快速生成自动化脚本。
通常情况下，RPA技术需要服务端软件配置安装、运行、维护等繁琐的技术门槛。另外，企业要实现业务流程自动化，仍然需要花费较长的时间，包括培训、沟通、技术支持等开支。这就是为什么现在越来越多的企业考虑采用“云+边缘+RPA”的方案来实现业务流程自动化。
