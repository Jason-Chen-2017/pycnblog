# 2.核心概念与联系

## 数据类型
### 1.关系型数据库（Relational Database Management System, RDBMS）
关系型数据库是一个建立在表格上面的数据库，它的主要特征就是数据是按照行和列的方式存储的，每一行代表一个记录，每一列代表一个字段。关系型数据库分为三类：

- 基于磁盘的关系型数据库：如MySQL、PostgreSQL、Oracle、SQLite等，通常使用关系模型，也就是数据都是存放在表里面的，它可以把复杂的数据组织成一种有机的结构。
- 分布式关系型数据库：如SQL Server、IBM DB2、MySQL Cluster等，这种数据库是以分布式的方式存储的，通过网络把数据复制到不同节点上去，让每个节点保存完整的数据。
- NoSQL数据库：NoSQL意指“Not Only SQL”，表示非关系型数据库，它的概念比较晚一些，主要是为了克服关系型数据库所面临的一些问题而提出的。典型的NoSQL数据库包括：MongoDB、Couchbase、Redis等。

### 2.文档型数据库（Document-oriented Database）
文档型数据库以JSON、XML、BSON格式存储数据，它不像关系型数据库那样使用表格结构存储数据，而是存储的是文档（document），也就是说数据被组织成了一个个的文档。典型的文档型数据库包括MongoDB、Couchbase等。

### 3.键值型数据库（Key-Value Store）
键值型数据库主要用来存储少量的、简单的数据，它没有固定的模式，只需要提供一个key就可以获取对应的值。典型的键值型数据库包括Memcached、Redis、Riak、LevelDB等。

### 4.图形数据库（Graph Database）
图形数据库是用来存储复杂的关系数据，比如关系型数据库中的数据，它也可以用来存储网络结构数据或者其他复杂的数据。图形数据库一般都采用图结构存储数据，用于高性能地处理复杂的数据关联问题。典型的图形数据库包括Neo4J、InfoGrid等。

### 5.搜索引擎数据库（Search Engine Database）
搜索引擎数据库主要用来存储和检索大量文本信息，具有快速索引、全文搜索等特性。典型的搜索引擎数据库包括Elasticsearch、Solr等。

## 操作方法
- 插入（insert）：往某个表或集合插入一条记录。
- 查询（select）：从某个表或集合中查询指定条件的数据。
- 更新（update）：修改某个表或集合中指定条件的数据。
- 删除（delete）：从某个表或集合中删除指定条件的数据。
- 创建（create）：创建一个新的数据库或表。
- 删除（drop）：删除一个已有的数据库或表。
- 事务（transaction）：一个事务是一个不可分割的工作单位，它包含了一系列的数据库操作，要么都成功，要么都失败，避免出现数据不一致的问题。
