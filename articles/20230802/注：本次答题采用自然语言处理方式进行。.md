         # 2.图像分类
         ## 2.1 概念
         图像分类（Image Classification），又称分类，属于计算机视觉（Computer Vision）的一个子任务，其目标是在一组图像中识别出某种特征并将其归类到预先定义好的不同类别之中，即对输入的图片进行分类。图像分类可以帮助用户快速了解、理解和处理大量的图像信息，提升工作效率，改善用户体验。
         ## 2.2 特点
         - 非监督型：不依赖标注数据或标签信息，仅利用图像本身进行分类；
         - 模型训练简单：图像分类模型通常不需要太复杂的网络结构或超参数，只需要很少的数据即可完成训练；
         - 模型易部署：训练好的模型能够直接部署到生产环境中，作为后端服务对外提供服务，满足用户的需求；
         - 适用范围广泛：图像分类任务可以在多个领域得到应用，例如电商、安防、交通安全、医疗、金融、人脸识别、运动行为分析、风险控制等。
         ## 2.3 基本流程
         1. 数据收集：首先需要收集足够数量的训练图片，这些图片应具有代表性。如果数据量过小，则模型的准确度会受到影响。通常情况下，训练图片应由具有相似主题的图片组成。
         2. 数据处理：图像分类的关键一步就是对图像进行预处理，包括图像增强（Data Augmentation）、归一化（Normalization）、数据划分（Train/Validation/Test Split）。
            Data Augmentation: 是指通过一些手段扩充原始图像数据集的方法。比如旋转、缩放、裁切、变化亮度、颜色等方法，目的在于增加数据集中的样本，提升模型在样本上的泛化能力。
            Normalization: 对图像像素值进行规范化，使得不同层之间的特征向量之间更加平稳。
            Train/Validation/Test Split: 将数据划分为训练集、验证集和测试集，分别用来训练、优化模型参数、评估模型效果。
         3. 模型设计：图像分类任务通常涉及到多个类别，因此需要选择一个合适的模型架构。目前常用的图像分类模型有卷积神经网络（CNN）、循环神经网络（RNN）、全连接网络（FCN）、单层感知机（SVM）等。根据数据大小、图片的复杂程度、是否有额外的信息等因素，选取不同的模型架构。
         4. 模型训练：训练过程是一个迭代过程，需要反复调整模型的参数，使得模型在验证集上的精度不断提升。通常，通过在训练集上采用交叉熵损失函数和SGD随机梯度下降法进行训练，当模型在验证集上性能不再提升时，停止训练。
         5. 模型测试：在测试过程中，模型对测试图片进行分类，计算正确率。最后输出分类结果。
