## 面试题与算法编程题解析

### 1. 阅读以下代码片段，并回答问题：

```go
package main

import "fmt"

func main() {
    m := map[string]int{"Alice": 25, "Bob": 30}
    for key, value := range m {
        if value > 30 {
            delete(m, key)
        }
    }
    fmt.Println(m)
}
```

**问题：** 该代码片段执行后，`m` 映射的内容是什么？

**答案：** 执行后，`m` 映射的内容为：`{"Bob": 30}`。

**解析：** 在这个例子中，我们遍历了 `m` 映射的所有键值对。当遇到 `value > 30` 时，通过调用 `delete(m, key)` 删除该键值对。由于 `Bob` 的值为 30，没有大于 30，因此没有被删除。

### 2. 给出以下代码片段的输出结果：

```go
package main

import "fmt"

func main() {
    c := make(chan int, 2)
    c <- 1
    c <- 2
    fmt.Println(<-c)
    fmt.Println(<-c)
}
```

**问题：** 该代码片段执行后，会输出什么？

**答案：** 执行后，会输出 `1` 和 `2`。

**解析：** 在这个例子中，我们创建了容量为 2 的带缓冲通道 `c`。首先，我们向通道中发送两个值。由于通道的缓冲区容量为 2，这两个发送操作不会阻塞。接着，我们使用 `<-c` 从通道中接收值。由于通道中有值，这两个接收操作不会阻塞。因此，输出结果为 `1` 和 `2`。

### 3. 阅读以下代码片段，并回答问题：

```go
package main

import (
    "fmt"
    "sync"
)

func main() {
    var mu sync.Mutex
    mu.Lock()
    fmt.Println("Lock acquired")
    mu.Unlock()
    fmt.Println("Lock released")
}
```

**问题：** 该代码片段执行后，会输出什么？

**答案：** 执行后，会输出：

```
Lock acquired
Lock released
```

**解析：** 在这个例子中，我们使用互斥锁 `mu` 保护了打印语句。由于 `mu.Lock()` 调用在打印语句之前，`mu.Unlock()` 调用在打印语句之后，因此会先输出 `Lock acquired`，然后输出 `Lock released`。

### 4. 给出以下代码片段的输出结果：

```go
package main

import "fmt"

func main() {
    var wg sync.WaitGroup
    wg.Add(1)
    go func() {
        defer wg.Done()
        fmt.Println("goroutine")
    }()
    wg.Wait()
    fmt.Println("main")
}
```

**问题：** 该代码片段执行后，会输出什么？

**答案：** 执行后，会输出：

```
goroutine
main
```

**解析：** 在这个例子中，我们使用 `sync.WaitGroup` 来等待一个 goroutine 完成。首先，我们添加了一个计数，然后启动了一个新的 goroutine。由于 `wg.Done()` 调用在 goroutine 中，它会在 goroutine 执行完成后减少计数。由于 `wg.Wait()` 调用在 goroutine 之后，它会等待计数变为 0，然后输出 `main`。
