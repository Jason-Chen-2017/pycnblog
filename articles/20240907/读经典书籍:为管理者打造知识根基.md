### 一线大厂面试题与经典书籍解析

#### 1. 函数是值传递还是引用传递？

**题目：** Golang 中函数参数传递是值传递还是引用传递？请举例说明。

**答案解析：** Golang 中所有参数都是值传递，这意味着函数接收的是参数的一份拷贝，对拷贝的修改不会影响原始值。

**举例代码：**

```go
package main

import "fmt"

func modify(x int) {
    x = 100
}

func main() {
    a := 10
    modify(a)
    fmt.Println(a) // 输出 10，而不是 100
}
```

**关联书籍：《卓有成效的管理者》** 该书强调管理者要掌握时间管理、人际关系、决策能力等，与本题中函数参数传递的值传递特性有异曲同工之妙。

#### 2. 如何安全读写共享变量？

**题目：** 在并发编程中，如何安全地读写共享变量？

**答案解析：** 可以使用互斥锁、读写锁、原子操作、通道等方法来保证共享变量的安全读写。

**举例代码：**

```go
package main

import (
    "fmt"
    "sync"
)

var (
    counter int
    mu      sync.Mutex
)

func increment() {
    mu.Lock()
    defer mu.Unlock()
    counter++
}

func main() {
    var wg sync.WaitGroup
    for i := 0; i < 1000; i++ {
            wg.Add(1)
            go func() {
                    defer wg.Done()
                    increment()
            }()
    }
    wg.Wait()
    fmt.Println("Counter:", counter)
}
```

**关联书籍：《深度学习》** 该书强调并行计算在深度学习中的应用，与本题中的并发编程有紧密联系。

#### 3. 缓冲、无缓冲 chan 的区别

**题目：** Golang 中，带缓冲和不带缓冲的通道有什么区别？

**答案解析：** 无缓冲通道发送操作会阻塞，直到有接收操作准备好接收数据；接收操作会阻塞，直到有发送操作准备好发送数据。带缓冲通道发送操作只有在缓冲区满时才会阻塞；接收操作只有在缓冲区为空时才会阻塞。

**举例代码：**

```go
// 无缓冲通道
c := make(chan int)

// 带缓冲通道，缓冲区大小为 10
c := make(chan int, 10) 
```

**关联书籍：《创新者的窘境》** 该书讲述了企业如何面对技术变革，这与本题中通道缓冲区的作用有相似之处。
