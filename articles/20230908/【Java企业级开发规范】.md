# 2. Java开发者规范概览
## 1. 编码规范
编码规范(Coding Standards)是一种详细规定程序源码的书写格式、命名约定、变量名称、文件结构等等规则的指南或规范,它使得阅读、修改和维护源代码的人对其更容易理解和快速熟悉,也减少了因编写代码引起的错误及差错。
编码规范可以分为两个层次:语法规范和命名规范。
### 1.1 语法规范
语法规范是在编程语言和开发环境上定义的一套编码规范,其中包括语句的排版方式、缩进、空白符号、关键字等内容,目的是使代码更加易懂和易读。语法规范通过自动化工具检查代码的一致性、健壮性和可读性,为程序员提供统一的编程风格,提高软件开发效率和质量。
一般来说,对于Java开发,推荐的语法规范有Google Code Style Guide、Sun Java Coding Convention、阿里巴巴Java开发手册。
### 1.2 命名规范
命名规范主要涉及到标识符的命名、变量名、函数名、类名、包名、参数名等命名风格,目的是为了让代码更容易被他人理解和记忆。命名规范既有助于代码的可读性,也有助于提高软件质量、降低维护难度,并降低风险。
一般来说,Java开发者应遵循驼峰命名法和大小写敏感的原则,一般不允许出现下划线、连字符、多个单词组合作为标识符。除此之外,还应该注意避免使用拼音、中文、英文混合的方式命名。
## 2. 命名规范
### 2.1 文件夹命名规范
文件夹命名规范是指按照标准的层级结构规范对不同类型的文件和文件夹进行分类归纳和组织的原则。采用这种方式能够让工程师更好的识别文件之间的关系和依赖关系。
一个合理的文件夹结构通常包括如下几种类型的文件或文件夹:

1. 源代码文件: 所有的源代码文件应该放在src文件夹内,包括java、xml、properties、jsp等文件。
2. 测试文件: 测试文件放在test文件夹中,包括junit测试文件和testng测试文件。
3. 配置文件: 配置文件放在config文件夹内,包括各种配置文件,如properties、xml、ini等。
4. 脚本文件: 脚本文件放在scripts文件夹中,包括shell脚本、perl脚本、python脚本等。
5. 编译后的文件: 在执行编译命令时,编译器产生的可执行文件、动态链接库文件、class字节码文件等放在bin文件夹中。
6. 数据文件: 数据文件放在data文件夹中,如csv、txt、sql数据文件。
7. 文档文件: 所有的文档文件放在docs文件夹中,包括word、ppt、excel文档等。
8. 资源文件: 所有的资源文件比如图片、视频、css样式表、js脚本文件等都放在resources文件夹中。
9. 第三方jar包: 如果项目依赖其他第三方jar包,那么这些jar包应该放在lib文件夹中。
### 2.2 类命名规范
类命名规范是指对类的命名要体现其功能特性、继承关系、实现接口的目的,并且要有描述性且易于理解。建议使用名词或名词短语来命名类。类名应全部小写,单词之间使用下划线连接。以下是一些常用的类命名规范:

1. Controller类: 控制器类主要负责处理请求,一般以Controller结尾。
2. Service类: 服务类用于封装业务逻辑,一般以Service结尾。
3. DAO类: 数据访问对象类用于操作数据,一般以DAO结尾。
4. Repository类: 仓库类用于管理业务数据,一般以Repository结尾。
5. DTO类: 数据传输对象类用于封装业务数据,一般以DTO结尾。
6. Entity类: 实体类用于保存业务数据,一般以Entity结尾。
7. VO类: Value Object类用于存放简单的值对象,一般以VO结尾。
8. Exception类: 异常类用于封装业务异常信息,一般以Exception结尾。
9. Enum类: 枚举类用于定义固定值集合,一般以Enum结尾。
10. Config类: 配置类用于配置一些全局变量、常量等属性,一般以Config结尾。
11. Bean类: Bean类用于实现实体类之间的交互,一般以Bean结尾。
12. Auxiliary类: 辅助类主要是一些工具类,一般以Auxiliary结尾。
13. Factory类: 工厂类用于创建对象,一般以Factory结尾。
14. Manager类: 管理类用于处理特定业务流程,一般以Manager结尾。
15. Util类: 工具类用于提供一些通用功能,一般以Util结尾。
16. Client类: 客户端类用于向服务器发送请求,一般以Client结尾。
17. Constants类: 常量类用于保存一些系统常量,一般以Constants结尾。
18. Handler类: 处理类主要用于处理客户端请求,一般以Handler结尾。
19. Message类: 消息类用于表示一个消息对象,一般以Message结尾。
20. Cache类: 缓存类用于缓存业务数据,一般以Cache结尾。
21. Log类: 日志类用于记录日志信息,一般以Log结尾。
22. Listener类: 监听类用于监听客户端请求,一般以Listener结尾。
23. Processor类: 处理器类用于处理请求,一般以Processor结尾。
24. Helper类: 助手类用于提供一些便捷的方法,一般以Helper结尾。
