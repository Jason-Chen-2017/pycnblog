# 2.核心概念术语介绍
## Web开发架构的演进过程
在Web开发的历史上，经历了几个阶段。从静态页面到动态页面，再到MVC，最后还有前后端分离。但随着互联网应用的发展，又有了新的架构设计趋势出现。比如React，Angular，Vue这些MVVM框架，SSR服务端渲染等技术，使得Web前端开发的架构演变成了一个整体。
### 单页应用SPA（Single-Page Application）
在SPA架构中，前端用JavaScript渲染出完整的页面，不向服务器请求HTML，所有的更新都是通过AJAX（Asynchronous JavaScript and XML）获取数据后直接更新页面，提升用户体验。这种架构解决了传统Web开发中“页面之间跳转”的问题，极大的减少了浏览器与服务器之间的通信次数，加快了页面响应速度。


### MV*架构
MV*架构是指Model-View-Controller（模型-视图-控制器）架构的缩写。其中，Model指的是数据模型，负责存储和管理数据；View指的是用户界面，负责呈现数据给用户；Controller指的是业务逻辑，负责处理用户请求并驱动Model和View进行交互。


### 框架与库
JavaScript的生态圈非常广泛，各种开源项目和框架层出不穷。为了更好地利用这些库和框架，提升开发效率，前端工程师们设计了一套规范，并制定了一整套架构设计方法论。这些规范包括前端工程的目录结构，模块化开发方式，技术选型指南等。

## 浏览器中的JavaScript
在浏览器中，JavaScript通过HTML DOM和BOM API访问DOM树和其他对象，可以实现各种功能，如表单验证，AJAX交互，事件处理，动画效果等。


当浏览器加载完页面后，会解析HTML代码，构建DOM树，然后解析CSS样式表，生成渲染树。JavaScript脚本在执行时，会操纵渲染树或执行修改DOM树的操作，重新渲染页面。

### 执行环境
浏览器执行JavaScript有两种环境：

#### 标准模式
这是默认的执行环境，也是最严格的执行环境。任何正常的JavaScript语句都可以在这种环境下执行，包括变量声明，条件判断，循环语句等。

#### 沙盒模式（又称严格模式）
这种模式的特点就是完全隔离全局作用域，限制了对外部作用域的污染，只能通过eval()函数来调用外部的JS代码。不过它依然保留了window对象的属性和方法，可以访问当前页面的DOM元素和属性。


## 模块化
模块化编程，即将一个复杂的应用程序分割成多个小的文件，并且每个文件只关注自己的功能和数据，而不依赖于其他模块。这样做的好处很多，首先可以提高代码的复用性，同时也让维护和开发更简单，避免了重复的代码，可以有效降低软件开发的成本。

### AMD(Asynchronous Module Definition)和CMD(Common Module Definition)
AMD和CMD是两个模块定义规范，用来定义模块的异步加载方式。它们的区别主要在于对依赖的模块的处理。AMD希望依赖的模块都预先加载完成，才能执行define的回调函数，如果出现某个依赖模块没有加载成功的情况，则使用require回调函数通知调用者加载失败。而CMD则是提倡在define时就指定依赖关系，也可以延迟执行某个依赖模块，直到真正执行的时候才检查依赖关系是否存在，不存在则加载失败。

目前主流的前端模块化方案有requirejs，seajs，webpack等。他们有各自的特色，但一般都会兼容AMD和CMD规范，并且提供了方便易用的API接口。因此，前端开发人员可以根据需要选择合适的模块化方案。

## 调试工具
Chrome DevTools和Firefox开发者工具，是调试JavaScript的利器。DevTools提供了强大的调试功能，包括断点设置，监控变量，查看调用堆栈，快速搜索并替换文本，网络性能分析等。
