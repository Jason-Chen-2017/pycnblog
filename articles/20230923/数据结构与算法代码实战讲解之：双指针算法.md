# 1.简介
  

双指针算法（two-pointer algorithm）是在计算机科学、编程语言、数学等领域中一个重要且经典的算法类别。它的主要思想就是利用两个游标或者指針在数组或链表中移动，从而达到遍历整个数组或链表的目的。双指针算法可以极大的减少对内存的占用，以及实现高效的查找和排序算法。因此，双指针算法应用非常广泛。本文将详细介绍双指针算法。
# 2.基本概念术语说明
## 2.1 数组与链表
对于算法来说，首先需要搞清楚的是数组和链表的区别和联系。数组是一系列相同的数据类型元素的集合，按照索引顺序存储，可以快速随机访问；链表则不同于数组，它是一种动态数据结构，支持动态添加、删除元素，并且可以在运行时动态调整数据位置。一般情况下，数组比链表更加适合静态数据查询，但对于动态数据更新要求比较高的场合，链表更加适合。

## 2.2 指针和引用
在数组和链表中都存在着指针和引用的概念。指针是一个变量，保存了数据地址，可以用来间接访问数组中的元素；引用则是一个指针变量，保存了数据的实际值。比如，有一个数组int arr[]={1,2,3}; int *p=arr; p=&arr[0];则p指向数组首元素的地址，如果把*p改成5，那么arr[0]的值也会变为5。但是，如果通过引用arr[0]=5方式修改，只改变了数组arr中的第一个元素的值，并不会影响指针p的指向。总结来说，指针是用于间接访问数组中的元素，而引用则是用于直接访问数组中的元素。

## 2.3 滑动窗口
滑动窗口（Sliding Window）是一种串行算法，即每次迭代都向前移动固定长度的窗口，完成固定数量的工作。滑动窗口通常用于解决字符串、数组、链表等特定类型的问题。常见的滑动窗口包括滑动最小值窗口、滑动最大值窗口、子数组的最大平均值、旋转数组的最小覆盖子集、连续子序列的最大乘积等。

## 2.4 分治策略
分治策略（Divide and Conquer）也是一种串行算法，其基本思想是将大任务划分为较小的子问题，然后递归求解每个子问题，最后合并结果得到大任务的解。一般情况下，分治算法既易于理解，又容易实现，而且效率很高。广义上讲，任何时候都可以将一个复杂的任务拆分成多个相互独立的子任务，然后再将这些子任务合并成最终结果。常见的分治算法有归并排序、快速排序、二分搜索树、动态规划等。

## 2.5 贪心算法
贪心算法（Greedy Algorithm）是一种局部最优的算法。在每一步选择时，都做出当前看起来最好的选择，从而希望导致全局最优的策略。贪心算法往往具有简单性、确定性和可行性。常见的贪心算法包括贪心选择法、Huffman编码、 Prim最小生成树算法等。

## 2.6 深度优先搜索与广度优先搜索
深度优先搜索（Depth First Search）是一种串行算法，即先搜索根结点，然后依次扩展左儿子和右儿子。广度优先搜索（Breadth First Search）则是另一种串行算法，它采用队列实现，即先搜索根结点的一层，然后扩展其各个孩子结点，再移至下一层，直到所有层均处理完毕。深度优先搜索一般用于无向图，广度优先搜索一般用于有向图。

## 2.7 次线性时间内的排序
欧拉通路（Eulerian Path）、汉密尔顿回路（Hamilton Cycle）都是一种次线性时间内的排序算法。所谓次线性时间内的排序，意味着找出一个线性序数，使得任意两点间至多只有一条通路。欧拉通路（Eulerian Circuit）就是在无向图中找出一条起点和终点都没有其他入射边和出射边的路径。汉密尔顿回路（Hamiltonian Cycle）就是在无向图中找出一条起始结点到每一个结点都恰好有一次出射的回路。有些算法比如DFS、BFS，通过生成邻接表或者邻接矩阵的方式，并不显式地维护图的度信息，所以它们的运行时间与节点数目相关而不是与节点的度相关。所以，这种情况下，为了找出次线性时间内的排序算法，可以通过构建含有度信息的图来保证运行时间的线性度量。
