# 1.简介
  

## 什么是HTTP?
HTTP(Hypertext Transfer Protocol)即超文本传输协议。它是基于TCP/IP协议族的应用层协议，其定义了一种用于从万维网服务器传输超文本到本地浏览器的协议。因此，HTTP是一个建立在TCP协议之上的协议。
## 为什么要学习HTTP?
任何涉及网络开发的工作，都离不开HTTP协议。在实际应用中，它扮演着至关重要的角色。比如，访问网站时，客户端需要向服务器发送HTTP请求；服务器响应请求并返回HTTP响应；浏览器解析HTTP响应，并展示出相应的内容给用户；当用户点击链接、输入搜索关键字等的时候，浏览器也会向服务器发送HTTP请求。学习HTTP协议对掌握网络开发技术有着重要意义。
## HTTP协议是如何工作的？
HTTP协议是客户端与服务器端之间的数据交换格式。一般情况下，客户端和服务器端通过HTTP协议通信之前，需要先建立连接。连接建立后，双方就可以按照约定的规则进行数据传输。HTTP协议是无状态的，也就是说，不会在客户端或服务器端保存关于之前事务的信息。HTTP协议支持持久连接，可以减少网络延迟，提高性能。
### 请求方法
HTTP协议支持各种不同的请求方法。常用的请求方法有GET、POST、PUT、DELETE、HEAD等。下面详细介绍一下这些方法。
#### GET
GET方法是最简单的HTTP请求方法。客户端通过HTTP请求URI获取资源。
例如，打开浏览器输入地址`https://www.baidu.com`，浏览器首先向域名系统DNS解析该域名，得到IP地址后，向对应的IP地址发送一个HTTP GET请求。如下图所示：
在请求行中，GET表示请求方法，URI表示请求资源路径。请求头中指定HTTP版本、内容类型等信息。请求体中为空，不需要携带额外数据。服务器收到请求之后，从文件系统或者数据库读取资源并将其返回给客户端，响应头中包括返回数据的大小、类型、编码方式等信息。响应体中就是资源本身。
#### POST
POST方法用来传输实体的主体。
比如，在百度搜索框里输入关键字"java"，按下回车键之后，浏览器首先向服务器发送一个HTTP POST请求。请求URI为`https://www.baidu.com/s`，请求头中包含表单提交的相关信息，如Content-Type、Content-Length等。请求体则是请求参数，查询字符串或者表单中的输入内容。服务器接收到POST请求后，验证请求参数并处理请求，然后生成HTML页面作为响应返回给客户端，响应头中包含返回数据的类型、大小、编码方式等信息。响应体则是返回的HTML页面内容。
#### PUT
PUT方法用来新建或者更新资源。
例如，把一段文字上传到网盘里，首先选择某个目录，然后点选"上传"按钮，浏览器首先向服务器发送一个HTTP PUT请求。请求URI为目标文件的URL，请求头中包含请求的数据长度、MIME类型等信息，请求体中则是待上传的文件内容。服务器接收到请求后，将文件保存到指定的位置。如果目标文件不存在，则创建新的文件；如果存在，则覆盖原文件。响应头中包含请求成功的信息，响应体为空。
#### DELETE
DELETE方法用来删除资源。
例如，在网盘中找到某些不想要的文件，点击右键选择"删除"，浏览器首先向服务器发送一个HTTP DELETE请求。请求URI为目标文件的URL，请求头中不包含额外的请求信息，请求体为空。服务器接收到请求后，删除目标文件。响应头中包含请求成功的信息，响应体为空。
#### HEAD
HEAD方法类似于GET方法，只不过客户端只接收响应头信息而不接收响应体。
例如，输入地址`https://www.baidu.com`，查看网站的标题、描述、关键词等元信息，浏览器首先向服务器发送一个HTTP HEAD请求。请求URI同GET请求一样，请求头和请求体为空。服务器收到请求后，从文件系统或者数据库读取资源的相关信息并将其返回给客户端，但不返回资源本身，仅返回响应头信息。比如，如果请求成功，响应头中可能包含Content-Type、Content-Length、Last-Modified、ETag等信息。
### URI、URL、URN
HTTP协议使用统一资源标识符(Uniform Resource Identifier, URI)来唯一地标识互联网上的资源，还可以使用统一资源定位符(Uniform Resource Locator, URL)表示资源的位置。
统一资源名(Universal Resource Name, URN)，是在URI的基础上，增加了一个命名空间标识符。
URI的示例如下：
- `http://example.com:8080/path/to/file`：表示HTTP协议的资源
- `mailto:<EMAIL>`：表示邮件地址
- `ldap://[2001:db8::7]/c=GB?objectClass?one`：表示LDAP协议的资源
- `tel:+1-201-555-0123`：表示电话号码
- `urn:isbn:978-0-395-06235-6`：表示书籍ISBN号
URL的示例如下：
- `http://example.com/path/to/file`：表示HTTP协议的URL
- `ftp://ftp.example.com/path/to/file.txt`：表示FTP协议的URL
- `mailto:<EMAIL>`：表示邮件地址URL
URN的示例如下：
- `urn:uuid:f81d4fae-7dec-11d0-a765-00a0c91e6bf6`：表示UUID值
