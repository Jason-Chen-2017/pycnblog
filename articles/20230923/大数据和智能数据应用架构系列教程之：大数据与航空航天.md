# 2.大数据概述
## 2.1 什么是大数据？
大数据是指按照一定规模、大小、复杂度存储的数据集合。它通常被定义为超过现有的数据仓库所能容纳的数据量。其特点主要包括以下几个方面：

1. 结构化与非结构化数据：大数据涉及到各种各样的形式和类型的数据，如结构化数据（如数据库中的表格）和非结构化数据（如文档、音频、视频、图像等）。
2. 海量数据：大数据通常是指海量数据，即数量极其庞大的结构化或非结构化数据。随着各种数据源的不断增长、收集速度的提升和数据处理技术的进步，每天都会产生海量的数据。
3. 异构数据：大数据还包含多种形式、种类丰富的数据，这些数据既有相同的数据模式又有不同的存储方式、数据质量、数据使用价值、数据处理手段等。
4. 时变、多样和动态：大数据中的数据呈现出时变性、多样性和动态性。由于数据呈现出时变性、多样性和动态性，使得传统的数据仓库模型难以应付此类数据。同时，大数据还需要实时地更新数据，保证数据的完整性和准确性。
5. 多样的应用需求：大数据所适用的应用范围十分广泛，如金融、电信、政府、媒体、医疗、制造等。大数据还涵盖了各行各业，包括科研、制造、电子政务、物流、广告、零售、健康保健、公共安全、制造业自动化控制、电力系统管理、航天航空领域。

## 2.2 大数据与航空航天
大数据在航空航天领域的应用主要有以下几个方面：

1. 数据采集与存储：航空航天领域数据量巨大，为了分析这些数据，必须将它们采集、存储、整理、分析和挖掘出来。这涉及到数据采集、存储、清洗、数据转换等方面技术。
2. 数据分析：航空航天领域的数据具有高度的多样性和复杂性，因此要通过对大数据进行分析发现有价值的信息，提取有用信息。目前主流的方法有机器学习、统计学习、数据挖掘等。
3. 数据挖掘工具开发：航空航天领域存在大量复杂的多维数据，如何能够直观、快速地发现有价值的数据特征，并对其进行分类、聚类、关联等分析，以及通过预测结果对航空飞机性能、作战效率进行评估，都是提升效率、改善效益的关键所在。
4. 业务流程自动化：在大数据领域，对业务流程进行自动化处理，可以有效提升效率和优化流程，降低人工重复性工作量。例如，根据航班起始时间、目的地等信息，推荐合适的交通工具、停靠站等；根据用户的浏览习惯和历史记录，推荐相关产品或者内容。
5. 预测模型训练：在航空航天领域，除了数据挖掘之外，还有基于历史数据的预测模型的训练。预测模型用于对将来的事情做出有利于投资者利益的预测，是获得竞争优势的关键因素。

## 2.3 大数据应用在航空航天领域的挑战
在航空航天领域采用大数据技术面临的最大挑战就是数据的数量和质量。目前航空航天数据尚处于爆炸阶段，其中各种相关数据如图像、卫星数据、遥感图像、无线信号等，已经涌现出海量的数据。数据采集、清洗、存储、分析、挖掘等环节都面临着巨大的挑战。

1. 数据量大：航空航天领域的数据量比传统互联网企业大几倍以上，且数据随时间变化很快，每天新增的数据量在亿级以上。
2. 数据采集难度大：航空航天领域的数据源繁多，包括卫星、无线信号、GPS等各种类型的数据，每种数据格式、数据量、数据质量都不尽相同。
3. 数据噪声大：航空航天领域存在大量噪声，如干扰信号、雷击、风暴、核辐射等。大数据技术需要对噪声进行有效识别和过滤。
4. 数据隐私保护：航空航天数据往往有高度的敏感性，比如个人隐私、军事隐私等。如何保护个人隐私、军事隐私、工业数据等个人敏感数据，是数据分析和使用过程中一个重大课题。
5. 数据加工分析困难：航空航天领域存在大量数据，经过长期积累，必然形成一定的规律性和模式，例如，同一种类型的飞机，其飞行轨迹与运行参数之间的关系是固定的，这种规律性数据的分析和挖掘就变得异常复杂。

## 3.3 数据清洗
数据清洗，又称数据清理，是对数据集进行初步处理，消除数据集中可能出现的错误、缺陷和不一致性。数据清洗可以实现以下功能：

1. 删除重复数据：删除整个数据集中完全相同的数据。
2. 删除缺失数据：删除整个数据集中缺失较多的值，以免影响分析结果。
3. 去除异常数据：对数据集进行偏离分布的检测，如果发现异常值，则删除。
4. 数据标准化：将原始数据进行统一化处理，以便进行后续分析。
5. 数据压缩：将数据进行压缩，以节省存储空间，提高查询效率。
6. 数据规范化：使数据字段满足某种命名规则，方便数据之间的链接、检索和关联。

## 3.4 大数据平台架构
大数据平台架构是指大数据平台的部署、配置、管理、监控和运营。其主要功能包括数据采集、存储、计算、分析、应用、安全、监控和运营等。如下图所示：


大数据平台架构中包含的组件主要有：

1. 数据源采集器：负责收集不同数据源的数据，包括卫星、无线信号、图像、地理位置、实时数据等。
2. 数据源缓存层：在数据源采集器输出的原始数据之间加入缓存层，避免源头的数据源过载。
3. 数据清洗层：对数据进行初步处理，消除数据集中可能出现的错误、缺陷和不一致性。
4. 存储层：将数据按照大数据平台的存储架构进行存储。存储介质包括磁盘、SSD、Hadoop Distributed File System (HDFS)。
5. 分布式计算引擎：对数据进行计算，比如机器学习、数据挖掘、图计算等。
6. 数据分析层：对存储层的数据进行分析，完成各种数据任务，如数据挖掘、预测、搜索、画像等。
7. 数据展示层：将数据进行展示和呈现，让用户可以使用多种方式查看数据。
8. 应用接口：根据不同的业务需求，提供API接口，允许第三方应用调用数据分析平台提供的能力。
9. 用户界面：提供友好的用户界面，方便用户管理大数据平台。
10. 日志审计：对用户操作记录进行审计和安全审核。
11. 运维中心：定期检查平台状态，进行数据备份、迁移、故障恢复等操作。
