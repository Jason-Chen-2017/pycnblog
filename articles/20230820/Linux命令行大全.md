# 2. 基本概念术语说明

## 2.1 操作系统

操作系统（Operating System，OS），又称作内核或宏观调控器，是一个运行在计算机上的控制程序，它负责分配处理机时间、管理存储器、控制输入输出设备等硬件资源，以及向上层软件提供抽象化的操作接口。目前，操作系统广泛应用于桌面端、服务器端、移动终端等领域。

Linux操作系统是一种开源的、免费的操作系统，由林纳斯·托瓦兹（<NAME>）开发，是一个基于POSIX和UNIX的多用户、多任务的多线程操作系统，由全球许多大学和IT企业开发，并以GPL授权条款发布。它是一个多用户、多任务、支持多种文件系统的操作系统。它的命令行接口（CLI）具有友好的可视化特性，可以直接在屏幕上操作，具有稳定、可靠、安全、快速的特点。

## 2.2 shell

shell（或命令提示符），是指用于访问操作系统的接口，用户通过键入命令的方式，就可以与操作系统进行交互。不同的shell类型代表着不同的用户接口样式，例如bash/sh，csh/tcsh等。

Linux系统自带的默认shell一般是bash，但也有部分Linux发行版修改过默认的shell类型，比如CentOS 7改用zsh作为默认shell。不同的shell类型在命令的执行、参数处理、变量设置、命令别名等方面的表现形式不同，对命令的调用方式也有影响。所以，熟练掌握常用shell的使用技巧，对于提升工作效率和掌握Linux命令行操作非常有帮助。

## 2.3 命令

命令（Command）是指对操作系统进行操作的指令，是最基本、最常用的操作单元。一个命令通常包含一个动词和多个名词或代词组成，表示对操作系统做什么操作。例如，命令“ls”用于显示当前目录的文件列表，“cd”用于切换到指定目录，“cp”用于复制文件或文件夹，“mv”用于移动文件或文件夹，“rm”用于删除文件或文件夹，“mkdir”用于创建新的目录等。命令是直接面向操作系统运行的。

## 2.4 终端

终端（Terminal）是指用户与操作系统之间的接口。终端可以是字符界面，也可以是图形界面的。用户可以通过终端输入命令、查看系统输出信息，也可以通过终端执行各种操作。

## 2.5 文件系统

文件系统（File System）是指操作系统管理数据的方式，它定义了如何组织、存储和管理文件。常见的Unix/Linux文件系统有ext2、ext3、ext4、NTFS、FAT32等。

文件系统可以分为两大类，一类是以树型结构组织的文件系统，如Ext2、Ext3、Ext4文件系统；另一类是以类似磁盘阵列的组织方式的文件系统，如NTFS、FAT32等。

文件系统结构决定了文件的存取速度，另外，不同的文件系统结构还有助于管理数据的冗余备份和容灾恢复。

## 2.6 路径

路径（Path）是指在一个文件系统里定位某个文件或目录的字符串，它是一种特殊的字符串序列，通过这个路径可以唯一确定一个文件或者目录。绝对路径（Absolute Path）是指以根目录（root directory）作为起始点，按照各个子目录或文件名连接起来得到的路径；相对路径（Relative Path）是指相对于当前位置（working directory）的路径。

## 2.7 权限

权限（Permission）是指对文件或目录的操作权限。文件和目录的权限包括读、写、执行三个属性，分别对应数字4、2、1，在文件属性的第一个字符中，第1位表示文件类型，第2-4位表示所有者的权限，第5-7位表示群组的权限，第8-10位表示其它用户的权限。
