# 1.简介
  

GitHub是一个面向开源及私有软件项目的托管平台，它提供了一个庞大的全球社区，并可以方便地分享你的工作成果。目前，GitHub已经成为开源世界中最流行的代码共享工具，拥有超过7亿个仓库、近50万名开发者，被越来越多的人使用。2021年，GitHub的全球用户突破了2.5亿。因此，无论您是否刚接触GitHub，都可以从这里了解到更多有关该平台的信息。
# 2.基本概念术语
在进行任何具体的操作之前，首先需要对GitHub的一些基础概念和术语有所了解。

1. 仓库（repository）：GitHub上的每个项目都是一个仓库，即代码仓库。仓库用于存放代码文件，还可以用来管理软件项目中的所有文件的版本历史记录。每个仓库都有一个唯一且完整的标识符，如https://github.com/username/projectname。

2. 克隆（clone）：克隆是指将远程仓库复制到本地电脑上，用户可以通过克隆仓库获得仓库中所有的提交记录、分支信息、issues等。

3. 分支（branch）：分支是Git的一个重要特性，它允许多人同时在同一个仓库工作，互不干扰。每当一个新的功能或修复需要独立测试时，就可以创建新分支，在这个分支上开发和测试，完成后再合并到主分支。

4. 提交（commit）：提交是Git最基本的单位，每次提交都会保存一次代码库的完整快照，使得代码回滚和版本比较变得容易。

5. 撤销（revert）：撤销是指在提交历史中删除某次提交，会导致之后的所有提交历史发生偏差。因此，建议在必要的时候只恢复或重新提交最近的一组提交。

6. 拉取请求（pull request）：拉取请求是一个由贡献者提出的形式的申请，希望得到审核人员的意见，以决定是否可以将其合并到主分支。

7. Git命令集：作为一款分布式版本控制系统，GitHub提供了丰富的Git命令集，帮助用户执行各种操作。其中常用的有add、commit、push、pull、checkout、merge等命令。

8. Issues：Issues是GitHub提供的一种跟踪任务和缺陷的方法，用户可以在仓库中创建Issues来报告错误、需求、改进和讨论。

9. Pull Request：Pull Request类似于提交的形式，用户可以在仓库中创建一个Pull Request，请求其他用户对自己提交的修改进行审查和评审。

# 3.核心算法原理和具体操作步骤
## 3.1.注册和登录
注册和登录GitHub账号是访问GitHub资源的第一步。在注册前，请确保通过认证的邮箱地址进行注册。


注册成功后，点击头像选择菜单中的Settings，进入设置页面。


设置页提供用户名更改、Email更换、SSH Key绑定、Two Factor Authentication安全验证等功能。


其中SSH Key绑定可以让你通过SSH协议连接GitHub服务器，可以使用多个SSH密钥实现代码推送和拉取。如果没有配置SSH密钥，则无法直接推送代码。

## 3.2.创建仓库
在GitHub创建仓库很简单，只需要填写相应信息即可。


创建成功后，会看到新建仓库的欢迎页面。


默认情况下，仓库仅限个人用户使用，如果需要给其他人分享您的代码或者进行协作开发，需要将仓库设置为公开（Public）。

## 3.3.创建分支
创建分支是为了让不同的开发者在同一个仓库里开发不同功能的补丁或功能。默认情况下，仓库只有一个主分支，用户可以随时创建新的分支并在其中进行开发。

在仓库的主页，选择“Branch: master”右侧的下拉框，然后输入分支名称，点击Create branch按钮即可创建新的分支。


创建完成后，将会切换至新创建的分支。

## 3.4.克隆仓库
克隆仓库是指将远程仓库复制到本地电脑上。可以通过克隆仓库获得仓库中所有的提交记录、分支信息、issues等。

在任意页面，点击“Clone or download”，然后点击复制按钮复制仓库的SSH地址。


打开终端，通过以下命令克隆仓库。

```bash
git clone <EMAIL>:username/projectname.git
```

## 3.5.上传文件
上传文件到仓库也是Git的一个重要操作，它可以帮助我们将本地的代码同步到远程仓库，并在不同的设备之间保持同步。

### 方式1：本地上传文件
首先，确认当前处在想要上传的文件夹。然后，使用如下命令添加文件到暂存区。

```bash
git add filename.txt
```

然后，提交文件到暂存区。

```bash
git commit -m "message"
```

最后，上传文件到远程仓库。

```bash
git push origin main
```

### 方式2：网页上传文件
除了可以通过终端上传文件外，也可以通过网页上传文件。点击仓库的“Upload files”按钮上传文件。


上传完成后，点击“Commit changes”提交上传。


## 3.6.管理分支
在Git中，分支是Git的一个重要特征，它允许多人同时在同一个仓库工作，互不干扰。每当一个新的功能或修复需要独立测试时，就可以创建新分支，在这个分支上开发和测试，完成后再合并到主分支。

查看分支列表可以通过如下命令。

```bash
git branch
```

创建分支命令如下。

```bash
git branch new_branch
```

切换分支命令如下。

```bash
git checkout new_branch
```

删除分支命令如下。

```bash
git branch -d new_branch
```

合并分支命令如下。

```bash
git merge other_branch
```

## 3.7.撤销提交
撤销提交是指在提交历史中删除某次提交，会导致之后的所有提交历史发生偏差。因此，建议在必要的时候只恢复或重新提交最近的一组提交。

### 方法1：直接删除提交记录
对于本机上已经push到远程仓库的提交记录，可以直接使用如下命令撤销提交。

```bash
git revert HEAD^
```

HEAD表示最新提交，HEAD^表示上一次提交。

### 方法2：新建分支重置提交历史
对于本机上未push到远程仓库的提交记录，可以通过新建分支来重置提交历史。

首先，切换至master分支。

```bash
git checkout master
```

然后，创建新的分支。

```bash
git branch temp
```

然后，切换至temp分支。

```bash
git checkout temp
```

此时HEAD指向temp分支。

```bash
git log --oneline --decorate
```

确认log正确。

```bash
git reset --hard xxxxxxxx # xxxxxxxx 为你要回退的提交号
```

注意，reset命令会覆盖掉指定分支的整个提交历史，慎用！

然后，切换回master分支。

```bash
git checkout master
```

最后，将temp分支合并到master分支。

```bash
git merge temp
```

这样，提交历史就恢复了。
