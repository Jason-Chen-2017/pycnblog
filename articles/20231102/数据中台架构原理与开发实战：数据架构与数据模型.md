# 1.背景介绍


## 数据是现代企业的信息资产，对数据的价值远远超过了其物理存储。数据的生命周期长，从产生到消亡，数据往往需要跨越多个业务线、多个维度、多个角色，才能被应用到实际生产环境中。如何通过数据中台架构来实现数据架构的优化？如何用最低成本实现“业务+计算”的全栈式数据分析？数据中台是解决这些问题的关键之处。
## “数据价值”还会扩展到“钱”，数据越多，价值就越高。如何把更多的数据价值付诸市场，推动财富效率和人类文明进步？传统的金融数据服务企业正在变得越来越碎片化，新型数据服务企业则正逐渐兴起。如何打造一个能够提供端到端数据服务的“数据中台”？
# 2.核心概念与联系
## 一、数据集市
数据集市，指的是存放、整合、共享、分析、交易、流通的一系列数据的集合体，一般包括各种类型的数据，如原始数据、半结构化数据、结构化数据、非结构化数据等，数据集市可以由各种数据源提供，也可以自行搭建或者购买第三方数据集市服务。数据集市中的数据包括原始数据、结构化数据、统计数据、网络数据、文本数据等。数据集市中的数据一般来源于各种类型的数据源，比如原始数据包括各个企业的数据、运营商的数据、第三方数据等；结构化数据是指将原始数据按照一定的标准进行清洗、分类、关联等处理得到的高质量、可查询的统一数据表；统计数据是指根据一定的数据规则，对原始数据进行统计、汇总、分析得到的一系列数字化信息；网络数据指的是通过网络传输的数据；文本数据指的是存储在文本文件或其他文本形式的数据。
## 二、数据湖
数据湖（Data Lake）是指储存在异地、本地、分散在不同服务器上的海量数据仓库，是一种面向主题的分布式存储技术架构。数据湖主要用来支持对大数据分析、数据挖掘、机器学习和数据仓库的需求，使得分析师可以快速获取、分析和处理数据。数据湖的构建和维护非常复杂，涉及大量的人力、物力、财力和技术资源，它需要构建专门的基础设施、工具和平台。
## 三、数据智能
数据智能（Data Intelligence）是基于云计算技术、人工智能、大数据分析等相关领域知识的科技产品、技术创新，旨在以更智能的方式对数据进行理解、加工、利用和管理。数据智能带来的改变不仅局限于数据的获取、处理、存储、搜索等，更主要是在数据处理、加工、分析等环节。数据智能技术要做到智能、简单、高效、便捷、实时，确保数据的准确性、完整性、可用性、及时性、安全性和隐私性。
## 四、数据中台
数据中台（Data Center）是一个综合性的技术体系，包括数据采集、存储、加工、处理、分析、呈现等一系列服务，基于数据中台，企业可以汲取互联网、移动互联网、大数据、云计算等新技术的优点，以提升公司的竞争力和能力，并降低企业的运营成本。数据中台的设计目标就是为了解决信息孤岛问题，构建一个能够提供端到端数据服务的平台，实现“业务+计算”的全栈式数据分析。
数据中台架构可以分为如下几个层次：
- 第一层：数据接入层。负责数据采集，主要通过各种方式获取数据，如关系数据库、NoSQL数据库、搜索引擎、日志文件、消息队列等。
- 第二层：数据存储层。主要用于数据的持久化存储，包括硬盘、SSD、内存等介质。
- 第三层：数据计算层。主要用于数据处理，包括数据传输、计算、分析等功能。
- 第四层：数据服务层。主要提供数据查询、报告、分析等能力，并提供数据接口给下游数据服务。
- 第五层：数据展示层。主要用于数据呈现，包括数据可视化、图形展示等能力。
- 第六层：数据工具层。主要用于数据工具、工具链、沙盒、调研等功能。
