# 1.背景介绍


数据资产管理（Data Asset Management）是一个重要的企业数字化进程，其中最重要的就是数据治理及价值分配过程。数据资产管理流程的目标是确保公司的各类数据能够准确无误地反映公司现有业务、客户需求、竞争对手、或者是市场上的真实情况。另外，在这个过程中还需要保证数据资产的可用性、可搜索性、可移植性、安全性和合规性等。因此，数据资产管理的目标主要分为两个方面：

1. 有效且科学地发现数据中的价值，提升企业数据价值的应用效益；
2. 提升数据资产的管理质量、效率和能力，并形成运营上合理、连贯、清晰的价值观念。

由于企业对数据的理解、价值定位和治理方法之间的差异，使得不同部门甚至不同的团队，对于数据资产管理往往会有一些不同的认识。比如说有的部门可能认为“数据驱动”，认为数据是生命线，数据资源可以支撑公司的成长和发展。而另一些部门则比较喜欢从“数据出发”的角度，认为数据只是工具、手段或手腕，数据资产的价值仍然在业务上下游环节、第三方数据平台等其他环节中体现。

基于以上考虑，相信很多企业都会选择采用一种模式——数据中台（Data Center of Truth）。数据中台，即一套完善的数据集成、加工、分析、应用的技术体系。数据中台模式由三个层级组成，分别是数据采集层、数据治理层和数据服务层。按照模式的不同阶段，数据中台也有着不同的发展方向。但是，无论是哪种阶段，数据中台都是解决企业如何有效发现、整合、管理、利用数据的问题，其核心是“连接、融合、治理”。

数据中台的核心作用就是能够将各个业务系统、应用系统、第三方数据源、企业数据仓库等数据源头汇聚到一起，并提供统一数据服务。数据中台可以覆盖数据采集、加工、存储、应用、加速、访问、控制等环节。数据中台的功能主要包括以下几点：

1. 数据连接：数据中台从多个来源获取数据，包括数据库、文件系统、API接口、消息队列等。它可以把这些数据连接起来，为业务提供整合的数据源。
2. 数据采集：数据中台可以根据不同的数据源类型进行数据采集，如数据库、文件系统、API接口。数据采集完成后，数据中台可以通过抽取、转换、加载的方式将数据导入到统一的数据源中。
3. 数据治理：数据中台通过元数据等多维数据描述符来定义数据实体，将不同的数据源按照约定的规则进行连接、规范化、匹配和关联。同时，数据中台也提供数据标准化、知识图谱构建、规则引擎配置等功能，确保数据准确、完整、有效。
4. 数据处理：数据中台通过流水线、脚本语言、机器学习等技术来处理数据，实现数据分析、挖掘、可视化、推荐等任务。通过这些分析，数据中台将数据结果输出给下游的应用系统、数据仓库等。
5. 数据共享：数据中台除了提供数据处理的能力之外，还可以通过数据服务层向外部提供数据服务。数据服务层提供数据查询、分析、计算、存储、传输、展示等功能，为公司提供更加丰富的价值洞察、决策支持、业务支撑等能力。

数据中台能够帮助企业实现以下四大目标：

1. 降低数据采集成本：数据中台能够简化数据采集难度，提高数据的收集效率，使企业减少重复性工作。
2. 提升数据价值应用效益：数据中台能够对数据进行多维度、多层次、多场景的价值分析，并通过数据服务层向外提供数据价值。
3. 提升数据资产的管理质量：数据中台能够通过数据治理层对数据进行元数据建设、数据标准化、数据质量评估、数据血缘追踪、知识图谱构建等方式，确保数据资产的全生命周期管理。
4. 提升公司整体的协同合作能力：数据中台能够提供数据共享和协同能力，实现不同业务系统之间、不同数据源之间、不同组织间数据的互通互用。通过数据中台的协同合作，企业可以获得更多的商业机会和收益。

总结一下，数据中台作为数据治理的重要技术体系，其核心功能是在多源、多种类、复杂性的数据环境下，构建统一、集成、高效的数据服务能力。其核心价值在于能够实现“连接、融合、治理”的功能，提供一个统一的数据中转站，让企业能够有效发现、整合、管理、利用数据。
