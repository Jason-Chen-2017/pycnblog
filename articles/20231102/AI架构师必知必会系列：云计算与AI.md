# 2.核心概念与联系
## 2.1 云计算基础知识
### 2.1.1 云计算简介
云计算（Cloud Computing）是一种基于网络的服务体系，利用计算机技术、自动化工具、存储空间、应用软件资源和网络服务的集合，按需提供计算和存储能力。它既可以覆盖企业内部运行的应用程序，也可以为外部客户提供高可用性的计算资源。它能够提供各种计算环境，包括网络虚拟机、云服务器、容器集群、弹性计算资源、GPU资源等。

云计算是一种通过网络连接的计算平台，提供高度可用的计算服务。它可让用户利用自己的电脑、笔记本或者手机，通过浏览器、APP或其他客户端界面使用公共云资源，实现云端存储、数据库、运算、网络等功能。云计算提供商通常以服务形式向消费者开放，并按照消费者付费的方式收取服务费用。

### 2.1.2 云计算的主要技术指标
云计算领域，最核心的两个技术指标分别是：

- 基础设施即服务（IaaS）：IaaS是云计算的一个重要支柱，为云厂商提供公有云计算基础设施，包括网络虚拟化、服务器租赁、存储设备、负载均衡等服务，帮助用户轻松构建、部署和扩展复杂的网络应用。IaaS主要包括Amazon Web Services（AWS），微软Azure，百度BCE，Google Cloud Platform（GCP）。
- 平台即服务（PaaS）：PaaS是云计算另一个重要支柱，为云应用开发人员提供服务，包括Web开发平台、消息队列服务、数据库服务、搜索服务、缓存服务、日志分析服务等。PaaS主要包括Heroku、Cloud Foundry、OpenShift等。

### 2.1.3 IaaS概述
IaaS（Infrastructure as a Service）即“基础设施即服务”，是一种基于云平台的网络服务，提供网络虚拟化、服务器租赁、存储设备等资源的一种服务形式。该服务允许用户按需购买网络，即可以拥有几台服务器的计算容量，也可以使用自己的服务器，甚至还可以自己搭建服务器集群。IaaS提供了物理硬件层面的虚拟化，用户可以通过标准接口操作虚拟网络，如创建、删除虚拟机，扩容或缩容网络，配置网络拓扑，安装、更新软件，磁盘阵列配置等。

### 2.1.4 PaaS概述
PaaS（Platform as a Service）即“平台即服务”，提供云端软件服务，包括Web开发平台、消息队列服务、数据库服务、搜索服务、缓存服务、日志分析服务等。PaaS平台通过标准接口提供服务，用户只需要上传源代码，就可以部署运行服务。PaaS服务对开发者来说，屏蔽了底层资源管理、网络配置、服务器操作、软件发布等细节，开发者可以专注于业务逻辑的实现。

### 2.1.5 IaaS的主要特点
- 按需计费：云计算中的IaaS服务都是按需付费的，用户只需支付实际使用到的资源费用即可。这意味着用户不用为长期空闲的资源付费，也不会影响其他用户的服务。例如，使用AWS EC2的用户只需为使用的每小时或每天的CPU、内存、存储空间付费。
- 高可用性：IaaS服务具备高度可靠性和可用性，用户可以从多个数据中心的服务器上获取计算服务，保证服务的连续性和稳定性。
- 灵活性：用户可以根据自己的业务情况调整服务规模，扩大或缩小服务器集群数量。IaaS平台还可提供备份、灾难恢复、软件升级等服务。
- 技术封闭：IaaS平台隐藏了服务器的技术细节，用户无需关注底层硬件、网络和软件，只需简单调用API接口即可使用服务。
- 可扩展性：IaaS平台的可扩展性使得用户可以根据自身业务和资源的变化随时增加或减少服务器集群规模。IaaS平台还支持负载均衡、弹性伸缩、弹性云服务器等技术，用户可以根据自己的需求选择合适的技术实现。

### 2.1.6 PaaS的主要特点
- 快速部署：PaaS平台通过标准接口提供服务，用户只需上传源代码，就可以部署运行服务，大幅度简化了云端软件的部署流程。
- 自动扩容缩容：PaaS平台具有自动扩容缩容的能力，用户只需修改配置文件或调用API接口即可实现动态的资源分配和释放。这样，平台就不需要用户手动重复的操作。
- 降低成本：PaaS平台将底层硬件和网络的管理工作委托给第三方，降低了IT部门的管理成本。
- 服务复用：PaaS平台提供的服务都是通用的，用户可以使用同样的服务来开发不同的应用，降低了开发人员的编程负担。
- 快速迭代：PaaS平台提供了自动化的发布机制，用户只需更新代码后，平台会自动完成软件发布、构建、测试和部署过程，提升了软件开发效率。
