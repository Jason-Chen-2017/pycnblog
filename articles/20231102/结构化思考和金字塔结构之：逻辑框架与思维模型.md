# 1.背景介绍


通常情况下，人们做决策时，都需要从各种事实、信息、证据中分析出正确的方向，根据决策需要制定策略或方案，最后再去执行。然而在复杂的决策场景下，人们往往很难准确理解并做出决策，原因可能有以下几点：

1. 不懂得分辨事物之间的联系，容易被局部因素牵着鼻子走
2. 没有对现实世界进行全面的认识和把握，因而无法形成全局性的思路
3. 缺乏对社会、经济、政治等多方面因素的考虑，不够理性
4. 对细节的掌控力弱

为了解决以上种种问题，科学家、哲学家、工程师们设计了一套思维模式——结构化思考法。这个方法基于现实世界，将问题分解为多个层次、模块，分别进行分析，以便更好地认识问题。因此，结构化思考可以帮助人们摆脱盲点，建立更好的决策能力。

结构化思考的方法有助于人们整体上理顺思路，突破自己的分析陷阱，因此可以改善自身决策、管理、创新等能力。它的理论基础主要来源于三条经典公理：第一条是可观察性（observability），即能够直接观测到某一事物的变化；第二条是可分解性（decompositionality），即一个事物可以分解成其他一些较小的组成部分；第三条是链接性（linkage），即不同事物之间存在着密切的联系。结构化思考还能让人们从更高的角度看待问题，观察复杂的现象，发现其中的规律和模式。

实际应用中，结构化思考方法的实现一般分为三个步骤：

1. 分解问题
2. 识别主题
3. 运用公式推导

其中，分解问题的目的是找出问题本质，识别主题的目的是找出最重要的问题之所在，而运用公式推导则是根据已有的知识和已知的规律，得出决策或方案。如此一来，结构化思考法提升了人们对复杂决策的理解、把握和决策能力。

对于计算机系统设计、管理等领域来说，结构化思考法也提供了一种有效的方法，帮助人们理清逻辑关系、梳理程序流程，找出关键环节，最终达到优化系统性能或降低成本的目的。

传统的系统开发过程往往没有充分利用结构化思考法，原因有很多：

1. 缺少有效的理论支撑，缺乏总体的系统框架
2. 缺少统一的工具和方法
3. 缺乏沟通的氛围，各个团队之间缺乏有效的沟通机制

如果要打造一套有效的结构化思考方法体系，需要解决以下四个问题：

1. 概念：如何定义“逻辑”？
2. 模型：如何构建抽象的模型？
3. 方法：如何有效地运用模型？
4. 技术：如何应用于工程实践？

结构化思考和金字塔结构是两种相互关联且紧密相关的思维模型，它们共同构成了一个系统化的思维模式，也是很多设计人员必备的思维工具。
