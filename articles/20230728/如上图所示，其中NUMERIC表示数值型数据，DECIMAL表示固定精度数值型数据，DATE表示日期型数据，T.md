# 1.简介
         
1、SQL 数据类型和对应范围

          SQL支持四种基本数据类型：

          1) NUMERIC：精确到小数点后几位，总共可以存储 15 到 38 位有效数字（存储范围受硬件不同而定）。通常用于货币、金融等金额计算；
          2) DECIMAL(p,s):精确到小数点后 s 位，共 p 位精度（存储范围受硬件不同而定），一般用于货币、金融等金额计算；
          3) DATE：年-月-日，包括：YYYY-MM-DD、YYYYMMDD、YYYY/MM/DD、YY/MM/DD等格式；
          4) TIMESTAMP：年-月-日 时:分:秒 的时间戳，包括：YYYY-MM-DD HH:MI:SS、YYYYMMDDHHMISS、YYYY/MM/DD HH:MI:SS、YY/MM/DD HH:MI:SS等格式；
          对于每一种数据类型，都有一个相应的取值范围，超出范围就会产生错误，例如，如果插入到字符串中就无法保存了。
          在使用 MySQL 或 Oracle 时，建议使用 NUMERIC、DECIMAL 和 DATE，因为其它两种类型在不同的数据库系统上可能存在差异性，这可能会导致数据异常和程序运行出错。
         2、数据库表设计

          如果设计的数据存储在关系型数据库中，那么需要给数据库表格字段定义清楚，才能保证数据的安全和准确。下面做一些简单的数据库表格设计建议：
          * 主键字段：设置主键字段可以方便快速定位单个记录，可以设置为自增长的 ID。但不能太长，最好不要超过 50 个字符。
          * 索引字段：可以加速查询速度，提高搜索效率。推荐建立组合索引（多列联合索引）或者唯一索引，避免全表扫描。
          * NOT NULL约束：非空约束能够帮助数据库更好的处理数据，并降低数据的缺失风险。但是对于数值类数据来说，空值可能意味着无穷大或者零，这时应该允许为空。
          * 默认值约束：对于不需要强制输入值的字段，可以指定默认值，这样可以减少用户输入的时间成本。
          * 字段长度：合理设置字段长度可以提高数据库性能。对于 varchar、char 字段，推荐设置字段长度为能够容纳实际数据的平均值，并避免过长或过短。
          * 流程字段：一些特定场景下会用到的字段，可以创建中间过程字段，方便以后的分析查询。
         3、MySQL 数据类型优化

          有些时候 MySQL 会自动优化某些数据类型，比如将 TEXT 数据类型转换成 MEDIUMTEXT、LONGTEXT，也可以将 int、float 转换成 decimal 来节省空间，等等。因此，尽量保持数据类型的精确性和空间利用率，可以提升数据库的整体性能。
         4、业务场景适配

          根据业务场景，确定适合的数据库类型和字段类型。如，对于金融业务，NUMERIC 和 DECIMAL 更适合金额计算，DATETIME 和 TIMESTAMP 可以用于保存时间信息。

         注意：
         1、数据类型一定要根据业务场景具体化，否则容易导致数据隐私泄露、性能下降、数据质量问题。
         2、数据库表设计要保持简单和易读性，并遵循数据规范，保证数据完整性。
