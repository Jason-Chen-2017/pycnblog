
作者：禅与计算机程序设计艺术                    

# 1.简介
         
1970年代末期，软件工程师研制出打孔卡片的机器人，到后来出现了计算机图形用户界面（GUI）的概念。计算机图形用户界面（GUI），它允许用户通过键盘、鼠标或者触摸屏来操作计算机系统中的各种功能，极大的方便了用户。然而，这种界面设计往往视觉上并不能直观地反映出系统的内部运行机制。因此，随着时间的推移，一些研究者提出了另外一种类型的用户界面——基于机器学习的用户界面（ML-UI）。这种界面能够更好地反映出系统的内部运行机制，并且可以通过自动化的方法来生成。
         在过去的几年里，围绕着机器学习的用户界面研究蓬勃发展。由于机器学习技术的广泛应用，越来越多的研究人员关注如何为机器学习模型提供更好的用户界面。其中最著名的就是Google的产品经理丹尼斯·格雷厄姆（Daniel Greene）博士，他曾创立并推动了谷歌的机器学习产品专利申请流程。最近，随着Kubeflow项目的发布，Kubernetes以及相关工具变得越来越流行，越来越多的开发者开始关注这些平台的集成与管理能力。
         2018年，随着Kubernetes的火爆，越来越多的企业、组织以及个人开始着力于使用云原生技术架构来构建容器化的应用程序。云原生是云计算的一个重要发展趋势，也是容器编排技术的集合体。云原生技术如此火热，已经成为IT界的热门话题。而其中的最佳实践方法也正是在社区中蓬勃发展。本文将从云原生技术领域最具代表性的几个活动——公众号、Open Source Summit、线下活动等进行介绍。

         # 2.公众号 
         某位开源社区的主页上吸引了众多读者，然而由于各种原因，这些阅读量并没有带来收益。更糟糕的是，因为内容质量太差，导致很多读者并没有持续阅读，甚至还曾被骚扰、恶意刷屏。为此，主办方不得不削减原计划，缩小举办范围。有些社区主动退订、停止运营。不过，还有一些公众号还在坚守阵地，继续开展各项活动。比如，KubeCon+CloudNative大会联合Linux基金会举办的“Cloud Native Computing Monthly”专栏，发布了《让Kubernetes服务网格更美好》一书，主题为“如何赋能应用构建者”，并邀请作者进行讨论。有一些公众号涉及到技术深入浅出的介绍，例如郭炜的“容器时代下的技术之路”。在这些公众号背后，还有一批技术专家努力传播开源技术、分享云原生技术、倡导敏捷软件开发等信息。这些公众号既是云原生技术的先驱者，又是高级技术人才的摇篮。阅读公众号文章可以帮助读者快速获取云原生技术的最新资讯，同时也能够建立起自己对云原生技术的认知和理解。

         # 3.Open Source Summit 
         在国外，每年都有很多开发者参加开源技术峰会，而在中国却鲜少听说。不过，在2019年，参加了第一届Open Source Summit，这是由华为主办的一场技术盛宴。会议共分为多个主题，其中云原生技术专场吸引了很多人，包括Ant Group CEO黄彦宁、阿里巴巴董事局主席马云、KubeEdge创始团队李响、Elastic官方总监庞博等。该专场从上午9点到晚上6点一直持续到深夜，难怪总有那么多工程师抱着笔记本、手机参加，可能因为他们比较忙碌吧。会议的内容覆盖开源技术、开源许可证、OpenStack、K8s、CNCF、微服务、DevOps、容器技术等多个领域，议程紧凑而精彩，出席嘉宾多元、演讲形式多样，讲座层次深入浅出，每个人都是受益匪浅的。由于时间跨度长，只有短时间内才能看到相关的资料。需要注意的是，由于会议主持人众多，不同公司负责同一个主题的演讲，可能会导致相互之间的矛盾或冲突。为了保证会议的正常召开，建议选择日历上的周末时间进行。

         # 4.线下沙龙 
         KubeCon + CloudNative大会在北京举行，主题为Cloud Native 101：Kubernetes。除了Kebelet、CoreDNS、Containerd、Fluentd、Istio、OpenTracing等关键技术的分享，还邀请了Cloud Foundry的技术专家分享。其他演讲主题包括“API Gateway vs Service Mesh”，“Serverless 的未来趋势”，以及“Kubernetes 多集群管理方案”等。为了扩大交流的影响，KubeCon大会除了定期举办各种技术分享外，还设立了一个线下活动中心，供与会人员线下聚集。有意向的读者也可以利用线下活动认识新朋友。线下活动一般举办一次，但在不同的城市可能会举办多次。对于国内的云原生技术爱好者来说，线下沙龙是一个很好的交流平台。但是，另一方面，需要注意的是，线下沙龙的门票费用较高，并且需要自理交通。因此，建议只作为一次性的技术交流活动，或者临时来回的聚会。

         # 5.未来趋势 
         云原生技术还处于蓬勃发展的阶段，未来的方向也不可预测。Open Source Summit已经走到了尽头，其后的云原生技术大会可能不会像KubeCon一样频繁举办。因此，必须坚持不懈地探索新的技术理念、新思维模式、新模式。无论是在软件开发还是云计算领域，不管是创业公司还是中大型公司，都应该保持对云原生技术的关注，并努力在这条技术轨道上取得进步。
