# 1.简介
         
2017年美国科技报告发布了《2017世界科技指数报告》，其中最重要的一项就是“物联网、云计算及其应用”报告。由于智能手机的普及和应用越来越广泛，物联网成为越来越多人关注的热点话题。而在物联网的发展过程中，“从混乱到秩序”成为一个老生常谈的话题。
         
         “From Chaos to Order”（混乱转化为秩序）作为物联网领域的一个里程碑式的工作，以另一种方法突出了“认知失调”这个关键词。作者在本文中认为，从理论上来说，物理系统的结构还处于无序状态，因而才会出现混乱状态。当这些系统受到外界环境影响时，自然会产生剧烈的行为变化，即混乱状态。反之，当环境条件发生变化后，自然会进入到一种稳定的秩序状态，这种状态下的物理系统将呈现良好的预测性、稳定性和可预测性。
          
         1997年亚马逊网络服务集团(AWS)公司总裁蒂姆·库布里克（Tim Cook）也曾提出过类似的观点：当混乱遇到一定的持续时间后，就会形成一个稳定的秩序。但是，仅仅只有物理系统才能在这个过程中自然地产生秩序。认知失调也是可以应用在整个社会经济系统中，而不只是物理系统中的。例如，企业经营活动中可能会出现严重的认知失调，因为企业管理者往往习惯于有限的资源、限制的时间和信息。因此，如何能够通过优化组织架构、流程和管理人员，以减少认知失调、消除障碍并提升业务效率，才是解决这个问题的关键。
         
         本文主要围绕“From Chaos to Order”这个主题展开讨论，对物联网领域的认知失调问题进行了专业的探讨。我们首先回顾了物理系统的无序状态，以及物理系统演变出来的混乱状态。然后介绍了认知失调的定义，阐述了认知失调对企业组织、流程、管理人员等方面造成的影响。最后，进一步讨论了认知失调在物联网领域的挑战，提出了在这一领域所需具备的基本能力，并分析了该领域的未来发展趋势和机遇。
         
         
         # 2. 基本概念术语说明
         ## 2.1 认知失调
         认知失调（cognitive dissonance），指的是人们在不同情况下，面临同样的状况或事件，却产生不同的判断或行为，称作“表象”。表象导致行动和决策产生困难，甚至可能导致错误的结果。
         
         有关认知失调的定义很模糊。上世纪60年代，加拿大心理学家沃尔特·皮查伊（Walter Pitts）提出了一个“注意失调症”（attentional disorder）的概念。他发现，不同的注意力分配方式会导致表象。比如，一个人的注意力分散在许多不同的事情上，就容易产生“注意失调”，这就是典型的“孤立”或“疏离”模式。同时，如果注意力高度集中在某些特定任务上，就容易产生“兴奋”或“紧张”模式，从而产生“表象”。
          
         20世纪70年代末，哈佛大学教授鲍华德·海默（Brian Hammond）把认知失调理论引入了医学领域。他认为，认知失调是一种多种原因造成的行为和态度上的差异。比如，对于正面事件，人们可能坚持“积极主动”的态度；但对于负面事件，他们可能表现出“恐慌”、“沉默”或“厌恶”的态度。更糟糕的是，由于缺乏真正的共鸣，人们可能产生“分神”、“空想”或“奇妙”的行为。
         
         随着认知科学的发展，对认知失调的研究日益丰富。如今，已有的认知失调模型有很多种类型，包括贫乏、焦虑、抑郁、躁狂等。而随着物联网、人工智能、大数据、云计算等新技术的发展，带来了新的物理系统特性。传统的物理系统可以采用导电或磁性的方式控制，而物联网系统则需要使用频率信号和微波等方式传输数据。因此，物联网系统将会产生新的认知失调问题。
         
         在本文中，我们只要了解一下认知失调在物联网领域的表现形式就可以了。认知失调在物联网领域有两个层次。第一个是“自然失调”，即物理系统自身的失灵；第二个是“社会失调”，即物联网系统的宿主环境给它造成的影响。
         
        ## 2.2 混乱状态
        在物理系统里面，系统的演变过程通常由以下三个阶段组成：

        1. 初始阶段：初态或者稀薄状态
        2. 混乱阶段：微小扰动引起的多种多样的非均匀行为
        3. 慢慢平静阶段：自然环境改变、随机应变后的平静状态
        
        混乱状态下，系统会呈现各种各样的行为。例如，一辆车突然失去控制，一个机器人做出类似昏昏欲睡的表情；但随着时间的推移，混乱逐渐消退，系统进入稳定的状态，稳定状态下，系统呈现合理且预测性的行为。

        ## 2.3 稳定状态
        在物理系统里面，稳定状态其实就是没有混乱状态。稳定状态下的系统，由于系统功能的稳定，预测性和稳定性得到了保证。稳定状态下的系统行为具有确定性和预测性。
        
        ## 2.4 可预测性
        可预测性（predictability）是指系统的行为具有某种预期，系统可以预测某种事件或状况，并能做出相应的行为。物联网系统中，可预测性体现在物联网设备数据的连续性、实时性、准确性、重复性等方面。例如，假设某个空气检测仪每隔几分钟发送一次数据到云端。如果云端接收到的数据出现差错，那么整个系统就无法实现预测性，所以必须考虑其性能、可用性、可靠性等因素。
        
        ## 2.5 延迟性
        延迟性（lateness）是指系统发出的命令或指令，实际上并没有及时执行。物联网系统中，延迟性体现在交互速度缓慢、数据传输距离长等方面。例如，假设一台路由器的设置选项不能实时保存，那么用户修改路由器配置的结果可能不能及时生效，这就造成了延迟性。
