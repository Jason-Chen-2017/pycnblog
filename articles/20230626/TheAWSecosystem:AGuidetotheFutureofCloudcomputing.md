2.1. Basic Concepts Explanation

AWS ecosystem is built upon a set of core principles and concepts. These concepts form the foundation of the AWS ecosystem and enable the development of powerful cloud-based applications.

2.2. Technical Overview

2.3. Comparison of Services
-----------------------

AWS ecosystem includes a wide range of services, each with its unique capabilities, features, and strengths. Here's a comparison of some of the key services in the AWS ecosystem:

### EC2

[ EC2 Documentation](https://docs.aws.amazon.com/AmazonEC2/latest/UserGuide/ec2-installation.html)

EC2 is a cloud-based computing service that provides resizable compute capacity in the cloud. It enables users to launch and manage virtual servers in the cloud with ease. EC2 supports various instance types, which are optimized for different workloads, including web, compute, and storage.

### S3

[ S3 Documentation](https://docs.aws.amazon.com/AmazonS3/latest/UserGuide/s3-getting-started.html)

S3 is a highly durable and scalable cloud-based storage service. It provides users with secure, fast, and cost-effective storage for their data. S3 supports various data types, including objects, buckets, and fixed-format objects.

### RDS

[ RDS Documentation](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/rds-getting-started.html)

RDS is a managed relational database service that makes it easy to set up, operate, and scale a relational database in the cloud. It supports various database engines, including MySQL, PostgreSQL, Oracle, SQL Server, Oracle X-DB, and Oracle for Oracle.

### DynamoDB

[ DynamoDB Documentation](https://docs.aws.amazon.com/AmazonDynamoDB/latest/UserGuide/dynamodb-getting-started.html)

DynamoDB is a fast and fully managed NoSQL database that provides single-digit millisecond latency at any scale. It can be used for a wide range of use cases, including data Warehousing, Shopping Carts, and游戏引擎.

### SQS

[ SQS Documentation](https://docs.aws.amazon.com/AmazonSQS/latest/UserGuide/sqs-getting-started.html)

SQS is a fully managed message queue service that enables decoupling of the components of an application by allowing messages to be sent and received without requiring an application to know who sent or received the message.

### Lambda

[ Lambda Documentation](https://docs.aws.amazon.com/Lambda/latest/dg/)

Lambda is a serverless compute service that runs code in response to events and automatically manages the underlying compute resources for you. It supports various programming languages and can be used for a wide range of use cases, including serverless web applications and event-based applications.
