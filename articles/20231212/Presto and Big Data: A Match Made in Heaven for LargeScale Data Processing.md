# 2.核心概念与联系

Presto是一个开源的分布式SQL查询引擎，它可以处理大规模的结构化和非结构化数据。Presto的核心概念包括：

1.分布式架构：Presto采用了分布式架构，可以在多个节点上并行处理数据，从而实现高性能和低延迟。

2.SQL查询：Presto支持标准的SQL查询语法，使得用户可以使用熟悉的语法来查询大数据集。

3.数据源集成：Presto可以连接多种数据源，包括Hadoop Hive、HBase、MySQL、PostgreSQL等。这使得Presto可以作为一个统一的数据处理平台，用于处理来自不同数据源的数据。

4.高性能：Presto的高性能是由其分布式架构、优化算法和硬件加速等因素共同决定的。

```sql

FROM sales
WHERE date >= '2020-01-01' AND date < '2020-01-02'
```

1. 首先，Presto会对查询进行解析，生成逻辑查询树。
2. 然后，Presto会对逻辑查询树进行转换，以便在分布式环境中执行。
3. 接下来，Presto会生成物理查询计划。
4. 最后，Presto会对物理查询计划进行优化，以便在分布式环境中执行。

在执行查询过程中，Presto会使用数据分区来提高性能。首先，Presto会根据`date`列对数据进行分区。然后，Presto会将分区数据发送到各个节点上进行并行处理。最后，Presto会将结果聚合起来，并返回给用户。
