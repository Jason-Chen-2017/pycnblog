# 2. 核心概念与联系

在进入具体的内容之前，我们需要了解一些关键的概念和联系。

## 2.1 代码复用的类型

代码复用可以分为以下几类：

1. 模块化复用：将代码拆分成多个模块，以便在不同的项目中重用。这种复用方式可以提高代码的可维护性和可读性。

2. 函数式复用：将相似的功能封装成函数或方法，以便在不同的项目中重用。这种复用方式可以提高代码的可重用性和可扩展性。

3. 对象式复用：将相似的功能封装成对象，以便在不同的项目中重用。这种复用方式可以提高代码的可维护性和可扩展性。

## 2.2 代码复用的原则

在进行代码复用时，需要遵循以下原则：

1. 单一职责原则：一个模块、函数或对象只负责一个职责，这可以提高代码的可维护性和可读性。

2. 开放封闭原则：一个模块、函数或对象对扩展是开放的，对修改是封闭的，这可以提高代码的可扩展性和可维护性。

3. 依赖倒转原则：一个模块、函数或对象不应该直接依赖于其他模块、函数或对象，而是依赖于抽象层次，这可以提高代码的可扩展性和可维护性。

# 4. 具体代码实例和详细解释说明

在这里，我们将通过一个具体的代码实例来说明如何进行代码复用。

假设我们有一个计算两个数之和的函数，我们可以将这个函数进行模块化复用，函数式复用，以及对象式复用。

## 4.1 模块化复用的实例

在这个实例中，我们将创建一个名为`math_module`的模块，并在其中定义一个`add`函数，用于计算两个数之和。

```python
# math_module.py
def add(a, b):
    return a + b
```

然后，我们可以在其他模块中导入`math_module`模块，并调用`add`函数。

```python
# another_module.py
from math_module import add

result = add(1, 2)
print(result)  # Output: 3
```

## 4.2 函数式复用的实例

在这个实例中，我们将创建一个名为`add`的函数，用于计算两个数之和。

```python
# add.py
def add(a, b):
    return a + b
```

然后，我们可以在其他文件中导入`add`函数，并调用它。

```python
# another_file.py
from add import add

result = add(1, 2)
print(result)  # Output: 3
```

## 4.3 对象式复用的实例

在这个实例中，我们将创建一个名为`Calculator`的类，并在其中定义一个`add`方法，用于计算两个数之和。

```python
# calculator.py
class Calculator:
    def add(self, a, b):
        return a + b
```

然后，我们可以在其他文件中创建一个`Calculator`对象，并调用`add`方法。

```python
# another_file.py
from calculator import Calculator

calculator = Calculator()
result = calculator.add(1, 2)
print(result)  # Output: 3
```
