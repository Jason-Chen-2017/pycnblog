# 1.简介
  

作为架构师是指能够设计、开发、部署和运维一款或多款系统应用产品的专门人员。一般来说，架构师对整个企业应用架构的设计、开发过程拥有较高的要求，它负责设计应用系统整体的结构、功能及其之间的交互关系等，通过对业务需求进行分析、设计、开发、集成、测试和部署，为公司提供可靠的服务。因此，在求职中，架构师往往需要具有丰富的专业技能，同时，还需要具备扎实的计算机基础知识、项目管理能力、沟通协调能力、执行力和敬业精神。

本文将结合我的实际经验以及从业心得出发，介绍我对“架构师”这个职位角色的理解和判断，希望能帮助到读者对此职位的定位更加准确。

# 2.架构师角色定位
## 2.1 定义
架构师是指具有工程学、经济学和社会学背景，在复杂系统的设计、开发、部署和运营中起到举足轻重作用，通过对复杂系统的把握和把控，提升产品质量、降低成本、优化资源利用率、促进产品创新、改善服务品质的专业技术人员。一般认为，架构师除了掌握复杂系统设计、开发、部署、运营的相关专业技能外，还应具备业务分析、产品规划、营销策划、组织管理、客户服务等领域的专业能力，才能成为一个成功的企业架构师。

## 2.2 职位介绍
### 2.2.1 开发工程师
开发工程师(Dev-Ops Engineer)，又称为运维工程师，可以分为程序员、数据库管理员、网络管理员、安全管理员、硬件维护工程师、系统工程师、测试工程师、项目经理、信息系统顾问等类型。一般来说，这类职位的人员主要负责对应用系统进行编码、调试和维护工作，涉及开发、测试、发布等环节。他们应具备扎实的编程能力、调试技巧、项目管理能力、系统架构设计能力、团队协作精神、持续交付意识等。

开发工程师一般从事的任务包括：

1. 研发阶段：负责应用系统的开发、测试、上线，提升产品质量、降低成本；
2. 运维阶段：负责应用系统的配置管理、日志审计、性能监测和问题排查，并确保系统运行正常；
3. 技术支持：负责对应用系统进行故障诊断、性能分析、并提供技术支持；
4. 架构设计：根据业务和竞争对手需求，制定系统架构设计文档；
5. 撤离、迁移或升级：当系统发生变更时，维护更新系统，减少或避免不必要损失。

### 2.2.2 架构师
架构师(Architect)是现代计算机科学中非常重要的一职，也是解决复杂系统架构问题的关键人才。一般来说，这类职位的人员是软件行业中最重要的人物之一，负责设计、创建并实施企业级软件系统的蓝图。架构师一般从事的任务包括：

1. 需求分析：充分理解用户需求、商业模式、市场情况，根据业务目标制定软件产品的架构设计和功能规格；
2. 系统设计：考虑系统的组件、模块、数据流、接口、通信协议、数据库设计、存储策略、安全性和可用性，将需求转化为系统架构设计；
3. 实现方案：选择正确的框架、工具、库、平台来构建软件系统，完成后验测试和集成；
4. 测试验证：应用测试用例、压力测试和生产环境测试，保证系统的稳定运行；
5. 发布部署：部署软件到生产环境，经过多个环节之后，确保系统的可靠性和可扩展性。

### 2.2.3 项目管理者
项目管理者(Project Manager)是指管理项目生命周期，以确保项目目标的达成，并最大限度地提高项目收益的专业人士。他/她需有很强的责任感、高度的组织和动手能力，擅长管理复杂的项目组织，以及分析项目的战略规划、资源分配、交付时间、风险管理、成本控制等各方面。

项目管理者一般从事的任务包括：

1. 项目规划：制定项目计划、目标、范围、激励机制和预算，明确项目范围和任务；
2. 执行管理：跟踪项目执行进度、问题、风险、影响、KPI等，确保项目按计划执行；
3. 投资决策：评估投资机会、资源配置、战略方向、预算约束等，形成决策意见并向部门主管汇报；
4. 品牌宣传：将项目管理理念、方法论、流程及工具推广至内部，确保项目的持续性和有效性。

# 3.本人理解和判断
作为软件开发行业的职场新宠——架构师，在日常工作中，要想做好架构设计、方案编写、项目实施、测试验证和发布上线等工作，都需要有不断学习和提升自己的能力。下面我将给读者一些参考建议：

## 3.1 基本要求

首先，作为一个软件架构师，应该具备如下基本要求：

1. 极强的逻辑思维能力：架构设计本身就是一门经验学科，涉及到的知识众多，理解这些知识之前，不能仅凭借已有的经验而盲目下结论。
2. 极强的设计能力：不管是细致的技术方案设计还是架构设计，都需要高度的思路能力和设计能力。
3. 深刻的业务理解和洞察力：架构设计不仅仅是技术活儿，更是一种深入了解业务、用户和使用场景的能力，同时对架构设计也要有独到的见解和直觉。
4. 良好的沟通和表达能力：与同事、上级、领导的沟通是架构设计不可替代的技能，如果不能很好地与他们沟通，就无法真正落地。
5. 高度的执行力：架构设计是一个长期工作，需要保持高度的执行力，始终紧绷头脑，并为之努力。

## 3.2 优先技能

其次，作为一个软件架构师，你的基本技能其实并不太难找。但是，如果你能更侧重于以下的专业能力，那么你可以获得更多的收获：

1. 软件工程师或程序员：软件架构师职位上的第一要务当然是软件的实现，所以，除了编程能力外，你还需要扎实的软件工程素养。比如，你需要熟悉各种编程语言、开发模式、版本管理、自动化工具等，以及对软件设计模式、体系结构的理解。
2. 数据库管理员：软件架构师的另一个要件就是数据库的管理，所以，对数据库管理、SQL语句编写、性能调优等方面的理解非常重要。
3. 网络管理员：软件架构师需要对网络系统的管理、配置、维护、性能调优等方面有比较深入的认识。
4. 安全管理员：软件架构师往往被要求负责整个系统的安全保护，需要有比较扎实的安全防范知识。
5. 硬件维护工程师：软件架构师需要对硬件设备（如服务器、网络设备）的维护、管理、安全等有比较深入的认识。
6. 系统工程师：软件架构师往往需要负责整个系统的集成、测试、发布等方面的工作，需要对系统工程有比较深入的理解。
7. 测试工程师：软件架构师的测试工作就是要保证软件系统的质量，测试工程师则是要对测试方法、工具、流程等方面有比较深入的理解。
8. 项目经理：软件架构师往往需要承担整个项目的管理工作，项目经理对项目管理有比较深入的理解。
9. 信息系统顾问：作为一个软件架构师，你需要与IT部门打交道很多，所以，对IT部门的信息系统有比较清晰的认识和理解是很重要的。

## 3.3 边界意义

最后，作为一个软件架构师，你的专业水平越高，边界意义也就越大。例如，你可能负责设计和研发一个微服务系统，那么，它的边界意义甚至比大型的单体架构系统还要大。你可能还需要配合前端、移动端开发工程师一起工作，也就是说，你的职责范围可能会涵盖多种技术和角色。

当然了，还有很多其他非常有价值的职位，比如UI设计师、数据库管理员、开发人员、云计算工程师、测试工程师等等，它们也能够帮助你更好的完成你的工作。
