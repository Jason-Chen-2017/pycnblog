# 2.基本概念术语说明
## 2.1 数据集市及应用场景
数据集市是指通过互联网、云计算等方式收集、整理、分析和处理海量数据，并提供给不同用户使用的一种新型的数据共享平台。在该平台中，用户可以自由下载、分享、上传和消费大量的数字数据，比如图像、视频、音频、文本、结构化数据、原始数据等，这些数据经过各种形式的加工和处理后，能够为企业提供有价值的信息和决策支持。当前，数据集市应用十分广泛，主要涵盖多个行业领域，如电信、金融、医疗、教育、交通、农业等。根据目前的数据集市提供服务的规模大小，分为小数据集市、大数据集市和超大数据集市三个层级。其中，小数据集市主要面向个人或非商业组织，提供的数据规模较小，但具有高价值；大数据集市面向具备一定规模和能力的企业或组织，拥有庞大的客户群和需求，提供的数据规模一般都在百亿以上，但通常需要付费才能使用；而超大数据集市则是针对各种规模、多样性、复杂度都很高的集中数据存储和分析需求的场景，其规模和数据量通常不超过100PB，但也往往收取巨额的预算费用。
## 2.2 机器学习
机器学习（Machine Learning）是一门关于计算机如何利用数据自动提取知识和进行有效预测的科学。它是借助于数据、统计方法、规则系统以及一些算法来制定规则、建立模型、实现预测与决策的计算机科学研究领域。它可以用于对数据进行分类、聚类、回归、降维等分析任务，也可以用于预测、描述和解决实际问题。机器学习技术是由人工智能与统计学相结合的一套用来解决很多自然界和社会问题的技术。其在许多领域都有着广阔的应用前景。
## 2.3 特征工程
特征工程是指从原始数据中提取有用信息，形成适合机器学习算法使用的有效特征或输入的过程。它的目标就是帮助数据科学家快速地开发出能够有效预测的模型，并通过参数调优来优化模型的性能。特征工程技术一般包括清洗、转换、抽取、选择、编码、标准化等几个阶段，包括了数据预处理、特征提取、特征选择、特征转换、特征缩放等几个环节。
## 2.4 大数据相关术语
**大数据定义：**大数据是一个集合，这个集合既有规模大，又非常复杂。由于收集、存储、管理、分析大量数据所需的时间和人力投入太大，因此，一味追求数量会导致数据集过大，难以真正地挖掘潜在的商机和价值。因此，现代数据科学家把大数据看作是过去几年产生的海量数据的综合产物，为了更好地挖掘数据中的价值，提升数据科学家的生产效率，产生了大数据技术。

**大数据采集方法:**

1. 数据采集方式

目前大数据采集方式主要分为三种:
- 文件传输协议(FTP):文件传输协议(File Transfer Protocol)是一种网络传输协议，允许两个计算机之间的文件传输。由于该协议简单、易于实现且易于管理，因此被广泛用于大数据采集。
- RESTful API:RESTful API(Representational State Transfer)是一种新的Web服务接口标准，是Web服务的一种风格，可以轻松的构建跨平台的、可复用的应用。目前，RESTful API已成为大数据采集的主流方式。
- 普通爬虫：普通爬虫是指一种非机器学习的基于网页抓取的大数据采集方式，适用于网站提供的原生API。

2. 数据采集工具

- Apache Nutch：Apache Nutch是一个开源的、基于Java编写的网络爬虫框架。它可以用于爬取整个互联网、某些网站或者网页。Nutch可以用于大数据采集，并且可以通过插件扩展功能，如数据清洗、分析、机器学习等。
- Hadoop MapReduce：Hadoop MapReduce是Apache Hadoop项目的一个子项目。它是一个分布式计算的编程模型和工具，可以让用户编写批量数据处理程序。MapReduce编程模型是一种将数据集切片并映射到不同的节点上的并行运算模式。因此，Hadoop MapReduce可以在集群上运行，提高大数据采集效率。
- Elasticsearch：Elasticsearch是一个开源搜索引擎。它可以实时分析数据，支持复杂的查询、过滤和聚合功能。通过Elasticsearch搭配Kibana界面，可以方便的查看、分析数据。
- Spark：Spark是由Apache基金会开发的一个快速、通用、内存计算的框架。它可以运行迭代算法、交互式查询、机器学习和图计算等大数据应用。Spark的运行速度快、容错性好，适用于海量数据处理。

**大数据分类：**

1. 海量数据(Volumetric Data)

- IoT数据:IoT(Internet of Things)技术使得物联网设备连接在一起，产生大量数据。随着互联网的普及、物联网的普及以及移动端设备的兴起，海量的IoT数据呈现爆炸式增长态势，使得传统数据库处理瓶颈越来越难以满足需求。
- 游戏数据:游戏行业为玩家提供了丰富的游戏数据，包括游戏运营数据、玩家行为数据、游戏角色数据、社交数据等。这些数据能够反映游戏的热度、玩家的喜好、竞技水平、内容热度等因素，有助于游戏策划、开发和运营等方面的决策。
- 手机位置数据:手机位置数据包括手机GPS定位数据、移动互联网数据、社交媒体数据等。这些数据可以帮助运营商了解用户的真实习惯，并通过反馈机制改善网络质量、服务体验。

2. 时变数据(Trending Data)

- 用户行为数据:用户行为数据包括用户浏览习惯、购买习惯、搜索习惯、评论、点赞、分享等。这些数据能够反映出用户的喜好偏好、行为轨迹，从而帮助推荐系统更好的向用户推送商品和广告。
- 物流数据:物流数据包括货运过程中各个环节的数据，如货物信息、车辆信息、路况信息、运输路径等。这些数据能够帮助物流公司实时掌握货源、运输动态、客户满意度等信息，提高效率并降低成本。

3. 结构化数据(Structured Data)

- 交易数据:交易数据包括购买订单、支付记录、退款记录、产品信息、顾客信息等。这些数据可以用于决策支持，如为用户提供更高质量的服务，促进商业活动的顺利开展。
- 身份验证数据:身份验证数据包括用户名、密码、信用卡信息、照片、指纹等。这些数据可以用于金融、证券、政务等金融行业的业务决策，如账户安全、风险评估、客户溢价等。

4. 半结构化数据(Semi Structured Data)

- 社交网络数据:社交网络数据包括微博、微信、知乎、论坛等用户生成的内容。这些数据可以用于监控用户行为、品牌营销等，为商业决策提供参考。
- 电子邮件数据:电子邮件数据包括邮件内容、发送时间、接收时间、附件等。这些数据可以用于监控用户行为、反垃圾邮件、促进客户关系等，为商业决策提供参考。

**大数据应用场景：**

1. 数据分析及挖掘
- 客户关系管理：电子商务、社交网络、搜索引擎、金融、保险等领域均有大量的客户数据，这些数据能够反映客户的购买行为、心理习惯、搜索习惯、浏览习惯、留存率、忠诚度等，为公司提供有效的客户关系维护、促进商业活动的推进和新老客户转化提供依据。
- 产品推荐：产品推荐系统能够帮助用户找到喜欢的产品、内容、服务等。大数据与推荐系统结合的应用非常广泛，如电商网站的推荐系统、微博、知乎的讨论推荐、移动APP的推送。
- 搜索排名：搜索引擎的关键词排名是一个典型的大数据应用场景。搜索引擎通过对海量数据进行索引和检索，能够快速准确地返回相关结果。此外，搜索结果的排序、过滤、推荐等行为也依赖于大数据。
- 广告投放：广告投放也是大数据应用的热点。电视广告、搜索引擎联盟广告等都是大数据应用领域的代表，它们通过对用户数据进行分析、挖掘和挑选，能够精准投放广告。

2. 数据采集及汇总
- 日志数据采集：在互联网环境中，日志数据无处不在。基于大数据分析技术的日志数据采集系统能够帮助公司提升运维效率，改善服务质量，监控业务变化。
- 电话营销数据采集：电话营销是广告的一种，其数据采集也属于大数据的一部分。通过大数据分析技术，电话营销数据能够帮助公司了解客户呼叫轨迹、客户反馈、客户满意度等。

3. 异常检测及异常预警
- 用户行为检测：基于用户行为数据，可以检测到用户在线行为的异常，如刷单、恶意注册等。异常检测系统能够帮助公司识别并阻止异常行为，提升客户服务和交易安全。
- 舆情监控：舆情监控系统能够实时监控国际舆论、经济形势、社会事件等社会动态。基于大数据分析技术的舆情监控系统能够帮助公司发现热点、趋势、威胁等信息，提前做好应对措施。
