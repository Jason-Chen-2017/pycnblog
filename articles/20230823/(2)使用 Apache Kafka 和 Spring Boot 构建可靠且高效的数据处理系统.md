# 2.基本概念术语说明
## 2.1 Apache Kafka简介

Apache Kafka 是由 LinkedIn 开源的分布式流处理平台，是一个快速、可扩展、持久化的消息队列服务。在实际生产环境中，Kafka 可以用于实现异步通信、日志聚集、网站活动跟踪、运营指标收集、实时事件分析等功能。

Apache Kafka 的主要特点包括:

1. 发布-订阅模式

   Kafka 通过 topic 来区分不同类型的消息，每个 topic 下可以有多个 partition，partition 负责存储和分配消息。生产者向其中发送消息后，会被分配到对应的 partition 中，消费者则从各个 partition 上按照指定的 offset 获取消息进行消费。通过这种方式，Kafka 可以实现多播、广播或订阅模式下的消息传递。
   
2. 高吞吐量

   Kafka 在设计上具有高吞吐量的特征。其消费和发送速度都非常快，每秒钟处理上万条消息。同时还支持水平扩展，可以在不影响性能的情况下动态增加集群节点数量。
   
3. 可用性

   Kafka 支持多副本机制，确保消息不丢失，即使当一个节点发生故障的时候仍然能够继续工作。另外，它提供了 Broker API，允许客户端和服务器之间使用不同的编程语言进行交互。

4. 容错性

   Kafka 提供自动数据备份，保证数据的安全性和一致性。同时还支持数据压缩和删除过期数据功能，有效地降低了磁盘空间占用率。
   
5. 高并发

    Kafka 使用多线程异步处理请求，在单机上可以轻松支撑数千个客户端同时读写。同时它通过 batch 机制优化网络 I/O ，进一步提升了并发处理能力。
    
## 2.2 Apache Zookeeper简介

Apache Zookeeper 是 Hadoop 和 Hbase 的依赖组件，它是一个开源的分布式协调服务，用来维护配置信息、命名服务、状态同步等。Zookeeper 是一个分布式锁服务，用来防止同时处理相同事务（对于事务的定义需要自己对照）。Zookeeper 也提供“临时”节点，这些节点在连接断开或者会话超时之后就会消失。

## 2.3 Spring Boot简介

Spring Boot 是由 Pivotal 团队提供的一套 Java 开发框架，目的是为了简化 Spring 框架的搭建过程。Spring Boot 可以做到开箱即用，只需要简单配置，就可以快速启动并运行一个微服务。 Spring Boot 采用了编码简单、内嵌 Tomcat 或 Jetty 以及无需 XML 配置等特性来达到最佳启动时间和较小内存占用率。
