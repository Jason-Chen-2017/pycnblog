## 2. 核心概念与联系

### 2.1 区块链基本概念

区块链是一种分布式账本技术，由一系列按时间顺序排列的区块组成。每个区块包含一组交易记录，区块之间通过哈希函数进行连接，形成一个不可篡改的链状结构。

#### 2.1.1 区块结构

- **交易记录（Transactions）**：每个区块包含多个交易记录，交易记录记录了资金的转移、数据的状态变更等操作。
- **区块头（Block Header）**：区块头包含区块的元数据，如区块版本、前一区块的哈希值、默克尔树根哈希值、时间戳等。
- **区块体（Block Body）**：区块体包含具体的交易记录数据。

#### 2.1.2 哈希函数

哈希函数是一种将任意长度的输入数据映射为固定长度的输出数据的函数。在区块链中，常用的哈希函数有SHA-256、SHA-3等。哈希函数具有以下特性：

- **单向性**：给定输入数据，可以快速计算出哈希值，但给定哈希值，无法反推出原始数据。
- **抗碰撞性**：两个不同的输入数据，其哈希值几乎不会相同。

#### 2.1.3 智能合约

智能合约是一种运行在区块链上的计算机程序，可以自动执行、控制或记录符合预定的条件或指令。智能合约实现了去中心化的自动化执行，减少了中介环节，提高了效率。

### 4.2 公式推导过程

以RSA加密算法为例，其数学模型如下：

- **密钥生成**：

  设大素数\( p \)和\( q \)互质，计算\( n = p \times q \)，\( \phi = (p - 1) \times (q - 1) \)。

  选择一个与\( \phi \)互质的整数\( e \)，计算\( d \)，使得\( d \times e \equiv 1 \pmod{\phi} \)。

  公钥为\( (n, e) \)，私钥为\( (n, d) \)。

- **加密过程**：

  设明文为\( M \)，加密后的密文为\( C \)，计算\( C = M^e \pmod{n} \)。

- **解密过程**：

  设密文为\( C \)，解密后的明文为\( M \)，计算\( M = C^d \pmod{n} \)。

2. **公钥交换**：

```go
    // 用户消息结构
    struct Message {
        address sender;
        string content;
        uint256 timestamp;
    }

    // 消息存储
    Message[] public messages;

    // 发送消息

        messages.push(Message(msg.sender, content, block.timestamp));
    }

    // 获取消息

        return messages;
    }
}
```

- **发送消息**：`sendMessage`函数用于发送消息，通过`push`操作将消息存储在`messages`数组中。
- **获取消息**：`getMessages`函数用于获取所有消息，返回`messages`数组。
