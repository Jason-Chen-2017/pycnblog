# 1.简介
  

NoSQL（Not Only SQL）是一个泛指非关系型数据存储技术，与传统的关系型数据库不同之处在于，NoSQL不满足关系模型的ACID原则，并没有严格的数据关系约束条件。但是它通过一些特有的功能特性提供了相对简单的查询方式，能够方便地存储海量数据。由于其分布式、弹性可伸缩、灵活的数据模型和高性能等优点，目前越来越多的公司、组织和个人开始使用NoSQL进行数据存储和分析。本文将会从NoSQL的背景介绍、特点、主要应用场景三个方面，以及SQL数据库与NoSQL数据库的比较，阐述它们之间的区别与联系。
# 2.背景介绍
## NoSQL概览
NoSQL，即“not only SQL”，一般用于描述非关系型数据库。如MongoDB、Cassandra、HBase等。这些数据库都不是关系型数据库，而是提供不同于传统RDBMS的存储机制，使得它们可以更好地处理大数据量和高速查询要求。

## MongoDB概览
MongoDB是一个基于分布式文件存储的开源NoSQL数据库。类关系型数据库管理系统，支持动态 schemas、自动索引、分片集群等功能。它是一个面向文档的数据库，最大特点是 schemaless，也就是说不需要提前定义 table 的结构。可以灵活地存储、查询、索引和复制不断增长的数据集合。它的优势包括高性能、易部署、易维护、自动平衡负载等。

## Cassandra概览
Apache Cassandra 是 Apache Software Foundation 下的一个开源项目，是一个高可用性、高吞吐量的分布式 NoSQL 数据库。相比于其他的 NoSQL 产品来说，Cassandra 有着独特的设计目标——高可用性和强一致性。Cassandra 将数据存储在一系列的节点上，每个节点都是数据中心内的一台服务器。它采用了“去中心化”架构，这样一来，即便一个节点所在的数据中心发生了故障，整个 Cassandra 仍然可以正常运行。Cassandra 也允许用户设置不同的复制策略，以实现数据的冗余备份，保证数据的高可用性。另外，Cassandra 支持按列存储数据，这使得 Cassandra 在处理分析时具有很好的性能优势。

# 3.基本概念术语说明
## 3.1 ACID属性
ACID（Atomicity、Consistency、Isolation、Durability）是关系数据库最基本的四个属性。ACID保证事务的原子性、一致性、隔离性、持久性。
- Atomicity（原子性）：一个事务要么全部执行，要么完全不执行。
- Consistency（一致性）：一个事务完成后，该事务所导致的数据库状态一定是正确的。
- Isolation（隔离性）：多个并发事务之间不会互相干扰，各自独立地执行。
- Durability（持久性）：一个事务一旦提交，它对数据库中数据的改变就永久保存下来。即使系统崩溃，数据库也能恢复到事务成功之前的状态。

## 3.2 CAP定理
CAP定理指出，对于一个分布式计算系统来说，不可能同时做到一致性（Consistency），可用性（Availability）和分区容错性（Partition Tolerance）。最多只能同时保证两个。
- Consistency（一致性）：在同一时间，所有节点的数据副本都相同。
- Availability（可用性）：一个客户端可以读取数据，无论这个数据是在哪个节点上。
- Partition Tolerance（分区容错性）：当网络分区出现的时候，系统仍然能够继续工作。换句话说就是，系统能承受任意个节点或网络分裂。

## 3.3 BASE原则
BASE是Basically Available(基本可用)，Soft state（软状态），Eventually consistent(最终一致性)三个短语的首字母。
- Basically Available（基本可用）：即使无法保证强一致性，但每个请求总会收到响应。
- Soft state（软状态）：允许系统中的数据存在中间状态，而这个中间状态不影响系统整体的功能，比如缓存。
- Eventual consistency（最终一致性）：一旦某个数据被更新，所有的读操作都会立即得到更新的数据，但是不保证具体的时间。

## 3.4 消息队列
消息队列又称为消息中间件，是一种应用程序间的通信方式。消息队列的作用是为应用程序提供异步的、松耦合的通信方式。消息队列是一种典型的发布/订阅模型，生产者（Publisher）发送消息到消息队列，消费者（Consumer）从消息队列订阅感兴趣的消息，这一模式简化了应用之间的依赖，降低了耦合度。消息队列的好处包括降低系统间通信成本、提升系统可靠性、解耦业务模块等。
