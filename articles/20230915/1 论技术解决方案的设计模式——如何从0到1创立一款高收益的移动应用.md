# 2.基本概念术语说明
## 2.1 移动互联网（Mobile Internet）
移动互联网指的是利用手机、平板电脑或其他移动终端等设备作为信息交换媒介的互联网。其特点是覆盖面广、速度快、成本低、无线信号覆盖广、基础设施差异小、服务多样化、用户群体多元。目前全球有上亿人口依赖移动互联网，包括消费者、制造商、企业用户、政府部门、银行等。

## 2.2 Android、iOS系统
Android和iOS是两个运行于移动设备上的操作系统。它们由谷歌、苹果公司和其它厂商开发，共同打造出基于这些平台的智能手机和平板电脑。两者都使用Java语言编写，并拥有自己的一套编程接口API。他们共享相同的代码库，可实现功能相同但不同外观和感觉的操作系统。其中Android支持谷歌的开源安卓系统，它是自由、开放源码、稳定性佳，用户界面设计精美；而iOS支持苹果的操作系统，它提供安全、流畅、功能丰富、视觉效果逼真的手机操作系统。

## 2.3 Hybrid开发框架
Hybrid开发框架是一种可以将Web技术和客户端技术结合起来构建跨平台、运行效率高的应用程序的方式。通过HTML5、CSS、JavaScript开发人员可以构建具有丰富功能的跨平台应用程序，并且可以使用原生的客户端API和第三方SDK访问硬件能力。Facebook的React Native、腾讯的Weex和阿里的UniApp均属于这种类型。

## 2.4 移动应用开发模式
移动应用开发模式又称为应用生命周期模型（Application Lifecycle Model），是一种用于开发移动应用的生命周期模型，主要包括以下几个阶段：
1. 需求分析阶段：需求分析一般分为市场调研、竞品分析、功能分析和可用性分析等四个部分。该阶段会收集用户的诉求，制定产品的目标，然后确定应用的功能需求、性能要求、界面设计、兼容性要求、用户学习成本、数据安全和保护要求等。
2. 设计阶段：设计阶段围绕着产品的目标和需求进行设计。涉及产品设计、界面设计、用户交互设计、程序逻辑设计、数据库设计、测试方案设计等方面。对产品的界面和交互设计非常重要，它决定了用户的使用体验。
3. 开发阶段：开发阶段把前面设计好的产品进行编码和测试，同时还要考虑适配不同的机型、进行单元测试、集成测试等。这里需要掌握多种语言的使用，例如Java、Objective-C、Swift、Python等。
4. 测试阶段：测试阶段是在最终发布之前，对产品进行严格的测试，包括功能测试、兼容性测试、易用性测试、性能测试、安全性测试、使用性测试等。测试时也要注意用户反馈，调整产品策略和优化细节。
5. 部署阶段：部署阶段是把产品部署到生产环境，通常需要上传应用到应用市场，用户才能下载安装使用。当产品达到一定规模后，可能还需要进行App Store审核、版本更新等工作。
6. 支持阶段：支持阶段是维持应用的正常运行和升级，在出现问题的时候及时进行维护，确保产品的稳定性、可用性、安全性和客户满意度。
