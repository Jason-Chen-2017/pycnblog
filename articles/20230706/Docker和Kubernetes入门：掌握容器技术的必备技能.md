# 2. 技术原理及概念

## 2.1. 基本概念解释

2.1.1. 容器

容器是一种轻量级、可移植的运行时实例。它包含了一个完整的操作系统和应用程序，可以在任何地方运行，只要有相应的环境。容器化技术可以将应用程序和操作系统打包成单个的可移植的打包文件，然后通过网络传输到另一个环境或主机。

## 2.1.2. Docker

Docker是一种开源的容器化平台，它提供了一种在不同环境中打包、发布和运行应用程序的方式。Docker使用轻量级、可移植的镜像来打包应用程序和操作系统，使得应用程序在任何地方都能够稳定运行。

## 2.1.3. Kubernetes

Kubernetes是一种开源的容器编排平台，用于管理和自动化容器化应用程序的部署、扩展和管理。它使用微服务架构来分解应用程序，使用容器化技术来打包、部署和管理这些微服务。

### 2.2.1. Docker的工作原理

Docker的工作原理包括以下几个步骤：

1. 构建镜像：使用Dockerfile文件构建镜像，该文件描述了如何构建一个Docker镜像。
2. 运行镜像：使用docker run命令运行镜像。
3. 获取镜像：使用docker images命令获取镜像列表。
4. 删除镜像：使用docker rmi命令删除不再需要的镜像。
5. 推送镜像：使用docker push命令将镜像推送至Docker Hub。
6. 拉取镜像：使用docker pull命令从Docker Hub拉取镜像。

### 2.2.2. Kubernetes的工作原理

Kubernetes的工作原理包括以下几个步骤：

1. 创建命名空间：使用kubectl create namespace命令创建一个命名空间。
2. 创建资源：使用kubectl get命令获取资源，然后使用kubectl apply命令创建或更新资源。
3. 部署应用程序：使用kubectl apply-async命令部署应用程序。
4. 扩展资源：使用kubectl scale命令扩展资源。
5. 删除资源：使用kubectl delete命令删除资源。
6. 获取资源：使用kubectl get命令获取资源列表。
7. 打印资源：使用kubectl logs命令查看资源日志。
