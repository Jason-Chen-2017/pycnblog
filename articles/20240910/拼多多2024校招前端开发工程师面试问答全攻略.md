## 前端基础问题

### 1. HTML 和 XHTML 的区别是什么？

**答案：** HTML 是超文本标记语言，而 XHTML 是可扩展超文本标记语言。XHTML 主要是对 HTML 的改进，更加严格遵循 XML 规范。具体区别如下：

- **语法严格：** XHTML 要求标签必须小写，属性值必须用引号括起来，标签必须成对出现等。
- **命名空间：** XHTML 需要为文档定义一个命名空间，以便与 XML 文档兼容。
- **兼容性：** HTML 在大多数浏览器中都可以正常显示，而 XHTML 需要浏览器支持 XML 解析。

### 2. CSS 的定位机制有哪些？

**答案：** CSS 的定位机制主要包括以下几种：

- **静态定位（static）：** 默认值，元素按照正常文档流进行布局。
- **相对定位（relative）：** 相对于元素在文档流中的正常位置进行定位。
- **绝对定位（absolute）：** 相对于最近的非 `static` 父元素进行定位，如果不存在这样的父元素，则相对于 `<html>` 元素进行定位。
- **固定定位（fixed）：** 相对于浏览器窗口进行定位，即使滚动页面，元素的位置也不会改变。

### 3. JavaScript 中的事件处理有哪些方式？

**答案：** JavaScript 中的事件处理有以下几种方式：

- **内联事件处理：** 在 HTML 元素的事件属性中直接绑定事件处理函数。
- **DOM0 级事件处理：** 通过元素的 `on` 属性绑定事件处理函数。
- **DOM2 级事件处理：** 使用 `addEventListener` 方法绑定事件处理函数。
- **事件冒泡和捕获：** 事件在 DOM 树中传递的过程，包括事件冒泡和事件捕获。

## JavaScript 问题

### 4. 什么是闭包？

**答案：** 闭包是函数和其 surrounding lexical environment 的组合体。它可以让函数访问并保持其定义时的环境状态。闭包的作用主要体现在以下几个方面：

- **保存变量：** 闭包可以访问并保存函数定义时的局部变量，即使函数已经返回。
- **实现封装：** 闭包可以隐藏函数的内部实现，只暴露有限的方法供外部访问。
- **实现模块化：** 闭包可以用来实现模块化编程，将函数和相关的数据封装在一起。

### 5. JavaScript 中的 this 是如何工作的？

**答案：** JavaScript 中的 `this` 关键字是指向函数执行时的上下文对象。`this` 的值取决于函数的调用方式：

- **独立函数调用：** `this` 指向全局对象（在严格模式下为 `undefined`）。
- **对象方法调用：** `this` 指向调用该方法的对象。
- **构造函数调用：** `this` 指向构造函数的实例。
- **调用函数时绑定：** 使用 `Function.prototype.bind` 方法绑定 `this` 的值。

### 6. 原型和原型链是什么？

**答案：** 原型是函数的属性，用于存储构造函数的原型对象，而原型链则是通过原型对象实现的继承关系。每个对象都有一个内部属性 `[[Prototype]]`，指向其原型对象。当访问一个对象的属性时，如果该属性不存在于对象本身，则会沿着原型链向上查找，直到找到该属性或达到原型链的末尾（`Object.prototype`）。

## Web 进阶问题

### 7. 什么是 RESTful API？

**答案：** RESTful API 是一种基于 REST（Representational State Transfer）风格的 Web 服务架构。它通过 HTTP 协议提供 Web 服务，使用动词（GET、POST、PUT、DELETE）表示操作，使用 URL 表示资源，使用 JSON 或 XML 表示数据。RESTful API 的主要特点包括：

- **无状态：** API 不保留客户端的状态。
- **统一接口：** 使用统一的接口设计，如 URL、HTTP 方法、状态码等。
- **可缓存：** API 允许缓存响应，提高性能。
- **无限制：** API 可以使用各种数据格式（JSON、XML、HTML 等）。

### 8. 什么是事件循环？

**答案：** 事件循环（Event Loop）是 JavaScript 中处理异步任务的关键机制。它允许 JavaScript 在等待异步任务完成时继续执行其他任务，从而实现并发执行。事件循环主要包括以下几个阶段：

- **宏任务队列：** 包括定时器、用户交互、I/O 操作等。
- **微任务队列：** 包括 `Promise`、`MutationObserver` 等。
- **事件循环：** 每轮事件循环从宏任务队列中取出一个任务执行，然后执行微任务队列中的任务，直到队列清空。

## 算法编程题库

### 9. 实现一个洗牌函数

**题目：** 编写一个函数，实现将数组进行随机洗牌。

**答案：** 使用 Fisher-Yates 算法实现。

```javascript
function shuffleArray(array) {
  for (let i = array.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [array[i], array[j]] = [array[j], array[i]];
  }
  return array;
}

// 示例
const arr = [1, 2, 3, 4, 5];
console.log(shuffleArray([...arr]));
```

### 10. 实现一个二分查找算法

**题目：** 编写一个二分查找函数，实现在一个有序数组中查找目标元素的索引。

**答案：**

```javascript
function binarySearch(arr, target) {
  let left = 0;
  let right = arr.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);

    if (arr[mid] === target) {
      return mid;
    } else if (arr[mid] < target) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }

  return -1;
}

// 示例
const arr = [1, 2, 3, 4, 5];
console.log(binarySearch(arr, 3)); // 输出 2
```

## 常见前端面试题

### 11. 什么是事件冒泡？

**答案：** 事件冒泡是指当某个元素上的事件触发时，该事件会先在该元素上触发，然后逐级向上传播到其父元素，直到传播到 document 对象。这是一个从子元素到父元素的过程。

### 12. 什么是事件捕获？

**答案：** 事件捕获是指当某个元素上的事件触发时，该事件会从 document 对象开始，逐级向下传播到目标元素。这是一个从父元素到子元素的过程。

### 13. 解释一下事件代理。

**答案：** 事件代理是指通过给父元素绑定事件处理函数，来处理子元素的事件。这样，当子元素上的事件触发时，会冒泡到父元素，从而在父元素上调用事件处理函数。

### 14. 解释一下什么是同源策略。

**答案：** 同源策略是一种安全策略，用于限制浏览器加载从其他源（协议、域名或端口不同）访问的文档或脚本。该策略可以防止恶意网站通过窃取其他网站的 cookie 或进行其他非法操作。

### 15. 什么是跨域？

**答案：** 跨域是指由于浏览器的同源策略限制，一个域下的文档或脚本无法与另一个域下的资源进行交互。例如，一个页面中的 JavaScript 无法与另一个域下的服务器进行 AJAX 请求。

### 16. 如何解决跨域问题？

**答案：** 解决跨域问题的常见方法有：

- **CORS（Cross-Origin Resource Sharing）：** 通过设置响应头 `Access-Control-Allow-Origin` 来允许跨域请求。
- **JSONP：** 通过动态创建 `<script>` 标签并加载跨域资源来绕过同源策略。
- **代理服务器：** 通过搭建一个代理服务器，将跨域请求转发到目标服务器，从而实现跨域访问。
