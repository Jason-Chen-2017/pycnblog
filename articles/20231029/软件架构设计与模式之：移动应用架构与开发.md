# 2.核心概念与联系
## **2.1. MVC模式**
MVC（Model-View-Controller）模式是一种经典的软件架构设计模式，它将应用程序分为三个部分：模型（Model）、视图（View）、控制器（Controller）。模型负责处理数据逻辑，视图负责展示数据，控制器负责接收用户输入并调用模型和视图进行处理。这种设计模式使得代码更加模块化和易于维护，同时也便于实现数据的持久化存储。

## **2.2. MVVM模式**
MVVM（Model-View-ViewModel）模式是MVC模式的一种衍生模式，它加入了ViewModel层。ViewModel是一个轻量级的对象，主要负责处理视图逻辑，减轻视图层的负担。MVVM模式相比MVC模式更加关注用户体验，能够实现更好的异步更新和数据绑定效果。

## **2.3. React Native、Flutter等新技术**
React Native和Flutter等新兴的技术可以用于移动应用架构的设计。它们基于原生语言编写，具有更高的性能和更快的开发速度。此外，它们还支持跨平台开发，能够在多种设备上运行相同的应用。
