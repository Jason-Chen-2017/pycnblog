 # 2.核心概念与联系
  ### 2.1 异常和错误的区别
   异常是程序运行时发生的意外情况，通常是程序的预期之外的情况。例如，除数为0是一个异常。错误是人类可以预测到的结果，通常是由于程序员或系统的错误导致的。例如，拼写错误是一个错误。
   在Go语言中，异常处理主要针对异常情况进行处理，而错误处理则主要针对错误情况进行处理。但是，在一些特殊情况下，异常可能会导致错误，反之亦然。因此，在实际编程中，我们需要对异常和错误进行适当的处理和管理。
  ### 2.2 捕获和忽略异常
   在Go语言中，我们可以通过使用try-catch语句来捕获和忽略异常。其中，try块包含可能抛出异常的代码块，而catch块包含处理异常的代码块。如果没有捕捉到异常，则会继续执行默认的错误处理函数，通常是print(“error: “+e)或者panic(“error:“+e)。
   当我们忽略一个异常时，可以使用continue语句跳过当前try块中的后续代码，然后继续执行下一个try块或者主程序逻辑。
  ### 2.3 错误码
   在Go语言中，我们可以使用错误码的方式来进行错误处理。错误码是一个整数值，用于表示特定类型的错误。例如，HTTP服务器返回的状态码404表示未找到请求资源，状态码500表示服务器内部错误。我们可以使用errors.New()函数根据错误码创建对应的错误类型，然后使用这些错误类型进行错误处理。
  ### 2.4 错误处理函数
   在Go语言中，我们可以自定义错误处理函数来处理特定的错误情况。这些函数可以在main()函数之外声明，然后在具体的地方被调用。使用googles擦除约定，可以在函数声明前添加“go”关键字来跳过函数的声明，直接将其作为参数传递给函数。例如：
    ```
    func handleError(err error) {
        if err != nil {
            log.Println(err)
        }
    }
    ```
 # 3.核心算法原理和具体操作步骤以及数学模型公式详细讲解
   ### 3.1 异常切换（Exception Switching）
   异常切换是一种处理异常的机制，它涉及到在多个try块之间切换不同的错误处理函数。每个try块都会有自己的捕获链，当一个异常发生时，会按照捕获链的顺序依次查找对应的通知函数。如果找到了通知函数，则会立即执行该函数。否则，会将异常信息记录下来，并在所有的通知函数中回调处理。这个机制的主要优点是可以方便地扩展错误处理函数，但同时也有一定的开销。
   ### 3.2 静态异常表（Static Exception Table）
   静态异常表是一种基于地址的异常处理方式，它在Go语言中被广泛使用。当一个函数被调用时，首先会在静态异常表中查找对应的异常信息，如果找不到，则会调用堆栈跟踪器进行异常处理。这个机制的优点是可以方便地进行异常处理，但同时也有一些限制，比如无法捕获人为错误。
   ### 3.3 自定义错误处理函数
