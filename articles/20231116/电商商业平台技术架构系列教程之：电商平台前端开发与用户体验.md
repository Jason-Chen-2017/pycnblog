# 2.核心概念与联系
为了更好地理解和掌握电商商业平台前端开发的技能，我们首先需要了解一些基本的前端技术概念。
## 什么是MVVM？
MVVM（Model-View-ViewModel）模式是一种用来开发WPF，Silverlight，Windows Store应用以及跨平台 Xamarin Forms 应用程序的设计模式。这种模式主要关注点分离，让开发者把精力集中于业务逻辑处理和数据展示层。其中的“V”指ViewModels，“M”指Models，“VM”指ViewModels。他们之间的关系如下图所示：
MVVM 模式中，ViewModel 是唯一与View通信的对象，它负责向 View 提供所需的数据。 ViewModel 不直接处理 UI 的渲染，而是转交给View的命令去做相应的事情。通过 ViewModel 将 Model 和 View 分开，可以使两者之间松耦合，并达到可测试性的作用。
## Web前端框架
目前流行的前端框架主要有 Angular，React，Vue，jQuery，BootStrap。其中，React 和 Vue 都是用于构建大型应用的框架，都是单页应用 (Single Page Application, SPA)。AngularJS 是基于 MVC（模型-视图-控制器） 模式的客户端JavaScript 框架，它采用了指令式编程，支持双向绑定，依赖注入和路由管理。 jQuery 在创建插件方面的能力很强，并且提供了一套良好的 API ，适合快速开发。 BootStrap 是 Twitter 提供的开源 CSS 框架，它提供丰富的基础样式和组件，可以快速开发响应式布局的界面。
## HTML / CSS
HTML （Hypertext Markup Language）是一门描述网页的内容结构的语言，CSS（Cascading Style Sheets）则用于控制网页的样式。HTML 标签定义了文档的结构，CSS 则通过选择器设置不同元素的显示方式，如颜色、大小、位置、背景等。
## JavaScript
JavaScript 是一门解释性语言，它的运行环境被广泛地应用于网页的前端开发，比如浏览器端和服务器端。JavaScript 通过 DOM（Document Object Model），可以操纵 HTML 文档的各种元素。除了 JavaScript，还有其他一些前端开发的技术，如 React，Vue，TypeScript 等，这些技术都可以帮助开发人员构建更复杂的前端应用。
## 浏览器渲染流程
当打开一个网页时，浏览会按照以下几个步骤渲染页面：

1. DNS解析，根据域名查询IP地址；
2. TCP连接，建立TCP链接，三次握手确认；
3. HTTP请求，发送HTTP请求获取资源；
4. 下载响应，接收服务器返回的资源文件；
5. 渲染页面，浏览器对HTML、CSS、JS进行解析，生成DOM树和CSSOM树，结合生成渲染树；
6. 解析Javascript，执行Javascript脚本，更新页面显示；
7. 渲染完毕，呈现给用户，完成一次完整的页面渲染流程。

以上是浏览器渲染页面的一个大概流程。至此，我们已经了解了一些前端相关的术语和概念，下面我们开始进入正题。

## 数据传输协议
由于浏览器只能识别文本格式的数据，因此要传输二进制数据就需要使用不同的编码格式，常用的包括 base64，utf-8 等。前端往往需要封装自己的业务接口，不同的业务接口用不同的协议，比如 HTTP，WebSocket，Socket 等。这里我们重点讨论一下 HTTP协议。
### HTTP 请求方法
HTTP请求共有七种方法，包括 GET，POST，PUT，DELETE，HEAD，OPTIONS，TRACE。其中，GET 方法通常用于只读的请求，也就是请求资源的目的只是获取信息，不修改资源。POST 方法通常用于新增资源或修改资源，它要求向服务器提交数据。PUT 方法类似于 POST 方法，但是用于更新资源，而不是新建资源。DELETE 方法用于删除服务器上的资源。HEAD 方法用于只获取 HTTP 头部信息，不返回实体内容。OPTIONS 方法用于检查服务器是否支持某些特殊功能，或者用于测试服务器的性能。TRACE 方法用于追踪经过代理或网关的请求。
### 状态码
状态码（Status Code）用于表示HTTP请求的成功或失败。HTTP状态码的分类有五种类型：
* 1xx：指示信息–表示请求已收到， continuing process 。
* 2xx：成功–表示请求已成功被服务器接收、理解、并接受。
* 3xx：重定向–要完成请求必须进行更进一步的操作。
* 4xx：客户端错误–请求有语法错误或请求无法实现。
* 5xx：服务器错误–服务器不能正常响应请求。
### Cookie和Session
Cookie和Session都是记录服务器端状态信息的方法。区别是，Cookie保存在客户机的本地磁盘上，可以保留多个会话，而Session保存在服务器内存中，仅在当前会话有效。Cookie一般用于保存登录信息，Session一般用于保存购物车信息、游戏分数等。
### HTTPS加密传输
HTTPS (HyperText Transfer Protocol Secure)，即 HTTP over TLS/SSL，是 HTTP 下一代安全协议。HTTPS 的主要特点是建立在 SSL/TLS 上，利用 SSL 认证用户的身份，确保数据在传输过程中不会被篡改。浏览器会自动识别并访问安全站点的 URL，例如HTTPS。
