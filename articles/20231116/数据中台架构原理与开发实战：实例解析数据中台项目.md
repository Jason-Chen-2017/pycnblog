                 

# 1.背景介绍


## 数据中台概念
数据的规模已经是复杂的海量，很多企业都在面临大量的数据管理、分析和应用需求。如何快速有效地提取、清洗、整合、存储、分析、挖掘这些数据，是构建一支高效的数据科技团队的关键因素。

随着互联网业务的发展，公司业务模式从简单单体模式向集成协同、模块化水平发展。服务中台(SOA)的出现使得应用与服务的解耦，但是数据管理却仍然存在多个数据中心，成为整个系统运行的瓶颈之一。为了解决这个难题，中央数据仓库(DW)的出现带来了一种全新的解决方案-数据中台架构。数据中台由三个组件构成:数据采集、计算平台和数据服务平台。它位于公司各个业务部门之间，汇聚不同数据源的历史数据、实时数据和实时计算结果，提供数据驱动的业务决策支持。


数据中台架构，包含三个主要部分：

1. 数据采集层（Data Ingestion）：采集层负责从业务系统中抽取、转换、传输数据到数据湖。数据湖是一个存储数据元数据的地方，数据湖中的数据可以被下游消费者或者其它应用消费。数据采集层通过标准接口将数据注入到数据湖中，并保证数据准确无误地进入数据湖。如图所示，数据采集层包含数据导入模块、数据转换模块、数据校验模块、数据推送模块等。

2. 计算平台层（Compute Platform）：计算平台为数据湖中的数据赋能，对数据进行处理并生成新的数据，如数据清洗、数据合并、数据分组、数据预测等，计算平台还会将计算结果持久化到数据湖中。计算平台分为离线计算平台和实时计算平台，离线计算平台适用于数据量较小，实时计算平台则更适合于对实时的需求。如图所示，计算平台包括数据清洗模块、数据转换模块、数据建模模块、数据预测模块、数据可视化模块、数据存储模块等。

3. 数据服务平台层（Data Service Platform）：数据服务平台提供了数据湖中数据的访问入口，为业务部门的应用提供数据服务。数据服务平台可以通过数据接口，按需获取数据湖中数据，并返回相应的结果。数据服务平台根据不同的业务诉求，提供不同的查询、报表、分析功能。如图所示，数据服务平台包含数据查询模块、数据报表模块、数据分析模块等。

数据中台架构能够帮助企业降低数据存储成本，简化数据治理流程，加快数据价值发现和应用上线周期，实现业务主导和数据驱动双赢。

## 数据中台架构设计原则

数据中台架构设计原则主要遵循以下四条规则：

1. 抽象数据源：不同数据源之间的差异尽可能隐藏起来。如企业内部的日志文件、关系型数据库、非关系型数据库等，不应该直接被接入数据中台。所以应当抽象出一个统一的数据模型，使得不同数据源之间的数据一致性得到保障。

2. 增量更新：数据中台需要支持增量更新，确保新加入或更新的数据能够及时反映到数据中台中。

3. 一致性：数据一致性问题是数据中台最重要的要求。要确保数据准确无误地同步，并且所有数据都能够被有效利用。

4. 流程自动化：由于数据中台的定位在于管理和自动化数据流转过程，所以流程自动化也是其最重要的设计原则之一。数据中台应当充分考虑业务流程的自动化设计，让流程流畅地执行，从而达到数据集中、数据精准、数据即用的目的。

## 数据中台架构设计思路

数据中台架构设计思路主要有以下几种：

1. 中心角色：数据中台架构将数据和计算能力放在中心位置，通过集成多个数据源、模块化服务、云端部署和自动化流程，提供集中化、自动化、可靠的数据服务。中心角色包括数据采集层、计算平台层和数据服务平台层。

2. 模块化服务：数据中台架构的核心优势之一就是模块化服务，通过对数据层面的业务理解，将不同的数据进行分类、归纳、匹配、关联，形成有效的数据服务。通过模块化服务，数据中台能够通过数据优化、数据融合等方式，将不同的数据进行整合和分析，满足不同用户的不同数据需求。

3. 混合存储：数据湖通常采用多种形式的存储介质，例如关系型数据库、NoSQL数据库、搜索引擎等。数据中台需要兼顾性能、成本、可用性和易用性，选择合适的存储介质、存储方案，才能最大限度地提升数据湖的服务能力。

4. 可扩展性：数据中台架构能够灵活地对数据进行采集、处理和服务，支持各种类型的业务系统、接口协议、开发语言、数据模型等，为业务创造新的商业价值。

# 2.核心概念与联系
## 定义
数据中台架构是一种基于集成的方式，将数据采集、计算、存储、查询、分析、可视化和数据服务等多个环节模块化，最终打通数据采集、数据处理、数据服务等多个环节，实现数据集中管理、自动化运维、数据共享和数据价值提升。数据中台架构将数据采集层、计算平台层、数据服务层和数据湖四个核心组件进行了有效的编排，彼此之间相互依赖，共同为最终业务需求提供服务。其基本特征是：中心化、集成、自治。

## 总揽
数据中台架构是指把数据仓库的管理，数据处理，数据共享和数据服务的多个环节模块化，并且与业务流程相结合，打通数据采集，数据处理，数据服务等多个环节，提升数据价值，建立起统一的数据视图，达到降低数据管理成本、提升数据工作效率、促进业务发展的目的。数据中台架构的核心组件为数据采集层、计算平台层、数据服务层和数据湖。其中，数据采集层负责数据收集，数据湖负责数据存储；计算平台层提供大数据分析的基础环境，数据服务层提供统一的访问入口，供业务系统调用。数据中台架构的设计原则包括抽象数据源，增量更新，一致性和流程自动化。

数据中台架构的优点主要有：

· 提升数据价值：数据中台架构能够通过对数据进行分类、归纳、匹配、关联，形成有效的数据服务，通过数据优化、数据融合等方式，将不同的数据进行整合和分析，提供数据驱动的业务决策支持。

· 提升数据工作效率：数据中台架构将数据采集层、计算平台层、数据服务层和数据湖四个核心组件进行了有效的编排，彼此之间相互依赖，共同为最终业务需求提供服务，能够降低数据管理成本、提升数据工作效率，增加数据价值。

· 降低数据管理成本：数据中台架构能够利用云计算和大数据分析技术，降低数据管理成本，促进企业数字化转型，提升组织竞争力。

· 促进业务发展：数据中台架构为企业搭建起统一的数据视图，能够基于数据洞察，做好各类定制化的业务开拓，提升组织能力。
