                 

# 1.背景介绍


## 1.1 面试官的期望
很多技术面试者都对项目管理比较感兴趣，因为作为技术人最重要的事情之一就是把自己的技术实力运用到实际应用中，而项目管理是把技术实力付诸行动的第一步。

当然，除了技术以外，项目管理也是非常重要的职业发展路径中的一环。任何一个技术人员，如果没有相关的项目经验，在面试中谈及项目管理一般会被认为是不自信甚至落后。原因如下：
- 首先，技术人员通常不具备独立完成一项完整项目的能力；
- 其次，项目管理知识需要针对不同的业务领域进行训练，有些项目管理理论或方法适用于某些业务领域但不能应用于其他业务领域，所以对于不同业务领域的项目管理要求是不同的；
- 最后，面向全栈工程师或者技术经理等岗位的人群，很难找到独立负责一个完整项目的工作岗位，项目经验往往是不可替代的。因此，即使面试官没有明确指出项目管理方面的需求，技术面试者也很可能会提到项目管理知识。

## 1.2 为什么要学习项目管理？
在项目管理中，可以学到以下几个重要的知识点：
- 技术导向：项目管理与技术密切相关，它强调采用“以客户为中心”的思想，关注客户需求、解决方案及其背后的商业模式，采用迭代和循序渐进的方式开发产品。
- 沟通能力：项目管理涉及多个角色、各部门协作、跨团队合作，需谦虚谨慎地与他人的沟通，尤其注意对信息的传递和交流。
- 执行能力：项目管理涉及各种资源、计划、预算、风险、质量、进度等多种因素，项目经理必须精益求精，认真细致地管理项目流程和资源。
- 业务理解：项目管理要清楚自己在做什么，明白客户需求，根据市场需求和竞争对手分析，并制定可行性方案，让客户满意。

# 2.核心概念与联系
## 2.1 项目管理概述
### 2.1.1 项目管理的定义
项目管理，是以组织、协同和决策为目的，为了实现组织目标、创造价值并维护其成功而实施的一系列活动的统称。项目管理的目的是为了达成组织目标，促进企业的持续发展，保持高效、有效、美观、可持续的运营。项目经理是一个带有管理意识、主导能力和领导才能的组织成员，他们的任务主要包括项目策划、设计、实施、评审、控制和监控等，帮助企业创建和持续成功的产品或服务。

项目管理（Project Management）是由德国计算机科学家约翰·本茨（John Benjamin）教授于1978年提出的新型管理理论。按照其定义，项目管理是一种领导、组织和协同的过程，旨在在给定的时间、资源和预算内，以客观、有序的方式，以可衡量的结果，改善企业、组织和个人的效率、生产力、质量和效益。

### 2.1.2 项目管理的层次结构
目前，按照项目管理的层次结构，可以分为甲级（初级）、乙级（中级）、丙级（高级）。
- 甲级项目管理：这一阶段的项目管理基本上不关注预算，主要关注工作量、质量、进度、范围，最常见的为燃尽图法。
- 乙级项目管理：项目经理的任务包括项目策划、资源分配、进度跟踪、质量保证、沟通协调等，其重点是以合理、有效的方法、工具来达成项目目标，并最终实现公司战略目标。此时，项目经理需具备较强的分析、决策能力，能够识别和解决突发事件、将商业价值转化为技术价值。
- 丙级项目管理：这种项目管理的级别更高一些，相比于乙级，其重心放到了财务预测、资源利用率以及合规法律等方面。丙级项目经理可以掌握大量的金融、财务、法律、文化、市场营销、人力资源、物料管理等专业知识，能够将复杂且艰苦的项目成功推向前进。

### 2.1.3 项目管理的六个基本特征
项目管理有六个基本特征，分别是：
- 正确性：项目经理应确保项目活动都符合企业的战略方向、计划、约束条件，确保工作的顺利进行。
- 专业性：项目经理应对所负责的领域进行专门的培训，获取高度专业知识和技能。
- 综合性：项目经理应充分考虑现有资源、外部环境和历史遗留问题，确保项目整体的科学性、系统性和可管理性。
- 服务性：项目经理应始终处于服务的立场上，为客户提供可靠的服务，并积极参与组织事务。
- 交流性：项目经理应善于和团队成员、客户、业务伙伴等之间互相沟通，提升沟通能力和协调能力。
- 创新性：项目经理应具有创新能力，主动探索新产品或服务的研发方法和流程。

### 2.1.4 项目管理的四种角色
项目管理的四种主要角色：
- 项目总监：项目总监的职责是在整个项目生命周期的各个层面负责管理、协调和支持所有项目的开展。
- 项目经理：项目经理的职责是将企业战略纳入到每个项目当中，并对项目的执行进行指导、监督和支持。
- 项目助理：项目助理的职责是配合项目经理进行项目管理的工作，如协调资源、分配任务、跟踪进度、收集反馈信息等。
- 项目顾问：项目顾问的职责是向客户提供项目管理咨询和建议，为企业寻找适合项目的经理人才，并帮助项目启动和推进。

## 2.2 项目计划
项目计划是项目管理的一个关键环节，它是指对计划期间内将要进行的所有活动以及项目的资源、计划、约束条件等进行明确定义，并做好相应的安排。

### 2.2.1 项目计划的作用
项目计划的作用是确保项目工作量、质量、进度、范围、资源、计划、约束条件等符合企业的战略方向、计划、约束条件，从而保证项目顺利进行。

项目计划能使项目的执行更加规范化、科学化，从而减少风险和浪费，更好地满足客户的需要。

### 2.2.2 项目计划的类型
根据项目的大小、复杂程度、范围、分工、频率以及资源投入的大小，项目计划可以分为如下三种：
- 短期项目计划：对短期内的项目进行计划，计划内容一般包括项目名称、重要日程、主要工作、资源列表、计划工期、依赖资源、目标成果。
- 中长期项目计划：对中期、长期的项目进行计划，计划内容一般包括计划概览、资源配置、进度安排、计划资源、风险管理、沟通协调、竞品分析等。
- 变更项目计划：对于那些原先存在的项目进行变更，需要调整、增加或取消一些计划项。

### 2.2.3 项目计划的步骤
项目计划的步骤一般包括以下几步：
- 项目准备阶段：包括收集和整理项目的背景信息、需求分析、商业模式、项目管理计划、预算预算核算、风险管理、发布会等。
- 项目计划阶段：包括制定项目计划，包括确定资源需求、安排项目任务、规划进度、监督检查、控制风险、追踪成果等。
- 项目评审阶段：包括对项目的计划进行评审，并依据评审结果制定后续的修正。
- 项目实施阶段：包括执行项目的具体步骤，包括各个组成部分的协调、人员分工、进度安排、风险处理、质量保证、沟通协调、回顾总结、交流反馈等。
- 项目完结阶段：包括项目收尾阶段，包括总结、统计、分析、监督、复盘等。

## 2.3 项目管理工具
项目管理工具是项目管理过程中使用的一系列工具、方法、模型及标准，帮助项目经理更有效、准确地管理项目。

### 2.3.1 项目管理工具分类
根据项目管理工具的用途、功能和目的，可以把项目管理工具分为五类：
- 会议工具：如会议室预定、会议通知、会议纪要、会议记录、会议视频、会议直播、网络会议等。
- 任务管理工具：如待办事项、任务管理、工作报告、进度跟踪、工作日志、时间管理等。
- 文件管理工具：如文件共享、文件分类、文件存储、文件备份、文档管理等。
- 通信工具：如即时通信、语音聊天、电子邮件、打卡签到、文档分享等。
- 统计工具：如统计分析、项目统计、资源分析、项目经理绩效评估等。

### 2.3.2 项目管理常用的软件
常用的项目管理软件有：
- Wrike：Wrike是美国一家科技企业为项目管理、团队协作和商业计划等提供了软件服务。
- Trello：Trello是一个基于Web的项目管理软件，可以管理看板、任务、日程、文件、计划等。
- Asana：Asana是一个项目管理软件，提供工作空间、项目管理、报告和反馈，同时还支持团队管理、沟通协作、安全性、合规性、数据分析等功能。
- Jira：Jira是一款开源的项目管理软件，支持敏捷开发和Scrum方式的项目管理，可用来跟踪bug、缺陷、任务等。
