软件架构就是指软件的整体结构、功能组成和设计模式。它由人工、过程化和计算机技术共同驱动，其目的是为了解决复杂性问题、提高效率和可靠性。软件架构的定义和作用一直深受软件开发人员的追求。如今，随着互联网、移动互联网、云计算、物联网等技术的普及，以及软件工程方法论的不断更新，软件架构的需求也越来越大。

# 2.核心概念与联系
## 1.什么是设计模式？
“软件设计模式”（Software Design Pattern）是一个用于面向对象软件设计中创建可重用的设计蓝图的总称。它是一套被反复使用、多数人知晓的、经过分类编目的、代码设计经验的总结。不同的设计模式有不同的适应场景、解决特定问题的方式、权衡利弊的方法、优缺点，以及实现模式的一般标准和结构。以下简要介绍一些常见的软件设计模式。
### 创建型设计模式
- 单例模式（Singleton Pattern）
- 工厂模式（Factory Pattern）
- 抽象工厂模式（Abstract Factory Pattern）
- 生成器模式（Builder Pattern）
- 原型模式（Prototype Pattern）
### 结构型设计模式
- 代理模式（Proxy Pattern）
- 桥接模式（Bridge Pattern）
- 组合模式（Composite Pattern）
- 装饰器模式（Decorator Pattern）
- 外观模式（Facade Pattern）
- 享元模式（Flyweight Pattern）
### 行为型设计模式
- 命令模式（Command Pattern）
- 解释器模式（Interpreter Pattern）
- 迭代器模式（Iterator Pattern）
- 中介者模式（Mediator Pattern）
- 备忘录模式（Memento Pattern）
- 观察者模式（Observer Pattern）
- 状态模式（State Pattern）
- 策略模式（Strategy Pattern）
- 模板方法模式（Template Method Pattern）
- 访问者模式（Visitor Pattern）

## 3.设计模式的六大原则
1. Single Responsibility Principle (SRP): 单一职责原则(Single responsibility principle)又叫作单一职责准则(Single reponsibility criteria)，指一个类应该只有一个引起它的变化原因。换句话说，一个类只负责完成自己的职责范围内的事情。单一职责使得类的实现，变得简单、易于理解和修改。

2. Open Close Principle (OCP): 开闭原则(Open-Closed principle)表示一个软件实体应该对扩展是开放的，而对修改是封闭的。即软件实体应尽量在不改变原有代码的情况下进行扩展。

3. Liskov Substitution Principle (LSP): 里氏替换原则(Liskov substitution principle)要求子类必须能够替换它们的基类。换言之，所有引用基类的地方必须能透明地使用子类对象。

4. Interface Segregation Principle (ISP): 接口隔离原则(Interface segregation principle)则要求客户端依赖的接口尽可能少，同时应该保证每个接口中的方法尽可能简单。

5. Dependency Inversion Principle (DIP): 依赖倒置原则(Dependency inversion principle)则认为高层模块不应该依赖低层模块，二者都应该依赖抽象。抽象不应该依赖细节，细节应该依赖抽象。

6. DRY (Don't Repeat Yourself) 原则: DRY原则意味着不要重复你自己编写的代码，相反，应该关注代码重用，保持代码的一致性。
## 4.设计模式的六种类型
1. Creational Pattern: 创造型模式，创建对象的模式，包括单例模式、工厂模式、抽象工厂模式、建造者模式、原型模式。

2. Structural Pattern: 结构型模式，处理对象的结构模式，包括代理模式、桥接模式、组合模式、装饰器模式、外观模式、享元模式。

3. Behavioral Pattern: 行为型模式，描述对象之间职责划分的模式，包括命令模式、解释器模式、迭代器模式、中介者模式、备忘录模式、观察者模式、状态模式、策略模式、模板方法模式、访问者模式。

4. Concurrency Pattern: 并发型模式，描述线程安全、同步、异步编程的模式。

5. Idiomatic Pattern: 习惯用法模式，反映了软件设计过程中的特有语言或语法。

6. Metaphorical Pattern: 隐喻模式，以某些看似抽象的实体为代表，展示一些软件设计过程中抽象现实世界的映射方式。
