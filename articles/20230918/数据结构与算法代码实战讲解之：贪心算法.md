# 1.简介
  

贪心算法（又称贪婪算法）是一种简单的最优选择算法，它在对某种最优化问题进行求解时采用局部最优策略。在贪心算法中，每一步都做出当前看起来最好的选择，也就是所谓的贪心法则。贪心算法可以帮助我们解决一些规模较小的问题，特别是在计算时间、资源利用率或其他方面受到限制的情况下。但是，对于复杂的或规模很大的问题，贪心算法往往并不能得到最优解。因此，贪心算法常常被用作辅助算法或者启发式算法。

# 一、贪心算法概述及其特点
## 1.1 什么是贪心算法？
贪心算法(Greedy algorithm)也叫做贪婪算法，是在对满足约束条件的最优化问题进行求解时采用局部最优策略的一类算法。贪心算法通常由两个要素组成：
- 贪心策略：以全局观察的方式考虑各个子问题的最优解，以找出一个全局最优解。
- 回溯法：使用局部最优解再递归地寻找全局最优解。

贪心算法基于这样的思路：每次都按照最优的方式做出选择，使得全局结果达到最优。换句话说，贪心算法希望找到一个局部最优解，然后把这个局部最优解带入到下一次迭代，直到得到全局最优解。 

贪心算法的特点：
- 贪心算法只适用于最优化问题，且最优子结构性质十分明显。
- 在最优化问题中，贪心算法可以产生全局最优解，但不一定保证这一解一定是最优的。
- 对一般的非最优化问题，贪心算法可能需要多次迭代才能得到最优解。
- 由于贪心算法依赖于局部最优，所以经常会遇到“盲目”行为。

贪心算法的主要应用场景：
- 对于求解单调函数最大值问题(如求解股票问题)。
- 用于生成最优解的数独求解问题。
- 用于求解哈夫曼编码问题。
- 在旅行商问题中，贪心算法可以帮助寻找一个全局最优解，而无需枚举所有情况。
- 用于网络流问题，如最小割模型(minimum cut problem)。

## 1.2 贪心算法的几何意义
贪心算法的名字起源于其对森林的最佳收获的定义——贪婪的生长方式。当我们说某种事物是“贪婪”的时候，就是指这种事物具有一定的生长规律，并且为了获得更多的产出，我们宁愿牺牲自己的一些优势或保全自己的一些劣势。贪心算法以“贪婪”的方式搜索问题空间，从而产生最优解。例如，假设有一个长方形平面上有若干条边，我们想在这个平面上画出一个矩形，使得这些边上的线段都恰好出现在矩形内部，并使得矩形面积最大。如果我们每次都把一条边取出，那么最终得到的矩形只能是正方形；如果我们每次都以矩形左上角为参照，在上下左右的方向上都取尽可能大的长度，那么最终得到的矩形可能会有非常奇怪的形状。但是，如果我们以每一次选择后都要求得到某个比例的面积，比如70%，那么就可以保证得到的矩形面积至少为70%。而贪心算法则是以类似的方式，在每一步选择后都计算一下新得到的形状的面积，并且每次都选择使得面积增加最多的那一步。 

## 1.3 贪心算法的历史演变
贪心算法可追溯到19世纪50年代，当时用来求解组合优化问题的人工智能领域刚刚兴起。1964年图灵奖获得者戴维森·韦恩发现了贪心算法。20世纪60年代，贪心算法成为公认的研究热点，并推广开来。最早应用于智能电路设计的贪心算法发表于1967年。20世纪70年代，贪心算法已然成为优化问题的关键算法。贪心算法已成为数理最优化方法的一个重要组成部分。

目前，贪心算法已经成为科技发展的必备工具。许多复杂的求解问题都可以使用贪心算法解决。在信息安全、网络流量分配等方面，贪心算法是计算资源有限的情况下，求解最优解的有效办法。
