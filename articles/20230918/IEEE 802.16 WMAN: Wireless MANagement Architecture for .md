# 1.简介
  

Wireless MANagement Architecture (WMAN) 是IEEE 802.16系列标准的一部分，它定义了无线网络管理、控制和数据采集系统。WMAN的设计目标是在UWB（Ultra-Wideband）频段上的全球性部署，使得中小型移动设备可以更有效地连接到无线网络，并享受到最佳性能。其主要功能包括：

1. 管理网络；
2. 无线主机控制；
3. 数据收集；
4. 安全性和隐私保护。

WMAN目前已被多个制造商和标准组织采用。在中国，华为、中国电信、中国联通等制造商、运营商、服务提供商已经在温州、上海、杭州、广州、天津等地开展了试点实验。然而，由于各个厂商各自实现方式的差异，同时标准的更新还不断增加新功能，因此在国际上推动 WMAN 的部署仍面临着很大的障碍。本文将从历史、现状、标准化、关键技术、应用场景、研究进展、未来规划五方面对 WMAN 的最新发展进行详细阐述。
# 2.基本概念术语说明
## 2.1 移动计算、移动宽带(UWB)、WI-FI
移动计算、移动宽带和WI-FI都是近几年热门的词汇。其中，移动宽带(UWB)是一种全新的无线通信技术，能够提供比传统的WI-FI(无限局域网)信号强度高很多的比特率。移动计算则是指通过无线通信技术的网络传输和处理能力对智能手机和平板电脑提供支持。
UWB频段一般采用长距离直线衰减(LDLS)的方式衰减，所占用的带宽也达到了数百MHz，这样就可以用于具有移动计算功能的终端设备。但同时，UWB技术本身也存在着一些缺陷，比如不能保证可靠传输和时延抖动，同时信号损耗也比较严重，所以目前还无法直接用于作为日常生活中的通讯工具。
WI-FI也是指无线局域网技术，早期它是由英特尔、高通、微软、诺基亚、博通等公司共同开发出来的，现在它也是由联盛德哥伦布大学、芬兰赫尔辛基大学、西班牙安特卫普理工大学、英国剑桥大学、香港中文大学等多所著名大学联合开发出来的。WI-FI能够提供数百Mbps的速率和低延迟，但是它的频谱资源却非常有限，主要用于厨房、居家、办公室等场景。

## 2.3 WMAN 组成
WMAN 由两个子层组成——Management Sublayer 和 Control Sublayer。
### Management Sublayer
Management Sublayer 又称为 MAC Sublayer，位于 802.16 Data Link Layer 下面，管理控制无线网络的各项服务，如连接建立、QoS 调度、链路质量监测、安全策略和认证、数据收集等。

#### WMAN 基本功能
管理子层中，WMAN 提供了以下基本功能：

1. 管理网络；
2. 无线主机控制；
3. 数据收集；
4. 安全性和隐私保护。
### Control Sublayer
Control Sublayer 又称为 PHY Sublayer，它位于 MAC Sublayer 上，负责上行无线信道的调控、同步、传输过程的错误检测和纠正。
#### PHY Sublayer 基本功能
PHY Sublayer 中的 PHY(Physical Layer) 模块负责无线信道的管理和调控，包括信道的选择、分配、调节、开关机、统计等。PHY Sublayer 提供了以下基本功能：

1. 时延抖动免疫：时延抖动是无线传输中常见的问题，其原因在于无线信道的传输速度和电磁波在频谱上的传播速度不匹配导致的，这种不匹配又引起了接收端的解调延迟。WMM 协议旨在通过设置不同的发送策略、设置过载的状态、降低传输速率来减少时延抖动，同时也提升信噪比。
2. 可靠传输：采用 RTS/CTS 或轮询模式发送数据帧，接收方确认后才认为数据已成功送达，确保了数据在传输过程中完整且不丢失。
3. 负载均衡：当多个设备同时请求无线信道时，需要根据队列长度或数据类型对请求者进行排序，使得具有较高优先级的请求获得共享信道资源，提高无线通信效率。
4. QoS 服务质量保证：QoS（Quality of Service）是 IEEE 802.16 中引入的机制，用于确保不同类别的应用或服务在网络上获得适当的服务质量。QoS 可以让网络资源得到更好的利用，避免某些应用程序因竞争无线信道而影响其他应用。QoS 可以提供以下几种服务质量保证：
    - 1、无差错传输（Best Effort Delivery）：对于不可预知的突发事件，即使发生了错误也不会有异常现象出现。例如，某条视频流的某一帧丢包，这个包只会影响该视频流的一个帧，其他视频流的其它帧依然可以正常播放。
    - 2、优先级高的帧（Time-Sensitive Traffic）：某些时间敏感型应用程序要求在特定时间段内可靠交付，例如金融交易、视频监控、视频会议等。QoS 可以通过优先级机制、调度机制或流控机制实现这些要求。
    - 3、保证最小的时延（Minimum Latency Guarantee）：QoS 可以确定某一数据包的最小时延，从而为该数据包分配更多的资源以保证可靠交付。
    - 4、实时性（Real Time Applications）：QoS 可以确保实时应用的实时性，例如语音、视频聊天等。QoS 还可以动态调整传输参数，确保应用的响应时间、吞吐量、可用性或数据一致性。
