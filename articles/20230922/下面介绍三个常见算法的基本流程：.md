# 1.简介
  

在计算机科学中，许多问题都可以用数值计算的方法进行求解。数值计算方法有很多种，包括线性代数、随机算法、搜索算法、动态规划等。这些算法有不同的特点和应用场景，本文将主要介绍以下三个常用的算法：

1. 贪婪算法（Greedy Algorithm）；
2. 分支定界法（Branch and Bound）；
3. 暴力破解法（Brute-Force）。

# 1.贪婪算法
贪婪算法是一种简单有效的解决问题的算法。其基本思想是对当前情况选择最优而没有考虑全局最优，因此得到的结果可能不是最优的全局最优解。但它可以快速找到可行解，并且得到解的近似解，较好的收敛速度。通常来说，贪婪算法对于求解线性规划问题比较有效。如求最大流最小割问题（Maximal flow problem in minimum cut）和最小生成树问题（Minimum spanning tree problem）。

## 1.1 基本概念
### 1.1.1 决策变量（Decision variables）
一个问题一般有一些待确定的决策变量。例如，在贪婪算法中，待确定的是一组活动(Activity)，需要从中选出一组活动，使得完成所有活动所需花费的最少时间或最低成本。在这种情况下，活动可以是一个任务、班级作业、实验、项目等。

### 1.1.2 个体（Individuals/Solutions）
一组已确定的决策变量称为一个个体。例如，在贪婪算法中，表示活动的一个个体即为一个组合方案。其中包含了活动的时间安排，每个活动是否进行，每项活动对应的工作量等信息。

### 1.1.3 策略（Strategies）
策略是指一个个体采取的动作集合。换句话说，就是给每个决策变量赋予的值。在贪婪算法中，策略就是每个个体采取的活动集合。例如，一个个体的策略可能为{A,C}，表示他首先考虑选择活动A，然后考虑选择活动C。

### 1.1.4 对象函数（Objective function）
对象函数又称目标函数，用来衡量某个个体或问题的好坏。在贪婪算法中，对象函数通常定义为完成所有活动所需的总时间或总成本。

## 1.2 基本算法步骤
### （1）输入阶段
输入阶段包括两个部分。第一部分是问题描述，包括目标函数、决策变量、约束条件等。第二部分是问题的初始解。

### （2）初始化阶段
初始化阶段就是设置一些参数，比如搜索树的高度限制、迭代次数等。

### （3）搜索阶段
搜索阶段通过执行搜索启发式算法，寻找满足约束条件且目标函数达到最优的解。常用的搜索启发式算法有广度优先搜索、深度优先搜索和最优优先搜索等。

### （4）选择阶段
选择阶段就是从搜索树中选择一条希望朝着更好的方向发展的路径。在贪婪算法中，选择最优子结构（Best Substructure），也就是选择具有最佳性能的局部最优解。

### （5）继续搜索阶段
继续搜索阶段往往会产生新解，需要重新搜索子节点。

### （6）输出阶段
输出阶段显示找到的最优解，并对其进行分析。

## 1.3 实例：求最大流最小割问题的贪婪算法
### 问题描述
假设有一个加权无向图G=(V,E),其中V为顶点集，E为边集，边上带有非负的权重w(u,v)。在这个图中，存在着一系列源点和汇点，分别为s和t。要确定在任意边不流通的条件下，从源点s流出的最大流量及流出的最小割。这个问题被称为最大流最小割问题。

### 算法步骤
1. 初始化：令F=0，E'=E，C=0。

2. 从E'中选取一条费用最小的边e∈E'(u,v)加入增广路（augmenting path）A，并按比例减少流量f，即：
   F+=f
   u->v: f=C/(out[v]-in[u])*(c[u][v]-(c[v][u]+f))
      C-=c[u][v]+f
    v->u: c[v][u]+f=c[v][u]-f+c[u][v]
     E'.remove((u,v))
    A.append((u,v))

3. 若E'为空，则结束；否则回到第2步。

4. 对流出的流量F，按边的容量对其归纳，即：
   for each (u,v) ∈ A:
        c[u][v]+=F*min(c[u][v],f)/c[u][v]

### 复杂度分析
由于贪心策略每次只考虑一个增广路，所以运行时间与流量有关。当初始流量很小时，贪心策略是高效的；当初始流量很大时，可能需要反复搜索才能找到最优解。因此，为了找到全局最优解，可以采用并行算法或分治算法。
