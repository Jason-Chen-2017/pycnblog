# 1.简介
  

HTTP（HyperText Transfer Protocol）即超文本传输协议，是一个用于从万维网上超媒体信息资源进行交换的协议，它使得 World Wide Web 上的数据都成为可能。HTTP 是由 HTTP 请求方法、状态码和 URL 组成的。这些元素共同定义了客户端和服务器端之间通信的规则和规范。本文档将对HTTP方法和状态码进行详细阐述。


# 2.基本概念术语说明
## 2.1 HTTP请求方法(Methods)
HTTP请求方法通常包括以下几种：
- GET：获取资源。GET 方法用来请求访问被标识的资源。
- POST：提交数据。POST 方法用来向服务器发送新数据。
- PUT：更新资源。PUT 方法用来上传文件或其他类型的内容到指定的资源地址。
- DELETE：删除资源。DELETE 方法用来删除服务器上的文件。
- HEAD：获取响应头部。HEAD 方法用来请求服务器返回指定资源的响应消息。
- OPTIONS：查看服务器支持的方法。OPTIONS 方法用来请求服务器返回服务器所支持的请求方法。

## 2.2 HTTP状态码(Status Codes)
HTTP状态码是指网页服务器根据客户端请求的处理结果，给客户端返回的一个数字代码。状态码的描述如下表所示：
| 状态码 | 描述                              |
|--------|-----------------------------------|
| 1XX    | 消息：表示请求已接收，继续处理      |
| 2XX    | 成功：表示请求已经成功接收、理解并接受 |
| 3XX    | 重定向：要完成请求必须进行更进一步的操作      |
| 4XX    | 客户端错误：请求包含语法错误或者请求无法实现   |
| 5XX    | 服务器错误：服务器在处理请求时发生内部错误        |

# 4.具体代码实例和解释说明
可以举个例子说明一下常用请求方法GET和状态码200的意义。

例如：
当用户输入网址http://www.baidu.com，然后按下回车，浏览器首先会向DNS服务器请求解析出IP地址。得到IP地址之后，就会打开一个TCP连接，向服务器发送一个GET请求。通过请求的方式传达想要获得什么资源。比如说“/index.html”代表用户想获得首页文件。

服务器收到请求后，会查找对应的资源，并将其返回给浏览器。比如说如果用户输入的是“/index.html”，那么服务器就会找到本地磁盘中的对应文件并将之发送给浏览器。

浏览器收到资源后，开始渲染页面，显示出网页的内容。但是，此时还没有完全结束，因为服务器并不知道用户是否已经看到完整的网页内容。所以，浏览器会向服务器再次发送请求。由于之前已经打开了TCP连接，浏览器就可以利用这个连接继续发出请求。只是这一次，服务器告诉浏览器“继续”。也就是说，只要服务器还没完全回复前一次请求，浏览器都不会停止发送请求。

最后，服务器把完整的网页内容送回给浏览器。浏览器开始解析HTML源码，并显示出网页。用户就可以在浏览网页的同时，继续发送HTTP请求获取更多的信息。

GET方法的含义：服务器只能通过URL获取资源，而不能通过BODY传递参数。

为什么需要GET？因为GET请求不应当产生副作用，不应该修改服务器上的任何东西。另外，GET请求比POST请求更安全。比如说，密码等敏感信息都可以在URL中看到。
