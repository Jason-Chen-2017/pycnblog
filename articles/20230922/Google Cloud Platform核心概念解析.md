# 1.简介
  

Google Cloud Platform（GCP）是一个基于Google开源云计算服务的一站式平台，通过提供各类基础设施、API接口、应用引擎、机器学习和其他工具等服务帮助企业快速构建和部署应用程序，降低IT成本、提升开发效率和运营效益。从功能上来说，GCP分为四大组件：
- GCP Compute：提供了运行虚拟机的能力，支持包括VMware、Amazon EC2、Azure VM在内的多种云服务商提供的虚拟化方案；
- GCP Storage：提供统一且高性能的数据存储、分析和检索能力；
- GCP Big Data & Machine Learning：提供了用于处理海量数据和进行机器学习的大数据解决方案；
- GCP Networking：提供了网络连接、负载均衡和安全防护的基础设施。
除了以上四大核心组件外，GCP还提供了很多其他服务，如容器编排、消息队列、数据库等，这些服务让用户可以快速的开发应用，并且具有可扩展性、高可用性、低延迟等特点，适合不同类型的业务场景。
# 2.核心概念术语说明
## 2.1.虚拟机实例(Instance)
实例(Instance)，是指在GCP中创建的虚拟服务器。一般情况下，每个GCP账户下最少会创建一个实例。实例包含一个或多个虚拟磁盘(Disk)，一个系统盘(System Disk)，以及一系列配置选项(如CPU核数、内存大小、网络带宽等)。实例可运行各种不同的操作系统，包括Linux、Windows Server等，并且可安装各种软件，比如MySQL、Apache、MongoDB等。实例的生命周期管理是通过GCP API或者命令行工具完成的。
## 2.2.磁盘(Disks)
磁盘(Disks)，是指GCP中的存储设备。包括两种类型：标准型SSD(Solid State Drive)硬盘和本地SSD硬盘。两种硬盘都可以在实例的生命周期内被动态添加和删除。每块硬盘最大可以存放16TB数据。在GCP中，所有硬盘都是高度可用的，由自动修复保护。因此，即使硬盘损坏，也可以在几秒钟内恢复正常。除此之外，GCP还提供备份和冗余机制，避免数据丢失。
## 2.3.区域和可用区(Regions and Zones)
区域(Region)是GCP上的逻辑隔离单元，每一个区域都由一个或多个物理的数据中心组成。GCP主要面向北美、欧洲、亚太地区和非洲等地区。每个区域都有自己的独立电力、水力和网络。区域可以有多个可用区(Zone)。可用区是在相同区域的一个子集，通常包含两个或更多的数据中心。可用区之间的距离取决于地理位置。可用区的划分允许集群跨越多个可用区，实现更高的可用性。
## 2.4.网络(Networks)
网络(Networks)，是指GCP中的逻辑网络结构，用来将虚拟机连接起来。GCP提供了四种网络：默认网络、自定义网络、VPC网络和Peering网络。其中，默认网络是每个项目都有一个默认网络，所有的实例都自动分配到这个网络中。自定义网络类似传统意义上的网络，用户可以使用IP地址范围、子网掩码、DNS服务器和路由表配置自定义网络。VPC网络是在指定范围内创建的私有网络，用户可以控制整个网络，例如设置子网、VLAN、路由策略等。Peering网络是一种灵活的方式，通过VPC间的连通性来建立跨VPC的连接。
## 2.5.路由器(Routers)
路由器(Routers)，是指在GCP中用来实现网络流量转发的设备。每个VPC都会配有一个默认路由器，该路由器与该VPC关联。默认路由器具备NAT、VPN、路由表等功能。用户可以通过配置路由规则来修改默认路由器的行为。除此之外，用户还可以创建自定义路由器并配置路由规则，实现更复杂的网络功能。
## 2.6.负载均衡(Load Balancing)
负载均衡(Load Balancing)，是指将网络流量分布到多台主机或云函数上的技术。负载均衡包括三种模式：静态模式、动态模式和组合模式。静态模式下，流量只发送给固定的一组目标。动态模式下，根据当前的负载情况动态调整流量的调度方式。组合模式下，可以同时使用静态和动态的负载均衡策略。GCP提供七层和四层负载均衡产品，以及HTTP/HTTPS负载均衡。
## 2.7.镜像(Images)
镜像(Images)，是指GCP中的预先配置好的操作系统模板。用户可以根据需要制作自己的镜像，也可以选择已经发布的公共镜像。当实例启动时，它将从镜像中读取启动文件，然后开始执行。用户也可以在某个实例上保存当前的状态作为新的自定义映像。
## 2.8.存储桶(Buckets)
存储桶(Buckets)，是指在GCP中用于存放对象存储数据的地方。存储桶提供了简单、经济、弹性的云端存储服务，用户可以自由地上传、下载、存储和访问数据。存储桶可以根据需要划分为不同的级别。公共存储桶可以供任何人免费访问，而私有存储桶则需要付费。公共存储桶的命名空间是全局唯一的，因此不能重复。
## 2.9.Cloud Functions
Cloud Functions，也称为serverless functions，是一种按需运行的代码片段。它无需预先配置或管理服务器，只要提供一些触发事件，就能够响应并执行特定任务。由于它是按需运行，所以它非常适合处理不频繁、计算需求不大的工作负载。它可以直接处理HTTP请求、事件通知、数据变化等。
## 2.10.容器集群(Container Clusters)
容器集群(Container Clusters)，是运行容器化应用的集群环境。容器集群由多个节点组成，可以提供弹性伸缩、高可用性和可靠性。容器集群可以部署多种服务，包括负载均衡、日志记录、监控告警、审计、备份等。容器集群可以运行各种容器化应用，包括基于Docker的应用、Kubernetes应用等。容器集群也可以与其他GCP产品结合，比如Cloud SQL、Bigtable、PubSub等。
## 2.11.云SQL(Cloud SQL)
Cloud SQL，是Google Cloud Platform提供的关系型数据库服务。它支持MySQL、PostgreSQL、SQL Server等主流关系型数据库。它提供完全托管、按需计费的数据库服务，帮助用户节省时间和资源，实现快速、便捷的数据库部署及扩展。Cloud SQL支持全球数据中心的部署，具备高可用性、安全性、性能优秀、自动备份恢复等特性。
## 2.12.BigQuery
BigQuery，是Google Cloud Platform提供的海量数据分析工具。它支持超过1PB的高容量数据集，并且具有快速查询、存储、和分析数据的能力。BigQuery的独特之处在于其自带的分析引擎，能够实时快速的分析海量数据。BigQuery可以很容易地集成到其他GPC产品中，包括数据仓库、机器学习、数据分析等。
## 2.13.PubSub
PubSub，是Google Cloud Platform提供的基于主题的消息传递服务。它支持消息发布与订阅模型，允许分布式系统异步通信。PubSub通过主题把消息分类，同样的消息会被发送到订阅了该主题的所有消费者。PubSub支持持久化存储，确保消息不会因为消费者失败而丢失。
## 2.14.IAM(Identity and Access Management)
IAM(Identity and Access Management)，是GCP提供的身份和访问管理服务。它允许用户管理权限，管理谁有权访问哪些资源，并对访问进行控制。IAM使用角色和权限控制访问，用户可以授予其角色所对应的权限，控制他们能做什么、不能做什么。用户可以创建自定义角色，并赋予自己希望使用的权限。IAM提供了细粒度的访问控制，并提供审计跟踪功能，帮助用户追溯发生了什么事情，提高安全性。
## 2.15.负载测试工具(Load Testing Tools)
负载测试工具(Load Testing Tools)，是用于模拟用户并生成高负载的工具。在生产环境中运行这些测试工具可以发现潜在的问题，改善系统的可靠性和性能。G Suite提供了Web Load Test、App Engine Flexible Load Test等多种负载测试工具。
## 2.16.API Gateway
API Gateway，是用于管理API的服务。它可以将许多不同的服务聚合到一起，并通过单个API网关接口暴露出来。通过API网关，可以对内部服务的访问进行统一管理，实现API的安全、认证和计费功能。API网关还可以缓存数据、过滤请求、加速流量，帮助提升API的性能。
