# 1.简介
  

“服务治理”是指企业在运营过程中，围绕业务需求，制定、执行和管理各项服务，包括流程服务、平台服务、工具服务、知识服务、人才服务、组织服务、信息服务等。“架构变革”是指通过技术创新或产品升级，提升公司的整体能力，优化公司的运营效率，改善服务质量，降低成本和风险，从而更好地为客户提供价值并赢得竞争优势。由于服务治理和架构变革具有颠覆性、复杂性和长远视野，因此，在最近几年中，越来越多的企业都尝试在服务治理和架构变革领域寻找突破口，寻求新思路和新理念。

此次分享的内容基于服务治理和架构变革的理论研究，结合实践经验，从多个维度探讨当前服务治理和架构变革领域的热点问题及关键技术，以及未来的发展方向。主要内容如下：

1.1 服务治理简史
服务治理的发展历史可以概括为两个阶段：第一阶段是物理服务治理时期，主要依靠人力、物力、财力投入实现服务的集约化；第二阶段则是数字服务治理时期，由于电子商务的发展和人工智能的应用，服务治理面临的新 challenges 日益突出，特别是对于中小型企业来说，如何将传统的面对面的沟通方式，转变为面对面的协作协同，实现真正意义上的全方位服务。

基于以上两阶段的发展历程，服务治理的发展趋势大致可以总结为：

1）物理服务治理向数字服务治理演进。随着互联网的普及，物理服务治理逐渐被数字服务治理所取代，以“移动互联网+”的方式运营，使得各种服务、活动、服务评估、团队管理、工具培训等也都成为一种数字服务，通过数字化的手段，实现服务的多样化、便捷化、精准化。

2）从“物业管理”向“企业数字化”转型。为了让客户得到最好的服务，“物业管理”不得不逐步淡出历史舞台，转型为“企业数字化”，通过数字化技术实现服务的自动化、标准化、智能化、个性化，为企业打造出一个数字化的服务生态系统。

3）面对新形势，服务治理应该充满活力和创新。服务治理是一个巨大的创新工程，需要持续不断的探索和开发，以满足需求的不断变化和客户的不断期待。

1.2 服务治理理论简介
服务治理理论是指对服务治理的社会科学研究，通过分析人类服务行为的规律，将其理论化、规范化，并提出可操作的解决方案，探索如何最大化服务效率，降低服务成本，促进服务的更加公平、有效和美好。它把服务管理分成三个层面：组织层面、资源层面和规则层面。

组织层面重点关注服务组织结构，包括服务提供者（如服务团队、服务提供商、服务组织），客户群体（客户角色、客户群体构成、客户群体理解），服务流程（服务过程、服务模型、服务流程），服务环境（服务场所、服务空间、服务条件）。资源层面重点关注服务人员（服务骨干、服务专家、服务助手），服务资源（服务资产、服务能力、服务费用），服务信息（服务知识、服务数据、服务信息）。规则层面重点关注服务准则（服务水平标准、服务品质标准、服务目标），服务法则（服务道德、服务规范、服务惯例）。

服务治理理论的产生对服务治理的发展至关重要，因为服务治理理论为服务的规划、设计、实施提供了一种客观、科学的视角，可以帮助企业高效地管理、发展和提升服务，避免走向极端，推动社会经济的进步和增长。

1.3 服务治理方法论简介
服务治理方法论是指基于对服务治理理论的理解，提出的一系列可操作的方法和工具，以实现服务的合理有效，从而有效地实现服务提供者与消费者之间的关系。方法论的核心是“基于目标导向”和“循序渐进”的原则。

基于目标导向的理念认为，服务治理的目标是确保服务生产者能够以最佳的方式为消费者提供服务，从而达到服务的完美。因此，服务治理方法论的首要任务是为服务管理制定明确的目标，并遵循科学的技术路线图、政策措施和工作方式，将目标转换为行动。

循序渐进的原则认为，任何成功的服务治理项目都是一条慢、曲折的道路，必须根据实际情况和目标进行迭代，逐步完善服务治理的方案。因此，服务治理方法论的第二个核心原则是循序渐进，先有策略再有计划，先有专业人员再有服务实施，先有工具再有服务模式，以尽可能小的投入获得最大的效果。

服务治理方法论是一种知识化的理论，是服务治理行业的理论基础，也是对服务治理的科学研究的基础。它有助于企业搭建起服务治理的理论框架，提升服务治理的整体水平和效率。

# 2.基本概念术语说明
## 2.1 服务网络服务治理
服务网络服务治理 (Service Network Management, SNM) 是指利用计算机网络技术和服务虚拟化技术实现企业内部、外部服务的自动化和协同，促进业务流程的优化、服务的一致性和价值的共享。服务网络服务治理的特点是：

1．功能整合
服务网络服务治理涉及许多不同的模块和服务，如何将这些服务整合到一起，并提供给用户，就显得尤为重要。例如，内部的服务系统如何与外部的服务系统相连接？内部的资源如何分配？业务流程是否可以自动化？供应商之间的合作是否可以协调？等等。

2．智能协同
服务网络服务治理的关键是将内部和外部的各种服务系统进行连接，在满足用户的不同需求时，提供适当的服务。因此，服务网络服务治理需要构建智能协同的机制，能自动发现、识别和发现新的服务并将它们添加到服务网络中。

3．价值共享
服务网络服务治理将服务提供者的服务和价值最大化，所以在服务交付上，服务网络服务治理必须保障服务的一致性和价值的共享。例如，如果用户的需求发生了变化，服务网络服务治理应该如何调整服务呢？服务之间的协调和契约如何保证服务的质量和效率？提供的服务是否符合用户的预期？

4．服务模型
服务网络服务治理在服务之间建立契约和协议，并将之作为基本的服务模型，由此构建服务的开放性、透明性和可复制性，促进服务的自动化和协同。例如，如何定义服务的属性、功能、范围和收费？服务的支持机构、法律条款、政策要求等如何体现？

## 2.2 服务场景服务治理
服务场景服务治理 (Service Scenario Management, SCM) 是指根据客户真实需求、消费习惯、商业模式、竞争对手、上下游关系等因素，为企业提供一系列面向特定消费群体或特定商业模式的服务。服务场景服务治理的特点是：

1．价值主张
服务场景服务治理考虑的是企业的独特利益，在服务场景中选择最有影响力的用户和商业模式，并为他们提供优质、高效的服务。服务场景服务治理理论认为，服务场景服务治理的目的是实现客户价值的最大化，因此必须将个人的需要放在第一位，考虑其需求、偏好、理解和需求，并在服务场景服务中注重与客户实际需求相关的差异化服务。

2．场景发展
服务场景服务治理涉及多个服务场景，需要不断迭代和完善，确保每个场景都提供有效、贴近用户的服务。此外，还需要考虑相关的商业模式和上下游关系，以确保服务的连续性和完整性。

3．服务品牌
服务场景服务治理对每种服务场景都会有一个品牌，品牌的创建需要充分关注服务场景的特点和客户群，以定位和塑造企业的服务形象。

4．服务动态
服务场景服务治理的每种服务场景都有自己的生命周期，在生命周期内，会呈现发展和变化的状态。服务场景服务治理需要时刻跟踪和预测客户需求，以快速响应客户的需求变迁，满足客户的需求。
