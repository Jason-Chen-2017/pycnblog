# 2.MySQL数据库概述及关键特性介绍
## 2.1 什么是MySQL？
MySQL是一个开源的关系数据库管理系统，由瑞典iaDB公司开发，属于关系数据库管理系统（RDBMS）范畴。MySQL使用C/S架构模式，客户端程序和服务器端程序均使用C语言编写，完全跨平台，因此可以在各种操作系统上运行。MySQL使用标准的SQL（结构化查询语言）作为接口语言，提供了完善的事务处理、 SQL 、存储过程等功能，支持多种编程语言和平台，支持高可用性、自动备份、灾难恢复等高级特性，适用于各种规模的Web应用、数据仓库和互联网企业。截止目前，MySQL已经成为开源界最热门的关系数据库管理系统，拥有超过十亿条记录的海量数据存储空间，拥有众多知名公司和组织的广泛支持，被世界各地的银行、保险、电信运营商、零售、科研机构、医疗机构等所采用。
## 2.2 MySQL的主要特点
### 2.2.1 MySQL与Oracle对比
MySQL和Oracle是两种主流的关系型数据库管理系统，都有自己的特点。但两者之间还是存在很多相似处，如都是关系型数据库、都遵循ACID规则、都支持SQL标准语法、都支持完整的事务处理等。但它们也有着不同之处，如MySQL支持分布式环境，支持实时查询，支持数据的压缩与备份等。由于Oracle基于商业化软件价格昂贵，学习曲线陡峭，而MySQL属于开源软件，无需购买许可证即可下载安装使用，易用性和便捷性更佳。因此，在选择MySQL或Oracle时，需要结合自身的需求，深入理解其特性及应用场景，然后再决定是否使用。
### 2.2.2 MySQL的优点
- MySQL支持分布式环境：MySQL集群可以利用多台服务器实现数据存储的分布式备份，提升数据安全性和可靠性。
- MySQL支持多线程处理：MySQL支持多线程处理，实现了真正的异步非阻塞IO模型。
- MySQL支持存储过程：MySQL支持存储过程，通过存储过程可实现复杂的业务逻辑，优化数据库性能。
- MySQL支持全文检索：MySQL支持全文检索，通过存储引擎MyISAM支持全文索引。
- MySQL支持事务：MySQL支持事务，确保数据的一致性，支持事务回滚，解决了传统数据库中的死锁问题。
- MySQL支持InnoDB引擎：MySQL支持InnoDB引擎，支持事务处理、崩溃修复和行级锁定，具有较高的并发处理能力和较高的安全性。
- MySQL支持XA事务：MySQL支持XA事务，允许多个数据库服务器协同工作，实现分布式事务。
- MySQL支持自定义函数：MySQL支持自定义函数，通过存储过程或触发器可以实现一些特定功能。
- MySQL支持视图：MySQL支持视图，可以创建虚拟表，隐藏复杂的数据结构，提高了数据的可视化效果。
- MySQL支持游标：MySQL支持游标，可以用来遍历结果集。
### 2.2.3 MySQL的缺点
- MySQL不支持分布式事务：由于MySQL支持XA事务，因此不支持分布式事务，这意味着如果两个节点之间插入相同的数据，会出现重复插入的问题。
- MySQL的连接数量受限：MySQL的最大连接数量默认限制为150个，当达到这个限制后，新来的连接将会被拒绝。
- MySQL不支持外键：MySQL不支持外键，只能通过检查约束来实现参照完整性。
- MySQL的备份及恢复速度较慢：MySQL的备份和恢复速度一般都比较慢，特别是对大数据量的表进行备份和恢复的时候。
- MySQL不支持空间数据类型：MySQL不支持空间数据类型，比如GIS数据类型。
