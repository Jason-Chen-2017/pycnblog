# 1.简介
  

对于计算机来说，数据存储、传输和处理的过程中会遇到各种各样的问题，比如字符集、编码格式等。如果不设置正确的编码格式，可能会导致程序不能正常运行或者产生乱码。本文将从以下三个方面进行阐述，帮助读者更好地理解编码格式这个概念以及如何选择合适的编码格式。
# 2.字符集和编码格式
首先，什么是字符集？什么是编码格式？
字符集（Character Set）指的是用来表示所有可打印或不可打印符号的一组规则。在计算机中，字符集通常以一个字符到整数的映射关系作为基础，这样就可以用整数来标识每种符号。不同的字符集代表了不同类型的字符，比如中文字符集、西欧语言字符集、日文字符集等。
而编码格式（Encoding Format）则是指根据字符集转换成计算机可以识别的二进制数据的方法。字符集和编码格式有着密切的联系，只有使用正确的编码格式才能将计算机的原始数据转换成可用于计算机内部计算的数字数据。一般情况下，计算机系统的默认字符集都是Unicode，它的编码格式也称为UTF-8。但是需要注意的是，UTF-8只是一种字符编码方案，并不是一种字符集。至于ASCII、GBK、BIG5这些历史遗留的字符集和编码格式，它们虽然已经不再被应用，但还是有必要了解一下。
# 3.基本概念
## 3.1 ASCII字符集及其编码格式
ASCII (American Standard Code for Information Interchange) 是美国信息交换标准代码，它是基于拉丁字母表的一套电脑编码系统。由于它只用7位二进制数（即 0~127）表示95个控制字符和94个可显示字符，所以属于比较简单的编码系统。它的编码格式很简单，每个ASCII字符占据8位二进制数，因此，ASCII兼容于Latin-1字符集。目前，UTF-8已经成为主流的字符编码格式。
## 3.2 ISO-8859字符集及其编码格式
ISO-8859 系列字符集是国际通用的字符编码方案，由一组单字节的编码组成，共收录了世界上几乎所有的主要语言字符。ISO-8859共收录了15种语言字符集，包括英语、法语、德语、意大利语、西班牙语、葡萄牙语、俄语、波兰语、罗马尼亚语、斯洛伐克语、瑞典语、芬兰语、匈牙利语、希腊语等。
除英语以外，其他语言均采用双字节编码。例如，中国的 GB2312 是最常用的 GB 编码，其中收录了汉字6763个，比全角字符集 GBK(国家标准) 和 Big5 多出3000多个。
## 3.3 Unicode字符集及其编码格式
Unicode 是一种字符编码，是业界共识，用来表示世界上所有文字。其编码范围涵盖了世界上所有语言的文字，包括希腊语、亚美尼亚语、泰语、中文、日语、韩语等等。与其他编码系统相比，Unicode 使用两个字节（16位）表示一个符号，所以理论上可以支持更多的符号。然而，实际上，Unicode 只被部分软件支持。除IE浏览器外，绝大多数浏览器都只支持Unicode BMP，也就是Basic Multilingual Plane，也就是最常用的平面，因为BMP里的字符基本上够用了。
UTF-8 就是 Unicode 的一种实现方式，它使用连续的字节序列来表示每个字符。它最常用的地方是互联网上的数据传输。
## 3.4 GBK/GB18030字符集及其编码格式
GBK/GB18030 是中国政府制定的关于计算机内文字显示的标准，由一系列预先分配好的码位组成。GBK采用双字节编码方案，而GB18030则采用四字节编码方案，这就保证了对繁体中文的全面支持。GB18030中文编码集是中国国家标准，与GBK兼容。GBK汉字区汉字数量超过GB2312汉字区的个数，因而具有更广泛的应用前景。
## 3.5 BIG5字符集及其编码格式
BIG5 是繁体中文的通行编码标准。它采用双字节编码方案，用于将中文编入五十音图符号系统中。BIG5编码最早在1986年颁布，历经数十年之后终于国际标准化。截至2019年，全球有5亿人口，大多数人口使用的仍然是繁体中文输入法。
## 3.6 UTF-8编码格式
UTF-8 是一种变长的字符编码，它使用一到四个字节序列来表示一个符号。它兼容ASCII，并且支持全世界所有语言的字符。但是，它与Unicode的兼容性差些，尤其是在非英语语言方面。
