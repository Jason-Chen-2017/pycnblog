# 1.背景介绍


## 1.1 电商技术平台建设背景介绍
电商作为目前国内知名的互联网金融模式，其在线零售、社交购物等业务占据了市场主流，因此各大电商平台纷纷涌现。近几年随着电商平台迅速崛起，有些平台为了获得更好的用户体验和流量转化效益，提出了多种增值服务和促销方式。由于卖家因违规或其他原因无法正常回复客户售后咨询的问题，导致客户对买家的满意度较低。而且卖家提交的售后工单也无法及时处理，进而影响平台利润。因此需要有一个系统化的解决方案来应对此类售后服务难题。  

针对这些痛点，本文将介绍如何建立一个面向电商平台的售后服务系统架构。首先，本文会介绍电商平台售后服务流程图，其主要包括订单创建、支付确认、发货处理、售后申请提交、售后处理结果反馈、维权跟进、评价等流程。其次，文章将阐述售后服务相关核心概念，如订单生命周期、售后分类、评价标准、投诉渠道、追溯记录、反馈记录等。最后，将分析不同售后服务需求对系统架构的影响。

## 1.2 售后服务系统架构介绍
### 1.2.1 概念与联系
上图为电商售后服务系统架构示意图，从上至下依次为：

1. **客服系统（客服中心）**：作为电商平台中的客服系统，负责与买家和卖家进行各种交流沟通，维护客户信息及提供售后服务。
2. **订单管理系统（订单系统）**：作为电商平台中的订单管理系统，负责处理订单的创建、支付确认、发货处理、售后申请提交、售后处理结果反馈、维权跟进、评价等流程。
3. **售后中心（售后中心）**：作为电商平台中的售后中心，负责接收来自买家和卖家的售后申请，并通过后台审核、排期、配送、协助、跟踪等方式为买家提供优质的售后服务。
4. **知识库系统（文档中心）**：作为电商平台中的知识库系统，负责收集电商平台的售后政策和规范，为平台的售后人员提供参考。
5. **数据统计分析系统（数据中心）**：作为电商平台中的数据统计分析系统，负责收集数据指标，用以评估平台的收益和成长情况。
6. **其它系统**：包含个人化系统、内容营销系统、支付系统等。

### 1.2.2 服务流程
#### 1.2.2.1 订单生命周期
一般情况下，订单的生命周期可分为以下7个阶段：

1. **订单创建（Order Created）**：买家下单，进入待付款状态；
2. **订单支付（Payment Confirmation）**：买家支付完成，订单进入待发货状态；
3. **订单发货（Delivery Handling）**：卖家发货，订单进入已发货状态；
4. **售后申请（AfterSale Request）**：买家提交售后申请，订单进入售后中状态；
5. **售后处理（AfterSale Process）**：售后中心处理售后申请，订单进入售后完成状态；
6. **售后维权（AfterSale Complaints）**：买家提交售后维权，订单进入维权中状态；
7. **订单完成（Order Completed）**：卖家发货之后，订单进入交易完成状态。

#### 1.2.2.2 售后分类
按照售后处理方式可以将售后服务分为如下五大类：

1. 客户服务：处理包括查询、咨询、退换货、取消订单、修改地址等相关问题；
2. 发货前售后：对订单商品品质、尺寸不符、包装破损、缺件、商品质量不达标等情况进行处理；
3. 发货中售后：在运输途中出现瑕疵、漏发、破损、错发商品等情况，包括顺丰快递与圆通、韵达快递、百世汇通等三方快递公司的差评、退款、退货等售后措施；
4. 签收后售后：在买家签收商品后，若存在质量问题或其他需要处理的情况，可以对其进行处理；
5. 其他类型售后：除以上四类外，还有些电商平台还会提供电话支持、EMS物流回单等售后服务。

#### 1.2.2.3 评价标准
根据国家相关法律法规的要求，在电商平台进行售后服务的时候，通常会给予如下评价：

1. **用户满意度：**包括对售后服务的满意度、在售后过程中对买家的帮助程度、售后返款情况、问题解决速度等；
2. **商户满意度：**包括对卖家的满意度、商品售后过程中的服务态度、货物配送及服务水平等；
3. **平台满意度：**包括平台运营成绩、产品质量、售后服务水平、商家服务质量等。

#### 1.2.2.4 投诉渠道
电商平台对用户的售后服务质量非常重视，所以一般都会设置专门的投诉渠道。比如，对于某些不理性、欺诈的行为，平台会封号、限制退款金额、甚至处罚严厉；对于服务态度差的卖家，平台可能会要求其承担一定的维权成本；对于质量问题，平台会组织检测部门对商品进行检测，确保商品的完好与安全；对于客户投诉过多的订单，平台可能采取一系列限制措施来减少用户投诉数量。

#### 1.2.2.5 追溯记录
电商平台需要记录每一次售后服务的处理记录，包括服务受害者、申请人、处理人、处理时间、处理事项、处理结果等信息。这样做的目的是为了方便买家和卖家追溯售后服务的整个过程，并提供有力的证据支撑案情。同时，该功能还可以为平台的售后工作带来新的效益，例如可以为平台的经营决策提供参考。

#### 1.2.2.6 反馈记录
电商平台的售后服务团队为了改善服务质量，会收集买家、卖家的售后反馈，用于及时了解消费者的满意程度、改进建议和改善措施。反馈信息可以通过网站、微信、QQ群、短信等方式直达到电商平台的售后团队成员，为售后服务提供更多的参考和帮助。另外，平台还需要及时响应来自用户的投诉举报，保证平台的公平正义。

### 1.2.3 售后服务系统架构演进
#### 1.2.3.1 时代背景
电商平台崛起于信息化时代，以线上线下结合的方式，为大众提供了更多便利。在当今信息化社会里，电子商务已经成为新的商业模式，传统的实体店被电子商务的模式所代替，越来越多的人选择在线购物。在电商平台建设的初期，售后服务是一个重要环节。因此，建设一套面向电商平台的售后服务系统，是电商平台建设的一个重要任务。

#### 1.2.3.2 新技术革命
2007年9月，马云在亚马逊创立了一个以“亚马逊网络服务”为名称的在线购物网站，这家网站引起了广泛的关注。该公司宣称要打造一个全新的电子商务领域。然而，20年后的今天，Amazon仍然是全球最大的在线购物网站，它已经拥有超过三千万的会员，并且在商品销售额、付款能力、物流能力等方面都取得了非凡的成就。而随着移动互联网的兴起和电商的发展，电商平台的售后服务需求也日益增加。

#### 1.2.3.3 小米与京东的售后服务系统架构演进
小米公司于2011年建立了自己的小米手机旗舰店，隶属于中国移动通信集团，主要为爱国主义者推出的品牌手机。小米手机的售后服务由供货商（华为）直接负责，但由于华为公司处于垄断地位，导致小米手机售后服务在当时相对来说比较简单。随着时间的推移，小米手机的份额越来越少，但售后服务却没有跟上。因此，小米手机引入第三方供货商来提供售后服务才得以发展。

2012年，小米手机发布了小米手机3代产品，这种手机的性能超过iPhone，但是售后服务却出现了一些问题。为了解决这一问题，小米公司与京东集团签订了一个战略合作协议，京东将自有的上门售后服务平台（JD-SOA）作为合作伙伴，帮助小米手机的售后服务更加高效、周到的进行。

京东的售后服务系统架构如下图所示：


1. **上下游供应链管理平台** 京东集团通过京东自身的供货链管理系统（PCSOA），协同供货商完成供应链的整合。京东集团向供应商提供一站式的配送服务，同时也接受供货商的订单和资料。

2. **供货商服务管理平台** 供货商通过独立的供货商服务管理平台（SMSP），完成售后保修、归还设备、上门服务等工作。京东集团通过第三方供货商平台，让供货商以最优质的服务为顾客提供优质的售后服务。

3. **客服系统** 京东集团和小米公司共同开发了一套客服系统，双方均可通过系统对接，实现即时沟通和问题反馈。同时，京东集团还设立了专门的售后服务电话，为客户提供全面的售后服务。

4. **售后中心** 在京东集团的帮助下，小米公司的售后中心（MiAfterSale）平台逐步形成，可以根据顾客的需求快速找到对应的售后服务专家进行沟通、协调、处理。售后中心和客服系统的双向沟通可以有效提升客户满意度。

#### 1.2.3.4 中国电商平台的售后服务架构演进
随着电商平台的发展，电商平台的售后服务也面临着许多新的挑战。如前所述，电商平台的订单流转周期较长，对于用户的售后请求也常常需要排队等候。在这种情况下，基于实时的数据分析和预测的系统是最好的解决方案。

2017年，华为技术有限公司推出了OceanLink ERP系统，可以为电商平台提供一个面向服务的系统，包括订单管理、仓库管理、客户关系管理、供应链管理、数据分析等。华为ERP系统帮助电商平台建立起完整的售后服务体系，并提供完整的售后服务流程。华为ERP系统通过基于数据的智能分析工具来优化售后服务，避免了传统售后服务流程的低效率，降低了售后服务成本。

此外，目前部分电商平台还通过智能客服系统（Intelligent Chatbot）来对售后服务进行管理。Intelligent Chatbot 是一种利用机器学习和自然语言处理技术，模仿人的对话交互方式的自动化终端，为客户提供即时的、可靠的售后服务。通过智能客服系统，电商平台可以在客服团队的辅助下，以最快的速度处理售后服务请求，提高客户满意度。
