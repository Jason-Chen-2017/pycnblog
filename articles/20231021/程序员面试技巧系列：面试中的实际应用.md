# 3.核心算法原理和具体操作步骤以及数学模型公式详细讲解



3.1 快速排序算法





## （1）选择主元

选择一个元素作为基准，把小于等于基准元素的放在左边，大于基准元素的放在右边。选择主元的方法有多种，比较简单的是随机选择，还有一种是三者取中法，即选择中间位置的元素作为主元。

```java
public static int partition(int[] arr, int low, int high) {
    // 随机选择主元
    Random random = new Random();
    int pivotIndex = random.nextInt(high - low + 1) + low;

    // 交换主元至数组末尾
    swap(arr, pivotIndex, high);
    
    // 设置基准值
    int pivotValue = arr[high];

    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivotValue) {
            i++;

            // 将小于等于基准值的元素交换至左侧
            swap(arr, i, j);
        }
    }

    // 将基准值交换至i+1处
    swap(arr, i + 1, high);

    return i + 1;
}

private static void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
```



## （2）快速排序过程

选主元，将小于等于基准值的元素放在左边，大于基准值的元素放在右边；递归排序左边的子数组和右边的子数组。

```java
public static void quickSort(int[] arr, int low, int high) {
    if (low < high) {
        int pivotIndex = partition(arr, low, high);

        quickSort(arr, low, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, high);
    }
}
```



## （3）快速排序的时间复杂度

最坏情况下的时间复杂度是 O(n^2)，当数组是反序排列时，此时时间复杂度为 O(n^2)。平均情况下的时间复杂度是 O(nlogn)。

- 每个元素只会被访问一次
- 每个元素都在同等概率被访问到

因此，快速排序是一个很好的选择。
