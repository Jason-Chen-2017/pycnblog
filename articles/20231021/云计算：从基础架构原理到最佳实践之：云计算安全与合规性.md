# 1.背景介绍


云计算已经成为主流IT技术之一，它是指通过网络提供虚拟化、远程计算和数据存储等能力，为用户提供可弹性伸缩的计算、存储和网络资源。随着云计算的蓬勃发展，越来越多的人开始关注云计算的安全问题和合规性问题。但是，云计算也同样面临着许多新的安全和合规性问题。下面就让我们一起来了解下云计算的安全与合规性。

1997年，美国联邦政府、商务部、国家信息管理局成立了信息安全委员会(ISAC)，ISAC提出“网络和信息系统（NIS）”安全领域的五项基本原则，包括保护通信、保护信息、识别威胁、防范攻击、审计跟踪。1998年，美国国土安全部成立了国土安全计算机中心(CSCC)并将其中的一个分支机构——国土安全计算机信息系统(CSI)，交由国家信息安全监督管理委员会(CSMILE)管理。1999年，加州大学洛杉矶分校设立国际研究所，美国中央情报局也设立了一个分支机构。这些机构在维护国家安全方面做了不懈努力，但安全问题也逐渐受到了各国政府、组织和社会的关注。而云计算的问题就暴露出来了。

近年来，国际上多个组织和政府陆续开始关注云计算的安全问题。包括美国国土安全部，阿里巴巴集团，微软，英国国家电网公司(Government Communications Headquarters)，以及日本国家ネットワーク推進センター等组织都发布了相关通知和白皮书，并采取多种形式开展云计算的安全相关研究。

2018年5月，英国国家网络安全中心(NCSC)发布了“云服务供应链的应用安全评估白皮书”，其中详细描述了云计算的安全风险、检测手段及控制措施，以及如何提高云服务供应链的应用安全水平。该白皮书还提供了有关云计算安全的若干建议，包括建立强大的基础设施和制度，优化云计算平台，改进应用程序安全配置和开发流程，增强管理和协作机制。

2018年12月，美国国防部发布了“2018年云漏洞、攻击和风险趋势分析”。该报告对云计算领域的安全问题进行了总结和研判，并结合当前的态势对未来进行了分析。同时，对当前存在的一些安全漏洞、攻击行为、云资源泄露、恶意程序入侵等问题进行了调查和追踪，并提出了相应的安全防范策略。

2019年1月，由中国工信部、信息产业部、公安部联合发布的《关于加强网络空间安全治理的指导意见》明确提出，要坚持网络空间安全建设的总体思路，统筹协调网络安全和经济社会发展；加强对安全事件的响应，深刻认识网络安全的新形势，针对突发安全事件及时处置，有效预防和化解网络安全风险。

2019年1月，由英特尔联盟安全委员会、微软安全基金会、诺亚方舟实验室、赛门铁克等机构共同发布的《中小企业云计算行业安全白皮书》，对云计算安全做出了全面的综合性分析，反映了安全技术人员对于云计算的理解程度、使用的感知、掌握技能、思维方式以及对安全的防御能力。

由此可见，云计算领域已经成为一个十分重要的研究热点。云计算的安全与合规性也必将成为一个复杂而严峻的课题。作为一名技术专家，本人具有丰富的行业经验，愿意倾听不同声音，给出独到的见解。
