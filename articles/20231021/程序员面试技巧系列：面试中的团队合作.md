# 1.背景介绍


“面试”作为技术人参加工作的重要途径之一，尤其在互联网行业快速发展的今天，“面试中技术能力突出”“面试难度不高”“面试机会多元”等诸多特征反映着越来越多的人们对该岗位的期望越来越高，越来越多的人参加了该职位的面试。而在实际的招聘过程中，很多人将这几点因素混淆，往往以“不知道怎么回答”、“说不上来”或“答非所问”等毫无解决方法的态度进行面试，导致很多人的心情和技术水平都受到影响，甚至导致考核体系崩塌。因此，作为一名资深技术专家，能够在面试环节中顺利的合作伙伴搭建和技能提升是非常必要的。
为了让更多技术人员更好地了解如何在面试中合作，帮助他们更快有效的提升自己的综合素质，我们邀请到了一些有经验的技术大牛，结合实际案例，分析面试中合作的常用手段及技巧，以及面试中可以遇到的各种“坑”。本文旨在分享自己的学习心得，并给想从事相关工作的程序员提供一份参考。
# 2.核心概念与联系
## 1) 综合性知识点（主要由候选者提问）
候选者可能会针对如下问题进行提问：
- 为什么要合作？/为什么选择我这个候选人？
- 您认为自己掌握哪些优秀的技术能力？
- 你有没有意识到自己缺乏某项技术能力？如果有，您如何克服？
- 有没有什么优秀的资源或者培训机构推荐给过你？
- 在您的团队中，有哪些角色和职责？
- 您觉得我适合这个团队吗？/是否愿意接受我的建议？
- 如果面试成功，您打算如何发展？/是否有进一步的合作计划？

## 2) 技术交流（主要由面试官和候选者双方互相交流）
面试中技术交流包括如下环节：
- 对话（一边候选者一边面试官用自然语言表达自己，很少听不到声音的情况下，候选者需要理解面试官的意图，有时候候选者会故意错开句子，让面试官难以捉摸；另一方面，面试官也需要做到清晰准确，避免使得候选者产生困惑或误导）。
- 反馈（候选者的反馈主要依赖于对问题的理解和分析，面试官需要和候选者多多沟通，询问细节、反馈偏差、提醒潜在风险点），并且可以根据候选者的反馈做进一步的指导。
- 反思（面试中候选者可能会对自己的答题方式或态度不满意，可以通过问面试官意见的方式进行调查，尤其是在复杂的问题上）。
- 提问（面试官和候选者之间存在一定程度的误会，比如候选者觉得面试官是在欺骗他，甚至在恐吓他，而面试官则会严厉批评候选者不负责任、不专业、提前结束面试等等。通过反复提问和调整对话气氛，能够缓解这种误会。另外，还有可能出现候选者提问某个问题后，面试官选择忽略掉这个问题，这时候候选者需要主动追问，通过反向提问的方式解决问题。）
