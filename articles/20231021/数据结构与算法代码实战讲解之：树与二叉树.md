# 1.背景介绍


## 概念介绍
树（Tree）是一种无限的抽象数据类型，用来表示层级关系的集合。它是一种抽象数据类型的数据结构，其中的每个元素都有一个父结点、零个或多个子结点。树可以用于表示组织结构、文件系统目录结构、语法分析树、表示游戏地图等等。本文所涉及到的树都是平衡树。
### 二叉树
二叉树（Binary Tree）是树的一种特殊形式。二叉树是一个有序树，每个节点最多有两个孩子节点，分别是左孩子和右孩子。二叉树的性质：
- 每个节点最多有两个孩子节点；
- 有左孩子的节点叫做“左子树”，有右孩子的节点叫做“右子树”；
- 如果某个节点没有孩子节点，则称为空节点（null node）。
#### 完全二叉树
如果一棵二叉树具有如下性质：
- 满足完全二叉树定义，即任意一个非空节点，都有两个子节点；
- 除了最后一层外，所有层上的节点都尽量是满的，最后一层可能只剩下左边的一些节点。
则这棵二叉树就是完全二叉树。
#### 满二叉树
如果一棵二叉树的所有叶子都在同一层上，并且每一个节点都有两个子节点，那么这棵二叉树就是满二叉树。
#### 完美二叉树
如果一棵二叉树是满二叉树，而且所有叶子都在最底层的两边连续分布，则称该树是完美二叉树。
## 树的分类
根据树的度和结点个数不同，树可分为不同的类别：
- 度为0的树称为叶子结点（leaf），它不再拥有孩子节点；
- 度为1的树叫做线性链表（single linked list），它由根节点、第一个孩子节点和第二个孩子节点构成；
- 度为2的树叫做二叉树（binary tree），它由根节点、左子树和右子树组成；
- 度不超过3的树叫做二叉查找树（binary search tree），它是一种排序树。
## 树的实现方式
树可以用数组或链表的方式实现，数组的方式把左右孩子节点作为两个索引值存储在当前结点中，链表的方式把孩子节点连接到当前结点中。当然，也可以用其他复杂的结构比如哈希表或者红黑树来实现。
## 应用场景
树主要用于解决以下三个方面的问题：
- 对树进行遍历，包括先序遍历、中序遍历和后序遍历；
- 求树的高度、宽度、深度、大小；
- 判断两个树是否相同。
# 2.核心概念与联系
## 二叉搜索树
二叉搜索树（Binary Search Tree）是指一棵二叉树，其中所有的键值（Key values）左子树的值均小于根节点的值，右子树的值均大于根节点的值。
二叉搜索树的一个重要特性就是：左子树的所有节点的值均小于根节点的值，右子树的所有节点的值均大于根节点的值。因此，在二叉搜索树中，搜索操作可以用二分法来快速找到指定的值。

## Trie树
Trie树（又称前缀树、字典树、 radix trie ）是一个树形结构，用来存储一系列的字符串。每个结点代表一个字符，结点之间的边代表字符之间的关联关系。
对于一个字符串，若从根节点出发，每个字符沿一条路径对应到相应的儿子结点，则这个字符串也存在于树中。因此，通过路径上的字符可以唯一确定一个字符串。
