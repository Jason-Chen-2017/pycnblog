# 1.简介
         

根据2020年医疗行业国民经济和产业结构调整后的战略部署要求，我司提出了由医疗垫付管理平台、医疗垫付决策支持系统（ADSS）、电子健康卡管理系统三部分组成的垫付管理体系。其中，医疗垫付管理平台作为垫付管理的信息中心和协同平台，负责数据共享、信息收集、数据分析及结果呈现；医疗垫付决策支持系统作为医疗垫付专项决策服务平台，实现高效的智能化结算方案生成，为医生提供有效的诊断信息和运营建议；电子健康卡管理系统则承担着发放电子健康卡、医院自助下单等一系列功能，确保患者在病情加重或多种疾病同时并发时能够迅速准确地做出医疗资源的分配和取舍，提升患者满意度。

随着“复工”的到来，新冠肺炎疫情在全球范围内快速蔓延。面对疫情复杂而严峻的形势，面临的是医疗服务的长期和终极危机。目前，世界各国正在抢修路、架设桥梁、扩建医院等举措，应对病毒的冲击。但是，如何更好地组织和运用科技手段，帮助患者在危机中更好地选择医疗资源和治疗方式，仍然是一个重要的问题。因此，我司基于自身经验和学习到的技术知识，结合医疗垫付管理、智能垫付系统及健康管理等领域的前沿研究，基于对ADSS的需求和体会，通过对垫付管理、智能结算、电子健康卡管理三个方面的相关技术的综合研究、系统设计、开发，希望能够设计出一套符合医疗垫付管理需求的自动化决策支持工具。

# 2.核心概念术语说明
## （1）ADSS概述
ADSS（Automated Discharge Decision Support System），即“自动出院决策支撑系统”。它是指通过计算机技术进行人机交互，以决策支持的方式进行出院过程及相关辅助决策的工具。它分为医生端、患者端和信息中心三个部分，用于完成出院决策的流程及相关事务的决策支持。ADSS可以对患者、医生及相关信息进行实时追踪，对出院患者或可能出现的风险问题作出有效的防护措施，减少出现意外发生的风险，并提供可信的出院决策依据。
## （2）信息采集
“信息采集”是ADSS的一项关键环节。信息采集是指将各类患者的患病史、既往史、临床表现、体检结果、就诊信息等相关信息进行汇总和整理，生成统一的数据格式。该阶段需要考虑数据的完整性、准确性、真实性、及时性。
## （3）特征抽取
“特征抽取”是ADSS的一个主要任务。特征抽取从事于从医嘱中或患者信息中提取出可用于出院决策的有价值信息。所谓特征，就是某一个有关的因素，如体征、实验室检查结果、药物分类等。ADSS可以通过不同的算法和模型对患者的特征进行分析，获取其最佳匹配程度、相关性及其他有用的信息。
## （4）候选区分
“候选区分”是ADSS的第三个关键环节。候选区分是指将所有病历记录的“符合标准”的患者进行划分。不同的标准，比如时间跨度、症状、手术类型等，可以分别设置不同的过滤条件。该阶段需要考虑各类因素的权衡和平衡。
## （5）精准推荐
“精准推荐”是ADSS的最后一步。精准推荐是指在候选列表中根据患者的具体情况、生理状态、过敏史、药物使用史等因素给出符合条件的推荐方案。该阶段还需要结合AI和ML技术，提高推荐准确率、降低推荐难度。
## （6）ADSS应用场景
ADSS的应用场景主要分为以下几类：

1. **心理健康出院辅助**: ADSS可以作为门诊环境中的服务导向系统，提高患者的心理健康状况，改善出院后生活质量。例如，通过远程诊断、心理测试、精神疾病筛查、早期警示及精准医疗咨询等方式，对患者进行诊断和适当治疗，帮助患者摆脱治疗依赖和焦虑情绪，达到心理健康出院的目标。

2. **智能出院决策支撑**: ADSS可以帮助医生根据患者的病情特点和诉求，智能地提供出院意见和辅助决策。例如，针对特定病情的患者，ADSS可以智能识别出其特有的临床特征，提供多种出院意见和方案，帮助医生快速做出最优选择，保证出院安全及患者满意度。

3. **预约挂号助手**: ADSS可以帮助患者预约和挂号，减少排队时间，提升患者的出院成功率。例如，ADSS可以将患者的个人信息、症状等通过手机短信发送至预先设置好的热线，等待医生的呼叫。也可以采用智能问答系统，自动检索患者的症状、用药史等信息，快速为患者提供医疗建议。

4. **电子健康卡管理**: ADSS可以完成电子健康卡发放、用户信息维护、患者信息查询、患者服务中心入口的构建等工作，确保患者在病情加重或多种疾病同时并发时能够迅速准确地做出医疗资源的分配和取舍，提升患者满意度。
