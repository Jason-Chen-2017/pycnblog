# 1.背景介绍


## 1.1为什么要做DevOps
随着互联网企业在市场竞争中处于一个快速增长期，市场竞争的激烈程度也越来越高。传统上，互联网公司依赖于内部人员进行开发、测试、运营等工作，但随着IT技术的发展，越来越多的应用都需要第三方服务支撑，比如电商网站的支付、物流等，这些第三方服务对公司的核心业务的支撑作用越来越重要，而对于企业来说，如何让自己能够更好的服务于客户，显得尤为重要。
为了能够提供更加可靠和可信的服务，这些第三方服务往往需要实施DevOps模式，或者说是敏捷开发流程（Agile Development Process）下的持续交付（Continuous Delivery）。其核心思想是通过自动化实现从代码编写到部署上线的全过程，提升效率和质量，释放生产力，进而满足用户需求。
## 1.2什么是DevOps
DevOps是一种现代软件开发方法论，旨在缩短软件开发周期、加快软件交付速度，并促进团队成员之间的沟通协作，降低软件出错率。它强调开发者、qa、运维、用户及相关利益相关者的高度协作，所有角色参与到开发过程中，共同构建和维护软件。DevOps运用最新的软件工程技术、人文社科的研究成果、模式及工具，围绕软件开发生命周期及其各个阶段进行优化，确保软件交付前后能够按时、准确、安全、可靠地运行。因此，DevOps是一种强调价值观，强调自动化运维的一套完善的方法论。
## 1.3为什么要做电商平台的DevOps
当今的电子商务领域正在经历一场技术革命。移动互联网、社交网络、物联网、云计算等新兴技术的驱动下，电子商务已经成为当今社会信息化、连接性增长的重要载体。不同行业、不同领域的电商平台，在功能、性能、界面等方面都有自己的特色。因此，为了能够更好的服务于客户，电商平台本身就需要实施DevOps模式，或者说是敏捷开发流程（Agile Development Process）下的持续交付（Continuous Delivery），这一切才会变得更加可靠和可信。
## 1.4DevOps有哪些好处
- 提升交付效率：通过自动化，降低了开发、测试、发布等环节的沟通成本；能够及时的发现bug并及时修复；提升了软件的响应速度和交付能力，从而减少了最终用户的等待时间；节约了不必要的人工资源。
- 提升业务连续性：通过自动化，可以保证代码库始终保持最新状态，从而保证了产品的稳定性和可用性；同时，由于所有的环节都是自动化的，因而不存在某个环节出现问题时其他环节无法正常运作的问题，从而提升了产品的容错能力。
- 更快地反馈客户需求：通过持续集成的方式，能及时发现和反馈客户需求的变化，帮助公司更快、更准确地调整产品策略；甚至可以用自动化测试工具检测出错误，帮助开发人员更快地定位错误位置，解决软件问题。
- 降低风险：DevOps的引入可以降低开发、测试、部署的风险，因为所有的环节都在自动化，不会导致因人为失误造成严重的问题。
## 1.5什么是电商平台DevOps实践？
按照《DevOps实践白皮书》（http://www.infoq.com/cn/news/2017/11/DevOps-Practice-Whitepaper）的定义，“DevOps实践”定义如下：DevOps实践是指通过应用敏捷开发方式、自动化运维工具和服务，将开发、测试、运营团队的职责划分合理，采用微服务架构、容器技术、持续集成、持续交付、持续部署等 DevOps 方法和工具，以更高的标准、更快的速度、更低的成本交付更高质量的产品或服务。此外，还应当关注用户需求，将用户故事转化为软件需求，并与开发人员一起进行持续改进。根据《DevOps实践白皮书》，电商平台DevOps实践包括以下五项基本步骤：
1. 开发流程优化
2. 测试流程优化
3. 持续集成（CI）和持续部署（CD）
4. 服务水平弹性伸缩
5. 用户体验改进和运营监控
