# 1.背景介绍


“信息架构”这个词汇在21世纪已经成为一个重要的词汇，它指的是通过合理安排、制定流程、提升效率和控制成本等方式建立起来的一整套有效的管理结构，用于管理企业内部或外部的所有信息，包括人力资源、财务信息、业务数据、产品目录及客户信息等。信息架构对公司的运营、产品开发、市场推广等各个环节都产生了重要的影响。所以，如何构建科学、清晰、精准的信息架构体系，对于任何一家规模的公司都是至关重要的。
随着互联网的飞速发展和海量信息数据的生成，信息架构的实践越来越多地应用到各个行业和领域中。不过，由于信息过载，构建清晰、科学的信息架构一直是一个难点和挑战。这就需要我们在信息架构的设计过程中更加注重结构化思维能力。
21世纪以来，“金字塔”一词逐渐被用来形容信息架构体系的构建方法。不同层级之间的划分，使得信息呈现出金字塔型的形态。金字塔一般由五个层次组成——顶层为核心信息，依次往下是分支信息、枝叶信息、横向关系信息以及立体关系信息。每一层中的信息数量较少，但层次间高度的联系性强。所以，研究人员经常将金字塔型的结构作为一种理论框架，来解释信息架构的组织形式。
正如其名，“金字塔”结构也叫做“结构化思维”。通过这个框架，我们可以清楚地看到信息的全貌，掌握信息的分层结构，理解各种信息之间的关系，从而达到信息的整合、梳理、传递和分析的目的。在这里，我们要探讨一下信息架构中如何运用结构化思维的方法，构建一个科学、清晰、精准的信息架构体系。
# 2.核心概念与联系
“信息”这个词汇非常宽泛，既可以指计算机程序、数据、文件、文本等物质，也可以指现实世界的事物，甚至还可以是人的灵魂。信息的层次和复杂度不断增加，如何让这些信息有条理地呈现在我们的视野中、实现信息的高效利用，才是信息架构的关键。我们首先需要明确一些核心概念，才能更好地理解结构化思维的内涵。
## 2.1 结构化思维
结构化思维是一种分析和处理复杂信息的有效办法。结构化思维的核心理念是，通过将复杂的事物拆分成简单易懂的元素并进行分类、整理、归纳，对整体状况有直观的认识。在设计信息架构时，采用这种思想往往能简化复杂度，提高效率和准确性。下面是结构化思维最主要的三个特征：
- 第一，习惯性思维：结构化思维要求我们对信息的处理过程保持一定的自然流畅性。我们总是习惯于以线性的方式处理信息，而忽略了其中的逻辑和关联。因此，了解信息的演变历史和模式对我们分析和处理信息很有帮助。
- 第二，抽象性思维：结构化思VELLING根据目标信息的特点，选择适合的方法和工具，对信息进行概括、归类、结构化、整理、呈现。抽象性思维能够帮助我们快速识别、把握信息的共性和关键特征。
- 第三，问题导向：结构化思维的一个重要原则是先发现问题再解决问题。结构化思维的第一步就是搞清楚信息究竟是什么、为什么、如何，然后再决定怎么办。这一步是找出问题的关键所在。通过这种方法，我们可以更容易地发现问题，找出问题的根源，并采取相应的措施去解决问题。
## 2.2 “五层”信息架构
“五层”信息架构是在构建信息架构时所采用的基本原则。“五层”由底层（Core）、分支层（Branch）、枝叶层（Fork）、横向关系层（Axis）、立体关系层（Spiral）五个层级构成，是结构化思维的一类方法论。
### （1）Core层
“Core”层是“金字塔”结构的核心。在Core层里，主要是由最重要的信息或者最具价值的知识呈现在用户面前。通常情况下，Core层的信息具有最高的可见性、最高的权威性，具有最核心的意义和价值。Core层的位置在金字塔的顶部，也是信息最原始、最原始的信息。
Core层的信息应该能够对用户产生直接的影响，如新闻、活动、产品和服务等。Core层的信息应当足够紧急且精准，用户需要马上获知。例如，电影的预告片往往在Core层出现；市场调研报告往往会提前发布在Core层。
### （2）Branch层
“Branch”层是Core层的子集，包含了与Core层相关的其他信息，同时又包含了相对独立的信息。Branch层的位置处于金字塔的中间，提供比较重要的信息。Branch层的信息具有一定程度的局限性和特殊性。比如，虽然我们可以获得广播新闻的快照，但是这些快照可能是采用某个局部的政治口径和方式制作的，我们并不能完全信任该消息。Branch层的信息可能来自某个局外的人士、某个区域的民众、某个组织机构，具有实时性和时效性。
Branch层的信息往往可以帮助Core层的信息更好地理解。Core层的信息会反映出用户的基本需求，而Branch层的信息则能够提供补充、辅助信息。例如，手机的定价政策通常都存在Branch层的描述；地铁路线的选择往往依赖Branch层的资讯，比如交通、天气、停车费用等。
### （3）Fork层
“Fork”层类似于Branch层，也包含了与Core层或Branch层相关的其他信息，但是信息量比Branch层小很多。Fork层的位置处于金字塔的底部，最具争议的信息。Fork层的信息往往反映了一个分歧，即不同的观点或立场。Fork层的内容很杂乱无章，我们需要花时间去阅读、理解、记忆。
比如，核武器是一个热门话题，并且有不同的观点。如果某个政府部门试图颠覆核工业，那么Core层就会发表声明，而Fork层可能会提供不同版本的核计划。另一方面，某些国家可能会支持进口核武器，那么Core层的声明就会受到质疑，而Fork层会列举支持进口核武器的国家。
### （4）Axis层
“Axis”层是横向关系的集合。它是指一条信息和另一条信息之间存在的关系。轴线表示两个信息之间的主题。Axis层中包含横向关系信息。比如，A报道说X发生了Y，B报道说Z发生了W，这时候，它们就属于同一个轴线。轴线层的存在与否，主要取决于两个信息之间的关系，比如X和Y是否具有因果关系，Y和W是否具有相关关系。Axis层中的信息与Core、Branch、Fork层中的信息存在差异，其存在意义在于提供上下文信息。
### （5）Spiral层
“Spiral”层是由上至下的层级结构。也就是说，Spiral层中的信息包含了上层级的信息。Spiral层的出现意味着某些信息开始转移到下一层级。如前所述，Core层的信息无法完全理解，需要转移到下一层级——Branch层，Branch层的信息没有孤立存在，需要转移到下一层级——Fork层。所以，信息的组织结构实际上就像一个金字塔一样，Spiral层只是起到了另一种安排形式上的作用。
