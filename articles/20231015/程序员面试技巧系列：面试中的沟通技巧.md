# 1.背景介绍


首先介绍一下这个系列的名字：“程序员面试技巧系列”。为什么要叫做“程序员面试技巧”呢？因为所有的面试都可以拆分成很多个小问题，而这些小问题也可以用一些技巧来解决。所以我觉得把这些技巧进行分类总结，并命名为“程序员面试技巧”或“技术面试中的技巧”可能更有助于对面试者的指导和辅助。 

这个系列的文章将会涉及到面试中遇到的各种面试官会问的问题和面试者应该如何回答的问题。文章的内容会从基础的一些面试技巧，比如应聘职位、提升自己的简历、软实力、工作经验、项目经验等；到一些具体的问题，比如HR和面试官要问什么样的问题，面试者应该如何准备，怎样面试才能得到最好的结果，这些都会在文章中详细的展开。 

# 2.核心概念与联系
## 2.1 提出需求之前先谈话技巧(Talking skills)
1、平时多主动询问面试官相关信息，不要耍无赖说不知道、不懂装懂。即使不能完全理解某个问题也要多提问，让面试官给予你足够的帮助和支持。

2、不要在电视上给别人提问，可以找一张明亮的桌子坐下来。电视屏幕反光太强可能会影响眼睛的适应性，建议用一个柔和的椅子或者沙发。同时，注意调整呼吸声音，避免喧哗，减轻噪音。

3、注意语言表达能力，一般来说英语水平越高越好。尤其是在表达复杂、专业的概念时，如果不能准确、清晰地表达出来，那么就无法胜任这个岗位。因此，一定要仔细斟酌每句话的措词、时态和语法，充分利用你的文笔技巧来表现自己。

4、以同理心对待面试官，面试官不会喜欢看到别人被欺骗。你需要讲解清楚自己的真实想法、价值观、优点、缺点，并且让面试官感受到你是一个真正的人。

5、如果不确定是否要接受邀请面试，可以尝试提前约定时间见面，这能让你有足够的时间去准备、沟通和了解更多的信息。另外，即使不接受邀请面试，也要勇敢面对面试官的“进一步说明”，否则很容易落空。

## 2.2 领导力（Leadership）技巧
领导力主要指管理技能，包括处理事务、协调资源、控制方向、冲突解决等方面的能力。

1、知识储备与团队建设。领导力需要广泛的知识储备，包括丰富的学习经验、有效的沟通技巧、有关领域的专业知识等。因此，如果你在应聘该职位时没有之前的相关经验，那么你就需要加强个人能力的培养，增强自信、独立、领导能力。

如果你正在工作中，就需要继续努力改善自己的管理、协调、沟通能力。比如，要保持良好的言行举止，认真负责，主动承担责任，避免让负面情绪积压在心里，如过度自责、过度沮丧等。

2、交流技巧。一般情况下，领导者之间的沟通都是直接的，通过语言文字进行交流。但有时候会出现一些不礼貌的语言，此时可以采取一些策略化的方法，比如采用隐晦手段、悬疑追问、猜测推断、假设风险等，来降低自己的嫉妒心、保护自己和维护团队气氛。

## 2.3 沟通技巧
沟通技巧就是指你在面试过程中，要善于倾听、理解和传达自己的意愿、观点和感受，并坦诚表达自己的想法、看法。

1、精炼自己的问题。要提问之前，先精炼好自己的问题，把握好要解决的问题的范围，缩小搜索的范畴，这样才能得到正确的答案。

2、注意表述习惯。不同的语言表达方式会引起不同程度的误解，因此你要选择恰当的方式和表述你的观点，才能够让面试官有所收获。

3、重申自身的立场。沟通时应该坦然直率，避免以个人私事和矛盾激怒他人。在回答问题时，你应该坚持自己的立场，并且根据面试官的回答进行判断，分析问题背后的原因。

4、尽量避免批评和攻击。在表达自己的观点时，切记不要表现得过于强硬和敌视面试官，这样可能会使得面试官产生反感甚至厌恶。

## 2.4 论证技巧（Reasoning skills）
为了证明自己的能力，需要经过艰苦的思维训练和精神洗礼。如果你已经准备好了，那就可以静下心来仔细分析和论证自己的想法和观点。

1、逻辑思维。在面试过程中，你要学会逻辑思维，从不同角度来分析问题，找出理论上的原因和解决方法，找到最优解。

2、直觉判断。作为一个聪慧的逻辑思考者，你还需要学着用直觉判断，判断不同信息的价值、意义、关联、重要性，以及当前的处境。

3、思路清晰。仔细阅读题目，直到找出关键信息为止。观察目标人的行为，了解他们的思维过程，以确定问题的根本原因。

4、质疑直觉。对于证伪性的陈述，你要反驳它，并对其质疑进行理智分析。通过质疑，你能够发现那些不可靠的假设和误导性的信息。

5、展示自己的技巧。论证时，要展示自己的见解、能力和技巧，以增强面试官的信服度和自信心。展示完毕后，你可以安静地听取面试官的回应。
