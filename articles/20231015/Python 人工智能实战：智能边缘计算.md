# 1.背景介绍


## 智能边缘计算简介
云计算、大数据、人工智能、物联网、IoT等新技术带来了海量数据处理、分析和应用的需求。目前，AIaaS平台层出不穷，包括亚马逊AWS，微软Azure，谷歌GCP等，这些云平台可以帮助用户轻松部署基于云端的机器学习服务，并提供超高算力运算能力。

然而，云端计算虽然降低了成本，提升了服务质量，但却无法满足信息孤岛（Edge）端设备的需求。特别是在边缘计算场景下，越来越多的应用需要在本地进行计算，以实现更快响应时间和更加隐私保护。相比于传统的云计算，边缘计算有着以下优势：

1. 移动性：边缘计算可以使应用能够及时响应移动用户的请求，无需等待云端服务器的响应；
2. 本地资源：边缘计算可以在边缘设备上运行，利用本地计算资源可以减少网络延迟和数据传输成本，提高效率；
3. 隐私保护：边缘计算可以对敏感数据的处理做到保密，不会向云端发送原始数据，保证用户的隐私安全；
4. 节约成本：由于边缘计算只用计算能力很强的设备完成计算任务，因此可以节省大量的云计算资源费用，实现云端与本地的整体利益最大化。

那么如何将云计算和边缘计算结合起来呢？答案就是“智能边缘计算”。

## 智能边缘计算概述
### 概念定义
智能边缘计算是一种新的计算模式，它将云计算和边缘计算技术有效地结合起来，通过分布式、轻量化、动态管理的边缘计算资源，可以让应用快速响应、解放用户本地设备的计算资源，同时通过可靠的低功耗的小型边缘终端设备，实现隐私保护、节约成本。其主要特征如下：

1. 分布式部署：智能边缘计算架构支持将应用程序部署在不同的位置，即分布式部署；
2. 低功耗：智能边缘计算架构支持部署在功耗较低的小型边缘终端设备，尤其是嵌入式设备，以实现超低功耗和长期稳定运行；
3. 可扩展性：智能边缘计算架构可以通过自动扩展机制来满足应用的高性能和弹性需求；
4. 灵活性：智能边缘计算架构支持灵活部署，允许在云端或者边缘部署应用，同时应用之间也可以互相调用和交流；
5. 无状态计算：智能边缘计算架构支持应用程序无状态，因此能够实现更高的可伸缩性和弹性，并降低部署和运维复杂度；
6. 动态管理：智能边缘计算架构支持动态管理，即根据应用的负载和使用情况，智能调配计算资源，实时分配资源；
7. 安全性：智能边缘计算架构支持完整的安全管理机制，包括数据加密、访问控制、审计跟踪等功能；
8. 边缘协同：智能边缘计算架构支持边缘设备之间的协作，能够实现异构设备上的业务逻辑。

### 边缘云联动方案


图1-1 边缘云联动方案示意图



边缘云联动方案是一种边缘计算架构，其中包括边缘计算设备、云服务提供商、应用程序开发者、云服务商、中间件和操作系统。从底层到顶层依次为：

1. 边缘计算设备：包含智能小型设备、嵌入式设备、车载终端设备等；
2. 云服务提供商：提供各种云服务，如计算、存储、数据库、消息队列等；
3. 应用程序开发者：编写应用代码并上传到云端或边缘端运行，然后通过API接口调用云服务；
4. 云服务商：提供基础设施服务，如机房、电力、网络等；
5. 中间件：包括OS、应用运行环境、编程语言环境等，用于应用程序的部署和运行；
6. 操作系统：包含硬件操作系统、应用操作系统等，负责云服务和应用程序的运行。

通过这种边缘云联动方案，可以把边缘设备上的计算资源作为云计算服务的一个组件，充分利用云服务提供商的计算、存储、数据库等能力，解决边缘计算中的很多难题，如：

1. 数据安全性：通过边缘计算架构，可以为客户提供全面的、高级的数据安全功能，例如数据加密、访问控制、审计跟踪等功能；
2. 边缘计算弹性：智能边缘计算架构可以有效应对边缘节点的不断增长和变化，比如新增节点或出现故障；
3. 成本节约：边缘计算架构可以节省云计算服务商的硬件成本，因为边缘计算节点的计算资源相对昂贵；
4. 用户体验：智能边缘计算架构可以为用户提供有限的、本地的计算资源，使得应用可以立即响应用户的请求，进一步提升用户体验。
