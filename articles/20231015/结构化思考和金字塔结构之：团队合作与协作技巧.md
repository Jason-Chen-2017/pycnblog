# 1.背景介绍


## 1.1 概述
公司内部上下级之间、部门之间、各个业务线之间的沟通和信息交流的过程中，出现了一种新的工作方式——**跨部门协作模式**。新模式源于多年来金字塔型组织架构的诞生及其复杂而严密的制度设计，以高度集中的权力制衡机制控制各个部门间的相互关系，形成了一系列高度依赖工作效率的日常运营。但是由于这种跨部门协作模式存在着较多的问题，如职责边界模糊、不清晰、易滥用等问题，因此出现了“跨部门协作”这个概念——作为替代，提出了“结构化思维”的理念和方法论。结构化思维可以帮助我们解决以上跨部门协作遇到的问题，特别是多个部门之间的信息和资源共享，提升了企业的整体竞争力。

## 1.2 结构化思维
结构化思维（Structured Thinking）也称为金字塔结构思维（Pyramid Structured Thinking），是指通过构建复杂系统所需要的多个层次的结构或模型的方式来解决问题。在一个组织中，往往存在着很多不同的层次的结构或者说模型，比如公司有各个部门、子公司、区域、市场等不同层次的结构；个人往往也会对自己的职业生涯建构出不同的层次结构。

### 1.2.1 什么是金字塔？
首先，我们要了解什么是金字塔，什么是金字塔结构思维。简单来说，就是在一定程度上，人们将世界分为不同的层次，由最高层到最低层依次为金字塔的上方和下方。金字塔是一个比喻，并不是实际存在的。但从某种意义上来说，它具有强烈的观念上的吸引力。

金字塔的顶部代表的是最重要的人物、事件、产品和理念，如CEO或领袖；中央有一个中心主题，如企业的主旨、宗旨或使命；底层则是由最低的工作单位组成，如基层员工、劳动者、学生和游客等。整个金字塔中间则是许多小的枢纽，如部门、业务线、区域等，它们彼此相连、相互支持。金字塔的结构在一定程度上反映了组织的结构、功能、任务、责任、角色等，从而使得员工之间的沟通更加顺畅，管理也更加有效。


### 1.2.2 为什么要进行金字塔结构思维？
在完成了一个项目后，员工通常需要将知识和经验传播给其他人。传播的方式有很多种，比如在座人员分享，参加培训课程，发布会上展示，或向上级汇报。这些传播的方法虽然有效，但却可能过于单一，无法充分展现组织的整体情况。而金字塔结构思维的作用正好弥补了传统的沟通方式。通过构建复杂系统所需的多个层次的结构或模型的方式，员工可以快速地理解一个复杂系统的整体架构和流程。如果能够通过这样的结构化思维，可以达到以下目的：

1. 提升信息的准确性，降低信息传递的误差。结构化思维能帮助组织内各层次的员工明白自己正在交流的信息，并且只传播必要的信息。
2. 促进工作的协调，减少沟通障碍。结构化思维能够帮助组织的各层次人员协调工作计划，减少组织成员之间或各个部门之间存在的信息不匹配的问题。
3. 提升组织的战略性，树立正确的方向。结构化思维能够帮助组织的各层级人员有针对性地做出决策，树立共同的目标和方向，增强团队的凝聚力和战略能力。
4. 提升业务的科学性和可复制性，改善管理风格。结构化思维能够提供一个分析和归纳全局的方法，使得每项工作都可以被细致地梳理，并且可以根据需要使用模型来解释现象。

### 1.2.3 如何实践结构化思维？
在实践结构化思维时，应该注意以下几点：

1. 建立层级关系。基于某个层级的大局观，构建该层级的结构模型。例如，为了解决组织内一个部门的问题，可以在该部门建立人员、业务、流程三个层级的结构模型。
2. 识别主干、支流。在每个层级的结构模型中，找出最主要的工作和关系，并分析这些关系以确定支流和主干。
3. 将目标域与行动域分离。在分析阶段，应该将目标域和行动域分开。目标域是在结构模型上面的，并试图描述所有的事情。行动域是在结构模型下的，它处理的只是个别任务。
4. 建立全面视图。在构建完所有层级的模型之后，总结出一个大的全景视图，即整个组织的结构。根据这个全景视图，可以评估当前组织的状况，判断是否需要增加或减少层级。
5. 持续改进。随着组织的发展和变化，需要不断重新构建金字塔结构，而且还需要随时调整模型以适应新的情况。

### 1.2.4 结构化思维在企业中的应用
结构化思维能够帮助企业解决以下两个问题：

1. **信息共享问题**：结构化思维可以帮助公司的各层级人员更有效地共享信息，并且减轻信息不匹配带来的负面影响。因为在结构化思维的框架下，各个层级的员工都是平等的，而不需要重复讲解相同的内容。这样一来，就可以避免重复、降低错误、节约时间、提升协作水平。
2. **管理层级优化问题**：结构化思维可以帮助管理层级的优化，从而简化管理和提升管理效率。公司的各层级人员一般都是紧密结合的，所以能在精细化的层级上建立起全面的视角，从而发现问题、寻求解决方案。再者，在结构化思维的框架下，不同层级的员工之间可以更直接地进行交流，避免了因各自工作职责不同导致的信息不匹配问题。
