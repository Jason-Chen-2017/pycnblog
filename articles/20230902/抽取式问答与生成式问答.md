# 1.简介
  

抽取式问答（Extractive Question Answering）与生成式问答（Generative Question Answering）是自然语言处理领域最火热的两大任务。两种方法虽然各有优缺点，但是在实际应用中往往存在着“鱼龙混杂”的局面——即既应用于数据集上训练出来的模型也会遇到任务特定的性质，如文本蕴含信息、多步推理等方面，难以统一处理。因此，如何选择合适的解决方案，是本文重点探讨的内容。
传统的机器学习方法往往是基于训练数据进行训练，而抽取式问答和生成式问答的方法则需要根据不同的数据类型制定不同的模型结构。而有了较强的理论支撑基础后，基于序列到序列（Seq2Seq）或条件随机场（CRF）等模型结构，可以进行多步推理的问题，以及涉及复杂实体连接、实体消岐等信息处理问题，从而提升模型效果。
2.抽取式问答与生成式问答的定义
2.1 抽取式问答
“抽取式问答”是指依据一段文字、图片或者视频中的信息，回答用户提出的简单问题。比如，对于文章“最近华为发布了全新的手机，售价为多少？”，回答问题“售价为多少？”就是抽取式问答。这种方法有时候比较简单直接，不需要构建复杂的模式识别模型；但由于依赖于关键词匹配，其结果往往不够准确。
2.2 生成式问答
相比于抽取式问答，“生成式问答”更加主观。它通过让系统生成符合用户意愿的答案来回答用户的问题。比如，给出一句话“你喜欢什么电影？”，生成系统可能输出“我喜欢《复仇者联盟》”。这种方法由于可以生成高质量的答案，所以应用范围更广。但由于需要独立实现文本生成模块，并基于此实现问答系统，因此相对比较复杂。
3.方法比较
前面我们提到，抽取式问答与生成式问答各有优缺点。那么它们之间有何区别呢？下面我们将详细介绍这两种方法的不同之处，并与当前主流方法比较。
3.1 语料库大小的影响
首先，因为抽取式问答和生成式问答都需要预先准备好的语料库，这就决定了它们所需数据的大小。一般来说，抽取式问答所需语料库小，生成式问答所需语料库大。但是，随着新闻报道的更新、新产品的发布、交互式助手的出现，语料库的大小也是日渐扩大的趋势。这时，抽取式问答模型可能会成为限制因素。
3.2 模型效率的影响
第二，抽取式问答模型往往采用规则、正则表达式或者其他简单方式进行特征工程，而不是深入地构造具有丰富表达能力的语言模型。因此，这些模型通常在效率方面表现不佳，需要更长的时间才能收敛。
相比于生成式问答，抽取式问答模型对于新闻文章等场景的效果要好一些，但是当遇到复杂的问题时，例如关于医疗健康的科普问题，它们的效果就会变得很差。
3.3 数据分布的影响
第三，抽取式问答模型对样本分布要求很高。一般情况下，需要训练的数据是用户提出的问题-答案对的语料库。对于一些数据集，即使抽取出了一部分样本，但如果模型没有足够的训练样本，最终的结果可能仍然会偏离目标值。
相比于生成式问答模型，抽取式问答模型对于数据分布的鲁棒性要求不太严格。只要原始数据满足一定要求，模型就可以训练出来。
3.4 任务复杂度的影响
第四，抽取式问答模型的任务一般简单些。因为主要目的在于直接回答简单问题。但如果遇到了复杂的问题，比如如何处理多个实体之间的关系、如何回答文本蕴含信息等，模型的性能就很难保证。
相比于生成式问答模型，抽取式问答模型能够处理多步推理的问题，如对话系统。
综上所述，抽取式问答模型虽然容易上手，但在某些特定领域和任务下，它的性能还不够理想。而生成式问答模型的优势在于可以处理复杂的问题，但是由于需要独立实现文本生成模块，它往往更耗费时间和精力，并且需要很强的硬件条件。不过，随着计算能力的增强，未来生成式问答模型的发展趋势还是会越来越多。
4.总结
综上所述，抽取式问答与生成式问答是自然语言处理领域两个重要且热门的任务。这两种方法虽然各有优缺点，但它们都能为我们的生活提供帮助，而且目前还有许多研究工作正在探索两者之间的融合、突破和优化。因此，掌握这两种方法，对于我们理解自然语言、人机交互、知识图谱、机器学习等领域都非常重要。
