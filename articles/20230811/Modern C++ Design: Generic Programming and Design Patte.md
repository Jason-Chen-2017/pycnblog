# 1.简介
         

C++ 是一门具有历史悠久、广泛使用的高级编程语言。在过去十多年里，C++ 在可移植性方面一直占有很大的优势，在许多领域都得到了应用。但是随着近几年来C++ 社区逐渐成熟，其中的一些设计模式也逐渐成为众人皆知的“经典范式”。本文将通过“通俗易懂”的方式带领读者学习与掌握这些经典设计模式的原理与运用方法，并提出一系列的问题，供读者反思与思考。

# 2.核心概念和术语
## 模板（Template）
模板是 C++ 的一个重要特性，它提供了一种类似函数重载的机制，可以在编译时期对类型进行参数化，实现泛型编程。模板可以定义一个泛型类或函数，这种类或函数能够接受不同的数据类型作为参数，从而在编译期间生成不同的代码，适应不同的需求。比如，可以定义一个容器类，该类可以接收不同类型的元素作为参数。

## 泛型编程
泛型编程是指编写高度抽象且可复用的代码，不依赖于任何特定数据类型或结构，而是可以适应任意类型的数据和算法。泛型编程的关键是通过泛型算法与容器等抽象机制来避免重复的代码，让代码更加模块化、可维护和可扩展。

## 抽象数据类型（Abstract Data Type，ADT）
抽象数据类型（Abstract Data Type，ADT）是指用来描述系统数据结构的集合，同时隐藏其内部细节。通过 ADT，可以实现数据与逻辑之间的分离，支持不同的数据结构之间互相转换，促进代码的模块化和可维护性。

## 迭代器（Iterator）
迭代器是一个对象，它用于访问集合中的元素。迭代器封装了集合中元素的顺序或者其他特征，使得我们可以通过统一的接口访问集合中的元素。通过迭代器，可以方便地遍历整个集合，获得集合中的元素。

## 策略模式（Strategy pattern）
策略模式是行为型设计模式之一，它允许创建表示各种算法的方法接口，并且可以根据不同的需要使用不同的算法。策略模式定义了算法族，分别封装起来，让它们之间可以互换。

## 命令模式（Command pattern）
命令模式是行为型设计模式之一，它是用来将请求封装为一个对象，从而使你可用不同的请求对客户进行参数化；对请求排队或者记录请求日志，以及支持可撤销的操作。命令模式通常与组合模式结合使用，用于实现详细的请求协议。

## 责任链模式（Chain of Responsibility pattern）
责任链模式是行为型设计模式之一，它使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。这种模式给予客户端添加新的处理动作的灵活性。

## 适配器模式（Adapter pattern）
适配器模式是结构型设计模式之一，它能把某个类的接口变换成客户端所期待的另一个接口。适配器模式主要涉及到两个角色——源（Adaptee）角色和目标（Target）角色。源角色是被适配的类，目标角色是要求客户端的接口。

## 桥接模式（Bridge pattern）
桥接模式是结构型设计模式之一，它是为了解决两个变化维度（即变化点）而产生的。桥接模式将一个大类对象与两个相互独立的子类对象之间建立一个中间层，这个中间层可在各个子类对象之间转化信息。通过桥接模式，子类的功能可以单独完成，但由于继承了中间层的通用功能，因此可以减少它们之间的耦合程度。

## 装饰器模式（Decorator pattern）
装饰器模式是结构型设计模式之一，它是为已有的对象增加新职责而动态地添加功能的方式。通过引入装饰器，可以在运行时动态地为对象添加额外的职责。

## 代理模式（Proxy pattern）
代理模式是结构型设计模式之一，它提供一个替代品或代表者，并由代理执行某些任务。代理模式通常与委托模式结合使用，提供对对象的访问权限。
