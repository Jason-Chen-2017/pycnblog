# 1.简介
         

云计算是一种新型的计算模式，使得用户可以像租用服务器一样，购买到配置好的计算环境，其核心特征是按需付费。云计算的大多数服务都依赖于分布式计算技术，包括集群运算、并行计算等。但随着云计算平台的日益壮大，平台上可供使用的计算资源也越来越多。因此，云服务提供商需要能够应对不断增长的用户需求，实现弹性计算资源的快速扩容或缩容，确保计算服务的可用性。

弹性计算（Elastic Computing）是指云计算中提供的一种计算资源的能力可以随时增加或者减少，适应用户的需求，使得计算服务始终保持高可用状态。弹性计算的主要目标是满足用户对计算资源的按需要求，提升资源利用率，降低成本。

## 2.弹性计算的定义及特征
弹性计算是一个云计算服务的核心特征，它的定义由AWS在2011年首次提出，它包括两个方面：

1. 弹性伸缩性：即云计算平台可以自动地向上或向下扩展计算资源的能力，从而响应用户的计算需求，保证计算服务的稳定运行。
2. 弹性可用性：即云计算平台对服务的故障和硬件故障具有一定的容错能力，保证计算服务的高可用性。

弹性计算可以分为三种类型：

1. 垂直弹性：即调整计算资源的规格，比如调整CPU核数、内存大小等。
2. 水平弹性：即添加或删除计算资源，比如添加新的计算节点或销毁已有的节点。
3. 混合弹性：既能实现垂直弹性，又能实现水平弹性。

目前，AWS、Google Cloud Platform、阿里云等主流云服务提供商均支持弹性计算。

## 3.弹性计算的特点
### 3.1 简单易用
弹性计算的特性要求云计算平台应当能实现用户操作界面简洁、易用。用户只需按照简单的规则设置即可自动弹性调整计算资源，不需要了解底层的操作指令。这种简化的操作方式将降低用户的学习难度，让更多的人参与到弹性计算的开发和部署中来。

### 3.2 抗住突发事件
弹性计算能够抵御突发事件，如服务器宕机、硬盘故障等。由于弹性计算平台采用了集群式架构，因此无论是服务器故障还是硬件故障，都可以及时发现并补救，避免造成不可挽回的损失。

### 3.3 节约成本
弹性计算能节省云计算平台的运营成本。无论是初期的计算资源投入，还是后期的小量订单，弹性计算都能帮助用户降低云计算平台的总体支出。

### 3.4 实现多样化应用场景
弹性计算能够兼顾弹性伸缩性和弹性可用性，能支持多种应用场景，如实时计算、分析计算、大数据处理等。用户只需选择合适的计算配置，就可以快速地启动计算任务。这样一来，弹性计算就能很好地满足云计算平台用户的不同诉求。
