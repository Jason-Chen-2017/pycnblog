# 1.简介
         

作为一名软件工程师或互联网从业者，除了会写代码外，也要具备良好的编码能力、逻辑思维能力、时间管理能力、沟通协调能力等方面的知识。为了更好的应对日益复杂的IT工作环境，提升自身的综合素质和竞争力，现在越来越多的人开始关注软件开发领域的软实力。

而“软实力”背后，往往隐藏着一个更为关键的问题——如何培养自己成为一个真正的“软件工程师”。毕竟，写出让别人能够快速读懂的代码并不能完全体现自己的编程能力；甚至，能够把一个看似简单的问题变得难上加难也是软件工程师的强项。

很多人在刚接触到这个方向时，都会感觉莫名其妙。对于很多软件工程师来说，成长的过程，似乎都没有什么大的意义。因为自己做不到“优秀”，所以，只好接受现状，为生计打拼。但其实，真正想要成长的是自己的思维方式、职业道德，以及学习能力等软实力。只有把这些因素投入到日常的工作中，才能有所收获。

因此，要想真正突破这一困境，首先需要了解一下这个行业的“理论基础”。只有了解了这些基本原理之后，才可能有助于你解决实际问题。否则，可能会陷入到“认知盲区”，不知道该怎么下手。

那么，《7.软件工程师成长之路——迷茫时刻如何突破困境》这篇文章，就将告诉你一些关于软件工程师成长、软技能训练、职业规划等方面的相关知识和方法论。希望通过阅读这篇文章，你能受益匪浅！

# 2.什么是软件工程？
“软件工程”是一个广义的词汇，它可以泛指软件开发及其相关的一系列活动，包括需求分析、设计、编码、测试、维护、运营等。“软件工程师”通常是指具有以上全部责任的专门人员。相对于一般工程师而言，软件工程师更多地负责整个项目生命周期中的一个环节，他们处理的是复杂度最高的技术性问题。

由于软件工程师面临的特殊需求，导致其专业化程度很高。比如，在需求分析环节，软件工程师需要充分考虑到用户需求、商业需求、技术需求等方面，以确保产品能够满足需求。在设计环节，软件工程师需要仔细观察目标用户的行为习惯、情绪特征、喜好偏好等，从而创造出符合用户需求的产品。在编码环节，软件工程师需要不断迭代和优化产品的实现方案，以保证最终达到预期效果。在维护环节，软件工程师需要不断修复Bug，确保软件的运行稳定可靠。

因此，要想成为一个合格的软件工程师，除了扎实的计算机科学基础外，还需掌握一些软件工程相关的专业技能。其中最重要的就是编程能力，也就是我们今天主要谈到的主题。

# 3.为什么要学编程？
虽然编程并不是软件工程师的一项必备技能，但是它无处不在。既然要写程序，那不如从一个小例子开始吧。

假设你要编写一个程序，能够根据输入的年龄打印出对应的生肖（鼠、牛、虎、兔、龙、蛇、马、羊、猴、鸡、狗）。你可能会这样思考：“怎么才能判断一个人是鼠还是猴呢？难道我要写一个万能的函数吗？”。

这种想法不错，不过你要想得太简单了。比如，岁月磨平后的“猴子补丁”，只适用于有一定计算机基础的程序员，且只能解决特定情况，无法应用到其他场景。因此，软件工程师需要更多地涉及技术问题，才能有较好的编程能力。

举个例子来说，假设你负责一个开源项目，你需要向用户展示某款软件的使用方法。你可能会选择先编写用户手册，然后再编写教程视频。这种方式虽然简单易行，但缺乏动手能力，只会束缚你的思维。软件工程师往往擅长通过编程的方式解决复杂的问题，以及利用编程语言自带的特性解决一些重复性的问题。

总结一下，学编程，并非一件孤立无援的事。理解程序的运行机制，研究各种算法和数据结构，掌握编程语言的各种特性，能够帮助你在实际工作中游刃有余地解决问题。同时，善于沟通和协调，能够让你的编程工作与团队成员相融洽，共同进步。
