# 2.前言
## 数据结构
在计算机程序设计中，数据结构是指存储、组织数据的方式。它是计算机中信息的逻辑结构，是指对数据的组织、存储方式的描述，并非实际存在的计算机实体。数据结构与算法是密不可分的两个部分，只有理解了数据结构才能更好地理解算法。数据结构又可以细分为以下几种类型：

1. 集合：集合（Collection）是指一些值的集合，比如数组、链表、队列、栈等，这些数据结构中的元素都可以重复出现，无序排列。典型的集合数据结构包括数组、列表、元组、集合。

2. 线性表：线性表（Linear List）是最简单的一种数据结构，它由相同类型的数据元素按顺序排列而成，每个元素称作节点（Node）。最基本的线性表就是数组，如同生活中使用的一样。数组中每个位置可以存放一个值，并且下标是固定的。典型的线性表数据结构包括数组、链表、队列、栈等。

3. 树形结构：树形结构（Tree Structure）是一种非线性结构，它由一组无限个节点组成，根节点到叶子节点的每一条路径都代表着一种分支。树形结构常用于表示层次化数据，如文件系统、文件夹目录等。树形结构包括二叉树、排序二叉树、平衡二叉树、完全二叉树、霍夫曼编码树、B树、B+树、B*树等。

4. 图论结构：图论结构（Graph Theory）是指用来研究相互关系的一系列对象，它包含结点（Vertex）、边（Edge）、路径、回路等概念。图论结构经常用以处理网络结构、计算最短路径、建模复杂系统等方面。图论结构中最常用的有邻接矩阵、邻接表、十字链表、邻接多重链表等。图论结构数据结构包括图、网格图、树状图等。

## 算法
算法（Algorithm）是指用来解决特定问题的一组指令或操作，它是完成某个任务所需的一系列步骤。算法通常采用抽象的形式定义，使得输入数据、输出结果的具体含义不变，便于不同人的理解和实施。由于算法有时需要依赖于数学理论或其他基础知识，所以它的正确性也容易被质疑。但任何算法都可以通过某些测试来验证其准确性和效率，因此，掌握各类算法、优化算法，并且在实际项目中运用它们，是成为一名优秀的软件工程师的必备技能。

算法又可以进一步分为三类：

1. 查找算法：查找算法（Search Algorithm）是指在已知数据集合中找到给定的值或者寻找满足指定条件的元素，其主要任务是在不需比较的情况下确定元素的位置或者索引。典型的查找算法包括顺序查找、二分查找、Hash查找等。

2. 插入算法：插入算法（Insertion Algorithm）是指向已排序好的元素序列中插入新元素，其目标是保持元素的有序性，同时还要保证插入的元素仍然处于正确的位置。典型的插入算法包括直接插入法、折半插入法、斜对称插入法等。

3. 排序算法：排序算法（Sorting Algorithm）是指将一组数据依照一定规则进行排列，使之方便检索、搜索和修改。排序算法可以归纳为内部排序算法（Internal Sorting Algorithm）和外部排序算法（External Sorting Algorithm），其中内部排序算法把记录存放在内存中进行排序，而外部排序算法则需要磁盘空间和外存资源。典型的内部排序算法包括冒泡排序、选择排序、插入排序、希尔排序、快速排序、堆排序等；典型的外部排序算法包括桶排序、基数排序、外部合并排序等。
