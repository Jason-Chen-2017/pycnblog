# 2.概览
## 2.1 大数据概述
随着互联网和移动互联网的发展，互联网用户的数据量越来越大，每天产生的数据量也在呈几何倍数增长。这些数据的处理通常依赖于一些高性能的大数据处理平台，如Hadoop、Spark等。数据处理平台的构建分为离线处理和实时处理两大类。
### 2.1.1 Hadoop
Hadoop(开放源代码的分布式计算框架)是由Apache基金会所开发的一个框架。该框架是一个基于HDFS(Hadoop Distributed File System)的分布式存储和计算平台，能够运行Hadoop所需的MapReduce运算程序。HDFS是一个高度容错的分布式文件系统，能够存储超大数据集的文件。Hadoop能够提供高吞吐量的数据访问能力，是构建大数据处理平台不可缺少的组件。Hadoop被广泛应用在电信、网络、金融、广告、搜索引擎、GPS定位、GIS（Geographic Information System）、推荐系统等领域。
### 2.1.2 Spark
Spark是基于内存计算的快速分布式数据处理框架，由UC Berkeley AMPLab开发，是一个开源的集群计算系统。Spark是建立在Hadoop MapReduce基础上的，它独特的高性能特性使其成为大数据分析的理想选择。Spark能够在毫秒或微秒级别内处理TB到PB的数据量，同时Spark提供Scala、Java、Python、R语言API接口，支持各种高级数据分析工具。目前，Spark已成为大数据领域最热门的框架，在各个行业均取得了成功。

## 2.2 数据仓库概述
数据仓库是企业用来收集、整理、分析和报告业务数据的信息系统。数据仓库是一个中心化的数据库系统，它包含多个主题建模实体、属性以及它们之间的关系，以支持复杂的分析工作。数据仓库的价值在于为决策制定人员、管理层和其他部门提供统一的、准确的、可靠的、透明的关于组织运营状况的信息。
数据仓库通常包括如下四个部分：
- 维度建模（Dimension Modeling）：就是对企业的关键业务活动、资源、产品、客户、市场的抽象。
- 事实表（Fact Table）：记录了企业的核心交易活动信息。
- 维度表（Dimension Table）：记录了企业中的各个维度的信息。
- 星型模型（Star Schema）：是一种较为简单的维度建模方法，其特点是使用一个星型模型来连接所有相关的维度表。
## 2.3 Apache Hive概述
Apache Hive是基于Hadoop的一种数据仓库工具。它能够将结构化的数据文件映射为一张表格，并提供完整的SQL查询功能，支持复杂的MapReduce计算框架及关联查询、联接、聚合等高级分析功能。由于其简单易用、部署容易、扩展性强、速度快等优点，Apache Hive已经成为众多大数据项目的重要组成部分。
Apache Hive的优点主要有以下几点：
- 简单易用：使用Hive简单而且熟悉，学习成本低，语法上与传统SQL语句相似。
- 部署方便：Hive不需要单独安装，只需要把jar包加入classpath即可。
- 扩展性强：Hive允许动态分区、动态加载表。支持自动索引生成，避免了手工创建索引的麻烦。
- 查询速度快：Hive采用Lazy Execution模式，即只有当查询语句执行时，才真正执行相应的查询操作，所以查询速度非常快。
- 可移植性好：Hive可以通过Thrift等协议与Hadoop平台无缝集成。
