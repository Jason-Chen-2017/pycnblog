# 1.简介
         

## 为什么我们的团队缺乏对安全计算领域的研究？
目前，在信息化、云计算等新时代背景下，互联网公司为了应对业务的不断增长和复杂性，越来越多地将重点转移到安全上，开发出了各种安全解决方案，包括加密、身份认证、访问控制、边界防护、网络安全等等。然而，由于安全领域是一个非常具有开放性和创造性的领域，各个公司都可以根据自身的业务特点制定适合自己的安全解决方案。因此，作为一家中型互联网公司，当我们面临新的安全威胁和问题时，往往需要从各个方面进行充分的调研和研究，才能找到最佳的解决方法。但实际上，我们的公司却没有足够的专业知识和研究力量来完整研究和实现安全计算领域的科研成果。这一缺陷导致了我们公司在相关领域的研究水平不高，严重影响了我们公司对相关领域的管理能力、决策过程、运营效率和产品质量的把控。因此，为了克服这个困难，我们提出了一个更加具体的要求：我们的团队是否缺乏对安全计算领域的研究？如果是这样的话，那么我们应该如何研究该领域，并且将自己的研究结果与其他行业的研究结果做结合？最后，我们希望通过这篇文章，能够引导大家对于安全计算领域的研究状态进行更深入地理解和分析，并提供相应的方向供大家参考。
## 定义
安全计算（Security Computing）是一个从物理层面上保障通信及计算系统安全的领域。在该领域中，专门研究系统的信息、数据和通信安全，并试图构建符合国家信息安全政策、法律要求、社会规范的安全计算平台。其目标是建立安全计算平台，使得各类机密计算任务可以在可信赖的环境中完成，同时保持系统的运行稳定、可用性及隐私保护。
## 安全计算的重要性
随着人工智能、云计算、区块链技术等新兴技术的日益普及，传统的服务器、存储设备等硬件已经无法满足信息处理、网络传输、计算需求的需求。为了突破硬件的限制，出现了虚拟机技术。但是，虚拟机的资源隔离机制可能导致安全风险。另外，多租户的云计算服务也要求不同租户间的资源隔离。基于此，出现了利用容器技术来构建安全计算平台的需求。虽然容器技术本身具有良好的隔离性，但仍存在一定的安全隐患，如恶意攻击、滥用权限等。因此，安全计算作为一种技术领域的研究，极大地促进了信息技术行业的发展。
# 2.基本概念术语说明
## 概念
安全计算的核心是一个可信执行环境（Trusted Execution Environment，TEE），它是一个独立于主存、CPU和外围设备的安全区域，用于存储敏感信息，并由其中的安全计算模块来执行指令。在TEE中，应用可以直接访问TEE内的数据和功能，并只能通过TEE中的安全计算模块来处理这些数据。安全计算的关键在于保障应用只能在信任的环境里运行，而不是任意的代码执行。除此之外，还需要保证TEE的数据在被泄露或篡改之前得到有效的保护。
## 术语
### TEE
Trusted Execution Environment，可信执行环境。在安全计算领域，TEE是一个独立于主存、CPU和外围设备的安全区域，用于存储敏告信息，并由其中的安全计算模块来执行指令。在TEE中，应用只能通过安全计算模块来处理这些数据，不能直接访问TEE内的数据和功能。

### Trusted System
可信系统。在安全计算领域，可信系统是一个指系统，它的操作系统和应用程序都受到高度信任，可以说它是完全受信任的。这样的系统可以部署在安全计算平台上，提供高级别的安全服务。

### Sandboxing
沙箱环境。沙箱环境是指将一个计算机系统划分成多个相互隔离的小系统，每个小系统只允许运行指定程序，且与外部隔离。这样做可以防止病毒和恶意程序从一个系统中渗透到另一个系统中。一般来说，沙箱环境属于微内核设计范畴。

### Formal Verification
正式验证。所谓的正式验证就是通过一系列测试用例来检测系统的正确性、安全性、完整性等特性。这通常是通过编写程序逻辑的形式描述语言，再使用自动化工具对生成的程序代码进行检查，从而确保程序的正确性、健壮性、安全性等。

### Dependability
可靠性。可靠性，顾名思义，就是指某个系统工作正常持续时间长短的能力。比如一个系统的可靠性就是指，当给定相同输入情况下，该系统在一段时间后依然表现出正确的行为。安全计算主要研究的是如何建立可靠性很高的安全计算平台。

### Trustworthy Computation
可信计算。在安全计算的概念中，可信计算即指一个计算任务或者程序可以运行在一个可信系统上，并且可以在这种系统中获得可信的结果。换句话说，可信计算是指，某个计算任务在某个特定环境下可以获得预期的输出，而且无需担心被恶意的攻击者操纵。

### Attacker Model
攻击模型。攻击者模型是指，假设有一个攻击者，他有某些目的，要对某种计算服务、资源或者数据的访问进行攻击。他可以采取什么样的手段，取得什么样的效果。

### Input-Dependent Output (IDO) Attack
输入依赖输出攻击。输入依赖输出攻击又称IDA，它是指，攻击者既拥有某种计算服务，也能够构造一些特殊的输入，使得计算结果发生变化，且不会对原始输入或计算结果造成任何影响。例如，当一个数据聚类算法完成后，可以向其中添加一些错误数据点，以达到欺骗聚类的目的。
