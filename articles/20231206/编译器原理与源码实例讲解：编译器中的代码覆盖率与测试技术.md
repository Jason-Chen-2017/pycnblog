### 3.1代码覆盖率的计算

代码覆盖率是一种衡量程序测试质量的指标，它表示测试用例中所覆盖的代码行数占总代码行数的比例。计算代码覆盖率的公式如下：

$$
覆盖率 = \frac{被覆盖代码行数}{总代码行数}
$$

### 3.2单元测试

单元测试是一种测试方法，它涉及到对程序的单个函数或模块进行测试。单元测试的目的是验证程序的单个函数或模块是否正确工作。

单元测试的具体操作步骤如下：

1. 编写测试用例：编写一个或多个用于测试程序的单个函数或模块的测试用例。
2. 执行测试用例：执行测试用例，并检查测试结果是否符合预期。
3. 修改程序：根据测试结果修改程序，直到所有测试用例都通过。

### 3.3集成测试

集成测试是一种测试方法，它涉及到对程序的多个函数或模块进行测试。集成测试的目的是验证程序的多个函数或模块是否正确工作。

集成测试的具体操作步骤如下：

1. 编写测试用例：编写一个或多个用于测试程序的多个函数或模块的测试用例。
2. 执行测试用例：执行测试用例，并检查测试结果是否符合预期。
3. 修改程序：根据测试结果修改程序，直到所有测试用例都通过。

### 3.4系统测试

系统测试是一种测试方法，它涉及到对整个程序进行测试。系统测试的目的是验证程序是否满足用户需求。

### 4.3集成测试

```python

    return a - b

    assert add(1, 2) == 3
    assert add(-1, 1) == 0
    assert add(0, 0) == 0
    assert sub(1, 2) == -1
    assert sub(-1, 1) == 0
    assert sub(0, 0) == 0

test_add_sub()
```

在这个例子中，我们编写了一个名为`test_add_sub`的函数，用于测试`add`和`sub`函数。我们使用`assert`语句来检查`add`和`sub`函数的结果是否与预期一致。如果所有的`assert`语句都通过，那么我们可以说`add`和`sub`函数是正确的。
