# 2.核心概念与联系

在深入探讨Kubernetes的设计原理之前，我们需要了解一些核心概念。这些概念包括容器、集群、节点、Pod、服务、部署、配置映射、状态设置、资源请求和限制等。

容器是一种轻量级的软件包装，它包含了应用程序的所有依赖项和运行时环境。容器可以在任何支持容器化的平台上运行，无需安装任何软件。

集群是一组相互连接的计算节点，它们共同组成一个大型的计算资源池。集群可以包含多个节点，每个节点都可以运行多个容器。

节点是集群中的计算机，它们负责运行容器化的应用程序。节点可以是物理服务器、虚拟服务器或云服务器。

Pod是Kubernetes中的基本部署单元，它包含了一个或多个容器。Pod是Kubernetes中最小的部署单位，它们可以在集群中的任何节点上运行。

服务是Kubernetes中的抽象层，它允许用户在集群中创建和管理应用程序的网络服务。服务可以将多个Pod连接在一起，以实现负载均衡和故障转移。

部署是Kubernetes中的一种资源，它用于定义和管理应用程序的多个版本。部署可以用于自动化应用程序的部署、滚动更新和回滚。

配置映射是Kubernetes中的一种资源，它用于存储和管理应用程序的配置信息。配置映射可以用于存储环境变量、文件和其他配置信息。

状态设置是Kubernetes中的一种资源，它用于存储和管理应用程序的状态信息。状态设置可以用于存储数据库连接信息、缓存配置和其他状态信息。

资源请求和限制是Kubernetes中的一种资源管理机制，它用于控制容器的资源使用情况。资源请求用于指定容器需要的最小资源，而资源限制用于指定容器可以使用的最大资源。
