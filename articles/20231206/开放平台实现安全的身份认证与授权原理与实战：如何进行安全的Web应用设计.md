# 2.核心概念与联系

在讨论身份认证与授权原理之前，我们需要了解一些核心概念。

## 2.1 身份认证

身份认证是一种验证用户身份的过程，通常涉及到用户提供凭据（如密码、身份证或驾驶证等）以证明自己是谁。身份认证的目的是确保用户是真实的，并且可以访问Web应用程序的资源。

## 2.2 授权

授权是一种确定用户可以访问Web应用程序的资源的过程。授权涉及到确定用户的权限，并根据这些权限来限制他们可以访问的资源。授权的目的是确保用户只能访问他们具有权限的资源，以保护Web应用程序的安全性。

## 2.3 联系

身份认证与授权是密切相关的，因为它们都涉及到确保Web应用程序的安全性。身份认证确保用户是真实的，而授权确保用户只能访问他们具有权限的资源。这两个过程一起工作，可以确保Web应用程序的安全性。
