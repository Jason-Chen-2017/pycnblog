# 2.核心概念与联系

在分布式系统中，我们需要关注以下几个核心概念：

1.分布式一致性：分布式系统中的多个节点需要保持一致的状态，以确保数据的准确性和完整性。

2.分布式事务：在分布式系统中，我们需要处理跨多个节点的事务，以确保事务的原子性、一致性和隔离性。

3.分布式存储：分布式系统需要一个分布式存储系统，以存储和管理数据。

4.分布式计算：分布式系统需要一个分布式计算框架，以实现并行和分布式计算。

5.分布式系统的故障容错：分布式系统需要具备故障容错的能力，以确保系统的可用性和稳定性。

- 准备阶段：在这个阶段，节点会向领导者发送准备消息，表示它们是否准备好接受这个值。如果领导者收到足够多的准备消息，它会进入决策阶段。
- 决策阶段：在这个阶段，领导者会向节点发送决策消息，通知它们接受这个值。节点会将这个值存储到本地状态中。

2.Raft算法：Raft是一种一致性算法，它可以在分布式系统中实现一致性和故障容错。Raft算法的核心思想是通过选举来实现一致性决策和故障容错。具体来说，Raft算法包括以下几个步骤：

- 日志复制阶段：在这个阶段，领导者会将日志复制到其他节点，以确保所有节点的日志是一致的。
- 状态转移阶段：在这个阶段，节点会根据领导者的状态进行状态转移。如果领导者发生故障，其他节点会选举出新的领导者。

- 投票阶段：在这个阶段，节点会向其他节点发送投票消息，表示它们是否支持这个值。如果足够多的节点支持这个值，它会被选为一致性决策。
- 决策阶段：在这个阶段，节点会将一致性决策存储到本地状态中。
