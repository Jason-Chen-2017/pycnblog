## 2.1 容器化技术的基本概念

# 4.具体代码实例和详细解释说明

在本节中，我们将通过详细的代码示例来帮助开发者更好地理解和应用容器化技术。

## 4.1 创建Docker文件

创建Docker文件是创建Docker镜像的第一步。Docker文件是一个用于定义Docker镜像的配置文件。Docker文件中可以定义软件应用和其依赖的运行时环境。

以下是一个简单的Docker文件示例：

```
FROM ubuntu:18.04

RUN apt-get update && \
    apt-get install -y python3

WORKDIR /app

COPY . /app

CMD ["python3", "app.py"]
```

在这个Docker文件中，我们定义了一个基于Ubuntu 18.04的Docker镜像。我们使用RUN命令来安装Python3。我们使用WORKDIR命令来设置工作目录。我们使用COPY命令来将当前目录复制到容器的工作目录。我们使用CMD命令来设置容器启动时的命令。

## 4.2 构建Docker镜像

根据Docker文件，可以使用Docker命令来构建Docker镜像。以下是构建Docker镜像的命令：

```
docker build -t my-image .
```

在这个命令中，我们使用docker build命令来构建Docker镜像。我们使用-t选项来设置镜像的标签。我们使用.选项来指定Docker文件所在的目录。

## 4.3 启动Docker容器

根据Docker镜像，可以使用Docker命令来启动Docker容器。以下是启动Docker容器的命令：

```
docker run -p 5000:5000 my-image
```

在这个命令中，我们使用docker run命令来启动Docker容器。我们使用-p选项来设置容器的端口映射。我们使用my-image选项来指定容器的镜像。
