# 2.基本概念术语说明
## 数据可扩展性架构（Scalable Data Architecture）
数据可扩展性架构(Scalable Data Architecture)是指能够有效应对变化的业务需求的技术体系。其目标是构建一个高度可靠、可扩展的存储、计算、网络、依赖管理等层级结构的架构，能够满足多种类型的计算任务，包括实时计算、批处理、流处理、机器学习、图数据库、搜索引擎、日志分析、消息队列、缓存系统、对象存储等。

目前的数据可扩展性架构主要由存储、计算、网络、依赖管理四个层级组成。其中，存储层包含大量数据集、结构化、半结构化甚至非结构化的数据存储系统；计算层则包含各种数据计算服务，如实时计算、批处理、流处理、机器学习、图数据库等；网络层则提供用户接口、客户端/服务器、消息传递、分布式事务等网络通信协议支持；依赖管理层则管理所有组件之间的依赖关系。

## 云原生架构
云原生架构（Cloud Native Architecture）是一种新的应用程序开发方法论和软件架构设计理念，它主要关注于如何通过基础设施即服务（Infrastructure as a Service，简称 IaaS），平台即服务（Platform as a Service，简称 PaaS），软件即服务（Software as a Service，简称 SaaS），以及自动化运维工具的结合，打造出具备可扩展性、弹性伸缩能力、灵活迁移能力的现代化应用程序。它是以云计算为核心，基于容器技术、微服务架构、康威定律和不可变基础设施等理念，构建分布式应用架构的一系列模式和原则。

云原生架构包括容器编排技术、微服务架构、弹性伸缩策略、服务网格、事件驱动架构、声明式API等技术。这些技术或方法可以帮助企业在更短的时间内获得竞争优势，降低创新和投资风险，提升整体效益。另外，云原生架构也能够降低开发和运维人员的技术门槛，使得业务人员可以聚焦于核心业务，而不需要关心底层架构。因此，云原生架构已经成为新一代应用架构的标杆。
