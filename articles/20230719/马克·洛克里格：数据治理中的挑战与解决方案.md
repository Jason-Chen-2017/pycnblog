# 2.基本概念术语说明
数据治理的定义：数据治理(Data Governance)是指数据的开放、可信、完整、准确、可追溯、价值导向和应用广泛的过程中，围绕数据产生的各种流程和机制，负责保障数据质量、服务实体、满足用户需求、保护用户隐私和个人权利，以及维护数据价值的一个综合性工作。

数据治理核心问题：数据治理的核心问题主要包括以下几个方面：

1. 数据质量：如何保障数据质量？什么是数据质量的定义？数据质量评估的标准、方法、工具是什么？

2. 服务实体：数据治理需要保障哪些实体的服务？数据的管理者、使用者、数据提供者之间的关系是怎样的？

3. 用户需求：数据管理应关注用户在使用数据时的需求，哪些数据是最重要的？如何满足用户的数据需求？

4. 用户隐私和个人权利：数据治理如何保护用户隐私和个人权利？用户的数据如何被泄露或篡改？数据需要具备哪些安全措施？

5. 数据价值：数据管理面临的最大挑战是如何保证数据价值？数据可以带来哪些价值？如何做到数据价值最大化？

# 6.附录常见问题与解答
1. 有哪些数据治理方法论？

数据治理方法论是指对数据治理过程和管理方法的整体观点和理解，它强调“道”（Government of Data）的重要作用。在数据治理的过程中，可以通过遵循一定的方法论指导原则，实现数据源头的纠错，加强数据的流通，以及制定数据管理制度、监督检查、协同配合等一系列工作。主要的方法论有四个方面：

1. 清晰的定义：数据治理是关于管理信息资源的行为规范，旨在确保数据的开放性、安全性、完整性、可信度、可用性、和效用性，即数据源头的正确性、完整性、有效性、时效性，并且能够适应业务发展的需要；

2. 优先考虑用户：数据治理是以用户为中心，充分尊重用户的意愿，优化数据管理和交付方式，保障数据主体的合法权益；

3. 迭代式管理：数据治理是一项具有生命周期、动态变化的活动，不同阶段的管理与使用目标可能存在差异，因此需要按实际情况不断调整；

4. 敏捷的融合：数据治理的前期是理论研究和方法论的研究，后期则是应用于实际运行的实际场景中。

总结来说，数据治理的目的是为所有用户提供一个高质量可靠的服务，既要尊重用户的需求，又要保障用户的合法权益。有效的、充满弹性的管理制度，是数据治理面临的关键难题。

2. 什么是数据仓库？

数据仓库是一个集成了企业所需信息的大型数据库，用于支持决策支持、报告和分析等应用。数据仓库是一个面向主题的仓库，通常由多个数据源（如交易系统、销售订单、库存系统等）汇总而成。其主要作用是进行复杂查询和分析，并使得存储的数据易于理解和使用。数据仓库的特点是集中存储、统一数据模型和抽取逻辑、集成不同数据源、保持历史记录、可读性强。

3. 数据治理中的数据质量体系有哪些？

数据质量体系主要分为数据质量建模、数据质量鉴别、数据质量控制、数据质量反馈、数据质量管理五大类。

1. 数据质量建模：对数据源头的质量建模，确定数据从源头到最终目的经历过什么样的过程；

2. 数据质量鉴别：检查数据质量的方法包括结构、编码、标准、一致性等方面，需要按照相应的标准验证数据质量；

3. 数据质量控制：对数据的质量进行监控，确保数据的质量得到有效控制；

4. 数据质量反馈：反馈数据质量的问题，提升数据质量，开展数据质量相关的研发投入；

5. 数据质量管理：根据数据质量建模、数据质量鉴别、数据质量控制、数据质量反馈等过程，制定数据质量管理政策和规范，对数据质量进行管理。

4. 为何数据治理很重要？

- 数据治理成为互联网时代数据治理的一个重要领域，数据的价值和价值主张直接决定着企业的竞争优势，数据治理保障了数据的可靠、准确和真实。

- 数据治理是为了提升组织效率、优化决策、改善业务流程、促进创新等，从而提升企业的竞争力和发展能力。

- 数据治理需要落实到每一个环节，包括数据源头、采集、转换、加载、存储、访问、分析、展示、发现、规划、审核、流通和消费等，这条路还需要继续走下去。
