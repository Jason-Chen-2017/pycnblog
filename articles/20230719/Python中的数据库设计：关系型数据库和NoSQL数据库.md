# 2.基本概念术语说明
首先，我们要搞清楚一些基本概念和术语。
## RDBMS(Relational Database Management System)关系型数据库管理系统
关系型数据库管理系统是建立在关系模型上的数据库，其中的数据表之间存在相互联系，通过键（Key）来关联。如下图所示:

![image.png](attachment:image.png)

如上图所示，关系型数据库管理系统可以划分为四层，分别是：

1. 数据定义层：负责对数据库对象的创建、修改和删除；
2. 数据操纵层：负责插入、更新、查询和删除数据；
3. 数据库操作层：向调用者返回结果或者影响记录条数；
4. 数据库独立性：该层为数据库提供了接口屏蔽，使得它无需关注底层硬件结构，只需要知道提供的接口就可以完成各种操作。

其中，数据模型是关系模型，关系模型是一个二维的表格模型，每个表格由若干字段构成，并且每行对应于某个实体或对象，每列对应于某个属性。而数据结构则指的是一个表格的组织形式，它包括表格模式、索引、外键等。关系型数据库管理系统通过三元组（Tuple）来描述信息：（表名，主键值，属性值） 。

## NoSQL(Not Only SQL)
NOSQL，即“不仅仅是SQL”，是一类非关系型数据库。在非关系型数据库中，不仅仅支持SQL语言，还可以支持其他非结构化数据模型。

1. Key-value stores(键值对存储):

键值存储一般用于存储小块数据，类似于字典，比如Memcached、Redis。它的优点是快速访问，占用的内存很少，缺点是不易扩展。由于是基于键值对的数据模型，所以它只能通过键来获取数据。

2. Column family stores(列族存储):

列族存储也是一种非关系型数据库，它的特点是按照列存储，而不是按行存储。也就是说，把多个列放在一起存储。它也被称作“宽表”。它的优点是通过列来过滤和查询数据，并且能够存储复杂类型的数据，比如JSON对象。缺点是写入时需要更多的IO，查找速度慢。

3. Document Stores(文档型存储):

文档型存储又称为“面向文档”数据库，它的基本单位是文档。文档型存储中，所有数据都被序列化到一个文档中。它支持灵活的数据模型，可以使用任意的结构。它既可以存储结构化数据，也可以存储半结构化数据。文档型数据库适合存储动态文档，例如博客、新闻等。但是，它的缺点是检索起来比较麻烦。另外，文档型数据库通常需要做大量的解析工作才能获得需要的信息。

4. Graph databases(图数据库):

图数据库也叫网络型数据库。它支持网络结构数据，允许节点之间的连接。它的优点是高性能的查询功能，适合用来存储复杂的关系数据，比如社交网络、推荐引擎等。缺点是插入、更新和删除操作比较复杂。

综上所述，关系型数据库和非关系型数据库是两种截然不同的数据库，这两者各有特点和适用场景。理解它们的区别和优缺点，并根据需求进行合理选择，才能更好地管理海量数据。
