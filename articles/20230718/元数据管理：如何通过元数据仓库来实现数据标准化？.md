　　数据标准化（Data Normalization）是指按照一定的规则把各个异构的数据集统一到一个形式上的过程，其目的是为了让不同系统中的同一数据在结构、格式、语言等方面保持一致性，从而使得不同来源的数据之间能够互相转换和对接，实现数据共享和整合。元数据管理就是为数据标准化提供元数据的过程。元数据就是描述数据的一组数据属性或特征。元数据仓库就是一种信息系统，用于存储、管理和保护元数据的集合。元数据管理是指定义、收集、分析、组织和存储元数据的过程。元数据管理可以分为以下三个阶段：元数据定义、元数据收集、元数据解析、元数据质量保证、元数据集成和元数据服务。

　　20世纪70年代后期，计算机技术和网络应用日益普及，海量的大型数据集开始涌现出来。随着社会经济发展，企业迫切需要这些数据做精细化的决策、运营和管理。然而，缺乏有效的元数据管理体系限制了企业对数据的利用效率和质量的影响，进一步削弱了企业的数据价值。

​    在这样的背景下，元数据管理的需求量越来越大，而开源界已经出现了许多元数据管理工具。但是，由于这些工具仅仅满足了企业内部的元数据管理需求，并没有很好地服务于企业外部的需求，因此，需要更加符合实际场景的元数据管理工具。

# 2.基本概念术语说明
## 2.1 数据标准化
　　数据标准化（Data Normalization）是指按照一定的规则把各个异构的数据集统一到一个形式上的过程，其目的是为了让不同系统中的同一数据在结构、格式、语言等方面保持一致性，从而使得不同来源的数据之间能够互相转换和对接，实现数据共享和整合。数据标准化包括了如下三个阶段：数据建模、数据准备、数据传输。

### 2.1.1 数据建模
　　数据建模（Data Modeling）是一个独立于具体应用程序的设计阶段，它包括确定数据结构、制定数据关系、定义实体、属性和实体间的联系，通过建立数据库模型、物理模型、逻辑模型、视图模型，完成数据建模工作。数据建模要考虑应用需求、数据规模、数据类型、数据访问模式、数据安全性、数据完整性、数据可用性、数据实时性、数据的可扩展性、数据迁移等因素。

### 2.1.2 数据准备
　　数据准备（Data Preparation）是一个基于标准化数据建模的过程，该过程将原始数据按照一定的规范进行清理、转换、拆分、合并、验证、编码等处理，完成对数据的准备工作。数据准备涉及数据字典构建、数据验证、数据标准化、数据抽取、数据汇总、数据补充、数据违规检测、数据审核等工作。

### 2.1.3 数据传输
　　数据传输（Data Transfer）是将经过数据准备后的数据转移到目标系统中进行应用的阶段。数据传输包括数据加载、数据复制、数据导入导出、数据转换、数据同步等工作。

　　　　　　　

## 2.2 元数据定义
　　元数据（Metadata）是描述数据的一组数据属性或特征。它是关于数据本身的信息，通常包括数据创建日期、创建者、用途、授权方式、适用的法律、联系方式等。元数据定义包括元数据分类、元数据元素和元数据集合。元数据分类又称为元数据模型或元数据框架。元数据元素是元数据集合中的组成部分。

### 2.2.1 元数据分类
　　元数据分类（Metadata Classification）是元数据的一种类型，它将元数据分为不同的层次，如全面、主题、核心、限定词和分类号。全面元数据指所有相关信息都包含在元数据中；主题元数据指只包含与数据主题相关的信息；核心元数据指核心数据元素的元数据；限定词元数据指数据范围内的一些关键词和短语；分类号元数据指包含标识数据类别的信息。

### 2.2.2 元数据元素
　　元数据元素（Metadata Element）是元数据集合中的组成部分。元数据元素一般包含以下几个元素：属性、属性名称、属性类型、约束、描述、注释、示例等。属性是数据中重要的数据项，其名称、类型、约束、描述、注释、示例等属于元数据元素。

### 2.2.3 元数据集合
　　元数据集合（Metadata Set）是由元数据元素所组成的集合。元数据集合作为一系列元数据元素集合的统称。

## 2.3 元数据收集
　　元数据收集（Metadata Collection）是获取、整理、存储、检索、分析、使用、流通和传播元数据的过程。元数据收集方法主要有两种：静态元数据收集和动态元数据收集。静态元数据收集是基于元数据模板生成元数据，而动态元数据收集则是通过技术手段对已有的数据进行元数据的采集、解析和整理。

### 2.3.1 静态元数据收集
　　静态元数据收集（Static Metadata Collection）是基于元数据模板的方法，该模板采用预先制订好的标准，将某些关键数据字段的元数据定义完备，如字段名称、数据类型、约束条件、用途、数据格式、示例等。静态元数据收集以表格或文件形式呈现，并随数据一同提交给第三方进行审核和验证。

### 2.3.2 动态元数据收集
　　动态元数据收集（Dynamic Metadata Collection）是对已有的元数据进行动态收集的过程。它通过使用技术手段对数据进行自动化的收集，提取、解析元数据信息。动态元数据收集既可以从源头数据中直接获取元数据，也可以结合上下游应用系统的接口等技术手段获取元数据。动态元数据收集方法不但需要更多的时间，而且还要求对数据有更高的理解能力，同时也增加了技术难度。

## 2.4 元数据解析
　　元数据解析（Metadata Parsing）是对元数据收集过程中获得的数据进行解析，以生成元数据集合。元数据解析方法分为两类，分别是基于正则表达式和基于自学习方法。基于正则表达式的方法将文本信息和元数据模板匹配，根据正则表达式匹配到的结果生成元数据集合。基于自学习方法使用机器学习算法训练模型，从源头数据中发现隐藏的元数据特征，然后根据这些特征生成元数据集合。

## 2.5 元数据质量保证
　　元数据质量保证（Metadata Quality Assurance）是确保元数据准确、完整、可信、正确、及时、及时的过程。元数据质量保证需要遵循相关规范，如ISO/IEC 15489、NIST SP 800-53等。元数据质量保证还包括元数据溯源，即追踪元数据的产生、变更、流通过程。

## 2.6 元数据集成
　　元数据集成（Metadata Integration）是将多个数据集的元数据融合到一起的过程，目的是为了方便查询和整合数据。元数据集成方法可以基于元数据映射、元数据链接、元数据聚合、元数据质量评估等技术手段。

## 2.7 元数据服务
　　元数据服务（Metadata Serivce）是提供元数据信息的服务，包括元数据查询、元数据管理、元数据鉴权、元数据共享、元数据应用开发、元数据接口、元数据工具等。元数据服务有助于降低系统部署成本、减少系统集成难度、提升用户体验、促进知识创新、保障数据治理。

## 2.8 元数据管理工具
　　元数据管理工具（Metadata Management Tool）是用来管理元数据的专用软件，它可以协助用户管理元数据，包括元数据的定义、收集、解析、质量保证、集成和服务。目前常用的元数据管理工具有Apache Atlas、Cloudera Data Hub、Dremio、Hive Metastore、InfoSphere Metabase、Infinispan Data Grid、MariaDB、MongoDB、MySQL MetaDatabase、Oracle GoldenGate、PostgreSQL Plus Advanced Server、RapidMiner Data Miner、SAS Visual Analytics、SAP HANA Cloud Platform、Splunk Enterprise Data Fabric 和 Tableau Desktop。
