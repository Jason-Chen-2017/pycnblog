## 计算机图形学（Computer Graphics）
在电脑屏幕上显示、渲染和交互的方式就是计算机图形学，而图形用户界面（Graphical User Interface, GUI）则是最主要的一种图形展示方式。GUI的出现促进了电脑图形学的革命性转变，也是使得我们看到如今这么美观，顺滑流畅的电脑界面之原因。
GUI将人机交互与计算机系统融合在一起，通过界面化的图形界面向用户提供各种功能选项及信息，并实现直接对数据或命令的输入输出，以达到提高效率、降低成本、提升工作质量、改善用户体验的效果。
## Java图形界面设计
GUI编程是面向对象编程（Object-Oriented Programming, OOP）的一个重要方面。Java作为世界上主流的多平台开发语言，提供了丰富的GUI库，如AWT（Abstract Window Toolkit）、Swing（Standard Widgets for the Java）等，用以方便地进行GUI编程。因此，掌握Java图形界面编程对于任一系统工程师都是非常必要的技能。然而，想要精通Java图形界面的开发，仍然需要综合掌握相关的基础知识。
本书《Java图形界面设计：GUI编程和用户界面组件》由Java核心技术系列教材中第一册《Java编程的概念》之后陆续出版，共分为七章。本书采用详细的学习法，从计算机图形学的基本原理出发，全面讲述Java图形界面的编程。
本书的作者自身具备丰富的软件开发经验，同时也热衷于分享自己的学习心得，希望能够给读者带来更加系统的Java图形界面编程学习教程。
# 2.基本概念术语说明
## 用户界面组件
用户界面（User Interface, UI）指的是一个系统的外观，即包括所有控件（Control）、菜单、工具栏、状态栏、任务栏等的总称。其中，控件是UI的构成要素，负责呈现信息或处理用户操作，如按钮、标签、列表框、组合框、文本框、滚动条等。
常见的控件类型有：按钮（Button）、标签（Label）、文本框（Text Box）、复选框（Check Box）、单选框（Radio Button）、下拉菜单（DropDown List）、滚动条（Scrollbar）、进度条（Progress Bar）、颜色选择器（Color Picker）等。
## AWT（Abstract Window Toolkit）
AWT是一个基于C/S架构的图形用户接口（Graphical User Interface, GUI）编程环境。它定义了一套窗口管理器、事件模型、图形呈现模型、字体和图标支持、资源加载等机制。虽然AWT已被证明可以构建功能完善的GUI应用，但其已经逐渐被JavaFX所取代。
## Swing（Standard Widgets for the Java）
Swing是一组用于创建图形用户界面应用程序的类库。Swing和AWT一样，具有良好的移植性和跨平台能力。Swing继承了AWT的所有优点，并添加了许多新的组件，如表格、树、层次结构、日历、颜色选择器等，使得其成为GUI编程领域中的必备技能。
## 模板方法模式
模板方法模式（Template Method Pattern）是一种行为型设计模式，它定义了一个算法的骨架，并允许子类修改算法的某些特定步骤，但仍然可以使用父类的其他功能。模板方法模式使得子类可以按需重写父类的方法，并且可以在不改变算法结构的情况下，增加新的功能模块。由于模板方法模式提供了一种封装结构，使得子类无需了解其父类实现的细节，从而简化了代码编写和维护。
