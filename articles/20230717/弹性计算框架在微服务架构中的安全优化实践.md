# 2.基本概念术语说明
## 2.1 Kubernetes简介
Kubernetes是Google内部开源的容器编排管理系统，由Google团队大规模生产环境经验所提炼而成。其主要功能包括：
- **自动化** − 自动部署应用、扩缩容，确保应用按需运行。
- **自我修复** − 当Pod因各种原因异常终止时，集群会自动重启Pod，确保应用始终可用。
- **弹性伸缩** − 支持水平扩展和垂直缩放，确保应用能够随着业务需求快速响应。
- **服务发现和负载均衡** − 通过DNS或插件，提供可变性的服务发现和负载均衡。
- **存储编排** − 为应用提供持久化存储，确保数据安全、共享和访问。
- **安全和策略控制** − 提供细粒度的权限控制和安全策略，确保应用运行安全。

## 2.2 Docker简介
Docker是一个开源的应用容器引擎，让开发者可以打包应用到一个轻量级的、可移植的容器中，然后发布到任何流行的Linux机器上，也可以作为一个远程服务端运行。Docker的优点包括：
- 轻量级 − Docker镜像的大小只有几十兆，并且启动容器的时间也很短。
- 可移植性 − 可以在多种平台上运行，无论是物理机、虚拟机、公有云、私有云甚至混合云都是一样的。
- 松耦合 − Docker通过工具和库，让开发者和 ops 或 SRE 人员可以更好地组织和管理复杂的应用。
- 自动化 − Docker可以使用 Dockerfile 来定义镜像，使得容器可以在部署前就进行自动化构建。

## 2.3 Alibaba Cloud Container Service（ACK）简介
Alibaba Cloud Container Service 是基于阿里巴巴集团 PaaS 领域技术能力和经验打造的，面向企业用户的云原生容器服务平台。旨在帮助客户轻松、高效地建立、管理和运营云原生应用。可以按需快速设置和部署容器化的应用程序。ACK 目前提供两种运行模式：
- 托管版：由 ACK 团队提供基础设施即服务 (IaaS) 和中间件服务 (MAS)，并对 Kubernetes 技术框架进行持续升级和优化，确保 Kubernetes 集群稳定、可靠。用户可以直接使用 ACK 中已有的系统组件和服务，不需要自己去做复杂的运维工作。
- 自主版：由用户根据自己的业务场景和技术要求自行选择和部署服务组件，通过 Helm Charts 方式进行配置。用户可以灵活地自定义集群的网络、存储、日志、监控等组件，实现业务快速迭代。
