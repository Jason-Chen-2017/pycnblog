# 2.核心概念与联系
在进行软件测试之前，我们需要了解一些基本的概念：
- **软件测试**：是指对软件系统进行检查或审查，以评估其是否满足预期功能和性能需求的过程。
- **质量保证**：是指在软件开发过程中采取的各种措施，以确保最终产品具有高质量和可靠性。
- **测试策略**：是指在进行软件测试时所采取的全局策略，包括选择哪些功能需要测试、选择哪些测试方法等。
- **测试计划**：是指针对特定项目制定的详细计划，包括哪些模块需要测试、何时进行测试等。
- **测试用例**：是指针对特定功能或模块编写的具体操作步骤，用于验证该功能或模块是否满足预期需求。
- **缺陷跟踪**：是指在进行软件测试过程中记录并跟踪找到的缺陷，以便及时修复并确保其不再影响产品质量。
- **自动化testing**：是指使用计算机程序自动执行某些或所有testing任务的过程，可以提高testing效率并减少人工错误。
