# 2.核心概念与联系

在规则引擎中，核心概念包括规则、事件、条件、操作、决策等。这些概念之间存在着密切的联系，我们将在后续的内容中详细解释。

- 规则：规则是规则引擎的基本组成单元，用于描述某种情况下的行为或操作。规则由条件和操作组成，当条件满足时，规则将执行相应的操作。
- 事件：事件是规则引擎中的触发器，用于引发规则的执行。事件可以是外部系统的数据变化、用户操作等。
- 条件：条件是规则的判断基础，用于描述规则执行的前提。条件可以是基于数据的比较、逻辑运算等。
- 操作：操作是规则的执行结果，用于描述规则执行后的行为。操作可以是数据的修改、系统的调用等。
- 决策：决策是规则引擎的核心功能，用于根据规则和条件来处理数据和执行操作。决策可以是基于规则的顺序执行、基于条件的选择等。

```python
    {
        "condition": "age >= 18",
        "action": "grant_access"
    },
    {
        "condition": "age < 18",
        "action": "deny_access"
    }
]

# 定义事件
event = {
    "age": 20
}

# 规则匹配
matched_rule = None
for rule in rules:
    if eval(rule["condition"]):
        matched_rule = rule
        break

# 规则执行
if matched_rule:
    matched_rule["action"](event)
else:
    print("No rule matched")

# 事件处理
def grant_access(event):
    print("Access granted")

# 运行结果
# Access granted
```
