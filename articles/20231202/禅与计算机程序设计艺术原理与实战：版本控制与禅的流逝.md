# 2.核心概念与联系
## 2.1 版本控制基础知识
版本控制是一种用于跟踪代码变更历史记录和协同开发团队工作的技术。主要包括：
- **版本库（Repository）**：存储所有项目历史记录的地方；
- **工作区（Working Directory）**：开发人员在电脑上进行编辑和提交代码的地方；
- **提交（Commit）**：将工作区修改后的内容保存到版本库中；
- **回滚（Rollback）**：从版本库恢复之前的某个状态；
- **分支（Branch）**：为了实现特定功能或任务而创建独立分支线路；
- **合并（Merge）**：将多个分支线路合并成一个整体。
## 2.2 禅宗对版本控制的看法
禅宗认为，程序设计需要精益求精、专注注意力、不断学习等原则。这些原则与版本控制相符合，因此可以从禅宗思想入手来提高程序设计水平。同时，禅宗强调“一心向无”和“不动心”等原则，这些原则也可以帮助我们更好地管理代码变更历史记录和协同开发团队工作。
