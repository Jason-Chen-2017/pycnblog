# 2.核心概念与联系
在MySQL中，数据类型与存储是两个重要的概念。数据类型决定了数据库中存储的数据类别，而存储则决定了如何将这些数据保存到磁盘上。在本节中，我们将介绍这两个概念之间的联系和关系。

## 2.1 数据类型
MySQL支持多种不同的数据类型，包括整数、浮点数、字符串、日期时间等等。每种数据类型都有其特定的长度和精度限制，这些限制决定了可以存储多少信息以及如何进行比较和运算。例如，整数类型可以根据其长度分为tinyint、smallint、mediumint和bigint等不同大小；浮点数类型可以根据精度分为float、double和decimal等不同精度；字符串类型可以根据长度分为varchar、text等不同长度。

## 2.2 存储
MySQL使用表（table）来组织数据，表由一组列（column）组成。每个列都有一个名称和一个对应的数据类型。当插入或更新记录时，MySQL会根据列的数据类型来确定如何存储该记录中的值。例如，如果某个列是整数类型，那么MySQL会将该值转换为相应的二进制格式并保存到磁盘上；如果某个列是字符串类型，那么MySQL会将该值转换为相应的文本格式并保存到磁盘上；如果某个列是日期时间类型，那么MySQL会将该值转换为相应的日期时间格式并保存到磁盘上。
