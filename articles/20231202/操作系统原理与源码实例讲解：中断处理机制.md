# 2.核心概念与联系
中断处理机制是操作系统调度程序的基础之一，它可以让操作系统在运行程序时能够响应外部设备请求和异常情况。当中断发生时，CPU会暂停当前正在执行的任务并切换到新的任务上去执行。这种切换过程称为上下文切换（context switch）。

中断处理机制主要包括：
- **硬件中断**：由硬件设备触发的中断请求信号，如键盘输入、鼠标移动等。当CPU检测到硬件中断信号时，会暂停当前任务并执行相应的中断服务程序（ISR）来处理该请求。
- **软件中断**：由软件触发的中断请求信号，如系统调用等。当CPU执行系统调用时，会暂停当前任务并执行相应的ISR来处理该请求。
- **异常**：由程序错误或异常情况引起的中断请求信号，如除数为零错误、访问非法内存地址错误等。当CPU检测到异常信号时，会暂停当前任务并执行相应的ISR来处理该错误。
