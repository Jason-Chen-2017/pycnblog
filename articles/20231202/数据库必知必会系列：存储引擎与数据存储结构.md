# 2.核心概念与联系

存储引擎是数据库系统的核心组件，它负责数据的存储和管理。不同的存储引擎有不同的特点和优缺点，因此选择合适的存储引擎对于数据库系统的性能和稳定性至关重要。

存储引擎可以分为两类：磁盘存储引擎和内存存储引擎。磁盘存储引擎将数据存储在磁盘上，而内存存储引擎将数据存储在内存中。磁盘存储引擎的优点是数据的持久性和安全性，而内存存储引擎的优点是速度快和实时性。

磁盘存储引擎可以进一步分为两类：顺序存储引擎和非顺序存储引擎。顺序存储引擎将数据按照顺序存储在磁盘上，而非顺序存储引擎将数据按照不同的策略存储在磁盘上。顺序存储引擎的优点是数据的紧凑性和查询速度，而非顺序存储引擎的优点是数据的灵活性和扩展性。

内存存储引擎可以进一步分为两类：基于页的内存存储引擎和基于段的内存存储引擎。基于页的内存存储引擎将数据存储在页上，而基于段的内存存储引擎将数据存储在段上。基于页的内存存储引擎的优点是数据的紧凑性和查询速度，而基于段的内存存储引擎的优点是数据的灵活性和扩展性。
