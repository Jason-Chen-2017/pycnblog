# 2.核心概念与联系
## 2.1 身份认证（Identity Authentication）
身份认证是指验证一个用户声称所属的实体（通常是人）确实存在并且该实体正在进行身份验证过程中。这通常包括密码输入、图像识别等多种方式来验证用户是否真正存在并且能够访问特定资源或服务。
## 2.2 授权（Authorization）
授权是指确定已经进行了身份验证的用户可以执行哪些操作或访问哪些资源。这涉及到对用户角色、权限等因素进行管理和控制，以确保他们只能访问他们拥有权限的资源或执行他们拥有权限的操作。
## 2.3 OpenID Connect（OIDC）
OpenID Connect是基于OAuth2.0协议构建而成立标准化身份提供者(IdP)和服务提供者(SP)之间的简单 identity layer(标识层)，它允许客户端向IdP请求用户信息并获取令牌来访问受保护的资源。OIDC使得跨域单点登录(SSO)变得更加容易和可靠，同时也提供了更好的安全性和隐私保护机制。
## 2.4 OAuth2.0协议
OAuth2.0是一种标准化授权协议，它允许第三方应用程序获得受限制访问某个HTTP服务器上的资源，而不需要获取该服务器上凭据(如密码)。OAuth2.0主要由四个组件组成：客户端、服务器端、资源所有者和API服务器端。它定义了一种简化流程来让第三方应用程序请求对特定资源进行读取/写入/删除等操作的权限，从而避免暴露密码给第三方应用程序并降低风险。
