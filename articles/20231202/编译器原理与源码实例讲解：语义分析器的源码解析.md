                 

# 1.背景介绍

编译器是计算机语言的翻译器，编译器实现将一种特定的源代码（例如 Python 或 Java）翻译成另一种形式（可执行代码或二进制代码），这些代码可以在目标运行时平台上运行。编译器主要由3个子部件组成：词法单元化(lexing)、语法分析(parsing)、中间表现(intermediate)以及代码生成(Code Generation)。它们分别完成词法单元分析、完成语法分析、中间表现生成、代码生成等功能。本文主要讨论如何在编译器内部通过语法分析器获取语法树并使用语义分析器进一步分析这些语法节点。

语义分析器对于某些计算机语言流行起来相对密集，可以帮助实现有意义的语法判断，例如在执行某个函数调用时能够进入函数体来获取函数的身份。

在大多数现代的同构编译器中，特别是 Go 语言来说，语法分析和语义分析分开进行。并且语义分析通常每次都从头开始。语法分析器对于编译器的代码生成阶段来说不是所有语义带有相同效应的所有情况都具有。在大多数情况下，语言设计者对编译器的源代码可以通过语言将输入的LL(1)或LR(1)可以重用来重写。因此，可以使用现有的语法分析器来完成。但是语言的分析，例如的状态复位的处理，细微的语义不能提供，如，C语言的未初始化的static变量，空指针的访问以othing可能在最后的代码しShapeの出来。
