# 2.核心概念与联系

在讨论Go语言的并发编程与并发模型之前，我们需要了解一些核心概念。

## 2.1 Goroutine

Goroutine是Go语言中的轻量级线程，它是Go语言的并发编程的基本单元。Goroutine是用户级线程，它们由Go运行时创建和管理。Goroutine之间可以相互独立地运行，并且可以在同一时间运行多个Goroutine。Goroutine的创建和销毁非常轻量级，因此可以创建大量的Goroutine，从而实现高性能的并发编程。

## 2.2 Channel

Channel是Go语言中的一种数据结构，用于实现并发编程的同步和通信。Channel是一个可以存储和传输数据的有序的数据流，它可以用来实现多个Goroutine之间的数据传递。Channel可以用来实现多个Goroutine之间的同步和通信，从而实现高性能的并发编程。

## 2.3 并发模型

并发模型是Go语言的并发编程的核心概念。并发模型是一种抽象的并发编程模型，它描述了如何实现并发编程的同步和通信。Go语言的并发模型是基于Goroutine和Channel的，它可以帮助我们更好地编写并发程序。
