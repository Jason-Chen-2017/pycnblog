                 

# 1.背景介绍

线程是操作系统中的一个基本概念，它是进程的一个独立单元，可以并发执行。线程与进程之间的关系类似于生物学中的细胞和组织之间的关系：进程是线程的容器，线程是进程内部的一个执行单元。

在多任务环境下，操作系统为了提高资源利用率和响应速度，需要实现并发执行多个任务。这时候就需要引入线程来实现并发执行。线程有许多特点，例如轻量级、独立、同步等。在本文中，我们将详细讲解线程的概念及其特性。

# 2.核心概念与联系
## 2.1 进程与线程的区别
### 2.1.1 进程（Process）：
- **进程是操作系统对于资源分配和管理的最小单位**；每个进程都有自己独立的地址空间和资源（如文件描述符、信号处理器等）；
- **进程在内存管理上具有独立性**：每个进程都有自己独立的内存空间，包括代码段、数据段和堆栈段；
- **进程在运行上具有独立性**：操作系统为每个进程分配一定数量的CPU时间片（Quantum），当前正在运行的进程被称为活动 progress（Active Process）或者当前 progress（Current Process）；当前正在运行的进程占据整个CPU资源，其他所有非活动状态下的所有其他进程都处于阻塞状态；当前正在运行的进程使用完毕后会释放CPU资源并转换到就绪队列中等待调度；
- **创建、销毁和通信成本较高**：由于每个进制拥有自己独立的地址空间和资源，因此创建、销毁或者通信相对较高昂。例如创建新子节点需要复制整个地址空间和重新初始化相关资源；而且由于不同子节点拥有不同地址空间，因此无法直接访问彼此之间变量或者数据结构；通信也需要借助IPC机制来实现数据传递。
