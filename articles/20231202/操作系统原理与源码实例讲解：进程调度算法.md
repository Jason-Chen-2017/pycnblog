                 

# 1.背景介绍

**进程调度**是操作系统中比较重要的环节之一，主要有两种，即抢占式进程调度和非抢占式进程调度。抢占式是指操作系统会根据某种策略决定运行哪个进程，而非抢占式进程则是由进程自身决定。目前主流操作系统采用的是 **非抢占式进程调度**，这一文章将主要介绍这种调度方式的算法。

进程调度算法是操作系统内部最重要的几个系统层面的系统 **内核** 的组成部分之一。对象是在已经被加载进内存的进程，也即系统中合法的进程。通过调度算法，操作系统可执行计算机在有限资源下的调度资源分配经济与公平性。择优调度能够实现计算机资源的优化并提高系统吞吐量。

公平性是指系统对所有进程的调度是公平的即性能差异在控制范围内，不会偏守高或偏轻降权，以公平的资源分配不能被任何一个进程或者某种进程欺凌区域和资源。

## 二、核心概念与联系

**进程和线程**

操作系统中的敏捷体的程序单位是进程，即运行中的程序。并非所有的进程都是独立的，系统中存在独立的进程和相互联系的依赖进程或线程。每一个进程中可以创建多个线程，这有以下两种途径：

1. 同步：此时进程中的线程上下文操作是原子的，虽然是单线程也相当于一个进程，但是并非每个进程都是原子的。需要重载成非原子的线程上下文，这就使得提高并发性变得不常常，使得进程管理变得困难。

2. 异步：线程上下文操作是在不同进程之间的原子操作，当每个线程上下文以至少一个操作当作原子操作并运行时，则可看作两个或多个子线程上下文并行运行的一个线程。这就使得可以**高效进行并发任务**，但是对跳行进行有解注意定，操作就得放到操作系统 privacy 了函数配置私有队列中， Зoning、Seeing、Dan相同

### 一Enumerable函数实现

这是一种可以用于集合对象以获取集合大小的函数，这可以帮助用户实现 **无限循环的可重复取值**。

``` python
class EnumLock:
    def __init__(self, data):
        self.data = data
        self.mut_cnt = 0

    def f(self, a, b):
        a += b
        self.mut_cnt += 1
        return self.mut_cnt

    def add(self, a, b):
        return self.f(a, b)
```
