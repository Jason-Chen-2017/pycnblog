# 2.核心概念与联系
在了解Go语言性能优化之前，我们需要了解一些基本概念：并发、并行和并发模型。这些概念对于理解Go语言中的性能优化至关重要。

## 2.1 并发与并行
并发（Concurrency）是指多个任务同时运行，但不一定会同时执行。而并行（Parallelism）则是指多个任务同时执行，每个任务都在不同的处理器上运行。通常情况下，并发可以通过多线程或协程来实现，而并行则需要多处理器或多核心来支持。

## 2.2 Go语言中的Goroutine和Channel
Go语言提供了两种特殊类型的轻量级线程：Goroutine（协程）和Channel（通道）。Goroutine是Go语言中最小的执行单元，它们由操作系统调度器管理，可以轻松地创建和销毁。Channel则是一种用于安全地传递数据之间 Goroutine 之间通信的机制。Channel 可以让 Goroutine 之间安全地共享数据，从而实现高效且易于维护的并发编程模式。
