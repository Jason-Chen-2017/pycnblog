# 2.核心概念与联系

在讨论身份认证与授权之前，我们需要了解一些核心概念。

## 2.1 身份认证

身份认证是一种验证过程，用于确认一个用户是否是他所声称的身份。通常，身份认证涉及到用户提供凭据，如密码或其他身份验证信息，以便系统可以验证用户的身份。

## 2.2 授权

授权是一种控制访问资源的过程，用于确定用户是否有权访问某个资源。授权通常涉及到对用户的身份进行验证，以及对用户的权限进行检查。

## 2.3 微服务

微服务是一种架构风格，它将应用程序划分为多个小的服务，每个服务都负责处理特定的功能。微服务的主要优点是它们可以独立部署和扩展，并且可以更容易地进行维护和修改。

哈希算法是一种用于将数据转换为固定长度字符串的算法。在身份认证中，我们可以使用哈希算法来存储用户的密码，而不是原始的密码本身。这样可以确保即使密码被泄露，攻击者也无法得到原始的密码。

## 3.2 公钥加密

公钥加密是一种加密方法，它使用一对公钥和私钥进行加密和解密。在身份认证与授权中，我们可以使用公钥加密来确保数据的安全传输。通过使用公钥加密，我们可以确保只有具有相应的私钥的用户才能解密数据。
