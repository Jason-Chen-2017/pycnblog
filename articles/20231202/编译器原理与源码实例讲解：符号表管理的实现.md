# 1.背景介绍

# 2.核心概念与联系

在编译器中，符号表是一个数据结构，用于存储程序中的各种符号及其相关信息。符号表的核心概念包括：

- 符号：程序中的变量、函数、类等。
- 符号表：一个数据结构，用于存储符号及其相关信息。
- 符号表项：符号表中的一个元素，包括符号及其相关信息。
- 作用域：符号的有效范围，用于限制符号的访问范围。
- 链接：符号之间的关系，用于描述符号之间的依赖关系。

符号表与其他编译器组成部分之间的联系如下：

- 词法分析：将源代码划分为一系列的词法单元（如标识符、关键字、运算符等），并将这些词法单元存储到符号表中。
- 语法分析：根据语法规则解析源代码，生成抽象语法树（AST），并在解析过程中更新符号表。
- 语义分析：根据语义规则分析抽象语法树，检查程序的语义正确性，并更新符号表。
- 代码生成：根据符号表中的符号及其相关信息，生成目标代码。

4. 修改符号：根据标识符名称，修改符号表中的符号及其相关信息。
5. 删除符号：根据标识符名称，删除符号表中的符号及其相关信息。

```python
class SymbolTable:
    def __init__(self):
        self.table = {}

    def add_symbol(self, name, value):
        self.table[name] = value

    def find_symbol(self, name):
        return self.table.get(name)

    def modify_symbol(self, name, value):
        if name in self.table:
            self.table[name] = value

    def delete_symbol(self, name):
        if name in self.table:
            del self.table[name]
```
