# 2.核心概念与联系
## 2.1 进程与线程
在操作系统中，进程是独立运行的程序实例，每个进程都有自己独立的内存空间、文件描述符等资源。线程则是进程内部的一个执行单元，线程共享进程的资源。多线程编程可以提高并发性能，但需要注意避免竞争条件和死锁等问题。

## 2.2 同步与异步
同步是指一段代码必须等待另一段代码完成后才能继续执行；异步则允许一段代码在另一段代码完成后自动继续执行。在进程间通信中，同步通信使得发送方进程必须等待接收方进程确认消息已经接收；而异步通信则不需要这样的等待。

## 2.3 阻塞与非阻塞
阻塞是指当一个请求尚未得到满足时，其他请求不能被处理；非阻塞则允许其他请求继续处理。在进程间通信中，阻塞调用会导致调用方暂停执行直到请求完成；而非阻塞调用则允许调用方继续执行其他任务。
