# 2.核心概念与联系

框架设计的核心概念包括模块化、组件化、抽象、依赖注入、反射等。这些概念在框架设计中起着关键作用，使框架更加灵活、可扩展和可维护。

模块化是指将软件系统划分为多个模块，每个模块负责特定的功能。模块化有助于提高代码的可读性、可维护性和可重用性。

组件化是指将软件系统划分为多个组件，每个组件负责特定的功能。组件化有助于提高软件系统的可扩展性和可维护性。

抽象是指将复杂的功能抽象为简单的接口，使开发人员可以更轻松地使用这些功能。抽象有助于提高代码的可读性、可维护性和可重用性。

依赖注入是一种设计模式，它将对象之间的依赖关系在运行时注入。依赖注入有助于提高代码的可测试性、可维护性和可扩展性。

反射是一种动态的代码执行机制，它允许程序在运行时查看和操作其自身的结构。反射有助于提高代码的可扩展性和可维护性。
