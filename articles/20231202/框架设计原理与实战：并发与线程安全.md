# 2.核心概念与联系
## 2.1并发与并行
在并发编程中，我们需要区分两个概念：并发（Concurrency）和并行（Parallelism）。**并发**是指多个任务在同一时间内被处理器执行；而**并行**是指多个任务同时执行，每个任务都由单独的处理器执行。尽管这两个概念似乎很相似，但它们之间存在一定的区别：即使只有一个处理器也可以实现并发（例如通过轮询或交替执行任务），而真正意义上的并行则需要多个处理器协同工作。

## 2.2线程与进程
在操作系统层面上，进程（Process）是资源分配和调度的最小单位；而线程（Thread）则是进程内部运行的最小单位。一个进程可以包含多个线程，每个线程都有自己独立的栈空间和寄存器状态等资源。由于线程共享进程内部的资源（如内存空间、文件描述符等），因此具有相对于进程而言较低的开销。因此，在大多数情况下使用线程是更高效且更合适的选择。

## 2.3锁与互斥量
锁（Lock）是一种同步原语，用于控制对共享资源的访问。当一个线程获取到锁后，其他线程无法访问该共享资源；当锁被释放后，其他线程才能够访问该资源。互斥量（Mutex）是锁机制中最基本且最常见的一种形式：它允许一个线 program at a time to access the shared resource, while preventing other threads from accessing it until the lock is released.
