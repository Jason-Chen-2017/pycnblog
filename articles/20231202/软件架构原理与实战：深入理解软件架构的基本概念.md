# 2.核心概念与联系
在深入探讨软件架构之前，我们需要了解一些基本概念：
- **软件设计**：软件设计是指为特定目标创建满足需求的软件结构。它包括选择合适的算法和数据结构以及确定程序结构。
- **软件架构**：软件架构是指为特定目标创建满足需求的软件组织形式。它包括组成部分之间的关系以及整体结构。
- **模块化**：模块化是指将大型问题划分为较小的子问题，每个子问题都可以独立地实现和测试。这样做可以提高代码可读性、可维护性和可重用性。
- **对象编程**：对象编程是一种编程范式，其中类似于现实生活中的对象表示数据和行为。这种方法使得代码更容易理解和维护。
- **面向对象编程**：面向对象编程（OOP）是一种编程范式，其中类似于现实生活中的对象表示数据和行为。这种方法使得代码更容易理解和维护。OOP包括封装、继承和多态等特征。
