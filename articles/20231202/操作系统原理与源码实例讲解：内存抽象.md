# 2.核心概念与联系
内存抽象是操作系统中的一个基本组成部分，它为应用程序提供了一种简化的方式来访问和管理内存空间。内存抽象包括以下几个核心概念：
- 虚拟地址空间：每个进程都有自己独立的虚拟地址空间，它允许进程在不同的内存区域之间自由移动和访问数据。
- 物理地址空间：物理地址空间是计算机硬件提供的实际内存空间，操作系统需要将虚拟地址转换为物理地址才能访问内存。
- 页面调度：当虚拟地址超出物理地址范围时，操作系统需要进行页面调度，将部分页面从磁盘加载到内存中或者从内存中移除。
- 保护和共享：操作系统需要确保每个进程只能访问自己的虚拟地址空间，并且对共享资源进行合适的保护和控制。
- 缓冲区管理：缓冲区是内存中动态分配给应用程序的一块连续空间，操作系统需要对缓冲区进行合适的管理和分配。
