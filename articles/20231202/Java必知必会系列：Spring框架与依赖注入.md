# 2.核心概念与联系

## 2.1 Spring框架的核心概念

### 2.1.1 依赖注入（Dependency Injection，DI）

依赖注入是Spring框架的核心概念之一，它是一种设计模式，用于解决对象之间的依赖关系。通过依赖注入，我们可以在运行时动态地为一个对象提供其所需的依赖关系，从而实现更加灵活的代码组织和维护。

### 2.1.2 控制反转（Inversion of Control，IoC）

控制反转是Spring框架的另一个核心概念，它是依赖注入的基础。通过控制反转，我们可以将对象的创建和依赖关系的管理交给Spring容器，而不是在代码中手动创建和管理这些对象。这样，我们可以更加灵活地控制对象之间的关系，从而实现更好的代码复用和维护。

### 2.1.3 面向切面编程（Aspect-Oriented Programming，AOP）

面向切面编程是Spring框架的另一个重要概念，它是一种设计模式，用于解决跨切面的问题。通过面向切面编程，我们可以将一些横切关注点（如日志记录、事务管理等）抽取出来，并在运行时动态地应用到目标方法上。这样，我们可以更加灵活地管理这些横切关注点，从而实现更好的代码组织和维护。

## 2.2 Spring框架与依赖注入的联系

Spring框架与依赖注入密切相关，它提供了一种通过依赖注入实现控制反转和面向切面编程的方法。通过依赖注入，我们可以将对象的创建和依赖关系的管理交给Spring容器，从而实现更加灵活的代码组织和维护。同时，通过面向切面编程，我们可以将一些横切关注点抽取出来，并在运行时动态地应用到目标方法上。

## 4.1 依赖注入的代码实例

```java
// 创建一个Spring容器
ApplicationContext context = new ClassPathXmlApplicationContext("bean.xml");

// 从Spring容器中获取一个Bean
UserService userService = (UserService) context.getBean("userService");

// 通过依赖注入将这个对象的依赖关系注入到这个对象中
userService.setDao(new UserDao());

// 调用对象的方法
userService.addUser();
```

在这个代码实例中，我们首先创建了一个Spring容器，并从中获取一个UserService对象。然后，我们通过依赖注入将UserDao对象注入到UserService对象中，并调用UserService对象的addUser方法。
