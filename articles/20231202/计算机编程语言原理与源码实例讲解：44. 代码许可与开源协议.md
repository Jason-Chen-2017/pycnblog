# 2.核心概念与联系

## 2.1 代码许可

代码许可是指软件开发者对其软件代码的一种授权，规定了代码的使用、分发和修改的范围。许可证可以是开放的，允许任何人使用、分发和修改代码，也可以是闭源的，限制了代码的使用和分发。许可证通常包括以下几个方面：

- 授权范围：许可证规定了代码的使用、分发和修改的范围，包括个人使用、商业使用、非商业使用等。
- 许可对象：许可证规定了代码的许可对象，可以是个人、企业、组织等。
- 许可条款：许可证规定了代码的使用、分发和修改的条件，包括保留声明、禁止使用声明、禁止分发声明等。

## 2.2 开源协议

开源协议是一种软件开发者对其软件代码的一种协议，确保了软件的开放性和可持续发展。开源协议通常包括以下几个方面：

- 自由性：开源协议要求软件代码的自由性，即任何人可以访问、使用、修改和分发软件代码。
- 透明性：开源协议要求软件代码的透明性，即软件代码的发布和审查。
- 协作性：开源协议要求软件代码的协作性，即任何人可以参与软件的开发和维护。
