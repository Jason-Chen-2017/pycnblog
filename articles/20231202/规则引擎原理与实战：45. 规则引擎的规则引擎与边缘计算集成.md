# 2.核心概念与联系

在边缘计算环境中，规则引擎的核心概念包括：规则、事件、事件处理器、规则引擎等。这些概念之间的联系如下：

- 规则：规则是一种条件-动作的对应关系，用于描述系统的行为。规则由条件部分（条件表达式）和动作部分（动作列表）组成。
- 事件：事件是系统中发生的异步动作，可以触发规则的执行。事件可以是外部输入（如传感器数据），也可以是内部生成（如计时器触发）。
- 事件处理器：事件处理器是规则引擎的组件，负责接收事件、触发规则的执行、执行动作等。事件处理器可以是单线程的，也可以是多线程的，以实现并发处理。
- 规则引擎：规则引擎是规则系统的核心组件，负责管理规则、事件、事件处理器等，以实现规则的自动化执行。规则引擎可以是内存型的，也可以是持久化型的，以实现数据的持久化存储。

```python
from rule_engine import RuleEngine
from rule import Rule
from event import Event

# 创建规则引擎实例
engine = RuleEngine()

# 创建规则实例
rule1 = Rule("temperature > 30", "turn_on_cooler")
rule2 = Rule("temperature < 20", "turn_off_cooler")

# 加载规则到规则引擎
engine.load_rules([rule1, rule2])

# 创建事件实例
event = Event("temperature", 25)

# 触发规则执行
engine.trigger(event)

# 获取执行结果
result = engine.get_result()
print(result)  # 输出：turn_off_cooler
```

在上述代码中，我们首先导入了`rule_engine`、`rule`、`event`模块。然后创建了规则引擎实例`engine`，并创建了两个规则实例`rule1`和`rule2`。接着，我们加载了规则到规则引擎，并创建了事件实例`event`。最后，我们触发规则执行，并获取执行结果。
