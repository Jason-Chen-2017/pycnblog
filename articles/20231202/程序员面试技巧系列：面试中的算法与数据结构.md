# 2.核心概念与联系
## 2.1 算法与数据结构的关系
算法和数据结构是计算机科学中两个密切相关的概念。算法描述了如何解决问题，而数据结构则提供了存储和组织数据的方式。算法通常需要使用某种类型的数据结构来实现，而不同类型的数据结构可以用于解决不同类型的问题。因此，了解算法和数据结构之间的联系非常重要。

## 2.2 常见的数据结构类型
1. **线性表**：包括数组、链表等。线性表是一种顺序存储结构，元素在内存中连续排列。它们主要应用于处理有序或无序的元素集合。
2. **树形结构**：包括二叉树、平衡二叉树等。树形结构是一种非线性存储结构，具有父子关系。它们主要应用于处理层次化或部分有序的元素集合。
3. **图**：包括无向图、有向图等。图是一种非线性存储结构，具有顶点和边之间的关系。它们主要应用于处理复杂网络或关系模型问题。
4. **栈**：后进先出（LIFO）特征；主要应用于函数调用、表达式求值等场景；典型实现为链表或数组；空栈和满栈是特殊情况；栈可以嵌套使用；时间复杂度为O(1)；空间复杂度为O(n)（当使用数组实现时）；空间换时间原则；适合处理递归问题；适合处理逆波兰表达式（后缀表达式）计算等场景；适合处理括号匹配问题（如判断一个字符串是否为有效括号序列）等场景；适合处理最近最少使用策略缓存场景（LRU缓存策略）等场景；适合处理动态规划问题（如 longest common subsequence problem）等场景；适合处理基本计算器功能（如简单加减乘除运算计算器功能）等场景；适合处理基本文本编辑功能（如剪切粘贴功能）等场景；适合处理基本堆栈功能（如 Undo/Redo 功能）等场景 ;
5. **队列**：先进先出（FIFO）特征；主要应用于任务调度、消息传输等场景 ; 典型实现为链表或数组 ; 空队列和满队列是特殊情况 ; 时间复杂度为O(1) ; 空间复杂度为O(n) (当使用数组实现时) ; 空间换时间原则 ; 适合处理任务调度问题 ; 适合处理消息传输问题 ; 适合处理贪心策略缓存场景 (FIFO缓存策略) ;
