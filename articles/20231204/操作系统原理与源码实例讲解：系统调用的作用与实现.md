# 2.核心概念与联系

## 2.1 系统调用的概念

系统调用是操作系统提供给用户程序的一种接口，用于实现对操作系统内部功能的访问。通过系统调用，用户程序可以向操作系统请求各种服务，如文件操作、进程管理、内存分配等。系统调用通常是通过特定的系统调用号和参数传递给操作系统内核，然后内核根据这些参数执行相应的操作。

## 2.2 系统调用与应用程序接口的关系

应用程序接口（Application Programming Interface，API）是一种软件接口，用于定义软件组件之间的交互方式。系统调用是操作系统提供的一种API，用于实现对操作系统内部功能的访问。API可以是用户级别的，如文件操作、网络通信等，也可以是内核级别的，如进程管理、内存分配等。系统调用通常是通过特定的系统调用号和参数传递给操作系统内核，然后内核根据这些参数执行相应的操作。

# 4.具体代码实例和详细解释说明

在这里，我们以Linux操作系统为例，展示一个简单的系统调用实例：

```c
#include <stdio.h>
#include <unistd.h>

int main() {
    int pid = fork();
    if (pid == 0) {
        printf("Child process: %d\n", getpid());
    } else {
        printf("Parent process: %d\n", getpid());
    }
    return 0;
}
```

在上述代码中，我们使用了`fork()`系统调用创建子进程。`fork()`系统调用返回一个整数，表示子进程的进程ID（PID）。如果返回值为0，表示当前是子进程；否则，表示当前是父进程。我们根据返回值打印当前进程是子进程还是父进程，并输出当前进程的PID。
