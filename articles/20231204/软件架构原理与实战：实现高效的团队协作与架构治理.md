# 2.核心概念与联系

## 2.1 软件架构的定义与特点

软件架构是软件系统的高层次组织结构和设计决策。它定义了系统的组件、它们之间的关系以及它们的行为。软件架构的主要特点包括：

1. 抽象：软件架构抽象了系统的细节，使得设计和实现过程更加简单和可管理。
2. 组件：软件架构将系统划分为多个组件，每个组件负责特定的功能。
3. 关系：软件架构定义了组件之间的关系，如依赖关系、通信关系等。
4. 行为：软件架构描述了组件的行为，包括它们的功能、状态和交互。

## 2.2 软件架构的分类

软件架构可以根据不同的维度进行分类，如：

1. 基于组件的架构：这类架构将系统划分为多个组件，每个组件负责特定的功能。例如，基于微服务的架构。
2. 基于数据的架构：这类架构将系统划分为多个数据库，每个数据库负责特定的数据。例如，基于分布式数据库的架构。
3. 基于协作的架构：这类架构将系统划分为多个协作组件，每个组件负责特定的任务。例如，基于消息队列的架构。

3. 组件组织：根据组件的功能和关系，定义组件之间的关系，如依赖关系、通信关系等。

接下来，我们需要定义这些组件之间的关系。例如，购物车组件需要访问商品信息组件，以获取商品的信息。同样，订单组件需要访问购物车组件，以获取用户的购物车信息。

最后，我们需要描述这些组件的行为。例如，商品信息组件需要提供一个接口，以便其他组件可以获取商品的信息。同样，购物车组件需要提供一个接口，以便其他组件可以操作购物车。
