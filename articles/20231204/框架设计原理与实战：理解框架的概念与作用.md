# 2.核心概念与联系

在框架设计中，有几个核心概念需要我们深入理解：

1.模块化：模块化是指将软件系统划分为多个模块，每个模块负责完成特定的功能。模块化有助于提高软件的可维护性和可扩展性，因为每个模块都是独立的，可以独立开发和测试。

2.组件化：组件化是指将软件系统划分为多个组件，每个组件实现特定的功能。组件化有助于提高软件的可重用性，因为每个组件都可以独立使用，可以被其他软件系统重用。

3.抽象：抽象是指将复杂的系统或概念简化为更简单的形式，以便更容易理解和操作。抽象有助于提高软件的可维护性和可扩展性，因为抽象可以将复杂的系统或概念分解为更小的部分，使得每个部分更容易理解和操作。

4.接口：接口是软件系统中的一个抽象层，它定义了一个对象或组件的行为和功能。接口有助于提高软件的可扩展性和可维护性，因为接口可以将不同的对象或组件之间的交互标准化，使得它们可以更容易地相互操作。

5.依赖注入：依赖注入是一种设计模式，它将对象之间的依赖关系从构建过程中分离出来，使得对象可以在运行时动态地获取依赖关系。依赖注入有助于提高软件的可维护性和可扩展性，因为它可以将对象之间的依赖关系分离出来，使得它们可以更容易地相互替换。

6.反射：反射是一种动态地获取和操作类的信息和功能的机制。反射有助于提高软件的可扩展性和可维护性，因为它可以将类的信息和功能动态地获取和操作，使得软件可以更容易地扩展和维护。
