随着互联网的不断发展，跨域请求技术已经成为了前端开发中的一个重要的技术。跨域请求是指从一个域名下的网页访问另一个域名下的资源。这种访问方式在传统的浏览器安全模型中是被禁止的，因为它可能会导致安全问题。

在传统的浏览器安全模型中，每个域名都是独立的，它们之间是相互隔离的。这意味着一个域名下的网页不能直接访问另一个域名下的资源。这是为了保护用户的隐私和安全，防止恶意网站窃取用户的数据。

然而，随着Web应用程序的复杂性和功能的增加，跨域请求技术变得越来越重要。例如，一个网站可能需要从另一个网站获取数据，或者需要与后端服务器进行通信。为了解决这个问题，浏览器提供了一些跨域请求的技术，如XMLHttpRequest、CORS（跨域资源共享）和Fetch API等。

## 1.1 背景介绍

跨域请求技术的发展与Web的演进有关。在早期的Web应用程序中，所有的资源都在同一个域名下，因此没有跨域请求的问题。但随着Web应用程序的复杂性和功能的增加，需要访问来自不同域名的资源也越来越多。

## 1.2 核心概念与联系

在本节中，我们将介绍跨域请求的核心概念和联系。这些概念包括：

- 跨域请求
- 同源策略
- CORS
- XMLHttpRequest
- Fetch API

### 1.2.1 跨域请求

跨域请求是指从一个域名下的网页访问另一个域名下的资源。这种访问方式在传统的浏览器安全模型中是被禁止的，因为它可能会导致安全问题。

### 1.2.2 同源策略

同源策略是浏览器的一个安全策略，它限制了从同一个域名下的网页访问另一个域名下的资源。这是为了保护用户的隐私和安全，防止恶意网站窃取用户的数据。

### 1.2.3 CORS

CORS（跨域资源共享）是一种浏览器安全机制，它允许一个域名下的网页访问另一个域名下的资源。CORS 是一种通过HTTP头部信息进行跨域请求的方式，它允许服务器决定是否允许跨域请求。

### 1.2.4 XMLHttpRequest

XMLHttpRequest是一种用于在浏览器中发送HTTP请求的API，它允许网页在后台与服务器进行通信。XMLHttpRequest可以用于发送GET和POST请求，并处理服务器的响应。

### 1.2.5 Fetch API

Fetch API是一种用于发送HTTP请求的API，它是XMLHttpRequest的一个更现代的替代方案。Fetch API提供了一种更简单和更灵活的方式来发送HTTP请求，并处理服务器的响应。
