# 2.量子通信中的信息编码
## 2.1 概念及术语说明
信息传输是量子通信中最基础的功能之一，目的就是使得接收者获得发送者发送的信息。在物理层、数据链路层或网络层等不同的层次上都可以实现信息的传输，通过信道传输的信息被送入接收者的缓存区，即使收不到信息，接收方也能获得有关的信息。

信息编码是指对要传送的数据进行信号化编码，使之符合信道的特性并可被接收方识别。如以二进制表示的数据会被编码成一个或多个高低电平状态构成的信号，用光载波或射频信号等。由于信道的本质是电磁辐射，因此需要一种方法对传输的数据进行编码，从而使得传输过程中能够抵抗干扰和抗电磁干扰，从而保证信息的准确性。

通信系统中信息编码可以分为两大类，即符号编码（如ASCII、BCH）和量子编码（例如BBPSSW、BPRL）。符号编码直接采用比特串来编码信息，这种方式简单易懂，且编码后的数据大小与信息本身相同；然而，由于符号编码存在纠错错误率过大的缺点，使得在传输过程中出现少量误码率较高的情况。另一方面，量子编码采用的是量子态，其编码效果更好，编码后的比特串比符号编码小很多，但仍然受到干扰影响，导致信息传输效率低下。

量子通信系统中常用的两种信息编码方案分别是BBPSSW和BPRL。

BBPSSW（Binary Block Pseudo-Shifted-Walsh-like code），是一种基于二维码生成矩阵的编码方案。该编码方案由伯恩斯坦和韦恩于1997年共同提出的，并被用于量子通信系统中。这种编码方案的基本思想是在通信线路上产生奇异的多粒子相互作用，从而生成所需的干扰。通过系统性地研究各种线路上可能产生的奇异多粒子相互作用，该方案首次把高阶非局域概率计算的问题转变为了实际可行的计算模型。

BPRL（Binary Pseudo Randomized Linear code），是一种基于线性熵源的编码方案。它由奥古斯特·康马斯提出，基于伽罗瓦空间上的随机变换及基于密钥分布函数的编码方法。该方案不需要生成矩阵，而是利用预定义的线性熵源模板，将原始数据打乱并编码。这样，编码得到的数据块长度与输入数据的长度相同，并且很难再被推测出来。与BBPSSW相比，BPRL的编码效率更高，但传播时延较长。

除此之外，还有其他一些量子通信系统使用的信息编码方案，如QPSK、Golay、CSS、SIC、Polar、PAM、Chase等。

## 2.2 基本概念
### 2.2.1 时变的编码
时变编码(time-varying coding)是指在通信信道内，把一段时间内的输入信号转化为一段时间内输出信号的过程，如将连续的脉冲信号变换为电流脉冲信号、带宽扩大后的脉冲信号或者其他编码形式的信号。时变编码起到的作用是抵消或减少信道中非理想的噪声效应。常见的时变编码有AWGN、高斯白噪声、退信道线性回波、频谱扭曲、比例失真、稳态跳变、施加微波干扰等。 

### 2.2.2 非标准编码(non-orthogonal encoding)
非标准编码(non-orthogonal encoding)，又称为非正交编码、特殊编码或相关编码，是指一种通信信号在不同基带信道间进行通信传输时的编码方式。一般情况下，在信道内同时传输两个以上不同频率的信号，则发生混叠。这些信号如果不经过编码就直接作为载波信号传输，那么就会出现衍生波，从而降低信号的可靠性。非标准编码的主要目的是使每个基带信道内的传输信号具有截然不同的特征，从而阻止它们之间的相互干扰，达到信号传输的最佳效果。非标准编码可分为如下几种类型:

1. 选通编码：选择性信道遮蔽(Selective Channel Oscillation)
2. 分散编码：多干扰因子选择性反向偶极制导(Multiple Distortion Factor Selectively Inverse Pairing Polarization Technique)
3. 拼接编码：功率分散序列(Power Density Sequences)
4. 框架编码：多角度整形(Majority Angle Shaping)
5. 耦合编码：时序标志干扰(Time-Tagged Interference)

### 2.2.3 模块化传输(modular transmission)
模块化传输(Modular Transmission)是指把信号拆分成若干个小片段，然后按照规定的编码方式分别处理。模块化传输具有以下几个优点:

1. 提高编码效率
2. 可提高传输速率
3. 便于实验测试
4. 可以提供更多信息量

## 2.3 BBPSSW量子编码方案
BBPSSW(Binary Block Pseudo-Shifted-Walsh-like code)是一个基于二维码生成矩阵的编码方案。BBPSSW编码方案首先构造了一个适当大小的二维码生成矩阵，根据输入数据流图生成对应的二维码矩阵。其次，利用二维码生成矩阵对输入数据进行映射，得到编码后的数据流。

BBPSSW编码方案的特点如下：

1. 编码块大小与输入数据块大小一致，编码效率高
2. 只支持单个比特的误码率，即使出现极端误码率，也可以在最大允许码长内识别正确信息
3. 支持中间节点的跳跃错误修正
4. 实时性强，可以在线上实时传输高速数据

BBPSSW编码方案的步骤如下：

1. 构建生成矩阵
2. 数据映射
3. 添加校验码
4. 编码译码器

### 2.3.1 构建生成矩阵
BBPSSW的生成矩阵由元素为 $ \pm I_N $ 的单位阵和 $ N^2 - 1 $ 个交叉阵构成，其中 $ I_N $ 表示 $ N \times N $ 的单位阵，$ \cdot $ 表示内积。构建生成矩阵的目的是希望生成的交叉阵与原始数据流图之间存在着互补关系，以方便交叉约束来消除模块间干扰，并对原始数据进行划分。生成矩阵的构造方法如下：

1. 根据输入数据流图设计一套交叉阵约束条件，比如：所有的边必须至少连通三个节点。
2. 用不同的权值代替交叉阵，构造生成矩阵。

### 2.3.2 数据映射
数据映射时对输入数据进行编码的过程。根据生成矩阵和相关性对输入数据进行映射，得到编码后的数据流。映射的方法有两种：

1. 使用置换矩阵乘法，根据生成矩阵生成所有可能的编码序列，然后对输入数据流应用这些编码序列。
2. 根据生成矩阵设计密集编码器，以图聚类的思想对数据流图进行划分，生成不同类型的二维码，再进行编码。

### 2.3.3 添加校验码
添加校验码的目的是在编码后的结果中增加校验信息，防止信息损坏。BBPSSW使用了单比特奇偶校验码。奇校验，即将输入比特串中的每一位与常数 $ c $ 进行XOR运算，运算结果为1则对应输出比特位的值为1，否则为0。偶校验，则只保留输入比特串中的1位，对输入比特串中的0位进行翻转，然后重复步骤1。

### 2.3.4 编码译码器
编码译码器(Encoder/Decoder)是把输入数据流分割成若干个模块，在每个模块之间设置连通性约束，然后依照相应编码方案对模块进行编码，最后将模块的输出重新组合成完整的数据流。编码译码器在收发双方间进行工作。

## 2.4 BPRL量子编码方案
BPRL(Binary Pseudo Randomized Linear code)是一个基于线性熵源的编码方案。BPRL编码方案可以说是BBPSSW编码方案的改进版，但与BBPSSW不同的是，它不需要生成矩阵，而是利用预定义的线性熵源模板，将原始数据打乱并编码。

BPRL的编码方法可以分为以下三步：

1. 对输入数据流图进行分块，每个分块内部的比特串随机排列。
2. 将随机排列的数据划分为多重熵源模板，每个熵源模板根据上一步的顺序进行混淆。
3. 通过均衡熵源模板和复杂混响模型模拟产生的混合信号，生成编码后的结果。

BPRL编码方案的特点如下：

1. 无需生成矩阵，编码效率高
2. 不支持跳跃错误，只能纠正单比特错误
3. 需要采用特别优化的傅里叶变换求逆的方式来计算反向信息
