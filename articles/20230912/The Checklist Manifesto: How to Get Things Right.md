# 2.基本概念术语说明
# 2.1 清单（checklist）
清单，又称计划表、工作表、待办事项清单、工作过程图等，是一种记录工作细节和作业安排的工作总结和检查手段。它通常由一个中心主题或主要目标，以及根据中心主题分组的一系列条件或过程项组成。每个条件或过程项列出一条线索或提示，并且带有“符合标准？”或者“该做哪些准备工作？”，每个任务列出清晰的指令或步骤，并且前后相互关联。清单有助于组织、计划、跟踪、审核、控制、监督和完成工作，从而减少损失、缩短完成时间、降低风险、提升工作效率。
# 2.2 模块（module）
在清单中，每个模块即表示完成某项任务所需的准备步骤，包括具体的工作指令和工具。模块可以划分成子模块，子模块又包括任务。每个任务是一个完整的、可以独立完成的事务，一般由任务名称、目标、预期结果、关注点、注意事项、检查清单等组成。任务有其功能、步骤和优先级三个方面属性，它也是分解清单的基本单元。
# 2.3 过程图（process map）
过程图是一种组织、流程和结构化的图形绘制方式。它通常是以流程的视角展现事物发展的全貌。过程图通常由流程节点和连线构成，用于描述产品或服务的各个阶段，并明确展示各个节点之间的关系和交流信息。过程图既可以作为工作流程的视觉呈现，也可以作为团队合作的依据。

# 3.1 分解清单
清单可分解成多个模块，每个模块是一条条目，清单是所有模块的集合。例如：创建项目的步骤，可能包括需求分析、业务模式定义、架构设计、数据库设计、前端设计、后端设计、测试设计、部署发布、运维支持等。每条步骤成为一个模块，每个模块的名称，就是一条清单上的条目名。这就可以很好地帮助团队成员对工作进行分解，找出障碍点，并用清晰、具体的方式完成工作。
# 3.2 执行任务时要掌握的技能
当清单中的任务列表被分配到团队成员身上时，他们就会开始逐个执行任务。执行任务时，应该具备以下几个基本技能：
- 理解清单条目中的目标、要求和预期结果，以便顺利完成工作。
- 以专注的态度，仔细审视任务中的各个方面。
- 细心耐心，认真细致地执行每项任务。
- 在执行过程中，要能够正确处理各种状况，避免出现意外情况。
- 抓住重点，密切关注被分配到的任务，并及时提供反馈。
- 不断完善自己的工作技能，提升自己的工作效率。
- 每周至少需要花费几个小时的时间，以提升自己的效率。
- 一定要遵守公司的内部规范和道德规范。
# 3.3 建立健康、协同的工作氛围
为了提升工作的有效性，需要团队中的成员具有高度的沟通、协作精神。每天早晚，大家要集中精力，组织协调工作。因此，团队成员应尽量设定共同的目标，制定共同的工作计划，并保持积极的沟通。这是实现目标的必要条件。建立健康的工作氛围，也会促使团队成员之间产生正向的影响，并逐渐形成团队凝聚力。
# 4.具体代码实例和解释说明
# 4.1 创建一个项目的清单
项目创建清单如下：

1. 确定客户需求
2. 收集客户反馈意见
3. 撰写业务方案
4. 提供技术方案
5. 招募团队成员
6. 项目总结和评估
7. 将项目方案提交给CEO批准

模块1: 确认客户需求
- 了解客户需求，收集信息，确定目标和范围。
- 收集客户反馈意见，确保客户需求满足范围和质量标准。
- 将客户需求整理成文档，提供给CEO进行审阅。

2.2 模块2: 收集客户反馈意见
- 通过现有的渠道，如电话、邮件、网络等，收集客户反馈意见。
- 使用户口舒适，并征求客户的建议。
- 确保收集到的信息可靠，确保准确把握需求。

2.3 模块3: 撰写业务方案
- 检查项目过程是否合理，并找出关键环节和功能。
- 用流程图或网络拓扑图，描述整体框架和各个环节之间的交流信息。
- 根据目标客户市场、业务类型、产品定位，编写完整的产品或业务方案。
- 将方案文档发送给CEO，对方案进行评审。

2.4 模块4: 提供技术方案
- 判断项目选择的技术栈、开发环境是否适合客户的业务规模和性能。
- 将项目技术路线图、功能模块列表、技术架构图、技术实现方式等，提供给客户。
- 对技术方案进行技术评审，讨论技术路线是否合理、实施方案是否可行。

2.5 模块5: 招募团队成员
- 熟悉客户需求，并与客户保持紧密联系。
- 选派专业技术人员，为项目做好准备。
- 为团队成员提供相关培训，确保合格、高效。

2.6 模块6: 项目总结和评估
- 对项目结果进行总结，包括产品生命周期结束后的反馈、运营效果、财务状况、法律义务等。
- 对项目成败原因进行分析，并与客户保持良好的沟通，达成一致的协议。
- 对项目进行评估，根据实际情况进行调整，确保项目按计划完成。

2.7 模块7: 将项目方案提交给CEO批准
- 当项目完成时，提交最终的方案给CEO审阅，接受或拒绝。
- 对项目方案进行总结，制定下一步工作计划。
- 在CEO和团队成员间传播项目方案。

# 4.2 检查一个产品是否符合销售规范的清单
产品检查清单如下：

1. 查看产品价格是否合理
2. 查看产品特性是否符合要求
3. 查看用户评论是否代表性
4. 查看产品试用情况
5. 查看产品品牌资产是否侵权
6. 提交问卷调查
7. 跟进反馈结果

模块1: 查看产品价格是否合理
- 核实产品价格与市场目标是否匹配，价钱能承受得住亏损的风险。
- 了解竞争对手的价格，判断产品价格的优势或劣势。
- 在售卖之前与客户签订购买协议。

2.2 模块2: 查看产品特性是否符合要求
- 阅读产品说明书，了解产品功能，使用场景，与竞品对比。
- 核实产品的可用性、易用性、安全性、兼容性、流畅性等特性是否符合规范。
- 与客户确认产品的使用限制和政策，确保合规。

2.3 模块3: 查看用户评论是否代表性
- 核实用户在网上发布的评论内容，与产品实际情况是否吻合。
- 从网站上下载评论数据，分析用户情感，判断用户满意度和满意程度。
- 将反馈转发给产品经理，与产品经理一起讨论，收集总结用户意见。

2.4 模块4: 查看产品试用情况
- 核实产品最新试用情况，如热门度、参与人数、试用反馈、质量、价格等。
- 了解客户试用反馈，判定产品是否需要进一步迭代。
- 如果产品需要进一步迭代，征求客户反馈意见，制定后续版本的优化方案。

2.5 模块6: 查看产品品牌资产是否侵权
- 查询商标注册状态，确认是否有侵权事宜。
- 与侵权方取得联系，举证报案。
- 立案调查，向法院起诉。
- 向公众披露情况。

2.6 模块7: 提交问卷调查
- 定期进行产品调研和用户访谈，提出问题，收集反馈。
- 分析客户反馈，获取更多的产品信息。
- 定期收集数据，进行统计，制作报告。

2.7 模块8: 跟进反馈结果
- 将反馈结果和数据呈现给部门主管，支持其决策。
- 听取CEO的汇报，组织后续工作。
- 持续更新数据，使产品特性有针对性地进行调整。

# 4.3 写推荐信的清单
推荐信撰写清单如下：

1. 填写邀请函模板
2. 添加联系方式
3. 描述产品优势
4. 描述为什么要联系
5. 添加合适的签名
6. 准备礼品卡
7. 寄送出去

模块1: 填写邀请函模板
- 查找模板，在模板上填入相关信息，如姓名、年龄、性别、年份、工作年限、地址、公司名称、联系方式等。
- 保存好模板，等待接收人准备好礼品卡。

2.2 模块2: 添加联系方式
- 使用有效的邮箱地址、手机号码，发送推荐信。
- 在邮件末尾添加联系方式，确保接收人能够收到消息。

2.3 模块3: 描述产品优势
- 描述公司产品的特色和优势，提高推荐信的知名度和社会影响力。
- 提及产品的核心价值，比如速度、稳定性、体验、性能、使用方便等。

2.4 模块4: 描述为什么要联系
- 选择正确的邮件主题，明确推荐人的需求。
- 简单描述对方的公司历史、产品经历、收入来源、职业规划等。
- 在主题行末尾添加“祝您工作顺利！”，加强宣传效果。

2.5 模块5: 添加合适的签名
- 多种符号和文字格式组合，增加不同气氛。
- 选择具有亲和力的个人签名，突出个人魅力。

2.6 模块6: 准备礼品卡
- 打印或制作一个礼品卡，并缴纳邮费。
- 填写内容包括赠品清单、发货地址、寄送时间、联系方式等。
- 告知接收人如何将礼品卡寄送到指定地址。

2.7 模块7: 寄送出去
- 将所有材料放进信封，轻轻放在快递箱里。
- 提醒接收人收件箱中保留地址。
- 选择其他快递公司，运送礼品卡。
- 等待收件人签收。

模块1: 安装PySftp模块
```python
pip install pysftp
```

2.2 模块2: 配置SSH连接参数
```python
import paramiko

host = "example.com" # ssh主机地址
port = 22             # ssh主机端口
username = "root"    # ssh用户名
password = "password"   # ssh密码

transport = paramiko.Transport((host, port))
transport.connect(username=username, password=password)
sftp_client = paramiko.SFTPClient.from_transport(transport)
```

2.3 模块3: 读取本地文件
```python
local_file = open("localfilename", "rb")
remote_path = "/home/user/remotepath/" + local_file.name 
```

2.4 模块4: 打开远程服务器目录
```python
sftp_client.chdir("/home/user/")
```

2.5 模块5: 上传文件
```python
sftp_client.putfo(local_file, remote_path)
```

2.6 模块6: 文件传输完成
```python
print("File uploaded successfully!")
transport.close()
```
