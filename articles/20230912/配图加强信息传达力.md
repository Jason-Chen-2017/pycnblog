# 1.简介
  

无论是一篇技术类文章还是科普类文章，配图都能极大提高信息的传达力。在不同的情景下，比如图文并茂、多图展示等，配图所传达的信息的表现形式也会不一样。而配图的应用场景也从早年简单到如今依次为网络直播、微博、微信朋友圈等，使用范围更广泛。因此，如果掌握了一些技巧，用对配图的运用方式，那么就能通过创意的手法来传达更多的信息。

本文先对配图的相关知识做个简单的介绍，然后再介绍其在机器学习领域中最重要的一环——数据可视化。最后还要结合实际案例，分享一些可以借鉴的经验，帮助读者加深对配图的理解。


## 2. 基本概念术语说明
2.1 配图概述
配图是一种将某些数据呈现的方式。通常情况下，配图用于描述一些抽象的主题，或者向读者展示某个过程的细节。根据它的引入位置不同，配图可分为以下几种类型:

- 描述性配图（Infographics）: 将复杂的数据以图形的方式展现，通过直观的视觉效果传递信息。例如公司报告、产品路线图、营销宣传海报等。
- 数据可视化图表（Data visualization charts）: 以图表或矩阵的形式显示数据，用来展示一些特定指标的变化趋势。例如气候变化图表、网页浏览流量图表、客户消费趋势图表等。
- 技术实现示意图（Technical illustration）: 用图表、流程图、树状结构等展示计算机系统的各个功能模块和连接情况。例如阿里巴巴内部业务系统的架构示意图、SVM模型的理解示意图、数据库索引原理图、RESTful API 请求流程图等。

2.2 配图类型
目前，关于配图的分类较为模糊。一些比较成熟的类型如下：

- 概览图（Overview maps）: 描述复杂数据的地理分布、空间特征，一般用于介绍一个区域的状况。例如谷歌地图、OSM地图等。
- 数据统计图（Data stats maps）: 在数据可视化中，可将某一维度（如时间、价格、地区）的数据绘制成热力图、蜂群图等，用来分析某项指标在不同时间、地点的分布及发展变化。
- 信息组织图（Information architecture maps）: 通过以图形的方式呈现信息架构中的节点关系，能够有效地展示信息之间的联系及信息的流向。例如，业务架构图、交通工程图、电信网络拓扑图等。
- 分布式系统图（Distributed system maps）: 使用图形表示分布式系统的组成和交互，从而促进对系统架构的理解，发现潜在的问题或瓶颈。
- 模型演化图（Model evolution graphs）: 对模型的训练过程进行可视化，展示模型随着时间的推移训练出来的效果，帮助用户更好地理解模型的性能指标，并发现模型的改进方向。

除此之外，还有一些特殊类型的配图，比如游戏配图、漫画配图等。同时，由于结合实际案例的方式需要结合专业知识才能够做出更具说服力的内容，因此，后续的文章中会涉及到一些机器学习中的实践经验，希望能给大家提供一些借鉴。
