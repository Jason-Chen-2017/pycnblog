# 1.简介
  

随着信息技术的飞速发展，计算机网络的出现成为IT界的一大热点话题。网络通信是互联网的基石之一，构建一个良好的网络通信基础设施可以让人们更好地进行交流、沟通和合作。因此，了解计算机网络通信的各个层级、协议及其功能对于开发者和系统管理员来说都是至关重要的。

《[4] 学习计算机网络协议——从物理层到应用层 [2017版]》一书根据作者多年丰富的网络通信技术经验，结合最新研究成果，为广大读者提供系统性、全面、深入、透彻的计算机网络协议知识体系。本书共分八章，分别介绍了物理层、数据链路层、网络层、传输层、应用层等七层网络协议的特性、模型、原理、功能和特点，并通过配套的代码示例介绍了这些协议的实现方法。本书还针对不同类型用户和不同场景进行了优化设计，力求让读者在阅读时能够获得最易于理解的讲解，并在实践中快速掌握相关技术技能。

本书适用于需要系统掌握计算机网络协议底层细节和关键技术能力的系统工程师或技术人员。

# 3. 一、介绍物理层
物理层的任务是传输比特流，即将0、1串行传输到另一端。物理层的数据传输单位是bit，它规定了电信号的传输规则，包括信号的构成、编码、调制、波形控制、信道分配、噪声抑制等过程。传输过程中会引入各种误码，因此物理层采用差错编码（error-correcting code）对传输的信息进行纠正。常用的物理层协议包括:

1. 单线信道(Synchronous Optical Network)

2. 双绞线(Twisted Pair Copper)

3. 以太网(Ethernet)

4. Wi-Fi(无线局域网)

5. 晶体管(Integrated Circuit Wireless)

6. 光纤(Fiber Optic)

# 4. 二、数据链路层
数据链路层(Data Link Layer)的任务就是传送数据帧。数据链路层负责把网络层传下来的分组发送到相邻结点的物理媒体上，并进行错误检查、重传、传输控制等机制。常用的协议包括:

1. HDLC(High-Level Data Link Control)

2. PPP(Point-to-point Protocol)

3. CSMA/CD(Carrier Sense Multiple Access with Collision Detection)

4. ATM(Asynchronous Transfer Mode)

# 5. 三、网络层
网络层(Network Layer)的任务就是选择一条可到达目的地址的路径。网络层确定了收到的包的正确性、顺序性和不重复性，并通过路由选路功能把数据包送往目标计算机。网络层的协议包括:

1. IP(Internet Protocol)

2. ICMP(Internet Control Message Protocol)

3. ARP(Address Resolution Protocol)

4. RARP(Reverse Address Resolution Protocol)

5. OSPF(Open Shortest Path First)

6. BGP(Border Gateway Protocol)

# 6. 四、传输层
传输层(Transport Layer)的任务就是建立两台计算机之间的数据传输通路，负责为应用进程之间的通信提供通用的数据传输服务。传输层协议包括:

1. TCP(Transmission Control Protocol)

2. UDP(User Datagram Protocol)

3. SCTP(Stream Control Transmission Protocol)

4. RTP(Real-time Transport Protocol)

5. RTCP(RTP Control Protocol)

# 7. 五、应用层
应用层(Application Layer)的任务就是向用户提供各种应用程序。应用层协议包括:

1. HTTP(Hypertext Transfer Protocol)

2. SMTP(Simple Mail Transfer Protocol)

3. SNMP(Simple Network Management Protocol)

4. DNS(Domain Name System)

5. DHCP(Dynamic Host Configuration Protocol)

6. TFTP(Trivial File Transfer Protocol)

7. FTP(File Transfer Protocol)
