# 2.基本概念术语说明
## 软件设计模式（Design pattern）
软件设计模式是一套被反复使用的、最佳实践的流程或方法，它提供了描述一个问题的方案，并隐含的提出了解决该问题的方法。通俗来讲，就是在不断重复实践中总结出来的一套经验。其目的是为了重用设计好的代码，避免重复造轮子，提高代码质量。比如，创建型模式如工厂模式、抽象工厂模式、单例模式，结构型模式如代理模式、桥接模式、组合模式，行为型模式如迭代器模式、命令模式、备忘录模式等。
## 可伸缩性（Scalability）
可伸缩性，即指当某系统需要增加处理能力时，可以快速而有效地增减资源，以适应新的需求。通常来说，可伸缩性分为两种类型：垂直可伸缩性和水平可伸缩性。
### 垂直可伸缩性
垂直可伸缩性又称为纵向伸缩性，意味着通过增加服务器的数量来实现处理能力的增加。这种方式比较简单直接，但是会影响到硬件配置、服务器布局、网络带宽等软硬件的配合问题，且可能会带来复杂的系统故障问题。
### 水平可伸缩性
水平可伸缩性又称为横向伸缩性，意味着通过增加服务器集群的数量来实现处理能力的增加。这种方式相较于垂直可伸缩性有明显的优点，如节省硬件成本、弹性扩容等。但同时也会带来一些复杂的问题，如数据同步、服务的路由分配、负载均衡等问题，因此水平可伸缩性往往要结合使用其它技术手段才能够实现。
## API接口（API Interface)
API接口，即应用程序编程接口。通常，API接口定义了一组函数、规则和协议，使得不同程序之间的交互变得简单和可靠。API接口是对外暴露给调用者的一系列服务的集合。目前最流行的两种类型的API接口分别是RESTful API和SOAP Web Service。
### RESTful API
RESTful API，全称Representational State Transfer，是一种用来开发Web API的风格。它定义了URI作为访问资源的路径，HTTP请求用于操作资源。RESTful API一般都具备如下特征：
- 1.客户端-服务器端分离：RESTful API将API的请求端和响应端分开，允许客户端和服务端独立开发。
- 2. Stateless：无状态，每次请求之间没有关联的信息。
- 3. Cacheable：可缓存。通过Cache-Control或者Etag头控制缓存。
- 4. Uniform interface：统一接口，资源的表现形式由URL表示，请求方法(GET/POST/PUT/DELETE)指定操作方式。
- 5. Client-side validation：客户端验证。服务器端只校验参数的正确性。
- 6. Layered system：层次化架构，每层提供不同的功能，可以做到互相独立扩展。
- 7. Code on demand（可选）：按需代码。允许客户端动态下载执行的代码。

### SOAP Web Service
SOAP（Simple Object Access Protocol），是基于XML的远程过程调用（Remote Procedure Call，RPC）协议。SOAP中定义了如何构造、序列化、编码、发送和接收SOAP消息。因此，SOAP可以实现跨平台、跨语言通信。与RESTful API不同，SOAP Web Service依赖于SOAP协议，而RESTful API不需要依赖任何协议。由于RESTful API一般都要通过HTTP协议传输，所以性能上存在一定的局限性。
