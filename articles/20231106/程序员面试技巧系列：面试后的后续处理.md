# 1.背景介绍


对于一个资深技术专家来说，面试官和面试者之间的交流往往最重要的环节之一就是面试后的后续处理。一般来说，面试官通常都会要求候选人在线上回答问题，但面试结束之后还需要进行面试后处理工作。

无论是不是招聘技术人员，或是面试别人，面试后的后续处理都是一个至关重要的环节。面试官会对你的表现给予评价、安排工作机会或是提供优惠券，但是面试后的后续处理能够帮助你更加有效地利用你所获得的信息。

因此，了解并掌握面试后的后续处理方法将会成为你面试中的关键。本文将基于计算机相关专业面试知识展开阐述。

# 2.核心概念与联系
## 2.1 面试官综合能力分析
- 个人能力: 面试官在对你进行综合能力分析时，主要从以下方面考虑:
  1. 态度：面试官会通过问一些开放性的问题，比如说：你为什么选择这个职业，你期望得到什么？
  2. 知识储备：面试官会让候选人介绍自己的一些编程经验和学习方法。同时，也会让他们描述一下公司的业务、产品及团队情况等等，以及询问候选人的求职意向。
- 工作绩效: 面试官会根据你的工作经历、工作表现以及其他同事反馈信息，做出相应的评价，比如说：工作上有哪些进步、不足之处、可以改善之处等等。
- 社会阅历: 在面试官看来，作为技术人员，你要具备一定社会阅历和竞争力。所以，在面试过程中，面试官会提出一些类似于“为什么你适合这个岗位”这样的非常实际的问题，这些问题能够更好地让面试者展示自己作为技术人员的潜力和资源。

## 2.2 后续处理的不同阶段
在面试完毕之后，面试官一般会根据候选人对自己的感觉、能力评估和面试过程中的反馈，给出相应的指导，包括面试成绩、推荐 offer 或直接辞退等。

一般而言，面试后的后续处理分为以下三个阶段：

1. **初试失败**

   当初试结果表明你没有过硬的技术基础或者技术功底的时候，一般会被面试官叫停甚至是被直接辞退。这个时候，如果没有特别强烈的自我否定，一般情况下，面试官不会很理解，毕竟这种情况很少发生，而且以后应该会越来越难。

   如果你能在这个阶段总结一下自己的错误、缺点和不足，能够更好地为自己找到突破口，并且能找老板或领导反映，那就太好了！不要着急、不要气馁，坚持下去。一定会有好结果的！

2. **技术面试胜利**

   在技术面试胜利的阶段，一般候选人已经具备了一定编程水平，一般也都能在企业内部拿到比较好的 offer。这个时候，面试官可能会根据候选人的优秀履历、完成的项目、职称、面试经验等因素，给出一些建议。

   有时候候选人通过技术面试后，可能会遇到一些尴尬的事情，比如被忽悠、挖苦，没准会导致技术面试失败，需要重新寻找另一份工作，这些都是值得倾听的，记住：只有充分认识自己的弱点，才能在技术面试中锲而不舍！

3. **offer 提交阶段**

   一般当候选人接受 offer 时，面试官就会给予面试官和候选人的面谈机会，并根据双方面的反馈信息，给出最终的决定。通常情况下，面试官会让候选人接受 offer 之后，在确认好薪酬福利之后立即安排入职培训等流程。
