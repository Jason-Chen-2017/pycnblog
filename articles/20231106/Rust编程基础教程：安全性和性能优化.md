## 2.1.什么是Rust？
Rust 是 Mozilla Research 创建的一个开源、静态类型的编程语言，由 Mozilla 基金会支持。Rust 是系统编程语言，编译时不需要运行时分配器（allocator），其运行速度也快于 C 和 C++。同时，Rust 提供了很多安全性保证和内存管理功能，使得编写不容易出现错误的程序变得十分简单。Rust 的主要特点如下：

1. 内存安全
   - Rust 通过所有权系统和借用检查来保障内存安全。在编译时检测并防止数据竞争和空指针引用；
   - 数据结构布局可以由编译器自行决定，避免手动管理内存；
2. 线程安全
   - Rust 中没有全局变量，所以线程之间完全隔离；
   - Rust 中的所有数据类型都是线程安全的，可以在多个线程之间共享；
3. 类型安全
   - 在编译时就能发现代码中的错误；
   - 可以在编译时做出内存分配和初始化的决定，避免运行时开销；
4. 并发
   - Rust 支持基于Actor模式的并发模型；
   - Rust 有异步编程模型；
   - 可以利用微任务（coroutine）库实现协作式调度；
5. 更低的启动时间
   - Rust 编译后的代码非常紧凑，启动速度非常快；
   - Rust 使用 LLVM 技术来进行优化；
6. 语法简洁
   - Rust 用易于理解的语法，消除了 C++ 中复杂的嵌套语法；
   - Rust 的标准库（stdlib）有丰富且全面的功能；
   - 支持泛型编程和 trait 对象机制；
7. 包管理
   - Rust 支持依赖管理和版本控制，可以轻松导入第三方代码；
   - 没有 Java 的复杂依赖管理，没有 Maven 之类的繁琐配置；

## 2.3.Rust与其他编程语言的比较
Rust 和 C++ 属于同类语言，也是静态类型语言，但是它们之间还有一些差异。

| 对比项       | Rust              | C++                   |
| ------------ | ----------------- | --------------------- |
| 定位         | 系统编程语言      | 通用编程语言          |
| 底层支持     | 操作系统接口和 ABI | 虚拟机                |
| 目标机器类型 | 大端（x86_64）     | 小端（i386）           |
| 编译器       | rustc             | g++ / clang           |
| 运行效率     | 高于 C 和 C++     | 比较慢                 |
| 运行时       | 无需分配器        | 需要分配器             |
| GC           | 可选              | 不可用                 |
| 异常处理     | 可选              | 默认                  |
| 编码习惯     | 与 C 类似         | 与 C++ 类似           |
| 函数重载     | 不可用            | 可用                  |
| 跨平台性     | 一般              | 良好                  |
| 开发环境     | 普通编辑器或IDE   | IDE                   |

Rust 和 Python 属于动态类型语言，而 Go 和 Ruby 则属于脚本语言。Rust 的语法上更接近 C++，而且有 GC 可以自动回收内存，使得 Rust 适合于编写服务器软件。除此之外，Rust 拥有丰富的生态，包括用于 web 应用开发的框架 Rocket、异步 I/O 库 Tokio 等等。
