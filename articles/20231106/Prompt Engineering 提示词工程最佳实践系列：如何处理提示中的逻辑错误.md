# 1.背景介绍


提示词工程（Prompt engineering）是构建和训练文本生成模型的过程。其目的是通过提取、组织、呈现关键信息来提供给用户有价值的提示信息，帮助用户完成日常任务。然而，开发者在设计和编写提示时容易出现逻辑或语法上的错误，影响提示的有效性和准确性。本文将对这些错误进行分类和讨论，并分享一些经验技巧，供开发者参考。
提示词不仅会影响文本生成模型的性能表现，还可能导致用户质疑和恶意评价。因此，识别并修复这些错误对于提升模型的效果和品牌形象具有重要意义。同时，它也能促进文本生成的创新和应用，让用户在生活中真正实现自助服务。
# 2.核心概念与联系
## 2.1 常见错误类型
目前，提示词工程存在以下常见错误类型：
### 缺少目的性
提示词缺乏目的性通常是因为没有充分定义它的目标和使用场景，没有清晰地阐述业务需求。此外，一些较短的句子也可能会发生这种情况。例如："本站的内容包括：新闻资讯、科技资讯、娱乐资讯等"。
### 没有明确指代对象
提示词没有明确指代对象往往是由于对主语、宾语或其他特定要素的理解不到位。例如："感谢您的参观！期待再次光临"，用户很难明白这个消息应该是针对网站还是用户。
### 使用了模糊不清的动词
一些过于简单或模糊的动词可能导致用户误解。例如："亲爱的朋友，欢迎来到本网站！"，用户很难明白"来到"的含义。
### 不够精准
提示词需要考虑到业务实际情况和需求，不能仅仅满足于自己的想法和幻觉。过于模糊或抽象的语言可能会引起误会，甚至造成欺骗。例如："打车可以选择最便宜的快捷方式，比去汽车中心用人力更省时间和费用。"，用户很难判断"快捷方式"具体指的是哪种。
### 缺少关联性
提示词缺乏关联性不仅会令用户困惑，而且可能使得模型无法理解该提示所关注的问题。例如："周末去长城遛弯比较方便"，用户不知道长城是什么地方，也不知道为什么要去长城。
## 2.2 常见解决方案
开发者应当注意以下几点，根据不同错误类型及时发现和纠正错误。
### 目的性
为了向用户传递信息，一定要定义清楚目的和使用场景，并且让用户能够清楚地理解其含义。例如，可以通过让用户在问询之前预先了解网站的主要功能，再解释提示内容的方式来增强提示的相关性。另外，还可采用较长的句子来更好地突出提示的目的。
### 对象指代
务必清楚地指出要提醒或回应的内容的对象，并提前介绍其基本知识。这有利于提高提示的准确性。例如，提示可以使用“亲爱的X，欢迎您访问XX网站！”之类的语句来具体指出，而不是泛泛而谈。
### 动词使用
除了注意动词的单一含义、时态和形式，还应当注意避免不必要的否定或肯定，尤其是在描述行为时。例如，如果提示使用动词“可以”，则要慎用。
### 精准度
提示词一定要达到业务实际情况的精准度。这要求设计人员不断迭代和反馈，以获取用户的反馈，然后根据用户的实际情况完善相应的提示。
### 关联性
提醒或者回应用户关注的问题必须与提示词相关。一般来说，关联性越强，提示词的效用就越大。所以，在设计提示词时，需时刻记住，所提醒的主题一定是用户关心的，通过给予足够的上下文和关联，让用户真正理解提示词的内容。
