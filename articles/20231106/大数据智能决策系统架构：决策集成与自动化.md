# 1.背景介绍


在企业应用系统中，决策支持系统(DSS)起到了关键作用。比如，当订单支付成功后，供应链管理系统需要根据历史订单数据做出相应的调整，才能确保公司产品质量的长期稳定性和竞争力。传统的决策支持系统一般通过人工的规则或规则引擎实现决策支持。随着业务规模的扩大、数据的不断积累、人的精力不断下降，传统的决策支持系统面临技术革新、效率提升、可靠性增强等挑战。为了应对这些挑战，大数据技术及其衍生技术正逐渐成为决策支持领域的主流技术手段。

基于大数据决策支持系统的架构设计，主要可以分为以下几个方面：

⒈ 数据采集与存储：收集、整合、存储海量数据，包括原始数据（日志、电子文档、网页）、结构化数据（数据库表、文件）、半结构化数据（XML、JSON）。

⒉ 数据清洗、转换、拆分：从多种数据源中获取信息，进行数据清洗、转换和拆分，生成易于理解的数据格式。

⒊ 数据分析：对收集到的数据进行统计分析、机器学习、文本挖掘等分析处理，将有效信息提炼出来，并形成决策依据。

⒋ 决策算法的设计与开发：采用机器学习、模式识别、复杂决策树、神经网络等算法，建立有效的决策模型，能够完成复杂的业务流程和决策规则。

⒌ 智能决策的实现：将决策模型部署到智能决策引擎中，实现自动化的决策过程，有效地支撑企业的日常运营和决策支持。

基于大数据决策支持系统的架构设计，对于企业来说具有重要的意义。一方面，它能更好地获取、整理、分析数据，有效地解决现实世界的问题；另一方面，它能够实现更加快速准确的决策，为企业节省时间和资源，提升工作效率。

本文将简要阐述大数据智能决策系统的架构设计及相关技术方案，希望能为读者提供一些参考。

# 2.核心概念与联系
## 2.1 DSS概述
DSS全称决策支持系统，指的是企业内部用来支持业务决策的一系列工具和服务，如规则引擎、数据库、决策树、大数据平台、知识库、上下文感知系统、自适应引擎等。该系统所涉及的各个模块之间关系密切，功能相互制约，构成一个整体。一般包括如下四大功能：

- 规则引擎：用来处理简单和重复性决策任务，如“是否给予抵扣”“是否推送优惠券”，以及“是否给予信用额度”。
- 决策树：一种高度决策自动化的方法，基于业务知识库、规则、数据等综合分析，系统atically determines the appropriate action to take in a particular situation by analyzing data and algorithms with a set of decision rules or tree structures. 
- 上下文感知系统：是一个专门的模块，能够对客观事件和主观信息进行匹配、关联、理解、分析，生成策略，形成有效的决策结果。例如，通过购买行为分析，推荐适合用户的商品组合，增强用户体验，提高营销转化率。
- 数据库：用于保存原始数据、业务数据、规则、决策模型等，可用于灵活查询、报告、分析等。
