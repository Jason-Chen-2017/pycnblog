# 2.核心概念与联系
Python的系统编程就是指利用Python语言进行程序的编写，并在运行时系统地控制硬件资源，如内存、CPU、文件、网络等。这些系统资源被称为系统调用接口，或简称为syscalls。不同类型的系统调用（如打开文件、创建进程），对应不同的方法，如open()、fork()等。

为了更好地理解系统编程，需要了解以下几个重要概念：

1. 进程（Process）

   是指正在运行的应用程序。每个进程都拥有一个独特的进程ID（Process IDentifier，PID）。它包括了指令指针、寄存器集合、内存空间、打开的文件描述符、环境变量、运行时间等信息。一个进程可以包含多个线程，也可以从父进程派生出子进程。进程间可以通过系统调用互相通信。

2. 线程（Thread）

   是进程的一组执行流，可以看作轻量级进程。线程共享进程的所有资源，如内存地址空间、全局变量、文件描述符、信号处理句柏、堆栈等。不同线程之间可以交替执行，每个线程都有自己独立的调用栈、寄存器集合、线程局部存储区。线程间只能通过主动放弃CPU的形式切换，不能任意进行调度。

3. 协程（Coroutine）

   是用户态的轻量级线程，比线程更加简洁，能够完成复杂的任务。协程的调度完全由用户控制，可以方便地把一个耗时的任务分成若干个小的任务交给其他的协程去执行，也无需像线程那样要维护线程上下文切换的开销。

4. I/O Multiplexing

   是指同时监控多个输入/输出请求的程序设计技术。I/O Multiplexing允许应用程序同时等待多个输入/输出操作的完成，并根据操作结果轮询每个请求是否完成。I/O Multiplexing可以减少内核进行系统调用的时间。

5. IPC（Inter-Process Communication）

   进程间通信（Inter-Process Communication，IPC）是指两个或多个进程之间的信息交换，主要有共享内存、管道（Pipe）、套接字（Socket）、消息队列（Message Queue）等方式。共享内存方式要求两个进程必须建立一个映射关系才能访问内存，速度快，但稍微麻烦些；管道方式是在内存中创建一段缓冲区，进程通过向其中写入或者读取字节数据，另一进程通过读取或写入同一端的数据，适用于不同进程之间的通信；套接字方式提供了一种更高效的进程间通信方式，可以实现全双工通信、支持面向连接的传输等，适用于跨网络通信；消息队列方式通常在客户机和服务器之间传递消息。

除了以上这些概念，还有很多系统编程的术语需要熟悉。但基本上，掌握这些核心概念就足够了，更进一步的概念还需要读者自己深入研究。
