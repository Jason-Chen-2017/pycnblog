# 2.核心概念与联系
## 网络编程基础知识
计算机网络（英语：Computer Networking）是指利用分组交换技术把地球上分布广泛的各类计算机连接起来，使得它们之间可以相互传递信息、共享资源、协同工作、网络管理、应用控制等功能的计算机网络系统。简而言之，网络是由各种计算机按照一定的规则相互通信所组成的集合体，它是支持许多重要应用的支柱性技术。网络工程就是计算机网络的设计、构建、维护和管理的一系列工程活动。网络应用程序是网络服务的提供者，包括互联网浏览器、即时通讯软件、电子邮件客户端、文件传输协议(FTP)客户端、远程打印机、视频会议系统、路由器等。
### TCP/IP模型
TCP/IP模型是一个互联网通信协议簇，由一系列网络协议组成，它们一起构成了互联网的骨干。TCP/IP协议族提供了一种统一且简单的方式来互连不同的计算机网络，并提供有效的可靠性服务。TCP/IP协议族由四层协议组成：应用层，传输层，网络层，数据链路层。每一层都承担不同的数据处理任务。应用层决定了向用户提供什么样的服务，如Web浏览器或电子邮件客户端；传输层提供可靠的、面向连接的通信，确保两台计算机之间的通信安全；网络层负责点对点（或者多对多）通信，选择合适的路由路径；数据链路层负责结点间的物理通信。
### IP地址和端口号
Internet Protocol (IP) 地址用来标识网络上的设备。每个设备都分配一个唯一的IP地址。
IP地址通常表示为四个数字的序列，每个数字用“.”隔开。例如，172.16.17.32是一个IP地址。
端口号用来区分不同进程在同一台计算机上的作用。不同的应用程序可以运行在同一台主机上，因此需要用端口号来区分不同的进程。每个网络服务都由一个确定的端口号来标识，如HTTP的端口号是80。
## Socket编程
Socket（套接字）是Berkeley sockets API标准的一个实现，用于不同机器之间进行通信。Socket允许应用程序在两个正在运行的进程之间建立一个双向通信信道。在网络编程中，Socket主要用于客户端-服务器通信，尤其是在Web开发中。
Socket编程有两种模式：TCP Sockets 和 UDP Sockets。
TCP Sockets 是一种面向连接的、可靠的、基于字节流的通信方式。客户端首先发起请求连接到服务器，然后等待服务器的确认。如果连接建立成功，则服务器和客户端就可以开始发送和接收数据包。如果连接中断，则需要重新建立连接。TCP Sockets 使用两个步骤完成整个过程：创建一个Socket，并用connect()函数指定目标地址。然后调用send()和recv()函数来收发数据。
UDP Sockets 是一种非连接的、不可靠的、基于数据报的通信方式。不需要先创建连接，只需要绑定到某个本地IP地址和端口号，即可开始发送和接收数据包。UDP Sockets 使用三个步骤完成整个过程：创建一个Socket，用bind()函数绑定本地IP地址和端口号，再用sendto()和recvfrom()函数发送和接收数据。
