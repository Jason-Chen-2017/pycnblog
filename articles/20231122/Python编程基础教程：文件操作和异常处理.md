# 2.核心概念与联系
## 文件操作
文件操作（File I/O）是指对计算机存储设备上的文件的读、写、创建和删除等操作。由于每种文件类型和存储方式都不同，所以文件操作涉及到许多细节问题。

### 操作对象
- 打开文件：使用`open()`函数可以打开指定路径的文件，并返回一个文件对象。
- 关闭文件：当不再需要文件资源时，可以使用`close()`方法释放占用的内存资源。
- 读取文件：使用`read()`、`readline()`或`readlines()`方法可以从文件中读取数据。
- 写入文件：使用`write()`方法可以向文件中写入数据。
- 删除文件：使用`remove()`方法可以删除文件。

### 操作模式
- 文本模式：使用该模式打开的文件只能进行文本形式的数据读写。
- 二进制模式：使用该模式打开的文件能够读取字节流数据。
- 更新模式（追加模式）：在更新模式下，如果文件已存在则在文件尾部进行追加操作。否则创建一个新的文件。

### 编码格式
- ASCII：最简单的编码格式，只包含英文字符和数字。
- Unicode：提供了全世界所有语言的字符集支持，覆盖了几乎所有的字符。但是，使用Unicode编码代价很高，通常只能用来处理少量文本数据。
- UTF-8：Unicode编码的一种变体，它使用1-4个字节来表示每个字符。

## 异常处理
异常处理（Exception Handling）是指通过某种手段捕获错误并予以处理，从而使程序能够继续运行而不是终止运行。

异常一般分为两种：系统异常和用户异常。

系统异常是由操作系统抛出的错误引起的，如除零错误、越界访问、内存溢出等；用户异常是应用逻辑代码自己抛出的错误，如除零错误、类型转换错误、语法错误等。

### try...except块
使用try...except块可以在可能出现异常的代码块前后添加异常处理代码。如下所示：

```python
try:
    # 有可能产生异常的代码
except ExceptionType as e:
    # 捕获异常发生时的处理代码
    print(e)   # 报错信息输出到控制台
    logger.error(str(e))    # 报错信息记录日志
```

其中，`ExceptionType`是指定的异常类（如ValueError、ZeroDivisionError），`as e`语句用于将捕获到的异常实例赋值给变量`e`。此外，还可以通过设置多个except子句处理不同的异常类型。

```python
try:
    # 有可能产生异常的代码
except ValueError as e:
    # 处理值错误异常
    pass
except ZeroDivisionError as e:
    # 处理除零异常
    pass
else:
    # 如果没有异常发生，则执行这里的代码
    pass
finally:
    # 不管是否有异常发生，都会执行这里的代码
    pass
```

最后，`else`和`finally`语句分别在没有异常发生和异常发生情况下执行的语句块。

### raise语句
使用raise语句可以主动抛出异常。如：

```python
if x < 0:
    raise ValueError("x should not be negative")
```

这样的话，如果表达式`x<0`成立，就会抛出一个ValueError异常，并显示自定义的错误信息。
