# 2.基本概念术语说明
## 数据结构
数据结构是在计算机中存储、组织、处理、检索和操纵数据的形式化建模，它定义了数据的存储结构、关系和操作的方式。常见的数据结构有栈、队列、数组、链表、树、图、集合和字典。其中，栈、队列和数组都是线性数据结构，链表是非线性数据结构，树和图都是复杂数据结构，而集合和字典是特殊的容器。根据数据元素之间的逻辑关系，不同的数据结构适用不同的访问方式。
## 算法
算法（英语：algorithm）是指用来解决特定类问题的一系列操作的清晰指令，其精确定义和描述了一个计算过程，并由计算机执行。任何算法都要遵循某种有效的顺序，否则就会导致错误或无法达到期望结果。算法应具有可重复性，即在相同的输入条件下，算法的输出必定相同；而且应具有确定性，也就是算法所实现的功能应该是一致的，且每一步都有确切的定义；同时算法应该高效，执行效率应该尽可能地高。常见的算法有排序算法、查找算法、动态规划算法、贪婪算法、回溯法、分治法、博弈论算法等。
## 时间复杂度
时间复杂度（Time Complexity）是一个函数，用于衡量算法中语句执行次数与数据规模的增长速度的一种测度标准。一般情况下，算法中语句的执行次数称为时间复杂度，记作 T(n)。T(n) 由以下两个部分组成：
- 一次项 O(1): 指数级数量级的低阶项，如 if/else 或赋值运算。
- 多次项 O(n^c): n 的某个常数 c 次方，表示数据规模随问题规模增长的急剧增长，如线性搜索、简单排序算法。
通常，算法的时间复杂度可以粗略地分为：
- 最优时间复杂度 (Optimal Time Complexity)，又称渐进时间复杂度，表示该算法在最坏的情况下的运行时间。例如，快速排序算法的最优时间复杂度是 O(nlogn)，选择排序算法的最优时间复杂度是 O(n^2)。
- 平均时间复杂度 (Average Time ComplexITY)，表示该算法在所有可能输入下的平均运行时间。例如，哈希表的平均时间复杂度是 O(1) 。
- 最坏时间复杂度 (Worst Case Time Complexity)，表示该算法在最差的情况下的运行时间。例如，二叉树遍历算法的最坏时间复杂度是 O(n)。
# 3.核心算法原理和具体操作步骤以及数学公式讲解
## 插入排序（Insertion Sort）
插入排序是一种简单直观的排序算法。它的工作原理是通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。插入排序在实现上，在从后向前扫描过程中，需要反复把已排序元素逐步向后挪位，为最新元素腾出空间。
### 操作步骤
首先，取第一个元素作为有序序列。然后，按照顺序比较第二个元素与有序序列的每个元素，如果该元素小于等于当前元素，则插入到对应位置之后；否则，继续下一轮比较。
### 算法概述
插入排序算法的主要思路是通过“插入”方法将一个数据元素插入到已经排好序的列表中去。其算法流程如下：

1. 从第一个元素开始，该元素可以认为已经被排序。
2. 取出下一个元素，在已经排序的元素序列中从后向前扫描。
3. 如果该元素（已排序）大于新元素，将该元素移到下一位置。
4. 重复步骤 3，直到找到已排序的元素小于或者等于新元素的位置。
5. 将新元素插入到这个位置后。
6. 重复步骤 2~5 ，直到排序完成。

插入排序算法的平均时间复杂度为 O(n^2) 。原因在于，对随机的数据进行排序时，平均情况下每次插入操作只移动相邻元素，因此需要比较 n−1 个元素，故每个元素都要比前面所有元素都要少。所以，平均情况下，插入排序的运行时间为 O(n^2) 。
