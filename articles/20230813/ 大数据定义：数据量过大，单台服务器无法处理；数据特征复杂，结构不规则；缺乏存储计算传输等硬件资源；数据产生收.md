# 1.简介
  

## 数据产生背景
- 物联网时代，大数据的产生已经成为可能。传感器、图像、位置信息、运动轨迹等各类数据产生率越来越高，数据的采集、处理、分析、储存、传输等环节也变得越来越复杂。单个设备无法完全负责处理海量的数据，因此需要分散到多台计算机上进行处理。而这就需要有一个能够快速存储、处理、分析、查询大量数据的系统。这一需求让大数据研究者们看到了一个机会，开发出一种分布式计算框架。
- 当然，大数据并不是银弹。在没有非常先进的硬件基础设施的情况下，其处理能力还是受限于单台计算机的处理能力。此外，数据采集、处理、存储、查询等环节同时发生时，还存在着数据不一致的问题，这就要求系统具备强大的容错、恢复、迁移能力，才能实现较好的处理效果。另外，由于数据的结构、特性、规模等都存在一些限制，为了提升性能，往往要采用更加复杂的算法。最后，不同类型的数据可能具有不同的特点，这就要求系统能够灵活地应对不同的数据类型，并且根据需要动态调整计算任务的分配，确保整体的效率最大化。
## 大数据分类及定义
- **互联网数据**：从互联网网站收集、存储的数据，包括但不限于文本、视频、音频、图片等。通过搜索引擎、社交媒体、微博客等渠道，可以获得海量的互联网数据。
- **移动互联网数据**：通过手机APP、微信、支付宝等应用收集的用户数据，如浏览记录、使用习惯、订单数据等。
- **社会网络数据**：包括各种社交媒体平台提供的公开信息，如微博、QQ空间、新闻网站等。这些信息可以反映用户的兴趣爱好、意见反馈、行为轨迹、社交关系等。
- **云端存储数据**：通过云端服务商如Amazon Web Services、Google Cloud Platform等提供的分布式存储和计算服务，可以获得海量的云端数据。
- **物联网数据**：包括由IoT设备采集的实时数据，如环境数据、用电量、空气质量数据、卫生数据等。
- **其他类型数据**：包括行业或领域内的特定类型数据，如医疗健康数据、金融数据、制造业数据等。
- **数据量**：指系统中所存储、处理的总数据量。
- **数据特征**：指数据的结构、模式、分布、大小等特征。它是系统可靠性、准确性、有效性、鲁棒性的关键。
- **数据采集方式**：指数据的采集、获取的方式，比如通过爬虫、API接口、摄像头采集、GPS定位等。
- **数据处理方法**：指数据的处理、计算的方式，比如统计分析、机器学习、图像识别、文本挖掘等。
- **数据安全策略**：指保护数据的完整性、可用性和隐私性的方法，比如加密、访问控制、权限管理等。
- **硬件资源**：指用于数据存储、计算、传输等的硬件设备、资源，如内存、CPU、磁盘等。
- **数据收集、收集时间、收集范围、样本选择策略**：指数据收集的手段、方式、时长、范围、质量保证机制。
- **数据预处理方法**：指对数据进行清洗、验证、转换等预处理方式，如去除无关数据、特征工程、异常检测、数据扩充等。
- **数据模型设计和构建方式**：指如何根据现实世界的业务特点，建立符合需求的数据库模型。
- **数据存储方案**：指如何选取合适的数据存储介质，如分布式文件系统、NoSQL、搜索引擎等。
- **计算框架和调度策略**：指如何选择计算框架和任务调度策略，如MapReduce、Spark、Storm等。
- **查询语言和接口支持**：指系统支持的查询语言和接口形式，如SQL、RESTful API等。
- **数据访问控制策略**：指数据的访问权限控制策略，如白名单、黑名单、角色权限等。
- **数据迁移策略**：指数据的迁移方式，如自动化复制、异地容灾等。
- **数据增量更新策略**：指数据的增量更新方式，如按需更新、定时批量更新等。
- **数据质量保证方案**：指数据的质量保证、评估、监控方式，比如数据延迟、数据丢失、数据重复等。
