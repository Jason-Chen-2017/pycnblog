# 2.核心概念与联系

## 2.1 压力测试

压力测试（Stress Test）是一种用于评估系统性能的方法，通过对系统进行大量请求和高负载的测试，以评估系统在极端情况下的稳定性和性能。压力测试的目的是为了确保系统在高负载下仍然能够正常运行，并且能够满足预期的性能要求。

## 2.2 高可用系统

高可用系统（High Availability System）是指一种能够在任何时候保持运行的系统，通常通过多个副本、冗余组件和自动故障转移等方式来实现。高可用系统的目的是为了确保系统在故障时能够继续运行，并且能够提供可靠的服务。

## 2.3 联系

压力测试和高可用系统之间的联系在于，压力测试可以帮助我们评估高可用系统在极端情况下的性能和稳定性。通过压力测试，我们可以发现系统的潜在问题，并在实际运行时避免故障。
