## 3.3 Altibase并发控制

Altibase并发控制采用了ACID规范，包括：

- 原子性：一个事务要么全部提交，要么全部回滚。
- 一致性：事务执行前后，数据保持一致。
- 隔离性：不同事务之间互相隔离，不影响彼此。
- 持久性：事务提交后，数据持久保存到磁盘。

# 4.具体代码实例和详细解释说明

## 4.1 创建游戏角色表

```sql
CREATE TABLE game_character (
    id INT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    level INT NOT NULL,
    exp INT NOT NULL,
    gold INT NOT NULL
);
```

## 4.2 插入游戏角色数据

```sql

(1, 'Warrior', 10, 1000, 100),
(2, 'Mage', 15, 1500, 150),
(3, 'Archer', 8, 800, 80);
```

## 4.3 查询游戏角色数据

```sql
SELECT * FROM game_character WHERE level > 10;
```

## 4.4 更新游戏角色数据

```sql
UPDATE game_character SET level = 11, exp = 1100, gold = 110 WHERE id = 1;
```

## 4.5 删除游戏角色数据

```sql
DELETE FROM game_character WHERE id = 3;
```
