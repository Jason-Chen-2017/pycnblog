                 

# 1.背景介绍


随着互联网的普及和信息化的应用，基于人工智能（AI）的人工智障（Chatbot）已经成为主流的信息服务方式之一。然而，仅仅通过Chatbot并不能有效地完成业务流程任务的自动化处理。企业级的业务流程往往复杂多变，用人工的方式去完成这些流程耗费时间、力量及成本大。而为了提升效率，人们倾向于采用流程自动化的方式来解决这个问题。其中最具代表性的就是Robotic Process Automation（RPA），它可以实现对业务流程的自动化管理，帮助企业节约时间、降低成本、提高工作质量。同时，也能够提供更加精准的服务，改善客户体验。在这一背景下，很多公司都开始着手研发基于RPA的人工智障（Chatbot）。如今，针对RPA技术的企业级应用开发已经成为一个热门话题，但是由于各个公司自身应用需求的不同导致了差异化的开发难点，并且还存在诸多实际操作中的困难。因此，如何根据不同业务特点、目标用户群体，以及供应商的技术能力等因素综合考虑，以最优方案完成一个可用的企业级应用开发产品，将是企业面临的重大挑战。本文旨在通过33点的内容深入讨论一下基于RPA技术的企业级应用开发的移动与跨平台问题。
# 2.核心概念与联系
## 2.1 RPA与Chatbot
### 定义
**人工智障（Chatbot）**：基于人工智能的聊天机器人，包括基于文本、语音交互、图像识别等技术，具有快速响应、自学习、自适应、群队协作等特性。其作用一般分为四类：语言技巧类、日程安排类、事务处理类、自我助理类。此外，它还可以通过第三方平台与其他应用程序或服务进行集成。

**流程自动化（RPA）**：一种在计算机控制下运行的软件程序，用于编制、优化、执行或监控业务过程和流程。其目的是让计算机代替人类执行繁琐且易出错的重复性、机械性、乏味的手动过程，从而达到提升效率、降低成本、改善客户满意度的目的。由第三方供应商提供的RPA工具软件可以满足不同行业的需求。

### 概念联系
**Chatbot和RPA并不是孤立的两个技术，它们之间有以下一些相似之处：**

1. 用途相同——为了实现业务流程的自动化处理；

2. 技术栈相同——采用机器学习、深度学习、文本分析、图像识别、语音识别等技术。

3. 操作对象相同——不管是内部的还是外部的客服人员、业务顾问、员工都是可以实现RPA与Chatbot功能的。

4. 部署环境相同——Chatbot可以通过Web和移动端的方式进行部署，RPA通常安装在服务器上。

5. 数据源相同——两种技术都依赖于大数据来进行训练，它们所需要的数据源基本一致。

**除了上面这些相似之处之外，还有一些重要的联系要明确：**

1. 侧重点不同——Chatbot主要是为了回答简单的问题，而RPA可以实现复杂的业务流程自动化管理；

2. 数据驱动策略不同——Chatbot通过一定规则来进行自学习，而RPA则是基于经验丰富的业务知识库来进行领域知识的理解、抽取、检索。

3. 发展方向不同——Chatbot目前更多的聚焦在客户服务类的产品中，而RPA越来越多的应用在金融、零售、采购、物流、生产制造等领域。

4. 用户群体不同——由于Chatbot的应用场景广泛，但受限于机器学习算法的限制，它只能处理非常简单的数据输入，无法处理复杂的数据；而RPA具有更强的数据驱动能力，可以使用更加复杂的数据进行训练。

**总结：Chatbot和RPA之间主要的区别在于侧重点不同，RPA更注重对复杂的业务流程自动化管理，可以处理更加复杂的数据，可以应用在多个领域中。Chatbot主要用于回答简单的问题，比如在线咨询、即时反馈等。</strong>
