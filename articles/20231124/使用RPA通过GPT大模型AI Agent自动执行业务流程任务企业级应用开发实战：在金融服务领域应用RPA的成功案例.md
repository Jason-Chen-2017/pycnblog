                 

# 1.背景介绍


## 概述
### GPT-3是什么？
GPT-3是一个开源的自然语言生成AI模型，由OpenAI团队于2020年11月3日开源发布。它的核心是一个基于transformer（BERT）的神经网络模型，其结构与BERT、GPT一样采用预训练语言模型的方式进行预训练。GPT-3的目标是成为一个可以生成连续自然语言文本的AI模型。而这一功能对于金融行业来说非常重要。

### 为什么要用RPA代替人工处理？
GPT-3这个模型生成的语言连贯性高，而且还能根据语境推断出用户的意图，这种语言理解能力是人工无法比拟的。所以在面对复杂且多变的金融业务流程时，传统的人工审批方式显得力不从心。基于此，OpenAI将提出一种新型的解决方案——由机器人自动执行金融业务流程。

人工审批存在以下缺点：

1. 需要人工审核整个业务流程，效率低下；

2. 审批过程不透明，容易出现遗漏或错误，影响效率；

3. 对财务、IT等岗位人员要求较高，管理成本较高。

而RPA则能够以系统化、自动的方式完成审批过程，不仅能提升效率，也降低管理成本。

### RPA如何实现自动执行业务流程？
RPA(Robotic Process Automation)即“机器人流程自动化”，它是指利用计算机软件，通过设定工作流自动完成重复性任务，并把这些任务作为自动化的任务交给机器人来运行。在实际运用中，机器人只需要跟随脚本走完指定的业务流程即可，不需要人工参与。通过RPA自动化业务流程的主要流程如下：

2. 模板识别：用于对传入的数据进行初步分析，判断其是否符合某个业务模板；

3. 数据解析：用于对已识别的模板数据进行解析，得到需要输入的数据；

4. 数据转换：将数据转化为适合输入RPA引擎中的格式，并向该引擎提供数据；

5. 执行引擎：根据RPA引擎所设置的流程，完成相关的业务流程；

6. 数据输出：将完成的业务流程结果输出到外部系统，供外部系统查看或者存储；

7. 服务反馈：根据机器人执行结果，向服务调用者返回处理结果。

总结来说，RPA可以通过预先设定好的业务流程，并将相应的数据输入进去，机器人自动执行业务流程，有效地节省人工成本，提升公司的工作效率。

### GPT-3和RPA是如何协同工作的？
GPT-3的模型能够生成连贯的语言描述，这种语言理解能力可以帮助RPA更准确地识别业务信息。例如，当业务部门将客户的存款申请单发送给银行的时候，GPT-3能够将这封信的内容翻译成流畅的语言，如"请确认您的存款金额是否正确？您确定要提取您的存款吗？"。RPA可以使用GPT-3的语言理解能力识别客户存款申请单上所填的信息，并作出相应的处理，如发送确认函给客户。
