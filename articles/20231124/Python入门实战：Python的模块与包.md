# 2.核心概念与联系
## 2.1 模块与包
### 什么是模块？
模块就是独立的代码文件，可以被其他程序或模块导入使用。每个模块都有一个自己独立的作用域，包含必要的变量和函数定义。模块之间可以互相引用，以实现功能的复用和可靠性。模块通常以.py结尾。

### 什么是包？
包是用来组织模块的一种方式，它类似于文件夹，里面可以存放多个模块，但是要注意的是，包也是一个模块。当我们导入一个包时，其实他就是一次性引入了该包下面的所有模块。

### 包的作用？
包的主要作用有以下几个：

1. 防止命名冲突：不同包可以设置不同的命名空间，避免不同模块的名称相同造成冲突。

2. 提高项目结构清晰度：通过划分包，可以更好的实现项目的结构和层次化管理。

3. 方便版本控制和共享：包可以打包成发布版本，并且可以方便的分享给他人。

4. 有利于重用代码：包可以集成不同模块的功能，提高代码的重用率。

5. 便于单元测试：可以通过包来完成单元测试。
