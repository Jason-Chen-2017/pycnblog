# 1.背景介绍


Python作为一种非常流行、方便快捷的脚本语言，它也拥有着众多优秀的第三方库、框架和工具，能够轻松解决很多问题。但是编写好的代码并不是完美无缺的，往往还需要考虑到代码的可重复使用性（Reusability），也就是说，通过一个函数或类实现了某功能的代码可以被其他地方调用执行。

当然，在实际工作中，我们是无法做到每个项目都可以复用代码的。代码的可重复利用率一般取决于以下三个因素：

1. 函数/类接口的一致性：保证输入输出参数类型、顺序等保持一致
2. 可读性：代码易于理解和维护
3. 扩展性：容易新增新功能

所以，如果要提升代码的可重复利用率，就需要我们自己动手去设计良好、可扩展性强的函数和模块。

# 2.核心概念与联系
## 2.1 什么是模块？
模块是组织代码的文件。每一个Python文件都是一个模块，包含了一个或者多个函数、变量、类、方法等。模块就是不同编程语言中的库（library）、包（package）或者module，它定义了一组相关的函数、变量、数据结构等。
```python
import math # import语句用于导入模块math
print(math.sqrt(9)) # 使用函数sqrt计算平方根9
```
## 2.2 为何要引入模块？
如果你写过一些代码，你会发现它们经常会出现重复代码。举个例子，比如两个程序都需要处理网络连接，其中必然存在相同的套接字创建、发送数据、关闭套接字的过程。那么，我们就可以将这些重复的过程封装成一个模块，然后再调用这个模块即可。这样做不仅能减少代码量，而且可以提高代码的可读性和可复用性。

## 2.3 模块的分类
根据模块被引用的方式，Python分为内置模块、第三方模块、自定义模块三种：

1. 内置模块（Built-in Modules）：由Python安装时自带的模块，直接导入即可使用，如`math`、`random`等；
2. 第三方模块（Third-party Modules）：由第三方提供的模块，安装后才能使用，如`requests`、`beautifulsoup`等；
3. 自定义模块（Custom Modules）：用户自己编写的模块，可以保存自己的代码和功能，以便复用。

在编写代码的时候，尽量不要直接调用内置模块和第三方模块的具体函数或方法，而应该通过模块名访问模块提供的接口。

## 2.4 模块的作用域
模块的作用域决定了模块中的变量、函数、类是否可以在当前脚本或模块之外进行访问。

### 全局作用域
全局作用域指的是模块顶层范围，在模块的整个生命周期内都有效。模块中的变量、函数、类等定义在全局作用域内，可以直接被其他模块中的函数、语句所引用。全局作用域只能被加载一次，因此可以降低内存占用。

### 函数作用域
函数作用域指的是在函数内部定义的变量，只能在函数内访问。除此之外，函数外部也可以访问，但访问到的都是该作用域的同名变量。

### 局部作用域
局部作用域指的是在函数或其他结构中定义的变量，只在其作用范围内有效。当作用范围结束时，变量也随之销毁。局部作用域只能被调用一次，因此适合临时使用。
