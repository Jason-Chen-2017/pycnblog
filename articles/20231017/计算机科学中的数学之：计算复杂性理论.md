# 1.背景介绍


## 1.1 计算复杂性
计算复杂性的定义是指一个问题所需的资源数量或时间，随着输入数据量的增加而增长的一种度量。对于计算机来说，输入数据的大小和复杂度决定了运行时间，所以计算复杂性也是衡量计算机性能的一个重要标准。通常将计算复杂性分为三个层次：P、NP、NP-hard等。
### 1.1.1 P类问题
又称“确定性问题”，指的是可以在多项式时间内解决的问题。这一类问题中的任何一个问题都可以被解决，且能在多项式时间内得到最优解。多项式时间复杂度指算法的运行时间为输入规模的n的k次幂。举个例子，一个经典的问题就是求两个随机序列表的第一个公共元素。其运行时间为O(n)，然而这个问题实际上难于在多项式时间内解决。因此，P类问题的应用范围十分广泛。
### 1.1.2 NP类问题
NP类问题的全称是Nondeterministic Polynomial Time Class (non-deterministic polynomial time class)。它是指可以在多项式时间内求解但不一定总能找到最优解的一些问题。在NP类中，大部分问题都是难于在多项式时间内解决的，但是其中很多问题的某种近似算法却具有着很高的多项式时间复杂度。因此，NP类问题对很多应用领域如图灵完备语言、计划算法等都是至关重要的。
### 1.1.3 NP-hard类问题
NP-hard问题又称“NP完全问题”，是指存在多项式时间内的算法来验证某个给定的问题是否属于NP类或者是是否可以由多项式时间内的算法来解决。该类问题是NP类问题的子集。因此，NP-hard问题也具有着与P类和NP类同样的重要地位。
### 1.2 计算复杂性理论
计算复杂性理论研究计算机算法、计算过程以及系统设计者在设计和分析复杂系统时会遇到的一些基本问题。主要包括：渐进时间复杂度（Asymptotic Time Complexity）、渐进空间复杂度（Asymptotic Space Complexity）、算术算法（Arithmetic Algorithm）、主定理（Master Theorem）等。下面介绍一下这几个概念的具体含义。
#### 渐进时间复杂度
渐进时间复杂度简称作TC，是用来描述一个算法运行时间与问题规模之间的关系。用TC来刻画算法运行时间的增长率，并在不同的数据规模下比较算法效率，从而选择具有最佳效率的算法。一般情况下，算法的时间复杂度分为两个部分，一个是单调递增的常数因子，另一个是问题规模n的多项式函数。所谓“渐进”指的是时间复杂度随着问题规模n的增加而增长的趋势，即用一个阶的多项式来近似表示算法的时间复杂度。例如，当n变得越来越大时，算法的运行时间就越来越长，称这种增长趋势为渐进时间复杂度。
#### 渐进空间复杂度
渐进空间复杂度简称作SC，是用来描述算法使用的空间与问题规模之间的关系。用SC来刻画算法内存占用的增长率，并在不同的问题规模下比较算法效率，从而选择具有最小空间需求的算法。一般情况下，算法的空间复杂度分为两个部分，一个是单调递增的常数因子，另一个是问题规模n的多项式函数。所谓“渐进”指的是空间复杂度随着问题规模n的增加而增长的趋势，即用一个阶的多项式来近似表示算法的空间复杂度。例如，当n变得越来越大时，算法的内存占用就会越来越大，称这种增长趋势为渐进空间复杂度。
#### 算术算法
算术算法是指能够在多项式时间内进行加减乘除运算，一般用于计算哈希值、字符串匹配、矩阵乘法等常见问题的算法。
#### 主定理
主定理是指多项式时间算法的最坏情况与平均情况之间的差距不能超过一个常数。该定理是指对于一个具有多项式时间算法的问题，通过修改问题使得其变成简单问题，所得到的算法的运行时间要么严格小于最坏情况，要么严格大于平均情况。这就要求算法的设计者注意如何修改问题来降低复杂度，从而获得一个运行时间在某些问题上的最坏情况。
