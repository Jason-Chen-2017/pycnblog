# 1.背景介绍


作为一名Java开发人员，在日常工作中我们需要处理各种各样的文件。比如读取、写入、搜索、复制、移动等等，这些功能都离不开IO流和文件的操作。所以本文将探讨Java中文件读写及其相关的知识。如果你是一个技术专家或对计算机底层有非常高级的研究，并希望有所作为，那么可以尝试阅读本文。
# 2.核心概念与联系
首先，我们必须了解以下的一些重要的概念和联系。如下图所示：

1.InputStream/OutputStream：这是两个抽象类，它们提供了读取字节流和写入字节流的方法。 ByteArrayInputStream用于从字节数组中创建输入流； ByteArrayOutputStream用于向字节数组中输出数据。 DataInputStream和DataOutputStream分别提供以“标准”类型化方式读取和写入基本数据类型的能力。如同其他的抽象类一样，FileOutputStream继承了OutputStream。

2.Reader/Writer：这个两个类与前面的InputStream/OutputStream类似。它们主要用来处理字符流（字符序列）而不是字节流。 InputStreamReader用于将字节流转换成字符流，BufferedReader则是BufferedReader类，它通过读取一个字符数组缓冲区来提升性能。 PrintWriter用于写入字符到PrintWriter对象，该对象可以通过PrintWriter.println()方法向输出流写出字符。

3.File：File类代表了一个文件或者目录。它提供了很多属性，比如文件名称、路径、大小、最后修改时间等。

4.FileInputStream/ FileOutputStream：它们分别继承自InputStream 和 OutputStream 。 FileInputStream 可以读取一个已存在的二进制文件中的字节，而 FileOutputStream 可以向文件中写入新的字节。

5.FileReader/FileWriter：它们也是BufferedReader/BufferedWriter 的替代品。FileReader 和 FileWriter 能够帮助我们读取和写入文本文件。他们可以读取和写入具有特定编码格式的Unicode文本文件。

6.BufferedReader/BufferedWriter：BufferedReader 是个可以从 BufferedReader 中读取数据的类。可以用 readLine() 方法一次性读取一行数据。BufferedWriter 是个可以把数据写入 BufferedWriter 中的类。也可以用 newLine() 方法把缓冲区的数据刷新到硬盘上。

7.Files：Files 类提供了一些文件和目录操作的便捷方法，如创建新文件、删除文件、检查文件是否存在等。其中，Path 接口提供了操作系统特定的文件路径，它可以与其他文件系统无缝地交互。URI 和 URL 都是描述资源位置的字符串形式。

8.内存映射文件(Memory Mapped Files)：一种比普通文件的访问速度快的访问方式。它允许应用程序直接访问磁盘上的某个文件的内容，而不需要把整个文件加载到内存中。MemoryMappedFile 支持按页访问文件内容，使得随机访问和顺序访问效率更高。
