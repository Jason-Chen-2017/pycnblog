         # 2. 基础知识
         ## 2.1 Istio 是什么？
         Istio 是由 IBM、Google 和 Lyft 联合推出的一款开源服务网格产品。它通过一个全面的功能集合，满足用户从分布式微服务架构向云原生应用架构的迁移和连接方面需求。
         Istio 提供了一种简单的方法来保护、连接和管理微服务，而不需要更改应用程序代码或重新构建、重构网络。通过在整个分布式系统中自动化运用流量控制、监控、日志和其他操作策略，它可以让开发人员轻松地对服务间通信进行管理。
         ## 2.2 服务网格架构
         服务网格（Service Mesh）是用于处理服务间通信的基础设施层。通常，服务网格会作为独立的组件部署在计算集群之上，用来调配流量、控制通信、实施访问策略等。它可以在整个环境中抽象出服务间通信的复杂性，并将其抽象为一套完整的解决方案。下图展示了服务网格架构模型：

         从上图可知，Istio Service Mesh 拥有以下几个特点：

         - **Sidecar代理**：每个 pod 都会注入一个 Sidecar 代理，用来拦截 pod 中的流量并处理请求。
         - **Mixer组件**：Mixer 负责在服务网格范围内执行策略决策和遥测数据收集。
         - **Pilot组件**：Pilot 根据服务注册表中的变化来动态调整路由配置，并为 Envoy sidecars 提供当前集群中可用服务的列表。
         - **Citadel组件**：Citadel 为集群内的所有 pod 和 service 安全地提供证书，并支持基于角色的访问控制（RBAC）。
         - **Tracing**：Istio 可以帮助用户跟踪整个分布式系统中的延迟和流量信息。
         通过以上特点，可以看出，Istio Service Mesh 具有以下优点：

         - **透明性**：无需更改应用程序代码，即可利用服务网格提供的功能。
         - **可观察性**：利用 Istio Mixer 可以获取服务网格中的各种遥测数据，如调用次数、响应时间、故障率等。
         - **安全性**：Istio Citadel 可以保证服务间通信的安全性，包括加密、认证和授权。
         - **可扩展性**：由于 Istio 使用 sidecar 代理的方式实现流量管控，所以可以很好地扩展服务网格。
         # 3. Istio 中文社区的成长情况
         ## 3.1 组织定位及目标
        在 2019 年 7 月 2 日，Istio 中文社区（https://istio.io/zh）正式开通。社区由一群热爱开源，关注 Service Mesh 技术，有着丰富经验的技术人员组成，欢迎有兴趣的同学加入社区一起讨论和分享！社区不断壮大，目前已成为众多公司、组织、个人共同推动、参与和支持 Service Mesh 技术的一站式平台。
        
        社区目前的组织结构如下图所示:

        “Istio 中文社区”有着丰富的协作能力，主要工作内容包括：
        1. 翻译计划与管理：社区每周五召开的 Community Meeting 会议就是翻译计划与管理的会议，每一次会议讨论的内容都是社区正在翻译的文档，以及相应的翻译进度；社区成员可以直接向社区提交 PR ，也可以申请导师来指导 PR 的审阅和合并。
        2. 撰写新手文档：社区还撰写了一些适合新手阅读的教程文档，如安装使用、配置流程等，这些教程文档能够帮助新老用户快速上手 Istio 。
        3. 分享演讲材料：社区每两周举办一次技术分享会，邀请业界领袖和技术专家参加，分享他们最新的技术研究和创新。
        4. 线上活动：社区每两周举办一个线下沙龙，聚集技术专家和布道者，分享他们的最新技术经验，吸引更多人参加。

        为了更好的促进社区成员之间的互动，提升社区的整体协作能力，社区决定建立健康、开放、包容的社区文化氛围。社区目前的维护模式是由社区自行选取的技术专家负责，社区每半年会定期举行 Community Sync ，推选一批社区核心成员出席，共同讨论社区各项事宜。同时社区也会定期举办 Meetup，邀请大咖云集，聚焦于特定技术方向，分享新鲜的技术资讯。

        最后，社区的目标是通过技术共享、知识传播，帮助更多的人掌握 Service Mesh 技术。我们的愿景是成为国内最大的 Service Mesh 技术社区，为 Service Mesh 技术的普及和推广做出积极贡献。
     
     ## 3.2 参与贡献方式
    如果您对 Service Mesh 有兴趣，想为社区做出自己的贡献，有很多途径可以参与贡献，具体如下：
    
    1. 演讲分享：社区每两周举办一次技术分享会，邀请业界领袖和技术专家参加，分享他们最新的技术研究和创新。参与分享，您可以获得与行业专家交流的机会，促进您的职业发展。
    2. 代码贡献：我们非常欢迎社区成员参与到代码贡献中，帮助我们完善现有的文档和示例，使得 Istio 更加易用和可靠。代码贡献不仅可以帮助我们熟悉项目，还可以结识志同道合的伙伴。
    3. 组织活动：您可以主动参与社区的活动组织，例如分享主题沙龙，举办线下活动，以及组织专场研讨会。这样既可以丰富自己，也能收获别人的参与。
    4. 浏览文档和示例：您可以浏览社区提供的文档和示例，充分理解 Istio 相关的知识。并不是所有的知识都可以记录成文档，但任何知识都可以通过文档的形式传播出来，为后续学习和实践提供参考。
    
    除此之外，社区还有几个比较重要的知识共享渠道：
    
    1. GitHub Issue：我们鼓励大家通过 GitHub Issue 跟踪社区的问题反馈、Bug 报告等，帮助社区解决问题。
    2. Wechat Group：加入社区微信群，参与技术讨论，分享您的学习心得，帮助更多的人掌握 Service Mesh 技术。
    3. Slack Channel：每周五晚上，社区都会在 ServiceMesher slack channel 上举行技术沙龙，邀请业界专家和布道者，分享最新的技术资讯。
    
    最后，感谢您花费宝贵的时间参与 Istio 中文社区的建设，我们相信您的参与对 Istio 社区的发展是十分重要的。
