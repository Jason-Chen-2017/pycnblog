# 2. 前言
## 2.1 Goland是什么
Goland是一个基于IntelliJ IDEA平台构建的Go语言专用的IDE。它整合了许多强大的工具，如代码完成、语法检查、运行/调试环境、版本控制系统(VCS)集成、单元测试、静态分析、动态分析等，帮助开发者更高效地编写Go代码。除了可视化编辑之外，还提供强大的命令行工具和API接口，让开发者能够轻松扩展或定制其工具箱。在性能方面，Goland提供了各种实时监控和分析工具，让你从细微处察觉到全局。
## 2.2 Go调试器有哪几种？分别有什么特点？
- Delve: 官方发布的Go语言调试器，使用Delve，你可以轻松跟踪你的代码的执行过程、调试你的Go程序。
- gdb: 是Unix和类Unix操作系统上的命令行调试工具。它支持断点设置、单步执行、表达式计算、数据查看和打印，让调试Go程序变得十分简单。
- lldb: Apple公司推出的Go语言调试器，功能类似于gdb，适用于MacOS和iOS平台。
- gobuild: 支持智能编译及可视化显示，简单方便。
- Visual Studio Code with the Go extension: 支持代码补全、语法检查和错误标记。

当然还有很多其他的调试器，如GDBTM、GoLand内置的调试器、GoLand的插件等，这里就不一一列举了。

## 2.3 为什么要在Goland下调试Go项目？
Goland支持全栈开发，包括代码编辑、编译运行、调试、版本管理、集成测试、静态分析、性能分析等。Goland的功能非常强大，但是由于其代码编辑器基于IntelliJ IDEA，因此它的调试功能也依赖于这个平台。在Go语言社区中，有一种说法认为Goland比Visual Studio Code要快得多，并且提供最佳的IDE体验。这是一件好事，因为Visual Studio Code作为一款跨平台代码编辑器，调试功能弱且不直观，使用起来比较麻烦。而Goland是JetBrains旗下的一款IDE，它的设计思想和Microsoft VS系列产品不同，使用起来更方便。
