# 1.简介
         
　　Docker是一个基于Go语言开发的开源容器引擎，其提供简单易用的容器技术。可从以下几个方面进行了解释：
         1、Docker是什么？
        Docker是一个开源的应用容器引擎，让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的Linux或Windows机器上，也可以实现虚拟化。Docker对应用程序的构建及运行过程进行封装，从而使得应用在部署时环境一致性大幅度提升。
        2、为什么要用Docker？
        1）资源隔离能力强。由于Docker利用Linux内核的cgroup和namespace技术实现了资源的隔离，使得容器间互相隔离，并具有较高的资源互相独立的能力。
        2）弹性伸缩能力强。Docker通过轻量级虚拟机（VM）镜像机制实现了动态扩容，可以快速部署服务，并保证运行性能。
        3）部署方便快捷。无需复杂配置，只需要Dockerfile文件就可以轻松定义自己的镜像，并可以把镜像发布到公共或私有仓库。
        4）持续交付能力强。基于Dockerfile的快速迭代能力，可以帮助开发团队迅速反馈产品需求，并及时修复bugs，提升用户体验。
        5）版本控制能力强。基于Docker镜像的版本管理，可以追踪每个组件的变更历史，并且可以在任何时候回滚到之前的状态。
        6）兼容性好。Docker支持各种Linux发行版，包括Ubuntu、CentOS、Debian等，并可以很容易地移植到其他平台。
         2、Docker容器技术概述
         　　　Docker的主要特征如下：
        1、容器是被彻底隔离的进程集合。
        2、容器之间相互独立，不能看到彼此的内核空间，也无法直接访问彼此的敏感信息。
        3、资源分配极其精确，容器的cpu、内存、网络等资源都是限定过的，严格限制资源使用，防止恶意使用。
        4、启动速度快，因为不必启动整台操作系统，只需启动容器即可。
         3、Docker 的分层存储
         　　　Docker 使用一种叫做 联合挂载（union mounts）的技术来实现分层存储。这种技术允许镜像中的每一层都是只读的，当容器运行时，实际上是在一个堆叠的视图中可读写的文件系统。这些堆叠视图之所以能够工作，就是因为底层的文件系统只能读写其自身的内容，而顶层的文件系统只能看到属于自己权限范围的目录。
         　　这就意味着，如果同一个镜像的两个不同容器都共享了一层文件系统，那么对其中一个容器修改会影响另一个容器，因此不同的容器之间应该是完全独立的。
         4、总结
         　　　Docker 提供了统一化部署环境，可以有效降低生产环境的成本，同时实现应用的可移植、可维护性及资源节约。相对于传统的虚拟机技术，使用 Docker 可以极大地提升效率和灵活性。
         　　　在云计算时代，容器技术又扮演着重要角色，它极大的加剧了开发和运维之间的协作，同时让云计算得以普及。希望本文对你理解Docker的相关知识点有所帮助，期待更多的干货文章~
