# 1.背景介绍


## 数据结构概述
数据结构是计算机科学中存储、组织、管理数据的抽象化表示法，是指相互之间存在一种或多种关系的数据元素的集合。数据结构可以分为以下几类:

1. 逻辑结构——对数据之间的关系进行分类和定义，将数据划分为不同的结构或模式，如树状结构、图状结构、线性结构等；
2. 物理结构——按照硬件特性和存储方式实现数据结构，如顺序表、链表、栈、队列、哈希表、B-树等；
3. 操作方法——对数据结构进行各种操作，如增删查改、排序、查找、遍历、转换等。

除了上述三大类，还有两种特殊的数据结构，即并行结构（Parallel）和分布式结构（Distributed）。

## 数组与链表
数组和链表都是非常经典的数据结构。下面就来对比一下它们的特点、应用场景及优劣。

### 数组
数组（Array），也称为一维数组或者数列，是由同一类型元素组成的按序排列的集合。它具有以下几个特征：

1. 用索引（Index）访问元素，时间复杂度为 O(1)。
2. 一次性分配内存空间，数组大小固定。
3. 能够动态扩容，扩容时需要重新分配内存空间。

一般情况下，数组的大小是在初始化的时候确定好的，后续如果需要改变数组大小，只能创建一个新的数组，然后把旧数组里的数据逐个复制到新数组里。这样的代价比较高昂。

### 链表
链表（Linked List），也称单链表，是一种线性数据结构，用节点的方式存储数据，每个节点都有一个指向下一个节点的引用。其特点如下：

1. 插入和删除操作不需要移动其他元素，只需要修改指针即可。
2. 可以灵活调整内存大小，不会像数组一样受限于固定的大小。

但是链表由于要保存指针信息，占用额外的存储空间，所以对于空间要求较高的地方不太适合。另外，链表的插入和删除操作可能导致链表节点间的连接断裂，失去连贯性。

### 数组和链表的区别
从上面两者的特点可以看出，数组和链表在不同情景下的应用场景和优缺点各有不同。下面总结一下两者的共同点：

1. 随机访问：数组支持通过索引快速找到对应元素，时间复杂度为 O(1)，而链表需要通过指针（next）逐个遍历找到对应元素，时间复杂度为 O(n)。
2. 插入和删除：数组因为预先分配了内存空间，因此插入和删除效率比较高，但不能动态扩容；链表可以在运行时随意调整内存大小，并且支持动态插入和删除，相对来说稍微慢一些。
3. 分配空间：数组需要一次性分配所有空间，且大小固定，无法动态扩展；链表可根据实际情况动态分配内存空间，也可采用预留机制预先分配一定量空间，提升性能。

数组和链表都是用于处理集合数据的结构，其本质都是用来存储、组织、管理数据的。但是由于其各自的特点和应用场景的差异，他们分别被广泛地运用于不同的领域。比如，图论中的邻接矩阵通常用二维数组表示，用邻接表表示更为适宜，文件系统中的目录结构用树形结构表示，数据库中的索引用散列表表示更为有效等。
