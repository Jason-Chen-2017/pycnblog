# 1.背景介绍


树（Tree）是一种抽象数据类型，用来模拟具有层次关系的数据集合。它由n个节点组成，并定义了各节点之间的链接关系，称为边（edge）。在任意两个不同的节点间都可能有一条路径相连。树有三种基本形态：根、内部节点和外部节点。在最简单的树中，只有一个根节点。如果某个节点没有子节点，则该节点称为叶子节点或终端节点；如果某个节点有且仅有一个子节点，则该节点称为父节点；如果某个节点有两棵子树，则该节点称为中间节点。树结构通常用来描述部分或者整体层次关系的数据，例如，文件目录结构、家庭关系等。树的另一种重要性质是可以方便地实现多种动态集合操作，如遍历、查找、插入、删除等。
二叉树（Binary Tree）是树的一种特殊形式。二叉树是每个节点最多有两个孩子树的树结构。在任意节点处，最多有两颗子树，分别为左子树和右子树。而一般的树中允许出现子树个数不唯一的情况，因此不能完全等同于二叉树。二叉树有两种基本形态：满二叉树和完美二叉树。满二叉树所有分支节点均填满，即每一个节点都有左右子树，并且所有的叶子都在同一层上。完美二叉树是一种高度平衡的二叉树，即任一节点到叶子节点的路径长度相同，而且左右子树也分别为二叉树。

# 2.核心概念与联系
## 2.1 树的术语和定义
### 2.1.1 树的术语和定义
树的术语有7个，分别是：结点（node）、顶部（root）、度（degree）、边（edge）、路径（path）、子树（subtree）和树的度序列（degree sequence）。

树的定义是：“树是由n(n>0)个结点或者无限个结点的集合T，它满足以下条件：

1. 每个结点有零个或多个子女。

2. 没有回路，即从根到某一结点的任何路径上经过的结点集恰好包含这个结点本身，也就是说不存在循环的边。

3. 没有多个回合的边，即不能出现两个结点之间的边。

其中，结点表示树中一个元素，边表示连接两个结点的线。树的根表示树的中心，没有父结点的结点叫做根，根有子女叫做根的子树。度是指一个结点拥有的子树个数。

### 2.1.2 有序树
在树结构中，当对树中每一结点按照大小顺序建立有序的索引时，这种树就叫做有序树。比如：按数字大小或字母大小排列的文件夹、磁盘分区表等都是有序树；按照广度优先或深度优先的次序建立的树，就是无序树。

### 2.1.3 森林（Forest）
森林是一个或多个有根树组成的集合，它们彼此独立，互不干扰。森林的术语是"forest"，其中“fore”是“林”，而“st”是“树”。它是由n棵互不相交的树构成的，每个树可能有根结点、0个、1个、2个、...个子树。在森林中，每个树的根结点不会出现在其他树的内部结点。

森林被广泛应用于分类、聚类、模式识别、图像处理、生物信息学、信号处理、数据挖掘等领域。

## 2.2 二叉树
二叉树又称为树状数组（trie），是一个重要的数据结构。二叉树是一种多叉树，它的每个节点最多有两个孩子，分别为左孩子和右孩子。若左子树不空，则左子树上所有结点的值小于根结点的值；若右子树不空，则右子树上所有结点的值大于根结点的值。二叉树中除了根结点外，每个结点都有左右两个子树，并且左右子树也都是二叉树。

### 2.2.1 二叉搜索树（BST）
二叉搜索树（BST）是指一棵空树或者具有下列性质的二叉树：

1. 若任意节点的左子树不空，则左子树上所有结点的值均小于它的根结点的值；

2. 若任意节点的右子树不空，则右子树上所有结点的值均大于它的根结点的值；

3. 任意节点的左、右子树也分别为二叉查找树。

这种数据结构被广泛用于各种算法和数据处理，如排序、检索、范围查询、插入和删除操作等。

### 2.2.2 满二叉树（Full Binary Tree）
满二叉树（full binary tree）是指除最后一层外，其他层的节点都达到最大数量，并且最下层的叶子节点都集中在最左边。如下所示：

```
    1
   / \
  2   3
 / \   \
4   5   6
       / \
      7   8
```

### 2.2.3 完全二叉树（Complete Binary Tree）
完全二叉树（complete binary tree）是指除了最底层外，其他层的节点都达到最大数量，最底层不完全。如下所示：

```
     1
    / \
   2   3
  /     \
 4       5
        / \
       6   7
          /
         8
```
