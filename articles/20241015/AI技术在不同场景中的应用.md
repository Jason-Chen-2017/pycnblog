### 第一部分：AI技术基础

#### 第1章：AI技术概述

##### 1.1 AI技术的基本概念

人工智能（Artificial Intelligence，简称AI）是指由人制造出来的具有一定智能的系统，能够模拟、延伸和扩展人类智能，实现感知、理解、学习和决策等功能。AI技术包括机器学习、深度学习、自然语言处理、计算机视觉等多个子领域。

**人工智能的类型**

- **弱人工智能（Narrow AI）**：专注于特定任务，如语音识别、图像分类等。
- **强人工智能（General AI）**：具备人类所有智能特征，能够进行复杂决策和问题解决。
- **混合智能（Hybrid AI）**：结合多种智能技术，如将自然语言处理与计算机视觉结合。

##### 1.2 AI技术的发展历程

- **萌芽时期（1950s-1960s）**：人工智能的概念首次被提出，早期研究主要集中在逻辑推理和知识表示。
- **黄金时代（1980s-1990s）**：专家系统和规则引擎得到广泛应用，但性能有限。
- **复兴时期（2000s至今）**：机器学习和深度学习的崛起，推动了AI技术的快速发展。

##### 1.3 AI技术的应用场景概述

- **工业自动化**：提高生产效率，降低人力成本。
- **金融服务**：风控、量化交易、智能投顾等。
- **医疗健康**：疾病诊断、药物研发、健康监测等。
- **制造业**：智能制造、智能物流、预测性维护等。
- **零售行业**：智能推荐、智能客服、供应链优化等。
- **交通运输**：自动驾驶、智能交通管理、无人机配送等。

### 第一部分：AI技术基础

#### 第2章：机器学习基础

##### 2.1 机器学习的基本原理

机器学习（Machine Learning，ML）是AI的核心技术之一，通过算法让计算机从数据中学习规律，进而进行预测或决策。机器学习主要包括以下三个要素：

- **数据（Data）**：作为训练样本，用于模型学习。
- **模型（Model）**：对数据进行分析和预测的算法结构。
- **优化（Optimization）**：通过调整模型参数，提高模型性能。

##### 2.2 监督学习算法

监督学习（Supervised Learning）是最常见的机器学习任务，其核心是利用标记数据进行训练，从而预测未知数据。

###### 2.2.1 线性回归

线性回归（Linear Regression）是最简单的监督学习算法，用于预测连续值。其基本原理是通过拟合一条直线，来描述因变量和自变量之间的关系。

**线性回归模型：**

$$
y = \beta_0 + \beta_1x_1 + \beta_2x_2 + ... + \beta_nx_n
$$

其中，$y$ 是因变量，$x_1, x_2, ..., x_n$ 是自变量，$\beta_0, \beta_1, \beta_2, ..., \beta_n$ 是模型参数。

**线性回归算法流程：**

1. 数据预处理：对数据进行归一化或标准化处理。
2. 模型初始化：随机初始化模型参数。
3. 训练过程：通过梯度下降等优化算法，不断调整模型参数，使损失函数最小化。
4. 模型评估：使用测试数据集评估模型性能。

###### 2.2.2 决策树

决策树（Decision Tree）是一种基于树形结构的分类算法，通过一系列规则进行决策，将数据集划分为不同的类别。

**决策树模型：**

决策树由一系列判断节点和叶子节点组成。判断节点表示对特征的测试，叶子节点表示最终的分类结果。

**决策树算法流程：**

1. 数据预处理：对数据进行编码或预处理。
2. 特征选择：选择具有区分性的特征。
3. 构建决策树：通过递归方式，将数据集划分为子集，并选择最优特征进行划分。
4. 模型评估：使用测试数据集评估模型性能。

**决策树优缺点：**

- **优点**：直观、易于理解，能够处理非线性数据。
- **缺点**：容易过拟合，对异常值敏感。

###### 2.2.3 支持向量机

支持向量机（Support Vector Machine，SVM）是一种线性分类算法，通过找到一个最优的超平面，将不同类别的数据分开。

**支持向量机模型：**

$$
w \cdot x + b = 0
$$

其中，$w$ 是权重向量，$x$ 是特征向量，$b$ 是偏置项。

**支持向量机算法流程：**

1. 数据预处理：对数据进行归一化或标准化处理。
2. 特征选择：选择具有区分性的特征。
3. 模型训练：通过求解最优化问题，找到最优的权重向量。
4. 模型评估：使用测试数据集评估模型性能。

**支持向量机优缺点：**

- **优点**：具有很好的泛化能力，能够处理高维数据。
- **缺点**：对异常值敏感，计算复杂度较高。

##### 2.3 无监督学习算法

无监督学习（Unsupervised Learning）不需要标记数据，其目标是从数据中发现潜在的规律和结构。

###### 2.3.1 K均值聚类

K均值聚类（K-Means Clustering）是一种基于距离度量的聚类算法，通过迭代优化，将数据划分为K个簇。

**K均值聚类模型：**

1. 初始化：随机选择K个初始中心点。
2. 分配：计算每个数据点到中心的距离，将其分配到最近的簇。
3. 重新计算：重新计算每个簇的中心点。
4. 重复步骤2和3，直至中心点不再发生显著变化。

**K均值聚类优缺点：**

- **优点**：简单、易于实现，能够处理大型数据集。
- **缺点**：对初始中心点敏感，可能陷入局部最优。

###### 2.3.2 主成分分析

主成分分析（Principal Component Analysis，PCA）是一种降维技术，通过正交变换，将原始数据转换到新的坐标系中，保留主要信息，减少冗余。

**PCA模型：**

1. 数据预处理：对数据进行中心化和标准化处理。
2. 特征提取：计算数据的相关矩阵，求解特征值和特征向量。
3. 降维：选择前几个特征值对应的特征向量，构成新的特征空间。
4. 数据转换：将原始数据映射到新的特征空间。

**PCA优缺点：**

- **优点**：有效降维，提高数据处理效率。
- **缺点**：对异常值敏感，可能丢失部分信息。

### 第一部分：AI技术基础

#### 第3章：深度学习入门

##### 3.1 深度学习的核心概念

深度学习（Deep Learning，DL）是机器学习的一个分支，通过构建深层次的神经网络模型，对数据进行建模和分析。深度学习的核心概念包括：

- **神经网络（Neural Network）**：模仿人脑神经元连接方式的计算模型。
- **深度（Depth）**：神经网络层数，深度越大，模型表达能力越强。
- **非线性变换（Non-linear Transformation）**：通过激活函数实现，增加模型非线性能力。

##### 3.2 神经网络基础

神经网络（Neural Network，NN）是由大量神经元连接而成的计算模型，用于模拟人脑神经元的工作方式。神经网络的基础概念包括：

- **神经元（Neuron）**：神经网络的基本单元，负责接收输入、进行计算并产生输出。
- **权重（Weight）**：神经元之间的连接强度，用于传递信息。
- **激活函数（Activation Function）**：对神经元输出进行非线性变换，常用的激活函数包括ReLU、Sigmoid、Tanh等。

**神经网络模型：**

$$
\text{output} = f(\sum_{i=1}^{n} w_ix_i + b)
$$

其中，$f$ 是激活函数，$w_i$ 是权重，$x_i$ 是输入，$b$ 是偏置项。

##### 3.3 深度学习架构

深度学习架构主要包括卷积神经网络（CNN）、循环神经网络（RNN）、生成对抗网络（GAN）等。

###### 3.3.1 卷积神经网络（CNN）

卷积神经网络（Convolutional Neural Network，CNN）是一种专门用于处理图像数据的神经网络，通过卷积层提取图像特征。

**CNN架构：**

1. 输入层：接收图像数据。
2. 卷积层：通过卷积操作提取图像特征。
3. 池化层：降低特征维度，减小模型参数。
4. 全连接层：对提取到的特征进行分类或回归。
5. 输出层：产生预测结果。

**CNN优势：**

- **并行计算**：卷积操作可以并行进行，提高计算效率。
- **参数共享**：卷积核在图像中共享，减少模型参数。

**CNN应用场景：**

- **图像分类**：例如，识别猫、狗等图像。
- **目标检测**：例如，识别图像中的车辆、行人等目标。

###### 3.3.2 循环神经网络（RNN）

循环神经网络（Recurrent Neural Network，RNN）是一种专门用于处理序列数据的神经网络，通过循环连接实现记忆功能。

**RNN架构：**

1. 输入层：接收序列数据。
2. 隐藏层：通过循环连接实现记忆功能。
3. 输出层：产生预测结果。

**RNN优势：**

- **序列建模**：能够处理任意长度的序列数据。
- **记忆功能**：能够记住历史信息，提高预测准确性。

**RNN应用场景：**

- **自然语言处理**：例如，语言模型、机器翻译等。
- **时间序列分析**：例如，股票价格预测、天气预测等。

###### 3.3.3 生成对抗网络（GAN）

生成对抗网络（Generative Adversarial Network，GAN）是由两个神经网络组成的对抗性模型，一个生成器生成数据，一个判别器判断数据真实性。

**GAN架构：**

1. 生成器（Generator）：生成与真实数据相似的数据。
2. 判别器（Discriminator）：判断输入数据是真实数据还是生成数据。
3. 对抗训练：生成器和判别器相互竞争，生成器试图生成更真实的数据，判别器试图更准确地判断数据。

**GAN优势：**

- **数据生成**：能够生成与真实数据相似的数据。
- **自适应优化**：通过对抗训练，生成器能够自适应地优化生成质量。

**GAN应用场景：**

- **图像生成**：例如，生成艺术作品、人脸生成等。
- **数据增强**：例如，在图像分类任务中，生成与训练数据相似的数据，用于训练和测试。

### 第二部分：AI技术在金融领域的应用

#### 第4章：AI技术在金融领域的应用

##### 4.1 金融风控

金融风控（Financial Risk Management）是金融领域中的一项重要任务，旨在识别、评估和控制金融风险，确保金融机构的稳健运营。AI技术在金融风控中发挥着重要作用，包括信用评分、交易预测等方面。

###### 4.1.1 信用评分

信用评分（Credit Scoring）是评估借款人信用风险的重要手段。传统的信用评分模型主要基于借款人的历史数据，如收入、工作经历、信用记录等。而AI技术通过引入机器学习和深度学习算法，可以更加准确地评估借款人的信用风险。

**AI技术在信用评分中的应用：**

1. **特征工程**：通过分析借款人的历史数据，提取与信用风险相关的特征，如还款历史、信用额度、消费习惯等。
2. **模型训练**：使用机器学习算法，如逻辑回归、决策树、随机森林等，对特征进行建模，评估借款人的信用风险。
3. **模型评估**：使用交叉验证等方法，评估模型的准确性和稳定性。

**AI信用评分优势：**

- **自动化**：自动分析借款人的特征，提高评估效率。
- **精准度**：通过学习大量历史数据，提高信用评估的准确性。

**AI信用评分应用场景：**

- **贷款审批**：银行和金融机构使用AI信用评分模型，快速评估借款人的信用风险，决定是否批准贷款。
- **信用卡审批**：信用卡公司通过AI信用评分模型，决定是否批准信用卡申请。

###### 4.1.2 交易预测

交易预测（Transaction Prediction）是金融领域中的一项重要任务，旨在识别和预测金融交易中的欺诈行为。传统的交易预测方法主要基于规则和统计模型，而AI技术通过引入机器学习和深度学习算法，可以更加准确地识别和预测交易欺诈。

**AI技术在交易预测中的应用：**

1. **特征工程**：通过分析交易数据，提取与欺诈行为相关的特征，如交易金额、交易时间、地理位置等。
2. **模型训练**：使用机器学习算法，如逻辑回归、决策树、支持向量机等，对特征进行建模，预测交易是否为欺诈。
3. **模型评估**：使用交叉验证等方法，评估模型的准确性和稳定性。

**AI交易预测优势：**

- **自动化**：自动分析交易特征，提高预测效率。
- **精准度**：通过学习大量历史数据，提高交易预测的准确性。

**AI交易预测应用场景：**

- **反欺诈系统**：银行和金融机构使用AI交易预测模型，实时监控交易行为，识别和预防欺诈行为。
- **信用评分**：将交易预测结果与信用评分模型结合，更全面地评估借款人的信用风险。

##### 4.2 人工智能在量化交易中的应用

量化交易（Quantitative Trading）是利用数学模型和计算机算法进行股票、期货等金融市场的交易策略研究。AI技术在量化交易中发挥着重要作用，包括股票市场预测和期货市场预测等方面。

###### 4.2.1 股票市场预测

股票市场预测（Stock Market Prediction）是量化交易中的一项重要任务，旨在预测股票价格的未来走势。传统的股票市场预测方法主要基于技术分析和基本面分析，而AI技术通过引入机器学习和深度学习算法，可以更加准确地预测股票价格。

**AI技术在股票市场预测中的应用：**

1. **特征工程**：通过分析股票的历史价格、交易量、财务指标等数据，提取与股票价格走势相关的特征。
2. **模型训练**：使用机器学习算法，如线性回归、支持向量机、深度神经网络等，对特征进行建模，预测股票价格。
3. **模型评估**：使用交叉验证等方法，评估模型的预测性能。

**AI股票市场预测优势：**

- **自动化**：自动分析股票特征，提高预测效率。
- **精准度**：通过学习大量历史数据，提高股票价格预测的准确性。

###### 7.1.1 协同过滤

协同过滤（Collaborative Filtering）是智能推荐系统中的一种常见算法，通过分析用户之间的行为模式，预测用户对未知商品的兴趣。

**协同过滤算法流程：**

1. **用户行为数据收集**：收集用户的历史行为数据，如购买记录、浏览记录等。
2. **相似度计算**：计算用户之间的相似度，常用的相似度度量方法包括余弦相似度、皮尔逊相关系数等。
3. **预测推荐**：根据用户之间的相似度，预测用户对未知商品的兴趣，生成推荐列表。

**协同过滤优势：**

- **个性化推荐**：根据用户的历史行为，实现个性化的商品推荐。
- **高效性**：通过计算用户相似度，快速生成推荐列表。

**协同过滤应用场景：**

- **电商推荐**：例如，电商平台通过协同过滤算法，为用户推荐相关商品，提高用户购物体验。

**内容推荐算法流程：**

1. **商品特征提取**：提取商品的属性特征，如类别、品牌、价格等。
2. **用户历史行为分析**：分析用户的历史行为数据，提取用户的兴趣特征。
3. **相似度计算**：计算商品特征与用户兴趣特征的相似度，生成推荐列表。

**内容推荐优势：**

- **相关性高**：根据用户兴趣和商品属性，实现相关性高的商品推荐。
- **适用性广**：适用于各种类型的商品推荐，如电商、视频、新闻等。

**内容推荐应用场景：**

#### 第8章：AI技术在自动驾驶中的应用

##### 8.1 感知系统

感知系统（Perception System）是自动驾驶（Autonomous Driving）的重要组成部分，通过多种传感器获取环境信息，为自动驾驶系统提供数据支持。AI技术在感知系统中发挥着关键作用，包括摄像头感知和激光雷达感知等方面。

###### 8.1.1 摄像头感知

摄像头感知（Camera Perception）是利用摄像头获取环境图像，通过图像处理和计算机视觉技术，实现目标检测、场景理解和行为预测等功能。

**摄像头感知技术：**

1. **图像预处理**：对获取的图像进行滤波、去噪和增强等预处理操作。
2. **目标检测**：通过卷积神经网络（CNN）等深度学习算法，实现车辆、行人、道路等目标的检测和定位。
3. **场景理解**：分析检测到的目标行为，如行人过马路、车辆并道等，预测可能的交通事件。
4. **行为预测**：根据场景理解结果，预测目标的行为轨迹，为自动驾驶系统提供决策依据。

**摄像头感知优势：**

- **广泛适用**：摄像头具有较低的成本和较高的分辨率，适用于多种环境。
- **实时性**：摄像头感知系统可以实现实时数据采集和处理。

**摄像头感知应用场景：**

- **城市自动驾驶**：例如，城市自动驾驶车辆通过摄像头感知系统，实现道路车辆和行人检测，确保行车安全。
- **无人商店**：例如，无人商店通过摄像头感知系统，实时监控顾客行为，提供购物体验。

###### 8.1.2 激光雷达感知

激光雷达感知（Lidar Perception）是利用激光雷达（Lidar）获取环境三维点云数据，通过点云处理和深度学习算法，实现环境感知和目标检测等功能。

**激光雷达感知技术：**

1. **点云采集**：激光雷达发射激光脉冲，测量目标物体的距离和位置，生成三维点云数据。
2. **点云处理**：对采集到的点云数据进行滤波、去噪和分割等处理，提取有用的环境信息。
3. **目标检测**：通过深度学习算法，如卷积神经网络（CNN）等，实现车辆、行人、障碍物等目标的检测和分类。
4. **场景理解**：分析检测到的目标行为，预测可能的交通事件。

**激光雷达感知优势：**

- **高精度**：激光雷达可以获取高精度的三维点云数据，实现高精度的环境感知。
- **广视角**：激光雷达具有较宽的视角范围，可以覆盖更多的环境信息。

**激光雷达感知应用场景：**

- **高速公路自动驾驶**：例如，高速公路自动驾驶车辆通过激光雷达感知系统，实现道路车辆和障碍物的精确检测，确保行车安全。
- **无人机导航**：例如，无人机通过激光雷达感知系统，实现三维地形感知和障碍物避让。

##### 8.2 控制系统

控制系统（Control System）是自动驾驶的核心模块，通过感知系统获取的环境信息，实现车辆的路径规划、轨迹跟踪和动态控制等功能。AI技术在控制系统中发挥着重要作用，包括路径规划、环境感知与决策等方面。

###### 8.2.1 路径规划

路径规划（Path Planning）是自动驾驶控制系统中的一个关键任务，通过分析道路环境信息和目标位置，生成一条最优的行驶路径。

**路径规划技术：**

1. **全局路径规划**：采用A*算法、RRT（快速随机树）算法等，生成从起点到终点的全局路径。
2. **局部路径规划**：采用动态窗口法、轨迹生成器等，根据当前环境信息，生成车辆的短期行驶路径。
3. **路径优化**：结合车辆动力学模型，优化路径的平滑性和安全性。

**路径规划优势：**

- **安全性**：通过全局和局部路径规划，确保车辆在复杂环境中行驶的安全性。
- **效率性**：优化路径，提高车辆行驶的效率。

**路径规划应用场景：**

- **自动驾驶车辆**：例如，自动驾驶车辆通过路径规划，实现从起点到终点的自动行驶。
- **无人配送车**：例如，无人配送车通过路径规划，实现高效、安全的货物配送。

###### 8.2.2 环境感知与决策

环境感知与决策（Environmental Perception and Decision Making）是自动驾驶控制系统中的一个关键任务，通过感知系统获取的环境信息，实现车辆对环境的适应和决策。

**环境感知与决策技术：**

1. **传感器数据融合**：融合摄像头、激光雷达、雷达等多种传感器数据，实现更全面的环境感知。
2. **多目标跟踪**：通过卡尔曼滤波、粒子滤波等算法，实现车辆、行人、障碍物等目标的同时跟踪。
3. **决策算法**：结合感知结果和车辆状态，采用深度强化学习、决策树等算法，实现车辆的控制决策。

**环境感知与决策优势：**

- **自适应**：通过环境感知与决策，实现车辆对复杂环境的自适应和响应。
- **智能性**：通过深度学习和强化学习算法，实现车辆智能化的控制决策。

**环境感知与决策应用场景：**

- **无人驾驶出租车**：例如，无人驾驶出租车通过环境感知与决策，实现自动驾驶和智能避障。
- **无人配送车**：例如，无人配送车通过环境感知与决策，实现智能配送和路径规划。

### 第三部分：AI技术应用实战

#### 第9章：AI应用开发实战

##### 9.1 实战项目介绍

###### 9.1.1 项目背景

随着人工智能技术的快速发展，越来越多的企业和组织开始尝试将AI技术应用于实际业务场景中，以提高效率和竞争力。本文将介绍一个实际应用AI技术的项目，旨在通过深度学习算法实现图像分类，从而帮助零售企业优化商品推荐系统。

###### 9.1.2 项目目标

- 利用深度学习算法对商品图像进行分类，提高商品推荐的精准度。
- 构建一个可扩展的AI模型，支持实时图像分类和预测。
- 提供一个用户友好的界面，方便企业员工和管理者使用和监控模型性能。

##### 9.2 数据准备与预处理

###### 9.2.1 数据收集

在项目初期，我们需要收集大量的商品图像数据，这些数据将用于训练和评估深度学习模型。数据来源包括电商平台、社交媒体和公开数据集。为了保证数据的质量和多样性，我们需要对数据进行筛选和清洗，去除重复、噪声和不合格的数据。

**数据收集步骤：**

1. **数据采集**：使用爬虫工具从电商平台上爬取商品图像和相关信息。
2. **数据筛选**：根据商品类别、质量、分辨率等条件筛选数据。
3. **数据清洗**：删除重复数据、填充缺失值、纠正错误数据。

###### 9.2.2 数据预处理

预处理是深度学习模型训练的重要步骤，通过对数据进行标准化、归一化和数据增强等操作，提高模型的学习效果和泛化能力。

**数据预处理步骤：**

1. **图像增强**：通过旋转、翻转、缩放、裁剪等操作，增加图像数据的多样性。
2. **数据归一化**：对图像数据进行归一化处理，将像素值缩放到[0, 1]范围内。
3. **标签编码**：将分类标签转换为整数编码，方便模型处理。

##### 9.3 模型训练与优化

在数据准备和预处理完成后，我们可以开始训练深度学习模型。本文选择卷积神经网络（CNN）作为图像分类模型，通过调整网络结构、优化算法和超参数，提高模型的分类准确率。

###### 9.3.1 模型选择

CNN是处理图像数据的常用神经网络结构，通过多层卷积层和池化层提取图像特征，实现高精度的分类。

**CNN模型架构：**

1. **输入层**：接收归一化的图像数据。
2. **卷积层**：通过卷积操作提取图像特征。
3. **池化层**：降低特征维度，减少计算量。
4. **全连接层**：将提取到的特征映射到分类结果。
5. **输出层**：产生分类预测结果。

###### 9.3.2 模型调参

模型调参是深度学习训练的重要环节，通过调整学习率、批次大小、正则化参数等超参数，优化模型的性能。

**调参方法：**

1. **网格搜索**：在给定的超参数范围内，逐一尝试所有可能的组合，选择最佳参数。
2. **随机搜索**：随机选择超参数组合，进行多次实验，选择最佳参数。
3. **贝叶斯优化**：使用贝叶斯优化算法，根据实验结果自动调整超参数。

##### 9.4 模型评估与部署

模型训练完成后，我们需要对模型进行评估，验证其在实际应用中的性能。同时，还需要将模型部署到生产环境中，实现实时图像分类和预测。

###### 9.4.1 模型评估指标

模型评估指标包括准确率、召回率、精确率、F1分数等，用于衡量模型在分类任务中的性能。

**评估指标计算：**

- **准确率（Accuracy）**：分类正确的样本数占总样本数的比例。
- **召回率（Recall）**：分类正确的正样本数占总正样本数的比例。
- **精确率（Precision）**：分类正确的正样本数占分类为正样本数的比例。
- **F1分数（F1 Score）**：精确率和召回率的加权平均，用于综合评估模型的性能。

###### 9.4.2 模型部署策略

模型部署是将训练好的模型部署到生产环境中，实现实时图像分类和预测的过程。

**部署策略：**

1. **容器化**：将模型和依赖库打包成容器镜像，提高部署的灵活性和可移植性。
2. **服务化**：将模型部署到服务器，提供API接口，方便其他系统调用。
3. **监控与维护**：实时监控模型性能，定期更新和优化模型。

###### A.1.2 基本使用方法

以下是一个简单的TensorFlow代码示例，实现线性回归：

```python
import tensorflow as tf

# 创建计算图
x = tf.placeholder(tf.float32, shape=[None, 1])
y = tf.placeholder(tf.float32, shape=[None, 1])
w = tf.Variable(0.0, name="weights")
b = tf.Variable(0.0, name="biases")
y_pred = w * x + b
loss = tf.reduce_mean(tf.square(y - y_pred))

# 梯度下降优化器
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.5)
train_op = optimizer.minimize(loss)

# 初始化变量
init = tf.global_variables_initializer()

# 训练模型
with tf.Session() as sess:
    sess.run(init)
    for i in range(1000):
        sess.run(train_op, feed_dict={x: x_train, y: y_train})
    # 输出模型参数
    print("weights:", sess.run(w))
    print("biases:", sess.run(b))
```

   ```

   ```

###### A.2.2 基本使用方法

以下是一个简单的PyTorch代码示例，实现线性回归：

```python
import torch
import torch.nn as nn
import torch.optim as optim

# 创建数据
x_train = torch.tensor([[1.0], [2.0], [3.0], [4.0]], requires_grad=True)
y_train = torch.tensor([[0.0], [1.0], [2.0], [3.0]], requires_grad=True)

# 创建模型
model = nn.Linear(1, 1)

# 损失函数和优化器
criterion = nn.MSELoss()
optimizer = optim.SGD(model.parameters(), lr=0.5)

# 训练模型
for epoch in range(1000):
    optimizer.zero_grad()
    y_pred = model(x_train)
    loss = criterion(y_pred, y_train)
    loss.backward()
    optimizer.step()

# 输出模型参数
print("weights:", model.weight)
print("biases:", model.bias)
```
