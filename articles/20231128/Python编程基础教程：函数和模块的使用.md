# 2.核心概念与联系

## 2.1 什么是函数？

函数（Function）是一种封装代码的功能，可以对输入参数进行预处理、执行一些逻辑运算或者计算得到输出结果，然后返回输出结果给调用它的地方。函数可以提高代码的复用性、可维护性、降低出错率、提高代码的可读性和可理解性。
函数的基本语法如下所示：

```python
def function_name(parameter):
    # 这里是函数体
    return output
```

- `function_name` 是函数的名称，我们习惯于首字母小写，多个单词之间使用下划线连接；
- `parameter` 是函数的参数列表，以逗号分隔，如果没有参数，则可以不写；
- `return` 表示函数返回值，可以返回一个或者多个值，如果没有返回值，则可以省略此行；
- `#` 表示注释，用于描述函数作用或实现过程。

例如：

```python
def my_sum(a, b):
    """
    计算两个数字的和。
    :param a: 第一个数字。
    :param b: 第二个数字。
    :return: 和。
    """
    c = a + b
    return c


result = my_sum(1, 2)
print(result)  # Output: 3
```

上述代码定义了一个名为 `my_sum()` 的函数，该函数接受两个参数 `a` 和 `b`，并返回它们的和。该函数还包含函数的文档字符串（docstring），这是一条帮助用户了解函数目的和用法的信息。当用户调用该函数时，可以通过按? 来查看函数的文档字符串。通过这种方式，我们可以在代码中添加注释来提供额外信息，为其他人阅读我们的代码做出贡献。

## 2.2 为何要使用函数？

使用函数可以提高代码的可读性、可维护性、可扩展性。

### 提高代码的可读性

函数的名字通常能清晰地表达函数的作用，并且易于理解。比如，函数 `increase_by_one()` 就是用来将输入的值增加一的函数。这样，函数调用者就可以清楚地知道该函数完成了哪些操作。

### 提高代码的可维护性

函数提供了一种抽象层，使得代码更容易修改和调试。比如，假设我们想改进上述函数，使之增加输入值的倍数。如果直接在原函数中修改代码，可能会造成影响，因为函数的调用方可能依赖于原来的行为。但通过创建新的函数来实现新功能，可以防止原有的函数被破坏。

### 提高代码的可扩展性

函数可以很方便地进行单元测试，这有助于验证函数是否按照预期工作。

## 2.3 类比生活中的例子

让我们把函数看作一台机器，它接收输入数据，对其进行加工，然后输出结果。举个例子，假如有一个工厂有两类产品 A 和 B，每类产品生产需要耗费不同的时间，假定工厂生产一件产品的时间分别是 tA 和tB 。那么，我们可以创建一个名为 `production()` 的函数，它会根据不同产品类型计算生产所需的时间，并返回总时间。如下所示：

```python
def production(product_type):
    if product_type == 'A':
        time = tA()  # 根据产品类型计算相应的生产时间
    elif product_type == 'B':
        time = tB()
    else:
        raise ValueError('Invalid input!')

    total_time = time * 2  # 假定两次加工相同数量的产品
    return total_time
```

这个函数有两种类型的产品，每种产品都有一个不同的生产时间，而且两次加工同样数量的产品花费的时间是一样的。这是一个非常简单的例子，真实世界里面的函数往往会复杂得多。比如，函数可能需要读取文件，进行某些数据处理，并生成报告。函数的输出结果可能要保存到文件或数据库中，甚至可能需要通过网络传输给别的计算机。总之，函数使得生活中的许多事情变得更加容易，我们可以利用函数来解决各种各样的问题。
