# 2.核心概念与联系
## 2.1 Bigtable基本概念
Google Bigtable是一种分布式的宽列式存储系统，它的设计目标是提供高性能、高可扩展性和高可靠性。Bigtable的核心概念包括：

- **表（Table）**：Bigtable中的表是一种类似于关系型数据库中的表，它由一组列组成。表的每一行都包含了一组列值，列值可以是任意长度的字节序列。
- **列族（Column Family）**：列族是一组连续的列，它们共享一个common key空间。列族的设计使得Bigtable能够在同一时刻对大量的读写请求进行并行处理。
- **列（Column）**：列是表中的一列数据，它由一组值组成。每个列的名称是一个字符串，列的值是一个字节序列。
- **单元格（Cell）**：单元格是表中的一个具体数据项，它由一行和一列定义。单元格的值是一个字节序列。

## 2.2 Bigtable与关系型数据库的区别
Bigtable与关系型数据库在设计理念和功能上有很大的不同。关系型数据库基于关系模型，它的核心概念是表和关系。关系型数据库使用SQL语言进行查询和操作，它的数据结构和操作都是基于行和列的。

而Bigtable则是一种宽列式存储系统，它的设计目标是提供高性能、高可扩展性和高可靠性。Bigtable使用列族来存储数据，这使得它能够在同一时刻对大量的读写请求进行并行处理。此外，Bigtable不支持SQL语言，而是使用自己的API进行数据操作。
