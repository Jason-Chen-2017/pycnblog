# 2.核心概念与联系

## 2.1列存储
列存储是Table Store的核心概念之一。它是一种数据存储方法，将数据按照列存储在磁盘上。这种存储方法有以下优点：

- 数据压缩：由于列存储将相同类型的数据存储在一起，我们可以对这些数据进行压缩。这有助于减少磁盘空间的使用，并提高数据查询的速度。
- 数据分区：列存储可以将数据按照某个键进行分区。这有助于减少数据查询的时间，并提高查询性能。
- 数据并行处理：列存储可以将数据按照列进行并行处理。这有助于提高数据处理的速度，并减少计算资源的使用。

## 2.2分区
分区是Table Store的另一个核心概念。它是一种将数据划分为多个部分的方法，以便更有效地存储和查询数据。分区可以按照不同的键进行划分，例如时间、地理位置等。这有助于减少数据查询的时间，并提高查询性能。

## 2.3压缩
压缩是Table Store的另一个核心概念。它是一种将数据存储在磁盘上的方法，以减少磁盘空间的使用。压缩可以通过将相同类型的数据存储在一起，并使用特定的算法对其进行压缩来实现。这有助于减少磁盘空间的使用，并提高数据查询的速度。
