# 2.核心概念与联系
# 2.1 Cover定理简介
Cover定理是一种用于评估程序测试覆盖率的方法，它通过对程序的所有可能执行路径进行评估来确保程序的正确性和完整性。Cover定理的核心概念包括：

- 可行路径：程序中的所有可能执行路径，包括条件语句、循环、函数调用等。
- 覆盖率：测试用例覆盖的可行路径的比例，用于评估测试的质量。
- 条件覆盖率：测试用例覆盖的所有可能的条件组合的比例。
- 路径覆盖率：测试用例覆盖的所有可能的执行路径的比例。
