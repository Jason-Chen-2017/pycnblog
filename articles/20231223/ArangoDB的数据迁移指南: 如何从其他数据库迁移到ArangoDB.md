# 2.核心概念与联系
# 2.1数据迁移的基本概念
数据迁移是将数据从一种数据库系统迁移到另一种数据库系统的过程。这可能是由于性能、可扩展性、成本或功能需求等原因。数据迁移可以涉及到不同的数据库类型，例如关系型数据库到非关系型数据库、非关系型数据库到关系型数据库、或者不同的非关系型数据库。

# 2.2ArangoDB与其他数据库的关系
ArangoDB是一个多模型数据库，支持文档、关系型和图形数据模型。因此，在迁移到ArangoDB时，可能需要考虑以下几种数据库类型：

- 文档类数据库：例如MongoDB、Couchbase等。
- 关系型数据库：例如MySQL、PostgreSQL等。
- 图形数据库：例如Neo4j、OrientDB等。
