# 2.核心概念与联系

在进行UI测试自动化框架的比较之前，我们需要了解一些核心概念。以下是一些关键术语的解释：

1. **UI测试自动化框架**：UI测试自动化框架是一种软件测试方法，它使用自动化工具来执行用户界面测试。这些框架通常包括一组API和库，用于创建、执行和维护测试用例。

2. **GUI测试**：GUI测试是一种软件测试方法，它旨在验证软件的图形用户界面（GUI）是否满足预期的需求和功能。GUI测试通常包括窗口、对话框、菜单、按钮等组件的测试。

3. **功能测试**：功能测试是一种软件测试方法，它旨在验证软件是否满足其预期的功能需求。功能测试通常包括输入、输出、数据处理等方面的测试。

4. **性能测试**：性能测试是一种软件测试方法，它旨在验证软件是否满足预期的性能要求。性能测试通常包括响应时间、吞吐量、并发性能等方面的测试。

5. **兼容性测试**：兼容性测试是一种软件测试方法，它旨在验证软件是否在不同的环境中正常工作。兼容性测试通常包括操作系统、浏览器、设备等方面的测试。
