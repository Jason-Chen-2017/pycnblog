# 1.简介
         
## PCI DSS 2.3 概述
PCI DSS(Payment Card Industry Data Security Standards)2.3，全称为“支付卡行业数据安全标准”，是一种风险管理准则，它定义了支付卡公司的运营、贸易、客户服务等方面的信息安全要求。PCI DSS 2.3已经推出了第一次修订版本，其目的是提供可行的、普适的、集成的、可重复使用的解决方案，帮助组织有效识别、分析和解决安全事件。PCI DSS 2.3共分12条规则和27个控制点，这些规则及控制点主要是为了保障支付卡公司的信息安全、业务数据的完整性、机密性和可用性。通过遵守PCI DSS 2.3中的规则和控制，可以降低支付卡公司可能发生的安全事故、泄露、攻击或破坏。
根据PCI DSS 2.3规定，支付卡公司应该采取以下措施来满足PCI DSS 2.3的要求：
- 使用通用加密技术保护个人身份信息、信用卡号码和卡片相关的数据；
- 为所有系统组件建立合理的访问权限控制；
- 限制特权用户的权限，防止他们执行系统权限提升攻击;
- 使用适当的网络安全策略和防火墙配置；
- 配置数据库审计功能，检测和记录对敏感数据的访问；
- 在开发过程中就系统安全性进行评估和测试；
- 测试人员应具有系统安全知识并负责系统漏洞的修复和更新；
- 使用更新且充分验证过的操作系统、应用程序和Web浏览器。
## PCI DSS 2.3管理应用程序的方法论
本文档旨在阐述PCI DSS 2.3规范中关于应用安全管理方法论。应用安全管理（Application Security Management，ASM）是指将对应用的安全建设作为一个整体进行管理，包括但不限于对应用源代码的安全编码、设计模式的采用、网络通信安全、外部接口的安全控制、应用数据的安全处理、应用运行环境的安全监测、安全报告、渗透测试等一系列工作。应用安全管理可以有效地保障企业数字化转型期间产生的应用系统的安全。
要实现应用安全管理，需要遵循如下几个关键步骤：
- 识别应用安全需求
- 制定应用安全计划
- 实现应用安全实施
- 持续改进应用安全
通过应用安全管理，组织可以有效降低应用系统的安全风险，缩短应用更新迭代周期，提升应用系统的可靠性和可用性。在本文中，将讨论PCI DSS 2.3规范中的几种管理应用的方法。
### 方法1：输入过滤
输入过滤是应用安全管理的一项重要手段。输入过滤是指对从外部传入应用的数据流进行检查，检查其是否符合预定义的安全规则。例如，对于银行卡交易系统来说，输入过滤可以用来验证来自终端用户银行卡交易请求中的卡号码，确保它们符合PCI DSS 2.3中规定的格式要求。输入过滤也可以用于检测恶意攻击和垃圾邮件的网络钓鱼行为。
### 方法2：输出过滤
输出过滤是应用安全管理的另一重要手段。输出过滤是指对应用发送到外部的响应进行检查，检查其是否符合预定义的安全规则。例如，对于电子商务网站来说，输出过滤可以用来验证支付请求是否成功接收，确保它们被正常记录和处理。输出过滤也可以用于检测潜在的反垃圾邮件、网络钓鱼和恶意攻击。
### 方法3：异常检测和容错机制
异常检测是应用安全管理的第三种方法。异常检测是指检测应用运行过程中突然出现的异常情况，比如时序攻击、重放攻击或者垃圾邮件攻击。异常检测可以帮助检测应用中的缺陷和弱点，并且可以帮助应用快速纠正和避免这些缺陷。异常检测还可以通过设置容错机制使应用遇到异常情况时可以自动平稳运行，从而避免造成灾难性的后果。
### 方法4：审计和日志记录
审计和日志记录是应用安全管理的第四种方法。审计和日志记录是指记录应用的运行过程中的各种活动，包括用户登录、操作、资源访问、错误消息、安全事件等。审计和日志记录可以帮助记录应用的所有操作记录，并且还能够分析应用的运行行为，找出其中的异常和威胁，并针对性地做出相应的防御措施。通过建立健全的审计记录和日志系统，可以使得应用安全管理更加顺利，有效地保障应用的安全。
### 方法5：认证和授权
认证和授权是应用安全管理的第五种方法。认证和授权是指确定用户的真实身份，并确保只有授权的用户才能访问应用和其相关的数据。通过实现应用级的认证和授权，组织可以更好地控制用户对应用的访问权限，从而保证应用数据的安全。
综上所述，PCI DSS 2.3规范中提到的应用安全管理方法，如输入过滤、输出过滤、异常检测、审计和日志记录、认证和授权，都可以有效地促进应用的安全建设。应用安全管理是使企业数字化转型过程顺利进行、有效应对数字化危机、保障应用系统的安全的重要环节。
