# 2.物联网概述
## 2.1 什么是物联网
物联网（Internet of Things，IoT）由两个词组成，即 Internet 和 Thing 。Internet 代表网络互联，把各种设备连在一起；Thing 表示的是由无数的实体组成的网络。这些实体，包括传感器、服务器、网关、路由器等都被称作“物体”。物联网就是将互联网和物体连接起来，让它们共同工作，形成自动化的信息处理、传输和控制机制。如此一来，物联网将数据采集、处理和控制放在了一起，实现了数据的共享、分析和利用，极大地提高了生产力和效率。

## 2.2 物联网的定义
物联网是一个非常广泛的概念，涵盖了多种不同的领域。其中最主要的三个定义如下：
1. 全球物联网（Global IoT）：这是定义物联网的第一个标准。它提出，物联网是一个世界范围内的信息系统。因此，这一定义涵盖了不同国家、不同地区以及不同行业之间的物联网交流。
2. 自组织网（Self-organizing Network）：这是定义物联网第二个标准。它描述物联网是一个由个体节点构成的自治网络，可以自主进行协同工作，确保信息安全。
3. 精细物联网（Fine-grained IoT）：这是定义物联网第三个标准。它强调物联网应该具有高度的灵活性，能够满足不同领域的需求。例如，运动监控、环境监测、金融监管等都属于精细物联网的范畴。

# 3.物联网平台架构
## 3.1 物联网平台的组成
物联网平台由两部分构成：前端和后端。
前端：负责用户界面，包括手机、平板电脑等终端设备上运行的应用程序，以及浏览器等网页浏览界面。前端向用户提供服务，包括数据采集、呈现、管理、远程访问等功能。
后端：负责数据存储、数据处理、业务逻辑处理等后台功能，如数据库、消息队列、Web服务器等。后端支持前端的各种功能，包括数据接入、处理、推送等功能。

## 3.2 数据采集
数据采集可以分为两种形式：
1. 实时采集：实时采集指的是收集物体当前的时间和空间数据，比如温度、湿度、压强、位置、速度等。这种数据是最精准、实时的。但它带来的代价是高延迟、低带宽，而且对网络的要求也很高。
2. 离线采集：离线采集指的是将物理设备的数据存储下来，再进行数据处理和分析。这种方式可以解决实时采集的问题，降低延迟、提高带宽。但它又存在数据存储的问题，且可能无法获取全部数据。

物联网平台的核心任务之一就是实现数据的采集，从而对物体产生的行为做出响应。目前，物联网平台通常采用两种类型的采集技术：
1. 智能终端设备采集：这种方法使用物联网设备作为采集源，直接采集到物体的数据。
2. 中间件采集：这种方法使用中间件系统作为采集源，比如云网关、服务器集群等。

## 3.3 数据处理
数据处理是物联网平台的关键环节，其目的是对采集到的数据进行加工处理。数据处理通常包括数据清洗、过滤、聚合、转换、计算等过程。一般情况下，数据处理会使用数据仓库、分布式数据库等技术，将处理好的数据存储下来。同时，平台还需要对数据进行展示、统计、报警、诊断等功能。

## 3.4 业务逻辑处理
业务逻辑处理是物联网平台的重要能力之一。它提供了一种可以自定义的编程语言，用来实现规则引擎、决策树等算法，根据设备上传的不同类型数据，自动执行特定任务。业务逻辑处理还包括应用控制策略、智能网关、物流管理、安防系统、视频监控等。

## 3.5 数据中心
物联网平台的中枢部分是数据中心。数据中心用于存储所有平台的原始数据、处理后的数据、规则配置、策略参数、历史数据、日志文件等。它还可以通过数据中心提供的接口，向外提供统一的服务。

# 4.边缘计算概述
## 4.1 边缘计算的定义
边缘计算是一种新的计算机计算模型，它将计算和应用分离。它将计算工作从数据中心传输到本地设备，这样可以降低数据的传输延迟、提升计算效率、节省网络带宽资源。

边缘计算主要分为三类：
1. 数据处理：指对来自智能终端设备或中间件的原始数据进行数据处理，如过滤、清洗、计算、存储等。
2. 分析处理：指对设备数据进行复杂的分析，如图像识别、机器学习等。
3. 通信处理：指对设备之间的数据交换进行优化，如短信、电信等。

## 4.2 边缘计算的优势
边缘计算的主要优势包括：
1. 低延迟：边缘计算可以将运算处理任务放置在靠近物体的位置，从而减少网络延迟，提高数据处理的速度。
2. 更小的带宽：边缘计算可以将数据传输、处理、存储等任务转移到本地，从而减少网络带宽资源消耗。
3. 更好的性能：边缘计算可以使用更加强大的硬件资源，提高计算性能。

# 5.云端计算 VS 边缘计算
## 5.1 云端计算
云端计算是一种服务器型的计算模型，其主要特征是计算和数据存储都在云端，通过网络进行交互。云端计算包括以下几类：
1. 公有云：指由互联网服务提供商提供的公共服务器，一般为大型企业或政府机构所拥有的。
2. 私有云：指由私营公司或个人所拥有的服务器，一般是内部使用的。
3. 混合云：指私有云与公有云相结合，既提供公共服务，也提供自身内部使用的服务。

## 5.2 云端计算的优缺点
### 优点
1. 按需付费：云端计算服务按消费量计费，只需支付实际使用量的费用，可以降低成本。
2. 可伸缩性：云端计算服务可以根据需求增加或减少计算资源，使得系统能够应对变化中的负载。
3. 技术支撑：云端计算服务所提供的技术框架、工具和服务都比较成熟。
4. 服务全面：云端计算服务包括数据存储、计算资源、网络、软件等多个方面的服务。

### 缺点
1. 隐私问题：云端计算服务可能会收集并处理用户数据，可能造成隐私泄露。
2. 服务稳定性：云端计算服务由于依赖于外部供应商，所以容易出现故障。
3. 安全问题：云端计算服务可能会受到恶意攻击，存在安全漏洞。
4. 数据中心距离限制：云端计算服务通常是部署在云端的数据中心，与物体距离较远，带宽和延迟较低。

## 5.3 边缘计算的优缺点
### 优点
1. 减少响应时间：边缘计算可以在本地计算，不必等待数据传输回来。
2. 更好的响应能力：边缘计算可以与边缘设备直接通信，获得实时数据，而不需要过多的数据传输开销。
3. 大幅降低成本：边缘计算可以将数据中心的计算资源转移到边缘设备，从而降低成本。
4. 更好的响应速度：边缘计算可以提升计算的响应速度，同时降低服务器端的资源消耗。

### 缺点
1. 更新缓慢：边缘计算的更新周期较长，只能适应实时数据的需求。
2. 不够成熟：边缘计算的相关技术和服务仍处于初期阶段，还没有完全取代传统的云端计算。
3. 数据安全：边缘计算依赖于本地通信，可能会存在数据安全风险。
4. 需要更多的硬件资源：边缘计算需要更多的硬件资源来进行计算，成本比云端计算高很多。
