# 1.简介
         
数据标准在信息安全领域具有重要作用。随着电子政务、数字货币和区块链等新兴技术的快速发展，越来越多的数据产生，需要遵循一些标准或准则对其进行保护。比如，支付场景下使用信用卡支付时，需要遵循PCI-DSS国际数据安全标准；电子公证平台上进行交易之前，需要遵守国家的电子公证法律法规。数据标准的制定和管理成为社会各个方面的共识和习惯。但是，如何确保数据标准的安全、有效且符合相关法律要求是一个复杂而又艰巨的任务。数据的安全问题无处不在，并面临着各种迫切需求。
# 2.标准的定义及分类
数据标准(Data Standard)是指由一组接口、规则和约束所确定的事实上可被所有用户接受或认同的一套处理、存储和通信方式、方法、格式、协议和格式、流程和要求。它作为一种系统级的规范，包括计算机系统、数据库系统、网络系统等各种各样的系统和设备间共享的通用约定，旨在确保数据在不同的应用领域、环境条件和上下游组织之间可以互相通信。数据标准通常由多个方面组成，如接口规范、业务规范、通讯规范、传输规范、数据模型、服务模型、安全规范、隐私规范、合规性规范等。数据标准分为技术标准和管理标准。

技术标准是指通过定义、结构化、规范化以及制订相关文档，对数据类型、传输格式、协议、加密机制、授权机制等方面进行技术层面的约束。技术标准通常基于行业内已有的标准，但也可能根据特定的应用领域、行业需求制定新的标准。例如，美国国家标准技术研究院（NIST）发布了FIPS PUB 199、RFC 4309、ANSI X9.31、ISO/IEC 18001、ISO/IEC 21017、ISO/IEC 27001、HIPAA、NIST SP 800-18、NIST SP 800-30、GDPR、CCPA、COPPA、GLBA等十几种技术标准。

管理标准是指通过制订相应的规范、要求、流程和管理机制，管理者应当遵循来自主管部门的审批流程、管理措施、权威机构评估等管理手段来管理数据标准的推行。管理标准对数据标准的开发、实施、维护、更新等方面均有相关要求。

根据数据标准的类型，可以将它们分为以下三类:

IT数据标准:IT数据标准用于确保在信息技术 (IT) 系统之间传递数据时的质量、安全和可靠性。它们往往适用于企业内部的信息系统，包括计算机系统、服务器、数据库、应用程序、硬件、网络设备等。在IT数据标准中，涉及到数据类型、传输格式、协议、授权机制、加密机制、数据完整性校验、保密性和可用性等方面。
业务数据标准:业务数据标准是企业在特定业务领域中制订的数据标准，主要用来确保数据传递时的完整性、一致性、正确性、及时性和可用性。例如，银行间支付数据标准就是一个典型的业务数据标准。业务数据标准一般会更加注重相关的法律义务。
法律数据标准:法律数据标准是关于法律法规、规则、指令的集合，主要用于保障数据、人员、财产或其它知识产权的合法性、有效性、保密性和完整性。例如，保险公司的医疗保险数据标准就是法律数据标准。
# 3.数据标准的工作原理
数据标准的制定过程可以分为三个阶段:

1.立项阶段:首先要确定数据的生命周期以及流向，收集相关的数据需求、业务模型、协议和标准，并编写项目计划。此后，制定并提交工作报告，以征求相关部门的意见。

2.审核阶段:审核过程中，会对项目计划和计划中的需求、业务模式、标准和协议进行详细审查，还要检查产品是否满足相关法律规定、规范和协议的要求。

3.批准阶段:当审核完成并且项目经费全部支付完毕之后，就可以正式发布数据标准了。数据标准发布之后，就进入了生效阶段。如果遇到任何问题，可以通过修订的方式来解决。

数据标准的执行过程可以分为如下四个环节:

1.支持阶段:支持阶段包括对数据标准进行风险分析、测评、监控、检测、评估等工作。

2.实施阶段:实施阶段包括制定标准操作手册、数据治理、数据安全等工作。

3.改进阶段:改进阶段主要是持续不断地对数据标准进行持续跟踪、调整和优化。

4.维持阶段:维持阶段是指根据新发展，对数据标准进行适当的更新、补充和调整。

数据标准除了支持IT数据标准之外，还有其他类型的标准，如业务数据标准和法律数据标准。其中，法律数据标准最为重要。因为数据保护、数据安全和数据合规等保障人民生活水平的重要需求依赖于法律数据标准。因此，在设计和实施数据标准时，需充分考虑数据保护和法律效力两个角度。
