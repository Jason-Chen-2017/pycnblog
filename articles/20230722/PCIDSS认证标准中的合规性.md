# 1.简介
         
在IT行业中，越来越多的公司要求自己提供“支付卡颁发机构（PCI）数据安全标准（DSS）”认证，其中就包括PCI-DSS自身的认证标准。由于PCI-DSS认证项目中含有很多重要的认证标准，比如ISO/IEC 27001、NIST SP 800-53、HIPPA、FISMA等，因此了解这些认证标准背后的基本概念、术语和算法原理对于对照理解和分析PCI-DSS认证标准的过程非常重要。而计算机专业人员，尤其是嵌入式系统工程师、安全专业人员都需要比较熟悉PCI、DSS、ISO/IEC 27001、NIST SP 800-53、HIPPA、FISMA这些国家相关的标准，对它们进行正确的理解和运用才能取得更好的效果。
# 2. PCI(Payment Card Industry) 支付卡行业协会
美国PCI协会于1996年成立，主要职责是在银行、金融机构、保险公司和支付卡公司之间建立信息交流、技术沟通和协作的桥梁，促进信息的共享和价值增长。PCI成员单位包括美国的国民身份证明中心、美国的商业银行、美国的储蓄银行、美国的支付机构、美国的保险公司、美国的信用卡和贷款公司。2006年，PCI正式成为美国国际支付领域的非政府组织。
# 3. PCI DSS (Payment Card Industry Data Security Standards) 支付卡行业数据安全标准
PCI-DSS是美国PCI协会推出的一系列数据安全标准，由欧盟、美国、加拿大、澳大利亚、新西兰、英国等五个国家的银行、金融机构、保险公司、支付卡公司、电信运营商和通信设备制造商联合编制，旨在提升支付卡行业的数据安全能力并推动信息技术应用和服务的改善。除了PCI-DSS本身之外，还存在其他数据安全标准如NIST SP 800-53、ISO/IEC 27001、HIPAA、FISMA等。
# 4. ISO/IEC 27001 (Information Technology - Security techniques - Information security management systems - Part 1: Architecture and requirements) 信息技术安全管理体系 - 信息安全技术 - 信息安全管理系统 - 第1部分: 体系结构与需求规范
ISO/IEC 27001是美国国际标准化组织（International Organization for Standardization）于2000年发布的最高级别的计算机安全管理标准，它是一个涵盖了信息安全管理体系各方面的规范，包括安全策略、技术和流程、管理方法、测试、监控、评估和报告等方面。ISO/IEC 27001包含的信息安全管理体系的第一部分与第二部分两个部分。
# NIST (National Institute of Science and Technology) 国家信息安全局(National Institute of Science and Technology，NIST) 是美国政府为公众提供安全保障的全球性科技实验室。NIST在其官方网站上共分为8大安全领域，包括电子邮件、密码学、网络安全、身份管理、生物识别、安全基础设施、信息系统和软件，以及IT物理安全。NIST秘书处曾经历过PCI-DSS的认证。
# HIPPA (Health Insurance Portability And Accountability Act) 医疗保险可移植性与责任法律（HIPAA） 是美国联邦政府为了保护消费者个人隐私和健康信息而制定的一项法律。它由联邦政府、州政府和地方政府联合制定，并授权国会监督，作为医疗保险的信托及责任依据。
# FISMA (Foreign Intelligence Surveillance Act) 欧洲情报监视法（Foreign Intelligence Surveillance Act，FISMA） 是1970年通过的联邦法案，它赋予美国政府在当时特别关注的物理空间目标的权力。美国政府的敏感地位和对情报界、军事部门以及国际关系部门所传达的指令的高度重视，使得此法成为了美国政府抵御恐怖主义、恐怖分子以及其它非法活动的有效工具。
# 5. NIST SP 800-53 (Rev. 4) 网络安全控制手册（NIST Special Publication 800-53 Revision 4）
NIST SP 800-53是一套美国国土安全部（Department of Commerce）主管部门用于管理、评估、跟踪和评估计算机信息系统（CIA）安全风险的概要技术指南。NIST SP 800-53将安全管理的各方面分为六大主题：
1. 物理访问控制
2. 身份验证
3. 访问控制
4. 错误处理
5. 数据备份和恢复
6. 漏洞管理

NIST SP 800-53提供详细的安全控制，从而为开发、部署、运维和维护信息系统提供可靠的指导。同时，NIST SP 800-53也要求组织履行信息安全管理制度，持续审查安全措施，不断完善防范措施，确保系统免受威胁和漏洞攻击。
# 6. HIPPA认证申请流程
## 6.1. 注册申请
首先需到HIPPA官网https://www.hhs.gov/hipaa/for-individuals/regulation/index.html，填写相关资料。并上传有效的身份证件、健康信息承诺表、符合HIPAA管辖的产品销售许可、商标注册、防伪声明、销售商品／服务合同等相关文件。
## 6.2. 资格审查
当提交资料审核后，收件地址即为完成评估的结果。如果符合HIPPA资格，即可申请HIPPA认证。
## 6.3. 准备材料
根据具体业务场景，做好合规义务清单，并准备完整的HIPAA协议、业务协议、服务协议、授权委托书、授权委托人资格证明、确认书等相关资料。
## 6.4. 投函递交
符合资格后，需打印并签名的申请表（必须）、授权委托书（可以）、证明材料（至少两封）、确认书（必需），按指定的邮寄地址寄送到指定的邮政局信箱。
## 6.5. 测试评估
一般在3～7天内完成HIPAA评估。会通过电话、短信或信件方式通知收件人的状态。
