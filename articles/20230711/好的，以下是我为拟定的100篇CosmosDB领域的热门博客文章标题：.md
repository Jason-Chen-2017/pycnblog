# 2. 技术原理及概念

## 2.1. 基本概念解释

Cosmos DB 采用了一些新的数据模型和数据结构，如分片、键值对、文档、列族、列和图形数据类型。

## 2.2. 技术原理介绍：算法原理，具体操作步骤，数学公式，代码实例和解释说明

### 2.2.1. 数据分片

Cosmos DB 使用数据分片来支持高效的键值对读取。数据分片是一种跨范式的数据存储和查询方法，它将数据根据键的哈希函数分成多个片段，每个片段都可以存储不同的数据类型。

### 2.2.2. 键值对

键值对是 Cosmos DB 中最基本的存储方式，它由一个键值对和多个文档组成。键值对支持高效的读取，但写入性能较低。

### 2.2.3. 列族

列族是一种类似于文档的存储方式，但列族中每个列的值都是不可变的。列族支持按列进行分片，但写入性能较低。

### 2.2.4. 列

列是一种非常简单的存储方式，它只包含一个值。列支持高效的写入和查询，但不支持分片。

### 2.2.5. 图形数据类型

图形数据类型支持复杂的图形数据结构，如路线、网络和形状数据。图形数据类型可以存储键值对，列族和列，以及文本数据。
