# 2. 技术原理及概念

## 2.1. 基本概念解释

关系型数据库（RDBMS）和 NoSQL 数据库的最大区别在于数据模型和数据存储方式。关系型数据库采用表格和行/列的结构来存储数据，而 NoSQL 数据库采用文件、键值对和文档等数据结构来存储数据。

MariaDB 和 MongoDB 都是 NoSQL 数据库。它们都支持键值对数据结构，并且都提供了类似于文档和集合的功能。然而，它们之间也有一些区别。

## 2.2. 技术原理介绍: 算法原理，具体操作步骤，数学公式，代码实例和解释说明

### 2.2.1. 数据存储方式

MariaDB 和 MongoDB 都采用分片和聚类的技术来存储数据。分片是指将数据分成多个片段，以便于并发访问。聚类是指将数据按照某种规则进行分组，以便于查询。

### 2.2.2. 数据访问方式

MariaDB 和 MongoDB 都支持查询，包括全文搜索、聚合和分片等。此外，它们还支持插入、删除和更新操作。

### 2.2.3. 数据操作

MariaDB 和 MongoDB 都支持数据操作，包括基本操作（GET、SET、ADD、REMOVE）和高级操作（CRUD、SELECT、JOIN、AGGREGATE等）。

| 数据存储方式 | 分片和聚类 | 分片和聚类 |
