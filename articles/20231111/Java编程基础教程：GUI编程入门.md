                 

# 1.背景介绍


## 一、Java GUI简介
图形用户界面（Graphical User Interface，简称GUI），是指一种通过人机交互方式与计算机进行信息交流的方式。一般情况下，GUI由窗口、菜单、控件组成。窗口提供一个框架，用于容纳控件；菜单栏则用于显示应用程序功能，在屏幕上方出现；控件则是一个用户交互接口元素，例如按钮、标签、文本框等，能够让用户进行各种输入或输出操作。GUI为用户提供了直观、易用、富有吸引力的图形界面，使得计算机应用具有更高的可用性和可视化能力，同时提升了用户体验。

Java语言中有多种图形组件供开发者选择，如Swing、AWT、 SWT、JavaFX等，它们都可以用来构造GUI界面，并且各自有其特有的特性。其中，Swing最初是Sun公司为开发JFC（Java Foundation Classes）时提供的一套图形用户界面组件集，由于其跨平台特性，Swing已经成为最主流的Java GUI开发工具之一。除此之外，还有其他几种Java GUI开发框架，如JavaFX、Java Web Start等。其中，JavaFX是Oracle公司推出的基于Java SE规范的新型GUI开发框架，目前已广泛应用于各种Java桌面程序和Web浏览器。另外，Spring Framework也支持构建Java GUI界面，并通过JavaServer Faces框架实现动态生成HTML页面作为前端展示。

## 二、为什么要学习GUI编程？
### （一）便捷使用
由于现代计算机系统普遍配备了多种不同形态的硬件设备，包括键盘鼠标、触摸板等，使得用户可以从不同的角度操作电脑。GUI技术可以将复杂的数据、信息、图像、视频等信息以可视化的方式呈现给用户，直接向用户展现计算机的功能及用途。这样不仅方便用户使用，而且能有效提升工作效率。

### （二）用户体验优化
今天的互联网产品越来越复杂，用户需求变动频繁，如移动互联网、智能手机、平板电脑、车载终端等。GUI技术能够有效地改善用户体验，提供便利的操作方式和流畅的响应速度，从而提升产品的受众群体。

### （三）提升软件质量
图形用户界面设计的好坏直接影响到软件的质量，一旦设计的不当、效果不好甚至丧失可用性，就很难避免软件烂尾。因此，掌握GUI技术对于软件开发人员来说尤其重要，它能够帮助工程师提升软件的设计水平、调试能力和性能表现，改善产品的运行效果和用户体验。

# 2.核心概念与联系
## 一、窗体组件
### （一）概述
窗体组件（Frame component）又称容器组件（Container component），是构成GUI的基本单位。所有的窗体组件都有一个父容器，它决定了该组件相对哪个容器的位置。如果没有父容器，那么就是顶层窗体（root frame）。

### （二）分类
#### ⑴顶级窗体
所有的Java应用程序都是以某个顶级窗体作为根节点启动的，它被称为“主窗口”。所有其他的组件都嵌套在该窗体下，如菜单栏、菜单项、按钮、对话框、下拉列表、滚动条等。

#### ⑵子级窗体
Java中除了顶级窗体外，还可以创建子级窗体。子级窗体通常位于主窗口之上，但也可以嵌套在其他窗体中。子级窗体可以是透明的或者非透明的，即“模态”还是“非模态”。

##### a) 模态窗体
模态窗体是指该窗体是一个独立的窗口，用户只能在该窗体内进行操作，无法操作其他任何窗口。只有当前窗口关闭后才能访问其他窗口。模态窗体一般用于处理一些比较复杂的操作，如编辑文件、打印文档、打开另存为等。

##### b) 非模态窗体
非模态窗体是指该窗体不是一个独立的窗口，它将自己的窗口覆盖在母窗体上。非模态窗体可以拖动，大小调整，还可以容纳其他窗口。非模态窗体一般用于处理一些简单、快捷的操作，如文件查看、消息提示、弹出窗口等。

#### ⑶对话框窗体
对话框窗体是指弹出的窗口，用于完成某些特殊任务，比如文件的打开、保存、打印等。对话框窗体中的控制部件（如文本框、按钮）和功能按钮（如“确定”、“取消”）往往绑定某个事件处理器。

#### ⑷子级对话框窗体
子级对话框窗体也是一种对话框窗体，但是它位于其它窗体的内部。在这种窗体中，用户只需要填充必要的信息就可以执行特定任务，不需要切换到别的窗口。这些窗体通常出现在一个窗口的内部，做为其控制逻辑的一部分。

#### ⑷组合窗体
组合窗体是多个窗体组件组装起来的窗口。它提供了统一的窗口管理和布局机制，方便用户管理多窗口间的关系。

## 二、组件与容器
组件（Component）是构成GUI的基本单元。每个组件都存在于一个容器中，容器决定了它的位置、大小、边界和外观。组件分为以下几类：

1. 基本组件

   基本组件是最简单的组件，它只是简单的包含一些文本、图片、声音等内容。例如，标签、按钮、单选框、复选框等都是基本组件。

2. 容器组件

   容器组件是构成GUI的基本单位，它可以容纳其他组件，并控制它们的布局。容器组件又分为两大类：容器类和窗体类。

3. 组合组件

   组合组件是可以容纳其他组件的容器。例如，面板、分隔符、表格、滚动面板、选项卡等都是组合组件。

容器类组件又分为四种类型：

1. 面板组件

   面板组件是一种容器组件，它提供了一个矩形的空间来显示其内容。面板本身也可以容纳其他组件。例如，JPanel类、BorderLayout类、BoxLayout类、GridBagLayout类等都是面板组件。

2. 分隔符组件

   分隔符组件是一种容器组件，它提供一个水平或垂直的分割线，用来划分不同区域。例如，JSeparator类、JSplitPane类、JToolBar类等都是分隔符组件。

3. 表格组件

   表格组件是一种容器组件，它以矩阵形式显示其内容。表格可以容纳其他组件。例如，JTable类、JTreeTable类、JComboBox类等都是表格组件。

4. 滚动面板组件

   滚动面板组件是一种容器组件，它可以滚动其内容，显示部分内容或者隐藏剩余内容。例如，JScrollPane类、JViewport类、JTextPane类等都是滚动面板组件。

窗体类组件又分为三种类型：

1. 对话框窗体组件

   对话框窗体组件是一种特殊的容器组件，它提供一个窗口，用于完成某些特殊任务。例如，JOptionPane类、JColorChooser类、JFileChooser类等都是对话框窗体组件。

2. 菜单栏组件

   菜单栏组件是一种容器组件，它提供一个菜单栏，用于显示应用程序功能。例如，JMenuBar类、JMenu类、JMenuItem类等都是菜单栏组件。

3. 工具栏组件

   工具栏组件是一种容器组件，它提供一个工具栏，用于放置按钮、滑块、文字标签等。例如，JToolBar类、JPopupMenu类等都是工具栏组件。
