# 1.简介
  

作为一个项目管理人员，我经常需要与其他部门或个人进行沟通，比如对需求、设计方案、开发进度等情况进行协调。但是在完成这些工作之前，首先需要有一个好的项目会议组织形式，才能更有效地进行沟通。

因此，本文将重点介绍一下项目会议应该遵循的一些基本规则及建议，让大家能够轻松地掌握这类会议的组织方法，提高会议效率和质量，最大限度地完成任务。

2. 会议准备
## 2.1 会议前准备

在会议前，首先要做好以下事情：

1. 明确目标：会议的目的是什么？讨论哪些内容？
2. 安排时间：确定好每天的会议时间，同时对于每个阶段都要设定具体的截止时间，防止出现无法按时结束的问题。
3. 下载议题文档：会议主要讨论某个议题，所以最好提前下载好相关文档，根据文件内容进行必要的准备。
4. 提供手机号码或者个人联系方式：如果有紧急情况发生，就一定要跟主持人取得联系，同时也为了后续的记录保存。

## 2.2 会议流程

通常一个项目会议分为以下几个流程：

1. 项目汇报：项目的主要管理者、开发者、测试者和客户代表通过现场视频讲解项目背景和任务，并演示最新版本的功能。
2. 需求讨论：项目经理和开发团队围绕需求进行开放性讨论，了解需求方的意见和需求的变化，确保需求方了解产品的功能和特色。
3. 设计评审：产品设计师和开发人员就项目设计草案进行讨论，寻找共同的观点。
4. 开发进度展示：开发人员展示当前项目的开发进度，分享新上线功能和已完成功能。
5. 测试计划书编写：测试人员根据产品验收标准和测试用例编写测试计划书，向项目管理人员汇报测试方案。
6. 测试执行：测试人员按照测试计划书中的测试方案进行自动化测试和手动测试，汇总测试结果并及时反馈。
7. 上线发布：经过测试确认无问题后，项目经理宣布项目上线，并进行相应的运营推广。

## 2.3 会议讨论气氛

每次项目会议都会受到许多不同的声音，不同的观点。会议气氛的整体氛围应保持积极向上、欢迎与包容的态度，可以尝试以下方法来让会议气氛更加融洽：

1. 布置会议纪律：每个项目参与者都需要制定自己的会议纪律，譬如每个成员应事先阅读会议纪律，对于会议纪律违反者，需做出严厉批评和道歉。
2. 使用虚拟身份：使用虚拟身份来增强气氛，会议室内的声音不容易被隔绝，也能让更多的人加入到会议中来。
3. 分享经验教训：对于成功经验、失败经验、值得学习的经验等，都可以进行分享和讨论。
4. 演练精神：面对复杂或重要的议题，应多多采用演练的方式进行探讨，提升讨论效果。
5. 用语言来拉近距离：不管是给下属讲话还是回答问题，都需要注意语言表达的技巧。

## 2.4 会议记录

在项目会议期间，每个人都需要做好记录。有些关键信息比如需求变动、Bug修正、项目进展等，需要记录下来并留存，以便日后查询。

1. 每个人都需要按照要求准备笔记本电脑，并安装录音设备。
2. 会议过程中需把会议内容用文字记录，这样能够让自己和他人理解。
3. 在记录时尽量详细，既包括当事人的说话，也要添加自己的想法和评价。
4. 对每件事都留个固定的主题标签，这样方便搜索。
5. 如果遇到敏感事件，可以直接通知上级领导，上级领导可决定是否公开此事。
