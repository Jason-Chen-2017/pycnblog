# 1.简介
  

Fujitsu Cloud（富士通云）是日本的一家独立公司，创立于2009年，总部设在东京都，注册资本10亿日元。Fujitsu Cloud提供包括IAAS、PAAS、SAAS、SDN、SmartFabric、DevOps和AI、Edge Cloud等系列产品及服务。其中，云计算市场份额排名第五的富士通公司最初的产品定位主要面向政府机关和金融机构。但是后来业务范围扩展到更多应用领域，云计算市场份额目前位列全球第十四。其在国际上也拥有多个代理商、大客户、合作伙伴。另外，其产品属于集成化的解决方案，可以帮助企业降低运营成本、提升效率、节省IT资源。因此，现在很多初创企业或小型企业也可以考虑将自己的业务部署到富士通云上，作为“零浪费”的一种方式。
# 2.混合云架构
Fujitsu Cloud根据云平台服务客户的不同需求，实现了多种类型的混合云架构。根据客户需要，Fujitsu Cloud提供了各种类型的混合云产品，如公有云/私有云、联邦云、私有联邦云、托管云、托管私有云、边缘云、混合云。各类混合云架构之间的关系如下图所示：
其中，私有云是指部署在数据中心内部的基础设施，由用户自行管理；公有云是指部署在第三方数据中心托管的数据中心内的云服务，可供所有用户使用；联邦云是指部署在两个不同区域的私有云之间建立的安全连接，使得用户可以在不同的区域中访问同样的数据；私有联邦云是指利用一个公有的云服务（如公有云或其他混合云）作为中转站，通过网络将用户的数据分散到两个或多个私有云，并在两端建立安全连接；托管云是在云提供商托管的数据中心内提供的一组计算机资源，通常由云提供商直接管理；托管私有云是指由用户自己提供一套服务器和存储设备，然后委托云提供商作为服务提供给用户；边缘云部署在数据中心外部，采用边缘计算的方式进行计算任务的处理，其特点是能够在本地完成数据的分析和运算，减少网络带宽压力和延迟，提高云端效率；混合云是指多个私有云、公有云或其他混合云服务共同工作，形成的统一整体，具有极高的弹性和可用性。
# 3.Fujitsu Smart Fabric与云集群
富士通采用的是基于硬件资源虚拟化的方式构建云集群，即通过主动创建、调度云服务器、网络带宽等资源的方式，动态分配和释放，以达到优化资源使用、提升资源利用率、降低成本的目的。其架构如图所示：
Fujitsu Smart Fabric 是一个分布式计算框架，基于开源的 Ceph 组件，利用 CPU、GPU 和 SSD 等硬件资源的多样性、性能、成本优势，构建了一个弹性、高容错、可扩展的云计算框架，能够快速响应用户的请求。该框架支持动态管理节点的增减配额，以及负载均衡和网络流量调控功能，可以有效地避免单点故障、提高服务质量，同时满足了性能要求。

# 5.Fujitsu Edge Cloud与基于浏览器的智能网关
Fujitsu Edge Cloud 是富士通提供的一种基于浏览器的智能网关，其架构如图所示：
该产品可以为用户提供包括视频监控、远程控制、工业互联网、智能仪表、自动驾驶、物联网和无人机等在线服务。基于 Chromium 浏览器及其在浏览器上的各类扩展开发技术，Fujitsu Edge Cloud 提供了一套完整且可靠的端到端的端到端智能网关系统。智能网关系统的设计目标是针对终端设备的不同特性，根据这些特性配置不同的网关策略，实现对终端设备的安全、智能和可控。Fujitsu Edge Cloud 在边缘端部署智能设备，通过浏览器连接到用户的浏览器，获取用户的个人信息、浏览记录、电子邮件、短信等信息，对终端设备进行管理和控制。
# 6.结论
Fujitsu Cloud 提供了丰富的云服务，涵盖公有云、私有云、联邦云、私有联邦云、托管云、托管私有云、边缘云、混合云等。其中，富士通 Smart Fabric 为企业提供分布式的云计算平台，具备高速的计算能力、弹性的资源管理能力、安全的网络防护能力，并支持多种编程语言的应用。通过其云容器服务，企业可以使用开源的 Kubernetes 或 OpenShift 来快速部署容器集群，可以快速运行和管理基于 AI 的机器学习模型。而富士通 Smart Compute Platform 提供了适用于 GPU 的库和工具包，为数据科学家提供了方便快捷的机器学习开发环境。另外，富士通 Edge Cloud 为终端设备提供了基于浏览器的安全、智能和可控的网关服务，可以在边缘设备上实现智能应用。综合而言，Fujitsu Cloud 是一个综合性的云服务平台，既可以帮助企业降低运营成本、提升效率、节省 IT 资源，又可以促进数据的共享和整合，并提升终端设备的利用率。
