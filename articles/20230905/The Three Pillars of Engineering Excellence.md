# 1.简介
  

我们在生活中常常会看到工程学领域的很多人物，比如结构设计师、焊接师、钳工等。他们都认为自己拥有丰富的工程知识，能够设计出比同行更好的建筑和工程作品。工程学作为科学的一部分，其理论知识和实践方法对社会发展和个人发展都有着重要的影响。然而，许多工程学者并不善于将自己的经验技能传授给他人，原因很可能就是缺乏传播的能力。今天，我们把目光投向了工程学领域最有力的三条法宝——制度、流程和工具。通过学习和应用这些原则，可以帮助工程师实现更高水平的工作成果。

# 2.理论概念
## 2.1.制度(Organization)
工程学的制度，即建立一个自上而下的管理机构，是工程项目从计划、设计到施工一系列活动的规范化执行过程。工程学家用制度构建了一个透明的流程系统，以保证工程实施的效率和质量。制度的作用包括：

- 规范管理：制度是指导各方人员完成工作目标和行动准则。它规定了组织内成员之间的工作职责，以及工作日程安排，确定每项任务所需的资源和能力，并使不同的人员有共同遵守的规则约束。制度对工作流程和作业质量提出了更高要求。
- 投资决策支持：制度强调工程建设投资的资金分配，项目的开支预算、经费预算制定和管理，以及其他相关的财务事务。制度的落实结果反映了投资者和管理者对项目投资计划的认可度。
- 沟通协调：制度包含多个层级，它包括项目管理、业务领域及科研部门的沟通协调等内容。制度确保工程人员的信息交流和决策闭环。
- 服务质量保证：制度设置独立的服务质量保证团队，负责监督和维护工程的合理性和可靠性。制度设置相应的评估机制，以确保工程的服务水平符合合同要求。

## 2.2.流程(Process)
工程学的流程，是指工程项目或产品的生产、运输、加工等环节中出现的复杂的处理方法和机制。工程学家倾向于考虑如何改进流程的操作流程，将流程过程化、自动化、标准化，从而达到效率最大化、产品质量提升、降低成本、降低风险、防止劳损等目的。流程的作用包括：

- 提高工作效率：流程使工程师能够更好地管理和分配工作时间，减少人力资源消耗，提高生产效率。
- 提高工作质量：流程保证了作业的一致性、可追溯性和可重复性，并降低了因流程错误导致的损失。
- 降低成本：流程能够有效地整合各类设备，降低单位面积制造成本，缩短产品开发周期。
- 降低风险：流程可以有效地监控设备运行情况，提前发现潜在风险并采取措施进行应对，降低了项目的尾气污染和危险事件发生概率。
- 防止劳损：流程可以提升员工的工作态度和自律意识，使他们保持身体健康，防止伤害、损害，降低了单位生产成本。

## 2.3.工具(Tool)
工程学的工具，是指用来辅助工程工作的各种设备、材料、工具和手段。工程师根据自身的专长选择适当的工具，并充分利用工具对工程项目进行辅助。工程学家建议工具要满足工程师的需求和能力，能轻松完成工作任务，易于安装、使用和维修。工具的作用包括：

- 提高工作效率：工具的使用可以节省工程师的时间和精力，提高工作效率。如3D打印机、钳工工具、冲压机、焊接工具等。
- 提高工作质量：工具的选取能提高生产力，改善产品质量。例如使用精准的测量工具、高级的锻造工艺技术、精密的切削工具、坚固耐用的胶水等。
- 降低成本：工具的低廉价格、广泛的应用范围、完善的保修政策等优点都可以为工程企业节省成本。
- 提升生产力：工具的使用可以提升工程师的创新能力、解决问题能力和学习能力。如应用电子绘图工具，建立虚拟模型，分析产品性能等。
- 减少劳损：工具的安装、维护、改装、调试等操作都需要时间和专业知识。通过工具化的管理，可以让工程师集中精力工作，避免因工具的问题影响工作进度，提高工作质量。

# 3.核心算法原理及操作步骤
## 3.1.投影法(Project Plotting Technique)
　　投影法是工程学的一个重要分类方法，它定义了一种工程进程可视化的方法。它采用一组标准特征进行建模，将项目的整个生命周期分为多个阶段，每个阶段都有一个标准特征对应它，一个标准特征又对应着一个具体活动。通过这种方式，工程师可以使用项目的生命周期模型来检视它的过程，识别问题和机会，并对其进行优化调整。

　　1.第一步：项目生命周期模型的构建。首先，项目生命周期模型是一个阶段顺序列表，它说明了工程项目从计划、设计到施工的全过程，并且每个阶段都有一个特定的特征，每个特征还有一个相关的任务清单。然后，建立阶段顺序列表时，工程师应该考虑到：

- 是否有必要将所有阶段都列入模型？
- 每个阶段是否具有实际意义？
- 是否有必要将所有特征都列入模型？
- 是否有必要将所有任务都列入模型？
- 模型中的任何元素是否具有足够的详细程度？

　　2.第二步：定义特征。工程师需要定义每个阶段所对应的特征，它们一般来自于该阶段的具体任务。特征的名称和描述要言之有物，便于理解。一个典型的特征示例如下：

- 数据收集：收集所有涉及到的有关材料、人员、场地、设备的数据。
- 工程准备：制订检查方案，寻找供应商，协助第三方技术咨询公司进行现场评估。
- 资金筹划：确定资金来源和使用，确保资金能够用于后续建设工作。

　　3.第三步：定义任务。为了使特征能够完整表达项目生命周期的每个阶段，工程师需要将每个特征对应的任务也定义清楚。每个任务的名称和描述应该简洁易懂，且突出重要性。一个典型的任务示例如下：

- 材料采购：收集、研究、购买项目所需的各类材料。
- 人员安排：安排专业人员、工程师、设备及资金专家进行施工队伍建设和人员培训。
- 总承包商确认：对项目总承包商进行初步确认，确认相关合同的签署情况、合同金额、质量要求等。

　　4.第四步：创建项目报告模板。在项目生命周期模型的基础上，工程师需要制作项目报告模板，并提交给项目管理人员审核。项目报告模板应详细说明每个阶段所属的主题、任务、执行者、文档和时间表。通过项目报告模板，工程师可以将项目生命周期的不同阶段划分清楚，并且在项目过程中对项目状态进行监控。

　　5.第五步：建立项目管理计划。除了项目生命周期模型和项目报告模板外，工程师还需要制订一份详细的项目管理计划，作为项目进展的管控工具。项目管理计划应包括项目进度核算、管理规定、经营规划、资金规划、人员配备等内容。项目管理计划的建立将有助于项目管理人员掌握工程项目的各项经营规划、工作计划和业绩考核。

　　6.第六步：评审。工程师可以通过反复迭代和评审，逐步完善项目管理计划和项目报告模板，直到项目成功完成。在成功完成工程项目之前，工程师应继续提升自己的专业水平，提高对项目管理的理解和技能。

## 3.2.SWEBO Methodology
SWEBO（Swiss Watch and Clock Operating System）为瑞士钟表和时钟操作系统。其方法将软件开发过程分解为以下几个主要阶段：

- 需求分析：对客户需求进行详细研究，开发人员明确自己的产品需求。
- 设计阶段：根据客户需求和技术路线图进行产品设计和模块设计，保证产品满足客户的需求。
- 编码阶段：依据开发规范编写代码，降低软件开发难度。
- 测试阶段：测试代码的正确性、兼容性和鲁棒性，验证软件的正确运行。
- 维护阶段：更新和修正已有的软件，完善新功能。
- 发布阶段：软件通过最终的测试之后，正式向用户提供使用。

# 4.实例代码和解释说明
## 4.1.C++面向对象编程
```c++
class Animal {
  public:
    void speak() const; // animal类的speak函数声明

  private:
    std::string type_;
};

void Animal::speak() const{
   cout << "Animal is speaking" << endl; 
}

class Dog : public Animal {
  public:
    void speak() const override; // dog类的speak函数重载，继承 Animal 的speak函数
};

void Dog::speak() const{
   cout << "Dog is barking" << endl; 
}

int main(){
   Dog d{}; // 创建dog类的对象

   d.speak(); // 对象调用狗类的speak函数

    return 0;
}
```
这段代码定义了一个父类Animal，声明了一个`void speak()`函数。然后再定义了两个子类，分别是Dog和Cat。其中，Dog类的`speak()`函数继承自Animal类，但是使用了override关键字重新定义，也就是说，Dog的这个函数覆盖了Animal的这个函数。然后在main函数中，创建一个Dog类型的对象，并调用其speak函数。输出显示的是“Dog is barking”，因为这是Dog对象调用的Dog类的speak函数。
## 4.2.Python面向对象编程
```python
class Animal:
    def __init__(self):
        self.type_ = ""

    def speak(self): # animal类的speak函数
        print("Animal is speaking")


class Dog(Animal):
    def speak(self): # dog类的speak函数重载
        print("Dog is barking")


d = Dog()
d.speak() # 输出Dog is barking
```
这段代码定义了一个父类Animal，初始化了一个类变量`type_`；再定义了一个Dog子类，继承自Animal类，然后再覆写父类Animal的speak函数，并增加了一个新的dog类的speak函数。在最后，创建一个Dog类的对象，并调用其speak函数。输出显示的是“Dog is barking”。
