# 2.核心概念与联系

## 2.1 记录类型

记录类型是一种新的数据类型，它允许程序员使用结构化的方式表示和处理数据。记录类型是一种结构体，它可以包含多个字段，每个字段都有一个名字和一个类型。

例如，我们可以定义一个表示人的记录类型，如下所示：

```java
public record Person(String name, int age, String gender) {}
```

在这个例子中，我们定义了一个名为`Person`的记录类型，它包含三个字段：`name`、`age`和`gender`。这些字段都有一个类型，分别是`String`、`int`和`String`。

记录类型有以下特点：

1. 它们是不可变的，这意味着一旦创建，它们的值就不能被修改。
2. 它们可以有构造函数，这意味着程序员可以为其指定值。
3. 它们可以有方法，这意味着程序员可以为其添加行为。

## 2.2 模块系统

模块系统是一种新的代码组织方式，它允许程序员更好地组织和管理代码。模块系统将代码划分为多个模块，每个模块都是一个独立的代码块，它可以与其他模块相互引用和依赖。

模块系统的主要特点是：

1. 它们是独立的，这意味着每个模块都可以单独编译和测试。
2. 它们可以互相引用，这意味着程序员可以在一个模块中使用另一个模块提供的功能。
3. 它们可以有依赖关系，这意味着程序员可以指定一个模块依赖于另一个模块。

# 4.具体代码实例和详细解释说明

## 4.1 记录类型的代码实例

以下是一个使用记录类型的代码实例：

```java
public record Person(String name, int age, String gender) {}

public class Main {
    public static void main(String[] args) {
        Person person = new Person("John", 30, "Male");
        System.out.println(person.name); // John
        System.out.println(person.age); // 30
        System.out.println(person.gender); // Male
    }
}
```

在这个例子中，我们定义了一个名为`Person`的记录类型，它包含三个字段：`name`、`age`和`gender`。然后我们创建了一个`Person`实例，并访问了其字段。

## 4.2 模块系统的代码实例

以下是一个使用模块系统的代码实例：

```java
module com.example {
    requires java.base;
    exports com.example;
}

package com.example {
    public class Main {
        public static void main(String[] args) {
            System.out.println("Hello, World!");
        }
    }
}
```

在这个例子中，我们定义了一个名为`com.example`的模块，它依赖于`java.base`模块。然后我们导出了`com.example`包，并定义了一个`Main`类。
