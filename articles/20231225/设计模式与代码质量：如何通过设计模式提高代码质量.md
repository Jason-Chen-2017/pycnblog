## 4.1 单例模式
单例模式是一种常见的设计模式，它确保一个类只有一个实例，并提供一个全局访问点。

### 4.1.1 代码实例
```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super(Singleton, cls).__new__(cls, *args, **kwargs)
        return cls._instance
```
### 4.1.2 解释说明
在上面的代码实例中，我们定义了一个`Singleton`类，它使用了`__new__`方法来实现单例模式。`__new__`方法是一个类方法，它在创建新实例时被调用。在`__new__`方法中，我们检查了`_instance`属性是否已经存在，如果不存在，则创建一个新实例并将其存储在`_instance`属性中。这样，我们可以确保`Singleton`类只有一个实例，并且可以通过`Singleton.instance`访问它。

单例模式的优势是它可以确保全局访问点，避免多次创建同一个实例，从而提高代码的效率。

## 4.2 工厂方法模式
工厂方法模式是一种创建型设计模式，它定义了一个用于创建对象的接口，但让子类决定实例化哪个类。

### 4.2.1 代码实例
```python
class Animal:
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

class AnimalFactory:
    @staticmethod
    def create_animal(animal_type):
        if animal_type == "Dog":
            return Dog()
        elif animal_type == "Cat":
            return Cat()
        else:
            raise ValueError("Invalid animal type")
```
### 4.2.2 解释说明
在上面的代码实例中，我们定义了一个`Animal`类和两个子类`Dog`和`Cat`。这些类实现了`speak`方法，用于返回各自的声音。然后，我们定义了一个`AnimalFactory`类，它包含一个`create_animal`静态方法，用于根据输入的`animal_type`创建对应的实例。通过使用工厂方法模式，我们可以在不修改`AnimalFactory`类的情况下添加新的动物类，从而实现代码的可扩展性。

工厂方法模式的优势是它可以将对象创建的过程从客户端代码中分离，使代码更加模块化和可维护。
