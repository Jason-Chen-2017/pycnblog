# 1.简介
  


## 1.1.什么是数据库?

数据库（DataBase，DB）是一个长期存储、组织、管理、安全访问和扩充数据集合的计算机系统。它是用来存储、检索和管理数据的仓库。数据库由数据结构和相关的数据控制语言组成，用于存储、处理和提取信息。目前，关系型数据库、非关系型数据库、对象数据库等多种类型数据库被广泛应用于各种行业、各个角落。

## 1.2.数据库的作用？

1、收集、组织、存储数据

   数据是大部分应用系统的基础，而在大数据时代，数据量将越来越庞大，对于数据的收集、组织、存储、检索和分析将成为一个复杂而严峻的任务。数据库可以实现数据的集中化、整合化和共享化，有效地降低成本和提高效率。

2、提供数据支持

   数据库是信息资源库，用来支持业务应用、数据分析、决策支持等功能。数据库可使得各种应用程序能够容易地存取、交换、共享数据。因此，数据库系统的设计与开发工作对企业的各项业务都有着至关重要的意义。

3、改进业务流程

   数据库能够加速公司内部及外部的协同工作，改善信息流通方式，提升工作效率，同时也减少了管理人员的工作量。同时，数据库还能够有效地处理海量数据，促进业务的创新与变革。

## 1.3.数据库分类

按照数据库的类型、体系结构、应用环境、运行时状态和实现模型，数据库可分为以下几类：

1. 关系型数据库

   是建立在关系模型上的数据库。其特点是通过三维表结构来组织数据，并且每个关系型数据库都定义了一种数据模型，即相互关联的表之间的关系或联系。关系型数据库包括Oracle、MySQL、SQL Server、PostgreSQL、SQLite等。

2. 非关系型数据库

   不依赖于关系模型的数据模型，没有固定的 schema 或 table 模型，一般采用键值对存储。这类数据库包括 NoSQL 数据库、列族数据库、图形数据库等。

3. 对象数据库

   将数据模型抽象成对象的形式。该类型的数据库主要用于管理大规模、复杂的数据集。对象数据库通常采用面向对象的方式表示和处理数据。典型的对象数据库包括 Java 数据库编程接口 (JDBC) 的 Hibernate 和 EJB。

4. XML数据库

   支持 XML 格式的数据存储。其特点是数据以 XML 文件的形式存在，可以灵活地扩展和修改数据。典型的 XML 数据库包括 Apache CouchDB、eXist-db 和 BaseX。

5. 分布式数据库

   以分布式网络的形式存储数据。分布式数据库可以跨越多个网络节点，具有更好的容错性和可伸缩性。典型的分布式数据库包括 Hadoop、MongoDB、Redis 等。

6. 大数据数据库

   提供处理海量数据、分析海量数据的方法。这类数据库面临着更高的计算、存储、传输需求。大数据数据库通常采用列式存储、压缩、并行计算等手段优化性能。典型的大数据数据库包括 Apache Cassandra、HBase、Spark SQL 等。

7. 行动数据库

   以嵌入式设备的形式运行，为移动应用提供数据支持。行动数据库适用于具有资源受限、功耗要求较高的场景。行动数据库不仅需要兼顾性能、价格和成本等方面考虑因素，而且还需要保证快速响应、低延迟的特性。典型的行动数据库包括 Android SQLite。

总之，数据库的种类繁多且变化多端，但它们都提供了一套完整的技术解决方案，可满足不同类型的应用环境。
