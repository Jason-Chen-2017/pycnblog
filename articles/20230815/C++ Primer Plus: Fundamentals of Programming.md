# 1.简介
  

## 1.1 为什么要学习C++？
### 1.1.1 1970年代的一些历史回忆
在1970年代初期，程序员们觉得很难从头开始编写一个程序。他们都是一名计算机系的学生，当时的计算机技术还不成熟，而且学习曲线陡峭。因此，编程教材中往往写得非常简单、易懂。但是由于技术的进步，至今仍然有很多优秀的程序员都放弃了学习古老的编程语言，转而选择了更现代化的语言如C、Java或Python等。因为学习这些新型语言需要用到各种新的语法知识、库函数和开发工具，所以学完这些语言才能真正地应用于实际工作。然而，对于那些刚接触计算机编程的人来说，学习现代化的编程语言还是一个艰难的过程。相反，程序员们常常会倾向于学习最初的、比较简单但功能完整的语言，如BASIC或FORTRAN。对于初学者来说，这一学习方式无疑增加了难度和时间成本。
### 1.1.2 20世纪90年代的变化
随着硬件性能的提高和互联网的普及，科技界急需一种能够快速开发应用软件的方法。为了满足这一需求，编程语言被广泛采用。那时候，C语言已经成为主流语言之一，它具有结构清晰、易读性强、易移植性好、支持多种平台特性、可嵌入其他编程语言等特点。1990年代，微软推出了Visual Studio IDE，使得C语言开发环境得以统一和简化。到了2000年代末期，C++迎来了它的黄金时期。C++由Bjarne Stroustrup和Donald Knuth合著，是C语言的一个超集。C++具有多态性、面向对象性、异常处理机制、静态类型检查、运行速度快、安全性高、跨平台特性等独有的特性。可以说，C++是当时最好的编程语言，也是最常用的编程语言。
## 1.2 本书的结构与目标读者
C++ Primer Plus的目的是向读者介绍C++语言及其编程技术。作者希望通过详细的教学案例和有针对性的问题讲解，让读者在短期内掌握C++的基础知识、熟练运用C++进行编程。本书共分八章，每章包括1-3节简短的知识介绍、3-4节较长的技术分析、1-2节练习题。读者在阅读时可以根据自己的理解水平，选择适合自己需要的内容。本书适合有一定C/C++编程经验，并且对编程有浓厚兴趣、渴望进阶的学生学习。由于本书面向的是C++编程语言，对熟悉其它编程语言的读者也有一定的参考价值。除此之外，本书也欢迎各级机关、培训机构、系统工程师、项目经理、技术顾问等从事技术研发、技术管理和技术咨询工作的技术人员阅读。
## 1.3 如何阅读本书
本书建议从第1章“C++简介”开始阅读。第1章介绍C++的基本概念和基本用法。第2章介绍C++的数据结构，包括数组、链表、队列和栈等。第3章介绍C++中的类和对象。第4章介绍面向对象编程的基本概念和实践方法。第5章介绍异常处理和调试技术。第6章介绍字符串和文本处理技术。第7章介绍文件输入输出技术。最后一章介绍总结、展望和参考资料。每章后都有练习题供读者自测。每章前面的思维导图（MMD）可以作为提示读者了解该章的主要内容。另外，本书提供完整的源码和示例，读者可以在自己电脑上实践和修改代码。
