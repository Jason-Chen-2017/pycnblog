# 1.简介
  

服务治理与运维管理（Service Governance and Operations Management）是指对企业IT服务进行持续优化、监控、评价、控制的过程。它是保持IT服务质量高效运行的关键环节。实现服务治理与运维管理可以有效地提升IT服务的可用性、稳定性和可靠性，保障业务连续性和客户满意度。服务治理与运维管理还可以从以下几个方面促进企业经济发展和社会发展：
* 提升企业竞争力：服务治理与运维管理是市场营销活动中的重要组成部分，能够影响企业的竞争力，促进企业在市场上的布局与竞争。
* 提升社会效益：通过提供优质的产品和服务，企业能够创造更多就业岗位，招聘的人才也会越来越多；通过优化营销渠道和方式，企业可提升社会满意度和经济效益；通过降低企业拥堵成本，可以释放更多资源投入到有利于环境改善的领域。
* 推动社会转型：科技革命席卷全球，物流、电子支付、医疗卫生、教育等各行各业都已经进入了数字化时代。服务治理与运维管理作为推动社会转型的一部分，将成为企业解决不同行业数字化带来的新问题的必要工具。

为了有效地实现服务治理与运维管理，企业需要做到以下几点：
* 服务管理：确定IT服务的目标、定义服务范围和交付条件、审查并更新服务计划、监控服务运行状态、分析问题及其原因、定期进行风险分析和安全维护。
* 团队建设：建立完善的管理机制、职责分工和技能培训制度，确保各部门人员能够专注于服务的提供上。
* 沟通协调：了解企业内部信息、客户需求、合作伙伴需求和政策法规，能够有效地进行沟通协调，保证服务的顺畅运行。
* 资源整合：采用精益创业的方法，积极探索新技术，激活资源的创新能力。充分利用云计算、大数据、人工智能等新兴技术，打造更加智能化、个性化的IT服务体系。
* 生命周期管理：持续改善服务质量和服务标准，通过质量保证、变更管理、工程质量管理、成本管理等方式，来减少服务的停机时间，提升企业的绩效。
服务治理与运维管理是管理和运营企业IT系统最基础也是最重要的环节，它的成功离不开高度专业化的技术支持和高度组织化的管理制度。因此，企业的IT部门需要有针对性地进行服务治理与运维管理，确保公司的IT系统始终处于最佳状态，提供优质的客户服务，最大限度地提升公司的竞争力和社会效益。

# 2.基本概念术语说明
## 2.1.服务治理术语与基本原则
### 服务管理（Service management）
服务管理（Service management）是指企业如何通过一系列措施，使企业的IT服务真正为用户提供价值、满足用户需求、达到企业目标，而不只是为了赚钱而忽略或轻视用户需求。它主要关注服务交付结果（包括服务水平、服务质量、服务时效性、服务范围）、服务成本的控制和优化、资源分配的合理化、人力资源管理、IT流程的优化及服务管理政策的制定。服务管理可以通过结构化、文档化的方式进行，其主要职能有：
* 指导方向：明确企业的服务战略，确定服务管理的目标和衡量标准，制定服务方向和目标，协调各部门的工作。
* 执行服务：指导企业实施服务管理政策，采取有效的管理手段，全面准确地收集、分析和处理信息，为企业提供优质的服务。
* 组织领导：鼓励和支持企业的服务经理、服务副总裁以及相关人员组成服务管理团队，有效整合人力、财力、信息资源，确保服务管理的目标、策略、方案和执行符合企业的战略目标。
### 四大支柱角色（Four Pillars of Service Management）
服务管理模式通常由四大支柱角色构成：

1. 客户关系管理（Customer Relationship Management，CRM）
顾客关系管理，即客服团队及相关的各种管理制度，主要用于跟踪客户需求、维护良好的客户关系，确保客户满意度与忠诚度。

2. 技术支持（Technical Support）
技术支持是指企业向客户提供专业技术人员的培训、咨询、解决方案，以及提供相关的设备和软件。它通过保证客户能够快速、有效地解决技术问题来提升客户满意度，并帮助企业提高竞争力。

3. 资源管理（Resource Management）
资源管理是指建立起有效的资源共享机制，并能充分发挥各个资源之间的协同作用。资源管理的目的是确保为用户提供所需的服务，同时应对资源的不足、挤占、过度、冲突等问题，确保整个IT系统的稳定运作。

4. 服务管理（Service Management）
服务管理，即企业管理者将IT服务转换成销售、支持、维护等形式，并通过一个系统化的平台来实施这些服务。服务管理的目的就是让客户满意、取得最高的服务质量和客户满意度。

### ITIL 4C 模型 （ITIL 4C Model）
ITIL 4C 模型（Information Technology Infrastructure Library 4 Critical Components）用于描述组织管理的信息技术基础设施中存在的四个关键支柱角色（Critical Component）。模型包括4个主要的工作职能，分别是战略计划（Strategic Planning），组织设计（Organizational Design），流程改进（Process Improvement），文化转型（Culture Change）。该模型适用于任何规模的组织，可帮助组织定义、执行、管理和改进其信息技术基础设施。


## 2.2.运维管理术语与基本原则
运维管理（Operation and Maintenance Management，简称 O&M）是指企业IT资产的日常管理，包括硬件、软件和网络设备的安装部署、运行维护、故障排除、性能分析、功能扩展、安全管理等。它不仅涉及硬件设备的配置与维护，而且还要考虑服务器的性能调整、操作系统的升级、数据库的备份恢复、存储容量规划、网络设备的维护等。与服务治理相比，运维管理更多强调事故的避免、故障的处理、问题的识别、问题的预测、问题的跟踪、优化的决策和过程的改进等。运维管理是IT系统的生命周期管理的一部分，是确定是否进行更改、将更改部署到生产环境中的关键一步。

### 管理层级
运维管理分为三层管理：横向管理（Infrastructure Management）、纵向管理（Application Management）、终端管理（End User Management）。
#### 横向管理（Infrastructure Management）
横向管理又称基础设施管理，主要负责IT资产基础设施的设计、部署、管理、运维。主要任务如下：
* 基础设施规划与设计
* 硬件设备选型与配置
* 操作系统选择与配置
* 软件包的集成与配置
* 网络设备规划与设计
* 数据中心布局设计与规划
* 测试环境与日常运维
#### 纵向管理（Application Management）
纵向管理又称应用管理，主要负责应用的发布、更新、运行与管理。主要任务如下：
* 应用开发与测试
* 应用的发布与部署
* 应用的性能优化
* 故障的诊断与定位
* 应用的生命周期管理
#### 终端管理（End User Management）
终端管理又称最终用户管理，主要负责最终用户的配置、使用、管理及服务。主要任务如下：
* 用户认证与授权
* 使用指南与帮助文件
* 技术支持与服务
* 数据备份与恢复
* 设备维护与管理
* 用户满意度调查与分析
* 品牌形象设计与推广

### O&M 的特点
O&M 是IT资产日常管理，属于IT服务的范畴，与服务治理完全不同。O&M 在各个级别、各个领域都会存在，但其主要目的都是为了保证IT资产的持续运作，以提升业务的成功率。所以，O&M 可以理解为一种管理活动，也可以说是IT服务的延伸。

O&M 有以下几个特征：
1. 以用户为中心：O&M 以用户为中心，并且把用户看作是 IT 资产的最终用户。
2. 以支持为目的：O&M 以支持为目的，以提升客户满意度、降低支持成本为目标。
3. 从细节到整体：O&M 从细节到整体，把复杂的 IT 系统分解成易于管理的组件，从而提升 O&M 的效率。
4. 对齐业务：O&M 对齐业务，关注并管理业务关键服务，从而提升业务的利润和利益。
5. 依据业界标准：O&M 根据各自业务和 IT 环境的要求，采用的方法和技术需要与其他行业保持一致。
