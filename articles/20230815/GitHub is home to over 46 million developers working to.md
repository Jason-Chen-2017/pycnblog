# 1.简介
  

GitHub是一个面向开源及私有软件项目的托管平台，由GitHub公司(WWW.GITHUB.COM)开发维护，最初于2008年4月10日上线。作为目前最大的Web版本代码托管服务提供者，拥有超过460万个注册用户，其中95%以上的用户都是在全球范围内从事计算机编程工作的个人或组织。它的产品帮助用户完成从代码编写到项目管理的一系列流程，让开发者和科研人员能够方便快捷地进行协作开发，解决各种各样的问题。截止目前，GitHub已经被全世界超过三分之一的公司采用，并在全球范围内成为开源及私有软件项目的主流的代码托管平台。
# 2.基本概念术语
## 2.1.注册账号与登录认证
GitHub网站提供了两种方式来注册账号、登录认证：用户名密码注册和OAuth第三方应用注册。
### 用户名密码注册
当用户第一次访问GitHub时，需要输入自己的用户名、邮箱地址、密码等信息，系统会自动生成一个唯一标识符（称为账户ID）和个人主页URL。然后，用户可以在该页面上创建新的仓库或者参与其他人的项目。
### OAuth第三方应用注册
如果要允许第三方应用直接登录GitHub，可以先申请到GitHub API的授权码，然后配置好回调地址，就可以使用OAuth协议第三方应用登录GitHub账号了。这里需要注意的是，注册第三方应用前需要先向GitHub公司提交应用程序的基本信息。
## 2.2.Git相关术语
Git是目前使用最广泛的分布式版本控制系统，是一个开源的版本控制软件。我们熟悉的SVN、Mercurial等都属于集中式版本控制系统，而Git完全不同。Git的特点包括以下几点：
* 快速：速度快，Git的内部数据结构只存储指向实际文件的SHA-1值，所以克隆一个仓库或者创建一个新仓库都非常快。对同一个文件多次提交可以节省磁盘空间，因为每一次提交仅保存差异。
* 分布式：每个仓库都是独立完整的，互不干扰。
* 轻量级：占用的空间小，克隆一个仓库通常仅需几秒钟，而且很少消耗额外的内存资源。
* 可靠性：Git保证数据的完整性，可以通过引用历史记录来恢复数据。
* Git工作流程：推送本地仓库到远程仓库->拉取远程仓库到本地仓库->合并冲突->提交本地修改到远程仓库->推送本地修改到远程仓库。
