# 2.基本概念术语说明
## 2.1什么是大数据？
大数据是指，具有以下特征的数据集合，它的规模超过了目前常用的关系型数据库或者海量数据能够存储的范围。这些数据包括文本、图像、视频、音频、网络活动数据、交易记录等。通过大数据分析技术，就可以从中提取有价值的信息，比如人群画像、个性化推荐、异常检测、风险预测、金融投资方向判断等等。
## 2.2什么是实时数据采集？
实时数据采集是指，按照数据产生的时间顺序收集、存储和传输数据，不延迟。根据这个特性，实时数据采集通常使用消息队列作为中间件。消息队列中的数据可以被消费者读取并分析，也可以用于后续的数据处理。
## 2.3什么是流式数据采集？
流式数据采集是指，按照数据产生的速度、大小或者数量等指标，实时地收集、存储和传输数据。流式数据采集可以满足实时计算需求，如即时监控和边缘计算。
## 2.4什么是数据仓库？
数据仓库是用来集成来自多个源头的数据，进行汇总整理、规范化、保护、报表生成、分析展示和应用的系统。数据仓库的构建需要ETL（抽取-转换-加载）过程，将不同来源的、异构数据进行统一、结构化、准确地存储、维护、查询。
## 2.5什么是Hadoop？
Apache Hadoop 是Apache基金会的一个开源的分布式计算平台，它提供高容错性、高可靠性的数据存储能力。Hadoop 将海量的数据分布存储在多个节点上，然后通过 MapReduce 框架进行分布式计算，并最终生成结果。
## 2.6什么是MapReduce？
MapReduce 是一种编程模型和计算框架，它提供了一种简单的方法来并行处理大量的数据，并且不需要复杂的底层编程。它分为两个阶段：Map 和 Reduce。其中，Map 阶段负责处理输入数据并生成中间数据，Reduce 阶段则负责处理中间数据并生成最终结果。
## 2.7什么是Spark Streaming？
Spark Streaming 是 Apache Spark 提供的实时流计算功能。它基于微批次时间窗口和 DStream（Data Stream 的缩写），使开发人员能够在任意时刻、任意地点处理大量数据。Spark Streaming 支持实时流数据处理和离线批量数据处理，同时支持 Scala、Java、Python、R 等多种语言。
## 2.8什么是Flume？
Flume 是 Cloudera 提供的开源分布式日志采集器，它允许你方便地采集、聚合和传输大量数据。Flume 通过对日志文件进行滚动解析、压缩、加密等操作，使其适应 HDFS、Kafka 或其他数据集成系统。
## 2.9什么是Kafka？
Apache Kafka 是 LinkedIn 在 2011 年创建的开源项目，它是一个分布式、高吞吐率、低延迟的分布式发布/订阅消息系统。它可以处理消费者机器的不可靠网络、temporary failure 和 data loss 。Kafka 主要由 Producer、Broker 和 Consumer 三部分组成。
## 2.10什么是Storm？
Storm 是 Distributed Realtime Computation System 的缩写，是一个开源的分布式计算系统。Storm 可以实时处理海量的数据流，并且提供了强大的事件驱动机制，可以实现实时的计算。Storm 运行在集群之上，通过保持超高的数据处理能力和高容错性，促进实时数据分析的应用场景。
## 2.11什么是Redis？
Redis 是开源的内存数据库，它支持高速读写，是最常用的 NoSQL 数据库。Redis 提供了键值对存储、字符串、列表、散列、集合和有序集合。它也支持发布/订阅模式、事务和 Lua 脚本。
## 2.12什么是Zookeeper？
Apache Zookeeper 是 Apache Hadoop 项目中的一个子项目，它是一个开源的分布式协调服务。它为分布式环境中的应用程序提供高可用性，协调分布式进程之间的相互通信，并且实现故障转移功能。Zookeeper 分布式文件系统、配置管理、命名注册、分布式锁等都依赖于 Zookeeper 来实现。
