# 1.背景介绍


对于一个组织来说，总是会有各种各样的数据需要存储、处理、分析和呈现。不同部门或业务之间往往有重复且无效的工作，造成了数据不一致性问题。为了解决这一问题，业务、IT和数据科学团队可以考虑建立数据中台。数据中台是一个集成、统一、协同、共享的多维数据存储平台，它能够有效地解决这些数据存储、管理、使用和分析的痛点。数据中台的产生主要分为以下四个原因：

1. 数据量增长: 大数据、云计算等新兴技术使得数据量越来越大，单机硬件无法满足快速增长的需要。

2. 不同部门或业务之间存在数据孤岛: 每个业务部门都要自行搭建数据仓库，独立完成数据的采集、清洗、加工等工作，导致数据价值低下、数据共享困难。

3. 信息孤岛: IT部门作为主力部门，对所有业务部门的信息都掌握在自己手里，但不能提供真正的价值。

4. 技术及工具孤岛: 由于业务各方差异过大、复杂度高、底层技术要求高，IT部门很难提供统一的技术支持。

那么数据中台到底是什么呢？数据中台由多个不同功能的数据库组成，数据通过统一的接口从数据源（比如企业数据或第三方数据）导入到数据中台。然后通过一系列数据处理过程，对数据进行清洗、转换、规范化、归一化、关联、过滤等操作，最后按照要求展现给用户。数据中台的意义在于降低数据壁垒，统一、标准化、提升数据质量，提升数据价值。其架构设计也应符合分而治之的原则。如下图所示：
数据中台按照一定层级结构分为数据源、数据仓库、数据湖、数据服务三层。

数据源层: 数据源是指企业或者其他组织产生的原始数据。数据源可以直接来源于互联网、应用服务器、工控系统等设备。数据源包括日志文件、IoT传感器数据、应用程序数据、天气数据等。

数据仓库层: 数据仓库用于存储海量数据，是数据中台最重要也是最基础的层。数据仓库通常会经历ETL（抽取-传输-加载）过程，将数据转移到数据湖。数据仓库层包括企业数据仓库、数据湖（数据中台的关键组件）。企业数据仓库维护着公司的核心业务数据，拥有完整的数据生命周期管理体系；数据湖负责存放所有非核心业务数据，具备数据共享能力和可扩展性。

数据湖层: 数据湖是一个能够持续存储和共享大量数据资源的大型分布式存储系统。它是数据中台的第二个重要层，其优点是能够按需访问数据，数据共享容易；缺点是架构复杂、成本高、数据可用性差。数据湖层主要包括Hadoop生态圈、开源商用组件等。

数据服务层: 数据服务层提供对外数据服务，包含数据查询、分析、报表等，为上层业务部门提供各种数据服务。数据服务层的核心是数据API，供其他业务部门调用。
