# 1.背景介绍


## 一、什么是数据库？
> 数据库（Database）指的是长期存储在计算机内、可被多用户共享的数据集合。一个数据库通常是一个文件结构或者关系型数据库管理系统中保存着组织化的数据，这些数据用于存放各种信息，并可供查询、修改、添加或删除。简单地说，数据库就是存放数据的地方。
数据库分为服务器端数据库和客户端数据库。按照存储位置，可以分为以下几类：

 - 基于磁盘的数据库：例如硬盘上安装的关系型数据库MySQL、Oracle等；
 - 基于内存的数据库：例如Oracle数据库中的内存表、Redis等；
 - 分布式数据库：包括NoSQL数据库和分布式文件系统HDFS。

## 二、为什么需要用数据库？
很多人或多或少都听说过，数据库很重要，但为什么我们需要它呢？原因主要有以下几个方面：

1. 数据的安全性保障
 - 大量数据涉及到隐私、个人信息、金融交易、社会情感等敏感数据，将这些数据存在硬盘上可能会导致严重的泄露隐私或安全事故。因此，引入数据库之后，我们可以把一些关键性数据存放在数据库中，避免发生泄露隐私的风险。
2. 数据的一致性和完整性
 - 当多个人或系统同时操作同一份数据时，由于各自的操作可能不精确，造成数据的不一致。通过引入数据库，我们可以保证数据之间的一致性和完整性。
3. 数据的可用性
 - 数据库降低了数据访问的延迟，提高了数据处理的效率。另外，数据库还能提供备份、恢复功能，使得数据更加可靠。
4. 数据的扩展性
 - 有些时候，数据量的增加带来了数据存储的压力。数据库提供了扩展性，使得数据容量可以根据业务的增长自动扩充。
5. 历史数据的存储和检索
 - 通过数据库，我们可以对历史数据进行有效的归档和查询，这对于分析数据和对比不同时间点的历史数据非常有帮助。

## 三、数据库的种类
数据库按其结构、组织形式、管理方式、应用领域、处理性能、成本、适用场景等因素可分为以下几类：

 - 文件数据库：这种数据库将所有数据都存储在文件中，如SQL Server、Access等，文件越大，查询速度就越慢。
 - 消除型数据库：这种数据库将数据存储在磁盘上，减轻主存的负担，但仍需考虑硬件性能和网络延迟的问题。如MongoDB、Couchbase等。
 - 关系型数据库：这种数据库将数据组织在表格中，每张表格对应于一个数据库对象，关系数据库可以方便地支持复杂的查询功能。如MySQL、PostgreSQL、Oracle等。
 - NoSQL数据库：这是一种非关系型数据库，它将数据以文档、图形或键值对的形式存储在数据库中，一般采用分布式集群的方式扩展性能。如MongoDB、Redis等。
 - 分布式数据库：这种数据库将数据分布在不同的机器或节点上，利用局域网或互联网连接，每个节点上运行数据库服务进程，以提升整体的处理能力和可靠性。如HBase、Hadoop等。
 
根据以上分类，可以将数据库分为服务器端数据库和客户端数据库两大类。
 
## 四、关系型数据库（Relational Database）简介
关系型数据库由表格组成，每张表格对应于一个数据库对象，这种数据库以行和列的形式存储数据，因此它强调数据的结构化，也要求每个字段必须有确切的定义，并遵循一定的规则。

1. 属性（Attribute）：数据项的名称，描述数据项的内容。比如，电子邮件地址、用户名、密码等属性可以用来建立用户表。
2. 主键（Primary Key）：唯一标识一条记录的属性或组合属性。主键总是唯一的，不能重复，且不能为空。比如，身份证号码可以作为主键建立用户表。
3. 外键（Foreign Key）：用来实现两个表之间关系的属性，该属性的值是另一表的主键。比如，产品表的外键指向价格表的主键，以此来维护数据的一致性。
4. 实体（Entity）：具有共性的属性集合。实体表示真实世界中某个对象的属性集合。比如，人员信息、订单信息等。
5. 关系（Relationship）：实体间的联系。关系表示实体之间的联系和依赖关系。比如，一个人的销售订单是另一个人的购买行为，所以这两个实体之间存在“属于”关系。

# 2.核心概念与联系
## 一、基本概念
1. 数据库（database）:一个存储在计算机内，可被多用户共享的数据集合。
2. 数据（data）:数据的原始状态，数字、文字、图像、声音、视频或其他类型。
3. 数据库管理系统（DBMS）:用来管理关系型数据库的软件。
4. 数据模型（Data Model）:是关于数据如何在数据库中组织、存储、操纵、管理的一系列规则。
5. 数据字典（Data Dictionary）:是一个包含关于数据库 schema 的描述的信息。
6. 数据表（table）:用来存储数据的二维平面表格。
7. 记录（record）:在表中，一条数据记录。
8. 字段（field）:在记录中，数据元素。
9. 主键（primary key）:每张表只能有一个主键。它是一个唯一标识一条记录的属性或组合属性。主键总是唯一的，不能重复，且不能为空。
10. 外键（foreign key）:用来实现两个表之间关系的属性，该属性的值是另一表的主键。
11. 索引（index）:一种特殊的查询数据的方式，它把随机读取变为顺序读取。它的作用是在一个包含许多记录的数据表里快速找到一个或几个特定的记录，从而缩短查找时间。
12. 事务（transaction）:一组sql语句组成的集合，并满足ACID特性（Atomicity、Consistency、Isolation、Durability）。事务是一组不可分割的工作单元，要么都执行，要么都不执行。
13. ACID特性:原子性（Atomicity），一致性（Consistency），隔离性（Isolation），持久性（Durability）。

## 二、关系型数据库的相关术语
1. 数据库实例：数据库的具体实现，例如MySQL的实例。
2. 数据库服务器：提供给数据库客户使用的计算机，它可以同时承载多个数据库实例。
3. 数据库引擎：用来存储、检索和更新数据库数据的核心组件。
4. 关系模型：以关系表的形式存储和管理数据的模型。
5. 关系表：具有二维数据结构的矩形表。
6. 关系数据类型：用于存储关系数据的一种数据类型，例如整数、小数、日期、字符串等。
7. SQL语言：结构化查询语言，用于访问和操作数据库。
8. 视图：虚拟的表，是现实世界的子集，它以某种方式从实际的关系表中抽象出来的虚构表。
