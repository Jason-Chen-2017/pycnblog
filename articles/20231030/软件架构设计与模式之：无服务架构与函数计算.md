# 1.背景介绍


## 一、什么是函数计算？
函数计算（Function Compute，FC）是阿里云推出的基于事件驱动架构的服务，可以帮助用户在无服务器架构中运行应用程序。它是一种按需计算平台，能够根据需求动态伸缩资源、按量计费、提供高可用性。用户只需要关注自己的业务逻辑，不需要管理和运维底层的服务器集群和基础设施，甚至不用购买虚拟机或容器即可部署应用。函数计算利用事件驱动架构，通过发布-订阅机制来接收事件并触发对应的函数执行。

目前主流的 FC 服务包括 FaaS、事件托管、函数工作流等产品形态，其中 FaaS 是最典型的无服务器计算产品，其他形态则提供了更高级的计算功能。

## 二、为什么要使用函数计算？
函数计算在无服务器架构下，通过发布-订阅的事件驱动架构，实现按需计算。它的优点主要体现在：

1. 按需计算：函数计算可以按使用量或请求次数计费，当用户无需运行某些函数时，也可以暂停运行以节省开销；
2. 灵活伸缩：函数计算通过弹性伸缩机制，随着业务的增长和消亡，能够自动扩容或缩容，满足用户需求；
3. 实时响应：函数计算支持秒级响应时间，能及时响应用户请求，降低了延迟和故障率；
4. 可靠性保证：函数计算提供多副本冗余存储，确保数据安全、完整性，防止因节点失效导致的数据丢失或错误；
5. 免运维：函数计算无需购买服务器、安装软件、配置网络环境、管理服务器集群等，即可快速部署应用；

## 三、无服务架构（Serverless Architecture）和函数计算（Function Compute）的区别？
函数计算是阿里云推出的一款新型的无服务架构，属于服务计算范畴。相比于传统的云函数（Function as a Service，FaaS），函数计算具有更高的性能、更可预测的价格、更灵活的伸缩方式和按量付费模式。

无服务架构与函数计算最大的区别就是，前者依赖于云厂商提供的服务，后者由阿里云自身提供的服务。云厂商提供的服务有两种形式：计算资源的共享、使用事件触发计算资源。函数计算采用的是事件触发计算资源的模式，无需预先购买计算资源，只需要上传代码和配置文件，便可立即获得计算资源。而云厂商的云函数（CFC）是一种按量计费的函数服务。
