# 1.背景介绍


## 一、背景
在我国，IT界对“信息”这个词语已经充满了误导性和歧视性的倾向。首先，“信息”一词在汉语里原本指的是有关物质的东西，如物品、工具、实物等；现代互联网时代的信息则是网络上海量数据产生的电子信息，它既有形状也有形式。其次，在国家的发展进程中，科技的发展日新月异，大批创新型公司涌入市场，数字经济成为当今社会的主流，网络经济的蓬勃发展正在带动整个产业的变革。因此，IT界不再以“信息”二字作为最重要的词语，而是更多地使用“信息技术”、“数字经济”、“网络经济”等描述当前的社会经济格局及信息技术发展方向。

随着信息技术的飞速发展，IT企业也越来越重视对信息的整合管理。传统的信息结构模式常常束缚了业务运营的创造力和决策效率，在数字化时代，新的信息架构模式势不可挡。结构化思维方法又成为信息架构设计的主要方式，虽然“信息”一词经历了种种流言蜚语，但是结构化思维却逐渐成为架构师必备的技能。

## 二、信息架构的定义
“信息架构”一词在IT界出现的时间并不长，但随着信息技术的兴起，很多人都把它看作是一个以文档为核心的管理体系，强调文档的创建、归档、管理、共享、检索、分析等流程。这种“信息架构”的理念其实很古老，源自于法国启蒙思想家卢梭。他认为人类活动本身就是生产信息，信息的处理和利用应该成为一切行为的中心。因此，在1994年，西方媒体发表了一篇题为《社会功能论》的著名论文，提出“架构即社会功能”。架构师是“提供和支持建筑物的运行、分配、消除障碍、保持平衡”的角色，也就是说，一个好的架构师必须懂得如何管理信息，让各种信息在架构中的流动顺畅，而不是使信息彼此纠缠不清。

2000年，斯坦福大学的安迪.格雷教授在《结构化思考与可视化》一书中提出了“信息架构”的概念，他认为“信息架构”是“通过将不同层级的信息组织成统一、可理解的视图，从而实现信息的有效整合、传递、使用、分析、存储和表达。”也就是说，“信息架构”实际上是关于信息系统设计和组织的一门学问。

目前，“信息架构”被广泛应用于各个领域，包括医疗、电信、金融、政府、商业、交通、公共事业、教育等。信息架构师负责对信息进行整理、分类、编制索引，以便更好地满足用户的需求。根据ISO9000认证标准，信息架构师一般需要具有以下五项能力：

- 抽象思维：能够发现复杂的信息和主题，并将其抽象为易于理解的内容。
- 概念模型：掌握信息的本质和联系，能够构建并管理信息的内部模型。
- 逻辑分析：具有深刻的洞察力，能用自然语言对信息进行分析，提出解决方案或建议。
- 业务理解：理解业务、产品或服务背后的信息需求，才能做到全面、准确且适宜。
- 数据处理：善于运用计算机软件、工具和数据库处理海量数据，实现数据采集、转换、挖掘和分析。

## 三、信息架构设计过程及关键环节
## （一）构思阶段
### 1. 确定目标
首先，信息架构设计人员需要对信息的整合有清晰的目标。一般来说，目标可以由五个维度来具体阐述，如数据分析、智能应用、产品开发、信息获取、价值交换等。例如，在电商平台网站的设计中，目标可能是获取用户满意度、增加订单量、提升产品购买率等。

### 2. 找出信息
然后，信息架构设计人员需要找寻信息。在这个过程中，他们需要首先确定自己要收集哪些信息，以及这些信息来源在哪里。这个过程通常称为“信息搜集”，包括观察研究、访谈调研、采访咨询、数据采集、线上线下培训、案例研究、竞品分析等。

### 3. 总结信息
紧接着，信息架构设计人员需要进行信息的总结。这一步主要是为了确认信息是否真正存在，并对其进行必要的整理和归纳。总结的信息通常包括：结构、主题、内容、分类、范围、关联、相关性、时序、量级、质量、权限、隐私等。

### 4. 建立模型
最后，信息架构设计人员需要建立信息模型。信息模型即一种结构化的方式来呈现信息，帮助信息架构师和相关工作者理解信息的含义、结构关系、行为特性和上下文关系。模型中往往会涉及信息的描述、实体之间的关系、功能特性、信息生命周期、用户需求、可用性、兼容性、扩展性、安全性、隐私性等。

## （二）策划阶段
### 1. 调研需求
首先，信息架构设计人员需要和业务、产品、用户等方面进行需求调研。调研的目的是为了明确信息架构设计人员的职责范围、对用户的期望、业务目标、可行性判断、所需技术和资源等方面的情况。

### 2. 分析信息
其次，信息架构设计人员需要分析信息。信息架构设计人员通常具有一定的专业知识，可以进行多角度分析，从多个维度去探究信息架构的作用。包括内容、形式、上下文、主题、关联、规模、空间、时效性、动态性、可靠性、完整性、适用性、可访问性、可复用性、灵活性、适应性、可伸缩性等多个方面。

### 3. 设计信息架构
最后，信息架构设计人员需要进行信息架构设计。信息架构设计人员通常具备强大的图示、思维逻辑以及组织能力，能够将分析得到的各项信息转化为信息架构蓝图。蓝图通常包括：实体、主题、关系、功能、过程、视图、界面、规则、约束、规则引擎、索引、搜索、导航、安全性、隐私性等。

## 四、信息架构的关键原则
信息架构是为了解决信息的问题而发展起来的，在设计时应该遵循以下原则：

1. 面向用户：信息架构的核心目标应该是为用户提供有价值的、有意义的信息，面向用户意味着考虑所有用户的视角，为用户提供符合其需要的信息。
2. 可见性：信息架构应该让信息可见、方便检索、分类、过滤、查询，同时信息不仅要提供给用户，还应该有助于支持业务运营，帮助决策。
3. 有序性：信息架构的结构应该有序，避免出现孤立点、混乱无章的情况。
4. 清晰性：信息架构应该保持清晰、简单、直观，让用户快速找到所需信息。
5. 及时性：信息架构需要及时反映业务变化、产品更新、政策变化，保证信息的准确性、完整性和及时性。

## 五、信息架构设计的注意事项
下面是一些需要注意的信息架构设计的注意事项：

1. 迭代更新：信息架构是不断演进的，每一次调整都需要重新审视信息架构设计，确保设计结果能够反映最新的业务需求。
2. 汇总信息：不同的信息需要汇总为一份，对信息的分类和组织方法有所不同。
3. 用语精准：信息架构设计人员需要用语精准，用正确的词语描述信息，避免不恰当的用语和歧义。
4. 协作配合：信息架构设计人员需要和其他部门的专业人员配合，从多个角度进行设计，确保信息架构最终达到一致。
5. 测试发布：信息架构不是凭空创造出来的，一定要测试发布，通过用户访谈、业务试点、业务审批、客户参观等方式验证信息架构的有效性。
