## 2.关键技术概念
### 1.应用服务器（Application Server）
应用服务器（Application Server）指的是处理客户端请求和应用程序逻辑的服务器软件，负责接收客户端发送的请求，解析请求中的指令并调用相应的业务逻辑进行处理，然后生成相应的响应结果返回给客户端。主要功能包括：

① 请求处理：应用服务器接收请求后，会首先验证身份，然后进行URL匹配，再把请求交给Servlet容器。

② Servlet容器：Servlet容器是运行在服务器上的一个应用，它负责读取编译后的Servlet类，并通过线程池为每个请求创建线程执行。

③ 业务处理：Servlet容器创建完成后，便可以调用业务对象的方法进行业务处理，比如用户登录、订单查询等。

④ 数据访问：当业务处理完成后，会把结果写入到数据库中。

⑤ 返回响应：最后，应用服务器把业务处理结果封装成HTTP响应返回给客户端。

应用服务器也是服务器软件的一部分，所以需要安装Java虚拟机（JVM），同时还需要安装Servlet引擎。

### 2.Servlet
Servlet（Server Applet）是一种运行在Web服务器上的小型程序，它属于服务器端的 Java 小程序，由 javax.servlet.Servlet 接口实现。它的主要作用是实现对HTTP请求的响应，即根据客户端的请求信息生成动态的内容，并传送回浏览器显示给用户。

Web服务器收到客户端请求后，根据请求报文信息中的URL寻找对应的Servlet，并调用Servlet的service方法来处理请求。当Servlet处理完业务逻辑之后，会把结果写入到输出流（response stream）中，然后Web服务器将响应内容发送给客户端。

### 3.JSP(Java Server Pages)
JSP（Java Server Pages）是一种运行在Web服务器上的Java小程序，它可以用来生成动态网页，其底层运行原理与Servlet类似。但JSP可以让您利用服务器端的强大功能，生成静态页面的同时，还能处理动态请求。JSP文件后缀名为.jsp，它们是Java代码和HTML标签的混合体，可以包含Java代码来执行各种业务逻辑。JSP页面可以在页面中插入Java表达式、脚本片段、自定义标记库、属性设置等。

JSP并不是真正意义上的Servlet，它只是一种运行在服务器上的小程序，实际上还是Servlet的实现。但是因为它们共享同一个命名空间，所以被设计出来时需要格外注意安全性。除此之外，它们还有些特性使得它们更适合用于Web应用程序开发。例如，它们支持动态标签，可以直接向前台输出数据库内容，而不是直接生成静态网页；JSP文件可以被缓存，可以提升访问速度；而且JSP提供了一个模板机制，可以方便地重用代码。

### 4.Spring Boot
Spring Boot是由Pivotal团队提供的全新框架，其设计目的是用来简化新Spring应用的初始搭建以及开发过程。该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的XML。通过这种方式，Spring Boot致力于减少配置时间，从而缩短开发周期。其核心特征包括：

- 创建独立的、生产级别的Spring版本
- 提供了一套自动配置项，可以快速开启常用功能
- 内嵌Tomcat和Jetty Web服务器，无需部署war包即可运行
- 提供命令行界面，可以快速运行Spring Boot应用

### 5.MySQL数据库
MySQL是一个开放源代码的关系型数据库管理系统，由瑞典MySQL AB公司开发，目前属于Oracle Corporation的一部分。MySQL是最流行的关系型数据库服务器端软件，被广泛应用于Internet网站、政府网站、金融网站、零售网站、学术网站、搜索引擎等。

### 6.Redis数据库
Redis是一个开源的高性能键值存储系统，它支持的数据结构有字符串、哈希表、列表、集合、排序 Set 和带范围查询的 Zset。Redis 提供了多种语言的 API 来处理键值存储，其中包括 PHP、Java、JavaScript、Python、C、Go 等。

### 7.Nginx
Nginx（Engine X）是一个免费、开源的Web服务器，它非常适合处理动态请求，高并发场景下提供良好的性能。Nginx采用事件驱动的异步框架，支持热插拔，允许模块的开发。

Nginx采用的是单进程单线程的模型，同时支持Linux和Windows操作系统，可以很好地与其他server软件配合使用，如Apache HTTP Server、Tengine等。

### 8.Apache Tomcat
Apache Tomcat (Tomcat服务器) 是Apache下的一个开源的Web服务容器（ Servlet 容器），是一个轻量级的Web应用服务器，非常适合用于开发和调试JSP、Servlet、Java EE 应用程序。Tomcat服务器是一个WAS（Web Application Server）容器，它提供了独立的Web容器，可以运行Servlet、JSP以及各种类型的WEB应用。

Tomcat服务器是一个标准的WAR（Web Application Archive）文件格式，可以直接运行其中的应用，不需要额外的部署容器。Tomcat是运行在Servlet规范下的，因此可以使用所有兼容的Servlet规范的Servlet和JSP。

### 9.Kafka消息队列
Kafka是一款开源的分布式发布订阅消息系统，它最初起源于LinkedIn，用于构建实时的数据管道和流处理应用。它是一个分布式、可持久化的日志存储系统，能够提供高吞吐量、低延迟的数据管道。Kafka的接口简单易懂、功能丰富，适用于大数据实时处理的场景。
