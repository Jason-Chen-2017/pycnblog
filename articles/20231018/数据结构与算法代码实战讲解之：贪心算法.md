# 1.背景介绍


## 什么是贪心算法？
贪心算法（Greedy algorithm）也是一种典型的动态规划方法。它总是对当前情况做出最优选择，也就是说，不从整体考虑，所作出的仅仅是局部最优或近似最优解。这种策略并不是贯穿始终，而是随着时间推移逐渐形成的，因此也被称为贪婪法、自适应法、自利法或贪心选择法。贪心算法能够产生全局最优解，但在很多时候并不能保证得到精确解。虽然贪心算法往往具有较高的时间复杂度，但同时其收敛速度也很快，可以解决一些复杂的问题。

比如图论中寻找最小生成树问题（Minimum Spanning Tree Problem, MST），就是一个经典的应用。

## 为何要用贪心算法？
当面对复杂的优化问题时，人们通常会采用启发式搜索或随机游走的方式来求解，但是这样的方法往往收敛慢且效率低下。另一种更为直接、高效的方法是贪心算法，它利用对所给问题的分析，通过一步步地执行某种操作来达到目标。每一步都会产生一个局部最优解，直至达到全局最优解。由于贪心算法简单易懂，且能得到全局最优解，因此有很多实际运用的地方。

比如最短路径问题、停车分配问题、分配任务给工人问题等。

## 贪心算法的适用场景
贪心算法一般都有两个基本条件：第一，所求问题是一个优化问题，即存在目标函数以及目标变量；第二，问题具有局部最优性质，即无论如何都要保证找到某个局部最优解。贪心算法通常具有最坏情况时间复杂度，但却是许多情况下的最优解，因此在一些有着复杂计算量要求的任务上表现得尤为突出。另外，贪心算法的求解过程通常比较容易理解和掌握。因此，许多领域都把它作为重要的算法，包括计算机科学、经济学、生物学、电路设计、图论、线性规划、运筹学、机器学习、人工智能等等。

# 2.核心概念与联系
## 概念
贪心算法背后的关键理念是，每次都做出在当前看来是最好的选择。换句话说，就是为了使之成为全局最优解而不采取任何过多的猜测或尝试。

举个例子，假设我们希望从十万条订单中选出其中价值最高的订单，那么我们可能会先按照订单金额排序，然后再次将排名靠前的订单归入某个“选择池”（例如，10%）。然后，我们就可以选择这些订单中的一条，因为这些订单已经都具有相似的价值了。

这是一种贪心算法的例子——我们首先就按订单的金额进行排序，然后再次从该列表中选择订单。这个选择往往比完全排序的结果要好，因为这意味着我们不需要再查看后面的订单。

## 相关术语
- 问题：待求解的一个或多个约束优化问题
- 变量：表示问题的对象的量
- 函数：确定变量的取值的集合或映射关系
- 目标函数：想要最小化或最大化的函数表达式
- 子问题：作为某个问题的组成部分的另一个问题，通常依赖于父问题的解
- 策略：由算法用来达到期望目标的指导准则
- 剪枝：跳过一些没有必要探索的分支或子问题

## 相关模式
贪心算法通常有三种主要模式：
1. 分治模式：该模式将问题分解为几个互相独立的子问题，然后递归地解决每个子问题。贪心算法本身就是采用这种模式的一种。
2. 贪心选择模式：在每一步中，贪心地选择最优的元素，然后依据此元素更新状态。贪心算法的主要工作原理就是基于这种模式。
3. 回溯模式：当一个解不能再改进时，回退到之前的状态，重新选择一个元素，直至得到可行解。贪心算法也可以实现回溯模式。

## 相关问题
贪心算法具有良好的最优子结构性质，所以它们经常被用在组合优化问题中。这些问题通常都可以通过贪心选择求解，即通过依次选择最优的子问题来获得整个问题的最优解。以下是一些典型的组合优化问题：

1. 单调队列问题：给定一个序列，希望从中删除一部分元素，使得剩下的元素仍然为非严格递增或非严格递减序列。这类问题可以转化为贪心选择模式。
2. 整数拼凑问题：给定一系列数字，希望组成这些数字的所有可能组合，而每个组合中任意两数字的差都是指定的整数。这类问题可以转化为贪心选择模式。
3. 括号匹配问题：给定一个字符串，希望匹配成一组合法的括号对。这类问题也可以转化为贪心选择模式。
