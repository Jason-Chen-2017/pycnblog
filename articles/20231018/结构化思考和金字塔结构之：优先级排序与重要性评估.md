# 1.背景介绍


什么叫做“结构化思维”？“结构化思维”其实就是根据特定的问题需要提炼出最优解决方案的方法论。那么如何在复杂的问题中找到问题的关键所在呢？通过对问题进行分类、分析、综合、归纳等方法就可以找到问题的关键点，然后再从这些关键点中提取最关键的因素作为我们的研究对象。而为了发现关键点，就需要建立起层次化、多级排序的知识体系，也就是我们所说的“金字塔”。那么，如何有效地运用“金字塔”这一数据结构解决具体的业务问题呢？

对于优先级排序与重要性评估这一问题，我们可以从以下两个方面入手：

① 优先级排序：优先级排序是指根据不同情况及其影响，对待办事项进行分类、排序并依次实施的过程。本文将基于企业管理者日常工作任务优先级的实际经验，结合结构化思考理论，阐述基于优先级的重要性评估的基本原理、方法、流程、工具以及注意事项。

② 重要性评估：重要性评估是指对于不同的信息资源或事件，在对资源进行整合和决策的过程中，识别其重要性的过程。在实际应用场景中，重要性评估既涉及到信息的收集、处理和分析，也涉及到对数据的总结、分析和可视化，具有十分重要的意义。本文将基于《结构化思考和金字塔结构之：优先级排序与重要性评估》一文，为企业管理者提供基于优先级的重要性评估服务方案，帮助企业更好地管理资源。

基于以上两个方面，本文将以一个企业管理者的视角，系统阐述优先级排序和重要性评估的基本原理，方法，流程，工具以及注意事项，力争让读者能正确理解优先级排序与重要性评估的机制，掌握相应的分析方法，能够为企业提供有力的决策支持。 

# 2.核心概念与联系
## 2.1 核心概念
**层次化、多级排序的知识体系**：为了有效地发现关键点，提炼问题的真相，企业管理者首先需要构建层次化、多级排序的知识体系，也就是我们所说的“金字塔”。金字塔是一个层次化的组织结构，它把复杂的信息和知识集中起来，通过不同层次的组织架构对复杂信息进行分类、归类、筛选和检索。


如图所示，金字塔由多个结点（结点通常采用图形符号表示），每个结点代表一个主题或领域。金字塔上最高的一层代表着整个的宇宙或者知识的本源；中间的各个层代表着各个阶段的主要知识主题；最底下的各个节点则是具体的事物或信息。通过对问题的分析和归纳，我们可以从最抽象的层面逐步深入，直到某个具体的事物或信息。通过这种层次化的组织结构，我们可以快速准确地定位到关键点，将不同层次的知识组织起来，最终得到一个有条理的整体视图，帮助我们识别问题的真相。

**优先级排序：**企业管理者日常工作任务优先级的实际经验，可以总结为四个原则：

① 必要性原则：如果某件事不是必要的，就不要做。例如：买房子只要有钱、有车、还贷款，不必考虑物质条件是否充足。

② 时效性原则：已经完成的工作不要重复做。例如：交通费缴纳只需预付一次，后续只按时缴纳即可。

③ 紧急程度原则：越紧急的工作越重要。例如：紧急事情尽快做完，平时事项安排妥当。

④ 需求性原则：对自己的能力、资源和条件都非常有利的工作才重要。例如：要求有项目经理参与的项目，往往需要有技术人才。

**重要性评估：**重要性评估是指对于不同的信息资源或事件，在对资源进行整合和决策的过程中，识别其重要性的过程。重要性评估可以用来对各种资源进行整合、分配和管理，包括人力、物资、设备等。在实际应用场景中，重要性评估既涉及到信息的收集、处理和分析，也涉及到对数据的总结、分析和可视化，具有十分重要的意义。

## 2.2 相关术语
**必要性（Necessity）**：是指某件事是否对个人、家庭、国家、社会或者企业来说都是必要的。如果某件事没有必要，一般不会受到重视，或甚至会被认为是一种浪费时间和精力。必要性原则要求我们不要过分依赖一切东西，只有一些必要的事情才值得花时间、花心思去做。

**时效性（Efficiency）**：是指完成某件事情所需的时间是否可以缩短。如果一件事不能快速完成，可能使人感到沮丧，甚至产生焦虑或恐惧。因此，时效性原则建议我们尽量避免做那些需要长时间才能完成的事情。

**紧急程度（Importance）**：是指工作的紧迫程度。优先处理紧急的工作，可以保证时间的最佳利用，同时降低了处理其他事项时的压力。

**需求性（Demand）**：是指对资源、人力、时间、机会、位置、财富、技术、经验等有利条件的事。需求性原则强调对个人的能力、资源、条件等多方面的考虑，而不是盲目追求工作上的任何福利或进步。

**关键路径法（Critical Path Method）**：是一种关键功能分析的方法，用于判断流程或项目中的关键路径及其对应的资源消耗。其基本思想是识别项目中最慢的环节，然后根据其所占用的时间确定关键路径。关键路径法是对传统项目管理方法的一种改进，它明显减少了资源浪费，同时保证了项目顺利完成。
