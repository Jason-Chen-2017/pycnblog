# 1.背景介绍


近年来，随着新技术的不断涌现、传感器设备的飞速普及、数据的呈爆炸增长，传统的数据处理方式已经无法满足海量数据的快速计算处理需求。尤其是在航空航天领域，飞机航线跟踪、空服识别、人员轨迹识别等高频场景需要实时分析和处理海量的数据，因此对大数据的快速处理提出了更高的要求。虽然云计算平台已成为实现大数据处理的一大关键技术，但在航空航天行业，由于采用的传感器设备和数据量的限制，采用云端数据处理方案仍然存在一定的局限性。为了突破这一难题，传感器厂商和运营商都面临着如何高效、低成本地处理海量数据的难题，同时也需要保持对数据私密性和安全性的保障。基于此，国际航空航天组织（IATA）发布了“Digital Innovation Platform for Air Traffic Management (DIPM)”(数字化航空航天管理平台)，旨在打造具有数据分析、机器学习能力的运维工具、决策支持系统、监控系统，通过建立统一的数据源，实现各类机场经验、风险管理和灾害预警数据共享和集成，实现海量数据、多种数据源之间的快速准确联合分析，进而实现对机场运行状态、经济状况、交通运输情况等综合评估和预警，并自动化决策制定。2019年3月2日，IATA将由英国伦敦道格拉斯•卡辛克主持举办的第二届“IATA-DIPM Workshop”(IATA-DIPM研讨会)正式启动。
本文将以“Digital Innovation Platform for Air Traffic Management (DIPM)”为主题，结合自身研究以及前期学习的成果，从数据采集到智能分析，分享我对于大数据、智能数据应用架构、航空航天大数据处理以及未来的设想。
# 2.核心概念与联系
## 2.1 大数据
大数据（Big Data）是指存储、处理、和分析海量数据，可以从各种渠道产生，包括互联网数据、电子邮件、移动应用程序、社交媒体、游戏、消费行为日志、科技产品和服务、医疗记录、车流数据等。这些数据通常被存储在分布式文件系统中，并由多个数据源生成。

### 2.1.1 数据量
据报告显示，2020年全球超过三分之一的新进入者可能具有以下特征：拥有至少百亿条、千亿条或万亿条的网络或应用活动信息；通过移动设备、手表、健康追踪器、虚拟现实设备、地图引擎、无人机、智能手机、社交媒体、IoT终端收集的数据量达到十亿条以上；以及在互联网上进行协作和信息交流。这为分析和处理海量数据提供了一个重要的机遇。

### 2.1.2 数据类型
大数据可以按数据抽象层次分类，主要分为结构化、半结构化和非结构化数据。

1. 结构化数据：也称为关系型数据，例如，数据库表中的字段、记录、对象；这种数据按照固定模式进行组织，能够精确而完整地描述事物间的联系。
2. 半结构化数据：包括文本、图像、视频、音频、地理位置数据等，这些数据被格式化成一些结构，但是没有严格的规则，数据之间存在着相当大的内在关联性。
3. 非结构化数据：非结构化数据是指数据缺乏结构化的特性，它们通常以非随机的方式存储，并且同样不能用简单的模式进行分类。例如，HTML网页源码、JSON文档、CSV文件、XML数据等。

### 2.1.3 系统架构
一般情况下，大数据系统的架构通常包括三个阶段：数据采集、数据存储、数据分析和决策支持。


1. 数据采集：首先，运营商和各个机场都会搜集海量的数据，比如，机场跑道、设施设备的运行数据、飞机空姐轨迹、社会交往数据、企业业务数据、航空公司资金使用情况数据等。
2. 数据存储：其次，这些数据需要进行存储才能便于后续的分析工作，如采用Hadoop或Spark进行分布式存储，比如HDFS、Hive等。
3. 数据分析：第三步，需要对这些海量数据进行分析和挖掘，找到有价值的信息，同时进行数据挖掘和机器学习，得到智慧的洞察力。
4. 智能决策支持：最后，运营商、机场及其相关部门通过利用大数据智能决策支撑系统，进行复杂的数据分析和决策支持，比如，机场运行情况的预测、路线优化、资源调配、维护策略调整、意外事件发现、风险防范等。

## 2.2 智能数据应用架构
智能数据应用架构（SmarterData Application Architecture）是指运用大数据、机器学习、人工智能等技术来改善大数据处理流程、提升数据分析和决策能力，打造智能数据处理系统，从而满足海量数据下的分析和决策需要。

智能数据应用架构的设计目标是为了应对大规模、高维、异质、动态和复杂的数据，通过实现数据采集、清洗、存储、分析、挖掘、关联、预测、推荐、决策支持等核心功能，推动企业从数据中获益，以获取更多的商业价值。目前，国际航空航天组织（IATA）定义智能数据应用架构如下：

- 数据采集：通过各种不同来源的原始数据，采集和转换成标准化的结构化数据。
- 数据清洗：对数据进行初步的预处理，消除脏数据、异常数据、重复数据等影响数据质量的因素。
- 数据转换：将异构的数据源进行统一、转换，确保数据处理过程中的一致性。
- 数据标准化：采用标准的表示形式，简化数据的存储、检索和交换。
- 数据挖掘：基于海量的数据进行高效、精准的挖掘，获得丰富的知识和 insights。
- 数据关联：结合不同数据源及其之间的关联性，识别出有意义的关联关系和模式。
- 数据预测：利用历史数据预测当前数据的走向和变化。
- 数据推荐：根据用户偏好、喜好和习惯推荐相关商品、服务或信息。
- 智能决策支持：运用大数据技术来构建智能决策支撑系统，自动识别和解决复杂的问题，提升业务的决策效率、降低人工成本，提升用户满意度。

基于这一定义，我认为我们应该关注以下几个方面的建设：

1. 数据采集：提升数据采集和转换的效率，支持数据源包括非结构化数据、半结构化数据、和非结构化数据。
2. 数据仓库：设计数据仓库，将数据采集后的结构化数据加载到数据仓库中，支持数据查询、分析、挖掘、关联、预测、推荐等。
3. 数据湖：设计数据湖，以数据湖为核心，将所有数据源汇聚到一起，支持数据分析、挖掘、关联、预测、推荐等。
4. 数据处理：实现海量数据处理的有效性，包括批处理、流处理和超大规模计算。
5. 智能分析：引入机器学习和统计方法，加强对数据模式的理解，支持数据分析、挖掘、关联、预测、推荐等。
6. 智能推荐：引入推荐算法，推动用户行为数据的收集和分析，做到个性化推荐和趣味推荐。
7. 智能决策支持：实现智能决策支撑系统，基于历史数据和相关数据源，对核心问题进行自动化的分析和决策，提升决策效率。
