# 2.核心概念与联系
首先，我们需要了解下web开发的一些基本概念。如图所示，Web应用是一个运行在互联网上的应用程序，它可以访问用户的浏览器进行页面显示、向用户提供服务（如电子商务网站）、记录用户信息等功能。Web应用分为前端（客户端）和后端（服务器端），前端负责呈现页面，包括HTML/CSS/JavaScript，后端负责处理后台逻辑，如数据处理、业务逻辑处理等。



其中，HTTP协议用于传输数据，WebSocket协议用于实现实时通信。RESTful API是目前最流行的API设计风格，它定义了资源、集合、方法三种主要组件。

前端开发通常使用HTML、CSS和JavaScript，而后端则更喜欢使用Python、Java或其他服务器端编程语言。因此，在做web开发时，需要明白前端和后端的工作模式、接口规范以及相应的工具集。

Web框架是在特定编程环境下用于构建Web应用的软件包或模块，它包含了开发Web应用所需的一系列工具、类库和模板文件。这些工具、类库和模板文件能够有效地简化Web应用的开发流程，并提供了可重用代码和便利的编程模型。在实际项目中，Web框架既可以直接使用，也可以集成到其它工具或系统中使用。比如，Django就是Python最常用的Web框架之一。

很多公司都会有自己的Web应用，不同公司的技术栈也不同，因此，选择适合自己团队的Web框架十分重要。所以，本文将从以下三个方面深入探讨Python web开发中的技术栈：

⒈ Python生态圈：Python作为一种通用编程语言，具有庞大的库和框架供开发者使用。本文将介绍一些主流的Python web框架，以及它们分别使用的技术栈及其优缺点。

⒉ Flask：Flask是一个基于Python的微型Web框架，轻量级且高性能。它本身非常简单，但它的简洁设计和可扩展性强，适合快速开发小型的、简单的应用。Flask支持WSGI(Web Server Gateway Interface)，因此可以部署在各种WSGI服务器上。本文将介绍Flask的基本概念和特性，以及如何使用Flask进行Web开发。

⒊ Django：Django是一个Python Web框架，它集成了丰富的特性，简约而灵活。它提供了自动生成后台管理页面的能力，适用于复杂的Web应用。Django自带的ORM（Object Relational Mapping）功能使得数据库操作变得方便，同时还支持RESTful API。本文将介绍Django的基本概念和特性，以及如何使用Django进行Web开发。
