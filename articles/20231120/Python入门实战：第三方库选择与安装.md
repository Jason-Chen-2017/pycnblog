                 

# 1.背景介绍


## 什么是第三方库？
在编程过程中，我们经常需要用到其他开发者编写好的函数或模块，称为**第三方库**。
比如，要实现一个网络爬虫程序，我们需要使用BeautifulSoup模块，它可以用来解析网页上的HTML、XML等文档，提取有效信息。
这些模块一般都已经有很多精心设计的算法和方法来解决相应的问题，免去了我们重复造轮子的麻烦。

然而，在日常使用中，还有一些模块我们可能不太熟悉，也没有找到合适的功能。因此，如何找到并安装合适的第三方库，成为我们每天都要面临的一个难题也是很重要的。

本文将带领大家了解以下知识点：

1. 为什么需要学习第三方库？
2. Python生态中有哪些优秀的第三方库？
3. 安装第三方库的方法及注意事项。

# 2.核心概念与联系
## 模块(module)
在Python中，一个.py文件就是一个模块（Module）。模块内部定义了各种变量、函数和类，通过导入其他模块的方式使用该模块中的对象。如：
```python
import module_name
obj = module_name.class_or_function() # 使用该模块的类或函数
``` 

## 包(package)
模块可能很多，如果希望管理起来更方便，可以把多个相关模块放在一个文件夹下，形成一个包（Package），这样就可以使用`import package_name`语句来导入整个包。如：
```python
from package_name import module_name
obj = module_name.class_or_function() # 使用该包下的某个模块的类或函数
``` 

## pip(Python Package Index)
pip是一个包管理工具，可以帮助我们轻松安装第三方库。pip官方网站：https://pypi.org/ 。其作用是：

1. 在官方发布的pypi镜像站点上搜索、下载、安装第三方库；
2. 可以自动处理依赖关系，自动下载安装依赖的第三方库；
3. 支持安装指定版本的库，避免不同库之间的兼容性问题。

## venv(virtual environment)
venv是一个创建独立环境的工具。一般情况下，我们会在系统目录下创建一个virtualenv文件夹，里面包含了一个完整的Python运行环境。venv允许我们创建多个虚拟环境，每个环境里面的Python都是独自安装的，互相之间不会影响。

## 搜索、安装、激活第三方库
为了安装第三方库，我们首先需要搜索一下它的名称。通常，搜索到的结果都会列出该库的名字、简介、作者、最新版本号、分类、支持平台、github地址等信息。可以通过PyPI网站或者搜索引擎进行搜索。

搜索到要安装的库后，可以按照如下方式安装：

1. 使用pip命令安装，语法为 `pip install package-name`，例如`pip install beautifulsoup4`。pip会自动检测依赖关系，并安装所有缺失的依赖。

2. 如果需要安装指定版本的库，可以使用`pip install package-name==version`命令。例如，`pip install beautifulsoup4==4.9.1`安装BeautifulSoup4最新版。

3. 如果要在虚拟环境中安装第三方库，则先进入虚拟环境，再执行上面两种安装命令即可。

安装完毕后，我们还需要激活才能调用模块内的函数和类。如果安装时提示已成功安装，则可以直接使用。但是，如果安装成功但无法正常使用，可能原因如下：

1. 由于安装路径未添加到环境变量中，导致调用时找不到模块。可以在Anaconda Prompt中输入`activate env_name`，激活当前虚拟环境，然后尝试调用第三方库的函数或类。

2. 第三方库的依赖库可能也没有安装成功。这种情况，可以先尝试安装依赖的库，再重新安装主库。也可以通过`pip freeze > requirements.txt`导出库依赖清单，再在新的虚拟环境中根据清单安装依赖库。

最后，我们可以查看Python环境中安装的所有第三方库信息，使用`pip list`命令。
