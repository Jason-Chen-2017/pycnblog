                 

# 1.背景介绍


## 企业需要解决的问题——使用RPA完成业务流程的自动化
企业在日益增长的需求驱动下，面临着越来越多复杂、繁琐、重复的业务流程任务，而现有的手动办公方式显然无法满足快速、准确、自动化的需求。同时，各个部门的业务人员也在越来越注重信息化建设，提升工作效率。如此背景下，企业需要将一部分重复性繁琐的流程自动化，降低人工成本，提高工作效率。
### 现状分析
随着商业模式的不断发展和对市场竞争的激烈态势，房地产市场也经历了一系列的变革和升级，如由传统房地产管理模式转型为数字化管理模式、以APP+电子围墙的方式实现互联网金融服务等。对于房地产企业来说，由于涉及房地产相关事务，他们每天都要处理大量的信息，如征收土地使用权证书、招标购买施工许可证、签订物业管理合同等。因此，各类房地产企业都倾向于采用集中管控的方式进行流程管理，而非手工操作。另外，由于各个环节往往存在不同的审批流程、规则和手续，使得每个阶段的处理时间长、效率低下。因此，房地产企业需要一个系统化的解决方案来自动化地管理这些流程，从而提高工作效率，降低人力资源消耗和管理成本。
### RPA（Robotic Process Automation）
使用RPA可以实现业务流程的自动化，而且它具有以下几方面的优点：
- 可靠：通过有效地模拟人的行为、键盘输入、鼠标点击、屏幕阅读、语音识别等，机器能够逼真模拟用户完成各种业务流程，降低因人为操作造成的错误。
- 便捷：使用RPA不需要去学习新的编程语言，只需按照简单的配置即可快速部署；可以在任何平台上运行，兼容各种软硬件环境，使其具备良好的灵活性和适应性。
- 准确：在模拟人的行为时，机器可以跟踪并精确地还原用户的操作过程，避免因模拟过程中的人为错误而产生误差。
为了解决以上业务需求，房地产企业可以选择使用RPA来实现业务流程的自动化。目前，已有一些研究者将RPA用于自动化地执行房地产投资决策、申请与审批环节、审计结算等。但这些研究结果尚未实际落地。如何通过使用GPT-3模型的AI agent来自动化执行房地产商业流程，成为亟待解决的问题之一。
# 2.核心概念与联系
## GPT-3模型
GPT-3，全称“Generative Pre-trained Transformer”，是一种基于Transformer的文本生成模型，目前已经发布了多个版本，主要用于文本生成领域，并且经过了进一步训练。GPT-3模型的结构和神经网络架构与其他预训练模型相似，包括一个编码器和一个解码器，两个模块共享参数，前者生成编码语句，后者根据编码语句生成输出文本。但是不同的是，GPT-3模型相较于其他模型更加关注语言模型和文本生成任务，同时为了达到更好的结果，引入了很多额外的技术，例如：条件流、混合语言模型、条件层次分布，以及优化目标的设计等。
## AI Agent
AI Agent是一个具有自我学习能力、能够做出反馈并改善自己行为的实体。它的表现形式可以是虚拟助手、聊天机器人、游戏引擎或智能体等，也可以是汽车、手机、平板电脑甚至是微控制器。与传统的人工智能系统不同，AI Agent由数据驱动，其学习能力来源于训练数据，而不是靠自主构造算法。它可以获取知识、学习新技能、扩展能力、规划未来的计划、处理事务、与其他实体交互、构建情景等。AI Agent主要分为三种类型：
- Rule-based Agent：规则代理，通常情况下是人类的助理型机器人，依赖于既定的策略或规则来处理任务，如微软小冰和Google助手等。
- Learning-based Agent：学习代理，能够学习新事物、掌握新技能、解决问题，且学习到的知识可以用于其他应用场景。如AlphaGo和Chatbot。
- Hybrid Agent：混合型代理，既可以像规则代理一样依据既定策略来处理任务，又可以学习新技能、处理事务、与其他实体交互等。如Siri、Alexa。

### 概览
基于GPT-3模型和AI agent的房地产商业流程自动化架构，整体架构如下图所示。首先，公司将人员、设备、资料、软件部署到中心机房，使用预先定义好的规则和指导，引导员工使用微信、邮件等方式，提交符合要求的文件和文档。中心机房自动启动RPA智能助手，读取企业微信或邮箱，识别提交文件，判断是否包含符合流程标准的材料，如果符合则把材料存储到共享目录中，并通知相关人员审核。审核人员登录共享目录，对材料进行初步检查确认后，再由审核人员发送最终确认指令给申请人，提示其核实审核意见，审批通过后会触发自动化流程，生成相应的手续费收据或缴纳清单等。整个流程全程无需人工干预，所有材料、工作票据均自动保存，流程数据与最终结果呈现，节省公司管理成本，提高工作效率。
