# 2.核心概念与联系
## 模块导入
模块(Module)是Python代码中可以被别处引用使用的代码片段。不同的模块可以提供不同的函数、类、变量或其他资源。在Python中，可以直接引入模块或通过导入路径指定模块文件路径引入模块。模块的导入语法如下：

```python
import module_name # 引入某个模块中的所有符号到当前命名空间
from module_name import symbol1[,symbol2[...]] # 从某个模块中导入指定的符号到当前命名空间
```

例如，要导入`math`模块，可以使用以下语句：

```python
import math
```

然后就可以通过`math.`前缀访问其内置函数，如`math.sqrt()`用于计算平方根。类似地，要从模块中导入多个函数或变量，可以使用逗号分隔符隔开，如：

```python
from math import sqrt, pi, pow
```

此时，可以通过`sqrt()`、`pi`、`pow()`等名称直接访问这些函数或变量。

## 包(Package)
包(Package)是包含多个模块的文件夹。每一个文件夹都有一个名为`__init__.py`的文件，这个文件决定了这个文件夹是一个包。在导入包的时候，Python会搜索目录下的所有`__init__.py`，然后把它们当作一个包处理。包除了可以包含多个模块外，还可以包含子包、资源文件等内容。包的导入语法如下：

```python
import package_name # 导入整个包
from package_name import module_name # 从包中导入某个模块
```

例如，要导入`matplotlib`包，可以使用以下语句：

```python
import matplotlib as mpl
```

其中，`as`关键字用于给`matplotlib`起一个别名，方便后续代码引用。之后就可以通过`mpl.`前缀访问`matplotlib`包内的所有模块和函数。

## `__name__`变量
每个模块都有一个内置的变量`__name__`。当模块被直接运行时，`__name__`的值为`"__main__"`；如果模块是被导入的，那么`__name__`的值就是模块的名字。例如，编写一个模块`test.py`，代码如下：

```python
def myfunc():
    print("This is a test function.")
    
if __name__ == "__main__":
    myfunc()
```

模块内部有一个`myfunc()`函数，只有当模块被直接运行时，才会执行函数体。这是一种模块编程的常见模式，即在模块内部添加一个判断条件，仅当外部代码通过`import`命令加载模块时，才执行某些特定代码。这样做的目的是避免多余的代码在模块内部运行时被调用，提高模块的性能。

## 创建自己的模块
除了直接使用现有的模块外，还可以创建自己的模块。只需要创建一个`.py`文件，在文件中定义函数、类或变量即可。创建好的模块文件可以被直接运行，也可以被导入到其他地方。对于复杂的模块，还可以在模块文件的开头加上文档字符串，用于描述模块的功能和使用方法。

## 使用 `__all__` 变量控制导入

有时候，模块内部可能定义了很多函数和变量，但实际上我们只希望其中的一部分被外界所使用，而其余的则应该被隐藏起来，防止被意外调用或修改。这种情况下，我们可以通过 `__all__` 变量来实现。我们可以将不想让外界使用的函数和变量放到一个列表 `__all__` 中，这样就不会被自动导入。例如，编写一个模块`calculator.py`，代码如下：

```python
def add(x, y):
    return x + y

def subtract(x, y):
    return x - y

__all__ = ["add"]
```

这里，我们只定义了两个函数`add()`和`subtract()`，并将`subtract()`函数加入到了`__all__`列表中。这样的话，只可以通过`from calculator import add`来导入`add()`函数，而不能导入`subtract()`函数。
