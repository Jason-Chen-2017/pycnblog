# 1.简介
  

## 一、项目背景及意义
近几年来，随着互联网的发展，线上支付成为许多公司绕不过的话题，例如拼多多的支付宝、微信支付、Apple Pay等。与此同时，由于线上支付交易越来越复杂，涉及多个环节，导致商户经营管理难度加大。因此，在这种情况下，对于支付系统的性能优化，安全性保障，数据分析监测，风险控制等方面也需要综合考虑和设计。为了满足企业级支付系统的需求，实现完整的支付系统，本项目基于Apache License进行开源，目前已上线运营中。
## 二、项目目的与要求
### 目标
本项目的目标是建立一个集成化的支付系统平台，包括支付网关（支付通道）、交易系统、结算系统、账务系统、风控系统、运营监控等相关子系统，提供高可用，可靠，可扩展的支付服务。
### 要求
- 实现整体功能流程和逻辑
- 按标准配置保证项目稳定性，可靠性
- 采用微服务架构，各个子系统独立部署
- 提供后台管理页面，支撑核心业务运营和管理
- 可视化监控数据展示及报警功能，提升运维效率
- 数据安全保障，采用安全认证和权限控制机制
- 支持国际化，支持更多种类支付渠道，支持多币种支付
- 在保证业务稳定运行的前提下，增加新功能和模块，保持快速迭代，持续改进

# 2.基本概念术语说明
## 一、支付系统概念
支付系统是在电子商务和互联网金融的基础上发展起来的产物。它是指利用计算机网络技术为消费者提供有效收付款方式的一系列计算机和信息系统。主要功能包括资金划转、支付指令的路由、结算、反洗钱、风险控制等，能够为消费者提供便捷、高效、安全、可靠的支付方式。其重要特点是通过对交易信息的收集、分析处理、存储和传输，实现支付功能。
## 二、常用术语
1. **支付通道**：支付通道是指公司开展交易所需的第三方支付机构或银行，用于为顾客提供各种支付方式，例如支付宝，微信支付，网银等。不同的支付通道具有不同支付服务模式、安全措施和收费标准，并能够为商家提供不同的服务收入。
2. **收单机构**：收单机构又称中介机构，即商户和交易网络中介机构，负责支付机构之间的信息交换，完成支付通道中用户支付指令的发送与接收，完成资金的划转。收单机构分为银行间收单中心、同业存款银行、第三方支付机构、代收货款支付机构等，通常由第三方支付机构担任。
3. **交易中心**：交易中心是指专门服务于支付机构的支付交易环境，提供商户交易信息上传、查询、交易统计、数据分析、风险识别、清算处理等服务。交易中心的作用不仅仅局限于支付领域，它还承担其他业务的支撑工作。
4. **支付终端**:支付终端指个人、商户等进行支付行为的设备或终端。包括手机APP、微信公众号、网页支付、扫码支付、POS机、刷卡机等。支付终端通过调用支付接口与支付机构通信，实现支付功能。
5. **支付接口**：支付接口是指由支付机构和支付终端共同遵守的一种协议，定义了双方之间如何进行交易以及交互的方式。支付接口定义了支付指令的格式、数据结构、安全加密方法、传输协议等。支付接口对支付系统的运行起到至关重要的作用。
6. **加密算法**：加密算法是指对信息进行编码、隐藏或是使其无法被直接读取的方法。包括MD5、SHA1、DES、AES、RSA等。其中，MD5、SHA1用于对信息生成“指纹”或签名，可用于确保信息完整性。DES、AES用于对信息进行加密和解密，可用于保护敏感数据。RSA则是一种公钥加密算法，可以用于信息的加密和解密，同时也解决了密钥传递的问题。
7. **数字证书**：数字证书（Digital Certificate）是电脑用来识别身份和建立加密通信信道的重要工具，它通常由证书颁发机构CA颁发给用户。数字证书包含两部分信息：公钥和私钥。公钥用于加密数据，私钥用于解密数据。每台服务器都需要安装对应的公钥才能与客户端进行数据交互。
