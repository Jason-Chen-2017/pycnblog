# 2.为什么要使用对比库与框架？
# 对比库与框架这个话题在日常的开发工作中经常出现。下面通过一个实际的案例，来看一下为何要使用对比库与框架。

1、理解问题
假设你是一个大学生，负责某个实验室的研制项目。该项目的目标是设计一种新型的激光切割机械。你刚接触到激光切割机械相关知识时，除了一些基本的物理知识外，还有一系列的知识点需要了解，例如：热传导材料、绝缘子串、冲击波捕获技术、照明系统等。但是由于时间关系，你只能集中精力学习这些知识点中的一小部分。另一方面，你公司的同事正好有一款激光切割机械相同的产品，他们已经具备了相应的经验和技术能力，可以帮你快速了解它的工作原理。但是，你却对公司的同事提到的知识点比较困惑，无法完全理解。所以你希望通过对比这两种方案，最终找到导致这种情况出现的原因。

2、定义对比库与框架
首先，我们应该清楚对比库与框架的定义。“库”指的是代码集合，它提供了一组用来完成特定功能的代码。比如，一个使用C++编写的库，可以通过调用函数实现字符串的拼接、加密等功能。而“框架”则是运行于应用程序上的引擎，它提供了一些帮助我们开发应用程序的组件。比如，Django是一个使用Python编写的Web开发框架，它内部封装了一套HTTP协议处理函数，使得我们可以快速搭建Web应用。

3、举个例子
比如说，你和公司的同事都采用了Python语言，并且想用TensorFlow库来做机器学习。但是你还没有研究过TensorFlow库的内部机制。因此，你便向公司的同事询问一下，TensorFlow提供哪些函数用于构建神经网络模型？公司的同事向你展示了如何构建一个简单的神经网络模型，并且给出了一个示例代码：
```python
import tensorflow as tf

x = tf.constant([[1.], [2.], [3.], [4.]])
y_true = tf.constant([[-1.], [-3.], [-5.], [-7.]])
w = tf.Variable(tf.random.normal((1, 1)))
b = tf.Variable(0.)

y_pred = tf.matmul(x, w) + b

loss = tf.reduce_mean(tf.square(y_pred - y_true))
optimizer = tf.keras.optimizers.SGD(learning_rate=0.01)
trainable_variables = [w, b]
training_op = optimizer.minimize(loss, var_list=trainable_variables)

with tf.Session() as sess:
    init_op = tf.global_variables_initializer()
    sess.run(init_op)

    for i in range(100):
        _, loss_val = sess.run([training_op, loss])

        if (i % 10 == 0):
            print("Step:", i, "Loss:", loss_val)
```

上面的代码主要就是创建一个线性回归模型，用于拟合数据集（x, y_true）上的点。代码里有一些新的概念，其中包括常量tensor、变量tensor、计算图、会话、优化器、全局变量初始化、训练循环等概念。在了解这些概念后，你便明白了TensorFlow的工作原理。

4、对比方法
下面，我们将介绍几种方法，用于比较不同库或者框架之间的差异。
- 查阅文档
这是比较常用的方法。当遇到不懂的地方时，最简单的方式就是查看相关库或者框架的文档。文档一般都会详细列出库或者框架所支持的功能，而且还有一些示例代码供你参考。
- 源码分析
另一种比较有效的方法，也是调试的时候常用的方法。一般来说，如果遇到性能问题、错误、漏洞等问题，首先应该检查源码。通过阅读源代码，你可以更全面地理解库或者框架的底层工作原理。
- API查阅
由于API的统一性，不同的库或者框架之间往往都有相同的接口名称。所以可以通过查询这些接口的文档，来判断其实现方式是否一致。此外，也可以通过源码解析，观察它们的实现细节。
- 测试用例
最后，通过测试用例，可以更全面地验证库或者框架的正确性。测试用例通常是在开发阶段引入的，目的是为了验证代码是否能够正常工作。如果发现错误，就可以根据测试用例的失败信息，进一步排查问题。

5、对比应用场景
- 速度
随着硬件的发展，CPU的处理能力越来越强，GPU的运算能力越来越强。在AI领域，GPU和TPU等加速芯片的普及使得大规模神经网络的训练变得越来越容易。所以，GPU、TPU等加速芯片所带来的性能提升，势必会影响到对比库与框架之间的选择。
- 灵活性
由于对比库与框架只涉及到了基础功能的实现，所以灵活性是关键因素。比如，你可以使用NumPy、SciPy等开源库，来完成一些高级特征工程的任务；也可以使用OpenCV、Pillow等图像处理库，来完成一些计算机视觉任务。另外，有的库或框架还提供了一些额外的扩展插件，使得它们具有更高级的功能。
