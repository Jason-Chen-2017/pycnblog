# 2.基本概念和术语
## Django 
Django是一个用Python编写的高级web框架，它是一个开放源代码的web应用框架，由吉多·范罗苏姆、马斯卡尔·瓦特纳和拉斐尔·克鲁兹创造。它最初被称为“瑞典语Django”，因为它的一部分名字来自瑞典语。Django可以让你通过模型-视图-模板(MVC)的方式来组织你的代码。Django提供了强大的ORM，还内置了身份验证系统和相关工具。你可以使用Django轻松地生成数据库表格，管理数据，实现用户认证登录等功能。 

## 模型-视图-控制器 (MVC) 
MVC是一个设计模式，它将应用程序分成三个层次：模型层、视图层、控制器层。其中，模型层负责处理业务逻辑，数据持久化；视图层负责向用户呈现信息，并接收用户的输入；控制器层则负责建立模型和视图之间的联系。

### Models 
Models是Django的一个模块，它定义了一个对象，用来存储和处理网站的数据。每个模型对应数据库中的一张表，每张表都包含多个字段，每条记录都是一条数据记录。你可以定义多个模型类来表示数据库中的实体。Django提供的模型类包括：User、Group、Permission、Message、Category、Page、Post等等。 

### Views 
Views是Django应用程序的处理器，它负责处理客户端的请求，响应HTTP请求，并返回响应结果。每当有用户访问服务器上的某个网页时，Django都会自动调用对应的View处理请求，然后把结果呈现给用户。你需要定义不同的Views来处理不同类型客户端请求，如：GET请求、POST请求、上传文件请求等。 

### Templates 
Templates是Django的模板系统，它负责渲染HTML页面。Django的模板语法类似Jinja2或Twig，但又比它们复杂许多，你可以通过模板标签和变量语言来动态生成HTML页面。Templates支持继承和宏等扩展特性，可以帮助你重用代码。 

## Restful API 
Restful API(Representational State Transfer)，即表述性状态转移，是一种架构风格，它定义了一组互相连接的标准接口，通过这些接口，客户端可以直接操作服务器端资源，而无需知道服务器内部的实现细节。

Restful API与MVC模式的区别主要有以下几点：

1. 地址的统一性。MV*模式下，各个URL代表着特定功能，而且这些URL一般是硬编码在客户端应用里的。但是Restful API可以根据功能实现的模块化程度，让不同的URL指向不同的资源，从而实现了更好的可扩展性。

2. 分离的关注点。MVC模式将网络请求的处理和数据的展示分开，这使得后续的维护和扩展变得十分容易。但是Restful API与前端、后端耦合在一起，前端只能看到后端返回的数据，没有办法做进一步的数据处理。

3. 资源之间的独立性。MVC模式将数据抽象为模型对象，因此模型之间没有强关联，不利于优化查询性能。而RESTFul API倡导的是资源的独立性，每个资源的URI应该是唯一标识符，这种方式能够减少对上下文的依赖，提升系统的可伸缩性和复用性。

4. 请求方式的多样性。Restful API允许客户端使用各种类型的请求，如GET、POST、PUT、DELETE、PATCH等。而MV*模式中只允许使用GET方法。

## PostgreSQL数据库
PostgreSQL是一个开源的关系数据库管理系统，它可以提供安全、ACID兼容、高性能、并行查询、内建全文搜索功能等优点。它还提供JSON、XML、地理位置、hstore、数组类型、时间戳类型、事务机制等额外的功能。
