# 1.简介
  

Docker 是一款开源的容器技术框架，可以让开发者快速构建、测试和部署应用程序。作为一种基于Linux内核的虚拟化技术，它利用namespace和cgroup技术将应用隔离在一个独立的环境中，并提供与操作系统隔离的方式。Kubernetes（K8s）是一个自动化的容器编排引擎，它能够管理和调度容器集群。通过它们的组合，可以实现更高级的功能和服务，比如负载均衡、服务发现、日志记录、监控等。相较于传统的虚拟机技术，Docker和Kubernetes都具有以下优势：
- 高效率：Docker容器可以实现即时启动和停止，而虚拟机需要多次启动和停止才能完成相同任务。
- 可移植性：Docker的可移植性非常好，可以在任何支撑Linux内核的平台上运行。Kubernetes可以跨云端、内部和混合环境部署，实现弹性伸缩和故障转移。
- 资源共享和隔离：Docker提供了按需分配CPU、内存和网络带宽等资源的方法，提升了资源利用率；Kubernetes对容器进行编排，将资源划分到不同的节点，实现共享和隔离。

这些优势使得容器技术正在成为开发和部署分布式应用的标准工具。无论是微服务架构还是基于云计算的应用，都在广泛地应用容器技术。2017年10月，CNCF基金会宣布成立，推出了Kubernetes项目，这是容器编排领域的事实标准。而随着CNCF基金会的成立，越来越多的公司开始关注和试用K8s技术，并将其部署到生产环境中。如今，Kubernetes已成为云计算、微服务、DevOps、机器学习和物联网等领域最热门的编排技术。

# 2.基本概念术语说明
## Docker
### 什么是Docker？
Docker是一个开源的容器技术框架，可以让开发者快速构建、测试和部署应用程序。它是一个基于Linux内核的轻量级虚拟化技术，利用namespace和cgroup技术将应用隔离在一个独立的环境中，并提供与操作系统隔离的方式。

### 为什么要用Docker？
- 更高效率：Docker利用namespace和cgroup机制，通过虚拟化的方式，把应用环境和运行环境隔离开来，从而提高了启动速度和资源利用率。
- 更多选择：Dockerfile允许用户自定义镜像，满足不同场景下的需求。而且支持多种编程语言，包括Java、Python、Ruby、PHP、Node.js、Go、C++等。
- 更简单的开发流程：开发者不需要关心基础设施配置，只需编写Dockerfile文件即可打包成镜像。再通过镜像运行容器，就可以直接运行应用。

### 容器与虚拟机的区别是什么？
- 容器与虚拟机的定义：容器就是一种轻量级的虚拟化技术，而虚拟机则是指真正运行在硬件上的完整操作系统，拥有自己的内核，可以运行任意程序。
- 容器与虚拟机的区别：容器与虚拟机最大的区别在于隔离性。容器没有操作系统层面的虚拟化，因此其启动时间比虚拟机短很多。但是容器利用了宿主机操作系统的某些特性，例如PID命名空间、用户/组ID映射、设备权限控制、cgroups资源限制等，因此可以在一定程度上达到和虚拟机一样的隔离效果。但是也存在一些差异，例如容器没有虚拟机那么高的性能开销，同时还存在诸如性能优化不足、容器之间通信困难等问题。
