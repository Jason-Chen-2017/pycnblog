# 1.简介
  

随着科技的飞速发展，数据量的爆炸、分布式系统的普及、计算资源的不断增长等等新一代信息技术的诞生已经带来了巨大的挑战。同时，随着互联网行业的蓬勃发展，互联网应用也越来越丰富，越来越复杂。由此产生了一个新的需求，如何利用数据驱动业务发展？如何实现机器学习平台的研发和架构设计？

近年来，随着云计算的广泛采用，越来越多的公司开始试图利用云平台搭建机器学习平台。基于云平台部署机器学习平台可以解决基础设施管理问题、降低运维成本、提升服务质量，因此成为越来越多企业考虑的方向。

但是，对于传统的数据中心机房部署机器学习平台就存在着一定的挑战。首先，硬件配置较低，无法满足计算密集型机器学习场景的需求；其次，数据中心网络环境不稳定、带宽限制、安全风险较高，机器学习平台的部署与运行需要跨地域、跨机房部署、动态分配计算资源等一系列配套设施的支持。

因此，云原生计算基金会(CNCF)推出了Kubernetes项目，使得容器化应用在分布式环境下的部署、管理变得简单可靠。基于Kubernetes打造的机器学习平台既具有弹性可伸缩性，又具有高可用性、易于维护、易于扩展的特点。

基于容器技术、Kubernetes框架、机器学习框架的融合，当前，机器学习平台的研发正在向前迈进。基于Kubernetes集群构建的机器学习平台，可以在不同集群之间通过调度算法进行资源动态分配、弹性伸缩。因此，机器学习平台可以有效应对计算密集型任务、处理海量数据的应用场景。

除此之外，机器学习平台还可以集成不同的模型训练框架、统一管理平台权限、日志收集与分析、数据集成、监控告警、交付和运营管理等一系列功能，帮助用户更好的实现机器学习平台的运维管理。

综上所述，我是一个热衷于机器学习领域技术的“津滨”人工智能工作室(AI Studio)创始成员，云原生计算基金会(CNCF)会员，Kubernetes Contributor。我在机器学习平台的研发和架构设计方面积累了丰富的经验。在此期待能够与各路英才充分讨论、探讨。期待您的参与！欢迎您的留言与建议。








2020年8月26日
　　云原生计算基金会官方网站宣布发布2020年云原生计算基金会工作报告并开源Cloud Native Computing Foundation annual report。

　　报告涵盖了云原生计算基金会自2017年创建至今，包括开源项目、研究人员、公司和其他组织在云原生计算领域的贡献，旨在展示云原生技术和社区在生产中使用的实际情况。该报告对CNCF技术领导力、整体愿景、价值观、盟友、活动、参与者和发展方向提供了全面的回顾。

　　报告认为，云原生计算基金会是一个独立的非营利组织，其目标是促进云原生技术的健康发展，并将其作为一个厂商主导的行业标准。“云原生计算基金会的立场是帮助各个云原生供应商和开源社区建立更紧密的合作关系，共同推进云原生技术的发展。”

　　云原生计算基金会的理事会每两周举行一次，讨论各种议题，其中一个重要议题就是云原生技术的发展方向。“我们希望借助最新的技术和方法，让云原生技术更加高效、更加便捷地被应用到实际的生产环境中。”

　　CNCF项目的数量激增，使得云原生技术的发展更加复杂。根据报告，“整个CNCF生态系统的规模正处于历史性增长的阶段，代表了云原生技术在整个行业中的占有率。”

　　另外，“除了CNCF基金会项目之外，还有许多社区成员、个人、公司和组织在这个生态系统中提供技术支持。这些开放社区的努力推动了CNCF项目的发展，并促进了整个生态系统的发展。”

　　报告指出，云原生技术已经成为主流，正在影响着许多行业和领域。“现在，越来越多的组织、企业和个人开始意识到，在拥抱云原生技术时，需要进行大胆尝试，而无需过多投入资金或时间，快速地获得收益。”

　　报告还提出了一系列建议，建议由CNCF工作组、技术委员会、云原生产品小组以及创业公司共同制定：

1. 把关注放在用户而不是项目上：“很多人抱怨云原生技术难用，其实只要注意引导用户使用它就可以解决这个问题。我们可以制定一份简单易懂的手册，教导用户如何快速入门，然后根据用户的反馈改进文档。这样，用户就不用再担心云原生技术是否适合他们了。”

2. 在社区和用户之间架起桥梁：“为了让更多的人了解云原生技术，我们需要做一些大胆尝试。我们可以邀请用户参与我们的社区，鼓励他们分享自己的知识和经验。我们也可以通过我们的推荐机制帮助用户找到他们感兴趣的项目。”

3. 完善技术规范：“尽管我们为开源项目提供很多指南和文档，但这些仍然不能覆盖所有的情况。为了提升云原生技术的完整性，我们需要制定更详细的技术规范和最佳实践，以确保开发者都知道如何正确地使用它。”

4. 支持与用户的沟通交流：“我们需要倾听用户的声音，反馈他们对技术的看法。并且我们应该重视文档的版本控制，从而避免旧版文档对用户的误导。”

5. 推动开源技术的发展：“如果云原生技术只能依赖于闭源软件，那么就没有太多人会真正使用它。因此，我们需要推动开源技术的发展，并支持那些致力于推动云原生技术发展的优秀组织。”

6. 构建生态系统和工具：“我们需要构建云原生技术生态系统，以连接不同开源项目之间的工具和技术。目前，只有少数云原生技术框架和工具被选中加入生态系统，这导致生态系统并不完善。”

7. 为创业公司提供支持：“由于创业公司的特殊性，它们往往不能受益于市场上的大量技术培训，因此我们需要帮助他们获得成功。”

8. 提供优惠政策：“许多初创公司或初级雇员可能不想花太多的时间去学习和掌握云原生技术，我们可以提供一些免费的培训课程或免费的咨询服务，帮助他们快速入门。”

9. 尊重不同角色和层次的人士：“云原生技术对所有人的价值都很重要，我们需要尊重那些刚刚接触云原生技术，正在学习它的工程师，以及那些对技术有强烈兴趣但又缺乏经验的用户。”
