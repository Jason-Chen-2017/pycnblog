# 2.核心概念与联系

在了解如何在Table Store中实现数据流处理与分析之前，我们需要了解一些核心概念和联系。

## 2.1 Table Store

Table Store是一种高性能的数据存储系统，它的核心特点是提供低延迟、高吞吐量的数据存取能力。Table Store通常用于存储大量数据，如日志、事件、传感器数据等。它的主要特点包括：

- 高性能：Table Store通常使用SSD存储设备，提供低延迟的数据存取能力。
- 高吞吐量：Table Store支持并行访问，可以支持高吞吐量的数据存取操作。
- 高可扩展性：Table Store支持水平扩展，可以根据需求增加更多的存储设备。

## 2.2 数据流处理与分析

数据流处理与分析是一种处理大规模、实时数据的方法，它的核心思想是将数据流看作是一个无限序列，并在这个序列上进行实时分析和处理。数据流处理与分析的主要特点包括：

- 实时性：数据流处理与分析需要在数据到达时进行处理，不能等待所有数据到手再进行处理。
- 大规模性：数据流处理与分析需要处理大量的数据，通常需要使用分布式系统来实现。
- 流式计算：数据流处理与分析需要在流式计算环境中进行，即数据在处理过程中不能被存储在磁盘上，而是在内存中进行处理。
