## 2. 核心概念与联系

在“软件系统架构黄金法则：API设计”方法中，我们将API设计分为以下几个核心概念：

- **可用性（Availability）**：API的可用性是指API在给定时间范围内能够正常工作的概率。可用性是API设计的一个关键要素，因为如果API不可用，那么它就无法被其他系统和应用程序使用。

- **性能（Performance）**：API的性能是指API在处理请求和响应的速度和效率。性能是API设计的另一个关键要素，因为如果API的性能不佳，那么它可能会导致用户体验不佳，甚至导致系统崩溃。

- **安全性（Security）**：API的安全性是指API在处理请求和响应时能够保护数据和系统资源的能力。安全性是API设计的一个重要要素，因为如果API不安全，那么它可能会导致数据泄露、系统攻击等严重后果。

- **可扩展性（Scalability）**：API的可扩展性是指API在处理更多请求和响应的能力。可扩展性是API设计的一个关键要素，因为如果API不可扩展，那么它可能会在处理更多请求和响应时导致性能下降、系统崩溃等问题。

- **易用性（Usability）**：API的易用性是指API的文档、示例和其他资源的质量。易用性是API设计的一个重要要素，因为如果API不易用，那么其他开发人员可能会困难地使用它，从而导致开发时间延长、代码质量下降等问题。
