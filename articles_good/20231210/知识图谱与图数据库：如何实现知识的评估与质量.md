## 2. 核心概念与联系

在本节中，我们将介绍知识图谱和图数据库的核心概念，以及它们之间的联系。

### 2.1 知识图谱

知识图谱是一种数据结构，它将实体（如人、地点、组织等）与实体之间的关系以及实体属性表示为图。知识图谱可以帮助人们更好地理解和查询数据，并为AI和ML提供了更强大的能力。知识图谱的主要组成部分包括：

- 实体：实体是知识图谱中的基本组成部分，它们可以是人、地点、组织等。实体可以具有属性，如名称、年龄、地址等。
- 关系：关系是实体之间的连接，它们可以表示实体之间的联系，如人与人的亲属关系、地点与地点的距离关系等。
- 属性：属性是实体的特征，它们可以用来描述实体的特征，如人的职业、地点的面积等。

### 2.2 图数据库

图数据库是一种非关系型数据库，它将数据存储为图形结构，以便更好地处理复杂的关系和连接。图数据库的主要组成部分包括：

- 节点：节点是图数据库中的基本组成部分，它们可以表示实体。节点可以具有属性，如名称、年龄、地址等。
- 边：边是节点之间的连接，它们可以表示实体之间的关系。边可以具有属性，如权重、方向等。
- 图：图是节点和边的集合，它可以用来表示实体之间的关系。图可以用来表示复杂的关系和连接。

### 2.3 知识图谱与图数据库的联系

知识图谱和图数据库之间的联系在于它们都使用图形结构来表示数据。知识图谱主要关注实体之间的关系和属性，而图数据库主要关注节点之间的连接和属性。知识图谱可以被看作是图数据库的一种特殊应用，它将实体、关系和属性表示为图形结构。

我们可以使用以下代码来构建知识图谱：

```python
from rdflib import Graph, Namespace, Literal
from rdflib.namespace import RDF, RDFS

# 创建一个知识图谱实例
kg = Graph()

# 创建一个命名空间
ns = Namespace("http://example.com/")

# 添加实体
kg.add((ns.entity1, RDF.type, RDFS.Class))
kg.add((ns.entity2, RDF.type, RDFS.Class))

# 添加关系
kg.add((ns.entity1, ns.relationship, ns.entity2))

# 添加属性
kg.add((ns.entity1, ns.property, Literal("value1")))
```

这段代码首先导入了`rdflib`库，然后创建了一个知识图谱实例。接着，我们创建了一个命名空间，并添加了实体、关系和属性。最后，我们使用`add`方法来添加实体、关系和属性到知识图谱中。
