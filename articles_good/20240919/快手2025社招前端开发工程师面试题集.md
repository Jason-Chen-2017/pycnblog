## 3. 核心算法原理 & 具体操作步骤

### 3.1 算法原理概述

**排序算法：** 对一组数据进行排序。

**查找算法：** 在数据结构中查找特定数据。

**数据结构：** 存储和组织数据的方式。

### 3.2 算法步骤详解

**排序算法：**
- 冒泡排序：重复遍历要排序的数列，一次比较两个元素，如果它们的顺序错误就把它们交换过来。
- 快速排序：通过一趟排序将待排序的记录分割成独立的两部分，其中一部分记录的排序码小于另一部分所有记录的排序码。

**查找算法：**
- 顺序查找：从数据结构的一端开始，依次扫描各数据元素，直到找到所需的元素或扫描结束。
- 二分查找：将数据结构分为两部分，根据待查找元素的关键字与中间元素的值进行比较，决定接下来应当在数据结构的哪一部分中继续搜索。

### 3.3 算法优缺点

**排序算法：**
- 冒泡排序：简单易懂，但效率较低。
- 快速排序：效率较高，但可能存在递归深度问题。

**查找算法：**
- 顺序查找：简单易实现，但效率较低。
- 二分查找：效率较高，但适用于有序数据结构。

### 3.4 算法应用领域

**排序算法：**
- 数据排序：对大量数据进行排序。

**查找算法：**
- 数据查找：在数据结构中查找特定数据。

## 4. 数学模型和公式 & 详细讲解 & 举例说明

### 4.1 数学模型构建

**冒泡排序：** 假设数据序列为\[a1, a2, ..., an\]，每次遍历从序列的一端开始，比较相邻的两个元素，如果它们的顺序错误就把它们交换过来。

**快速排序：** 假设数据序列为\[a1, a2, ..., an\]，选择一个基准元素，将序列分为两部分，一部分记录的排序码小于基准元素，另一部分记录的排序码大于基准元素。

### 4.2 公式推导过程

**冒泡排序：** 时间复杂度 \(T(n) = O(n^2)\)。

**快速排序：** 平均时间复杂度 \(T(n) = O(n\log n)\)，最坏时间复杂度 \(T(n) = O(n^2)\)。

### 4.3 案例分析与讲解

**冒泡排序：**
```javascript
function bubbleSort(arr) {
  for (let i = 0; i < arr.length; i++) {
    for (let j = 0; j < arr.length - i - 1; j++) {
      if (arr[j] > arr[j + 1]) {
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
      }
    }
  }
  return arr;
}

const arr = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5];
console.log(bubbleSort(arr)); // [1, 1, 2, 3, 3, 4, 5, 5, 5, 6, 9]
```

**快速排序：**
```javascript
function quickSort(arr) {
  if (arr.length <= 1) {
    return arr;
  }
  const pivot = arr[arr.length - 1];
  const left = [];
  const right = [];

  for (let i = 0; i < arr.length - 1; i++) {
    if (arr[i] < pivot) {
      left.push(arr[i]);
    } else {
      right.push(arr[i]);
    }
  }

  return [...quickSort(left), pivot, ...quickSort(right)];
}

const arr = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5];
console.log(quickSort(arr)); // [1, 1, 2, 3, 3, 4, 5, 5, 5, 6, 9]
```
