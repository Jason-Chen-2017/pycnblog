## 3.1 实体关系模型
实体关系模型的核心概念包括实体、属性、关系、主键、外键等。实体是数据库中的基本数据对象，它可以表示实际世界中的一个事物或概念。属性是实体的一种特征，它可以表示实体的属性值。关系是实体之间的联系，它可以表示实体之间的关系。主键是实体的唯一标识，它可以用来唯一标识实体。外键是关系的唯一标识，它可以用来唯一标识关系。

实体关系模型的数学模型公式如下：

$$
E = \{e_1, e_2, ..., e_n\} \\
A = \{a_1, a_2, ..., a_m\} \\
R = \{r_1, r_2, ..., r_k\} \\
PK = \{p_1, p_2, ..., p_l\} \\
FK = \{f_1, f_2, ..., f_m\}
$$

其中，$E$ 表示实体集合，$A$ 表示属性集合，$R$ 表示关系集合，$PK$ 表示主键集合，$FK$ 表示外键集合。

## 3.2 三范式
三范式是数据库设计的一个重要原则，它要求数据库中的表应该是第一范式、第二范式和第三范式。

1. **第一范式**：第一范式要求数据库中的表应该是不可分割的，即表中的每一列都应该是不可分割的。

2. **第二范式**：第二范式要求数据库中的表应该是完全依赖性，即表中的每一列都应该完全依赖于主键。

3. **第三范式**：第三范式要求数据库中的表应该是最小化依赖性，即表中的每一列都应该最小化依赖于非主键的非主键属性。

## 3.3 数据库设计模式
数据库设计模式包括基本模式、组合模式和高级模式等。

1. **基本模式**：基本模式是数据库设计中的基本单位，它包括一张表和一组查询语句。基本模式可以用来表示数据库中的基本数据结构和操作。

2. **组合模式**：组合模式是数据库设计中的一种复合模式，它可以用来表示数据库中的复杂数据结构和操作。组合模式可以通过组合基本模式来实现。

3. **高级模式**：高级模式是数据库设计中的一种高级模式，它可以用来表示数据库中的高级数据结构和操作。高级模式可以通过组合基本模式和组合模式来实现。

## 4.1 实体关系模型
假设我们要设计一个学生管理系统，其中包括学生、课程和成绩三个实体。我们可以通过以下代码实现实体关系模型：

```sql
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(255),
    age INT
);

CREATE TABLE course (
    id INT PRIMARY KEY,
    name VARCHAR(255),
    credit INT
);

CREATE TABLE score (
    student_id INT,
    course_id INT,
    grade INT,
    FOREIGN KEY (student_id) REFERENCES student(id),
    FOREIGN KEY (course_id) REFERENCES course(id)
);
```

在上述代码中，我们创建了三个表：`student`、`course` 和 `score`。`student` 表包含学生的基本信息，`course` 表包含课程的基本信息，`score` 表包含学生和课程之间的成绩信息。`student` 表和 `course` 表的主键分别是 `id`，而 `score` 表的主键是 `student_id` 和 `course_id`。`score` 表中的 `student_id` 和 `course_id` 都是外键，它们分别引用 `student` 表和 `course` 表的主键。

## 4.2 三范式
假设我们要设计一个销售管理系统，其中包括客户、订单和订单详情三个实体。我们可以通过以下代码实现三范式：

```sql
CREATE TABLE customer (
    id INT PRIMARY KEY,
    name VARCHAR(255),
    address VARCHAR(255)
);

CREATE TABLE order (
    id INT PRIMARY KEY,
    customer_id INT,
    order_date DATE,
    total_amount DECIMAL(10,2),
    FOREIGN KEY (customer_id) REFERENCES customer(id)
);

CREATE TABLE order_detail (
    order_id INT,
    product_id INT,
    quantity INT,
    price DECIMAL(10,2),
    FOREIGN KEY (order_id) REFERENCES order(id),
    FOREIGN KEY (product_id) REFERENCES product(id)
);
```

在上述代码中，我们创建了三个表：`customer`、`order` 和 `order_detail`。`customer` 表包含客户的基本信息，`order` 表包含订单的基本信息，`order_detail` 表包含订单详情的基本信息。`customer` 表的主键是 `id`，`order` 表的主键是 `id`，而 `order_detail` 表的主键是 `order_id` 和 `product_id`。`order` 表中的 `customer_id` 是外键，它引用 `customer` 表的主键。`order_detail` 表中的 `order_id` 和 `product_id` 都是外键，它们分别引用 `order` 表和 `product` 表的主键。
