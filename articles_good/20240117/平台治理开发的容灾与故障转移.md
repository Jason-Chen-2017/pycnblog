# 2. 核心概念与联系

## 2.1 容灾

容灾是指在系统故障时，通过预先制定的备份和恢复计划，快速恢复系统的能力。容灾涉及到数据备份、系统恢复、故障检测等方面。通过容灾措施，我们可以确保系统在遇到故障时能够快速恢复，从而提高系统的可用性和稳定性。

## 2.2 故障转移

故障转移是指在系统故障时，将故障请求从故障发生的地方转移到其他地方以便进行处理的过程。故障转移涉及到故障检测、故障定位、故障处理等方面。通过故障转移措施，我们可以确保系统在遇到故障时能够快速恢复，从而提高系统的可用性和稳定性。

## 2.3 容灾与故障转移的联系

容灾与故障转移是平台治理开发中相互联系的两个环节。容灾涉及到数据备份、系统恢复等方面，而故障转移涉及到故障检测、故障定位、故障处理等方面。在实际工作中，我们需要将容灾和故障转移相结合，以便更好地处理系统的故障，并确保系统的可用性和稳定性。
