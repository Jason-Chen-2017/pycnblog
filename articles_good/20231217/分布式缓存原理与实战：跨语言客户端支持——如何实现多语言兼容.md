## 2.1 分布式缓存的核心概念

### 2.1.1 缓存数据结构

缓存数据结构是分布式缓存中的核心组成部分，主要包括以下几种：

- **键值对**：缓存数据通常以键值对的形式存储，其中键用于唯一地标识数据，值用于存储数据本身。
- **链表**：当缓存数据过多时，可以使用链表来存储数据，以提高查找效率。
- **哈希表**：哈希表是一种常用的缓存数据结构，可以通过键值来存储和查找数据。
- **二叉树**：二叉树是另一种常用的缓存数据结构，可以通过键值来存储和查找数据。

### 2.1.2 缓存一致性

缓存一致性是分布式缓存中的重要概念，主要包括以下几种：

- **强一致性**：强一致性要求在任何时刻，缓存中的数据都必须与原始数据一致。
- **弱一致性**：弱一致性允许缓存中的数据与原始数据不完全一致，但是在大多数情况下，数据仍然能够被正确使用。
- **最终一致性**：最终一致性要求在一段时间内，缓存中的数据会与原始数据一致。

### 2.1.3 缓存策略

缓存策略是分布式缓存中的重要组成部分，主要包括以下几种：

- **LRU**：LRU（Least Recently Used，最近最少使用）策略是一种常用的缓存策略，它根据数据的访问频率来决定哪些数据需要被缓存。
- **LFU**：LFU（Least Frequently Used，最少使用）策略是一种另一种常用的缓存策略，它根据数据的使用频率来决定哪些数据需要被缓存。
- **LRU-K**：LRU-K策略是一种基于LRU策略的缓存策略，它根据数据的访问频率和大小来决定哪些数据需要被缓存。

class DistributedCache:
    def __init__(self):

    def set(self, key, value):

    def get(self, key):
        if key in self.cache:
            return self.cache[key]
        else:
            return None
