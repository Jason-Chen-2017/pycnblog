#### 3. 使用协程和通道实现并发下载

**题目：** 使用协程和通道实现并发下载多个文件，将每个文件的内容拼接成一个字符串，并打印出来。

**答案：**

```go
package main

import (
    "fmt"
    "io/ioutil"
    "net/http"
    "strings"
)

func download(url string, c chan<- string) {
    resp, err := http.Get(url)
    if err != nil {
        c <- err.Error()
        return
    }
    defer resp.Body.Close()

    body, err := ioutil.ReadAll(resp.Body)
    if err != nil {
        c <- err.Error()
        return
    }

    c <- string(body)
}

func main() {
    urls := []string{
        "http://example.com",
        "http://example.org",
        "http://example.net",
    }
    c := make(chan string)
    var wg sync.WaitGroup

    for _, url := range urls {
        wg.Add(1)
        go func(u string) {
            defer wg.Done()
            download(u, c)
        }(url)
    }

    go func() {
        wg.Wait()
        close(c)
    }()

    result := ""
    for msg := range c {
        result += msg + "\n"
    }

    fmt.Println(result)
}
```

**解析：** 在此例中，使用协程并发下载多个文件，并将每个文件的内容拼接成一个字符串。通过通道 `c` 传递下载结果，主 Goroutine 接收通道数据，并将结果打印出来。

### 实例分析

**题目：** 分析以下代码，说明其执行过程和结果。

```go
package main

import (
    "fmt"
)

func main() {
    channels := []chan int{
        make(chan int, 2),
        make(chan int, 0),
    }

    go func() {
        channels[0] <- 1
        channels[0] <- 2
        channels[1] <- 3
    }()

    fmt.Println(<-channels[0])
    fmt.Println(<-channels[1])
}
```

**答案：** 

```plaintext
1
3
```

**解析：**

1. 创建两个通道 `channels[0]` 和 `channels[1]`，其中 `channels[0]` 是一个带缓冲的通道（容量为 2），`channels[1]` 是一个无缓冲的通道。
2. 启动一个匿名协程，该协程向 `channels[0]` 发送两个值 `1` 和 `2`，然后向 `channels[1]` 发送值 `3`。
3. 主 Goroutine 从 `channels[0]` 接收一个值，输出 `1`。
4. 主 Goroutine 从 `channels[1]` 接收一个值，输出 `3`。

由于 `channels[0]` 是带缓冲的通道，可以同时进行发送和接收操作，因此第一个发送的值 `1` 能够立即被接收。而 `channels[1]` 是无缓冲的通道，发送操作会导致 Goroutine 阻塞，直到有接收操作准备好接收数据。因此，在主 Goroutine 接收 `channels[1]` 的值之前，匿名协程会等待，直到发送操作完成。

由于 `channels[0]` 是带缓冲的通道，可以同时进行发送和接收操作，因此第一个发送的值 `1` 能够立即被接收。而 `channels[1]` 是无缓冲的通道，发送操作会导致 Goroutine 阻塞，直到有接收操作准备好接收数据。因此，在主 Goroutine 接收 `channels[1]` 的值之前，匿名协程会等待，直到发送操作完成。
