### 4. 算法编程题库

#### **题目：** 编写一个Python函数，实现基于用户行为数据的社区活跃度分析。

**题目描述：** 
假设我们有一个包含用户行为数据的数据集，数据集包括以下字段：用户ID（uid）、行为类型（type，如发帖、评论、点赞）、行为时间（timestamp）。请编写一个函数，计算每个用户在一定时间窗口内的活跃度，活跃度定义为该用户在时间窗口内的行为次数。

**输入：** 
- 用户行为数据列表，每个行为数据是一个字典，如：`[{uid: 1, type: 'post', timestamp: 1625347823}, {uid: 2, type: 'comment', timestamp: 1625347824}, ...]`
- 时间窗口长度，以秒为单位

**输出：** 
- 一个字典，键为用户ID，值为用户在时间窗口内的活跃度，即行为次数。

**示例：** 
```python
data = [
    {'uid': 1, 'type': 'post', 'timestamp': 1625347823},
    {'uid': 2, 'type': 'comment', 'timestamp': 1625347824},
    {'uid': 1, 'type': 'post', 'timestamp': 1625347825},
    {'uid': 2, 'type': 'comment', 'timestamp': 1625347826},
    {'uid': 3, 'type': 'like', 'timestamp': 1625347827},
]

window_length = 60

result = calculate_activity(data, window_length)
print(result)
```

**期望输出：** 
```python
{1: 2, 2: 2, 3: 1}
```

#### **答案：**

```python
def calculate_activity(data, window_length):
    activity = {}
    for item in data:
        uid = item['uid']
        timestamp = item['timestamp']
        if uid not in activity:
            activity[uid] = []
        activity[uid].append(timestamp)
    
    result = {}
    for uid, timestamps in activity.items():
        count = 0
        timestamps.sort()
        for i in range(len(timestamps) - 1):
            if timestamps[i+1] - timestamps[i] <= window_length:
                count += 1
            else:
                break
        result[uid] = count
    return result

# 测试代码
data = [
    {'uid': 1, 'type': 'post', 'timestamp': 1625347823},
    {'uid': 2, 'type': 'comment', 'timestamp': 1625347824},
    {'uid': 1, 'type': 'post', 'timestamp': 1625347825},
    {'uid': 2, 'type': 'comment', 'timestamp': 1625347826},
    {'uid': 3, 'type': 'like', 'timestamp': 1625347827},
]

window_length = 60

result = calculate_activity(data, window_length)
print(result)
```

#### **解析：**

该函数首先遍历输入数据，根据用户ID（uid）将行为时间（timestamp）存储在一个字典中。接着，对每个用户的行为时间进行排序，并计算在一定时间窗口（window_length秒）内的行为次数。具体步骤如下：

1. **初始化一个空的字典activity，用于存储每个用户的行为时间列表**。
2. **遍历输入数据，将每个用户的行为时间添加到对应的列表中**。如果用户ID不在字典中，则添加一个新的列表。
3. **对每个用户的行为时间列表进行排序**。
4. **遍历排序后的行为时间列表，计算在时间窗口内的行为次数**。如果在时间窗口内有两个连续的行为时间，则计数增加1。如果下一个行为时间超过了时间窗口，则停止计数。
5. **将计算得到的活跃度（行为次数）存储在结果字典中**。

该函数的时间复杂度主要在于对行为时间列表的排序操作，为O(nlogn)，其中n是用户总数。其余步骤的时间复杂度为O(n)，因此整体的时间复杂度为O(nlogn)。
