# 1.Linux 
## 1.1 Linux发行版
​    Linux发行版是基于Linux内核的开源操作系统，它可以让用户得到最好的兼容性与稳定性。不同发行版之间差异很大，比如有些发行版采用滚动升级（Redhat Enterprise Linux）的方式，而有些发行版采用全盘升级（Ubuntu）的方式，在配置上也存在较大的差异。不过总体来说，linux发行版分为两大类，主要区别在于它们各自支持的组件及其数量上。

1. Debian系列
​      Debian(又称GNU/Linux Distributions，缩写为GNU+Linux)是一个基于GNU通用公共许可证版本3（GPLv3）的Linux发行版。它的独特之处在于专注于开放源代码的自由软件理念，所有软件都免费，用户可以任意使用、修改或再发布。Debian是一个精心设计的系统，使它成为世界上安装最多的操作系统。它有着庞大的软件仓库，且允许第三方软件的安装。而且由于其使用GPLv3授权协议，因此任何人都可以自由地修改源代码。它的桌面环境是GNOME，网络浏览器是Mozilla Firefox。

2. Ubuntu
​     Ubuntu是一个基于Debian的Linux发行版。它的定位与Debian截然不同，Ubuntu着重于轻量化，安装速度快，启动速度快，同时还针对个人用户进行了优化。它的桌面环境是Unity，网络浏览器是Firefox，支持中文输入法。另外，除了默认支持命令行外，Ubuntu还提供了图形界面支持——Canonical公司推出的LXDE、KDE Plasma和XFCE桌面环境，以及GNOME桌面环境。

## 1.2 Linux命令
​    在日常生活中，你肯定会使用到各种命令，包括搜索、删除、复制、压缩文件等等。Linux的命令行工具也是如此，用户可以通过终端窗口输入命令，完成相应的任务。不同的命令代表不同的功能，比如ls命令用于显示当前目录的文件列表；cp命令用于复制文件或文件夹；rm命令用于删除文件或文件夹等。了解Linux命令，可以提高工作效率，节约时间，提升产品ivity。以下是一些常用的命令：

1. ls 命令

​        ls 命令是list的缩写，用于列出目录中的内容。这个命令不仅可以查看目录里的文件，也可以查看文件夹属性信息，甚至还可以使用通配符。例如：

- 查看当前目录下的文件：`ls`
- 查看指定目录下的所有文件：`ls /path/to/directory/`
- 使用通配符查找多个文件：`ls *.txt`

2. cd 命令

​       cd 命令是change directory的缩写，用于切换当前工作目录。如果不加任何参数，则直接进入用户的主目录；如果指定路径，则进入该路径所在的目录。例如：

- 进入当前用户的主目录：`cd`
- 进入上级目录：`cd..`
- 进入根目录：`cd /`

3. mkdir 命令

​         mkdir 命令是make directory的缩写，用于创建新的目录。例如：

- 创建名为"test"的新目录：`mkdir test`
- 创建指定路径的子目录：`mkdir -p /path/to/parent/subdirectory/`

4. rm 命令

​          rm 命令是remove的缩写，用于删除文件或目录。注意，删除目录时，需先清空目录后才能删除成功。例如：

- 删除单个文件：`rm filename`
- 删除整个目录：`rm -r dirname`

5. cp 命令

​           cp 命令是copy的缩写，用于复制文件或目录。例如：

- 复制单个文件到指定目录：`cp file_name target_dir/`
- 复制整个目录到指定位置：`cp -r source_dir/ target_dir/`

6. mv 命令

​           mv 命令是move的缩写，用于移动文件或目录。mv与cp类似，不同之处在于mv只能改名，不能覆盖已有的同名文件。例如：

- 把文件file1改名为file2：`mv file1 file2`
- 把目录dir1移动到dir2目录下：`mv dir1 dir2/`

7. chmod 命令

​            chmod 命令用来改变文件的权限，管理文件访问权限。它有三种模式：

- `chmod u+x file`，添加用户执行权限
- `chmod go-w file`，删除其他人写权限
- `chmod 755 file`，将文件权限设置为755（用户读、写、执行；组读、执行；其它读、执行）。

### 2.1.7 编码规范
​    本小节介绍一些编码规范，帮助大家更好的编写python代码，并避免错误发生。
#### PEP8编码规范
​    PEP8是一种官方编程规范，建议遵循PEP8风格的python代码。PEP8定义了一些风格指南，比如命名规范、注释规范、空格规范等等。其中，最重要的是以下几条：

1. 命名规范

   模块名全部小写，包名全部小写，类名首字母大写，方法名和函数名均使用小写字母加下划线，实例变量以单个小写字母表示。

2. 注释规范

   每个模块、类、方法需要添加文档注释，详细描述模块、类的用途、方法的功能和调用方法等信息。类注释不需要写完整的文档描述，但是一定要包含作者姓名、联系方式、版本信息等必要的信息。

3. 空格规范

   每个语句后面增加一个空格，不要使用Tab键。每行末尾不要有多余的空格。

4. 行长限制

   每行代码控制在79个字符以内。

#### 单元测试
​    单元测试是用来保证代码正确运行的方法。通过编写测试用例，自动测试程序的某一特定功能是否正常，从而找出程序中的bug和漏洞。测试用例可以保障软件质量，降低软件开发成本，提高软件维护效率。

unittest模块提供了强大的测试框架，能够有效测试程序模块和函数的行为，并且可以生成测试报告。下面给出一个示例：

```python
 import unittest

 class TestStringMethods(unittest.TestCase):

     def setUp(self):
         self.string = 'this is a sample string'

     def tearDown(self):
         pass

     def test_upper(self):
         self.assertEqual('THIS IS A SAMPLE STRING', self.string.upper())

         self.assertNotEqual('sample', self.string.lower()[::])

     def test_isalpha(self):
         self.assertTrue(self.string.isalpha())

         self.assertFalse('@#$%^'.isalpha())

     def test_split(self):
         words = self.string.split()
         self.assertListEqual(['this', 'is', 'a','sample','string'], words)

         numbers = '1,2,3,4'.split(',')
         self.assertListEqual(['1', '2', '3', '4'], numbers)

         separator = '-_'
         sentence = 'This_is_a-sentence'
         words = sentence.split(separator)
         self.assertListEqual(['This', 'is', 'a','sentence'], words)


         expected = [{'first': 'John', 'last': 'Doe'},
                      {'first': 'Jane', 'last': 'Smith'}]
         actual = [{'first': 'John', 'last': 'Doe'},
                   {'first': 'Jane', 'last': 'Smith'}]
         self.assertDictEqual({'John': 'Doe',
                               'Jane': 'Smith'},
                              {'John': 'Doe',
                               'Jane': 'Smith'})

         self.assertAlmostEqual(1.23, 1.234, delta=0.01)

         self.assertRaises(ValueError, int, 'a')


 if __name__ == '__main__':
     unittest.main()
```

##### 测试用例命名规范
​    测试用例名必须以test开头，紧跟着模块名或者类名。如上例中的TestStringMethods测试类中，测试用例名前缀为test。

##### setup和teardown方法
​    在每个测试方法前后分别设置setup和teardown方法，用来准备和清理测试数据。例如，在setUp方法中，通常会创建一个测试对象，在tearDown方法中，则销毁测试对象。这样做的目的是，确保测试环境干净利落，避免因上一次测试造成影响。

##### assert方法
​    assert方法用于判断实际结果和预期结果是否一致。常用的断言方法如下：

1. assertTrue(expr)：判断表达式expr的值是否为True，为False时触发AssertionError。
2. assertFalse(expr)：判断表达式expr的值是否为False，为True时触发AssertionError。
3. assertEqual(a, b)：判断a和b是否相等，不相等时触发AssertionError。
4. assertNotEqual(a, b)：判断a和b是否不相等，相等时触发AssertionError。
5. assertIs(a, b)：判断a和b是否指向相同的对象，不相同时触发AssertionError。
6. assertIsNot(a, b)：判断a和b是否指向不同的对象，相同时触发AssertionError。
7. assertIsNone(obj)：判断对象obj是否为None，不是时触发AssertionError。
8. assertIsNotNone(obj)：判断对象obj是否不为None，为None时触发AssertionError。
9. assertIn(member, container)：判断成员member是否属于容器container，不属于时触发AssertionError。
10. assertNotIn(member, container)：判断成员member是否不属于容器container，属于时触发AssertionError。
11. assertGreater(a, b)：判断a是否大于b，不满足时触发AssertionError。
12. assertLess(a, b)：判断a是否小于b，不满足时触发AssertionError。
13. assertRaises(excClass, callableObj, \*args, \*\*kwds)：判断函数调用callableObj(\*args, \*\*kwds)是否抛出指定的异常excClass，不抛出时触发AssertionError。
