## 2. 核心概念与联系

在工作流引擎中，核心概念包括：工作流、任务、节点、流程、触发器、事件等。这些概念之间的联系如下：

- **工作流**：工作流是一种由一系列相关任务组成的流程，它描述了如何完成一项业务活动。工作流可以是线性的、循环的、并行的等多种形式。
- **任务**：任务是工作流中的基本单位，它表示一项具体的操作或活动。任务可以是人工执行的，也可以是自动执行的。
- **节点**：节点是工作流中的一个具体的位置，它表示一个任务的执行环节。节点可以是开始节点、结束节点、分支节点、合并节点等多种类型。
- **流程**：流程是工作流中的一个完整的执行过程，它包括一系列相关的任务和节点。流程可以是线性的、循环的、并行的等多种形式。
- **触发器**：触发器是工作流中的一个特殊节点，它可以根据一定的条件来触发任务的执行。触发器可以是时间触发器、事件触发器、条件触发器等多种类型。
- **事件**：事件是工作流中的一个特殊的信息，它可以触发任务的执行或者改变工作流的执行流程。事件可以是数据事件、状态事件、操作事件等多种类型。

```python
class WorkflowEngine:
    def __init__(self):
        self.tasks = []
        self.nodes = []
        self.triggers = []
        self.events = []

    def add_task(self, task):
        self.tasks.append(task)

    def add_node(self, node):
        self.nodes.append(node)

    def add_trigger(self, trigger):
        self.triggers.append(trigger)

    def add_event(self, event):
        self.events.append(event)

    def execute(self):
        for trigger in self.triggers:
            if trigger.is_triggered():
                task = trigger.get_task()
                node = trigger.get_node()
                event = trigger.get_event()
                task.execute(node, event)
```

- `add_task`：用于添加任务。
- `add_node`：用于添加节点。
- `add_trigger`：用于添加触发器。
- `add_event`：用于添加事件。
- `execute`：用于执行工作流。
