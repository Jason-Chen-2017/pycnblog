## 2. 核心概念与联系

### 2.1 图的基本概念

图是由节点（vertices）和边（edges）组成的数学结构。节点表示实体，边表示实体之间的关系。图可以是有向的，也可以是无向的。

*   **有向图**: 边具有方向，表示关系的方向性。
*   **无向图**: 边没有方向，表示关系的双向性。

### 2.2 图的类型

根据图的特性，可以将其分为不同的类型：

*   **简单图**: 不存在自环（连接同一节点的边）和重边（连接同一对节点的多条边）。
*   **多重图**: 允许存在重边。
*   **加权图**: 每条边都有一个权重，表示关系的强度或代价。
*   **二分图**: 节点可以分为两个不相交的集合，每条边连接两个集合中的节点。

### 2.3 图的表示方法

图可以用多种方式表示，例如：

*   **邻接矩阵**: 使用一个矩阵表示节点之间的连接关系。
*   **邻接表**: 使用链表存储每个节点的邻居节点。
*   **关联矩阵**: 使用一个矩阵表示节点和边之间的关联关系。

## 3. 核心算法原理具体操作步骤

图论中包含许多重要的算法，用于解决各种问题，例如：

### 3.1 图遍历算法

图遍历算法用于访问图中的所有节点，常见的算法包括：

*   **深度优先搜索 (DFS)**: 从一个起始节点开始，沿着每条路径尽可能深地搜索，直到到达没有未访问邻居节点的节点为止。
*   **广度优先搜索 (BFS)**: 从一个起始节点开始，先访问其所有邻居节点，然后再访问邻居节点的邻居节点，以此类推。

### 3.2 最短路径算法

最短路径算法用于找到图中两点之间的最短路径，常见的算法包括：

*   **Dijkstra算法**: 用于求解单源最短路径问题，即从一个起始节点到所有其他节点的最短路径。
*   **Bellman-Ford算法**: 用于求解单源最短路径问题，可以处理负权边。
*   **Floyd-Warshall算法**: 用于求解所有节点对之间的最短路径。

### 3.3 最小生成树算法

最小生成树算法用于找到图中连接所有节点的最小权重边集，常见的算法包括：

*   **Prim算法**: 从一个起始节点开始，逐步添加连接到当前树的最小权重边，直到所有节点都连接到树上。
*   **Kruskal算法**: 按照边的权重从小到大排序，依次添加不构成回路的边，直到所有节点都连接起来。

$$

### 4.2 邻接表

邻接表使用链表存储每个节点的邻居节点。对于每个节点 $i$，都有一个链表存储所有与节点 $i$ 相邻的节点。

### 4.3 关联矩阵

关联矩阵是一个 $n \times m$ 的矩阵，其中 $n$ 是节点的数量，$m$ 是边的数量。如果节点 $i$ 和边 $j$ 相关联，则矩阵的 $(i, j)$ 元素为 1，否则为 0。

b_{n1} & b_{n2} & \cdots & b_{nm}

```python
class Graph:
    def __init__(self):
        self.graph = {}

    def add_edge(self, u, v):
        if u not in self.graph:
            self.graph[u] = []
        self.graph[u].append(v)

    def dfs(self, start):
        visited = set()
        def dfs_helper(node):
            visited.add(node)
            for neighbor in self.graph[node]:
                if neighbor not in visited:
                    dfs_helper(neighbor)
        dfs_helper(start)

# 创建一个图
graph = Graph()
graph.add_edge('A', 'B')
graph.add_edge('A', 'C')
graph.add_edge('B', 'D')
graph.add_edge('C', 'E')

# 从节点 A 开始深度优先搜索
graph.dfs('A')
```
