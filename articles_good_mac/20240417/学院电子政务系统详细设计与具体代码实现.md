## 4.数学模型和公式详细讲解举例说明

在数据库设计中，我们使用了ER模型。ER模型是一种数据模型，它用实体（Entity）、属性（Attribute）和关系（Relationship）的概念来描述现实世界的数据。

一个实体可以用一个元组 $(E, A)$ 来表示，其中 $E$ 是实体的名称，$A$ 是实体的属性集合。一个关系可以用一个元组 $(R, E1, E2)$ 来表示，其中 $R$ 是关系的名称，$E1$ 和 $E2$ 是关系的两个实体。

在ER模型中，我们可以通过一些公式来描述实体和关系。例如，我们可以用以下公式来描述“学生选课”的关系：

$$
R_{选课} = \{(s, c) | s \in S, c \in C, s \text{ 选课 } c\}
$$

其中，$S$ 是学生实体集合，$C$ 是课程实体集合，$s \text{ 选课 } c$ 表示学生 $s$ 选了课程 $c$。

CREATE TABLE Enrollment (
  student_id INT,
  course_id INT,
  grade FLOAT,
  PRIMARY KEY (student_id, course_id),
  FOREIGN KEY (student_id) REFERENCES Student(id),
  FOREIGN KEY (course_id) REFERENCES Course(id)
);

  @Autowired

  public List<Student> getAllStudents() {
    return studentService.getAllStudents();
  }

  @PostMapping
  public Student createStudent(@RequestBody Student student) {
    return studentService.createStudent(student);
  }
}

@RestController

    return courseService.getAllCourses();
  }

  }
}
