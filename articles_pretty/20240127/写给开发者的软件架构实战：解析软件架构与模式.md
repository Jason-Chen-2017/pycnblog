### 4.1 模块化设计

模块化设计是一种将软件系统划分为多个独立模块的方法，每个模块都负责一部分功能。这种设计方法可以提高代码的可维护性和可扩展性。

以下是一个简单的模块化设计示例：

```python
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

class Order:
    def __init__(self, user, product, quantity):
        self.user = user
        self.product = product
        self.quantity = quantity
```

在这个示例中，我们将用户和订单两个功能划分为两个独立的模块。这样，我们可以更容易地维护和扩展这些功能。

### 4.2 单一职责原则

单一职责原则是一种将软件系统的功能划分为多个独立功能的方法。每个功能应该只负责一部分功能，这样可以提高代码的可维护性和可扩展性。

以下是一个简单的单一职责原则示例：

```python
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

    def update_email(self, new_email):
        self.email = new_email

class Order:
    def __init__(self, user, product, quantity):
        self.user = user
        self.product = product
        self.quantity = quantity

    def update_quantity(self, new_quantity):
        self.quantity = new_quantity
```

在这个示例中，我们将用户和订单两个功能划分为两个独立的模块。每个模块只负责一部分功能，这样可以提高代码的可维护性和可扩展性。

### 4.3 依赖注入

依赖注入是一种将依赖关系从代码中抽取出来的方法。这种方法可以提高代码的可维护性和可扩展性。

以下是一个简单的依赖注入示例：

```python
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

class Order:
    def __init__(self, user, product, quantity):
        self.user = user
        self.product = product
        self.quantity = quantity

class EmailService:
    def send_email(self, user):
        print(f"Send email to {user.email}")

class OrderService:
    def __init__(self, email_service):
        self.email_service = email_service

    def create_order(self, user, product, quantity):
        order = Order(user, product, quantity)
        self.email_service.send_email(order.user)
```

在这个示例中，我们将依赖关系从代码中抽取出来。这样可以提高代码的可维护性和可扩展性。
