## 2. 核心概念与联系

软件系统架构黄金法则是一种设计原则，它包括以下几个核心概念：

- 单一职责原则：一个模块应该只负责一个功能。
- 开放封闭原则：一个模块应该对扩展开放，对修改封闭。
- 里氏替换原则：一个子类应该能够替换其父类，而不会影响系统的正常运行。
- 接口隔离原则：一个接口应该只提供必要的功能。
- 依赖反转原则：高层模块不应该依赖低层模块，两者之间应该依赖抽象；抽象不应该依赖详细设计，详细设计应该依赖抽象。

这些原则有助于我们设计出可维护、可扩展、可靠的软件系统架构。

```python
    def add(self, a, b):
        return a + b

class CalculatorFactory:
    @staticmethod
    def create_calculator():
        return Calculator()

class Strategy:
    def execute(self, calculator):
        pass

class AddStrategy(Strategy):
    def execute(self, calculator):
        result = calculator.add(1, 2)
        print(f"The result is {result}")

class SubtractStrategy(Strategy):
    def execute(self, calculator):
        result = calculator.add(3, 4)
        print(f"The result is {result}")

def client_code(strategy):
    calculator = CalculatorFactory.create_calculator()
    strategy.execute(calculator)

if __name__ == "__main__":
    add_strategy = AddStrategy()
    client_code(add_strategy)

    subtract_strategy = SubtractStrategy()
    client_code(subtract_strategy)
```
