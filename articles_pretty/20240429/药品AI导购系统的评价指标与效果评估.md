## 2. 核心概念与联系

### 2.1 评价指标的分类

药品 AI 导购系统的评价指标可以分为以下几类：

* **准确性指标：** 衡量系统推荐的药品是否与用户的实际需求相符，例如准确率、召回率、F1 值等。
* **相关性指标：** 衡量系统推荐的药品与用户症状的相关程度，例如 NDCG（Normalized Discounted Cumulative Gain）等。
* **多样性指标：** 衡量系统推荐结果的多样性，避免推荐过于单一或集中，例如覆盖率、基尼系数等。
* **用户满意度指标：** 衡量用户对系统推荐结果的满意程度，例如满意度评分、点击率、转化率等。
* **安全性指标：** 衡量系统推荐的药品是否安全，例如不良反应率、禁忌症匹配率等。

### 2.2 评价指标之间的联系

不同的评价指标之间存在一定的联系。例如，准确率和召回率之间存在一定的权衡关系，提高准确率可能会导致召回率下降，反之亦然。因此，需要根据具体的应用场景选择合适的评价指标组合，综合评估系统的效果。

## 4. 数学模型和公式详细讲解举例说明

### 4.1 准确率和召回率

* **准确率（Precision）：** 衡量系统推荐的药品中，有多少是用户真正需要的。

$$
Precision = \frac{TP}{TP + FP}
$$

其中，TP 表示真正例（True Positive），即系统推荐的药品是用户真正需要的；FP 表示假正例（False Positive），即系统推荐的药品不是用户真正需要的。

* **召回率（Recall）：** 衡量用户真正需要的药品中，有多少被系统推荐出来。

$$
Recall = \frac{TP}{TP + FN}
$$

其中，FN 表示假负例（False Negative），即用户真正需要的药品没有被系统推荐出来。

### 4.2 F1 值

F1 值是准确率和召回率的调和平均值，可以综合考虑模型的准确率和召回率。

$$
F1 = \frac{2 * Precision * Recall}{Precision + Recall}
$$

### 4.3 NDCG

NDCG 衡量系统推荐结果的相关性，考虑了推荐结果的排序位置。

$$
NDCG = \frac{DCG}{IDCG}
$$

其中，DCG（Discounted Cumulative Gain）表示推荐结果的累积收益，IDCG（Ideal Discounted Cumulative Gain）表示理想情况下推荐结果的累积收益。

## 5. 项目实践：代码实例和详细解释说明

以下是一个简单的 Python 代码示例，演示如何计算准确率和召回率：

```python
def calculate_precision_recall(true_labels, predicted_labels):
    """
    计算准确率和召回率
    :param true_labels: 真实标签列表
    :param predicted_labels: 预测标签列表
    :return: 准确率、召回率
    """
    tp = 0
    fp = 0
    fn = 0
    for i in range(len(true_labels)):
        if true_labels[i] == 1 and predicted_labels[i] == 1:
            tp += 1
        elif true_labels[i] == 0 and predicted_labels[i] == 1:
            fp += 1
        elif true_labels[i] == 1 and predicted_labels[i] == 0:
            fn += 1
    precision = tp / (tp + fp)
    recall = tp / (tp + fn)
    return precision, recall
```
