**2.1 什么是正交性？**

正交性源于几何学，描述的是两个向量垂直的关系。在更广泛的语境中，正交性指的是不同事物之间相互独立、互不影响的特性。

**2.2 正交性与软件设计**

在软件设计中，正交性意味着不同的功能模块、组件或代码片段之间具有清晰的边界和独立性。修改一个模块不会影响其他模块的功能，从而降低了代码的耦合度，提高了系统的可维护性和可扩展性。

实现正交性并没有一个固定的算法或步骤，它更像是一种设计思想和指导原则。以下是一些常用的方法：

* **模块化设计:** 将系统分解成独立的模块，每个模块负责特定的功能。
* **接口定义:** 使用清晰的接口来定义模块之间的交互方式，避免直接访问内部数据。

* **单一职责原则:** 每个模块只负责一项职责，避免功能交叉。

```python

    def area(self):
        raise NotImplementedError

# 实现不同的形状类

    def __init__(self, side):
        self.side = side

    def area(self):
        return self.side * self.side

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14159 * self.radius * self.radius

# 使用多态性计算面积

for shape in shapes:
    print(shape.area())
```

在这个示例中，`Shape` 接口定义了 `area()` 方法，但没有具体的实现。`Square` 和 `Circle` 类分别实现了 `area()` 方法，计算各自的面积。通过使用接口，我们可以将计算面积的逻辑与具体的形状类解耦，实现了正交性。
