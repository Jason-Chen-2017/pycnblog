## 1. 背景介绍

### 1.1. 传统搜索引擎的局限性

传统的搜索引擎主要依赖于关键词匹配技术，即根据用户输入的关键词，在数据库中查找包含这些关键词的文档。这种方法存在着以下局限性：

* **无法理解语义**: 关键词匹配只能识别字面上的相同，无法理解用户搜索的真正意图。例如，用户搜索“苹果”可能是想查找水果，也可能是想查找苹果公司。
* **对错别字和同义词不友好**: 如果用户输入了错别字或者使用了同义词，关键词匹配就无法找到相关的文档。
* **结果排序不精准**: 关键词匹配只能根据关键词出现的频率进行排序，无法根据文档的相关性进行排序。

### 1.2. 语义搜索的兴起

为了克服传统搜索引擎的局限性，语义搜索应运而生。语义搜索的目标是理解用户搜索的意图，并返回与用户意图最相关的结果。

### 1.3. GLM和Embedding技术的优势

近年来，随着深度学习技术的快速发展，GLM (General Language Model) 和 Embedding 技术在语义搜索领域取得了显著成果。

* **GLM**: GLM 是一种强大的语言模型，能够理解自然语言的语义，并生成高质量的文本。
* **Embedding**: Embedding 技术可以将文本转换成向量表示，从而方便地计算文本之间的语义相似度。

## 2. 核心概念与联系

### 2.1. GLM (General Language Model)

GLM 是一种基于 Transformer 架构的语言模型，通过预训练学习大量的文本数据，从而掌握语言的语法、语义和上下文信息。GLM 可以用于各种 NLP 任务，例如：

* 文本生成
* 文本分类
* 问答系统
* 机器翻译

### 2.2. Embedding

Embedding 是一种将离散型变量（例如单词、句子、文档）映射到连续向量空间的技术。通过 Embedding，我们可以将文本转换成向量表示，从而方便地计算文本之间的语义相似度。常见的 Embedding 方法包括：

* Word2Vec
* GloVe
* FastText

### 2.3. GLM + Embedding 实现语义搜索

GLM + Embedding 可以有效地实现语义搜索，其基本思路如下：

1. 使用 GLM 对用户查询进行语义理解，提取查询的意图和关键信息。
2. 使用 Embedding 技术将查询和文档转换成向量表示。
3. 计算查询向量和文档向量之间的相似度，并根据相似度进行排序，返回最相关的结果。
