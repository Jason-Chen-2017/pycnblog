## 1. 背景介绍

### 1.1 同学录的价值

同学录，承载着青春的回忆，是连接同窗情谊的纽带。随着互联网的发展，传统的纸质同学录逐渐被电子化的同学录所取代。电子化的同学录不仅方便快捷，而且可以永久保存，方便随时查阅。

### 1.2  系统目标

本项目旨在设计和实现一个功能完善、易于使用的同学录管理系统，满足用户以下需求：

*   **信息管理:** 用户可以方便地添加、删除、修改同学信息，包括姓名、联系方式、毕业院校等。
*   **班级管理:**  系统支持创建和管理多个班级，方便用户查找和联系同班同学。
*   **搜索功能:**  用户可以通过姓名、毕业院校等关键词快速查找同学。
*   **数据安全:**  系统采用安全措施保护用户数据，防止信息泄露。

## 2. 核心概念与联系

### 2.1 实体关系图 (ERD)

同学录管理系统主要涉及以下实体：

*   **用户:**  系统的使用者，可以是学生、老师等。
*   **班级:**  用户所属的班级。
*   **同学:**  班级内的成员。

实体之间的关系如下：

*   一个用户可以属于多个班级。
*   一个班级可以有多个同学。
*   一个同学只属于一个班级。

### 2.2  核心功能模块

系统主要功能模块包括：

*   **用户注册/登录:**  用户可以通过注册账号登录系统。
*   **个人信息管理:**  用户可以管理自己的个人信息。
*   **班级管理:**  用户可以创建、加入、退出班级。
*   **同学录管理:**  用户可以添加、修改、删除同学信息。
*   **搜索功能:**  用户可以通过关键词搜索同学。

## 3. 核心算法原理具体操作步骤

### 3.1 用户注册/登录

1.  用户输入用户名、密码等信息进行注册。
2.  系统验证用户信息，如果用户名已存在，则提示用户重新输入。
3.  系统将用户信息保存到数据库。
4.  用户输入用户名、密码进行登录。
5.  系统验证用户信息，如果用户名或密码错误，则提示用户重新输入。
6.  系统生成用户登录凭证，并将用户重定向到系统首页。

### 3.2 班级管理

1.  用户点击“创建班级”按钮，输入班级名称、毕业年份等信息。
2.  系统验证班级信息，如果班级名称已存在，则提示用户重新输入。
3.  系统将班级信息保存到数据库，并将用户设置为班级管理员。
4.  用户点击“加入班级”按钮，输入班级名称或班级ID。
5.  系统验证班级信息，如果班级不存在，则提示用户重新输入。
6.  系统将用户添加到班级成员列表中。
7.  用户点击“退出班级”按钮。
8.  系统将用户从班级成员列表中移除。

### 3.3 同学录管理

1.  用户点击“添加同学”按钮，输入同学姓名、联系方式、毕业院校等信息。
2.  系统验证同学信息，如果同学姓名已存在，则提示用户重新输入。
3.  系统将同学信息保存到数据库。
4.  用户点击“修改同学”按钮，修改同学信息。
5.  系统验证同学信息，并将修改后的信息保存到数据库。
6.  用户点击“删除同学”按钮。
7.  系统将同学信息从数据库中删除。

### 3.4 搜索功能

1.  用户输入关键词，例如姓名、毕业院校等。
2.  系统根据关键词查询数据库。
3.  系统将查询结果展示给用户。

```python
# models.py

from django.db import models

class User(models.Model):
    username = models.CharField(max_length=100, unique=True)
    password = models.CharField(max_length=100)

class Class(models.Model):
    name = models.CharField(max_length=100, unique=True)
    graduation_year = models.IntegerField()
    members = models.ManyToManyField(User, related_name='classes')

class Student(models.Model):
    name = models.CharField(max_length=100)
    contact = models.CharField(max_length=100)
    graduation_school = models.CharField(max_length=100)
    class_id = models.ForeignKey(Class, on_delete=models.CASCADE, related_name='students')
```

```python
# views.py

from django.shortcuts import render, redirect
from .models import User, Class, Student

def register(request):
    if request.method == 'POST':
        username = request.POST['username']
        password = request.POST['password']
        user = User(username=username, password=password)
        user.save()
        return redirect('login')
    else:
        return render(request, 'register.html')

def login(request):
    if request.method == 'POST':
        username = request.POST['username']
        password = request.POST['password']
        user = User.objects.filter(username=username, password=password).first()
        if user:
            request.session['user_id'] = user.id
            return redirect('index')
        else:
            return render(request, 'login.html', {'error': '用户名或密码错误'})
    else:
        return render(request, 'login.html')
```
