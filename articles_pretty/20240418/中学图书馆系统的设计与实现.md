在设计和实现图书馆系统时，有几个核心概念和联系需要明确：

### 2.1 实体和关系

在系统中，主要有三个实体：图书（Book）、用户（User）和借阅记录（BorrowRecord）。这三个实体之间存在着一定的联系：用户可以借阅图书，每次借阅会生成一个借阅记录。

### 2.2 数据库设计

数据库设计是系统设计的重要部分，需要根据实体和关系来设计数据库的表结构。在本系统中，我们需要设计三个表：图书表（books）、用户表（users）和借阅记录表（borrow_records）。

### 2.3 权限管理

权限管理是任何系统都不能忽视的部分，我们需要为管理员和普通用户分配不同的权限。

### 5.3 权限的管理

权限的管理主要包括为管理员和普通用户分配不同的权限。在Spring Security中，我们可以通过角色（Role）来管理用户的权限。下面是一些代码实例：

```java
// 为用户分配角色
public void assignRole(Long userId, String roleName) {
    User user = userRepository.findById(userId).orElseThrow(() -> new RuntimeException("User not found"));
    Role role = roleRepository.findByName(roleName).orElseThrow(() -> new RuntimeException("Role not found"));
    user.getRoles().add(role);
    userRepository.save(user);
}
```
