## 4.数学模型和公式详细讲解举例说明

在设计航空公司管理系统时，我们可以使用数学模型来帮助我们理解和解决问题。例如，我们可以使用线性规划模型来优化航班的调度。

假设我们有一组航班$F=\{f_1, f_2, ..., f_n\}$，每个航班$f_i$都有一个利润$p_i$和一个时间窗$[s_i, e_i]$。我们的目标是选择一组航班，使得总利润最大，同时每个航班的时间窗不重叠。

这个问题可以通过线性规划模型来求解：

$$
\begin{align*}
\max & \sum_{i=1}^{n} p_i x_i \\
\text{s.t.} & x_i + x_j \leq 1, \forall (i, j) \text{ s.t. } e_i \leq s_j \text{ or } e_j \leq s_i, i \neq j \\
& x_i \in \{0, 1\}, i = 1, 2, ..., n
\end{align*}
$$

在这个模型中，$x_i$是一个决策变量，表示是否选择航班$f_i$。第一个约束条件保证选择的航班的时间窗不重叠。

```python
# models.py
from django.db import models

class Flight(models.Model):
    number = models.CharField(max_length=10)
    departure = models.CharField(max_length=100)
    arrival = models.CharField(max_length=100)
    time = models.DateTimeField()

class Ticket(models.Model):
    flight = models.ForeignKey(Flight, on_delete=models.CASCADE)
    price = models.DecimalField(max_digits=6, decimal_places=2)

class Passenger(models.Model):
    name = models.CharField(max_length=100)
    ticket = models.ForeignKey(Ticket, on_delete=models.CASCADE)
```

这个代码示例定义了三个模型：Flight、Ticket和Passenger。Flight模型表示航班，Ticket模型表示机票，Passenger模型表示乘客。每个模型都对应一个数据库表，用来存储航班、机票和乘客的信息。
