## 1. 背景介绍

### 1.1 高校毕业设计管理的痛点

传统的高校毕业设计管理模式，往往依赖于纸质文档和人工操作，存在着诸多痛点：

* **信息孤岛**: 学生、导师、学院之间信息流通不畅，导致沟通效率低下。
* **管理繁琐**: 选题、开题、中期检查、答辩等环节流程复杂，耗费大量人力物力。
* **数据安全性**: 纸质文档易丢失、损毁，存在数据安全隐患。
* **缺乏透明度**: 学生对毕业设计进程缺乏了解，导师对学生进展难以全面掌握。

### 1.2  毕业设计管理系统的意义

开发高校毕业设计管理系统，旨在解决上述痛点，实现以下目标：

* **提高管理效率**: 自动化流程，减少人工操作，提升工作效率。
* **信息共享**: 建立信息平台，实现学生、导师、学院之间信息互通。
* **数据安全**: 电子化存储，保障数据安全可靠。
* **过程透明**: 实时追踪毕业设计进程，增强透明度。

## 2. 核心概念与联系

### 2.1 系统角色

* **学生**: 提交选题申请，查看导师信息，上传论文，参与答辩等。
* **导师**: 发布课题，审核学生选题，指导学生毕业设计，评阅论文等。
* **学院管理员**: 管理学生和导师信息，发布通知，统计数据等。

### 2.2 系统功能模块

* **用户管理**: 实现用户注册、登录、权限管理等功能。
* **课题管理**: 导师发布课题，学生选择课题，系统进行匹配。
* **论文管理**: 学生上传论文，导师进行评阅，系统进行查重。
* **答辩管理**: 安排答辩时间地点，记录答辩成绩。
* **统计分析**: 对毕业设计相关数据进行统计分析，生成报表。

## 3. 核心算法原理具体操作步骤

### 3.1 用户登录

1. 用户输入用户名和密码。
2. 系统验证用户名和密码是否正确。
3. 若正确，则登录成功，跳转到相应界面；否则，提示错误信息。

### 3.2 课题匹配

1. 导师发布课题，设置课题人数限制。
2. 学生选择课题，提交申请。
3. 系统根据学生成绩、专业等因素，进行智能匹配。
4. 导师审核学生申请，确定最终名单。

### 3.3 论文查重

1. 学生上传论文。
2. 系统调用第三方查重接口，对论文进行查重。
3. 将查重结果反馈给学生和导师。

    // 验证用户名和密码

        // 生成token

    }
