## 2. 核心概念与联系

在设计和实现医院管理系统时，我们需要理解以下核心概念：

- **患者信息管理**：包括患者基本信息、病历、检查结果、诊断结果等管理。
- **医生信息管理**：包括医生基本信息、出诊时间、挂号、诊断等管理。
- **药品信息管理**：包括药品基本信息、库存、供应商等管理。
- **收费管理**：包括挂号费、药品费、检查费、住院费等收费管理。

这些核心概念之间存在着紧密的联系，例如，患者的诊断结果会影响到药品的使用和收费，医生的出诊时间会影响到患者的就医安排等。

## 4. 数学模型和公式详细讲解举例说明

在医院管理系统中，我们主要使用了ER模型（实体-关系模型）来设计数据库。ER模型是一种数据模型，它以实体和实体之间的关系为基础，描述了数据的逻辑结构。

其中，实体可以看作是现实世界中可以区分的事物或对象，关系则描述了实体之间的联系。例如，我们可以把“患者”、“医生”、“药品”看作是实体，“诊断”、“开药”、“收费”看作是关系。

在ER模型中，我们常用E-R图来表示实体和关系，其中实体用矩形表示，关系用菱形表示。以下是一个简化的E-R图示例：

```
患者 - 诊断 -> 医生
医生 - 开药 -> 药品
患者 - 收费 <- 药品
```

在这个示例中，我们可以看到，“患者”通过“诊断”关系与“医生”相联系，“医生”通过“开药”关系与“药品”相联系，“患者”通过“收费”关系与“药品”相联系。

在数据库操作方面，我们使用了JDBC（Java Database Connectivity）来实现Java与MySQL的连接。以下是一段查询患者信息的代码示例：

```java
// 查询患者信息
public List<Patient> getPatients() {
    List<Patient> patients = new ArrayList<>();
    
    try {
        Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/hospital", "root", "password");
        Statement stmt = conn.createStatement();
        ResultSet rs = stmt.executeQuery("SELECT * FROM Patient");
        
        while (rs.next()) {
            String id = rs.getString("id");
            String name = rs.getString("name");
            String diagnosis = rs.getString("diagnosis");
            
            patients.add(new Patient(id, name, diagnosis));
        }
        
        rs.close();
        stmt.close();
        conn.close();
    } catch (SQLException e) {
        e.printStackTrace();
    }
    
    return patients;
}
```

这段代码首先建立了与MySQL服务器的连接，然后执行了一个SQL查询语句，获取了所有患者的信息，并将其封装为患者对象，最后将这些患者对象添加到了一个列表中。
