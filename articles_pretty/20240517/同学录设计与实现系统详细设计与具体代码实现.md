## 1. 背景介绍

### 1.1 同学录的意义

同学录，承载着青春的记忆，是连接同窗情谊的纽带。在信息时代，传统的纸质同学录逐渐被电子化的同学录所取代。电子同学录不仅方便快捷，更能借助互联网的力量，将分散各地的同学们重新连接在一起。

### 1.2 系统设计目标

本系统旨在设计和实现一个功能完善、用户体验良好的电子同学录系统，满足以下目标：

*   **信息存储与管理:**  提供便捷的用户信息录入、修改、查询功能，有效管理同学信息。
*   **班级管理:**  支持创建和管理不同班级，方便用户查找和联系同班同学。
*   **互动交流:**  提供留言板、照片墙等功能，促进同学之间的互动交流，增进情谊。
*   **数据安全:**  保障用户信息安全，防止数据泄露和滥用。

## 2. 核心概念与联系

### 2.1 用户

*   **注册用户:**  通过注册账号，获得使用同学录系统的权限。
*   **用户信息:**  包括姓名、性别、生日、联系方式、毕业院校等基本信息，以及个人照片、兴趣爱好等个性化信息。

### 2.2 班级

*   **班级信息:**  包括班级名称、入学年份、毕业年份、班主任姓名等信息。
*   **班级成员:**  每个班级包含多个注册用户，用户可以选择加入自己所在的班级。

### 2.3 互动功能

*   **留言板:**  用户可以在留言板上发布留言，与其他同学交流互动。
*   **照片墙:**  用户可以上传照片，分享生活点滴，回忆青春岁月。

## 3. 核心算法原理具体操作步骤

### 3.1 用户注册

1.  用户填写注册信息，包括用户名、密码、邮箱等。
2.  系统验证用户信息，确保用户名和邮箱的唯一性。
3.  系统将用户信息存储到数据库中，并发送激活邮件到用户邮箱。
4.  用户点击激活链接，完成注册流程。

### 3.2 用户登录

1.  用户输入用户名和密码。
2.  系统验证用户名和密码是否匹配。
3.  验证成功后，系统将用户信息存储到 session 中，并跳转到用户主页。

### 3.3 班级创建

1.  用户选择创建班级，填写班级信息。
2.  系统验证班级信息，确保班级名称的唯一性。
3.  系统将班级信息存储到数据库中，并自动将创建者加入该班级。

### 3.4 留言板发布

1.  用户在留言板输入留言内容。
2.  系统将留言内容和用户信息存储到数据库中。
3.  系统将留言内容展示在留言板页面上。

### 3.5 照片上传

1.  用户选择上传照片，选择照片文件。
2.  系统将照片文件存储到服务器，并生成照片链接。
3.  系统将照片链接和用户信息存储到数据库中。
4.  系统将照片展示在照片墙页面上。

    end_year = models.IntegerField()

#### 5.2.3 留言模型

```python
from django.db import models

class Message(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.user.username}: {self.content}"
```

#### 5.2.4 照片模型

```python
from django.db import models

class Photo(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    image = models.ImageField(upload_to='photos/')
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.user.username}'s photo"
```
