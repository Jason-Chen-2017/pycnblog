## 1. 背景介绍

动态规划 (Dynamic Programming) 是一种算法设计技术，用于解决具有 **最优子结构** 和 **重叠子问题** 的问题。它将问题分解为更小的子问题，并存储子问题的解，以便在需要时重复使用，从而避免重复计算，提高算法效率。

### 1.1 最优子结构

当一个问题的最优解包含其子问题的最优解时，称该问题具有最优子结构性质。例如，求解最短路径问题时，从起点到终点的最短路径必然包含从起点到中间节点的最短路径。

### 1.2 重叠子问题

如果一个递归算法在解决问题的过程中反复求解相同的子问题，则称该问题具有重叠子问题性质。动态规划通过存储子问题的解来避免重复计算，从而提高算法效率。

## 2. 核心概念与联系

### 2.1 状态

状态表示一个子问题的解，通常用一个数组或表格来存储。例如，在求解最短路径问题时，状态可以表示从起点到某个节点的最短距离。

### 2.2 状态转移方程

状态转移方程描述了如何从一个或多个已知状态计算出新的状态。它通常是一个递归公式，表示当前状态与之前状态之间的关系。

### 2.3 边界条件

边界条件是状态转移方程的初始条件，用于确定最小子问题的解。

### 2.4 决策

决策是指在每个状态下选择下一步行动，例如选择走哪条路径。

## 3. 核心算法原理具体操作步骤

动态规划算法通常遵循以下步骤：

1. **定义状态**: 确定问题的状态表示，以及状态的含义。
2. **确定状态转移方程**: 推导出状态之间的关系，建立状态转移方程。
3. **确定边界条件**: 确定最小子问题的解，作为状态转移方程的初始条件。
4. **计算状态**: 按照状态转移方程，从边界条件开始，逐步计算所有状态的解。
5. **构造解**: 根据计算出的状态，回溯决策过程，构造问题的最终解。
