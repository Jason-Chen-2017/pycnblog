### 3.3 任务完成

在任务管理子系统中，我们使用了一种称为“负载均衡”的算法来分配任务。这个算法的数学模型如下：

$$
\text{负载指数} = \frac{\text{未完成的任务数量}}{\text{团队成员数量}}
$$

我们的目标是使得每个团队成员的负载指数尽可能接近。为了达到这个目标，我们将新的任务分配给负载指数最低的团队成员。

```python
class Task:
    def __init__(self, name, assignee):
        self.name = name
        self.assignee = assignee
        self.status = "未完成"

    def complete(self):
        self.status = "已完成"

class Team:
    def __init__(self):
        self.members = []
        self.tasks = []

    def create_task(self, name, assignee):
        if assignee not in self.members:
            raise ValueError("Assignee not in team")
        task = Task(name, assignee)
        self.tasks.append(task)

    def complete_task(self, task):
        task.complete()
```

这段代码定义了两个类：Task和Team。Task类代表一个任务，它有三个属性：name（任务名）、assignee（被分配者）和status（状态）。Team类代表一个团队，它有两个属性：members（团队成员）和tasks（任务列表）。Team类中的create_task方法用于创建新任务，complete_task方法用于完成任务。
