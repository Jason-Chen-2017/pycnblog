## 4. 数学模型和公式详细讲解举例说明

### 4.1 订单匹配模型
#### 4.1.1 问题描述
#### 4.1.2 模型建立
#### 4.1.3 求解方法

订单匹配可以抽象为一个二分图匹配问题。设毕业生集合为 $U=\{u_1,u_2,...,u_n\}$,订单集合为 $V=\{v_1,v_2,...,v_m\}$。若毕业生 $u_i$ 可以接受订单 $v_j$,则连接一条边 $(u_i,v_j)$。目标是找到一个最大匹配,使得尽可能多的毕业生被分配到订单。

该问题可以使用匈牙利算法求解。设 $M$ 为当前匹配,初始为空集。对于任意未匹配点 $u\in U$,从 $u$ 开始寻找增广路:
1. 令 $S=T=\emptyset$
2. 令 $S=S\cup \{u\}$,对 $S$ 中每个顶点 $u'$,考虑与其相连的每个未匹配点 $v$,若 $v\notin T$,则:
   - 若 $v$ 是未匹配点,则找到增广路,更新匹配 $M$,算法结束
   - 否则 $v$ 已匹配,设 $v$ 在 $M$ 中的匹配点为 $u''$,令 $S=S\cup \{u''\},T=T\cup\{v\}$
3. 若找不到增广路,则算法结束

匈牙利算法的时间复杂度为 $O(nm)$。

### 4.2 路径规划模型
#### 4.2.1 问题描述
#### 4.2.2 模型建立 
#### 4.2.3 求解方法

路径规划可以抽象为一个最短路问题。设道路网络为有向图 $G=(V,E)$,其中 $V$ 为路口集合,$E$ 为道路集合,每条道路 $(u,v)$ 有长度 $w(u,v)$。给定起点 $s$ 和终点 $t$,目标是找到一条从 $s$ 到 $t$ 的最短路径。

该问题可以使用 Dijkstra 算法求解。设 $d(v)$ 为起点 $s$ 到顶点 $v$ 的最短距离,初始 $d(s)=0,d(v)=\infty$。算法过程如下:
1. 初始化 $S=\emptyset,d(s)=0,d(v)=\infty$
2. 从 $V-S$ 中选择 $d(u)$ 最小的顶点 $u$,加入 $S$
3. 对每条边 $(u,v)\in E$,若 $d(v)>d(u)+w(u,v)$,则更新 $d(v)=d(u)+w(u,v)$
4. 重复2-3,直到 $t\in S$ 或 $S=V$

Dijkstra算法的时间复杂度为 $O(|V|^2)$,使用优先队列优化可达到 $O(|E|+|V|\log |V|)$。

### 4.3 服务评价模型  
#### 4.3.1 问题描述
#### 4.3.2 模型建立
#### 4.3.3 求解方法

服务评价可以使用协同过滤算法。设有 $m$ 个用户和 $n$ 个服务,评分矩阵 $R\in \mathbb{R}^{m\times n}$,其中 $R_{ij}$ 表示用户 $i$ 对服务 $j$ 的评分。协同过滤算法分为以下步骤:
1. 计算用户相似度矩阵 $W\in \mathbb{R}^{m\times m}$,其中
$$
W_{ij}=\frac{\sum_{k\in S_{ij}}(R_{ik}-\overline{R}_i)(R_{jk}-\overline{R}_j)}{\sqrt{\sum_{k\in S_{ij}}(R_{ik}-\overline{R}_i)^2}\sqrt{\sum_{k\in S_{ij}}(R_{jk}-\overline{R}_j)^2}}
$$
其中 $S_{ij}$ 为用户 $i$ 和 $j$ 共同评分的服务集合,$\overline{R}_i$ 为用户 $i$ 的平均评分。

2. 计算用户 $i$ 对服务 $j$ 的预测评分
$$
\hat{R}_{ij}=\overline{R}_i+\frac{\sum_{k\in S_j}W_{ik}(R_{kj}-\overline{R}_k)}{\sum_{k\in S_j}|W_{ik}|}
$$
其中 $S_j$ 为对服务 $j$ 有评分的用户集合。

协同过滤算法的时间复杂度为 $O(m^2n)$,可以使用降维、局部敏感哈希等技术进行优化。

以订单匹配模块为例,后端接口设计如下:

```java
@PostMapping("/orders/{orderId}/dispatch")
public ResponseEntity<Order> dispatchOrder(@PathVariable Long orderId, @RequestBody DispatchRequest request) {
    // 参数校验
    if (!orderService.exists(orderId)) {
        return ResponseEntity.notFound().build();
    }
    
    // 调用服务
    Order order = orderService.dispatch(orderId, request);
    
    // 返回结果
    return ResponseEntity.ok(order);
}
```

其中`DispatchRequest`为请求体,包含订单匹配需要的参数,如毕业生ID、服务时间等。`orderService.dispatch`方法内部调用匈牙利算法进行订单匹配,返回匹配后的订单详情。
