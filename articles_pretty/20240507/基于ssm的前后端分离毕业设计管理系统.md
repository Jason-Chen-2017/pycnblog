## 1. 背景介绍

### 1.1 毕业设计管理的痛点

传统的毕业设计管理模式往往依赖于纸质文档和人工操作，存在着诸多弊端：

* **信息流通不畅:** 学生、导师和学院之间信息传递效率低下，容易出现信息滞后、遗漏等问题。
* **管理效率低下:** 人工处理流程繁琐，耗费大量时间和人力成本，难以满足日益增长的毕业设计管理需求。
* **数据统计困难:** 纸质文档难以进行数据统计和分析，无法有效掌握毕业设计整体进度和质量情况。

### 1.2 前后端分离的优势

随着互联网技术的发展，前后端分离架构逐渐成为主流。其优势在于：

* **职责分离:** 前端专注于用户界面和交互体验，后端专注于业务逻辑和数据处理，提高开发效率。
* **技术栈灵活:** 前后端可以使用不同的技术栈，方便开发者选择最合适的工具和框架。
* **可维护性强:** 前后端代码解耦，方便维护和扩展，降低系统升级的难度。

### 1.3 SSM 框架的优势

SSM 框架是 Spring + SpringMVC + MyBatis 的缩写，是 Java Web 开发的常用框架组合，具有以下优势：

* **轻量级:** 框架结构简单，易于学习和使用。
* **功能强大:** 提供了丰富的功能模块，满足各种开发需求。
* **社区活跃:** 拥有庞大的开发者社区，提供丰富的学习资源和技术支持。

## 2. 核心概念与联系

### 2.1 系统架构

本系统采用前后端分离架构，前端使用 Vue.js 框架开发，后端使用 SSM 框架开发，数据库采用 MySQL。

### 2.2 功能模块

系统主要包含以下功能模块：

* **学生管理:** 学生信息管理、选题管理、论文提交、进度汇报等。
* **导师管理:** 导师信息管理、学生指导、论文评阅、成绩录入等。
* **学院管理:** 毕业设计安排、进度监控、数据统计、系统设置等。

### 2.3 技术栈

* **前端:** Vue.js、Element UI、Axios
* **后端:** Spring、SpringMVC、MyBatis、MySQL
* **开发工具:** IntelliJ IDEA、Maven、Git

## 3. 核心算法原理

### 3.1 用户权限管理

系统采用基于角色的访问控制 (RBAC) 模型进行用户权限管理，将用户分为学生、导师、管理员等角色，并为每个角色分配不同的权限。

### 3.2 论文查重

系统集成第三方论文查重接口，实现论文的自动查重功能，帮助导师和学生检测论文的原创性。

### 3.3 进度监控

系统通过定时任务和消息队列机制，实时监控毕业设计的进度情况，并及时提醒学生和导师。

## 4. 数学模型和公式

本系统不涉及复杂的数学模型和公式。
