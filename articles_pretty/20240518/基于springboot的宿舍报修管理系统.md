## 1. 背景介绍

### 1.1 高校宿舍管理的现状与挑战

随着高校规模的不断扩大，学生人数逐年增加，宿舍管理工作面临着越来越大的压力。传统的宿舍管理模式主要依靠人工记录和处理报修信息，存在着效率低下、信息不透明、难以追溯等问题。为了提高宿舍管理效率和服务质量，迫切需要一种信息化、智能化的宿舍报修管理系统。

### 1.2 Spring Boot框架的优势

Spring Boot 是一款基于 Spring 框架的快速开发框架，它简化了 Spring 应用的初始搭建和开发过程。其优势主要体现在以下几个方面：

* **自动配置:** Spring Boot 会根据项目依赖自动配置 Spring 应用，减少了大量的 XML 配置文件。
* **嵌入式服务器:** Spring Boot 内置了 Tomcat、Jetty 等 Servlet 容器，可以直接运行 Spring Boot 应用，无需单独部署 Web 服务器。
* **简化依赖管理:** Spring Boot 提供了一系列 starter 依赖，可以方便地引入项目所需的依赖，简化了依赖管理。
* **易于上手:** Spring Boot 提供了丰富的文档和示例，易于上手和学习。

### 1.3 本系统的目标

本系统旨在利用 Spring Boot 框架的优势，构建一个高效、便捷、安全的宿舍报修管理系统，实现以下目标：

* **简化报修流程:** 学生可以通过系统方便快捷地提交报修信息。
* **提高处理效率:** 管理员可以及时查看和处理报修信息，提高工作效率。
* **增强信息透明度:** 学生可以随时查看报修进度和处理结果。
* **方便数据统计:** 系统可以自动统计报修数据，为宿舍管理提供数据支持。

## 2. 核心概念与联系

### 2.1 系统用户角色

本系统主要涉及以下用户角色：

* **学生:** 提交报修信息，查看报修进度和处理结果。
* **管理员:** 接收和处理报修信息，管理报修类型和维修人员。
* **维修人员:** 接收指派的报修任务，更新维修进度和结果。

### 2.2 系统功能模块

本系统主要包括以下功能模块：

* **用户管理:** 管理学生、管理员和维修人员信息。
* **报修管理:** 提交、处理、查询和统计报修信息。
* **宿舍管理:** 管理宿舍楼宇、房间和床位信息。
* **系统设置:** 设置报修类型、维修人员等系统参数。

### 2.3 模块间关系

各模块之间存在如下关系：

* **用户管理** 模块为其他模块提供用户信息。
* **报修管理** 模块依赖于 **用户管理** 和 **宿舍管理** 模块的信息。
* **系统设置** 模块为其他模块提供配置参数。

## 3. 核心算法原理具体操作步骤

### 3.1 报修流程

学生提交报修信息后，系统会自动将报修信息分配给相应的维修人员。维修人员收到报修信息后，可以更新维修进度和结果。学生可以随时查看报修进度和处理结果。

### 3.2 报修信息分配算法

系统采用以下算法将报修信息分配给维修人员：

1. 根据报修类型确定负责该类型的维修人员列表。
2. 比较各维修人员的当前工作量，选择工作量最少的维修人员。
3. 如果所有维修人员的工作量都已饱和，则将报修信息加入待分配队列。

### 3.3 报修数据统计

系统可以根据报修信息自动统计以下数据：

* 各类报修的數量和占比。
* 各宿舍楼宇的报修情况。
* 各维修人员的维修效率。
