## 4. 数学模型和公式详细讲解举例说明
### 4.1 SUS量表模型
SUS(System Usability Scale)是一种经典的用户评估系统可用性的工具, 通过10个问题测量用户对系统的主观感知。
SUS得分的计算公式如下:
$$SUS=2.5\times (\sum_{i=1}^{10} (x_i-1)_{奇数项}+\sum_{i=1}^{10} (5-x_i)_{偶数项} ) $$
其中$x_i$代表用户对第$i$个问题的评分。奇数项为正向陈述,得分为用户评分减1;偶数项为反向陈述,得分为5减去用户评分。最终SUS分数在0-100之间,分数越高代表可用性越好。

### 4.2 任务完成时间模型
任务完成时间是评估交互效率的重要指标。Fitts定律描述了人机交互中目标选择任务完成时间与目标大小、距离之间的关系:
$$MT = a + b \log_2 (1+\frac{D}{W})$$
其中$MT$为任务完成时间,$D$为鼠标移动距离,$W$为目标宽度,$a$和$b$为经验参数。根据Fitts定律,优化界面布局、缩短操作距离、放大目标尺寸,都有助于提高交互效率。

### 4.3 眼动注视时间分布模型
眼动追踪实验中,注视时间分布是分析用户关注点的重要指标。kullback-Leibler散度可用于度量两个概率分布之间的差异:  
$$D_{KL}(P||Q)=\sum_i P(i)\log\frac{P(i)}{Q(i)}$$
其中$P$为实际眼动注视时间分布,$Q$为理想注视时间分布。$D_{KL}$的值越小,代表实际注视分布与理想分布越接近,交互界面设计越合理。

## 5. 项目实践:代码实例和详细解释说明
下面以Python为例,展示如何实现SUS量表计算的代码:

```python
def sus_score(scores):
    """
    计算SUS得分
    :param scores: 用户对10个问题的评分列表
    :return: SUS得分(0-100)
    """
    assert len(scores) == 10
    odd_scores = [score-1 for score in scores[::2]] 
    even_scores = [5-score for score in scores[1::2]]
    sus = 2.5 * sum(odd_scores + even_scores)
    return sus

# 示例:用户评分为[3, 4, 2, 2, 4, 1, 3, 4, 3, 2]
scores = [3, 4, 2, 2, 4, 1, 3, 4, 3, 2]  
sus = sus_score(scores)
print(f"SUS score: {sus:.2f}")
```

代码说明:
1. `sus_score`函数接收一个长度为10的列表`scores`,存储用户对10个SUS问题的评分。
2. 通过切片操作提取奇数项和偶数项,注意Python索引从0开始,因此奇数项对应偶数索引。
3. 根据SUS计算公式,对奇数项评分减1,偶数项用5减评分,再乘以2.5求和,得到最终的SUS分数。
4. 示例部分准备一个评分列表,调用`sus_score`计算得分,并用f-string打印输出,保留2位小数。

输出结果:
```
SUS score: 57.50
```
该用户对系统的可用性评分为57.5,属于中等水平。我们可以进一步分析用户在哪些问题上评分较低,有针对性地优化交互界面设计。

## 9. 附录:常见问题与解答
### 问题1:如何选择适合的可用性评估方法?
答:可结合以下因素进行选择:
1. 评估目的:是全面评估还是针对性优化
2. 时间和成本预算:不同方法的资源投入不同
3. 评估对象的复杂程度:复杂系统建议多种方法结合
4. 评估人员的经验和能力:缺乏经验时选择标准化程度高的

### 问题2:眼动追踪实验如何避免偏差?
答:主要措施包括:
1. recruiting 有代表性的实验参与者
2. 合理设计实验任务与流程,避免引导性
3. 控制实验环境,排除干扰因素
4. 采用科学的数据处理与统计方法
5. 谨慎解读实验结果,结合其他定性或定量数据
  
### 问题3:如何平衡LLM Chatbot的功能复杂度与界面简洁性?
答:需要权衡以下几点:
1. 对核心功能进行突出,次要功能适当隐藏
2. 保持术语、交互方式的一致性,降低认知负荷
3. 提供新手引导与渐进式的学习曲线
4. 界面美观大方,布局层次感强,减少无关视觉元素
5. 适当利用界面个性化,简化不常用的功能入口
