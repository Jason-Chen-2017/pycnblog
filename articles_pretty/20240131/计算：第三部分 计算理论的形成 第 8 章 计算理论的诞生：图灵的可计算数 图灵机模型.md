## 2. 核心概念与联系

图灵机模型是一种抽象的计算模型，它由一个无限长的纸带和一个读写头组成。纸带被划分为一个个格子，每个格子上可以写入一个符号。读写头可以在纸带上移动，并读取或写入符号。图灵机还有一个状态寄存器，用于记录当前的状态。图灵机的计算过程可以分为以下几个步骤：

1. 读取当前格子上的符号；
2. 根据当前状态和读取的符号，执行相应的操作，例如写入符号、移动读写头、改变状态等；
3. 转移到下一个格子，重复上述步骤。

图灵机模型可以模拟任何计算机算法，因为它具有以下特点：

1. 可以读取和写入任意长度的数据；
2. 可以根据读取的数据和当前状态执行任意操作；
3. 可以模拟任何有限状态自动机。

图灵机模型的提出，使得计算理论可以研究计算的本质和极限，例如可计算性、不可计算性、复杂性等问题。

图灵机模型的计算过程可以用数学模型来描述。假设图灵机的纸带上写有一个字符串 $w$，图灵机的初始状态为 $q_0$，读写头的初始位置为纸带的第一个格子。图灵机的计算过程可以表示为一个五元组 $(Q,\Sigma,\Gamma,\delta,q_0)$，其中：

- $Q$ 是状态集合；
- $\Sigma$ 是输入符号集合；
- $\Gamma$ 是纸带符号集合，其中 $\Sigma \subseteq \Gamma$；
- $\delta$ 是转移函数，它的定义域是 $Q \times \Gamma$，值域是 $Q \times \Gamma \times \{L,R\}$，表示从当前状态和读取的符号转移到下一个状态、写入的符号和读写头的移动方向；
- $q_0$ 是初始状态。

图灵机的计算过程可以用一个转移序列来表示，例如 $(q_0,w) \vdash (q_1,w') \vdash \cdots \vdash (q_n,w_n)$，其中 $\vdash$ 表示一次转移，$w'$ 和 $w_n$ 是纸带上的字符串。如果图灵机在某个状态下无法进行转移，则称其为停机状态。

图灵机模型的可计算数是指可以被图灵机计算的数，它包括所有可以用有限步计算得到的数。图灵机模型的不可计算数是指无法被图灵机计算的数，例如某些无限小数和某些无法判定的问题。

class TuringMachine:
    def __init__(self, states, symbols, tape_symbols, transitions, start_state):
        self.states = states
        self.symbols = symbols
        self.tape_symbols = tape_symbols
        self.transitions = transitions
        self.start_state = start_state
        self.current_state = start_state
        self.tape = ['B'] * 100
        self.head = 0

    def run(self, input_string):
        self.tape = ['B'] * 100 + list(input_string) + ['B'] * 100
        self.head = 100
        while True:
            symbol = self.tape[self.head]
            if (self.current_state, symbol) not in self.transitions:
                break
            new_state, new_symbol, direction = self.transitions[(self.current_state, symbol)]
            self.tape[self.head] = new_symbol
            if direction == 'L':
                self.head -= 1
            else:
                self.head += 1
            self.current_state = new_state
        return ''.join(self.tape).strip('B')

# Example usage

    tape_symbols={'0', '1', 'B'},

        ('q0', '0'): ('q1', '1', 'R'),
        ('q0', '1'): ('q2', '0', 'R'),
        ('q1', '0'): ('q1', '0', 'R'),
        ('q1', '1'): ('q1', '1', 'R'),
        ('q1', 'B'): ('q3', 'B', 'L'),
        ('q2', '0'): ('q2', '0', 'R'),
        ('q2', '1'): ('q2', '1', 'R'),
        ('q2', 'B'): ('q3', 'B', 'L'),
    },
    start_state='q0'

$$
