# 2.基本概念术语说明
## 2.1 加密算法
加密算法（英语：Encryption Algorithm），又称密钥交换算法或加解密算法，是将明文（plaintext）转换为密文（ciphertext）的一套算法。加密算法在加密过程中采用了各种数学方法，保证信息在传输过程中的保密性、完整性和可用性。其目的是为了防止第三方读取通信内容或者利用通信内容进行通信诈骗等。目前，主流的加密算法有：
### 2.1.1 分组密码算法
分组密码（block cipher）是一种加密算法，它在每一次运算时仅处理一个固定大小的数据块，即所谓的分组。分组密码一般用于处理长消息，比如密钥长度超过128位的AES算法就是典型的分组密码算法。分组密码算法可以分为两类，即对称分组密码算法和不对称分组密码算法。对称分组密码算法，又称为电码本体密码算法，在加密和解密过程中使用的密钥相同；而不对称分组密码算法，在加密和解密过程中使用的密钥不相同。常用对称分组密码算法包括：
- DES(Data Encryption Standard)：美国IBM公司于1977年开发的分组密码算法。它是一个块密码算法，工作模式为ECB、CBC和CTR三种，密钥长度为56比特，加密速度快，但限制密钥长度。
- 3DES(Triple Data Encryption Standard)：是由IBM公司于1999年发布的对称分组密码算法。它是对DES算法的改进版本，增加了一个密钥用来抵御多次密钥过渡攻击。密钥长度仍然是56比特。
- AES(Advanced Encryption Standard)：高级加密标准，是美国NIST（National Institute of Standards and Technology）公司在2001年发布的对称分组密码算法。该算法采用了混合列混合运算器结构，使得算法更加复杂，同时提高了加密速度。密钥长度可达到256比特，具有很高的安全性。
- IDEA(International Data Encryption Algorithm)：国际数据加密算法，又称Camellia加密算法，是一种对称分组密码算法，由德国的维尔纳·冯·莱茨瓦纳设计。它的密钥长度可达到128比特。
不对称分组密码算法，又称为公私钥算法，加密时使用公钥，解密时使用私钥。常用不对称分组密码算法包括RSA、ECC(Elliptic Curve Cryptography)。
- RSA(Rivest–Shamir–Adleman)：是美国计算机科学家罗纳德·李维斯特（Ronald Liden）、阿兰·图灵（Allen Turing）和马修·麦卡洛克（Mark McCaul）一起提出的基于整数因子分解的公钥加密算法。是目前最常用的公钥加密算法之一。
- ECC(Elliptic Curve Cryptography)：椭圆曲线加密算法是一种新型的公钥加密算法，由美国国家标准与技术研究院（NIST）提出。该算法同样采用整数因子分解的方法，可以生成公钥和私钥。椭圆曲LINE，LWE（Lightweight Elliptic Curves）等都是利用椭圆曲线密码学研究的研究方向。
### 2.1.2 流密码算法
流密码（stream cipher）是一种加密算法，它把任意长度的明文流作为输入，经过一个确定规则的运算得到密文流，而且不同的明文流一定不会产出相同的密文流。流密码通常采用反馈结构，将密钥流也加入加密计算流程。流密码算法可以分为一类，即RC4(Rivest Cipher 4)，是一种简单有效的流密码算法。
- RC4: Rivest Cipher 4，是由Rivest加密实验室于1987年设计的基于线性链接结构的流密码算法。它的密钥长度可选128/192/256比特。
### 2.1.3 对称密钥加密算法
对称密钥加密算法（symmetric-key encryption algorithm）是指对称加密算法，它只用一个密钥就能够加密和解密数据，并且这个密钥只能用于解密后续密文。常用对称密钥加密算法包括：
- AES: Advanced Encryption Standard，高级加密标准。
- DES: Data Encryption Standard，数据加密标准。
- Triple DES: 是三重DES算法。
- CAST: Compartmented Array Security Technique，分区数组安全技术。
- Blowfish: 是Bruce Schneier设计的一种对称加密算法。
- IDEA: International Data Encryption Algorithm，国际数据加密算法。
- SEED: Symmetric Encryption for the Enterprise，用于企业级的对称加密算法。
### 2.1.4 非对称密钥加密算法
非对称密钥加密算法（asymmetric key encryption algorithm）是指用两个密钥对数据进行加密和解密的算法。公钥加密系统的加密密钥对所有接收者均公开，任何接收者都可以通过这个密钥对数据进行解密。而私钥加密系统的加密密钥只有发送者知道，不能被任何人获取。常用非对称密钥加密算法包括：
- RSA: Rivest–Shamir–Adleman，是一种基于整数因子分解的公钥加密算法。
- ECC(Elliptic Curve Cryptography): 椭圆曲线加密算法是一种新型的公钥加密算法，通过椭圆曲线计算公钥和私钥。
### 2.1.5 哈希算法
哈希算法（Hash Function）是将任意长度的数据输入，经过算法运算，生成固定长度的输出值。由于数据量可能非常大，所以哈希算法经常被用于信息检索、摘要校验、数据库索引等场合。常用哈希算法包括：
- MD5: Message Digest Algorithm 5，MD5是一种哈希算法，是最早的被广泛使用的散列函数。它的输出是固定的128比特字节序列。
- SHA-1: Secure Hash Algorithm 1，SHA-1也是一种哈希算法，是美国国家安全局（NSA）设计的一种较新的 hashing 函数，速度较快，适用于数字签名、身份认证、数据完整性验证等应用。
- SHA-2: Secure Hash Algorithm 2，SHA-256，SHA-384，SHA-512是SHA-2系列的不同哈希算法，它们输出的值都是固定的256/384/512比特的字节序列。
## 2.2 哈希值
哈希值的概念源自于密码学中密码的理论，将任意长度的信息压缩成特定长度的摘要。通过摘要值就可以快速比较两个文件是否一致，也可以将文件存储起来方便查找。哈希值又称“指纹”，通过哈希值可以判断一个文件的内容是否被修改过。哈希值在网络传输中起着重要作用，可以检测数据是否被篡改或产生错误，从而保证数据的完整性。常见的哈希算法包括MD5、SHA-1、SHA-256、SHA-512等。
## 2.3 Base64编码
Base64是一种用64个字符表示任意二进制数据的方法。一般情况下，Base64编码后的字符串由A-Z、a-z、0-9、+和/组成，在URL、Cookie、网页内容、文本、配置文件、加密密钥等场景会用到。Base64编码实际上是在6bit为一个单位的原始数据前添加一个64的余数，从而将原始数据分割成若干组6bit数据。每个6bit的数据在64个字符中可以找到对应的表示字符，这样就可以用ASCII字符表示二进制数据了。
## 2.4 摘要算法
摘要算法（digest algorithm）是指通过一个函数计算任意长度的信息摘要的算法，常见的摘要算法有MD5、SHA-1等。摘要算法的主要功能是对任意长度的信息做一个固定长度的结果，这个结果可以唯一标识这个信息。与哈希算法不同的是，摘要算法不需要反复计算，因此速度更快，而且摘要长度可以控定。常见的摘要算法包括MD5、SHA-1、SHA-256、SHA-512等。
## 2.5 数字签名
数字签名（Digital Signature）是指利用私钥对消息或文档进行加密、解密并产生相关信息摘要的过程，然后将此摘要与公钥配对。数字签名有助于确认消息的完整性，因为只有拥有私钥的人才能产生正确的签名。数字签名还可以防止数字信封伪造。常见的数字签名算法包括DSA、ECDSA、RSA、HMAC-MD5、HMAC-SHA1等。
