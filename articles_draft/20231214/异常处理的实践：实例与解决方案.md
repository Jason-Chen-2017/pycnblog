# 2.核心概念与联系
异常处理的核心概念包括异常的类型、异常的抛出和捕获、异常处理机制等。

## 2.1 异常的类型
异常的类型可以分为两种：检查异常（Checked Exception）和运行异常（Runtime Exception）。

### 2.1.1 检查异常（Checked Exception）
检查异常是指程序在编译时可以检测到的异常，这类异常必须在程序中进行处理，否则会导致编译错误。检查异常通常是由于程序员在编写代码时未处理或未捕获的异常。例如，文件操作异常、网络操作异常等。

### 2.1.2 运行异常（Runtime Exception）
运行异常是指程序在运行时可能发生的异常，这类异常不需要在编译时进行处理，但在运行时可能会导致程序崩溃或数据损失。运行异常通常是由于程序在运行过程中遇到了未预期的情况导致的异常。例如，数组越界异常、空指针异常等。

## 2.2 异常的抛出和捕获
异常的抛出和捕获是异常处理的基本操作，涉及到的步骤包括：异常的抛出、异常的捕获和异常的处理。

### 2.2.1 异常的抛出
异常的抛出是指当程序在运行过程中遇到了异常情况时，程序会抛出一个异常对象，并将这个异常对象传递给调用者。异常的抛出可以通过 throw 关键字来实现。例如：

```java
try {
    // 可能会发生异常的代码
    // ...
} catch (Exception e) {
    // 异常的处理代码
    // ...
}
```

### 2.2.2 异常的捕获
异常的捕获是指当程序在运行过程中遇到了异常时，程序会捕获这个异常对象，并进行相应的处理。异常的捕获可以通过 catch 块来实现。例如：

```java
try {
    // 可能会发生异常的代码
    // ...
} catch (Exception e) {
    // 异常的处理代码
    // ...
}
```

### 2.2.3 异常的处理
异常的处理是指当程序在运行过程中遇到了异常时，程序会执行相应的处理代码，以避免程序崩溃或数据损失。异常的处理可以通过 catch 块来实现。例如：

```java
try {
    // 可能会发生异常的代码
    // ...
} catch (Exception e) {
    // 异常的处理代码
    // ...
}
```

## 2.3 异常处理机制
异常处理机制是一种用于处理异常的方法，包括 try-catch 块、finally 块等。

### 2.3.1 try-catch 块
try-catch 块是一种用于处理异常的结构，它包括一个 try 块和一个或多个 catch 块。try 块中的代码可能会发生异常，如果发生异常，程序会跳出 try 块，并执行相应的 catch 块中的代码。例如：

```java
try {
    // 可能会发生异常的代码
    // ...
} catch (Exception e) {
    // 异常的处理代码
    // ...
}
```

### 2.3.2 finally 块
finally 块是一种用于执行清理操作的结构，它可以与 try-catch 块一起使用。无论是否发生异常，finally 块中的代码都会被执行。例如：

```java
try {
    // 可能会发生异常的代码
    // ...
} catch (Exception e) {
    // 异常的处理代码
    // ...
} finally {
    // 清理操作
    // ...
}
```
