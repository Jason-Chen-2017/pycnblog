### 4.1 分布式计数器示例

分布式计数器是分布式系统中的一个重要概念，它用于解决分布式系统中的数据存储问题。分布式计数器主要包括：

1. 初始化：首先需要初始化一个分布式计数器，初始值为0。
2. 增加：当需要增加计数器值时，需要在多个节点上执行增加操作。
3. 查询：当需要查询计数器值时，需要在多个节点上执行查询操作。

以下是一个Go语言实现的分布式计数器示例：

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

type Counter struct {
	value int
	mu    sync.Mutex
}

func (c *Counter) Increment() {
	c.mu.Lock()
	c.value++
	c.mu.Unlock()
}

func (c *Counter) Get() int {
	c.mu.Lock()
	defer c.mu.Unlock()
	return c.value
}

func main() {
	counter := &Counter{value: 0}

	var wg sync.WaitGroup
	wg.Add(2)

	go func() {
		for i := 0; i < 10; i++ {
			counter.Increment()
			time.Sleep(time.Second)
		}
		wg.Done()
	}()

	go func() {
		for i := 0; i < 10; i++ {
			counter.Increment()
			time.Sleep(time.Second)
		}
		wg.Done()
	}()

	wg.Wait()
	fmt.Println(counter.Get())
}
```

在上述示例中，我们创建了一个Counter结构体，它包含一个value字段和一个mu同步锁。Counter结构体提供了Increment和Get方法，用于增加计数器值和查询计数器值。

在main函数中，我们创建了一个Counter实例，并启动两个goroutine来模拟多个节点的操作。每个goroutine会增加计数器值10次，并在每次增加后休眠1秒。

最后，我们调用Counter实例的Get方法来查询计数器值，并输出结果。

### 4.2 详细解释说明

在上述示例中，我们使用了Go语言的sync包来实现分布式计数器。sync包提供了一些同步原语，如Mutex、WaitGroup等，用于解决并发问题。

Mutex是Go语言中的互斥锁，它用于保护共享资源。在上述示例中，我们使用Mutex来保护计数器的value字段，以确保在同一时刻只有一个goroutine可以访问它。

WaitGroup是Go语言中的同步原语，它用于等待多个goroutine完成后再继续执行。在上述示例中，我们使用WaitGroup来等待两个goroutine完成后再输出计数器值。
