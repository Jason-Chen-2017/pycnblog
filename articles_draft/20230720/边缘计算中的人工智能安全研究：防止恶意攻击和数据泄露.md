# 2.基本概念术语说明
## 2.1 边缘计算
边缘计算（Edge Computing），又称作边缘设备计算或端到端计算，是一种新型计算机系统架构模式。其基本思想是将计算任务放置在本地进行，从而缓解网络的带宽、延迟和计算负载，缩短距离和传输时间。它属于分布式计算范畴。主要特点是部署在物理网络边缘的服务器上，具有低延迟、高带宽、低成本等优点，因此被认为是一种关键技术。边缘计算与传统的数据中心网络相比，具有更快、更可靠的响应速度和较少的延迟。2019年，英特尔宣布其可穿戴产品将采用边缘计算架构。

边缘计算主要分为三个层级：计算、传输和网络。

- **计算层**：在边缘节点进行计算运算，解决复杂的问题，并将结果返回到用户终端。
- **传输层**：对数据进行加密、压缩、编码等，传输到网络边缘，并由本地计算节点进行解码、解压、解密、验证等处理过程。
- **网络层**：负责网络管理、流量调度等功能。

![边缘计算模型](https://i.loli.net/2021/12/07/mVcjkV9KiaPxkD6.png)

典型的边缘计算架构包括：

1. Client Layer：客户端层。主要实现应用交互、数据采集、传输等功能。客户端也可以理解为应用层，但由于边缘计算部署在用户终端设备上，其应用范围受限，通常只支持特定应用。
2. Gateway Layer：网关层。包括网络接入层、协议转换层、安全控制层、流量调度层等。该层位于客户端和边缘计算设备之间，处理边缘计算设备到用户终端设备的数据收发，网络协议转换，流量过滤等功能。
3. Core Layer：核心层。包括计算资源管理层、计算资源调度层、任务管理层等。该层主要承担计算任务的执行，完成不同计算任务之间的任务调度，以及处理任务产生的中间结果。
4. Analytics Layer：分析层。包括实时数据分析层、存储数据分析层、模型训练层等。该层可以接收来自不同边缘计算设备或终端的实时数据，并对数据进行统计、分析，然后生成模型，用于预测未来的行为或事件。

## 2.2 边缘计算中的设备安全
目前，边缘计算的系统架构都是公共云计算服务提供商所托管，因此，边缘计算中的设备安全依赖于公共云服务提供商的安全策略。但是，由于边缘计算的特殊性，边缘计算设备的安全可以进一步加强。

### 2.2.1 边缘计算设备类型
边缘计算中的设备类型主要分为两类：专用设备和嵌入式设备。

- 专用设备：专用设备指的是运行边缘计算任务的机器，比如具有单独处理器或处理单元的服务器，具有独立内存、固态硬盘或 SSD 的磁盘阵列，甚至具有专用的通信接口卡。
- 嵌入式设备：嵌入式设备是在系统中嵌入的小型处理器，具备某种特定的功能特性，可直接被应用软件调用。它们有较大的功耗、少量内存、不适宜做深度学习等任务。

### 2.2.2 边缘计算设备认证
为了保证边缘计算设备的安全，必须首先认证其身份，确保设备合法。一般情况下，边缘计算设备会被赋予唯一标识符或标签，并且边缘计算框架将其映射到一个唯一的实体上。这一过程称为 Device Authentication 或 Device Attestation。边缘计算设备认证可以利用两种方法：

1. 签名验证：设备制造商可以在每次启动时提供预先签名的数字证书。边缘计算框架可以获取该证书，并用它对设备的所有配置参数进行签名验证。如果验证失败，则会拒绝启动设备。这种方式比较简单，容易实现，但可能会受到攻击者的操控。
2. 漏洞防护：边缘计算框架可以对边缘计算设备进行持续监控，检测是否存在已知安全漏洞。如果发现漏洞，则会立即通知设备制造商，要求升级软件。这种方式相对安全，不容易受到攻击者的操控。

### 2.2.3 边缘计算设备更新
边缘计算设备每隔几周就会发布一次安全补丁或者版本更新，对边缘计算设备进行更新可以获取最新漏洞修复。边缘计算设备更新主要有两种方法：

1. 自动更新：边缘计算设备可以设置定时自动更新，或者由边缘计算框架定时检查软件版本。这种方式不需要手动干预，自动进行更新。
2. 手动更新：边缘计算设备可以通过网络更新程序来下载更新包，安装更新后，会自动启动。这种方式比较麻烦，需要用户手动更新。

### 2.2.4 边缘计算设备安全隔离
对于边缘计算设备来说，安全隔离意味着将其软硬件环境隔离，使得设备仅可访问授权的应用，且不能直接访问外网。边缘计算设备安全隔离可以采用两种方法：

1. 操作系统层面的安全隔离：将边缘计算设备运行的操作系统进行划分，只允许部分软件进行运行，其他软件不允许运行，达到隔离效果。
2. 容器化隔离：利用容器技术将边缘计算设备上的多个软件应用隔离，每个容器只能访问自己指定的资源，达到隔离效果。

### 2.2.5 边缘计算设备加固
加固是一种手段，旨在增强边缘计算设备的安全性，抵御攻击、规避风险。边缘计算设备的加固可以分为两步：

1. 配置项检测：检查配置项中的错误或不安全设置，从而避免边缘计算设备遭受攻击。
2. 漏洞扫描：定期扫描边缘计算设备的安全漏洞，发现后立即进行补丁和更新。

### 2.2.6 边缘计算数据安全
边缘计算数据安全问题的根本原因在于边缘计算设备的缺乏可靠性和完整性。因此，在边缘计算中，数据安全需要考虑以下几个方面：

1. 数据存储安全：对于边缘计算设备中的数据，需要将其安全地存储起来，避免数据被篡改或泄露。
2. 数据传输安全：在边缘计算过程中，数据的传输可能会发生各种异常情况，需要对其进行有效的保护，避免数据泄露、恶意篡改等风险。
3. 数据完整性验证：边缘计算中的数据存储、传输、处理都会受到潜在的攻击者的攻击。因此，需要引入完整性验证机制，确保数据的完整性，并阻止篡改数据。
4. 权限管理：针对不同的业务场景，需要设定不同的权限和访问控制策略，才能确保边缘计算中数据的安全。

## 2.3 边缘计算网络安全
边缘计算网络安全主要基于分布式计算模型。由于边缘计算设备部署在距离用户较远的地方，这就给边缘计算带来了新的安全挑战。要保证边缘计算网络的安全，主要有以下五个层面的考虑：

1. 边缘计算设备的安全：保证边缘计算设备的安全，主要有以下三方面考虑：
   - 使用安全的操作系统：选择对边缘计算设备进行安全配置的操作系统，例如，禁止 root 用户登录，启用防火墙等。
   - 设置安全的账户密码：不要给设备设置默认账户密码，设置随机账户密码，确保账户密码安全。
   - 安装安全的应用：仅安装必要的应用，排除危险应用。
2. 服务间通信的安全：由于边缘计算采用分布式计算模型，不同边缘计算设备之间的通信通常需要进行加密，才能确保数据的安全。
3. 边缘计算应用的安全：在边缘计算应用中，需要对应用的权限、输入输出进行控制，避免出现安全漏洞。
4. 边缘计算设备的配置管理：边缘计算设备的配置管理需要严格按照正确的流程进行，确保边缘计算设备配置的完整性和一致性。
5. 边缘计算网络的运营安全：边缘计算网络的运营安全需要建立良好的运营记录，规范操作，保持网络安全。

## 2.4 边缘计算应用程序安全
边缘计算应用程序是指部署在边缘计算设备上运行的软件。应用程序的安全性很难得到保证，因为它们是在边缘计算设备上运行的，并且会受到边缘计算设备的安全漏洞的影响。因此，应用程序安全主要有两个层面的考虑：

1. 运行时安全：应用程序应该进行有效的编程、开发和测试，确保其运行时安全。要注意以下安全问题：
   - 检查指针使用：检查指针是一种错误处理机制，其目的是跟踪内存分配和释放，帮助程序员追踪内存使用情况。应谨慎使用，防止程序的内存管理错误。
   - 数据类型溢出：数据类型溢出是一种常见的安全漏洞，当程序处理的数据超过数据类型容量时，会导致数据损坏或崩溃。要注意整数溢出、浮点数溢出等。
   - 缓冲区溢出：缓冲区溢出是另一种常见的安全漏洞，当程序写入或读取数据时，超出了缓冲区的边界，可能会导致数据泄露或恶意攻击。
   - 堆栈溢出：堆栈溢出是指，函数调用发生嵌套过深，导致栈空间被耗尽，导致缓冲区溢出。要注意检查递归函数的调用次数，防止栈溢出。
2. 开发人员安全意识：边缘计算设备上运行的应用程序通常由第三方开发者提供。为了确保应用程序的安全，应建立起良好的开发人员习惯和态度，使其对安全性有充分的关注。

## 2.5 边缘计算平台安全
边缘计算平台就是提供运行和管理边缘计算资源的系统，如计算集群、存储集群等。边缘计算平台的安全性也十分重要。主要有四个层面的考虑：

1. 平台集成程度：边缘计算平台与云服务商的合作关系决定了平台的集成程度。边缘计算平台应经过充分安全评估，确保其能够满足云服务提供商的安全要求。
2. 平台配置的安全：边缘计算平台的配置管理应进行详细、精细的审计和控制。平台所有者应当对边缘计算平台的配置管理和维护进行持续的安全培训和演练。
3. 平台基础设施的安全：边缘计算平台运行的基础设施包括物理设备、软件组件和网络结构。为了确保平台的安全，平台所有者应当对其进行持续的安全加固，保障平台的正常运行。
4. 边缘计算平台管理体系的安全：边缘计算平台管理体系包括整个平台的安全管理、数据管理、应用管理、基础设施管理等。边缘计算平台管理体系应当向云服务提供商提供符合安全要求的体系和管理框架。

## 2.6 边缘计算云服务安全
云服务安全是边缘计算的一个重要的方面。云服务商为边缘计算设备提供计算、存储、网络等基础设施服务。云服务安全的主体是云服务提供商，他们需要确保服务的安全性，尤其是其提供的服务需要满足边缘计算设备的安全要求。

1. 数据安全：云服务商应确保其提供的存储服务的安全性。云存储服务包含数据存取、数据安全、数据完整性、数据可用性和数据可移植性等模块。
2. 计算资源安全：云服务商应确保其提供的计算资源的安全性。云计算服务包含虚拟机、云函数、容器引擎、网络服务等。
3. 网络安全：云服务商应确保其提供的网络服务的安全性。云网络服务包含基础设施即服务（IaaS）、平台即服务（PaaS）、软件即服务（SaaS）。
4. 身份认证：云服务商应确保其提供的身份认证系统的安全性。云服务提供商的身份认证系统应当具备高水平的安全性，应对身份验证和鉴权事件提供真正的安全保障。
5. API 和 SDK 安全：云服务商应确保其提供的 API 和 SDK 的安全性。API 是云服务的编程接口，SDK 是云服务的软件开发工具包。云服务提供商应当保证其提供的 API 和 SDK 安全，以免被恶意使用，造成损失。
