# 1.简介
  

编程是一项能力，并不是每个程序员都需要掌握。很多人只是想拿到毕设或者证书，然后就把这些时间投入到学习知识和技能上，而忘了花更多的时间去体验编程的乐趣。所以，很多程序员都不知道什么是编程，只是习惯用自己的语言来描述自己的想法。如果你准备好从事编程工作，但还是被一些概念束缚住了，那么可以了解一下如何成为一名自由软件开发者。

本文通过阅读**《如何成为一名自由软件项目的贡献者》**这本书（作者：Richard Lindh）的精华，整理出“编程者如何成为自由软件开发者”系列文章的框架结构，使读者更容易理解该主题的重要性，帮助大家正确认识并掌握这一领域的知识和技能。

作者认为，为了让开源社区能够发展壮大、繁荣昌盛，开发者要有一种独立思考的态度。在这种态度下，他们会参与到自由软件项目中，共同完善软件的功能、性能和可用性，促进软件开发的多样化、开放和协作。因此，自由软件开发者必须有好的专业素养，对计算机科学和相关学科有丰富的研究和实践经验。他们懂得代码编写的艺术，擅长将自己的技术成果传播到其他开发者手中，为软件创新保驾护航。

# 2.基本概念术语说明

## 2.1 自由软件
自由软件（Free software）是指由用户共享的、源代码可供所有人修改的软件。它属于无版税的自由、免费、共享的软件类别，其核心是遵循共享的原则。也就是说，自由软件向所有用户提供了使用、复制、研究、修改或再分发软件的自由权利。其著名的软件之一就是GNU操作系统，它是一个自由软件，你可以随意下载安装，并且在任何时候使用。

## 2.2 自由软件开发者
自由软件开发者（Free software developer）是指独立于政府、企业和组织之外，以尽可能广泛的、充满活力的方式来提升个人的能力、技术和创造力的人群。在自由软件项目中，自由软件开发者通过使用及修改开源软件，为公众提供服务、解决问题，并贡献自己的力量，将源代码提供给社区进行改进。


## 2.3 GPL许可证
GPL(General Public License)是自由软件基金会（FSF）所创建的一款著名的开源许可证，允许自由软件在法律约束范围内以源代码和二进制形式自由分享。GPL协议要求基于GPL许可的开源软件必须也以相同方式提供源码。而任何修改后的版本必须继续以GPL许可发布。该协议的主要目标是在开源领域建立一个共赢的环境。但是由于GPL协议的复杂性和保障不足，目前已经越来越少的使用。

## 2.4 源代码（Source Code）
源代码是一组指令集合，它们告诉电脑或其它设备如何运行。一般情况下，源代码文件具有特定格式，并由文本编辑器打开查看。如果某些程序没有源代码，则称其为编译后的机器码。


## 2.5 社区（Community）
社区是指软件开发者们为了共同完成某个项目而聚集的地方。开发者之间往往倾向于彼此熟悉，并且分享自己对项目的看法。社区通常拥有一个明确的规划、需求和目标，围绕着这个目标，每个开发者都尝试着做出独特的贡献。


## 2.6 倡议（Initiative）
倡议是一种基于行动的、以社会化的方式推动变化的过程。自由软件开发者应当鼓励开发者们积极倡议新的软件或硬件标准，帮助开源社区规范软件开发的流程，以及为各个群体提供便利。


## 2.7 文档（Documentation）
文档是记录软件使用方法、注意事项、特性等信息的文件。对于开源软件来说，最好的文档就是代码本身。文档是其它开发者不可缺少的一部分。


## 2.8 发行版（Release）
发行版是指将源代码作为压缩包或安装档，提供给用户使用的过程。用户可以自行下载安装或直接购买。


## 2.9 测试（Testing）
测试是指确认软件是否按预期工作的过程。测试人员按照测试计划检查软件的各个方面，确认其满足所有要求。


## 2.10 版本控制系统（Version Control System）
版本控制系统（VCS）用于跟踪和管理软件的不同版本。它可以帮助开发者追踪源代码的变动、恢复到过去的版本、比较差异，以及备份代码。

## 3.1 创建开源项目
首先，你需要了解什么叫做开源项目。开源项目就是你有一个代码库，其他人可以在上面进行更改、提出建议，你的项目就是属于自由软件的一个成员。

接下来，你需要找到志同道合的朋友，一起决定一起创建一个开源项目。在制定计划的时候，你应该考虑到以下几个细节：

1. 项目名称：项目名称即你项目的主页地址上的名字，应该具有吸引人的含义，不能太冗长。
2. 项目目的：你的项目目的是什么？要清晰地陈述你的项目的目的。比如，你想要构建一个杀毒软件，那就应该清楚它的目标是什么。
3. 使用的编程语言：你希望用什么编程语言来开发这个项目？例如，Python、JavaScript、Java等。
4. 社区支持：你需要选择一个开源社区，并加入到社区的讨论中。
5. 文档撰写：除了代码注释外，还需要撰写项目文档。

除了上面的五个步骤，你还需要考虑一下：

1. 开发环境：你需要配置开发环境。包括安装必要的工具、软件、插件，并进行必要的配置。
2. 技术栈：了解你正在使用的技术栈。比如，你希望用Django、Flask、Bootstrap开发网站，那就需要对相关技术有一定的了解。
3. 任务分配：任务分配是一门大学问，需要你有足够的能力分配给不同的人负责不同的模块。同时，你也需要有较高的耐心和持续性。
4. 任务协调：当团队成员处于不同时区、语言、背景等情况时，需要进行日常的沟通协调。
5. 分享经验：你的经验很宝贵，需要分享给大家，帮助他人。

## 3.2 提交贡献
开源项目的贡献包括提交代码、翻译、编写文档、回答问题、提供反馈、组织活动、测试等等。每一次的贡献都对项目的开发起到积极作用。

### 提交代码
开源项目的核心就是代码。你可以使用GitHub或GitLab这样的托管平台，通过Pull Request机制来完成自己的贡献。首先，你需要Fork仓库，在本地Clone副本后，在副本中编写代码。一旦完成编码，就可以Push到远程仓库，并向项目管理员发起Pull Request。项目管理员审阅代码后，合并到主分支。整个流程如下图所示。


### 翻译
开源项目中的文档也是非常重要的资源。而且，有些国家和地区的程序员还可以从事翻译工作。如果你有兴趣，可以将英文文档翻译成中文，提交到项目中。项目管理员可以对其进行审阅，并将其合并到主分支。

### 编写文档
项目的文档就是项目的知识库。如果你觉得某个部分的描述不够清晰、准确，可以通过提交补充材料的方式来完善文档。项目管理员可以对你的补充材料进行审核，并将其合并到项目中。

### 回答问题
有些时候，用户可能会遇到问题，需要向项目提交Issue。项目管理员可以看到用户的问题，并给予回复。当然，你也可以回答其他用户的问题。

### 提供反馈
用户提供的反馈往往会有建设性的意义。项目管理员可以收到用户反馈，并根据情况进行处理。

### 组织活动
在开源项目中，我们可以组织各种活动。你可以举办线下聚会、举行教程、组织开发者小聚等。当然，你也可以组织自己的活动。

### 测试
开源项目一定要经过测试。有些项目会设置自动测试，有些项目的测试由第三方组织进行。所有的测试都是开源项目的必要环节。

## 3.3 分享经验
经验是成功的关键。在开源项目中，你可以分享你的知识和经验。通过写博客、分享知识、讲座、分享自己的工作等方式，你可以提升自己的影响力和声誉。你的经验对他人也会产生帮助。

## 3.4 增强影响力
开源项目为世界带来了巨大的财富，你的成功也正是间接的获得了他人的帮助。要促进项目的健康发展，你可以参加各种活动，通过参与项目的开发来增强自己的影响力。

## 3.5 拓展视野
开源项目是多元化的，你可以加入多个领域，开拓眼界。加入开源项目的讨论组，与志同道合的朋友一起交流。与其它开发者一起分享你的技术见解，参加各种技术会议。
