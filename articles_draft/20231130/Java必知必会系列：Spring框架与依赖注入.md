## 2.核心概念与联系

### 2.1 依赖注入的类型

依赖注入有两种主要类型：构造器注入和设置方法注入。

- 构造器注入：这种类型的依赖注入是通过在对象的构造器中注入依赖的。这种方式的优点是，它可以确保对象在创建时就已经具有所需的依赖，因此无需在运行时进行额外的操作。
- 设置方法注入：这种类型的依赖注入是通过在对象的setter方法中注入依赖的。这种方式的优点是，它允许开发人员在运行时动态地更改对象的依赖关系。

### 2.2 依赖注入的实现方式

依赖注入的实现方式有两种主要方式：直接实例化和反射实例化。

- 直接实例化：这种方式的依赖注入是通过直接创建依赖对象的实例，并将其传递给依赖接口的实现类的构造器或setter方法。这种方式的优点是，它简单易用，但是它可能会导致耦合性较高。
- 反射实例化：这种方式的依赖注入是通过使用反射API来动态创建依赖对象的实例，并将其传递给依赖接口的实现类的构造器或setter方法。这种方式的优点是，它可以降低耦合性，但是它可能会导致代码复杂性较高。

### 2.3 依赖注入的应用场景

依赖注入的应用场景有两种主要类型：单例模式和多例模式。

- 单例模式：这种应用场景的依赖注入是通过使用单例模式来确保整个应用程序中只有一个实例的依赖对象。这种应用场景的优点是，它可以降低内存占用，但是它可能会导致并发性能问题。
- 多例模式：这种应用场景的依赖注入是通过使用多例模式来确保整个应用程序中可以有多个实例的依赖对象。这种应用场景的优点是，它可以提高并发性能，但是它可能会导致内存占用较高。
