# 2.核心概念与联系

在可视化调试技术中，核心概念包括：可视化调试器、调试器界面、调试器功能等。

## 2.1 可视化调试器

可视化调试器是可视化调试技术的核心组件，负责收集程序执行过程中的各种信息，并将这些信息以图形化的方式展示给开发者。可视化调试器可以收集程序的执行流程、变量的值、内存的分配情况等信息，并将这些信息以图形化的方式展示给开发者，从而帮助开发者更直观地理解程序的运行流程。

## 2.2 调试器界面

调试器界面是可视化调试器的用户界面，负责展示程序执行过程中的各种信息。调试器界面通常包括以下几个部分：

- 调试器控制部分：负责控制程序的执行，包括启动、暂停、继续等操作。
- 调试器窗口部分：负责展示程序执行过程中的各种信息，包括执行流程、变量值、内存分配情况等。
- 调试器工具部分：负责提供各种调试工具，如断点、步入、步出、步过等。

## 2.3 调试器功能

可视化调试器具有以下主要功能：

- 程序启动：可视化调试器可以启动程序，从而开始对程序进行调试。
- 程序暂停：可视化调试器可以暂停程序的执行，从而让开发者查看程序的当前状态。
- 程序继续：可视化调试器可以让程序继续执行，从而让开发者观察程序的执行流程。
- 断点设置：可视化调试器可以设置断点，从而让程序在达到某个条件时暂停执行。
- 步入、步出、步过：可视化调试器可以让程序逐行执行，从而让开发者更直观地观察程序的执行流程。
- 变量查看：可视化调试器可以让开发者查看程序中的变量值，从而帮助开发者理解程序的运行状态。
- 内存分配查看：可视化调试器可以让开发者查看程序中的内存分配情况，从而帮助开发者理解程序的内存使用情况。

4. 然后，我们可以使用步入、步出、步过等功能来逐行执行程序，从而更直观地观察程序的执行流程。在本例中，我们可以使用步入功能来逐行执行程序，从而观察变量 c 的值。
5. 最后，我们可以让程序继续执行，从而完成调试。在本例中，我们可以让程序继续执行，从而输出结果。
