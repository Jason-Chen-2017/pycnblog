# 2.核心概念与联系

在Java中，代码质量与重构技术的核心概念包括：

1.代码可读性：代码可读性是指代码对其他开发人员的可读性。好的代码可读性可以让其他开发人员更快地理解代码的功能和逻辑，从而提高开发效率。

2.代码可维护性：代码可维护性是指代码对于修改和更新的可维护性。好的代码可维护性可以让开发人员更容易修改和更新代码，从而提高软件的稳定性和性能。

3.代码可扩展性：代码可扩展性是指代码对于扩展和添加新功能的可扩展性。好的代码可扩展性可以让开发人员更容易添加新功能，从而提高软件的灵活性和适应性。

重构技术是一种修改代码的技术，可以帮助开发人员提高代码的可读性、可维护性和可扩展性。重构技术包括：

1.提取方法：提取方法是一种重构技术，可以将重复的代码提取出来，形成一个独立的方法。这可以让代码更加简洁和可读性。

2.替换方法：替换方法是一种重构技术，可以将一个方法替换为另一个方法。这可以让代码更加简洁和易于理解。

3.移动代码：移动代码是一种重构技术，可以将代码从一个类或方法移动到另一个类或方法。这可以让代码更加组织和可维护。

4.修改参数：修改参数是一种重构技术，可以将方法的参数修改为其他参数。这可以让代码更加灵活和可扩展。
