# 1.简介
  

GitHub是一个面向开源及私有软件项目的版本控制平台，在这里，你可以不受限地进行协作开发、管理仓库以及分享你的工作成果。而作为一名初级的Github用户，你或许对如何使用这个平台有些困惑。那么，本教程将带领大家一起学习并掌握GitHub的基础知识。

# 2.GitHub概述
GitHub是一个面向开源及私有软件项目的版本控制平台，目前已经成为全球最大的同性交友网站。这是一个非常有意义的平台，因为它能够帮助像程序员一样，为其创造一个共同的地方，可以互相提供帮助、探索新技术、分享想法，并且鼓励成长。

## 2.1 GitHub是什么？
GitHub是一个基于Git版本控制系统的托管服务，用来存储所有的版本历史记录，包括源代码、文本文件、图像等各种类型的文件。与其他版本控制系统不同的是，GitHub还提供了额外的功能，例如允许同事们可以协作完成任务、用图表展示项目进展等等。同时，GitHub也拥有着强大的社交功能，让用户可以跟踪关注的项目、发现有趣的人物、建立起个人品牌等。

## 2.2 为什么要用GitHub?
当今世界，互联网环境日益复杂，产品快速迭代，创新能力显著提高。但是互联网信息的分布式产生了一系列新的问题。比如说，代码散布于众多仓库中，阅读这些代码可能需要经历多个步骤才能找到正确的版本；而且，各个团队可能采用了不同的编程语言，很难保持沟通，使得协同工作变得困难。因此，越来越多的人开始倾向于使用集中式的代码托管工具，如GitHub。

GitHub可以为项目的所有方面提供协作平台，通过GitHub，你可以方便地管理自己的代码库、跟踪问题、分享想法、和他人合作开发。除此之外，GitHub还提供了一个很好的博客平台，可以在上面展示你的个人品牌，分享你的故事以及提升技能。另外，GitHub还支持与很多程序员交流。你可以通过议题、评论、讨论来与程序员们一起讨论你的项目，还可以通过GitHub Actions实现持续集成。最后，GitHub是一个开放平台，你可以在上面建设自己喜欢的项目，分享你的成果。所以，无论你身处何处，都可以使用GitHub来实现你的目的！

# 3.基础知识
## 3.1 Git与GitHub的关系
Git是一款开源的分布式版本控制系统，GitHub是Git的托管服务。两者之间有如下关系：
- Git: Git是一个命令行工具，用于管理项目的源码。你可以使用它把项目代码从一个地方复制到另一个地方，对代码做改动，或者撤销修改，还可以查看每一次提交的内容。
- GitHub: GitHub是一项托管服务，它是一个基于Git的仓库托管站点，为用户提供方便的界面，让你能够快速发布和分享你的代码。GitHub为你提供了Web界面、代码搜索、 issue系统、wiki等丰富的功能。并且，GitHub可以和其它版本控制服务集成，如Bitbucket、GitLab等，提供更多的功能。

## 3.2 用户账户与组织机构
GitHub提供了两个类型的用户账户，分别是个人账号（User）和组织账号（Organization）。
### 3.2.1 个人账户
对于个人账号，你可以创建一个独立的账户，并可以自由地创建、分享、编辑、删除仓库，以及对整个账户下所有仓库的权限进行管理。你可以邀请朋友加入你的项目组队，帮助你和他人协作完成任务。你也可以选择付费购买私人仓库，享受高级权限和功能。
### 3.2.2 组织机构
对于组织账号，你可以创建一组共享仓库，让多个成员协作开发。你可以直接邀请外部用户加入你创建的组织，赋予其相应权限，协助你管理仓库。

## 3.3 创建仓库
GitHub提供了两种方式来创建一个仓库：

1. 在GitHub主页上点击“New repository”按钮。
2. 在命令行里输入`git clone <repository URL>`，克隆远程仓库到本地。

第一种方式是最简单的方式，但仅适用于个人账号的仓库，不能创建组织账号的仓库。第二种方式则可以创建任何类型的仓库，无论是个人账号还是组织账号。

一旦你创建好仓库，就可以往里面添加文件、编辑代码、上传图片、提交注释、创建分支、合并分支、新建pull requests、设置仓库设置等等。

## 3.4 分支管理
在项目中，通常存在着分支（branch）和标签（tag）这两个概念。分支就是一条直线上的多个不同路口，你可以把它比喻成你的职业生涯。每个分支代表着一条独特的道路，只要走过这条路，就会留下一个独特的记忆，而这条路不会影响其它路径。分支可以用来尝试新功能、修复bug、实验性质的代码等等。而标签则是指向某一特定提交点的指针。它可以帮助你归档某个重要的里程碑版本，并随时回到该版本。

GitHub的分支管理系统让你更加轻松地进行分支操作。你可以在任意时刻创建、切换和删除分支，创建标签来标记重要的提交点，以及与他人分享你的工作成果。

# 4.核心算法和具体操作步骤
## 4.1 登录GitHub



## 4.2 创建仓库
登录成功后，点击右上角的"+"号按钮，选择"New Repository"创建一个新仓库。


进入“Create a new repository”页面，填写相关信息即可。


其中必填选项是“Repository name”，即仓库名称。建议使用英文名称，容易辨识，唯一标识性强。

“Description”可以填写一些描述信息。

“Public/Private”决定了仓库的可见性，如果选为私有，则只有授权的用户才可访问，默认值为公开。

“Initialize this repository with a README”会生成一个初始的README文件。

“Add.gitignore”可以选择忽略哪些文件，建议不要选。

“Choose a license”可以选择开源许可证，一般选择MIT License。

填写完毕后，点击底部的绿色按钮“Create repository”。


等待几秒钟，页面会提示刚刚创建的仓库已经准备就绪，点击“go to repository”即可进入仓库的主页面。

## 4.3 克隆仓库
首先，如果你没有SSH密钥，需要先生成SSH密钥。

```
$ ssh-keygen -t rsa -C "your_email@example.com" # 生成密钥
```

如果你已有SSH密钥，请跳过这一步。然后，复制SSH公钥，粘贴至GitHub对应的仓库的SSH公钥区域。


打开命令行窗口，执行如下命令，克隆远程仓库：

```
$ git clone git@github.com:<用户名>/<仓库名>.git
```

例如：

```
$ git clone git@github.com:yourname/myrepo.git
```

克隆完成后，会自动进入克隆的目录，你可以看到仓库内的文件结构。

## 4.4 推送改动
在克隆的目录中，新增、修改文件，然后执行以下命令：

```
$ git add.   // 添加改动文件
$ git commit -m "commit message"   // 提交改动
$ git push origin master   // 将改动推送至远程仓库
```

最后，刷新GitHub仓库的主页面，可以看到新提交的改动。

## 4.5 切换分支
在GitHub的仓库主页面，点击"Branch:"旁边的"master"字样，可以查看当前分支情况。


点击左侧的"Branch: master"，可以切换到其他分支，也可以创建新分支。点击"New Branch: xxx"，即可创建一个新分支。


## 4.6 删除分支
点击分支名称旁边的“Delete branch”按钮，即可删除该分支。


## 4.7 设置仓库属性
点击仓库名称旁边的设置按钮，可以更改仓库的描述、名字等属性。


## 4.8 查看日志
点击仓库名称旁边的"Commits"按钮，可以查看提交历史。


## 4.9 创建Pull Requests
点击仓库名称旁边的"Pull Request"按钮，可以创建和合并PR。
