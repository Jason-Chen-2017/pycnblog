# 2.数据特征
## 2.1数据特征介绍
数据特征，英文名称feature，是指某个对象的某种特征或属性。在机器学习中，数据特征通常指的是关于个体的某些信息，比如说，人的身高、体重、年龄、性别等。但也可以是关于整个系统的某些信息，比如，在一个电子商务网站中，数据特征可能包括顾客的购买历史记录、浏览记录、收藏夹等。数据特征通常由多个维度构成，也可能只包含一个维度。

## 2.2数据特征类型
根据数据特征不同，分为以下几种类型：

1. 连续变量（Continuous variable）：连续变量指的是该变量的值可数无限小数值。如，人的身高、体重、温度、价格等。

2. 离散变量（Discrete variable）：离散变量指的是该变量的值只能有若干个离散点。如，职业、性别、部门、类别、国家等。

3. 标称变量（Categorical variable）：标称变量指的是该变量的值属于多选或者二元组形式。如，职业、性别、国家等。

4. 序变量（Ordered variable）：序变量指的是变量的值按一定的顺序排列，并且可以用来排序、比较。如，成绩分等级。

5. 时间序列变量（Time series variable）：时间序列变量指的是时间有先后顺序的变量。如股票价格、销售量、运动轨迹等。

6. 结构变量（Structured variable）：结构变量指的是一些有结构的变量，比如矩阵、图像、文本、视频等。

## 2.3数据特征识别
数据特征识别，英文名称data profiling，是指通过探索和分析数据集的特性来获取有关数据特征的重要信息。数据特征识别主要有一下几个步骤：

1. 数据描述统计：首先对数据集做基本的统计分析，包括计数统计、质量统计、范围统计、平均值、标准差、众数及占比等。

2. 数据探索性分析：对数据进行探索性分析，目的是找出其中的异常、缺失值、不匹配项、数据倾斜等问题。

3. 数据分布图：绘制数据分布图，用于直观地查看各个特征的分布情况。

4. 特征选择：选择重要的特征，通过挖掘相关性、互信息、卡方检验、皮尔森相关系数等方法确定。

5. 数据归一化：对数据进行标准化或最小-最大缩放，确保所有数据在同一量纲下进行处理。

6. 数据编码：对 categorical 特征进行编码，例如将职业、性别、国家等特征转换为数字表示。

7. 生成报告：生成报告，汇总上述步骤得到的数据特征信息，方便查阅。

## 2.4数据特征归纳
数据特征归纳，英文名称data summarization，是指从大量的原始数据中总结出重要的、有意义的信息。数据特征归纳的方法有很多，最常用的有以下几种：

1. 数据汇总：将原始数据集按照一定的规则进行汇总，如将年龄区间划分为青少年、中年、老年；按年份进行数据汇总，形成年度统计图表；对不同用户的行为数据进行汇总统计，形成用户画像；将产品价格区间进行合并，形成商品价格段。

2. 数据粒度提升：在较低的数据粒度上进行数据汇总，再提升到较高的数据粒度。如将不同城市的销售额汇总到省级，再到国家级；将订单中的物品分类汇总到单品级别，再到类别级别；按不同时期、不同季节、不同区域的销售额进行数据汇总，形成环比、同比、累积图。

3. 主题挖掘：将数据集中的文档主题模型应用到数据中，找出其中相似的文档，形成主题词集合。如，在零售网站的搜索日志中，挖掘出热销商品的主题词集合。

4. 数据采样：对数据进行随机抽样，产生新的样本。

5. 数据聚类：将数据集中的样本聚类，形成不同群体之间的联系。如，在客户群体中，根据购买习惯、兴趣爱好等因素进行聚类。

# 3.数据类型
## 3.1数据类型分类
一般情况下，数据类型分为两大类：结构化数据与非结构化数据。

结构化数据(Structured data)是指具有固定模式的数据集合，这些数据被组织成一系列字段，每个字段代表一种数据类型，并且字段之间存在着严格的对应关系。比如XML，它定义了一种树状结构，这种结构包含若干层标签以及这些标签下的节点。关系数据库中，每张表都是结构化的。

非结构化数据(Unstructured data)是指具有不规律性、无固定模式的数据集合，这类数据没有严格的结构，数据之间的关联通常没有任何规则可言。例如，电子邮件中的无结构文本，网页上的图片，音频、视频文件中的声音、图像流，这些都是非结构化数据。非结构化数据需要用一些分析方法才能得到有价值的信息。

## 3.2结构化数据分析方法
对于结构化数据，可以根据不同的场景选择不同的分析方法，如：

1. 事务处理系统(Transaction processing system)：TPS可以看作是一个账本，记录了多次数据输入和更新过程。分析TPS中的数据，通常会涉及时间维度的分析。

2. 高度结构化数据(Highly structured data)：HTM，即Hierarchical Temporal Memory Model，是一种基于时间的结构化数据模型。HTM将复杂的数据对象建模为一种树状结构，每个节点存储了有关时间范围内的数据。HTM通过向后链接的网络结构连接不同节点，可以方便地对数据进行查询、挖掘、分析。

3. 面向主题的分析(Topic modeling)：主题模型是一种无监督学习方法，可以自动发现数据集中的共同主题。主题模型利用数据的潜在结构，建立一个包含主题的主题空间。每个数据点可以表示为它所属主题的概率分布。分析HTM或TPS的数据时，往往需要进行主题建模。

4. 行/列式存储数据(Row and column-oriented storage data)：ROLAP，即Relational Online Analytical Processing，是一种基于关系的分析处理系统。ROLAP将数据组织成表格，每行表示一个事实，每列表示一个维度。可以对表格的任意一行或任意一列执行高性能的分析运算。如OLAP Cube、MDX，它们可以支持联机分析处理功能。

## 3.3非结构化数据分析方法
对于非结构化数据，通常需要进行一些特殊的分析方法，如下：

1. 文本分析(Text analysis)：对于文本数据，可以使用诸如主题模型、情感分析、新闻事件抽取等方法进行分析。

2. 图像分析(Image analysis)：对于图像数据，可以采用专门针对图像的分析方法，如图像分类、检测与跟踪。

3. 信号与噪声分析(Signal and noise analysis)：对于信号和噪声混杂的场合，可以采用信号与噪声分离方法，如基于傅里叶变换的光谱分析法。

4. 网络与关联分析(Network and association analysis)：对于网络数据，可以采用网络分析方法，如PageRank、度中心度、社会网络分析。

5. 时空分析(Spatio-temporal analysis)：时空分析是指对空间位置及时间维度同时进行分析。对于以经纬度为坐标的时空数据，可以采用GIS、网络分析、轨迹分析方法。
