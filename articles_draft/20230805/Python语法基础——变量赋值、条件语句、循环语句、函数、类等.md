         # 3.Python语法基础——变量赋值
         ## 3.1 变量赋值
         变量是存储数据的工具。在程序执行过程中，可以将不同的值赋给同一个变量，然后再访问该值。变量是动态类型的，可以保存不同类型的数据，如整数、浮点数、字符串、布尔值等。
         
         ### 3.1.1 变量命名规则
         Python变量名只能包含字母数字以及下划线，且不能以数字开头。变量名应简短、明确、描述性强。
         
         ### 3.1.2 注释
         用井号“#”表示单行注释，并对齐到当前所在位置；“'''...'''”或“"""..."""”表示多行注释，可以在代码块前后加以标识，便于理解。
         
         ## 3.2 基本赋值操作符
         有以下几种基本赋值操作符：
         
         | 操作符 | 描述 | 示例 |
         |:----|:---|:-----|
         | =    | 简单的赋值运算符 | x=y   |
         | +=   | 加等于赋值运算符   | x+=y  |
         | -=   | 减等于赋值运算符   | x-=y  |
         | *=   | 乘等于赋值运算符   | x*=y  |
         | /=   | 除等于赋值运算符   | x/=y  |
         | %=   | 求模等于赋值运算符| x%=y  |
         | //=  | 整除等于赋值运算符 | x//=y |
         
         使用赋值操作符为变量分配内存空间，并把右侧表达式的值赋值给左侧变量。如：a=b+c，则a所指向的内存地址存放的是b+c的结果。
         
         **注意**：Python 3 中字符串默认使用 unicode，而 Python 2 中使用 str 进行字符串处理，两者的区别和联系将在第四章介绍。
         
         # 4.Python语法基础——条件语句
         ## 4.1 什么是条件语句？
         “条件语句”是在判断某个特定条件是否成立的情况下，执行不同的代码片段。条件语句的一般形式包括 if-else 语句、if-elif-else 语句和 for-while 语句。
         
         ## 4.2 if-else语句
         ```python
         a = 10
         b = 20
         
         if a > b:
             print("a is greater than b")
         else:
             print("a is not greater than b")
             
         Output: a is less than b
         ```
         上面的例子展示了一个简单的 if-else 语句。首先，a 和 b 被赋值为 10 和 20。然后，一条 if 语句检查 a 是否大于 b。由于 a 不大于 b，因此执行 else 分支中的代码，输出 "a is not greater than b"。
         
         ## 4.3 if-elif-else语句
         如果希望条件语句具有更多的分支选项，可以使用多个 elif 子句。elif 子句用来指定多个可能的情况，如果第一个条件不成立，就依次尝试其他条件。如果所有条件都不满足，就会执行 else 分支的代码。
         
         下面的例子展示了一个 if-elif-else 语句：
         ```python
         a = 10
         b = 20
         
         if a == b:
             print("a and b are equal")
         elif a < b:
             print("a is less than b")
         else:
             print("a is greater than or equal to b")
             
         Output: a is less than b
         ```
         这里有三个条件，第一个是 a 和 b 是否相等，第二个是 a 是否小于 b，第三个就是 else 分支了，当 none of the conditions above are satisfied 时执行该分支。由于 a 小于 b，因此执行 elif 分支中的代码，输出 "a is less than b"。
         
         ## 4.4 for-while语句
         有两种循环语句，即 for 循环和 while 循环。for 循环在指定次数内重复执行一段代码块；while 循环则是只要指定的条件成立，就一直循环执行一段代码块。
         
         下面是一个 for 循环的例子：
         ```python
         for i in range(5):
             print(i)
             
         Output: 
         0
         1
         2
         3
         4
         ```
         从上面的例子可以看出，range() 函数生成了一个序列，然后 for 循环在每次迭代时，依次取出序列中的元素，并把它赋值给变量 i 。在此例中，for 循环会打印 0 到 4 这五个数。
         
         下面是一个 while 循环的例子：
         ```python
         count = 0
         while (count < 5):
             print(count)
             count += 1
             
         Output: 
        ... (重复5次输出)
         ```
         从上面的例子可以看出，初始化变量 count 为 0，然后 while 循环只要 count 小于 5 ，就会一直循环执行 print(count) 语句，并把 count 加 1 ，直至 count 达到 5 ，才退出循环。
         
         # 5.Python语法基础——循环语句
         ## 5.1 break语句
         break 语句用于跳出循环体，直接进入下一次迭代。
         
         下面是一个简单的例子：
         ```python
         for i in range(1, 11):
            if i == 5:
                break
            print(i)
         
         Output: 
         1
         2
         3
         4
         ```
         从上面例子可以看出，当 i 等于 5 时，break 语句被触发，导致循环终止，直接进入下一次迭代。
         
         ## 5.2 continue语句
         continue 语句用于跳过当前循环体的剩余语句，直接进入下一次迭代。
         
         下面是一个简单的例子：
         ```python
         for i in range(1, 11):
            if i == 5:
                continue
            print(i)
         
         Output: 
         1
         2
         3
         4
         6
         7
         8
         9
         10
         ```
         从上面例子可以看出，当 i 等于 5 时，continue 语句被触发，导致 i 的值增加 1 （而不是正常的输出），然后继续下一次循环。
         
         ## 5.3 pass语句
         pass 语句是空语句，是为了保持程序结构完整性的占位符。它不做任何事情，一般来说可以用一个 pass 来替代一些暂时不想实现或缺少功能的代码。
         
         下面是一个简单的例子：
         ```python
         for i in range(5):
             pass
         
         Output: 
         None
         ```
         从上面例子可以看出，pass 语句什么也不做，只是保持程序结构完整性。
