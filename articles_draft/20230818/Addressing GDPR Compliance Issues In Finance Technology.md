# 1.简介
  

## 概述
本文通过对GDPR数据保护法(General Data Protection Regulation)的详细理解、清晰表达和实现技巧，帮助技术人员和业务领域人员在遵守该法规的同时，提升个人信息和用户隐私权保护水平。主要面向金融类互联网企业和服务提供者，旨在展示在不同的数据保护要求下，如何处理敏感数据并防止数据泄露。
## 目的与范围
### 目标
- 提供一般性的知识和技能，能够让读者了解GDPR法律条文的概况及其对商业模式和产品设计的影响；
- 为技术人员和业务领域人员提供了清晰明了的解决方案，能够制定具有自主可控性的隐私保护措施；
- 提供信息安全行业内尚缺乏的专业背景和方法论，帮助企业在顺应GDPR时期进行更有效的合规工作；
- 将文章呈现给广大技术人员和金融业从业者阅读，分享成长经验和管理最佳实践。

# 2. 数据收集与存储
## （一）定义
收集（data collection）是指某些特定目的或条件下，通过采集信息、观察行为、或者以其他方式创建的数字数据。收集的数据可以来自任何渠道，包括人类、技术和自动化设备。数据收集的方式和手段多种多样，有手动输入、设备采集、网页浏览记录等。收集数据的对象也不尽相同，既可以是个人也可以是机构。

## （二）类型
根据收集数据的对象、过程和目的分，数据收集可以分为四类：

1. 直接收集：通过直接与被收集者通信、接触、携带或获得某些信息、文档、文件、图像、视频、音频、位置信息等获取数据。例如通过调查问卷、公开招聘信息、市场调研、网络爬虫等直接获取数据。
2. 间接收集：通过与他人合作、交易、支付、交换或授权第三方获取数据。例如通过合作伙伴关系获取数据、提供付费服务获取数据、共同研究获取数据等。
3. 集合收集：通过网络搜索、社交媒体、社群网站等各种方式获取数据。例如通过网络搜索引擎或社交媒体获取相关信息。
4. 机器收集：通过计算机程序、设备、软件等模拟生成的数据，即在电脑中运行的软件或硬件获取数据。例如通过GPS定位、摄像头拍摄、机器学习模型训练等获取数据。

## （三）目的
根据数据收集的目的是什么，分为两大类：

1. 使用数据：使用数据的目的往往是为了完成某个任务。例如需要对客户数据进行分析、筛选，还可以用于预测和推荐。
2. 留存数据：通过保留数据，可以提高后续数据使用效率、降低数据使用成本，以及促进数据资产转移。

## （四）隐私权衡取
随着全球化和数字化进程的加快，越来越多的人都开始通过互联网进行各种活动。这个过程中，收集的数据量变得越来越大。数据收集带来的隐私权问题不容忽视。

1. 数据安全：数据收集越多，其价值就越高。如果数据流失或者泄露，那么可能造成经济损失和个人危害。因此，当收集到的数据越多，就越容易出现数据泄露事件。此外，由于采用了各种收集手段，使得数据被收集者无法准确知晓自己收集了哪些数据，导致数据安全和隐私权受到威胁。
2. 用户隐私权：当数据被收集到的时候，用户对收集数据的具体用途、收集者身份、访问和披露权限等一系列隐私权利义务是十分敏感的。用户如果不知情，很可能会侵犯自己的权益。因此，当收集到的数据越多，就越容易造成隐私权问题。
3. 财产权：当收集到的数据越多，就越容易侵犯用户的财产权。用户需要知悉这些数据是否真的用于偿还债务，但很多时候用户并不一定能做出决定。因此，数据收集者会收受过多的利息和其它不必要的损失。

因此，在数据收集上，当必须收集的个人信息量达到一定数量级时，需要考虑如何保障用户的隐私权。

# 3. 数据传输
## （一）定义
数据传输（data transfer）指用户或个人向另一个组织、个人或设备发送、接收、存储、上传、下载、移动、复制、储存、显示、传输或接收数据或信息的一系列行为，它涉及个人信息的静态或动态的收集、存储、传输，并可能通过公共网络、无线网络、计算机网络、传真机、邮件、短信、微信、微博、即时通讯、应用程序等多种方式进行。数据传输通常由数据控制器或处理者负责。

## （二）类型
根据数据传输的内容，数据传输可以分为以下几类：

1. 服务提供者之间的传输：服务提供者之间的数据传输一般是基于同一份协议进行传输。
2. 第三方平台间的传输：第三方平台间的数据传输往往采用双边数据传输协议，其中一种协议为标准协议，另一种协议为私有协议。私有协议的数据传输，主要用于平台之间内部数据传输。
3. 平台之间的传输：平台之间的传输包括跨平台应用、设备之间的数据同步等。
4. 第三方服务平台之间的传输：第三方服务平台之间的数据传输可以满足第三方服务的需求。

## （三）安全机制
数据传输的安全保障，首先要依赖于数据传输协议的建立，其次才是采用加密、SSL/TLS协议加密传输。

1. 数据传输协议：数据传输协议的选择不仅要遵循数据传输中的基本原则，而且要注意相应的安全防范措施。比如，可以使用HTTPS协议代替HTTP协议，可以在传输过程中使用SSL/TLS协议对数据进行加密。
2. 加密传输：加密传输是通过对传输的消息进行加密来保护用户信息的安全。这样即便数据传输中存在安全漏洞，也只能看到加密后的结果，无法获知原始的信息。

## （四）隐私权衡取
数据传输中会产生隐私权问题。

1. 数据主体权益：数据主体在数据传输过程中，会承担一些额外的责任。如接收数据是否合法、数据真实性、服务质量、违约情况、违法成本、法律诉讼等。这些都是数据主体应承担的权利。
2. 数据利用偏好：数据主体的意愿和偏好会影响到数据使用的偏好。如果数据主体认为一个组织或服务的收集和使用可能违背自己或公司的利益，就会选择不同的组织或服务。这也是数据主体应负有的责任。
3. 信息控制权：数据主体拥有自己的信息控制权，他们可以撤回自己的个人信息，也可以限制第三方获取个人信息的能力。但是，这种能力往往不能覆盖所有场景下的信息。也就是说，个人仍然可以依据个人的需求索取更多的信息。

因此，数据传输过程中，应该充分关注数据主体的权益，保证数据主体的合法权益得到最大化保障。

# 4. 数据共享与使用
## （一）定义
数据共享与使用（Data Sharing and Usage）是指个体或组织收集、存储、处理和传输数据并在日常生活和工作中共享和使用这些数据的行为。数据共享与使用分为个人数据共享与使用、企业数据共享与使用以及公共数据共享与使用。

## （二）类型
根据数据共享与使用的目的、行为以及使用对象，数据共享与使用可以分为以下几类：

1. 个体与个体之间的共享与使用：个人数据共享与使用是指不同人的个人数据可以相互共享、转让。如，个人消费习惯、位置信息、职业信息等。
2. 组织与组织之间的共享与使用：企业数据共享与使用是指两个或多个组织之间共享、转让数据的行为。如，财务数据、员工信息、客户信息等。
3. 公共数据共享与使用：公共数据共享与使用是指公共部门在做公共事项时所收集、储存、处理和使用的数据。如，天气数据、公共安全数据、新闻报道数据等。

## （三）目的
根据数据共享与使用的数据的目的是什么，数据共享与使用可以分为以下几类：

1. 数据分析：数据分析的目的一般是为了提供决策支持、解决业务问题或优化流程。
2. 数据交换：数据交换的目的是为了促进资源共享和合作。
3. 数据推送：数据推送的目的是为了满足用户的个性化需求。
4. 数据共享：数据共享的目的是为了促进数据的共享。
5. 数据传递：数据传递的目的是为了向其他实体传递数据。

## （四）隐私权衡取
数据共享与使用过程中，也会产生隐私权问题。

1. 目的限制：数据的使用者往往无法预见到数据的使用目的，他们只能按照自己的想象做出决定，而这可能因数据使用目的的特殊性而不同。所以，在数据共享与使用前，需要向数据使用者征询其数据使用目的和使用权利。
2. 信息主体责任：个人、组织、政府和公众在数据共享与使用过程中，都要承担一定的责任。他们要确保数据使用者有权利请求、更正、删除、暂停使用个人信息，并且保证对处理个人信息的机构、单位和个人负责。
3. 保护个人信息：数据共享与使用过程中，个人信息的保护尤为重要。在数据共享与使用过程中，应采取必要的保护措施，如合理选择保管环境，限制不必要的交流，定期清理个人信息。

因此，数据共享与使用过程中，应充分保护个人信息，尤其是在与公众共享信息时。
