# 3.1 排序算法

排序算法是计算机程序设计中非常重要的一种算法，用于对数据进行排序。常见的排序算法有：冒泡排序、选择排序、插入排序、希尔排序、快速排序、归并排序等。

## 3.1.1 冒泡排序

冒泡排序是一种简单的排序算法，它的基本思想是通过多次对数据进行交换，使较小的元素逐渐向前移动，较大的元素逐渐向后移动。

冒泡排序的时间复杂度为O(n^2)，其中n为数据的个数。

### 3.1.1.1 算法步骤

1. 从第一个元素开始，与后续的每个元素进行比较。
2. 如果当前元素大于后续元素，则交换它们的位置。
3. 重复第1步和第2步，直到整个数据序列有序。

### 3.1.1.2 代码实例

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr
```

## 3.1.2 选择排序

选择排序是一种简单的排序算法，它的基本思想是在未排序的数据中找到最小（或最大）元素，并将其放在已排序的数据的末尾。

选择排序的时间复杂度为O(n^2)，其中n为数据的个数。

### 3.1.2.1 算法步骤

1. 从未排序的数据中找到最小（或最大）元素。
2. 将找到的元素放在已排序的数据的末尾。
3. 重复第1步和第2步，直到整个数据序列有序。

### 3.1.2.2 代码实例

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_index = i
        for j in range(i+1, n):
            if arr[min_index] > arr[j]:
                min_index = j
        arr[i], arr[min_index] = arr[min_index], arr[i]
    return arr
```

# 3.2 搜索算法

搜索算法是计算机程序设计中非常重要的一种算法，用于在数据中查找满足某个条件的元素。常见的搜索算法有：线性搜索、二分搜索、深度优先搜索、广度优先搜索等。

# 3.2.1 线性搜索

线性搜索是一种简单的搜索算法，它的基本思想是从数据的第一个元素开始，逐个比较每个元素与查找的目标元素，直到找到匹配的元素或者遍历完整个数据。

线性搜索的时间复杂度为O(n)，其中n为数据的个数。

### 3.2.1.1 算法步骤

1. 从数据的第一个元素开始。
2. 比较当前元素与查找的目标元素。
3. 如果当前元素与查找的目标元素相等，则返回当前元素的索引。
4. 如果当前元素与查找的目标元素不相等，则继续比较下一个元素。
5. 重复第2步至第4步，直到找到匹配的元素或者遍历完整个数据。

### 3.2.1.2 代码实例

```python
def linear_search(arr, target):
    n = len(arr)
    for i in range(n):
        if arr[i] == target:
            return i
    return -1
```

# 3.3 数据结构

# 3.3.1 栈

栈是一种后进先出（LIFO，Last In First Out）的数据结构，它的基本操作有：入栈（push）、出栈（pop）和查看栈顶元素（peek）。

### 3.3.1.1 算法步骤

1. 创建一个空栈。
2. 将数据的第一个元素入栈。
3. 如果栈不为空，则将栈顶元素出栈。
4. 重复第2步和第3步，直到所有数据被处理完毕。

### 3.3.1.2 代码实例

```python
class Stack:
    def __init__(self):
        self.stack = []

    def push(self, item):
        self.stack.append(item)

    def pop(self):
        if self.is_empty():
            return None
        return self.stack.pop()

    def peek(self):
        if self.is_empty():
            return None
        return self.stack[-1]

    def is_empty(self):
        return len(self.stack) == 0

    def size(self):
        return len(self.stack)
```

## 6.1.1 为什么冒泡排序的时间复杂度为O(n^2)？

冒泡排序的时间复杂度为O(n^2)，因为在每次迭代中，它需要遍历整个数据序列，以便找到最大（或最小）元素并将其放在已排序的数据的末尾。因此，冒泡排序的时间复杂度随着数据的大小而线性增长。

## 6.1.2 为什么选择排序的时间复杂度也为O(n^2)？

选择排序的时间复杂度也为O(n^2)，因为在每次迭代中，它需要遍历未排序的数据以找到最小（或最大）元素，并将其放在已排序的数据的末尾。因此，选择排序的时间复杂度也随着数据的大小而线性增长。
