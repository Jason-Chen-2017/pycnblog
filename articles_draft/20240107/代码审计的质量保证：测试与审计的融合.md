# 2. 核心概念与联系

在了解新的代码审计方法之前，我们需要了解一些基本概念。

## 2.1 代码审计

代码审计是一种通过人工或自动方式检查代码的过程，旨在确保代码的质量、安全性和可靠性。代码审计可以包括以下几种方法：

1. 代码审查：通过人工阅读和评审代码，以检测和修复错误、漏洞和不良行为。
2. 静态代码分析：通过自动工具分析代码，以检测代码中的错误、漏洞和不良行为。
3. 动态代码分析：通过自动工具在运行时分析代码，以检测代码中的错误、漏洞和不良行为。

## 2.2 测试

测试是一种通过创建和执行测试用例，以验证软件系统是否满足需求和质量标准的过程。测试可以包括以下几种方法：

1. 单元测试：通过测试单个代码块或函数，以检测和修复错误、漏洞和不良行为。
2. 集成测试：通过测试多个单元组合成的子系统，以检测和修复错误、漏洞和不良行为。
3. 系统测试：通过测试整个软件系统，以检测和修复错误、漏洞和不良行为。

## 2.3 测试与审计的融合

测试与审计的融合是一种将测试和审计方法结合使用的新方法，旨在提高代码审计的质量和可靠性。通过融合测试和审计，我们可以利用测试的自动化优势，提高审计的效率和准确性。同时，通过融合审计和测试，我们可以利用审计的专业知识，提高测试的覆盖率和准确性。
