# 1.背景介绍


当你拿到一个编程题目之后，面试官一般会让你写一下自己对这个问题的理解、尝试用自己的代码实现了吗？如果没有回答出来，或者没有给出很好的代码解决方案，那么你就会被问到一些复杂的问题，比如时间复杂度，空间复杂度，怎么优化等。很多技术人员不了解这些知识，只能自我安慰，觉得面试过不好，却无法在行中获得提升。他们往往就此落寞，过着看不到光明的日子。

为了帮助技术人员走出这种困境，我们整理了一份《程序员面试技巧系列：面试后的后续处理》(下简称《TPL})，主要分享技术人员面试过程中的后续处理策略和方法。阅读本文之前，推荐先了解《程序员简历编写秘籍》《程序员头条效应：如何通过人头和热度赢得技术人的青睐》两本书。这两本书将极大的帮助你更加精准地制作简历和自我介绍，帮助你在面试过程中脱颖而出。

面试官很重要，但也不是绝对的。如果你真的准备好迎接面试，并且面试官真诚相待，那么面试后的后续处理依然可以起到良好的效果。通过与面试官交流，树立正确的态度，善于分析反馈，积极主动地寻求建议，能帮助你在面试中更上一层楼。

# 2.核心概念与联系
## 2.1后续处理策略及其分类
面试后的后续处理分为三个阶段：
1. 激励/感谢/祝福：面试通过、入职、晋升都会给面试者带来激励和感谢；
2. 提升：当面试者能够形成合理的规划并努力完成面试任务时，需要将面试的结果转化为工作上的优势；
3. 问询：当面试者能够清楚的认识到自己的薄弱点、需要改进的地方时，需要根据面试结果向面试官提出宝贵建议。

## 2.2主要关注点
后续处理的主要关注点如下：
1. 收集面试者信息：通过收集信息进行调查、参考面试者对问题的理解程度，为下一步后续处理提供更全面的信息；
2. 确认候选人：针对不同的情况，选择合适的人才做后续处理；
3. 提升技能：提升面试者能力、学习新技术、准备面试；
4. 为竞争对手做准备：做好本职工作后，积极备战面试前的竞争对手的比赛，从而吸引更多优秀人才；
5. 建立个人品牌：展示自身的才能和专长，为公司和团队打造个人档案，吸引更多业务伙伴；

## 3.1收集面试者信息
收集信息的方法一般有两种：
- 电话访谈：亲自与面试官面谈、记录下对方的详细介绍、面试经历、语言表达能力、热情沟通能力等；
- 网络搜索：利用互联网搜索引擎、社交媒体平台或企业内部信息共享工具搜集面试者的信息，包括姓名、性别、年龄、联系方式、教育背景、工作经验、项目经历、专业技能、兴趣爱好等；

收集到的信息应包含以下部分：
1. 基本信息：姓名、性别、年龄、所在城市、当前工作、教育经历、项目经历、专业技能、兴趣爱好等；
2. 面试经历：包括历次面试的机构、职位、评分、评论、面试内容、面试效果、面试心得等；
3. 技术能力：包括计算机基础知识、编程语言、开发工具、数据库、云计算、云服务等相关技能的掌握情况；
4. 软实力：包括团队协作能力、沟通协调能力、领导风格、组织能力等软实力的培养情况；
5. 其他信息：包括面试官对面试者的印象、为什么选择你、期望的工作环境、职业发展方向等。

## 3.2确认候选人
可以通过以下方式确认候选人：
1. 根据面试官的要求判断：有些面试官可能会根据你提供的项目经历、学历、技能等信息进行筛选；
2. 通过自我评估：在面试前进行自我评估，列出你的优缺点、将来想要达成的目标、意愿、加分项、减分项、可改进的地方等，在面试中可以进行综合调整，从而确认最具竞争力的人；
3. 从职位需求角度考虑：不同公司对于不同类型的人才都有不同的需求，你可以结合公司的招聘政策，了解不同类型的人才需求；
4. 关注公司动态：在招聘网站上关注该公司近期的招聘活动，了解公司招聘方向和规模的变化，随时跟踪该公司的最新消息；
5. 从实习生角度考虑：作为实习生，你也许只知道自己公司和面试官的相关信息，但实际上你还可以结合自己的实践经验，和面试官探讨工作细节。因此，即使是初级的实习生也可以成为你后续的发展重点。

## 3.3提升技能
提升技能的一些方式如下：
1. 提升个人能力：有些面试官可能会为你考核个人能力，包括编程能力、解决问题的能力、沟通协调能力等；
2. 倾听面试官的意见：你可以和面试官一起探讨自己的优缺点、意见，以及自己想得到哪方面的帮助，促进双方的沟通和了解；
3. 提升学习能力：除了在面试中获得反馈外，也可以通过参加面试官组织的相关课程、交流群，提升自己对相关技术的掌握和理解；
4. 寻找志同道合的同事：与志同道合的同事建立沟通关系，可以共同分享自己的心路历程、工作经验，也可以倾诉面试的心酸与苦闷；
5. 增加项目经验：加入有价值的项目，增强自己对项目管理、沟通技巧的掌握，提高项目的完工率、质量；
6. 进修学习：参加大学、研究所、公开课等培训班，提升职场竞争力、个人能力；
7. 拓展资源：除了上述提升之外，你还可以用自己的时间和金钱投入到开源社区、合作社区、技术论坛等公益性资源中，在提升自己能力的同时丰富自己的经验。

## 3.4为竞争对手做准备
为竞争对手做准备的一些建议如下：
1. 调整时间：把面试的时段安排得合适，避免影响到对方的正常工作和生活；
2. 提前准备材料：收集要发送的宣传材料，如名片、面试前材料、个人介绍、联系方式等；
3. 设置路线图：制定预计的时间表，记录下要准备的面试题目和项目，在关键的时候刻意放慢脚步；
4. 不轻易挂科：如果面试失败，尽快调整计划，重新安排时间；
5. 注意伤害：不要让不良印象伤害到自己，控制住自己的情绪，冷静处理面试官的指控；
6. 提升个人声誉：多向邻居或朋友介绍自己的工作经历、个人能力和项目经验，提升自己的名声，在面试结束后与对方面谈，彰显个人魅力。

## 3.5建立个人品牌
建立个人品牌的一些建议如下：
1. 展示自我价值：在面试的过程中，不要局限于技术本身，多提及自己的价值观和理念，帮助对方接近自己；
2. 勇于挑战：面试之前不要抱怨自己不合适，也不要怕失去竞争力，适时接受挑战，主动向面试官证明自己；
3. 主动传播：定期发布面试题解，撰写技术博客等，主动推广自己的技能和经验；
4. 保持持续更新：除了发布面试题解和技术博客外，也应该多关注竞争对手的最新招聘信息，保持竞争力；
5. 与同行交流：结识更多业界的朋友，分享自己的经验、心得和感悟，帮助自己成长。

# python代码示例
def bubble_sort(arr):
    n = len(arr)
    
    for i in range(n):
        # Last i elements are already sorted
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
                
    return arr


arr = [64, 34, 25, 12, 22, 11, 90]
sorted_arr = bubble_sort(arr)
print("Sorted array is:")
for i in range(len(sorted_arr)):
    print("%d" %sorted_arr[i])
```

// c++代码示例
#include <bits/stdc++.h>
using namespace std;

int main() {
  int arr[] = {64, 34, 25, 12, 22, 11, 90};
  
  // Bubble sort implementation 
  int n = sizeof(arr)/sizeof(arr[0]);
  for (int i=0; i<n-1; i++)
      for (int j=0; j<n-i-1; j++)
          if (arr[j] > arr[j+1])
              swap(arr[j], arr[j+1]);
              
  cout << "Sorted array:\n";
  for (int i=0; i<n; ++i)
      cout << arr[i] <<'';

  return 0;
}
```

# 5.未来发展趋势与挑战
面试后的后续处理策略随着技术的发展和产业的变化而变迁。作为技术人员，面试前后对自己的提升与否，都离不开各种因素的影响，尤其是在面试前期应付面试官提出的各种“难”题时。在这里，我们总结下面试后的后续处理策略的发展趋势与挑战。

1. 云计算的发展：云计算给IT人员提供了更多的创造力和选择。面试后的后续处理也要相应地升级，面试者的云技术知识及掌握程度，以及解决面试问题的能力，都会成为云计算相关人员的主要考察点。
2. AI时代：AI技术的崛起将改变整个行业的方向，面试官需要更好的应对。后续处理手册也需要同步更新，增加对AI知识的考察。
3. 大数据和人工智能的普及：大数据和人工智能技术也将是新的技术考察点。因此，后续处理手册也会跟上这一趋势，需要相应地扩充内容。
4. 中国经济的恢复和社会的进步：中国经济的恢复和社会的进步，让考生们有机会享受到更多的收获。在后续处理策略上，面试官不仅需要更多的注意面试者的成长潜力，更需要注重学业成长。
5. 职场的不景气：在工作岗位频繁更换的大背景下，职场的不景气会对考生们的后续处理产生一定影响。因此，后续处理策略的设计上也不能完全依赖于一次面试。面试后需要根据面试者的自身情况进行适时的调整和改进。
