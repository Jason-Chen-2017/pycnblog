## 4.1 代码实例

我们假设有一个简单的API，用于获取用户信息。这个API有两个版本，分别是v1和v2。我们将通过一个简单的Python程序来实现这个API的版本控制。

```python
import requests

class UserAPI:
    def __init__(self, version):
        self.version = version
        self.base_url = 'https://api.example.com/user'

    def get_user_info(self, user_id):
        url = f'{self.base_url}/{self.version}/{user_id}'
        response = requests.get(url)
        if response.status_code == 200:
            return response.json()
        else:
            return None

user_api_v1 = UserAPI('v1')
user_info_v1 = user_api_v1.get_user_info('1')
print(user_info_v1)

user_api_v2 = UserAPI('v2')
user_info_v2 = user_api_v2.get_user_info('1')
print(user_info_v2)
```

在这个代码实例中，我们首先定义了一个`UserAPI`类，它有一个构造函数，接受一个版本参数，并设置了一个基本URL。然后，我们创建了两个`UserAPI`对象，分别对应v1和v2版本的API。最后，我们调用了`get_user_info`方法来获取用户信息，并打印了结果。

## 4.2 详细解释说明

在这个代码实例中，我们通过Python程序来实现API的版本控制。首先，我们定义了一个`UserAPI`类，它有一个构造函数，接受一个版本参数，并设置了一个基本URL。这个类表示一个API，可以根据不同的版本来实现不同的API。

然后，我们创建了两个`UserAPI`对象，分别对应v1和v2版本的API。这样，我们可以根据需求选择合适的API版本来获取用户信息。

最后，我们调用了`get_user_info`方法来获取用户信息，并打印了结果。这个方法通过发送HTTP GET请求来实现API的调用。如果请求成功，它会返回用户信息；如果请求失败，它会返回`None`。
