# 2.基本概念术语说明
## 2.1 开放源码软件（Open Source Software）
开放源码软件（Open Source Software，缩写 OSS），是一种允许他人免费使用、修改、再发布的计算机软件源代码。一般情况下，OSS 是指源代码可以自由获得、修改、共享的软件。其目的在于降低软件成本，提高软件质量，降低软件的集成成本、部署难度，增加开源的普及率，加强对软硬件的创新性开发，以及促进全球化协作。

## 2.2 开源协议（Open Source License）
开源协议，通常称之为“许可证”，是用户可以自由下载、使用或修改代码的权利与义务。它是将开发者关于软件代码（包括源代码、设计图纸、样机程序、测试用例、用户手册、参考材料）的使用、复制、分发、传播以及再许可等权限和义务规范化的一种文件。根据软件是否以开放源代码形式进行授权，开源协议分为四种类型：
- 消费型开源许可证（Consumer-Oriented Open Source Licenses）：如 GNU GPL、Mozilla Public License等。允许任何人使用该许可证发布的代码；但要求软件的修改、衍生品或者衍生版本必须继续遵循相同的许可协议；
- 宽松型开源许可证（Permissive Open Source Licenses）：如 BSD、MIT、Apache等。允许个人、公司以及其他组织使用，但限制软件的改动范围；
-  copyleft 型开源许可证（Copyleft Open Source Licenses）：如 GNU LGPL、Mozilla Public License v2.0等。要求软件的修改、衍生品或者衍生版本必须继续遵循相同的许可协议，而且这些版权声明必须保持与原始版权声明一致；
- 专有闭源软件（Proprietary Software）：这是一种非开放源代码的软件。例如，苹果公司的 macOS 操作系统就是一种专有闭源软件。

## 2.3 开源项目（Open Source Project）
开源项目，又称“社区”。它是由开发者共同参与的开发、维护、并最终形成的一套基于开放协议的软件产品。社区围绕着某个开源主题建立，不断接受、修正、完善代码，促进软件的发展和更新。开源项目具有以下特征：
- 全球性：开源项目已经遍布全球，用户分布于各个国家；
- 开放性：开源项目的所有资源都是开放的，任何人都可以访问、下载、阅读、修改、扩展、分享。这是让志愿者们快速学习并加入到开发团队中的好机会；
- 透明性：所有的活动都可以追溯到社区成员的公开贡献记录。社区成员的贡献总结、文档、决策过程等都可以被所有参与者看到，这确保了社区的透明度；
- 分布式协作：开发人员可以在不同的时间、地点和角色上参与到开源项目中。这可以确保不同人的贡献得到尊重和认可，确保项目的持续开发。

## 2.4 开源软件（Open Source Software）
开源软件，又称“软件”。它是根据开源协议发布的，可以直接使用的软件。开源软件一般情况下都具有高水平的质量、可靠性和可用性。比如，Linux 操作系统就是开源软件。

## 2.5 托管网站（Hosting Website）
托管网站，是用于存放开源项目的网站，一般情况下，开源项目都会将代码托管到托管网站。托管网站提供众多服务，如代码管理、构建环境设置、静态页面托管、动态页面托管、性能监控、安全防护等。开源项目通过托管网站可以轻松地接收、验证、处理提交的贡献，提高开源项目的吸引力和易用性。

## 2.6 版本控制系统（Version Control System）
版本控制系统（Version Control System，VCS），用于跟踪、管理和发布软件的变更。常用的 VCS 有 Git、Mercurial、SVN。VCS 提供了记录每次修改、历史记录、并提供回滚功能等诸多便利功能，可以有效地管理软件的开发进程，实现项目代码的协调、共享、测试和迭代等。

## 2.7 bug 追踪系统（Bug Tracking System）
bug 追踪系统，用于跟踪和记录软件运行过程中出现的问题。当软件遇到问题时，可以通过 bug 追踪系统记录错误信息，方便开发人员和测试人员排查和修复 Bug。

# 3.核心算法原理和具体操作步骤
## 3.1 Github Pages
GitHub Pages 是 GitHub 提供的静态网页托管服务。它的特点是在线更新、无限空间、安全稳定、免费域名。你可以通过 DNS 配置，绑定你的域名，将自己创建的静态网站发布到互联网上。

### 3.1.1 创建仓库
首先，你需要登录到 GitHub 账户，然后点击右上角的 New repository 按钮，创建一个新的仓库。新建仓库的页面如下：


填写仓库名称，描述，公开或者私有。这里我们创建一个名为 “yourusername.github.io” 的仓库，表示这个仓库是用来发布你的个人网站的。

接下来，选择你的仓库模板。这里建议选取一个纯文本的模板，这样你就可以开始编写你的网站了。

### 3.1.2 克隆本地仓库

打开命令行窗口，输入以下命令克隆仓库：

```bash
git clone https://github.com/[你的用户名]/[你的仓库名].git
```

例如，如果你的用户名是 "YourUsername"，仓库名是 "blog",那么你应该输入：

```bash
git clone https://github.com/YourUsername/blog.git
```

成功克隆仓库后，进入目录：

```bash
cd blog
```

### 3.1.3 在本地编写网站
在完成仓库的克隆和切换工作目录后，你可以在此目录下编写网站的文件。可以使用任何编辑器（如 Sublime Text、Notepad++ 或 VS Code）。

如果你要发布的网站是一个 HTML 文件，那么只需把它放入根目录就行。如果你要发布的网站不是一个 HTML 文件，例如是一个 Jekyll 站点，那么还需要额外配置才能发布成功。

配置好网站后，你可以通过命令 `jekyll serve` 来预览网站，也可以通过编译后的文件来发布网站。

### 3.1.4 提交修改并推送
编写完成网站后，你可以通过命令 `git status` 查看当前目录下的所有修改，确认无误后，使用命令 `git add.` 将所有修改添加到暂存区。

然后，使用命令 `git commit -m "[提交注释]"` 将修改提交到本地仓库，"[提交注释]" 请换成详细的提交信息。

最后，使用命令 `git push origin master` 将修改推送到远程仓库。

至此，你的网站已经发布成功！你可以在浏览器中访问 `[你的仓库名].github.io`，查看效果。当然，你也可以绑定自己的域名来访问网站。

### 3.1.5 添加自定义域名
要给网站绑定自己的域名，需要先购买一个域名，然后修改 DNS 设置。

1. 购买域名：一般来说，想要绑定自己的域名，首先需要有一个域名。如果你还没有域名，那么可以去注册一个域名服务商。
2. 修改 DNS 设置：DNS （Domain Name System）服务器负责把域名转换为 IP 地址。你需要将你的域名解析到 GitHub Pages 服务的 IP 上，这样才能正常访问网站。不同的域名服务商提供了不同的 DNS 服务器，但是它们都使用了类似的方法。
3. 添加 CNAME 文件：如果你的域名是在 GitHub Pages 服务之外购买的，则需要添加 CNAME 文件。CNAME 文件的内容应该是你的域名，类似于 www.example.com。

完成以上设置后，你的域名就可以正常访问你的网站。

## 3.2 Jekyll 静态网站生成器
Jekyll 是一种简单的、博客aware、可快速生成静态网站的工具。它使用了 Ruby 语言编写，拥有丰富且灵活的插件支持。它还支持 Markdown 和 Liquid 模板语言，可以轻松地生成复杂的网页。

### 3.2.1 安装 Jekyll
安装 Jekyll 非常简单。你只需要下载安装包，然后按照提示一步步安装即可。如果你的系统中没有 ruby 或 gem，请先安装它们。

安装完成后，你可以通过终端或命令行，执行以下命令检查 Jekyll 是否安装成功：

```bash
jekyll -v
```

输出结果类似于：

```bash
jekyll 3.8.5 -- Python-based static site generator
```

### 3.2.2 使用 Jekyll 创建站点
创建 Jekyll 站点，需要先创建一个新的仓库，然后克隆到本地。

```bash
jekyll new myblog
```

然后进入 myblog 目录，启动本地服务器预览站点：

```bash
cd myblog
bundle exec jekyll serve
```

这样，你的 Jekyll 站点就启动成功了，你可以通过 http://localhost:4000 访问。

### 3.2.3 编写 Jekyll 页面
Jekyll 使用 Markdown 语法来撰写页面，默认使用.md 后缀的文件名。页面的元数据放在 YAML 块中，字段之间使用冒号 : 分隔。

一个典型的 Jekyll 页面可能长这样：

```yaml
---
layout: post
title: Hello World!
categories: My Blog
tags:
  - hello
  - world
---

This is my first post on this blog powered by Jekyll!
```

第一行的 layout 指定了布局模板，这里指定的是 post。

第二行的 title 是页面的标题。

第三行的 categories 指定了页面所属分类。

第四行的 tags 指定了页面的标签。

第六行之后是页面的内容。

Jekyll 会自动生成相应的网页，你可以预览页面效果。
