# 2.核心概念与联系

在可重构计算中，我们需要关注两个核心概念：可重构子问题（subproblem）和可重构计算图（recomputation graph）。

## 2.1 可重构子问题

可重构子问题是指在计算过程中，某个子问题的计算结果可以被多次重复利用的问题。这种子问题通常具有以下特征：

1. 子问题之间存在依赖关系，即计算某个子问题的结果时，需要依赖其他子问题的结果。
2. 子问题的计算结果具有稳定性，即在某个时刻计算出的结果在后续计算中仍然有效。

通过识别可重构子问题，我们可以减少计算次数，从而提高计算效率。

## 2.2 可重构计算图

可重构计算图是一种表示可重构计算过程的图形结构。它由节点（nodes）和边（edges）组成，其中节点表示计算子问题，边表示子问题之间的依赖关系。

可重构计算图的主要特点是：

1. 图中的节点表示可重构子问题。
2. 图中的边表示子问题之间的依赖关系。
3. 图中的节点可以被分为多个层次，每个层次对应一个计算阶段。

通过构建可重构计算图，我们可以更好地理解计算过程，并找到可重构子问题以提高计算效率。
