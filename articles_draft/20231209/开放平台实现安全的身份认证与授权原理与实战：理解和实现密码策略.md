# 2.核心概念与联系

在本节中，我们将介绍身份认证与授权的核心概念，并探讨它们之间的联系。

## 2.1 身份认证

身份认证是指验证用户是否具有合法的身份，以确保用户是真实的人。身份认证通常包括以下几个步骤：

1. 用户提供身份认证信息，如用户名和密码。
2. 系统验证用户提供的身份认证信息是否与预期的信息匹配。
3. 如果验证成功，则用户被认为是合法的用户，可以进行后续操作。

## 2.2 授权

授权是指确定用户在系统中可以执行哪些操作。授权通常包括以下几个步骤：

1. 用户请求执行某个操作。
2. 系统检查用户是否具有执行该操作的权限。
3. 如果用户具有执行该操作的权限，则系统允许用户执行该操作。

## 2.3 身份认证与授权之间的联系

身份认证与授权是两个相互依赖的过程，它们之间存在以下联系：

1. 身份认证是授权的前提条件。只有通过身份认证后，用户才能进行授权。
2. 授权是身份认证的扩展。身份认证只能确定用户是否合法，而授权则可以确定用户在系统中可以执行哪些操作。

```python
import hashlib

def authenticate(username, password):
    # 使用用户名和密码生成密码摘要
    digest = hashlib.sha256((username + password).encode('utf-8')).hexdigest()

    # 检查密码摘要是否与预期的摘要匹配
    if digest == 'expected_digest':
        return True
    else:
        return False
```

def authorize(username, action):
