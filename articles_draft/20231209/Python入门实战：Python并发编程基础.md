### 2.3 异步IO

异步IO是一种I/O操作方式，允许程序在等待I/O操作完成的同时继续执行其他任务。这种方式可以提高程序的执行效率，因为它避免了程序在等待I/O操作的过程中的阻塞。在Python中，异步IO可以通过`asyncio`模块实现。

### 4.1 线程的代码实例

以下是一个使用Python线程编程的代码实例：

```python
import threading

def worker():
    print("线程正在执行任务...")

threads = []
for i in range(5):
    t = threading.Thread(target=worker)
    t.start()
    threads.append(t)

for t in threads:
    t.join()
```

在这个代码实例中，我们创建了5个线程，并启动它们。每个线程都执行一个名为`worker`的目标函数。我们使用`join`方法等待所有线程完成执行，并获取它们的结果。

### 4.2 进程的代码实例

以下是一个使用Python进程编程的代码实例：

```python
import multiprocessing

def worker():
    print("进程正在执行任务...")

if __name__ == "__main__":
    processes = []
    for i in range(5):
        p = multiprocessing.Process(target=worker)
        p.start()
        processes.append(p)

    for p in processes:
        p.join()
```

在这个代码实例中，我们创建了5个进程，并启动它们。每个进程都执行一个名为`worker`的目标函数。我们使用`join`方法等待所有进程完成执行，并获取它们的结果。

### 4.3 异步IO的代码实例

以下是一个使用Python异步IO编程的代码实例：

```python
import asyncio

async def worker():
    print("异步任务正在执行...")
    await asyncio.sleep(1)
    print("异步任务已完成")

if __name__ == "__main__":
    loop = asyncio.get_event_loop()
    tasks = []
    for i in range(5):
        t = loop.create_task(worker())
        tasks.append(t)

    loop.run_until_complete(asyncio.gather(*tasks))
    loop.close()
```

在这个代码实例中，我们创建了5个异步任务，并启动它们。每个异步任务都执行一个名为`worker`的目标函数。我们使用`gather`函数将所有任务组合成一个Future对象，并使用`run_until_complete`方法等待所有任务完成执行。
