# 1.简介
  

## 什么是Web应用安全
Web应用安全(英文：Web Application Security)，是一个广义上的概念，指的是基于网络的应用程序的安全管理，其主要关注以下四个方面：

1、网络攻击防护

2、数据安全保障

3、身份认证和访问控制

4、安全日志管理

随着互联网的快速发展、信息化建设加快、信息泄露频发等众多安全事件的发生，越来越多的人开始担忧网络安全和信息安全问题。因此，建立一个全面的、健全的Web应用安全保障体系是一件非常重要的事情。

Web应用安全可分为静态Web应用安全、动态Web应用安全、移动端Web应用安全、云计算平台Web应用安全等。本文仅对Web应用安全进行讨论，不涉及其他类型安全问题。

## 为什么要做Web应用安全
无论在企业还是个人都需要为自己的网站、应用建立一个良好的Web应用安全保障体系。由于Web应用具有高度的用户隐私性和敏感性，需要采取一系列的安全措施来确保用户数据的安全、信息的准确性、系统的可用性和性能。

建立一个全面而严格的Web应用安全体系需要解决以下几个关键点：

1、识别风险威胁

2、评估安全风险

3、识别并分类出高危漏洞

4、制定合理的安全控制措施

5、持续维护和更新安全措施

做好Web应用安全保障，可以有效防范企业和个人用户的各种安全风险，保障公司或个人在线业务的信息安全，从而促进经济的发展。

# 2.基本概念术语说明
## 计算机安全攻防术语表
### 概念理解
- 主动攻击（Active Attack）：黑客通过攻击者构建的恶意程序或者被动入侵，利用资源对目标主机实施攻击，通常要在连接目标主机之前完成整个攻击过程。

- 被动攻击（Passive Attack）：黑客借助目标主机上运行的正常服务，诱骗该主机去执行一些攻击活动。在没有得到主机权限的情况下实施的攻击方法。

- 免疫攻击（Defense Evasion Attack）：攻击者利用一些漏洞或缺陷使得防御系统无法正确识别攻击，或者当防御系统检测到攻击后采取的一种缓解措施。

- 盲目的攻击（Blind Attack）：攻击者只知道自己的目的，并不知道自己在干什么。他也不会判断对方是否有能力防御。盲目的攻击用于暴力破解密码，获取明文文件等。

- 垂直渗透攻击（Vertical Attack）：攻击者主要通过系统漏洞入侵，得到系统的管理员权限，如SSH登录，上传、下载文件，获得操作系统权限等。

- 水平渗透攻击（Horizontal Attack）：攻击者通过多个系统同时入侵，达到渗透系统的目的，如入侵内网。

- 社工攻击（Social Engineering Attack）：攻击者通过非法获取信任关系，与受害者勾结，诱导受害者相信攻击者所提供的信息，制造恐惧心理，希望他们同意向攻击者索取信息、交付物品或对其进行伤害。

- 暴力破解攻击（Brute Force Attack）：攻击者通过尝试所有可能的密码组合来破解加密系统，通常采用字典攻击的方式，将密码输入系统多次，直至成功。

- 字典攻击（Dictionary Attack）：攻击者通过穷举方式猜测出整个密码列表中的正确密码，或是通过一些猜测密码攻击者通过重复试错的方式将字典中的单词或字符作为密码输入系统，直至成功。

- 拒绝服务攻击（Denial of Service Attack）：攻击者向系统发送大量的请求，使系统超负荷运转，导致正常服务无法响应。包括SYN Flood、UDP Flood、Smurf attack等。

- 入侵检测系统（Intrusion Detection System）：ID系统是一种网络安全机制，它可以监控网络流量，根据规则和启发式算法检测异常行为，将异常行为引起的危害降低到最低程度。

- 漏洞扫描工具（Vulnerability Scanner Tool）：漏洞扫描工具是一种能够识别系统中存在的安全漏洞，并对其进行评估和预警的安全工具。

- 反病毒软件（Anti-virus Software）：反病毒软件可以帮助用户避免因恶意软件侵害系统的安全。

- 防火墙（Firewall）：防火墙是一种网络安全策略，可以过滤网络流量，阻止恶意流量进入系统，并实施相应的控制措施。

- 入侵检测（Intrusion Detection）：入侵检测系统就是为了辅助管理员识别和分析系统里的异常活动，并发现有威胁的入侵行为，并及时给予应对和补救措施。

- 应用层审计（Application Level Auditing）：应用程序审计是对应用程序在系统中的运行状况和安全性进行全面的检查，包括检查应用程序对网络的使用情况、数据传输情况、文件处理情况、数据库操作情况等。

- 文件完整性验证（File Integrity Verification）：文件完整性验证是指对文件的完整性进行核验，如检测文件是否被篡改或遭到破坏。

- 访问控制列表（Access Control List，ACL）：访问控制列表是一种控制用户访问资源的安全机制。

- 访问控制矩阵（Access Control Matrix）：访问控制矩阵是一种简易、低级别的访问控制机制。

- 会话管理（Session Management）：会话管理是通过某种手段，允许用户访问某个网络应用时，验证用户身份并记录用户的操作，避免恶意用户冒充其它用户进行操作。

- 敏感数据加密（Sensitive Data Encryption）：敏感数据加密是对用户数据进行加密，以提高数据的机密性、完整性和可用性。

- 应用白名单（Application Whitelist）：应用白名单是一种访问控制机制，以限制特定应用的访问权限。

- 集成应用安全测试（Integrated Application Security Testing）：集成应用安全测试（IAST）是一种软件测试方法，它是在实际应用环境下模拟恶意攻击行为，检测软件系统及其组件的潜在安全风险，包括设计、开发、测试、部署等环节。

- 渗透测试（Penetration Test）：渗透测试是指评估网络安全产品或系统的安全性和可靠性的一项技术活动，其中包含攻击者对网络系统的攻击，目的是找寻系统的安全漏洞，然后修补它们，以保证系统长久运行、不受损害。

- 测试人员（Security Tester）：测试人员可以是政府官员、安全专业人士或软件工程师，他负责测试安全系统的安全性和可靠性，以确保其能符合公司或组织的要求。

- 安全开发生命周期（Secure Development Lifecycle）：安全开发生命周期（SDLC）是指以实现可靠的安全产品或系统为目标，定义和规范安全开发流程，并经过多轮迭代之后，开发出的产品或系统具有良好的安全性、可用性和可靠性。

- 对抗回归测试（Adversarial Regression Testing）：对抗回归测试（ART）是一种软件测试技术，旨在识别产品或系统中已知安全漏洞，并在生产环境中重新引入这些漏洞，以确定它们是否已被修复。

- 黑盒测试（Blackbox Testing）：黑盒测试又称结构测试、边界测试、内部测试或外观测试。在黑盒测试中，参与测试的对象不是产品的源代码，而是产品的功能或接口。

- 白盒测试（Whitebox Testing）：白盒测试又称逻辑测试、内在测试或内部函数测试。白盒测试是指参与测试的对象是产品的源代码。

- 代码级安全测试（Code-based Security Testing）：代码级安全测试是一种软件测试技术，它使用静态代码扫描工具对产品的源代码进行分析，查找其中的安全漏洞。

- 命令注入（Command Injection）：命令注入是指攻击者在向运行中的应用程序传递恶意命令代码的攻击行为。

- SQL注入（SQL Injection）：SQL注入是指攻击者通过在表单、链接、cookie或其他输入字段中输入恶意的SQL指令，绕过了防火墙并直接在数据库服务器上执行命令，从而获取或修改敏感数据。

- XSS跨站脚本（Cross-Site Scripting）：XSS跨站脚本攻击是指攻击者通过插入恶意脚本代码，欺骗用户点击链接或者提交表单，对用户浏览器上的信息进行篡改，从而盗取用户数据，或者窃取用户Cookie、甚至执行任意代码。

- CSRF跨站请求伪造（Cross-Site Request Forgery）：CSRF跨站请求伪造攻击是一种恶意网站诱导受害者进入第三方网站，在第三方网站中，受害者登录状态下向网站发起恶意请求，利用双重验证的特点，比如口令验证，个人信息确认等，进行非法操作。

- 正向代理（Forward Proxy）：正向代理隐藏客户端的真实IP地址，通过代理服务器发送请求到网站服务器。

- 反向代理（Reverse Proxy）：反向代理是指以服务器自身来接受客户端的请求，并将客户请求转发给内部网络上的其他服务器，此时客户端不知道请求真实的被代理服务器。

- OAuth协议（Open Authorization）：OAuth协议是一种授权标准，允许用户赋予第三方应用或网站代表自己授权某项操作，而不需要将用户名和密码提供给第三方应用或网站。

- OpenID协议（OpenID Connect）：OpenID Connect是OpenID的升级版，它增加了安全层级、认证参数、和扩展属性，用于更好地满足安全需求。

- TLS/SSL协议（Transport Layer Security / Secure Sockets Layer）：TLS/SSL协议是安全套接层（Secure Socket Layer）和传输层安全（Transport Layer Security）的缩写，是一种安全协议，用于两个通信应用程序之间提供保密性和数据完整性。

- JWT（JSON Web Tokens）：JWT（JSON Web Tokens），是为了在网络应用间进行鉴权而设计的一种声明性的安全tokens（令牌）。

- OWASP（Open Web Application Security Project）：OWASP是一个开放的社区，致力于推动软件安全领域的发展。

- PCI DSS（Payment Card Industry Data Security Standard）：PCI DSS是美国商务部为支付卡行业开发的一个安全标准。

- HSTS（HTTP Strict Transport Security）：HSTS是一种由网景工作室提出的Web安全增强功能。通过它可以在web浏览器上强制客户端只能通过HTTPS连接，从而可以有效防止中间人攻击、数据劫持和会话劫持。
