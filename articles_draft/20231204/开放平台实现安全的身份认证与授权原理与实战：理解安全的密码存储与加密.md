# 2.核心概念与联系

在开放平台中，身份认证与授权的核心概念包括：

- 用户身份：用户在系统中的唯一标识，通常是用户名或电子邮件地址。
- 密码：用户在系统中设置的安全密码，用于验证用户身份。
- 授权：用户在系统中的权限，用于确定用户可以访问哪些资源。
- 密码存储：一种安全的方法，用于存储用户的密码，以便在用户尝试登录时进行验证。
- 密码加密：一种安全的方法，用于加密用户的密码，以便在传输和存储过程中保护其安全性。

这些概念之间的联系如下：

- 用户身份与密码存储：用户身份是密码存储的基础，因为密码存储用于验证用户身份。
- 密码与密码加密：密码加密是密码存储的一部分，因为密码加密用于保护密码的安全性。
- 授权与密码存储：授权与密码存储密切相关，因为授权用于确定用户可以访问哪些资源，而密码存储用于验证用户身份。
