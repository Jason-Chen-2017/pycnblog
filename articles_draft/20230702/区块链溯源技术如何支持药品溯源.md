区块链技术是一种分布式账本技术，可以实现药品信息的去中心化存储和共享。药品溯源技术可以通过区块链技术实现药品信息的不可篡改、可追溯、可验证，提升药品溯源的效率和安全性。

2.3. 相关技术比较
常见的药品溯源技术包括数据库、文件系统、分布式数据库等。与区块链技术相比，这些技术都存在一些局限性，如数据篡改、数据隐私等问题。而区块链技术具有不可篡改、可追溯、可验证的特点，能够有效解决这些问题。

```
@RestController
@RequestMapping("/api")
public class溯源控制器 {

    @Autowired
    private coreService coreService;

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    @Autowired
    private springRepository repository;

    public String getAll药品信息() {
        String url = "http://localhost:8080/api/all";
        return restTemplate.getForObject(url, String.class);
    }

    public String get药品信息(String id) {
        String url = "http://localhost:8080/api/药品信息/" + id;
        return restTemplate.getForObject(url, String.class);
    }

    public void add药品信息(String id, String name, String price) {
        String url = "http://localhost:8080/api/药品信息";
        coreService.add药品信息(id, name, price);
    }

    public void update药品信息(String id, String name, String price) {
        String url = "http://localhost:8080/api/药品信息/" + id;
        coreService.update药品信息(id, name, price);
    }

    public void delete药品信息(String id) {
        String url = "http://localhost:8080/api/药品信息/" + id;
        coreService.delete药品信息(id);
    }
}
```

4.4. 代码讲解说明
- `@RestController`：声明本类是一个RESTful Web服务的控制器。
- `@RequestMapping("/api")`：声明本类是一个RESTful Web服务的请求处理类，对应的请求URL为"/api"。
- `@Autowired`：声明本类中的所有变量都注入了依赖关系。
- `@Bean`：声明本类中的所有属性都注入了依赖关系。
- `restTemplate`：声明一个RestTemplate对象，用于调用RESTful Web服务的API。
- `springRepository`：声明一个SpringRepository对象，用于调用数据库操作。
- `@GetMapping`：声明一个用于获取所有药品信息的GET请求方法。
- `@GetMapping("/api/{id}")`：声明一个用于获取特定药品信息的GET请求方法，其中`{id}`是一个URL参数。
- `@PostMapping`：声明一个用于添加药品信息的POST请求方法。
- `@PutMapping`：声明一个用于修改药品信息的PUT请求方法。
- `@DeleteMapping`：声明一个用于删除药品信息的DELETE请求方法。
