# 2.Java编程思想
## 2.1 概念

### 2.1.1 Java平台
Java平台（Java SE，Java EE，Java ME）是由Sun Microsystems公司于1995年推出的计算机操作系统。Sun Microsystems把自己打造成一个功能完整的商用平台，提供各种Java技术产品和服务，包括Java语言、Java虚拟机、JavaBeans、Java组件、Java开发工具和应用程序服务器。
目前，Java平台分为三个层次：

- Java SE（Standard Edition）：标准版，只提供Java运行环境，不包含开发工具；
- Java EE（Enterprise Edition）：企业版，包括服务器端开发技术和分布式计算技术；
- Java ME（Micro Edition）：微型版，提供移动设备和嵌入式设备上使用的Java API。

Sun Microsystems的Java SE平台由四个主要组成部分组成：

- Java SE平台类库：提供最基本的API，包括集合、I/O、日期时间、网络、图形用户界面、多线程、SQL支持等；
- Java编译器（javac）：用于将源代码编译为字节码，然后运行在Java虚拟机上；
- Java调试器（jdb）：用于调试Java程序；
- Java工具箱（java工具箱）：包含许多实用的工具，如查找替换字符串、验证数字序列等。

Sun Microsystems同时拥有Oracle Corporation的Java EE平台、Intel Corporation的Java ME平台和IBM Corporation的WebSphere Application Server产品，是业界领先的Java平台。

### 2.1.2 Java虚拟机
Java虚拟机（JVM，Java Virtual Machine）是一种独立于具体硬件和操作系统的程序执行引擎，它负责执行被加载的Java程序。JVM屏蔽底层操作系统的复杂性，使得Java程序具有与系统无关的移植能力。每当需要运行Java程序时，就需要启动JVM，在JVM上运行的Java程序称为虚拟机上的程序（Virtual Machine Program）。JVM提供了一个与操作系统无关的环境，使得Java程序能够不受限地运行在各种各样的系统平台上。由于JVM是在原始机器指令之上仿真执行的，因此Java程序的运行速度非常快。

### 2.1.3 Java程序
Java程序就是用Java语言编写的可以直接运行在JVM上的源码文件或字节码文件。Java程序一般通过创建主类来定义自己的逻辑和行为。Java程序是按照Unicode字符集编码而成的，并以“.class”为后缀名。通常情况下，编译后的Java程序会产生一个以“.class”结尾的文件，该文件就是字节码文件。

### 2.1.4 Java类
Java类（Class）是面向对象编程的基本单位。类是描述具有相同属性和方法的对象的蓝图。Java类可以看做是一种数据类型，但又比一般的数据类型要复杂得多，因为Java类可以包含状态信息（成员变量），行为信息（成员函数），构造器和析构器。类可以扩展另一个类，并重写父类的成员函数，因此Java是单继承结构的语言。

Java类也可以实现接口，因此一个Java类可以有多个接口，这意味着可以根据需要组合不同的接口。

### 2.1.5 对象
对象（Object）是指具有相关状态和行为的“事物”。对象可以通过方法调用和访问其状态和行为。在Java中，所有事物都是对象，包括基本数据类型、数组、枚举、类等。所有的对象都有唯一的标识符——一个称为对象的引用（Reference）。

### 2.1.6 方法
方法（Method）是一种行为，它代表某个对象所能执行的动作。方法在语法上类似于函数，但是它们与其他的方法没有区别。在Java中，方法是一个独立的执行单位，只能作用于某个特定的对象。方法可以有参数，可以返回值，并且可以在任意位置调用。

### 2.1.7 抽象类
抽象类（Abstract Class）是不能够实例化的，它的作用是定义一个基类或者接口，其中的方法存在多个具体实现。抽象类不能用于创建对象，只能被继承和扩展，因此，抽象类一般用来作为基类，用于定义公共的属性和方法，而子类可以选择性地实现这些方法。

### 2.1.8 接口
接口（Interface）是抽象类类型的一种特殊形式，接口是完全抽象的，除了方法声明外，不能有方法体。接口用于定义某些类的行为约束，它允许定义一些方法签名，子类必须实现这些方法。接口可用于实现多重继承，并且可以使得某些实现变得更加灵活。

### 2.1.9 内部类
内部类（Inner Class）是指定义在另外一个类中的类。内部类可以直接访问外部类的成员，包括私有的成员，因此内部类提供了对数据的封装，降低耦合度。

### 2.1.10 多态
多态（Polymorphism）是指同一个方法调用可以在不同的情景下表现出不同行为。Java中多态主要通过方法重载和方法重写来实现。方法重载是指两个方法名称相同，参数列表不同，在同一个类中进行方法定义，这样，编译器就能够判断调用的哪个方法，这是通过编译时的静态绑定来实现的。方法重写则是指子类重新定义了父类的方法，这样，当父类的对象调用这个方法的时候，就会调用子类的定义，这是通过运行时的动态绑定来实现的。

### 2.1.11 继承
继承（Inheritance）是面向对象编程的一个重要概念。Java的继承机制是单继承结构。一个Java类只能有一个直接父类，通过 extends 关键字指定。

### 2.1.12 包
包（Package）是指一个目录树，里面包含了一组相关类、接口、资源文件和子目录。包可以防止命名冲突，并且可以避免过多的目录层次。每个包都对应于一个编译单元，因此，Java文件必须置于相应的包下才能编译成功。

### 2.1.13 泛型
泛型（Generics）是指在Java 1.5版本引入的特征，它允许程序员创建可以应用于不同类型参数的类或方法，而不是只能使用 Object 来表示类型。泛型通过在编译期间检查类型参数的类型是否正确，来确保类型安全。

### 2.1.14 异常
异常（Exception）是Java编程中广泛使用的机制。当程序遇到无法预料的错误时，抛出一个异常，可以通知调用者发生了错误，并提供错误原因的相关信息。

## 2.2 安装配置

### 2.2.1 安装JDK
JDK全称Java Development Kit，是Java SE的一套软件开发工具包。JDK包含了JRE（Java Runtime Environment）和Java开发环境。JRE是运行JAVA程序的环境，而JDK则是为JAVA开发提供的开发环境。

JRE和JDK的安装很简单。如果已经安装了Java开发环境，则可以跳过这一节。否则，可以从Oracle官网下载最新版本的JDK并安装。

如果下载速度很慢，可以使用国内的阿里云镜像站点：http://maven.aliyun.com/nexus/content/groups/public 。


下载完成后，双击运行“jdk-8uXXX-linux-x64.rpm”，安装JDK。

### 2.2.2 配置环境变量
配置JDK的环境变量可以让命令行使用JDK的任何命令和工具。

打开“控制面板” -> “系统和安全” -> “系统”，点击左侧的“高级系统设置”，再点击“环境变量”按钮。

在“用户变量”区域找到“Path”变量，点击编辑。将C:\Program Files\Java\jdk1.8.0_XX\bin追加到变量值的末尾。


配置好环境变量之后，就可以在命令行中输入java或javac来启动Java虚拟机了。

## 2.3 Hello World

创建一个文本文件，写入以下代码：

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, world!");
    }
}
```

保存为“HelloWorld.java”。双击运行该程序，命令行窗口输出：

```
Hello, world!
```

如果你成功看到上面打印出的消息，证明你已经成功运行了一个简单的Java程序。
