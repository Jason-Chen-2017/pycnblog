## 2.核心概念与联系

### 2.1 应用安全测试

应用安全测试是一种针对应用程序安全性的测试方法，旨在找出应用程序中的安全漏洞。应用安全测试可以分为以下几种类型：

- 静态应用安全测试（Static Application Security Testing，SAST）：在此类型的测试中，无需运行应用程序。而是通过分析应用程序的源代码来找出潜在的安全漏洞。
- 动态应用安全测试（Dynamic Application Security Testing，DAST）：在此类型的测试中，需要运行应用程序。通过模拟黑客攻击来找出应用程序中的安全漏洞。
- 混合应用安全测试（Hybrid Application Security Testing，HAST）：混合应用安全测试是一种结合了静态和动态应用安全测试的方法。

### 2.2 漏洞扫描

漏洞扫描是一种自动化的工具，通过对应用程序进行扫描来发现潜在的安全漏洞。漏洞扫描可以分为以下几种类型：

- 网络漏洞扫描：通过对网络服务进行扫描来找出潜在的安全漏洞。
- 应用程序漏洞扫描：通过对应用程序的代码进行扫描来找出潜在的安全漏洞。
- Web应用程序漏洞扫描：通过对Web应用程序进行扫描来找出潜在的安全漏洞。

### 2.3 结合应用安全测试和漏洞扫描

结合应用安全测试和漏洞扫描可以在软件开发过程中提高应用程序的安全性。这种结合方法可以在软件开发的早期阶段发现并修复安全漏洞，从而降低软件的安全风险。

结合应用安全测试和漏洞扫描的方法可以包括以下几种：

- 在应用安全测试过程中使用漏洞扫描工具：在进行应用安全测试时，可以使用漏洞扫描工具来自动化地发现潜在的安全漏洞。
- 在漏洞扫描过程中使用应用安全测试方法：在进行漏洞扫描时，可以使用应用安全测试方法来手动地发现潜在的安全漏洞。
- 结合静态应用安全测试和网络漏洞扫描：结合静态应用安全测试和网络漏洞扫描可以在软件开发的早期阶段发现并修复安全漏洞，从而降低软件的安全风险。
