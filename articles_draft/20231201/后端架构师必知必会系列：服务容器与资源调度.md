# 2.核心概念与联系

在服务化架构中，容器技术和资源调度是两个重要的概念。接下来我们将详细介绍这两个概念的定义和联系。

## 2.1 容器技术

容器技术是一种轻量级的应用程序运行环境，它可以将应用程序和其依赖项打包成一个独立的运行环境，从而使得应用程序可以在任何支持容器的环境中运行。容器技术的出现使得部署和扩展服务变得更加简单和高效。

容器技术的核心概念有以下几点：

- 镜像：容器镜像是一个特殊的文件系统，包含了应用程序需要的所有文件。镜像可以被复制和分发，从而实现应用程序的部署和扩展。
- 容器：容器是镜像的实例，是一个运行中的应用程序。容器可以运行在任何支持容器的环境中，从而实现应用程序的迁移和扩展。
- 注册中心：容器注册中心是一个用于存储和管理容器的中心。容器注册中心可以帮助我们找到和管理容器，从而实现应用程序的发现和管理。

## 2.2 资源调度

资源调度是指将应用程序分配到可用资源上的过程。资源调度的目标是确保应用程序可以充分利用资源，同时避免资源的浪费。资源调度是一个复杂的问题，需要考虑到多种因素，如应用程序的性能需求、资源的可用性、应用程序之间的依赖关系等。

资源调度的核心概念有以下几点：

- 资源：资源是计算机系统中的一种物质或非物质资源，如CPU、内存、磁盘等。资源是应用程序运行所需的基本条件。
- 任务：任务是一个需要执行的操作，可以是一个应用程序或一个计算任务。任务需要在计算机系统中的某个资源上执行。
- 调度策略：调度策略是指将任务分配到资源上的方法。调度策略可以是基于性能、基于资源、基于优先级等不同的策略。
