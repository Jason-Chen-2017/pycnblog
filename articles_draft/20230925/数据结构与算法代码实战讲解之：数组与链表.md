# 2.基本概念
## 2.1 数组（Array）
数组（Array）是一种存储相同类型元素的一组连续内存空间，可以按索引直接访问每个元素。在C++中，可以通过`vector`或`array`进行声明。

数组的优点：
1.随机访问——数组中的元素可以根据下标直接访问。
2.插入删除元素方便——可以在数组的任意位置插入或者删除元素。
3.节省空间——由于数组中所有元素都是连续存储的，因此占用的内存空间远小于链接表或其他动态分配的数据结构。

缺点：
1.大小固定——创建数组时需要指定数组的大小，不能随意增加或缩减。
2.插入删除效率低——当数组满时，只能向后添加元素，如果需要插入或删除中间的元素则效率较低。

## 2.2 链表（Linked List）
链表（Linked List）是一种物理存储单元上非连续、非顺序的存储结构，数据元素之间靠指针连接，每个数据元素由两个部分组成，一是存储数据的变量，称作数据域（data field），二是指向下一个元素地址的指针域（pointer field），即链表中下一个元素存储区的起始位置。

链表的优点：
1.灵活性——链表中的元素可以自由增加和删除。
2.扩容容易——只需更改指针就可以扩充链表的长度。
3.插入删除效率高——可以在链表的任何位置插入或者删除节点，不会像数组那样造成元素移动。

缺点：
1.额外空间开销——指针占用的额外空间。
2.操作复杂——需要遍历链表才能找到特定的元素。

## 2.3 栈与队列
栈（Stack）又称堆栈，是一种抽象数据类型，用来存储和管理元素的集合。栈是一种容器，类似于其他容器，但其操作受限，只能在顶端（最先进入的位置）进行添加或删除元素的操作。栈具有以下两个主要操作：

1.入栈：从栈顶压入一个新的项，同时也称push或insert。
2.出栈：移除栈顶的项，同时也称pop或delete。

栈是一种 Last In First Out （LIFO）的结构，即最先进入的元素最后一个被释放出来。栈可以用数组或链表来实现，两种方式的区别就是空间上的差异。栈的例子有计算器的括号匹配、函数调用栈、浏览器前进后退的历史记录。

队列（Queue）又称先进先出，是另一种抽象数据类型，用来存储和管理元素的集合。队列是一种容器，类似于其他容器，但其操作受限，只能在队尾（最先进入的位置）进行添加元素，在队头（最近进入的位置）进行删除元素的操作。队列具有以下两个主要操作：

1.入队：将一个新元素放到队尾，同时也称enqueue或put。
2.出队：从队首移除第一个元素，同时也称dequeue或get。

队列是一种 First In First Out （FIFO）的结构，即先进入的元素先离开。队列也可以用数组或链表来实现，两者的区别是空间上的差异。队列的例子有排队买票、打印任务、CPU调度、进程间通信等。

## 2.4 递归（Recursion）
递归（Recursion）是指在一个函数体内部调用自身的过程叫做递归。递归函数通常都存在以下特点：

1.基线条件：当函数的参数满足一定条件时，递归停止运行。
2.递归条件：当函数的参数不满足基线条件时，才会继续执行递归。
3.多态性：递归函数的返回值可以是任何类型，函数也可以针对不同的输入参数采用不同策略处理。

递归的优点：
1.简单易懂——函数的定义很直观易懂，递归相当于循环，可使得问题变得简单易懂。
2.效率高——递归函数在某些问题上可以达到较高的效率，如求阶乘。

缺点：
1.过深层次调用——当递归函数层次太深时，会导致系统崩溃或栈溢出。
2.栈空间消耗大——每一次递归调用都会保存调用信息，可能会占用大量的栈空间。

## 2.5 搜索（Searching）
搜索（Searching）是指在已知的序列（如列表、数组、字符串等）中，找出特定元素的一个过程。搜索的算法一般分为两种：

1.顺序搜索（Linear Searching）：通过检查每个元素是否符合要求，直到找到特定元素为止。
2.有序搜索（Ordered Searching）：首先确定序列的最小值和最大值，然后依据一定规则将序列划分为两个子序列，再分别搜索两个子序列。

搜索的优点：
1.简单快速——搜索算法的平均时间复杂度往往要低于 O(n)，因此在大规模数据中可以获得优秀的性能。
2.准确度高——搜索算法能找到准确的值，而不是近似的值。

缺点：
1.效率低下——顺序搜索、有序搜索的时间复杂度均为 O(n)。
2.缺乏灵活性——搜索算法的设计往往取决于具体的问题，无法应对各种特殊情况。

## 2.6 排序（Sorting）
排序（Sorting）是指对一组元素按照一定规则进行重新排列，使得组内的元素满足一定的关系。排序的目的在于将序列中元素之间的比较关系确定下来，使之变成按特定顺序排列的状态。排序算法通常分为两大类：

1.内部排序（In-place sorting）：指不需要占用额外内存的排序算法。如插入排序、希尔排序、冒泡排序、归并排序、堆排序。
2.外部排序（External sorting）：指需要占用额外内存的排序算法。如归并排序，合并排序，基数排序。

排序的优点：
1.简洁清晰——排序算法的设计很简洁，流程也很清晰，对于初级算法学习者来说，可以简单明了地看懂。
2.稳定性保证——对相同元素的排序结果是一致的，避免因输入数据顺序变化导致的结果变化。

缺点：
1.时间复杂度高——排序算法的平均时间复杂度一般都为 O(n log n)，而比较排序的平均时间复杂度为 O(n^2)。
2.空间复杂度高——排序算法的空间复杂度往往为 O(1) 或 O(n)。

## 2.7 树（Tree）
树（Tree）是一种抽象数据类型，用来表示具有树状结构关系的元素的集合。树是一种非线性数据结构，由结点和边组成。树的数据结构可以分为三种：

1.带权重的树（Weighted Tree）：树的边上可以赋予权值，称为带权重的树。比如，用于表示距离的树，边上的权值代表两结点之间的距离。
2.二叉树（Binary Tree）：树的每个结点最多拥有两个孩子，左孩子称为左子树，右孩子称为右子树，称为二叉树。
3.多叉树（Multi-way tree）：树的每个结点可以有多个孩子，这种树称为多叉树。比如，一个节点可以有三个孩子，四叉树；一个节点可以有四个孩子，五叉树。

树的优点：
1.树型结构清晰——树是一种结构清晰、易于理解的数据结构。
2.容易扩展——树结构允许对其进行轻松的扩展。

缺点：
1.高度不固定位决——树的高度取决于树的宽度，导致树的高度不固定。
2.需要占用额外内存——树结构需要额外的内存空间来存储指针。

## 2.8 图（Graph）
图（Graph）是一种多维结构，由节点和边组成，由点与点之间的链接关系来定义。图的数据结构可以分为无向图和有向图，两种图各有特点，下面介绍一下它们的概念。

1.无向图（Undirected Graph）：无向图是图论中的一种基本模型。在无向图中，任意两个节点之间都存在一条边。在无向图中，边的方向是没有约束的，可以双向流通，并且同一个节点既可能接收消息，又可能发送消息。

2.有向图（Directed Graph）：有向图是图论中的另一种基本模型。在有向图中，任意两个节点之间都存在一条边。在有向图中，边的方向是有约束的，只能单向流通。例如，在互联网中，A 网页可以链接到 B 网页，但 A 网页无法链接到 C 网页。

图的优点：
1.表达能力强——图的结构能够完整地反映现实世界中复杂的关系。
2.便于理解——图是一种直观、形象且易于理解的抽象模型。

缺点：
1.空间占用大——图结构需要存储节点和边的信息，因此，图的存储空间往往比树大。
2.计算复杂度高——对于复杂的图，算法的运行时间往往会长且费时。
