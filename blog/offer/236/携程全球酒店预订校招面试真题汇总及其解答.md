#### 1. 酒店预订系统的高并发设计如何考虑？

**题目：** 请简要描述一个高并发酒店预订系统的设计思路。

**答案：**

- **分布式架构：** 系统采用分布式架构，将数据库读写分离，前端、后端、数据库服务分别部署在不同的服务器上。
- **缓存层：** 在热门酒店页、预订结果页等高访问量页面使用 Redis 等缓存技术，减少数据库访问压力。
- **限流器：** 使用 Redis 实现限流器，控制对数据库的访问量，避免数据库瞬间被大量请求冲击。
- **队列：** 将用户请求放入消息队列，如 RabbitMQ，实现异步处理，降低系统响应时间。
- **数据库优化：** 对数据库进行索引优化、SQL 优化，提高查询效率。
- **微服务架构：** 系统采用微服务架构，将不同功能模块（如用户服务、订单服务、支付服务）分离，独立部署和扩展。
- **负载均衡：** 使用 Nginx 等负载均衡器，将请求分配到不同的服务器上，提高系统处理能力。

**解析：** 高并发设计主要考虑分布式架构、缓存层、限流器、队列、数据库优化、微服务架构和负载均衡等方面，以保证系统在面临大量请求时能够稳定运行。

#### 2. 如何保证酒店库存数据的实时性？

**题目：** 请简要描述一种保证酒店库存数据实时性的方法。

**答案：**

- **实时同步：** 酒店库存数据通过 API 接口实时同步到预订系统，确保库存数据与酒店端一致。
- **消息队列：** 将酒店库存变更消息（如预订、取消、修改）发送到消息队列，如 Kafka，预订系统通过订阅消息队列来获取库存变更信息。
- **定时同步：** 预订系统定时从酒店获取库存数据，如每天凌晨进行全量同步。
- **缓存刷新策略：** 当库存发生变更时，及时刷新相关页面的缓存，确保用户看到最新的库存信息。

**解析：** 保证库存数据的实时性可以通过实时同步、消息队列、定时同步和缓存刷新策略来实现，确保库存数据与酒店端保持一致，提高用户预订体验。

**题目：** 请简要描述一种处理预订系统订单冲突的方法。

**解析：** 订单取消率高的原因主要包括价格变动、预订错误、行程变更、预订信息不准确和支付问题等，针对这些原因可以采取相应的措施来降低取消率。
