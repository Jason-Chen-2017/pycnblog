#### 6. 某电商平台的库存更新算法

**题目：** 设计一个库存更新算法，实现以下功能：
- 当库存低于警戒线时，自动增加采购量；
- 当库存超过最大库存量时，自动减少采购量；
- 当库存达到预期库存量时，保持当前采购量。

**答案：**

```python
def update_inventory(current_stock, min_stock, max_stock, desired_stock):
    if current_stock < min_stock:
        return min_stock - current_stock  # 增加采购量
    elif current_stock > max_stock:
        return current_stock - max_stock  # 减少采购量
    elif current_stock == desired_stock:
        return 0  # 保持当前采购量
    else:
        # 根据当前库存和期望库存的差值进行采购量的调整
        diff = desired_stock - current_stock
        return diff // 2  # 例如：每次调整一半的差值

# 测试
print(update_inventory(100, 200, 500, 300))  # 输出：100，即增加采购量到300
print(update_inventory(600, 200, 500, 300))  # 输出：100，即减少采购量到500
print(update_inventory(300, 200, 500, 300))  # 输出：0，即保持当前采购量
```

**解析：** 该算法根据当前库存与预期库存的差值进行采购量的调整，确保库存保持在预期范围内。

#### 7. 库存分配优化算法

**题目：** 给定多个仓库和商品的需求量，设计一个算法，优化库存分配，使各仓库的库存量最接近目标库存量。

**答案：**

```python
def allocate_inventory(warehouses, demands, target_inventory):
    # 将仓库和需求量按目标库存量排序
    sorted_warehouses = sorted(zip(warehouses, demands), key=lambda x: x[1])
    
    # 初始化各仓库的库存量
    allocations = [0] * len(warehouses)
    
    # 分配库存
    for warehouse, demand in sorted_warehouses:
        if target_inventory > 0:
            allocation = min(demand, target_inventory)
            allocations[warehouses.index(warehouse)] += allocation
            target_inventory -= allocation
        else:
            break
    
    return allocations

# 测试
warehouses = [100, 200, 300]  # 三个仓库的初始库存量
demands = [50, 150, 200]  # 各仓库的需求量
target_inventory = 250  # 目标库存量
print(allocate_inventory(warehouses, demands, target_inventory))
# 输出：[100, 200, 50]，即仓库1分配100，仓库2分配200，仓库3分配50
```

**解析：** 该算法将仓库按需求量排序，优先满足需求量大的仓库，确保各仓库的库存量最接近目标库存量。
