## 第一部分：C#基础

### **变量与数据类型**

在C#编程中，变量是用于存储数据的容器。每个变量都有其类型、名称和值。C#提供了丰富的数据类型，包括基本数据类型、引用数据类型和值数据类型。

**基本数据类型**包括整型（如`int`、`long`）、浮点型（如`float`、`double`）、布尔型（`bool`）和字符型（`char`）等。基本数据类型在内存中直接存储值，速度快，但存储空间有限。

**引用数据类型**包括类（`class`）、结构（`struct`）、接口（`interface`）和数组。引用数据类型在内存中存储的是对象的引用地址，而不是数据本身，因此占用的内存空间相对较大。

**值数据类型**包括`enum`（枚举类型）和`decimal`（高精度数值类型）。值数据类型在内存中直接存储值，既具有基本数据类型的速度优势，又具有引用数据类型的空间优势。

变量的声明和初始化是编程的基础，C#提供了便捷的语法。例如：

```csharp
int number = 10;
double price = 19.99;
bool isAvailable = true;
char grade = 'A';
```

### **运算符与表达式**

C#提供了丰富的运算符，包括算术运算符（如`+`、`-`、`*`、`/`）、逻辑运算符（如`&&`、`||`、`!`）、比较运算符（如`==`、`!=`、`>`、`<`）和位运算符（如`&`、`|`、`^`）等。

表达式是由运算符和变量或常量组成的计算式，C#可以根据表达式的值返回一个结果。例如：

```csharp
int a = 5;
int b = 10;
int sum = a + b; // 返回15
bool isEqual = a == b; // 返回false
```

### **控制结构**

控制结构是编程中用于控制程序流程的语句。C#提供了条件语句（如`if`、`else if`、`switch`）、循环语句（如`for`、`while`、`do-while`）和跳转语句（如`break`、`continue`、`return`）。

**条件语句**用于根据条件执行不同的代码块。例如：

```csharp
if (a > b)
{
    Console.WriteLine("a大于b");
}
else
{
    Console.WriteLine("a小于或等于b");
}
```

**循环语句**用于重复执行一段代码。例如：

```csharp
for (int i = 0; i < 5; i++)
{
    Console.WriteLine(i);
}
```

**跳转语句**用于在程序中跳转执行。例如：

```csharp
while (true)
{
    Console.WriteLine("无限循环");
    break; // 跳出循环
}
```

### **面向对象编程**

面向对象编程（OOP）是C#编程的核心概念。OOP将数据和操作数据的方法封装在一起，形成对象。C#提供了类（`class`）和结构（`struct`）两种类型来定义对象。

**类**是一种引用类型，用于表示抽象的概念。例如：

```csharp
public class Student
{
    public string Name { get; set; }
    public int Age { get; set; }
}
```

**结构**是一种值类型，用于表示具体的事物。例如：

```csharp
public struct Point
{
    public int X { get; set; }
    public int Y { get; set; }
}
```

面向对象编程还包括继承、封装、多态等特性，这些特性使得代码更加模块化、可复用和易于维护。

```csharp
public class Teacher : Person
{
    public string Subject { get; set; }
}
```

通过面向对象编程，C#能够更好地组织和管理代码，提高开发效率。

### **泛型**

泛型是C#的一个重要特性，它允许我们编写可重用的代码，同时保持类型安全。泛型通过使用类型参数来定义类、接口、方法和委托，从而实现类型的参数化。

**泛型类**：泛型类是一种可以接受类型参数的类，例如：

```csharp
public class Stack<T>
{
    private T[] items = new T[100];
    private int position = 0;

    public void Push(T item)
    {
        items[position++] = item;
    }

    public T Pop()
    {
        return items[--position];
    }
}
```

**泛型接口**：泛型接口与泛型类类似，也是一种可以接受类型参数的接口，例如：

```csharp
public interface IEnumerable<T>
{
    IEnumerator<T> GetEnumerator();
    IEnumerator GetEnumerator();
}
```

**泛型方法**：泛型方法是一种可以接受类型参数的方法，例如：

```csharp
public static T Max<T>(T a, T b) where T : IComparable<T>
{
    return a.CompareTo(b) > 0 ? a : b;
}
```

泛型大大提高了代码的复用性和灵活性，使得我们可以编写更加安全和高效的代码。

### **委托**

委托是C#中用于封装方法的类型，它类似于函数指针。委托可以传递方法作为参数，并在需要时调用这些方法。

**定义委托**：委托的定义使用`delegate`关键字，例如：

```csharp
public delegate void MyDelegate(string message);
```

**创建委托实例**：创建委托实例的过程类似于创建对象，例如：

```csharp
MyDelegate myDelegate = new MyDelegate(ShowMessage);
```

**调用委托**：调用委托的方法使用`Invoke`方法，例如：

```csharp
myDelegate.Invoke("Hello, World!");
```

**匿名方法**：C#允许我们直接在委托定义时使用匿名方法，例如：

```csharp
myDelegate += delegate(string message)
{
    Console.WriteLine($"Received message: {message}");
};
```

委托在事件处理、回调函数等方面有着广泛的应用。

### **事件**

事件是C#中用于描述对象之间通信的一种机制。事件通常与委托一起使用，使得对象能够在特定情况下触发相应的方法。

**定义事件**：事件的定义使用`event`关键字，例如：

```csharp
public delegate void MyEvent();
public event MyEvent MyEventHandler;
```

**添加事件处理程序**：可以通过`+=`操作符向事件添加处理程序，例如：

```csharp
MyEventHandler += ShowMessage;
```

**引发事件**：引发事件使用` Raise`方法，例如：

```csharp
MyEventHandler?.Invoke();
```

事件在用户界面编程、系统通知等方面有着重要的应用。

### **属性**

属性是C#中用于封装数据访问的一种机制，它允许我们控制对变量的访问和修改。

**定义属性**：属性的声明使用`prop`关键字，例如：

```csharp
public class Person
{
    private string name;

    public string Name
    {
        get { return name; }
        set { name = value; }
    }
}
```

**只读属性**：只读属性只有`get`访问器，例如：

```csharp
public string FullName
{
    get { return $"{firstName} {lastName}"; }
}
```

**只写属性**：只写属性只有`set`访问器，例如：

```csharp
public void SetFullName(string fullName)
{
    string[] parts = fullName.Split(' ');
    firstName = parts[0];
    lastName = parts[1];
}
```

属性使得类的数据访问更加安全和灵活。

### **索引器**

索引器是C#中用于表示集合类型的一种特性，它允许我们通过索引访问集合中的元素。

**定义索引器**：索引器的声明使用`this`关键字，例如：

```csharp
public class List<T>
{
    private T[] items = new T[100];

    public T this[int index]
    {
        get { return items[index]; }
        set { items[index] = value; }
    }
}
```

索引器使得数组、列表等集合类型的使用更加灵活和方便。

### **异常处理**

异常处理是C#中用于处理程序运行时错误的一种机制。通过异常处理，我们可以在程序出现错误时进行适当的处理，从而避免程序崩溃。

**捕获异常**：使用`try`和`catch`语句捕获异常，例如：

```csharp
try
{
    // 可能出现异常的代码
}
catch (Exception ex)
{
    // 异常处理代码
}
```

**抛出异常**：使用`throw`关键字抛出异常，例如：

```csharp
if (condition)
{
    throw new ArgumentException("参数无效");
}
```

**异常过滤**：C# 6.0和C# 7.0引入了异常过滤特性，允许我们在`catch`语句中指定异常类型，从而更灵活地处理异常，例如：

```csharp
catch (ArgumentException ex when ex.Message.StartsWith("参数"))
{
    // 特定异常处理代码
}
```

异常处理使得程序的健壮性得到提高。

C# 6.0和C# 7.0作为C#语言的最新版本，引入了许多新的特性，这些特性旨在提高编程效率、增强代码的可读性，并进一步丰富C#的功能。在这一部分，我们将详细介绍这些新增特性，包括局部函数、throw表达式、name of和空条件运算符、using static指令和异常过滤器等。

### **局部函数**

局部函数是C# 6.0引入的一个新特性，它允许在方法内部定义另一个方法。局部函数的定义和使用与常规方法类似，但它们仅存在于外部方法的上下文中。

**定义局部函数**：

```csharp
public void Main()
{
    Console.WriteLine("Hello, World!");
    
    void PrintMessage()
    {
        Console.WriteLine("This is a local function.");
    }
    
    PrintMessage();
}
```

在上述示例中，`PrintMessage`是一个局部函数，它只能在`Main`方法内部调用。

**优点**：

- 减少方法调用的开销，提高性能。
- 提高代码的可读性，避免方法层次的过多嵌套。
- 增强代码的组织性，使得相关功能逻辑更加集中。

### **throw表达式**

throw表达式是C# 7.0引入的一个新特性，它允许在异常处理过程中返回一个值。这为异常处理提供了更多的灵活性和强大的功能。

**使用throw表达式**：

```csharp
public int TryDivide(int a, int b)
{
    if (b == 0)
    {
        throw new ArgumentException("除数不能为零", nameof(b));
    }
    
    return a / b;
}
```

在上述示例中，当除数为零时，`TryDivide`方法抛出`ArgumentException`，并在异常中传递了错误消息和错误参数名称。

**优点**：

- 可以在异常处理过程中返回值，提供更丰富的错误处理逻辑。
- 提高代码的可读性和可维护性，使得错误处理逻辑更加清晰。

### **name of和空条件运算符**

name of运算符是C# 7.0引入的一个新特性，它用于获取表达式的名称。而空条件运算符（?.）则是C# 6.0引入的一个新特性，它允许对可能为空的引用类型进行安全访问。

**使用name of运算符**：

```csharp
public void Validate(int value)
{
    string name = nameof(value);
    if (value <= 0)
    {
        throw new ArgumentException($"{name} must be greater than zero", name);
    }
}
```

在上述示例中，`nameof(value)`获取了变量`value`的名称，并在异常消息中使用该名称。

**使用空条件运算符**：

```csharp
public string GetDescription(Product product)
{
    return product?.Description ?? "未知";
}
```

在上述示例中，`product?.Description`使用空条件运算符检查`product`是否为null。如果`product`为null，则返回"未知"。

**优点**：

- 提高代码的可读性，使得逻辑更加清晰。
- 避免因引用类型为null导致的空指针异常。
- 提供更加灵活和安全的引用访问方式。

### **using static指令**

using static指令是C# 6.0引入的一个新特性，它允许我们引入一个静态类或静态成员，使得这些静态成员可以直接使用，而无需实例化类。

**使用using static指令**：

```csharp
using static System.Math;

public void Main()
{
    double result = Sqrt(9);
    Console.WriteLine(result); // 输出3
}
```

在上述示例中，我们使用`using static`指令引入了`Math.Sqrt`方法，可以直接使用，而无需调用`Math`类。

**优点**：

- 提高代码的可读性，使得静态成员的调用更加直观。
- 减少代码冗余，提高代码的简洁性。

### **异常过滤器**

C# 6.0和C# 7.0引入了异常过滤器，它允许我们在`catch`语句中指定异常类型，从而更精确地处理异常。

**使用异常过滤器**：

```csharp
try
{
    // 可能出现异常的代码
}
catch (ArgumentException ex when ex.Message.StartsWith("参数"))
{
    // 特定异常处理代码
}
catch (Exception)
{
    // 其他异常处理代码
}
```

在上述示例中，我们使用异常过滤器` ArgumentException ex when ex.Message.StartsWith("参数")`来处理特定的异常。

**优点**：

- 提高代码的可读性，使得异常处理逻辑更加清晰。
- 允许我们为不同的异常类型提供定制化的处理方法。
- 提高程序的健壮性，避免因错误处理不当导致的问题。

C# 6.0和C# 7.0的新增特性为开发者提供了更多的便利和功能，使得C#编程更加高效、灵活和安全。局部函数、throw表达式、name of和空条件运算符、using static指令和异常过滤器等特性不仅提高了代码的可读性和可维护性，还丰富了C#语言的功能。
